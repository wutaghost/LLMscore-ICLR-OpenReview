009                                              A BSTRACT
010
011            Many graphs can be represented as Text-attributed Graphs (TAGs). Due to the
012            rich textual information present in each node of TAGs, traditional graph neu-
013            ral networks (GNNs) often struggle to deliver satisfactory performance. Recent
014            advancements leveraging large language models (LLMs) to augment new node
015            text features have notably enhanced node representations, resulting in significant
016            performance improvements. However, these methods typically require extensive
017            annotations or fine-tuning on all nodes, which are both time-consuming and ex-
018
               pensive. To address this challenge, we propose GAGA, a novel and lightweight
               framework for TAG representation learning. GAGA employs a more efficient
019
               strategy by annotating only representative nodes and edges, thereby reducing both
020
               annotation time and cost. It further capitalizes on these annotations by construct-
021            ing an annotation graph that captures the topological relationships among them.
022            Additionally, GAGA introduces a two-level alignment module to integrate the an-
023            notation graph with the TAG, ensuring effective alignment of their underlying
024            structures. Experiments demonstrate that GAGA achieves classification accura-
025            cies comparable to or exceeding state-of-the-art methods while requiring only 1%
026            of the data to be annotated, making it highly efficient.

029   1   I NTRODUCTION
030
031   In real-world scenarios, many graphs can be effectively represented as text-attributed graphs (TAGs)
032   (Yang et al., 2021), such as paper citation networks and product purchase graphs. In TAGs, nodes
033
      represent pieces of text, and edges signify connections between these nodes, such as citations or
      purchases. A classic example is the Ogbn-arxiv (Hu et al., 2020a) dataset, where nodes represent
034
      the titles and abstracts of academic papers, and edges denote citation relationships. TAGs hold
035
      significant practical value in areas such as text classification (Yang et al., 2015), recommendation
036   systems (Juan et al., 2023), social networks (Li et al., 2022), and fake news detection (Kananian
037   et al., 2024).
038
039
      In recent years, numerous models have been proposed to tackle challenges related to node classifi-
      cation (Maekawa et al., 2022) and link prediction (Zhang & Chen, 2018) in TAGs. For example, tra-
040
      ditional graph neural networks (GNNs) like Graph Convolutional Network (GCN) (Kipf & Welling,
041
      2016a) and GraphSAGE (Hamilton et al., 2017) have been widely used for node classification tasks
042   on TAGs. However, compared to classical graph data, directly leveraging these GNN architectures
043   often struggles to achieve satisfactory performance on TAGs, particularly on large-scale datasets
044   like Ogbn-arxiv. A key challenge with TAGs is that canonical GNN models tend to emphasize the
045   graph structure while potentially underutilizing the rich textual information associated with each
046   node, which is essential for capturing the context of nodes (Yan et al., 2023).
047
      To further improve the performance, some recent works have started to augment information on
048   graph data. Specifically, with the advent of large language models (LLMs), several studies started
049   to leverage LLMs to generate new textual features for TAGs. The primary motivation for incor-
050   porating LLM-generated textual features is to utilize the vast world knowledge embedded in these
051   models. This allows for the extraction of high-quality features that surpass those derived from raw
052   text alone (He et al., 2023; Duan et al., 2023). For example, TAPE (He et al., 2023) prompts LLMs
053   to predict node categories and to provide explanations for their own predictions. These predictions
      and explanations can be seen as summarization or an enhancer for each node and utilized as new

091
092   To address the limitations outlined before, we propose GAGA: Graph Alignment Guided Text-
093
      attributed Graph Learning. Instead of generating summarization or refining features for each node,
      GAGA focuses on a more efficient strategy by only annotating representative nodes (edges), thereby
094
      reducing both the time and cost of annotation via LLMs. To further utilize these annotations, GAGA
095
      recognizes and incorporates the topological relationships between them by constructing an annota-
096   tion graph. Then, an alignment module between the annotation graph and the TAGs is introduced,
097   ensuring that the underlying structures of both graphs are effectively integrated.
098
099
      There are three stages in GAGA. We aim to obtain a representative set of nodes or edges in the first
      stage. Then, we generate annotations for these nodes or edges by prompting LLMs. These anno-
100
      tations are constructed into an annotation graph. In the second stage, through contrastive learning
101
      between these sub-annotation graphs and their corresponding sub-textual graphs, we simultaneously
102   fine-tune an LLM and a GNN. During our novel-designed and lightweight two-level contrastive
103   learning process, the LLM captures semantic information from the text, while the GNN processes
104   the topological structure information of the annotation graphs and textual graphs. In the third stage,
105   which involves downstream tasks, we only fine-tune the GNN while keeping the LLM frozen. Ex-
106   perimental results demonstrate that GAGA achieves a classification accuracy that is comparable to
107   or even exceeds SOTA methods while requiring only 1% of the data to be annotated, making it
      3 ∼ 100 times more time-and-cost-efficient. Our contributions can be summarized as follows:

      2   R ELATED W ORK
125   Language Models for Text-Attributed Graph. Traditional GNN approaches typically handle
126   TAGs by converting textual attributes into features via shallow neural networks, such as bag-of-
127   words, which limits the comprehensive understanding of textual semantics. Recent research has
128   focused on embeddings based on language models (LMs) like BERT (Devlin, 2018) to address this
129   issue. These fine-tuning pre-trained models can effectively generate deep embeddings to capture
130
      the rich semantic information within TAGs. There are two main architectures for empowering TAG
      tasks using LMs. The first is the cascaded architecture, where the textual information of nodes is
131
      independently encoded by LMs, and GNN models then aggregate the outputs to obtain the final
132
      embeddings, including TextGNN (Zhu et al., 2021a), GEAR (Zhou et al., 2019), GIANT (Chien
133   et al., 2021), GPT-GNN (Hu et al., 2020b), and SimTeG (Duan et al., 2023). However, this ap-
134   proach separates text encoding from graph aggregation, which prevents a unified integration of the
135   two processes. As a result, the nested architecture has also been widely studied. This approach in-
136   tegrates text encoding and graph aggregation, performing these tasks iteratively to better unify both
137   processes. For example, Graphormer (Yang et al., 2021), GLEM (Zhao et al., 2022) and DRAGON
138   (Yasunaga et al., 2022) follow this nested architecture.
139   LLMs such as ChatGPT (Brown et al., 2020) have shown tremendous potential in various NLP tasks.
140   However, how to apply LLMs to graph-structured data, such as TAGs, remains a challenge (Chai
141   et al., 2023; Qin et al., 2023). Chen et al. (2024) investigated the potential of LLMs in node classifi-
142   cation tasks. Some works have already attempted to use LLMs for TAGs. He et al. (2023) proposed
143   TAPE, which leverages LLMs to use the explanations generated during zero-shot classification as
144   informative features for the graph. Another category of methods utilizes LLMs’ strengths in text
145   understanding to improve the graph’s topology by enhancing the semantic understanding of node
146   information. A representative example of this approach is the work by Sun et al. (2023), where they
147
      utilize LLMs to generate pseudo-labels and compute the semantic similarity between node attributes
      to either remove or add edges, thereby improving the graph’s topology. Yu et al. (2023) propose ENG
148
      that enhances TAGs by using LLMs to generate additional nodes and the corresponding additional
149
      information. Additionally, Pan et al. (2024) employs a knowledge distillation approach to distill
150   LLMs into graph models specifically for TAG learning. However, focusing solely on either nodes
151   or edges has its limitations. Concentrating only on nodes (textual information) can result in the
152   loss of the network’s topological structure, while focusing exclusively on edges makes it difficult to
153   fully capture the semantic information of the nodes themselves. Therefore, we need a structure that
154   combines both nodes and edges to enhance the TAG representation, which is a strength of GAGA.
155   Moreover, as mentioned above, all previous SOTA methods are time-consuming and costly, while
156   our GAGA is highly lightweight and achieves almost the same performance.
157   Explanation-Guided Learning. With the advancement of AI, the importance of research in Ex-
158   plainable Artificial Intelligence (XAI) has become increasingly prominent. Although researchers
159   have made significant progress in XAI techniques, leading to more attempts to generate explanations
160   for Deep Neural Networks (DNNs), more profound issues, such as how to apply XAI techniques to
161   improve the performance of DNN models, warrant further attention as research progresses (Ross
      et al., 2017; Gao et al., 2024). In the field of computer vision, there has been extensive research on

      using explanation supervision to guide model training (Das et al., 2017; Linsley et al., 2018; Qiao
163   et al., 2018; Mitsuhara et al., 2019; Zhang et al., 2019; Gao et al., 2022; Zhang et al., 2023; Saha &
164   Roy, 2023). In NLP tasks, explanations are primarily generated through attention mechanisms or by
165   utilizing auxiliary generative models (Bao et al., 2018; Strout et al., 2019; Zhong et al., 2019; Choi

175   (V, A, {sn }n∈V ), where V is a set of N nodes, A ∈ RN ×N is the adjacency matrix, sn ∈ DLn

177   Ln as the sequence length.

      In this section, we introduce our novel framework, GAGA. As illustrated in Figure 2, it primarily
209
      comprises three stages: annotation graph generation, two-level (subgraph) alignment, and down-
210   stream task fine-tuning. In the first stage, we first obtain a small but important set of nodes or edges
211   through representative-based selection. Then, we generate annotations for these nodes or edges by
212   prompting LLMs. These annotations are constructed into an annotation graph. In the second stage,
213   we employ a two-level (subgraph level and prototype level) contrastive learning between the sub-
214   annotation graphs and their corresponding sub-textual graphs, enabling lightweight fine-tuning of
215   both the language model (LM) and GNNs using information from the selected nodes. In the third
      stage, which involves downstream tasks, we only fine-tune the GNN while keeping the LM frozen.

239   Representative Node (Edge) Selection. As we mentioned earlier, using LLMs to annotate data
240   can enhance model performance on TAGs. However, given that a typical graph contains tens of
241
      thousands of nodes, annotating each one can be both time-consuming and costly (Chen et al., 2023).
      Thus, to maintain high performance while annotating only a small number of nodes, it is crucial to
242
      select nodes (edges) with high information density (Cai et al., 2017). Information density, denoted
243

244   derlying data distribution in the embedding space. To locate nodes situated in dense regions of the
245   embedding space, we first perform k-means clustering on the embeddings of all unlabeled nodes.
246   We then compute each node’s Euclidean distance || · || to its nearest cluster center. The density score
247   for a node vi is determined by converting this distance into a similarity score:

      As we have only annotated a subset of nodes (edges), how to generalize these annotations becomes
284
      the key question, which is the motivation of our two-level alignment. Our alignment consists of two
285   levels: one is for aligning sub-textual graphs and sub-annotation graphs, while the other one is for
286   aligning sub-textual graphs and the prototypes induced by sub-annotation graphs.

289

531   6   C ONCLUSIONS
532
      We investigated node classification and link prediction tasks in Text-attributed Graphs (TAGs) and
533
      introduced GAGA, a lightweight and highly efficient graph representation learning framework.
534   GAGA employs a streamlined approach by annotating only representative nodes and edges, thereby
535   significantly reducing both annotation time and cost. It leverages these annotations by construct-
536   ing an annotation graph that captures the topological relationships among the annotated elements.
537   Furthermore, GAGA features a two-level alignment module to effectively integrate the annotation
538   graph with the TAG, ensuring the alignment of their underlying structures. Experimental results
539   demonstrate that GAGA achieves classification accuracies comparable to or exceeding state-of-the-
      art methods while requiring only 1% of the data to be annotated, underscoring its high efficiency.

826   The details of the datasets are as follows:
827
      Cora. The Cora dataset (McCallum et al., 2000) consists of 2,708 scientific publications categorized
828   into seven distinct classes: case-based, genetic algorithms, neural networks, probabilistic methods,
829   reinforcement learning, rule learning, and theory. This dataset includes a citation network compris-
830   ing 5,429 links. The selection process ensured that each paper in the final corpus either cites or is
831   cited by at least one other paper.
832
      PubMed. The PubMed dataset (Sen et al., 2008) comprises 19,717 scientific publications from
833
      the PubMed database, all related to diabetes. These publications are categorized into three distinct
834   classes: Experimental Induced Diabetes, Type 1 Diabetes, and Type 2 Diabetes. The dataset also
835   includes a citation network with 44,338 links.
836
837
      ogbn-arxiv. The ogbn-arxiv dataset (Hu et al., 2020a) is a directed graph representing the citation
      network among all computer science papers on arXiv, as indexed by MAG (Wang et al., 2020). In
838
      this dataset, each node corresponds to an arXiv paper, and each directed edge signifies a citation
839
      from one paper to another. The primary task is to predict the 40 subject areas of the arXiv computer
840   science papers, such as cs.AI, cs.LG, and cs.OS, which are manually assigned by the authors and
841   arXiv moderators.
842
843
      Citeseer. The CiteSeer dataset (Giles et al., 1998a) consists of 3312 scientific publications classified
      into one of six classes. The citation network consists of 4732 links. Each publication in the dataset
844
      is described by a 0/1-valued word vector indicating the absence/presence of the corresponding word
845
      from the dictionary. The dictionary consists of 3703 unique words.
846
847   ogbn-products The ogbn-products Hu et al. (2020a) dataset is an Amazon product co-purchasing
848   network with products as nodes and edges indicating co-purchases. The task is to predict the product
849
      category in a multi-class setup using 47 top-level categories as labels.
850   tape-arxiv23 The tape-arxiv23 He et al. (2023) dataset is a directed graph of citation networks
851   among computer science arXiv papers from 2023 onwards. Each node represents a paper, and the
852   edges show citation links. The task is to predict the 40 subject areas of these papers, such as cs.AI,
853   cs.LG, and cs.OS, based on author and moderator labels.

      A.2   BASELINES
856
857   Details about the baseline models for node classification are as follows:
858
859   MLP. As a baseline model, MLP does not consider graph structure information and relies solely
860
      on node features for classification. Its performance provides a reference point for assessing the
      improvements offered by other models.
861
862   GCN. GCN (Kipf & Welling, 2016a) is a classical graph neural network model that effectively
863   captures local neighborhood information through graph convolution operations. We chose GCN due
      to its strong performance and widespread application across many graph datasets.


                                                         16
      Under review as a conference paper at ICLR 2025


864
      GraphSAGE. GraphSAGE (Sun et al., 2021) generates node embeddings by sampling and aggre-
865   gating information from neighboring nodes, making it suitable for handling large-scale graph data.
866   We used GraphSAGE to evaluate its generalization capability across different datasets.
867
868
      RevGAT. RevGAT (Li et al., 2021) combines graph attention mechanisms with reversible network
      structures, reducing memory consumption while maintaining performance. We selected RevGAT to
869
      explore the effectiveness of attention mechanisms on graph data.
870
871   InstructGLM. InstructGLM Ye et al. (2023) integrates the strengths of graph neural networks and
872   language models, enhancing generalization through instruction learning. We here use InstructGLM
873   using Llama-7b (Touvron et al., 2023) as the backbone.
874   Graphormer. Graphormer (Ying et al., 2021) leverages the Transformer architecture to process
875   graph data, capturing global dependencies. We selected Graphormer to test the performance of
876   Transformer-based approaches in graph neural networks.
877
      TAPE. The TAPE model (He et al., 2023) leverages LLMs to capture textual information as features,
878   which can be used to boost GNN performance on downstream tasks. A key innovation is its use of
879   explanations as features: TAPE prompts an LLM to perform zero-shot classification, requests textual
880   explanations for its decision-making process, and designs an LLM-to-LM interpreter to translate
881   these explanations into informative features for downstream GNNs. For fairness, we use TAPE with
882   the backbone of the GCN model, which is the same as our model.
883
      GLEM. GLEM (Zhao et al., 2022) combines graph models with LLMs, like DeBERTa [12], in
884   a variational EM framework, alternating between updating the LLM and GNN in the E-step and
885   M-step to enhance downstream task performance.
886
887
      OneForAll. OneForAll (Liu et al., 2023) represents all nodes and edges as human-readable texts
      and encodes them from various domains into a unified space using LLMs. The framework then
888
      adapts to different tasks by adding task-specific prompts into the input graph.
889
890   ENGINE. ENGINE (Zhu et al., 2024) introduces a lightweight, tunable G-Ladder module to each
891   LLM layer, using a message-passing mechanism to incorporate structural information. This allows
892   token-level outputs from each LLM layer to pass through the G-Ladder, enhancing node representa-
893
      tions for node classification.
894   GIANT. GIANT (Chien et al., 2021) utilizes XR-Transformers (Zhang et al., 2021a) for neighbor-
895   hood prediction, producing an LLM that generates more effective feature vectors for node classifi-
896   cation than bag-of-words and vanilla BERT embeddings.
897   SimTEG. SimTeG (Duan et al., 2023) fine-tunes a pre-trained LM with parameter-efficient methods
898   for a task like node classification, then uses the LM’s last hidden states as node embeddings for GNN
899   training, significantly boosting performance on graph benchmarks.
900
      For the link prediction tasks, we use the baseline models from HeaRT (Li et al., 2023; Mao et al.,
901
      2023).
902
903   Heuristic methods: Common Neighbor (CN) (Newman, 2001), Adamic Adar (AA) (Adamic &
904   Adar, 2003), and Resource Allocation (RA) (Zhou et al., 2009) use common neighbors, while Short-
905   est Path (Liben-Nowell & Kleinberg, 2003) and Katz (Katz, 1953) rely on path information to score
906
      link existence.
907   Embedding methods: Matrix Factorization (MF) (Menon & Elkan, 2011), Multilayer Perceptron
908   (MLP), and Node2Vec (Grover & Leskovec, 2016) learn low-dimensional node embeddings to pre-
909   dict link likelihood.
910   GNN methods: Graph Convolutional Network (GCN) (Kipf & Welling, 2016a), Graph Attention
911   Network (GAT) (Veličković et al., 2018), GraphSAGE (SAGE) (Sun et al., 2021), and Graph Au-
912   toencoder (GAE) (Kipf & Welling, 2016b) integrate multi-hop graph structures via message passing.
913
      GNN + Pairwise Information: SEAL (Zhang et al., 2021b), BUDDY (Chamberlain et al., 2022),
914
      and Neural Bellman-Ford Network (NBFNet) (Zhu et al., 2021b) use subgraph features; Neo-GNN
915
      (Yun et al., 2021), Neighborhood Contrastive Network (NCN) (Wang et al., 2023), and NCNC
916   (Wang et al., 2023) utilize common neighbor information, while Position Encoding Graph Neural
917   Network (PEG) (Wang et al., 2022) employs positional encoding.

1059   In this section, we present several case studies, as shown in Figure 4, to illustrate how the annota-
1060   tion prototype enhances the model’s performance. When the input textual information is processed
1061   by GAGA, directly predicting the category of the node can sometimes result in inaccurate predic-
