009                                             A BSTRACT
010
011           Language-based agentic systems have shown great promise in recent years, transi-
012           tioning from solving small-scale research problems to being deployed in challeng-
013           ing real-world tasks. However, optimizing these systems often requires substantial
014           manual labor. Recent studies have demonstrated that these systems can be repre-
015           sented as computational graphs, enabling automatic optimization. Despite these
016           advancements, most current efforts in Graph-based Agentic System Optimization
017           (GASO) fail to properly assign feedback to the system’s components given feed-
018
              back on the system’s output. To address this challenge, we formalize the con-
              cept of semantic backpropagation with semantic gradients—a generalization that
019
              aligns several key optimization techniques, including reverse-mode automatic dif-
020
              ferentiation and the more recent TextGrad by exploiting the relationship among
021           nodes with a common successor. This serves as a method for computing direc-
022           tional information about how changes to each component of an agentic system
023           might improve the system’s output. To use these gradients, we propose a method
024           called semantic gradient descent which enables us to solve GASO effectively. Our
025           results on both BIG-Bench Hard and GSM8K show that our approach outperforms
026           existing state-of-the-art methods for solving GASO problems. A detailed ablation
027           study on the LIAR dataset demonstrates the parsimonious nature of our method.

030   1   I NTRODUCTION
031
032   Language-based agentic systems are being hailed as a major breakthrough in artificial intelligence,
033
      with real-world deployment well underway and numerous companies already being founded based
      on this technology (e.g., Pythagora-io (2023) ). Such agentic systems typically consist of multiple
034
      components. These components are selected to perform specific tasks, such as question answering,
035
      implementing and executing computer programs, or performing web searches (Wang et al., 2024;
036   Guo et al., 2024). Due to the strength of Large Language Models (LLMs) in doing a wide array of
037   tasks, agentic systems typically have most of their key components rely on querying LLMs. This
038   results in communication between the components of such systems being handled with free-form
039   natural language (Zhuge et al., 2023). However, while relying on LLMs does partially alleviate the
040   engineering burden of building such systems, designing agentic systems remains nontrivial.
041
      Agentic systems are often modeled as computational graphs, with components involving frozen large
042   models having auxiliary optimizable parameters (Zhuge et al., 2024). When the graph topology is
043   fixed, the challenge of optimizing these parameters to enable an agentic system to solve a specific
044   problem can be modeled as the Graph-based Agent System Optimization (GASO) problem.
045
      Famous methods that attempt to solve the GASO problem include DSPy prompt optimization meth-
046
      ods (e.g., COPRO) (Khattab et al., 2024), GPTSwarm’s node optimization (Zhuge et al., 2024),
047
      Textgrad (Yuksekgonul et al., 2024), and Trace’s OptoPrime (Cheng et al., 2024). DSPy prompt op-
048   timization methods and GPTSwarm’s node optimization both do so by optimizing a prompt template
049   for each node given the input-output pairs of each node that were observed during predictions. In
050   contrast, both TextGrad and OptoPrime adopt a backpropagation-inspired approach, wherein both of
051   them attempt to assign a feedback to each of the components in a system by message passing in the
052   reverse topological order (backward) of the graph given an output feedback. However, OptoPrime
053   fails to build a compact and explicit representation of how each component can be improved to pass
      backward. On the other hand, TextGrad omits neighborhood nodes while computing the backward

103   of delivering precise answers to a variety of user queries through a structured computational ap-

      proach. We formalize this system using a directed acyclic computational graph (V, E, H, Θ). In this
      graph, V represents the set of vertices or nodes, with each node v ∈ V being a variable within the

      practical applications, vθ is often a prompt prefix or suffix optimized to improve the LLM’s response

131
      a file, executing a command line and reading the result, and managing some internal thought (Wang
132
      et al., 2024; Zhang et al., 2024; Jin et al., 2024).

151   Building on the foundational work of Linnainmaa (1970; 1976), Pryzant et al. (2023), and Yuksek-
152   gonul et al. (2024), we introduce the concept of Semantic Backpropagation over Semantic Gradients.
153   In the context of the GASO problem, the semantic gradient of a loss function l(Q, A(Q, Θ)) with
154   respect to a variable v, denoted as ∇v lQ , provides directional information on how altering v can
155   improve system performance for a query Q. Semantic backpropagation employs the final semantic
156   gradient ∇A lQ —typically derived from the answer feedback F (Q, A)—to generate semantic gra-
157   dients ∇v lQ for all variables v ∈ V . Specifically, for a given variable v, the backward functions—
158   acting as a generalization of the chain rule—are used to partially compute the semantic gradients

306

307

310   a query Q, TextGrad can be implemented as a special case of semantic backpropagation by having
311   Av as an identity function and the backward functions ĥ satisfying:
312

319   of how v changes that would lead to a change of w—does not depend on any other predecessors of

320   w . These are not the case in reverse-mode automatic differentiation, where ĥvw implements ∂w  Jhvw

321   depending on v, and hw is the function hw with predecessors other than v fixed at their current

323   of the forward functions, which is typically not true in computational graphs. This limitation is

      importance in order for complimentary and synergistic behavior to emerge (Zhuge et al., 2023).
325   A prominent example is the importance of having system prompts that synergize well with users’
326   prompts in a conversational LLM, which allows it to successfully approach virtually any objective
327   (e.g., OpenAI, 2024). It is neither necessary, nor justified, to ignore nodes with a common successor
328   during backpropagation.
329
      Consider, for a concrete example, a variable va with predecessors vq and vθ , where vq is a user
330
      query, va is an answer to the query, and vθ is an instruction that helps produce a good answer to
331   the user query with relation va = hva (vq , vθ ) for some hva . Following TextGrad’s formulation,
332   the gradient with respect to instruction vθ , would only explicitly depend on vθ and va , and not the
333   question vq . It keeps any possibly useful information about the question unavailable when updating
334   vθ . This could potentially lead to an instance learning an instruction vθ that completely ignores the
335   nature of the approached question, e.g., attempting to memorize the last suited answer as the exact
336   instruction to follow. Moreover, when computing the gradient with respect to the question vq , an
337   identical function is applied as when computing the gradient with respect to vθ , which disregards
338   the difference between the role of these variables in the system.
339   Semantic backpropagation solves this issue by incorporating dependency and heterogeneity between
340   neighboring nodes into our formulation. In Section 5.2, we show empirical evidence that this issue
341   can decrease the optimization performance.

      4.1   T EXTUAL G RADIENTS FOR P ROMPT O PTIMIZATION
346
347   The concept of textual gradient was first introduced in the context of prompt optimization as “a local
348   loss signal which contains information on how to improve the current prompt” (Pryzant et al., 2023).
349   Furthermore, Pryzant et al. (2023) propose ProTeGi, an optimization method that improves prompt
350   parameters that are used to instruct an LLM to produce an answer. Given some query-expected-
351   answer pairs, ProTeGi computes a textual gradient for each pair and applies the textual gradients to
352   optimize the prompt parameter. By refining the prompt through a series of textual gradient-informed
353   edits, an improvement is observed over baseline methods including APE (Zhou et al., 2023) and
354
      AutoGPT (Significant Gravitas, 2023). GRAD-SUM (Austin & Chartock, 2024) extends ProTeGi by
      introducing a gradient summation procedure to prevent the prompt updates from being too specific to
355
      a query-expected-answer pair. GRAD-SUM also generalizes ProTeGi by incorporating an LLMs-
356
      as-a-judge module (Zheng et al., 2024; Fu et al., 2023; Chen et al., 2024) to relax the expected-
357   answer requirement of ProTeGi. While textual gradient (Significant Gravitas, 2023), ProTeGi and
358   GRAD-SUM introduce first-order optimization-like methods to text-based functions, these methods
359   are not defined for variables not directly connected to the output variable. In addition, we observe
360   an interesting analogy between optimizing textual prompts using textual gradients and optimizing
361   numerical prompts using numerical gradients, as done by Schmidhuber (2015), where numerical
362   prompts are used to query a world model (Schmidhuber, 1990) through a numerical interface.
363
364   4.2   OTHER BACKPROPAGATION -I NSPIRED M ETHODS FOR GASO
365
366   Trace (Cheng et al., 2024) models the execution trace of computer programs as computational graphs
367   (i.e., trace graphs). In Trace, the authors introduce OptoPrime as an optimization algorithm similar
368   to TextGrad and semantic backpropagation that uses a feedback signal propagated backwards in the
369
      trace graph and then used for optimization. For each node v in the trace graph, a subgraph that
      contains all nodes u such that there is a path from v through u and then to the output node is as-
370
      signed as feedback to v. Conceptually, this subgraph includes all computations influenced by v that
371
      have an effect on the output. Then, an optimizer (typically based on large language models) lever-
372   ages the subgraphs as feedback signals to improve the optimizable nodes. Although the subgraph
373   of v contains all relevant information on how v influences the output, it does not directly indicate
374   how improvements can be made (i.e., there is no gradient-like information). Without gradient-like
375   information gained from a backpropagation-like process, an optimizer must itself implicitly try to es-
376   timate such information—a very non-trivial task. Another issue with the aforementioned subgraphs
377   is that the size of these graphs scales linearly with the depth of the overall computational graph. This
      makes Trace incompatible with large graphs when the optimizer uses transformer-based language


                                                         7
      Under review as a conference paper at ICLR 2025


378
      models (Vaswani, 2017; Schmidhuber, 1992; Schlag et al., 2021). Unlike in Trace, semantic back-
379   propagation explicitly backpropagates this gradient-like information and is not limited by the same
380   linear scaling requirement. Zhou et al. (2024)—a concurrent work—introduces a backpropagation-
381   like method for GASO which includes edge optimization. However, this approach is limited to
382   chain-structured agentic systems, with edge optimization involving the addition, removal, or rear-
383   rangement of agents within the chain.
384
385   4.3   OTHER M ETHODS FOR GASO
386
387   DSPy (Khattab et al., 2024) attempts to abstract executions of language model pipelines into text
388   transformation graphs. Each node in such a graph is defined as a declarative module that takes a
389   type of text as input and transforms it to another type of text as output. These nodes are parameter-
390   ized by the instructions and sets of demonstrations that are prefixed to the input text before querying
391
      an LLM. DSPy has many implementations of optimizers for finding the best values for these pa-
      rameters. Two such optimizers are BootstrapFewshot, which provides few-shot demonstrations for
392
      each step in the pipeline, and COPRO, which does coordinate-ascent optimization on each of the
393
      optimizable instruction prompts in the pipeline. These two optimizers are locally focused, i.e. each
394   step is only implicitly aware of the entire pipeline.
395
396   GPTSwarm’s node optimization method Zhuge et al. (2024) solves GASO when the edges are fixed,
397
      which is the focus of this paper. Compared to COPRO, in each iteration of GPTSwarm’s node
      optimization method, each node’s parameter is updated with respect to a local objective function
398
      specific to the node. Such local objective functions are not always available and require an accurate
399
      understanding of the function of each specific node. Optimizing such a local objective function
400   also limits the possibility that a node could change its function through global optimization. On the
401   other hand, GPTSwarm offers an edge optimization method that can be used as a complement of our
402   method when approaching the GASO problem with optimizable edges.

532
533
      In this work, we tackled the challenge of optimizing language-based agentic systems by introduc-
534   ing semantic gradients and semantic backpropagation. These concepts generalize existing credit-
535   assignment methods, such as reverse-mode automatic differentiation and TextGrad, by incorporating
536   neighborhood conditioning to compute directional information which can be leveraged to improve
537   each optimizable component of the system. This framework enabled us to propose semantic gra-
538   dient descent, effectively solving the Graph-based Agentic System Optimization (GASO) problem.
539   Altogether, our results indicate that semantic gradients can significantly reduce the manual effort
      needed to optimize agentic systems, paving the way for greater scalability in AI solutions.
