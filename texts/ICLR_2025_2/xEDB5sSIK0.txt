                                                  A BSTRACT
010
011
012            Class imbalance is a pervasive issue in many real-world datasets, particularly in
013
               graph-structured data, where certain classes are significantly underrepresented.
               This imbalance can severely impact the performance of Graph Neural Networks
014
               (GNNs), leading to biased learning or over-fitting. The existing oversampling
015
               techniques often overlook the intrinsic properties of graphs, such as Label Infor-
016            mativeness (LI), which measures the amount of information a neighbor’s label
017            provides about a node’s label. To address this, we propose Label Informativeness-
018            based Minority Oversampling (LIMO), a novel algorithm that strategically over-
019            samples minority class nodes by augmenting edges to maximize LI. This tech-
020            nique generates a balanced, synthetic graph that enhances GNN performance with-
021            out significantly increasing data volume. Our theoretical analysis shows that the
022            effectiveness of GNNs is directly proportional to label informativeness, with mu-
023            tual information as a mediator. Additionally, we provide insights into how varia-
024
               tions in the number of inter-class edges influence the LI by analyzing its deriva-
               tive. Experimental results on various homophilous and heterophilous benchmark
025
               datasets demonstrate the effectiveness of LIMO in improving the performance of
026
               node classification for different imbalance ratios, with particularly significant im-
027            provements observed in heterophilous graph datasets. Our code is available at
028            https://anonymous.4open.science/r/limo-12CC/

032   1   I NTRODUCTION
033
034   The emergence of Graph Neural Networks (GNNs) has pushed the boundaries of graph structure
035   analysis (Joshi & Mishra, 2021). These networks harness node attributes and graph topology to
036   enhance learning outcomes. Approaches like Graph Convolutional Networks (GCNs) and Graph
037   Attention Networks (GATs) have shown marked improvements in tasks such as node classification
038   and link prediction (Kipf & Welling, 2017; Velickovic et al., 2018). By utilizing both node features
039
      and edge information, these methodologies capture intricate relationships within graphs, thereby
      boosting performance on graph tasks (Hamilton et al., 2017).
040
041   Class imbalance is a prevalent issue in many real-world datasets Kim et al. (2020), where certain
042   classes are significantly underrepresented compared to others in a dataset. That is, the number of
043   samples belonging to one class (the majority class) far exceeds that of another (the minority class).
044   Consider an example of classifying medical images for a certain disease. The classifier tends to
045
      fail to precisely classify if the dataset is skewed towards any one of the positive or negative classes
      for the patient having the disease Tasci et al. (2022). The classifier is likely to be biased towards
046
      predicting the class labels of the majority of the images in the dataset. Such imbalance skews the
047
      performance of machine learning models, and the model tends to favor the majority class because
048   it dominates the training process He & Garcia (2009). This is particularly problematic when the
049   minority class represents rare but critical cases, such as fraud or disease detection Batista et al.
050   (2004). It becomes difficult to use traditional machine learning algorithms while working with a
051   class-imbalanced dataset because they often assume an equal distribution of classes. In the presence
052   of class imbalance, the algorithms are likely to give biased predictions Shwartz-Ziv et al. (2024).
053   Specifically, models may achieve high overall accuracy by simply predicting the majority class more
      frequently, but their performance on the minority class remains poor.

084   Imbalanced node classification presents significant challenges for existing Graph Neural Networks
085   (GNNs). In scenarios where the majority class dominates, the loss function becomes skewed, caus-
086   ing the GNN to overfit to the majority class while neglecting the minority class. This leads to poor
087
      predictive performance on minority class samples, limiting the effectiveness of GNNs in real-world
      applications characterized by imbalanced class distributions, such as malicious account detection.
088
      Addressing this issue is crucial for improving the adoption of GNNs in such tasks. Many previ-
089
      ous works have tried addressing these challenges Chen et al., 2021; Zhao et al., 2021; Ashmore
090   & Chen, 2023; Wang et al., 2022b; Hsu et al., 2024. These approaches either add synthetic nodes
091   based on the features of the existing nodes in the graph or train the node classifier to learn with the
092   class-imbalanced dataset.
093
      In our work, we take a different approach to overcome the class imbalance. Our algorithm LIMO
094
      uses the concept of label informativeness of the given graph to mitigate the issue of class imbalance
095
      by strategically adding the edges to the graph. Empirically, it has been established that a positive
096   correlation exists between LI and model performance Platonov et al. (2024). In our work, we fur-
097   ther extend it and formally establish the positive correlation. In general, increasing the LI of the
098   graph increases model performance. Hence, we add the synthetically generated nodes and edges to
099   improve the graph’s label informativeness. Additionally, LIMO acts on the class imbalanced dataset
100   before it is given as an input to the GNN, thereby reducing the overhead cost of training the classifier
101   to learn on the imbalanced dataset.
102   Our main contributions are: First, we propose Label Informativeness-based Minority Oversam-
103   pling (LIMO). We theoretically establish the relationship between the label informativeness and the
104   accuracy of the model predictions in GNN. Additionally, we analyze the influence of change in
105   the number of inter-class and intra-class edges on LI. Finally, we empirically validate our proposed
106   method on the node classification task and observe that it outperforms state-of-the-art approaches
107   by a significant margin. Figure 1 gives us an illustration of how different components of LIMO help
      in generating the synthetic node features and edges the graph to include the synthetic nodes.

168   balanced graph G′ = {V,′ A,′ F,′ Y ′ }. To achieve this, the minority classes ci s are first identified
169   within the dataset. A matrix P is then defined to quantify the distribution of edges over the classes,
170   where P = [p(c1 , c2 )]C−1

238   In our study, we consider the LI of a subgraph by taking the nodes belonging to two classes of
239   interest (say c1 and c2 ). In equation 2 we substitute p(c1 , c2 ) = pbc , p(c1 , c1 ) = pwc , p(c1 ) = p1
240   and p(c2 ) = p2 , the formula for LI becomes:
241
242                                                 pbc log(pbc ) + pwc log(pwc )
                                     LI(G) = 2 −                                                               (5)

244   where:
245
                           ebc                  ewc                2ebc + ewc                  ewc
246            pbc =             ,   pwc =             ,   p1 =                 ,   p2 =
247                    ebc + ewc             ebc + ewc            2(ebc + ewc )            2(ebc + ewc )
248   In this equation, ebc represents the number of inter-class edges (between classes), and ewc represents
249   the number of intra-class edges (within class). To understand how changes in the number of inter-
250   class edges (denoted by ebc ) affect LI, we perform a differentiation of LI with respect to ebc . Upon
251   differentiating LI and evaluating it at ( ebc = 1.31167627 and ewc = 1 ), we find that the derivative is
252   greater than 0. This positive derivative indicates that an increase in the number of inter-class edges
253   increases LI. This result has significant implications for our understanding of graph structures and
254   their label distributions. Specifically, it suggests that enhancing the connectivity between classes
255
      (increasing inter-class edges) can improve the informativeness of the labels. This improvement in
      LI can lead to better performance in tasks such as node classification, where the quality of label
256
      information is crucial.

      4.2   R ELATIONSHIP BETWEEN L ABEL I NFORMATIVENESS AND ACCURACY WITH M UTUAL
259
            I NFORMATION AS A M EDIATOR
260
261   In the field of Graph Neural Networks (GNNs), grasping the connection between label informa-
262   tiveness and accuracy is essential for enhancing model efficacy. When node labels contain highly
263   informative data, GNN models are expected to generate more precise graph representations. This
264   theorem has been developed to formalize this relationship, offering a mathematical framework to
265   examine how label informativeness influences accuracy.
266   Theorem 2. Let I(Y, Z) be the Mutual Information between the node labels Y and Z. H(Y ) be
267   the entropy of the node labels, and H(Y |Z) be the conditional entropy of the node labels. Then, the
268   accuracy of the GNN model is directly proportional to the label informativeness.

      The proof is described in Appendix A.2.

526   6          R ELATED W ORKS
527
528   Data-level techniques on imbalanced graph data, such as oversampling and undersampling, aim
529   to balance the dataset by increasing the number of instances in the minority class or reducing the
530   number of instances in the majority class, respectively. However, undersampling may lead to the
531   loss of potentially useful data (Khan & Chandra, 2024). Oversampling methods like SMOTE (Syn-
532   thetic Minority Over-sampling Technique) Chawla et al., 2002 generate synthetic examples of the
533   minority class by interpolating the feature space of the nodes to achieve a more balanced distribu-
534   tion. However, interpolation in the feature space can generate out-of-context synthetic nodes, which
535   might lead to the biased learning of GNNs. Also, synthetic nodes borrow edges from their parent
536
      node, which might mislead the GNN. GraphSMOTE Zhao et al., 2021 mitigates this issue by using
      a GNN to predict the edges of synthetic nodes by learning from the graph itself. GraphSMOTE can
537
      be computationally intensive and complicated, which might lead to longer GNN training times.
538
539   ReNode Chen et al. (2021) addresses the problem of class imbalance from the perspective of topol-
      ogy imbalance and proposed a model-agnostic method designed to tackle the issue of topology


                                                                                       10
      Under review as a conference paper at ICLR 2025


540
      imbalance. It achieves this by adaptively re-weighting the influence of labeled nodes according to
541
      their relative positions to the class boundaries. G2 GNN Wang et al. (2022b) alleviates the graph
542   imbalance issue by deriving extra supervision globally from neighboring graphs and locally from
543   stochastic augmentations of graphs. The recent work HOVER Ashmore & Chen (2023) involves a
544   simple yet effective edge removal method to mitigate heterophily and learn distinguishable node em-
545   beddings. These are then used to oversample minority bots to generate a balanced class distribution.
546   FincGAN Hsu et al. (2024) employs a Generative Adversarial Network (GAN) to generate synthetic
547   samples for minority classes, avoiding over-fitting issues common with traditional oversampling
548   methods.
549   ImGAGN Qu et al., 2021 is an adversarial network-based architecture that adds a set of synthetic mi-
550   nority nodes to overcome the class imbalance. TAM Song et al., 2022 loss, which is topology-aware
551   margin loss for class imbalanced node classification, performs well by comparing the connectivity
552   pattern of each node with the class-averaged counterpart and adaptively adjusting the margin accord-
553   ingly. mGNN Wang et al., 2022a mitigates the class imbalance by oversampling after performing
554   the feature aggregation.
555   Recent studies such as Hsu et al. (2024) and Jing et al. (2024) tackle graph imbalance through
556   oversampling methods. Hsu et al. (2024) utilizes GANs to create synthetic nodes and edges, but
557   this approach is computationally expensive. Jing et al. (2024) employs dual-feature aggregation to
558   address heterophily and conducts oversampling in the embedding space, avoiding edge synthesis. In
559   contrast, the proposed LIMO directly increases a graph property LI through strategic node and edge
560   augmentation. This increases model performance and improves minority class representation.
561   In the field of graph-based learning, several innovative approaches have emerged to address class
562   imbalance. These include Park et al. (2022), which generates ego networks for minority class nodes
563   while maintaining structural consistency and employing saliency-based node mixing to avoid intro-
564   ducing class-specific features. Another method, Song et al. (2022), implements a topology-aware
565   margin loss to enhance the separation of minority class nodes while preserving graph structure. Ad-
566   ditionally, Li et al. (2023) creates more challenging samples for underrepresented classes, thereby
567   enhancing training effectiveness in imbalanced scenarios. While these techniques offer innovative
568
      solutions, LIMO sets itself apart by directly utilizing Label Informativeness (LI) to guide both node
      and edge augmentation. This approach results in balanced graph representations that are specifically
569
      optimized for downstream GNN tasks.
570
571
572   7   C ONCLUSION
573
574   In this work, we introduced Label Informativeness-based Minority Oversampling (LIMO), a novel
575   approach to addressing class imbalance in graph-structured data. By augmenting edges in a manner
576   that maximizes Label Informativeness (LI), LIMO strategically oversamples minority class nodes
577   without significantly inflating the dataset. Our theoretical analysis revealed that the performance
578   of Graph Neural Networks (GNNs) is strongly correlated with label informativeness, with mutual
579   information acting as a key intermediary. The analysis of the derivative of LI further provided a
580   deeper understanding of the impact of inter-class edges on informativeness. Experimental results
581   on various benchmark datasets, both homophilous and heterophilous, demonstrated that LIMO sub-
582
      stantially improves node classification accuracy, particularly in heterophilous settings where class
      imbalance is more pronounced.
583
584   Despite its strengths, LIMO has certain limitations. First, while the algorithm balances the dataset
585   without inflating it excessively, there is still an inherent computational cost associated with gener-
586   ating and evaluating new edges. Second, LIMO’s effectiveness depends on the accuracy of label
587   informativeness estimation, which may be less reliable in graphs where the node labels exhibit low
588
      correlation with their neighbors. Overcoming these limitations remains an open area for future in-
      vestigation.

762

769                                                         H(Y |Z)
770                                          Error Rate =
                                                             H(Y )

773

777

792   A.3   C OMPLEXITY A NALYSIS
793
794   In this section, we note the computational cost of the LIMO algorithm, accounting for both node
795   and edge additions.
796
797   S PACE C OMPLEXITY
798
             • LIMO (Node Addition): The number of nodes added by LIMO is equal to the number of
799
               minority class nodes, denoted as nminority . Thus, the space required for additional nodes is
800            O(nminority ).
801
802
             • LIMO (Edge Addition): The number of edges added depends on the original LI value and
               the adjacency matrix of the graph. This number, denoted as nedges , can vary across datasets.
803
               Hence, the space complexity for edges is O(nedges ).
804
805
806
      T IME C OMPLEXITY
807          • LIMO (Node Addition) For each new node added by SMOTE, we need to find the nearest
808            nodes in the feature space across all the nodes in the graph. Finding the nearest nodes
809            requires O(n), where n is the total number of nodes in the graph. Since we add nminority
               nodes, the total time complexity for SMOTE is: O(nminority · n)


                                                       15
      Under review as a conference paper at ICLR 2025


810
             • LIMO Edge Addition For every edge added, calculating pc1 and pc2 (graph-specific prop-
811            erties) requires O(n2 ). This is calculated for all the classes: O(C · n2 ), where C is the
812            number of classes.
813
               For each newly generated node, which is nminority in number, we have to compare edge
814            probability pc1 and pc2 across all classes. time complexity for this is O(C · nminority ). Since
815            O(C · nminority ) is smaller than O(C · n2 ), it can be ignored.
816
817   Overall time complexity by combining the above: Total Time Complexity = O(nminority · n) + O(C ·
818   n2 )

822
823   B.1   E VALUATION M ETRICS
824
      We have used Accuracy, AUC-ROC, and F1 scores to judge the performance of the GNN on imbal-
825
      anced datasets. The details of these metrics are as follows:
826
827   Accuracy
828
829
      Accuracy measures the proportion of correct predictions made by the model out of all predictions
      made. It is calculated using the formula:
830
831
832                                             Number of Correct Predictions
                                  Accuracy =
833                                              Total Number of Predictions
834
      While this is a good metric to infer the overall performance of a model, it fails to tell the whole
835
      story when there is a stark imbalance in the class distribution of the nodes. Even if all the minority
836
      is classified as majority class the accuracy score will be high. It also fails when the detection of one
837   class correctly is more important than the other classes.
838
839   Area under the receiver operating characteristic curve (AUC-ROC)
840
      It is a performance measurement for a classification problem defined as the area under the true
841
      positive rate versus the false positive rate plot, ranging from 0 to 1. The true positive rate, also
842
      known as sensitivity or recall, is the ratio of the correctly predicted positives to the sum of the
843   correctly predicted positives and incorrectly predicted negatives. The false positive rate is the ratio
844   of incorrectly predicted positives to the sum of incorrectly predicted positives and correctly predicted
845   negatives.
846
847   F1-Score
848
      The metric is defined as the harmonic mean of precision and recall, ranging from 0 to 1. Precision
849   is the ratio of correctly predicted positives to all positives. Recall is the same as was defined for
850   AUC-ROC.
851
852
                                                         Precision × Recall
853                                    F1-Score = 2 ×
                                                         Precision + Recall
854

      B.2   BASELINES
856
857   To evaluate LIMO’s performance, we compared it against several state-of-the-art oversampling tech-
858   niques, including Oversampling (OS), Re-weight (RW), SMOTE (SM), Embed-up (ES), and Graph
859   SMOTE (GS). A brief description of the baselines is given below:
860
861         1. Over-sampling (OS) is a classical approach for imbalanced learning problems by repeating
862            samples from minority classes. We implement it in the raw input space by duplicatingns
863            minority nodes along their edges. In each training iteration, V is over-sampled to contain
               n + ns nodes, and A ∈ R(n+ns )×(n+ns ) .Zhao et al. (2021)


                                                         16
      Under review as a conference paper at ICLR 2025


864
            2. Re-weightYuan & Ma (2012) (RW) is a cost-sensitive approach that gives class-specific
865            loss weight. In particular, it assigns higher loss weights to minority samples to alleviate the
866            issue of majority classes dominating the loss function.
867
868
869
            3. SMOTE (Synthetic minority over-sampling technique)Chawla et al. (2002) (SM) in the
               feature space is the interpolation of the synthetic data point between the target node and the
870
               node that is nearest to it in the feature space, nn(v) as given by
871
872
873                                     nn(v) = arg min ∥fu − fv ∥, s.t. Yu = Yv
                                                       u
874
875
876            where fu and fv are the features of u and v nodes, respectively, and Yv are the labels of u
877            and v vertices. The features of the synthetic node are given by
878
879
                                            fv′ = (1 − δ) × fv + δ × fn n(v)
880
881
882
883         4. Embed-SMOTE (ES) Ando & Huang (2017) A variant of SMOTE adapted for deep learn-
884            ing, designed to oversample data within the intermediate embedding layer of a GNN. This
885            approach eliminates the need for edge generation by operating directly on the learned rep-
886
               resentations.
887
888         5. GraphSMOTE (GS) Zhao et al. (2021) generates synthetic nodes similar to smote, but here,
889            it interpolates in the embedding space to create the embedding of the synthetic node. There
890            is an edge generator that learns using a neural network whether there exists an edge between
891            given nodes. A GNN then does the classification; some versions also use the loss in the
892
               classification to train the embedding and the edge generator.
893

896
897   We experiment on each dataset using a standard split of 20 nodes for training, 25 for validation,
898   and 55 for testing in the majority class. For minority classes with an imbalance ratio i ∈ [0, 1], we
899   sampled 20 × i nodes. When the minority class had fewer than three nodes, we allocated one node
900   each for training, validation, and testing. To evaluate LIMO, we compared it with several baselines:
901   Over-sampling (OS), Reweight (RW), SMOTE (SM), Embed-SMOTE (ES), and Graph SMOTE
902   (GS). All experiments were performed on NVIDIA GeForce RTX 3090, A100-SXM4-80GB, and
903   RTX 6000 Ada Generation GPUs in Python using PyTorch and PyG. We employed GraphSAGE as
904
      the GNN architecture for training on the balanced datasets created by LIMO and the baselines. We
      conducted experiments with three random seeds (10, 20, 30) to mitigate randomness and averaged
905
      the results. The GraphSAGE model used two layers with a linear layer output dimension of 64
906
      for both layers. ReLU activation was employed, and we used Adam optimizer for training with
907   a learning rate of 0.001. We either terminate the training after 5000 epochs or when validation
908   performance plateaued.
