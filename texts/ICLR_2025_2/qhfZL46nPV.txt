009                                                 A BSTRACT
010
011            Counterfactual explanations in reinforcement learning (RL) aim to answer what-
012            if questions by showing sparse and minimal changes to states, which results in
013            the probability mass moving from one action to another. Although these expla-
014            nations are effective in classification tasks that look for the presence of concepts,
015            RL brings new challenges that current counterfactual methods for RL still need
016            to solve. These challenges include defining similarity in RL, out-of-distribution
017            states, and lack of discriminative power. Given a state of interest called the query
018
               state, we solve these problems by asking how long the agent can execute the query
               state action without incurring a negative outcome regarding the expected return.
019
               We coin this outcome-based semifactual (OSF) explanation and find the OSF state
020
               by simulating trajectories from the query state. The last state in a subtrajectory
021            where we can take the same action as in the query state without incurring a nega-
022            tive outcome is the OSF state. This state is discriminative, plausible, and similar
023            to the query state. It abstracts away unimportant action switching with little ex-
024            planatory value and shows the boundary between positive and negative outcomes.
025            Qualitatively, we show that our method explains when it is necessary to switch
026            actions. As a result, it is easier to understand the agent’s behavior. Quantita-
027            tively, we demonstrate that our method can increase policy performance and, at
028            the same time, reduce how often the agent switches its action across six environ-
029
               ments. The code and trained models are available at https://anonymous.
               4open.science/r/osf-explanation-for-rl-E312/.

      1    I NTRODUCTION
033
034
      Reinforcement learning (RL) has shown incredible performance in several domains. These include
035
      surpassing human performance in various games like Go, chess, poker, Atari (Mnih et al., 2013;
036   Schrittwieser et al., 2020; Silver et al., 2016; Brown & Sandholm, 2017), robotics (Tang et al.,
037   2024), and healthcare (Yu et al., 2023). Much of these accomplishments are due to neural networks
038   being flexible function approximators. However, the flexibility is obtained by sacrificing other de-
039   sirable properties, such as explainability (Arrieta et al., 2020). Explainability is crucial since it helps
040   stakeholders, from the end users to researchers, understand and trust RL systems. By understanding
041   the systems, they can be improved, corrected, and be safely deployed. Furthermore, they can be
042   used in high-stake domains such as healthcare, criminal justice, and finance (Yang et al., 2023).
043   Explainability in RL has become increasingly popular recently and resulted in the research field
044   known as explainable reinforcement learning (XRL) (Qing et al., 2023; Glanois et al., 2024; Amitai
045   & Amir, 2024; Milani et al., 2024; Hickling et al., 2024; Bekkemoen, 2024). The XRL commu-
046   nity has mainly focused on leveraging methods from the supervised learning explainability field and
047   often does not consider the specific challenges that make explainability in RL difficult. These chal-
048   lenges include delayed rewards, stochastic environments, and large state spaces (Amitai & Amir,
049   2024). For example, feature importance is one of the most popular methods in XRL but does not
050
      specifically tackle sequential decision-making (Bekkemoen, 2024). We need additional explainabil-
      ity tools tailored towards RL to better understand the behavior in sequential decision-making.
051
052   Counterfactual explanations are relatively new in XRL and can potentially help us understand RL
053   agents. However, the existing counterfactual methods in RL have problems, which we illustrate with
      an example from Huber et al. (2023). In Fig. 1, we notice two problems with a generative approach to

      Semifactual explanations describe how much we can modify input features without changing the
101
      agent’s action (Aryal & Keane, 2024). Semifactual states are states furthest away from the query
102   state without crossing the decision boundary, shown in Fig. 3a, but suffer from a lack of discrimi-
103   native power like counterfactual explanations. To alleviate the problems mentioned, we propose the
104   outcome-based semifactual (OSF) explanation and introduce a simulation-based approach to find
105   OSF states to explain the agent’s behavior. Our OSF explanations differ from semifactual explana-
106   tions used in supervised learning, we aim to look at it from the value space perspective rather than
107   looking at the probability distribution over actions. Given a query state and simulated trajectories
      from it, we ask how long we can keep executing the query state action and expect a similar (but

      not necessarily optimal) outcome. For example, Fig. 3b illustrates a OSF state in the mountain car
109   environment where we keep accelerating left beyond the decision boundary while still being able

115   In summary, we propose: 1) Outcome-based semifactual (OSF) explanations that focus on the out-
116   come of an agent. They enable us to understand which action switches are critical to the agent and
117   which can be abstracted away. Furthermore, they are important states on the boundary between pos-
118   itive and negative outcomes that further help us understand the agent. 2) We show the effectiveness
119   of OSF explanations via various case studies and compare them to counterfactual explanations using
120   the nearest neighbor search. 3) We demonstrate that we can use OSF explanations to minimize the
121   number of times the agent switches action during rollouts. Furthermore, we show that this mini-
122   mization can improve policy performance in six environments while abstracting away unimportant
123
      behavior with little explanatory value.

128   Reinforcement Learning. In RL, we model the environment as a Markov decision process (MDP)
129
      consisting of a tuple ⟨S, A, T, R, γ⟩ ( Sutton & Barto, 2018; Achiam, 2018; Chapter 2 in Albrecht

130

131
      discount factor. Additionally, we have the starting state distribution defined by µ : S → [0, 1]. An
132   agent interacts with the environment via a policy π : S → A that takes a state and outputs a corre-
133   sponding action. The policy can either output a distribution over actions or be indirectly defined via
134   a state-action value function. The trajectory is defined by τ = (s1 , a1 , s2 , a2 , . . .) where s1 ∼ µ.

144   Counterfactual and Semifactual Explanation. We define a counterfactual explanation with re-
145   spect to a policy π as a tuple ⟨s, s′ ⟩ where s is the query state and s′ is the counterfactual
146   state (Guidotti, 2022). Given the policy π and query state s, we get π(s) = a. The counterfactual
147
      state s′ is similar to s but where the features have small sparse changes such that π(s′ ) = a′ ̸= a.
      The “optimal” counterfactual state has certain desirable properties such as validity, sparse changes
148
      with respect to the query state, and similarity. A semifactual explanation is similar where the fea-
149
      tures are altered but where the query state s and the semifactual state s′ maintain the same action,
150   π(s′ ) = a = π(s). We want the change between the query and semifactual states to be as large as
151   possible to observe the states’ differences while not crossing the decision boundary. Like counter-
152   factual explanations, semifactual explanations should satisfy certain desirable properties (Aryal &
153   Keane, 2023). Figure 3a shows an example of a query state, a counterfactual state, and a semifactual
154   state.

190     State Importance. State importance methods are among the most popular in XRL. As far as we
191     know, Amir & Amir (2018) and Huang et al. (2018) proposed the earliest work in this category of
192     methods. They show stakeholders a summary of important states to understand an agent’s global
193     behavior. The importance definition varies between studies. Some studies use Q-values to measure
194     state importance, while others use quantities like interestingness (Sequeira & Gervasio, 2020). User
195
        studies have shown that these explanations are effective at aligning mental models (Huber et al.,
        2021; Amitai & Amir, 2024). One of their strengths is their ease of use and applicability. They
196
        discover and find these important states via simulations. The downside is the need for a state transi-
197
        tion model, which can partially be fixed by learning world models (Ha & Schmidhuber, 2018). New
198     importance measures have been proposed in later studies (Huang et al., 2019; Lage et al., 2019;
199     Sequeira & Gervasio, 2020). Our method similarly leverages Q-values as an importance measure. It
200     uses a state transition model like the methods above. However, our goal is to abstract away complex
201     agent behavior, and find states that are important and on the boundary between positive and negative
202     outcomes.
203
        Counterfactual Explanations in RL. Both Olson et al. (2021) and Huber et al. (2023) use genera-
204     tive adversarial networks to generate counterfactual states. In the introduction, we mentioned prob-
205     lems with these approaches, namely out-of-distribution states, how to handle the similarity measure,
206     and a lack of discriminative power. To solve these problems, we improve upon these counterfactual
207     methods by proposing a different type of explanation, namely OSF explanations.
208
        Semifactual Explanations. Semifactual explanations are not new and have been researched in the
209
        context of supervised learning (Kenny & Keane, 2021; Kenny & Huang, 2023). However, there has
210     been little work on semifactual explanations in RL (Gajcin et al., 2024). Semifactual explanations
211     deal with even-if explanations (Aryal & Keane, 2024). For instance, imagine we have a system
212     that predicts the risk of diabetes. An explanation could be that a person has a low risk of diabetes
213     according to the system and that even if they doubled the sugar intake, the risk would still be low.
214     This is a semifactual explanation since the decision stays the same while the input changes. We
215     extend semifactual explanations and use OSF explanations to explain the outcome of actions as seen
        in Fig. 3b.

218   We introduce the concept of outcome-based semifactual (OSF) explanations and how we can find
219   OSF states algorithmically.
220
221
      What is an OSF explanation? An OSF explanation consists of a tuple ⟨st , s′t+n , δ⟩ that is con-
      structed with respect to a policy. st is the query state, s′t+n is the OSF state, and δ is the stopping
222
      criterion. We assume the policy is represented as a Q-function. Given at = arg maxa Q(st , a),
223
      s′t+n is the state n timesteps after st by only executing the action at and where maxa Q(st+n , a) −
224   Q(s′t+n , at ) ≤ δ. st+n is the state n timesteps after st by following the policy. Because s′t+n is
225   produced by a rollout from st , we assume that changes in features are sparse. Additionally, the OSF
226   state is visually similar to the query state given that δ is not too large.
227
      Finding an OSF state. We want to understand how much a query state st can be modified by execut-
228
      ing the same action without affecting the future outcome negatively. We define negative outcomes as
229
      states where the return is significantly lower than what is achieved by following the policy. We seek
230   an OSF state, as seen in Fig. 3b. To find the OSF state, we assume access to a Q-function and a state
231   transition probability function T that enables us to simulate trajectories. OSF states we find depend
232   on whether the state transition probability function is stochastic or deterministic. The experiments
233   use deterministic state transition probability functions and are therefore unaffected by stochasticity.
234   Using the Q-function, we define the importance gap between two states s and s′ conditioned on the
235   action a by
236                                  IG(s, s′ | a) = max′
                                                           Q(s, a′ ) − Q(s′ , a).                        (1)
                                                             a

238   The method starts by first following the policy from the state st to find the trajectory τ =
239   (st , at , st+1 , at+1 , . . . , sN ). After obtaining the trajectory τ , we simulate a new trajectory from
240
      the state st , but instead of following the policy, we execute the action at = arg maxa Q(st , a) to all
      states that follow and obtain the trajectory τ ′ = (st , at , s′t+1 , at , s′t+2 , at , . . . , s′T ). The OSF state is
241
      found by applying the following criterion
242
243                                    n = (arg min IG(st+i , s′t+i | at ) > δ) − 1                                      (2)
                                                  i

      to the trajectories. Hence, s′t+n is the OSF state. The criterion says to keep executing action at as
      long as the gap is lesser than or equal to δ. The criterion only needs to be one-sided since a negative
246
      gap tells us we are reaching states that are better than following the policy. The gap will generally
247
      be positive as the policy is “optimal”. We allow a small gap since a small loss in expected return
248   will not affect the outcome negatively. For example, a small δ will still allow the policy in mountain
249   car to reach the goal, albeit taking a few more timesteps.
250
251
      The δ value depends on the environment because the expected return varies based on the reward,
      which differs between environments. On the one hand, a small δ value results in the query state and
252
      OSF state being very similar and lowers the discriminative power. On the other hand, a large δ value
253
      makes the OSF state dissimilar to the query state and ends up being a state that the policy rarely
254   encounters. The δ value has to be set by a human stakeholder that interactively explores different δ
255   values.

281   We look at OSF explanations for two query states across three environments: mountain car, Pong,
282   and Breakout. For Pong and Breakout, in addition to OSF explanations, we inspect the counterfac-
283   tual explanations for the same query states. The counterfactual states are found by looking for the
284   closest states based on the policy’s embedding space. To accomplish that, we generate 100 000 states
285   from simulations to find the closest states. From those 100 000, we select the 100 closest states and
286   sample 4 states from those 100 at random to avoid selecting states too visually similar. These 4 are
287   visualized as the counterfactual states in Figs. 6, 8, 10 and 12.

289   5.2.1    M OUNTAIN C AR
290
      In the first case study, we look at specific parts of the decision boundary that are unsmooth in
291
      the mountain car environment. We ask why they exist and whether they are necessary. Since the
292
      mountain car environment only has two features, we can easily visualize its state space and the
293   corresponding decision boundaries.
294
295   In Fig. 4a, the query state is in a region where the policy accelerates right but changes quickly to
296
      do nothing before changing to accelerate left. In this specific case, we see that the OSF state is
      in the accelerate left region of the state space. This means we can skip the do nothing action and
297
      keep accelerating right until gravity pulls the car into the accelerate left region. By only using the
298
      query state action, the agent can reach the goal quicker than by following the policy. It is difficult to
299   conclude that the do nothing region is useless. However, in this case, the complexity of the decision
300   boundary can be reduced.
301
302
      Fig. 4a shows that specific parts of the orange region are not necessary, while Figs. 4b and 4d depict
      that other parts are needed. Figs. 4b and 4d show that the orange region helps reduce the car’s
303
      acceleration when going towards left. Although we want the car to get high up on the left to get
304
      enough speed to go right, too high is unnecessary. Going too high up to the left wastes time because
305   the car can get to the goal with less speed. Fig. 4c shows the small blue island next to the query state
306   is unneeded as the OSF trajectory after the island is similar to the original trajectory. Figs. 4e and 4f
307   show two situations where a small patch of orange area next to the query state decreases the time it
308   takes to reach the goal if we use it and increases the time if we do not use it.
309
      To conclude, we have observed that the complexity of the decision boundaries are sometimes needed.
310
      While in other cases, they add unnecessary complexity without increasing the policy’s performance.

516   6   D ISCUSSION AND C ONCLUSION
517
518
519
      We presented a new method to understand the long-term behavior of RL agents. The method finds
      outcome-based semifactual (OSF) explanations that focus on explaining the outcome rather than
520
      actions. Given a state of interest, called the query state, the OSF explanation asks how long an
521
      agent can execute the same action before it has to switch due to a negative outcome. We achieve
522   this by simulating trajectories and keeping track of the expected return if the agent keeps using the
523   policy versus executing the same action. When the difference between these two estimates reaches
524   a threshold, we stop and set the state as the OSF state. Like state importance methods, we show
525   the entire rollout from the query state to the OSF state to better understand the agent’s behavior. To
526   demonstrate the usefulness of our method, we qualitatively show examples of explanations produced
527   by our method in three environments. The results show many situations where the agent unneces-
528   sarily changes actions, increasing the complexity of understanding its behavior. To emphasize that
529   the usefulness extends beyond these few examples, we quantitatively show how our method can
530
      improve policy performance while reducing the number of action switches across six environments.
531   Our method depends on selecting effective query states. However, the same applies to most local
532   explanations, such as counterfactuals or saliency maps. Another limitation of our method is the need
533   for a state transition probability function. This can be improved by training world models. Because
534   our method does not need to forecast far into the future, we believe small compounding errors in the
535   state transition probability model should not significantly impact our method. Finally, our method
536
      can be computationally intensive. We need to have two environments initialized to the same query
      state for rollouts and comparisons.
537
538   In the future, we should perform evaluations through user studies and consider how they should be
539   set up so that the results are comparable to other works. Finally, we should investigate how using
      world models affects the method.
