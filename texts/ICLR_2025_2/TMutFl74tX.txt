009                                                 A BSTRACT
010
011            We propose a meta-learning method for learning from multiple noisy annotators.
012            In many applications such as crowdsourcing services, labels for supervised learn-
013            ing are given by multiple annotators. Since the annotators have different skills or
014            biases, given labels can be noisy. To learn accurate classifiers from such data, ex-
015            isting methods require many noisy annotated data. However, sufficient data might
016            be unavailable in practice. To overcome the lack of data, the proposed method uses
017            labeled data obtained in different but related tasks. The proposed method embeds
018
               each example in tasks to a latent space by using a neural network and constructs a
               probabilistic model for learning a task-specific classifier while estimating annota-
019
               tors’ abilities on the latent space. This neural network is meta-learned to improve
020
               the expected test classification performance when the classifier is adapted to a
021            given small amount of annotated data. This classifier adaptation is performed by
022            maximizing the posterior probability via the expectation-maximization (EM) al-
023            gorithm. Since each step in the EM algorithm is easily computed as a closed-form
024            and is differentiable, the proposed method can efficiently backpropagate the loss
025            through the EM algorithm to meta-learn the neural network. We demonstrate the
026            effectiveness of the proposed method with real-world datasets with synthetic noise
027            and real-world crowdsourcing datasets.

030   1    I NTRODUCTION
031
032   Supervised learning requires labeled data for learning classifiers. In real-word applications, the
033   labels are often given from multiple annotators. For example, in crowdsourcing services, we can
034   obtain the labels by outsourcing annotation tasks to multiple workers (Zhang et al., 2016; Sheng &
035
      Zhang, 2019). In medical care or cyber security, multiple experts often perform annotation because
      annotation is quite time-consuming and difficult (Mimori et al., 2021; Salem et al., 2021).
036
037   In such situations, an essential challenge is that the obtained labels are generally noisy. This is be-
038   cause the performance of different annotators can vary widely. For example, in a crowdsourcing
039   service, some workers are simply spammers that provide random labels to easily earn money (Kazai
040   et al., 2011). In medical care or cyber security, even experts have much variability in annotation
041
      performance (Raykar et al., 2010; Mimori et al., 2021). When such noisy labels are used, standard
      supervised learning methods cannot perform well (Song et al., 2022). Thus, many methods have
042
      been proposed for learning classifiers from noisy labeled data obtained from multiple annotators
043
      without knowing ground truth labels (Raykar et al., 2010; Kajino et al., 2012; Rodrigues & Pereira,
044   2018; Chu et al., 2021). These methods usually require a large amount of annotated data to deal
045   with noisy labels. However, such data might be difficult to collect in some applications. For exam-
046   ple, since the crowdsourcing services usually charge a fee on the basis of the number of data and
047   annotators, enough annotated data are difficult to collect when budgets are constrained. In medical
048   care or cyber security, due to the scarcity of data and expertise requirements, it is difficult to collect
049   both sufficient data to be labeled and experts who can annotate.
050   Even if a large amount of data is difficult to collect on a task of interest, called a target task 1 ,
051   sufficient data with ground truth labels might be available in different but related tasks, called

      In the inner problem of the proposed method, we first embed the given task-specific examples to a
      latent space by a neural network and build a probabilistic latent variable model for learning clas-
076
      sifiers from multiple annotators on the latent space. By using the high expressive capability of
077
      neural networks, we aim to meta-learn embeddings suitable for our problem. Since source tasks
078   contain only clean labeled data, we artificially introduce label noises to the task-specific examples at
079   every meta-training iteration to simulate the test environment. We will demonstrate that this pseudo-
080   annotation strategy is critical to improve performance in our experiments. With our probabilistic
081   model, a ground truth label for each given example is treated as a latent variable, and the embed-
082   ded task-specific examples are modeled by the Gaussian mixture model (GMM) on the latent space.
083   Each component of GMM corresponds to a ground truth class label (latent variable). Further, each
084   annotator’s ability is modeled as an annotator-specific confusion matrix, whose entries are the prob-
085   ability of the observed noisy label given a ground truth label. This modeling allows the GMM to
086
      be fitted to the embedded data while accounting for each annotator quality. The GMM parameters
      and annotator-specific confusion matrices are task-specific parameters and are obtained by maxi-
087
      mizing the posterior probability given the pseudo-annotated task-specific data and the priors with
088
      the expectation-maximization (EM) algorithm. By using the adapted GMM parameters, the task-
089   specific classifier is obtained as the posterior probability of the ground truth label given an example
090   in a principled manner.
091
092
      In the outer problem, we meta-learn the parameters of the neural network such that the expected test
      classification performance, which is directly calculated with clean labeled data, is improved when
093
      the adapted classifier is used. Since each step in the EM algorithm is easily obtained as a closed-
094
      form and is differentiable, we can efficiently solve the bi-level optimization with a standard gradient
095   descent method. By meta-learning the neural network on various tasks, we can obtain example
096   representations suitable for learning from multiple noisy annotators on unseen tasks.

100
101   Many methods have been proposed for learning from multiple noisy annotators (Sheng & Zhang,
102   2019; Zhang et al., 2022). Early methods focused on estimating ground truth labels for given train-
103
      ing data (Dawid & Skene, 1979; Whitehill et al., 2009; Venanzi et al., 2014; Welinder et al., 2010).
      Majority voting (MV) is the simplest method. However, it often does not work well because it ig-

      nores per-annotator characteristics (Raykar et al., 2010). The Dawid and Skene (DS) model (Dawid
109   & Skene, 1979) is a famous probabilistic model that explicitly models the annotator’s ability as
110   a class-dependent and input example-independent confusion matrix. It estimates the ground truth
111   labels via the EM algorithm. Due to its effectiveness, the DS model has been the basis for subse-
112   quent studies (Whitehill et al., 2009; Venanzi et al., 2014; Welinder et al., 2010; Li & Yu, 2014).
113   Later studies focused on directly learning classifiers from noisy labeled data given by multiple an-
114   notators (Raykar et al., 2010; Kajino et al., 2012; Rodrigues et al., 2013). Among them, neural
115   network-based methods have gained attention for their high expressive capability (Albarqouni et al.,
116   2016; Rodrigues & Pereira, 2018; Chu et al., 2021; Chu & Wang, 2021; Wei et al., 2022; Li et al.,
117
      2020; Gao et al., 2022; Guo et al., 2023; Li et al., 2024). Although these methods are promising,
      when there are insufficient data, they do not work well since overfitting easily occurs. Although our
118
      method also uses neural networks, they are shared across all tasks and are learned with all data in
119
      source tasks. Therefore, our method can learn from a few data without overfitting.
120
121   Meta-learning aims to learn how to learn from a few data by using data in related tasks (Hospedales
122   et al., 2020; Vanschoren, 2018). Although most works focus on few-shot classification with clean
123
      labeled data, a few methods use the meta-leaning for learning from multiple noisy annotators (Zhang
      et al., 2023; Xu & Zhang, 2022; Han et al., 2021b;a). Zhang et al. (2023) and Han et al. (2021a) use
124
      a model pre-trained by ordinary meta-learning or transfer learning for estimating ground truth labels
125
      of only given examples in target tasks. Unlike the proposed method, these methods cannot directly
126   learn classifiers on target tasks. One method requires additional annotators for testing examples in
127   target tasks (Han et al., 2021b). Most importantly, all these methods use pre-trained models without
128   noisy annotations as feature extractors and simply apply learning methods from noisy annotators to
129   only target tasks. In contrast, the proposed method incorporates the classifier learning from noisy
130   annotators during the meta-learning phase to mimic the test environment. By constructing appropri-
131   ate probabilistic models in the inner optimization, the proposed method can adapt to data given from
132   noisy annotators via the closed-form EM steps, which leads to effective and fast meta-learning. We
133   demonstrate that this learning strategy dramatically improve performance in our experiments.
134   In the meta-learning methods, gradient-based methods such as model-agnostic meta-learning
135   (MAML) (Finn et al., 2017) are a representative approach, which solves the inner problems by
136   using gradient descent methods. They require second-order derivatives of the parameters of neu-
137   ral networks to solve the bi-level optimization and thus impose high computation cost (Rajeswaran
138   et al., 2019; Bertinetto et al., 2018). The proposed method can solve the inner problem via the EM
139   algorithm, where each EM step is easily obtained as a closed-form. Thus, the proposed method
140
      is more efficient than MAML. In addition, unlike MAML, the EM algorithm does not require the
      step size of gradient descents to be determined, which greatly affects meta-learning performance.
141
      Another representative approach is the embedding-based methods that meta-learns neural networks
142
      for embeddings, such as prototypical network (Snell et al., 2017). It calculates the prototype of each
143   class as the average of the embedded training data in the same class and classifies a new example on
144   the basis of the distance between each prototype and the embedded example. The proposed method
145   can be regarded as a natural extension of the prototypical network. Specifically, in our probabilistic
146   model, each prototype is obtained as the weighted average of the training data, where the weights are
147   estimated from noisy labeled data in a principled manner. A few studies formulate the inner prob-
148   lem as probabilistic modeling for topic modeling or clustering (Iwata, 2021a;b; Lee et al., 2020).
149   Our method also incorporates the probabilistic model for learning from multiple annotators in the
150   meta-learning.

      4.1   DATA
355
356   In the main paper, we used three real-world datasets: Omniglot, Miniimagenet, and LabelMe. Om-
357   niglot and Miniimagenet are commonly used in meta-learning studies (Snell et al., 2017; Finn et al.,
358   2017; Rajeswaran et al., 2019; Zhang et al., 2023), and LabelMe is a real-world crowdsourcing
359   dataset that is commonly used in crowdsourcing studies (Rodrigues & Pereira, 2018; Chu et al.,
360   2021; Chu & Wang, 2021). Omniglot consists of hand-written images of 964 characters (classes)
361   from 50 alphabets (Lake et al., 2015). Miniimagenet consists of images of 100 classes obtained
362   from the ILSVRC12-dataset (ImageNet) (Vinyals et al., 2016). Omniglot and Miniimagenet con-
363
      sist of only clean labeled data. Since meta-learning requires many tasks (classes), we used these
      datasets. LabelMe consists of 2,688 images of 8 classes. One-thousand of them were annotated
364
      from 59 workers in Amazon Mechanical Turk (Rodrigues et al., 2017). Each image was labeled by
365
      an average of 2.5 workers. The details of these datasets are described in Section F. We additionally
366   evaluated the proposed method with the CIFAR-10H dataset (Peterson et al., 2019) in Section I.9.
367
368   For Omniglot, we randomly used 764 classes for meta-training, 100 classes for validation, and 100
369
      classes for testing. For Miniimagenet, we randomly used 70 classes for meta-training, 10 classes for
      validation, and 20 classes for testing. For both datasets, we generated 50 tasks for both validation
370
      and testing data. For each task, we first randomly select four classes. Then, for each class of a task,
371
      we randomly used a few examples (one, three or, five examples per class) for support data and 10
372   examples for query data. For LabelMe, we used all eight classes for testing and created 10 tasks. We
373   used the same number of support and query data as for Omniglot and Miniimagenet. Since LabelMe
374   does not have many classes, we used Miniimagenet for meta-training and validation. Since LabelMe
375   consists of classes not included in Miniimagenet and has a small number of annotators per example
376   (2.5), this dataset is suitable for our problem setting. Thus, we used this dataset. Note that using
377   different datasets for source and target tasks is important since such a situation would be common
      in practice. For each dataset, we created five different meta-training/validation/testing splits.

      Under review as a conference paper at ICLR 2025


378
      Since Omniglot and Miniimagenet do not have data annotated by multiple annotators, we artifi-
379   cially create them. Such simulated annotation is commonly used in previous studies (Han et al.,
380   2021a; Tu et al., 2020; Tanno et al., 2019; Chu et al., 2021; Li et al., 2020; Cao et al., 2019)
381   since it enables us to investigate the methods in controlled environments. For simulated annota-
382   tions for target tasks, we considered three types of annotators: expert, hammer, and spammer. The
383   expert returns ground truth labels with probability q (0.8 < q ≤ 1) or otherwise chooses wrong
384   labels uniformly at random. The hammer also returns labels on the basis of the same mechanism
385   as the experts with a low probability for returning ground truth labels q (0.5 < q ≤ 0.8). The
386   spammer returns labels uniformly randomly from all classes. These types of annotators are typ-
387
      ical worker types in the real crowdsourcing service and thus are commonly used in experiments
      of previous studies (Kazai et al., 2011; Han et al., 2021a; Tu et al., 2020). For the support set in
388
      each target task, we randomly draw R annotators from a predefined annotator’s distribution p(A),
389
      where A takes expert (E), hammer (H), or spammer (S). Specifically, we considered four cases
390   (p(E), p(H), p(S)) = (0.1, 0.8, 0.1), (0.1, 0.7, 0.2), (0.1, 0.6, 0.3), (0.1, 0.5, 0.4). After selecting
391   the annotator type, we uniformly randomly select accuracy rate q in each range when the type is
392   expert or normal. All support data were annotated from all R annotators. For our method, pseudo-
393   annotators are generated at each meta-training iteration from only the single annotator’s distribution
394   (p(E), p(H), p(S)) = (0.1, 0.7, 0.2) 4 . The confusion matrix distribution for pseudo-annotators p(B)
395   was constructed from the (p(E), p(H), p(S)) = (0.1, 0.7, 0.2) and accuracy rate q.
396   For Omniglot and Miniimagenet, we evaluated mean test accuracy on target tasks over four differ-
397   ent target annotator distributions mentioned above with different numbers of support data per class
398   within {1, 3, 5} and annotators R within {3, 5, 7}. For LabelMe, we evaluated mean test accuracy
399   on target tasks with different numbers of support data per class within {1, 3, 5}. In Section I.4, we
400   evaluated the proposed method with other annotator’s types on target tasks such as pair-wise flippers
401   or class-wise spammers (Tanno et al., 2019; Khetan et al., 2018) when the same annotator distribu-
402   tion (p(E), p(H), p(S)) = (0.1, 0.7, 0.2) was used for meta-learning. Even when the annotator’s
403   type is different between target and source tasks, our method worked well.

412

413
414   Here, methods with the symbol ‘MV’ used majority voting for determining the label of each support
415   example. Methods with the symbol ‘DS’ used the DS model for estimating labels of support data
416   (Dawid & Skene, 1979). Although the DS model is simple, it has been reported to perform better
417
      than many existing methods (Zheng et al., 2017). Thus, we chose it as a comparison method. Al-
      though the original DS model does not consider the priors, we used them as in the proposed method
418
      because it improved the performance. In the DS methods, the initial value of responsibility λnk in
419                                             PR
      the EM algorithm was set to λnk = R1 r=1 δ(ynr , k), where R is the number of annotators in a task,
420
      as in the proposed method. When the number of the EM step J is one, the initial values (soft la-
421
      bels) are directly used for classifier learning. CL is a famous neural network-based learning method
422   from multiple annotators, which have annotator-specific layers. CNAL is a recently proposed neural
423   network-based learning method from multiple annotators, which models both an annotator-invariant
424   confusion matrix and annotator-specific confusion matrices. CL and CNAL learn the whole neural
425   networks with gradient decent methods. LRMV, LRDS, RFMV, RFDS, CL, and CNAL used only
426   target support data obtained from multiple annotators for learning classifiers. We included these
427   methods to investigate the effectiveness of using data in source tasks. If these methods outperform
428   the proposed method, there is no need to perform meta-learning in the first place. Thus, it is impor-
429   tant to include them in comparison for extensive experiments.

532
533
      We proposed a meta-learning method for learning classifiers with a limited number of labeled data
534   given from multiple annotators. Our experiments showed that our method outperformed various
535   existing methods on real-world datasets. Although the proposed method assumes input example-
536   independent confusion matrices for modeling annotators, it is interesting to extend our framework
537   to handle input example-dependent confusion matrices as in (Gao et al., 2022; Guo et al., 2023;
538   Li et al., 2024). In addition, it is also interesting to extend our framework to other crowdsourcing
539   problems such as active learning from multiple annotators. We also plan to use variational Bayesian
      inference, which is also differentiable, instead of the EM algorithm in our framework.

      Since the proposed method uses data from multiple tasks for meta-learning, biased data might be
543   included. This might result in biased results. We encourage researchers to develop methods to detect

751
752   There are some meta-learning methods for learning from a few noisy labeled data without consid-
753   ering multiple annotators (Liang et al., 2022; Mazumder et al., 2021; Chen et al., 2022; Killamsetty
754   et al., 2022). Unlike the proposed method, these methods do not use information of the annotator’s
755   id for training. When the annotators’ ids are available, learning methods that model multiple an-
      notators tend to perform better than methods that do not consider annotators (Tanno et al., 2019;


                                                       14
      Under review as a conference paper at ICLR 2025


756
      Gao et al., 2022). Therefore, this paper considers modeling multiple annotators. We experimentally
757   compared the proposed method with this approach in Section I.7.
758
759   Transfer learning is related to our work (Pan & Yang, 2009). Meta-learning is a type of fine-tuning
760
      method in transfer learning, particularly effective in very small data regimes (Hospedales et al.,
      2020). Since our aim is to learn from a small amount of noisy data, we focused on meta-learning.
761
      Although the standard fine-tuning methods pre-train models with clean labeled source data, we
762
      showed that it is insufficient to our setting. The proposed method performed well with pseudo-
763   annotation during the meta-learning. Domain adaptation is a well-known transfer learning approach
764   (Pan & Yang, 2009; Ganin & Lempitsky, 2015; Long et al., 2018). Since it simultaneously uses
765   source and target data for training, it requires time-consuming training as new target tasks appear.
766   In contrast, meta-learning methods do not because the meta-learned model with source data can
767   perform rapid and efficient adaptation with only target data.

881   D    I NTUITIVE E XPLANATION OF H OW EM A LGORITHMS C AN C OPE WITH
882        N OISY L ABELS
883
884   This section explains how our EM algorithm described in Section 3.2 contributes to the robustness.
885
886   Suppose we know the ground truth labels of data. In that case, we can easily estimate annotators’
887
      confusion matrices (i.e., the probability that the r-th annotator returns label l when the ground truth
                   r
      label is k; αlk ) using the ground truth and annotators’ labels. Conversely, if we know true annota-
888
      tors’ confusion matrices, we can estimate the ground truth labels from the confusion matrices and
889
      annotators’ labels. That is, ground truth labels and confusion matrices are interdependent. Thus,
890   if we initialize the labels of data, we can alternately estimate the labels and confusion matrices.
891   Here, we can make the label estimation more robust by considering multiple annotators’ confusion
892   matrices. If the initialization is not far from ground truth labels, we can expect to perform accurate
893   estimation.
894
      The EM algorithm is a systematical way to perform this procedure from a given probabilistic model.
895
      Specifically, in our setting, it iteratively estimates the label probabilities of the data (λnk ) using the
896   current estimate of annotators’ confusion matrices (αlk  r
                                                                  ) and classifier parameters (µk and πk ) in Eq.
897   (6), then uses the estimated labels to further refine the estimation of the annotator confusion matrices
898   and classifier parameters in Eq. (7). In our experiments, simple and commonly used initialization
                 PR
899   λnk = R1 r=1 δ(ynr , k), where R is the number of annotators that labeled example, worked well
900   as described in Section 3.3. This is the intuitive reason why the EM algorithm can obtain accurate
901   parameters from multiple noisy annotators.
902
      However, this is still a challenging estimation problem since no true labels exist. The proposed
903   method meta-learns example embeddings that make this problem easier to solve by explicitly maxi-
904   mizing the performance of the estimated classifier in source tasks.

907   E    D ISCUSSION OF THE SITUATION WHERE THERE ARE MANY SPARSE
908        ANNOTATORS
909
910   In some real-world applications, there are many annotators and each of them only labels a few
911   support data. Here, we discuss this situation in two cases.
912   When each annotator labels ’the same’ support data in a target task, accurate labels of the data
913   can be estimated since each instance has many annotations. Thus, in such cases, the proposed
914   method and conventional meta-learning methods that simply use support data with the estimated
915   labels (e.g., PrMV, PrDS, MaMV, and MaDS in our experiments) would perform well. In contrast,
916   since the number of support data is small, ordinary methods for learning from multiple annotators
917   (crowdsourcing methods) that do not use source data cannot learn accurate classifiers even if they
      can estimate accurate labels of support data.


                                                           17
      Under review as a conference paper at ICLR 2025


918
      When each annotator labels (generally) ’different’ support data in a target task, the number of whole
919   support data becomes large since there are many annotators. Since we focus on a small data regime,
920   this situation is out of the scope of our work. Ordinary crowdsourcing methods such as (Zhang et al.,
921   2024) are more appropriate for these situations.

1068   In the prior distributions of our model in Eq. (3), (b, c, τ ) are treated as hyperparameters. As for
1069   c and τ , we set c = 1 and τ = 1 for all experiments, which performed well. By tuning these
1070   parameters, the performance of the proposed method might be further improved. As for b, we
1071   evaluated the proposed method by changing b. Table 5 shows the average test accuracy on each
1072   dataset. We found that large b slightly performed well. This is because b controls the class prior of
1073   the support set. Specifically, as b increases, the class priors π = (πk )Kk=1 become uniform. Since
1074   the true class prior of the support set was uniform in our experiments, which is a standard setting in

       2018; Iwata & Kumagai, 2020), large b worked well. We note that even small b = 1 outperformed
       other comparison methods (see Tables 1 and 2 in the main paper).

1276   J     L IMITATIONS
1277
1278   The proposed method assumes that source tasks have clean labeled data as in almost all existing
1279   meta-learning methods (Snell et al., 2017; Finn et al., 2017; Garnelo et al., 2018; Rajeswaran et al.,
1280   2019; Bertinetto et al., 2018; Han et al., 2021a; Liang et al., 2022). Although this assumption might
1281   be restrictive in some cases, there are many applications where the proposed method can be applied
1282
       as described in Section 1. When each source task has enough noisy labeled data, we can accurately
       estimate their ground truth labels by applying existing methods for learning from noisy labels (Han
1283
       et al., 2018; 2020; Zheng et al., 2017). Therefore, in this case, we can use the source tasks with their
1284
       estimated labels for the proposed method.
1285
1286   Although the proposed method worked well even when different datasets (e.g., Miniimagenet and
1287   LabelMe) or noise (annotator) models are used for source and target tasks in our experiments, when
1288
       there is a significant discrepancy between the target and source tasks, meta-learning methods, includ-
       ing the proposed method, may not work well. This is a common challenge for existing meta-learning
1289
       methods. To deal with this problem, using task-augmentation methods such as (Yao et al., 2021) in
1290
       our framework would be one of the promising research directions.
