009                                               A BSTRACT
010
011            Large Language Models (LLMs) often generate incorrect or outdated infor-
012            mation, especially in low-resource settings or when dealing with private data.
013            To address this, Retrieval-Augmented Generation (RAG) uses external knowl-
014            edge bases (KBs), but these can also suffer from inaccuracies. We introduce
015            MAC-CAFE, a novel Multi-actor, Centralized Critic Architecture for Feedback-
016            driven Editing approach that iteratively refines the KB based on expert feedback
017            using a multi-actor, centralized critic reinforcement learning framework. Each
018
               document is assigned to an actor, modeled as a ReACT agent, which performs
               structured edits based on document-specific targeted instructions from a central-
019
               ized critic. Experimental results show that MAC-CAFE significantly improves
020
               KB quality and RAG system performance, enhancing accuracy by up to 8% over
021            baselines.

024   1   I NTRODUCTION
025
026   Large Language Models (LLMs) often produce incorrect or outdated information, particularly in
027   low-resource settings or when handling private data. Even if the information provided is accurate,
028   LLMs can generate hallucinated or imaginary content alongside it (Maynez et al., 2020; Zhou et al.,
029   2021). A promising solution to address these issues is the integration of retrieval components that
030   extract relevant information from external knowledge sources, known as Retrieval-Augmented Gen-
031   eration (RAG) (Chen et al., 2017; Khandelwal et al., 2020; Guu et al., 2020; Izacard et al., 2022;
032   Shi et al., 2023). For clarity, we will refer to these external knowledge sources as Knowledge Bases
033
      (KBs). However, KBs themselves can suffer from inaccuracies, incompleteness, or outdated con-
      tent. To address these challenges, there is growing interest in Knowledge Editing (KE) techniques
034
      to enhance LLMs with up-to-date and accurate knowledge.
035
036   Advancements in KE have focused on updating the model’s parameters (De Cao et al., 2021a; Meng
037   et al., 2022; 2023), adding new parameters to model (Huang et al., 2023; Yu et al., 2024), and holding
038   additional memory (Madaan et al., 2022; Wang et al., 2024a;b). Contrary to approaches that either
039
      update model parameters or add new parameters that require white-box access to LLMs, memory-
      based approaches can work with black-box access to LLMs. In similar line of thought, recently,
040
      KE approaches have also focused on refining the KBs themselves (Li et al., 2024). For example,
041
      the method proposed in Li et al. (2024) continuously updates KBs with new information, such as
042   the current identity of the British Prime Minister. This approach demonstrates that directly editing
043   the KB is more effective than simply adding new documents, which may coexist with outdated or
044   inaccurate ones. Removing older documents is often not feasible, as only certain sections may be
045   incorrect, while other parts could still provide valuable information for different queries. However, in
046   applications like chatbots or code generation using API documentation, where updated information
047   might not be readily available in document form, expert intervention can be crucial (Ramjee et al.,
048   2024; Afzal et al., 2024). In such cases, expert feedback can be used to directly update the KB with
049   accurate information when the LLM produces erroneous results.
050   To leverage expert or oracle feedback, we propose MAC-CAFE, a Multi-actor, Centralized Critic
051   Architecture for Feedback-driven Editing technique. Our contributions are as follows:
052
053         1. Introduction of Feedback-Driven KB Editing: We present MAC-CAFE, a novel frame-
               work that refines the KB using structured edits based on expert feedback. This approach

               allows for direct, document-level updates without requiring access to LLM parameters,
055            making it applicable to both white-box and black-box LLMs.

081   2   R ELATED WORK
082
083
      The MAC-CAFE framework addresses a key limitation of current RAG systems: the inability to
084   dynamically update Knowledge Bases (KBs) without retraining or altering model parameters. Our
085   work draws from research in Retrieval-Augmented Generation (RAG), Continual Learning, Model
086   Editing, and feedback-driven prompt optimization, incorporating insights from Multi-Agent Rein-
087   forcement Learning (MARL) to propose an effective solution for KB editing.
088
      Retrieval Augmented Generation (RAG): RAG systems enhance LMs by retrieving relevant
089
      knowledge from a KB based on the input query and appending it to the context, thereby addressing
090   the limitations of standalone LMs that lack sufficient context and produce inaccurate answers (Chen
091   et al., 2017; Khandelwal et al., 2020; Guu et al., 2020; Izacard et al., 2022; Shi et al., 2023). These
092   systems dynamically construct contexts from unstructured KBs without modifying the LM’s inter-
093   nal parameters. MAC-CAFE further enhances RAG systems by refining the KB itself based on
094   feedback, ensuring more accurate and up-to-date information.
095
      Continual Learning: Continual Learning (CL) methods address the challenge of updating LMs in
096   non-stationary environments by ensuring that new information is learned without forgetting previ-
097   ously acquired knowledge (Jin et al., 2022; Xu et al., 2023; Padmanabhan et al., 2023; Akyürek
098   et al., 2024). These methods are often computationally intensive and require large-scale retraining,
099   making them less suitable for scenarios requiring frequent updates or minimal computational re-
100   sources. MAC-CAFE, by contrast, leverages expert feedback to perform direct edits to the KB,
101   avoiding the need for extensive retraining.
102   Knowledge Editing: Knowledge Editing approaches fall into two categories: Model Editing, which
103   modifies the LM parameters directly, and Input Editing, which updates the knowledge supplied to
104   the model. While Model Editing efficiently alters specific facts using specialized secondary models
105   or altering parameters (De Cao et al., 2021b; Meng et al., 2023), it struggles to ensure consistent
106   updates across contexts (Onoe et al., 2023; Hua et al., 2024). In contrast, Input Editing modifies the
107   KB itself, enabling updates to be reflected in outputs without changing model parameters (Madaan
      et al., 2022; Wang et al., 2024a;b; Li et al., 2024). MAC-CAFE builds on input editing techniques

                                                        2
      Under review as a conference paper at ICLR 2025


108
      by leveraging expert feedback to refine the KB systematically, ensuring more accurate and consistent
109   responses.
110
111   Prompt Optimization: With the advent of LMs, some recent works approximate gradients in text-
112
      based environments using LMs (Pryzant et al., 2023; Wang et al., 2023; Juneja et al., 2024; Gupta
      et al., 2024) for optimizing task prompts. MAC-CAFE is inspired by these approaches and gen-
113
      erates textual reflections, similar to MetaReflection (Gupta et al., 2024) and Shinn et al. (2023), as
114
      proxies for gradients. It provides actionable guidance for document updates without the need for
115   differentiable models. Additionally, MAC-CAFE adopts clustering strategies for feedback aggre-
116   gation from works like UniPrompt (Juneja et al., 2024)- ensuring that actors receive coherent and
117   non-redundant instructions.
118
      Multi-Agent Reinforcement Learning (MARL): Multi-agent reinforcement learning (MARL) has
119
      been applied to various domains, with early research focusing on tabular methods (Busoniu et al.,
120
      2008; Canese et al., 2021; Gronauer & Diepold, 2022) and later expanding to deep learning tech-
121   niques for high-dimensional inputs (Tampuu et al., 2017; Leibo et al., 2017). Studies have explored
122   independent Q-learning (Tan, 1993), agent communication (Foerster et al., 2016; Das et al., 2017),
123   and centralized training with decentralized execution (Gupta et al., 2017). However, most of these
124   approaches do not address the critical challenge of multi-agent credit assignment. Actor-critic meth-
125   ods have been introduced to overcome this limitation by employing centralized critics with decen-
126   tralized actors (Foerster et al., 2018; Iqbal & Sha, 2019; Wang et al., 2021; Chen et al., 2023).
127   MAC-CAFE extends such actor-critic framework to operate directly on textual content, using
128   the centralized critic to decompose feedback into actionable textual gradients for each document-
129
      specific actor.

131   an overview of MAC-CAFE.

146   builtin-array.md. The system retrieves relevant documents based on the given task (e.g.,
147   counting non-inversions in an array) and generates a program, which is evaluated by the compiler,
148   resulting in feedback (e.g., compile errors).
149    2 Centralized Feedback Analysis: We analyze compile errors to generate reflections that explain
150   why the errors occurred. For instance, if the apply method in the Array class is partial and may
151   raise an error, the reflection suggests adding a ? to handle potential failures. These reflections are
152   matched to the documents they pertain to, refining the understanding of errors.
153
       3 Distributing Gradients: Reflections are generalized into gradients, which summarize modifica-
154
      tions needed for each document. For example, the theme might be the partial nature of functions like
155   apply and update, which need better error handling in the documentation.
156
157   4 Generating Edit Actions: Gradients are converted into structured edit actions, such as adding or
158
      modifying content in specific sections of the documents.
159    5 Re-evaluation and MCTS Search: After edits are applied, the KB is re-evaluated, generating
160   new feedback and a reward score. This score guides a Monte Carlo Tree Search (MCTS) to explore
161   different states of the KB, iterating through steps 1 - 4 to progressively refine the KB and improve
      the system’s overall performance.

      4        M ETHODOLOGY
190
191
      We will start by describing a typical Retrieval-Augmented Generation (RAG) system over unstruc-
192
      tured Knowledge Bases.
193
194   Errors in such systems can arise from multiple components: 1) the LLM B might fail to reason
195   correctly over the provided information, 2) the retriever R might not select the right set of relevant
196
      documents from K, or 3) the knowledge base K itself might contain incorrect or incomplete in-
      formation. We assume an expert is monitoring the system, identifying when answers are incorrect,
197
      determining which component is at fault, and providing feedback on why the answer is incorrect
198
      and what the correct answer must be.
199
200   This work focuses on scenarios where incorrect answers result from issues in the Knowledge Base
201   (K). Our goal is to improve K by addressing mistakes in K and filling in missing information based
202
      on expert feedback, thus enhancing the RAG system’s performance on future queries.
203

      4.2   K NOWLEDGE BASE E DITING AS S TATE S EARCH
217
218
      In our problem setting, the Knowledge Base (K) is defined as a collection of documents K =
219   {Di }ni=1 . We assume each document consists of a number of chunks of text and can be represented
220   as Di = [cij ]. The state s ∈ S of the system is represented by the current configuration of the KB,
221   i.e., the content of all documents in K.
222
      Given a query qi and a set of retrieved documents Γ(qi , K), the LLM B generates an answer oi .
223
      When errors arise due to incomplete or incorrect information in the retrieved documents, our goal
224
      is to identify the optimal configuration of K that improves the accuracy of the system’s responses.
225   Thus, we define our state search problem as finding the best state s∗ of the KB.
226
227   State Space: The state space S encompasses all possible configurations of the KB. Each state s cor-
228
      responds to a particular set of document contents, represented as: s = {Di }ni=1 , where Di denotes
      the content of document i and n is the number of documents in K. The state s captures the overall
229
      structure and content of the KB at any given point. We set s0 = K.
230
231   State Transition Function: The state transition function T (s, u) defines how the KB changes in
232   response to the action u taken by the agent. Each action contains modifications to one or more
233   documents within the KB, resulting in a new KB configuration. The state transition is formalized as:
234
      s′ = T (s, u), where s′ is the new state of the KB after applying u.
235   Action Space: The action space A consists of list of diffs di corresponding to each document Di .

269   the LM would need to output the entire edited KB, which is challenging due to the inherent difficulty

375   5   E VALUATING K NOWLEDGE BASE E DITING Q UALITY
376
377   A Knowledge Base should be complete with respect to a task - it should contain all the information
      necessary to assist the RAG system to solve the task at hand. Given the open-ended nature of tasks


                                                                7
      Under review as a conference paper at ICLR 2025


378
      that typical RAG agents are designed for, it is hard to quantify a closed-form metric of completeness.
379   That said, an ideal Knowledge Base editing system should at least be able to incorporate as much
380   external feedback as possible.
381
382
      Further, It will be extremely undesirable for any Knowledge Base to only help the RAG system for
      a small subset of tasks. Given the tendencies for data-driven techniques to over-fit on the train-set
383
      distribution, it is important that knowledge base edits are generalizable to unseen examples.
384
385   Lastly, given the semantic and textual nature of the Knowledge Base, it is important that the doc-
386   uments in the Knowledge base are coherent and consistent throughout. This not only makes the
387   document interpretable for human consumption, it also help reduce in-context noise during LLM
388
      inteference, which has been shown to affect LLM performance (Liu et al., 2024).

393   6.1     BASELINE
394
395   While there has been a rich body of works in the area of knowledge editing and prompt optimization,
396   to the best of our knowledge, MAC-CAFE is the first work targeting the feedback-driven textual
397   Knowledge Base Editing problem. Therefore, to perform a holistic evaluation of MAC-CAFE we
398   implement - P ROMPTAGENT-E, an extension of P ROMPTAGENT Wang et al. (2023) for the KB
399   editing task. P ROMPTAGENT formulates prompt optimization as a strategic planning problem using
400
      Monte Carlo Tree Search (MCTS). At a high-level our baseline approach, P ROMPTAGENT -E cre-
      ates separate P ROMPTAGENT -style agents to optimize specific document in the KB. To minimize
401
      spurious edits in the Knowledge Base, we restrict P ROMPTAGENT -E to only optimize documents
402
      that were part of the retrievals for more than 2 training sample. After identifying the best nodes
403   for each of the document-wise runs, we put them back in the knowledge base to generate the new
404   version of the KB. In contrast to MAC-CAFE, P ROMPTAGENT -E can be seen as a collection of
405   document-wise Independent Actor-Critic models (Foerster et al., 2017). We present in-depth com-
406   parisons between P ROMPTAGENT -E and MAC-CAFE in Section 7

449   6.3     E VALUATION M ETRICS
450
451   In section 5 we discussed the desirable properties of a Knowledge Base edit. We leverage these
452
      properties to design 3 metrics for the KB Editing problem as follows:
453   Completeness: We use the train set accuracy to estimate the degree of expert feedback incorporated
454   in the learnt Knowledge Base.
455
      Generalization: To estimate the degree of generalization of our Knowledge Base edits, we use the
456   held out test set accuracy.
457
458   Coherence: To quantify the degree of coherence of the KB, we first calculate a document-wise
459
      coherence score using G-Eval (Liu et al., 2023) with GPT4-1106- PREVIEW as the judge model.
      The G-eval prompt assigns a 1-5 score to the diff of changes with respect to the original document,
460
      checking for thematic similarity of the diff. We pool all the edited documents for a KB edit and
461
      average there respective coherence score to define the KB coherence metric.
462
463
464   6.4     S YSTEM C ONFIGURATIONS
465
      MCTS parameters: We use the Upper Confidence bounds applied to Trees (UCT) algorithm for
466
      selecting expansion nodes, enabling effective exploration and exploitation of the KB state space. For
467
      our experiments, we set a maximum search depth of 3, an expansion width of 3, and a maximum of 5
468   iterations. The UCT exploration constant is set to 2.5. These parameters were chosen to balance the
469   computational cost and the need for adequate exploration. A depth of 3 ensures that the search can
470   explore sufficient variations in the KB states without unnecessary expansion, while an expansion
471   width of 3 allows a moderate number of candidate states to be evaluated at each step. Similarly,
472   5 iterations provide enough opportunity to refine the state search, and the UCT constant of 2.5
473   encourages sufficient exploration in early stages while converging towards high-reward states in
474   later stages. For unstructured data, the documents are chunked after every 50 lines and then edit the
475   chunks.
476   RAG System: For the purpose of our evaluations, we setup a generic RAG system which uses an
477   embedding similarity for semantic retrieval. Additionally, in lines with prior works like (Zhang
478   et al., 2023) for coding related tasks, we use an iterative retrieval setup wherein we first generate a
479   code using naive retrieval and then query the database again with both the question and generated
480   code to improve the quality of retrieval before generating the final result.
481   LLM configs: We use O PENAI-T EXT-E MBEDDING -3-L ARGE as the embedding model with di-
482   mensions size of 3072 and use cosine similarity as a metric of embedding match for ranking. To
483   account for the 8191 max input limit, we create document chunks of at most 7500 tokens. For the
484   reasoning model, we use GPT4-1106- PREVIEW, with a temperature of 0. Since LLMs are known
485   to perform poorly with longer context input (Liu et al., 2024), we restrict the max token budget for
      retrievals at 18000 tokens and remove any lower ranked retrieval to fit this token budget.

      8     C ONCLUSION
531
532
      We introduced MAC-CAFE, a novel framework for refining Knowledge Bases (KBs) in
533
      Retrieval-Augmented Generation (RAG) systems using a multi-actor, centralized critic architec-
534   ture. MAC-CAFE enables efficient KB updates without retraining or altering model parameters by
535   leveraging feedback-driven structured edits and textual gradients.
536
537
      Our approach achieved superior performance in preserving knowledge base (KB) coherence, consis-
      tency, and completeness, resulting in enhanced RAG system responses. Nonetheless, there remains
538
      considerable potential for further advancements. Future work will focus on refining these three met-
539
      rics to elevate system performance even further.
