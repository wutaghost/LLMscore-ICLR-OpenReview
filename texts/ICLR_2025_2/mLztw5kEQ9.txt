009                                                A BSTRACT
010
011
               Source-Free Domain Adaptation (SFDA) aims to adapt a model trained in an
012            inaccessible source domain S to a different, unlabelled target domain T . The crux
013            is to remove the source bias, which leads to catastrophic failure on T samples
014            that are dissimilar to S. Unfortunately, without any prior knowledge about the
015            bias, the current SFDA setting has no mechanism to circumvent it. We introduce a
016            practical setting to address this gap—Debiased SFDA, where the model receives
017            additional supervision from a pre-trained, frozen reference model. This setting
018            stays in line with the essence of SFDA, which accommodates proprietary source-
019            domain training, while also offering prior knowledge that is unaffected by source-
020
               domain training to facilitate debiasing. Under this setting, we propose 1) a simple
               contrastive objective that debiases the source-trained model with the guidance from
021
               the reference model. 2) a diagnostic metric that evaluates the degree to which
022
               an adapted model is biased towards S. Our objective can be easily plugged into
023            different baselines for debiasing, and through extensive evaluations, we demonstrate
024            that it engenders consistent improvements across standard benchmarks. Code is
025            supplied under supplementary material.

      1    I NTRODUCTION
030
031   Domain adaptation (DA) aims to adapt a model trained on a labeled source domain S (e.g., computer-
032   synthesized images) to generalize to a novel, unlabeled target domain T (e.g., real-world images),
033
      where S and T exhibit significant domain gap. In particular, recent studies look at source-free
      domain adaptation (SFDA), a more practical setting that provides a pre-trained source model in
034
      lieu of data, as rising concerns regarding data privacy and intellectual property render the source
035
      data unavailable (Liang et al., 2020; Chen et al., 2023). However, the source model is inevitably
036   biased to spurious correlations that only hold in S (Zhang et al., 2022). The crux of SFDA is thus to
037   debias the source model, such that transferable, pertinent knowledge (e.g., object shape) is retained as
038   discriminative cues, and source-specific traits (e.g., object orientation) irrelevant to the target domain
039   are discarded. One common approach treats local clusters in T (based on features extracted by the
040   source model) as transferable knowledge (Yang et al., 2021a; 2022; Hwang et al., 2024). Following
041   the intuition that the source pre-trained model has learned good class representation, this enforces
042   prediction consistency between each T sample and its k nearest neighbors.
043   Unfortunately, debiasing a biased model without additional prior is paradoxical (Caron et al., 2020;
044   Huang et al., 2017). Consider the aforementioned approach. The source pre-trained model may
045   associate similar samples feature-wise based on orientation (i.e., spurious) as opposed to shape (i.e.,
046   transferable). The use of such spurious correlations may cause samples of different classes to have
047   the same prediction, as they are erroneously considered among the nearest neighbors.
048   To show this, we design a diagnostic experiment in Figure 1, where we select test samples with
049   confident correct predictions by the source model as the easy split, and those with confident wrong
050   predictions as the hard split. We exhaust the experiment design space of SFDA by investigating
051   various changes and observe that there is a consistent trade-off between model performances on the
052   easy and hard splits. Our experiments include changing the algorithm in Figure 1(a), changing the
053   number of epochs used in source model training in Figure 1(b), and changing the number of epochs
      during adaptation in Figure 1(c). We provide more details regarding the experiment in the Appendix.

073
      This behavior bears strong similarity to the precision-recall trade-off in machine learning: a model
074
      either predicts too liberally and over-identifies correct samples, or too conservatively and risks missing
075   positive samples. This trade-off can be attributed to the lack of knowledge on spurious features (Yue
076   et al., 2021), as reliance on them results in increased false positives and/or false negatives. Similarly,
077   without the same type of prior during target adaptation, a model either excels with source-like samples
078   in T by exploiting source-specific (i.e., spurious) knowledge, or prioritize performance on difficult
079   samples by over-adjusting to T and risk losing transferable knowledge. This analysis outlines a
080   fundamental flaw of conventional SFDA: any adapted model is simply a point on the trade-off curve
081   and performs unreliably on classifying samples of arbitrary difficulty. It highlights the dangers of
082   deploying such models that abuse spurious correlations into the wild, especially when it involves use
083
      cases that risk perpetuating harmful prejudices (Ji et al., 2020; Dimanov et al., 2020).
084   To address this deficiency, we propose a practical setting—Debiased SFDA—where a frozen refer-
085   ence model fr , pre-trained on a diverse dataset (e.g., ImageNet classification), provides additional
086   supervision during the adaptation of the source model fs . We highlight our rationales:
087
         • Practical. Open-source pre-trained models are widely available. In real-world SFDA scenarios,
088
           such models can be leveraged to improve the performance of a proprietary fs .
089
090      • Debiased. fr is pre-trained on large, diverse datasets, granting it strong generalization abilities.
091
           They offer useful debiasing cues by identifying predictions conflicting fs (Yue et al., 2023).
           For instance, an image of a car and a truck photographed in similar orientations may be deemed
092
           similar by the biased fs but dissimilar by fr , suggesting that orientation is a spurious attribute.
093
094
         • Frozen. fr does not receive any (spurious) supervision from fs in order to preserve the general
           knowledge. This is a key distinction from the ensemble approach implemented in Zhang et al.
095
           (2022), where fr is updated by the pseudo-labels generated by fs (Figure 2(c)).
096
097   Under the proposed setting, we introduce a simple contrastive objective that leverages fr to debias fs .
098   We term our approach Debiased Contrastive learning (DeCo), which first clusters samples in T using
099   the general knowledge of fr , and then adapts the classifier of fs to align with the general knowledge,
100   such that samples with similar predictions should come from the same cluster.
101
      For the first step, we employ an unsupervised clustering algorithm used in generalized category
102   discovery (Vaze et al., 2022; Wen et al., 2023) and leverage the following concepts: (a) the number
103   of T classes, which is known in SFDA, and (b) classes in T can be considered novel to fr , as it is
104   pre-trained on a different dataset. Next, we take inspiration from the InfoNCE loss (van den Oord
105   et al., 2019) and implement it such that for each training sample, positive keys in a mini-batch are
106   defined as those from the same cluster, and negative keys as those from a different cluster. We also
107   further propose a diagnostic metric, Hardness Accuracy (HAcc), to quantitatively measure an adapted
      model’s accuracy in relation to initial source bias (i.e., after debiasing). Our contributions:

119   2    R ELATED WORK
120
121   Domain adaptation. DA aims to improve performance on unlabeled target domains by leveraging
122   knowledge learned from a labeled source domain; the main challenge at hand is the reduction of
123
      domain shift. Conventional DA methods assume that there is access to both the source and target
      domain data during the adaptation process. Unsupervised domain adaptation (UDA) methods mainly
124
      fall under two categories: domain alignment methods, which seek align the source and target domain
125
      distributions (Cui et al., 2020; Liu et al., 2021a), and self-training methods, which utilize adversarial
126   learning to learn domain-invariant feature representations (Long et al., 2018; Tzeng et al., 2017; Shu
127   et al., 2018).
128
129   Source-free domain adaptation. In SFDA, the pre-trained source model is adapted to an unlabeled
130   target dataset. There are two main families of approaches to SFDA: model-based methods, which
131   consists of fine-tuning modular models for domain adaptation, and data-based methods, which follows
132   the intuition of mimicking or exploring pertinent source domain data properties (Yu et al., 2023).
133   We focus on the latter and specifically study neighborhood clustering for SFDA. SHOT (Liang
134   et al., 2020) freezes the source classifier and clusters target features using information maximization
135   and self-supervised pseudo-labeling. G-SFDA (Yang et al., 2021b), NRC (Yang et al., 2021a),
136
      and AaD (Yang et al., 2022) utilize local neighborhood features in the feature space to enforce
      consistency in predictions. Following this line of work, SF(DA)2 (Hwang et al., 2024) leverages
137
      data augmentations by building augmentation graphs and identifying class partitions with spectral
138
      neighborhood clustering. Zhang et al. (2022) adopts an ensembling approach to improve pseudo-label
139   reliability by integrating a pre-trained network during target adaptation.
140
141   Spurious correlation in domain adaptation. A model that is trained on a source domain is likely
142   to learn spurious correlation in the data and subsequently perform poorly on target domains. To
143   reduce spurious correlation, ATDOC (Liang et al., 2021) and CST (Liu et al., 2021a) propose to
144   generate target-oriented pseudo-labels using an auxiliary classifier. Notably, these methods initialize
145   the aforementioned classifier with source-trained classifiers, the latter of which brings both the
146   knowledge and spurious correlation of the source domain to the former. ICON (Yue et al., 2023)
147   eliminates this issue by employing a randomly-initialized cluster head that solely learns from the
148   target distribution.
149
150   Contrastive SFDA. Contrastive learning has become an increasingly popular approach in SFDA
151
      (VS et al., 2023). VS et al. (2023) uses an instance relation graph network that exploits objects
      relations to guide contrastive loss. SiLAN (Wang et al., 2024) contrasts augmented views in the latent
152
      space. UC-SFDA (Chen et al., 2023) uses evidence theory to find high-confidence pseudo-labels
153
      for contrasting. Our method extends the work of ICON (Yue et al., 2023), which is designed for
154   debiasing in the UDA setting and requires access to source data during training. We overcome this
155   requirement by contrasting feature similarity of in-domain samples between the source-trained model
156   and a large-scale pre-trained model.
157
158   Bias metrics. Studies have established numerous methods to quantify the unfairness of machine
159   learning models (Brodersen et al., 2010; Besse et al., 2020; Mehrabi et al., 2021). We propose HAcc
160   as we are concerned with measuring performance changes based on the reduction of source-specific
161   biases as opposed to the correction of statistical biases in general. This provides a more tapered view
      for assessment.

176   3.1     S OURCE - FREE DOMAIN ADAPTATION (SFDA)
177
      The goal is to adapt a model trained in a source domain S (i.e., source model) to a target domain T ,
178
      using only an unlabelled dataset D = {xi }N   i=1 from T , where xi is the i-th sample (e.g., an image).
179   Its paradigm is illustrated in Figure 2(a). We focus on the closed-set setting in this paper, under which
180   the target domain shares the same C classes as the source domain (C is known). The source model
181   fs consists of a feature extractor that maps a sample x to the feature space, and a classifier that maps
182   the feature to its softmax-normalized probability of belonging to each class. The objective of existing
183   methods comprise of two parts: an adaptation loss Ladp to adjust fs to the target distribution, and a
184   regularization loss Lreg to prevent model degeneration. We detail the two losses next.
185   Existing approach. We take a representative approach AaD (Yang et al., 2022) as an example. It
186   fine-tunes fs by minimizing the following loss for each x ∈ D:
187
                                                               Ladp                                    Lreg
188                                     z X }|           {    zX   }|         {
189                                               T     ′              T     ′
                             Laad (x) = −   fs (x) fs (x ) +λ    fs (x) fs (x ),                                                         (1)
190                                            x′ ∈K(x)                                x′ ∈Q(x)
191
192   where K(x) is the set of k nearest neighbors measured by cosine similarity in the feature space, Q(x)
193
      is given by D \ x and λ is the regularization weight. In particular, AaD assumes that fs , with source
      training, has learned transferable feature, i.e., x and its nearest neighbors K(x) should come from
194
      the same class, hence its adaption loss increases the dot product similarity between their predictions.
195
      However, this objective could lead to a collapsed solution, e.g., Ladp is also minimized by mapping
196   all samples in D to the same feature. Hence, to prevent this, Lreg encourages different samples (x
197   and any sample in Q(x)) to have different predictions. The two loss components can take different
198   forms in other SFDA methods: e.g., Ladp can be implemented as a cross-entropy loss calculated from
199   the prediction fs (x) and pseudo-label argmax fs (x) (Chu et al., 2022; Li et al., 2022), and Lreg can
200   be implemented by an entropy term to encourage the diversity of predictions over D (Liang et al.,
201   2020).
202   Unfortunately, we have shown in Figure 1 that the adapted fs is still prone to spurious correlations in
203   S, which cannot be resolved within the current design space of SFDA. This motivates us to propose a
204   more practical setting in Section 3.2.

206   3.2     D EBIASED SFDA
207
208   As shown in Figure 2, unlike (a) SFDA, where the adaptation uses only a single source model fs
209   trained on hidden source domain S, (b) debiased SFDA additionally leverages a reference model
210   fr , which satisfies the following points: 1) It is an open-source model, hence is generally available.
211   2) Its pre-training dataset should be diverse to learn general knowledge, while also related to the
212
      SFDA task to allow knowledge transfer. 3) During adaptation, fr should be frozen to prevent it
      from absorbing spurious correlation in S (from the source model) and T . In this work, we drop the
213
      pre-trained classifier in fr , and considers it as a mapping from the sample space to a feature space.
214
215   In particular, debiased SFDA is fundamentally different from the previous ensemble approach (Zhang
      et al., 2022) shown in Figure 2(c). In debiased SFDA, the adaptation of fs receives guidance from

      Based on the debiased SFDA setting, we propose Debiased Contrastive (DeCo) learning, which has
238   two steps: 1) cluster the samples in D based on the features extracted by the reference model fr ; 2)
239   then use a loss inspired by InfoNCE (van den Oord et al., 2019) to align the source model fs with the
240   clusters. Our overall pipeline is depicted in Figure 3. We detail each step below.
241
242   4.1          C LUSTERING
243
244   Cluster head. Our cluster head g consists of two parts, an MLP that maps the feature space of fr to
245   a projection feature space, and a linear head that classifies a projected feature into a prediction, i.e.,
246   pi = g(fr (xi )) where pi ∈ RC is the softmax-normalized probabilities of belonging to each of the
247   C clusters, and xi ∈ D is a sample. The cluster head g is randomly initialized during training.
248   Training of the cluster head. Inspired by SimGCD (Wen et al., 2023), which is originally proposed
249   for novel class discovery, we adapt the unsupervised objective of the algorithm as our cluster training
250   objective. Specifically, we implement an unsupervised representation loss and self-distillation loss.
251   The loss for training g per mini-batch B in D is given by:

295   5.1   S ETTINGS
296
297   Datasets. We evaluate our method against three benchmark datasets for image classification: Office-
298   Home (Venkateswara et al., 2017), VisDA-C 2017 (Peng et al., 2017), and DomainNet-126 (Peng
299
      et al., 2019). Office-Home consists of 4 domains (Art, Clipart, Product, Real) with 65 classes
      per domain and 15,500 images. VisDA evaluates synthesis-to-real object recognition across 12
300
      classes, containing 152,000 synthetic source images and 55,000 real target images. DomainNet is a
301
      large-scale benchmark with 6 domains and 345 classes. Following Chen et al. (2022), we select 4
302   domains (Clipart, Real, Sketch, Painting) containing 126 classes.
303
304   Implementation details. We conduct experiments under similar settings as described in previous
305   work for fair comparison (Liang et al., 2020; Yang et al., 2022; Hwang et al., 2024). We follow
306   Hwang et al. (2024) as closely as possible when evaluating against DomainNet. The architecture
307   consists of a feature backbone and the classification head, which is a fully-connected layer with
308   weight normalization. ResNet-50 is adopted as the backbone for Office-Home and DomainNet,
309   and ResNet-101 is used for VisDA. We adopt SGD with momentum 0.9 and train 40 epochs for
310   Office-Home, and 15 epochs for VisDA and DomainNet. Batch size is set to 64 for Office-Home
311   and VisDA, and 128 for DomainNet. We adhere to the standard practice of training the last two
312
      layers of the model at a learning rate 10 times of the backbone – the learning rate is set to 1e-3 for
      Office-Home, 1e-4 for VisDA, and 5e-5 for DomainNet. We set the learning rate to 0.1 for g across
313
      all benchmarks. For fairness, all reference models used in our experiments are pre-trained on the
314
      ImageNet-1K dataset to match the feature backbone in the source model.

      H-scores and HAcc. As there are currently no standardized measures to quantify model bias vis-à-
317
      vis source bias for SFDA, we propose Hardness Accuracy (HAcc), a novel metric that finds the mean-
318   class accuracy across K unequal (in size and class distribution) partitions of T test samples. Firstly,
319   we begin by defining the hardness score (i.e., H-score) of a sample xi as the degree of uncertainty to
320   which fs predicts xi as the ground-truth label yi ; it is represented as h(xi ) = 1 − ϕ(fs (xi ))[idxyi ],
321   where ϕ refers to the softmax function and idxyi the yi -th index. In other words, the higher the
322   H-score, the more uncertain fs evaluates xi as the correct label. To generate unequal splits, sample
323   ratio per partition is determined based on the H-score using the Power Law (Clauset et al., 2009).

      5.4     L IMITATIONS & T ECHNICAL C OMPUTE
517
518   We recognize the lack of and need for further study on the theoretical guarantee of debiasing SFDA.
519   Additionally, our experiments only include studies on two baselines with DeCo. It would be beneficial
520   to expand our investigation towards additional baselines, as well as debiasing for methods outside of
521   neighborhood clustering. We focus our evaluations heavily on the VisDA dataset, as we believe that
522   it is sufficiently challenging in comparison to Office-Home and DomainNet. More work on larger
523   benchmarks of higher complexity is required to further validate our claims.
524   All experiments are run on a single NVIDIA RTXA6000. The runtime varies depending on the size
525   of the dataset. For Office-Home, model training for each subtask usually takes less than two hours. It
526   took about 10 hours to train on VisDA. DomainNet is much larger than Office-Home; the smallest
527   subtask takes 5 hours and the largest can take up to 12 hours to train.

      6      C ONCLUSION
530
531
      In this paper, we study the deficiency of current Source-Free Domain Adaptation (SFDA) methods in
532
      addressing source bias. We present a practical setting called Debiased SFDA as a solution, where
533
      model adaptation can leverage a pre-trained, frozen reference model. Building on this setting, we
534   propose a simple approach, Debiased Contrastive learning (DeCo). The objective of DeCo is to
535   align the source model with the clusters identified by the reference model, such that its classification
536   boundary respects the cluster boundary. To evaluate its debiasing capabilities, we design a Hardness
537   Accuracy (HAcc) metric that focuses on a model’s performance on hard samples outside the source
538   domain distribution. Extensive evaluations suggest that our proposed DeCo improves baselines on
539   both accuracy and HAcc, especially with a strong reference model. As future work, we will study
      Debiased SFDA from a theoretical perspective and analyze its generalization bound.

741
      Larger and more complex models are likely to outperform smaller counterparts in classification
      tasks as they have an increased ability to capture visual nuances. Thus, using a large pre-trained
742
      foundation model as fr is quintessential in achieving strong results. However, we demonstrate that
743
      using smaller models with DeCo still conducts debiasing to provide performance gains, especially on
744   biased datasets. Table 9 reports the results of our experiments.
745
746   HAcc shows the benefits of DeCo more clearly – DeCo-RN50 increases HAcc in split 11 by 4.4%
747
      and the overall average by 1.5%. Similarly, DeCo-SwinT leads to a 3.1% improvement in average
      HAcc. This indicates that smaller, less complex models still provide useful clues for mitigating source
748
      bias. Additionally, we observed that Swin variants of DeCo outperform those of ResNet. Swin-Tiny
749
      (28.3m parameters) is comparative to ResNet-50 (25.6m) in size and smaller than the ResNet-101
750   (44.5m), which is the standard visual backbone used in VisDA evaluation. Notably, DeCo-SwinT
751   achieves stronger performance than DeCo-RN101 on the standard benchmark. This improvement can
752   be attributed to the differences in architecture — while ResNet models are efficient and effective at
753   capturing local features and spatial hierarchies, Swin transformers excel at modeling global context
754   and long-range dependencies. The representation power of Swin transformers may indicate less
755   likelihood to rely on spurious correlations to make predictions, thereby acting as better references for
      the adapting model.

      A.2.1    DATA AUGMENTATIONS
772
      For data augmentations, training samples are resized to 256 × 256, randomly cropped to 224, then
773   random horizontal flip is applied. We implement a strong augmented view of x in addition to the
774   standard augmentation, which we consider the weakly augmented view. Strong augmentations include
775   the same transformations as the original, but further includes a random application of color jitter, as
776   well as RandAugment (Cubuk et al., 2019).
777
778
      A.2.2    U NIFORM D ISTRIBUTION IN C LUSTERS
779
780   Lg avoids the possibility of uniform distribution (i.e., C1 , C1 , ..., C1 ) by using sharpened target pre-
781   dictions and mean-entropy maximization, H(p̄). We utilize an exponential moving average-updated
782   teacher model to sharpen soft cluster pseudo-labels for loss calculation. Sharper target predictions
783   guide the model to produce confident low entropy anchor predictions; they also make them unequal
784   to the uniform distribution. Additionally, H(p̄) seeks to minimize the negative entropy of the average
785   prediction across all anchor views in a given mini-batch. In using sharpened target predictions, the
786
      average prediction used in entropy regularization is also not equal to the uniform distribution. Further
      information on the proof is provided in the appendix of Assran et al. (2022).
787
788
789   A.3     D IAGNOSTIC E XPERIMENT
790
791   We create easy and hard splits using the VisDA dataset, where Seasy contains samples with features
792   that are most similar to the source domain, and Shard contains those most dissimilar. Note that the
793
      sets are identical in size and do not overlap. Similarity is determined by the cosine similarity between
      the mean class representation of S and a given sample in T , as evaluated by the source pre-trained
794
      model fs . This means that sample similarity is measured in relation to source bias. Then, we run
795
      inference with models trained on the full dataset on these splits. We experiment with varying the size
796   of the split by taking the top-K target test samples of highest/lowest similarity to the source domain
797   for every class; the resulting split is balanced in class distribution. We report the average per-class
798   accuracy for two baselines in Table 10.
799
800
      A.4     E VALUATIVE PARTITIONS
801
802
      To assess a model’s robustness towards source bias and simulate the difficulty of real-world scenarios,
803
      we aim to generate datasets that simulate long-tailed distribution for testing. Specifically, we
804   experiment with using the Power Law (Clauset et al., 2009) and Zipf’s Law (Powers, 1998) to
805   calculate the hardness ratio, which is used to determine the composition of difficult and easy samples
806   in a given split. For the Power Law, we set αpow = 6.0 based on the size of the dataset and generate
807   20 partitions of unequal length and class distribution. We also tried partitioning using Zipf’s law
808   by ranking all T samples based on the defined hardness score to create 200 partitions. We proceed
809   default to using the Power Law as it illustrates performance dichotomy more prominently than Zipf’s
      law.
