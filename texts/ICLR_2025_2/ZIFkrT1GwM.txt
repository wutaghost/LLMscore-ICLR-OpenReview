009                                                A BSTRACT
010
011            We investigate the transformer’s capability for in-context learning (ICL) to simulate
012            the training process of deep models. Our key contribution is providing a positive
013            example of using a transformer to train a deep neural network by gradient descent
014            in an implicit fashion via ICL. Specifically, we provide an explicit construction of a
015            (2N + 4)L-layer transformer capable of simulating L gradient descent steps of an
016            N -layer ReLU network through ICL. We also give the theoretical guarantees for
017
               the approximation within any given error and the convergence of the ICL gradient
               descent. Additionally, we extend our analysis to the more practical setting using
018
               Softmax-based transformers. We validate our findings on synthetic datasets for 3-
019
               layer, 4-layer, and 6-layer neural networks. The results show that ICL performance
020            matches that of direct training.
021
022   1 I NTRODUCTION
023   We study transformers’ ability to simulate the training process of deep models. This analysis is not
024   only practical but also timely. On one hand, transformers and deep models (Brown, 2020; Radford
025   et al., 2019) are so powerful, popular and form a new machine learning paradigm — foundation
026   models. These large-scale machine learning models, trained on vast data, provide a general-purpose
027   foundation for various tasks with minimal supervision (Team et al., 2023; Touvron et al., 2023;
028
      Zhang et al., 2022). On the other hand, the high cost of pretraining these models often makes them
      prohibitive outside certain industrial labs (Jiang et al., 2024; Bi et al., 2024; Achiam et al., 2023). In
029
      this work, we aim to advance the “one-for-many” modeling philosophy of foundation model paradigm
030
      (Bommasani et al., 2021) by considering the following research problem:
031
032    Question 1. Is it possible to train one deep model with the ICL of another foundation model?
033
      The implication of Question 1 is profound: if true, one foundation model could lead to many others
034   without pertaining. In this work, we provide an affirmative example for Question 1. Specifically,
035   we show that transformer models are capable of simulating the training of a deep ReLU-based
036   feed-forward neural network with provable guarantees through In-Context Learning (ICL). Our
037   analysis assumes that we have well-pretrained the transformer using the data generated by the deep
038   network. We require the deep network to maintain consistent hyperparameters (e.g., model width
039   and depth) during the pretraining and testing. However, during the testing, we vary the parameter
040   distribution and input data distribution of the deep network to generate data for the transformer.
041   In ICL, the models learn to solve new tasks during inference by using task-specific examples provided
042   as part of the input prompt, rather than through parameter updates (Wei et al., 2023; Bubeck et al.,
043   2023; Achiam et al., 2023; Bai et al., 2023; Min et al., 2022; Garg et al., 2022; Brown, 2020). Unlike
044   standard supervised learning, ICL enables models to adapt to new tasks during inference using only
045   the provided examples. In this work, the new task of our interest is algorithmic approximation via
046   ICL (Bai et al., 2023; Zhang et al., 2023; Wang et al.). Specifically, we aim to use transformer’s ICL
047   capability to replace/simulate the standard supervised training algorithms for N -layer networks. To
048
      be concrete, we formalize the learning problem of how transformers learn (i) a given function and (ii)
      a machine learning algorithm (e.g., gradient descent) via ICL, following (Bai et al., 2023).
049
050   (i) ICL for Function f . Let f : Rd → R be the function of our interest. Suppose we have a dataset
051   Dn := {(xi , yi )}i∈[n] , where {xi }i∈[n] ⊆ Rd and {yi }i∈[n] ⊆ R are the input and output of f ,
052   respectively. Let xn+1 be the test input. The goal of ICL is to use a transformer, denoted by T , to
053   predict yn+1 based on the test input and the in-context dataset autoregresively: ybn+1 ∼ T (Dn , xn+1 ).
      The goal is for the prediction ybn+1 to be close to yn+1 = f (x).

      (ii) ICL for Gradient Descent of a Parametrized Model f (w, ·). Bai et al. (2023) generalize (i)
055   to include algorithmic approximations of Gradient Descent (GD) training algorithms and explore
056   how transformers simulate gradient descent during inference without parameter updates. They term
057   the simulated GD algorithm “In-Context Gradient Descent (ICGD).” In essence, ICGD enables
058   transformers to approximate gradient descent on a loss function Ln (w) for a parameterized model
059   f (w, ·) based on a dataset Dn . Traditional gradient descent updates w iteratively as wt+1 = wt −
060   η∇Ln (wt ). In contrast, ICGD uses a transformer T to simulate these updates within a forward
061   pass. Given example data Dn and test input xn+1 , the transformer performs gradient steps in an
062   implicit fashion by inferring parameter updates through its internal representations, using input
063
      context without explicit weight changes. Please see Section 2 for explicit formulation.
064   In this work, we investigate the case where f (w, ·) is a deep feed-forward neural network. We defer
065   the detailed problem setting to Section 2. In comparison to standard ICGD (Bai et al., 2023), ICGD
066   for deep feed-forward networks is not trivial. This is due to two technical challenges:
067   (C1) Analytical feasibility of gradient computation for these thick networks.
068   (C2) Explicit construction capable of approximating ICGD for such layers and their gradients.
069   A work similar to ours is (Wang et al.). It demonstrates that the transformer implements multiple steps
070   of ICGD on deep neural networks. However, it requires more layers in the transformer model and
071   fails to consider the Softmax-transformer. We provide a more detailed comparison in Appendix B.1.
072   To this end, we present the first explicit expression for gradient computation of N -layer feed-forward
073   network (Lemma 1). Importantly, its term-by-term tractability provides key insights for the detailed
074   construction of a specific transformer to train this network via ICGD (Theorem 1).
075   Contributions. We offer a positive early investigation of Question 1. Our contributions are threefold:
076    • Approximation by ReLU-Transformer. For simplicity, we begin with the ReLU-based trans-
077      former. For a broad class of smooth empirical risks, we construct a (2N + 4)L-layer transformer
078      to approximate L steps of in-context gradient descent on the N -layer feed-forward networks with
079      the same input and output dimensions (Theorem 1). We then extend this to accommodate varying
080
         dimensions (Theorem 5). We also provide the theoretical guarantees for the approximation within
         any given error (Corollary 1.1) and the convergence of the ICL gradient descent (Lemma 14).
081
       • Approximation by Softmax-Transformer. We extend our analysis to the Softmax-transformer to
082
         better reflect realistic applications. The key technique is to ensure a qualified approximation error
083
         at each point to achieve universal approximation capabilities of the Softmax-based Transformer
084      (Lemma 16). We give a construction of a 4L-layer Softmax transformer to approximate L steps of
085      gradient descent, and guarantee the approximation and the convergence (Theorem 6).
086    • Experimental Validation. We validate our theory with ReLU- and Softmax-transformers, specifi-
087      cally, ICGD for the N -layer networks (Theorem 1, Theorem 5, and Theorem 6). We assess the ICL
088      capabilities of transformers by training 3-, 4-, and 6-layer networks in Appendix G. The numerical
089      results show that the performance of ICL matches that of training N -layer networks.
090   Organization. We show our main results in Theorem 1 and the informal version in Appendix A.
091   Section 2 presents preliminaries. Section 3 presents the problem setup and ICL approximation to
092   GD steps of N -layer feed-forward network based on ReLU-transformer. The appendix includes the
093   related works (Appendix B.1), the detailed proofs of the main text (Appendix D), ICL approximation
094   to GD steps of N -layer network based on Softmax-transformer (Appendix F), the experimental
095   results (Appendix G), and the application to diffusion models (Appendix H).

      B     R ELATED W ORKS , B ROADER I MPACT AND L IMITATIONS
769
770   In this section, we show the related works, broader impact and limitations.
771
772   B.1    R ELATED W ORKS
773
774   In-Context Learning. Large language models (LLMs) demonstrate the in-context learning (ICL)
775   ability (Brown, 2020), an ability to flexibly adjust their prediction based on additional data given in
776   context. In recent years, a number of studies investigate enhancing ICL capabilities (Chen et al., 2022;
777   Gu et al., 2023; Shi et al., 2023), exploring influencing factors (Shin et al., 2022; Yoo et al., 2022), and
778   interpreting ICL theoretically (Xie et al., 2021; Wies et al., 2024; Panwar et al., 2023; Li et al., 2023;
779
      Bai et al., 2023; Dai et al., 2022). The works most relevant to ours are as follows. (Von Oswald et al.,
      2023) showed that linear attention-only Transformers with manually set parameters closely resemble
780
      models trained via gradient descent. (Bai et al., 2023) providing a more efficient construction for in-
781
      context gradient descent and established quantitative error bounds for simulating multi-step gradient
782   descent. However, these results focused on simple ICL algorithms or specific tasks like least squares,
783   ridge regression, and gradient descent on two-layer neural networks. These algorithms are inadequate
784   for practical applications. For example: (i) Approximating the diffusion score function requires
785   neural networks with multiple layers (Chen et al., 2023). (ii) Approximating the indicator function
786   requires at least 3-layer networks (Safran and Shamir, 2017). Therefore, the explicit construction
787   of transformers to implement in-context gradient descent (ICGD) on deep models is necessary to
788   better align with real-world in-context settings. Our work achieves this by analyzing the gradient
789   descent on N -layer neural networks through the use of ICL. We provide a more efficient construction
790
      for in-context gradient descent. Furthermore, we extend our analysis to Softmax-transformer in
      Appendix F to better align with real-world uses.
791
792
      In-Context Gradient Descent on Deep Models (Wang et al.; Panigrahi et al., 2023). A work
793
      similar to ours is (Wang et al.). It constructs a family of transformers with flexible activation functions
794   to implement multiple steps of ICGD on deep neural networks. This work emphasizes the generality
795   of activation functions and demonstrates the theoretical feasibility of such constructions. Our work
796   adopts a different approach by enhancing the efficiency of transformers and better aligning with
797   practical applications. We introduce the following novelties:
798    • More structured and efficient transformer architecture. While the work (Wang et al.) uses a
799      O(N 2 L)-layer transformer to approximate L gradient descent steps on N -layer neural networks,
800      our approach achieves more efficient simulation for ICGD. We approximate specific terms in the
801      gradient expression to reduce computational costs, requiring only a (2N +4)L-layer transformer for
802      L gradient descent steps. Our method focuses on selecting and approximating the most impactful
803      intermediate terms in the explicit gradient descent expression (Lemmas 3 to 5), optimizing layer
804      complexity to O(N L).
805    • Less restrictive input and output dimensions for N -layer neural networks. The work (Wang
806      et al.) simplifies the output of N -layer networks to a scalar. Our work expands this by considering
807      cases where output dimensions exceed one, as detailed in Appendix E. This includes scenarios
808      where input and output dimensions differ.
809    • More practical transformer model. The work (Wang et al.) discusses activation functions in the
         attention layer that meet a general decay condition ((Wang et al., Definition 2.3)) without consid-


                                                          15
      Under review as a conference paper at ICLR 2025


810
        ering the Softmax activation function. We extend our analysis to include Softmax-transformers.
811     Our analysis reflects more realistic applications, as detailed in Appendix F.
812
      • More advanced and complicated applications. The work (Wang et al.) discusses the applications
813
        to functions, including indicators, linear, and smooth functions. We explore more advanced and
814
        complicated scenarios, i.e., the score function in diffusion models discussed in Appendix H. The
815     score function (Chen et al., 2023) falls outside the smooth function class. This enhancement
816     broadens the applicability of our results.
817
      Another work similar to ours is (Panigrahi et al., 2023). It proposes a new efficient construction,
818
      Transformer in Transformer (TINT), to allow a transformer to simulate and finetune more complex
819
      models (e.g., one transformer). The main distinction between ours and (Panigrahi et al., 2023)
820   lies in the different aims: Our approach focuses on using a standard transformer for the simulator
821   (with a minor modification: the “element-wise multiplication layer”), and we provide a theoretical
822   understanding of how a standard transformer can learn the ICGD of an N -layer network using ICL.
823   In contrast, the work (Panigrahi et al., 2023) aims to build even stronger transformers by introducing
824   several structural modifications that enable running gradient descent on auxiliary transformers. While
825   it demonstrates in-context gradient descent for a more advanced model, i.e., one transformer, our
826   work offers the following potential advantages:
827   • Explicit transformer construction. We provide an explicit construction of the transformer,
828     whereas the work (Panigrahi et al., 2023) does not detail the explicit construction of model
829     parameters within their transformer.
830   • Exact gradient descent. We compute the exact and explicit gradient descent for an N -layer
831     network (Lemma 1). Building on this, we employ the transformer’s ICL to perform gradient descent
832     on all parameters. However, the work (Panigrahi et al., 2023) stops the gradient computation
833     through attention scores in the self-attention layer and only updates the value parameter in the
834     self-attention module. Additionally, it uses Taylor expansion to approximate the gradient.
835   • Rigorous error and convergence guarantees. We provide rigorous gradient descent approxima-
836     tion errors (for multiple steps) and convergence guarantees for the ICGD on an N -layer network
837     (Corollary 1.1 and Lemma 14). However, the work (Panigrahi et al., 2023) only presents the
838     gradient approximation error for each specific part of the parameters in a single step.
839   • Attention layer better aligned with practice. Our analysis is based on ReLU-attention (Theo-
840     rem 1) or Softmax-attention (Theorem 6), whereas the work (Panigrahi et al., 2023) utilizes linear
841     attention. Our choice of attention layer better aligns with practical applications.

      B.2   B ROADER I MPACT
843
844   This theoretical work aims to shed light on the foundations of large transformer-based models and is
845   not expected to have negative social impacts.
846
847   B.3   L IMITATIONS
848
849   Our work has the following four limitations:
850
      • Although we provide a theoretical guarantee for the ICL of the Softmax-Transformer to ap-
851     proximate gradient descent in N -layer NN, characterizing the weight matrices construction in
852     Softmax-Transformer remains challenging. This motivates us to rethink transformer universality
853     and explore more accurate proof techniques for ICL in Softmax-Transformer, which we leave for
854     future work.
855   • The hidden dimension and MLP dimension of the transformer in Theorem 1 are both O(N    e K 2) +
856     Dw , which is very large. The reason for the large dimensions is that if we use ICL to perform
857     ICGD on the N -layer network, we need to allow the transformer to realize the N -layer network
858     parameters. This means that it is reasonable for the input dimension to be so large. However, it is
859     possible to reduce the hidden dimension and MLP dimension of the transformer through smarter
860     construction. We leave this for future work.
861   • The generalization capabilities are limited compared with traditional transformers. In our setting,
862     the pretraining task refers to using in-context examples generated by an N -layer network for a
863     given N . Specifically, during pretraining, the distribution of the N -layer network parameters is
        predetermined (e.g., N (0, I)). The input data distribution of N -layer network for generating the


                                                       16
      Under review as a conference paper at ICLR 2025


864
        in-context examples is also predetermined (e.g., N (−2, I)). The generalization capabilities include
865     the following two aspects: (i) Varying the input data distribution for the N -layer network to generate
866     the in-context examples. For example, we change the input data distribution from N (−2, I) to
867     0.9N (−2, I) + 0.1N (2, I) during the testing in Appendix G.1. (ii) Varying the distribution of the
868     N -layer network parameters. For example, we change the distribution from N (0, I) to N (0.5, I)
869     in Appendix G.2. The above points lead to differences between the distributions of in-context
870     examples during pretraining and testing. However, we must generate the in-context examples by
871     the N -layer network with the same hyperparameters, including the network width and depth. We
872     leave the theoretical analysis of broader generalization capabilities for future work.
873   • In theory, the FLOPs (Hoffmann et al., 2022) required to perform one forward pass of the trans-
874     former are greater than those required for the direct training of an N -layer network. (i) For the
875     forward pass of the transformer, the FLOPs for in-context learning (ICL) are O(nLN 3 K 5 /ϵ2 ),
876
        where ϵ is the approximation error in the sum of ReLU. (ii) For direct training of the N -layer
        network, the FLOPs without ICL are O(nLN K 2 ). Therefore, the FLOPs required for ICL ex-
877
        ceed those needed for direct training of the N -layer network. However, experimental results in
878
        Appendix G demonstrate that the transformer with ICL can achieve the performance of a trained
879     6-layer network using fewer FLOPs in practice (3.3 billion vs. 7.6 billion FLOPs). This finding
880     encourages further exploration of more efficient architectures. We also leave this topic for future
881     research.

2485
2486   In this section, we conduct experiments to verify the capability of ICL to learn deep feed-forward
2487   neural networks. We conduct the experiments based on 3-layer NN, 4-layer NN and 6-layer NN using
2488   both ReLU-Transformer and Softmax-Transformer based on the GPT-2 backbone.
2489
2490   Experimental Objectives. Our objectives include the following three parts:
2491   • Objective 1. Validating the performance of ICL matches that of training N -layer networks, i.e.,
2492     the results in Theorem 1, Theorem 5, and Theorem 6.
2493   • Objective 2. Validating the ICL performance in scenarios where the testing distribution diverges
2494     from the pretraining one or where prompt lengths exceed those used in pretraining.
2495
       • Objective 3. Validating the ICL performance in scenarios where the distribution of parameters in
2496     the N -layer network diverges from that of the pretraining phase.
2497
       • Objective 4. Validating that a deeper transformer achieves better ICL performance, supporting the
2498     idea that scaling up the transformer enables it to perform more ICGD steps.

       H        A PPLICATION : ICL FOR D IFFUSION S CORE A PPROXIMATION
2701
2702   In this part, we give an important application of our work, i.e., learn the score function of diffusion
2703   models by the in-context learning of transformer models. We give the preliminaries about score
2704   matching generative diffusion models in Appendix H.1. Then, we give the analysis for ICL to
2705   approximate the diffusion score function in Appendix H.2.
2706
2707   H.1       S CORE M ATCHING G ENERATIVE D IFFUSION M ODELS
2708
2709   Diffusion Model. Let x0 ∈ Rd be initial data following target data distribution x0 ∼ P0 . In essence,
2710   a diffusion generative model consists of two stochastic process in Rd :
2711
        • A forward process gradually add noise to the initial data (e.g., images): x0 → x1 → · · · → xT .
2712
2713
        • A backward process gradually remove noise from pure noise: yT → yT −1 → · · · → y0 .
                                                                                     d
2714   Importantly, the backward process is the reversed forward process, i.e., yt ≈ xT −t for i ∈ 0, . . . , T .2
2715   This allows the backward process to reconstruct the initial data from noise, and hence generative. To
2716
       achieve this time-reversal, a diffusion model learns the reverse process by ensuring the backward
       conditional distributions mirror the forward ones. The most prevalent technique for aligning these
2717
       conditional dynamics is through “score matching” — a strategy training a model to match score
2718
       function, i.e., the gradients of the log marginal density of the forward process (Song et al., 2020b;a;
2719   Vincent, 2011). To be precise, let Pt , pt (·) denote the distribution function and destiny function of
2720   xt . The score function is given by ∇ log pt (·). In this work, we focus on leveraging the in-context
2721   learning (ICL) capability of transformers to emulate the score-matching training process.
2722

2743
