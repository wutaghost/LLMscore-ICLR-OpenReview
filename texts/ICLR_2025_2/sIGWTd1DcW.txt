                                                        A BSTRACT
010
011
                   The attention mechanism is a critical component of Large Language Models
012
                   (LLMs) that allows tokens in a sequence to interact with each other, but is order-
013                invariant. Incorporating position encoding (PE) makes it possible to address by
014                position, such as attending to the i-th token. However, current PE methods use
015                token counts to derive position, and thus cannot generalize to higher levels of
016                abstraction, such as attending to the i-th sentence. In this paper, we propose a
017                new position encoding method, Contextual Position Encoding (CoPE), that allows
018                positions to be conditioned on context by incrementing position only on certain
019                tokens determined by the model. This allows more general position addressing
020                such as attending to the i-th particular word, noun, or sentence. We show that CoPE
021
                   can solve the selective copy, counting and Flip-Flop tasks where popular position
                   embeddings fail, and improves perplexity on language modeling and coding tasks1 .

      1       I NTRODUCTION
025
026
      Many common data sources such as text, audio, code, and timelines of events are ordered sequences.
027   When processing such sequences, the ordering information is clearly critical. In the case of text,
028   position information is vital not only for decoding meaning between words, but is necessary at every
029   scale, such as the sentence and paragraph level. The Transformer architecture, which is the main
030   backbone of current Large Language Models (LLMs), relies on the attention mechanism (Bahdanau
031   et al., 2014) that inherently lacks ordering information and treats sequences as sets. Thus, it is
032   necessary to have an additional mechanism for encoding position information. Position encoding
033   (PE) (Collobert and Weston, 2008; Sukhbaatar et al., 2015) achieves this by assigning an embedding
034   vector to each position and adding that to the corresponding token representations. Position itself can
035
      be measured in two ways: absolute PE that counts tokens from the start of a sequence, and relative
      PE that counts backward starting at the current token. PE methods have become an integral part of
036
      LLMs with several proposed variations of these basic themes (Dufter et al., 2022).
037
038   One common feature of existing PE methods is the use of tokens as the unit of measurement. However,
039   a token is a variable unit that can be a whole word, or part of it, or even a character depending on the
040   tokenization method. For Byte-Pair Encoding (BPE) tokenization (Sennrich et al., 2016), a word can
041
      be 1 or many tokens depending on the word itself. This position variance increases for more abstract
      elements like a sentence, which can have from ten to hundreds of tokens. Therefore token position is
042
      not suited for general position addressing such as finding the i-th word or sentence.
043
044   In order to tie position measurement to more semantically meaningful units such as words, or
045   sentences, one needs to take context into account. However, this is impossible with current PE
046   methods as position addressing is computed independently of the context, and then later merged with
047
      context addressing. We argue that this separation of the position and context addressing is the core
      problem, and instead we propose a new PE method that integrates context and position addressing
048
      together. In particular, we are interested in position encoding that is context dependent, so it can
049
      represent various levels of position abstraction at the same time, from token positions to sentence
050   positions. This way, it is possible for example to use token positions to attend to the previous few
051   tokens, while using sentence positions to attend to previous sentences for better understanding of the
052   current sentence. We call our method Contextual Position Encoding (CoPE).

075
      CoPE first determines which tokens to count using their context vectors. Specifically, given the
      current token as a query vector, we compute a gate value for each previous token using their key
076
      vectors. Then we aggregate those gate values to determine the relative position of each token with
077
      respect to the current token, as shown in Fig. 1. Unlike token positions, this contextual position can
078   take fractional values, thus cannot have an assigned embedding. Instead, we interpolate embeddings
079   that are assigned to integer values to compute position embeddings. Like the other PE methods, these
080   position embeddings are then added to the key vectors, so a query vector can use them in the attention
081   operation. Since contextual position can vary from query-to-query and layer-to-layer, the model can
082   simultaneously measure distances in multiple units.
083
      We first apply CoPE to several toy tasks: symbolic counting, selective copying and the Flip-Flop task,
084   where it outperforms token-based PE methods, especially in the case of out-of-domain generalization.
085   To test real-world applicability, we use a language modeling task on Wikipedia text where we show
086   CoPE also leads to better performance. The same performance gain is also observed when trained on
087   code. Further, we perform pre-training of 1.4B models from scratch and find that CoPE significantly
088   improves perplexity, and leads to better accuracy on word counting task.

121   3.1   S TANDARD POSITION ENCODING FAILS ON SIMPLE TOY TASKS
122
123   Here we analyze a simplified attention mechanism and a toy task to illustrate shortcomings of current
124   position addressing techniques that are based on token positions. Let us consider simple sequences
125   consisting of two types of tokens x and y to illustrate the interplay of the context and position
126
      addressing mechanisms. Given a sequence yyyyxyyy, for example, context addressing can focus the
      attention on token x by producing key and query vectors such that
127
128                                    q⊤ kx = q⊤ ky + ∆       where ∆ > 0.                                  (2)
129
130   This will give attention weights ax /ay = exp ∆. Suppose ∆ = 1, then the attention on x will be
131   about e ≈ 2.7 times larger than of y. Similarly, position addressing allows us to extract the i-th token
132
      (in relative position so i = 0 is the last token) using position embeddings such that
133

137   representation (i.e. the same key vectors), their attention difference will only depend on their positions
138   i and j:

187   In CoPE, positions are measured in a context dependent way rather than being a simple token count.
188
      The method works by first deciding which tokens should be included when measuring distance using
      their context vectors. To do that, a gate value is computed for every query qi and key kj pair
189
190                                                 gij = σ(q⊤

192

193   tokens i and j, the gate value should be 1 for only sentence separation tokens such as “.”. The gates
194   also condition on the query, so each query can have different position measurements if needed. The
195   soft gating function allows differentiation so that the system can be trained with backpropagation.

224   Limited positions From Eq. (4), we can see the maximum value for pij is the context size T , which
225   means we need T + 1 position embeddings (including position 0). However, if the gates are sparsely
226   activated (e.g. counting sentences), we can cover the whole context T with much fewer positions.
227
      Thus we can set a limit pmax < T on the maximum possible position by setting pij ← min (pij , pmax ).

229
      Multi-head attention So far, CoPE is defined for single-head attention. The multi-head extension
      is straightforward as each head will do their own CoPE independently. The keys and query vectors
230
      are different between heads, so that means they can implement different position measurements.
231
      For example, head 1 can have keys that turn all gates on so that the position counts tokens, while
232   head 2 gates are on only for word-beginning tokens, to count words as positions. While the position
233   embeddings e[p] are shared between the heads only, we also experiment with position embeddings
234   that are shared across the layers as well.
235
236   Computation The most computationally expensive operation in the self-attention module is the
237   key (or value) and query multiplication that has O(T 2 dh ) FLOPS, where dh is the head dimension.
238   The most expensive operation of CoPE is the gate computation in Eq. (3), but we can benefit from the
239   query and key multiplication that was already computed during attention, and reduce gate computation
240   to simply applying the softmax function. The next most expensive operation in CoPE is the matrix
241
      multiplication in Eq. (7) that has O(T pmax dh ) FLOPS. This computation can be reduced by selecting
      a small pmax , which we show works well in our experiments.
242
243
      Computing gates Note that the same keys are used in computing the gates in Eq. (3) as the final
244
      attention computation of Eq. (9). This biases highly attended tokens to be counted in the position
245
      computation as well. To disentangle position from attention itself, we can use separate keys that are
246   computed with an additional projection ki = Wg hi when computing gates. We denote this version as
247   sep-keys in our experiments. Another option is to use the value vectors instead so that gij = σ(q⊤
                                                                                                       i vj ),
248   which we refer to as val-gates. However, these versions will require more computation as we cannot
249   reuse the key query multiplication.

      5.1   F LIP -F LOP TASK
258
259   The Flip-Flop language modeling task was introduced in Liu et al. (2024) to expose the failure
260   of Transformer models to capture robust reasoning over long-range input sequences. The input
261   strings consist of alternating sequences of instructions {w, i, r} ("write", "ignore", and "read"), each
262   followed by one bit of information (0 or 1) that the model needs to memorize if it follows w, or recall
263   the last memory if it follows r. It is guaranteed that all strings start with w and end with r. For
264   example, given string ”w0i1r0w1i0i1i1r”, the expected output is 1, since the last w operation is
265   followed by 1. To solve this task, the model has to be able to sharply attend to the latest occurrence
266
      of the w symbol, the position of which varies between sequences due to ignore instructions. The task
      defines two test sets: in-distribution and out-of-distribution (OOD), where the latter increases the
267
      distance to the last w by increasing the number of ignore instructions.
268

304   5.2    S ELECTIVE C OPY TASK
305
306
      The selective copy task introduced by Gu and Dao (2023) requires context-aware reasoning for
      selective memorization. In this task the model is given a sequence of tokens and asked to copy all
307
      tokens except a denoted blank token. For example, when the input is DBBCFBFBE where B is the
308
      blank, the model is expected to output DCFFE. In our experiments, we set the vocabulary size to 16,
309   and the output sequence length (number of non-blanks) to 256, and vary the number of blank tokens.
310   The training and in-distribution test data have 256 blanks whereas the dense and sparse OOD test
311   data have 128 blanks and 512 blanks, respectively. We train models with dimension 64, 2 layers and
312   2 heads, and report test error after 100k steps. The results, given in Table 2 (right), show that on the
313   in-distribution test set our method CoPE can solve the task while others fail to do so. Similarly, CoPE
314   generalizes better on both dense and sparse OOD test sets. The presence of blank tokens makes it
315   harder to locate the next token to copy, but CoPE can count only non-blank tokens, and hence be
316   more resilient to blanks. At each step, it can then simply copy the non-blank token a distance of 256
317
      (non-blanks) away. Repeating this 256 times will copy the entire sequence of non-blanks.

367

497   6    R ELATED W ORK
498
499   While the attention mechanism was proposed in Bahdanau et al. (2014) for processing sequences of
500   tokens, the model was still based on RNNs so position encoding (PE) was not necessary. The Memory
501   Network (Weston et al., 2015) architecture moved away from RNNs when processing sequences,
502   instead using multiple layers of attention, and first introduced using PE together with the attention
503   mechanism (Sukhbaatar et al., 2015). They added learnable embedding vectors that correspond to
504   each relative position to the hidden representations. A similar position embedding was used earlier
505   in a convolution-based architecture (Collobert and Weston, 2008), and later in an architecture that
506   combines convolution with attention (Gehring et al., 2017). The latter used an absolute PE because
507
      relative position cannot be defined on the source text in machine translation.
508   PE became in an important topic of research with the popularity of the Transformer architecture.
509   The original paper by Vaswani et al. (2017) employed an absolute PE with fixed vectors, but the
510   relative position embedding was later used in Shaw et al. (2018). Relative PE is especially suited to
511   processing unbounded sequences (Dai et al., 2019). Since then, many different variations of relative
512   and absolute PE have been proposed. In Raffel et al. (2020), each relative position is assigned a
513
      simple bias scalar that gets added to the attention logits. While being efficient, this makes position
      addressing independent of the current token. Press et al. (2022) further simplifies the bias terms
514
      by making them fixed in a decaying pattern instead of learning for generalization to longer context.
515
      Haviv et al. (2022) takes it to the extreme by removing PE and demonstrated that position information
516   can be recovered by counting previous tokens with causal attention.
517
518   While absolute PE was used in early LLMs (Radford et al., 2019), relative PE is more common in
519
      recent LLMs (Touvron et al., 2023b;a; Jiang et al., 2023). In particular, RoPE (Su et al., 2024) made
      it possible to do relative PE without modifying the self-attention code. It relies on the fact that query
520
      and key dot product only depend on the angle between those vectors and are agnostic to their absolute
521
      angles. Thus if they are rotated by angles proportional to their absolute position, then its effect on the
522   attention logit will only depend on their difference in position. However, CoPE differs from all these
523   PE methods as it measures position in a context dependent way instead of simply using token counts.
524
525
      While RNNs can be inserted into the Transformer architecture to represent position information in
      an implicit way (Wang et al., 2019; Neishi and Yoshinaga, 2019), the sequential nature of RNN
526
      operations breaks the parallelization of Transformer training, making it slower and less practical. In
527
      comparison, the only sequential operation in CoPE is a cumulative sum, which is lightweight and can
528   be partially parallelized. For more details on different PE methods, see the survey by Dufter et al.
529   (2022). Zhao et al. (2023) also provides a survey focused on length generalization of PE methods.
530
531
532   7    C ONCLUSION
533
534   In this paper, we proposed a novel position encoding method called CoPE that measures position in
535   a context dependent way, thus moving away from the current token-based position paradigm. This
536   approach allows more freedom when addressing by position, and brings gains on several tasks. While
537   this paper only focused on text and code domains, CoPE has the potential to improve domains such
538   as video and speech where token position seems intuitively even less appropriate. Another avenue to
539   explore is training even larger models with CoPE beyond the 1.4B parameter models in this paper,
      which requires considerable pre-training compute budget.

819   18             logits_floor = logits_int.gather(-1, pos_floor)
      19             w = pos - pos_floor
820
      20             return logits_ceil * w + logits_floor * (1 - w)

823   23     def __init__(self, npos_max, head_dim):
824   24         super().__init__()
      25         self.cope = CoPE(npos_max, head_dim)
825
      26         self.head_dim = head_dim
826   27
827   28         def forward(self, query, key, val, mask):
828   29             # q, k, v have dimensions batch x seq_len x head_dim
829   30             attn_logits = torch.bmm(query, key.transpose(-1, -2))
      31             attn_logits = attn_logits / math.sqrt(self.head_dim)
830
      32             attn_logits += mask.log()
831   33             attn_logits += self.cope(query, attn_logits)
832   34             attn = torch.softmax(attn_logits, dim=-1)
833   35             out = torch.bmm(attn, val)
834   36             return out

           Following Liu et al. (2024), we experiment with Transformer models of different sizes, varying head
841        dimension in {128, 256}, and number of heads and layers in {2, 4}. We utilize AdamW optimizer
842        with linear learning rate decay (lr = 3e − 4, β1 = 0.9, β2 = 0.999, ε = 10−8 ). We train on 8 GPUs
843        with batch size 16 for 10,000 steps. For the main results, we ran 3 seeds and reported their average
844        along with standard deviations as can be seen in Table 8.
845
           In our ablations, we experiment with hard attention, as in this task for each sequence model is required
846
           to attend to a single specific token. Furthermore, we experiment with incorporating contextual infor-
847        mation into positional encoding through a multilayer perceptron (MLP). In particular, instead of using
848        interpolation (Eq. (5)) we learn the positional encodings by training an N -dimensional MLP layer,
849        and denote this approach as CoPE _MLP. This change significantly increases memory and runtime
850        load on the training (by 30-50 times in our experiments compared with regular positional encodings),
851        but allows for more flexibility in positional in-context learning. We vary N ∈ {32, 64, 128, 256}, and
852        report results in Table 8 for N = 64 to strike the balance between model’s efficiency and performance.
853        We also experiment with ingesting CoPE _MLP only in the first layer of the transformer model:
854        this helps to reduce runtime by the order of magnitude, but hurts the performance, especially on the
855
           out-of-distribution (OOD) task.
856        Similarly to the AL I B I approach proposed by Press et al. (2022), we can treat the cumulative sum of
857        the gates as learned biases (while in the original paper authors used static bias). Specifically, Eq. (6)
858        will be simplified to:
859                                           aij = Softmax(q⊤  i kj + m · pij ),                              (10)
860
861        where m is head-specific slope fixed before training. In our experiments on the FlipFlop task,
862        we train model with 4 heads, and experiment with three sets of pre-fixed slopes: {1, 21 , 212 , 213 },
863        { 212 , 213 , 214 , 215 }, { 214 , 215 , 216 , 217 }. We also train a model where m is a learned parameter, specific for
           each head and layer, and initialized from 0. No other positional embeddings are added to the model.
