011                                               A BSTRACT
012
013            While large language models (LLMs) have been thoroughly evaluated for deduc-
014            tive and inductive reasoning, their proficiency in abductive reasoning and holis-
015            tic rule learning in interactive environments remains less explored. We intro-
016            duce RULEARN, a novel benchmark specifically designed to assess the rule-
017            learning abilities of LLM agents in interactive settings. In RULEARN, agents
018
               strategically interact with simulated environments to gather observations, discern
               patterns, and solve complex problems. To enhance the rule-learning capabilities
019
               for LLM agents, we propose IDEA, a novel reasoning framework that integrates
020
               the process of Induction, DEduction, and Abduction. The IDEA agent gener-
021            ates initial hypotheses from limited observations through abduction, devises plans
022            to validate these hypotheses or leverages them to solve problems via deduction,
023            and refines previous hypotheses using patterns identified from new observations
024            through induction, dynamically establishing and applying rules that mimic human
025            rule-learning behaviors. Our evaluation of the IDEA framework, which involves
026            five representative LLMs, demonstrates significant improvements over the base-
027            line. Furthermore, within this framework, our comparison with 50 human partic-
028            ipants reveals notable discrepancies in rule-learning behaviors. LLM agents tend
029
               to generate plausible initial hypotheses but struggle to refine them through interac-
               tion. Conversely, humans, despite sometimes overlooking initial details, excel at
030
               incorporating feedback and continuously improving their hypotheses. We believe
031
               our benchmark, RULEARN, will serve as a valuable and challenging resource,
032            and that the IDEA framework will provide crucial insights for the development of
033            LLM agents capable of human-like rule learning in real-world scenarios. We will
034            release our code and data upon acceptance of the paper.

      For such cyclic reasoning to function effectively, it requires an interactive foundation. In the real
079   world, the observations we reason from are gathered through interaction. Then, through alternating
080   cycles of refining hypothesis and interaction, we iteratively collect evidence and refine our hypothe-
081   ses, drawing ever closer to the truth. The dynamic interplay between interaction and different stages
082   of the reasoning loop within an interactive environment is the key to humans’ reasoning effective-
083   ness. However, recent studies focusing on reasoning often lack this dynamic and interdependent
084   nature, as abduction, induction, and deduction are tested one at a time in non-interactive environ-
085   ments (Bowen et al., 2024; Wang et al., 2023; Saparov et al., 2024; Liu et al., 2024).
086   To address this gap, we introduce RULEARN, a benchmark designed to evaluate the rule-learning
087   abilities of LLM agents in interactive environments. RULEARN consists of puzzles with hidden
088   rules set in a text-based environment, where agents begin exploration without prior knowledge of
089   these rules. The agent’s objective is to solve the puzzle by interacting with the environment. At each
090   step, the agent selects an action from a predefined set, executes it, and gathers observations to gain
091   insights into the environment. Successfully solving the puzzles requires the agent to strategically
092   select actions, efficiently gather pattern-revealing observations, and accurately reason from them to
093
      infer the hidden rules. RULEARN presents substantial challenges, as agents must rely entirely on
      observations generated by their chosen actions to discern rules. If these observations fail to reveal a
094
      clear pattern, the agent is likely to fail.
095
096   We design three types of environments within RULEARN to evaluate the rule-learning ability in
097   different scenarios: (1) The Function Operator: The agent is tasked with determining the coeffi-
098   cients of mathematical functions defined by hidden expressions. Agents can assign various values
099
      to the input variables and observe the outputs, using this information to hypothesize the function’s
      form. The challenge lies in efficiently selecting input values that reveal the underlying structure and
100
      accurately computing the coefficients based on limited observations. (2) The Escape Room: The
101
      agent is tasked with deciphering the passcode to exit an escape room. A hidden rule determines
102   how the objects in the room infer the passcode digits. Agents interact with these objects to gather
103   clues and input passcodes into the door. The door provides hints indicating which digits are correct.
104   Based on these hints, agents formulate hypotheses and infer the relationship between the objects and
105   the passcode. (3) The Reactor: The agent is tasked with synthesizing target strings using a reactor
106   with a hidden string-combining rule. Agents need to experiment with different inputs and analyze
107   outputs to deduce the reactor’s transformation rule and achieve the desired outcome. Detailed puzzle
      definitions are provided in § 3.

      To tackle the challenge in RULEARN, we introduce IDEA, a novel reasoning framework that
109   integrates the process of Induction, DEduction, and Abduction. The IDEA agent employs these
110   reasoning processes iteratively to explore the environments, learn rules, and achieve goals. In the
111   abduction phase, the IDEA agent generates an initial hypothesis from limited observations. During
112   the deduction phase, the IDEA agent creates and executes plans to meet objectives or test its hy-
113   pothesis. In the induction phase, the IDEA agent refines its hypothesis based on new observations,
114   enhancing their accuracy and robustness. This iterative cycle enables the LLM agent to continually
115   improve the learned rules through environmental feedback. An overview of how the IDEA agent
116   solves puzzles in RULEARN is shown in Figure 2.
117   We evaluat the IDEA framework using five popular LLMs—GPT-3.5-Turbo, GPT-4o, Gemma-
118   7B, Llama3-8B and Llama3-70B. The experimental results demonstrate that IDEA significantly
119   improves rule-learning proficiency and puzzle-solving performance of the LLM agent within the
120   RULEARN benchmark. The IDEA agent increases success rates by approximately 10% compared
121   to the baseline LLM agent. Specifically, the action of the IDEA agent aims to validate or leverage
122   current hypothesis, providing important guidance in an environment with hidden rules. Without the
123
      guidance of the hypothesis, the LLM agent will always choose the most direct actions attempting
      to quickly solve the puzzle. Such actions, taken without understanding the hidden rules, are akin
124
      to navigating without a compass—efforts are made, but they often lead to ineffective or misguided
125
      outcomes. Based on our experiments, the IDEA agent achieves 47.6% fewer repeated actions,
126   yielding more diverse observations and finally solving the puzzles with a better understanding of the
127   hidden rule. These improvements suggest that IDEA enables LLMs to better adapt and learn rules
128   in unfamiliar environments, narrowing the gap between human and AI rule-learning abilities.
129
      Within the IDEA framework, we conduct a study to compare the rule-learning behaviors and perfor-
130
      mances between LLMs and 50 human participants solving puzzles. Our findings reveal that, despite
131
      the improvements brought by the IDEA framework, LLMs still face the following challenges when
132   compared to human participants: (1) LLM agents continue to struggle with efficiently exploring un-
133   familiar environments. The new observations gathered usually do not provide enough information
134   to help reveal the underlying rules, moreover, LLM agents tend to repeat previous actions that do
135   not yield new information. (2) LLMs fall short in deducting valid plans to verify current hypotheses
136   and guide future exploration. (3) LLMs fall short to correct previous hypotheses when they con-
137   tradict new observations and are less capable of refining a hypothesis to make it more robust and
138   encompassing all observations. These findings provide valuable insights for the development of
139   LLM agents capable of human-like rule learning in real-world scenarios.

144   2.1   LLM AGENT
145
146   Agents driven by large language models equipped with memory, tool use, planning, and reason-
147   ing capabilities have shown significant advancements in understanding complex tasks and achieving
148   notable performance (Wang et al., 2024b). Techniques like memory summarization (Chen et al.,
149   2023; Zhou et al., 2023; Wang et al., 2024a) and retrieval (Andreas, 2022; Park et al., 2023; Zhong
150   et al., 2023) assist LLM agents in filtering relevant information based on the current context (Zhang
151   et al., 2024). Additionally, the integration of tool use enables LLM agents to perform better in
152   knowledge-grounded tasks. For instance, some studies (Nakano et al., 2022; Lu et al., 2023; Shi
153
      et al., 2023a) have enabled LLM agents to access up-to-date information from external databases to
      provide correct answers, while others (Schick et al., 2023) have facilitated the use of various tools
154
      like calculators to enhance solution accuracy. Planning modules (Yuan et al., 2023; Shen et al.,
155
      2023) allow these agents to decompose complex tasks into simpler components, achieving signifi-
156   cant improvements. Moreover, employing search algorithms (Yao et al., 2023; Besta et al., 2024)
157   helps LLM agents reach solutions in fewer steps. Building on this foundation, we are interested in
158   whether LLM agents can derive new knowledge–learning rules through interacting with the envi-
159   ronment, rather than merely answering questions. Recent studies have begun investigating different
160   types of reasoning in LLMs, including abduction, deduction, and induction (Bowen et al., 2024;
161   Wang et al., 2023; Saparov et al., 2024; Cheng et al., 2024; Yang et al., 2024). However, existing
      research mostly focuses on isolated aspects of the rule-learning process. Consequently, the compre-

163   topic that will be further examined in Section 2.2.

      2.2   RULE L EARNING
166
167   Rule learning—involving evidence gathering, pattern identification, hypothesis formulation, and
168   validation—is gaining focus in recent research. Yang et al. (2023) and Zhu et al. (2024) improved
169   question-answering systems by formulating optimal response rules, while Shi et al. (2023b) ad-
170   vanced event prediction by enabling LLMs to hypothesize causes of previous events. Some works
171   also evaluate LLMs rule-learning abilities (Liu et al., 2024). However, existing studies adopt static,
172   non-interactive settings for rule learning, where LLMs are provided with complete information to
173   formulate rules. This approach has two significant limitations: it contrasts with real-world rule
174
      learning, which involves actively and strategically gathering evidence, and it restricts agents from
      conducting experiments to validate the rules they learn. In real-world scenarios, the process of gath-
175
      ering observations, validating hypotheses, and refining hypotheses is iterative. These simplified,
176
      static settings fail to adequately assess an agent’s capabilities in real, dynamic environments. Even
177   in studies with interactive environments (Xu et al., 2024; Montes et al., 2022), information gathering,
178   rule generation, and application are treated as independent objectives or are incomplete. In reality,
179   these processes are dynamic and interdependent: generated hypotheses guide actions, and new ob-
180   servations refine those hypotheses. In contrast, RULEARN provides a realistic setting that allows
181   LLMs to engage in dynamic, interdependent rule-learning processes, closely mirroring real-world
182   scenarios. To effectively navigate and learn within this environment, IDEA equips LLM agents with
183   the ability to emulate human reasoning by managing these interdependent processes.

185   3     T HE RULEARN B ENCHMARK
186
187   We develop three puzzle sets, the Function Operator, the Escape Room, and the Reactor, each
188   comprising 20 puzzles associated with environments of varying complexity.
189
190
      The Function Operator. This puzzle type simulates scenarios where systemic theories or estab-
      lished knowledge (in this case, mathematics) is applicable. The agent interacts with a set of uni-
191
      variate multi-term equations involving integer parameters from [0,9] and elementary functions of
192
      the variable x, selected from f (x) ∈ {x0 , x1 , x2 , sin(x), x1 , |x|, −x}. The agent is provided with
193   the number of functions, the presence of specific parameters in each function (the exact numerical
194   values of these parameters are unknown and represented by letters), and the types of elementary
195   functions involved in the current puzzle. The goal of the agent is to deduce the values of these pa-
196   rameters. For example, in one puzzle, the ground truth is F1 (x) = a sin(x) + b × x1 , F2 (x) = ax2
197   where a = 3 and b = 2. The agent knows the following information: There are three elementary
198   functions in this puzzle {sin(x), x1 , x2 }, there are two functions F1 (x) and F2 (x), F1 (x) has 2
199   terms and parameters a, b in it, and F2 (x) has 1 term and one parameter a. To solve the puzzle, the
200   agent must interact with the environment through a defined action space: selecting a function and
201
      assigning values to x, then observing the resultant output. For example, assigning values 1 and 2 to
      F2 reveals a quadratic increase in output, indicating the presence of x2 in F2 . Similarly, assigning
202
      a value of 1 to F1 results in a floating-point output, rather than an integer, suggesting the inclusion
203
      of trigonometric components, confirming that sin(x) is a component of F1 . Each puzzle features a
204   different number of functions, unknown parameters, and elementary functions in use.
205
206   The Escape Room. This environment simulates scenarios where no established knowledge is ap-
207
      plicable. We create a fictitious environment: an agent is in an escape room–an art gallery, tasked
      with deciphering a password to unlock a code-secured door and escape. The password is a 3-digit
208
      number where each digit represents the count of paintings of a specific type—watercolor, oil, or
209
      acrylic—that share a given color. The agent can explore the environment and investigate paintings,
210   receiving brief descriptions such as This is an acrylic painting of a green jungle, providing informa-
211   tion about the painting’s type and color. The agent only knows that the password is a 3-digit number
212   and receives a hint to focus on a specific color relevant to the password. After forming an initial
213   hypothesis and inputting the password into the door, the door provides hints indicating which digits
214   are correct. Based on these hints, the agent refines its hypotheses and infers the relationship between
215   the painting attributes and the passcode. To prevent the problem from being solved merely by brute
      force, the specific color associated with the password changes after every three failed attempts. Each

251   4      T HE IDEA AGENT
252
253   We introduce IDEA, a novel reasoning framework that integrates the process of Induction,
254   DEduction, and Abduction to learn rules in interactive environments. The IDEA agent consists of
255   the following components: Goal(G), Action Space(A), Memory(M), Hypothesis(H), and Plan(P),
256   which are elaborated in Table 2.
257
      Upon beginning to explore a puzzle, we initialize the agent memory with an initial observation of the
258   environment. The agent’s goal is initialized with the objective of the puzzle, e.g., synthesize a target
259   string for a Reactor puzzle. The agent’s action space is initialized as the set of interactive actions
260   defined by the puzzle, such as choosing two strings and running the Reactor, as well as establish
261   the initial hypothesis (abductive action), devises a plan to validate or leverage hypothesis (deductive
262   action), and refining the current hypothesis (inductive action).
263
      The IDEA agent begins with an abductive action to generate an initial hypothesis, followed by a
264   deduction step to create a new plan. Based on this plan, the agent interacts with the environment.
265   Upon receiving feedback from the environment as a new observation, the agent may take an induc-
266   tive action to refine the hypothesis or perform another interaction with the environment. Deductive
267   action is invoked to adjust the plan every time the hypothesis changes. This reasoning loop con-
268   tinues until the puzzle is solved or a maximum number of steps is reached. After each step, the
269   results are appended to the agent’s memory, including interaction outcomes and any modifications
      to the hypothesis or plan. We provide a simplified algorithm demonstrating how the IDEA agent

303
304   To evaluate the effectiveness of IDEA, we respectively initialize it with three popular open-source
305   LLMs, including Gemma-7B (Team et al., 2024), Llama3-8B, and Llama3-70B (Dubey et al., 2024),

307   IDEA agent against the following two variants:
308

309
        tion and does not generate hypotheses or plans. At each step, it selects an interactive action solely
310     based on its current memories and the goal.
311   • Oracle-rule Agent: Even if the agent could successfully learn the correct rule, applying the
312     learned rule to solve the puzzle is non-trivial. The Oracle-rule agent serves as a special control
313     group to establish the Oracle performance with the ground-truth rule provided at the beginning.
314     Specifically: 1) For the Function Operator puzzles, agents are given the exact forms of the func-
315     tions. Their task is to derive the values of the coefficients. 2) For the Escape Room puzzles, agents
316     are provided with how the password is constructed from the objects in the room. Their task is to
317     derive the password using the provided rule. 3) For the Reactor puzzles, the reaction rule is given
318     to the agents in natural language accompanied with examples. The agents only need to synthesize
319
        the target strings using the provided rules.
320   Each variant is evaluated on all three puzzle types (totaling 60 puzzles), with five trials per puzzle,
321   resulting in 300 tests per variant. In our initial experiments, we found that using a higher temperature

439   each interaction yields deterministic results, and repeating the same action generally does not pro-

441   could result in different outcomes after the rule changes following three failed attempts.
442
      We calculate the average number of repeated actions performed while solving each puzzle, with
443
      detailed statistics presented in Table 7 in Appendix A.2. We observe that most LLMs commonly
444
      repeat actions in the Baseline agent. Although models like GPT-4o and Llama3-70B exhibit this
445   issue to a lesser degree, even these advanced models sometimes repeat actions that fail to provide
446   further insights. The IDEA agent effectively reduces this tendency by explicitly generating plans
447   during the deduction phase. By outlining a clear plan, the IDEA agent can better assess whether
448   the current observations are sufficient or if further specific evidence is needed to reveal the under-
449   lying rule. For example, in the Escape Room puzzle, the IDEA agent avoids unnecessary attempts
450   at entering passwords when the evidence gathered is sufficient to determine the correct code (see
451   Figure 11 in Appendix A.4).
452   IDEA relies on the reasoning ability of underlying LLMs. The effectiveness of IDEA depends
453   on the underlying LLMs’ ability to reason from hypotheses and observations. Particularly, if an
454   agent generates a false hypothesis and fails to properly refine it, being guided by this incorrect
455   hypothesis can lead the agent to perform even worse than the baseline. During our experiments, we
456   observed that current LLMs tend to hallucinate, especially in the Escape Room puzzles and more
457   severely in the Reactor puzzles. This results in smaller performance improvements compared to
458   those seen with the Function Operator puzzles. This is likely because such fictitious scenarios are
459

      level patterns, and their reasoning capabilities still require significant enhancement. Examples of

      hallucination can be seen in Appendix A.4.4).

463   5   H UMAN EVALUATION
464
465   We employ three computer science graduate students (our co-authors) to evaluate all the hypotheses
466   and plans generated by both the IDEA agent and human participants during the abduction, deduc-
467   tion, and induction stages within a randomly selected 50% subsample of the puzzles.
468   Abduction stage. During the abduction stage, agents formulate an initial hypothesis based on initial
469   observations. Since some puzzles are simple, agents can potentially guess the ground truth rule
470   at this stage. Figure 5(a) shows that LLMs such as GPT-4o have approximately a 30% success
471   rate in correctly identifying the rule during abduction. Notably, humans exhibit the lowest success
472   rate in this stage. We believe this is because humans often lack the patience to thoroughly review
473   all textual information initially and may overlook certain details. In contrast, LLMs meticulously
474   process every word of the prompt and leverage their comprehensive pre-training data to generate
475   plausible hypotheses. Additionally, humans tend not to formalize hypotheses when operating under
476
      uncertainty.
477   Deduction stage. After establishing an initial hypothesis—or each time the agent refines a hypothe-
478   sis—the agent derives a plan to either validate this hypothesis or use it to solve the puzzle. As shown
479   in Figure 5(b), humans generally outperform LLMs in creating high-quality plans. However, in the
480   Function Operator puzzles, Llama3-70B and GPT-4o perform better than humans, suggesting that
481   these models may have superior mathematical intuition compared to our human participants, who
482
      were undergraduate students from computer science-related majors.
483   Induction stage. Figure 5(c) illustrates the proportion of inductions where the refined hypothesis
484   improved over the previous one, which we define as the effective induction rate. Humans outper-
485   form LLMs by a significant margin in this process, with 40% of their refined hypotheses being better
      than the previous ones. Surprisingly, GPT-4o performs worse than Llama3-70B and GPT-3.5-Turbo.

531
      In this work, we introduce RULEARN, a benchmark designed to evaluate LLM agents’ rule-
532
      learning abilities in interactive environments. We propose IDEA, an agent framework that mimics
533
      human reasoning through abduction, deduction, and induction. Comprehensive experiments involv-
534   ing five prominent LLMs and human participants reveal that while IDEA significantly improves the
535   rule learning ability of LLM agents, there is still a large gap between LLM and humans particu-
536   larly in refining hypotheses and adapting strategies. Despite these advancements from the IDEA
537   framework, LLMs still face challenges in generating valid hypotheses and avoiding repetitive ac-
538   tions in complex scenarios. Our findings underscore the need for further development of LLMs
539   that can emulate human cognitive processes more effectively in explorations of novel environments.
      RULEARN provides a foundational resource for future research aimed at closing these gaps.

1113
1114   A.3.1     E NVIRONMENT E NTITIES
1115
1116           • Agent: Represents the entity focused on rule-learning and problem-solving, comprising
1117             the following components:
1118               * Goal (G): The objective of the agent, articulated in natural language.
1119
                   * Buffer Memories (M̃ := {m̃1 , m̃2 , . . . , m̃n }): This temporary storage holds all
1120                 newly generated information during the agent’s exploration, including actions taken,
1121                 outcomes of each action, and observations. This is where the most recent activities
1122                 are initially recorded.
1123               * Memories (M := {m1 , m2 , . . . , mn }): This permanent memory stores all signif-
1124                 icant observations and facts from the beginning of the task. When the agent forms
1125                 new assumptions and plans, the contents of the Buffer Memories are evaluated; non-
1126                 essential details like are discarded, while important facts and observations are trans-
1127                 ferred to the permanent Memories. This architecture ensures that each time the agent
1128                 revises its hypotheses, it can clearly distinguish which observations occurred under
1129                 the new assumptions and plan.
1130               * Hypotheses (H): The current hypotheses formulated by the agent to explain all the
1131                 observations, are expressed in natural language.
1132               * Plan (P): The current strategy devised by the agent to either test the correctness
1133                 of the existing hypotheses or to leverage these hypotheses to achieve the goal, also
                     represented in natural language.


                                                                 21
       Under review as a conference paper at ICLR 2025


1134
                    * Action Space (A): A set of potential actions available to the agent, determined by its
1135                  current hypotheses and plan. The Action Space is dynamic and can change in response
1136                  to interactions with the environment. For example, after investigating a fridge, the
1137                  agent gains the additional option to open the fridge and inspect its contents.
1138
1139
1140           • Objects (O): Represents all interactive entities within the environment that provide the
1141             agent with a means to interact with the world. A single object in this set is denoted as O.
1142
1143
                    * Description (Do ): A concise description of the object, detailing its nature and poten-
1144
                      tial uses, presented in natural language.
1145
1146                * Predefined interactive actions (OA := {ã1 , ã2 , . . . , ãn }): A set of actions that are
1147                  predefined for each object. Each action is described in natural language, explaining its
1148                  purpose. Additionally, each action is associated with a coded function that processes
1149                  the agent’s inputs and produces an effect, potentially altering the environment based
1150                  on these inputs.

       A.3.2     I NTERACTIVE F UNCTIONS
1154
1155           • Perceptual Action:= â(O): An action automatically added to the agent’s action space for
1156             all objects within the same scope. Upon perceiving an object, the agent gains the ability to
1157             interact more detailedly with it, adding its interactive actions to the S.
1158
1159
1160
               • Interactive Action:= ã(Do , G, H, P, I, M̃, M): A predefinec action that triggers a pre-
                 coded effect based on the agent’s input I. For example, in using a reactor, the agent decides
1161
                 the materials and their order of addition, and the reactor processes these inputs based on
1162
                 pre-coded rules to synthesize new materials.
1163
1164
1165           • Abductive Action:= ā(G, M̃): An action based on initial observations, allowing the agent
1166             to formulate the first hypotheses and generate a new plan.
1167
1168
1169           • Inductive Action:= ȧ(G, H, P, M̃, M): An action based on the current observations,
1170             goals, prior hypotheses, and previous plans, allowing the agent to refine hypotheses and
1171             generate new plans.
1172
1173
1174
               • Deductive Action := ä(G, H,A, M, M̃): An action based on the current memories, hy-
                 pothesis, and action space that generates a plan to either validate the current hypothesis or
1175
                 leverage it to solve problems.
1176
1177
1178           • Action select:=Fa (G, H, P,M̃, M, A) → a: A function where the agent selects an action
1179             from the action space, considering all gathered information.
1180

1182   With the definitions and entities described above, we can formalize our interactive, rule-learning
1183   process. The sequence begins with the agent selecting an action from the available action space.
1184   The agent then decides on an input based on the selected action. Once the action is executed,
1185   the environment responds by providing feedback to the agent. The outcome of this action results
1186   in changes to M̃, M, S, H, P and O, making the environment dynamic as the exploration process
1187   progresses. These changes reflect the agent’s interactions and adaptations to the evolving conditions
       within the environment.
