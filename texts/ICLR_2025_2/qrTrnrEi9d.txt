009                                                A BSTRACT
010
011            Large language models (LLMs) combined with instruction tuning have shown
012            significant progress in information extraction (IE) tasks, exhibiting strong general-
013            ization capabilities to unseen datasets by following annotation guidelines. However,
014            their applicability to low-resource languages remains limited due to lack of both
015
               labeled data for fine-tuning, and unlabeled text for pre-training. In this paper,
               we propose TransFusion, a framework in which models are fine-tuned to use En-
016
               glish translations of low-resource language data, enabling more precise predictions
017
               through annotation fusion. Based on TransFusion, we introduce GoLLIE-TF, a
018            cross-lingual instruction-tuned LLM for IE tasks, designed to close the perfor-
019            mance gap between high and low-resource languages. Our experiments across
020            twelve multilingual IE datasets spanning 50 languages demonstrate that GoLLIE-
021            TF achieves better cross-lingual transfer over the base model. In addition, we show
022            that TransFusion significantly improves low-resource language named entity recog-
023            nition when applied to proprietary models such as GPT-4 (+5 F1) with a prompting
024            approach, or fine-tuning different language models including decoder-only (+14
025            F1) and encoder-only (+13 F1) architectures.
026
027   1    I NTRODUCTION
028
      The task of information extraction (IE) is challenging due to fine-grained annotation guidelines
029
      for span-level annotations. Fortunately, recent advances in instruction-following large language
030
      models (LLM) (Ouyang et al., 2022; Gemini et al., 2023) such as GoLLIE (Sainz et al., 2024) have
031   demonstrated the ability to perform zero-shot IE without labels using annotation guidelines. However,
032   these models are often pre-trained on English-centric data (Touvron et al., 2023; Roziere et al., 2023).
033   Even state-of-the-art proprietary models such as GPT-4 exhibit significant performance degradation
034   from 80 English F1 to 55 F1 on low-resource African languages, as shown in Figure 1 (right).
035
036
      To improve NLP on low-resource languages, the research community has turned to machine translation
      to translate fine-tuning datasets (translate-train) and translate test data into high-resource languages
037
      for easier processing (translate-test) (Hu et al., 2020). Recent studies (Shi et al., 2022; Huang et al.,
038
      2023) on prompting LLMs with translated data have shown improvements on diverse tasks such
039   as math reasoning and summarization. Prior work has explored the use of machine translation to
040   improve multilingual instruction-following on traditional NLP benchmarks, such as natural language
041   inference, and sentiment analysis, however, the use of MT to improve instruction-following IE models
042   is less explored, as there is not a trivial alignment between labels in the native language and translated
043   texts (Ahuja et al., 2023). With recent efforts to develop machine translation (MT) models such as
044   M2M (Fan et al., 2021) and NLLB-200 (Costa-jussà et al., 2022) that better support low-resource
045   languages, we study how to teach LLMs to leverage an external MT system in a resource-efficient
046   manner to improve low-resource IE.
047
      In this paper, we propose a Translation and Fusion (TransFusion) framework, which aims to teach
048   models to use translation data from an external MT system to make better predictions. The framework
049   includes three steps: (1) translating low-resource data into English at inference time, to be annotated
050   by a high-resource model. Next, (2) these span-annotated English translations are combined with
051   low-resource language text in a fusion model that is trained to make predictions conditioned on both
052   types of data. Finally (3), the language model generates a TransFusion reasoning chain (annotate
053   and fuse) in a single autoregressive decoding pass. To train TransFusion models, we construct
      cross-lingual instruction fine-tuning data by translating and projecting labels from English IE datasets

081   2       BACKGROUND : A NNOTATION G UIDELINE F OLLOWING LLM S FOR IE
082
      In this paper, we employ the GoLLIE model (Sainz et al., 2024), which has been instruction-tuned on
083   English Information Extraction (IE) tasks using label schema guidelines, to achieve state-of-the-art
084   zero-shot IE on unseen datasets. GoLLIE utilizes a Python code representation for both inputs and
085   outputs, providing a clear and human-readable structure that unifies various IE annotation tasks. Each
086   label schema is encapsulated as a Python class object, with the annotation guidelines embedded as
087   strings within these objects (an example of a GoLLIE prompt is provided in the the Appendix in
088   Figure 6.
089
      Limitation of Cross-lingual Transferbilitiy: Despite GoLLIE’s impressive performance, it is
090   designed for use on English, as it is primarily fine-tuned on English data. This limitation is shown in
091   Figure 1 (right), where we see a significant drop in performance on low-resource African languages,
092   from 95 to 48, compared to English. In this study, we experiment with cross-lingual transfer, where
093   human-labeled data in the target languages are assumed to be unavailable. Collecting such data
094   is costly and time-inefficient, as it requires well-trained native language speakers. Recent efforts,
095   such as NLLB-200 (Costa-jussà et al., 2022), have focused on gathering low-resource translation
096   data to train multilingual MT models capable of translating across 200 languages. Building on this
097   progress, we explore whether an instruction-tuned information extraction model can learn to use an
098
      external translation model (Schick et al., 2024) to enhance performance on low-resource language IE
      tasks. This offers an efficient and effective alternative to computationally intensive pre-training based
099
      methods for adapting to new languages (Scao et al., 2022; Xue et al., 2021; Alabi et al., 2022; Üstün
100
      et al., 2024).

      Training a Decoder-only LM (GoLLIE-TF). To implement our TransFusion framework within the
163   instruction-following GoLLIE model, we represent the framework as natural language instructions,
164   providing the model with supplementary English translation text of the original target language
165   sentence, which is illustrated in Figure 1 (left). The TransFusion instruction specifies the output
166   format, guiding the model to first generate annotations for the English translation and subsequently
167   for the target language data, using the English annotations as context (an example can be found
168   in Appendix Figure 6 ). This autoregressive approach enables the model to perform the annotate
169   and fuse steps concurrently during inference. During training, we fine-tune the GoLLIE model to
170   adhere to these instructions, ensuring it generates annotations for both the English and target language
171
      data sequentially. We apply the next token prediction loss to the tokens following the TransFusion
      instruction. At inference time, x is the low-resource language and xtrans is the English translation:

196   Named Entity Recognition (NER) datasets from MasakhaNER2.0 (Adelani et al., 2022) (20 African

200

201
      ERD (Tedeschi & Navigli, 2022) (10 high-resource languages), MultiCoNER2 (12 high-resource
202

248   4.1 L ANGUAGE M ODELS AND BASELINES
249   Models: We adopt GoLLIE-7B as our primary starting checkpoint. GoLLIE is an instruction
250   fine-tuned version of CodeLLaMA (Roziere et al., 2023) that is trained on approximately 500,000
251   English demonstrations. Although the model was not explicitly pre-trained on multilingual data,
252   its pre-training corpus includes a substantial amount of high-resource language content, such as
253   Wikipedia, covering a diverse linguistic range (Touvron et al., 2023). This makes GoLLIE-7B an
254   appropriate testbed for examining the adaptation of English-centric LLMs to low-resource languages
255   that may be underrepresented in pre-training. In addition to this decoder-only LLM, we explore
256   encoder-only models specifically pre-trained on African languages, such as AfroXLM-R (Alabi et al.,
257
      2022) in Section 5.3.
258   Training Setup: Initilized from GoLLIE-7B, we continue fine-tuning the model on a dataset of
259   20,000 TransFusion training examples using QLoRA (Dettmers et al., 2024). QLoRA has been shown
260   to better maintain the base model’s performance (Biderman et al., 2024) and offers faster training
261   times compared to full fine-tuning. To implement this, we freeze the transformer model weights
262   and apply LoRA (Hu et al., 2021) to all linear layers within all the transformer blocks. We set the
263   LoRA rank to 128 and the alpha parameter to 16 based on preliminary experiments as we found
264
      smaller alpha leads to more stable training and higher rank for fast convergence. We use the AdamW
      optimizer (Kingma & Ba, 2015) with a batch size of 16 and a learning rate of 1e-4, managed by a
265
      cosine scheduler. The training process was conducted on a setup of 2 NVIDIA A40 GPUs, each
266
      equipped with 48GB of memory. The entire experiment session spanned approximately 6 hours. We
267   use greedy decoding at inference time.

529
530   We introduce TransFusion, a framework that bridges the performance gap between high and
531   low-resource languages in information extraction by leveraging machine translation. Our experi-
532
      ments demonstrate that TransFusion significantly improves the cross-lingual transfer capabilities
      of instruction-tuned LLMs, surpassing both proprietary models and encoder-only architectures on
533
      low-resource languages NER. This work demonstrates the potential of translation-based techniques
534
      to unlock the power of LLMs for a wider range of low-resource languages, paving the way for more
535   inclusive and equitable IE capabilities across diverse linguistic communities.

      B        L IMITATIONS AND BROADER IMPACT
908   The NER experiments conducted on GPT-4 have yielded promising results for low-resource languages.
909   However, concerns remain regarding potential data contamination resulting from the possibility that
910   GPT-4 was pre-trained or fine-tuned on the test data.1 The Translation-and-fusion framework, while
911   effective in enhancing cross-lingual transfer, does introduce additional inference costs during test time
912   inference. These additional steps include translation using an external MT system and annotation
913
      processes, which can contribute to an increased number of token generation. This is similar to
      chain-of-thought prompting or retrieval augmented generation, which uses additional computational
914
      cost at inference for better quality generation. Thus, practitioners should consider the trade-off
915
      between performance and efficiency when deciding to adopt the Translation-and-fusion approach.
