009                                                 A BSTRACT
010
011            Vision-Language Models (VLMs) have demonstrated strong performance in
012            single-image understanding, supported by many high-quality instruction datasets.
013            However, multi-image reasoning tasks remain under-explored in the open-source
014            community due to two major issues: (1) scaling up datasets with multiple cor-
015            related images and complex reasoning instructions is resource-intensive and dif-
016            ficult to maintain quality and (2) there is a shortage of robust multi-image eval-
017            uation benchmarks. To address these issues, we introduce SM I R, an efficient
018            synthetic data-generation pipeline for multi-image reasoning, and a high-quality
019
               SM I R dataset generated using this pipeline. Our pipeline efficiently extracts
               highly correlated images using multimodal embeddings, combining visual and
020
               descriptive information and leverages open-source LLMs to generate quality in-
021
               structions, offering a cost-effective alternative to expensive closed-source solu-
022            tions. Additionally, we present SM I R-B ENCH, a novel multi-image reasoning
023            evaluation benchmark comprising 100 diverse examples across 7 complex multi-
024            image reasoning tasks. Unlike existing benchmarks, SM I R-B ENCH is multi-turn
025            and allows for free-form responses, providing a more comprehensive evaluation
026            of model expressiveness and reasoning capability. We demonstrate the effective-
027            ness of SM I R dataset by fine-tuning several open-source VLMs and evaluating
028            their performance on SM I R-B ENCH. Our results show that models trained on our
029            dataset outperform baseline models in multi-image reasoning tasks. Furthermore,
030
               we observe enhanced model expressiveness and more nuanced reasoning in free-
               form responses, highlighting the value of our approach for advancing open-source
031
               VLM research. 1

      1    I NTRODUCTION
036
037   Vision-Language Models (VLMs) have shown impressive capabilities in tasks involving single im-
038
      ages, particularly open-source models that have benefited from high-quality instruction datasets
      (Laurençon et al., 2024b; Zhang et al., 2023; Xu et al., 2022). However, when it comes to multi-
039
      image tasks, such as comparing or analyzing relationships between multiple images, the perfor-
040
      mance of open-source VLMs (Liu et al., 2024b;a; Li et al., 2024a; Awadalla et al., 2023; Yao
041   et al., 2024; Wang et al., 2023b) lags significantly behind their closed-source counterparts in GPT-4
042   (Achiam et al., 2023), Claude 3.5 Sonnet (Anthropic, 2024a), Claude 3 (Anthropic, 2024b), and
043   Gemini 1.5 (Reid et al., 2024). One of the crucial problems is that constructing large-scale, compli-
044   cated multi-image reasoning datasets and evaluation benchmarks is challenging.
045
      First, collecting and curating large-scale multiple images with high correlations is hard. Identify-
046
      ing correlated semantic information or entities across images requires large-scale images and so-
047   phisticated algorithms. Thus, most existing multi-image instruction tuning datasets do not have
048   highly correlated images. For example, MANTIS (Jiang et al., 2024) often includes unrelated im-
049   ages within the same multi-image reasoning question, potentially undermining the complexity of
050   the task. MMDU-45K (Liu et al., 2024e) attempts to address this by clustering image captions.
051   MMInstruct (Liu et al., 2024d), on the other hand, only considers one image at a time, falling short
052

055   featuring related images within multi-image scenarios.
056
057   Second, scaling up the number of highly correlated images presents significant challenges. Existing
058
      datasets such as MANTIS, MMDU-45K, and MultiInstruct require extensive human curation and
      annotation, resulting in a labor-intensive and time-consuming process. To minimize human effort,

062   However, this method proves expensive and difficult to scale effectively.
063
064
      Third, evaluating multi-image reasoning is complicated. Given the increased complexity of multi-
      image reasoning tasks, using multi-turn free-response evaluation instead of the multiple-choice for-
065
      mat employed by previous benchmarks such as (Fu et al., 2024; Ying et al., 2024; Wang et al.,
066
      2024a; Yue et al., 2024; Singh et al., 2019; Hudson & Manning, 2019; Antol et al., 2015). Free-
067   response evaluations are more challenging, requiring models to articulate their thought processes,
068   providing insight into their reasoning abilities, and allowing for a more nuanced assessment of their
069   capabilities.
070
      To address these challenges, we propose a synthetic data generation pipeline, SM I R, for multi-image
071
      reasoning and a human-annotated evaluation benchmark for multi-image reasoning, SM I R-B ENCH.
072
      SM I R aims to generate correlated and challenging multi-image reasoning questions, while SM I R-
073   B ENCH evaluates models on free-response, difficult multi-image scenarios.
074
075   To summarize, we address these issues with our contributions:
076
            • Two novel sampling algorithms: Cluster Sampling for data quality robustness and Graph
077           Iteration Sampling for diversity. Both use multimodal embeddings (combining image and
078           caption) to group correlated images for challenging multi-image instruction tuning datasets.
079
            • A scalable synthetic multimodal data generation pipeline utilizing open-source LLMs such
080           as Meta Llama 3.1 70B Instruct Turbo (Dubey et al., 2024), eliminating the need for expen-
081           sive closed-source models, reducing costs by up to 50 times (Kirkovska, 2024) and speeds
082           up to by 10 times (Kirkovska, 2024), while significantly minimizing human annotation
083           efforts.
084         • A new multimodal evaluation benchmark with free-form responses, assessing both final
085           answers and reasoning processes in complex multi-image tasks. Using GPT-4-Turbo and
086           other open-source models as reference, we see up to 11% improvement with the SM I R
087           dataset.

089   2   R ELATED W ORKS
090
091   Vision Language Models We focus on instruction tuning Vision-Language Models (VLMs) that
092   utilize a pretrained Large Language Model (LLM) backbone because this approach is cost-effective
093   and more accessible for the open-source community. Since the backbone responsible for language
094   understanding is already trained, the overall training process becomes simpler and requires fewer
095   resources. Our primary task involves aligning the vision encoder—typically architectures like Vi-
096   sion Transformer (ViT) (Dosovitskiy, 2020), SigLIP (Zhai et al., 2023), or CLIP (Radford et al.,
097   2021)—with the LLM backbone. This alignment is facilitated through linear layers that connect
098
      the vision encoder to the backbone, enabling the integration of visual and textual information. For
      instance, BLIP-2 (Li et al., 2023b) uses OPT (Zhang et al., 2022) and FLAN-T5 (Chung et al., 2022)
099
      as backbones, MiniGPT-4 (Zhu et al., 2023) utilizes Vicuna (Chiang et al., 2023), and Qwen-2-VL
100
      (Wang et al., 2024b) employs Qwen-2-1.5B (Yang et al., 2024) as the language backbone. In this pa-
101   per, we focus on creating a high-quality multi-image reasoning dataset for instruction tuning instead
102   of large-scale interleaved pretraining datasets like OBELICS (Laurençon et al., 2024a), MINT-1T
103   (Awadalla et al., 2024), and LAION-5B (Schuhmann et al., 2022).
104

107   to improve reasoning capabilities in VLMs. However, these datasets have limitations in their ap-
      proaches. MANTIS randomly concatenates single image pairs from LLaVA-665k (Liu et al., 2024a),

184   3.1   M ULTIMODAL E MBEDDING C ONSTRUCTION
185
      To identify correlated images effectively, we developed a method that incorporates both visual and
186
      textual information from image-caption pairs. Our approach utilizes SigLIP and CLIP image em-
187
      beddings alongside corresponding caption embeddings. We formulated a multimodal embedding by
188   combining these components with a small constant, c, as follows:
189
190
191
                                     Emultimodal = Eimage + c · Ecaption                                (1)
192
      where Emultimodal is the multimodal embedding, Eimage is the image embedding, and Ecaption
193   is the caption embedding. For the ShareGPT4V (Chen et al., 2023), a c = 0.2 worked well, but
194   this parameter may vary depending on the quality of individual image-caption pairs in other data
195   sources.
196
      Importantly, relying solely on either image or caption information would limit our ability to
197
      concurrently consider both visual and textual contexts, which is crucial for establishing a
198
      comprehensive understanding of the images. This multimodal approach enables us to capture
199   the nuanced relationships between visual content and its associated descriptive text, thereby enhanc-
200   ing our capacity to identify and group correlated images effectively.
201
202
      Following the generation of multimodal embeddings, we employed UMAP (McInnes et al., 2018) to
      reduce the dimensionality of the vectors. This technique allowed us to project the high-dimensional
203
      embeddings into a lower-dimensional space, facilitating more efficient analysis and visualization of
204
      the data while preserving its essential structure.

      3.2   G ROUPING I MAGES
207
208   We present two novel algorithms designed to group correlated images prior to leveraging an open-
209   source Large Language Model (LLM) for synthetic data generation in multi-image reasoning tasks.
210   The emphasis on correlated images is crucial, as it facilitates challenging multi-image reasoning
211   scenarios. These scenarios require the model to identify intricate relationships and differentiate
212   between visually similar scenes, thus enhancing the complexity and realism of the reasoning process.
213
214   Clustering We employed HDBSCAN (Malzer & Baum, 2020), a density-based clustering algo-
215   rithm, to group the SigLIP and CLIP multimodal embeddings into coherent clusters. To establish
      meaningful relationships between the two embedding spaces, we developed a greedy algorithm that

296   In our data generation approach, we focused on creating two distinct types of questions: shorter,

305   4.3   DATA S OURCE
306
      Our study leveraged the ShareGPT4V (Chen et al., 2023) dataset as the primary source for generating
307
      synthetic examples. This comprehensive dataset comprises of better image-caption pairs derived
308   from LLaVA-Instruct (Liu et al., 2024a) and COCO (Lin et al., 2014). To maintain diversity from the
309   data source, we synthetically generated 5,000 data points from each 20,000-image batch, resulting
310   in a total of 160,000 synthetic examples. SM I R pipeline can also be applied easily to other data
311   resources in the future.

426   6   C ONCLUSION
427
428   This paper introduces a synthetic data pipeline designed to enhance multi-image reasoning capabil-
429   ities on open-source VLMs. By leveraging multimodal embeddings and grouping algorithms, the
430   pipeline generates high-quality synthetic multi-image reasoning instruction tuning data. The ap-
431   proach yields up to 11% improvement on SM I R-B ENCH for popular open-source models, demon-
      strating the significant potential of synthetic data in advancing open-source VLM models.

432
      Limitations Our methods have several limitations. Random sampling with iteration is time-
433   intensive due to the need for recalculating distance embeddings for each new image sampled. Fur-
434   ther investigation is needed to determine the scalability of our synthetic data. Future research should
435   focus on developing more time-efficient algorithms and optimizing data mixtures.
436
437   Broader Impact This paper introduces a method for generating high-quality, cost-effective data
438   for VLMs, addressing the growing challenge of data scarcity. By advancing these open-source tech-
439   niques, we contribute to narrowing the performance gap between open and closed-source models,
440   promoting more accessible and powerful multimodal AI.

      A.1   G REEDY C LUSTER M ATCHING A LGORITHM
651
652   We present the pseudocode for the Greedy Cluster Matching in Algorithm 1.
653
      Let CS = {S1 , ..., Sm } and CC = {C1 , ..., Cn } be cluster sets from SigLIP and CLIP embeddings
654
      respectively. The algorithm proceeds as follows:
655
656   1. Select the largest cluster from either set: Xmax = arg maxX∈CS ∪CC |X|
657   2. If Xmax ∈ CS , find the best match in CC : Ybest = arg maxCj ∈CC score(Xmax , Cj )
658
659
      3. If Xmax ∈ CC , find the best match in CS : Ybest = arg maxSi ∈CS score(Xmax , Si )
660   Where the score function is defined as:
661
                                                          |A ∩ B|
662                                         score(A, B) = |A|+|B|
663                                                              2
664
665   This process is repeated, greedily selecting the largest remaining cluster and finding its best match,
666
      until all clusters are matched or one set is exhausted.
667
      Algorithm 1 Greedy Cluster Matching Algorithm
668
669   Require: Two lists of clusters c1 and c2
670
      Ensure: List of matched cluster pairs
       1: c1 ← sort(c1, key = len, reverse = True)
671
       2: c2 ← sort(c2, key = len, reverse = True)
672
       3: matched pairs ← []
673    4: num samples ← 0
674    5: while c1 is not empty and c2 is not empty do
675    6:     if len(c1[0]) ≥ len(c2[0]) then
676    7:          larger cluster ← c1.pop(0)
677    8:          smaller list ← c2
678    9:     else
679   10:          larger cluster ← c2.pop(0)
680   11:          smaller list ← c1
681
      12:     end if
      13:     best match ← None
682
      14:     best score ← −1
683
      15:     for i, cluster in enumerate(smaller list) do
684   16:          overlap ← len(set(larger cluster) ∩ set(cluster))
685   17:          avg size ← (len(larger cluster) + len(cluster))/2
686   18:          score ← overlap/avg size
687   19:          if score > best score then
688   20:               best score ← score
689   21:               best match ← (i, cluster)
690   22:          end if
691   23:     end for
692
      24:     if best match is not None then
      25:          best index, best cluster ← best match

      27:          matched pairs.append(union)

697   30:     end if
