009                                               A BSTRACT
010
011            Pre-trained Vision-Language Models (VLMs) are becoming increasingly popular
012            across various visual tasks, and several open-sourced VLM variants have been
013            released. However, selecting the best-performing pre-trained VLM for a specific
014            downstream task is challenging since no single VLM can achieve promising per-
015            formance on all downstream tasks, and evaluating all available VLMs is impossi-
016            ble due to time and data limitations. To address this problem, this paper proposes a
017            novel paradigm to select and reuse VLM for downstream tasks, called Model La-
018
               bel Learning (MLL). The proposal contains three key modules: model labeling,
               which assigns labels to each VLM to describe their specialty and utility; model
019
               selection, which matches the requirements of the target task with model labels;
020
               and model reuse, which applies selected VLMs to the target task in an ensem-
021            ble manner. The proposal is highly computationally efficient and growable since
022            the model labeling process is completed target task independent and the ability
023            could grow with the number of candidate VLMs. We also introduce a new bench-
024            mark for evaluating VLM selection methods, including 49 VLMs and 17 target
025            task datasets. Experimental results clearly demonstrate the effectiveness of the
026            proposed method for selecting and reusing VLMs.

029   1   I NTRODUCTION
030
031   Vision-Language Models (VLMs), such as CLIP (Radford et al., 2021), ALIGN (Jia et al., 2021),
032   etc, which are pre-trained on large-scale image-text datasets, have recently attracted significant at-
033
      tention due to their remarkable zero-shot prediction capabilities on visual tasks. However, though
      VLM shows impressive general ability, as highlighted in Radford et al. (2021), VLMs often fall
034
      short of supervised expert models in many downstream tasks. To address this limitation, numerous
035
      studies (Dosovitskiy et al., 2021; Yu et al., 2022; Fang et al., 2023) have sought to enhance the zero-
036   shot performance of VLMs by studying model architectures, pre-training datasets, and training/fine-
037   tuning methods. This effort has led to the development of many open-source pre-trained VLMs with
038   diverse structures and parameters, contributing to VLM model hubs like open-clip (Ilharco et al.,
039   2021), which currently hosts more than 100 pre-trained VLMs.
040
      As more and more VLMs are open-sourced, the problem of how to select a VLM to reuse for
041
      specific downstream tasks naturally occurs. Although we can directly utilize the best-performing
042   model on a universal dataset such as ImageNet, previous work (Fang et al., 2022) has shown that
043   the performance of VLMs can vary greatly depending on dataset domain. For example, we evaluate
044   the performance of various pre-trained VLMs in the open-clip library across several downstream
045   tasks (1(a)) and within different classes of a specific task (1(b)). Figure 1(a) reveals that each VLM
046   demonstrates distinct strengths in zero-shot visual tasks, with no single model outperforming all
047   others across every task. Interestingly, models that perform worse on general tasks can sometimes
048   surpass stronger models in specific downstream tasks. Furthermore, even in the same task, different
049   VLMs exhibit varying levels of performance across specific classes, as illustrated in Figure 1(b).
050   Therefore, it is important to design VLM selection methods, and it would be better if we could
051   achieve more fine-grained selection, i.e., select different VLMs to handle different classes. The
052   direct way to select a model is to evaluate all candidate models’ performance on the target task.
053   However, it is unrealistic due to time and computational resource limitations. Additionally, previous
      works on model selection (Tran et al., 2019; You et al., 2021) primarily focus on single-modal

      to focus on VLM selection, proposing to evaluate VLM performance using textual information.
      However, their selection strategy heavily depends on the models’ ground-truth performance on large-

078
      specific tasks, selection strategy effectiveness drops, as shown later in our experiments.
079
080   To this end, we introduce a novel paradigm to select and reuse VLMs called Model Label Learning
081   (MLL). The core idea is to organize candidate pre-trained VLMs into a model hub and describe
082
      the specialty and utility of each VLM as the model’s label in some manner. When facing a new
      downstream task, we can match the task requirements with the model labels to select and reuse
083
      models. Specifically, the proposal contains three key interconnected modules: model labeling, model
084
      selection, and model reuse. In the model labeling process, we construct a semantic graph with
085   commonly occurring visual concepts and representative samples, and each model undergoes pre-
086   testing on the semantic graph to generate its model label, which describes its capability on these
087   semantic classes. In the model selection process, we generate caption descriptions for both the
088   nodes in the semantic graph and the categories to be classified in the target task to compare their
089   similarity. This enables us to evaluate the model’s performance on the target classes by aligning the
090   matched semantic nodes with the model labels. In the model reuse process, we apply an ensemble
091   strategy that combines the selected models’ predictions on a single class and chooses the highest
092   confidence across all classes as the final prediction.
093   The model labeling process is completed immediately when the candidate VLM is added to the
094   model hub, therefore, it is target task independent, which means the proposal is both data and com-
095   putationally efficient in the model selection process. Moreover, the proposal is highly growable
096   since the capability could grow with the number of candidate models in the model hub and the
097   model labels are also scalable since more semantic nodes can be added continually. Moreover, we
098   introduce a comprehensive benchmark for evaluating VLM selection methods, aiming to facilitate
099
      related research. The benchmark includes 49 pre-trained VLMs and 17 target datasets as down-
      stream tasks. The ground-truth model ranking for each target task is provided for evaluation. We
100
      construct a semantic graph that contains more than 9000 commonly used visual concepts to pre-
101
      test each VLM. The experiments conducted demonstrate the effectiveness of our approach in both
102   selecting and reusing VLMs, while also validating the scalability of the model hub.
103
104   In summary, our contributions are as follows:
105
106        1. We highlight that the performance of pre-trained VLM varies across different downstream
107           tasks and even among classes within the same task. Therefore, it is important to study the
              VLM selection problem which is usually neglected by related researchers.

      2.1   V ISION -L ANGUAGE M ODEL
119
120   In recent years, there have been significant advances in the field of Vision-Language Models
121   (VLMs), including notable models such as CLIP (Radford et al., 2021), ALIGN (Jia et al., 2021),
122   BLIP (Li et al., 2022), etc. These models leverage large-scale datasets containing image-text pairs,
123   such as WIT (Srinivasan et al., 2021), to align visual and text features within a shared embedding
124   space, which has led to impressive capabilities in feature extraction, particularly in the realm of
125   zero-shot visual tasks. Tremendous works (Dosovitskiy et al., 2021; Yu et al., 2022; Fang et al.,
126   2023) attempted to improve the zero-shot capabilities of VLMs by focusing on model architecture,
127   pre-training datasets, and training/fine-tuning methods, which lead to the emergence of numerous
128
      open-source pre-trained VLMs. As a result, several VLM model hubs are constructed, such as open-
      clip (Ilharco et al., 2021) and HuggingFace (Wolf et al., 2020), which provide access to numerous
129
      VLMs. However, these model hubs lack effective model selection mechanisms; users can only select
130
      models based on some quantitative indicators, such as download volume, popularity, etc.
131
132
      2.2   M ODEL S ELECTION
133
134   As pre-trained models become increasingly diverse, how to select appropriate pre-trained models
135   to tackle specific tasks has become a significant challenge. Many researchers have started to focus
136   on this aspect. For example, Negative Conditional Entropy (NCE) (Tran et al., 2019) proposes an
137   information-theoretic quantity to learn the transferability and hardness between classification tasks;
138   LEEP (Nguyen et al., 2020) utilizes source prediction probabilities instead of hard labels compared
139   with NCE; LogME (You et al., 2021) estimates the correlation between source model features and
140
      the target outputs by maximum evidence; MetaGL (Park et al., 2023) solves the model selection
      problem on graph data by introducing a meta-learning method; EMMS (Meng et al., 2023) uses
141
      weighted linear regression to estimate the transferability of candidate models; Model Spider (Zhang
142
      et al., 2024) uses a re-ranking mechanism to enhance the task-model co-embedding. Although these
143   methods achieve well-performing in different settings, most of them focus on single-modal which
144   cannot be directly used for VLM selection. Moreover, the training data for VLM is inaccessible,
145   which introduces more challenges. Model selection for VLM is still a relatively new topic. LOVM
146   (Zohar et al., 2023) uses a text dataset to describe the prediction task to train a linear model to predict
147   the performance of the VLM. However, this method can only exploit text information and becomes
148   less effective when there is a domain shift between the downstream tasks and the training tasks.
149
150   2.3   L EARNWARE
151
152   Learnware (Zhou & Tan, 2022) is a novel paradigm that explores more effective model selection
153
      by constructing specifications to describe the capabilities of the model, closely aligning with our
      idea of model labeling. Compared with previous selection methods, learnware enables scalable and
154
      efficient model selection across diverse architectures and input types within a unified framework,
155
      improving as the system expands. Model specification is central to the learnware paradigm. Re-
156   cent works (Tan et al., 2024) on learnware paradigm are built on Reduced Kernel Mean Embedding
157   (RKME) (Wu et al., 2021), which maps training data distributions to points in Reproducing Kernel
158   Hilbert Space (RKHS) and identifies models by comparing similarities in the RKHS. Furthermore,
159   Guo et al. (2023) enhanced RKME for heterogeneous label spaces, while Tan et al. (2023) addressed
160   challenges in heterogeneous feature spaces. However, learnware requires training data to construct
161   specifications. Considering the scale of VLM pre-trained datasets, it is unrealistic to construct spec-
      ifications for learnware to select models due to limited time and computational resources.

181   3.2   P ROBLEM S ETUP
182                                                                          M
      Assume the model hub has M pre-trained VLMs {fm = {Im , Tm }}m=1 , where Im and Tm denote
183   image encoder and text encoder of the VLM fm . There are two stages in our setting: the submission
184   stage for developers to upload models and the identification stage for users to select models.
185
186
      In the submission stage, the model developer submits a VLM fm to the model hub, and the model
      hub assigns a label Sm to the model to describe its specialty and utility. It is particularly emphasized
187
      that uploaded models are anonymous, meaning we do not have access to their training data.
188
189   In the identification stage, the user attempts to select VLMs from the model hub for the zero-shot
190   downstream task T , by uploading general information about the task, such as classes, domain type,
191   and task type, to describe their requirements. We subsequently utilize this information to select and
192
      reuse suitable VLMs, based on the model labels established in the submission stage.
193   The two main problems in our settings are: 1) In the submission stage, how can we design a label
194   to that fully characterize the capabilities of the submitted VLM? 2) In the identification stage, how
195   can we select and reuse appropriate VLMs from the model hub to address users’ downstream tasks
196   based on their requirements and the model labels?

200   4.1   F RAMEWORK
201
202   As illustrated in Figure 2, the MLL paradigm consists of three key modules: model labeling, model
203   selection, and model reuse. In the model labeling process, MLL constructs a semantic graph G with
204   commonly occurring visual concepts and representative samples as the evaluation datasets. When
205
      models are submitted to the model hub, they are pre-tested on the semantic graph and assigned
      labels Sm , which describe their capability on these semantic classes. In the model selection process,
206
      we generate caption descriptions for both the nodes in the semantic graph and the categories in the
207
      target task to compare their similarity. This enables us to evaluate the model’s performance on the
208   target classes by aligning the matched semantic nodes with the model labels. In the model reuse
209   process, we apply an ensemble strategy that combines the selected models’ predictions on a single
210   class and chooses the highest confidence across all classes as the final prediction.
211

408   5.2     E XPERIMENT S ETUP
409
410   Semantic Graph Construction. We construct a semantic graph G containing 9055 nodes using
411   the WordNet synsets, which contains a wide range of items, such as animals, tools, clothing, vehi-
412
      cles, plants, and more. Each node is represented by up to 75 randomly selected images from the
      sample datasets, reflecting the distribution of the node’s concepts. We use OpenAI text-embedding-
413
      3-large model to obtain caption embeddings of semantic graph nodes and downstream task class
414
      nodes, we then match the similar node between them by cosine similarity between the embeddings.
415
416
417
      Compared Methods. Initially, we compare our proposal with ImageNet Baseline (INB), which
      employs the performance of VLMs on the ImageNet to select which model to reuse. Additionally,
418
      we compare it with a VLM selection method called ModelGPT (Zohar et al., 2023). ModelGPT
419
      employs generated captions and synonyms for target task classes as substitutes for images of those
420   classes, then evaluates the performance of VLMs by measuring their ability to correctly classify
421   the captions and synonyms into their corresponding classes, which serves as the reuse metric in
422   combination with INB. A linear model is then learned between the reuse metric and ground-truth
423   performance on training downstream tasks. Finally, the zero-shot ability of VLMs on the target task
424   is predicted using this linear model and the reuse metric.
425
426   Implementation Details. We adopt the official code to implement ModelGPT. For a fair com-
427   parison, the experiment utilizes the ground-truth performance of VLMs on Sample Datasets for
428   ModelGPT to train its linear model, and then evaluate it on the benchmark. For both INB and
429   ModelGPT, the experiment selects the model with the highest predictive performance given by the
430   method for reuse in the target task. Specifically, we employ the same prompting strategy outlined
431   in the work of Radford et al. (2021), which uses the prompt “a photo of {class}”, where “{class}”
      is replaced by the task class. All selected models are utilized without any further fine-tuning, given

      that all downstream tasks are conducted in a zero-shot manner. Additionally, the weight α for model
433   selection in our setting is set to 0.7. All experiments are conducted on NVIDIA A800 GPUs.

437   Zero-shot Performance In our experimental setup, the goal is to optimize the performance of
438   VLMs on downstream zero-shot visual tasks. Therefore, in Table 1, we compare the performance
439   of different model selection methods across 17 benchmark datasets. We set two values for the
440   count k of reused models, specifically 1 and 3, to test the effects of using a single model versus an
441   ensemble of three models per class. The results show that our method achieves high performance on
442   most downstream tasks. ModelGPT largely aligns with INB, indicating a strong correlation in
443
      their selection strategies. When INB fails to select a well-performing model, ModelGPT also
      struggles with selection. By comparing different counts k of reused models, MLL demonstrates
444
      that reusing the model with the best performance per class is often sufficient to outperform baseline
445
      methods in most downstream tasks, highlighting the practicality of the MLL paradigm. We also find
446   that in datasets with a limited number of classes, such as PCam and MNIST, employing a single
447   model for each class tends to yield better results. Additionally, when the models available in the
448   model hub are generally weak, as seen in several datasets, such as CLEVR-D and DMLab, relying
449   on ensemble methods may introduce more noise than benefit. In these cases, a single model per
450   class often provides the ultimate balance between simplicity and effectiveness.

468   6     C ONCLUSION
469
470   In this paper, we explore how to select and reuse pre-trained VLMs for a specific downstream task.
471   To the best of our knowledge, this problem has been rarely studied. To address this, we propose
472   a novel paradigm called Model Label Learning (MLL) that assigns each VLM a label to describe
473   its utility on representative visual concepts. The MLL paradigm contains three key modules: model
474   labeling, model selection, and model reuse. The proposal is highly efficient, scalable, and convenient
475
      for both model developers and users. Moreover, we introduced a benchmark for evaluating pre-
      trained VLM selection and reuse methods that contain 49 pre-trained VLMs and 17 target datasets,
476
      with ground-truth ranking for each target task. Experiments demonstrate the proposal can achieve
477
      state-of-the-art model selection performance for VLMs and the ability to deal with downstream tasks
478   could grow with the scale of the model hub, showing the potential of building large model hubs with
479   advanced model selection mechanisms.
480
481
      In future work, we will endeavor to develop a novel model hub based on the MLL paradigm pre-
      sented in this paper, allowing valid VLM developers from all over the world to submit their models.
482
      When users work on visual classification tasks, they will be able to select and reuse models from
483
      the hub. The limitation of this paper is that the current implementation focuses solely on VLMs and
484   visual classification tasks. We will further attempt to extend our paradigm to more model types that
485   have significant architectural differences compared with VLMs, and more complex tasks.
