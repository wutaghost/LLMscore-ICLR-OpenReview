011                                               A BSTRACT
012
013            Skill discovery methods equip an agent with diverse skills necessary for solving
014            challenging tasks through an unsupervised learning manner. However, making
015            the pre-learned skills expandable for new tasks remains a challenge in existing
016
               research. To handle this limitation, we propose a scalable skill discovery algo-
               rithm, a playbook, which can accommodate unseen tasks by training new skills
017
               while maintaining previously learned ones. The playbook, characterized by dis-
018
               crete skills and an extendable structure, enables the extension of the skill set to
019            cover new datasets. Since we design the playbook to have a finite number of
020            skills, we can interpret a decision-making problem as a sequential skill classifica-
021            tion problem, so we aim to learn additional skills of the playbook by applying the
022            techniques of class-incremental learning. In addition, we also introduce skill plan-
023            ning schemes that can leverage both previously and newly learned skills to solve
024            challenging tasks compounded by multiple sub-tasks. The proposed method is
025            evaluated in the complex robotic manipulation benchmarks, and the results show
026            that the playbook outperforms existing state-of-the-art methods that learn contin-
027
               uous skills.

      1   I NTRODUCTION
031
032   Recent studies on skill discovery have successfully addressed challenging decision-making tasks
033
      such as maze navigation (Pertsch et al., 2020; Shi et al., 2022; Kim et al., 2023), locomotion (Sharma
      et al., 2020; Kim et al., 2021), and robotic manipulation (Ajay et al., 2021; Hong et al., 2024). Skill
034
      discovery, a hierarchical policy learning method, equips an agent with the skills necessary to solve
035
      complex tasks by identifying and acquiring useful and diverse skills through an unsupervised learn-
036   ing manner. These methods learn the skill space or skill set by embedding sampled trajectories from
037   task-agnostic datasets, which are collected by actively exploring the environment (Jiang et al., 2022;
038   Mazzaglia et al., 2023) or pre-collected using a behavior policy (Gupta et al., 2019; Lynch et al.,
039   2019; Rosete-Beas et al., 2022). The discovered skills can be leveraged for solving downstream
040   tasks, e.g., reaching goals or maximizing rewards designed for a specific task.
041
      In order to apply skill discovery methods to more general and everyday tasks, the learned skill space
042   or skill set must be scalable. For example, a cooking robot in a kitchen should be able to learn
043   additional recipes using new cooking tools or ingredients. However, there is a lack of research
044   on learning new skills and expanding an available skill set for unseen tasks. Existing studies such
045   as Eysenbach et al. (2019); Lee et al. (2020); Peng et al. (2019); Laskin et al. (2022); Park et al.
046   (2022; 2023) also solve downstream tasks using pre-acquired skills, while they often struggle to
047   solve entirely unseen tasks because their learned skill space cannot be expanded. To address this
048   issue, we propose a playbook, a novel algorithm with a scalable structure that allows us to add skills
049   for new tasks while maintaining previously learned skills.
050   If the skills are discrete, we can interpret a goal-conditioned decision-making problem as a sequential
051   skill classification problem. Also, adding skills implies increasing the number of classes the agent
052   can select. From this perspective, the main idea of the playbook is to extend the finite skill set
053   by applying the techniques of class-incremental learning for image classification. To do this, we
      design the playbook to select a skill based on current and goal states. As a result, the playbook


                                                        1

054
      can be extended to accommodate new tasks by learning additional skills through class-incremental
055   learning. To mitigate the problem of losing previous skills when learning new skills, which is called
056   catastrophic forgetting, we utilize the gradient boosting method of Wang et al. (2022), which fixes
057   previous skills while training new skills. By using the extended playbook, we can solve compounded
058   problems, which are a mixture of old and new tasks. For instance, if we have a pre-trained playbook
059   that can open a drawer and extend it for the new task of picking up a block, an extended playbook
060   can pick up the block in the closed drawer.
061   The playbook focuses on training a set of discrete skills. However, it is challenging to express multi-
062   modal behavior distributions with only a finite number of skills. To solve this issue, the playbook
063   utilizes the MCP (Peng et al., 2019) structure. MCP has several behavior primitives, each of which
064   represents an independent action probability distribution. MCP generates a wide range of behaviors
065   by combining primitives with a weight vector. We let the playbook learn finite skills, which are used
066   as weight vectors of MCP and primitives. Then, we can extend the playbook by adding new skills
067   and primitives to increase its expressive power over the raw action space for solving unseen tasks.
068   Our primary contribution is to propose a scalable skill discovery method that can accommodate new
069   tasks by expanding the discrete skill set. The playbook has the following strengths: 1) The playbook
070   covers multi-modal behavior distributions with a small number of skills. We have experimentally
071   verified that the playbook with a finite number of skills shows better performance than existing
072   baselines that train the continuous skill space. Specifically, on the CALVIN benchmark (Mees et al.,
073   2022), the playbook achieves a success rate of 21.4% for challenging robotic problems that require
074   an agent to decide and perform several hidden tasks. The existing state-of-the-art methods hardly
075
      solve these problems (success rate of 1.3% or less). 2) The playbook can be extended to adopt
      new skills. We also have experimentally verified that when new datasets are provided on CALVIN,
076
      the playbook successfully addresses new tasks included in datasets through structural extension.
077
      3) The extended playbook can solve compounded problems by mixing skills learned from different
078   datasets. We have verified that the extended playbook records a success rate of 24.4% for challenging
079   compounded problems on CALVIN. The source code is provided in the supplementary material.

084   2.1   H IERARCHICAL P OLICY L EARNING U SING AN O FFLINE TASK -AGNOSTIC DATASET
085
      Recent research on skill discovery has successfully solved long-horizon tasks by acquiring skills
086
      from an offline task-agnostic dataset (Lynch et al., 2019; Singh et al., 2021; Hakhamaneshi et al.,
087   2022). These studies generally aim to solve intricate tasks by deploying skills and address down-
088   stream tasks by reusing or fine-tuning previously learned skills. Existing methods learn the skill
089   space by encoding actions (Pertsch et al., 2020), states (Gupta et al., 2019), and state-action pairs
090   (Ajay et al., 2021; Shi et al., 2022) of the dataset and generate raw actions using a skill-conditioned
091   policy. On the other hand, the playbook trains a set of skills by embedding a state and an action
092   sequence and uses the skill as a weight vector of the MCP structure rather than a direct input of the
093   policy. Skill discovery studies such as Mazzaglia et al. (2023); Ju et al. (2024) train discrete skills
094   to represent multi-modal behaviors of offline datasets, but they do not expand the skill set for new
095
      tasks. In contrast, the playbook extends their skills and structure to cover completely new tasks.

127   3     L EARNING P LAYBOOK FROM U NSTRUCTURED DATASETS
128
129   In this section, we propose a novel scalable skill discovery method, a playbook, which aims to
130   learn a finite number of skills capable of representing multi-modal behavior distributions included
131   in unstructured datasets consisting only of states and actions without any task description. In this
132   paper, we refer to a skill of the playbook as a play. As show in Figure 1, the playbook utilizes a
133   state-action embedding model to select a play from the set of N plays according to a given state
134   and an action sequence. The selected play becomes the weights of M primitives to let the weighted
135   combination of primitives represent the given raw action sequence. We design the playbook as a
136
      structure of individual and independent components (i.e., multiple plays and primitives) to facilitate
      structural extension. Consequently, the playbook can improve the expressive power by increasing
137
      the number of plays and primitives it owns. Section 3 introduces the playbook structure and explains
138
      how the playbook is trained. Section 4 presents the process of extending a playbook through class-
139   incremental learning. Finally, Section 5 describes the play plan method for reaching the given goal
140   state using a trained playbook.

223   4     P LAYBOOK E XTENSION BY C LASS -I NCREMENTAL L EARNING
224
225   When a new dataset is given, we can accommodate it efficiently by reusing existing skills while
226   training new skills. For example, reusing the skill that reaches a specific position or object can be
227   helpful because it is frequently performed for various manipulation tasks. Therefore, we aim to
228
      extend the playbook by adding new skills while reusing previously learned ones.
229   We assume that new datasets for the playbook extension are sequentially given. Due to memory
230   limitations, we cannot store all previous data, so the dataset used for training is left with only a
231   small amount. Then, we extend the playbook by adding new plays and primitives to accommodate
232   both the remaining dataset and the given new dataset. As a result, the extended playbook owns
233   diverse plays learned from different datasets. Finally, we focus on solving compounded problems,
234
      which are a mixture of old and new tasks, using an extended playbook.
235
236   4.1   C ONTINUAL P LAY L EARNING FOR N EW DATASET
237
238   Since the play is a discrete variable, we can interpret a goal-conditioned RL problem using a play-
239   book as a sequential play classification problem that selects play indices. Therefore, extending
240   a playbook can be considered as a case of class-incremental learning problem. However, when
241
      performing class-incremental learning, we can face the problem of losing previously learned knowl-
      edge, which is called catastrophic forgetting. To extend the playbook while mitigating catastrophic
242
      forgetting, we apply the gradient boosting technique for a class-incremental learning method in-
243
      spired by FOSTER (Wang et al., 2022). When the new dataset is given, FOSTER fixes previously
244   trained models and adds new parameterized models to cover the new dataset. FOSTER aims to train
245   additional models for the new dataset while maintaining the output of the original model for the
246   remaining dataset. After training additional models, FOSTER compresses the entire model grown
247   to the original model size through knowledge distillation. Since the state-action embedding model
248   of the playbook selects plays, we perform class-incremental learning for the embedding model.
249
      We extend the playbook in the following order. First, we freeze a pre-trained playbook consisting of
250   a state-action embedding model and multiple plays and primitives. Next, we add a new parameter-
251   ized embedding model and the fixed number of plays and primitives. Then, we define an extended
252   embedding model that adds the output of the new and existing embedding models. Finally, the ex-
253   tended embedding model and the added plays and primitives are trained to minimize loss (3) for
254   the new dataset. By training newly added plays and primitives, the expressive power of the play-
255   book over the raw action space can be improved. Since we have the fixed original playbook, the
256   extended embedding model can choose previously learned plays or train new plays when learning
257   new tasks. After training the extended model, the embedding model is reduced to its original net-
258
      work size through knowledge distillation of FOSTER. In summary, when the playbook extension is
      completed, the size of the embedding model is maintained, and the number of plays and primitives
259
      increases. We refer to the above process of extending a playbook as a continual play learning.

263
264   In this section, we explain the process of the play plan inference to reach a given goal state using
265   a trained playbook. In general, all planning methods applicable in the discrete action space can
266   be utilized for the playbook. In this paper, we propose two sampling-based methods, beam search
267   and Monte Carlo tree search (MCTS) planners. The beam search planner is suitable for a playbook
268   trained with a single dataset, and the MCTS planner addresses an extended playbook. These two
269   planners have a rollout step and a selection step, as shown in Figure 3. First, in the rollout step, we
      imagine several future state-play sequences using the trajectory generation model ∆. Next, in the


                                                          5
      Under review as a conference paper at ICLR 2025


270
      selection step, we select the best sequence that has reached the closest to the given goal state among
271   the imagined sequences using the distance metric Ψ. We denote a planning set by {∆, Ψ}.
272

274
      the original dataset consisting of state-action se-
      quences into state-play sequences using a trained

      playbook. With this dataset transformation, we

      gain two advantages. First, unlike raw actions,
277   play is a discrete variable, so the complexity of
278   state-play trajectories is reduced. Second, since
279   action sequences are compressed into plays, the
280   length of the state-play sequence is reduced.
281   Therefore, as the number of inference steps re-

287   5.1   P LAY P LAN G ENERATION THROUGH B EAM S EARCH
288
      We describe the beam search planner for inferring play sequences to reach a given goal state using
289
      a playbook, which is trained with a single dataset. In the rollout step of beam search, we use TT
290
      (Janner et al., 2021) as a trajectory generation model ∆. The original TT learns the conditional
291   probability distribution of state-action-reward sequences from the offline dataset. However, we only
292   model state-play sequences because the task-agnostic dataset we use has no rewards. By utilizing
293   TT, we obtain diverse and reasonable future state-play sequences conditioned on the given state.
294
      In the selection step of beam search, we choose the best play sequence among the generated se-
295
      quences. To this end, the playbook measures the dynamical distance in the state space between the
296
      last state of each plan and a given goal state. In goal-conditioned RL, the agent estimates Q-value,
297   Q(s, sg , a), which indicates the discounted sum of rewards that can be obtained in the future if ac-
298   tion a is performed in the current state s given the goal state sg . If we use a sparse reward function
299   for achieving goals, we can consider that a state with a higher Q-value reaches the goal in fewer time
300   steps. Therefore, we use the Q-value estimated by the goal-conditioned offline RL algorithm with
301   sparse rewards as a distance metric to measure the distance between two states.
302
      We use IQL (Kostrikov et al., 2022), an offline RL algorithm, for a distance metric Ψ by modifying
303   IQL to fit the goal-conditioned RL setting, i.e., all states are concatenated with the goal state as the
304   input for all parameterized models. To train goal-conditioned IQL, we need not only current states,
305   plays, and the next states that can be sampled from the dataset but also goal states and rewards that
306   are not given. Then, we sample goal states by setting the time step of the goal state as tG = t + ηH,
307   which presents a time step after η plays are performed. H is a fixed window size, and η ∈ N is a
308   random variable sampled along the geometric distribution. We use sparse rewards, which become 1
309   if the goal state is reached within one play from the current state (i.e., η = 1) and 0 if not.
310   Finally, through beam search with a planning set, the playbook can infer the play plan to reach
311   a given goal state. The playbook converts the first play of the best plan into raw actions using
312   primitives and performs the raw actions. Until the playbook achieves the goal state, the play plan
313   process is repeated.

315   5.2   M ONTE C ARLO T REE S EARCH FOR M IXED -P LAY P LAN
316
317   We aim to find the mixed play plan to solve the compounded problems using an extended playbook.
318   To this end, we propose an MCTS-based play planner that allows us to mix plays learned from
319   different datasets freely. For performing MCTS, we train and retain the same planning sets as beam
320
      search, {{∆1 , Ψ1 }, · · · , {∆P , ΨP }}, for each of P datasets. Note that the extended playbook must
      maintain all planning sets for each dataset, which is a limitation of the playbook extension.
321
322   We conduct a fixed number of tree searches, and each tree search generates one play plan. The
323   process of tree search is as follows. First, MCTS repeats selecting one planning set {∆i , Ψi }, which
      is trained from the dataset Di among the owned sets until it reaches a leaf node or the maximum tree


                                                            6
      Under review as a conference paper at ICLR 2025


324
      depth. For all selections, MCTS infers a play and next state using ∆i , i.e., it is a rollout step. Next,
325   when each tree search is completed, the value of the generated play plan is determined by measuring
326   the distance between the last state and a given goal using Ψi , i.e., it is a selection step. By iteratively
327   performing the above tree search, we can effectively generate mixed play plans. Finally, we choose
328   the trajectory with the highest value among the inferred mixed plans. A more detailed explanation
329   of the MCTS planner is presented in Appendix E.

      7    L IMITATIONS
518
519   Remaining all planning sets for mixed play plan. As mentioned in Section 5.2, we must preserve
520   planning sets for all datasets in order to generate mixed play plans using an extended playbook.
521   Since planning sets are composed of offline RL algorithms, this limitation can be solved through
522   continual RL for planning sets, but it is still a challenging problem. This limitation can be addressed
523   in the future work for the playbook.
524
      Trade-off between the number of plays and planning efficiency. The playbook has a trade-off be-
525   tween the number of plays and the efficiency of the play plan. As the number of plays increases, the
526   ability of the playbook to cover multi-modal actions improves, but as the complexity of the play plan
527   increases, the required computational cost increases and the efficiency decreases. Conversely, as the
528   number of plays decreases, the play plan becomes simple, but the expressive power of the playbook
529   decreases. Therefore, the playbook must explore the appropriate number of plays experimentally.

532
533
      In this paper, we propose a novel scalable offline discrete skill discovery algorithm, a playbook, for
534   long-horizon decision-making problems. The playbook provides a straightforward way to expanding
535   the skill set by utilizing discrete skills and the extensible structure. Furthermore, the playbook
536   effectively expresses multi-modal behavior distributions included in the dataset with only a finite
537   number of skills. Experimentally, we confirm that the playbook with a discrete skill set performs
538   better than existing baselines, which utilize the continuous skill space. In addition, we verify that
539   the extended playbook not only successfully solves tasks included in new datasets but also carries
      out compounded tasks, which are a mixture of old and new tasks.

      Before training the planning sets, we convert pre-collected demonstrations into state-play trajectories
793
      using a pre-trained playbook, as depicted in Figure 7. The playbook selects a play ŵt ∈ B by
794

795   action sequences of the unstructured dataset are converted into state-play sequences. The converted
796   sequence has fewer steps than the original one, but the play maintains information about the original
797   action sequence, making it easier to predict distant future states than a one-step dynamics model. In

      D    P LAY S ELECTION P ROCESS THROUGH B EAM S EARCH

803
      This section deals with the state-play sequence inference process using trained planning sets. The
804   planning process follows beam search proposed by Janner et al. (2021). Figure 8 shows the process
805   of selecting the best play index using a pre-trained playbook to reach a goal state. First, the playbook
806   uses the trajectory generation model ∆ to generate diverse and reasonable state-play sequences from
807   the current state. Next, the playbook uses the state distance metric Ψ to select the closest trajectory
808   to a given goal state. The first play of the selected sequence is converted into raw actions using a
809   low-level policy and is performed in the current state. This process is repeated until the goal state is
      reached.

840   E.1   T REE S EARCH P HASE
841
842   We generates play sequences with an extended playbook by mixing plays learned from multiple
843   datasets using the MCTS planner, as shown in Figure 9. MCTS is performed through iterative
844   tree search processes. In each tree search process, the planner sequentially chooses a planning set
845
      {∆i , Ψi } among all planning sets at the current node. Next, the planner determines the next node
      by deciding a play index to be performed and predicting the next state using ∆i . Then, the depth
846
      of the tree search increases by one. If the planner reaches a leaf node, the value of the leaf node is
847
      measured by Ψi using a given goal state. Each tree search process is performed until a leaf node is
848   reached or the maximum depth of the tree is achieved.
849
850
      E.2   O PTIMAL P LAY P LAN G ENERATION P HASE
851
852   After MCTS is finished, the planner finds the best play sequence within the searched tree based on
853   the stored node values. By starting from the root node, the planner selects the play index with the
854   highest node value among all plays owned by the current node. Until a leaf node is reached, the
855   planner generates the best play sequence by sequentially selecting the play indices. In practice, we
856   execute only the first play in the inferred play plan. Then, we obtain the next state by executing raw
857   actions in the environment and perform MCTS again until the goal state is achieved.
