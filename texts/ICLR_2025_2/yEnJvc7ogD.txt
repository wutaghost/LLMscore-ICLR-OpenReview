009                                                A BSTRACT
010
011            Many machine learning algorithms and classifiers are available only via API queries
012            as a “black-box” — that is, the downstream user has no ability to change, re-train,
013            or fine-tune the model on a particular target distribution. Indeed, a downstream user
014            may not have any knowledge of the training distribution or performance metric used
015            to construct and optimize the black-box model. We propose a simple and efficient
016            method, CWP LUGIN, which takes as input arbitrary multiclass predictions, and
017            post-processes them in order to adapt them to a new target distribution and to
018
               optimize a particular metric of the confusion matrix. Importantly, CWP LUGIN is a
               post-hoc method which does not rely on feature information, only requires a small
019
               amount of probabilistic predictions along with their corresponding true label, and
020
               optimizes metrics by querying. We empirically demonstrate that CWP LUGIN has
021            performance competitive with related methods on a variety of tabular and language
022            tasks.

025   1    I NTRODUCTION
026
027   Consider the following common scenario: A machine learning practitioner would like to adapt a
028   public, open source model to a particular target task with only small set of labeled target examples.
029   There are a plethora of approaches in domain and task adaptation for working in this setting, including
030   model fine-tuning (Han et al., 2024; Dodge et al., 2020), low-rank adaptation (Hu et al., 2022),
031   classical importance weighing techniques (Azizzadenesheli, 2021; Lipton et al., 2018; Sugiyama
032   et al., 2007), and more (see, e.g., Ganin & Lempitsky (2015); Sun & Saenko (2016); You et al.
033   (2019)). These methods have been relatively successful, and show that the underlying base model can
034
      be improved or modified in order to adapt its performance to the target distribution quite efficiently.
035   The modern machine learning landscape, however, has become rife with proprietary and black-box
036   models. For example, there are numerous image and language APIs which allow for only query
037   access to the models of interest. For example, developers using Google’s vision API (Google, 2024),
038   Amazon’s Rekognition (Amazon, 2024), or Clarifai’s platform (Clarifai, 2024) are usually restricted
039   from accessing or tuning the underlying model, and can only interact with it via API requests. In
040
      light of this more challenging setting, we revisit the fundamental question of model adaptation:
041
                If a machine learning practitioner has only black-box query access to a model,
042            when and how can they adapt the model to a particular target task with only a
043            small number of labeled examples?
044
045   We will assume that the only information which the model designers share is class probability
046   estimates for any queried data point — particular details about the training distribution, training loss,
047   model weights, or even the model architecture itself are unknown. In this more restricted setting,
048   most fine-tuning or re-training approaches are immediately disqualified since the underlying model
049   architecture, weights, or training data are all unavailable to the practitioner.
050   In addition to distribution shift, we also consider how a practitioner can adapt the predictions of a
051   black-box model in order to optimize a specific metric of interest other than the one the model was
052   trained to optimize. The cross-entropy loss is the de-facto objective optimized in order to achieve
053   good performance on metrics such as accuracy and calibration; however, at test or production time,
      system designers may also desire prioritizing other metrics such as F-measures (Ye et al., 2012;


                                                         1
      Under review as a conference paper at ICLR 2025


054
      Puthiya Parambath et al., 2014), geometric mean and classifier sensitivity (Monaghan et al., 2021),
055   Matthews Correlation Coefficient (Chicco & Jurman, 2020), and more (Müller et al., 2022). As an
056   example, a practitioner utilizing models for downstream tasks such as sorting patients to receive
057   clinical attention (Hicks et al., 2022) or utilizing a closed-source language model to screen CVs (Gan
058   et al., 2024), the performance of the classifier on a particular metric of interest — e.g., minimizing a
059   particular mix of false-positives and true-positives — may be more important than simply obtaining
060   good accuracy. Indeed, some performance metrics of interest may not even have a closed form, and
061   can only be estimated by deploying a production grade system to a target population (Huang et al.,
062   2021; Hiranandani et al., 2021).
063   Taken together, methods which can adapt classifiers in a post-hoc and black-box manner to (1)
064   account for distribution shift; and (2) optimize specific metrics have broad applicability. Both tasks
065   are especially salient given the recent history and potential evolution of the model landscape (Maslej
066   et al., 2024).
067
068   Contributions. We propose a simple and effective coordinate-wise plugin method CWP LUGIN
069   for post-processing the probabilistic predictions of a black-box predictor in order to simultaneously
070   achieve both (1) improved performance on a shifted distribution; and (2) improvement on a specified
071   metric of interest. CWP LUGIN method is broadly applicable since it only assumes query access to
072
      the metric, and is not defined inherently defined by assuming any structure of the metric itself.
073   We introduce CWP LUGIN in Section 3.1. As input, the algorithm takes in (1) a set of probabilistic
074   multiclass predictions on a target domain along with their true labels; and (2) query access to a
075   particular metric of interest (e.g., accuracy, recall, F-measure). We consider metrics which can
076   be defined as simple functions of the confusion matrix, as standard in the black-box classification
077   literature (Hiranandani et al., 2020; Jiang et al., 2020). The output of CWP LUGIN is a set of m
078
      class weights, one for each of m classes. These weights are then used at inference time in order to
      appropriately re-weigh each of the classes in order to maximize the metric of interest.
079
080   In Section 3.2, we demonstrate that for a certain class of metrics — linear diagonal metrics — plugin
081   is a consistent classifier in that it will eventually recover the Bayes optimal predictor under the
082   metric of interest. We also demonstrate that the design of CWP LUGIN allows for its run-time to
083   be substantially improved when data is class balanced or the metric it is optimizing obeys a certain
084
      quasi-concavity property (Section 3.3).
085   Since the only inputs to CWP LUGIN are raw multiclass predictions — and not feature data — it is an
086   extremely flexible method which can be applied to a variety of both classical and modern domains.
087   To demonstrate this, in Section 4 we provide experimental evidence of its superior performance for
088   metric optimization across multiple tabular and language classification tasks under distribution shift.
089   For an illustrated setting of where CWP LUGIN may be applied, we refer to Figure 1.

      Classfier metric optimization is a well studied problem in both theory and practice (Ye et al., 2012;
      Koyejo et al., 2014; Narasimhan et al., 2014; Yan et al., 2018). Most related, however, is the line of
094
      work investigating optimizing black-box metrics, e.g., when no closed form of the metric is known
095
      Zhao et al. (2019); Ren et al. (2018); Huang et al. (2019); Hiranandani et al. (2021). This line of work
096   utilizes a variety of approaches, including importance weighed empirical risk minimization, or model
097   retraining for robustness. The most relevant work is that of Hiranandani et al. (2021), which is a
098   purely post-hoc method which does not require retraining or fine-tuning classifiers. The authors there
099   propose a post-hoc estimator which is learned via a “probing classifier” approach. Their approach
100   solves a particular, global linear system in order to find the weights which optimize a particular
101   metric. Our proposed method is instead local in that it considers only pair-wise comparisons between
102   classes. We demonstrate the superior performance of our method on a variety of real-world black-box
103   prediction tasks, suggesting that a global, linear system approach may not always be necessary.
104   There is a long history of work in machine learning on domain adaptation, or generalization under
105   distribution shift. These fall into a few main categories: Distributionally Robust Optimization
106   (DRO, Rahimian & Mehrotra (2019)), Invariant Risk Minimization (IRM, Arjovsky et al. (2019)),
107   various importance weighing methods (Lipton et al., 2018), and many more (Wilkins-Reeves et al.,
      2024; Gretton et al., 2008; Nguyen et al., 2010). As far as we are aware, however, there are few

108
      methods other than calibration which operate using only (probabilistic) predictions and labels for the
109   target distribution, and further do not require re-training or fine-tuning of the original model. These
110   properties are essential, as they allow methods to be applied on top of closed-source models (Geng
111   et al., 2024). One such example is the work of Wei et al. (2023), who propose re-weighing predictions
112   in the face of distribution prior shift with DRO.
113
      Calibration has long been a staple method within the machine learning community (Platt et al., 1999;
114
      Niculescu-Mizil & Caruana, 2005; Guo et al., 2017; Minderer et al., 2021; Carrell et al., 2022). Any
115   probabilistic classification model can be provably calibrated in a post-hoc manner, even for arbitrarily
116   distributed data (Gupta et al., 2020). Recently, Wu et al. (2024) demonstrated that a stronger version
117   of calibration from the algorithmic fairness literature, multicalibration (Hébert-Johnson et al., 2018),
118   has deep connections to robustness under distribution shift, and proposed a post-processing algorithm
119   which adapts a predictor under both co-variate and label shift for regression tasks.
120
      It is worth mentioning that language models have their own set of domain adaptation techniques,
121   such as fine-tuning from supervised (Han et al., 2024) or human feedback (Tian et al., 2023), prompt
122   tuning/engineering (Liu et al., 2023), in-context learning (Dong et al., 2022), etc. Our method is
123   agnostic to the choice of underlying base model; nonetheless, we include fine-tuning as a suitable
124   baseline where applicable.

139   2        P RELIMINARIES
140
141   Let X be the data domain and Y = {1, 2, . . . , m} = [m] be the set of labels in a multiclass
142
      classification problem. Let ∆(Y) denote the set of all distributions over labels. A (probabilistic)
      predictor b : X → ∆(Y) maps data points to distributions over classes. We call b a black-box
143
      predictor if we do not have any knowledge of how b was created, its particular architecture, or how it
144
      functions. Indeed, we may not even know or have access to the source distribution that b was trained
145   on: all we have is query access to obtain b(x) for any given x ∈ X . Typical examples of black-box
146   predictors include closed-source models of classification API services such as Google VisionAI or
147   Amazon Rekognition (Google, 2024; Amazon, 2024), custom text classification solutions provided
148   by a company like Clarifai (Clarifai, 2024), or models trained on proprietary health data and made
149   available to us via API by independent entity (see, e.g., Dandelion (2024)).
150
      We call S = {(b(xi ), yi )}i∈[n] a sample of n data points, and assume that (xi , yi ) ∼ D i.i.d. for
151   a target distribution D supported on X × Y. Notice that we adopt the convention of using the
152   predictions of b to define the sample S; this is purely to simplify notation since our proposed method
153   will operate using only the predictions of b (and disregard any feature information). We work in the
154   scenario where |S| is small, say, on the order of tens or hundreds of examples. Therefore, given that
155   the target and source domains have non-trivial overlap, we expect that training or fine-tuning a new
156   model from scratch using only the sample S will give sub-par performance on the target domain.1
157   Metrics and Confusion Matrices. Before discussing how we plan to improve b by re-weighing
158   its predictions, we first provide background on the metrics we seek to optimize. As is standard
159   in the black-box classification literature (Hiranandani et al., 2021; Jiang et al., 2020), we consider
160   post-processing b in order to optimize for metrics defined as functions of the confusion matrix. We

174   3       R EWEIGHING P REDICTIONS USING L EARNED C LASS W EIGHTS
175
176   In Section 3.1, we propose CWP LUGIN: a method for learning weights w to re-weigh the predictions
177   from a black-box predictor b in order to optimize a metric f , potentially under distribution shift
178   between the source domain that b was trained on and the novel target domain. We argue that
179   CWP LUGIN is simple to implement, and can be analyzed in a certain restricted setting (Section 3.2).
180
      We also show that it is generally parallelizable, and with certain additional structure of the metric f ,
      enjoys sizable efficiency improvements (Section 3.3).
181

      3.1      T HE CWP LUGIN R E - WEIGHING M ETHOD
184
      Our proposed method will learn a vector w ∈ Rm of m weights, one to re-weigh each of the m
185   classes predicted by b. Simply re-weighing the predictions is surprisingly expressive: Not only does
186   it allow for provably optimizing certain families of metrics (Section 3.2), it also describes the Bayes
187   optimal learner under certain kinds of distribution shift such as label shift and label noise (see, e.g.,
188   Hiranandani et al. (2021, Table 1)). In addition, there are a variety of post-hoc model adaptation
189   methods from the calibration and robustness literature which show surprising potential improvements
190   by modifying the output of a predictor b with only m or m2 parameters (Guo et al., 2017; Kull et al.,
191   2019; Wei et al., 2023; Wang, 2023); We use these as motivation in our design of CWP LUGIN.
192   A naive approach to learning the optimal weights w∗ maximizing the metric f on the sample set
193   S = {(b(xi ), yi )}i∈[n] is to perform a brute-force m dimensional grid search over [0, 1]m . For binary
194   classification problems, this simplifies to tuning the decision threshold to optimize a metric f using a
195   hold-out validation set.2 However, this approach quickly becomes infeasible as the number of classes
196   m grows beyond two and the required precision ϵ increases. For example, finding w∗ with m = 5
197   classes and precision ϵ = 0.1, or m = 3 and ϵ = 0.01, both require a search over at least 105 grid
198   points — and hence, metric evaluations — of f (Ch ).
199   To ameliorate this, we instead propose a coordinate-wise search approach, which we call CWP LUGIN.
200   Instead of performing a grid search over all O(1/ϵm ) grid points, CWP LUGIN fixes one of the classes
201   — say, class m — as a reference class. It then restricts consideration to the m − 1 classifiers which
202   output either class k or class m everywhere (for k ∈ [m − 1]). It will use these restrictions in order
203   to find an optimal relative weight between each pair of classes.
204   Before formalizing this, we introduce the following necessary assumption on the black-box predictor
205   b in order to guarantee convergence of CWP LUGIN. Let b(x)k be the probability of class k ∈ [m].
206   Assumption 1. For each k ∈ [m], there exists xj ∈ S such that b(xj )k > 0.
207
208   This assumption simply states that the sample S is non-trivial over all m classes. This is w.l.o.g.: if b
209   did not satisfy this for some class k, we could simply drop that class from all predictions.

253   3.2   A NALYSIS
254
      In this section, we sketch the guarantees of the CWP LUGIN algorithm within the framework of metric
255
      weight elicitation (Zhao et al., 2019). Most details are deferred to Appendix A.2, but we give an
256   informal overview of our results here. The metric weight elicitation framework assumes that the
257   metric f is only available via oracle query. The goal is to learn the metric f , by assuming that it has a
258   specific functional form (linear, diagonal, etc.), and fitting the relevant coefficients using a sample S.
259
      In our first result, Proposition 6, we show that CWP LUGIN is a consistent estimator for the family
260
      of linear-diagonal metrics: it elicits the optimal weights and learns the Bayes optimal predictor
261
      when given access to population quantities. Afterwards, in Proposition 9, we show that with a finite
262   (polynomial) number of samples, CWP LUGIN can still obtain approximately optimal weights for
263   the underlying linear-diagonal metric Both results illustrate that CWP LUGIN may provide rigorous
264   statistical guarantees in the presence of metric shift; this is not normally provided by standard post-hoc
265   post-processing methods like calibration.

306   4       E XPERIMENTS
307
308   We provide preliminary empirical evidence that the CWP LUGIN method can be used post-hoc to
309   improve the metrics of black-box predictors in various distribution shift / metric optimization settings.
310
311   Experimental Setup. In our experimental setup, we will work with three different sets of data.
312   The training set is sampled from the source distribution, and is what we use to train the black-box
313   predictor b. After initial training of b, we cannot modify or access its weights/architecture, re-train it,
314   etc. We then tune the black-box predictions in a post-hoc manner in order to perform well on the
315   out-of-distribution test set by using a (small) validation set S. Generally, the size of |S| ≪ the size of
316
      the training set, and so the practitioner stands to gain from using some of the power of b. Finally, we
      report results of the adapted model on the hold-out test set.
317
318   To simulate this setup in our experiments, in each setting we fix a certain model to be the “base
319   black-box classifier” b. Then we investigate how much we can improve upon b by only modifying its
320   predictions and not the model itself.
321   To measure statistical significance and better understand how sensitive each evaluated method is
322   to the individual samples which appear in the validation set S, we run each experiment multiple

      are calibration methods. This is because many calibration techniques are post-hoc and operate
336
      using only (multiclass) black-box predictions and true labels. Note, however, that most calibration
337   techniques have goals slightly orthogonal to ours: they seek to increase accuracy or produce calibrated
338   probabilities by minimizing the negative log likelihood (NLL) or similar quantities, and do not
339   explicitly optimize for a particular metric of interest. On the other hand, CWP LUGIN takes as input
340   the metric of interest (e.g., Accuracy, F-measure, etc.) and optimizes for it explicitly.4
341
      We list out the baselines we include, deferring their additional implementation details to Appendix B.1.
342
343   Clean. Throughout, clean represents the raw hold-out test performance of the black-box predictor
344   with no post-processing applied. If a method improves upon clean, then it means that the small
345   validation set S was helpful in adapting or improving the base black-box classifier b.
346   Vector. We include a variant of vector scaling (Guo et al., 2017), a standard in post-hoc calibration.
347
      Dirichlet Calibration. Introduced by Kull et al. (2019), Dirichlet calibration is a family of methods
348
      which can be implemented directly on top of class probabilities. We include two versions amongst
349
      our baselines: DiagDirich and FullDirich, which roughly correspond to learning post-hoc estimators
350   with m weights for the former, and m2 for the latter.
351
352   Probing Classifier. The post-hoc “probing classifier” approach from Hiranandani et al. (2021) can
353
      also take in an arbitrary (confusion matrix-based) metric as input and optimize for it. We use the
      authors’ original implementation, but restrict to the version which does not use feature-defined groups
354
      in order to refine the estimates.
355
356   Metrics Evaluated. Note that only our CWP LUGIN method and the probing classifier method take
357   as input the metric to be optimized as input. We generally run our experiments with Accuracy and
358   macro variants of F-measure, G-mean, and Matthews Correlation Coefficient (MCC). We use the
359
      scikit-learn (Kramer & Kramer, 2016) F-measure implementation, the imbalanced-learn (Lemaı̂tre
      et al., 2017) implementation of G-mean, and our own implementation of MCC.

      In this section, we show that CWP LUGIN can also perform well in the presence of label shift (Lipton
474   et al., 2018; Storkey, 2008) or label noise (Natarajan et al., 2013; Patrini et al., 2017). Let D′ be the
475   source distribution, and D the target distribution. We test for learning under knock-out label shift.
476   This setting is motivated by, for example, disease classification, where during an outbreak D(y|x)
477   may be larger than historical data D′ (y|x), but the manifestations of the disease D(x|y) = D′ (x|y)
478   may not change (Lipton et al., 2018). In our experiments, we model label shift by randomly deleting
479   a fraction of a subset of classes in D relative to the original source distribution D′ . We also test for
480   symmetric, class-dependent label noise. That is, for a certain subset of classes, datapoints of that
481   class have their labels in the validation set S flipped to another class — chosen uniformly at random
482
      — with probability p.
483    We test these two types of noise on two language classification tasks: SNLI (Bowman et al., 2015)
484    and ANLI (Nie et al., 2020). For both the label shift and label noise settings, we utilize a model
485    trained on GLUE (Wang et al., 2019) and ANLI as our base, black-box predictor (Wong, 2023;
       Li et al., 2023). Details about the specific parameters of label noise and label shift are deferred to

525    5         L IMITATIONS AND C ONCLUSIONS
526
527    One limitation of CWP LUGIN is that it may be very dependent on the available number of samples
528    for the selected fixed class. Throughout our discussion, we chose class m as the fixed class; however,
529    in practice we found that choice of this fixed class can impact performance and the ability to fit a
530
       meaningful signal in the data. Another limitation is that since the post-processing method utilizes
       solely the probabilistic multiclass predictions — and not any feature information — the quality
531
       of these predictions is quite important in determining the outcome of the method. For example,
532
       predictions which are not calibrated, or do not represent meaningful probabilities may allow for less
533    expressiveness of post-hoc estimators, which limits this class of post-processing methods. We leave
534    investigating both these directions more rigorously to future work.
535
536
       We believe that our work represents an important direction in the ever-changing model marketplace.
       As black-box predictors potentially become more common solutions to machine learning practitioner
537
       application domains, post-hoc methods like CWP LUGIN may eventually allow practitioners some
538
       degree of model adaptation to particular tasks of interest.

975    B.1    A DDITIONAL BASELINE D ETAILS
976
977    Here we give additional implementation details for the baseline methods we compare against. Post-
978    hoc multiclass calibration techniques fall into two categories: techniques which operate on logits
979
       (raw, unscaled probabilities), and techniques which take as input class probabilities. We assume that
       only class probabilities are available to us as outputs of black box models, and as such, we mainly
980
       focus on the latter.
981
982    Vector Scaling. Let σSM : Rm → ∆m be the softmax function. Given a black-box predictor b, vector
983    scaling (Guo et al., 2017) learns a transformed estimator of b, given by σSM (W · b(xi ) + c). The
984    weight matrix W ∈ Rm×m and bias vector c ∈ Rm are chosen in order to minimize the NLL on the
985
       calibration set. Note that the weight matrix W is restricted to be diagonal, and hence, the method is
       essentially learning 2m parameters. Furthermore, the original formulation actually fits the parameters
986
       on top of the model logits, which are unavailable to us. We modify the formulation to fit the class
987
       probabilities given as the output of b(xi ). We use the vector scaling implementation given by NetCal
988    in Küppers et al. (2020), which uses cross validation to select the best internal parameters.
989
990    Dirichlet Calibration. Introduced by Kull et al. (2019), Dirichlet calibration is a family of methods
991
       which can be implemented directly on top of class probabilities. The method is built on the assumption
       that the underlying prediction vectors are sampled from a Dirichlet distribution. Formally, Dirichlet
992
       calibration also learns a weight matrix W and bias c learn a classifier given by σSM (W · ln b(xi ) + c).
993
       In order to choose appropriate W and c, Dirichlet calibration minimizes log loss combined with
994    Off-Diagonal and Intercept Regularisation (ODIR). ODIR takes two hyperparameter values: λ and µ.
995    We search over all combinations of (λ, µ) ∈ {10−1 , 10−2 , 10−3 , 10−4 , 10−5 }2 . We select the best
996    performing hyperparameter pair on the validation set S.
997
       Throughout our experiments, we noticed similar performance of Diagonal Dirichlet and Vector
998
       scaling, even though the implementations are very separate. Given that we select the optimal
999
       Diagonal Dirichlet calibrator based on performance on the validation set S, the resulting solution may
1000   look nearly identical to Vector scaling at smaller regularization values. As the larger regularization
1001   values were rarely selected, the performance and optimized solution of both methods are quite similar.
1002
1003
       Probing Classifier. In addition to calibration measures, we also report the performance of the
       “probing classifier” introduced in Hiranandani et al. (2021). This classifier is constructed via post-
1004
       processing a black-box predictor b by learning m class weights, similar to plugin. However, these m
1005
       weights are found by solving a particular linear system which maximizes the metric of interest. We
1006   use the authors’ original implementation, but restrict to the version which does not use feature-defined
1007   groups in order to refine the estimates. The method also takes in a step-size parameter ϵ. We select
1008   the best performing parameter amongst ϵ ∈ {0.1, 0.05, 0.01, 0.005, 0.001} by taking the one with
1009   the best metric value on the (validation) set S.
1010
       BERT-FT. Our fine-tuning baseline takes the original open-source BERT-cased model from Hug-
1011
       gingFace (Devlin et al., 2018), and fine-tunes it using AdamW on the validation set S with batch size
1012   64 over 100 epochs. We use a linear learning rate decay which kicks in after 500 warmup steps, and
1013   also utilize the default pre-trained BERT-cased tokenizer. We select the best performing model across
1014   all epochs (using only the set S, not any hold-out data). Then, we report the predictions of the model
1015   on the hold-out test set.
