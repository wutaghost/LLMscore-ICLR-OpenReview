011                                               A BSTRACT
012
013            Recently, large language models (LLMs) have achieved significant progress in
014            automated code generation. Despite their strong instruction-following capabilities,
015            these models frequently struggled to align with user intent in the coding scenario. In
016            particular, they were hampered by datasets that lacked diversity and failed to address
017
               specialized tasks or edge cases. Furthermore, challenges in supervised fine-tuning
               (SFT) and reinforcement learning from human feedback (RLHF) led to failures
018
               in generating precise, human-intent-aligned code. To tackle these challenges and
019
               improve the code generation performance for automated programming systems, we
020            propose Feedback-driven Adaptive Long/short-term memory reinforced Coding
021            OptimizatioN (i.e., FALCON). FALCON is structured into two hierarchical levels,
022            from the global level, long-term memory improves code quality by retaining and
023            applying learned knowledge, while from the local level, short-term memory allows
024            for the incorporation of immediate feedback from compilers and AI systems.
025            Additionally, we introduce meta-reinforcement learning with feedback rewards
026            to solve the global-local bi-level optimization problem and enhance the model’s
027            adaptability across diverse code generation tasks. Extensive experiments are
028
               conducted and it is found that our technique achieves state-of-the-art performance,
               leading other reinforcement learning methods by more than 4.5 percentage points
029
               on the MBPP benchmark and 6.1 percentage points on the Humaneval benchmark.
030
               The open-sourced code is publicly available at https://anonymous.4open.
031            science/r/FALCON-BFE0/README.md.

034   1   I NTRODUCTION
035
036
      The development of Large Language Models (LLMs) has significantly advanced automated code
037   generation (Zan et al., 2023). Models like CodeLLaMA (Roziere et al., 2023) and DeepSeek-Coder
038   (Guo et al., 2024), tailored for code-centric tasks, have demonstrated outstanding performance
039   across programming challenges. While LLMs excel in instruction-following through tuning (Jiang
040   et al., 2024), they often misalign with user intent, making feedback-based adjustments critical. For
041   example, InstructGPT (Ouyang et al., 2022) leverages reinforcement learning with human feedback
042   (RLHF), and CodeRL (Le et al., 2022) uses compilation feedback to refine model performance.
043   Similarly, CompCoder (Wang et al., 2022) enhances code compilability with compiler feedback,
044   and RLTF (Liu et al., 2023a) offers fine-grained feedback on compiler errors. However, current
045
      RL frameworks generate compilation errors and overlook non-differentiable features (e.g. coding
      style) that affect the performance significantly (Jiang et al., 2024). To address these challenges, we
046
      propose a reinforcement learning system combining long-term and short-term memory feedback.
047
      From the global level, long-term memory tracks trends over time for higher-quality code retrieval,
048   while from the local level, short-term memory captures recent errors and immediate feedback. The
049   main contributions of this paper are as follows:
050
051          • Short-Term and Long-Term Memory for Reinforcement Learning: We propose a dual-
052            memory approach for reinforcement learning in code generation, where short-term memory
053            enables real-time corrections and long-term memory accumulates knowledge from past runs
               to improve code quality and reduce repetitive mistakes.

             • Non-Differentiable Code Features into Feedback Loops: Our approach addresses the

066   2.1   P RE - TRAINED M ODELS FOR C ODE G ENERATION
067
068   In recent years, pre-trained language models have made significant progress in the field of code
069   generation. Trained on large-scale code corpora, these models have demonstrated powerful code
070
      generation capabilities. For example, CodeBERT Feng et al. (2020), a model based on an encoder-
      only architecture, has shown impressive results. With the advent of in-context learning methods,
071
      decoder-only Transformer models have become the dominant technology for language modeling
072
      Vaswani et al. (2017). Several models, such as CodeGPT Lu et al. (2021), CodeGeeX Zheng
073   et al. (2024), and DeepSeek-Coder Guo et al. (2024), use Causal Language Modelling (CLM)
074   pretraining, while others like CodeT5 Wang et al. (2021) and AlphaCode Li et al. (2022) utilize an
075   encoder-decoder architecture. Additionally, models like CodeFusion Singh et al. (2023) leverage
076   diffusion-based techniques for code generation. These pre-trained models exhibit great potential in
077   code generation tasks, achieving notable improvements in accuracy through various architectures
078   and training strategies. However, they still face challenges in ensuring the syntactical and functional
079   correctness of the generated code.
080
081   2.2   R EINFORCEMENT L EARNING ON CODE
082
083   Reinforcement learning (RL) is a method that learns optimal strategies through reward signals from
084   interacting with the environment Fujimoto et al. (2019). It excels in sequence generation tasks, such
085   as enhancing performance in translation and summarization models by improving BLEU and ROUGE
086
      scores Ahn et al. (2019). Unlike traditional natural language processing tasks, code generation is more
      complex. Beyond syntactical correctness, the generated code must be functionally accurate, meaning
087
      it should compile and perform as expected in various scenarios. Passing unit tests is important for
088
      verifying correctness, safety, and precision, but not sufficient unless the tests are comprehensive and
089   well-designed. Unit tests must cover diverse use cases and edge cases to ensure the code adheres to
090   standards and meets requirements. For instance, the RL-based code generation fine-tuning framework,
091   CodeRL Le et al. (2022), guides fine-tuning by integrating unit test signals with reinforcement learning.
092   PPOCoder Shojaee et al. (2023) improves on CodeRL by employing Proximal Policy Optimization
093   to refine the approach, while RLTF Liu et al. (2023a) incorporates specific error information from the
094   code and multi-granularity feedback alongside an online framework for model training. StepCoder
095   Dou et al. (2024) enhances the code generation process through compiler feedback and segmental
096   optimization. However, current RL-based methods still face some limitations. Primarily, these
097
      methods lack detailed feedback on specific types and distributions of programming errors, which is
      crucial for identifying patterns, understanding root causes, and creating targeted interventions. This
098
      makes addressing recurring issues difficult. Additionally, the corrective mechanisms are not robust
099
      enough, often failing to provide timely, specific guidance, hindering effective learning. Additionally,
100   the diversity of input-output examples in existing benchmarks is limited, restricting the model’s
101   ability to adapt and generalize to different or unseen problems Liu et al. (2023a).

106   We aim to enhance the automated code generation capabilities of Large Language Models (LLMs) by
107   addressing key challenges in accuracy, diversity, error correction, and code quality. Formally, given a
      high-level specification D, the task is to generate a sequence of code tokens W = {w1 , w2 , . . . , wT }

      that maximizes the conditional probability P (W |D, θ), where θ represents the model parameters.
109   The optimization objective is defined as θ∗ = arg maxθ ED∼D [log P (W |D, θ)].
110
111   Challenges. Effective code generation is impeded by several factors. Accuracy requires minimizing
112
      syntactical or logical errors in W to ensure correct functionality. Diversity of input-output examples
      X is often limited, restricting the model’s ability to generalize across varied programming tasks.
113
      Efficient error correction mechanisms are necessary to identify and rectify errors in W , ensuring
114
      robust performance. Maintaining high code quality, which encompasses adherence to coding
115   standards, style guidelines, and managing code complexity C(W ), remains a persistent challenge.
116   Additionally, adaptability refers to the model’s capacity to adapt to new tasks and incorporate
117   feedback for continuous improvement, which is constrained without robust memory mechanisms.
118
      FALCON Framework. To address these challenges, we propose the FALCON framework. The
119
      framework leverages both a long-term memory buffer Mlong = {(Di , Wi , Ti , Fi )}N i=1 and a short-
120
      term memory buffer Mshort = {(Dj , Wj , Tj , Fj )}M j=1 to utilize diverse feedback, enabling fine-
121
      tuning of θ. Formally, FALCON seeks to optimize θ by maximizing a composite reward function
122
      R(W, F ) = αT (W ) + βS(W ) + γC(W ) + δE(W ), where T (W ), S(W ), C(W ), and E(W )
123   represent unit test, code style, complexity,  and error feedback, respectively.  The optimization
      objective is defined as θ∗ = arg maxθ ED∼D EW ∼P (W |D,θ) [R(W, F )] .

126
      Assumptions. The effectiveness of FALCON is based on the following assumptions: 1. Exchange-
      ability of the dataset D = {Di }N i=1 , implying that the order of tasks does not influence model
127
      performance; 2. Independence of feedback signals F given the generated code W ; 3. Sufficient
128
      memory capacity in both Mlong and Mshort to store relevant interactions without significant data
129   loss; and 4. Feedback efficacy, ensuring that integrated feedback mechanisms provide meaningful
130   and actionable information to guide the optimization of θ.
131
132
      Optimization Objective. Given a set of tasks T = {Di }N         i=1 , the goal is to learn optimal
      parameters θ∗ that maximize the expected composite reward across all tasks, formulated as

      and Fi represents the aggregated feedback. The formulation ensures continuous improvement by
135
      leveraging both historical and recent feedback to enhance code generation quality.

139
140
      In this section, we explore the FALCON framework, which integrates comprehensive unit testing
141   with reinforcement learning, supported by both long-term and short-term memory buffers. During the
142   code generation process, the system stores task descriptions, generated code, and various feedback
143   (e.g., compilation results, code style, and complexity) in the long-term memory buffer. By retrieving
144   this information, the model references high-quality code, avoids past mistakes, and ensures adherence
145   to required standards. After generating the code, a judge model evaluates it and calculates rewards
146   based on the feedback, which are then used to update the model’s parameters through reinforcement
147   learning. All generated code and feedback are stored for future reference and optimization. The
148   combination of long-term and short-term memory feedback in the FALCON framework allows the
149
      model to not only learn from a wide range of historical data but also adapt quickly to new tasks based
      on recent performance. The overall framework is illustrated in Figure 1.

527   6   C ONCLUSIONS AND F UTURE W ORK
528
529
      In this work, we propose FALCON, a novel framework that enhances automated code generation
      by integrating long-term and short-term memory feedbacks within a meta-reinforcement learning
530
      strategy. Long-term memory retains past interactions to improve code quality and reduce repetitive
531
      mistakes, while short-term memory enables immediate adjustments based on recent feedback from
532   compilers and AI systems. This dual-memory approach addresses limitations in existing models
533   that struggle to align code generation with user intent, especially in specialized tasks or edge cases.
534   By incorporating non-differentiable code features like style and complexity into the feedback loop,
535   FALCON ensures that the generated code is not only functionally correct but also adheres to real-
536   world programming standards. Extensive evaluations on benchmarks including APPS, HumanEval,
537   and CODAL-Bench demonstrate that FALCON outperforms existing RL-based methods, achieving
538   higher functional correctness and better coding style adherence. In future work, we aim to expand
539   FALCON’s capabilities by incorporating a broader diversity of programming languages and tackling
      more complex code generation challenges.
