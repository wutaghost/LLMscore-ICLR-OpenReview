011                                              A BSTRACT
012
013            The performance of neural networks improves when more parameters are used.
014            However, the model sizes are constrained by the available on-device memory
015            during training and inference. Although applying techniques like quantization
016            can alleviate the constraint, they suffer from performance degradation. In this
017            work, we introduce NeuZip, a new weight compression scheme based on the en-
018            tropy of floating-point numbers in neural networks. With NeuZip, we are able to
019
               achieve memory-efficient training and inference without sacrificing performance.
               Notably, we significantly reduce the memory footprint of training a Llama-3 8B
020
               model from 31GB to less than 16GB, while keeping the training dynamics fully
021
               unchanged. In inference, our method can reduce memory usage by more than half
022            while maintaining near-lossless performance.

026   1   I NTRODUCTION
027
028   Deep learning with neural networks has become the backbone of numerous artificial intelligence
029   applications. The search for better-performing networks is a longstanding topic in deep learning.
030   Without modifying the design, scaling up the number of parameters (e.g., number of hidden dimen-
031   sions or layers) has been demonstrated as an effective practice to boost the performance of neural
032   networks of the same kind (Kaplan et al., 2020). This idea has been successfully applied to text, im-
033
      age, audio, and multi-modal tasks with a wide range of model architectures (Yu et al., 2022; Radford
      et al., 2019; Brown et al., 2020). Recently, the number of parameters in the state-of-the-art models
034
      has become more than 100 billion or even a trillion parameters. For example, one of the state-of-the-
035
      art language models in 2020, GPT-3, has 175B parameters (Brown et al., 2020), growing by nearly
036   100 times compared with the largest Transformer architecture in the 2017 paper (Vaswani et al.,
037   2017).
038
039
      Despite the growth in the model size, the hardware capacity is not keeping up with the pace: the
      largest on-device memory of GPUs was 32GB in 2017, and is 80GB to this date in 2024, growing
040
      by only 2.5 times. The available hardware supply poses a limitation on the trainable model size,
041
      bottlenecking the scaling capacity. Although this problem can be alleviated by using more GPUs
042   and sharding the model in multiple devices (Rajbhandari et al., 2019), such a practice introduces
043   more communication overheads among GPUs, making large-scale distributed training less efficient.
044   Therefore, saving the total memory usage is critical in scaling up neural networks.
045
      The peak memory usage is dominated by three relatively independent parts: the optimizer, the saved
046
      activations for back-propagation, and the model itself. For the optimizer, there are already memory-
047
      efficient optimizers achieving a sublinear space complexity (Shazeer & Stern, 2018; Hao et al.,
048   2024); for the activations, the memory can be saved by enabling activation checkpointing (Chen
049   et al., 2016), which saves the storage by recomputing the forward activations during the back-
050   propagation. For the model parameters, there has not been an effective method to save the memory
051   while preserving the ability to train the model. Recently, Dettmers et al. (2023) proposed the quan-
052   tized low-rank adaptation (QLoRA), which freezes the parameters using a 4-bit data type for the
053   backbone pre-trained model. While significantly saving the memory for the model, it imposes a
      constraint on the overall change of the model to be low-rank, limiting the model capacity.

054
      In this paper, we propose NeuZip, an algorithm to compress the neural networks while maintaining
055   their full abilities. Specifically, each floating-point number is represented by three parts: the sign
056   bit, the exponent bits, and the mantissa bits. Following the observation that weights are concentrated
057   around zero (Kalamkar et al., 2019), we demonstrate that this corresponds to the low-entropy nature
058   of the exponent bits. We hence compress the exponent bits using the asymmetric numeral system
059   (ANS Duda (2013)), a lossless compression algorithm that achieves a high throughput on parallel
060   computing devices like GPUs. Since the compression is lossless, the memory reduction comes
061   without compromising any precision loss and enables full-parameter training.
062   In addition to lossless compression for training, we also propose a lossy variant of NeuZip for
063   inference that further reduces the memory footprint. Specifically, we control the relative change of
064   each parameter by storing only the top-k significant bits of the mantissa. We empirically show that
065   lossy NeuZip lies at the Pareto frontier of the memory–performance trade-off when compared with
066   several state-of-the-art quantization baselines.

      The Shannon entropy (Shannon, 1948) is used to measure the “stochasticity” of a random variable
071
      with the following definition:
072
073                                           H(X) :=        E       [− log2 p(X)]                       (1)
                                                         X∼p(X)
074
075   for a random variable X with probability p. A lower entropy indicates a less stochasticity of a
076   random variable. In fact, the entropy equals the minimum number of bits required, in expecta-
077   tion, to represent a random variable, therefore corresponding to data compressibility. For the non-
078   concentrating random variable with all possible values sharing an equal probability, the entropy of
079   which reaches the maximum value log2 n, where n is all possible values X can take. On the other
080   hand, for highly-concentrating (e.g., fully deterministic) random variables, the entropy can be as low
081   as 0.
082
083   2.1       L OW-E NTROPY NATURE OF N EURAL N ETWORK PARAMETERS
084
085   We argue that the parameters in neural network tend to have low entropy. First, parameters are typ-
086   ically initialized with Gaussian distribution for matrices (Glorot & Bengio, 2010; He et al., 2015).
087
      This encourages all weights to be centered around zero, effectively reducing the entropy (or random-
      ness). In addition, regularization is also applied for better generalization ability. For example, the
088
      weight decay technique reduces the magnitudes of weights at every update iteration. Similarly in
089
      Bayesian inference, prior distributions (e.g., Gaussian and Laplace distributions) are often applied,
090   imposing a zero-concentrated preference over the parameters.Even without explicit regularization,
091   stochastic gradient descent (SGD) or its variants are shown to have the implicit regularization ef-
092   fect on neural networks, meaning the model parameters are implicitly encouraged to have smaller
093   magnitudes during training (Soudry et al., 2018; Vardi & Shamir, 2021). All the above effects and
094   techniques lead to the following observation:
095   Observation 2.1 Assuming neural network parameters are i.i.d. random variables, the entropy of
096   the distribution is likely to be low.
097
098   Specifically, each parameter is represented is represented by three components: the sign bit, the
099   exponent bits, and the mantissa bits in the IEEE 754 standard (IEEE, 2019).1 Therefore, we conduct
100   a fine-grained analysis and investigate the distribution of each component of a floating-point number
101   in neural networks.
102   As shown in Figure 1, the sign bit has a high entropy as it is evenly distributed; hence, it is not
103   compressible. For the exponent bits, there is a clear pattern that they demonstrate a low-entropy
104   nature, carrying only less than 3 bits of information with 8 bits of capacity. For the mantissa bits,
105   they store nearly 7-bit information with 7-bit capacity. In fact, we shown in Appendix A that this is
106   common in deep learning.

      Compressed representation. Based on observation, we see that the number of bits per exponent
      is largely inflated compared with the information entropy. However, previous research demonstrates
131
      that the dynamic range provided by the 8-bit exponents are critical for neural networks (Kalamkar
132
      et al., 2019). We therefore propose to compress the exponent bits in a lossless manner based on the
133   entropy. This practice mainly has three benefits: (1) it increases the throughput of compression as
134   only part of the bits are processed by the compression; (2) it reduces the burden of maintaining the
135   statistics of a large set of symbols (e.g., 256 symbols for 8-bit exponents versus 65,536 symbols for
136   16-bit representations), enabling a great efficiency of compression algorithms; (3) most importantly,
137   it recovers most of the compressibility as shown in Figure 1.
138
139   Multi-layer neural networks. The compression alone does not save any memory for maintaining
140   a single array. This is because, either compression or decompression, requires at least one buffer
141   of the same size as the uncompressed array. In the scope of neural networks, the whole model is
142
      prohibitively large and it is infeasible to duplicate the memory. In NeuZip, however, we exploit the
      multi-layer structure of modern neural networks to avoid creating a large buffer. Without loss of
143
      generality (see Appendix F), we focus on the linear function as a common building block in neural
144
      networks at layer l:
145
146                                           xl ← Wl xl−1 + bl ,                                        (2)
147   where Wl ∈ Rm×n is the weight matrix, bl ∈ Rm is the bias vector of layer l, and xl is the input of
148   layer l. We propose to modify the compressed forward pass in the following form
149
150
                                             Ŵ ←decompress(cl )                                         (3)
151                                           xl ←Ŵ xl−1 + bl ,                                         (4)
152
      where cl is the compressed storage of the matrix Wl . In this way, we only need to store ci for
153   each layer, enjoying low-memory usage. During each forward pass, weight matrices stay in the
154   compressed form until the original data is needed, in which case it is decompressed into a temporary
155   space Ŵ for computation. As a result, the entire network is never fully decompressed at any point
156   in time, making the overall forward pass memory efficient. The per-layer procedure is shown in

190   Compression algorithm. In our implementation, we choose to use the asymmetric numeral sys-
191   tems (ANS) (Duda, 2013) as our backbone compression algorithm because it can be easily paral-
192   lelized and achieves a high throughput with parallel execution, making it an ideal candidate on deep
193   learning accelerators like GPUs. Specifically, ANS encodes a sequence of symbols by treating them
194   as base-n numbers. However, unlike the common numerical system that uses a uniform base for
195
      each digit, ANS treats every single digit with a different base ⌈1/p̂i ⌉, where p̂ is the frequency of
      symbols. As a result, it achieves a near-optimal compression rate by suing around 1/p̂i bits for the
196
      ith symbol.

      2.3      L OSSY N EU Z IP : A DDITIONALLY T RUNCATING M ANTISSA FOR I NFERENCE
200
      In the algorithm above, we show that the training of neural networks can be completely unaffected
201   by lossless compression. On the other hand, inference is known to be less sensitive to precision loss
202   compared with training (Dettmers et al., 2022; Dettmers & Zettlemoyer, 2023). This enables further
203   memory reduction of NeuZip by reducing the precision. In our study, we conduct a pilot experiment
204   that perturbs each weight with a noise proportional to the weight magnitude. We observe that with
205   a small noise ratio there is little or no effect on the overall performance (Appendix C). Motivated by
206   this, we propose a variant of NeuZip that compresses mantissa in a lossy way during inference.
207   In its core, we simply round and truncate the mantissa to fewer bits. Specifically, we assume the
208   original floating-point number f has an exponent e and mantissa m. After rounding, the mantissa is
209   denoted by m̂ and the resulting floating-point number is denoted by fˆ.
210
211   The rounding introduces an error expressed as:
212                                                   m            m̂
213                                |f − fˆ| = 2e−127 · 7 − 2e−127 · 7 = 2e−134 · |m − m̂|                                        (7)
                                                      2            2
214
215   where e − 127 interprets the exponent bits e as an integer, which can be either positive, negative,
      or 0. In the fraction m/27 , m is the significand (an unsigned integer) and 7 is the precision. It is


                                                                     4
      Under review as a conference paper at ICLR 2025


216
      straightforward to see that the relative error is given by
217
218                                 |f − fˆ|   2e−134 · |m − m̂|   |m − m̂|
219                                          =                   =          .                              (8)
                                      |f |       2e−134 · |m|         m
220
221   Suppose our rounding keeps k most significant bits in m̂, the earliest point where m̂ could differ
222   from the original number m is at the (k + 1)th bit. This means that the maximum possible relative
223   change introduced by this rounding is 1/2k . Given that the mantissa bits are highly uniform as
224
      shown in Figure 1, such a practice resembles the weight perturbation based on relative magnitudes,
      justifying the rounding trick applied to mantissas.
225
226   In our implementation, we store the sign and mantissa bits together as a signed integer to minimize
227   the requests of memory write. Further, given that modern architectures are mostly byte (8-bit)
228   addressable, we pack multiple such signed integers into a single byte for memory efficiency. To
229   align with an 8-bit byte, we let the precision after rounding to be {0, 1, 3}, ensuring that all the bits
230
      in a byte are utilized efficiently. We illustrate the process in Figure 8b.
231   Lastly, we enable a block-wise normalization technique (Dettmers et al., 2023), where a block is
232   a chunk of weights that are stored contiguously in memory. Such block-wise normalization makes
233   sure that the weight with the largest magnitude in a block will always be normalized to 1, invariant
234   to mantissa rounding and truncation. The normalization coefficient—which handles mantissa while
235   ignoring the exponent—is stored with 8 bits, and is used for de-normalization during the decom-
236
      pression of the weight. This strategy is based on the observation that larger weights play a more
      important role in neural networks (Han et al., 2015).

241   We empirically verify the effectiveness of NeuZip across different model architectures and datasets.
242   Given the success of large language models, we mainly consider Transformer-based models for our
243   experiments. We choose two designs of Transformer, decoder-only and encoder–decoder models, to
244   show the generality of our method. Each experiment is conducted on a single RTX A6000 GPU to
245   avoid complications with the memory-usage in the multi-gpu scenario. The uncompressed data type

493   4   R ELATED W ORK
494
495   Model compression. Previous work has explored different techniques to reduce the memory usage
496   of neural networks, including knowledge distillation (Hinton et al., 2015) and pruning (Kwon et al.,
497   2022). Most related to our work is the quantization technique, which represents each parameter
498   with fewer bits; common approaches include k-means-based quantization (Han et al., 2016), lin-
499   ear quantization (Han et al., 2016), mixed precision quantization (Dettmers et al., 2022; 2023), and
500
      non-uniform grid quantization (Chikin & Antiukh, 2022). Our NeuZip differs from these methods
      by utilizing the exponent entropy and achieving lossless compression (see Appendix G for details).
501
      When training data are available, one may incorporate the quantization into the training process to
502
      improve performance (Xiao et al., 2023; Frantar et al., 2023). In this paper, our NeuZip compression
503   is a zero-shot method, and therefore, our experiments consider the widely used zero-shot quantiza-
504   tion methods (Dettmers et al., 2022; 2023) for fair comparison. We leave the utilization of additional
505   data of NeuZip to future work.
506
507
      Memory-efficient optimizers. The optimizer also occupies a considerable amount of memory
508
      during training (Rajbhandari et al., 2019). To address this, memory-efficient optimizers (Shazeer
509
      & Stern, 2018; Zhao et al., 2024; Hao et al., 2024) are developed to reduce the memory footprint
510   of training. Our NeuZip is orthogonal to these optimization techniques, as it can be seamlessly
511   combined with any of these methods for further memory saving. In particular, the lossless NeuZip
512   is expected to have exactly the same results with less memory.
513
514
      Parameter-efficient training. Another line of research saves memory by training a subset of pa-
515
      rameters (Houlsby et al., 2019; Zaken et al., 2022) so the optimizer only stores information about
516   a small set of trainable parameters. One notable example is the low-rank adaptation (LoRA (Hu
517   et al., 2022)). However, such a practice restricts the optimization space of parameters, and thus usu-
518   ally leads to significant performance degradation. Moreover, low-rank methods are unsuitable for
519   pre-training.
520
      It is important to mention that memory-efficient optimizers and parameter-efficient training cannot
521
      reduce the memory cost during inference. By contrast, our NeuZip is suitable for both training and
522   inference.
523
524
525   5   C ONCLUSION
526
527   Summary. In this work, we present NeuZip, a novel compression scheme for neural networks that
528   achieves memory-efficient training and inference. By analyzing the floating-point structures, we
529   propose to compress the exponent in a lossless way and to compress the mantissa in a lossy way.
530   The lossless variant of our NeuZip may be applied to both training and inference, while yielding
531   exactly the same result as the uncompressed model. The lossy NeuZip provides additional memory
532   saving for inference, achieving superior memory–performance trade-off.
533
534   Limitations and future work. Due to the hardware constraint, the largest model that we consider
535   in this paper is 70B. We would like to verify our NeuZip on even larger models like GPT-3 (Brown
536   et al., 2020) with more capable hardware. Another limitation of NeuZip is that the throughput is
537   lower than the vanilla model. However, it has a comparable speed with highly optimized quanti-
538   zation methods while achieving significantly better performance. By using NeuZip, we expect to
539   create opportunities for researchers to explore and study large models on more accessible or edge
      devices.

      In this section, we describe the forward-backward procedure of NeuZip. First, we compress all
804   the linear layers in the original model and store the compressed information on-device. During
805   the training iterations, we decompress the compressed weights in a layer-by-layer manner for the
806   forward pass. For the backward pass, the input is recalculated again following the forward pass like
807   activation checkpointing (Chen et al., 2016). A linear operation calculates the gradients for both
808   the weight matrix and input. To do so, we need to decompress the weight again, which is used
809   to calculate the gradient of input. After the gradient is calculated, we directly update the weight
      without storing it similar to LOMO (Lv et al., 2023).

      G        D ISCUSSION ON Q UANTIZATION AND C OMPRESSION
965
966
      Our NeuZip resembles quantization in the sense that both methods save the parameter memory.
      However, their mechanisms for memory saving are fundamentally different. Specifically, quantiza-
967
      tion is inherently lossy regardless of the chosen quantization values. The introduced error in this
968
      lossy process hinders its applicability in scenarios where precision plays an important role. As a
969   concrete example, quantization-based training methods underperform NeuZip in both quality and
970   memory saving in Section 3, displaying the importance of precisions.
971
          4
              This might only hold for certain implementations.


                                                               18
       Under review as a conference paper at ICLR 2025


972
       Undoubtedly, some quantization schemes may have lower levels of precision loss. For example, if
973    the quantized values are uniformly distributed (i.e. linear quantization Han et al. (2015); Dettmers
974    et al. (2022)), the overall precision loss could be enormous because it overlooks the distribution of
975    the raw data. On the other hand, using non-uniform quantization could alleviate this issue by pro-
976    viding tighter approximations for common values (Chikin & Antiukh, 2022; Dettmers et al., 2023).
977    However, even non-uniform schemes cannot entirely eliminate precision loss and often rely on sub-
978    optimal mixed-precision approaches to compensate for the reductions (Dettmers et al., 2022; 2023).
979    More importantly, recent studies have highlighted that precision becomes increasingly critical in
980    large-scale settings (Kumar et al., 2024). This observation reinforces the importance of applying
981
       NeuZip, which enables precision control up to a lossless level.
982    It is important to note that NeuZip is specifically designed for the raw parameters of neural networks
983    and is not practical for application to quantized values. This is because optimal quantization schemes
984    yield (approximately) equal frequency distributions across all quantized values (Chikin & Antiukh,
985    2022; Dettmers et al., 2023). As a result, applying additional compression would neither improve
986    the compression rate nor preserve efficiency, instead halving the throughput.
