017                                              A BSTRACT
018
019           In modern web design, ensuring adaptability and user engagement through dy-
020           namic layouts is increasingly important. With the growing demand for person-
021           alized user experiences, traditional static web layouts are insufficient for meeting
022           user preferences. This paper introduces an innovative approach that leverages gen-
023           erative AI to dynamically adapt web layouts in real-time. With the help of data
024
              that is collected under the banner of user interactions through technologies such
              as java script and node.js, we are able to save those interactions which not only
025
              include the click patterns, but also the timestamps, user’s name, day and date and
026
              number of clicks. These clicks correspond to interactions of users with different
027           react components. This data is being stored as a csv file as it is more easier to read
028           when it comes to parsing it to an AI model. Once every designated cycle, the data
029           is fed to a python script which does an API call to the Chat GPT 4o model which
030           then analyzes the data and re-writes the CSS to create a new web layout which
031           is based on the user’s interactions. This successfully gives a web interface that
032           adapts its layout in real-time, which is somewhat similar to many recommended
033           systems of popular applications like netflix, amazon prime etc. Its significance
034           extends across multiple fields, as this approach can enhance user engagement by
035
              dynamically displaying components based on user interaction patterns. Addition-
              ally, it offers potential revenue growth for companies, allowing them to charge
036
              higher rates for ads strategically placed in high-engagement areas of the layout,
037
              based on inferred user data.

      4     DATA A NALYSIS VIA GPT4 O

056
      After successfully collecting user interaction data, we developed a Python script to analyze this data
057   and update the webpage’s CSS dynamically. The script utilizes libraries such as Pandas, OpenAI,
058   and OS to facilitate the analysis and modifications. The process begins by using Pandas to read
059   and analyze the CSV file containing user interaction data. The script focuses on the last row of the
060   CSV file to determine the recently logged in user so as to know if the username in this row exists
061   elsewhere in the data.
062
063   4.1   U SERNAME IN THE DATA -S ET
064
065   If the username is found in the contents of CSV except the last row via a function
066
      check username in csv, the script invokes the function update css from interactions. This function
      takes three arguments: the username, the location of the CSS file, and the user query. Next, the CSS
067
      file, which contains the web page layout information, is opened and its contents are loaded into a
068
      temporary variable named css file. The function interact with csv is then called with the user query,
069   the address of the CSV file, and the contents of the CSS file as arguments. This function performs
070   the following steps:
071
072          • Convert CSV to Data-Frame: It uses Pandas to convert the CSV data into a Data-Frame.
073          • Generate Prompt: A prompt is created based on the user query.
074
             • Generate Custom CSS: The prompt is sent to the generate text function, which utilizes the
075            GPT-4 model to generate custom CSS.

110   The state st at time t represents the current layout of the webpage along with the user’s interaction

112
113          • Component positions and sizes (as defined by CSS),

      5.3   ACTIONS ( A )
121
122   The actions at represent the potential modifications to the webpage’s CSS layout. These actions can
123   include:
124
125          • Repositioning components (e.g., moving a button or banner),
126          • Resizing components (e.g., changing the size of images or text areas),
127          • Recoloring or restyling elements (e.g., adjusting font size or colors to improve visibility).
128
129                   a(t) = {layout modification: component resizing, repositioning, etc.}
130
131   5.4   R EWARD (R)
132
133   The reward R(st , at ) is a function that quantifies the success of an action based on the change in
134   user engagement. This reward can be constructed by considering the following factors:
135
             • Maximizing user time on the website,
136
137          • Encouraging specific actions, such as clicks on ads or buttons,
138          • Minimizing the bounce rate (i.e., keeping users engaged and preventing them from leaving),
139          • Increasing overall revenue generated from user interactions.
140
141   You can define the reward function as:
142
      R(st , at ) = α·(user time on website)+β·(click-through rate)−γ·(bounce rate)+δ·(revenue from ads)
143
144   where α, β, γ, and δ are tunable coefficients that represent the relative importance of each metric.
145

      8   R ESULT
176
177
      Reinforcement learning (RL), a subfield of unsupervised learning, provides a goal-driven framework
      ideal for optimizing dynamic website layouts. By integrating supervised learning, using existing user
178
      data for predictions, this approach enhances the accuracy of layout optimization. The combination
179
      allows the system to not only adapt based on user interactions but also predict user preferences,
180   moving toward more effective outcomes.
181
182   While human behavioral patterns serve as core inputs to the model, the reward system plays a crucial
183
      role in guiding layout adjustments. It addresses the challenge of accurate predictions, ensuring that
      the layout aligns with user behavior. This cyclical process continuously refines the user experience,
184
      making the website more responsive to individual preferences.
185
186   Beyond increasing engagement and revenue, this method offers valuable insights into how users
187   interact with components, allowing developers to design layouts that cater more effectively to user
188   attention. As layouts dynamically evolve in response to real-time data, this system fosters a person-
189
      alized user experience, advancing both design quality and user satisfaction.
