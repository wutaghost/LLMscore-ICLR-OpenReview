009                                              A BSTRACT
010
011            Large deep neural networks, often fine-tuned from foundation models, dominate
012            modern machine learning, but their high memory requirements limit deployment
013            on resource-constrained devices. Strong lottery tickets (SLTs) offer a promising
014            solution by significantly reducing memory usage, as they are fully characterized
015            by a seed for generating a random source network and a binary mask. Notably,
016
               multiple models can share the same source network without increasing its width
               requirement. As we show, this source sharing can lead to memory savings when
017
               experts share specific sparsity patterns. Based on novel insights into optimized
018
               subsetsum approximations, we also show how the masks can be adjusted to fur-
019            ther reduce memory overhead. To validate these theoretical findings, we provide
020            explicit SLT constructions in experiments.

023   1   I NTRODUCTION
024
025   The rapid advancements in deep learning have led to the widespread development of large neural
026   network models, particularly those fine-tuned from foundation models (Bommasani et al., 2022).
027
      Such models, which dominate many areas of modern machine learning, are often tasked with solv-
      ing highly specialized problems. However, their immense memory requirements pose a significant
028
      challenge, especially when deploying these models on resource-constrained devices. This is particu-
029
      larly relevant in real-world scenarios, where numerous models might need to interact and collaborate
030   to solve complex tasks. As we move toward an era where multiple AI models must work together
031   seamlessly, the cumulative memory footprint of these models becomes a critical bottleneck.
032
033
      While each model might be specialized to a given task, many are derived from the same or similar
      foundation models, resulting in structural similarities. Such similarities might also occur by design
034
      when some of the model components solve tasks jointly like mixtures of experts (MoEs) (Jiang
035
      et al., 2024; Lepikhin et al., 2020) or ensembles (Tabik et al., 2020; Kilimci et al., 2018). We argue
036   that structural overlaps can be leveraged to optimize memory usage across multiple models or their
037   components and propose concrete strategies to do so in the context of Strong Lottery Tickets (SLTs).
038
039
      SLTs can naturally reduce memory footprints even of single models (Ramanujan et al., 2020; Otsuka
      et al., 2024), as they only need to save a seed for generating a random source network and a binary
040
      mask that selects a subnetwork. An overlooked fact is that this principle (Lueker, 1998) encom-
041
      passes also model families. Multiple models can share the same source network without demand-
042   ing any increase of the width requirement in state-of-the-art lottery ticket existence constructions
043   (Pensia et al., 2020; Burkholz, 2022b; Ferbach et al., 2022). Sharing a source network has several
044   advantages: a) Multiple models can access the same memory of the source network and potentially
045   reuse computations at (parallel) inference time. b) Saving source model weights instead of a seed
046   could solve issues with reproducibility on different devices. In that case, storing a single source is
047   preferable over multiple ones. c) Beyond the overlap of sparsity patterns of target networks, we can
048   optimize subsetsum approximations to gain more memory-efficient mask representations. Sharing
049   the same source increases such memory efficiency.
050   Our analysis is based on novel insights into subsetsum approximation, which forms the basis of
051   SLT constructions and defines the masks that approximate given target models. Utilizing the fact
052   that subsetsum approximations have multiple solutions among which we can choose different can-
053   didates, we discuss alternative selection criteria. The most memory-efficient approach is neither just
      maximizing the overlap of the masks nor their individual sparsity. This implies a trade-off between


                                                        1

054
      the objective to reduce memory and inference speed, which would both be desirable in practice.
055   Furthermore, perhaps counter-intuitively, larger source models support more memory-efficient rep-
056   resentations, thus, inducing a trade-off between the size of the source model and the masks and the
057   joint memory requirements of the masks.
058
      Overall, our contributions can be summarized as follows:
059
060          • Up to our knowledge, we are the first to identify the potential of the Strong Lottery Ticket
061            (SLT) concept to reduce the memory footprint of neural network model families like en-
062            sembles or mixtures of experts (MoEs).
063          • We identify multiple advantages of sharing a single source network across multiple models.
064            In particular, we prove that sharing a source network induces a lower width requirement
065            (on the source) in lottery ticket existence statements.
066          • We obtain conceptual and theoretical insights into subsetsum approximations and utilize
067            them to optimize memory usage across structurally similar models.
068
             • We identify fundamental trade-offs between memory and inference speed in joint SLT con-
069            structions of multiple models.

      Ensembles and mixture of experts The main application of our theory is representing and saving
      families of models, such as ensembles (Ganaie et al., 2021) and mixture of experts (MoEs) (Cai et al.,
075
      2024; Jacobs et al., 1991). Traditional ensemble methods such as bagging (Breiman, 1996), stacking
076
      Wolpert (1992) and boosting (Freund & Schapire, 1996) have been applied to deep ensembles (K.
077   et al., 2005; Liu et al., 2014; Low et al., 2020). leading to better generalization performance, effi-
078   ciency, and less need for stronger models. Nevertheless, neural networks incur high training costs.
079   Proposed alternatives are homogeneous and heterogeneous ensembles, which distribute the data set
080   over the models or use a diverse set of (smaller) models (Tabik et al., 2020; Kilimci et al., 2018). A
081   modern approach is a mixture of experts (MoEs) architecture, contributing to the scaling and success
082   of Large Language Models (LLMs) (Jiang et al., 2024; Lepikhin et al., 2020). However, while MoEs
083   aim to improve inference efficiency, the increasing size of MoE leads to storage issues, for example,
084   when the device capacity is constrained (e.g. in the case of edge devices). Recently, hierarchical
085
      storage solutions have been proposed to alleviate this issue (Yu et al., 2024; Hwang et al., 2024; Yi
      et al., 2023). For example, off-loading inactive experts with selection forecasting has the goal of
086
      minimizing the overhead from data transfer. Moreover, specific GPU kernels have been designed to
087
      address the issue of memory movement and computational efficiency (Rajbhandari et al., 2022; He
088   et al., 2021; Nie et al., 2022; Hwang et al., 2023). To add to this, PIT (Zheng et al., 2023) introduces
089   a sparse tiling mechanism based on permutation equivariance leading to computational savings.
090
091   Model compression Besides infrastructural improvements, dense-to-sparse model training (Ko-
092   matsuzaki et al., 2023), sparse-to-dense model training (Fedus et al., 2022b) and merging experts
093   (Li et al., 2022) also lead to inference and memory savings by reducing the size of the involved
094   models. Model families can also be sparsified jointly according to Allingham et al. (2023). Yet, up
095   to our knowledge, the potential of SLTs to further reduce the memory footprint of such models, has
096   not been explored. Utilizing SLTs, we show it is possible to save multiple models with one seed and
097
      their respective binary masks. Furthermore, we show that the storage of the masks can be optimized
      based on novel insights into subsetsum approximation.

      Strong lottery tickets Many neural network compression algorithms are inspired by the lottery
100
      ticket hypothesis (Frankle & Carbin, 2019), which conjectures the existence of subnetworks of ran-
101
      domly initialized neural networks, the source network, which can approximate a target network if
102   trained in isolation. Ramanujan et al. (2020) found empirical evidence for an even stronger hy-
103   pothesis, the SLT, that there exist subnetworks that do not require any training but pruning the
104   source network is sufficient to solve complex tasks. Malach et al. (2020) proved this hypothesis
105   formally but required a high overparameterization, namely that the source network is wider than
106   a target network by a polynomial factor in the approximation error, which can yield potentially
107   optimally sparse lottery tickets Fischer & Burkholz (2022). Two subsequent works brought this fac-
      tor down to logarithmic overparameterization (Orseau et al., 2020; Pensia et al., 2020) suggesting


                                                         2
      Under review as a conference paper at ICLR 2025


108
      practical relevance of SLT constructions, which could be exploited for reduced memory represen-
109   tations, as mentioned by Otsuka et al. (2024). (Pensia et al., 2020) have achieved their practical
110   bound by utilizing an older theoretical result on subsetsum approximations (Lueker, 1998) in their
111   SLT construction. Subsequent work has extended this proof strategy to cover modern deep neural
112   network architectures with nonzero biases (Fischer et al., 2021), convolutional layers (da Cunha
113   et al., 2022; Burkholz, 2022b) and residual blocks, (Burkholz, 2022b), various activation functions
114   (Burkholz, 2022c), more general equivariant architectures including graph neural networks (Ferbach
115   et al., 2022), specific universal architectures (Burkholz et al., 2022), and construct structured SLTs
116   or more flexible approximations (Xiong et al., 2023). Also, the distribution of source networks has
117
      been generalized to binary (Diffenderfer et al., 2021; Zhou et al., 2019; Sreenivasan et al., 2022) or
      randomly sparse (Gadhikar et al., 2023) weights. Our results apply to all of these architectures and
118
      conditions, as we base our insights on the underlying subsetsum approximation.

120
121   Subsetsum approximation Subsetsum approximation is a fairly old result that states that any
122   target variable in a bounded interval can be approximated as the sum of a subset of a sufficiently
123   large source size (Lueker, 1998), which can be used to prove lottery ticket existence (Pensia et al.,
124   2020). This statement has been generalized to larger target intervals (Burkholz et al., 2022), multi-
125   dimensional targets and sources (Becchetti et al., 2022), randomly dropped source elements (Gad-
126   hikar et al., 2023), epsilon perturbations (Xiong et al., 2023), and partially frozen sources (Otsuka
127
      et al., 2024). In this work, we analyze the properties of the approximating subsets and exploit our
      flexibility in choosing candidates that reduce the joint memory requirements of multiple targets.

131   3   SLT EXISTENCE WITH SHARED VERSUS SEPARATE SOURCES
132
133   Our first objective is to prove the existence of strong lottery tickets (SLTs) for a family of target
134   models such as MoEs and ensembles. A largely overlooked fact is that current lottery ticket con-
135   structions already support simultaneously all possible target models that are smaller than a specific
136   upper bound on their size (width and depth) if they share the same random source model, which is
137   pruned to approximate our target models. As a consequence, we only have to store the corresponding
138   masks of each target model and the seed of the random source model generation.
139   Thus, we only have to compare this setting to the alternative, where we use separate source net-
140   works for each expert model. At first glance, separate sources could have the advantage that the
141   corresponding models can be developed independently and do not require synchronization of mul-
142   tiple servers, for instance, if they are trained on different machines (and data potentially). Saving
143   multiple seeds for different source models incurs only negligible additional memory and the com-
144   putational overhead of generating multiple source models can be parallelized.
145   So why do we still propose to share the same source network? As we prove, the first disadvantage
146   of separate sources is that this increases our bound on the source width. Thus, each of the multiple
147   source models would need to be larger – to ensure the success of a higher number of subsetsum
148   approximations in the independent LT constructions. However, the width factor is only logarithmic
149   in the number m of experts log(m), but this can still be significant if our target experts are large.
150   More importantly, we show that sharing a source model allows us to find more memory-efficient
151
      lottery tickets. Furthermore, overlapping target sparsity patterns can become more easily exploited.
      To back these statements up theoretically, we present an analytically tractable comparison of sub-
152
      setsum representations of multiple targets that highlights the memory benefits of sharing the same
153
      source. Moreover, we show that sparsifying each target mask individually is suboptimal and propose
154   another criterion to find more memory efficient representations of multiple targets.

283   Let us consider a variant of the subsetsum approximation problem with m target models of length
284   p. As we saw in Theorem 3.1, we can approximate each target of a model with the same source
285   set. We will now have p source sets of size n and m target models. This gives us a target matrix
                                                                                     p×n
286   Z ∈ [− 12 , 12 ]p×m , mask M ∈ {0, 1}p×m×n and source networks X ∈ U [−1, 1]       . Then we want
287   to optimize the subsetsum problem for each target, given a source network. We refer to this problem
288   as the multi-subsetsum approximation problem.
289
290   Improving memory savings In the previous section, we have seen how that we only need a single
291   source set to represent m different targets with no additional width requirements. Since the widely
292
      used random number generators are deterministic with respect to the seed s, we only need s and
      the binary subset masks to reconstruct target values to a certain precision ϵ. Since binary matri-
293
      ces require lesser memory compared to floating point parameters of the model, this is an efficient
294
      alternative to compress multiple models of the same architecture during deployment. In addition,
295   efficient algorithms like bitwise compression, run-length encoding, etc. can be used to efficiently
296   store sparse binary matrices. Having sparse matrices can improve the memory savings and also
297   lower the inference time computations.
298
      To save memory, we prefer smaller source set sizes n, sparser masks M , and also care about a low
299
      computational overhead to reconstruct a model. The choice of the source set size n depends on the
300
      permissible target error ϵ and the probabilistic error of finding a satisfying subset δ. A s ϵ, δ decrease,
301   the necessary source set size that satisfies the errors increases. Together, the errors have an impact
302   on how identical the performance of the approximated network is to the target network. At the same
303   time, for a given Xi for i ∈ [p], several subsets Mi,j for j ∈ [m] satisfy the target requirement. As
304   n increases, so do the choices of Mi,j . Based on how we choose subsets from the choices we can
305   gain further memory improvements. We will next discuss different storage formulations and then
306   evaluate approaches that pick optimal subsets according to different metrics and as a result, increase
307   the sparsity of M and reduce the number of operations to reconstruct the model.
308

311
      plane, the bits common to all m planes are set. And m extra planes contain the remaining set bits.

517   6   D ISCUSSION
518
519   We investigate the potential of the strong lottery ticket (SLT) concept to reduce the joint memory
520   footprint of model families that have similar deep neural network architectures and have SLT ap-
521   proximations, which can be subnetworks of the same source network. If the models overlap in
522   their sparsity patterns in addition, they can be further exploited for compressed representations, if
523   they share the same source network in the SLT construction, as we have established theoretically.
524
      Beyond similarity in the original parameterization, we have propose and analyze multiple criteria
      to select jointly optimized masks that characterize individual models, which we derive from novel
525
      insights into subset-sum approximation. This enabled us to identify fundamental trade-offs between
526
      computationally relevant objectives like the inference cost and memory footprint. At first consider-
527   ation surprisingly, we further find that larger source sizes, which imply larger masks, can achieve
528   higher memory savings, as they offer more masks to choose from during the optimization. Beyond
529   a certain source size threshold, however, we expect that this effect saturates so that increasing the
530   overparameterization further can only have a negative impact. Yet, this threshold is not attainable
531   in a computationally effective way because the space of all potential subsetsum approximations in-
532   creases exponentially in the source set size. However, we identify potential for already great memory
533   improvements for moderate, practical sizes of source sizes. With our analysis, we have thus laid the
534   theoretical basis for utilizing the SLT principle to obtain memory savings of multiple large-scale
535
      models.

895   C.2   C OMPUTATION TIME AND STORAGE EFFICIENCY
896
897
      Traditional strong lottery ticket finding process involves sampling the weights of an entire network at
      random and finding masks that approximate target parameter values by performing multiple subset-
898
      sum approximation operations(SSA). We mimic the same and sample different source sets randomly
899
      for each target parameter vector of size m=8. A multi-subset-sum approximation(MSSA) is then
900   performed to identify ideal subsets that optimize some objective(sparsest individual subsets, total
901   set bits length in (m + 1)-scheme, etc.).
902
903
      Time complexity of algorithms: MSSA comprises two operations. Firstly, we find candidate sets
      that approximate each of m targets by parsing through the 2n possible subsets. The time complexity
904
      of repeating this for each of p target vectors in an MoE layer is O(p.2n ). Secondly, we choose the
905
      ideal subset. Finding the sparsest individual subsets like in the case of P artition and Local − bin
906   does not incur any comparable time complexity. Whereas, Local−bin+opt incurs a lot of additional
907   overhead that scales exponentially with the number of experts m. Let c be the average number of
908   candidates that approximate each target to ϵ. Then the additional overhead is O(cm ). The encoding
909   times per an 8 expert MoE layer observed for the algorithms and their corresponding number of set
910   bits are shown in Table 2. To ensure a fair comparison between algorithms we consider them in
911   the memory storage scheme that they are tailored for. Therefore, for P artition&Local − bin, we
912   follow (m)-scheme and for Local − bin + opt, we follow (m + 1)-scheme.
913   As we see in Table 2, both operations are costly over millions of parameters. Especially, comparing
914   across candidate sets of 8 targets to find subsets that optimize total length in (m + 1)-scheme seems
915   to be more computationally intensive than generating candidate sets. But as a result, we do observe
916   an improvement in the number of set bits per parameter. While we acknowledge that 4hrs. is
917   a significant amount of time to encode, it is still negligible compared to the hundred/millions of
      GPU hours that large foundation models train for(Marcińczuk (2023)). Even finetuning them takes

919   be performed once.

      Storage Compression Formats: To analyze the extent to which the mask sparsity and similarity
      can be exploited, we compare four different compression/file storage schemes. They are:
942
943
            • Float-32: This is the default way models are saved where each parameter is stored as 32-bit
944           float data. This will serve as the baseline.
945
946
            • N-bit Mask: The n-bit masks, that are identified by the multi-subset-sum approximation
              over a source set of size n, and the associated seed are saved.
947
948         • Huffman Encoded - Set Bit Positions(HE-SBP): The main goal of this storage format is
949           to save the positions of set bits within the n-bit mask efficiently and to exploit the sparsity
950           of the binary masks. We use Huffman encoding to optimize the binary code representation
951
              of each of the n bit positions based on the frequency with which they occur across all
              parameters in an MoE layer. It is important to note that a delimiter token must also be
952
              included within this style of encoding since the number of set bits within approximating
953
              subsets of different parameters can vary. Within the encoded file, a delimiter marks the end
954           of a list of set bit positions applicable to the parameter that is being parsed. HE-SBP can be
955           applied to store masks following either the (m)-scheme or the (m + 1)-scheme and any of
956           the three proposed algorithms. It is important to note that the code block that compresses
957           a target vector of size m would have m delimiter tokens in the (m) − scheme, whereas,
958           (m + 1) delimiter tokens in the (m + 1)-scheme. To decode, one must save the seed, the
959           encoded message as well as the Huffman tree that was used.
960         • Huffman Encoded - Bin Representatives(HE-BR): Using Huffman encoding, this
961           method encodes the unique approximating subset masks directly. It couples well with
962           the masks found by the Partition algorithm operating on a single source set for all target
963           parameter vectors. This is because the number of unique approximating subset masks are
964           restricted (= 1/epsilon bin representatives). Its compression performance suffers as the
965           number of unique approximating subsets increases.
