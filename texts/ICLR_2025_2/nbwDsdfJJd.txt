029   1    I NTRODUCTION
030
031   Estimating string distances is one of the most fundamental problems in computer science and
032   information theory, with rich applications in high-dimensional geometry, computational biology and
033   machine learning. The problem could be generically formulated as follows: given a collection of
034   strings A1 , . . . , Am ∈ Σn where Σ is the alphabet, the goal is to design a data structure to preprocess
035   these strings such that when a query B ∈ Σn is given, the data structure needs to quickly output
036   estimates of ∥Ai − B∥ for all i ∈ [m], where ∥ · ∥ is the distance of interest. Assuming the symbols in
037   Σ can allow constant time access and operations, a naı̈ve implementation would be to simply compute
038   all the distances between Ai ’s and B, which would require O(mn) time. Designing data structures
039
      with o(mn) query time has been the driving research direction in string distance estimations. To
      make the discussion concrete, in this work we will focus on binary alphabet (Σ = {0, 1}) and for
040
      distance, we will study Hamming and edit distance. Hamming distance (Hamming, 1950) is one
041
      of the most natural distance measurements for binary strings, with its deep root in error detecting
042   and correction for codes. It finds large array of applications in database similarity searches (Indyk
043   & Motwani, 1998; Charikar, 2002; Norouzi et al., 2012) and clustering algorithms (Huang, 1997;
044   Huang & Ng, 1999).
045
      Compared to Hamming distance, edit distance or the Levenshtein distance (Levenshtein, 1966) could
046
      be viewed as a more robust distance measurement for strings: it counts the minimum number of
047
      operations (including insertion, deletion and substitution) to transform from Ai to B. To see the
048   robustness compared to Hamming distance, consider Ai = (01)n and B = (10)n , the Hamming
049   distance between these two strings is n, but Ai could be easily transformed to B by deleting the first
050   bit and adding a 0 to the end, yielding an edit distance of 2. Due to its flexibility, edit distance is
051   particularly useful for sequence alignment in computational biology (Wang et al., 2015; Young et al.,
052   2021; Berger et al., 2021), measuring text similarity (Navarro, 2001; Sidorov et al., 2015) and natural
053   language processing, speech recognition (Fiscus et al., 2006; Droppo & Acero, 2010) and time series
      analysis (Marteau, 2009; Gold & Sharir, 2018).

      In addition to data structures with fast query times, another important consideration is to ensure the
055   database is secure. Consider the scenario where the database consists of private medical data of m
056   patients, where each of the Ai is the characteristic vector of n different symptoms. A malicious
057   adversary might attempt to count the number of symptoms each patient has by querying 0n , or
058   detecting whether patient i has symptom j by querying ej and 0n where ej is the j-th standard basis
059   in Rn . It is hence crucial to curate a private scheme so that the adversary cannot distinguish the case
060   whether the patient has symptom j or not. This notion of privacy has been precisely captured by
061   differential privacy (Dwork, 2006; Dwork et al., 2006), which states that for neighboring databases1 ,
062   the output distribution of the data structure query should be close with high probability, hence any
063
      adversary cannot distinguishable between the two cases.
064   Motivated by both privacy and efficiency concerns, we ask the following natural question:
065
066      Is it possible to design a data structures to estimate Hamming and edit distance, that are both
067                               differentially private, and time/space-efficient?

095   Before diving into the details, we would like to make several remarks regarding our data structure
096   results. Note that instead of solving the exact Hamming distance and edit distance problem, we
097   impose the assumption that the query B has the property that for any i ∈ [m], ∥Ai − B∥ ≤ k. Such
098   an assumption might seem restrictive at its first glance, but under the standard complexity assumption
099   Strong Exponential Time Hypothesis (SETH) (Impagliazzo & Paturi, 2001; Impagliazzo et al.,

      Regarding privacy guarantees, one might consider the following simple augmentation to any fast
113
      data structure for Hamming distance: compute the distance estimate via the data structure, and add
114
      Laplace noise to it. Since changing one coordinate of the database would lead to the Hamming
115   distance change by at most 1, Laplace mechanism would properly handle this case. However, our
116   goal is to release a differentially private data structure that is robust against potentially infinitely
117   many queries, and a simple output perturbation won’t be sufficient as an adversary could simply
118   query with the same B, average them to reduce the variance and obtain a relatively accurate estimate
119   of the de-noised output. To address this issue, we consider the differentially private function release
120   communication model (Hall et al., 2013), where the curator releases an ϵ-DP description of a function
121   eb(·) that is ϵ-DP without seeing any query in advance. The client can then use eb(·) to compute eb(B)
122   for any query B. This strong guarantee ensures that the client could feed infinitely many queries to
123
      eb(·) without compromising the privacy of the database.

      2    R ELATED W ORK
127
128   Differential Privacy. Differential privacy is a ubiquitous notion for protecting the privacy of
129   database. Dwork et al. (2006) first introduced this concept, which characterizes a class of algorithms
130   such that when inputs are two neighboring datasets, with high probability the output distributions are
131
      similar. Differential privacy has a wide range of applications in general machine learning (Chaud-
      huri & Monteleoni, 2008; Williams & McSherry, 2010; Jayaraman & Evans, 2019; Triastcyn &
132
      Faltings, 2020), training deep neural networks (Abadi et al., 2016; Bagdasaryan et al., 2019), com-
133
      puter vision (Zhu et al., 2020; Luo et al., 2021; Torkzadehmahani et al., 2019), natural language
134   processing (Yue et al., 2021; Weggenmann & Kerschbaum, 2018), large language models (Gao et al.,
135   2023; Yu et al., 2022), label protect (Yang et al., 2022), multiple data release (Wu et al., 2022),
136   federated learning (Sun et al., 2023; Song et al., 2023a) and peer review (Ding et al., 2022). In
137   recent years, differential privacy has been playing an important role for data structure design, both in
138   making these data structures robust against adaptive adversary (Beimel et al., 2022; Hassidim et al.,
139   2022; Song et al., 2023b; Cherapanamjeri et al., 2023) and in the function release communication
140   model (Hall et al., 2013; Huang & Roth, 2014; Wang et al., 2016; Aldà & Rubinstein, 2017; Coleman
141   & Shrivastava, 2021; Wagner et al., 2023; Backurs et al., 2024).
142
143   Hamming Distance and Edit Distance. Given bit strings A and B, many distance measurements
144   have been proposed that capture various characteristics of bit strings. Hamming distance was first
145   studied by Hamming (Hamming, 1950) in the context of error correction for codes. From an
146   algorithmic perspective, Hamming distance is mostly studied in the context of approximate nearest-
147   neighbor search and locality-sensitive hashing (Indyk & Motwani, 1998; Charikar, 2002). When it
148   is known that the query B has the property Dham (A, B) ≤ k, Porat & Lipsky (2007) shows how
149   to construct a sketch of size O(k)
                                       e   in O(n)
                                               e      time, and with high probability, these sketches preserve
150   Hamming distance. Edit distance, proposed by Levenshtein (Levenshtein, 1966), is a more robust
151   notion of distance between bit strings. It has applications in computational biology (Wang et al.,
152
      2015; Young et al., 2021; Berger et al., 2021), text similarity (Navarro, 2001; Sidorov et al., 2015) and
      speech recognition (Fiscus et al., 2006; Droppo & Acero, 2010). From a computational perspective,
153
      it is known that under the Strong Exponential Time Hypothesis (SETH), no algorithm can solve edit
154
      distance in O(n2−o(1) ) time, even its approximate variants (Belazzougui & Zhang, 2016; Chakraborty
155
      et al., 2016a;b; Naumovitz et al., 2017; Rubinstein et al., 2019; Rubinstein & Song, 2020; Goldenberg
156   et al., 2020; Jin et al., 2021; Boroujeni et al., 2021; Kociumaka et al., 2021; Bhattacharya & Kouckỳ,
157   2023; Kouckỳ & Saks, 2024). Hence, various assumptions have been imposed to enable more
158   efficient algorithm design. The most related assumption to us is that Dedit (A, B) ≤ k, and in this
159   regime various algorithms have been proposed (Ukkonen, 1985; Myers, 1986; Landau & Vishkin,
160   1988; Goldenberg et al., 2019; Kociumaka & Saha, 2020; Goldenberg et al., 2023). Under SETH,
161   it has been shown that the optimal dependence on n and k is O(n + k 2 ), up to sub-polynomial
      factors (Goldenberg et al., 2023).

      3.2   D IFFERENTIAL P RIVACY
185
186   Differential privacy (DP) is the key privacy measure we will be trying to craft our algorithm to possess
187   it. In this paper, we will solely focus on pure DP (ϵ-DP).
188
      Definition 3.3 (ϵ-Differential Privacy). We say an algorithm A is ϵ-differentially private (ϵ-DP) if for
189   any two neighboring databases D1 and D2 and any subsets of possible outputs S, we have
190
191                                  Pr[A(D1 ) ∈ S] ≤ eϵ · Pr[A(D2 ) ∈ S],
192   where the probability is taken over the randomness of A.
193
194   Since we will be designing data structures, we will work with the function release communication
195   model (Hall et al., 2013) where the goal is to release a function that is ϵ-DP against any sequence of
196   queries of arbitrary length.
197   Definition 3.4 (ϵ-DP Data Structure). We say a data structure A is ϵ-DP, if A is ϵ-DP against any
198   sequence of queries of arbitrary length. In other words, the curator will release an ϵ-DP description
199   of a function eb(·) without seeing any query in advance.
200
201   Finally, we will be utilizing the post-processing property of ϵ-DP.
202   Lemma 3.5 (Post-Processing). Let A be ϵ-DP, then for any deterministic or randomized function g
203   that only depends on the output of A, g ◦ A is also ϵ-DP.

      To start off, we introduce our data structure for differentially private Hamming distance. In particular,
208   we will adapt a data structure due to Porat & Lipsky (2007): this data structure computes a sketch of
209
      length O(k)
               e     bit string to both the database and query, then with high probability, one could retrieve
210   the Hamming distance from these sketches. Since the resulting sketch is also a bit string, a natural
211   idea is to inject Laplace noise on each coordinate of the sketch. Since for two neighboring databases,
212   only one coordinate would change, we could add Laplace noise of scale 1/ϵ to achieve ϵ-DP. However,
213   this approach has a critical issue: one could show that with high probability, the magnitude of each
214   noise is roughly O(ϵ−1 log k), aggregating the k coordinates of the sketch, this leads to a total error
215   of O(ϵ−1 k log k). To decrease this error to O(1), one would have to choose ϵ = k log k, which is too
      large for most applications.
