009                                                A BSTRACT
010
011            Standard supervised learners attempt to learn a model from a labeled dataset.
012            Given a small set of labeled instances, and a pool of unlabeled instances, a bud-
013            geted learner can use its given budget to pay to acquire the labels of some unla-
014            beled instances, which it can then use to produce a model. Here, we explore bud-
015            geted learning in the context of survival datasets, which include (right) censored
016
               instances, where we know only a lower bound ci on that instance’s time-to-event
               ti . Here, that learner can pay to (partially) label a censored instance – e.g., to
017
               acquire the actual time ti for an instance [e.g., go from (3yr, censor) to (7.2yr,
018
               uncensored)], or other variants [eg, learn about 1 more year, so go from (3yr,
019            censor) to either (3.2yr, uncensored) or (4yr, censor)]. This serves as a model
020            of real world data collection, where follow-up with censored patients does not
021            always lead to uncensoring, and how much information is given to the learner
022            model during data collection is a function of the budget and the nature of the data
023            itself. Many fields – such as medicine, finance, and engineering – contain survival
024            datasets with a large number of censored instance, and also operate under budget
025            constraints with respect to the learning process, thus making it important to be
026            able to apply this budgeted learning approach. Despite this importance, very few
027
               other projects have explored this. We provide both experimental and theoretical
               results for how to apply state-of-the-art budgeted learning algorithms to survival
028
               data and the respective limitations that exist in doing so. Our approach provides
029
               bounds and time complexity asymptotically equivalent to m: the standard active
030            learning methodm: sBatchBALD. Moreover, empirical analysis on several survival
031            tasks show that our model performs better than other potential approaches on sev-
032            eral benchmarks.

061   obtain the labels m: forof) in order to acquire an effective model, at minimal cost (Ren et al., 2021).
062   Its goal is to minimize the amount of data needed to reach a target accuracy while also enabling the
063   model to achieve higher accuracy as quickly as possible, saving time and compute (less compute is
064   needed to train the model m: as there exists less samples in the trainingbecause fewer samples are included

066
      AL has made significant strides as a field, yet its definition remains vague. Researchers often train
067   models until “convergence”, but this term lacks a clear definition, as it relies on arbitrary thresholds.
068   Typically, convergence is declared when the loss changes minimally (m: by an epsilonϵ) over a certain
069   number of steps (patience) (Ren et al., 2021). This imprecision arises because both ϵ and patience
070   are heuristically chosen and can vary by task and model, resulting in an inconsistent understanding
071   of true convergence.
072

073   model loss, offering a more precise framework for active learning (Lizotte et al., 2003). Budgeted
074   learning attempts to choose the best instances for the model given a budget constraint. If all data
075   instances had the same cost and the task was to minimize the budget m: needed rather than work within a
076   given budget, we would have ALrather than working within a predefined one, we would essentially have
077   AL.. Budgeted learning is more attuned to real world problems where a predefined budget is oftenm:
078   initially given prior to data collection. We show that once this problem is solved, one can easily also
079   account for real-world situations where data instances may potentially cost different amounts.
080   Traditional AL makes several assumptions that often don’t hold in real-world scenarios. We have

478   7     E XPERIMENTS
479
480   For each dataset, we first divide it into training and test sets. We assign the labels to time bins as
481   quantiles of the time to event label. We use the same bins as the training data to assign the labels of
482   the test data. We artificially censor points in the training data so that we can decensor them in our
483   experiments. Hiding information from the model to test its performance is common for uncensored
484   data, however, we can do this for censored data as well. For censored data, we can artificially
485   “further” censor them for this purpose, the only difference is that they cannot be decensored until
      the time to event.


                                                              9
      Under review as a conference paper at ICLR 2025


486
      To artificially censor the dataset, a number of data instances in the training data are further censored
487   at a uniformly random time between 0 and their current “true time” — regardless of whether that
488   time is censored or uncensored. This process assigns each instance a “fake time” after the additional
489   censoring. The true time for each instance is kept hidden from the model (only the oracle can see
490   it). Instances where the fake time is less than the true time are considered queryable, meaning
491   the m: y can provide new information when decensoredoracle can provide new information about the in-
492   stance when queried. On the other hand, instances from which no new information can be gained
493   are unqueryable. m: In the dataset, there is a column called censored, which traditionally holds a
494   value of 1 or 0 in standard Active Learning (AL) settings. However, in our setup, this column may
495
      also take on the value -1. Specifically, a value of 1 indicates that the time of the event is known, 0
      signifies that the time is not yet known but could potentially be determined with further querying,
496
      and -1 implies that the time is unknown and cannot be obtained through additional queries (e.g., in
497
      the case of a subject being hit by a bus).
498
499   At the time of querying, the current fake time is compared to the hidden true time by the oracle
500   and updated accordingly. When an instances time becomes decensored by the oracle, it can still be
501
      queryable if the resulting fake time is still less than the true time. But, if the fake time is now equal
      to the true time, that instance is no longer able to be queried.

610   9   C ONCLUSION
611
612   In this paper, we discussed a generalized form of active learning that incorporates budget constraints
613   and, when necessary, accounts for the individual costs of queried instances. We explored methods
614   to extend acquisition functions for use with censored data and to account for scenarios where only
615   partial information is gained during queries. Our proposed method was evaluated across three real-
616
      world datasets; however, we anticipate that this emerging area of research will inspire many future
      studies.
617
618   In particular, one promising direction is to combine BBsurv with a semi-supervised approach. As
619   noted in Kirsch et al. (2019), such an approach may enhance performance, and given the success
620   of MCtH, we believe this is a worthwhile avenue to explore. Additionally, it would be valuable
621   to revisit some of our assumptions, such as whether the increment changes over time or if data is
622
      censored in ways other than randomly.
623   Finally, there is a substantial body of literature on methods for approximating the maximum cover-
624   age problem. Alternative approximation schemes beyond the greedy approach may prove advanta-
625   geous and warrant further investigation in future work.

      A.1   R EDUCING BATCH S ELECTION TO W EIGHTED M AX C OVER
759
760   The maximum coverage problem is a well known combinatorial problem that involves choosing k
761   sets, from a group of N sets of integers. The task is to choose the k sets whose union is maximal.
762   For example if the sets are as follows: S1 = {1, 2, 3}, S2 = {2, 3, 4}, S3 = {4, 5}, S4 = {6},
763   where here N=4 and k=2, then the optimal choice of sets here are sets S1 and S3 , as their union
764   {1, 2, 3, 4, 5} is larger than the union of S1 and S2 : {1, 2, 3, 4} or S2 and S3 = {2, 3, 4, 5}, and all
765   pairs with S4 make at max 4. This problem is known to be np-hard to solve optimally.
766   There is an extension of this problem called the ”weighted” maximum cover problem where every-
767   thing is the same except each integer has attached to it a weight, and the goal is now to maximize
768   the sum of the weight of the union rather than simply the size of the union. In the example above, if
769   the integers 1,2,3,4, and 5 all had weight 1, but 6 had weight 10, then now we certainly would wish
770   to include S4 as part of one of the sets we choose, in this case you could choose S1 and S4 , or S2
771   and S4 , as both would give you a total highest weight of 13.
772   If you have a set of data points D, then each data point di ∈ D, provides some amount of informa-
773   tion. Any two data points di and dj also have an intersection to consider. Indeed for any batch of
774   data points,we consider information as area that is covered by an area in the information space.
775
      If we now label each unique intersection as an integer, then we now have a bunch of sets, each
776   filled with integers. Furthermore we can give each integer a weight as the amount of area they cover
777   in information space (the amount of information they are expected to give the model). In the batch
778   active learning case, the task now becomes choosing the k data points (which are sets in this case) out
779   of the N total data points that give maximal information cover maximal area). Thus we can simplify
780   the active learning problem using this formulation of mutual information provided by BatchBALD
781   into the weighted maximum coverage problem discussed above.
782   This representation allows us to see that this problem, at least with the information given, is NP-hard
783   as the maximum coverage probelm is NP-hard. In future works maybe it turns out that most datasets
784   have an underlying structure that makes batch selection easier, however in this general case we can
785   argue NP-hardness This is a very useful representation of the problem as it allows an easier way to
786   think about the machine learning problem as a simpler combinatorial one. Furthermore there exists
787   literature in this field that we can use.
788   One such insight from literature is that if the information function is a submodular function, then
789   the greedy algorithm does very well.
790
791
      A set function f : 2N → R defined on the subsets of a finite set N is called submodular if for every
      A, B ⊆ N ,
792
                               f (A) + f (B) ≥ f (A ∪ B) + f (A ∩ B).
793
794
      Equivalently, f is submodular if it satisfies the diminishing returns property: for every A ⊆ B ⊆ N
795   and x ∈ N \ B,
796
                              f (A ∪ {x}) − f (A) ≥ f (B ∪ {x}) − f (B) .
797
798   The greedy approach (Khuller et al., 1999), gives the highest known guaranteed lower bound of all
799   polynomial time approximation schemes (Khuller et al., 1999). In fact proving there is a polynomial
800   time approximation scheme that achieves a higher lower bound is equivalent to proving P = NP.
801
      The greedy algorithm achieves a lower bound of 1-1/e which is about 63% of the optimal. And
802
      indeed is the algorithm used in BatchBALD for selecting it’s batch.
803
804   Another advantage of this formulation is it allows us extend to the non-uniform case. There is
805   an extension of the weighted maximum cover problem known as the ”budgeted” maximum cover
806
      problem where along with the constrained from the weighted problem a budget is also given and
      each set has a given cost attached.
807
808   Khuller et al. (1999) have provided a modified greedy algorithm to the budgeted problem that also
809   meets the lower bound of 1-1/e. They further showed that finding a polynomial time algorithm that
      achieves a better lower bound in this setting would be equivalent to proving N P ⊆ Dtime(nlog log n ).

838     In our own notation, the pool of points S = Dpool , and the weights wi is estimated using our ac-
839   quisition function aBBsurv . k is meant to be a parameter chosen by the user where a higher k yields
840   better performance however at a higher computational cost. We can take k=3 which is the low-
841   est k that guarantees the optimal 1 − 1/e bound. This new greedy algorithm is very computationally
842   expensive. The first part of the algorithm relies on finding the set out of all sets of size less than k = 3
843
      size (via brute force) that maximizes the weight, and assigning it to H1 which has a complexity of
      O(n2 ) if we denote n = |Dpool |. The rest of the algorithm involves for every possible initial set
844
      of size k = 3 instances, greedily adding instances to this selected batch based off of the ratio of
845
      the weight to cost of an instance, which is of order O(n3 ). We argue that for the purposes of
846   our settings, we do not need the first step and we can greatly reduce the complexity of the second
847   step. The arguments we are making are not guaranteed or proven, however it has shown results
848   experimentally and serves as a strong approximation of this algorithm in most settings.
849
      For the first part of the algorithm, in deep learning models, often two or 3 points are not significant
850
      enough to significantly alter the models loss Domingos (2012). The first part of the algorithm is
851
      only relevant if it turns out that a set of size 3 points gives more information than a set of larger size.
852   In deep learning models all data instances provide some information, and when accounting for the
853   weight to cost ratio, it is very likely that the points within H1 are also selected in sets of larger size.
854   Thus we omit this step in the algorithm, especially with larger budgets it is very unlikely to help.
855
      The second part of the algorithm which involves for every possible initial set of size k = 3 instances,
856
      greedily adding instances to this selected batch based off of the ratio of the weight to cost of an
857
      instance. The main part that dramatically increases the computational complexity here is having
858   to consider all n3 starting positions. In Khuller et al. (1999) they discuss one main setting where
859   considering all possible starting states is useful, is if the points you are choosing from are dense in
860   information space, that is that the sets share a very high amount of similar elements between them
861   and learning about any one instance implies you learn about the larger group. In our setting, we start
862   out with a small number of instances and wish to learn about a complex function that covers a large
863   amount in information space, the instances in the pool we can learn from are often very sparse and
      give us vastly different information from different parts of the information space rather than all being

865   multiple instances from a clump of points but rather want to spread out your queried instances. Thus
866   considering all possible starting points is not efficient and we recommend omiting it all together.

871   A.3    C HOOSING THE S AME I NSTANCE M ULTIPLE T IMES IN O NE Q UERY
872
      Since in our setting the Oracle does not give full information. It opens up the question as to if you
873
      can choose the same instance multiple times in one query. For example, if the Oracle gives 5 years of
874
      new information for Alice, you might want to ask the Oracle to give you 10 years but for double the
875   cost of course. This is not often seen in the real world, perhaps if you are working with a software
876   that has all the instances but only gives incremental updates. Or maybe when dealing with ”label
877   delays”, covered in Dedja et al. (2023).
878
      Fortunately, our formulation allows for generalization over this as well. Instead of considering how
879
      many times we should ask for the ”Alice” instance, we can see it such that asking for two increments
880
      of Alice, is a seperate instance with its own cost as asking for one increment of Alice. Furthermore,
881   the entire information that can be gained from for the one increment instance is contained in asking
882   for the 2 increment instance. The question now becomes is the extra information work the cost?
883   Since the costs between these two instances is different, we again have a case of the budgeted
884   maximum coverage problem, for which we show in the paper that there already exists an optimal
885   approximation greedy algorithm for this setting. A final note is that there is never an incentive to
886   ask for both the Alice with 2 increments instance and the Alice with 1 increment instance, but the
887   optimal greedy algorithm does not assume this and may take this action. Thus we can actually
888   create an even better algorithm than greedy in this specific case by only allowing it to ever choose
889
      the maximum number of increments instance each time.
