                                                 A BSTRACT
008
009
               Neural algorithmic reasoning aims to capture computations with neural networks
010
               via learning the models to imitate the execution of classic algorithms. While
011            common architectures are expressive enough to contain the correct model in the
012            weights space, current neural reasoners are struggling to generalize well on out-
013            of-distribution data. On the other hand, classic computations are not affected by
014            distributional shifts as they can be described as transitions between discrete com-
015            putational states. In this work, we propose to force neural reasoners to maintain
016            the execution trajectory as a combination of finite predefined states. To achieve
017            that, we separate discrete and continuous data flows and describe the interaction
018            between them. Trained with supervision on the algorithm’s state transitions, such
019
               models are able to perfectly align with the original algorithm. To show this,
               we evaluate our approach on multiple algorithmic problems and get perfect test
020
               scores both in single-task and multitask setups. Moreover, the proposed architec-
021
               tural choice allows us to prove the correctness of the learned algorithms for any
022            test data.

025   1   I NTRODUCTION
026
027   Learning to capture algorithmic dependencies in data and to perform algorithmic-like computations
028   with neural networks are core problems in machine learning, studied for a long time using vari-
029   ous approaches (Roni Khardon, 1994; Graves et al., 2014; Zaremba & Sutskever, 2014; Reed &
030   De Freitas, 2015; Kaiser & Sutskever, 2015; Veličković et al., 2020b).
031   Neural algorithmic reasoning (Veličković & Blundell, 2021) is a research area focusing on building
032   models capable of executing classic algorithms. Relying on strong theoretical guarantees of algo-
033   rithms to work correctly on any input of any size and distribution, this setting provides unlimited
034   challenges for out-of-distribution generalization of neural networks. Prior work explored this setup
035   using the CLRS-30 benchmark (Veličković et al., 2022), which covers classic algorithms from the
036   Introduction to Algorithms textbook (Cormen et al., 2009) and uses graphs as a universal tool to
037   encode data of various types. Importantly, CLRS-30 also provides the decomposition of classic al-
038
      gorithms into subroutines and simple transitions between consecutive execution steps, called hints,
      which can be used during training in various forms.
039
040   The core idea of the CLRS-30 benchmark is to understand how neural reasoners generalize well
041   beyond the training distribution, namely on larger graphs. Classic algorithms possess strong gen-
042   eralization due to the guarantee that correct execution steps never encounter ‘out-of-distribution’
043   states, as all state transitions are predefined by the algorithm. In contrast, when encountering in-
044
      puts from distributions that significantly differ from the train data, neural networks are usually not
      capable of robustly maintaining internal calculations in the desired domain. Consequently, due to
045
      the complex and diverse nature of all possible data that neural reasoners can be tested on, the gen-
046
      eralization performance of such models can vary depending on particular test distribution (Mahdavi
047   et al., 2023).
048
049   Given that, it is becoming important to interpret internal computations of neural reasoners to find
050
      errors or to prove the correctness of the learned algorithms (Georgiev et al., 2021).
051   Interpretation methods have been actively developing recently due to various real-world applications
052   of neural networks and the need to debug and maintain systems based on them. Especially, the
053   Transformer architecture (Vaswani et al., 2017) demonstrates state-of-the-art performance in natural
      language processing and other modalities, representing a field for the development of interpretability


                                                        1
      Under review as a conference paper at ICLR 2025


054
      methods (Elhage et al., 2021; Weiss et al., 2021; Zhou et al., 2024; Lindner et al., 2024). Based on
055   active research on a computational model behind the transformer architecture, recent works propose
056   a way to learn models that are fully interpretable by design (Friedman et al., 2023).
057
058
      We found the ability to design models that are interpretable in a simple and formalized way to be
      crucial for neural algorithmic reasoning as it is naturally related to the goal of learning to perform
059
      computations with neural networks.
060
061   In this paper, we propose to force neural reasoners to follow the execution trajectory as a combina-
062   tion of finite predefined states, which is important for both generalization ability and interpretability
063   of neural reasoners. To achieve that, we start with an attention-based neural network and describe
064
      three building blocks to enhance its generalization abilities: feature discretization, hard attention
      and separating discrete and continuous data flows. In short, all mentioned blocks are connected:
065
066          • State discretization does not allow the model to use complex and redundant dependencies
067            in data;
068
             • Hard attention is needed to ensure that attention weights will not be annealed for larger
069            graphs. Also, hard attention limits the set of possible messages that each node can receive;
070
             • Separating discrete and continuous flows is needed to ensure that state discretization does
071
               not lose information about continuous data.
072
073   Then, we build fully discrete neural reasoners for different algorithmic tasks and demonstrate their
074   ability to perfectly mimic ground-truth algorithm execution. As a result, we achieve perfect test
075   scores on the multiple algorithmic tasks with guarantees of correctness on any test data. Moreover,
076   we demonstrate that a single network is capable of executing all covered algorithms in a multitask
077   manner with perfect generalization too.
078   In summary, we consider the proposed blocks as a crucial component for robust and interpretable
079   neural reasoners and demonstrate that trained with hint supervision, discretized models perfectly
080   capture the dynamic of the underlying algorithms and do not suffer from distributional shifts.

      2     BACKGROUND
083
084
      2.1   A LGORITHMIC REASONING
085
086   Performing algorithmic-like computations usually requires the execution of sequential steps and the
087   number of such steps depends on the input size. To imitate such computations, neural networks are
088   expected to be based on some form of recurrent unit, which can be applied to a particular problem
089   instance several times (Kaiser & Sutskever, 2015; Zaremba & Sutskever, 2014; Vinyals et al., 2015;
090   Veličković et al., 2020b).
091   The CLRS Algorithmic Reasoning Benchmark (CLRS-30) (Veličković et al., 2022) defines a general
092   paradigm of algorithmic modeling based on Graph Neural Networks (GNNs), as graphs can natu-
093   rally represent different input types and manipulations over such inputs. Also, GNNs are proven to
094   be well-suited for neural execution (Xu et al., 2020; Dudzik & Veličković, 2022).
095
      The CLRS-30 benchmark covers different algorithms over various domains (arrays, strings, graphs)
096   and formulates them as algorithms over graphs. Also, CLRS-30 proposes to utilize the decom-
097   position of the algorithmic trajectory execution into simple logical steps, called hints. Using this
098   decomposition is expected to better align the model to desired computations and prevent it from
099   utilizing hidden non-generalizable dependencies of a particular train set. Prior work demonstrates a
100   wide variety of additional inductive biases for models towards generalizing computations, including
101   different forms of hint usage (Veličković et al., 2022; Bevilacqua et al., 2023), biases from standard
102   data structures (Jürß et al., 2024; Jain et al., 2023), knowledge transfer and multitasking (Xhonneux
103   et al., 2021; Ibarz et al., 2022; Numeroso et al., 2023), etc. Also, recent studies demonstrate sev-
104
      eral benefits of learning neural reasoners end-to-end without any hints at all (Mahdavi et al., 2023;
      Rodionov & Prokhorenkova, 2023).
105
106   The recently proposed SALSA-CLRS benchmark (Minder et al., 2023) enables a more thorough
107   OOD evaluation compared to CLRS-30 with increased test sizes (up to 100-fold train-to-test scaling,
      compared to 4-fold for CLRS-30) and diverse test distributions. Despite significant gains in the


                                                         2
      Under review as a conference paper at ICLR 2025


108
      performance of neural reasoners in recent work, current models still struggle to generalize to out-
109   of-distribution (OOD) test data (Mahdavi et al., 2023; Georgiev et al., 2023; Minder et al., 2023).
110   While de Luca & Fountoulakis (2024) prove by construction the ability of the transformer-based
111   neural reasoners to perfectly simulate graph algorithms (with minor limitations occurring from the
112   finite precision), it is still unclear if generalizable and interpretable models can be obtained via
113   learning. Importantly, the issues of OOD generalization are induced not only by the challenges
114   of capturing the algorithmic dependencies in the data but also by the need to carefully operate
115   with continuous inputs. For example, investigating the simplest scenario of learning to emulate
116   the addition of real numbers, Klindt (2023) demonstrates the failure of some models to exactly
117
      imitate the desired computations due to the nature of gradient-based optimization. This limitation
      can significantly affect the performance of neural reasoners on adversarial examples and larger input
118
      instances when small errors can be accumulated.
119
120
121   2.2   T RANSFORMER INTERPRETABILITY AND COMPUTATION MODEL
122
      Transformer (Vaswani et al., 2017) is a neural network architecture for processing sequential data.
123
      The input to the transformer is a sequence of tokens from a discrete vocabulary. The input layer maps
124
      each token to a high-dimensional embedding and aggregates it with the positional encoding. The
125   key components of each layer are attention blocks and MLP with residual connections. Providing a
126   detailed description of mechanisms learned by transformer models (Elhage et al., 2021) is of great
127   interest due to their widespread applications.
128
      RASP (Weiss et al., 2021) is a programming language proposed as a high-level formalization of the
129
      computational model behind transformers. The main primitives of RASP are elementwise sequence
130
      functions, select and aggregate operations, which conceptually relate to computations performed
131   by different blocks of the model. Later, Lindner et al. (2024) presented Tracr, a compiler for con-
132   verting RASP programs to the weights of the transformer model, which can be useful for evaluating
133   interpretability methods.
134
      While RASP might have limited expressibility, it supports arbitrary complex continuous functions
135
      which in theory can be represented by transformer architecture, but are difficult to learn. Also, RASP
136
      is designed to formalize computations over sequences of fixed length. Motivated by that, Zhou et al.
137   (2024) proposed RASP-L, a restricted version of RASP, which aims to formalize the computations
138   that are easy to learn with transformers in a size-generalized way. The authors also conjecture that
139   the length-generalization of transformers on algorithmic problems is related to the ‘simplicity’ of
140   solving these problems in RASP-L language.
141
      Another recent work (Friedman et al., 2023) describes Transformer Programs: constrained trans-
142
      formers that can be trained using gradient-based optimization and then automatically converted into
143   a discrete, human-readable program. Built on RASP, transformer programs are not designed to be
144   size-invariant.

149   3.1   E NCODE - PROCESS - DECODE PARADIGM
150
151
      Our work follows the encode-process-decode paradigm (Hamrick et al., 2018), which is usually
      employed for step-by-step neural execution.
152
153   All input data is represented as a graph G with an adjacency matrix A and node and edge features
154   that are first mapped with a simple linear encoder to high-dimensional vectors of size h. Let us
155   denote node features at a time step t (1 ≤ t ≤ T ) as X t = (xt1 , . . . , xtn ) and edge features as
156   Et = (et1 , . . . , etm ). Then, the processor, usually a single-layer GNN, recurrently updates these
157
      features, producing node and edge features for the next step:
158                                  X t+1 , E t+1 = Processor(X t , E t , A).
159
160   The processor network can operate on the original graph defined by the task (for graph problems) or
161   on the fully connected graph. For the latter option, the information about the original graph can be
      encoded into the edge features.


                                                        3
      Under review as a conference paper at ICLR 2025


162
      The number of processor steps T can be defined automatically by the processor or externally (e.g.,
163   as the number of steps of the original algorithm). After the last step, the node and edge features are
164   mapped with another linear layer, called the decoder, to the output predictions of the model.
165
166
      If the model is trained with hint supervision, the changes of node and edge features at each step are
      expected to be related to the original algorithm execution. In this sense, the processor network is
167
      aimed to mimic the algorithm’s execution in the latent space.

212   3.3   C ONTINUOUS INPUTS
213
214   Clearly, most of the algorithmic problems operate with continuous or unbounded inputs (e.g.,
215   weights on edges). Usually, all input data is encoded into node and edge features and the pro-
      cessor operates over the resulting vectors. The proposed discretization of such vectors would lead to


                                                          4
      Under review as a conference paper at ICLR 2025


216
      the loss of information necessary for performing correct execution steps. One possible option to op-
217   erate with such inputs (we will call them scalars, meaning both continuous or size-dependent integer
218   inputs, such as node indexes) is Neural Execution Engines (Yan et al., 2020), which allows one to
219   operate with bit-wise representations of integer and (in theory) real numbers. Such representations
220   are bounded by design, but fully discrete and interpretable.
221
      We propose another option: to maintain scalar inputs (denote them by S) separately from the node
222
      and edge features and use them only as edge priorities sij in the attention block. If scalars are related
223   to the nodes, we assign them to edges depending on the scalar of the sender or receiver node. Now,
224   we can consider the hard attention block as a selector which for each node selects the best edge
225   based on an ordered pair of ‘states priority’ (attention weights described above, which depend only
226   on states of the corresponding nodes and edges) and sij . We note that this selector is related to the
227   theoretical primitive select best from RASP. We implement this simply by augmenting key vectors
228   Kij of each edge with the indicator if the given edge has the “best” (min or max) scalar among the
229   other edges to node j. Thus, scalars affect only the attention weights, not the messages and the node
230   states.
231   For multiple different scalar inputs (e.g., weighted edges and node indexes), we use multi-head
232   attention, where each head operates with separate scalars.
233
      Given that, the interface of the proposed processor can be described as
234
235                                 X t+1 , E t+1 = Processor(X t , E t , A, S),
236   where X t , X t+1 and E t , E t+1 are from the fixed sets. State sets are independent of the execution
237   step t and the input graph (including scalar inputs S).

280
281   In this section, we perform experiments to evaluate how the proposed discretization affects the
282   performance of neural reasoners on diverse algorithmic reasoning tasks. Our main questions are:
283
284
            1. Can the proposed discrete neural reasoners capture the desired algorithmic dynamics with
               hint supervision?
285
286         2. How does discretization affect OOD and size-generalization performance of neural reason-

               ers?
288         3. Is the proposed model capable of multi-task learning?

      to utilize the given amount of node and edge states to solve the problem. We discuss no-hint experi-

      4.1   DATASETS
294
295   We perform our experiments on the problems from the recently proposed SALSA-CLRS bench-
296   mark (Minder et al., 2023), namely BFS, DFS, Prim, Dijkstra, Maximum Independent Set (MIS),
297   and Eccentricity. We believe that the proposed method is not limited by the covered problems, but
298   we leave the implementation of the required data flows (e.g., edge-based reasoning (Ibarz et al.,
299   2022), graph-level hints, interactions between different scalars) for future work.
300   The train dataset of SALSA-CLRS consists of random graphs with at most 16 nodes sampled from
301   the Erdös-Rényi (ER) distribution with parameter p chosen to be as low as possible while graphs
302   remain connected with high probability. The test set consists of sparse graphs of sizes from 16 to
303   1600 nodes.
304   We slightly modify the hints from the benchmark without conceptual changes (e.g., we have modi-
305   fied the hints for the DFS problem to remove graph-level hints). Discrete states are fully described
306   by the non-scalar hints and scalars are exactly the hints of the scalar type (we refer to Veličković
307   et al. (2022) for the details on hint design).

      5   I NTERPETABILITY AND TESTING
393
394
      In addition to the empirical evaluation of the trained models on diverse test data, the discrete and
395
      size-independent design of the proposed models allows us to interpret and test them manually. The
396   main idea is to show that the model will perform the exact discrete state transitions (including dis-
397   crete operations with scalars) as the original algorithm.
398
399
      First, we note that due to the hard attention, each node receives exactly one message. Also, the
      message depends only on the discrete states of the corresponding nodes and edges. Thereby, as each
400
      node and edge states after a single processor step depend only on the current states and received
401
      message, all the possible options can be directly enumerated and tested if all states change to the
402   correct ones.
403
404   The only remaining part to fully interpret the whole model is the attention block. We note that
405
      our implementation of the select best selector (Section 3.3) does not necessarily produce the top-1
      choice over the ordered pairs of ‘states priority’ and scalars sij as it simply augments key vectors
406
      with indicators if the given edge has the best scalar among others. For example, it may happen
407
      that for some state, the maximum attention weight is achieved for an edge without the indicator.
408   However, given the finite number of discrete states, we can manually check if the mentioned “best”
409   indicator increases the attention weight between every pair of states. Combined with hard attention,
410   this would imply that the attention block attends depending on the predefined states and uses scalar
411   priorities only to break ties. Please see the particular example with more detailed explanation for the
412   BFS problem in the Appendix D.
413
      Given that, we can unit-test all possible state transitions and attention blocks. With full coverage
414   of such tests, we can guarantee the correct execution of the desired algorithm for any test data.
415   We tested our trained models from Section 4 manually verifying state transitions. As a result, we
416   confirm that the attention block indeed operates as select best selector, as the model actually uses
417   these indicators to increase the attention weights. Thus, we can guarantee that for any graph size,
418   the model will mirror the desired algorithm, which is correct for any test size.

      6   T OWARDS NO - HINT DISCRETE REASONERS
421
422
      In this section, we discuss the challenges of training discrete reasoners without hints, which can be
423
      useful when tackling new algorithmic problems.
424
425   Training deep discretized models is known to be challenging: without hyperparameter search, dis-
426   crete models are only slightly improved over the untrained models. Therefore, we focus only on
427   the BFS algorithm, as it is well-aligned with the message-passing framework, has short roll-outs,
428
      and can be solved with small states count (note that for no-hint models node/edge states count is a
      hyperparameter due to the absence of the ground truth states trajectory).
429
430   We recall that the output of the BFS problem is the exploration tree pointing from each node to
431   its parent. Each node chooses as a parent the neighbor from the previous distance layer with the
      smallest index.

447   step of the processor and note that the states correspond to the distances from the starting node. More
448
      formally, we note that the model with four states uses the first state for the starting node, the second
      state for its neighbors, the third state for nodes at distance two from the starting node, and the last
449
      state for all other nodes and such states-based classification of distance has accuracy > 98% when
450
      tested on 1000 random graphs with 16 nodes. Then, we note that for the nodes that are from the first
451   four distance layers from the starting node, the pointers are predicted with 100% accuracy and these
452   pointers are computed layer-by-layer as in the ground truth algorithm (we refer to Appendix B for
453   illustrations). The mistakes of the model are on the distance ≥ 4 from the starting node (we did not
454   reverse-engineer the specific logic of computations on larger distances).
455
      We found this behavior well-aligned to the BFS algorithm and indicating the possibility of achieving
456
      the perfect validation score with enough states count. However, this algorithm does not generalize
457   since it fails at distances larger than those encountered during training.
458
459   On the other hand, one can demonstrate that for a small enough state count (for BFS, it is two node
460
      states and two edge states) and diverse enough validation data, the perfect validation performance
      implies that the learned solution will generalize to any graph size.
461
462   Therefore, we highlight the need to achieve perfect validation performance with models that use as
463   few states as possible, which corresponds to the minimum description length (MDL) theory (Myung,
464   2000; Rissanen, 2006) and is related to the notion of Kolmogorov complexity (Kolmogorov, 1963).
465   Finally, we note that for sequential problems, such as DFS, obtaining a good no-hint model can
466   be even more challenging and can require additional effort. One possible way to overcome this
467   limitation is to implement a curriculum learning setup.

470   7   L IMITATIONS AND FUTURE WORK
471
472   Limitations In this work, we propose a method to learn robust neural reasoners that demonstrate
473   perfect generalization performance and are interpretable by design. In this section, we describe
474   some limitations of our work and important directions for future research.
475   First, several proposed design choices strictly reduce the expressive power of the model. For exam-
476   ple, due to the hard attention, the proposed model is unable to compute the mean value from all the
477   neighbors in a single message-passing step, which is trivial for attention-based models (note that this
478   can be computed in several message-passing steps). Thus, the model in its current form is unable to
479   express transitions between hints for some algorithms from the CLRS-30 in a single processor step.
480   However, we believe that the expressivity of the proposed model can be enhanced with additional
481   architectural modifications (e.g., edge-based reasoning (Ibarz et al., 2022), global states, interactions
482
      between different scalars) that can be combined with the proposed discretization ideas.
483   Second, while we report the perfect scores for the covered tasks, we cannot guarantee that the train-
484   ing will converge to the correct model for any initialization/training data distributions. However, we
485   empirically found the proposed method to be quite robust to various architecture/training hyperpa-
      rameter choices.


                                                         9
      Under review as a conference paper at ICLR 2025


486
      Future work Our method is based on the particular architectural choice and actively utilizes the
487   attention mechanism. However, the graph deep learning field is rich in various architectures exploit-
488   ing different inductive biases and computation flows. The proposed separation between discrete
489   states and continuous inputs may apply to other models, however, any particular construction can
490   require additional efforts.
491
      Also, we provide only one example of the ScalarUpdate block. We believe that utilizing a general
492
      architecture (e.g., some form of discrete Neural Turing Machine (Graves et al., 2014; Gulcehre et al.,
493   2016)) capable of executing a wider range of manipulation is of interest for future work.
494
495   With the development of neural reasoners and their ability to execute classic algorithms on abstract
496
      data, it is becoming more important to investigate how such models can be applicable in real-world
      scenarios according to the Neural Algorithmic Reasoning blueprint (Veličković & Blundell, 2021)
497
      and transfer their knowledge to high-dimensional noisy data with intrinsic algorithmic dependen-
498
      cies. While there are several examples of NAR-based models tackling real-world problems (Beurer-
499   Kellner et al., 2022; Numeroso et al., 2023), there are no established benchmarks for extensive
500   evaluation and comparison of different approaches.
501
502
      Lastly, we leave for future work a deeper investigation of learning interpretable neural reasoners
      without hints, which we consider essential from both theoretical perspective and practical applica-
503
      tions, e.g., combinatorial optimization.

506   8   C ONCLUSION
507
508   In this paper, we force neural reasoners to maintain the execution trajectory as a combination of
509   finite predefined states. To achieve that, we separate discrete and continuous data flows and de-
510   scribe the interaction between them. The obtained discrete reasoners are interpretable by design.
511
      Moreover, trained with hint supervision, such models perfectly capture the dynamic of the under-
      lying algorithms and do not suffer from distributional shifts. We consider discretization of hidden
512
      representations as a crucial component for robust neural reasoners.

      In this section, we investigate if the proposed ScalarU pdate module can be successfully extended
      to support more complex manipulations with scalars.
807
808   First, we note that simple manipulations with scalars cover a significant part of the classical al-
809   gorithms. In this work, we use the minimum set of the required functions, but it can be directly
      extended by other functions. Importantly, as ScalarU pdate can be viewed as a separate module,


                                                            15
      Under review as a conference paper at ICLR 2025


810
      we can separately check if it is possible to train it with any given set of predefined manipulations for
811   any problem only with supervision on the results.
812
813   Let us formalize the problem: each input of the ScalarU pdate module can be described as an
814
      object with a discrete state si (from a fixed predefined set) and several scalar values (we consider
      two scalars xi and yi ). Note that we omit the separation between nodes and edges and consider
815
      objects with several scalar values. For each discrete state si , there exists a ground truth update of
816
      scalars, e.g., f (s, x, y) = x + cos(y). The output of the scalar updater can be viewed as a sum:
817                                                       X
818                            ScalarU pdate(s, x, y) =         g(x, y) · activateg (s),
819                                                     g∈OPS

820   where OPS is a predefined set of operations and activateg is a 0-1 function representing if a specific
821   operation should be applied. Note that activateg depends only on a discrete state of the input.
822   For our additional experiment, we train several different ScalarU pdate modules with the extended
823   set of operations:
824
825          • g0 (x, y) = 1;
826          • g1 (x, y) = x;
827          • g2 (x, y) = cos(x);
828
             • g3 (x, y) = x · y;
829
             • g4 (x, y) = atan2(x, y)
830
831   to learn the following set of the ground truth updates simultaneously:
832
833          • f0 (x, y) = x;
834          • f1 (x, y) = cos(x);
835          • f2 (x, y) = cos(x) + x · y;
836
             • f3 (x, y) = atan2(x, y);
837
             • f4 (x, y) = 1 + x + atan2(x, y).
838
839   In particular, we consider a set of 16 discrete states (numbered from 0 to 15 and sampled uniformly)
840   and the ground truth scalar update is derived from these states by taking the remainder of the division
841   by 5 (updates count).
842
      The learnable parameters of the ScalarU pdate are state’s embeddings and linear projections for
843   each indicator. We train ScalarU pdate to minimize the MSE loss between the ground truth and
844   predicted outputs with 5000 optimization steps. Additionally, we train a non-discrete scalar updater
845   (2-layer MLP), similar to our ablation experiments. We refer to the source code for the experiment
846   details.
847
      Inspired by Klindt (2023), we generate training scalars X and Y from Uniform[0.5, 1.0] and gener-
848
      ate test set by sampling scalars from the Uniform[0., 0.5] distribution.
849
850   We report the evaluation results in Table 7. The proposed discrete ScalarU pdate module success-
851   fully learned the correct operations for updates f1 , ..., f4 for all seeds and 3 times out of 5 for f0
852
      (note that the model was trained to predict different manipulations for different states simultane-
      ously). For unsuccessful runs, when f0 was not learned correctly, the learned operation for f0 was
853
      g2 for some states (i.e. x · y instead of x), which can be explained by optimization challenges as the
854
      distribution of y is close to 1.
855
856   Our experiment demonstrates that the proposed ScalarU pdate module can be extended to support
857   a wider range of manipulations with scalars. We note that this complicates the optimization problem
858
      of selecting the correct operations/operands from the operations results (e.g., such decomposition
      might not be unique).

      As an example, consider the BFS algorithm. First, recall the pseudocode of the algorithm:
873
874     Starting node ← visited
875     All other nodes ← not visited
876     for step in range(T) do
877         for node U in a graph do
878
                if U is visited on previous steps then
                    continue
879
                end if
880
                if U has a neighbor P that visited on previous steps then
881                 U ← visited on this step
882                 U select the smallest-indexed such neighbor P as parent:
883                 Edge (U, P ) ← pointer
884                 Self-loop (U, U ) ← not pointer
885             end if
886         end for
887     end for
888     return a BFS tree described by pointers
889   Now let us describe how we can verify that the trained DNAR model will perfectly imitate this
890   algorithm for any test data.
891
892
      First, we note that for each node U , the node state on the step t + 1 (denoted by Ut+1 ) is the function
      of Ut and Vt , where V is the node that sends the message to U on step t:
893
894                                Ut+1 = StateU pdate(Ut , message f rom Vt )
895
      How does the node U select a node that will send a message to it? For any node V connected to U ,
896
      the node U computes attention scores depending on discrete states of V of each node and a discrete
897
      indicator if each node has the smallest (or largest) scalar among all neighbors of U with the same
898   discrete state as V . Then, the node U selects the node V with the largest attention score.
899
900   In our (slightly simplified) case, the attention scores only depend on the tuples (Ustate , Vstate ,
901
      indicator if u has the smallest index) and there are only 8 such tuples. We can directly com-
      pute these attention scores and verify the required invariants, e.g.,
902
903         Attention(not visited, visited, smallest) > Attention(not visited, ∗any other∗),
904
      which would imply that the not visited node will receive the message from the smallest-indexed
905
      visited neighbor if such exists independently of the graph size and distribution. If there is no such
906
      neighbor, the node U will receive the message from another not visited node (or from itself).
907
908   After verifying the correctness of the message flows, we need to ensure that the state updates are
909   computed correctly, e.g.,
910                     visited = StateU pdate(not visited, message f rom visited)
911
912                                  visited = StateU pdate(visited, ∗any∗)
913                not visited = StateU pdate(not visited, message f rom not visited)
914   The main idea is that due to the finite states count and discrete manipulations with scalars, there are
915   only finite amounts of such checks that can cover all possible state transitions and all of them should
916   be evaluated only once.
