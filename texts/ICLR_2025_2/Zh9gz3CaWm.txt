009                                                A BSTRACT
010
011            Federated learning (FL) is a popular distributed machine learning framework for
012            edge computing. However, it faces a significant challenge: the communication
013            overhead caused by frequent model updates between clients and the central server.
014            Previous studies have overlooked a crucial piece of information: the central server
015            already knows the initial model on each client before local training begins in every
016            round. This oversight leads to significant redundancy in communication, as full
017            model information are transmitted unnecessarily. To address this, we propose a
018
               novel framework called model update distillation (MUD), which leverages this
               prior knowledge to decouple model parameters from the network architecture. In-
019
               stead of transmitting raw parameter updates, our method synthesizes and transmits
020
               compact tensor sequences that encode only the essential information for synchro-
021            nization. This dramatically reduces communication overhead while still allowing
022            recipients to accurately reconstruct the intended model updates. Extensive ex-
023            perimental results demonstrate that FedMUD achieves substantial improvements
024            in communication efficiency, making it a highly effective solution for federated
025            learning in bandwidth-constrained environments. The PyTorch-like core code can
026            be found in 3.

029   1    I NTRODUCTION
030
031   Federated learning (FL) (McMahan et al., 2017; Shokri & Shmatikov, 2015) has become a promising
032   approach for privacy-preserving machine learning by allowing model training directly on edge devices,
033
      such as smartphones and IoT sensors, without requiring centralized data storage. This decentralized
      framework ensures that sensitive data remains on-device, mitigating privacy and regulatory concerns
034
      (Ching et al., 2018; GDPR, 2016; ADPPA, 2022). With the rise of edge computing, FL has gained
035
      further traction, enabling real-time data processing and decision-making at the source (Feng et al.,
036   2021; Nguyen et al., 2021). By leveraging the growing computational power of mobile devices, FL
037   not only enhances scalability and efficiency in diverse fields like healthcare, finance, and smart cities
038   but also reduces latency and the need for transmitting raw data to centralized servers.
039
      However, one of the persistent challenges in FL is the growing imbalance between computing power
040
      and communication bandwidth. As shown in Table 1, advances in bandwidth have not kept pace with
041
      computing power. For instance, MediaTek and Qualcomm chips have improved by 53% and 31%,
042   respectively, over their previous generations (PrimateLabs, 2024). In contrast, the global median
043   upload bandwidth for mobile devices increased by only 7% from 2023 to 2024 (Ookla, 2024). This
044   growing disparity highlights a key challenge in FL: modern neural network models, even lightweight
045   ones like MobileNet, still consist of millions of parameters. Given that edge devices typically rely on
046   wireless or long-distance connections, with bandwidth often limited to tens of Mbps, this bandwidth
047   bottleneck severely restricts FL’s potential. As computational power continues to rise, optimizing
048   communication efficiency in bandwidth-constrained environments is crucial for maintaining FL
049   performance.
050   To address this challenge, communication-efficient FL has emerged as an active research area. Re-
051   searchers have developed various strategies to reduce the communication burden, including techniques
052   like model compression (e.g., quantization (Liu et al., 2023; Sun et al., 2022) and sparsification (Aji
053   & Heafield, 2017; Dai et al., 2022)) that minimize the size of model updates. Additionally, methods
      like delayed gradient averaging allow devices to perform more local computation before transmitting

      reduction in communication overhead remains limited due to the fundamental relationship between

069   In this paper, we uncover a critical yet often overlooked information in federated learning: in each
070
      training round, the central server has prior knowledge of the initial model that each edge client
      will use for local training. By exploiting this valuable information, we propose a novel federated
071
      learning framework called Model Update Distillation (FedMUD). Inspired by gradient inversion
072
      (Zhu et al., 2019) and dataset distillation (Wang et al., 2018), FedMUD constructs a synthetic tensor
073   sequence, which is fed into the initial model of the current round. Specially, after local update,
074   we fixed the initial model parameters and iteratively optimize this tensor sequence to ensure that
075   the resulting parameter differences closely match the actual local updates produced by the clients.
076   Through model update distillation, we condense these parameter differences into a compact tensor
077   sequence, allowing recipients to accurately reconstruct the intended model updates from the sender.
078   This decoupling of model updates from the full parameter set significantly reduces communication
079   overhead, as it eliminates the need to transmit the entire array of raw parameters. By treating the
080   model’s parameter updates as a whole and compressing them, our approach offers a promising
081
      solution to improve communication efficiency.
082   The main contributions of this paper are summarized as follows:
083
084              • We point out and exploit a crucial piece of information that has been overlooked in previous
085
                   research: the central server in federated learning knows the initial model on each client
                   before local training begins in every round.
086
087              • We propose FedMUD, a new framework that treats network parameter updates as a whole,
088
                   decoupling them from the network architecture and ensuring a more efficient learning
                   process.
089
090              • Experimental results demonstrate that model update distillation significantly reduces the
091                amount of communication compared to baselines, without significant accuracy degradation,
092
                   enabling a communication-efficient and training-accelerated FL process.

      3.1   M OTIVATION
173
174   In the standard FL process, the central server distributes the aggregated global model from the
175   previous round to each participating device. These devices then perform local training using the
176   global model as a starting point, updating their local models accordingly. The FedAvg algorithm
177   a cornerstone of FL, involves sending model updates between the new and old models back to the
178   server. However, this approach has a significant inefficiency: the data volume transferred in each
179   communication round remains constant, irrespective of the extent of changes in model parameters, as
180   long as the network architecture does not change. This results in redundant transmission of update
181
      information. The root of this issue lies in the inherent interdependence between the model parameters
      and the network architecture in traditional FL approaches. Since the network’s structure and its
182
      parameters are closely linked, even minor changes in parameters require retransmitting the entire
183
      model, leading to inefficiency in communication.
184
185   3.2   O UR S TRATEGY
186
187   We introduce a novel approach that involves approximating model updates across different training
188   periods within a single model. This approximation is represented as a synthetic tensor sequence,
189   drawing inspiration from concepts in dataset distillation (Wang et al., 2018) and gradient inversion
190   (Zhu et al., 2019). The core idea is for the sender to transmit this tensor sequence, which encapsulates
191   only the indispensable information required for model updates, thus eliminating the need to transmit
192
      the entire set of raw parameter differences. Upon receiving the tensor sequence, the recipient
      can accurately reconstruct the original model update by performing a single gradient descent step,
193
      integrating the information from the received tensor sequence with the state of the previous model.

230   As the model size increases, the approximation error between the virtual gradient obtained by the
231   MUD method and the real model parameter difference tends to grow, especially for large-scale neural
232   networks. To address this, we introduce a modular alignment approach that segments the network into
233   modules. Each module uses an independent synthetic dataset to approximate its gradient change. The
234
      number of modules is kept below the local training batch size to ensure it stays within the device’s
      computational capacity. By modularizing the network, model update distillation for each sub-module
235
      can be processed in parallel, reducing delays. We will analyze the error introduced by MUD in the
236
      convergence analysis section.

      3.3   D IFFERENCES WITH D ISTILLATION - RELATED M ETHODOLOGIES
239
240   We delve into the differences between the MUD and the popular distillation-related methodologies:
241
      – Knowledge Distillation (KD) is a process involving two distinct neural network sets: a larger,
242
      complex “teacher" model, and a smaller, efficient “student" model (Hinton et al., 2015). The primary
243
      goal of KD is to transfer the knowledge from the teacher to the student, enabling the student model to
244   reach performance levels similar to the teacher. This transfer is generally accomplished by minimizing
245   the differences in predictions (probabilities or logits) between the teacher and student models when
246   given the same input data.
247
      – Dataset Distillation (DD) represents an extension of KD where knowledge transfer occurs at the
248
      dataset level rather than between models (Wang et al., 2018). The core idea behind DD is to condense
249
      a large and comprehensive training dataset into a much smaller, but highly representative, synthetic
250   dataset, given a fixed network initialization. This smaller dataset is designed to retain the essential
251   characteristics and information of the original dataset, enabling models to be trained effectively on
252   this distilled dataset instead of the full, larger dataset.
253
      – Differences with KD and DD: From the description provided, it is evident that our proposed
254
      MUD method diverges significantly from both KD and DD in its approach and objectives. Unlike
255
      KD, which is centered on the relationship between a larger, more complex “teacher" model and a
256   smaller, more efficient “student" model, our MUD focuses on the update within a single model across
257   different training periods. The primary input in MUD is the parameter disparity observed between
258   two consecutive training stages of the same model. This approach is fundamentally different from the
259   input used in DD, which involves a large-scale dataset that typically contains thousands to millions of
260   images. The output of MUD contrasts sharply with the output of KD, which is the student model
261   trained to mimic the teacher model. Although MUD’s output bears a resemblance to DD in terms of
262   its form, the two are fundamentally different in their design intentions.

      Baselines. We evaluate FedMUD against five state-of-the-art methods, covering quantization, sparsi-
394
      fication, and knowledge distillation approaches: 1) Top-k (Aji & Heafield, 2017) is a sparsification
395
      technique that reduces communicated gradients by selecting the largest k elements, with k set to 50%
396   of the total parameters. 2) FedPAQ (Reisizadeh et al., 2020) is the first federated learning quantization
397   scheme, which compresses models uploaded by clients and distributed by the server to 8-bit. 3)
398   DAdaQ (Hönig et al., 2022) employs a dual adaptive quantization algorithm that adjusts quantization
399   levels dynamically across rounds and clients. 4) AdaGQ (Liu et al., 2023) adapts quantization levels
400   in each round based on gradient norms and client bandwidth. 5) FedKD (Wu et al., 2022) improves
401   communication efficiency by creating a smaller messenger model via knowledge distillation between
402   the server and clients. While recently published methods like FedDST (Bibikar et al., 2022) and
403   FedCS (Jiang & Borcea, 2023) are relevant, their original papers only test on a three-layer CNN.
404
      Thus, we exclude them from our comparison, as their effectiveness on more complex models remains
      unclear.
405
406   Datasets. Experiments are conducted on two benchmark datasets: CIFAR-10 and a real-world
407   medical image dataset CRCSlides (Kather et al., 2018; 2019), which is collected for predicting
408   survival from colorectal cancer histology slides. It contains 100,000 images for training and 7,180
409   images for testing. The image size is 3 × 32 × 32. Similar to prior work (Liu et al., 2023), we use σd
410
      to denote the level of Non-IID data, which corresponds to the fraction of data that belongs to only one
      class at each device. In our experiments, we set σd = 0.2, representing the scenario where 20% of
411
      each local dataset contains samples from only one class, while the remaining 80% contains samples
412
      from other classes. Detailed experiment settings are included in the Appendix A.

      5.2                                            C OMPUTATION AND C OMMUNICATION T IME A NALYSIS
509
510   To analyze how FedMUD reduces total training time, we dissect both communication and computation
511   times in Fig.3. For comparison, we also show the proportions of these times for other methods.
512   Our approach requires relatively longer computation time due to the MUD process but benefits
513   from significantly reduced communication time, improving overall training efficiency and device
514   utilization. FedAvg has the shortest computation time, as it transmits complete model information,
515   allowing it to reach target accuracy in fewer rounds. Among quantization-based methods, FedPAQ
516
      has the longest computation time, needing more rounds to compensate for information loss from
      quantization. DAdaQ and AdaGQ use adaptive quantization strategies that reduce communication
517
      rounds but require longer communication times. These results highlight the trade-offs between
518
      computation and communication times in different federated learning methods, underscoring the
519   effectiveness of FedMUD in optimizing overall training efficiency and device utilization.

522   6                                      L IMITATIONS D ISCUSSION
523
524   The MUD procedure requires additional computational cost, and it may be unrealistic to use synthetic
525   tensor to precisely approximate the model parameter update for ultra-large networks. However,
526   in the context of edge computing, models like ResNet-18 are already considered quite large, as
527   most commonly deployed architectures, such as MobileNet and GhostNet, are designed to be more
528   lightweight and efficient.Moreover, the additional computation is offset by the significant reduction in
529
      communication overhead. Despite these limitations, we believe our method offers valuable insights
      and a novel perspective on communication-efficient federated learning.

532   7                                      C ONCLUSION
533
534   In this study, we introduced model update distillation-based communication-efficient federated
535   learning (FedMUD), a novel approach where devices and the server synthesize tensor sequences to
536   represent small model updates, rather than transmitting raw model differences. Our key innovation
537   lies in distilling the structural essence of model updates, as opposed to directly compressing them.
538   This enables the transmission of only essential information for synchronization, bypassing the need to
539   transmit the entire set of raw parameter differences. Experimental results demonstrate that FedMUD
      substantially reduces communication overhead without sacrificing accuracy significantly.

650
      Implementations details. We implement FedMUD alongside baseline methods utilizing the PyTorch
651   framework on a system outfitted with four Nvidia RTX 3090 GPUs. Our simulation involves 10
652   virtual devices, with a device sampling rate of 1, ensuring each device’s participation in model
653   updating during every communication round. The data transmission rate for each device is randomly
654   set between 50 Mbps and 100 Mbps by default. All methods employ stochastic gradient descent for
655   local training, with a learning rate set at 0.01. Specifically, for FedMUD, the LBFGS optimization
656   technique is utilized for synthesizing the tensor sequence, offering the advantage of adaptively
657   controlling the step size to efficiently identify an optimal tensor sequence. The tensor length for each
658   module is set to 10, with the number of iterations also set to 10. For the ResNet-18 architecture,
659
      we segmented it into 18 modules, while the GoogLeNet model was partitioned into 11 modules.
      The batch size is set as 32, and devices are configured to upload model updates at the end of every
660
      epoch. All compared methods initialize their networks by pre-training using FedAvg for 30 rounds.
661
      Since they share the same initialization process, the experimental results and analyses presented
662   subsequently include only the costs after initialization. It is worth noting that, the initialization step is
663   crucial for achieving training stability. For example, the accuracies of Top-k and AdaGQ drop by as
664   much as 13% and 10% respectively if the initialization step is removed.

       syn_labels)

                syn_dw = torch.autograd.grad(loss, syn_model.parameters(),
691                      create_graph=True, allow_unused=True)
692             syn_gradients = torch.cat([v.flatten() for v in syn_dw])
693             loss = torch.nn.MSELoss()(syn_gradients, real_gradients)
694             loss.backward()
                return loss

         optimizer.step(closure)
