009                                                A BSTRACT
010
011
               Scalable distributed differentially private learning would benefit notably from re-
012            duced communication and synchronization overhead. The current best methods,
013            based on gradient averaging, inherently require many synchronization rounds. In
014            this work, we analyze blind model averaging for convex and smooth empirical
015            risk minimization (ERM): each user first locally finishes training a model and then
016            submits the model for secure averaging without any client-side online synchroniza-
017            tion. This setting lends itself not only to data point-level privacy but also to flexible
018            user-level privacy, where the combined impact of the user’s trained model does not
019            depend on the number of data points used for training.
020            In detail, we analyze the utility side of blind model averaging for support vector
021            machines (SVMs) and the inherently multi-class Softmax regression (SoftmaxReg).
022            On the theory side, we use strong duality to show for SVMs that blind model
023
               averaging converges toward centralized training performance if the task is robust
               against L2-regularization, i.e. if increasing the regularization weight does not
024
               destroy utility. Furthermore, we provide theoretical and experimental evidence that
025
               blind averaged Softmax Regression works well: we prove strong convexity of the
026            dual problem by proving smoothness of the primal problem. Using this result, we
027            also derive the first output perturbation bounds for Softmax regression. On the
028            experimental side, we support our theoretical SVM convergence. Furthermore, we
029            observe hints of an even more fine-granular connection between good utility of
030            model averaging and mid-range regularization weights which lead to compelling
031            utility-privacy-tradeoffs for SVM and Softmax regression on 3 datasets (CIFAR-
032            10, CIFAR-100, and federated EMNIST embeddings). We additionally provide
033            ablation for an artificially extreme non-IID scenario.

      1   I NTRODUCTION
037
038   Non-interactivity in distributed learning, with just one user message sent, offers minimal commu-
039
      nication overhead for privacy-preserving learning, which enables high scalability. Yet, Smith et al.
      (2017) have shown that non-interactive learning has limitations: for any non-interactive learning
040
      method, there are problems for which the excess risk increases exponentially with the number of
041
      model parameters, where excess risk is the loss-difference to the optimal model for the union of all
042   local data. By contrast, for a class of convex empirical risk-minimization (ERM)-based learning
043   objectives, Jayaraman et al. (2018) have explored blind model averaging (BlindAvg): each user first
044   locally finishes training a model and then submits the model for secure averaging non-interactively.
045   Their work, however, does not capture the key utility improvement of BlindAvg. Their result leaves
046   open whether blindly averaged ERMs converges beyond the best local model. More importantly, prior
047   work leaves the following question open, which we tackle in this work: Under which circumstances
048   does BlindAvg achieve a compelling utility-privacy tradeoff comparable to the centralized setting?
049   To achieve privacy amplification for blind averaging, prior work has identified a necessary condition:
050   a bound of the trained model s (output sensitivity). In contrast to sensitivity bounds only for global
051   optima (Chaudhuri et al., 2011; Jayaraman et al., 2018), the work by Wu et al. (2017) accounts for
052   leakage of the optimization algorithm and provides such bounds for each training iteration. They
053   show that for output sensitivity with stochastic gradient descent (SGD) learning it suffices that we
      have constants β, Λ, and L s.t. the objective function has a lower and upper bound on the second

129   Preliminaries of secure summation, pretraining, dual SVM representation via the representer theorem,
130   and strong convexity, Lipschitzness, and smoothness definitions are available in Appxs. D.2 to D.5.
131   Differential Privacy (DP) (Dwork et al., 2006b) quantifies the protection of any individual’s data
132   within a dataset against an arbitrarily strong attacker observing the output of a computation on the
133   said dataset. Strong protection is achieved by bounding the influence of each individual’s data on the
134   resulting ML model. For the (standard) definition of DP utilized in our proofs, we refer to Appx. D.1.
135
      DP ERMs via SGD Training We consider machine learning algorithms (learners) that have a
136
      strongly convex objective function, like Support Vector Machines (SVMs), logistic regression (LR),
137   or a softmax-activated single-layer perceptron (SoftmaxReg). These algorithms display a unique local
138   minimum and a lower bound on the objective function’s growth, making them ideal for computing
139   tight DP bounds. If we train these algorithms with stochastic gradient descent (SGD) for a bounded
140   learning rate and their objective function is smooth and Lipschitz, then they are uniformly stable
141   (Hardt et al., 2016) which implies a DP bound on the learner (Wu et al., 2017). For that, the sensitivity
142   of the model is bounded in any iteration and does not scale with the number of iterations if we choose
143   a linearly decreasing learning rate schedule.

145   is trained with SGD on learning rate τm = min(1/β , 1/Λm) for iteration m and has a Λ-strongly
146   convex, β-smooth, and L-Lipschitz objective function J , then the output model f = Tξ (D) has the
147
      sensitivity bound s = 2L/Λn. A mini-batch SGD extension follows by Wu et al. (2017).

151   and R influence the Lipschitzness L and the sensitivity for most objective functions.

      Threat model & security goals. We assume that a fraction of at least t users are honest (say
      t = 50 %), i.e. they follow the protocol including honestly generated noise and don’t collude with
368
      the adversary. In contrast, untrustworthy users can collude with a passive, collaborating adversary
369
      by exchanging information about the randomness used in their local computation. The adversary
370   is assumed to have full knowledge about each user’s dataset, except for one data point of one user.
371   Our privacy goals are (ε, δ)-differential privacy (protecting single samples) and (ε, δ)-Υ-group DP
372   (protecting all samples of a user at once) respectively, depending on whether each user is a local
373   aggregator or a person. To compensate for untrustworthy users, we adjust the noise added by each
374   user by t; e.g., if t = 50%, then we double the noise to satisfy our guarantees.
375
376   Non-interactive protocol. BlindAvg is agnostic to a specific secure summation protocol πSecSum
377   (cf. Alg. 2): e.g. we can use a non-interactive extension of the SecAgg protocol (Bell et al., 2020).
      SecAgg requires 4 communication rounds, but the user only shares their local model in the third

      Under review as a conference paper at ICLR 2025

      round. Following Bogetoft et al. (2009), we introduce J computation servers that aggregate the
379
      model parameters on behalf of the users. Specifically, each user i sends their model f (i) in fixed-
380
      point arithmetic in shares r(j,i) to server j, where for j < J, each r(j,i)
                                                                             P is drawn randomly from
381
      { 1, . . . , B } for a sufficiently large B and where r(J,i) = (f (i) − j<J r(j,i) ) mod B. The

512   Here, and in Tbl. 3 we discuss the most related work and continue the discussion in Appx. C.
513   For interactive ERM via gradient perturbation, Jayaraman et al. (2018) show strong utility-privacy
514   tradeoffs. It requires O(log(nw)) invocations of secure summation (SecSum) for w users with n data
515   points each. For non-interactive learning, however, Jayaraman et al. (2018) show output perturbation
516   results using Chaudhuri et al. (2011) for which they only show that the convergence bounds from the
517   local training are preserved, thus leaving a gap of 1/w to centralized learning. We prove that SVM
518   learning converges in the limit, closing the 1/w gap, and we experimentally show that SoftmaxReg
519
      leads to strong results. Moreover, their work does not exclude leakage from the learner (only from
      the optimum), while we use bounds on the learner SGD (Wu et al., 2017).
520
521   In differentially private federated learning (FL) (McMahan et al., 2017; Abadi et al., 2016) and also in
522   follow-up work (Chen et al., 2022; Choquette-Choo et al., 2024), each user protects its local data by
523   only submitting noisy local model updates protected via DP guarantees (DP-SGD). A central server
524   then aggregates all incoming updates. However, DP-FL is interactive as its communication rounds
525
      increase with the number of training iterations. Moreover, FL achieves a weaker utility-privacy
                                                                                                    √
      tradeoff than centralized learning, already for hundreds of users. FL’s noise scales with O( w) and
526
      has a prohibitively high number of communication rounds.

531   For datasets that work best with small regularization Λ like the deliberately designed S YN FAIL,
532   BlindAvg fails to capture a strong task-performance. Yet, with privacy even the global model works
533   best with mid-range Λ ≥ 1 as the noise scales with O(1/Λ). This points to a more general limitation
534   of small Λ’s for DP ERMs. For unfavorable like non-IID datasets, blind averaging leads to a reduced
535   signal-to-noise ratio, i.e. model parameters smaller than in the centralized setting. This may explain
536
      why blind averaged SoftmaxReg works better for less noise (cf. Tbl. 2). For unbalanced datasets,
      increasing Λ too much to help convergence can lead to poor accuracy as the margin grows so much
537
      that not the same number of support vectors is chosen per label. For a detailed discussion, cf. Appx. B.
538
539   Future work can theoretically capture the utility-link of BlindAvg to the global model for any Λ or
      extend our insight into BlindAvg convergence via the dual problem to other learners like SoftmaxReg.

736   Distributional shift between the public and sensitive dataset.             For pretraining, we leverage
737   contrastive learning. While very effective generally, it is susceptible to performance loss if the shape
738   of the sensitive data used to train an SVM or SoftmaxReg is significantly different from the shape of
739   the initial public data.
740   Input Clipping. We require bounded input data for a bounded sensitivity. In many pretraining
741   methods like SimCLR, no natural bound exists thus we norm-clip the input data by a constant c. To
742   provide a data-independent c on CIFAR-10 and EMNIST, c is based on CIFAR-100 (here: 34.854);
743   its similar data distribution encompasses the output distribution of the pretraining reasonably well.
744   For CIFAR-100, c is based on CIFAR-10 (here: 34.157).
745   Hyperparameter Search. In SVM-SGD, we have two important hyperparameters that influence
746   the noise scale: the regularization weight Λ and the predictor radius R. In the noise scale subterm
747   “c/Λ + R”, the maximal predictor radius is naturally significantly smaller than c/Λ due to the regular-
748   ization penalty. Thus, a bad-tuned R often does not have as large of a utility impact as a bad-tuned Λ.
749   Estimating parameters for a fixed ε from public data is called hyperparameter freeness in prior work
750   (Iyengar et al., 2019). For the other ε values, we can estimate Λ by fitting a (linear) curve on related
751   public data (proposed by Chaudhuri et al. (2011)) or synthetic data (proposed by AMP-NT (Iyengar
752
      et al., 2019)) as smaller ε prefer a higher Λ and vice versa.
753   Blind averaging – Signal-to-noise ratio. For unfavorable yet balanced local datasets, we identify
754   as a main limitation of blind averaging a reduced signal-to-noise ratio: the model is not as large as in
755   the centralized setting w.r.t. the sensitivity analysis. This effect would explain why our experiments
      show that blind averaged SoftmaxReg works better with very little noise (cf. Tbl. 2).

756
      For SVMs, our formal characterization of the effect of blind averaging enables us to describe its
757   limitations more precisely. In summary, we see two effects that reduce the signal-to-noise ratio. One
758   effect comes from the requirement of the SVM training that the model with the smallest norm shall
759   be found that satisfies the soft-margin constraints of the training data points. The local SVM training
760   has fewer data points and, thus, fewer constraints. Hence, unfavorable local data sets will lead to
761   a smaller model. Another effect comes from the averaging itself. Unfavorable local data sets can
762   lead to local models that point in very different directions. When averaging these models, their norm
763   naturally decreases as for any two vectors a, b ∈ Rp we have 0.5∥a + b∥2 ≤ 0.5(∥a∥2 + ∥b∥2 ), and
764   this discrepancy is larger the smaller the inner product is.
765   Blind averaging – Unbalanced data. Convergence holds if all data points are support vectors
766   (SV) which implies a large margin. Yet a regularly trained SVM chooses roughly equally many SVs
767   per class: by the dual problem, we have the constraint y T α = 0 for labels yj ∈ { −1, 1 } and dual
768   coefficients α. If we have an SV inside the margin then αj = Λ−1 . Hence, enlarging the margin such
769   that all data points are SVs can lead to poor utility performance. Moreover, unbalanced local data can
770   deteriorate the performance of blind averaging as observed in the EMNIST experiments (cf. Fig. 7
771   (d)) as we favor privacy, i.e. a constant sensitivity per user, above utility, i.e. optimal local scaling.
772   Active attacks. Active attackers may deviate from the protocol or send maliciously construed local
773   models. If the used secure summation protocol is resilient against active adversaries and can still
774   guarantee that only the sum of the inputs is leaked, privacy is preserved. This follows from analyzing
775   our algorithm for just the honest users and then leveraging the post-processing property of differential
776   privacy. Secure summation protocols such as Bell et al. (2020) leak partial sums under active attacks
777   and will diminish the privacy offered by our work against such adversaries as well.

      Other Privacy-preserving Distributed Learning Protocols. The noise overhead of FL can be
      completely avoided by protocols that rely on cryptographic methods to hide intermediary training up-
784
      dates from a central aggregator. Several secure distributed learning methods protect the contributions
785
      during training but do not come with privacy guarantees for the model such as DP: an attacker, e.g. a
786   curious training party, can potentially extract information about the training data from the model. As
787   we focus on differentially private distributed learning methods, we will neglect those methods.
788
789
      cpSGD (Agarwal et al., 2018) is a protocol that utilizes secure multi-party computation (MPC)
      methods to honestly generate noise and compute DP-SGD. While cpSGD provides the full flexibility
790
      of SGD, it does not scale to millions of users as it relies on expensive MPC methods. Truex et al.
791
      (2019) relies on a combination of MPC and DP methods which also does not scale to millions of
792   users.
793
794   Another line of research aims for the stronger privacy goal of protecting a user’s entire input, called
795
      local DP, during distributed learning (Balle et al., 2020b; Girgis et al., 2021). Due to the strong
      privacy goal, federated learning with local DP tends to achieve weaker accuracy. With Cor. E.3,
796
      evaluated in Fig. 6, we show how BlindAvg achieves a comparable guarantee via group privacy and
797
      MPC-based aggregation overhead: given enough users, any user can protect their entire dataset at
798   once while we still reach good accuracy.
799
800   For DP SVM training, other methods besides output perturbation Chaudhuri et al. (2011); Wu et al.
801
      (2017) such as objective perturbation (Chaudhuri et al., 2011; Kifer et al., 2012; Iyengar et al., 2019;
      Bassily et al., 2019) and gradient perturbation (Bassily et al., 2014; Wang et al., 2017; Feldman et al.,
802
      2018; Bassily et al., 2019; Feldman et al., 2020; Yu et al., 2021) exist. Output perturbation noises a
803
      trained model once calibrated to the model’s sensitivity which marks it an ideal candidate for non-
804   interactive learning. In contrast, objective perturbation noises the objective function while gradient
805   perturbation noises the intermediary gradient updates. Yet, DP requires objective perturbation and the
806   privacy amplification by iteration variant of gradient perturbation Feldman et al. (2018; 2020) to leak
807   no intermediary model updates which marks in the convex SVM setting their amplification above
808   DP-SGD. To protect all of these updates in a distributed setting, the training has to be performed in
809   expensive protocols like MPC or homomorphic encryption. Other gradient perturbation methods that
      leak intermediary gradient Bassily et al. (2014); Abadi et al. (2016); Yu et al. (2021) need one MPC

      invocation per iteration. All these variants have a large communication overhead and are thus highly
811   interactive.

      D.1     D IFFERENTIAL P RIVACY
840
841   To ease our analysis, we consider a randomized mechanism M to be a function translating a database
842   to a random variable over possible outputs. Running the mechanism then is reduced to sampling
843   from the random variable. With that in mind, the standard definition of differential privacy looks as
844   follows.
845   Definition D.1 (≈ε,δ relation). Let Obs be a set of observations, and RV(Obs) be the set of random
846   variables over Obs, and D be the set of all databases. A randomized algorithm M : D → RV(Obs)
847   for a pair of datasets D, D′ ∈ D, we write M (D) ≈ε,δ M (D′ ) if for all tests S ⊆ Obs we have
848   Pr[M (D) ∈ S] ≤ exp(ε) Pr[M (D′ ) ∈ S] + δ with ε, δ ∈ R+ .
849   Definition D.2 (Differential Privacy). Let Obs be a set of observations, and RV(Obs) be the
850   set of random variables over Obs, and D be the set of all databases. A randomized algorithm
851   M : D → RV(Obs) for all pairs of databases D, D′ ∈ D that differ in at most 1 element is a
852   (ε, δ)-DP mechanism if we have M (D) ≈ε,δ M (D′ ).
853
854   In our proofs, we utilize a randomized variant of the sensitivity as proposed by Wu et al. (2017) to
855   achieve DP ERMs via SGD Training.
856   Definition D.3 (Randomized Sensitivity). Let q : (D, r) → R be a randomized function on dataset
857   D and randomness r. The sensitivity of q is defined as s = maxD∼1 D′ maxr ∥q(D, r) − q(D′ , r)∥,
858   where D ∼1 D′ denotes that the datasets D and D′ differ in at most one element. We say that q is an
859   s-sensitivity-bounded function.
860
      In the context of machine learning, the randomized algorithm represents the training procedure of a
861
      predictor. Our distinguishing element is one data record of the database.

881   of roughly 109 users without biasing towards computationally equipped users. Additionally, it offers
882   resilience against client dropouts and colluding adversaries, both of which are substantial features for
883   our distributed setting.

1179   I.2   S ECURE S UMMATION
1180
1181   Before formulating the security of the secure summation protocol, we define a network execution
1182   against a global network attacker that is active and adaptive. For self-containedness, we briefly
1183   present the notion of interactive machines and a sequential activation network execution. More
1184
       general frameworks for such a setting include, e.g., the universal composability framework (Canetti,
       2000).
1185
1186   We rely on the notion of interactive machines. For two interactive machines X, Y , we write ⟨X, Y ⟩
1187   for the interaction between X and Y . We write ⟨X, Y ⟩ = b to state that the machine X terminates
       and outputs b.


                                                         22
       Under review as a conference paper at ICLR 2025


1188
       The network execution Realπ . Next, we define a network execution against a global network
1189   attacker that is active and adaptive. Given a protocol π with client and server code, we define an
1190   interactive machine Realπ that lets each client party run the client code, lets the servers run the
1191   server code, and emulates a (sequential-activation-based) network execution, and interacts with
1192   another machine, called the attacker A. The interaction is written as ⟨A, Realπ ⟩. Whenever within
1193   this network execution a party B sends a message m over the network to a party C, the interactive
1194   machine Realπ , sends this message m to the attacker, activates the attacker, and waits for a response
1195   m′ from the attacker. Realπ then lets this response m′ be delivered to party C, and activates party
1196   C. Moreover, the attacker A can send a dedicated message (compromise, P ) for compromising a
1197
       party P within the protocol execution. Whenever the attacker sends the message (compromise, P )
       to the network execution Realπ , the network execution marks this party P as compromised and sends
1198
       the internal state of this party to the attacker A. For each compromised party P , the attacker decides
1199
       how P acts. Formally, the network execution redirects each message m that is sent to P to the
1200   attacker A and awaits a response message (m′ , P ′ ) from the attacker A. Upon receiving the response
1201   (m′ , P ′ ), the network execution Realπ sends on behalf of P the message m′ to the party P ′ .
1202
1203
       For convenience, we write that a party P runs the client code of a protocol π on input m when the
       network execution runs for party P the client code of π on input m.
