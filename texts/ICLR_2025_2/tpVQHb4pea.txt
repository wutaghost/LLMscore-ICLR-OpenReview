009                                               A BSTRACT
010
011            Large language models (LLMs) have made significant progress in natural lan-
012            guage understanding and generation, driven by scalable pretraining and advanced
013            finetuning. However, enhancing reasoning abilities in LLMs, particularly via re-
014            inforcement learning from human feedback (RLHF), remains challenging due to
015            the scarcity of high-quality preference data, which is labor-intensive to annotate
016            and crucial for reward model (RM) finetuning. To alleviate this issue, we introduce
017            CodePMP, a scalable preference model pretraining (PMP) pipeline that utilizes
018            a large corpus of synthesized code-preference pairs from publicly available high-
019
               quality source code. CodePMP improves RM finetuning efficiency by pretraining
               preference models on large-scale synthesized code-preference pairs. We evaluate
020
               CodePMP on mathematical reasoning tasks (GSM8K, MATH) and logical reason-
021
               ing tasks (ReClor, LogiQA2.0), consistently showing significant improvements
022            in reasoning performance of LLMs and highlighting the importance of scalable
023            preference model pretraining for efficient reward modeling.

      Large language models (LLMs) have made remarkable progress in natural language understanding and
046
      generation, benefiting from scalable pretraining and finetuning techniques like supervised finetuning
047
      (SFT) (Wang et al., 2022; 2023a) and Reinforcement Learning from Human Feedback (RLHF) (Bai
048   et al., 2022a; Lightman et al., 2023b; Bai et al., 2022b; Gulcehre et al., 2023; Schulman et al., 2017;
049   Rafailov et al., 2024). However, enhancing LLMs’ reasoning abilities, particularly in complex logical
050   and mathematical tasks, remains a significant challenge (Wang et al., 2023b; Zhang et al., 2024b).
051   Although RLHF is effective, it relies heavily on high-quality preference data, which is costly and
052   labor-intensive to annotate (Cobbe et al., 2021b; Zheng et al., 2024). This limitation impedes the
053   scalability of reward model (RM) finetuning, which is essential for guiding LLMs toward optimal
      outputs.

072
      To alleviate this issue, prior works like Anthropic’s Preference Model Pretraining (PMP) (Askell et al.,
      2021) have proposed improving reward modeling data efficiency by pretraining preference models on
073
      large-scale preference data from public sources like Reddit and Wikipedia, followed by an efficient
074
      finetuning on limited high-quality human-annotated data. However, this approach is less effective for
075   reasoning tasks due to the scarcity of reasoning preference pairs available online. Compared to other
076   tasks, manually annotating preference data for reasoning is inherently more challenging and difficult
077   to scale (Zhang et al., 2024b; Zhou et al., 2023), highlighting the urgent need for a scalable PMP
078   approach for reasoning tasks.
079
      In this paper, we propose CodePMP, a scalable preference model pretraining pipeline that enhances
080
      LLM reasoning abilities using synthesized preference pairs derived from high-quality, publicly
081   available source code. Code, with its inherently logical and structured nature, provides rich data
082   suitable for reasoning tasks. Recent works (Zhang et al., 2024b; Aryabumi et al., 2024) also show a
083   strong correlation between code training and reasoning improvements in LLMs. By leveraging the
084   huge amount and diverse coverage of source code available on platforms like GitHub, CodePMP offers
085   a scalable solution for pretraining preference models, thereby improving RM finetuning efficiency
086   and enhancing LLMs reasoning performance.
087   Specifically, CodePMP generates preference pairs by synthesizing chosen and rejected code responses
088   for a given code-related prompt or description using CodeLLMs. A strong CodeLLM produces
089   higher-quality (chosen) responses, while a weaker model generates suboptimal or even low-quality
090   (rejected) responses. These <chosen, rejected> pairs, accumulated in the millions, form a large-scale
091   synthesized preference dataset. This dataset is then used to pretrain the preference model with
092   pairwise ranking objectives (Cobbe et al., 2021b; Charniak & Johnson, 2005), providing an good
093   initialization for further fine-tuning the reward models.
094   We evaluate CodePMP on widely studied reasoning tasks, including mathematical reasoning tasks
095   such as GSM8K (Cobbe et al., 2021b) and MATH (Hendrycks et al., 2021), as well as logical
096   reasoning tasks like ReClor (Yu et al., 2020) and LogiQA2.0 (Liu et al., 2023). Our experiments
097   demonstrate that CodePMP significantly improves RM fine-tuning accuracy and Best-of-N perfor-
098   mance in reasoning tasks, outperforming direct RM fine-tuning, as highlighted in Figure 1. Moreover,
099   additional experimental results reveals that RMs initialized with CodePMP exhibit greater robustness
100   across different tasks. These results indicate that code-derived preference data provides a scalable,
101
      cost-effective solution for enhancing LLM reasoning capabilities while reducing reliance on extensive
      preference annotation, achieving more effective reward modeling for reasoning tasks.
102
103   In summary, our main contributions are:
104
105   1. We introduce CodePMP, a scalable method that uses code-derived preference pairs to pretrain
106
         preference models, improving sample efficiency and robustness for downstream RM finetuning.
107   2. We validate that CodePMP significantly improves performance on reasoning tasks, demonstrating
         that a scalable PMP process positively impacts LLM reasoning abilities.

115
      ing, aimed at modeling sequences of language. This is typically achieved through Causal Language
      Models (Causal LM), where the model is trained to maximize the likelihood of predicting the next
116
      word wt given the preceding words w1 , w2 , . . . , wt−1 . The training process minimizes the negative
117
      log-likelihood of the predicted word sequence:

123   This loss function LLM encourages the model to capture the underlying patterns in the data. Trans-
124   former architectures (Vaswani, 2017) have become the standard for Causal LM due to their ability to
125   handle long-range dependencies effectively.
126
127   Reward Modeling Reward modeling (RM) is crucial in reinforcement learning from human
128   feedback (RLHF), providing a scalar reward signal that guides the learning process based on output
129   quality. The reward model Rθ predicts the quality of an output y given a context x as s = Rθ (x, y),
130
      where s is the scalar reward score. In preference modeling, RM predicts the relative quality of outputs
      by comparing pairs. A common method is the Pairwise Ranking Loss, where the model assigns
131
      higher scores to preferred (chosen) outputs:
132
133
134                                      LRM = − log (σ(schosen − srejected ))
135
      , where schosen = Rθ (x, ychosen ) and srejected = Rθ (x, yrejected ), and σ(·) is the sigmoid function.
136
137
      Best-of-N Sampling Best-of-N (BoN) sampling improves LLM reasoning (Cobbe et al., 2021b;
138
      Lightman et al., 2023b). In this approach, N candidate solutions {y1 , y2 , . . . , yN } are generated
139
      by sampling from the LLM’s output distribution for a given problem. A reward model scores each
140   candidate and selects the highest-scoring one as the final answer:

                                                     yi ∈{y1 ,y2 ,...,yN }
144
145   , where Rθ (x, yi ) represents the reward score for each candidate yi . This technique is especially
146   effective in tasks like mathematical problem-solving and logical inference, where selecting the most
147   plausible solution from a diverse set of outputs improves overall accuracy (Wang et al., 2022).

151   3.1   M ODEL D ESIGN
152
153   Code Preference Model Pretraining (CodePMP) is designed to enhance the sample efficiency of reward
154   models, particularly for reasoning tasks where high-quality preference data is scarce. Traditionally,
155   reward models are finetuned on small, curated datasets, which limits their effectiveness in complex
156   tasks like mathematical reasoning or logical deduction. CodePMP mitigates this limitation by
157   introducing a pretraining phase between basic language model pretraining and finetuning on domain-
158
      specific reasoning datasets. This phase leverages a large, diverse dataset of code-preference pairs,
      enabling the model to learn generalizable patterns and ranking strategies.
159
160   CodePMP training involves two key components: Reward Modeling (RM) and Language Modeling
161   (LM). In RM, the model is trained on code-preference pairs, learning to assign higher scores to the
      chosen code through a pairwise ranking loss. In LM, only the chosen code is used for autoregressive

180   3.2     DATA C ONSTRUCTION
181
182   To enable scalable preference model pretraining, we construct a dataset sourced from GitHub, which
183
      includes a diverse range of repositories and associated metadata. The dataset consists of two primary
      components: Repository Data comprises over 1.3 billion code files from GitHub repositories, while
184
      GitHub Metadata includes information such as commit histories, discussions, pull requests, and
185
      issues.
186
187   The CodePMP dataset is constructed through a systematic process. First, raw source code is processed
188   by a description summarizer, typically an instruction-tuned CodeLLM, to generate prompts that
189
      describe the functionality of the code.
190   These prompts are then used by two CodeLLMs of different capabilities to generate code snippets:
191
192           • Chosen response: Generated by a more advanced CodeLLM (e.g., 6.7B parameters).
193           • Rejected response: Generated by a less capable CodeLLM (e.g., 1.3B parameters).
194
195   This process yields pairs of code responses—one chosen and one rejected—which are used for
196   preference modeling. This scalable approach significantly enhances pretraining efficiency, improving
197   performance on downstream tasks.
198   The steps of the CodePMP methodology are outlined systematically in Algorithm 1.

231   4.1.3    E VALUATION S ETTINGS
232
233
      Following (Zhang et al., 2024a), we adopt two evaluation metrics:
234
235   RM Accuracy This metric measures the accuracy of the reward model in distinguishing chosen
236   from rejected solutions on the holdout test sets. It provides insight into the model’s ability to classify
237
      individual sequences.
238
239   Best-of-N (BoN) Accuracy This metric evaluates the proportion of correct solutions selected
240
      by the finetuned RM from N candidate responses. It assesses the model’s group-wise ranking
      performance, focusing on its ability to select the correct answer from a set of candidates. We use
241
      MetaMath-Mistral-7B (Yu et al., 2023) as the generator for this evaluation.
242
243   For mathematical reasoning, we use the GSM8K (Cobbe et al., 2021b) and MATH (Hendrycks et al.,
244   2021) test sets. For logical reasoning, we evaluate on the ReClor (Yu et al., 2020) and LogiQA2.0 (Liu
245   et al., 2023) test sets. Further details can be found in Appendix D.
246   Note that logical reasoning questions typically involve a paragraph followed by statements to be
247   judged true or false, making Best-of-N evaluation challenging. Therefore, we use multiple-choice
248   accuracy, where the reward model ranks four manually annotated options and selects the best one.
249   This metric is equivalent to Best-of-4, and thus, for logical reasoning tasks, multiple-choice accuracy
250   and Best-of-N are used interchangeably.

      4.2.3   RM S AMPLE E FFICIENCY C OMPARISON
303
304   One key advantage of CodePMP is its ability to improve the sample efficiency of RM finetuning.
305   To assess this, we conduct experiments with progressively larger sample sizes for RM finetuning.
306   As indicated by (Kaplan et al., 2020), optimal results are achieved when the learning rate scheduler
307   completes its decay at the end of training. Therefore, rather than evaluating intermediate checkpoints,
308   we retrain models with varying sample sizes for optimal results. Figure 4 and 10 (Appendix E)
309   show that as the sample size increases, RMs with CodePMP initialization consistently outperforms
310
      others in both BoN and RM accuracy. Notably, RMs finetuned with CodePMP initialization using
      just 0.5k samples surpasses RMs finetuned without CodePMP initialization using 40k samples on
311
      mathematical tasks, demonstrating CodePMP’s significant advantage in sample efficiency. However,
312
      as sample size increases, this advantage diminishes slightly, suggesting that with much larger datasets,
313   CodePMP’s benefit may become less pronounced, but the cost of manual labeling remains a key
314   consideration.

482   Bradley-Terry and Plackett-Luce to capture human preferences (Bradley & Terry, 1952; Plackett,
483   1975; Cobbe et al., 2021b; Saunders et al., 2022; Lightman et al., 2023b; Wang et al., 2023b; Uesato
484   et al., 2022; Luo et al., 2024; Yu et al., 2024; Stiennon et al., 2020; Nakano et al., 2021). Recent
485   advancements introduced probability-based methods (Zhao et al., 2023; Jiang et al., 2023), offering
      more refined predictions. Innovations such as the Critique-out-Loud model (Ankner et al., 2024)

      Under review as a conference paper at ICLR 2025


486
      integrate natural language critiques to enhance RMs. Generative reward models (GRMs) (Yang
487   et al., 2024b) further improve sample efficiency. Preference Modeling Pretraining (PMP) (Askell
488   et al., 2021) introduces a pretraining phase, leveraging large-scale pairwise ranking data to boost RM
489   performance. However, many of these methods rely on costly manual annotations or limited data,
490   limiting scalability. CodePMP addresses this issue by automating preference data generation from
491   code, improving RM sample efficiency and reducing dependency on manual data collection.
492
493   Code Training Incorporating code into LLM pretraining has significantly improved performance in
494   tasks such as commonsense reasoning (Madaan et al., 2022) and mathematical reasoning (Liang et al.,
495   2022; Shao et al., 2024; Yang et al., 2024a). Code also enhances general reasoning abilities (Muen-
496   nighoff et al., 2023; Fu & Khot, 2022; Ma et al., 2023). Recent research (Dong et al., 2023; Ma
497   et al., 2023) shows that integrating code during supervised finetuning strengthens LLMs in complex
498
      decision-making tasks. CodePMP pioneers the use of scalable, synthetically generated code prefer-
      ence pairs, reducing reliance on manual annotations (Dubey et al., 2024; Gemini-Team et al., 2024;
499
      Groeneveld et al., 2024; Bi et al., 2024). This approach improves sample efficiency and scalability in
500
      reasoning-intensive tasks, opening new possibilities for LLM performance improvements.
501
502
      LLM Reasoning Improving reasoning in LLMs remains a challenge, and various advanced tech-
503
      niques have been proposed. Chain of Thought (CoT) prompting (Wei et al., 2022; Fu et al., 2023)
504   improves reasoning by generating intermediate steps, while supervised finetuning (SFT) with CoT
505   further boosts performance (Cobbe et al., 2021a; Liu et al., 2024; Yu et al., 2023). Other methods
506   focus on increasing inference time computation, such as problem decomposition (Zhou et al., 2022),
507   search-based approaches like MCTS (Xu, 2023), and using LLMs as verifiers (Huang et al., 2022;
508   Luo et al., 2023). Reward models, including outcome-based (ORM) and process-based (PRM), also
509   improve performance, with PRM showing stronger results (Lightman et al., 2023a; Wang et al.,
510   2023b). Unlike these approaches, CodePMP introduces a scalable preference model pretraining stage,
511   which is compatible with all the aforementioned methods.
512
513   7   C ONCLUSION & F UTURE W ORKS
514
515   This paper introduces CodePMP, a scalable pretraining approach that leverages code-preference
516   pairs to improve reasoning capabilities in large language models. Experimental results validate that
517   CodePMP significantly improves sample efficiency and boosts performance on reasoning tasks.
518
      For future work, we aim to extend CodePMP in two key directions. CodePrMP will focus on utilizing
519
      compiler and interpreter verifiability to provide low-cost process supervision signals. GenPMP
520
      will explore how to improve sample efficiency and the performance of generative reward models by
521   integrating code data.

774
775   The RM finetuning for mathematical reasoning uses the MathShepherd dataset( (Wang et al., 2023b)),
776   which contains 444k query-response samples, with some queries having multiple distinct responses.
777
      We divide the dataset into a 400k training set and a 44k test set. For RM finetuning, we construct
      preference pairs by selecting both correct and incorrect responses for the same query. To form the
778
      4.3k test set, we combine one positive and negative sample for each query from the original test set.
779
780   We also create two training sets of different sizes: MathShepherd-preference-800k and MathShepherd-
781   preference-40k. The 800k training set is built by combining multiple positive and negative samples
782   for each query in the original training set, resulting in 800k samples. In contrast, the 40k training set
783
      randomly selects one positive-negative pair for each query, totaling 40k samples.

905   F     L OGICAL R EASONING E VALUATION E XAMPLES
906
907   We randomly select and present examples from the Reclor test set, which consists of multiple-choice
908   questions based on a given passage. While it is possible to have the model generate additional
909   candidate answers to create a Best-of-N test, it becomes difficult to ensure that the original correct
910
      answer remains among the options after introducing new candidates, and to identify the new correct
      answer. We attempt to use GPT-4o to annotate the correct answers for 32 responses, but the
911
      consistency with manual inspection is low, as is the consistency of GPT-4o’s own multiple annotations.
912
      It can be inferred that the consistency rate would worsen if expanded to 256 responses. Therefore, after
913   careful consideration, we decide to use RM to score only the original four manually annotated answer
914   options, match the top-ranked option with the manually annotated correct answer, and calculate
915   accuracy. In principle, this method is equivalent to the Best-of-4 test.
