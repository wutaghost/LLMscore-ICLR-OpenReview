007                                               A BSTRACT
008
009            Existing crowd-counting methods rely on the manual localization of each per-
010            son in the image. While recent efforts have attempted to circumvent the anno-
011            tation burden through vision-language models or crowd image generation, these
012            approaches rely on pseudo-labels to perform crowd-counting. Simulated datasets
013            provide an alternative to the annotation cost associated with real datasets. How-
014            ever, the use of large-scale simulated data often results in a distribution gap be-
015            tween real and simulated domains. To address the latter, we introduce knowledge
016            retrieval inspired by knowledge-enhanced models in natural language processing.
017
               With knowledge retrieval, we extract simulated crowd images and their text de-
               scriptions to augment the image embeddings of real crowd images to improve
018
               generalized crowd-counting. Knowledge retrieval allows one to use a vast amount
019
               of non-parameterized knowledge during testing, enhancing a model’s inference
020            capability. Our work is the first to actively incorporate text information to regress
021            the crowd count in any supervised manner. Moreover, to address the domain gap,
022            we propose a pre-training and retrieval mechanism that uses unlabeled real crowd
023            images along with simulated data. We report state-of-the-art results for zero-shot
024            counting on five public datasets, surpassing existing multi-model crowd-counting
025            methods. The code will be made publicly available after the review process.

027   1   I NTRODUCTION
028
029   Crowd-counting has garnered significant interest owing to its extensive applications in safety and
030
      population management (Sindagi & Patel, 2018; Kang et al., 2018). Accurately estimating counts
      becomes particularly challenging, especially in densely populated areas.
031
032
      Most prominent crowd-counting methods either estimate a density map (Sindagi & Patel, 2017;
      Ranasinghe et al., 2024; Han et al., 2023) or localize head positions (Song et al., 2021; Liang et al.,
033
      2022b) to estimate the count. However, these methods require point-level annotations for human
034
      heads, which is an expensive and laborious process. Recently, to relieve the cost of annotation, the
035   field has been moving towards using vision-language models and synthetic images. An illustrative
036   example of this trend is observed in the introduction of the CrowdCLIP (Liang et al., 2023) model,
037   which integrates the CLIP (Radford et al., 2021) architecture for crowd-counting showcasing a con-
038   temporary approach in merging vision and language models for this specific task. While CrowdCLIP
039   is positioned as an unsupervised model requiring no explicit count labels, evaluating the test set in-
040   volves determining the optimal count label structure for performance assessment. In contrast, the
041   AFreeCA (D’Alessandro et al., 2024) model proposes a fully supervised crowd-counting strategy by
042   synthesizing crowd images using stable diffusion and multi-modal supervision. However, a notable
043
      challenge arises in AFreeCA, where the actual crowd count in the synthesized images diverges from
      the count provided as the text condition to the model, introducing inherent noise into the pipeline.
044
045
      However, CrowdCLIP and AFreeCA demonstrate the transferability and generalizability of incor-
      porating text knowledge and a vast amount of data to annotator-free crowd-counting. Consequently,
046
      we can address the annotation cost involved in crowd-counting by training a model with simulated
047
      data to perform zero-shot crowd-counting on real images. The benefits of using simulated data are
048   two-fold: 1. we can create point annotations without any human labor. 2. We can create a huge
049   amount of data for the model to train. Naturally, models need more capacity to parameterize a large
050   corpus of data, as evidenced by large language models. However, recently developed retrieval aug-
051   mented generation (Lewis et al., 2020) for natural language processing demonstrated the advantage
052   of using non-parametric knowledge (external information) for more updated, reliable response gen-
053   eration. Following this, RA-CLIP (Xie et al., 2023) illustrated the advantage of using a reference
      database for zero-shot performance with vision-language models for classification. However, the

094   2   R ELATED WORKS
095
096   Annotator-free crowd-counting. Existing crowd-counting methods use real-world images with
097
      manually annotated ground truth, a labor-intensive and costly process. To mitigate the depen-
      dence on these human annotations, recent research has explored annotator-free approaches to crowd-
098
      counting. For example, CSS-CCNN utilizes self-supervised learning by pretraining the image en-
099
      coder with a rotation prediction task before fine-tuning the encoder and a density decoder using
100   Sinkhorn matching, completely bypassing ground truth annotations. Similarly, CrowdCLIP lever-
101   ages the CLIP architecture to train an image encoder for crowd interval prediction by contrasting
102   image features with count interval labels. In contrast to these methods, AFreeCA performs fully
103   supervised crowd-counting by training its network on synthetic images generated using stable dif-
104   fusion, enabling the model to learn crowd counts directly from artificially generated data.
105
106   Real and simulated crowd images. Text descriptions about images establish the multi-modal re-
107   lationship among image-text pairs. However, these text descriptions generally include information
      about objects present in the image and the context of the image. But specific information like the

132
      to get the crowd count, which is a tedious and time-consuming task (D’Alessandro et al., 2024). In
133   contrast, using simulated data (Wang et al., 2019) eliminates the necessity of labor for annotation

      augmenting the input image embeddings with image-text pair information from an external database.
      In addition, Chen et al. (2024) and Liu et al. (2023) utilize the knowledge-enhanced models to im-
142
      prove classification performance with diffusion models and customized visual models. However,
143
      the above methods cater to classification for a given image. In our work, we use external knowledge
144   retrieval to improve the performance of crowd-counting in a weakly supervised learning manner.

148   database for a given query image to enhance the inference performance for crowd counting as shown

153
      Text descriptions for simulated images. We utilize the crowd locations, weather conditions, and
154   time conditions available for each crowd image in the GCC dataset. For each crowd image, we
155   construct a text description like,
156            “The image has a [weather condition] weather with
157            [crowd count] people in the [time of day].”
158   For the weather conditions, we use {clear, cloudy, rainy, foggy} as the labels, and for
159   the time of day, we use {morning, evening, night} as the prompts. These text descriptions
160   are only constructed for the images that will be included in the reference database. We use 80% of
161   images from the GCC training set for the reference database. However, including only simulated
      data in the reference set introduces a domain gap between simulated and real-world test images.

      4.1   I MPLEMENTATION DETAILS
297   We use the Vision Transform (ViT-B/16) (Dosovitskiy et al., 2021) as the image encoder with pre-
298   trained weights on ImageNet-21K (Deng et al., 2009) with the hidden dimension size set to 768.
299   The input image size to the image encoder is 224 × 224. For the text encoder we use the Sentence
300   Transformer (SentenceT) (Reimers & Gurevych, 2019). The SentenceT architecture has 6 Trans-
301   former block layers and outputs a 384 dimensional vector for each sentence. To reconcile the image
302   embeddings with the text embeddings, we project the output of the image encoder from a 768 di-
303   mensional vector to a 384 dimensional vector. Moreover, we use an MLP for the count decoder to
304   map the augmented image embeddings to the crowd count following Liang et al. (2022a).
305   We implement our framework with PyTorch (Paszke et al., 2019). All experiments are conducted
306   on 4 NVIDIA RTX A6000 GPUs, and we use a batch size of 32 for pre-training the image encoder
307
      and training the KAM. First, the image encoder is trained for 200 epochs with unlabeled real crowd
      images and simulated crowd images. To pre-train the image encoder with the ranking loss, we use
308
      the AdamW optimizer (Loshchilov & Hutter, 2018) with a learning rate of 1e-3 and a weight decay
309
      of 0.01 factor and a linear warm-up over ten epochs. We use five ranked crops with a 1 : 0.75 scaling
310   ratio between consecutive crops on real crowd images following Liu et al. (2018), whereas we use
311   four ranked crops with a 2 : 1 scaling ratio for the simulated crowd images. During pre-training of
312   the image encoder, we perform RandAugment (Cubuk et al., 2020), random horizontal flip, random
313   Gaussian blur, and random color distortions. To train the KAM and the count decoder for generalized
314   crowd-counting, we adopt the same optimizer with a learning rate of 1e-5 and perform training for
315   150 epochs using simulated data. To assess few-shot performance, we fine-tune the image encoder,
316   KAM, and count decoder on real labeled crowd images.

      F    E FFICIENCY ANALYSIS
895
896   For the retrieval process, we use the naive maximum inner product search. This involves computing
897   the similarity between image embeddings and crop embeddings in the reference database and sorting
898
      the similarity scores to find the closest neighbors.
899   Suppose the reference database is of size N , the embedding dimensionality is of size d, and we
900
      need to find the nearest k neighbors. Then, the computational efficacy of the whole process is
      O(N · d + N · log k). Accordingly, as the retrieval space scales, the time it takes for the retrieval
901
      process will increase. However, for larger reference databases, using approximation methods such
902
      as the k-d tree, the computational complexity can be reduced to O(log N ) for smaller dimensional
903   sizes, but still, the time consumed will increase with the size of the reference database.
