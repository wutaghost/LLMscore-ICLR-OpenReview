008                                              A BSTRACT
009
010            There is growing demand for large language models which can process hundreds
011            of thousands or even millions of input tokens. Inference at this extreme scale
012            demands significant computational resources and costs. To address the inference
013            time costs associated with running self-attention based transformer language mod-
014            els on long contexts, we propose a tunable mechanism that reduces the cost of the
015            forward pass by attending to only the most relevant tokens at every generation step
016            using a top-k selection mechanism. We showcase the efficiency gains afforded by
017            our method by performing inference on context windows up to 1M tokens us-
018
               ing approximately 16GB of GPU RAM. Our experiments reveal that models are
               capable of handling the sparsity induced by the reduced number of keys and val-
019
               ues. By attending to less than 1% of input tokens, we achieve over 95% of model
020
               performance on common long context benchmarks (LM-Eval, AlpacaEval, and
021            RULER).

      Models (LLMs) are commonplace. Leading this charge is Ring Attention (Liu et al., 2023), which
055   uses round-robin communication between many servers to scale up computation. However, this
056   approach is not a sustainable or commercially viable long-term solution due to the extremely high
057   per-token inference costs.
058
      In this paper, we observe that modern LLMs only require the top contributors to each attention
059
      computation to perform well. This opens the door to do long-context inference with very little GPU
060
      computing power. We build an implementation of attention in which keys and values are stored in
061   a vector database in CPU memory. When attention is computed using a query vector, we retrieve
062   only the keys with the largest attention scores. This retrieval can be done in sublinear time using an
063   approximate k-nearest neighbor search, enabling long-context inference using plentiful and cheap
064   CPU memory, and without the heavy computational overhead required for full attention.
065
      Our proposed approach bears similarities to Retrieval Augmented Generation (RAG), in which a
066
      large corpus of text is analyzed by retrieving a small subset of the most relevant documents from
067   a vector database and placing them into the context window. However, retrieval models lack the
068   sophisticated zero-shot and reasoning capabilities of a full-scale language model. Our proposed
069   approach is a middle ground between long-context inference and RAG; we retain the inherent ca-
070   pabilities of language models, but also leverage a vector database to reduce computation. By seam-
071   lessly integrating the capabilities of long-context inference and RAG, we make it possible to perform
072   generation over context lengths in the millions of tokens with a single, inexpensive GPU.
073   Our primary contributions are summarized as follows:
074
075                      • We propose a simple method for sublinear long-context generation by using top-k attention
076                        over preprocessed states in a vector database.
077
                         • We show that this technique achieves high fidelity on common benchmarks and long-
078
                           context tasks.
079
080                      • We provide empirical analysis on why top-k attention works and prescriptive recommen-
081                        dations for choosing the optimal k for a given task.

      These experiments motivate the idea of exploiting model sparsity to speed up attention computations.
169   Empirically, very few keys are needed to perform inference, but it is important to select keys that are
170   primary contributors to the attention computation. The clear way forward is to use a vector database
171   to retrieve the top-k most influential tokens, enabling the GPU to perform matrix computations
172   while the keys and values live in CPU memory. This approach alleviates both the computational and
173   memory barriers to doing long-context inference on a small GPU.

      3     M ETHODOLOGY
176
177   In this section, we elaborate on our method for reducing memory costs at inference time using top-k
178   attention. But first, we introduce some notation and common terms related to attention blocks.
179
180   3.1    C AUSAL S ELF -ATTENTION
181
182   The first step of self-attention as implemented in a standard transformer block is to project the se-
183   quence of token embeddings x ∈ RN ×D via learned weight matrices into queries, keys, and values:
184   Q = xWQ , K = xWK , V = xWV ∈ RN ×D . The canonical scaled softmax attention is imple-
185   mented by “scoring” the relevance of each key vector to each query vector via inner products, and
186
      then the resulting “attention” scores are renormalized via a softmax function. Finally, at each query
      position, the scores are used to weight a summation over value vectors to produce the output hidden
187
      state at every position. This is often compactly represented using the following matrix notation:
188
                                                                     QK T

189                                 Attention(Q, K, V ) = Softmax √           V.                        (1)
190                                                                     D
191                                                                      √
192   We refer to S = QK T as the score matrix, and A = softmax(S/ D) as the attention matrix. The
193   transformer block performs a few more operations including normalization and position-wise feed-
194
      forward transformations, although our implementation leaves these operations intact and we do not
      consider them further.

      3.2    I NFERENCE W ITH KV C ACHES
197
198   Autoregressive transformer language models generate text by repeatedly predicting the most likely
199   next token conditioned on a sequence of previous tokens, and then concatenating the newly chosen
200   token to the original input sequence before repeating the process. Because the key and value com-
201   putations of past tokens are unaffected by the addition of future tokens, it is typical to cache key and
202   value pairs from each token and re-use them each time a token is generated.

240   3.3   T OP -k ATTENTION
241
      Even when using a KV Cache, as the length of the sequence grows, the cost of computing the
242
      attention scores for just the single newly generated token becomes non-trivial. We cut down on this
243   growing cost by only considering a subset of the most relevant keys in the KV cache by performing
244   a k-nearest neighbor search over the key vectors for the new query vector, returning only those value
245   vectors corresponding to the k-largest attention scores:
246
247
                                                                    1

                                                                              T

248                     top-k-Attention(Q, K, V ) = Softmax        √ top-k QK       V.                  (2)
                                                                    D
249
250   This can be visualized as a mask selecting only the k largest values in each row of the attention
251   matrix (Figure 5 center). This increases the sparsity of the attention matrices while retaining the
252   most important score values. The weighted sums of value vectors are taken only over the k value
253   vectors corresponding to the k largest attention scores, a final operation with complexity only O(k).
254
      An optimal implementation of this method abstracts the implementation and, critically, space-time
255   complexity of the k-NN search over the KV cache. In our experiments we offload the KV cache and
256   vector search operation to the CPU while performing the Q, K, and V projections on the GPU (as
257   well as other matrix operations in the transformer model).
258
      Algorithm 2 concretely describes how our method takes in a KV Cache and constructs a nearest
259
      neighbor vector search data structure with the keys as the “database” vectors. When a new query
260
      vector is generated, it searches over the cache for the top-k score values, and densely computes the
261   score values with respect to any previously generated tokens. K and V projections for the newly
262   generated tokens can optionally be added to the database. However, our experiments operate on
263   the scale of millions for the database and thousands for generated tokens, so we maintain a dense
264   KV cache for newly generated tokens on the GPU, while using a sparse top-k cache for keys in the
265   pre-computed context cache.

      5.1   E FFICIENT ATTENTION M ECHANISM
451
452   The memory usage of the classic “self-attention” operation (Vaswani et al., 2017) grows quadrati-
453   cally with the input sequence length. This high demand for GPU memory impedes scaling LLMs
454   to longer sequences, hindering many practical applications. To overcome this hurdle, recent works
455   have developed various efficient alternatives to the attention operation. Here, we divide existing
456   efficient attention mechanisms into two categories: one is “exact” attention (Dao et al., 2022), and
457   the other is “approximate” attention (Beltagy et al., 2020; Han et al., 2023).
458   Exact attention algorithms strive to maintain near mathematical equivalence to the original attention,
459   and the memory efficient approaches usually leverage efficient CUDA implementations (Dao et al.,
460   2022), blockwise computation (Liu et al., 2023; Kwon et al., 2023), and sequence parallelization (Li
461   et al., 2023). On the other hand, approximate attention algorithms save memory by reducing the size
462   of the actual attention matrix at the loss of some of the original fidelity. A straightforward solution
463   is sliding window attention (Beltagy et al., 2020), which assumes that local interactions dominate
464
      and long range dependencies can often be ignored. StreamingLLM (Xiao et al., 2024) discovered
      the “attention-sink” phenomenon and proposed a modified sliding window attention that alleviates
465
      the performance degradation in windowed attention. Other works explore the sparsity of attention
466
      matrices and propose different sparse attention operations to improve efficiency (Child et al., 2019;
467   Qiu et al., 2020; Nawrot et al., 2024; Jiang et al., 2024). Our work falls into the approximate attention
468   category; while similar ideas have been explored in previous work (Gupta et al., 2021; Klett & Ahle,
469   2024), we are the first to implement and run our top-k attention at a million token context-length
470   scale.
471
472   5.2   S YSTEM S OLUTIONS TO L ONG -C ONTEXT IN LLM S
473
474   There is a long line of proposed system solutions to address the long-context problem in LLMs
475   at inference. Flash Attention, and later extended to Flash-Attention 2, provides theoretical linear
476   complexity over the sequence length, providing two to four times the memory savings over standard
477
      attention at inference time (Dao et al., 2022; Dao, 2024). vLLM’s implementation of Paged At-
      tention (Kwon et al., 2023) optimizes for throughput over many requests, but, like Flash Attention,
478
      this Paged Attention implements a block matrix multiplication algorithm that allows for memory
479
      savings at inference time. With a more retriever-like system, Klett & Ahle (2024) propose using
480   sliding window attention to build a cache and k nearest neighbor (kNNs) using its utilization up to
481   16k context length. For KV cache management, Wu et al. (2022) alter the attention mechanism to
482   memorize the internal representations of past inputs to acquire new skills and knowledge. Li et al.
483   (2024) propose SnapKV, a method that reduces KV cache size without the need for fine-tuning by
484   carefully selecting clustered important KV positions for each attention head reducing memory while
485   increasing speed. Adnan et al. (2024) introduce Keyformer, which reduces the size of the cache
      by utilizing the sparsity in language models and only considering specific keys to decrease latency


                                                         9
486
      and increase throughput. Singhania et al. (2024) construct a low-cost proxy to full attention using
487   PCA, which informs the token subset for the attention computation. However, these solutions do not
488   support context lengths past a few hundred thousand tokens. Thus, Liu et al. (2023) propose Ring
489   Attention, which employs blockwise computation for the self-attention and feedforward operations,
490   distributing long sequences over multiple devices and overlapping key-value block communication
491   with the computation of attention. However, this method is computationally demanding, both in
492   terms of memory usage and processing speed, as it requires substantial resources to manage the
493   parallelization of large sequences across devices while maintaining communication overhead. To
494   the best of our knowledge our approach is the first to achieve inference on million token context
495
      windows on a single commodity GPU.
496
497   6   C ONCLUSION
498
499   We have demonstrated the capability of a top-k attention mechanism to operate at the million token
500   scale on a single GPU. In addition, our investigation of attention distributions across layers points
501   to future directions where the choice of k can be adapted across layers. We achieve sublinear com-
502   plexity and evaluate at over 95% accuracy on common benchmarks while using only 1% of the
503   context length on average in the attention block. This exploitation of attention sparsity opens up
504   new directions for efficient and viable solutions to long-context in language models.
