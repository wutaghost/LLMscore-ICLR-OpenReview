009                                                      A BSTRACT
010
011             This paper explores continual learning (CL) using Vision Transformer (ViT) in
012             streaming tasks under the challenging exemplar-free class-incremental (ExfCCL)
013             setting. We formulate ExfCCL as a learning problem consisting of two key sub-
014             systems: (i) task ID inference for test data, which selects appropriate task-specific
015             head classifiers to accounting for varying class distributions across tasks and
016             streams, and (ii) a dynamic learning-to-grow feature backbone that balances stabil-
017
                ity and plasticity, mitigating catastrophic forgetting through task synergies. Follow-
                ing the common protocol that the first task can train a ViT sufficiently well as the
018
                base model, we address these sub-systems from a continual memory learning per-
019
                spective. To support task ID inference, we utilize an external memory mechanism
020             that maintains task centroids computed by the base ViT throughout CL. For the
021             feature backbone, we identify optimal placements for internal (parameter) memory
022             to enable a dynamic, task-synergy guided growing feature backbone. We propose a
023             Hierarchical Exploration-Exploitation (HEE) sampling-based neural architecture
024             search (NAS) method that effectively learns task synergies by continually and struc-
025             turally updating internal memory with four basic operations: reuse, adapt,
026             new, and skip. Our approach, dubbed Continual Hierarchical-Exploration-
027             Exploitation Memory (CHEEM), is evaluated on the challenging Visual Domain
028
                Decathlon (VDD) and ImageNet-R benchmarks, demonstrating its effectiveness.

      Recently, the emergence of powerful pretrained Transformer models (Vaswani et al., 2017; Doso-
      vitskiy et al., 2021; Radford et al., 2021) has driven significant interest in ExfCCL using pretrained
082
      and frozen Vision Transformers (ViTs) (Dosovitskiy et al., 2021), primarily explored through the
083
      lens of prompt-tuning or prefix-tuning (Wang et al., 2022d;c;a; Smith et al., 2023a). However,
084   this approach presents two main drawbacks: (i) A single frozen pretrained Transformer backbone
085   cannot accommodate all streaming tasks of diverse nature, such as those found in the VDD bench-
086   mark (Rebuffi et al., 2017a), shown in Fig. 2. While this method maximizes the stability of the
087   feature backbone, it relies on input or prefix prompts to address plasticity by explicitly leveraging the
088   self-attention mechanisms. (ii) Due to the quadratic complexity of ViTs concerning the number of
089   input tokens, incorporating task-specific input prompts or layer-wise prefix prompts into a frozen
090   pretrained Transformer backbone significantly increases computational demands. Continual learning
091   where computational requirements always grow for new tasks by design regardless of task complexity
092
      fails to reflect true intelligence.
093   The challenge of structurally and dynamically updating a pretrained Transformer backbone for
094   ExfCCL remains an open problem, as it requires achieving a balance between the backbone’s stability
095   and plasticity while enabling dynamic computation and mitigating catastrophic forgetting.
096   Moreover, ExfCCL involves tackling the challenge of designing task head classifiers, which can
097   either explicitly infer task IDs to select task-specific head classifiers (Wang et al., 2022a) or share a
098   growing, common head classifier (Wang et al., 2022d;c; Smith et al., 2023a). The former approach is
099   straightforward and can help in learning task-specific components and enhancing the plasticity of the
100   backbone, but may suffer from low average precision of the task ID inference for certain tasks. In
101   contrast, the latter approach resorts to more sophisticated prefix prompt tuning to induce plasticity
102   in the feature backbone and can create a discrepancy between training, where head segments from
103   previous tasks are masked out in the softmax function of loss computation, and testing, where the
104
      entire head is used to infer class labels. This discrepancy can lead to instability when dealing with
      streaming tasks that involve a varying number of classes among tasks, such as those encountered in
105
      the VDD benchmark.
106
107   Our aim in this paper is to study ExfCCL using ViTs. As illustrated in Fig. 1, we formulate it
      as a problem of learning continual memory in ViT, which has two components: (i) The internal

      memory enabling a dynamic learning-to-grow feature backbone that balances stability and plasticity,
109   mitigating catastrophic forgetting through task synergies, in which a new task learns automatically
110   to reuse/adapt modules from previous similar tasks, or to introduce new modules when needed,
111   or to skip some modules when it appears to be an easier task (see the bottom of 2). The internal
112   parameter memory learning presents alternative perspectives to the input and prefix prompting based
113   methods (Wang et al., 2022d;c; Smith et al., 2023a; Wang et al., 2022a). (ii) The external memory
114   enabling task ID inference for test data, for which we adopt a method proposed in (Wang et al.,
115   2022a) for its simplicity.
116   We follow the common protocol in the prior art that the first task (e.g., ImageNet-1k (Russakovsky
117   et al., 2015)) can train a ViT sufficiently well as the base model. To enable a dynamic, learning-to-grow
118   feature backbone starting from the base model, we identify and provide simple yet effective solutions
119   for two key challenges: (i) Which modules in a ViT should be reused, adapted, newly created, or
120   skipped for a new task in ExfCCL? It is computationally impractical, and counterproductive to
121   the stability-plasticity trade-off, to make all ViT components dynamic (as opposed to a fully frozen
122   model). We designate the output projection layer following multi-head self-attention (MHSA) as the
123
      task-synergy internal (parameter) memory that will be structurally and dynamically updated. (ii) How
      can we represent and continually learn this task-synergy internal memory to enable dynamic
124
      memory structures across streaming tasks? We propose organizing the memory using a mixture
125
      of experts (MoEs) similar in spirit to (Riquelme et al., 2021), as depicted in Fig. 1 and illustrated
126   through the “task-factorized” visualization in Fig. 2. To learn to select the optimal synergy operation
127   from the four choices (reuse, adapt, new, and skip), we introduce an effective hierarchical
128   exploration-exploitation (HEE) sampling-based neural architecture search (NAS) method. This
129   approach is inspired by the task-incremental learn-to-grow method (Li et al., 2019). Our proposed
130   method is termed CHEEM (Continual Hierarchical-Exploration-Exploitation Memory).
131
      Our Contributions. This paper makes three main contributions to the field of exemplar-free
132   class-incremental continual learning (ExfCCL) using ViT. (i) It presents a hierarchical task-synergy
133   exploration-exploitation sampling based NAS method for learning task-aware dynamic models
134   continually with respect to four operations: Skip, Reuse, Adapt, and New to mitigate catastrophic
135   forgetting. (ii) It identifies a “sweet spot” in ViT as the task-synergy internal (parameter) memory,
136   i.e., the output projection layers after MHSA in ViT. It also presents a new usage for the class-token
137   CLS in ViT as the internal memory updating guidance, in addition to leveraging it in maintaining the
138   external (task-centroid) memory for task ID inference on the fly. (iii) It is the first work, to the best of
139   our knowledge, to evaluate continual learning with ViTs on the large-scale, diverse and imbalanced
140
      VDD benchmark (Rebuffi et al., 2017a), with better performance than the prior art.

175
      We choose to explicitly infer task IDs for test data (see Appendix A for the analyses). We adopt the
176   method proposed in S-Prompts (Wang et al., 2022a). For a task t, we leverage K-mean clustering of
177   the CLS tokens of its training images computed by the query function q(·) (i.e., the base model f0 ).
178   Denote by Zt = {zt1 , · · · , ztK } the clustered task centroids for task Tt , where for simplicity we use
179   the same K = 5 across tasks. We have the external memory, Ψ = ∪N         t=1 Zt after training. In inference,
180   for a testing sample x, we first compute its CLS token using the same query function q(x), denoted
181   by z. The task ID is then inferred via K-NN retrieval, KN N (z, Ψ), either by retrieving the class ID
182   of the Top-1 NN centroid in the external memory Ψ, or by majority voting of the class ID from the
183   K-NN centroids.
184   We note that the such constructed external memory does not break the exemplar-free protocol since
185   we do not use exemplars in any forms from previous tasks. We also note that we focus on the
186   learning of the internal parameter memory. The decoupled design between the external memory
187   and the internal memory will enable us to integrate and/or develop more advanced task ID inference
188   approaches in future work, while potentially promoting our proposed CHEEM in the field of ExfCCL.

446   5    R ELATED W ORK
447
      Experience Replay Based approaches aim to retain some exemplars, in the form of either raw data or
448
      latent features, from the previous tasks and replay them to the model along with the data from a new
449   task (Aljundi et al., 2019b; Rebuffi et al., 2017b; Aljundi et al., 2019a; Balaji et al., 2020; Bang et al.,
450   2021; Chaudhry et al., 2021; Pham et al., 2021), or generative replay methods (Shin et al., 2017; Cong
451   et al., 2020) which replay exemplars sampled from the learned generators along with the data from
452   the current task. For exemplar-free continual learning, Regularization-based Approaches explicitly
453   prevent model parameters from deviating too far from their stable values learned on previous tasks
454   when learning a new task (Aljundi et al., 2018; 2019c; Douillard et al., 2020; Nguyen et al., 2018;
455   Kirkpatrick et al., 2017; Li & Hoiem, 2018; Zenke et al., 2017; Schwarz et al., 2018).
456   Dynamic Models aim to use different parameters for each task to eliminate the use of stored exemplars.
457   Dynamically Expandable Network (Yoon et al., 2018) adds neurons to a network based on learned
458   sparsity constraints and heuristic loss thresholds. PathNet (Fernando et al., 2017) and RPSNet
459   (Rajasegaran et al., 2019) learn task-specific submodules or paths. Progressive Neural Networks (Rusu
460   et al., 2016) learn a new network per task and adds lateral connections to the previous tasks’ networks.
461   The L2G (Li et al., 2019) uses Differentiable Architecture Search (DARTS) (Liu et al., 2019a) to
462   determine if a layer can be reused, adapted, or renewed for a task, which is tested for ConvNets
463
      and the learning-to-grow operations are applied uniformly at each layer in a ConvNet. Our method
      is motivated by the L2G method, but with significant differences. NAS has also been explored by
464
      (Gao et al., 2023a) for continual learning on small scale tasks. Approaches that learn task-specific
465
      components on top of a backbone include (Ge et al., 2023; Verma et al., 2021; Wortsman et al., 2020;
466   Abati et al., 2020).
467
468   Recently, there has been increasing interest in continual learning using Vision Transformers (Wang
469
      et al., 2022d;c; Xue et al., 2022; Ermis et al., 2022; Douillard et al., 2022; Pelosin et al., 2022; Yu
      et al., 2021; Li et al., 2022; Iscen et al., 2022; Wang et al., 2022a;b; Mohamed et al., 2023; Gao et al.,
470
      2023b; Zhou et al., 2023). Prompt Based approaches learn external parameters appended to the data
471
      tokens that encode task-specific information useful for classification (Wang et al., 2022d;a; Douillard
472   et al., 2022; Smith et al., 2023a; Jung et al., 2023; Tang et al., 2023; Zhou et al., 2024). Our proposed
473   method is complementary to prompting-based based approaches.
474
475   6    C ONCLUSION
476
477
      This paper presents a method of transforming Vision Transformers (ViTs) for exemplar-free class-
      incremental continual learning (ExfCCL), dubbed CHEEM (Continual Hierarchical-Exploration-
478
      Exploitation Memory). Our CHEEM consists of the external (task-centroid) memory and the internal
479
      (parameter) memory. The former is for task ID inference for test data based on clustered task
480   centroids in training. The latter is realized by a proposed Hierarchical-Exploration-Exploitation
481   (HEE) sampling based neural architecture search algorithm. The external and internal memory are
482   maintained in a decoupled way. Our CHEEM is tested on two challenging benchmarks, the VDD
483   benchmark and the continual ImageNet-R benchmark. It obtains state-of-the-art performance on
484   the VDD outperforming the prior art by a large margin, with sensible CHEEM strutures continually
485   learned. It obtains on-par performance on the imbalanced ImageNet-R benchmark, while performs
      worse on the balanced one, for which we analyze the reason thoroughly due to the external memory.

       A    TASK -S PECIFIC H EAD S ELECTION VIA TASK ID I NFERENCE VS S HARING A

            G ROWING , C OMMON TASK H EAD
975
976
       As aforementioned in the introduction of main text, explicitly inferring task IDs from the external
977    memory for test data is to select the task head classifiers as done in (Wang et al., 2022a), while
978    also enabling selecting task-specific feature backbones (exploited in our CHEEM). An alternative
979    approach is to share a growing, common task head as done in (Wang et al., 2022d;c; Smith et al.,
980    2023a). The former can suffer from low average precision, while for the latter we observe serious
981    instability caused by imbalanced classes in stream tasks due to the discrepancy in training and testing
982    (see Table 2 and Table 3).
983    To show the instability, let C = (z1 , z2 , · · · , zK ) be the logits output of the common head. During
984    training, the task ID for each zk ∈ C is known, and the logits of previous tasks are masked out
985    (using −∞) before computing the softmax (i.e., task-specific heads are used in training). During
986    testing, the label for a testing sample is predicated by arg maxk∈[1,K] zk , across the entire head. This
987    discrepancy between training and testing imposes a very strong assumption of the stability of logits
988    in training and testing, which entails that the task-specific head segment for a testing example will
989    underlyingly resemble the 1-hot vector, such that it can hold across the entire head. Although (Wang
990    et al., 2022d;c; Smith et al., 2023a) show strong performance using the same number of classes per
991
       task (20) in ImageNet-R (Hendrycks et al., 2021), we observe that slightly varying the number of
       classes per task, while keeping everything else the same, can lead to around 10% drop of performance
992
       in ImageNet-R. For more challenging task-class distributions as in the VDD (Fig. 2), (Wang et al.,
993
       2022d;c; Smith et al., 2023a) have a catastrophic drop of performance.
994
995    To analyze this further, we probe the model after the training on the final task is complete. For each
996    task, we first make predictions using the task specific part of the head (i.e., the task-incremental
997
       setting) and calculate the entropy of predicted probabilities of all the samples with the task-specific
       head. Then, we make predictions with the full head (i.e., the class-incremental setting), including
998
       all the classes in the previous tasks, and track samples for which the logit with the maximum values
999
       changes to one of the classes from the previous tasks, which we refer to as switched samples.
1000
1001   As shown in Fig. 5, we observe that the entropy for the switched samples is consistently higher than
1002   those of the un-switched samples. Furthermore, the variance of the the entropy consistently goes
1003
       down as the tasks progress, which indicates that for later tasks, only the predictions that are highly
       confident remain unswitched. However, the entropy for the switched samples shows a large variance.
1004
       This indicates that sharp predictions are not a sufficient condition to avoid switching.

       D.5      I MPLEMENTATION D ETAILS OF THE Adapt O PERATION
1312
1313   How to Adapt in a sustainable way? The proposed Adapt operation will effectively increase the
1314   depth of the network in a plain way. In the worst case, if too many tasks use Adapt on top of each
1315   other, we will end up stacking too many MLP layers together. This may lead to unstable training due
1316   to gradient vanishing. Shortcut connections (He et al., 2016) have been shown to alleviate the gradient
1317   vanishing and exploding problems, making it possible to train deeper networks. We introduce the
1318   shortcut connection in adding a MLP Adapt operation. We test two different implementations: with
1319
       shortchut in all the three components (supernet training, target network selection and target network
       finetuing) versus with shortcut only in target network finetuning (i.e., without shortcut in the NAS
1320
       including both supernet training and target network selection).

1751
1752   It consists of 10 tasks: ImageNet-1k (Russakovsky et al., 2015), CIFAR100 (Krizhevsky et al., 2009),
1753   SVHN (Netzer et al., 2011), UCF101 Dynamic Images (UCF) (Soomro et al., 2012; Bilen et al.,
1754   2016), Omniglot (Lake et al., 2015), German Traffic Signs (GTSR) (Stallkamp et al., 2012), Daimler
1755   Pedestrian Classification (DPed) (Munder & Gavrila, 2006), VGG Flowers (Nilsback & Zisserman,
1756   2008), FGVC-Aircraft (Maji et al., 2013), and Describable Textures (DTD) (Cimpoi et al., 2014). All
1757
       the images in the VDD benchmark have been scaled such that the shorter side is 72 pixels. Table 10
       shows the number of samples in each task. Fig. 15 shows examples of images from each task of the
1758
       VDD benchmark. In our experiments, we use 10% of the official training data from
1759
       each of the tasks for validation (e.g., used in the target network selection in Section 3.2.3 in main
1760   text), and report the accuracy on the official validation set due to the unavailability of
1761   the ground-truth labels for the official test data. In Table 10, the train, validation
1762   and test splits are thus referred to 90% of the official training data, 10% of the official training data,
1763   and the entire official validation data respectively. When finetuning the learned architecture (i.e., the
1764   searched target network) for each task, we use the entire official training data to
1765   train and report results on the official validation set.

1768   E.2     I MAGE N ET-R B ENCHMARK
1769
1770   The ImageNet-R(etention) (Hendrycks et al., 2021) dataset contains art, cartoons, deviantart, graffiti,
1771   embroidery, graphics, origami, paintings, patterns, plastic objects, plush objects, sculptures, sketches,
1772   tattoos, toys, and video game renditions of ImageNet classes. It has renditions of 200 ImageNet
1773
       classes resulting in 30,000 images. The Split ImageNet-R benchmark proposed by (Wang et al.,
       2022c) uses the ImageNet-R dataset to build a continual learning benchmark specifically for studying
1774
       methods that use model pretrained on ImageNet (Russakovsky et al., 2015). ImageNet-R poses
1775
       challenges for such methods because of the diversity within the same class. The Split-ImageNet
1776   benchmark proves to be challenging for experience-replay based approaches because of this intra-
1777   class variance, as well as replay-free methods that use a frozen backbone from ImageNet as the
1778   accuracy of the standard models on ImageNet-R is low. We use the same training and validation
1779   splits as those used by (Smith et al., 2023a). For the balanced evaluation, we divide the data set into
1780   10 tasks with 20 classes each, and report results across 3 runs with random class orders. For the
1781   imbalanced evaluation, we construct 6 tasks with 5, 10, 15, 20, 50 and 100 classes. We report results
       across 3 runs with tasks, with varying task orders.

       F      T HE BASE V ISION T RANSFORMER : V I T-B/8
1783
1784
       We use the base Vision Transformer (ViT) model, with a patch size of 8 × 8 (ViT-B/8) model
1785
       from (Dosovitskiy et al., 2021). The base ViT model contains 12 Transformer blocks. A Transformer
1786   block is defined by stacking a Multi-Head Self-Attention (MHSA) block and a Multi-Layer Perceptron
1787   (MLP) block with resudual connections for each block. ViT-B/8 uses 12 attention heads in each of
1788   the MHSA blocks, and a feature dimension of 768. The MLP block expands the dimension size to
1789   3072 in the first layer and projects it back to 768 in the second layer. For all the experiments, we use
1790   an image size of 72 × 72 following the VDD setting. We base the implementation of the ViT on the
1791   timm package (Wightman, 2019).
1792
1793   Training the Base Model To train the ViT-B/8 model, we use the ImageNet data provided by
1794   the VDD benchmark (the train split in Table 10). To save the training time, we initialize the
1795   weights from the ViT-B/8 trained on the full resolution ImageNet dataset (224×224) and avail-
1796   able in the timm package, and finetune it for 30 epochs on the downsized version of ImageNet
1797   (72×72) in the VDD benchmark. We use a batch size of 2048 split across 4 Nvidia Quadro
1798
       RTX 8000 GPUs. We follow the standard training/finetuning recipes for ViT models. The file
       cheem/artifacts/imagenet_pretraining/args.yaml in our code folder provides all
1799
       the training hyperparameters used for training the the ViT-B/8 model on ImageNet. During testing,
1800
       we take a single center crop of 72×72 from an image scaled with the shortest side to scaled to 72
1801   pixels.
