009                                                A BSTRACT
010
011              In the past few years, Language Models (LMs) have shown par-human capabil-
012              ities in several domains. Despite their practical applications and exceeding user
013              consumption, they are susceptible to jailbreaks when malicious inputs exploit the
014              LM’s weaknesses, causing it to deviate from its intended behavior. Current defen-
015              sive strategies either classify the input prompt as adversarial or prevent LMs from
016              generating harmful outputs. The primary challenge is that the current defense
017              techniques are built against known and established jailbreaking patterns while
018              work poorly against novel attacks. In this research, we propose an end-to-end
019
                 framework for generating novel attack patterns and demonstrate how the proposed
                 defense approach can generalize over known and unknown attack patterns. Attack
020
                 patterns are generated using a novel self-learning large language model (LLM)-
021
                 based multi-agent system with closed loop feedback called ALMAS, which stands
022              for Attack using LLM-based Multi-Agent Systems. We demonstrate that system-
023              prompt attention from Small Language Models (SLMs) can be used to character-
024              ize adversarial prompts providing a novel explainable and cheaper defense ap-
025              proach called AttentionDefense. The proposed AttentionDefense is evaluated
026              against existing jailbreak benchmark datasets as well as the novel jailbreaks gener-
027              ated using ALMAS. Ablation studies demonstrate that SLM-based AttentionDe-
028              fense has equivalent or better jailbreak detection performance as compared to text
029              embedding based classifiers and GPT-4 zero-shot detectors. Our research suggests
030
                 that the attention mechanism is an integral component in understanding and ex-
                 plaining how LMs respond to malicious inputs that is not captured in the semantic
031
                 meaning of text embeddings. Additionally, for practical purposes AttentionDe-
032
                 fense is an ideal solution as it has the computation requirements of a small LM
033              but the performance of a LLM detector.

      Recent statistics show that ChatGPT alone has ∼1.5 million daily interactions1 and there are roughly
039
      750 million apps that use a Language Model (LM)2 . LM are powerful tools for natural language gen-
040

041   harmful or misleading content (Greshake et al., 2023; Perez and Ribeiro, 2022; Shen et al., 2024;
042   Zou et al., 2023). These attacks are called jailbreaks, which are specially crafted inputs that exploit
043   the model’s weaknesses and cause it to deviate from the intended behavior or instructions. Jailbreaks
044   are input user prompts that consists of two parts: (1) mechanism: how the attack is induced and (2)
045   payload: the generated content or following action that is produced by the attack. Figure 1 shows
046   the example of benign prompt and malicious prompt containing a harmful payload and jailbreak
047   mechanism.
048   Successful jailbreak mechanisms depend on the LM application, such as its audience, connected
049   data sources and accessibility. Mechanisms can be complex, such as using a single pixel that uses
050   markdown to send user inputs to a website (Greshake et al., 2023). The most popular known mecha-
051   nisms are text strings that try to override safety mechanisms. Examples are a simple prompt injection

091   With LM applications that use a system prompt, jailbreaks are successful when the user input causes
092   the LM to either disregard or override system prompt instructions with new instructions. Multiple
093   alternative safety mechanisms have been proposed (Phute et al., 2024; Xie et al., 2023; Zeng et al.,
094   2024; Bai et al., 2022; Bianchi et al., 2024; Wallace et al., 2024), however, many of them are still

102   volume of input prompt requests.
103
      3. Generalizability: The existing defense solutions are extensively trained and evaluated on pub-
104
      lic benchmark datasets but do not perform well on novel, unknown jailbreak attack patterns. For

133   LMs are autoregressive, where tokens are chosen partly on how the previous tokens are attended
134   to (Vaswani et al., 2017), which is quantified by the attention layer weights. As illustrated in Fig-
135   ure 1, the LM attends to the system prompt differently depending on the input when generating an

      responds to the input; a signal that is not found in semantic meaning with prompt text embeddings
137

138

139   strong enough signal that a small LM (SLM) can be used, reducing required compute and making

144
      based Multi-Agent Systems), to generate novel jailbreak attack patterns. ALMAS is self-learning
145   framework and with the help of a critic agent, it iteratively improves the jailbreak to make it complex
146   and compulsive. We experimentally demonstrate that AttentionDefense is generalizable to the novel
147   attack patterns from ALMAS.
148
      The key research contributions of this paper are as follows:
149
150   1. An end-to-end framework including (1) generating novel jailbreak attacks using a multi-agent
151   framework, ALMAS and (2) an explainable, generalizable defense solution against unknown jail-
152   breaks using AttentionDefense.
153   2. AttentionDefense (F1-score of 0.87) outperforms other prompt embedding based classifiers (F1-
154   score of 0.67) in detecting jailbreaks. AttentionDefense can provide explanation and insights on the
155   jailbreak attack as compared to the black-box classifiers.
156
      3. AttentionDefense proves to be a much cheaper alternative as compared to pre-trained and safety
157
      fine-tuned GPT4 as a jailbreak detector (F1-score of 0.86). This demonstrates that system-prompt
158   attention is a critical component in LM generations.
159
160   4. We provide evidence that examining system prompt attention is more robust than common de-
161
      fense methods, where AttentionDefense has roughly equal F1 scores for both known and novel
      jailbreaks, whereas competing methods have lower novel jailbreak detection.

178   2.1     ALMAS: ATTACK USING LLM BASED M ULTI -AGENT S YSTEMS
179
180   Jailbreak detection techniques are built on top of public benchmark datasets, which contain well
181   known and understood attack patterns. Such approaches show reduced performance while used in
182
      real-world use-cases. To overcome this challenge, we propose an adaptive self-learning framework,
      ALMAS, for generating novel jailbreak prompts. LLM based agents are used in ALMAS for jail-
183
      break prompt generation.
184
185   As shown in Figure 3, a multi-agent system comprising three different agents is designed:
186
              1. StrategyAgent: The role is to generate a novel strategy of attack. The strategy agent has
187
                 access to and builds upon public benchmark datasets with known strategies and templates.
188
189           2. AttackAgent: The attack agent generates the jailbreak prompt using the input strategy. The
190
                 jailbreak prompt is tested against an LM model, and if the attack was successful, we add
                 this prompt as a novel attack from ALMAS. If the jailbreak is not successful, the critic agent
191
                 is invoked to investigate the entire conversation, and provide suggestions for improvement.
192
193           3. CriticAgent: The role is to provide critical feedback to the strategy or attack agent.
194
      The multi-agent architecture is inspired by the Self-Reflection framework (Renze and Guven, 2024)
195   which has shown to substantially increased the performance of LMs for any given task. Detailed
196   definition of the agents are provided in Appendix E
197
198   2.2     ATTENTION D EFENSE
199
200   AttentionDefense consists of two components: an SLM and a classifier as seen in Figure 2. Using
201   an LM with low parameter size will reduce computation enough for most applications to be put into
202   production. For example, most SLMs can be run on a single GPU. However, SLMs tend to have low
203   quality output. For example, in the HuggingFace leadership board, top models have 70B parameters
204   or more 4 . Applying a classifier to the system prompt attention may be able to create usable output
205
      other than the low quality SLM generation.
206   For AttentionDefense, we compare performance of attention weights extracted from Phi-2 and Phi-
207   3.5 SLMs. The Phi-3.5 models have shown to have similar performance to leading models such as
208   Llama-3.1 and Gemma-2-9B but with fewer parameters 5 . However, Phi-3.5-mini is only available
209   with safety fine-tuning (called Phi-3.5-mini-instruct), while Phi-2 is available pre-trained (Haider
210   et al., 2024; Hughes, 2023). In addition, Phi-2 has fewer parameters than Phi-3.5-mini (2.7B vs
211
      3.8B) which makes for less inference time. The Phi models also have small context windows, where
      inputs with 8.5K token size are only considered. While this is a limitation for using long inputs,
212
      continued model development will improve the size of the context window.

282   4.1       D ESIGNING THE S YSTEM P ROMPT
283
284   The primary aim of these results discussion is to inform other researchers and developers on how
285   to design system prompts for their respective LM applications. For a system prompt to be well-
286   designed, the commands in the system prompt should be able to identify adversarial behavior in the
287   user input. Additionally, we use AttentionDefense to verify the determinant of the input jailbreak
288   that is integral to define an attack: jailbreak payload, mechanism, or both.
289   The effect of statements in the system prompt that warn the SLM to avoid the jailbreak mechanism
290   or payload are observed. Three different payload and mechanism instructions are used in the system
291   prompt for AttentionDefense, which are listed in Tables 4 and 5 in Appendix C. These instructions
292   vary in wording and length.
293   In addition, the four classification models are run for each possible system prompt. Thresholds for
294   the models are chosen based on optimal F1 score or to have very high precision (greater or equal
295   to 0.99) to reflect the demand for low false positive rates that are necessary to launch a model into
296   product without affecting users. If high precision is not possible the performance is not considered
297   in the final analysis.

299   4.2       C OMPARING ATTENTION D EFENSE TO C OMMON D EFENSES
300
301   When running an LM inference, there are two components: the inner workings of the model the
302   input is processed through, and the final generated output. In this work, multi-modal attacks are not
303   considered and the only input and output observed is text.
304
305   4.2.1       E MBEDDINGS AS T RAINING DATA
306
      When an LM call is initiated, the input prompt is converted into text embeddings. These embeddings
307   are then processed through the layers of the LM. Embeddings capture semantic meaning, or how the
308   words in the prompt can be interpreted. In Figure 1, attention is the focus the LM gives to prior
309   tokens from the current token. It can be argued that attention captures how the LM responds to the
310   input. Both embeddings and attention represent different components of an LM generation. The
311   jailbreak mechanism may not be captured by semantic meaning since it does not contain how the
312   model responds to the jailbreak. In addition, attention may be such a critical component in how
313   jailbreaks are processed that using SLM attention may be comparable to using LLM detectors or
314   fine-tuned classifiers.
315   AttentionDefense is compared to classifiers that are trained on the embeddings using the previously
316   described classfiers in Section 4.1. The TrustLLM jailbreaks and GPT-Generated WikiText prompts
317   text embeddings are used as training data. Three different embeddings are considered: TF-IDF,
318   Sentence Transformer all-MiniLM-L6-v2, and OpenAI ada-2 embeddings. These embeddings vary
319   in their simplicity and performance.
320   TF-IDF stands for Term Frequency-Inverse Document Frequency, which uses both the frequency
321   terms that appear across all documents and how many documents contain the terms (Spark Jones,
322   1972). Sentence Transformer embeddings enhance BERT transformers by focusing on sentence-

      4.2.2       SLM ATTENTION VS . LLM D ETECTOR
333
334   LMs become detectors through zero-shot learning, which is when an instruction is added to the
335   system prompt to return the predicted class (Phute et al., 2024). This is similar to classification
336   models, where a score between 0 and 1 is given based on the input. For LM detectors, no training
337   data is required. Only a system prompt defining the task and the class to return depending on
338   the input is needed. While this approach is attractive because no model training is needed, the
339   computation is much larger than other classification methods. Additionally, explainability of LMs
340
      for predictive tasks is a well known issue in machine learning, making it difficult to understand and
      determine the cause of predictive errors.
341
342   AttentionDefense is compared to the performance of an LLM detector. The LLM detector is tested
343   with the same system prompts as AttentionDefense but with additional instructions on detection
344   (Appendix D). For this experiment, we use pre-trained and safety fine-tuned GPT-4 as the LLM
345   detector for comparison since it is currently considered the SOTA (OpenAI et al., 2024).
346   The GPT-4 detectors are expected to perform better than AttentionDefense because the LLMs are
347   higher quality models than the Phi models. For example, GPT-4 has 1.8T parameters compared to
348   Phi-2’s and Phi-3.5-mini-instruct 2B and 3.8B parameters. If AttentionDefense has similar or better
349   performance to the LLMs, this implies that attention is an integral signal within LMs.

461   6       R ELATED W ORK
462
463   There are many methods to prevent jailbreaks that exist today. A common strategy is using the
464   LM itself, either by including a system prompt with the user prompt to mitigate jailbreaks or using
465   a separate LM call to classify the output (Phute et al., 2024; Xie et al., 2023; Zeng et al., 2024).
466   However, these each have their downsides. Using system prompts as a mitigation is brittle (Shen
467   et al., 2024) and approaches that require multiple LM calls are expensive and not practical in most
468
      production settings.
469   There has also been success in fine-tuning the model to give more emphasis to system instructions
470   and alignment (Bai et al., 2022; Bianchi et al., 2024; Wallace et al., 2024). However, it has been
471   shown that fine-tuning can be “fine-tuned out” (Qi et al., 2023; Zhan et al., 2024) and reduce task
472   performance and output quality (Mohammadi, 2024; Wei et al., 2023). Fine-tuning is also compu-
473   tationally expensive and therefore is not always a feasible solution.
474   Embeddings have also been proposed to compare incoming prompts as malicious using similarity
475   metrics 8 . While embeddings are simpler to generate since they do not require an LM inference
476   call, they capture semantic meaning rather than mechanisms within the LM. Here, the power in
477   using system prompt attention weights over input embeddings is established, demonstrating the
478   generalizability of system prompt attention to detecting adversarial inputs.
479   Mitigations have begun to incorporate latent representations into solutions. A few methods include
480   extra steps to altering the generated output (Xu et al., 2024; Sabir et al., 2023), but they are lim-
481   ited by known prior information such as the scope of the jailbreaks or safety tokens. Similar to
482   AttentionDefense, extracting layer activations has also been used to detect adversarial content with
483   classification models (Abdelnabi et al., 2024; Kawasaki et al., 2024; MacDiarmid, 2024). Most
484   of these approaches use an LLM, while AttentionDefense can achieve high performance using an
485
          8
              https://whylabs.ai/blog/posts/navigating-threats-detecting-llm-prompt-injections-and-jailbreaks

      Under review as a conference paper at ICLR 2025


486
      SLM. Additionally, using system prompt attention can be more interpretable than layer activations
487   in identifying attention weight shifts with alternate instructions.
488
489   Often, LLMs are used because of their higher performance and quality, as seen in the HuggingFace
490
      leadership board where top models have 70B parameters or more 9 . SLMs have fewer parameters, as
      low as 2-3B parameters (Abdin et al., 2024; Hughes, 2023). The difference in computation between
491
      an SLM and an LLM can be significant enough to enable more widespread use. However, the lower
492
      parameter size also comes at a cost with lower performance. With AttentionDefense, this trade-off
493   is handled by using SLM attention to classify prompts instead of the SLM generation.
494
495   To our knowledge, AttentionDefense is the first mitigation that uses system prompt attention to
496
      detect adversarial attacks, and the first open-box jailbreak detection classifier that uses an SLM.
      With AttentionDefense, it is also demonstrated how system prompt attention can be used for system
497
      prompt design, is more generalizable than embeddings, and has similar performance to an LLM
498
      detector with the computation of an SLM.

501   7       C ONCLUSIONS
502
503   In this work, we have demonstrated how AttentionDefense improves explainability, scalability and
504   generalizability of jailbreak detection approaches. Modeling system prompt attention can be used to
505   investigate how LMs respond to instructions, which we illustrated by observing the responses to a
506   variety of jailbreak mechanism and payload instructions in the system prompt. We have reduced the
507   scale of computation of a detection by showing how SLM system prompt attention classifiers can
508
      yield similar results to those of LLM detectors. Lastly, we have demonstrated how system prompt
      attention is robust to both novel and known jailbreaks compared to competing defenses by observing
509
      performance between known In-the-Wild known jailbreaks and ALMAS novel jailbreaks.
510
511   Limitations for this work stem from AttentionDefense being anchored by a system prompt. If the
512   system prompt is not prompt engineered well, the attention weights will not show any meaningful
513   difference between the benign and malicious prompts. Any change to the system prompt requires
514
      the training data to be regenerated. In addition, any constraints on the SLM, such as small context
      windows, will also be extended to AttentionDefense.
515
516   Future work involves investigating if the system prompt attention can be used for other detector use
517   cases outside of jailbreaks. In addition, exploring if a similar approach to AttentionDefense can be
518   built using unsupervised learning to measure out-of-distribution detection, which would remove the
519   requirement for labeled data.
