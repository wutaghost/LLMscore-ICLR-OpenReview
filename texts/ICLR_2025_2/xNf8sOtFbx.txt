011                                                A BSTRACT
012
013            Precisely annotating instances with multiple labels is costly and has emerged as
014            a significant bottleneck in the real-world multi-label learning tasks. To deal with
015            this problem, the most straightforward strategy is partially-annotating, which aims
016            to reduce the cost by annotating only a subset of labels. Existing works mainly
017            include label-level partially-annotating (LPA), where each instance is assigned a
018            subset of positive labels, and instance-level partially-annotating (IPA), where all
019            positive labels are assigned to an instance, but only a subset of instances are an-
020
               notated. However, these methods tend to focus on improving model performance
               under each type of partial annotation, often neglecting a fundamental question:
021
               which method is more cost-effective? In this paper, we empirically evaluate which
022
               partially-annotating method achieves better model performance at the same anno-
023            tation cost. To make a fair comparison, we manually annotated images in the MS-
024            COCO dataset using two partially-annotating methods and recorded their average
025            annotation time per image. This allows us to train models on two types of partial
026            annotations with the same annotation cost and to compare their performance. Em-
027            pirical results show that even when the number of examples annotated with IPA is
028            only one-fifth of that of LPA, models trained on IPA annotations significantly out-
029            perform those trained on LPA annotations, yielding that IPA is considerably more
030            cost-effective than LPA. To explain its superiority, our causal reasoning frame-
031
               work shows that compared to LPA, IPA preserves complete co-occurrence rela-
               tionships, enabling the model to capture correlative patterns, which is useful for
032
               improving model performance.

036   1    INTRODUCTION
037
038
039
      In single-label supervised learning, each instance is assigned with only one class label, whereas
      the real world may be multi-labeled, where multiple objects are usually present in a realistic sce-
040
      nario. For example, an image of a street scene often contains objects such as pedestrians, cars, and
041
      billboards. Multi-label learning provides a fundamental framework for handling data with multiple
042   semantics by training a MLL model (Liu et al., 2021b). It has been successfully applied to numerous
043   real-world applications such as image annotation (Kuznetsova et al., 2020), medical image analysis
044   (Ihler et al., 2024) and protein subcellular localization (Liu et al., 2022).
045
      Conventional MLL typically assumes that each training instance has been precisely annotated with
046
      all of its relevant labels. Unfortunately, it is difficult and costly to collect a large number of precise
047
      annotations. To save labeling expense, a feasible solution is to adopt the partially-annotating strat-
048   egy, i.e., annotating a subset of labels. Specifically, there are two main types of partially-annotating
049   methods, label-level partially-annotating (LPA) and instance-level partially-annotating (IPA). The
050   former annotates all training instances but only a subset of the labels for each instance; while the
051   latter annotates all labels for an instance but only a subset of the instances. Many recent studies
052   have developed specialized learning algorithms for these two types of partial annotations, known
053   as multi-label learning with partial labels (MLLPL) (Durand et al., 2019; Cole et al., 2021) and
      semi-supervised multi-label learning (SSMLL) (Wang et al., 2020; Xie et al., 2024).

054
      Although these methods have shown improvements in practical performance for their respective
055   learning tasks, a fundamental question remains: which partially-annotating method is more cost-
056   effective, that is, which achieves more model performance under the same annotation cost? The
057   answer to this question is not straightforward, as each method has its own advantages. LPA anno-
058   tates only a subset of labels for each instance, which often results in more examples being annotated.
059   This allows the learner to gain supervised information from a larger number of examples, even if that
060   information is incomplete. In contrast, IPA annotates all positive labels for a subset of examples, en-
061   abling it to leverage valuable prior information, such as the class distribution and label correlations,
062   for model training. Therefore, it is crucial to investigate which partially-annotating method offers
063
      better cost-effectiveness.
064   In this paper, we conduct a systematic analysis to study which partially-annotating method is better
065   for annotating a dataset. We consider an extreme LPA method, where each instance is annotated with
066   only one label, allowing us to annotate all instances at the lowest cost. In the practical annotation
067   process, we group all classes into several super-classes and then perform hierarchical annotation,
068   i.e., annotating the super-classes first and then the individual classes, to avoid checking every class.
069
      We show that the annotation cost and the value of the annotated labels for the LPA method depend on
      the randomness in the annotation process. Generally, greater randomness leads to higher annotation
070
      costs and increased label values, and vice versa. According to the different levels of randomness in
071
      the annotation process, we develop three variants of the LPA method. To make a fair comparison
072   between the LPA and IPA methods, we develop a unified learning framework to train models by
073   incorporating the pseudo-labeling technique, with the goal of leveraging the information of unan-
074   notated data. Extensive experimental results verify that models trained on IPA data significantly
075   outperform those trained on LPA data, even when the number of examples annotated with IPA is
076   only one-fifth of that of LPA. To explore the underlying mechanisms, we construct a causal reason-
077   ing model that discloses how, compared to LPA, IPA effectively captures co-occurrence relationships
078   during the annotation process, playing a crucial role in enhancing model performance.

081   2   R ELATED W ORKS
082
083   With the development of deep learning, deep neural networks (DNNs) have become the popular
084   learning models for tackling the MLL problems. Existing deep MLL methods can be roughly di-
085   vided into three groups. In traditional MLL studies, label correlations have been treated as fun-
086   damental information for solving the MLL tasks. Building on these findings, the first group of
087
      methods focuses on leveraging label correlations through the development of specialized architec-
      tures or training strategies. For example, Chen et al. (2019) used a graph convolutional network
088
      (GCN) to model the correlation among different labels. The second group aims to improve the com-
089
      monly used binary cross entropy (BCE) loss function. For example, the ASL loss (Ridnik et al.,
090   2021) incorporates dynamic down-weighting and hard-thresholding techniques to address the issue
091   of positive-negative imbalance in MLL. The last kind of methods aim to improve classification per-
092   formance by identifying objects of interest. For example, the asymmetric loss (ASL) incorporates
093   dynamic down-weighting and hard-thresholding techniques to address the issue of positive-negative
094   imbalance in MLL. The final group seeks to improve classification performance by identifying ob-
095   jects of interest. A notable example is Query2Label (Liu et al., 2021a), which employs the attention
096   technique to achieve this objective.
097   Although these methods have demonstrated improvements in the practical performance of MLL,
098   they require that every instance has been annotated with its all relevant labels. To handle this prob-
099   lem, a feasible solution is to adopt cost-effective annotation methods to reduce the annotation cost.
100   Among them, partially-annotating reduces overall costs by annotating only a subset of labels. This
101   approach can be categorized into label-level partial annotation and instance-level partial annotation.
102   Existing works focus on improving model performance under these two types of partial annotation,
103   known as multi-label learning with partial labels (MLLPL) and semi-supervised multi-label learning
104
      (SSMLL).
105   To solve the MLLPL problems, a pioneering method was to design a partial BCE loss (Durand et al.,
106   2019), which down-weights the loss of annotated labels. The subsequent works designed special-
107   ized modules to leverage the information from unannotated labels. Pu et al. (2022) developed a
      unified semantic-aware representation blending framework to recover unknown labels by exploit-


                                                         2
      Under review as a conference paper at ICLR 2025


108
      ing instance-level and prototype-level feature representation simultaneously. Chen et al. (2024)
109   proposed to produce pseudo-labels for unknown labels by exploring within-image and cross-image
110   semantic correlations. Sun et al. (2022); Hu et al. (2023) designed positive and negative prompts
111   to adapt the knowledge embedded in the large pretraining model CLIP to MLLPL. Additionally,
112   Shen et al. (2024) proposed an annotation strategy to allow different annotators to focus on different
113   subset of classes, aiming to reduce the annotation workload of each annotator.
114
      An extreme approach of label-level partially annotating is to assign only one label to each instance,
115   and the corresponding problem is referred to as single positive multi-label learning (SPML) (Cole
116   et al., 2021). An intuitive method was to treat unannotated labels as negative and to train a model
117   with the traditional BCE loss. Unfortunately, such a method would introduce a large number of false
118   negative labels, resulting in unfavorable model performance. To solve this problem, several attempts
119   have been made to recovering potential positive labels by either utilizing the regularization (Zhou
120   et al., 2022) or exploiting manifold structure information (Xie et al., 2022).
121   Compared to MLLPL, there has been relatively limited research aimed at enhancing the performance
122   of SSMLL. Early work (Wang et al., 2020) utilized a deep neural network (DNN) to extract features,
123   followed by a linear model for classification. While this two-stage approach significantly improved
124   SSMLL performance over traditional methods, its lack of end-to-end training constrained further
125   enhancements. To address this limitation, Xie et al. (2024) introduced an end-to-end DNN approach
126   specifically designed for the SSMLL problems. Their method developed a class-distribution-aware
127   thresholding strategy to perform pseudo-labeling in a class-wise manner, thereby eliminating the
128   need to estimate thresholds for each instance.

132   In this section, we begin by introducing some notations, followed by a discussion on how to anno-
133   tate the data and train the models, ensuring a fair comparison between the two partially-annotating

      feature space and Y = {0, 1}q is the label space with q possible classes. In the traditional MLL

138   our setting, we use two methods to partially-annotate training data. By using the LPA method, we
139   obtain a partially-labeled dataset D = {(xi , ŷi )}ni=1 , where ŷ is a partial label vector. Here, ŷk = 1
140   indicates that the k-th label is an annotated positive label for instance x; while ŷk = 0 indicates the

147   3.1   A NNOTATION E NGINE
148
149   Given that our empirical studies are mainly based on multi-label benchmark dataset MS-COCO (Lin
150
      et al., 2014), which is the most commonly used dataset in MLL literature, we adopt the annotation
      method that has been applied to annotate the MS-COCO dataset. Following their approach, we first
151
      group the original 80 classes into 12 super-classes (see details in Appendix). For a given instance,
152
      an annotator is asked to determine whether the instance contains any semantic super-class. This step
153   can significantly reduce the time required to check all the 80 classes. For example, an annotator can
154   easily determine that no vehicle is presented in the image of an indoor scene, so the annotator can
155   quickly skip the super-class vehicle. When an annotator determines the super-class that an image
156   may belong to, the annotator must check the classes within the super-class one by one and mark the
157   semantic class that is present in the image. Below, we will discuss how to perform the IPA and LPA
158   methods based on the annotation engine.
159

182   a balance between annotation costs and label values, we propose the following three LPA methods,
183   which progressively increase the randomness of the obtained labels. These three LPA methods are
184   listed as follows.

             • LPA-I This method directly annotates the most prominent object class in an image. Al-
               though this method can minimize annotation costs, it typically results in less informative
187
               labels. As shown in Figure 1, all three annotators provide the label truck using the LPA-
188
               I method. This object can be easily recognized from the image, resulting in a relatively
189            minor impact on improving model performance.
190
             • LPA-II For each image, the annotator is provided with super-classes in a random order by
191
               the annotation engine. The task of annotators is to identify the first super-class that the
192
               image contains and to annotate the most prominent object class within that super-class. As
193            shown in Figure 1, the randomness of the super-classes may lead annotators to focus on
194            some difficult-to-recognize objects, such as chair and handbag. These labels are clearly
195            more valuable than truck and have a significant impact on improving model performance.
196
             • LPA-III For each image, the annotator is provided with super-classes in a random order.
197            The annotator is asked to identify the first super-class contained in the image. Then, the
198            annotator is provided with the classes within the super-class in a random order. The task
199            is to check classes within the super-class in a random order and identify the first object
200            class that is present in the image. By further introducing randomness into the classes, the
201            range of annotations provided by the annotators will continue to expand, increasing label
202            diversity and enhancing model performance. For example, in Figure 1, using the LPA-
203            II method, annotators are unlikely to annotate motorcycle because trucks are clearly the
204            easiest objects to annotate within the super-class vehicle. In contrast, using the LPA-III
205
               method, annotators are likely to annotate motorcycle due to the randomness of classes.
206
      Generally, from the perspective of annotation costs, LPA-I has the lowest cost, as it annotates the eas-
207
      iest class each time, while LPA-III has the highest annotation cost because it often requires checking
208   multiple super-classes/classes to obtain the final annotation; from the perspective of the label value,
209   LPA-I provides the least valuable labels because it consistently annotates the most prominent ob-
210   jects, leading to significant information redundancy. In contrast, LPA-III offers the most valuable
211   labels by introducing randomness at both the super-class and class levels, encouraging annotators to
212   focus on difficult-to-recognize objects, which is useful for enhancing model performance.

262   To fairly compare two partially-annotating methods, we use a unified learning framework for model
263   training. It can be summarized as two procedures: i) warmup the model on the annotated labels. ii)
264   use the model to generate pseudo-labels for the unannotated parts and retrain the model on all labels.
265   Note that we repeat the second procedure until the model converges.

269   which treats all unknown labels as negative. For the IPA method, we train the model directly on
      the labeled dataset. During the warmup stage, we utilize the commonly used ASL loss, which is an

298   4       E XPERIMENTS
299
300   Dataset The experiments are mainly performed on benchmark dataset MS-COCO 2014 (MS-
301   COCO for short) 1 (Lin et al., 2014). It consists of 82,081 training examples and 40,504 validation
302   examples, which are categorized into 80 classes. Considering that all examples in the MS-COCO
303   dataset have true labels, we simulated manual annotation using three different LPA methods (LPA-I,
304   LPA-II and LPA-III) to generate three distinct datasets, each with instances that are each assigned
305   a single positive label. For the LPA-III method, which are not affected by the preference of anno-
306   tators, we can directly use true labels for annotation; for LPA-I and LPA-II methods, which require
307
      selecting the most salient object during annotation and are often influenced by the preference of
      annotators, we simulate this process by selecting the largest object according to sizes of bounding
308
      boxes. To fairly compare the IPA method with the three LPA methods, based on the previous ob-
309
      tained ratio of per-instance annotation costs between the different LPA methods and the IPA method,
310   IPA allows us to obtain three datasets, with the proportions of annotated examples being 20%, 42%,
311   and 55%, respectively. For notational simplicity, we denote these three experimental settings as
312   Mode-I, Mode-II and Mode-III.
313
314   Methods To compare the cost-effectiveness of the LPA and IPA methods, we train models on
315   the LPA and IPA datasets with equivalent annotation costs, and then compared their performance.
316   Specifically, for LPA, we employ two methods: i) LPA-AN: training models on LPA data with
317   AN loss by assuming unannotated labels as negative. ii) LPA-PL: training models on LPA data
318   using the PL method. For IPA, we employ two methods: i) IPA-ASL: training models only on
319
      labeled data with the ASL loss. ii) IPA-PL: training models on labeled and unlabeled data using
      the PL method. By comparing LPA-AN and IPA-ASL, we can evaluate which annotation method
320
      is more cost-effective when using only the annotated data. By comparing LPA-PL and IPA-PL, we
321
      can evaluate which annotation method is more cost-effective when leveraging the information of
322   unannotated data additionally.

      6   C ONCLUSION
520
      This paper studies an important question: which method is more cost-effective between label-level
521
      partially-annotating and instance-level partially-annotating? To answer this question, we conducted
522
      a systematic empirical study to compare the annotation cost and label value of these two annotation
523   methods. Regarding the annotation cost, we manually annotate images in MS-COCO using two
524   types of partially-annotating methods and record their averaging annotation time per image. Re-
525   garding the label value, we train models on the two types of partially-annotated data under the same
526   annotation budget and compare their performance. Our empirical studies show that IPA achieves
527   better model performance than LPA under the same annotation cost. From the perspective of causal
528   reasoning, we analyze why IPA is a better annotation method: it ensures complete co-occurrence
529   information during the annotation process, which facilitates the learning of co-occurrence patterns,
530   thereby enhancing model performance. In future, we will compare the cost-effectiveness of the two
531
      partial-annotating methods on larger-scale datasets.
