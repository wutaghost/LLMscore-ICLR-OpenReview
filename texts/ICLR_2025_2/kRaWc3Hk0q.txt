009                                               A BSTRACT
010
011            We present ReHub, a novel graph transformer architecture that achieves linear
012            complexity through an efficient reassignment technique between nodes and virtual
013            nodes. Graph transformers have become increasingly important in graph learning
014            for their ability to utilize long-range node communication explicitly, addressing
015            limitations such as oversmoothing and oversquashing found in message-passing
016
               graph networks. However, their dense attention mechanism scales quadratically
               with the number of nodes, limiting their applicability to large-scale graphs. ReHub
017
               draws inspiration from the airline industry’s hub-and-spoke model, where flights
018
               are assigned to optimize operational efficiency. In our approach, graph nodes
019            (spokes) are dynamically reassigned to a fixed number of virtual nodes (hubs) at
020            each model layer. Recent work, Neural Atoms (Li et al., 2024), has demonstrated
021            impressive and consistent improvements over GNN baselines by utilizing such
022            virtual nodes; their findings suggest that the number of hubs strongly influences
023            performance. However, increasing the number of hubs typically raises complex-
024            ity, requiring a trade-off to maintain linear complexity. Our key insight is that each
025            node only needs to interact with a small subset of hubs to achieve linear complex-
026            ity, even when the total number of hubs is large. To leverage all hubs without
027
               incurring additional computational costs, we propose a simple yet effective adap-
               tive reassignment technique based on hub-hub similarity scores, eliminating the
028
               need for expensive node-hub computations. Our experiments on long-range graph
029
               benchmarks indicate a consistent improvement in results over the base method,
030
               Neural Atoms, while maintaining a linear complexity instead of O(n3/2 ). Re-
031            markably, our sparse model achieves performance on par with its non-sparse coun-
032            terpart. Furthermore, ReHub outperforms competitive baselines and consistently
033            ranks among the top performers across various benchmarks.

036   1   I NTRODUCTION
037
038   Learning on graphs is essential in numerous domains, including social networks for influence pre-
039
      diction, biological networks for understanding protein interactions, molecular graphs for predicting
      chemical properties, knowledge graphs for recommendation systems, and financial networks for
040
      fraud detection and risk assessment. Graph neural networks (GNNs) have emerged as powerful
041
      tools in these areas, operating via message passing between connected nodes. However, a signifi-
042   cant challenge with GNNs is their limited communication range. While stacking message-passing
043   layers can increase the communication distance, it comes at a computational cost and can cause
044   issues such as oversmoothing and oversquashing (Alon & Yahav, 2020; Topping et al., 2021).
045
      Inspired by the success of transformers in natural language processing (Vaswani et al., 2017), graph
046
      transformers offer a solution by enabling global node communication through attention mechanisms
047
      (Dwivedi & Bresson, 2020; Shehzad et al., 2024). This overcomes the communication bottlenecks
048   of GNNs, but it comes at a significant computational cost. The quadratic complexity of dense atten-
049   tion operations limits the scalability of graph transformers, as even modest-sized graphs can exhaust
050   GPU memory. Several methods have been suggested to reduce the complexity of global atten-
051   tion. For example, GraphGPS (Rampášek et al., 2022) combines sparse attention mechanisms like
052   Performer (Choromanski et al., 2020) or Big Bird (Zaheer et al., 2020). Originally designed for pro-
053   cessing sequences rather than graph structure, these linear-memory transformers induce significant
      computational time and do not match the performance of dense attention (Shirzad et al., 2023).

054
      Recently, transformer-based graph networks have utilized the addition of virtual global nodes,
055   through which graph nodes communicate to sparsify the attention. By constraining the attention
056   to be between the graph nodes and these virtual nodes, the attention complexity is reduced to the
057   number of nodes times the number of virtual nodes, allowing the overall complexity to be governed
058   by the number of virtual nodes. Exphormer (Shirzad et al., 2023) maintains linear complexity by
059   using a fixed number of virtual nodes, while Neural Atoms (Li et al., 2024) explores both a fixed
060   number and a ratio relative to the number of nodes. An important finding in Neural Atoms is that
061   adding more virtual nodes increases prediction accuracy, creating a trade-off between computational
062   complexity and accuracy.
063   In this work, we introduce ReHub, a novel graph transformer architecture that achieves linear com-
064   plexity by dynamically reassigning graph nodes to virtual nodes. We are inspired by complex sys-
065   tems where efficient connectivity and adaptability are crucial for optimal performance. A pertinent
066   example is the airline industry, where flights are dynamically assigned to a limited number of major
067   airports (hubs) to optimize operational efficiency. We identify the graph nodes with spokes and the
068   virtual nodes with hubs. The key insight of our approach stems from noting that the transformer’s
069
      complexity is driven by spoke-hub attention. Therefore, it is not necessary to reduce the total num-
      ber of hubs to achieve linear complexity; instead, using a fixed, small number of connected hubs
070
      per spoke is sufficient. To effectively utilize all hubs without increasing computational cost, we
071
      introduce a simple yet efficient adaptive reassignment mechanism. In order to so and yet avoid the
072   costly computation the entire set of spoke-hub interactions, our reassignment mechanism is based
073   on hub-hub similarity scores, which are cheap to compute.
074
075
      In summary, then, our primary contribution is a novel graph transformer architecture that integrates
      global attention with an efficient spoke-hub reassignment strategy, significantly enhancing scala-
076
      bility while maintaining performance. Our experiments on long-range graph benchmarks indicate
077
      a consistent improvement in results over the base method, Neural Atoms, while keeping a linear
078
      complexity instead of O(n3/2 ). Remarkably, our sparse model achieves performance on par with
079   its non-sparse counterpart. Furthermore, ReHub outperforms competitive baselines and consistently
080   ranks among the top performers across various benchmarks.

083   2   R ELATED WORK
084
085   Learning on large graphs Graph learning architectures are a well-established and highly active
086   field of research (Wu et al., 2020). Common GNNs, such as GCN/GCN2 (Kipf & Welling, 2016;
087
      Chen et al., 2020), GAT/GATv2 (Velickovic et al., 2017; Brody et al., 2021), GIN/GINE (Xu et al.,
      2018; Hu et al., 2019), and GatedGCN (Bresson & Laurent, 2017), rely on a message-passing ar-
088
      chitecture that aggregates information. In each layer every graph node updates its representation by
089
      aggregating the neighboring nodes. This architecture inherently limits their ability to accumulate
090   information over large distances due to phenomena such as over-smoothing (Alon & Yahav, 2020),
091   where node representations become indistinguishable, and over-squashing (Topping et al., 2021) ,
092   where the capacity to propagate information is restricted by bottlenecks. Consequently, learning on
093   large graphs remains a persistent challenge (Duan et al., 2022). To address this issue, some methods
094   focus on reducing the memory footprint. One approach involves dividing graphs into mini-batches
095   (Wu et al., 2024), while another uses only segments of the graph for training (Cao et al., 2024).
096   Transformer architectures have recently gained popularity for graph-based tasks (Müller et al.,
097   2023). These methods address the over-smoothing and over-squashing issues by enabling all nodes
098   to interact with each other through attention (Velickovic et al., 2017; Ying et al., 2021). However,
099   this approach is computationally inefficient, with quadratic time and memory consumption in the
100   number of nodes. To mitigate these inefficiencies, more efficient transformer architectures have
101   emerged which utilize different approaches to reduce the number of computations such as approxi-
102   mations in Performer (Choromanski et al., 2020), predefined attention patterns in BigBird (Zaheer
103   et al., 2020) and better parallelism and partitioning in FlashAttention (Dao et al., 2022). GraphGPS
104
      (Rampášek et al., 2022) proposes a general framework for combining message-passing neural net-
      works (MPNNs) with attention at each layer, facilitating the use of such attention mechanisms.
105
106   Recent works have introduced approaches that leverage the structure and inherent informa-
107   tion of the graph, such as tokenization of hops of node’s neighbors in NAGphormer and
      applying structure-preserving attention on encoded sequences of sub-graphs in Gophormer


                                                       2
      Under review as a conference paper at ICLR 2025


108
      (Chen et al., 2022; Zhao et al., 2021). Some of these works ensure that the computational complex-
109   ity remains linear in the number of nodes (Shirzad et al., 2023; Chen et al., 2022; Wu et al., 2024;
110   2022) which allows them to be applied to larger graphs. We also propose a linear complexity archi-
111   tecture that passes nodes’ information through a medium that efficiently orchestrates the passage of
112   information in the graph and between nodes.
113
      Finally, we note that recently, State Space Models (SSMs) (Gu & Dao, 2023) have emerged as
114
      a promising approach for efficiently processing large sequences, with their adaptation to graphs
115   showing notable results (Wang et al., 2024; Behrouz & Hashemi, 2024). However, in this work, we
116   focus on transformer-based architectures, which remain the current go-to approach.
117
118
      Virtual nodes The concept of virtual nodes involves introducing new external nodes that interact
      with the graph to facilitate information exchange between existing nodes (Gilmer et al., 2017). Re-
119
      cent studies (Shirzad et al., 2023; Cai et al., 2023; Hwang et al., 2022) utilize this concept to extend
120
      the graph’s capability to capture long-range information through message-passing. Other research
121   explores the integration of virtual nodes within the context of transformers (Shirzad et al., 2023;
122   Li et al., 2024; Fu et al., 2024). Similarly, we use virtual nodes that communicate with both each
123   other and the graph nodes through attention. However, unlike prior methods, we dynamically up-
124   date the connectivity between the virtual nodes and graph nodes at each layer, enhancing the flow of
125   information.

226   A key component of ReHub’s architecture is the long-range spoke update layer, which leverages
227   both the spoke graph and the connections between spokes and hubs. Specifically, to maintain ef-

296
297   Methods in comparison. We compare ReHub against leading graph transformer based methods.
298
      GraphGPS (Rampášek et al., 2022) offers a framework to integrate MPNNs of different types (Kipf
      & Welling, 2016; Chen et al., 2020; Hu et al., 2019; Bresson & Laurent, 2017) and transformers.
299
      Transformer indicates a straightforward adaptation of the standard transformer architecture Vaswani
300
      et al. (2017) to graphs. Spectral Attention Networks (SAN) (Kreuzer et al., 2021) employ attention
301   on the fully connected graph in addition to graph attention using the original edges. Closest to our
302   method is Neural Atoms (Li et al., 2024) which utilizes a set of different, learned, virtual nodes
303   at each layer. Neural Atoms is able to propagate long-range information, improve performance
304   across various tasks and can be modularly applied to different MPNNs. As opposed to Neural
305   Atoms, in this work we aim to tackle the efficiency aspect, i.e. to maintain linear complexity in
306   the number of nodes in a graph without a loss of performance. Exphormer (Shirzad et al., 2023)
307   introduces a transformer architecture that achieves linear computational complexity by leveraging
308   expander graphs (Alon, 1986) to define sparse attention patterns. In this model, each node attends
309
      only to a fixed number of neighbors specified by a fixed expander graph, and a few global virtual
      nodes connected to all nodes are used to capture global context. In contrast, our proposed method
310
      ReHub employs a dynamic model where virtual nodes (hubs) are connected to subsets of nodes
311
      (spokes) rather than to all nodes. ReHub allows for rewiring connections between layers, enhanc-
312   ing adaptability while avoiding bottlenecks associated with fully connected global nodes, all while
313   maintaining linear computational complexity.

458
459
      Long-range spoke update layer components. As described in Section 3, our long-range spoke
460   update layer is designed to handle long range communication while maintaining memory efficiency.
461   The ablation provided in Table 4 highlights the contributions of the primary design choices. (1)
462   Initialization of hub features from spokes vs. learned hub features as parameters, where the lat-
463   ter is analogous to the initialization process of Neural Atoms. The initialization scheme, out-
464   lined in Section 3.3, is compared to a configuration where hub features are learned parameters,
465   i.e, h0ih = Pih ∈ Rd . Initializing the hubs from spokes significantly improves performance, while
466   using learned hubs results in reduced performance compared to the GNN baseline. (2) The use of
467   a fixed vs. dynamic number of hubs. For the fixed case, we set the
                                                                       √ number of hubs to 22, which is
468   approximately the square root of the average
                                               √     number of nodes,    479.4 ≈ 21.9. This is compared
469   to setting the number of hubs as Nh = Ns dynamically, according to graph size. (3) Reassigning
470
      spokes to hubs at each layer vs. keeping the same connections fixed across the layers, is shown to
      improve performance. (4) Including an encoding layer for the spokes prior to aggregation further
471
      improves performance.
472
473   We provide additional experiments for varying values of hubs ratios and k in Appendix A.4.
474   Hub utilization.
475
476   To gain insights into the reassignment dynamics, we measure the level of hub utilization. We define
477
      hub utilization U as the number of hubs that have at least one spoke connected to them. Formally,
      U = |{ih | E:,ih · 1Nh ≥ 1}| where E:,ih represents the set of all spokes connected to hub ih . The
478
      percentage of unused hubs is then given by 1 − U/Nh , which reflects the proportion of hubs that
479
      are not connected to any spoke. Figure 3 presents a cumulative graph illustrating the percentage of
480   graphs with an unused hub percentage below a given threshold. Hub utilization is shown per layer
481   for different configurations of hub ratio r and connected hubs k, based on the validation split of
482   the PascalVOC-SP dataset. The results consistently demonstrate that approximately 10% of hubs
483   remain isolated in each graph, across layers, regardless of the number of hubs or the number of
484   spokes per hub. This finding suggests that the network maintains robust information flow between
485   spokes and hubs. We also present an additional metric based on the Bhattacharyya coefficient in
      Appendix A.4.

      5                         C ONCLUSION AND FUTURE WORK
522
523   In this paper, we introduced ReHub, a novel graph transformer architecture designed to enhance
524   long-range communication in large graphs through dynamic reassignment of virtual nodes. This ap-
525
      proach facilitates efficient information passage, enabling effective learning on large graphs while
      maintaining linear computational complexity and reduced memory usage compared to existing
526
      methods. Our experimental results demonstrate that ReHub consistently outperforms Neural Atoms
527
      and ranks among the top two methods against various baselines. It achieves competitive accu-
528   racy compared to Exphormer with lower memory consumption across all evaluated scenarios. Re-
529   Hub achieves its efficiency through our proposed reassignment mechanism, which maintains sparse
530   spoke-hub connectivity, as highlighted by our ablation studies. The modular design of ReHub allows
531   for easy integration with various MPNNs.
532
      Limitations While consistently improving computational efficiency, our spoke-hub communica-
533
      tion maintained strong performance, though it did not establish a new state of the art. Additionally,
534   our current design lacks inherent support for positional information available in geometric graphs.
535   In future work, we aim to extend our method to support tasks requiring long-range communication
536   on geometric graphs by incorporating positional information into the spoke-hub attention and re-
537   assignment mechanisms. We also plan to make the reassignment module learnable, and optimize
538   it towards the prediction task to further boost accuracy. Finally, integrating ReHub into architec-
539   tures like Exphormer to enable the reassignment of expander graph edges between layers presents a
      promising direction to further enhance long-range communication.

      A.3      I MPLEMENTATION DETAILS
865
866   In the following we describe in more detail how the architecture is implemented.
867
      We use the open-source code provided by GraphGPS (Rampášek et al., 2022) and available on
868
      https://github.com/rampasek/GraphGPS. We merge into that code parts from Exphormer (Shirzad
869
      et al., 2023) which are relevant for the training and evaluation of the OGBN-Arxiv and Coauthor
870   Physics datasets.
871
872   Clustering. During preprocessing we use the METIS partitioning algorithm (Karypis & Kumar,
873   1998) to divide each graph to clusters according to the required number of hubs. In practice, we use
874   the python wrapper PyMetis3 which allows us to map each spoke to a single hub.
875
876   Hub features. Throughout the implementation we use sparse data structures that allows us to keep
877   the complexity linear even when running on multiple graphs simultaneously (i.e. in a batch). The
878   hubs are stored in a similar fashion to how spokes are stored in a batch. i.e. the features are kept in
879   a 2D matrix X ∈ RN ×d together with a 1D graph index matrix B ∈ {0, . . . , #Graphs}N indicating
880   which graph each node belongs to, where N is the number of nodes in the whole batch.
881   When aggregating the spokes to calculate hub features we use the scatter functions which allows
882   us to aggregate the spokes to two matrices. (1) a 2D matrix H ∈ RH×d and (2) a 1D graph index
883   matrix BH ∈ {0, . . . , #Graphs}H indicating which graph each hub belongs to, where H is the
884   number of hubs in the whole batch.
885
886   Hub assignment and reassignment. Note that the initial hub assignment is implemented identi-
887   cally to the reassignment algorithm but with i∗h set to the initial hub found in the clustering step.
888
889   Attention. Opposed to other transformer architectures which require the conversion of the spokes
890   to a dense representation, we can keep both spokes and hubs in their sparse representation. For each
891
      attention module we use GATv2Conv implementation provided in pytorch geometric which accept
      as input this type of sparse representation. Moreover, this implementation accept as input a bipartite
892
      graphs and here we use it to pass information between the graph of spokes and graph of hubs.
893
894
      Large random regular graph. As mention in Section 4.2, we would like to construct graphs of
895   arbitrary size. To achieve this, we generate a dataset of large random regular graphs, which can be
896   produced at any scale while ensuring connectivity between nodes (i.e. , avoiding isolated subgraphs)
897   due to the regularity property, i.e. A d-regular graph is a graph where each node has d number
898   of neighbors. To do that we use the random_regular_graph (Kim & Vu, 2003; Steger &
899   Wormald, 1999) function from the open-source NetworkX library (Hagberg et al., 2008), which
900   takes as an input the number of nodes to be constructed and the degree of each node d. Additionally,
901   to ensure compatibility with the models, we assign random values to the graph’s edge attributes,
902   node features, and prediction labels.
903
904   Peak memory usage. To sample the peak memory usage of the models we use the function
905
      torch.cuda.max_memory_allocated. This function returns the peak memory allocation
      since running the torch.cuda.reset_peak_memory_stats function, which we call just
906
      before the call to the model.

971   Bhattacharyya Coefficient vs. Uniform Distribution The Bhattacharyya Coefficient for a dis-
      crete probability distributions P and Q is measurement of how similar the two probability distribu-

       tions are. It is defined as:                                              Xp

974                                                                              x∈X
975
       BC(P, Q) lies between 0 and 1, where the higher the coefficient the more similar the distributions
976
       are. In our setup, P is the distribution over spokes connection per hub; i.e. it is the number of
977    spokes connected to each hub, divided by the overall number of connections from spokes to hubs
978    (so that the result is indeed a probability distribution). We then set Q to be the uniform distribution,
979    i.e. 1/Nh for each hub. By doing so, we can see how close the actual distribution P is to the uniform
980    distribution, where uniform indicates the optimal balanced assignment of spokes to hub – where
981    every hub has exactly the same number of spokes. For convenience, we define the Bhattacharyya
982    Percentage as the Bhattacharyya Coefficient multiplied by 100.

984    below a given threshold for the PascalVOC-SP dataset. As in Section 4.3 this is shown for each
985    layer, and for varying values of hubs ratios and k on the validation split of the dataset. The results
986    demonstrate that regardless of the number of hub and number of hubs per spoke, most graphs have
987    a Bhattacharyya Percentage above 80%. This suggests that our reassignment method spreads nodes
988    quite evenly across the various hubs, and does not create high concentration of spokes which remain
989    connected to only few hubs.
