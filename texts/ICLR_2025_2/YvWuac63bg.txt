028                                                                   A BSTRACT
029
030
                               Contrastive loss is a powerful approach for representation learning, where larger
031                            batch sizes enhance performance by providing more negative samples to better
032                            distinguish between similar and dissimilar data. However, scaling batch sizes is
033                            constrained by the quadratic growth in GPU memory consumption, primarily due
034                            to the full instantiation of the similarity matrix. To address this, we propose a
035                            tile-based computation strategy that partitions the contrastive loss calculation to
036                            arbitrary small blocks, avoiding full materialization of the similarity matrix. Fur-
037                            thermore, we introduce a multi-level tiling strategy to leverage the hierarchical
038                            structure of distributed systems, employing ring-based communication at the GPU
039
                               level to optimize synchronization and fused kernels at the CUDA core level to re-
                               duce I/O overhead. Experimental results show that the proposed method scales
040
                               batch sizes to unprecedented levels. For instance, it enables contrastive training
041
                               of a CLIP-ViT-L/14 model with a batch size of 4M or 12M using 8 or 32 A800
042                            80GB without sacrificing any accuracy. Compared to SOTA memory-efficient
043                            solutions, it achieves a two-order-of-magnitude reduction in memory while main-
044                            taining comparable speed. The code will be made publicly available.

074   Despite the above benefits, scaling batch size in contrastive learning is severely limited by GPU
075   memory. The memory required for computing and storing image-text similarity matrices (see Fig-
076
      ure 2(a)) grows quadratically with batch size, making further scaling impractical and limiting the
      potential performance gains, even with advanced hardware. Several methods have been proposed
077
      to address memory limitations in scaling batch sizes for contrastive learning. Gradient-Cache (Gao
078
      et al., 2021) reduces memory usage by decoupling model and loss computations, but the mem-
079   ory cost of the loss remains a significant bottleneck. OpenCLIP (Ilharco et al., 2021) and DisCo-
080   CLIP (Chen et al., 2023) improve efficiency by distributing contrastive loss computation across n
081   GPUs, reducing memory consumption by a factor of n. However, despite advances in memory-
082   efficient techniques, most studies are limited to a batch size of 128k, restricting the potential of
083   contrastive learning and the scaling demands of modern models and datasets (Saunshi et al., 2019;
084   Chen et al., 2022; Kaplan et al., 2020).
085   In this paper, we introduce Inf-CL, a novel approach to mitigate the quadratic memory cost in
086   contrastive learning, which is caused by the full instantiation of the similarity matrix for log-sum-
087   exp (LSE) computation. Instead of storing the entire matrix, Inf-CL partitions the LSE calculation
088   into smaller, sequentially computed tiles, leveraging the cumulative property of LSE. This confines
089   memory usage to the tile size and the number of parallel tiles, allowing for a trade-off between
090   memory and computational efficiency. To enhance practical efficiency, we propose a multi-level
091   tiling strategy. At a coarse-grained level, image and text batches are distributed across multiple
092   GPUs, with each GPU performing serial LSE computations on multiple rows. As computations pro-
093
      ceed, asynchronous column-wise data exchange minimizes communication overhead, as illustrated
      in Figure 2(b). At a fine-grained level, row-wise computations are parallelized across CUDA cores
094
      within each GPU, consolidating iterations into a single kernel to reduce I/O overhead. Theoretically,
095
      Inf-CL can compute contrastive loss with nearly infinite batch sizes using a small tile size, albeit
096   with reduced speed. The multi-level tiling strategy is crucial to achieving practical scalability and
097   efficiency, balancing memory reduction with computation speed.
098
099

      reduces space complexity from quadratic (e.g., O(b2 ) for CLIP, O(b2 /n) for OpenCLIP) to lin-
100
      ear (O(b/n2 ) for Inf-CL), where b is the batch size and n is the number of GPUs. This significant
101
      reduction in memory cost enables efficient training of large batch sizes. For instance, training a ViT-

105   terms of computation time, Inf-CL matches the performance of prior methods, requiring approxi-
106   mately 59 hours to process a 64k batch size on 8 A800 GPUs. The time cost scales nearly linearly
107   with batch size, as demonstrated by a batch size increase from 64k to 256k resulting in a roughly 4×

122   2.1   D ISTRIBUTED TRAINING SYSTEM
123
124   Cross-GPU Communication: For scaling batch size, training across multiple GPUs is crucial to
125   handle memory and computational demands. However, communication overhead between GPUs
126   can limit performance. Techniques like hierarchical all-reduce and ring-based communication al-
127   leviate such overhead by optimizing synchronization between GPUs (Liu et al., 2023). Blockwise
128   parallelism, as employed in methods like ring attention, further improves efficiency by overlapping
129
      computation and communication.
130   GPU Memory and Execution: The performance of modern deep learning models relies heavily
131   on hardware resources, particularly GPU memory and execution capabilities. GPUs, like A100s,
132   typically have two different types of memory: HBM (High Bandwidth Memory) and SRAM (Static
133   Random Access Memory). HBM serves as the primary memory with a capacity of up to 80GB. In
134   contrast, SRAM is much smaller (usually measured in megabytes) but offers a significantly faster
135
      access speed, acting as a vital cache for frequently accessed data and enabling rapid computations.
      Techniques like FlashAttention (Dao et al., 2022) show that fine-grained control over the memory
136
      access of HBM and the fuse the operations can achieve faster training and less memory usage.

143   text encoders are trained with contrastive loss after extracting features. For brevity, we only discuss
144   image-to-text contrastive loss as an example in the following sections, since the implementation of
145   text-to-image contrastive loss is symmetric. Specifically, given a batch size of b and the in-batch
146   c-dimensional visual feature I ∈ Rb×c and textual feature T ∈ Rb×c , the image-to-text contrastive

      The vanilla implementation first computes the similarity matrix X ∈ Rb×b = I · T ′ and stores it

154
      of negative log-likelihood is applied to the similarity matrix. The memory required to store X and
155

159   calculation still requires 66GB. This indicates that, with batch size scaling, the memory bottleneck
160   during training lies in the loss calculation. Although large batch sizes are necessary for improving
161   model performance (Saunshi et al., 2019; Chen et al., 2022), the traditional implementation struggles
      to support them due to excessive memory consumption in the loss calculation.

252
253   The scaling of batch size is usually accompanied by the scaling of the number of GPUs. In order to
254   fully utilize the parallelism between multiple GPUs while exploiting partially serial computation on
255   a single GPU to reduce the memory cost, we propose a multi-level tiling method that distributes the
256
      above LSE calculation to coarse-grained cross-GPU tiles and fine-grained in-GPU tiles.
257
      Cross-GPU Tile. As shown in Algorithm 1, in data parallel training with n GPUs, the i-th GPU
258   first processes a portion of images and texts to visual features I i ∈ Rbs ×c and textual features
259   T i ∈ Rbs ×c , where bs = b/n is the batch size in one GPU. Then for the calculation of the contrastive
260   loss, we distribute computations of different rows to different GPUs and synchronize the columns
261   between GPUs step-by-step, considering the row-wise characteristic. Specifically, the i-th GPU is
262   responsible for calculating X i,: and the corresponding li . For memory considerations, based on
263   the tiling strategy described in Section 3.1 where only one tile X i,j is computed at a time, X i,: is
264   further divided into X i,j for n step to calculate li following Equation 4, where the local LSE li,j is
265   calculated by in-gpu tiling as described in the next part.
266   Moreover, since the computation of X i,j while i ̸= j requires the textual feature T j stored in other
267   GPUs, additional communication overhead is inevitable, especially as the number of GPUs grows.
268   In order to reduce or even eliminate the communication overhead, we associate all GPUs with a
269   ring topology, based on the idea of overlapping communication time and computation time overlap
      as much as possible. Concretely, starting with T i , each GPU process sends the current textual

      5   R ELATED W ORK
502
      Contrastive Learning: The core idea of contrastive learning is to learn better representations by
503
      distinguishing between positive and negative pairs of samples (van den Oord et al., 2018; Chen et al.,
504
      2020b). This approach demonstrates strong effectiveness across diverse tasks, as the nature of the
505   paired samples varies depending on the specific application. In image foundation models, such as
506   SimCLR (Chen et al., 2020a) and MoCo (He et al., 2020), positive pairs are created by augmenting
507   the same image in different ways. For cross-modal retrieval, as exemplified by CLIP (Radford et al.,
508   2021) and ALIGN (Jia et al., 2021), the positive pairs consist of aligned image and text samples.
509   Similarly, for dense text retrieval (Karpukhin et al., 2020; Wang et al., 2022; Zhang et al., 2022),
510   the positive pairs are composed of query and document pairs. Several works improve contrastive
511   learning performance by enhancing dataset quality, modifying the loss function, or refining negative
512   sample selection (Vasu et al., 2024; Zhai et al., 2023; Zhang et al., 2023). Moreover, several studies,
513
      both empirical and theoretical, have demonstrated from various perspectives that larger batch sizes
      contribute to learning better representations (Saunshi et al., 2019; Chen et al., 2022). Due to the
514
      quadratic growth of memory usage with batch size in classical contrastive loss, most existing studies
515
      have stopped scaling their batch sizes to 128k, even when leveraging hundreds of GPUs (Radford
516   et al., 2021; Jia et al., 2021; Yang et al., 2022).
517
518   Memory-efficient Training: As deep learning models continue to grow in size and complexity, the
519
      demand for computational resources, particularly GPU memory, has increased significantly. Tech-
      niques such as Gradient Checkpointing (Sohoni et al., 2022) recompute activations during back-
520
      propagation to save memory at the expense of additional computation. Flash Attention (Dao et al.,
521
      2022) reduces memory overhead by computing attention in blocks without storing large interme-
522   diate states. Ring Attention (Liu et al., 2023) distributes long sequence activations across multiple
523   devices, overlapping computation and communication to train sequences far longer than previous
524   methods. For contrastive learning, GradCache (Gao et al., 2021) and BASIC (Pham et al., 2021) in-
525   troduce a gradient caching technique that decouples backpropagation between contrastive loss and
526   the encoder, which reduces memory usage in the model by accumulating gradients per mini-batch.
527   OpenCLIP (Ilharco et al., 2021) and DisCo-CLIP (Chen et al., 2023) reducing memory consumption
528   by distributing the computation of contrastive loss across multiple GPUs.

      6   C ONCLUSION
531
532
      This paper addresses the GPU memory bottleneck in scaling batch sizes for contrastive loss. To
533
      overcome the quadratic memory consumption resulting from the full instantiation of the similarity
534   matrix, we proposed a tile-based computation strategy that partitions the calculation into smaller
535   blocks, thus avoiding full matrix materialization. Furthermore, we introduced a multi-level tiling
536   strategy that leverages ring-based communication and fused kernels to optimize synchronization
537   and minimize I/O overhead. Our experiments demonstrated that our method scales contrastive loss
538   batch sizes to unprecedented levels without compromising accuracy or training speed. This approach
539   marks a significant advancement in large-scale contrastive learning, shedding light on further devel-
      opments in areas such as self-supervised learning and dense text retrieval.

        A.3                 FACTORS INFLUENCING PERFORMANCE WHEN SCALING BATCH SIZE
785
786     While larger batch size is theoretically expected to enhance performance Chen et al. (2022), our
787     experimental results deviate from this expectation. To better understand this discrepancy, we analyze
788     the factors that impact performance when scaling up batch size.
789
        Hyperparameters. Although larger batch sizes provide more diverse negative samples for con-
790
        trastive learning, potentially improving the embedding space, careful tuning of hyperparameters is
791
        necessary to ensure model convergence. Previous research indicates that when increasing batch
792     size, the learning rate should be scaled proportionally to maintain a consistent parameter update
793     norm throughout training (Goyal, 2017). Since a fixed learning rate is used across all experiments,
794     this may have contributed to the reduced performance observed with larger batch sizes. Moreover,
795     prior studies suggest that large batch sizes require longer training epochs to ensure sufficient pa-
796     rameter updates and avoid suboptimal convergence (Hoffer et al., 2017). Overall, the performance
797     gains from larger batch sizes are contingent on the careful tuning of multiple hyperparameters be-
798     yond just learning rate and epochs, highlighting the importance of comprehensive hyperparameter
799     optimization to fully exploit the benefits of scaling.
800     Data Scale. Increasing batch size improves the precision of gradient estimation for the representa-
801     tion distribution defined by the dataset Chen et al. (2022). Larger datasets more accurately capture
802     real-world distributions, and thus, employing a larger batch size enables contrastive loss to generate
803     more precise gradients, enhancing the model’s ability to learn discriminative representations. As
804     shown in Figure 5, our experiments on different data scales (e.g., CC12M and Laion400M) indi-
805     cate that the optimal batch size increases with dataset size. Specifically, performance on CC12M
806
        saturates at a batch size of 32k, whereas Laion400M achieves saturation at a batch size of 256k.
807     In summary, while scaling up batch sizes is critical for enhancing contrastive learning, our findings
808     suggest that performance does not monotonically improve with batch size increases. As seen in
809     our previous experiments (Table 3), extremely large batch sizes (e.g., 1024k) can lead to a decline
        in performance, indicating that factors such as hyperparameter tuning and dataset scale are among
