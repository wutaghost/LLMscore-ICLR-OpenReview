009                                                       A BSTRACT
010
011                Recent advancements have enabled Large Language Models (LLMs) to function
012                as agents that can perform actions1 using external tools. This requires register-
013                ing, i.e. integrating tool information into the LLM context prior to taking ac-
014                tions. Current methods indiscriminately incorporate all candidate tools into the
015                agent’s context and retain them across multiple reasoning steps. This process re-
016                mains opaque to LLM agents and is not integrated into their reasoning procedures,
017                leading to inefficiencies due to increased context length from irrelevant tools. To
018                address this, we introduce EcoAct, a tool-using algorithm that allows LLMs to
019
                   selectively register tools as needed, optimizing context use. By integrating the tool
                   registration process into the reasoning procedure, EcoAct reduces computational
020
                   costs by over 50% in multi-step reasoning tasks while maintaining performance,
021
                   as demonstrated through extensive experiments. Moreover, it can be plugged into
022                any reasoning pipeline with only minor modifications to the prompt, making it
023                universally applicable to LLM agents now and in the future.

      1       I NTRODUCTION
027
028
      Large language models (LLMs) have been conceptualized as agents and have demonstrated their
029
      capability to perform a broad range of complex tasks. When augmented with external tools (Yuan
030   et al., 2023; Qu et al., 2024; Zhang et al.), LLM agents can extend their functionality beyond con-
031   ventional natural language processing (Qin et al., 2023). For example, LLM agents equipped with
032   scientific tools can conduct scientific research (Bran et al., 2023; Ghafarollahi & Buehler, 2024),
033   while those integrated with physical robotic systems are capable of performing robotic manipula-
034   tions (Ahn et al., 2022; Huang et al., 2023). External tools essentially expand the action space of
035   LLM agents, enabling them to leverage existing functionalities to accomplish a variety of complex
036   tasks (Xi et al., 2023; Wu et al., 2023a; Peng et al., 2023; Wu et al., 2023b; Shridhar et al., 2020).
037   To equip LLM agents with external tools, they must undergo a tool registration procedure. Specifi-
038   cally, information about the candidate tools needs to be added to the context of the LLMs that support
039   the agents. This information represents essential details for tool usage, including tool names, de-
040   scriptions in natural language, and instructions for input parameters. The current practice in tool
041   registration indiscriminately incorporates all candidate tools into the agent’s context, where these
042   candidate tools are preemptively selected by users or retrieved automatically through external algo-
043   rithms (Ocker et al., 2024; Qin et al., 2023; Gao et al., 2023). LLM-based agents will then process
044
      contextual information from all registered tools and select the appropriate tool for each reasoning
      step. However, this paradigm, which involves preparing all tools in advance and keeping the full
045
      information of the registered tools within the LLM’s operational context, introduces one key issue:
046
      the tool registration process is opaque to the agents and not fully integrated into their autonomous
047   reasoning pipelines. Each time the LLM is invoked, information from all passively registered tools
048   is processed, even though not all tools are necessary and only one single tool can be utilized in
049   each step, which drives inefficiencies in both cost and inference time (see Figure 1a). The problem
050   becomes more pronounced as the number of pre-registered tools grows, imposing an even greater
051   burden on the agent’s decision-making process. The agent possesses the capacity to reason to act
052   with their intrinsic reasoning mechanism but lacks the ability to reason to register.

      methods. (a) In ReAct, all tools are registered in advance, retaining full information of these tools
079   within the LLM’s operational context at each reasoning step. This leads to unnecessarily long con-

084
085   In this work, we present EcoAct, a general tool-using paradigm that integrates the tool registration
086   procedure into the LLM agents’ reasoning procedure, granting them discretionary authority, which
087   is the freedom to register any tools they wish to use at any time through their intrinsic reasoning
088   mechanisms (see Figure 1b). For any potentially useful tool, EcoAct prompts the agent to reason
089   about registering the function before utilizing it, rather than passively accepting pre-prepared tools
090   at each reasoning step. EcoAct gives agents the flexibility to register tools according to actual
091   needs, thereby retaining only the necessary tools in the context and reducing costs. While the ef-
092   fectiveness of this tool registration process can be further enhanced through other agent reasoning
093
      methods (Yao et al., 2022; 2024; Qin et al., 2023; Wei et al., 2022), ensuring that tools are registered
      appropriately is essential for maximizing the agent’s task-solving capabilities. Specifically, before
094
      the agent begins taking actions to solve the user’s query using its intrinsic reasoning algorithms, we
095
      only provide the agent with one single meta-tool named tool register, which enables the agent to
096   register any tools deemed useful based on lightweight but easily-identifiable contextual information
097   - tool names at any time step. The agent here will rely on this meta-tool to extend its skill library
098   and solve the problem with its own self-registered tools. Additionally, the agent’s intrinsic reasoning
099   algorithms are seamlessly integrated into EcoAct. This integration enables the agent to employ its
100   own reasoning logic to determine when and which actions to register.
101
      We conduct extensive experiments on the ToolBench benchmark (Qin et al., 2023), which involves a
102   diverse array of large-scale tools. We utilize EcoAct to enhance both the classic single-trace reason-
103   ing method ReAct (Yao et al., 2022) and the complex tree-structured reasoning method DFSDT (Qin
104   et al., 2023), applying it across multiple models, The results show that the enhanced reasoning al-
105   gorithms even can achieve monetary cost savings of over 50% on queries involving large tools
106   from ToolBench, without compromising performance. Additionally, we conduct further analysis to
107   demonstrate the effectiveness of key design choices in the proposed algorithm, regarding aspects
      such as the granularity of tool registration and the concise context used during tool registration.

      Our contributions are summarized below: (1) We highlight a key limitation in the current tool-
109   utilization paradigm of LLM agent systems: tool registration is essentially opaque to the LLM
110   agents. Indiscriminately maintaining information about all registered tools within the LLM’s op-
111   erational context imposes a significant burden on the agent’s decision-making process. (2) We in-
112   troduce EcoAct, a plug-and-play algorithm that could seamlessly integrate tool registration into
113   the agent’s intrinsic reasoning procedures. The agent could reason to determine when to register
114   what tools based on its needs, thereby mitigating the burden of processing all accessible tools in
115   the backed LLM calling by only maintaining necessary tools. (3) We conduct comprehensive ex-
116   periments using the ToolBench benchmark, which encompasses a wide range of large-scale tools.
117
      Our results demonstrate that the enhancement of EcoAct enables significant cost savings through
      various reasoning methods. Notably, for queries involving large tools from ToolBench, we observe
118
      cost reductions exceeding 50% across multiple models.

123   In this section, we present EcoAct, a general tool-using algorithm designed to mitigate efficiency
124   issues in agent tool-using scenarios. We begin by formulating the research problem and then provide
125   the details of each component designed in EcoAct.

127   2.1   PROBLEM SETUP
128
      We start by defining the relevant notations and outlining the research problem. Consider a language
129
      agent and a set of tools Z = {zi }Ii=i that the agent could access. The agent’s objective is to address
130
      user queries according to a specific policy π. At any given decision time step t, the agent receives
131   two types of information: (1) the historical context ct which includes all previous action-observation
132   pairs, and (2) a set of accessible tools Z that can be used in this time step. The agent then must
133   determine the next action to take. Formally, this decision process can be expressed as:
134
135                                          π(ct , Z̃) → at , s.t. at ∈ A,                               (1)
136   where at denotes the action that been taken at time step t. It represents one specific tool-calling from
137   accessible tool set Z̃. A denotes the action space of this language agent. Consequently, the size of
138   the tool space is equivalent to the size of the action space, i.e., |A| = |Z|.
139
140
      In evaluating a specific agent algorithm, the total token consumption required to complete user
      queries, which encompass both input and output tokens—serves as a general metric for assessing
141
      the algorithm’s cost (Chen et al., 2023; Wang et al., 2023; Hidvégi et al., 2024; Cheng et al., 2023).
142
      This is because token consumption is positively correlated with budget expenditure and latency,
143   particularly in the context of large language models as a service (Gan et al., 2023; Sun et al., 2022).

188   Overview. The core concept behind EcoAct is to empower agents to autonomously register tools
189
      they find useful, rather than passively relying on pre-assigned tools. Two key questions arise in the
      design of this algorithm: (1) Is there a short yet distinct context that can assist agents in filtering and
190
      selecting the necessary tools from the available options without adding extra computational cost?
191
      (2) Given the answer to the first question, how can tool registration be seamlessly integrated into the
192   agent’s intrinsic reasoning process, enabling agents to determine, at each reasoning step, which tool
193   to register and when, based on this distinctive information?
194
195
      To address the first question, we propose utilizing tool names as easily recognizable tags to assist
      agents in determining which tools should be registered. To operationalize this, we introduce a meta-
196
      tool, tool register, and define an action enabling agents to register any tools they consider relevant
197
      based on these tool names. The workflow is as follows: (1) Initialization: Prior to task execution,
198   agents are equipped solely with the tool register meta-tool. Simultaneously, users provide queries
199   that include all available tool names along with instructions on how to use the tool register. Once
200   the agent registers a tool by name using tool register, detailed information about the registered tool
201   becomes available. (2) Reasoning: At each reasoning step, the agent can either register a new tool
202   or invoke a previously registered one, depending on the task requirements. We then detail the design
203   of this process and the intuition behind each step.
204
205   Tool name as informer. The initial phase of EcoAct involves identifying a concise yet distinct
206   context from the candidate tools, allowing agents to determine which tools are essential and which
207   are not, utilizing their intrinsic reasoning capabilities without increasing computational load. As
208   illustrated in Figure 2b, we present the average token consumption associated with each component
209
      of an external tool. We could observe that the majority of token consumption is attributable to tool
      descriptions and input parameter instructions, whereas tool names account for only 6.6% of the total
210
      token usage. Inspired by the efficiency of human tool using, wherein the utility of a tool can often
211
      be inferred from its name without recalling every specific detail, we propose leveraging tool names
212   as the most easily identifiable markers to identify which tools require in-depth learning and which
213   can be bypassed. Since tool names are often highly recognizable, this approach intuitively imposes
214   minimal burden on the language model’s context processing.

      tool register as meta-tool. To achieve the objective of retaining only the necessary tools based on
217   their names, we propose enabling agents to actively register tools using their intrinsic reasoning ca-
218   pabilities. Specifically, before the agent engages in a task, we (1) provide it with a list of tool names,
219   which incurs minimal token usage, and (2) introduce a single tool, tool register, which facilitates
220   an action allowing the agent to register tools deemed useful based on their names at each time step.
221   When the agent invokes tool register with a selected tool name, it receives the complete information
222   about the tool and adds it to its skill library. Essentially, we initialize the action space A with a
223   single ”meta-action,” i.e., At=0 = ã. Here, ã represents the action of using tool register to register
224   one tool, thereby expanding the action space over time. This strategy prevents the indiscriminate
225
      incorporation of all candidate tools into the context, fostering more efficient tool utilization.
226   Since tool registration has been integrated as a specialized action within the agent’s action space,
227   our algorithm offers several distinct advantages: (1) Orthogonal to Agent Reasoning Algorithms:
228   Our method essentially forges a meta-tool capable of registering any tool deemed useful across
229   various agent reasoning algorithms. As demonstrated in Section 3.3.1, it is agnostic to the specific
230   reasoning algorithms employed and performs effectively with diverse reasoning techniques. (2)
231
      Efficiency with Large-Scale Toolsets: When dealing with queries with a vast number of tools,
      our method significantly reduces operational costs, achieving notable cost savings as detailed in
232
      Section 3.2. This efficiency arises because directly integrating a large number of tools into the agent
233
      is more costly. EcoAct minimizes the number of tools registered. Thus, our method is particularly
234   beneficial in scenarios involving extensive toolsets, as it ensures that only the essential tools are
235   being utilized. (3) Simplicity and Intuitiveness: Our method mirrors human problem-solving
236   strategies involving multiple tools: it first filters tools based on simple identifiers (tool names) and
237   then examines the details of the selected tools before use. This approach not only simplifies the
238   process but also provides a general framework that could inspire the design of other agent algorithms.

      3.1   EXPERIMENTAL SETUP
251
252
253   Data preparation. We mainly conduct experiments on the ToolBench (Qin et al., 2023), which
254   is large-scale dataset for tool use. It involves 16,464 tools in total which has been widely used as
255
      the benchmark to make evaluations of tool use algorithm (Du et al., 2024; Ye et al.). ToolBench
      comprises six subsets G1-Instruction (G1-I), G1-Tool (G1-T), G1-Category (G1-C), G2-Instruction
256
      (G2-I), G2-Category (G2-C), and G3-Instruction (G3-I). These subsets are classified according to
257
      varying levels of complexity in tool use, with differences in ’Instruction’, ’Category’, and ’Tool’
258   reflecting the relationships between tool categories in these test subsets and those in the training
259   sets. Following the same setting with AnyTool (Du et al., 2024), we adopted the filtered benchmark
260   which excludes all non-solvable queries in ToolBench. The remaining queries in these six subsets
261   are 115, 132, 142, 107, 98, and 38, respectively. Unless specified otherwise, for each query, we
262   use the state-of-the-art method AnyTool (Du et al., 2024) to retrieve tools for each query in all
263   experiments across the paper. More details of this benchmark could be found in Appendix B.
264   Evaluation metrics. We primarily use two metrics to make evaluations: the pass rate and the cost,
265   with the latter measured in monetary terms. Pass rate essentially measures LLM’s ability to success-
266   fully execute an instruction within limited budgets. We utilize the evaluation script from (Du et al.,
267   2024) to get the pass rate results in all experiments of our paper, which addresses issues related to
268   artificially inflated pass rates (Du et al., 2024). We utilize GPT-4-turbo to make the pass rate evalua-
269   tions, applying the same prompts as those used in ToolBench. Unless specified otherwise, we report
      the cost in US cents. More details about the evaluation prototype can be found in Appendix B.2.

298     3.2                  MAIN RESULTS
299
300     EcoAct essentially serves as a plug-and-play component for different agent reasoning algorithms.
301     Here, we mainly evaluate the impact of EcoAct on the classical reasoning method ReAct (Yao
302     et al., 2022). Our aim is to assess both its effect on the overall performance and its influence on
303
        the token cost associated with solving user queries. We mainly investigate our algorithm on ReAct
        because ReAct as the classic single trace agent reasoning algorithm could be regarded as the basic
304
        component of different agent reasoning algorithms (Yao et al., 2024; Qin et al., 2023) with multiple
305
        reasoning traces. We expect the conclusions drawn from ReAct experiments could potentially be
306     generalized to more complex algorithms in the future.
307
308     Overall results. In this study, we compare ReAct with its variant augmented with EcoAct and
309
        present the aggregated results of different subsets in G1, G2, and G3 from ToolBench benchmark
        in Table 1, focusing on both performance and monetary costs, with different models. Our findings
310
        indicate that EcoAct significantly reduces the costs associated with ReAct while maintaining, and
311
        in some cases exceeding, its performance. This suggests that EcoAct serves as a ”free lunch” com-

      4   R ELATED W ORKS
450
451   Large language models (LLMs) represent a major breakthrough in artificial intelligence, prompting
452   an increasing body of research dedicated to employing LLMs in the construction of autonomous
453   agents capable of performing complex tasks (Xi et al., 2023; Wu et al., 2023b; Peng et al., 2023;
454   Shridhar et al., 2020; Song et al., 2024; Wu et al., 2024; Zhang et al., 2023; Ma et al., 2024). In
455   these LLM-based agents, the ability to leverage external functions, tools, or actions to interact with
456   the environment or solve sub-tasks is crucial. These external tools empower agents to go beyond
457   natural language processing. For instance, LLM agents equipped with scientific tools can conduct
458   scientific research (Bran et al., 2023; Ghafarollahi & Buehler, 2024), while those integrated with
459
      robotic systems can perform robotic manipulation tasks (Ahn et al., 2022; Huang et al., 2023).
460   To enable agents to use external tools, they must undergo a process called tool registration, where
461   relevant tool information is integrated into the LLM’s context prior to the agent taking action. This
462   process becomes challenging when the number of available tools exceeds the context limits. One
463   approach to mitigate this limitation is through retrieval-augmented generation (RAG) (Lewis et al.,
464   2020; Gao et al., 2023). For example, Patil et al. (2023); Li et al. (2023) use a pre-trained text
465
      embedding model to retrieve relevant tools from a large tool pool. Similarly, Qin et al. (2023)
      trained an additional API retriever to identify essential tools using curated tool retrieval data.
466
467   To handle user queries with registered tools, various reasoning algorithms for LLM agents have
468   been explored recently (Yao et al., 2022; Qin et al., 2023). Specifically, Yao et al. (2022) propose an
469   approach that interleaves the generation of reasoning traces with tool-using actions, leading to more
470   reliable and factual responses. Qin et al. (2023) introduce DFSDT, a decision tree-based method that
471
      expands the search space, increasing the likelihood of identifying a valid tool-using path. However,
      these reasoning algorithms do not integrate with the tool registration process, which can result in
472
      unnecessary costs due to the registration of irrelevant tools. In contrast, our approach seamlessly in-
473
      corporates tool registration into these reasoning algorithms, allowing agents to autonomously reason
474   about and register only the necessary tools, thereby avoiding such inefficiencies.
475
476
477   5   C ONCLUSION
478
479   In this work, we propose EcoAct, a simple yet effective approach that seamlessly integrates tool
480
      registration into the intrinsic reasoning processes of LLM agents. The core concept involves initial-
      izing the agent with a meta-tool named tool register, which enables the agent to selectively register
481
      tools deemed useful based on their names at each time step. This action allows the agent to avoid
482
      indiscriminately incorporating all candidate tools into its context, instead retaining only relevant in-
483   formation across reasoning steps, thereby achieving significant cost savings. We evaluate EcoAct
484   on the ToolBench dataset, augmenting various reasoning methods, and demonstrate that EcoAct
485   significantly reduces computational costs while maintaining comparable performance.
