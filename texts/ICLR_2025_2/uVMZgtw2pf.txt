011                                               A BSTRACT
012
013            Understanding the decision-making process of machine learning models is cru-
014            cial for ensuring trustworthy machine learning. Data Shapley, a landmark study
015            on data valuation, advances this understanding by assessing the contribution of
016
               each datum to model performance. However, the resource-intensive and time-
               consuming nature of multiple model retraining poses challenges for applying Data
017
               Shapley to large datasets. To address this, we propose the CHG (compound of
018
               Hardness and Gradient) utility function, which approximates the utility of each
019            data subset on model performance in every training epoch. By deriving the closed-
020            form Shapley value for each data point using the CHG utility function, we reduce
021            the computational complexity to that of a single model retraining, achieving a
022            quadratic improvement over existing marginal contribution-based methods. We
023            further leverage CHG Shapley for real-time data selection, conducting experi-
024            ments across three settings: standard datasets, label noise datasets, and class im-
025            balance datasets. These experiments demonstrate its effectiveness in identifying
026            high-value and noisy data. By enabling efficient data valuation, CHG Shapley
027
               promotes trustworthy model training through a novel data-centric perspective.

      1   I NTRODUCTION
031
032   The central problem of trustworthy machine learning is explaining the decision-making process of
033
      models to enhance the transparency of data-driven algorithms. However, the high complexity of ma-
      chine learning model training and inference processes obscures an intuitive understanding of their
034
      internal mechanisms. Approaching trustworthy machine learning from a data-centric perspective
035
      (Liu et al., 2023) offers a new perspective for research. For trustworthy model inference, a rep-
036   resentative algorithm is the SHAP (Lundberg & Lee, 2017), which quantitatively attributes model
037   outputs to input features, clarifying which features influence specific results the most. SHAP and its
038   variants(Kwon & Zou, 2022b) are widely applied in data analysis and healthcare. For trustworthy
039   model training, the Data Shapley algorithm (Ghorbani & Zou, 2019) stands out. It quantitatively
040   attributes a model’s performance to each training data point, identifying valuable data that improves
041   performance and noisy data that degrades it. Data valuation reveals how much each training sample
042   affects model performance, serving as a foundation for tasks like data selection, acquisition, and
043   cleaning, while also facilitating the creation of data markets (Mazumder et al., 2023).
044   The impressive effectiveness of both SHAP and Data Shapley algorithms is rooted in the Shapley
045   value (Shapley, 1953). The unique feature of the Shapley value lies in its ability to accurately
046   and fairly allocate contributions to each factor in decision-making processes where multiple factors
047   interact with each other. However, the exact computation of the Shapley value is O(2n ), where n
048   is the number of factors. Even with estimation techniques such as linear least squares regression
049   (Lundberg & Lee, 2017) or Monte Carlo methods (Ghorbani & Zou, 2019), the efficiency of SHAP
050
      and Data Shapley algorithms struggles to scale with high-dimensional inputs or large datasets.
051   In this paper, we focus on the efficiency problem of data valuation on large-scale datasets. Instead
052   of investigating more efficient and robust algorithms to approximate the Shapley value as in (Lund-
053   berg & Lee, 2017; Wang et al., 2024b), we concentrate on estimating the utility function. The key
      intuition is that if the performance of a model trained on a data subset can be expressed analytically,

075   Data-centric AI. For an extended period, the machine learning research community has predomi-
076   nantly concentrated on model development rather than on the underlying datasets (Mazumder et al.,
077   2023). However, the inaccuracies, unfairness, and biases in models caused by data have become
078
      increasingly severe in real-world applications (Wang et al., 2023). Conducting trustworthy machine
      learning research from a data-centric perspective has garnered increasing attention from researchers
079
      (Liu et al., 2023). Research on data valuation can also be seen as an effort to understand the machine
080
      learning model training process from a data-centric perspective.
081
082   Shapley value. The Shapley value (Shapley, 1953) is widely regarded as the fairest method for
083   allocating contributions (Rozemberczki et al., 2022; Algaba et al., 2019). Given a set of players N
084

088
089          • Linearity: For utility functions U1 , U2 and any α1 , α2 ∈ R, ϕ(i; α1 U1 + α2 U2 ) =

               α1 ϕ(i; U1 ) + α2 ϕ(i; U2 ).

      Data valuation. Data valuation aims to quantitatively analyze the impact of training data on the
109   performance of machine learning models, particularly deep neural networks. Determining the value
110   of data can facilitate tasks such as data selection (Wang et al., 2024c), acquisition(Mazumder et al.,
111   2023), and trading(Wang et al., 2024a). Since the performance of machine learning models is de-
112   rived from the collective contribution of all data points, a fair data valuation algorithm is essential to
113   accurately attribute each data point’s contribution to the model’s performance.
114
      A milestone in data valuation research is the Data Shapley algorithm developed by Ghorbani & Zou
115   (2019). Data Shapley builds upon the Shapley value formula by treating all data in the training
116   dataset as players in a cooperative game of “training the model together”. U (S) represents the
117   accuracy achieved by training the model on dataset subset S. Consequently, the Shapley value
118   computed for each data point serves as an indication of its contribution to model performance. Data
119   points with lower values are considered more likely to be harmful, while those with higher values
120   are deemed to be lacking in the dataset. Removing low-value or acquiring more high-value data can
121   improve accuracy. Following Data Shapley, several subsequent have been proposed.
122   KNN Shapley (Jia et al., 2019) focuses on the Shapley value for k-nearest      neighbors, offering
                                                                               √
123   a closed-form solution that improves computational efficiency to O( n log(n)2 ). Beta Shapley
124   (Kwon & Zou, 2022a) addresses the uniform weighting issue in Data Shapley by relaxing the ef-
125   ficiency axiom and using a beta distribution, improving performance in tasks like mislabeled data
126   detection. Data Banzhaf (Wang & Jia, 2023) reduces noise in model training by utilizing the Banzhaf
127   value, shown to be robust in noisy environments. LAVA (Just et al., 2023) developed a proxy for
128   validation performance, enabling the evaluation of data in a manner that is agnostic to the learning
129
      algorithms. DVRL (Kwon & Zou, 2023) and Data-OOB (Yoon et al., 2020) utilize reinforcement
      learning and out-of-bag estimation, respectively, to assess data values.
130
131   Several other works use gradient information to compute the value of training data. Xu et al. (2021)
132   propose the cosine GradE Shapley value (CGSV) to measure each agent’s contribution in a col-
133   laborative machine learning scenario. CGSV approximates an agent’s Shapley value during the
134   aggregation process by using the cosine similarity between the agent’s gradient and the aggregated
135
      gradient of all agents. Pruthi et al. (2020) introduced TracIn, a method that measures the influence
      of a training example on a test example by computing the inner product between their respective
136
      gradients.
137
138   Finally, Jiang et al. (2023) introduced the OpenDataVal benchmark, which focuses on data valuation
139   algorithm design by abstracting model training. We also use this benchmark to evaluate our proposed
140   algorithm in data valuation experiments.
141   Data selection. There are two main streams in the data selection field. The first focuses on deter-
142   mining the scores of individual data points, employing methods such as influence functions (Koh
143   & Liang, 2017; Yang et al., 2023; Chhabra et al., 2024), the EL2N score (Paul et al., 2021), and
144   loss (i.e., hardness) metrics (e.g., InfoBatch (Qin et al., 2024), worst-case training (Huang et al.,
145   2022)). These approaches typically use the information from a single data point and model parame-
146   ters to calculate scores for that point, subsequently selecting the data points with the highest scores.
147
      Recently, Xia et al. (2024) proposed LESS, which sets the gradient mean of data within a specific
      downstream task as the target gradient and employs the inner product or cosine similarity to select
148
      data with the most similar gradients from a large collection of instruction datasets for model fine-
149
      tuning. Their experiments (Xia et al., 2024) show that training on just 5% of the data selected by
150   LESS often outperforms training on the full dataset across various downstream tasks.
151
152   The second approach, including coreset methods such as GradMatch (Killamsetty et al., 2021a)
153
      and Glister (Killamsetty et al., 2021b), as well as data distillation (Zhao et al., 2021), emphasizes
      the utility of data subsets. These methods aim to identify a subset that effectively represents the
154
      original dataset while achieving comparable model accuracy. They typically leverage the gradients
155
      of individual training data points to form a subset whose gradient direction aligns with that of the
156   validation set.
157
158   Our data selection method, CHG Shapley, which is grounded in data valuation principles, represents
159
      a fusion of these two streams: it leverages the utility of different training subsets to calculate a
      score (i.e., Shapley value) for each individual data point. We will compare our method with these
160
      two approaches in data selection experiments, demonstrating its ability to identify high-value (i.e.,
161
      representative) data and low-value (i.e., noisy) data.

      3.1   P RELIMINARIES
165
166   Let N = {1, . . . , n} denote a labeled training set. The objective is to assign a scalar value to
167   each training data point based on its contribution to the model’s performance. A utility function
168   U : 2N → R maps any subset S of the training set (including the empty set) to the performance of
169   the model trained on that subset, denoted as U (S). In Data Shapley (Ghorbani & Zou, 2019), the
170   model’s accuracy on a hold-out test set is utilized as U (S). Besides, similar to Data Shapley, we do
171   not impose any distributional assumptions on the data, and independence among the data points is
172
      not required.

176

177

181   the model training time and expedite the calculation of each data’s value. To achieve this, we first
182   investigate the impact of training the model using a subset of data S.
183

187   training loss as much as possible. Therefore, in our study, the utility function is defined as the
188   degree of decrease in training loss.

      Differences from Data Shapley: Although our approach shares the same underlying principle as
      Data Shapley, we do not try to approximate it. Data Shapley utilizes a model trained on a subset of
275
      the training set, evaluated on an additional validation set, as its utility function. In CHG Shapley,
276
      our utility function captures the extent of training loss reduction when using a subset of the training
277   dataset. Furthermore, in Data Shapley, the required size of the validation set can grow by orders
278   of magnitude to achieve the desired utility precision (i.e., the model’s accuracy on the validation
279   set). For example, in a classification task, achieving a precision of 0.001 for the utility function
280   may require 1,000 validation samples, while a precision of 0.0001 could demand 10,000 samples.
281   Another challenge lies in the noise affecting the utility function, which arises from the model train-
282   ing process—particularly due to stochastic gradient descent—as discussed in Data Banzhaf (Wang
283   & Jia, 2023). In contrast, CHG Shapley relies solely on the training set and incorporates both loss
284   and gradient information from the model during training. This richer and more precise information
285
      allows CHG Shapley to achieve lower computational costs and higher precision. Moreover, by av-
      eraging the Shapley values over multiple epochs, CHG Shapley is somewhat resistant to the impacts
286
      of utility noise.

352   4.2   CHG S HAPLEY- BASED DATA S ELECTION A LGORITHM
353
354   In this section, we introduce several implementation strategies and practical techniques to improve
355   the scalability and efficiency of CHG Shapley, incorporating common tricks from data selection
356
      methods (Killamsetty et al., 2021a;b), and all the compared methods (except Full, Random, and
      AdaptiveRandom) also adopt these tricks:
357
358   Interval Data Selection: Instead of selecting new data at every training step, we perform data
359   selection at intervals. For example, during a 300-epoch training process, data selection is done
360   every 20 epochs, while in the remaining epochs, the previously selected subset is used for training.
361   This approach significantly reduces the computational overhead associated with data selection.
362   Last-Layer Gradients: Given the high dimensionality of gradients in modern deep learning models,
363   we adopt a last-layer gradient approximation. By focusing solely on the gradients of the last layer,
364   this technique accelerates the computation for all methods.
365
      Per-Class Approximations: To further enhance model accuracy, we apply a per-class approach,
366   running the algorithm separately for each class by considering only the data points from that class in
367   each data selection iteration. This reduces both memory usage and computational cost, improving
368   the overall performance of all methods.
369
      We detail the CHG Shapley Data Selection Algorithm in Algorithm 2, integrating the aforemen-
370
      tioned techniques to ensure efficient and scalable data valuation-based data selection.

      6   D ISCUSSION AND F UTURE W ORK
493
      In the data selection experiments, the introduction of hardness improves the model’s performance
494
      when no label noise is present. This suggests that hardness can serve as a proxy for data value
495
      to some extent. However, in cases with significant label noise, incorporating hardness into GradE
496   Shapley diminishes its effectiveness. This indicates a need for further investigation into how hard-
497   ness can be fairly integrated into data value assessments, particularly when it becomes an unreliable
498   metric.
499
      Besides, CHG Shapley performs well at small fractions but struggles with larger ones. This chal-
500
      lenge may arise from the difficulties of selecting data based on Shapley values (Wang et al., 2024c),
501
      rather than the limitations of CHG Shapley itself. Specifically, once a subset of data s is selected,
502   the Shapley value of the remaining data may change, as the set Sπi in Equation 2 will include s.
503   This affects the effectiveness of the original Shapley values in evaluating the remaining data. If we
504   must recalculate the Shapley values for the remaining data each time a new data point is selected,
505   the time complexity becomes unmanage able even with our proposed method. Thus, we leave the
506   exploration of iterative Shapley value calculations for data selection to future work.
