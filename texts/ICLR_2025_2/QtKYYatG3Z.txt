009                                                 Abstract
010
011              A common approach for assessing whether generative models develop world mod-
012              els is by studying the behavior of fixed models. However, many of the benefits of
013              having a world model arise when transferring a model to new tasks (e.g. few-shot
014              learning). In this paper, we ask: what does it mean to test if a learner has a world
015              model embodied in it? We consider a simple definition of a true world model: a
016
                 mapping from inputs to states. We introduce a procedure that assesses a learner’s
                 world model by measuring its inductive bias when transferring to new tasks. This
017
                 inductive bias can be measured in two distinct dimensions: does a learner ex-
018
                 trapolate to new data by building functions of state, and to what degree do these
019              functions capture the full state? We use this procedure to study the degree to which
020              pretrained models extrapolate to new tasks based on state. We find that models
021              that perform very well on next-token prediction can extrapolate to new tasks with
022              very little inductive bias toward state. We conclude by assessing the possibility
023              that these models learn bundles of heuristics that enable them to perform well on
024              next-token prediction despite preserving little of state.

      1     Introduction
027
028
      A growing body of research investigates whether large language models (LLMs) and other foundation
029
      models form internal representations of the data they’re trained on (Abdou et al., 2021; Li et al., 2023).
030   Methods that uncover world models from sequential data would be valuable in many settings: they
031   could be used to uncover scientific breakthroughs in domains such as protein generation, genetics, and
032   chemistry (Chowdhury et al., 2022; Benegas et al., 2023; Jablonka et al., 2024; Boiko et al., 2023).
033
      One of the biggest advantages of a model with an implicit world model is effective few-shot learning;
034
      with a correct world model, the same model can be transferred to different but related tasks with
035
      minimal modifications. However, much of the literature studying world models has focused on
036   assessing the outputs of a fixed model (Toshniwal et al., 2022; Vafa et al., 2024). A true world model
037   should manifest not just in making valid predictions, but in how a system learns and adapts to new
038   situations using a generalizable representation of the domain. This is particularly important because
039   many of the purported benefits of world models—like few-shot learning and transfer—specifically
040   arise from how models learn new tasks. Here, rather than studying a fixed model, we ask: what
041   does it mean to test if a learner has a world model embodied in it?
042   We consider a simple definition of a world model: a real-world representation of inputs in a
043   low-dimensional state space. Meanwhile, a learner is any procedure that takes a dataset and returns
044   a model that relates inputs to outputs. We then propose a procedure to test if a learner has a given
045   world model: when a learner is applied to a new dataset, to what degree does it learn functions of
046   this low-dimensional representation? For a learner to rely on a given world model, every dataset it is
047   applied to should only be a function of this low-dimensional representation of reality. This definition
048   is not just abstract; a world model in language could correspond to underlying concepts, so a learner
049   that has the correct concepts should have an inductive bias to learn new functions of these concepts.
050   We introduce two related definitions that capture properties for whether a learner uses the world
051   model. The first definition is about whether a learner respects state. This corresponds to whether
052   a learner’s predictions of points across new datasets obey state structure. For example, if the state
053   space corresponds to a board in the game Othello, the learner respects state if all sequences that
      map to the same board have the same prediction within any given dataset. However, respecting state

054
      doesn’t convey the full story; for example, a model can make predictions using very coarse functions
055   of state, such as how many pieces a particular Othello board has. To provide a fuller picture, we also
056   provide a definition of what it means for a learner to fully reconstruct state. Learners that use coarse
057   functions of state will not satisfy this second definition.
058
      We present a computationally efficient method for estimating these quantities. Our method involves
059
      repeatedly applying a learner to small amounts of data on random outputs that obey state and
060
      studying how it extrapolates. We then build a model that predicts these extrapolations as a function
061   of state. We quantify two properties to measure the learner’s world model: first, the learner’s
062   inductive bias toward state is how well the extrapolations can be predicted from state. Next, the
063   degree to which the learner recovers full state is given by how predictable original states are from
064   a shared representation that is predictive of extrapolations.
065
      We use this procedure to study the extent to which pretrained models use world models when
066
      fine-tuning. We consider several applications where the true world model is known. In the first
067   application, we study a setting where orbital data obeys the world model of Newtonian mechanics.
068   We pretrain a transformer on trajectories of planetary motion and ask: can the model transfer to other
069   tasks that rely on Newtonian mechanics? We show that while the model appears to obey Newtonian
070   mechanics for the task it’s trained on, our metrics reveal poor inductive bias. We show that instead
071   of recovering a compact world model, the learner is relying on piecemeal heuristics; while Newton’s
072   law of gravity can be recovered when the model is fine-tuned on narrow kinds of transfer data, the
073   model implies nonsensical laws when fine-tuned on more general sequences.
074   We also perform analogous exercises in two other areas where the true world model is known:
075   lattice problems (Liu et al., 2022; Vafa et al., 2024) and Othello games (Li et al., 2023; Nanda et al.,
076   2023b; Hazineh et al., 2023). On lattices, we find that sequence models have strong inductive biases
077   toward true state. On Othello, we find smaller inductive biases toward state. By way of calibration,
078   we also consider oracle models that are directly pretrained on state, in order to calibrate the degree
079   to which a model’s extrapolative properties are limited by architecture. These oracle benchmarks
080   show that while simple recurrent models like RNNs (Elman, 1990) and LSTMs (Hochreiter,
081
      1997) have about as strong an inductive bias as their respective oracles, there is a large gap for
      transformer (Vaswani et al., 2017) and Mamba (Gu & Dao, 2023; Dao & Gu, 2024) models. We
082
      next demonstrate the implications of these metrics; our inductive biases have a strong correlation
083
      with transfer performance across tasks.
084
085   Our results show that models, despite performing well on next-token prediction, can have poor
086   transfer properties and low inductive bias towards state. We conclude by assessing the possibility
087
      that these models — instead of learning compact representations of world models — learn bundles of
      heuristics (Karvonen et al., 2024) that enable them to perform well on next token prediction despite
088
      having poor transfer properties for new problems.

091   2   Framework
092
093   In this section, we lay out our framework for defining whether an algorithm learns from data using
094   an underlying world model. Let x ∈ X denote some input and y ∈ Y denote some output. In
095   our framework, the underlying world model is summarized by some state space Φ and a mapping
096   ϕ : X → Φ that associates each input with some state ϕ(x) ∈ Φ. An example is any pair (x, y) and a
097   dataset D = {(x1 , y1 ), . . . , (xn , yn )} is any finite collection of examples. A dataset D is consistent
098   with the underlying world model if for any pair (x, y), (x′ , y ′ ) ∈ D with ϕ(x) = ϕ(x′ ) then y = y ′ .
099
      When evaluating a learner against a world model, we assume the learner is applied to datasets that
      are consistent with the world model. Let DΦ denote the collection of all consistent datasets.
100
101   A learning algorithm, when given a dataset D, returns a prediction function m(·;
                                                                                  b D) that relates inputs
102   x to outputs y. We next state two definitions that capture properties related to whether a learning
103   algorithm uses a world model ϕ. Let P (·) be some chosen distribution over the inputs with x ∼ P (·).
104   Definition 2.1. The learning algorithm respects state Φ if for all D ∈ DΦ there exists some function
105   f (·; D) : Φ → Y such that m(x;
                                 b    D) = f (ϕ(x); D) for all x ∈ X with P (x) > 0.
106
107   In other words, the learning algorithm respects state if its learned prediction function returns the
      same predictions on inputs mapped to the same state by the world model. While this captures an

      intuitive property, it is nonetheless a weak requirement. As an extreme case, consider a learning
109   algorithm that returns a constant prediction function when applied to any dataset; this trivial learning
110   algorithm mechanically respects state. To distinguish such cases, we introduce a second property.
111
      Definition 2.2. Consider a learning algorithm that respects state Φ. The learning algorithm fully
112
      reconstructs state if there exists no non-injective r : Φ → Φ such that m(x;
                                                                              b    D) = f (r(ϕ(x)); D) for
113   all x ∈ X with P (x) > 0 and D ∈ DΦ .
114
115   The learning algorithm fully reconstructs state if its predictions cannot be expressed by coarsening the
116   state space of the underlying world model. If not, the learning algorithm partially reconstructs state.

207   3   Orbital Mechanics
208
209
      Here, we illustrate these metrics on a simple example where learners are applied to data that obey
210   Newtonian mechanics. Specifically, we simulate trajectories of a planet in motion and train a
211   transformer to predict the next location of the planet. We then ask: does fine-tuning a pretrained
212   model to new tasks demonstrate an inductive bias toward the states dictated by Newtonian mechanics?
213   Despite the model performing well on next-token prediction, our metrics reveal a low inductive bias
214   toward state. We demonstrate that the model has recovered piecemeal heuristics rather than a compact
215   world model; while it can recover Newton’s law of gravity for narrow slices of the data, it forms
      other laws for other types of sequences.

236   Data and pretraining. We begin by simulating a dataset of sequences where each sequence
237   describes a planet in motion around a sun, i.e. a two-body problem. To do this, we randomly sample
238   initial conditions (e.g. the masses and positions of each planet and their initial relative velocity)

240   the x and y coordinates of the lighter planet for 200 time-steps with 5-minute intervals, resulting in
241   sequences of length 400 (e.g. x1 , y1 , x2 , y2 , . . . , x200 , y200 ). We generate 1M such sequences for
242
      training (i.e. 400M training tokens) and 10,000 sequences for a held-out set. We train a 12-layer
      transformer decoder (Vaswani et al., 2017) to predict the next token of each sequence in the training
243
      set. We use a modified transformer to model continuous data; see Appendix A for more details.
244
245   We evaluate the model’s predictions on held-out data. The held-out R2 is above 0.99, indicating
246   very good prediction. The model outperforms a baseline model that always predicts the most recent
247   position, especially as it’s forced to generate more of the orbit (Figure 5). The left two panels of
248
      Figure 1 shows examples of orbits; given only a few data points for the beginning of the orbit, the
      model can complete the orbit with high accuracy.

250   Is the model a physics learner? The pretrained model’s predictions appear to obey fundamental laws
251   of physics. Here we ask: does the model use laws of physics as an inductive bias when transferring
252   to other problems? To test this, we note that each observation in a sequence of orbits is governed by
253   an 8-dimensional state vector consisting of the masses, relative velocities, and relative positions of
254
      each planet. Given the current state of a trajectory, the next position of an orbit is deterministic. If a
      learner’s inductive bias depends on the laws of orbital physics, it must be extrapolating based on state.
255
256   We use the metrics described in Section 2 to assess the model’s inductive biases. We implement these
257   metrics by simulating small datasets with the same inputs as the pretraining data but different outputs;
258   instead of training the model to predict next-token, we fine-tune it to predict random Gaussian noise
259   on each dataset. The inductive bias (IB) test assesses whether a model’s extrapolations across datasets
260
      can be predicted by the 8-dimensional state vector; meanwhile, the state recovery (SR) test aims to
      predict the state vector from the shared representation used by the IB test. We perform the test by
261
      generating 5 datasets and fine-tuning the pretrained model on each one to measure its extrapolations.
262
      See Appendix B for more details on how we implement these metrics.
263
264   We report R2 for both metrics (1 is perfect prediction, 0 is equivalent to a constant baseline). The
265   IB R2 is 0.65, showing that the pretrained model does not have a large inductive bias toward state
266
      when it transfers. Meanwhile, the SR R2 is 0.62 for the next-token-pretrained model. This shows
      that not only do the inductive biases not relate to state, but also that state isn’t fully captured.

291   4   Other Applications
292
293   We now apply our metrics to evaluate the world model properties of learners in other applications.
294   Evaluating world models requires using datasets where ground-truth states are known, and we study
295   two such common types of datasets: lattice problems and the board game Othello.
296   Lattice. One paradigm for assessing world models is studying a model’s behavior when it’s trained
297   on sequences that arise over lattices (Vafa et al., 2024; Liu et al., 2022). We study a lattice setting
298   similar to the Gridworld example considered in Liu et al. (2022). This setting simulates an agent
299   moving along a line segment with a finite number of positions. Specifically, there is a true state space
300   consisting of S states: Φ = {1, 2, . . . , S}. The language x consists of sequences with three tokens:
301   Σ = {L, ⊥, R}. The initial state of the sequence is 1. For a token σ = R, the state increases by 1,
302   while the state decreases by 1 for σ = L and stays the same for σ =⊥. When the state is 1, the state
303
      is at the boundary, so σ = L is not a valid token; similarly, when the state is S, σ = R is not a valid
      token. All tokens are valid for all other states. The last token of the sequence indicates the final state.
304
      We randomly generate sequences over the language by sampling a move uniformly at random over the
305
      set of valid moves for each timestep. We initialize the state at 1 and then sample sequences of length
306   100 over S = 5 states. We create a training set of 9.9M tokens and a hold-out set of 44K tokens.
307
308   Othello. We also study the board game Othello, a common testbed for evaluating the world models
309
      of sequence models (Li et al., 2023; Nanda et al., 2023b; Hazineh et al., 2023; Vafa et al., 2024).
      The game consists of two players taking turns placing tiles on an 8x8 board. Each game of Othello is
310
      tokenized into a sequence of at most 60 moves, where each token indicates which of the 60 squares
311
      the most recent tile was placed on (the middle four tiles are always occupied). The true state space
312   Φ corresponds to all 8x8 boards and the mapping ϕ converts game sequences into states. Following
313   Li et al. (2023), we consider two different sequence generating processes: championship, which
314   corresponds to true gameplay from Othello championships, and synthetic, which corresponds to
315   synthetic games generated randomly where each move is sampled uniformly at random from the set
316   of valid moves. We randomly split each dataset into train and hold-out sets. Our training sets contain
317   7.9M tokens for championship and 60M tokens for synthetic, along with 6K hold-out tokens.
318   Models. We study the world model learning properties for five classes of pretrained sequence
319   models: RNNs (Elman, 1990), LSTMs (Hochreiter, 1997), transformers (Vaswani et al., 2017),
320   Mamba (Gu & Dao, 2023), and Mamba-2 (Dao & Gu, 2024). We use the same number of layers
321   and embedding dimensions for each model so each model has approximately 20M parameters. The
322   only exception is that we find that smaller LSTM and RNN models perform better when trained on
323   lattices, so we use 2-layer models for LSTMs and RNNs for the lattice example. See Appendix A for

360   4.1   Inductive bias metrics
361
362   We now use the metrics described in Section 2 to assess whether these models have inductive bias
363
      toward state. Our metrics involve transferring each model to small datasets of randomly generated
      outputs and assessing how related each model’s extrapolation patterns are to the true state. The
364
      inductive bias (IB) test aims to predict a model’s extrapolations from the true state using a shared
365
      representation across datasets, while the state recovery (SR) test aims to predict the original state
366   from the shared representation. Each metric we report is a normalized prediction accuracy so that
367   0 corresponds to a model with as good predictive performance as a baseline model that makes the
368   same prediction for all inputs and 1 corresponds to perfect prediction (e.g. of state or of extrapolation
369   pattern). For the inductive bias measure, this is held-out R2 , and for the state recovery measure, it’s
370   1 minus normalized cross entropy. Appendix B contains more details about how we implement the
371   metrics across datasets and Appendix C contains more ablations.
372   The results are depicted in Table 1. For the lattice example, almost all inductive biases toward state
373   are close to 1, reaching as high as 0.996 for the LSTM pretrained on state. However, it’s not just the
374   models pretrained on state that achieve high inductive bias toward state; the transformer pretrained
375   on next-token prediction has an inductive bias toward state of 0.971. Similarly, all models achieve
376   near perfect state recovery. This shows that not only does the inductive bias of these learners contain
377   information about state; it contains all information about state. These results show that our metrics
      are not unachievable — models can perform well on them.

456   4.3   Bundles of Heuristics
457
458   The results in this section show that models can perform quite well on pretraining objectives yet have
459
      low inductive bias toward state (as measured by our metrics and transfer properties). Here, we try to
      make sense of this discrepancy. Specifically, we explore the hypothesis that these models are relying
460
      on “bundles of heuristics” (Karvonen et al., 2024); functions of the input that lead them to perform
461
      well on next-token prediction yet deliver poor inductive biases toward state.
462
463   We begin by fine-tuning models pretrained on next-token prediction on Othello to predict the true state
464   (i.e. the board) of each position in the subsequence. Throughout fine-tuning, we reconstruct the fine-
465
      tuned model’s predicted board for each sequence on held-out data, and record two kinds of metrics.
      The first is whether the predicted board exactly matches the true board. For the second, we measure
466
      how well the set of valid moves in the predicted board matches the set of valid moves in the true board.
467
      This is motivated by the fact that even if a predicted board is incorrect, it can still have the same set
468   of valid legal moves. The results are depicted in Figure 2. They point to an intriguing phenomenon:
469   even when the predicted board is incorrect, the set of legal moves from the predicted board tends
470   to match the set of legal moves from the true board. These results show that although learners may
471   not carry information about the full board, they carry enough information about the board to perform
472   well at next-token prediction. These findings carry implications for metric design and illustrate a
473   broader principle: models can learn representations that satisfy training objectives without capturing
474   complete world models. This phenomenon parallels observations in LLMs, where models can
475   sometimes answer questions correctly without demonstrating deeper conceptual understanding (Vafa
476
      et al., 2024). Future work should investigate similar patterns in domains like physics and navigation.
477   To further explore this hypothesis, we calculate a variant of our inductive bias metric. Our original
478   inductive bias metric measures how well the extrapolations of a learner can be predicted by a
479   shared function of state (State IB). We calculate another metric which measures how well these
480   extrapolations can be predicted by a shared function of the input sequence. If a shared function of
481   input sequence can predict extrapolations across datasets while a shared function of state cannot, it
482
      suggests that a learner is using the same heuristic to guide its extrapolations. We refer to this metric
      as Heuristic IB and calculate it analogously to State IB (we provide more details in Appendix B).
483
484   The results are depicted in Table 3, along with difference in state and heuristic inductive biases (larger
485   implies a larger inductive bias toward heuristics). For almost all models, the heuristic inductive bias
      is larger than the state inductive bias. For the transformer and Mamba models, the difference


                                                             9
      Under review as a conference paper at ICLR 2025


486
      between heuristic and state inductive biases is larger for models pretrained on next-token prediction
487   than models pretrained on oracle state information. The fact that these differences are smaller for
488   state-pretrained models shows that heuristic avoidance is possible for these architectures. However,
489   their next token pretraining encourages them to rely on bundles of heuristics.

492   5   Related Work
493
494   One strand of world model research studies whether the outputs of a fixed model accord with a
495   known world model by studying the fixed model’s outputs (Vafa et al., 2024). For example, one way
496   that Toshniwal et al. (2022) and Li et al. (2023) study world models is by assessing whether a model
497   trained on sequential game data always predicts legal moves in the underlying game. The question
498
      we study is a different yet related question: rather than studying the world model properties of a fixed
      model, we study what it means to test if a learner has a world model embodied in it. This framework
499
      could be used, for example, to study how large language models perform in few-shot learning.
500
501   Another strand of the literature assesses whether a model’s representations encode world models
502   without directly studying learning properties. For example, a common method uses probes to assess
503   whether an intermediate representation used by a neural network is predictive of state (Hewitt &
504
      Liang, 2019; Li et al., 2021; Abdou et al., 2021; Jin & Rinard, 2023; Li et al., 2023). However, there
      are open questions about the reliability of probes (Belinkov, 2022), such as appropriate function
505
      complexity (Alain & Bengio, 2018; Cao et al., 2021; Li et al., 2023). Our method sidesteps these
506
      issues by asking how a model learns, rather than what’s encoded in its fixed representations.
507
508   The methods in this paper are also related to the study of mechanistic interpretability of ML models
509   (Nanda et al., 2023a; Cunningham et al., 2023; Bereska & Gavves, 2024). Closely related to us,
510
      Karvonen et al. (2024) find that a GPT model trained on Othello performs internal computations corre-
      sponding to “bags of heuristics” rather than a coherent world model. Our procedures differ in aim be-
511
      cause 1) we study the world model capabilities of a learner rather than of a fixed model and 2) we do not
512
      seek to understand the internal mechanisms governing world model recovery. However, these findings
513   support our analysis of the Othello model relying on heuristics, rather than state, as its inductive bias.
514
515   Our examples with orbital mechanics also relates to the large body of work studying AI and physics
516
      (Hao et al., 2022; Wu & Tegmark, 2019). The example we study is most closely related to works
      studying whether AI models can uncover physical laws (Chen et al., 2022; Belyshev et al., 2024).
517
      We adapt tools from this literature — such as using symbolic regressions to evaluate AI models —
518
      to study the inductive biases of transfer learners (Liu & Tegmark, 2021; Wu & Tegmark, 2019).
519
520   This paper studies the problem of evaluating whether the world models of learning algorithms reflect
521   the world models of the real world. There is also a literature on world models in reinforcement
522
      learning (RL); while these literatures use similar terms, the goals are distinct. In RL, world models
      refer to representations (or even the specific neural network) learned by an agent, and their quality is
523
      typically measured by how well they perform policy optimization (Ha & Schmidhuber, 2018; Chen
524
      et al., 2024). In contrast, the LLM literature focuses on evaluation: a learning algorithm is evaluated
525   by how well it can recover an externally defined mapping from a true world model.
526
527
528   6   Conclusion
529
530   In this paper, we developed a framework for evaluating whether learning algorithms develop world
531   models by measuring their inductive bias toward state when transferring to new tasks. Our results
532   across applications reveal that while many models perform well on next-token prediction, they can
533   have limited inductive bias toward state and poor transfer properties to new state-based tasks. This
534   suggests that these models may be relying on bundles of heuristics rather than coherent world models.
535   As described in Section 2, our metrics depend on the implementation of the multitask learner used to
536   model extrapolations and recover state. While we use the same multitask learner for all models within
537   each task to ensure fair comparison, the measures may be sensitive to how this learner is implemented.
538   However, we find similar performance across the different kinds of multitask learners we consider
539   (Appendix C) and for a procedure based on correlation matrices (Appendix G). Further work should
      prioritize studying the most effective multitask learners for measuring these inductive biases.

679            12 layers, 16 attention heads, and 512 hidden dimensions. We also modify the transformer
680            so that it can take as input continuous data. Instead of using an embedding lookup table as
681            the first layer, we use a multi-layer perceptron to transform coordinates in Euclidean space
682            to 512-dimensional embeddings.
683
             • Mamba (Gu & Dao, 2023): We first encode inputs with a 512-dimension embedding layer.
684            We then pass inputs through 16 Mamba layers (analogous to 8 layers in a transformer due
685            to how Mamba layers are defined). We use 512 embedding dimensions, 16 for the SSM
686            state expansion factor, 2 for the block expansion factor, and 4 for the convolutional width.
687
             • Mamba-2 (Dao & Gu, 2024): We use the same architecture as for Mamba except the
688
               mixer in each block is a Mamba-2 module. We use the same specifications as well: 512
689
               embedding dimensions, an SSM state expansion factor of 16, a block expansion factor of 2,
690            and a convolutional width of 4.
691
692   We use Adam (Kingma & Ba, 2014) to optimize each model. We use a learning rate of 6e-4 and
693   decay the learning rate with with 2000 warmup iterations. We use weight decay of 0.1 and gradient
694   clipping at 1 for each model.
695
      When we pretrain models on next-token prediction, we include a head to predict next tokens (tying its
696   parameter weights to the initial embedding layer parameters). When we fine-tune to predict functions
697   of state, we discard the next-token head and randomly initialize a state head. How we predict state
698   depends on the type of state for each problem:
699
700          • Lattice: For the lattice problems, the state corresponds to a categorical variable between 1
701            and the number of states. We include a state prediction head that forms logits for each state
               and minimize cross-entropy loss.

703            represent the board as a 64-dimensional vector (corresponding to an 8x8 grid), where each
704            value takes on one of 3 categorical values (white, black, or unnoccupied). We predict this
705            state using a state prediction head that forms 64x3 logits and minimizing cross-entropy loss
706            summed across all board positions. When we transfer to functions of state, all functions use

708            we use a state prediction head forming two logits.

      B    Metric Implementation Details
717
718   Lattice. For the lattice example, we create 5 datasets of 500 new examples, D1 , . . . , D5 . For each
719   dataset, we sample sequences uniformly at random among the set of data points and sample outputs
720   from a Bernoulli(0.5) random variable. In our construction we make sure that any two sequences
721
      with the same state are mapped to the same output variable. We then fine-tune a model separately
      for each dataset, resulting in five fine-tuned models m̂(·; D1 ), . . . , m̂(·; D5 ). We then calculate the
722
      associated prediction functions across all inputs xi from the original training dataset, resulting in
723
      new datasets of the form {(xi , m̂(xi ; D1 )}, . . . , {(xi , m̂(xi ; D5 )}.
724
725   For the inductive bias test, we train a model to learn a representation the jointly predicts
726   (m̂(xi ; D1 ), . . . , m̂(xi ; D1 )) from the true state ϕ(xi ). Since each state is a different categori-
727
      cal variable, the neural network begins with an embedding layer, followed by L feedforward layers
      with H hidden dimensions and a ReLU nonlinearity (we find the best performance for 3 layers and 64
728
      hidden dimensions). The last layer of the neural network uses a linear transformation to predict the 5
729
      outputs simultaneously. We perform l1 penalization on the penultimate representation to encourage
730   sparsity. We consider l1 penalty values among [0.0, 0.0001, 0.1, 1.0] and choose the penalty with
731   the best validation loss. Since m̂(xi ; Dj ) is real-valued (corresponding to the predicted probability
732   of the binary output variable), we train this model for 5000 iterations using a batch size of 600 to
733   minimize the mean-squared error and report the held-out R2 .
734
735
      To perform the state reconstruction test, we predict the original state ϕ̂(xi ) from the penultimate
      representation of the network used for the inductive bias test. We perform this prediction by training
736
      a feedforward neural network trained to perform multiclass classification (since each original state is
737
      a single class). We find that 2 layers and 512 dimensions does best in practice. We train the model
738   for 5000 iterations with a batch size of 600 to minimize the cross entropy between the predicted and
739   true states. As a baseline, we consider the cross-entropy of a model that always predicts the same
740   value for each class, and report the difference between the baseline model and the model that uses
741   the representation.
742
      Othello. Our procedure for Othello follows the same steps as for the lattice example, except we
743
      perform adjustments to account for the fact that Othello’s state is a 64-dimensional board instead of
744   a single categorical variable. We begin by transferring each model to a dataset of 5 randomly chosen
745   inputs xi and 5 randomly chosen Binary outputs for each x. We perform this transfer exercise for 1000
746   iterations for 10 seeds, giving us new datasets of the form {(xi , m̂(xi ; D1 )}, . . . , {(xi , m̂(xi ; D10 )}.
747
      The implementation of the inductive bias test is the same as for the lattice example except we modify
748
      the neural network to account for the fact that our input (state) is an Othello board. Instead of using
749

752   or no tile has been placed on the corresponding square. The first layer of the network begins with
753   an embedding layer, followed by convolutional layers. We follow the convolutional layers with two
754   feedforward layers to predict the output. We again perform l1 penalty on the final layer to encourage
755   sparsity considering the same values as for the lattice example. We find that two convolutional layers,

922   G     Correlation-based Metrics
923
924
      Here we implement an additional procedure for estimating the inductive bias and partial reconstruc-
      tion metrics from Section 2.2. This procedure begins by following the basic setup in Section 2.2:
925
      we construct synthetic datasets that obey the state representation, and apply the learning algorithm.
926
      However, instead of estimating inductive bias and state recovery with a multitask learner, we estimate
927   them nonparametrically.
928
929   Specifically, after applying the learning algorithm across the J datasets, we collect the extrapolations
930
      {xi , m̂(xi , Dj )} for some set of held-out points x1 , . . . , xn that are shared across datasets. The
      inputs x1 , . . . , xn should satisfy the following two properties:
931
932         1. There exists an i ̸= j such that xi and xj have the same state, i.e. ϕ(xi ) = ϕ(xj ).
933
            2. There exists an i′ ̸= j ′ such that xi′ and xj ′ have different states, i.e. ϕ(xi′ ) ̸= ϕ(xj ′ ).
934
935   We then construct the n × n correlation matrix Σ such that Σi,j = corr(m̂(xi , D), m̂(xj , D)), where
936   m̂(xi , D) = (m̂(xi , D1 ), . . . , m̂(xi , DJ )) is the vector of extrapolations across datasets. Intuitively,
937   Σi,j describes how similar the extrapolations are for datapoints xi and xj ; how predictable is xj ’s
938   extrapolation from that of xi ’s?
939
      If a learner respects state, its extrapolations for two points in the same state will be perfectly correlated.
940   Similarly, if a learner is fully reconstructing state, it will have zero correlation for pairs of points that
941   are not in the same state. Therefore, to estimate inductive bias, we compute the average correlation
942   between points that have the same state:
943
                                            E[Σi,j |ϕ(xi ) = ϕ(xj ), i ̸= j].                                      (10)
944
945   A value of 1 implies perfect inductive bias toward state. Similarly, estimating state recovery involves
946   computing the average absolute correlation between points that don’t have the same state:
947                                            E[|Σi,j ||ϕ(xi ) ̸= ϕ(xj )].                                        (11)
948
      Nonzero values mean that a learner is extrapolating based on only partial functions of state. Our
949
      definition of state recovery in Section 2 would involve negating Equation (11) so that higher values
950
      are better. However, because Equation (10) and Equation (11) are directly comparable, it is easier to
951   compare them without negating Equation (11). Instead we report Equation (11) directly and refer to
952   it as state coarseness (SC).
953
954
      Because correlation-based measures of inductive bias and state coarseness are directly comparable,
      we additionally report the ratio of the two values: IB/SC. The ratio summarizes how much more
955
      correlated extrapolations of same-state pairs are than different-state pairs. Larger values mean larger
956
      levels of same-state correlation relative to different-state.
957
