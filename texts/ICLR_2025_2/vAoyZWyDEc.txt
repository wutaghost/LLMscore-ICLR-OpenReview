009                                               A BSTRACT
010
011            We study the computability of approximating optima of non-convex functions.
012            We give a simple proof to show that the problem of finding the optimal value (and
013            optimal point) or its approximation is not even computable in the oracle setting.
014            We also give a property a function has to satisfy if its global optima can be ap-
015            proximated. Next we give an example of such a global property we call basin of
016            attraction. Then we give a simple algorithm which converges to the global optima
017            when this is known. Finally, we give some numerical results.
018

      1   I NTRODUCTION AND P RELIMINARIES
021
      The problem of computing the global minima of a non-convex continuous function f : C ‚Üí R,
022
      where C ‚äā Rd is a closed, compact subset is well studied (Horst & Tuy). . Global minima is the
023
      point x‚ąó ‚ąą C which satisfies the following property: f (x‚ąó ) ‚Č§ f (x) for all x ‚ąą C. The function f
024   attains this minimum at least once by extreme value theorem. Our goal is to find one such point.
025
026   We note that we consider an oracle setting, where the function values are given by an oracle. This is
027
      different from the computablity of optima computable real functions studied for example in (Pour-El
      & Richards, 1989). We would like to point to the fact that our oracle setting is more general than one
028
      considered for example in (Lee et al., 2023) and suitable for any non-convex optimization problem.
029
      In their setting, it is shown that the optimal value may be computable but the optimizer (optimal
030   point) is not computable. We show that neither the optimizer nor the optimal value is computable in
031   our setting.
032
033
      The problem of non-convex minimization can be shown to be NP-hard by reducing the NP-complete
      subset-sum problem to a non-convex optimization problem (Murty & Kabadi, 1987). Let us define
034
      S = {x|, |f (x) ‚ąí f (x‚ąó )| ‚Č§ ŌĶ} and term the members of S as ŌĶ-optima. In optimization literature
035
      (Foster et al., 2019; Zhang et al., 2020), it is also known that finding approximations to optima is not
036   tractable for non-convex functions. For non-convex functions, ŌĶ-stationary point which is weaker
037   than ŌĶ-optima is also known to be not tractable (Zhang et al., 2020). We show more in this paper,
038   that this set S is not computable. This is much stronger than saying they it is intractable.
039
      It is easy to see that a simple grid search will output a sequence of points converging to the global
040
      optima. And for a Lipschitz continuous function, it requires an exponential number of oracle calls
041
      (Nesterov) if the Lipschitz constant is known. We show that Lipschitz constant is an example of
042   a property a function must satisfy if its approximate optima is computable. We also see that if the
043   Lipschitz constant is not known the (approximate) optima is not computable.
044
045
      Our main contributions in this paper are:
046
            1. An oracle setting which is applicable to many real-world optimization problems and differ-
047            ent from the computable real-functions setting studied in computable analysis (Pour-El &
048            Richards, 1989).
049
            2. In the oracle setting we consider, the problem of approximating both the minimal value and
050
               the minimizer of non-convex functions or their approximations are not computable.
051
052
            3. A simple property the function has to satisfy if its global optima can be approximated.
053         4. We give an example of global optima property-basin of attraction. And if this is known, we
               give an algorithm which converges to the global optima.

109
110   We assume there is an oracle for our continuous function f . This oracle gives the value f (x) up to

      1.4.1    S UPERVISED L EARNING
144
145   In supervised learning algorithms including Neural Networks (NN) there is a loss or error function
146   that needs to be minimized. In real-life problems like image classification, we have a rugged func-
147   tion, having many local optima. Hence simple methods like gradient search can not find the global
148   optima. And many state-of-the-art NN architectures like CNN‚Äôs still use the backpropagation algo-
149   rithm which is based on gradient search and can not converge computably to the global minima as

152   We are given images x, x ‚ąą DT of the digits 0 to 9. The image x is itself a matrix M of individual

161   labels of these images in the training set, i.e., C(x) = y(x), x ‚ąą DT . And then using probability by

299   4   A N A LGORITHM WITH K NOWN BASIN OF ATTRACTION
300
301   In this section, we also assume the function f to be differentiable. Let us denote the gradient by
302   ‚ĖĹf (x). The algorithm takes as input the lower bound m on the basin of attraction of the global
303   minimizer. By basin of attraction we mean the following: if we let the initial point to be in the
304   hypercube of length m in all co-ordinates, i.e., in the basin of attraction around the global minima
305   Bm (x‚ąó ) then the gradient descent algorithm will converge to the minima. This is another example
306   of a global optima property (independent of the last argument),
307
                          Q(m, x, ¬∑) := If x ‚ąą Bm (x‚ąó ) and x Őł= x‚ąó then ‚ĖĹf (x) Őł= 0.
308
309   Note that there exists a m such that Q(m, x, ¬∑) is True. If this basin of attraction size m can be
310   computed or its lower bound known then we can compute the global minima (Theorem 3.4) to any
311   approximation. In this section, we given an algorithm which converges to the global minima if this
312   basin length m or its lower bound is known.
313   The algorithm finds the point zk where the function takes a minimum amongst all points at a distance
314   of m from each other and does a gradient descent step from the point zk . The algorithm outputs an
315   ŌĶ-approximation to the global optima (see Theorem 5.4 for the value of ŌĶ). In this algorithm for
316   simplicity, we do not consider line searches and use constant step-size t > 0. The figure 1 shows
317   the gradient descent step taken at the point which has the minimum function value amongst all the
318   points in the grid.
319   We note the similarity of our algorithm with the one considered in paper (D‚ÄôHelon et al., 2007),
320   where the basin of attraction of global minimizer is first found by searching then a gradient descent
321   is performed. In our algorithm, these two steps are interleaved. The major issue with their algorithm
322   is that they assume the value of the global minima is known which they assume to be zero. But
323   this need not be known in real-world problems. This assumption is not needed with our approach.
      Moreover, we have formally shown the convergence of our algorithm.

534
535   We have proved that there is no algorithm with only oracle access to the function to compute a ŌĶ‚ąí
536   approximation to the optimizer (and optimal value) of a nonconvex function f . This result holds
537   even if the function has higher-order derivatives. We have given a necessary and sufficient property
538   a function has to satisfy if its global minima is computable. Then, we give basin of attraction as an
539   example of this property and an algorithm which approximates the global minima with this property.
      Finally, some numerical results were presented.
