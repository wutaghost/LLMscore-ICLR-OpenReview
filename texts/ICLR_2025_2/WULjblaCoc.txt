008
009
010
               Large language models based on the transformer architecture can solve highly
               complex tasks. But are there simple tasks that such models cannot solve? Here
011
               we focus on very simple counting tasks, that involve counting how many times to-
012
               kens in the vocabulary appeared in a string. We show that if the dimension of the
013            transformer state is linear in the context length, this task can be solved. However,
014            the solution we propose does not scale beyond this limit, and we provide theo-
015            retical arguments for why it is likely impossible for a size-limited transformer to
016            implement this task. Our empirical results demonstrate the same phase-transition
017            in performance, as anticipated by the theoretical argument. Our results demon-
018            strate the importance of understanding how transformers can solve simple tasks.

021   1   I NTRODUCTION
022
023   Large language models (LLMs) have demonstrated striking performance on a wide array of tasks,
024   from creative writing to solving complex math problems. Given these successes, a key question
025   arises: what can these models do, and just as importantly what can they not do. There are multiple
026   ways to address this question of expressiveness of LLMs. First, it can be studied empirically by
027
      probing LLMs and seeking relatively simple tasks that they cannot perform successfully. Indeed re-
      cent work has found several such tasks, including “needle in haystack” (Kamradt, 2024; Ivgi et al.,
028
      2023) as well as extrapolating tasks to longer sequences (Levy et al., 2024). A second, comple-
029
      mentary, approach is theoretical studies which chart the computational capabilities of transformers
030   (Sanford et al., 2023; Wei et al., 2022a).
031
032   In the current work, we focus on a simple task that transformers often struggle with, and analyze
033
      it theoretically. Specifically, we consider a very simple “Query Counting” task, defined as follows.
      The model is presented with a sequence of tokens, and is then asked how many times a given query
034
      token appears in the sequence. For example:
035
036            Consider the sequence a a b b a c c d a. How many times does the letter “a”
037            appear in the sequence?
038
039   Our interest in this problem is that it seems like a basic module needed for working with long input
040   streams. Indeed, there is a long line of work on sketching and streaming, that studies similar tasks
041   (Alon et al., 1996; Cohen, 2014). Furthermore, this task can be viewed as an extension of the
042   “needle in haystack” problem which has recently garnered much attention, since it was shown that
043   most LLMs struggle with it for long context. In the needle in haystack problem, the goal is to find a
044
      specific string in a long text. In our counting problem, the model is tasked with counting how many
      times a given string has appeared, which is a harder task than finding one appearance.
045
046   There is, however, a key difference between the needle in haystack problem and the counting prob-
047   lem we consider above. The needle in haystack problem is clearly solvable by transformers, re-
048   gardless of the context length. This is because detecting a similar token and extracting it (or nearby
049   tokens) is a simple task for single attention head (e.g., using induction heads Olsson et al., 2022).
050
      On the other hand, for the counting problem, as we argue here, it is much less clear that transformers
      can solve it for arbitrary context length. Note that when we write that transformers cannot perform a
051
      task, we are referring to transformers whose number of parameters is not dependent on context size.
052
053   Modern LLMs indeed struggle with counting tasks (e.g., see Section 6 and (Barbero et al., 2024)).
      Of course if these models can use code, the task becomes easy, but our focus is on understanding the

054
      capabilities of the transformer architecture itself. Specifically, we ask whether transformers have an
055   architectural limitation related to the counting task.
056
057   We next turn to ask when transformers can count. We show a construction that works as long as the
058
      transformer embedding size d is greater than the vocabulary size m. Our construction uses one-hot
      embeddings, or more generally orthogonal embeddings, of the vocabulary, which allow the model
059
      to maintain a histogram of counts of tokens previously observed. When d < m this orthogonal
060
      construction is no longer possible. A natural approach is to consider the “most orthogonal possible”
061   embedding (a notion formalized in Welch bounds (Welch, 1974)), and try to use it in a similar
062   scheme. However, we show that this does not allow implementing the histogram solution if the
063   dimension is smaller than the vocabulary.
064
      The above discussion suggests that in the d < m regime,1 the naive histogram approach does not
065
      seem to work. Our study of this regime reveals both a positive and a negative result. On the positive
066
      side, we show that there does exist a construction that allows counting, which can be done with
067   a single transformer layer. On the negative side, we prove that this construction requires an MLP
068   width that grows with context size, meaning it is not applicable to arbitrarily long contexts. Indeed,
069   when training transformers on these counting tasks, we find that they fail in the d < m regime.
070
      We next study a somewhat more complex counting task, which we refer to as “Most Frequent El-
071
      ement”. Here we present the model with a sequence of tokens, and ask for the count of the most
072
      frequent token. This is the same as taking the maximum of the histogram of the counts. Similarly to
073   the Query Counting task, we show that in this case, a solution exists for d > m based on an orthog-
074   onal construction. However, for d < m we show, using a communication complexity argument, that
075   no solution exists for a one layer transformer. Thus again we obtain a phase transition for counting
076   at d = m.
077
      Taken together, our results reveal an interesting landscape for simple counting task, where the d = m
078
      threshold separates between transformers that can count and those that cannot.
079
080   Our results highlight the importance of studying basic counting problems, and their dependence on
081   vocabulary size. They also point to limits of solving seemingly simple problems using transformers,
082
      and further emphasize the advantages of using code as a tool to sidestep these issues.

085   2       R ELATED W ORK
086
087   Since the introduction of transformer architectures (Vaswani et al., 2017) and the success of LLMs,
088   there has been much work on evaluating their performance on various tasks (e.g., see Srivastava
089   et al., 2023). In particular, much recent work has explored performance on long context tasks, the
090   dependence of accuracy on context length (Levy et al., 2024), and the ability of models to extrapolate
091   to lengths beyond those seen at training (Anil et al., 2022).
092   The fact that models often do not perform well on these evaluations has prompted works that try to
093   pinpoint the inherent limitations of transformer models. One line of work is to use computational
094   complexity lower bound approaches to show that transformers of a certain size and architecture
095   cannot implement a given function of interest. For example, Sanford et al. (2023) show that certain
096   functions cannot be implemented without the transformer size scaling with input size. A similar
097   limitation is also shown in (Peng et al., 2024) for certain compositions of functions.
098   A related line of work is to relate transformers to known complexity classes. For example it has been
099   shown that transformers can efficiently approximate Turing machines (Wei et al., 2022a), and that
100   transformers with bounded percision can only solve problems in uniform T C 0 (Merrill and Sabhar-
101   wal, 2023). Chain-of-Thought (Wei et al., 2022b) has also been analyzed from an expressiveness
102   viewpoint, demonstrating it can substantially improve the expressive power of transformers (Feng
103   et al., 2024).
104   Our focus here is not on the general capabilities of transformers but rather on a specific, seemingly
105   simple problem, and on the ability of transformers to solve it.

      4       A NALYZING Q UERY C OUNT (QC)
163
164   In this section we focus on the QC problem, and ask which transformer architectures can implement
165   it successfully. We first remark on a general limitation of transformers without positional embed-
166   dings for the counting problems we consider.2 We next show in Section 4.2 that if d > 2m a
167   one-head one-layer transformer can implement QC. We refer to this as the histogram solution. We
168   then show that the histogram solution stops working if d < m. The natural question is then whether
169   there are other solutions for the d < m case. We argue that in this case, solutions are likely to require
170   calculating the function 1/x, and show that this function would require an MLP layer of width n.
171   This means we cannot expect the transformer to extrapolate to long context sizes, and therefore a
172
      one-layer transformer is unlikely to be able to implement QC, at least using the two most natural
      solutions.
173
174
175
      4.1       T HE N EED FOR P OSITIONAL E MBEDDINGS
176
      Transformers use self attention to average over previous representations. The fact that they aver-
177   age, rather than sum, leads to an interesting limitation on their ability to count. Specifically, it
178   is easy to see that for variable context size, they cannot perform any counting task without the
179   use of positional embedding. Consider the QC task and an input sequence S1 = x1 , . . . , xn ,
180   where the goal is to return the count of xn in the sequence. Now consider the length 2n sequence
181   S2 = x1 , . . . , xn , x1 , . . . , xn . The correct output for this sequence is twice the correct output for
182   S1 . However, a transformer without positional embeddings that is applied to S1 will have exactly
183   the same output as the one for S2 . This follows because averaging is invariant to input duplication.
184   The above restriction no longer holds when positional embeddings are used, and it is easy to see
185   that it can be rectified with even a simple positional embedding that just signifies the last position
186   (see our construction in Section 4.4). This implies that if a transformer has access to the legnth of
187   the sequence, it may make it easier to count. Another thing to note is that while the above difficulty
188   arises for counting, it does not arise if we are interested in calculating proportions (e.g., what is the
189   fraction of the items of the sequence that are equal to xn ).

275   4.4     T HE C OUNTATTEND SOLUTION FOR ALL d AND m
276
277   In the previous section we considered a histogram based solution, which required d > m. Here
278   we provide an alternative approach to solve the counting problem which works for any d and m.
279   However, as we shall see, this solution requires a large MLP, that must scale with the length of the
280   input n. As a result, a transformer with a fixed MLP size will not be able to learn such a solution
281
      that will work with arbitrary n values.
282   We first present a high level description of this construction, which uses a 1-layer transformer with
283   a single head. The idea explicitly uses attention to count (hence the name CountAttend) as follows
284   (see also Figure 1b). Assume that the query token is xn = 4, so that we are seeking the number
285   of elements in x1 , . . . , xn that are equal to 4, and assume that the number of these elements is 7.
286   Then the token xn can attend to all other tokens, such that attention weight is high for all i such that
287
      xi = 4 (and same for all these) and near-zero otherwise. Thus, the attention weight will be 71 for all
      the xi = 4 tokens, including xn . We next need to extract this number and invert it, in order to get
288
      the answer 7.
289
290   Extracting the attention weight can be done by using a single coordinate in the positional embedding
291   that is one for position n and zero otherwise. The value aggregation of self-attention can then extract
292   the number 17 in this coordinate. Finally, to invert this number we need an MLP to implement the
293
      function 1/x. If the smallest number we need to invert is 1/n then this can be done with an MLP
      with 4n neurons.
294
295   The following proposition, proved in Appendix A, summarizes the properties of this construction.
296   Proposition 4.3. For any d, m, n there exists a transformer that solves the corresponding QC prob-
297   lem. The transformer has one layer, one attention head, dimension d, and an MLP of size O(n).
298   Furthermore, its matrix K ⊤ Q is diagonal with elements of magnitude O(log n).
299

344   5     A NALYZING M OST F REQUENT E LEMENT
345
346   In this section we consider the task of finding the most frequent element (MFE) in a given sequence
347
      of tokens. This problem is very much related to the counting problem, as intuitively it requires to
      count every token separately and compute the token that appears the maximum number of times. We
348
      show that there are strict bounds on the size of the embedding compared to the size of the dictionary,
349
      in order for transformers to be able to perform this task.
350
351
      5.1   MFE M ODELS M UST HAVE d ≥ Ω(m)
352
353   We begin by showing a lower bound on the required size of a 1-layer transformer solving the
354   MFE task. The following result establishes that MFE can be implemented only when dhp =
355   Ω(min{m, n}). This means that at a fixed precision p and if n > m, the dimension d must grow
356   linearly in the vocabolary size in order to implement MFE. This is the same trend that we saw for
357   the QC problem.
358   Theorem 5.1. Suppose that there is a 1-layer transformer with h heads, embedding dimension d,
359   and p bits of precision, followed by an MLP of arbitrary width and depth, that solves the MFE
360   task for sequences of length n. Then, we must have that dhp ≥ Ω(min{m, n}), where m is the
361   vocabulary size.
362
363
      The full proof can be found in Appendix B. The proof uses a communication complexity argument
      that is inspired by a lower bound of Sanford et al. (2023). This lower bound implies that transformers
364
      which solve the MFE task need to have an embedding size that scales with the size of the dictionary,
365
      or have many attention heads. Note that increasing the size of the MLP which follows the attention
366   cannot break the lower bound.
367
368
      5.2   MFE C AN BE I MPLEMENTED WHEN d = O(m)
369
370   The previous result showed that MFE cannot be implemented with a one layer transformer when d is
371   smaller than m. Here we show that it is possible to implement MFE when d = O(m). This implies
372   that d = O(m) is tight for the MFE problem. The result is described below.
373   Theorem 5.2. There exists a 1-layer transformer that solves the MFE task for sequences of size n
374   and dictionary size m, where the parameters d, h, p are equal to: d = m, h = 1, p = log(n), and
375   the MLP has d2 neurons.
376
377   The construction is again based on the histogram approach. Because d > m, one can compute the
      histogram of counts in the last position (as for QC). The only part left to be done is to extract the


                                                        7
      Under review as a conference paper at ICLR 2025


378
      maximum from the histogram, which can be done via a one layer MLP with m2 units (each unit
379   performs a maximum between two distinct elements in the histogram).
380
381   One limitation of the above result is that it requires an MLP that grows with m. This can be avoided
382
      if using two layers of attention. A two layer implementation is simple: use the first layer to calculate
      the “Query Count” for each element, and then use softmax to calculate the maximum over tokens.
383
      This construction does not need an MLP at all. Another option is to use a depth log(m) MLP with m
384
      neurons at each layer to calculate the maximum from the histogram (see Safran et al., 2024), how-
385   ever having depth which relies even logarithmically on the dictionary size is infeasible for practical
386   implementations.
387
388
      To summarize the above results, we have shown that MFE cannot be implemented by a one layer
      transformer if d < m, and that if d > m MFE can be implemented either with a one layer trans-
389
      former with wide MLP, or a two layer transformer without an MLP.

464
465
      We focus on the basic task of counting using a transformer architecture. When the dimension of
      the model is sufficiently large, we show that this task can be easily implemented by letting the
466
      transformer calculate the histogram of the input sequence. For smaller dimensions, we provide
467
      theoretical support suggesting that a one layer transformer cannot implement this function. Our
468   empirical results support this phase transition.
469
470   Understanding such limitations of transformers are key to developing new architectures. For exam-
471
      ple, our results show that in a sense it would be impossible to have transformers count arbitrarily
      well and for long contexts, without increasing the architecture size considerably. Concretely, this
472
      suggests that for counting tasks it may be important to delegate to tools (Schick et al., 2024) such as
473
      code execution that do not have the same limitations.
