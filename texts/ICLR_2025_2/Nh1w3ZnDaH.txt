009                                               A BSTRACT
010
011            Unpaired point cloud completion explores methods for learning a completion map
012            from unpaired incomplete and complete point cloud data. In this paper, we propose
013            a novel approach for unpaired point cloud completion using the unbalanced optimal
014            transport map, called Unbalanced Optimal Transport Map for Unpaired Point
015            Cloud Completion (UOT-UPC). We demonstrate that the unpaired point cloud
016            completion can be naturally interpreted as the Optimal Transport (OT) problem and
017            introduce the Unbalanced Optimal Transport (UOT) approach to address the class
018
               imbalance problem, which is prevalent in unpaired point cloud completion datasets.
               Moreover, we analyze the appropriate cost function for unpaired completion tasks.
019
               This analysis shows that the InfoCD cost function is particularly well-suited for
020
               this task. Our model is the first attempt to leverage UOT for unpaired point cloud
021            completion, achieving competitive or superior results on both single-category and
022            multi-category datasets. In particular, our model is especially effective in scenarios
023            with class imbalance, where the proportions of categories are different between the
024            incomplete and complete point cloud datasets.

027   1    I NTRODUCTION
028
029   The three-dimensional (3D) point cloud is a fundamental representation of 3D geometry processing
030   (Guo et al., 2020). However, obtaining complete point cloud data is challenging because of the
031   limitations of the scanning process (Yuan et al., 2018). In this respect, many methods have been
032   proposed for point cloud completion, which aims to recover a complete point cloud from incomplete
033
      (partial) data (Yu et al., 2021; Wang et al., 2022; Tchapmi et al., 2019; Chen et al., 2020; Hong
      et al., 2023). These previous approaches can be categorized into paired (supervised) and unpaired
034
      (unsupervised) methods. In the paired approach, the completion model is trained using paired data,
035
      which consists of incomplete point clouds and their corresponding completions (Yu et al., 2021;
036   Wang et al., 2022; Tchapmi et al., 2019; Xia et al., 2021; Zhou et al., 2021). However, acquiring this
037   paired training data is often difficult in practice. Therefore, the unpaired point cloud completion aims
038   to train a completion model from the independently sampled incomplete and complete point clouds,
039   leveraging shared semantic information, such as object class (Ma et al., 2023; Chen et al., 2020; Wen
040   et al., 2021), or through domain adaptation using paired synthetic data (Liu et al., 2024).. In this
041   regard, the unpaired point cloud completion is a challenging task of significant practical importance.
042   Optimal Transport problem (OT) problem (Villani et al., 2009; Peyré et al., 2017) investigates the
043   cost-minimizing transport map that bridges two probability distributions. Since the introduction of
044   WGAN (Arjovsky et al., 2017), the OT-based Wasserstein distance has been widely adopted as a loss
045   function in various machine learning tasks, including unpaired point cloud completion (Chen et al.,
046   2020; Wu et al., 2020). Recently, several works introduced alternative approaches based on OT (Rout
047   et al., 2022; Fan et al., 2022). Instead of estimating the Wasserstein distance, these works focus on
048   learning the optimal transport map (OT Map) from the source distribution to the target distribution
049   using neural networks. Intuitively, the optimal transport map T serves as a generator of the target
050
      distributions which minimizes the given cost function. In this respect, this cost function plays a
      crucial role for T , because it determines how each input x is transported to T (x).
051
052   In this paper, we introduce a novel unpaired point cloud completion model based on the unbalanced
053   optimal transport map. We refer to our model as the Unbalanced Optimal Transport Map for
      Unpaird Point Cloud Completion (UOT-UPC). We formulate the unpaired point cloud completion

055   that the completion model is required to generate the correct complete point cloud corresponding

058   exists in unpaired point cloud completion. Then, we verify that the Unbalanced Optimal Transport

073   Notations and Assumptions Let X , Y be two compact complete metric spaces, µ and ν be
074   probability distributions on X and Y, respectively. µ and ν are assumed to be absolutely continuous
075   with respect to the Lebesgue measure. Throughout this paper, we denote the source distribution as µ
076   and the target distribution as ν. Since the focus of this paper is on point cloud completion, µ and
077   ν represent the distributions of the incomplete and complete point clouds, respectively. For a
078   measurable map T , T# µ represents the pushforward distribution of µ. Π(µ, ν) denote the set of
079   joint probability distributions on X × Y whose marginals are µ and ν, respectively. Additionally, f ∗
080
      indicates the convex conjugate of a function f , i.e., f ∗ (y) = supx∈R {⟨x, y⟩ − f (x)} for f : R →
      [−∞, ∞].

085   Optimal Transport The Optimal Transport (OT) problem investigates the task of transporting
086   the source distribution µ ∈ P(X ) to the target distribution ν ∈ P(Y). This problem was initially
087   formulated by Monge (1781) using a deterministic transport map T : X → Y such that T# µ = ν:
088                                                    Z
089                              Cot (µ, ν) := inf         c(x, T (x))dµ(x) .                       (1)
                                                T# µ=ν       X
090
091   Intuitively, Monge’s OT problem explores the optimal transport map T ∗ that connects two distribu-
092   tions while minimizing the given cost function c(x, T (x)). Although Monge’s OT problem offers
093   an intuitive understanding, it has theoretical limitations: this formulation is non-convex and the
094   optimal transport map T ∗ may not exist depending on the conditions on µ and ν (Villani et al.,
095   2009). To overcome these issues, Kantorovich introduced a relaxed formulation of the OT problem
096
      (Kantorovich, 1948). Formally, this Kantorovich formulation is expressed in terms of a coupling π
      rather than a transport map T , as follows:
097
                                                        Z
098
099                             Cot (µ, ν) := inf              c(x, y)dπ(x, y) .                      (2)
                                              π∈Π(µ,ν)       X ×Y
100
101   where c is a cost function and π ∈ Π(µ, ν) is a coupling of µ and ν. In contrast to the Monge
102   problem, the minimizer π ⋆ of Eq 2 always exists under some mild assumptions on (X , µ), (Y, ν) and
103   the cost function c (Villani et al., 2009). Note that under our assumptions that µ and ν are absolutely
104
      continuous with respect to the Lebesgue measure, the deterministic optimal transport map T ∗ exists
      and the optimal coupling is given by π ⋆ = (Id × T ⋆ )# µ (Villani et al., 2009).
105
106   Rout et al. (2022); Fan et al. (2022) proposed a method for learning the optimal transport map
107   T ⋆ using the semi-dual formulation of OT. This neural network-based approach for learning the
      optimal transport map is referred to as Neural Optimal Transport (Neural OT). These works applied

114   Unbalanced Optimal Transport The classical OT problem assumes an exact transport between
115   two distributions µ and ν, i.e., π0 = µ, π1 = ν. However, this exact matching constraint results in
116
      sensitivity to outliers (Balaji et al., 2020; Séjourné et al., 2022) and vulnerability to class imbalance in
      the classical OT problem (Eyring et al., 2024). To mitigate this issue, a new variation of the optimal
117
      transport problem is introduced, called Unbalanced Optimal Transport (UOT) (Chizat et al., 2018;
118
      Liero et al., 2018). Formally, the UOT problem is expressed as follows:
119

121            Cuot (µ, ν) =        inf                c(x, y)dπ(x, y) + DΨ1 (π0 |µ) + DΨ2 (π1 |ν) ,            (4)
                               π∈M+ (X ×Y)      X ×Y

123   where M+ (X × Y) denotes the set of positive Radon measures on X × Y. DΨ1 and DΨ2 rep-
124           two f -divergences generated by convex functions Ψi , and are defined as DΨi (πi |η) =
      resents

      R

              dη(x)    dη(x). These f -divergences penalize the discrepancies between the marginal distribu-
126   tions π0 , π1 and µ, ν, respectively. Hence, in the UOT problem, the two marginal distributions
127   are softly matched to µ, ν, i.e., π0 ≈ µ and π1 ≈ ν. Intuitively, the UOT problem can be seen as
128   the OT problem between π0 ≈ µ and π1 ≈ ν, rather than between the exact distributions µ and ν
129   (Choi et al., 2023). This flexibility offers robustness to outliers (Balaji et al., 2020) and adaptability
130   to class imbalance problem between µ and ν (Eyring et al., 2024) to the UOT problem (See Sec
131   3.2 for details). We refer to the optimal transport map T ⋆ from π0 to π1 as the unbalanced optimal
132   transport map.
133   Choi et al. (2023) introduced a Neural OT model for the UOT problem into generative modeling,
134   called UOTM (See Sec 3.2 for details). In this paper, we introduce the unbalanced optimal transport
135   map to unpaired point cloud completion. Unlike generative modeling, in unpaired point cloud
136   completion, each incomplete source sample x should be transported to its corresponding complete
137   target sample y. Therefore, it is important to set an appropriate cost function c(x, y) in Eq 4, because
138   this cost determines how each x is transported to y in the optimal transport map. In Sec 3.1, we
139   investigate the optimal cost function for unpaired point cloud completion.

205    definition, the optimal transport map T ⋆ is (1) a generator of the complete point cloud samples,
206    i.e., T (x) ∼ ν for x ∼ µ that (2) optimally minimizes the given cost function c(x, T (x)). Thus,

258   Unbalanced Optimal Transport Map for Class Imbalance Problem In this paragraph, we clarify
259   the motivation for considering the unbalanced optimal transport map, instead of the classical optimal
260   transport map. Our goal in this paper is unpaired point cloud completion. Since the training data
261   X and Y are not given as pairs, there may be a class imbalance problem. For instance, consider
262   point cloud data consisting of ’Chair’ and ’Table’ classes. The ratio of these two classes may differ
263   between the incomplete point cloud distribution µ and the complete point cloud distribution ν. While
264   the incomplete point cloud data might consist of 50% ’Chair’ and 50% ’Table,’ the complete point
265   cloud data could be composed of 70% ’Chair’ and 30% ’Table.’
266   Unfortunately, the standard optimal transport problem (Eq. 1) is susceptible to this class imbalance
267   problem (Eyring et al., 2024). The standard optimal transport map transports each source sample
268   x ∼ µ to a target sample y ∼ ν without any rescaling. Consequently, in this class imbalance case,
269   20% of the ’Table’ incomplete point cloud data would be transported to 20% of the ’Chair’ complete
      point cloud. This behavior is undesirable for a point cloud completion model. In practice, this

331   practice, we set the source distribution µ̃ as a mixture of the incomplete point cloud distribution µ
332   and complete point cloud distribution ν, with a mixing probability of 50%, i.e., µ̃ = 0.5µ + 0.5ν.
333   Then, we train the unbalanced optimal transport map between µ̃ and ν. This mixture trick helps our
334   generator to produce high-fidelity complete point clouds. We conducted ablation studies on the
335   mixture trick and the cost function in Sec 5.3.

      4       R ELATED W ORKS
338
339
      Unpaired Point Completion Model Unpaired point completion models have developed following
340   recent advancements in unsupervised learning. Unpaired (Chen et al., 2020) is one of the first
341   approaches for unpaired point completion. This model introduces a GAN-based model that maps
342   the latent features of the incomplete point cloud to the latent features of the complete point cloud.
343   Wu et al. (2020) proposes a conditional GAN model that generates multiple plausible complete
344   point clouds conditioned on the incomplete point cloud. ShapeInv (Zhang et al., 2021) employs an
345   optimization-based GAN-inversion approach (Xia et al., 2022). ShapeInv finds the optimal generator
346   input noise to reconstruct the complete point cloud from the given incomplete point cloud. This
347   is conducted by minimizing the distance between the input incomplete point cloud, which is for
348
      completion, and the partial point cloud, which is obtained by degrading the generator’s output. Cycle4
      (Wen et al., 2021) proposes two simultaneous cyclic transformations between the latent spaces of
349
      incomplete point cloud and complete one through missing region coding. USSPA (Ma et al., 2023)
350
      proposes a symmetric shape-preserving method based on GAN. This method utilizes a two-part
351   generator. The first part is a coarse predictor with a symmetry learning module. The second part is an
352   autoencoder with local feature grouping and an upsampling module. In this paper, we propose an
353   unbalanced optimal transport approach for point cloud completion. To the best of our knowledge, this
354   is the first attempt to introduce the optimal transport map for the unpaired point cloud completion.

445   5.2   ROBUSTNESS TO C LASS I MBALANCE OF UOT APPROACH
446
      In this section, we explore the robustness of our model in class-imbalanced settings. As described
447
      in Sec 3.2, a key advantage of the UOT framework is its robustness and stability in handling class
448
      imbalance scenarios (Eyring et al., 2024). When the proportions of data classes between the source
449   and target distributions differ, UOT can rescale the mass to compensate for this imbalance, ensuring
450   that the learned transport map remains meaningful and accurate. Furthermore, note that this class
451   imbalance is neither an unusual nor a contrived scenario. As we observed in Table 2, this class
452   imbalance exists in even our multi-category experiment in Sec. 5.1.
453
454   Experimental Settings To explore the effects of class imbalance, we observe how the performance
455   of existing point cloud completion models changes with different class imbalance ratios. To be more
456   specific, we select two categories of datasets: Data1 (category: TV) and Data2 (category: Table).
457   These categories are selected because of their relatively abundant training samples and the distinct
458   differences in their shape. For the incomplete point cloud samples, we use the entire training data
459   for both Data 1 and Data2, maintaining their ratio of 6.4 : 21.3 in Table 2. For the complete point
460
      cloud samples, we manipulate the imbalance ratio r, i.e., Data1 and Data2 are sampled at a ratio of
      6.4 : 21.3 × r. Then, each model is evaluated across diverse values of r to explore the effects of class
461
      imbalance. We compare our model to (i) the standard OT counterpart of our model (OT-UPC) and
462
      (ii) USSPA, the state-of-the-art method for unpaired point cloud completion. Note that, as discussed
463   in Sec. 3.2, our model corresponds to the standard OT counterpart when Ψ∗i = Id. For detailed
464   hyperparameter settings, please refer to Appendix A.
465
466   Discussion As shown in Table 5, our model outperforms the two alternative models across various
467   class imbalance settings. (See Table 8 in the Appendix for results on other class combinations.) Note
468   that we tested r ≤ 1, because Data2 has a significantly larger total number of training samples, more
469   than three times that of Data1 (Table 2). Hence, setting r > 1 would result in discarding too many
470   training data samples. Our model consistently demonstrates stable performance, ranging between
471   6.65 and 6.78 across various class imbalance ratios r, while USSPA shows considerably greater
472   variance. In contrast, the standard OT generally performs poorly, with its best result appearing in the
473
      balanced case (1:1 ratio). We hypothesize that this phenomenon occurs due to the unstable training
      dynamics of the standard OT. The stable training dynamics in learning the transport map is also
474
      another advantage of the UOT over OT (Choi et al., 2024). In summary, these results indicate that our
475
      UOT-UPC offers strong robustness to class imbalance problem.

518   should ideally learn the identity mapping, which is relatively easier compared to completing the input
519   incomplete point cloud. We hypothesize this property encourages the training process, enabling the
520   model to generate complete point clouds more efficiently. Therefore, we empirically observed an
521   improvement in the fidelity of the point cloud completion when using this source mixture trick.

529   performance deteriorates by approximately 10% when τ is either too large (τmax ) or too small (τmin ).

531   6      C ONCLUSION
532
      In this paper, we introduce UOT-UPC, an unpaired point cloud completion model based on the UOT
533
      map. To the best of our knowledge, our work is the first attempt to introduce the unbalanced optimal
534   transport map to the point cloud completion task. We formulated the unpaired point cloud completion
535   task as an (unbalanced) optimal transport problem and investigated the optimal cost function for
536   this task. Our experiments demonstrated a strong correlation between cost function selection and
537   the model’s point cloud completion performance. When combined with the InfoCD cost function,
538   our UOT-UPC attains competitive performance compared to both unpaired and paired point cloud
539   completion models. Moreover, our experiments showed that UOT-UPC presents robustness to the
      class imbalance problem, which is prevalent in the unpaired point cloud completion tasks.

      A.1       N ETWORK
708
709   We adopt the generator and discriminator architectures from the USSPA framework as completion
710   model Tθ and potential vϕ . For the potential vϕ , the final sigmoid layer of the discriminator is omitted
711   to allow for the parameterization of the potential function, enabling outputs to assume any real values.
712   Additionally, we remove the feature discriminator to streamline the architecture. In the potential
713   vϕ , we implement the encoder proposed by Yuan et al. (2018) in their Point Cloud Networks (PCN).
714   Following the encoder, we employ an MLPConv layer specified as MLPConv(Cin , [C1 , . . . , Cn ]) =
715
      MLPConv(1024, [256, 256, 128, 128, 1]), which indicates that the output y is computed as follows:
716
717                  y = Conv1DC4 =128,C5 =1 (ReLU(. . . ReLU(Conv1DCin =1024,C1 =256 (x)) . . .))          (9)
718
719   Here, Conv1DCin ,Cout represents a 1D convolutional layer with Cin input channels and Cout output
720
      channels.
721
      The completion model Tθ receives as input a concatenation of the incomplete point cloud
722
      and a complete point cloud. These inputs are processed independently to generate distinct complete
723   point cloud samples. The completion model Tθ follows an Encoder-Decoder architecture, augmented
724   by an upsampling refinement module (upsampling module) in sequence. The upsampling module is
725   implemented using a 4-layer MLPConv network, where the final MLPConv layer is responsible for
726   refining and adding detailed structures to the output (Ma et al., 2023). Specifically, the inputs to the
727   last MLPConv layer are composed of the skeleton point cloud produced by the Encoder-Decoder
728   structure and the features extracted from the third MLPConv layer.

      Motivation - Optimal Cost Function The incomplete and complete point clouds utilized in the
      optimal cost function outlined in Sec 3.1 are sourced from the dataset proposed by Ma et al. (2023).

      This dataset consists of paired incomplete and complete point clouds. For a fair comparison, we
734
      shuffle the complete point clouds to create an unpaired setting. We then use these shuffled point
735   clouds as artificial complete data to train the USSPA model.

737   Training Concerning the loss function Lv,T . We employ Infocd as the cost function c with a
738   coordinate value of τ = 0.05. For the hyperparameters of InfoCD, we set τinfocd to 2 and λInfoCD
739   to 1.0 × 10−7 . The functions Ψ∗1 and Ψ∗2 are defined using the Softplus activation, SP(x) =
740   2 log(1 + ex ) − 2 log 2.2 As a regularization term, we incorporate the density loss dl proposed by Ma
741   et al. (2023), and we designate a coordinate value of 10.5 for dl. The objective of Potential vϕ is to
742   assign high value to target sample y while assigning lower values to generated sample ŷ. We utilize
743   the Adam optimizer with β1 = 0.95, β2 = 0.999 and learning rates of 2.0 × 10−5 , 1.0 × 10−5 for
744   the potential vϕ and completion model Tθ , respectively. The training is conducted with a batch size 4.

      The maximum epoch of training is 480. We report the final results based on the epoch that yields the
      best performance.
