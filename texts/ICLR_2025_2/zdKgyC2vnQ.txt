011                                               A BSTRACT
012
013            Vision-language models (VLMs) serve as general-purpose end-to-end models in
014            autonomous driving, performing subtasks such as prediction, planning, and per-
015
               ception through question-and-answer interactions. However, most existing meth-
               ods rely on computationally expensive visual encoders and large language models
016
               (LLMs), making them difficult to deploy in real-world scenarios and real-time
017
               applications. Meanwhile, most existing VLMs lack the ability to process multi-
018            ple images, making it difficult to adapt to multi-camera perception in autonomous
019            driving. To address these issues, we propose a novel framework called MiniDrive,
020            which incorporates our proposed Feature Engineering Mixture of Experts (FE-
021            MoE) module and Dynamic Instruction Adapter (DI-Adapter). The FE-MoE ef-
022            fectively maps 2D features into visual token embeddings before being input into
023            the language model. The DI-Adapter enables the visual token embeddings to
024            dynamically change with the instruction text embeddings, resolving the issue of
025            static visual token embeddings for the same image in previous approaches. The
026
               DI-Adapter enables the FE-MoE to further extract and process 2D visual features
               based on user instructions, focus on attention regions, and reduce redundancy.
027
               Compared to previous works, MiniDrive achieves state-of-the-art performance in
028
               terms of parameter size, floating point operations, and response efficiency, with
029            the smallest version containing only 83M parameters.

032   1   I NTRODUCTION
033
034   As large-scale pretraining techniques develop, vision-language models (VLMs), due to their pow-
035   erful visual reasoning capabilities, become the primary choice for visual question answering tasks
036   across various domains. Similarly, in the field of autonomous driving, question-answering reason-
037   ing based on VLMs has the potential to become a new method of interaction between drivers and
038   vehicles. This natural language question-answering approach enhances the interpretability of au-
039
      tonomous driving. VLMs integrate perception, prediction, and decision-making during driving into
      a unified model within autonomous driving systems, functioning as an end-to-end general model
040
      for solving various sub-tasks in autonomous driving. Numerous VLMs applications in autonomous
041
      driving systems already exist, where these models begin to perform tasks such as closed-loop con-
042   trol, scene perception, and traffic agent behavior analysis in autonomous systems (Chen et al., 2023;
043   Mao et al., 2023; Sima et al., 2023; Xu et al., 2023).
044
045

      ation. This implies that deploying VLMs in a system requires high computational costs and hardware
046
      resources. In autonomous driving systems, developing VLMs that consume fewer resources, have
047
      lower computational costs, and respond faster becomes a key consideration for practical deploy-
048   ment. However, current research on multimodal large models in autonomous driving mainly focuses
049   on models with over a billion parameters, such as BLIP-2 (Li et al., 2023), LLaMA-7B (Touvron
050   et al., 2023), GPT-3.5, and GPT-4 (Achiam et al., 2023), with the vision encoders relying on pre-
051   trained models based on the Transformer architecture, like CLIP (Radford et al., 2021). This con-
052   sumes substantial computational resources and hardware costs, and requires longer response times,
053   making them challenging to apply and deploy in practice. Recently, EM-VLM4AD(Gopalkrishnan
      et al., 2024) introduces a lightweight architecture, attempting for the first time to apply lightweight

123   2.1   V ISION -L ANGUAGE M ODELS
124
125   The success of the Transformer architecture (Vaswani, 2017) drives the development of LLMs. In
126   the field of computer vision, Dosovitskiy et al. (2020) propose the Vision Transformer (ViT), which
127   divides images into patches and processes them based on the Transformer architecture, adapting it
128   to computer vision tasks with success. Both images and natural language can be effectively learned
129   and represented by the Transformer architecture. A pioneering work is CLIP (Radford et al., 2021),
130
      which uses contrastive learning for image-text alignment training, demonstrating superior zero-shot
      capabilities in image classification tasks. Llava (Liu et al., 2024b) freezes CLIP’s vision encoder
131
      (ViT) and adds a linear projection layer between the vision encoder and LLMs, aiming to map
132
      visual output representations into textual space. Similarly, BLIP-2 (Li et al., 2023) aligns visual
133   and textual representations through a more complex Q-Former. InstructBLIP (Panagopoulou et al.,
134   2023) builds on BLIP-2 with instruction fine-tuning on public visual question-answering datasets.
135   MiniGPT-4 (Zhu et al., 2023) combines a frozen vision encoder and Q-Former with the similarly
136   frozen LLM Vicuna, aligning them with a single projection layer. Llava-1.5v (Liu et al., 2024a)
137   achieves state-of-the-art performance in 11 benchmarks by using CLIP-ViT-L-336px with a mul-
138   tilayer perceptron (MLP) projection layer and adding VQA data tailored for academic tasks with
139   simple response formatting prompts, significantly improving data efficiency. Phi-3-mini (Abdin
140   et al., 2024) features a default 4K context length and introduces a version extended to a 128K con-
141
      text length using LongRope technology, while employing a block structure similar to Llama-2 and
      the same tokenizer, enabling a lightweight multimodal model. Despite the powerful capabilities of
142
      these multimodal large models and their trend toward lightweight design, their parameter counts
143
      exceed one billion, making deployment and real-time use on many hardware platforms challenging.
144   Therefore, research and development of efficient vision-language models with smaller parameter
145   sizes and lower computational costs are necessary.
146
147   2.2   AUTONOMOUS D RIVING BASED ON LLM S
148
149   LLMs effectively enhance both the explainability of autonomous driving systems and their interac-
150   tion with humans (Greer & Trivedi, 2024). These advantages lead researchers to incorporate multi-
151   modal data from autonomous driving into LLMs’ training, aiming to build multi-modal large models
152   for autonomous driving. Chen et al. (2023) aligned vectorized modal information with LLaMA-7B
153
      (Touvron et al., 2023) to train a question-answering model for autonomous driving. The training
      process follows a two-stage approach: in the first stage, vector representations are aligned with a
154
      frozen LLaMA, while in the second stage, LoRA (Hu et al., 2021) is used to fine-tune the language
155
      model. DriveGPT4 (Xu et al., 2024) also employs LLaMA as its large language model, using CLIP
156   as the visual encoder. It generates corresponding answers by inputting both visual and textual in-
157   formation. DriveGPT4 leverages ChatGPT/GPT-4 to generate an instruction dataset and trains on
158   this dataset. However, DriveGPT4 only uses single-perspective images, limiting its ability to handle
159   more comprehensive understanding in autonomous driving scenarios. Wang et al. (2023) developed
160   DriveMLM , which uses LLaMA-7B as the foundational language model and ViT-g/14 as the image
161   encoder. This model processes multi-view images, LiDAR point clouds, traffic rules, and user com-
      mands to achieve closed-loop driving. Inspired by the chain-of-thought approach in large language

203   using a vision encoder based on large convolutional kernels (LKNet), further extracting and learning
204   these features to map them as tokens for the input language model. CNNs with large convolutional
205   kernels are more efficient and lightweight (Ding et al., 2022; 2024). Meanwhile, changes in the
206   user’s query dynamically alter the visual tokens.

      3.4   DYNAMIC I NSTRUCTION A DAPTER
325
326   In previous vision-language models, image representations are fixed before being input into the
327   language model, and they correspond to various text representations before entering the language
328   model for computation. To enable image representations to dynamically transform according to
329   different text representations before being input into the language model, thereby improving cross-
330   modal understanding, we introduce the Dynamic Instruction mechanism and design the Dynamic
331   Instruction Adapter. We use the text input sequence T as the key (k) and value (v), and the image
332
      input sequence V as the query (q). Through cross-attention, we compute the fused sequence V ′ that
      incorporates textual contextual information. The formula is shown as follows:
333
334                                  V ′ = CrossAtt.(q = V, k = T, v = T ).                                (6)

      The sequence in the residual channel is connected via a residual connection with the output sequence
336   of the projection layer, serving as the visual representation prior to the input into the language model.

      4.1   E XPERIMENTAL S ETTINGS
346
347   Datasets We conduct experiments on the DriveLM dataset. To ensure the fairness of the experi-
348   ments, we use the same training and evaluation protocol as EM-VLM4AD on the DriveLM dataset,
349   which includes the same training, validation, and test sets. The training set contains approximately
350   340,184 different multi-view/QA pairs, while the test set and validation set each contain 18,899 dif-
351   ferent multi-view/QA pairs.
352   Models We construct different versions of MiniDrive based on various UniRepLKNet models
353   as the vision backbone, with the main difference being their ability to learn visual token embed-
354
      dings. We use UniRepLKNet-A as the vision backbone for processing images with a resolution of
      224×224, and UniRepLKNet-S for processing images with a resolution of 384×384. We use the
355
      T5-small language model as the foundation.
356
      Evaluation metrics To ensure the fairness and reproducibility of the evaluation, we use the same
357   evaluation method as EM-VLM4AD on the DriveLM dataset, assessing the model from four differ-
358   ent perspectives: BLEU-4 (Papineni et al., 2002), ROUGE-L (Lin, 2004), METEOR (Banerjee &
359   Lavie, 2005), and CIDEr (Vedantam et al., 2015).
360   Implementation details Each model is trained on a single RTX 4090 GPU. The vision encoder is
361   frozen, while the other parameters are trained with an initial learning rate of 1e-4 and a weight decay
362   of 0.05. Each model is trained for 6 epochs on the training set. Note that in subsequent experiments,
363   MiniDrive refers to the MiniDrive224 version by default, with the number of tokens per image set to
364   16 and the number of experts set to 4.
