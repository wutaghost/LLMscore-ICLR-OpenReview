009                                               A BSTRACT
010
011            Long-context modeling is one of the critical capabilities of language AI for di-
012            gesting and reasoning over complex information pieces. In practice, long-context
013            capabilities are typically built into a pre-trained language model (LM) through a
014            carefully designed context extension stage, with the goal of producing generalist
015            long-context capabilities. In our preliminary experiments, however, we discov-
016            ered that the current open-weight generalist long-context models are still lacking
017            in practical long-context processing tasks. While this means perfectly effective
018
               long-context modeling demands task-specific data, the cost can be prohibitive. In
               this paper, we draw inspiration from how humans process a large body of informa-
019
               tion: a lossy retrieval stage ranks a large set of documents while the reader ends
020
               up reading deeply only the top candidates. We build an automatic data synthe-
021            sis pipeline that mimics this process using short-context LMs. The short-context
022            LMs are further tuned using these self-generated data to obtain task-specific long-
023            context capabilities. Similar to how pre-training learns from imperfect data, we
024            hypothesize and further demonstrate that the short-context model can bootstrap
025            over the synthetic data, outperforming not only long-context generalist models
026            but also the retrieval and read pipeline used to synthesize the training data.

029   1   I NTRODUCTION
030
031   The field of Artificial Intelligence (AI) and Natural Language Processing (NLP) have made sub-
032   stantial progress in building and teaching neural language models (LMs) to understand and generate
033
      language (Radford et al., 2019; Brown et al., 2020; OpenAI, 2023; Anthropic, 2023; 2024; Touvron
      et al., 2023a;b; MetaAI et al., 2024). Large-scale deep learning has enabled large LMs to learn from
034
      massive amounts of human-generated text (Radford et al., 2019; Brown et al., 2020). However, new
035
      challenges emerge as researchers consider building capabilities beyond those of humans. One popu-
036   lar example, also the focus of this paper, is understanding long-contexts of text (Dai et al., 2019; Su
037   et al., 2024; Xiong et al., 2023). Despite the advancements in modeling (Dao et al., 2022; Su et al.,
038   2024; Xiong et al., 2023), data keeps being lacking simply because humans no longer produce them
039   naturally. Specifically, while longer contexts give more degrees of freedom in forming possible lan-
040   guage sequences and long-interaction/complex-information tasks, the existing long texts are limited
041   to organized and compact ones like novels and code (Fu et al., 2024).
042   A common methodology in building long-context LM is to incorporate a context extension phase
043   into the model building cycle after the general pre-training phase and before the task-aware post-
044   training phase (Xiong et al., 2023; Rozière et al., 2024; MetaAI et al., 2024). As the standard
045   pre-training stage builds into the model the general capabilities over diverse text distributions, the
046   long-context extension phase is designed with the hope to extend generalist capabilities further to
047   long-context patterns. The subsequent post-training is supposed to activate and align these extra
048   long-context capabilities to task instructions.
049   Despite the exciting promise of this context extension scheme, the limited portion of model training
050   dedicated to long-context contradicts the aforementioned increasingly more complex space language
051   and task patterns when the text gets longer (Xiong et al., 2023; MetaAI et al., 2024). In other words,
052   using a context extension phase to cover all long-context understanding tasks can be mathematically
053   intractable, and consequently, building a long-context generalist may not succeed. To investigate
      this, in this paper, we conduct experiments and demonstrate empirically that practical tasks like

080   creating domain-specific supervised training may help fix this. Unlike short-context training data
081   that can be collected from everyday people (Kopf et al., 2023), long-context training can be much
082   harder to curate. Reading long pieces of text is inherently hard for humans, which could make the
083   annotation process not only costly but also very demanding and, therefore, less reliable.
084
      In this paper, to alleviate this problem and provide an intermediate solution, we propose a new
085
      approach which Automates Context Extension via Retrieval (ACER; Figure 1). Overall, ACER is
086   a two-stage method. We start with synthesizing imperfect data by combining retrieval with an LM
087   excels in short context. In the subsequent stage, we will fine-tune a large LM to bootstrap over this
088   data. ACER will start with some pairs of question and its long context, while no labeling is required.
089   In the synthesis pipeline, the long context is broken into chunks and a retrieval model will score and
090   rank the chunks. A small set of top-ranked chunks will be fed into the short-context LM to produce
091   answer with chain-of-thought (CoT; Wei et al. (2022)) reasoning. Then, in the fine-tuning stage, we
092   train the model using the context, question, and the CoT. On the other hand, the retrieval-based data
093   synthesis process will be hidden from the model. We desire that the deep model will learn a generic
094
      long-context understanding function by fitting over the full context and the CoT reasoning. We
      hypothesize that the model may discover a better latent function that transcends the original ranking
095
      mechanism used in the first stage. This also shares some spirit with LM pre-training. Whereas
096
      typically pre-training bootstrap from existing human data, due to the apparent data scarcity, ACER
097   will bootstrap from synthetic long-context data generated using retrieval.
098
099   Our experiments demonstrated the effectiveness of ACER. We found model trained with ACER
100
      without supervision can outperform contemporary generalist long-context models. It also outper-
      forms its own retrieval-based answering pipeline if applied to the test sets.

      describe how each of these stages work.

      2.1   AUTOMATIC DATA S YNTHESIS
109
110   Our data synthesis process combines a heuristic-based retrieval pipeline and a short-context genera-
111   tor. We use the following ingredients,
112
113          • Prompts: a set of prompts/problems, {p1 , p2 , .., pN }, each of which consists of a pair of
114            context and question pi = (ci , qi )
115          • Short-Context Ranker: A ranker model r(q, t) → R which takes a question q and a piece
116            of short text t. This ranker determines a relevance score corresponding to how helpful the
117            text t in answering the question q.
118          • Short-Context Generator: A generator model g(x) → a that takes some short prompt x
119            and returns an answer a. This will be an aligned instruction-tuned model like many existing
120            contemporary LM.
121
      We start the data synthesis process from the set of prompts. For one prompt pi = (ci , qi ), we break
122
      the context into a set of text chunks {ti1 , ti2 , ti3 , ...}. The ranking model will assign chunk tij a
123
      relevance score sij = r(qi , tij ). Based on these scores, we can produce a ranking of the text chunk
124   indices using the estimated helpfulness, [ri1 , ri2 , ri3 , ...]. We collect the top M ranked text chunks
125   while making sure that their concatenation can still fit in the short generator. These “most helpful”
126   chunks will together be fed into a generator to produce an answer,
127
                                    âi = g(INST ◦ qi ◦ tri1 ◦ tri2 ◦ .. ◦ triM )                          (1)
128
129
      Here INST denotes an instruction to the model to elicit an explicit reasoning in a chain-of-thought
130
      like form. This makes sure that the model can describe how it compares the information in the
131
      provided text pieces to identify the most useful ones, as well as how it combines them to arrive at
132   the final answer. This reasoning process will be used in the next stage to help train the model in the
133   fine-tuning stage.
134
135
      This data synthesis process leverages two critical capabilities in the original LM, relevance/useful-
      ness analysis and understanding of short pieces of text. We combine them heuristically to build a
136
      surrogate long-context pipeline. Readers familiar with the concept of map-reduce may recognize
137
      our data synthesis as such a process.
138

      2.2   F INE - TUNING
140
141   During data synthesis, we produce for each prompt, a full document ranking, a small set of helpful
142   documents and an answer. For fine-tuning, we discard the ranking and the document set and use
143   only the generated answer, because we do not want our model to be exposed to and learn from the
144   lossy retrieval pipeline. We fine-tune an LM fθ to produce the CoT answer aˆi , i.e.,
145                                                    X
                                          θ̂ = argminθ    f (aˆi |(ci , qi ))                        (2)
146
                                                         i
147
148   In this setup, we provide the model a lossless, unfiltered access to the full context. In addition, we
149   pair it with an answer with extra training signals, a CoT describing an information extraction process
150   of picking up useful pieces of information and sorting through. As with any other deep learning
151   application, we desire that the large over-parametrized LM can learn during the optimization process
152   to fit a long-context understanding function that will lead to similar reasoning and answer. To keep
153   it simple, we fine-tune the model with teacher forcing using a log likelihood loss. Only the answer
154   tokens participate in loss computation with the context token loss masked out during training.

244
      We directly fine-tune the ACER models over long context without approximation. This section
245
      describes the training details as well as contemporary training technologies/techniques we adopted.
246

      3.3.1     I MPLEMENTATION
248
249   We train over the synthesized data for 1 epoch with a batch size of 128 examples. We use a max
250   sequence length of 32k for RAG and 64k for reading comprehension, and therefore this amounts
251   up to 4M tokens and 8M tokens batch, respectively. These lengths are picked to cover most of the
252   test lengths while at the same time keeping training on our accelerators viable and efficient. At the
253   test time, RoPE extrapolation (Su et al., 2024) is utilized for sequences of longer length. We use a
254
      3e − 6 learning rate with Adam optimizer (Kingma & Ba, 2015). For model initialization, we still
      require a model that has a sufficiently long context length. This can be achieved unsupervisedly by
255
      training on long-context corpora. To save cost, we borrow Llama-3-8B-ProLong-Base model (Gao
256
      et al., 2024)1 , an open model unsupervisedly context-extended from our data synthesizer model
257   Llama-3-8B-Instruct (The authors made decision to build a base model from an instruction tuned
258   version.)
259
260   3.3.2     I NFRASTRUCTURE
261
262   We train our models using JAX (Bradbury et al., 2018) on cloud TPUv4s (Jouppi et al., 2023)
263   provided by TPU Research Cloud. These 4th generation TPUs are built with relatively small-size
264
      32GB HBM per chip while interconnected with high bandwidth fibers. We therefore opt to partition
      computation and perform our training with 3D parallelism of data parallelism, sequence parallelism,
265
      and tensor parallelism. We train on 32 TPUv4 chips in a (2, 4, 4) configuration mesh and map these
266
      axes to data, sequence, and tensor parallelism axes respectively. Optimizer states as well as the
267   full-precision copy of model weights are kept fully sharded over the mesh, over the entire course of
268   training. We manually shard model, optimizer, and critical activation in attention and feed-forward

280
281   For comparison, we consider models that are relatively open and closely related to the base model
282   we use for ACER, Llama-3.1-Instruct. We acknowledge the effectiveness of proprietary models
283   such as GPT4 OpenAI (2023). We do not include them here, because our focus is to evaluate a
284   method of data synthesis and the models and data they use are not directly comparable.
285   Specifically, we compare ACER with long-context LMs fine-tuned on open data as well as those
286   fine-tuned on closed data. Specifically, we consider the following,
287
288          • Together-Llama-2-7B-32K-Instruct (Together, 2023): A model by TogetherAI by ex-
289            tending Llama2 to a 32k context size. It is fine-tuned on an open mixture of long-context
290            question answering and summarization data.
291          • Llama-3-8B-ProLong-512k-Instruct (Gao et al., 2024): This is a Llama3-8B-Instruct
292            context extended on open long context dataset. It is then fine-tuned on UltraChat (Ding
293            et al., 2023), a large fine-tuning dataset generated by GPT-4. The creator found it most
294            helpful amongst open instruction-tuning datasets. This model closely relates to ours and
295            share the same base model, offering a straightforward comparison.
296          • Llama3-8B-Instruct (Truncation) (MetaAI et al., 2024): Llama3-8B-Instruct is a 8k
297            context model pre-trained and instruction-tuned by MetaAI with closed data. In this setup,
298            we truncate the input to fit it into the context.
299          • Llama3-8B-Instruct (RAG) (MetaAI et al., 2024): This is Llama3-8B-Instruct reading a
300            small set of input chunks generated with the same retrieval pipeline used in the ACER data

302          • Llama3.1-8B-Instruct (MetaAI et al., 2024): This is the second iteration of the Llama3
303            model by MetaAI. Its context is systematically extended by Meta GenAI team.
304          • Mistral-Nemo-Instruct-2407 (MistralAI, 2024): A larger 12B model trained by Mis-
305            tralAI and Nvidia using closed data. It has a native 128k context length.

308   the full context and extrapolate when reading longer context than the original training-time length

368   retrieved passages fed into the model for generating the final answer. Recall that Llama-3.1 and

376   seemingly suffers from the longer contexts with performance decaying. This shows one other ex-
377   ample that ACER achieved the goal we set for it and successfully extended the model’s capability
      presented in a shorter context onto a much longer context.

420   systems remain the same. Our ACER method still outperforms the other systems by decent mar-
421   gins. In fact, when looking at the the absolute numbers, we can observe that ACER with BM25
422   attains close or even better performance than best-performing baseline systems. The results suggest
423   that ACER is relatively agnostic to the retriever used and attains good performance to warm-start a
424
      system without requiring extra supervision.

427
428   Language Modeling Recent advancements of large language models have shown strong language
429   understanding ability and ace a wide range of natural language processing tasks. Based on the Trans-
430   former structure (Vaswani et al., 2017), LMs can be broadly categorized as decoder-only models
431   (e.g., GPT (Radford et al., 2018; 2019)), encoder-only models (e.g., BERT (Devlin et al., 2019)
      and RoBERTa (Liu et al., 2019)), and encoder-decoder models (e.g., BART Lewis et al. (2020a)

                                                        8
      Under review as a conference paper at ICLR 2025


432
      and T5 Raffel et al. (2020)). Most recently, the large language models such as the GPT fam-
433   ily (Brown et al., 2020; OpenAI, 2023), Gemini (GeminiTeam et al., 2024a;b), Llama Touvron
434   et al. (2023a;b); MetaAI et al. (2024), and Claude Anthropic (2023; 2024) have attract great at-
435   tention by achieving human-level performance on various tasks and showing structure-following
436   ability. These models are typically trained in several stages, including pre-training on web-scale
437   text data with auto-regressive language modeling, supervised fine-tuning on specific applications,
438   and human preference alignment, where the last step plays the pivotal role to steer the LMs as a
439   dialogue system to answer human’s instruction and generate responses that are in desired quality
440   and style, safe, and ethical. Some recent LM alignment methods include reinforcement learning
441
      from human feedback (RLHF) (Ouyang et al., 2022), proximal policy optimization (PPO) (Schul-
      man et al., 2017), direct policy optimization (DPO) (Rafailov et al., 2023), and Kahneman-Tversky
442
      optimization (KTO) (Ethayarajh et al., 2024).
443
444
      Long Context Modeling Transformer (Vaswani et al., 2017) is the foundation architecture for re-
445
      cent advancements in language models. However, its quadratic temporal and computation complex-
446   ity to the sequence length poses great challenge to scale to long input sequence, and the lack of ro-
447   bust position embeddings also degrades the performance of long context understanding. Therefore,
448   tremendous efforts are made to improve the long-context modeling of language models from dif-
449   ferent aspects, including more efficient attention mechanism for long context Beltagy et al. (2020);
450   Kwon et al. (2023); Dao et al. (2022); Dao (2024); Xiao et al. (2024a), a (recurrent) internal or exter-
451   nal memory bank (Dai et al., 2019; Wu et al., 2022a;b), and length extrapolation through positional
452   encoding (Press et al., 2022; Su et al., 2024; Peng et al., 2024; Zhu et al., 2024). To evaluate the
453   long-context modeling ability of LLMs, several synthetic benchmarks are proposed, such as Lost
454
      in the Middle (Liu et al., 2024), Needle in a Haystack (Kamradt, 2023), and Ruler (Hsieh et al.,
      2024). Because we are proposing new data synthesis method for long-context LM training, we do
455
      not include the synthetic benchmarks.
456
457
      Retrieval Augmented Generation Retrieval-augmented generation, or RAG, enhances LLMs’
458
      ability in knowledge intensive tasks. Originated from open-domain question answering (Chen et al.,
459   2017; Xiong et al., 2021b), RAG will enrich the prompt (i.e., a question) with additional rele-
460   vant context retrieved from an external corpus to help a reader better answer the question (Lewis
461   et al., 2020b; Guu et al., 2020; Izacard et al., 2024). The retrieval methods range from sparse fea-
462   tures (Robertson & Zaragoza, 2009; Roberts et al., 2020) and deep-learning based dense represen-
463   tations (Karpukhin et al., 2020; Lewis et al., 2020b) to direct generation by LLMs (Sun et al., 2023;
464   Yu et al., 2023). Recent studies have explored various ways to enhance RAG, such as better query
465   understanding (Kim et al., 2023; Chan et al., 2024), a better retriever (Karpukhin et al., 2020; Xiong
466   et al., 2021a; Yao et al., 2023), and a better reading model (Izacard & Grave, 2021; Cheng et al.,
467
      2021; Borgeaud et al., 2021). Specifically, Self-RAG (Asai et al., 2024) trains a critique model that
      reflects the retrieved passages and the generation to adaptively decide whether the retrieval is nec-
468
      essary. SuRe (Kim et al., 2024) proposes summarized retrieval which generate multiple summaries
469
      of retrieved context based on answer candidates. RAPTOR Sarthi et al. (2024) builds a hierarchi-
470   cal structure by iteratively cluster text chunks and generate summaries for the clusters, which can
471   aggregate text chunks to better answer summarizing questions.
472
473
474
      7   C ONCLUSION
475
      In this paper, we propose ACER, a method for automatically extending language model capabilities
476
      to longer contexts without using human generated supervision. ACER pivots through a retrieval
477
      pipeline to help a short-context model to heuristically process long pieces of text and produce a
478   synthetic imperfect answer. We demonstrate that this model can then bootstrap over this synthetic
479   answer to gain even stronger long context capabilities, often outperforms carefully built long-context
480   generalist models. We believe ACER demonstrate an effective unsupervised approach to extend
481   short context capabilities into longer contexts. It can be a useful tool to improve specific long-
482   context task performance where little training data exists. Users only need to write a few prompts
483   to run the pipeline. Broadly, we introduce automating capabilities extension onto longer context
484   as a new research problem; future works may consider better data synthesis processes and paths to
485   produce better extension results.
