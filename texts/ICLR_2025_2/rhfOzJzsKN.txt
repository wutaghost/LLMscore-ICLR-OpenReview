009                                                A BSTRACT
010
011            Federated learning (FL) enables collaborative model training across distributed
012            clients without centralizing sensitive raw data while benefiting from diverse data
013            sources. Despite recent advancements in FL, the communication overhead re-
014            mains a significant challenge, especially for large-scale models. Recent low-rank
015            adaptation (LoRA) techniques have shown promise in reducing these burdens in
016
               FL, but they are typically applied to each layer individually and depend on the
               model architecture, which limits their performance. To address these shortcom-
017
               ings, we propose Model-Agnostic Projection Adaptation (MAPA), a novel ap-
018
               proach that applies factorization to the entire model parameter space, which we
019            view as a single vector, regardless of the number of layers and model architec-
020            ture. MAPA factorizes the single-vector model update into a fixed reconstruction
021            matrix and a trainable projection vector, with the reconstruction matrix being ran-
022            domly initialized using a shared seed at each round. This ensures that only the
023            projection vectors need to be communicated to the server, thereby reducing the
024            communication cost. Furthermore, MAPA’s vector-based representation and re-
025            laxed rank constraints allow for a larger reconstruction matrix and smaller projec-
026            tion vector dimensions compared to LoRA, enhancing the expressiveness of model
027
               updates while significantly reducing communication overhead. Experimental re-
               sults demonstrate that MAPA outperforms existing FL methods in both commu-
028
               nication efficiency and model performance, effectively coupling optimization and
029
               communication efficiency in FL environments.

032   1   I NTRODUCTION
033
      Federated learning (FL) is a distributed machine learning framework that enables model training
034
      across numerous devices, referred to as clients, without the need to collect or process client data on
035
      a server. In a typical FL process, each client downloads an initialized model from the server, trains it
036   using local data, and then uploads the updated model back to the server. The server aggregates these
037   updates to refine the global model, employing techniques such as federated averaging (FedAvg)
038   (McMahan et al., 2017). This iterative process is repeated over multiple communication rounds,
039   enabling clients to improve the model collaboratively without data sharing.
040
      Despite the notable benefits of FL, a primary challenge is the substantial communication overhead
041
      involved in transmitting model updates between clients and the server, especially when dealing with
042   resource-constrained clients and large-scale models with numerous parameters. This communica-
043   tion overhead can become a significant bottleneck, limiting the scalability and efficiency of FL.
044
045
      To address the communication burden in FL, various strategies have been developed that focus on re-
      ducing either the communication frequency or the communication load per round. To decrease com-
046
      munication frequency, methods such as performing multiple local epochs on clients (Stich, 2018)
047
      and selecting a subset of clients to participate in each training round (Sattler et al., 2019; Li et al.,
048   2020) have been proposed. On the other hand, methods aiming to reduce the communication load
049   per round have been more extensively studied. Konečnỳ (2016) broadly classified these methods
050   into two categories: (i) sketched updates, where the local model is first optimized, and then the up-
051   date is compressed before transmission, and (ii) structured updates, where the model is optimized
052   in a subspace with fewer trainable parameters, which are then transmitted to reduce communication.
053   These strategies are complementary and can collectively contribute to enhancing the scalability and
      efficiency of FL.

      Low-rank adaptation (LoRA) (Hu et al., 2021; Ou et al., 2023; Bertsimas et al., 2023) is a popu-
      lar structured update method that decomposes parameter updates for each layer independently as
067
      ∆wd1 ×d2 ≈ Ad1 ×q Bq×d2 , with the rank constrained by q ≤ min(d1 , d2 ). Recently, many re-
068
      searchers have applied LoRA in FL to enhance training efficiency Yi et al. (2023); Sun et al. (2024);
069   Cho et al. (2024); Kuo et al. (2024); Yang et al. (2024); Qi et al. (2024). However, the layer-wise
070   approach and the rank constraint in LoRA restrict the ability to fully capture the low-rank structure
071   of the global gradient, thereby limiting the performance of these methods.
072
      This paper proposes Model-Agnostic Projection Adaptation (MAPA). MAPA treats the entire
073
      model parameters as a single vector and factorizes the model update ∆Wd×1 into a fixed recon-
074
      struction matrix Ad×p and a trainable projection vector Bp×1 , where d denotes the number of model
075   parameters and p ≤ d is the reduced dimension. In contrast to Freeze A LoRA (FA-LoRA) methods
076   Sun et al. (2024); Zhang et al. (2023); Zhu et al. (2024); Hao et al. (2024), the reconstruction matrix
077   is initialized randomly with a shared seed on every FL round, and it is not frozen during training.
078   Our approach still eliminates the need to transmit A and limits the communication to the projection
079   vectors. Compared to LoRA-based methods, MAPA’s vector-based representation and relaxed rank
080   constraints allow for a larger reconstruction matrix A and smaller projection vector B dimensions,
081   enhancing the expressiveness of model updates while reducing communication costs.
082   The high compression rate of MAPA comes from its large expressive capacity by relaxing the low-
083   rank condition q ≤ min(d1 , d2 ) of LoRA and factorizing the gradient signal into a single vector.
084   However, this incurs the overhead of generating a larger reconstruction matrix A, which results in
085   a higher memory and computation burden on clients. Motivated by this, we also propose an exten-
086   sion to MAPA called MAPAX, which mitigates this overhead and balances the trade-offs between
087   communication, computation, and memory costs depending on the client’s resources. Additionally,
088   we show that MAPAX can cover the whole space of communication-efficient factorization, bridging
089
      the gap between various techniques and fostering a better understanding of their methods. Figure 1
      visualizes the architectural differences between these methodologies in matrix manipulation forms.
090
091   Overall, we make the following key contributions:
092    • Introduction of MAPA. We present MAPA, a novel matrix factorization that operates indepen-
093      dently of the model architecture. By treating the entire model parameter as a vector, MAPA
094      constructs a larger reconstruction matrix, resulting in an expressive subspace that requires fewer
095      trainable parameters than low-rank layer-wise methods.
096    • Enhancement of Communication Efficiency in FL. By integrating MAPA into FL, we achieve
097      substantial reductions in communication by optimizing in a lower-dimensional subspace.
098    • Extension to MAPAX. We introduce MAPAX, an extension of MAPA, to address the computa-
099      tional and memory overhead associated with the larger reconstruction matrix. MAPAX creates
100      a trade-off between communication, computation, and memory costs, making it adaptable to
101      clients with varying resource constraints. We show that MAPAX bridges the gap between differ-
102      ent factorization techniques, offering a unified understanding and approach.
103    • Theoretical Analysis. We provide a thorough theoretical analysis establishing the convergence
104      of MAPA. We also show that MAPA outperforms LoRA-based methods in maintaining training
105      performance while reducing communication costs.
106    • Empirical Evaluation. We conduct extensive experiments on diverse datasets and model archi-
107      tectures, showing that MAPA surpasses SOTA methods in both communication efficiency and
         model performance.

161   et al., 2014; Lebedev et al., 2014; Denil et al., 2013), which is widely used because of its solid

163   reducing the rank and, consequently, the number of trainable parameters of each layer. Therefore,

      In contrast, our technique will introduce a novel black-box factorization independent of the model

      a higher representation at the same rate of communication. This factorization reshapes the entire
169   parameter matrix to the form of a single vector as Wt ∈ Rd×1 . Therefore, the update matrix
170   ∆W ∈ Rd×1 is computed as ∆W = AB, where A ∈ Rd×p and B ∈ Rp×1 having p ≪ d.

      In contrast to Freeze A LoRA methods, (Sun et al., 2024; Zhang et al., 2023; Zhu et al., 2024; Hao
      et al., 2024), we initialized B = 0 and A ∼ N (0, I), at the beginning of each round, we update
173
      model parameters W and reset B and generate A afresh and independently. This allows the explo-
174
      ration of various subspace configurations without any communication overhead and performance
175   improvement (See Appendix B). The sub-optimality of having a frozen A was also discussed in Guo
176   et al. (2024), although we provide an alternative solution from Guo et al. (2024), which does not
177   require training and transmission of matrix A, thus preserving communication efficiency. The next
178   difference between MAPA and LoRA-based methods lies in the condition of the factorization rank.
179   For a given matrix W ∈ Rd1 ×d2 , LoRA aims to reduce the number of parameters by factorizing the
180   update as ∆W = AB, where A ∈ Rd1 ×q and B ∈ Rq×d2 , requiring the factorization rank to satisfy
181   q < min(d1 , d2 ). However, since the size of the random matrix A does not add communication
182   overhead, we focus on keeping the projection vector B smaller than model parameters W .
183   To summarize, we introduce a unique matrix factorization method to reduce communication over-
184   head in FL. In contrast to compression techniques, our approach optimizes a low-dimensional pro-
185   jection vector directly in the subspace, demonstrating greater effectiveness than projecting already
186   computed gradients, especially in low-bandwidth scenarios. In comparison to existing low-rank
187   factorization techniques, MAPA enables a much larger reconstruction matrix by treating model pa-
188   rameters as a single vector, relaxing the low-rank condition, and employing a model-agnostic fac-
189
      torization independent of the number of layers and their architecture. Finally, MAPA enhances the
      subspace exploration by initializing the reconstruction matrix at each turn. All contributions collab-
190
      oratively result in a more expressive subspace where less information needs to be communicated,
191
      achieving greater flexibility, performance, and communication efficiency in FL.

194   In this section, we present MAPA, MAPAX, and their application in FL. We begin by elaborating
195   on the MAPA factorization technique, demonstrating the theoretical basis for proving its higher
196   representation capacity while facilitating lower gradient dimensions. Then, we explain how MAPAX
197   can be seen as the general factorization form and discuss its benefits. Subsequently, we describe the
198   detailed process for effectively leveraging MAPA factorization within the FL framework.

      3.1 M ODEL -AGNOSTIC L OW-R ANK A DAPTATION (MAPA)
200
201
      Recent literature studied the effect of low-rank factorization on FL communication efficiency (Sun
      et al., 2024; Zhang et al., 2023; Zhu et al., 2024; Hao et al., 2024). In each layer W ∈ Rd1 ×d2 , the
202
      idea of LoRA is to factorize the model update as ∆W = AB, where A ∈ Rd1 ×q and B ∈ Rq×d2 for
203
      q < min(d1 , d2 ). They take advantage of freezing the reconstruction matrix A, limiting the trainable
204   parameters to projection matrix B, thus reducing communication. While low-rank factorization
205   shows a promising direction in FL, MAPA aims to answer a more general question: How can we
206   design a factorization that achieves higher representation capacity with lower trainable parameters?
207
      MAPA Intuition and Description. MAPA works toward a factorization resulting in a large recon-
208
      struction matrix and small projection matrix, leveraging the fact that random reconstruction matrices
209
      do not need to be communicated, achieving higher representation capacity without communication
210   overhead, and resulting in a smaller projection matrix needed to be communicated. An analogy for
211   this purpose can be seeing the reconstruction matrix A as a shared vocabulary and the size of the
212   projection matrix B as the number of words used to communicate a message. A richer vocabu-
213   lary (larger A) allows for conveying complex ideas more concisely, reducing the number of words
214   (smaller B) needed to be communicated. To achieve this, MAPA treats the entire update of the
215   model as a single vector and applies a black-box factorization, regardless of the number of layers
      or the network architecture. Let d denote the total number of parameters across all layers of the

405
      MAPA’s effectiveness is assessed on image classification datasets: FMNIST Xiao et al. (2017),
406
      MNIST LeCun et al. (1998), CIFAR-10 and CIFAR-100 Krizhevsky et al. (2009). MNIST and
407
      FMNIST contain 60,000 training samples and 10,000 test samples, whereas CIFAR-10 and CIFAR-
408   100 each comprise 50,000 training samples and 10,000 test samples. Unlike other tasks, CIFAR-100
409   includes 100 classes, each containing 500 training and 100 test samples. We employ a CNN model
410   with 11k parameters for the MNIST and FMNIST datasets, a more substantial model with 1.4M
411   parameters for CIFAR-10, and a ResNet model with 5.5 parameters for CIFAR-100.
412
      We distribute the training set of each dataset among clients for model training, and the performance
413
      of the final global model is evaluated using the original test set. Our experimental setup contains
414   N = 100 clients with non-IID data distribution. The non-IID distribution is created by splitting
415   class data into 20 shards and then randomly assigning 5 shards from all class shards to each client
416   by finding a permutation that uses the whole dataset while assigning two to five classes for each
417   client. Similarly, for CIFAR-100, we attain twenty to fifty classes for each client.
418
      Our MAPA framework is built using JAX (Bradbury et al., 2018), which facilitates extensive paral-
419
      lelization and, in particular, consistent random number generation across a large number of nodes
420   and is designed for decoupled model parameters and architectures that ease the implementation of
421   MAPA for factorization of parameters independent of the model architecture. MAPA is configured
422   with 128 trainable parameters for MNIST and FMNIST while using 1024 for CIFAR-10 and CIFAR-
423   100 and trains over 500 global rounds. We compare the performance of the proposed MAPA with
424   FedAvg, FedAvg with Sparsification (Sparse), FedAvg with Quantization (Quant), EvoFed (Rahimi

531
532
      We introduced Model-Agnostic Projection Adaptation (MAPA), a novel technique for enhancing
      communication efficiency in FL by treating the entire model parameter space as a single vector and
533
      factorizing it into a fixed reconstruction matrix and a trainable projection vector. This approach
534
      can utilize a new reconstruction matrix at each round, increasing the expressiveness of model up-
535   dates without additional communication costs. Our theoretical analysis established the convergence
536   of MAPA and demonstrated and provided MAPAX extension as a solution for high memory con-
537   sumption. Extensive experiments on different datasets showed that MAPA provides a significant
538   advancement in improving the efficiency and practicality of FL, offering a promising direction for
539   future research in distributed machine learning.
