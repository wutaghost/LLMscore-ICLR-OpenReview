                                                  A BSTRACT
008
009
               Visual navigation takes inspiration from humans, who navigate in previously un-
010
               seen environments using vision without detailed environment maps. Inspired by
011            this, we introduce a novel no-RL, no-graph, no-odometry approach to visual nav-
012            igation using feudal learning to build a three tiered agent. Key to our approach is
013            a memory proxy map (MPM), an intermediate representation of the environment
014            learned in a self-supervised manner by the high-level manager agent that serves
015            as a simplified memory, approximating what the agent has seen. We demonstrate
016            that recording observations in this learned latent space is an effective and efficient
017            memory proxy that can remove the need for graphs and odometry in visual nav-
018            igation tasks. For the mid-level manager agent, we develop a waypoint network
019
               (WayNet) that outputs intermediate subgoals, or waypoints, imitating human way-
               point selection during local navigation. For the low-level worker agent, we learn
020
               a classifier over a discrete action space that avoids local obstacles and moves the
021
               agent towards the WayNet waypoint. The resulting feudal navigation network of-
022            fers a novel approach with no RL, no graph, no odometry, and no metric map; all
023            while achieving SOTA results on the image goal navigation task.

026   1   I NTRODUCTION
027
028   Visual navigation is motivated by the idea that humans likely navigate without ever building detailed
029   3D maps of their environment. In psychology, the concept of cognitive maps and graphs Tolman
030   (1948); Chrastil & Warren (2014); Peer et al. (2021); Epstein et al. (2017) formalizes this intuition,
031   and experiments have shown the validity of the idea that humans build approximate graphs of their
032   environment, encoding relative distances between landmarks. In vision and robotics, these ideas
033
      have translated to the construction of topological graphs and latent maps based primarily on visual
      observations. These visual navigation paradigms seek new representations of environments that
034
      are rich with semantic information, easy to dynamically update, and can be constructed faster and
035
      more compactly than full 3D metric maps Gupta et al. (2017); Savinov et al. (2018); Chaplot et al.
036   (2020b); Mirowski et al. (2018); Chen et al. (2019a); Gervet et al. (2023).
037
038   In this work, we focus on visual navigation in environments where odometry information is not
039
      readily available, which limits the efficacy of current SOTA work that assumes access to noise-less
      GPS+compass sensors as in popular image-goal navigation challenges Yadav et al. (2023). This lack
040
      of odometry data also limits the efficacy of SLAM based methods that require the camera pose to be
041
      known Kwon et al. (2023); Chaplot et al. (2019), graph based methods that rely on distance to define
042   edge features Kim et al. (2023), and reinforcement learning (RL) methods which use distance-based
043   rewards Wijmans et al. (2019). Inspired by NRNS Hahn et al. (2021), which questions the necessity
044   for RL and simulation to create an effective visual navigation agent, this work leverages feudal
045   learning and latent maps as a memory proxy to show that it is possible to create a performant visual
046   navigation agent that uses no odometry, no RL, no training in simulators, no graphs, and no metric
047   maps.
048   To do this, we take advantage of a three-tiered, feudal learning network structure and show its ben-
049   efits under supervised and self-supervised learning paradigms. Feudal learning Vezhnevets et al.
050   (2017) decomposes a task into sub-components, providing performance advantages that we find
051   particularly well-suited for visual navigation. The feudal framework identifies workers and man-
052   agers, and allows for multiple levels of hierarchy (ie. mid-level and high-level managers). Each of
053   these entities observes different aspects of the task and operates at a different temporal or spatial
      scale. For navigation in unseen environments, this dichotomy is ideal to make the overall task more

054
      manageable. The worker-agent can focus on local motion, while manager-agents direct navigation
055   and assess when to move to new regions.
056
057   Key to our approach is representing the traversed environment with a learned latent map (instead
058
      of a graph) that acts as a sufficient memory proxy during navigation. This memory proxy map
      (MPM) is obtained using self-supervised learning. The high-level manager of our feudal learning
059
      agent maintains the MPM as the agent navigates in novel environments, and the MPM’s density
060
      is used to determine when a region is well-explored, and it’s time to move away from the current
061   region. A second key aspect to our approach is a waypoint network (WayNet) for the middle-level
062   manager which outputs waypoints (visible sub-goals that act as stepping stones towards a certain
063   goal) for the worker agent to move towards. We train WayNet to imitate human exploration policies
064   in a supervised manner using point-click navigation trajectories from the LAVN dataset Johnson
065   et al. (2024). The intuition is that when humans navigate a simulated environment using point-click
066   teleoperation, they use the skill of choosing a single point in the observation to move toward. For
067   example, the chosen point may be toward the end of a hallway, toward a door, or further into a
068   room. We demonstrate that this skill is easily learnable and generalizes to new environments with
069
      zero-shot transfer. Finally, we train a low-level worker to choose actions that avoid obstacles in the
      environment while following this point-wise navigation supervision. We show SOTA performance
070
      on the image-goal navigation task in previously unseen Gibson environments Xia et al. (2018) in
071
      Habitat AI Savva et al. (2019) (a simulation environment comprised of scans of real scenes).
072
073   Our contributions are fourfold: 1) A self-supervised memory proxy map (MPM) that enables lean,
074   no-odometry, no-graph, no-RL navigation, 2) A waypoint network (WayNet) for local navigation
075
      through supervised learning of human exploration policies, 3) A hierarchical navigation framework
      using agents operating at different spatial scales, and 4) SOTA performance on the image-goal task
076
      in Habitat indoor environments (testing and training on different environments).

081   Visual Navigation Visual navigation aims to build representations that incorporate the rich infor-
082   mation of scenes by injecting image-based learning into traditional mapping and planning naviga-
083   tion frameworks Gupta et al. (2017); Chaplot et al. (2019); Devo et al. (2020); Shah et al. (2021a);
084   Seymour et al. (2021). Early work focused on creating full metric maps of a space using SLAM
085
      augmented by images Chaplot et al. (2019; 2020b). While full metric maps can be ideal, especially
      if the space can be mapped before planning, the representations are computationally complex. Topo-
086
      logical graphs and maps can lighten this load and provide image data at nodes and relative distances
087
      at edges Savinov et al. (2018); Chen et al. (2019b). While easier to build, these methods require
088   odometry to be readily available and have the potential for large memory requirements, especially if
089   new nodes are added every time an agent takes an action Shah et al. (2021a; 2022); He et al. (2023).
090   One solution to this problem is sparser topological graphs Hahn et al. (2021); Shah et al. (2021b)
091   where visual features of unexplored next-nodes are sometimes predicted or hallucinated He et al.
092   (2023). Some methods go a step further by pairing semantic labels with the graph representation
093   Kim et al. (2023); Chang et al. (2023). However, these methods break down in environments that
094   are sparse or featureless, have many duplicate objects, or contain uncommon objects that may not
095   appear in popular object detection datasets. Another solution is to only use graphs during training
096
      to build 2D embedding space representations of environments, i.e. potential fields or functions, that
      preserve important physical Morin et al. (2023); Ramakrishnan et al. (2022); Bono et al. (2023b),
097
      visual Henriques & Vedaldi (2018); Bono et al. (2023a); Ramakrishnan & Nagarajan (2022), or se-
098
      mantic Georgakis et al. (2021); Chaplot et al. (2020a); Al-Halah et al. (2022) relationships between
099   regions in the environment. Our method more closely aligns with this line of work, but we do not use
100   any graph networks or graph inference and instead build our 2D latent map using self-supervised
101   contrastive methods. dditionally, unlike many of the methods above, we do not require the agent
102   to have information about the test environment (odometry) before deployment, do not use RL or
103   graphs, or learn 3D metric maps.
104

146   efits to other methodologies outside of RL. In navigation, hierarchical networks are commonly used
147   to propose waypoints as subgoals during navigation Chane-Sane et al. (2021), typically working in
148   a top-down view of the environment Xu et al. (2021) with only two levels of agents Wöhlke et al.
149   (2021). Our work uses multiple agent levels, operates in the first person point of view for predicting
150   waypoints, and reaps the benefits of this feudal relationship without using reinforcement learning.

      High-Level Manager: Memory We contrastively learn a latent space that is used to build an ag-
155
      gregate memory proxy map (MPM) that serves as a memory module for our feudal navigation agent.
156   This self-supervised latent space is learned using Synchronous Momentum Grouping (SMoG) Pang
157   et al. (2022) which combines instance level contrastive learning and clustering methods. This model
158   is chosen for its momentum grouping, which enables it to perform both instance-level and group-
159   level contrastive learning simultaneously. We demonstrate empirically that this model improves the
160   quality of the learned latent space comprising the MPM (in Section 6). Instead of using typical data
161   augmentations (e.g. rotation, scale, shift) to determine positive pairs for contrastive learning, we dy-
      namically build clusters of images observed along the training trajectories based on visual similarity


                                                                    3
      Under review as a conference paper at ICLR 2025


162
      determined by Superglue Sarlin et al. (2020) robust keypoint matching and use these clusters to de-
163   fine positive and negative pairs during training. This step is vital for the success of the agent as using
164   this positive pair definition creates a latent space that preserves the relative distance between images
165   without the need for ground truth odometry data. For each trajectory in the training data, the first
166   image observation is chosen to represent the first cluster center. Each successive image along the
167   trajectory is compared to a memory bank of previously seen cluster centers as the agent travels along
168   its trajectory. If the confidence αc of these keypoint matches is high, the current image is added to
169   the corresponding cluster. Otherwise, it is used as a new cluster center. We build these clusters per
170   environment for all training trajectories and randomly sample images from the same cluster to act
171
      as positive pairs to contrastively train the network.
172   During inference, observation images are dynamically placed in this contrastively learned latent
173   space to build a memory proxy map (MPM) of previously visited locations on the fly as the agent
174   navigates in novel environments. For greater interpretability, our approach is to build an “isomap
175   imitator network” for a 2D interpretable projection. We train an MLP to map the learned SMoG
176   feature space (128 dim) to a 2D map representation.Specifically, we compute isomap 2D embed-
177
      dings for the learned SMoG features for all of the training data to reduce the dimensionality of the
      data while preserving the relative feature distances. Because isomap produces different embedding
178
      coordinates each time it is run, a simple MLP network (isomap imitator) is trained to reproduce
179
      the isomap embeddings from the SMoG features.To update the MPM during inference, a gaussian
180   weighted circular window with σ = 1 is added to the corresponding predicted 2D location in the
181   map from the isomap imitator for each image observation, thus creating a density map with values
182   corresponding to the amount of exploration that has occurred in each location. The more an area in
183   an environment is explored, the larger the region in the MPM corresponding to this area, providing
184   valuable information for downstream networks. This allows the agent to roughly localize itself in
185   its environment with respect to its previous observations as well as providing a signal to quantify
186   the amount of exploration that has taken place in a given area. Using this approach allows for adept
187   navigation without the need for graphs or SLAM, resulting in a demonstrably effective and efficient
188
      approach via this memory proxy.

      Mid-Level Manager: Direction For the mid-level manager, we leverage human knowledge to
217   learn optimal navigation and exploration policies directly from demonstrations collected in a human
218   navigation dataset. The intuition is that the human point-click navigation decisions in the LAVN
219   dataset Johnson et al. (2024) are learnable and generalize to new environments with zero-shot trans-
220   fer. For each trajectory in LAVN, we use the HLM to create a memory proxy map. Then, using the
221   RGBD observations from the dataset and the corresponding crops of the MPM centered around the
222   agent’s current location as input, we finetune Resnet-18 He et al. (2016) to predict a pixel coordinate
223   directing the navigation agent’s motion in the environment. To make the feudal agent goal-directed
224   (which is necessary for the image-goal task used for evaluation), keypoint matches between the cur-
225
      rent observation and a goal image are computed by Superglue Sarlin et al. (2020). If the confidence
      αk of this keypoint match is high, the average of the matched keypoints is used in the navigation
226
      pipeline instead of the waypoint prediction for the mid-level manager. In this manner the agent
227
      mimics human navigation in novel environments while checking if the goal location has been found.
228   While the high-level manager reasons about the navigation task at a more global view with a coarser
229   grained spatial scale, the mid-level manager (WayNet) has a first person, fine-grained view of the
230   environment. Breaking the problem into these two scales introduces spatial abstraction which al-
231   lows these two networks to work in tandem and compound their solutions to smaller pieces of the
232   overall problem to perform more efficiently than an end-to-end implementations (see Section 5).
233   This simplification also allows for faster training with smaller amounts of data (see Section 4).
234
235   Low-Level Worker: Action The low level worker agent takes actions in the environment based
236   on the current depth map and the waypoint predicted by WayNet. We define the following action
237   space in accordance with other SOTA methods Hahn et al. (2021); Yadav et al. (2022); Wasserman
238
      et al. (2023): “turn left 15◦ ”, “turn right 15◦ ”, and “move forward 0.25m”. Although an RL agent
      is typically used for this type of task, we find an MLP classifier works well to enable effective
239
      navigation. This classifier is trained to learn a mapping between the human-chosen action from the
240
      LAVN dataset Johnson et al. (2024) and the corresponding depth map and waypoint input. The agent
241   chooses to stop navigation when the confidence threshold αm for matching goal image features to
242   the current observation is high and either the agent’s depth measurement indicates it is sufficiently
243   close to the goal location (≤ 1m) or the ratio ψ of the area of the matched keypoints to the total
244   image size is relatively large.

      Training and Testing Procedure We train our method with the LAVN Johnson et al. (2024)
264   human navigation dataset, using a subset of 117 trajectories with a total of 36834 frames. The
265   average number of contrastive learning training data clusters per trajectory is 23 (median 25). We
266   train all models on either a GTX TITAN X or RTX 2080 Ti using a learning rate of 1e-4. The
267   confidence thresholds αc , αk , and αm are chosen empirically to be 0.7, and ψ is similarly chosen
268   as 0.85. The isomap imitator network used to make the MPM is a two layer MLP with ReLU
269   activations trained for 2000 epochs with batch size= 32, and we train SMoG Pang et al. (2022) for
      50 epochs with batch size = 16. We use the current observation and a 480 × 640 pixel crop of

      the MPM centered around the agent’s current observation as input to WayNet, which is a modified
271   Resnet-18 He et al. (2016) that accepts 7 channel input trained for 250 epochs with batch size = 16.
272   The classifier network for the low-level worker is a four layer MLP with PReLU activations where
273   the depth map and waypoint input have distinct projection heads trained for 2 epochs with batch
274   size = 128. In total, our entire method trains in 3M iterations. Compare this to other SOTA that uses
275   RL and simulators and trains for anywhere from 10-100M iterations Wijmans et al. (2019); Al-Halah
276   et al. (2022); Hahn et al. (2021) (or even 50 GPU days Yadav et al. (2022)) using anywhere from
277   14.5-500M frames on up to 64 GPUs. We use several orders of magnitude less data, significantly
278   less GPUs, and a fraction of the total iterations to train our network compared to other SOTA.
279   We test our network using the testing procedure and baselines outlined in NRNS Hahn et al. (2021).
280   Testing trajectories come from a publicly available set of Gibson Xia et al. (2018) environments
281   listed in Hahn et al. (2021). They consist of approximately 6K point pairs (start and goal loca-
282   tions) that are uniformly sampled from fourteen environments and divided between two curvatures
283   (straight/curved) and three goal distances (1.5 − 3m, 3 − 5m, 5 − 10m). We compare our method
284   performance against flat RL DD-PPO Wijmans et al. (2019) trained for varying lengths of time ,
285
      behavior cloning (BC) with a resnet-18 backbone and either a GRU or a metric map from Hahn
      et al. (2021), NRNS Hahn et al. (2021) with and without noise, ZSEL Al-Halah et al. (2022), OVRL
286
      Yadav et al. (2022), and NRNS and OVRL enhaced by SLING Wasserman et al. (2023). These
287
      methods are either trained directly in a simulator or for extended periods of time (ie. 100M time
288   steps, 53 GPU days), require odometry, or use graphs in their implementations. There are other
289   recent works that test on the image-goal task that require testing in previously seen environments,
290   full scene reconstruction Kwon et al. (2023), or full panoramic or semantic images Kim et al. (2023)
291   that are very reliant of the fixed spacing or semantic contextual information of residential dwellings.
292   These are excluded as unfair comparisons since our method does not assume prior knowledge of
293   semantic context (since it is not readily available in many applications).

479
      Our work extends prior research on visual navigation for the image goal task by providing a high
480
      performance, no-RL, no-graph, no-odometry solution.1 The resulting methodology is efficient,
481   lightweight and demonstrably effective. Our frameworks accomplishes this by putting emphasis on
482   building representations for agent memory, i.e. the memory proxy map (MPM). Such an emphasis is
483   critically important for no-graph solutions and can be extended for new paradigms of visual naviga-
484   tion that include continual learning. With the ubiquity of advanced SLAM algorithms, the questions
485

486
      arise: If SLAM is solved, why learn?, and why use visual navigation instead of building a metric
487   map? Certainly, there are many applications where SLAM is the best solution. But, SLAM is typi-
488   cally computationally intensive and requires considerable development time and compute resources
489   since each new environment requires management of a large-scale optimization problem. Visual
490   navigation holds the promise of a lightweight solution, suitable for select applications. Addition-
491   ally, visual and learning-based navigation enables systems to learn about the dynamic environments
492   agents are charged with navigating, including learning the nuances of navigating in social spaces.
493   Visual navigation also holds the promise of a low-cost, low-latency solution suitable for small mo-
494   bile agents at an accessible price point. In the long term, visual navigation applications with positive
495
      societal impact can enable intelligent navigation agents for applications in unseen environments
      such as disaster assessment, environment monitoring in hazardous areas (volcanic, subterranean),
496
      and indoor navigation of previously unmapped buildings.
