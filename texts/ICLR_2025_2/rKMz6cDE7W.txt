011                                                A BSTRACT
012
013              Attention computation takes both the time complexity of O(n2 ) and the space com-
014
                 plexity of O(n2 ) simultaneously, which makes deploying Large Language Models
                 (LLMs) in streaming applications that involve long contexts requiring substantial
015
                 computational resources. In recent OpenAI DevDay (Nov 6, 2023), OpenAI re-
016
                 leased a new model that is able to support a 128K-long document, in our paper,
017              we focus on the memory-efficient issue when context length n is much greater
018              than 128K (n ≫ 2d ). Considering a single-layer self-attention with Query, Key,
019              and Value matrices Q, K, V ∈ Rn×d , the polynomial method approximates the
020              attention output T ∈ Rn×d . It accomplishes this by constructing U1 , U2 ∈ Rn×t
021              to expedite attention Attn(Q, K, V ) computation within n1+o(1) time executions.
022              Despite this, computing the approximated attention matrix U1 U2⊤ ∈ Rn×n still
023              necessitates O(n2 ) space, leading to significant memory usage. In response to
024              these challenges, we introduce a new algorithm that only reads one pass of the data
025              in a streaming fashion. This method employs sublinear space o(n) to store three
026
                 sketch matrices, alleviating the need for exact K, V storage. Notably, our algorithm
                 exhibits exceptional memory-efficient performance with super-long tokens. As
027
                 the token length n increases, our error guarantee diminishes while the memory
028
                 usage remains nearly constant. This unique attribute underscores the potential of
029              our technique in efficiently handling LLMs in streaming applications.

032   1       I NTRODUCTION
033
034   Large Language Models (LLMs) such as ChatGPT (ChatGPT, 2022), InstructGPT (Ouyang et al.,
035   2022), Palm (Chowdhery et al., 2022; Anil et al., 2023), BARD (BARD, 2023), GPT-4 (OpenAI,
036   2023), LLAMA (Touvron et al., 2023a), LLAMA 2 (Touvron et al., 2023b), Adobe firefly (Adobe,
037
      2023), have revolutionized various aspects of human work. These models have shown remarkable
      capabilities in dialog systems (Ni et al., 2023; Deng et al., 2023a;b), document summarization (Huang
038
      et al., 2023; Ghadimi & Beigy, 2023; Zhang et al., 2023; Krishna et al., 2023), code completion
039
      (Zheng et al., 2023; Liu et al., 2023a; Allal et al., 2023), and question-answering (Rogers et al., 2023;
040   Budler et al., 2023; Roy et al., 2023). However, their performance in these applications is often
041   constrained by the context length.
042
043
      To prepare for the coming of artificial general intelligence (AGI) (Bubeck et al., 2023), one of
      the crucial bottlenecks for nowadays LLM is about how to handle super long context. In recent
044
      OpenAI DevDay (Nov 6, 2023) (Altman, 2023) 1 , OpenAI released a new model that is able to
045
      support a 128K-long document. In other words, you can feed a 300-page textbook into LLM. This is
046   already quite surprising. However, to finally achieve AGI, we might need to feed some data that is
047   significantly larger than the memory in a model. For example, what if we can’t even store the entire x
048   pages of a book in memory when x is super large?
049
      A longer context length allows the LLM to incorporate more information, potentially leading to more
050
      accurate and contextually appropriate responses. This increased capacity for information processing
051
      can enhance the LLM’s understanding, coherence, and contextual reasoning abilities. Therefore, to
052   optimally utilize pretrained LLMs, it’s crucial to efficiently and accurately generate long sequences.

      Despite the advantages of a long context length, LLMs, especially those based on transformers, face
074
      significant computational challenges. Inference with a long context in LLMs is computationally
075
      intensive, requiring both O(n2 ) space complexity and O(n2 ) time complexity to compute the
076   attention output. This computational demand can limit the practical application of LLMs in real-
077   world scenarios, making it a crucial area for further research and optimization.
078
079
      Previous work (Alman & Song, 2023a;b) has conducted an in-depth study on the fast approximation
080
      of attention computation within n1+o(1) time executions without space requirements. Below is a
      formal definition:
081
082   Definition 1.1 (Static Attention Approximation without Space Requirement (Alman & Song, 2023a)).
083   Let ϵ ∈ (0, 1) denote an accuracy parameter. Given three matrices Q, K, V ∈ Rn×d , the goal is to
084   construct T ∈ Rn×d such that
085
                                          ∥T − Attn(Q, K, V )∥∞ ≤ ϵ
086
087   where
088
089           • Att(Q, K, V ) := D−1 AV
090
              • A ∈ Rn×n is a square matrix A := exp(QK ⊤ /d), here we apply exp() function entry-
091
                wisely.
092
093           • D ∈ Rn×n is a diagonal matrix D := diag(A1n ) where 1n ∈ Rn is a length-n vector
094             where all the entries are ones.
095
096   However, the memory requirement of caching attention matrix D−1 A for LLM’s inference is still a
097   considerable issue that consumes O(n2 ) space complexity. In this paper, we study the computation-
098   efficiency problem in the context of transformer-based LLMs with super long context. We consider
099
      the following problem:
100
           How can we compute the attention with super-long context in space complexity of o(n)?
101
102   This question is crucial as it directly relates to the computational efficiency of LLMs, particularly
103   when dealing with super-long context lengths. In response to this question, our goal is to develop
104   an effective streaming algorithm. We aim to define and solve the streaming version of approximate
105   attention computation, which is a critical aspect of our research. By addressing this problem, we hope
106   to significantly enhance the computational efficiency of transformer-based LLMs, thereby expanding
107   their applicability in various real-world scenarios. We define the streaming version of approximate
      attention computation, which is also the problem we aim to solve in this paper:

      data structures (Jayaram & Woodruff, 2021), heavy hitter detection (Larsen et al., 2019), and sparse

      recovery (Nakos & Song, 2019). These studies focus on developing efficient algorithms for various

177   hitters, and recovering sparse signals. The advancements in these areas contribute to the development
178   of efficient and scalable algorithms for real-time analysis of streaming data.

180   Improving LLMs’ Utilization of Long Text Extensive research has been conducted on the
181   application of Large Language Models (LLMs) to lengthy texts (Su et al., 2021; Press et al., 2021;
182   Chen et al., 2023; Dao et al., 2022; Dao, 2023; Zaheer et al., 2020; Beltagy et al., 2020; Wang et al.,
183   2020; Kitaev et al., 2020; Peng et al., 2023). These studies aim to optimize LLMs to effectively
184
      capture and utilize the content within longer contexts, rather than treating them solely as inputs.
      However, despite advancements in these two directions, competent utilization of lengthy contexts
185
      within LLMs remains a challenge, as highlighted by recent works (Liu et al., 2023b; Li et al., 2023a).
186
187   The effective usage of prolonged contexts poses a significant challenge in the development and
188   application of LLMs. While research has focused on improving LLMs’ understanding of longer
189   texts, successfully leveraging this understanding for improved performance is not guaranteed. The
190
      challenge lies in effectively incorporating and utilizing the information contained within lengthy
      contexts, ensuring that LLMs can make accurate and meaningful predictions based on this additional
191
      context.

      A    L IMITATIONS
811
812
      In our work, we propose a single-pass streaming algorithm for the computation of very long sequences
813   of attention, but we recognize some limitations. Limitations relate to the algorithm’s reliance on
814   specific assumptions, limitations of the test scope, sensitivity to input quality and data characteristics,
815   and changes in performance as the data size increases.
816
817
818
      B    S OCIETAL I MPACT
819
      In this paper, we introduce an innovative single-pass algorithm, which can achieve efficient approx-
820
      imation of ultra-long sequence attention computing under sublinear space complexity, and solve
821
      the problem of high time and space complexity in current attention computing. Our paper is purely
822   theoretical and empirical in nature (mathematics problem) and thus we foresee no immediate negative
823   ethical impact.
824
825
      By constructing a specific matrix to approximate the attention output, the algorithm only needs one
      data traversal and uses sublinear space to store three summary matrices, which greatly reduces the
826
      memory requirement. It is especially suitable for processing extremely long sequences. As the
827
      sequence length increases, the error is guaranteed to decrease while the memory usage is almost
828   constant, showing excellent memory efficiency when streaming super long tokens.
