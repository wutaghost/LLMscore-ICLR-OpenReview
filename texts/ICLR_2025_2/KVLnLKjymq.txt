                                                    A BSTRACT
010
011                Inference with Transformer-based Large Language Models (LLMs) on long se-
012                quences is both costly and slow due to the quadratic complexity of the self-
013                attention mechanism. We introduce Star Attention, a two-phase block-sparse ap-
014                proximation that improves computational efficiency by sharding attention across
015                multiple hosts while minimizing communication overhead. In the first phase, the
016                context is processed using blockwise-local attention across hosts, in parallel. In
017                the second phase, query and response tokens attend to all prior cached tokens
018                through sequence-global attention. Star Attention integrates seamlessly with most
019
                   Transformer-based LLMs trained with global attention, reducing memory require-
                   ments and inference time by up to 11x while preserving 95-100% of accuracy.

022   1       I NTRODUCTION
023
024   Recent Large Language Models (LLMs) can support contexts up to millions of tokens in length
025   (Gemini-Team, 2024; Anthropic, 2024; Meta-AI, 2024), unlocking applications such as repository-
026   level code analysis, multi-document summarization, and large corpus retrieval. However, processing
027   such long sequences with LLMs requires substantial computational and memory resources due to
028   the quadratic complexity of the self-attention mechanism.
029   To address these challenges, various techniques have been proposed to reduce memory usage and
030   increase inference speed. For example, Flash Attention introduces an efficient GPU block-wise
031   implementation of the global attention, achieving significant reductions in memory overhead and
032   runtime (Dao et al., 2022; Dao, 2024). Ring Attention further extends this idea by distributing
033   the computation of self-attention and feed-forward modules across multiple devices, cleverly over-
034   lapping communication with shard-local attention computations to enhance scalability (Liu et al.,
035   2024a; Beltagy et al., 2020). More broadly, distributed strategies such as tensor, pipeline, sequence,
036
      and data parallelism have been proposed to divide compute effectively across multiple machines
      (Shoeybi et al., 2019; Huang et al., 2019; Li et al., 2023; Meta-AI, 2021).
037
038   Several prior works have shown that the attention matrix can be approximated with sparse atten-
039   tion mechanisms reducing the algorithmic complexity from quadratic to linear or log-linear. Child
040   et al. (2019) significantly reduces the complexity of attention by leveraging sparse factorizations
041   and (Choromanski et al., 2021) approximates attention using kernel-based methods. (Beltagy et al.,
042
      2020) employs sliding window attention and global tokens for efficient long-sequence processing
      while Xiao et al. (2024) adapts it for real-time long-sequence generation utilizing attention sinks.
043
      Complementing these approaches, memory-efficient techniques have also emerged. Key-value (KV)
044
      cache compression (Dai et al., 2019; Ge et al., 2024; Munkhdalai et al., 2024; Sun et al., 2024; Liu
045   et al., 2024b) and low-rank approximations (Srebro & Jaakkola, 2003) trade precision for reduced
046   memory usage.
047
048
      We introduce Star Attention, a novel algorithm for efficient LLM long-context inference 1 . This
      method is based on the observation that LLM inference usually has two stages: (1) prompt encod-
049
      ing, where the model processes input and stores KV vectors in the cache and (2) token generation,
050
      where model attends to the KV cache and autoregressively generates new tokens while updating the
051   cache with the new KV vectors. In many long-context tasks, the input consists of a long context fol-
052   lowed by a short query and a short answer. The information needed for answering the query is often

      localized within small parts of the context, meaning context tokens need only attend to nearby to-
055   kens, while query tokens need to attend to all prior tokens. Based on this observation, Star Attention
056   utilizes a two-phase approach shown in Figure 1:
057
058              1. Context Encoding: The context is divided into contiguous blocks and distributed across
059                 “context” hosts, with each host also receiving a copy of the first block (an “anchor block”).
060
                    Hosts compute self-attention only for their assigned blocks, without communicating with
                    each other, reducing attention complexity from quadratic to linear with respect to context
061
                    length. This distributed processing is similar to Ring Attention (Liu et al., 2024a) but
062
                    without the “ring” communication during context encoding (Figure 1a).
063
                 2. Query Encoding and Token Generation: The query is replicated across all hosts where
064
                    it initially attends to the KV cache on each host. Global attention is then computed by
065
                    aggregating the results at a designated “query” host by efficiently communicating a single
066                 vector and scalar per token from each context host. Only the query host updates its KV
067                 cache during this stage (Figure 1b).
068
069   Star Attention enables the context length to scale linearly with the number of hosts by distributing
070   the context processing across multiple hosts. Star Attention is compatible with most Transformer-
071   based LLMs trained with global attention, operating seamlessly out-of-the-box without additional
072
      model fine-tuning. We evaluate Star Attention for Llama3.1-8B and Llama3.1-70B (Meta-AI, 2024)
      on several long-context benchmarks. Star Attention achieves up to 11 times faster inference while
073
      maintaining 95-100% of the baseline accuracy. Furthermore, Star Attention can be combined with
074
      other LLM optimization methods like Flash Attention or KV cache compression, allowing for addi-
075   tional speedup enhancements during inference.

         2.1                   P HASE 1: C ONTEXT E NCODING
140
141      Given an input sequence comprising a context c followed by a query q, the context c is divided into
142      n contiguous blocks: c = [c1 , c2 , . . . , cn ], where each block ci contains b tokens. We introduce an
143      anchor block mechanism, in which, each block—except the first—is prefixed with the first block c1
144      of the sequence, referred to as the anchor block. This concatenation forms an augmented context c′ :
145
146                                                                    c′ = [c1 , (c1 c2 ), (c1 c3 ), . . . , (c1 cn )]
147
         where each augmented block c′i contains 2b tokens: b tokens from the anchor block c1 followed by b
148
         tokens from the current block ci (Figure 2). The positional indices of c1 are preserved, ensuring that
149
         its tokens retain their original position indices [0, 1, . . . , b−1]. The augmented blocks are distributed
150      across compute hosts, where each host computes attention over the 2b tokens from its assigned block
151      c′i and generates the corresponding key-value (KV) vectors. While KVs for the anchor block c1 are
152      discarded, the KVs for the current block ci are retained in the cache.
153
         We observe that, without anchor blocks—i.e., applying blockwise attention only to the original con-
154
         text c—the model fails to generate correct outputs. We conjecture this failure is due to the incorrect
155
         approximation to the attention patterns observed during phase 2 (Figure 3b), where multiple atten-
156      tion spikes, known as attention sinks (Xiao et al., 2024), are distributed across the sequence. These
157      spikes occur because each block is processed independently, creating an attention sink at the start of
158      each block. As a result, the model struggles to effectively focus on relevant parts of the context. To
159      address this issue, we prefix the blocks with the anchor block c1 , shifting the attention sinks to the
160      anchor tokens. By discarding the KVs of the anchor tokens the intermediate attention sinks are re-
161      moved ensuring the attention distribution of block-local attention (Figure 3c) closely approximates
         global attention (Figure 3a) while maintaining the computational efficiency of blockwise processing.

      4.1           P OSITION AND C ONTENT OF A NCHOR B LOCK
399
400
      In this section, we explore the role of anchor blocks during Phase 1 that enables Star Attention
401
      to approximate global attention behavior. As outlined in Section 2.1, anchor blocks are crucial in
402   managing the attention spikes generated at the start of each context block, helping Star Attention
403   approximate global attention (see Table 3 ) Drawing from the hypotheses on sink tokens in Xiao
404   et al. (2024), we consider two potential explanations for the effectiveness of anchor blocks: (1) the
405   model may develop a bias toward the absolute position of the anchor block, or (2) the semantic
406   content of the anchor block is essential for maintaining performance. To better understand how
407   anchor blocks enable Star Attention to approximate global attention distribution, we test both the
408   hypotheses. We conduct experiments on the Llama-3.1-8B-Instruct model, varying both the position
409   and content of the anchor block. We evaluate two configurations: a block size of 16K for sequences
410
      of length 64K, and a block size of 32K for sequences of length 128K, in both the cases, with anchor
      block size matching the context block size.
411
412   Position of anchor block: Here, we fix the content of the anchor block to the first context block and
413   vary its position IDs. We test three scenarios : (1) the position IDs are randomly sampled from the
414   range [0, starting position of the current block] (e.g., for a block starting at position 32K, position
415   IDs are sampled from [0, 32K] ); (2) the position IDs are derived from the previous block (e.g., for
416
      a block of size 16K starting at position 32K, position IDs are sampled from [16K, 32K] ); (3) the
      position IDs are fixed to the first block (our proposed approach). As shown in Table 3, varying the
417
      position of the anchor block has minimal impact on accuracy.
418
419   Content of anchor block: We fix the position IDs of the anchor block to that of the first block but
420   vary its content. We explore several configurations (as shown in Table 3): (i) a single repeated token
421   (e.g., ‘ ’, ‘ the’, or ‘.’); (ii) random tokens; (iii) shuffling the tokens of the first block; and
422
      (iv) using the original first block content (the proposed approach). Our results show that the content
      of the anchor block significantly impacts performance, with the original first block content yielding
423
      the best results. This outcome suggests that since global attention is performed during Phase 2, it
424
      is important for the local context blocks to attend to anchor blocks whose content reflects what the
425   model would see during global attention.
426
427   Previous block as anchor block: To examine the roles of both position and content, we experiment
428
      with using the previous block as the anchor block. For example, for a block of size 16K starting
      at position 32K, the anchor block would be the block with position IDs from 16K to 32K. This
429
      configuration has lower accuracy comparing to using the first block as the anchor(Table 3).
430
431   In summary, we found that while the positional placement of the anchor block is not important , its
      content is critical for optimal performance.

474   4.2   S IZE OF A NCHOR BLOCK
475
476   As discussed in Section 3.3, larger block sizes improve the accuracy of Star Attention. In this sec-
477
      tion, we analyze the impact of varying anchor block size while maintaining a fixed block size of
      32K for a sequence length of 128K. As illustrated in Figure 7, increasing the anchor block size en-
478
      hances model accuracy, with the best performance observed when the anchor block size equals the
479
      context block size. Although Figure 3b demonstrates that attention spikes predominantly occur in
480   the first few tokens, reducing the number of tokens in the anchor block leads to a substantial drop
481   in performance. This suggests that a larger anchor block is critical for maintaining model accuracy,
482   despite attention spikes being concentrated at the beginning of the sequence. This observation im-
483   plies that the anchor block’s effectiveness is not solely due to its role in managing attention sinks
484   but may involve other underlying factors. These findings remain consistent across both base and
485   instruct models, as well as for all sequence lengths. Further investigation into why the anchor block
      size must be equivalent to the context block size is left for future work.

      5   C ONCLUSION
487
488
      In this paper, we introduced Star Attention, a novel block-sparse attention mechanism designed to
489   enable efficient inference on long sequences in transformer-based LLMs. The method operates in
490   two phases: (1) context tokens are processed using blockwise-local attention, with the context seg-
491   mented into blocks where each block is prefixed with an anchor block; and (2) then the query and
492   response tokens attend to all prior cached tokens through sequence-global attention. Star Attention
493   delivers up to 11x speedup over Ring Attention while maintaining 95-100% accuracy, significantly
494   enhancing both memory efficiency and inference speed. Scaling Star Attention to longer sequences
495   (up to 1M) and larger models, we observe even greater speedups while preserving similar levels of
496   accuracy. Despite these advances, several open questions remain. The role and optimal size of an-
497
      chor blocks relative to context blocks require further exploration. Additionally, while Star Attention
      performs effectively with block sizes set to one-quarter of the sequence length, accuracy degrades
498
      when using smaller blocks on longer sequences. Future work will focus on refining the anchor
499
      block mechanism and improving performance on more complex long-context tasks to enhance the
500   scalability and robustness of Star Attention.
