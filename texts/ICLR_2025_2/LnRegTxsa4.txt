013                                              A BSTRACT
014
015            Cross-view object geo-localization has recently gained attention due to poten-
016            tial applications. Existing methods aim to capture spatial dependencies of query
017
               objects between different views through attention mechanisms to obtain spatial
               relationship feature maps, which are then used to predict object locations. Al-
018
               though promising, these approaches fail to effectively transfer information be-
019
               tween views and do not further refine the spatial relationship feature maps. This
020            results in the model erroneously focusing on irrelevant edge noise, thereby af-
021            fecting localization performance. To address these limitations, we introduce a
022            Cross-view and Cross-attention Module (CVCAM), which performs multiple
023            iterations of interaction between the two views, enabling continuous exchange
024            and learning of contextual information about the query object from both perspec-
025            tives. This facilitates a deeper understanding of cross-view relationships while
026            suppressing the edge noise unrelated to the query object. Furthermore, we inte-
027            grate a Multi-head Spatial Attention Module (MHSAM), which employs con-
028
               volutional kernels of various sizes to extract multi-scale spatial features from the
               feature maps containing implicit correspondences, further enhancing the feature
029
               representation of the query object. Additionally, given the scarcity of datasets
030
               for cross-view object geo-localization, we created a new dataset called G2D for
031            the ”Ground→Drone” localization task, enriching existing datasets and filling the
032            gap in ”Ground→Drone” localization task. Extensive experiments on the CVOGL
033            and G2D datasets demonstrate that our proposed method achieves high localiza-
034            tion accuracy, surpassing the current state-of-the-art.

037   1   I NTRODUCTION
038
039   Cross-view object geo-localization involves locating the specific position of an object in a refer-
040   ence image based on the object’s coordinates in a query image, as illustrated in Figure 1. This
041   task has diverse applications, including robot navigation (McManus et al., 2014), 3D reconstruc-
042   tion (Middelberg et al., 2014), and autonomous driving (Häne et al., 2017). Unlike traditional object
043   geo-localization (Krylov et al., 2018; Nassar et al., 2019; Chaabane et al., 2021), which deals with
044   localizing objects under similar viewpoints, cross-view object geo-localization presents significant
045
      challenges due to drastic changes in viewpoints, scale variations, and partial occlusions between
      ground-view and aerial-view images. To address these challenges, Sun et al. (2023) introduced
046
      the CVOGL dataset and proposed a cross-view object localization model called DetGeo. DetGeo
047
      uses a dot product between globally averaged query image features and reference image features to
048   direct the model attention to the spatial location of the query object. Although this approach es-
049   tablishes spatial correspondences between different views to some extent, it lacks sufficient feature
050   interaction due to the simplicity of the computation. Additionally, the use of global average pool-
051   ing discards a significant amount of spatial information, hindering the model’s ability to effectively
052   learn and associate contextual information about the query object across different views. As a result,
053   the model may focus erroneously on edge noise similar to the query object, impairing localization
      performance.

      To overcome these limitations, we draw inspiration from the core concepts of the Trans-
      former (Vaswani et al., 2017) architecture and propose a Cross-view and Cross-attention Module
071
      (CVCAM). This module applies multiple iterations of cross-attention directly between the query im-
072
      age features and reference image features, promoting effective interaction between the two views.
073   Additionally, it performs an extra cross-attention operation to integrate spatial information from
074   both views. Such interaction and fusion operations of CVCAM enable the model to fully learn the
075   contextual information of the query object across views, establish implicit correspondences, and
076   suppress non-relevant edge noise. Furthermore, we propose a Multi-head Spatial Attention Module
077   (MHSAM) to further process the spatial relationship feature maps output by CVCAM to enhance
078   the representation of query object features. MHSAM consists of three attention heads that perform
079   convolution and deconvolution operations with different kernel sizes, allowing the model to capture
080   multi-scale spatial information from varying receptive fields. This approach significantly improves
081
      the model ability to capture local features of the query object while effectively reducing the repre-
      sentation of non-target features.
082
083   Currently, there is a notable lack of comprehensive datasets for the cross-view object geo-
084   localization task. The only existing dataset is the CVOGL dataset, introduced by Sun et al. (2023),
085   which consists of two sub-datasets: one designed for the ”Ground→Satellite” localization task and
086   the other for ”Drone→Satellite” localization task. To expand the scope of cross-view object local-
087
      ization, we developed a new dataset specifically for the ”Ground→Drone” localization task, named
      G2D. This dataset addresses a critical gap in the field by extending geo-localization to include inter-
088
      actions between ground and drone perspectives, enriching the available resources for this challeng-
089
      ing problem. Our contributions can be summarized as follows:
090
091   1) We propose a Cross-view and Cross-attention Module (CVCAM) that leverages cross-attention to
092   associate contextual information about the query object across different views, establishing implicit
093
      correspondences while suppressing irrelevant edge noise.
094   2) We introduce a Multi-head Spatial Attention Module (MHSAM) that utilizes convolutional ker-
095   nels of varying sizes to capture multi-scale spatial information, further enhancing the feature repre-
096   sentation of the query object.
097   3) We create the G2D dataset, enriching the data resources for cross-view object geo-localization
098   and filling the gap in the ”Ground→Drone” localization task.
099
100
      4) Extensive experiments on the CVOGL and G2D datasets demonstrate that our proposed method,
      AttenGeo, achieves superior localization accuracy, significantly surpassing the state-of-the-art.

106
107   Existing methods for object geo-localization aim to develop algorithms that merge repeated detec-
      tions of each object into a single prediction for each ground truth object. Chaabane et al. (2021)


                                                           2
      Under review as a conference paper at ICLR 2025


108
      and Wilson et al. (2022) implemented a tracker-based approach that recognizes and merges the
109   same object across multiple frames. Krylov et al. (2018) employed a triangulation-based method
110   to find and merge nearby objects using triangulation algorithms. Nassar et al. (2019) utilized a re-
111   identification approach, employing an object detector that implicitly merges repeated detections
112   from input frames by receiving multiple frames as input. Although these methods are effective, they
113   primarily focus on localizing the same object from similar viewpoints and neglect the localization
114   of the same object under significantly different viewpoints. Sun et al. (2023) were the first to pro-
115   pose a cross-view object geo-localization task, aiming to locate objects under significant viewpoint
116   differences. They created the CVOGL dataset for this task and introduced the DetGeo method to
117
      determine the position of query objects in reference images. DetGeo (Sun et al., 2023) uses a dual-
      stream CNN architecture and incorporates a query-aware cross-view fusion module—an attention
118
      mechanism to integrate feature maps from two different views to find the spatial correspondence of
119
      query objects.

122   2.2   T RANSFORMER AND ATTENTION M ECHANISM
123
124
      The Transformer model, first introduced by Vaswani et al. (2017), has revolutionized natural lan-
      guage processing (NLP) through its innovative use of the cross-attention mechanism. This mech-
125
      anism establishes dependencies between sequences of different lengths, making it particularly ef-
126
      fective for sequence-to-sequence tasks. Given the remarkable success of Transformers, attention
127   mechanisms have been widely adopted across fields such as NLP, image classification, and object de-
128   tection. For example, Hu et al. (2018a) introduced the Squeeze-and-Excitation (SE) module, which
129   focuses on recalibrating channel-wise feature responses, thereby implementing a channel-level at-
130   tention mechanism. Woo et al. (2018) proposed the Convolutional Block Attention Module (CBAM),
131   which sequentially infers attention maps for spatial and channel dimensions. More recently, atten-
132   tion mechanisms have been combined with multi-scale feature extraction, enabling models to better
133   handle objects of various sizes. A prime example is the Feature Pyramid Network (Lin et al., 2017),
134   which constructs feature pyramids to achieve superior performance in object detection. In this con-
135
      text, for cross-view object geo-localization, we propose the use of a Cross-view and Cross-attention
      Module to establish spatial correspondences of the query object across different views, followed by
136
      the introduction of a Multi-head Spatial Attention Module to enhance the feature representation of
137
      the query object.

185   This section begins with outlining the feature fusion process and then elaborates on the key compo-
186   nents of the Cross-view and Cross-attention Module (CVCAM). The structure of CVCAM is shown
187   in Figure 3(a). Specifically, features from the query image Fq and the reference image Fr are flat-
188   tened and passed through a series of two CABs for k iterations of feature interaction. This process
189   produces the updated feature representations F′q ∈ RC×Hq Wq and F′r ∈ RC×Hr Wr . Following this
190   interaction stage, we employ an additional CAB to merge features from the two views, yielding the
191   fused representation Ff used ∈ RC×Hr ×Wr .

196
      attention operation is denoted as:

      CVOGL dataset. CVOGL (Sun et al., 2023) dataset can be used to study two types of cross-view

353   pairs. Specifically, each sub-dataset is divided into 4,343 pairs for training, 923 pairs for validation,
354   and 973 pairs for testing.

357   localization task, where ground-view images serve as query images and drone-view images are used
358   as reference images. The G2D dataset contains a total of 2,753 sample pairs, with 1,951 pairs
359   designated for training, 432 pairs for validation, and 370 pairs for testing.
360

      5     C ONCLUSION
532
533
      In this paper, we propose the CVCAM to enable the model to fully learn the contextual information
534   of the query object across different views, establishing implicit correspondences while suppressing
535   edge noise. Furthermore, we introduce the MHSAM to enhance the feature representation of the
536   query object, improving the model ability to capture the local features of the query object. In addi-
537   tion, we create the G2D dataset, which enriches the datasets for cross-view object geo-localization
538   and fills the gap for ”Ground→Drone” localization task. Extensive experiments on the CVOGL
539   and G2D datasets demonstrate that our method (AttenGeo) achieves superior object localization
      performance, reaching the state-of-the-art level.
