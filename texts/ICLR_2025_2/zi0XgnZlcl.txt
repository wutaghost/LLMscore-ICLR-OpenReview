009                                               A BSTRACT
010
011            Quantization has become one of the most effective methodologies to compress
012            LLMs into smaller size. However, the existing quantization solutions still show
013            limitations of either non-negligible accuracy drop or system inefficiency. In this
014            paper, we make a comprehensive analysis of the general quantization principles
015            on their effect to the triangle of accuracy, memory consumption and system effi-
016            ciency. We propose MixLLM that explores the new optimization space of mixed-
017
               precision quantization between output features based on the insight that different
               output features matter differently in the model. MixLLM identifies the output fea-
018
               tures with high salience in the global view rather than within each single layer,
019
               effectively assigning the larger bit-width to output features that need it most to
020            achieve good accuracy with low memory consumption. We present the sweet spot
021            of quantization configuration of algorithm-system co-design that lead to high ac-
022            curacy and system efficiency. To address the system challenge of this sweet spot,
023            we design the two-step dequantization to make use of the int8 Tensor Core easily
024            and fast data type conversion to reduce dequantization overhead significantly. Ex-
025            tensive experiments show that MixLLM achieves the best accuracy on a variety
026            of tasks for the popular LLMs than a set of state-of-the-art works. It shows 0.31
027            lower perplexity and 0.43% improvement on zero shot tasks for Llama 3 8B than
028
               QoQ, with similar memory consumption and system efficiency.

      1   I NTRODUCTION
031
032   Large language models (LLMs) (Bubeck et al., 2023; Meta, Cited Sep. 2024) have shown remark-
033   able performance on various tasks. But their large memory consumption and massive computation
034   cost have become an obstacle for the efficient deployment (Xia et al., 2023; 2024). Quantization
035   has become one of the most sufficient solution to compress LLMs into smaller size (Frantar et al.,
036   2022; Lin et al., 2024a; Xiao et al., 2023; Yao et al., 2022), by representing the weight or activation
037   with smaller bit-width. However, the existing quantization solutions still show limitations of either
038   non-negligible accuracy drop or system inefficiency.
039   There is a triangle of characteristics for efficient LLM quantization: accuracy, memory consumption
040   of parameters, and system efficiency of execution, which we call effectiveness triangle of quantiza-
041   tion. The existing quantization solutions have different focus and trade-off in the triangle:
042
      • The weight-only methodologies target to solve the memory consumption problem, and can
043
        speedup the small-batched decoding execution that faces the memory-wall problem (Xia et al.,
044     2023; Kim et al., 2024). But their accuracy drop of 4-bit quantization can be a challenge for the
045     production workloads sensitive to accuracy, as illustrated in recent studies (Wu et al., 2023). Be-
046     sides, the weight-only method can lead to system performance drop for large-batched workloads
047     (e.g., the SOTA W4A16 kernel only achieves 83% performance of its float16 counterpart at batch
048     size 512 with hidden size 4096, shown in Fig.2).
049   • The weight-activation quantization represents the activation with low-bit values along with the
050     weights, potentially lead to higher system efficiency. But it can lead to larger accuracy drop
051     than the weight-only method as the activation is usually harder to quantize (Zhao et al., 2024;
052     Ashkboos et al., 2024; Lin et al., 2024b). Besides, it introduces more dequantization overhead for
053     the activation that can hurt the system efficiency. The transformation optimizations in some works
        can make the system efficiency even worse.

054
      • Outlier separation and mixed-precision technologies emerge to improve the accuracy of low-
055     bit quantization by either excluding the unstructured high-salience weights from quantiza-
056     tion (Dettmers et al., 2024; Kim et al., 2024) or assigning larger bit-width for the quantization
057     of structured high-salience weights (Zhao et al., 2024). The former shows system efficiency prob-
058     lem due to the low efficiency of half precision (i.e., float16/bfloat16) sparse tensor processing.
059     The state-of-the-art mixed-precision solution (Zhao et al., 2024) aims for low-bit quantization but
060     shows non-negligible accuracy drop, even inferior to the 4-bit weight-only quantization.
061   Contributions. In this paper, we provide an extensive analysis of the general quantization prin-
062   ciples. To address the limitations of the previous works and cover the three characteristics in the
063   effectiveness triangle, we propose MixLLM, which makes the following contributions:
064
      ▶ High accuracy with low memory consumption: mixed-precision between output features on
065
      the weight, with global salience identification. Given that different neurons matter differently to
066
      the model’s output, we use different bit-width for different output features (i.e., output channels) for
067   the weight quantization, 8-bit for output features with high salience and 4-bit for others. Rather than
068   using a uniformed number of outliers within each layer according to the estimated salience w.r.t.
069   each single layer (Zhao et al., 2024), MixLLM identifies the salience of different output features
070   globally according to the estimated loss to the model’s output. This is because different layers
071   can have different importance to the model. Besides, the mixed-precision between output features
072   makes the system design easier than between input features because the calculation of different
073   output features are disjoint sub-problems.
074   ▶ High accuracy with good system efficiency: the co-designed quantization configuration and
075   GPU kernel optimization. We observe the sweet spot of several quantization decisions to achieve
076   both good accuracy and system efficiency. MixLLM uses 8-bit for activation quantization as it can
077   retain a good accuracy. Besides, MatMul execution tends to be bound more on the larger weight ten-
078   sor rather than the smaller activation tensor, which weakens the need to push the activation smaller
079   (refer to Sec.3.1). MixLLM uses symmetric quantization for 8-bit and asymmetric for 4-bit for good
080   accuracy, both in group-wise manner. Such configuration makes it challenging to achieve good sys-
081
      tem efficiency. We design the two-step dequantization to enable using fast int8 Tensor Core for such
      configuration, along with the fast integer-float conversion to decrease the dequantization overhead.

087   2.1       BACKGROUND OF Q UANTIZATION
088
089
      The quantizaiton maps the tensor X into the target range with smaller bit-width representation
      through affine transformation: Xq = clamp(⌊ Xs ⌉ + z, range), where s is the scale and z is the
090
      zero point. The value can be recovered (i.e., dequantization) through: X ′ = (Xq − z) × s. X ′ is
091
      pushed to the discrete chunks rather than recovered to the original value, thus has accuracy loss. The
092   bit-width is essential for the accuracy of quantization as it determines the number of chunks for the
093   quantized values (2bit width ). Take an example, enlarging the bit-width from 4 to 5 can double the
094   number of chunks, so that the 5-bit RTN quantization can easily beat the 4-bit quantizations with
095   advanced techniques (Tab.1).
096
      The scale and zero point can be calculated from the whole channel/token vector or a small group
097
      within the channel/token, the former is called per-channel/token quantization and the latter is group-
098   wise quantization. The group-wise scheme results in smaller accuracy loss due to the smaller chunk
099   scale, but requires more complex GPU kernel design1 .
100
101
      The symmetric quantization uses 0 as the zero point value, which simplifies the computations (Xq =
                                ′
102
      clamp(⌊ W s ⌉, range), X = Xq × s). This simplification enables many works to design the per-
      channel/per-token quantized linear kernels by multiplying the scales at the epilogue of the whole
103
      MatMul (matrix multiplication) for dequantization (Xiao et al., 2023; TensorRT-LLM, Cited Sep.
104
      2024). However, the symmetric quantization usually leads to larger loss than the asymmetric one as
105   the data distribution can be usually asymmetric, especially for smaller bit-width like 4-bit.

      2.2   R ELATED W ORKS AND D ISCUSSION OF G ENERAL Q UANTIZATION P RINCIPLES
109
110
111   This paper mainly focuses on post-training quantization (PTQ).
112   Systems that affect the quantization requirement. The continuous batching technology (Yu et al.,
113   2022) enables to batch the decoding tasks from different requests together to enlarge the batch di-
114   mension of MatMul during LLM inference. The SplitFuse method (Holmes et al., 2024) advances
115   the continuous batching by merging the prefill and decoding tasks into the same batch, further en-
116   larging the MatMul shapes. These technologies pushes the server side LLM jobs to become the
117   compute-bound workloads and further motivate the demand to reduce the massive computation.
118   Weight-only quantization and its limitation. There emerges a wide range of technologies to im-
119   prove the accuracy of weight-only quantization. GPTQ (Frantar et al., 2022) advances OBC (Frantar
120   & Alistarh, 2022) on OBS-based (Hassibi et al., 1993) weight compensation with blocked updating
121   and reordering. AWQ (Lin et al., 2024a) proposes to scale the weight according to the characteristic
122   of activation. OminiQuant (Shao et al., 2024)) proposes the learnable scaling and weight clipping
123   factors. SpQR (Dettmers et al., 2024), SqueezeLLM (Kim et al., 2024) and OWQ (Lee et al., 2024)
124   separate the outliers from the quantiation and with half precision. QuiP (Chee et al., 2023) aims
125   to achieve extreme low-bit quantization with incoherence processing. ZeroQuant(4+2) (Wu et al.,
126
      2023) aims to improve accuracy with medium-sized FP6 quantization.
127   The weight-only quantization does not reduce the computation but introduces the extra dequantiza-
128   tion operations. The low-bit weight will be dequantized to float16 to execute the MatMul in float16
129   datatype. The current weight-only quantization faces two challenges: 1) From the accuracy as-
130   pect, there is still an accuracy gap between the 4-bit quantization and the float16 model, especially
131   for many real business scenarios sensitive to the small accuracy drop, as discussed in the recent
132
      works (Wu et al., 2023; Xia et al., 2024). 2) It can lead to system efficiency drop on busy servers
      as the recent LLM inference serving systems will usually batch the processing of different requests
133
      together on the server and form large MatMuls. The large MatMuls are compute-bound and will
134
      suffer from the dequantization overhead (Lin et al., 2024b).
135
136   Weight-activation quantization and the challenges. The weight-activation quantization helps to
137   make use of the low-bit computing unit. LLM.int8() (Dettmers et al., 2022) observes the activa-
138
      tion outlier problem and separates outliers from quantization with half precision. ZeroQuant (Yao
      et al., 2022) proposes the per-token activation quantization and group-wise weight quantization.
139
      SmoothQuant (Xiao et al., 2023) addresses the activation outlier problem through smoothing,
140
      and AffineQuant (Ma et al., 2024) proposes the general affine transformation for quantization.
141   RPTQ (Yuan et al., 2023) reorders the channels to cluster similar scaled values together. Spin-
142   Quant (Liu et al., 2024) and QuaRot (Ashkboos et al., 2024) leverages matrix rotation properties
143   to alleviate the outlier phenomenon. Atom (Zhao et al., 2024) uses the mixed-precision between
144   input features to improve accuracy of 4-bit activation quantization. QoQ (Lin et al., 2024b) is a
145   holistic weight-activation-KV quantization solution with progressive group quantization, attention
146   smoothing, and channel reordering.
147   Even though the weight-activation quantization has the advantage of reduced MatMul computation
148   (i.e., MatMul in smaller bit-width to make use of the smaller bit-width computing unit with higher
149   computation throughput2 ), it faces the challenge of accuracy drop caused by the activation quantiza-
150   tion, especially that the activation is usually harder to quantize than the weight. The SOTA low-bit
151   weight 8-bit activation solution (W4A8) (Lin et al., 2024b) still have a gap to the 4-bit weight only
152   quantization. Beside the accuracy drop, the activation quantization will introduce more dequantiza-
153   tion overhead than the weight-only one, which is another challenge from system side.
154   The existing solutions focus on partial of the effectiveness triangle, but cannot cover all of them well.
155   MixLLM is orthogonal to the above works by exploring the mixed-precision between output features
156   with global salience identification, and the co-designed quantization decision and GPU kernels.

      3     M ETHODOLOGY
175
176
      3.1   Q UANTIZATION D ESIGN AND D ECISION IN M IX LLM
177
178   To cover the three aspects of the effectiveness triangle simultaneously, we make the following design
179   and decision of weight and activation quantization according to the analysis in Sec.2.2.
180
      A. Mixed-precision between output features of weight, with global salience identification.
181
182   It is known that different elements of the weight show different salience to the network’s loss when
183   being quantized (Kim et al., 2024; Dettmers et al., 2024). The outlier separation method can im-
184   prove the accuracy by using float16 to store the high-salience elements, but suffers from the system
185   efficiency problem (detailed analysis in Sec.A.4). We observe that the elements with high salience
186
      tend to show distribution along the output channels for most of the linear layers in many LLMs.
      Based on this observation, we can assign larger bit-width to the output channels of high salience,
187
      and smaller bit-with to the others, forming structured mixed-precision quantization. Through the
188
      experiments, we get the same conclusion with the existing works (Kim et al., 2024; Dettmers et al.,
189   2024) that there is only a small set of elements with high salience contributing significantly to the
190   model’s accuracy drop. Thus we only need to assign the large bit-width to a small portion of the
191   output channels to achieve good accuracy and retain a small memory consumption at the same time.
192
      The structured mixed-precision between different output channels can be friendly to the system
193
      efficiency and kernel development, due to the nature that different output features are disjoint in the
194
      MatMul and the computation of them are different sub-problems. Fig.1 shows how the linear layer
195   computes with the mixed-precision between output features. It divides the linear into independent
196   sub-problems, and finally gathers the output of the sub-problems together to form the final result.
197   This optimization space is orthogonal to the existing quantization optimizations, e.g., GPTQ (Frantar
198   et al., 2022), and can be applied together with them.
199
      One critical problem is how to identify the high-salience output channels in the model. The fixed
200
      threshold (Dettmers et al., 2024) or the fixed number/ratio (Zhao et al., 2024; Lee et al., 2024) of
201   high salience elements computed by the local loss of layers can be sub-optimal to the end-to-end
202   model, as different layers can show different importance to the model’s final output (Gromov et al.,
203   2024; Men et al., 2024; Dong et al., 2019). A high salience channel w.r.t. a layer may not be a
204   high salience channel of the end-to-end model. In MixLLM, we compute the high salience channels
205   globally according to their impact to the model’s final loss (Sec.3.2). As a result, different layers
206   will have different number of high salience channels.
207   Note that this design is different from the mixed-precision in Atom (Zhao et al., 2024) from two
208   aspects. 1) MixLLM first addresses the problem of identifying the high-salience channels globally
209   rather than locally. 2) MixLLM applies the mixed-precision between output features rather than
210   input features, which is more system performant and algorithm flexible3 as the output features are
211   disjoint naturally.
212
      B. Sweet spot of quantization decision with algorithm-system consideration: 8-bit symmetric
213   activation and 4-bit asymmetric weight quantization in group-wise manner.
214
          3

      MixLLM makes the same decision with QoQ (Lin et al., 2024b) on activation quantization to use
217   8-bit, as the 4-bit activation can lead to a large accuracy drop but does not lead to significant system
218   efficiency improvement as MatMul execution tends to be bound more on the larger weight tensor
219   rather than the smaller activation tensor. It can be partially indicated from the compute intensity
220   of the linear layer. Given token number M and input and output features K and N , the compute
221   intensity I = M KBact2M     NK
                               +KN Bweight . Bact and Bweight are the bytes per element of activation and
222   weight. Given M = 512 and N = K = 4096, reducing Bweight from 8 to 4 will results in a 80%
223   increasement of I, while reducing Bact from 8 to 4 will only achieve 5.88% increasement.
224
      Instead of using per-token and smoothing on the activation quantization, MixLLM uses group-wise
225   RTN method. On the one hand, Tab.1 shows that simple group-wise RTN quantization performs
226   better than token-wise smoothing method. On the other hand, the weight is already group-wise in
227   MixLLM, and the group-wise activation does not lead to significant more dequantization overhead in
228   the system. We observe symmetric quantization is enough for the 8-bit activation (refer to MixLLM
229   W8A8 in Tab.1), while asymmetric can be essential for the 4-bit weight. The group-wise method
230   with asymmetric can lead to a difficulty for the kernel to make use int8 Tensor Core, for which
231   QoQ (Lin et al., 2024b) introduces the two-step quantization method. Instead, we design a two-step
232   dequantization with the property of the mix of symmetric and asymmetric (Sec.3.3).

300
      Parallel execution of sub-problems of different bit-width. As for the execution shown in Fig.1,
301
      MixLLM puts different sub-problems onto different threads on the GPU to make them execute in
302
      parallel. Finally, the MatMul execution of the two parts write to the same target tensor with different
303   channel indices to generate the final output. We implement this function with the fused epilogue of
304   MatMul to scatter the output to the corresponding indices, which is basically costless.
305
306
      Two-step dequantization to make use of int8 Tensor Core. As for the W4A8 computation, the
      dequantized weight and activations are (Wq −z)sw and Aq sa , where Wq and z are uint4 datatype, Aq
307
      is int8 datatype, and sw and sa are float16 datatype. Directly dequantizing the tensors into float16
308
      datatype before the MatMul computation will prevent us using the fast 8-bit Tensor Core on the
309   GPU. Instead, MixLLM uses a two step dequantization within each group. Specifically, MixLLM
310   first partially dequantizes the weight into (Wq − z), and then multiply it by Aq with the 8-bit Tensor
311   Core. Finally, it multiplies this MatMul result by the two scales within each group. Note that we use
312   int8 datatype for (Wq − z) that covers the data range correctly.
313

314

      4.1      S ETUP
337
338   As for MixLLM evaluation in this paper, we use 0%, 10%, 20%, 50% and 100% percent of 8-bit
339   based on the 4-bit quantization, respectively. Meanwhile, we use 8-bit for activation quantization.
340   Both the weight and activation are group-wise quantized with group size 128. The 4-bit part is asym-
341   metric quantized and the 8-bit part (including that in weight) is symmetric, which is a good trade-off
342   between accuracy and system efficiency. Note that any other bit-width percentage configuration can
343   be used for real scenarios to trade-off memory usage, system efficiency and accuracy in practice.
344
      We enable GPTQ (without reorder) and clip search in MixLLM for the models except for Llama 3
      8B, as which shows poor performance without reorder in GPTQ. We also disable the clip search for
345
      the 70B and 72B models as the clip search tasks too long time.
346
347   Baselines and configurations. We compare MixLLM with the state-of-the-art (SOTA) quantization
348   solutions of both weight-only and weight-activation methods. As for the weight only quantization,
349   we compare MixLLM with the basic round-to-nearst (RTN) 4-bit and 5-bit quantization, and the
350

      5       S UMMARY
530
531
      We have presented MixLLM, achieving high accuracy with low memory consumption and high sys-
      tem efficiency with the rarely explored optimization space of mixed-precision quantization between
532
      output features. MixLLM identifies the salience of each output feature according to the loss distance
533
      estimation w.r.t. the global model loss rather than local layer loss. By assigning larger bit-width to
534   the features need it most, MixLLM achieves the superior accuracy to SOTA with low memory con-
535   sumption. The sub-problems of different bit-widths are disjoint and can run in parallel efficiently
536   on the GPU. We have identified the sweet spot of the quantizaiton configuration that is friendly
537   to both accuracy and system efficiency. To address the challenge of system efficiency, we design
538   the two-step dequantization to enable using int8 Tensor Core computation and the fast integer-float
539   conversion to reduce the dequantization overhead. Experiment results show that MixLLM achieves
      superior accuracy to SOTA with low memory cost and high system efficiency.

763   A.2    O NE - PASS VS . P ROGRESSIVE S EARCH
764
765   As described in Sec.3.2, MixLLM searches the high-salience features within a single pass. We have
766   tried the progressive procedure on Llama 2 7B and Mistral 7B models, which identifies smaller
767   portions of the high-salience features iteratively. Results show that the accuracy is the same to
768   the one-pass method to two decimal places. However, the progressive method shows much higher
769
      search time due to the repeated procedure. The one-pass method takes 7 minutes for each of the
      two models to search 10% high-salience features, while the progressive method that searches 2%
770
      high-salience iteratively takes 30 minutes to find top 10% features.

      A.3    OVERHEAD OF G LOBAL P RECISION S EARCH
773
774   Tab.6 shows the global precision search overhead described in Sec.3.2. As noted in Sec.4.1, the
775   calibration dataset has 128 samples with sequence length of 2048. We use a single A100 GPU
776   for the 1.5B, 7B and 8B models, and 4 A100 GPUs for the 70B and 72B models to perform the
777   search. We make use of device map in huggingface for multi-GPU execution, which is sequential
778   execution of layers. The 7B models require about 7 minutes and the 70B models require less than
779   60 minutes to complete the search. Considering that the quantization only needs to be preformed
780   once, the searching algorithm is practical for the real workloads.

782   A.4    P ERFORMANCE CHALLENGE OF THE FLOAT 16 OUTLIER SEPARATION
783
      Outlier separation with half precision works to improve the accuracy while using small bit-width
784
      for the non-sensitive weights (Kim et al., 2024; Dettmers et al., 2024), by separating the outliers
785
      into an extra sparse tensor in float16. However, it is hard to achieve the peak performance due
786   to the inefficiency of the sparse computation on the GPU, especially when the batch size is large
787   and the linear layer becomes compute-bounded. (As discusseded in Flash-LLM (Xia et al., 2023),
788   the hardware utilization can be lower than 10% for the sparse MatMul, while its dense counterpart
789   can usually achieve more than 60%.) This is because the unstructured tensor computation cannot
790   make use the fast Tensor Core easily, but has to use the SIMT Core in float16 for computation and
791   float32 for accumulation5 . Note the peak performance of int8 Tensor Core is 8× higher than that of
792   float16 SIMT Core on A100 GPU (NVIDIA, Cited Sep. 2024), and 32× higher than float32 SIMT
793   Core. Moreover, sparse computing makes it more difficult to fully utilize the hardware due to the
794
      non-continuous memory pattern and the extra index computation.
