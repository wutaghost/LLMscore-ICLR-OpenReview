011                                               A BSTRACT
012
013            Efficiently searching for target objects in intricate environments poses a significant
014            challenge for mobile robots, due to perception errors, limited field of view (FOV),
015            and visual occlusion. These factors cause the problem to be partially observed.
016            Therefore, we formulate the object-search task as a high-dimensional Partially
017
               Observable Markov Decision Process (POMDP) with hybrid (continuous and dis-
               crete) action spaces. We propose a novel sampling-based online POMDP solver
018
               named Neural Process Filtered k-Center Clustering Tree (NPF-kCT). The optimal
019
               action is selected using Monte Carlo Tree Search (MCTS) in conjunction with a
020            neural process network to filter out ineffective primitive actions (i.e., basic robot
021            operations), alongside k-center clustering hypersphere discretization to efficiently
022            refine high-dimensional continuous sub-action spaces. Adhering to the hierar-
023            chical optimistic optimization (HOO) concept, we leverage an upper-confidence
024            bound (UCB) on the action value function within the hypersphere with estimated
025            diameters to guide the MCTS expansion. We extensively tested our approach in
026            Gazebo simulations using Fetch and Stretch robots across diverse target-finding
027            scenarios. Comparative results show higher success rates and faster target de-
028
               tection than baseline methods, with no additional computational cost. We also
               validated our method on a physical robot in an office environment. Project page:
029
               https://sites.google.com/view/npfkct.

      1   I NTRODUCTION
033
034
      Object-searching tasks for mobile robots with manipulators in cluttered, partially known environ-
035
      ments, such as warehouses or living rooms, pose significant challenges. In scenarios such as re-
036   trieving an item from a warehouse box or fetching a Coke bottle from a kitchen table, the target
037   object is often occluded. Known as the “Mechanical Search” problem, these tasks typically rely on
038   partial knowledge of the environment—such as a rough map with large furniture or appliances like
039   shelves and tables—alongside many unknown small objects, e.g. mugs or snack boxes. To succeed,
040   the robot must adapt its configurations, strategically remove obstacles, localize the target, and ulti-
041   mately plan and execute the necessary grasp actions. In this paper, we assume the robot operates in
042   a home environment with multiple rooms and workspaces (furniture surfaces) and has access to 3D
043   point cloud and 2D occupancy grid maps, along with photos of the target object (Fig. 1).
044   Object search with mobile manipulators with on-board sensors relies on a comprehensive support
045   system encompassing modules for object segmentation, object detection, pose estimation, task-level
046   planning, and motion planning. Significant advancements have been made in these areas, particu-
047   larly with advanced learning technologies. For instance, Shaban et al. (2017) introduces a highly
048   efficient one-shot learning method leveraging a Fully Convolutional Network for pixel-level image
049   segmentation. Other methods employ diverse networks, such as the Neural Radiance Field architec-
050
      ture, to address challenges related to optimal grasp pose generation Sóti et al. (2023) and manipu-
      lation planning Qureshi et al. (2020). This work focuses on task-level planning to effectively select
051
      primitive actions and achieve long-horizon goals for target object search in complex environments.
052
053   In this paper, we propose a novel POMDP framework and solver, NPF-kCT, for object search tasks
      using mobile robot manipulators with only onboard sensors. We train a neural process-based net-

055   into hyperspheres via clustering, and MCTS is used to construct a belief tree for optimal action se-

092   ten integrated into the planning process, such as updating probabilities, rather than being treated as

094   layer, focusing on high-level cognitive tasks such as commonsense inference, contextual under-

096   strategies. The work Giuliari et al. (2023) demonstrates a reasoning module to infer plausible object

099

107   planning capabilities, allowing the system to remember the potential locations of the occluded target

      POMDP Formulation and Solvers: For a POMDP agent, the process commences with a brief,
109   where the agent deduces and executes the optimal action to transition to a new state. This new state is
110   concealed within an updated belief, refined through Bayesian inference based on observed data. Fol-
111   lowing each step, the agent receives an immediate reward and contributes to a discounted cumulative
112   reward, fostering a long-term objective. This iterative process persists until terminal conditions are
113   met. This framework has found extensive application in the field of robotics, effectively addressing
114   diverse tasks such as autonomous underwater vehicle navigation Hou et al. (2021), robot manipula-
115   tion Pajarinen et al. (2022), and Human-Robot collaboration Burks et al. (2023). Attaining the exact
116   optimal strategy for a POMDP problem is generally deemed computationally intractable Papadim-
117
      itriou & Tsitsiklis (1987). Due to the cheaper memory demands compared with the offline solvers,
      the online methods, particularly sampling-based approaches, have emerged as predominant solu-
118
      tions, striking a favorable balance between approximate optimality and manageable computational
119
      load. Sampling-based solvers, like Partially Observable Monte Carlo Planning (POMCP) Silver &
120   Veness (2010), Adaptive Belief Tree (ABT) Kurniawati & Yadav (2016), and Determinized Sparse
121   Partially Observable Tree Somani et al. (2013), adopt a common approach of representing belief as
122   particles and employ MCTS to expand the belief tree within constrained computational resources.
123   Despite notable progress, addressing POMDPs with high-dimensional continuous action spaces re-
124   mains a formidable challenge. The key point in existing continuous-action POMDPs is to refine the
125   action subset incrementally to improve the possibility that the selected subset of actions contains
126   the best action. Some continuous-action POMDP approaches, like partially observable Monte Carlo
127   planning with observation widening (POMCPOW) Sunberg & Kochenderfer (2018), use the Pro-
128
      gressive Widening (PW) strategy to continuously add new randomly sampled actions once current
      actions have been sufficiently explored. Other approaches incorporate technologies such as Voronoi
129
      Optimistic Optimization Lim et al. (2021) and Bayesian Optimization Mern et al. (2021) to enhance
130
      PW-based methods. These methods commonly employ the UCB1 Auer et al. (2002) algorithm for
131   action selection during exploration and leverage Monte Carlo backup for value estimation.
132
133   Neural Process in Robotics: Neural processes (NPs) offer a powerful alternative to Gaussian pro-
134
      cesses (GPs) for function regression, capturing uncertainty as a stochastic process. In robotics,
      effectively managing prediction uncertainty is crucial for enhancing the robustness and applicability
135
      of systems in real-world scenarios. For example, Chen et al. (2022) introduces a meta-learning al-
136
      gorithm using Conditional Neural Processes (CNPs) to accurately estimate grasp points from depth
137   images with minimal trials. Additionally, CNPs are utilized in a variety of applications, including
138   6D pose estimation Li et al. (2022) and social navigation Yildirim & Ugur (2022). In our work, we
139   utilize neural processes to address the uncertainty in scoring function regression with high dimension
140   input, like 3D point clouds and high-resolution images, and filter primitive actions effectively.

      General POMDP Model: In this paper, we consider a POMDP P with a hybrid action domain.
145
      Formally, defined as an 8-tuple < S, A, O, T, Z, R, b0 , γ >, where the state space S represents
146
      the state space, the action space A ≜ Ac × Ad denotes the set of all actions the robot can perform,
147
      where sub-domain Ac is assumed to be continuous and embedded in a bounded metric space with
148   distance metric function and sub-domain Ad is discrete; the observation space O means the set of all
149   observations the robot can perceive; the transition function T (s, a, s′ ) = P r(s′ |s, a) represents
150   the nondeterministic effects of actions a ∈ A working from the current state s ∈ S to the resulting
151   state s′ ∈ S; the observation function Z(s′ , a, o) = P r(o|s′ , a) is commonly a conditional
152   probability function that represents the observation the robot may perceive after performing action
153   a ∈ A in state s′ ∈ S; the immediate reward function R(s, a, s′ ) maps from a state, an action,
154   a state–action pair, or a tuple of state, action, and subsequent state to a value; the state s ∈ S is
155   initially hidden in a initial belief b0 , which is a probability distribution on the state space S; γ is a
156
      discount factor following 0 < γ < 1, set as 0.9 in this paper. The goal of solving a POMDP problem
      is to find an optimal policy Π∗ (b) = argmaxa∈A Q(b, a) for belief b, where the Q(b, a)-value is
157
      the value of executing action a when the agent is at belief b and continuing optimally afterwards.

181
      action in R9 ), declaring an object as a target or obstacle, and removing the i-th object using the

183

186   robot. Once an object is removed, it is moved outside the workspace, and its status uoi is set to -2,
187   indicating it will no longer block the view of other objects. The observation space O and function
188   Z focus on objects within the robot’s head camera FOV frustum V . Observations are assumed to
189   update the log-odds of the 4 nearest grids 5 within goi corresponding to the observed objects with
190   noise-adjusted values: −co + η for negative log-odds, co + η for positive, and η for near-zero,

192
      a positive threshold νp , the updated value will be c0 + η. For values below a negative threshold νn ,

229   NPF-kCT is an anytime online POMDP solver. We assume that the Q-value of the considered
230   POMDP problem follows Lipschitz continuity in the action space. NPF-kCT follows a standard
231   procedure with four alternating stages: planning, execution, obtaining observations, and filtering, as
232   shown in Algorithm 1 and Fig. 13. It primarily focuses on the planning stage, aiming to identify
233
      the optimal action based on the current belief b0 . To reduce the complexity of the high-dimension
      action domain and improve the refining efficiency, a pre-trained neural process-based network pre-
234
      dicts the feasibility of sampled actions with associated uncertainties, filtering out the useless action
235
      domains. The actions are then clustered into hyperspheres using k-center clustering. A belief tree
236   T is constructed, where nodes represent beliefs and actions. Each belief node nodeo ∈ T is linked
237   to a dynamic list L(nodeo ), which is initially formed from the previous step’s tree and then fuses
238   with the newly generated hyperspheres, displaying all the connected action nodes. During episode
239   simulations, L(nodeo ) expands as the action space is refined further using k-center clustering.

268   Motivation about Network Filtering: In POMDPs, many primitives within the continuous sub-
269   action domain Ac may be inefficient or even unreasonable for pursuing long-term goals or even
      short-term ones, like completing a primitive action. If the optimal action a∗ = argmaxa∈Ac Q(b, a)


                                                         5
      Under review as a conference paper at ICLR 2025


270
      lies within Ac , there exists a smaller but more efficient feasible region X ⊂ Ac such that a∗ ∈ X
271   and Q(b, a) is relatively large. An intuitive idea is, given s and α(P), to quickly identify this region
272   X using a score function g(a, s, α(P)) > 0, ∀a ∈ X , which, despite some uncertainty, can be
273   computed efficiently using a Gaussian process (GP), where α(P) is all configuration settings of the
274   problem P. However, in high-dimensional POMDPs, GPs are computationally intensive and limited
275   in applicability. Instead, we aim to use NPs, which are defined as distributions over functions, to
276   estimate the uncertainty in their predictions. Rather than training a complex network for long-term
277   goals, as seen in Q-learning Hausknecht & Stone (2015), with extremely large minimum description
278   length Zhao et al. (2024b) to pursue X = a∗ , we use a simpler scoring network focused on short-
279
      term (even one step), physically meaningful goals. This approach filters out ineffective actions,
      allowing a compact POMDP to select the optimal action based on current beliefs. Inheriting the
280
      advantage of the neural networks, NPs are computationally efficient during training and evaluation,
281
      and also flexible for different inputs. Hence, we use neural processes nn(µ(a), σ(a)) to learn this
282   scoring function, where µ(a) is the mean function and σ(a) is the kernel function.

534   6    C ONCLUSION
535
536   We propose NPF-kCT, a novel POMDP framework and solver for 3D object search with hybrid
537   actions. Combining MCTS, NPs, k-center clustering, and a revised UCB strategy, it selects optimal
538   actions based on maps, photos, and onboard sensors. Simulations and real-world tests show it out-
539   performs classical solvers, achieving higher rewards, fewer steps, and better success rates within the
      same computational resources.

      B.3    O BJECT DETECTOR
811
812   State-of-the-art real-time object detection systems, like YOLO, are commonly designed to divide
813   the objects into different classes and they are not matched with the target images. Meanwhile, we
814   have the 3D point clouds of the objects, which are helpful to divide the objects in the image. So
815   as to complete the given object detection task using several given images and some semantic words
816   (optional), we fuse the traditional feature-matching method and YOLO toolbox to complete the
817   object detection task.
818   Based on the previous point cloud segmentation, we perform it on the current visual local frame and
819   the separated point clouds in the local frame are re-projected to the image to bound the objects in the
820   RGB camera image forming a set of sub-images {Iip , i = 1, 2, · · · , n} using the camera configu-
821   ration and the perspective projection. Similar sub-images of this image {Iiy , i = 1, 2, · · · , m} and
822   their corresponding semantic scores {s̄yi , i = 1, 2, · · · , m} can also be bounded and generated
823   using the YOLOv5 model with pre-trained parameters. Commonly, we have m ̸= n. A simple
824
      data association method with the nearest images and enough common areas is presented to match
      these two sets of sub-images. For the successful data association pair, we use the sub-image in the
825
      local frame as the image corresponding to this object. These sub-images in the detected and asso-
826
      ciated 2D boxes corresponding to different objects are matched with the target object using SIFT
827   descriptor. The rate between the number of matched scale-invariant features and the number of all
828   features is defined as the probability of object detection, denoted {s̄di , i = 1, 2, · · · , n}. If this
829   task offers the target type, like cup and laptop, we use the mean values between the semantic scores
830   {s̄yi , i = 1, 2, · · · , n} and the probability of object detection {s̄di , i = 1, 2, · · · , n}. The main
831   process of the object detector is given in Fig. 11. We find the current object matching is not robust
832   enough in the real world environment and the perception is not our main focused point, so we also
833   add the color matching for the detected object in real world experiments 7 when the offered object
834   detector fails.

      It is easy to know that, in the real-world environment, some objects in the workspace are not move-

852

861   frames Trg (pgi ) based on the pre-visited robot poses pri during the task process. Here, it is noted

1070   POMDP solvers, like POMCP, POMCPOW, and VOMCPOW. The Bellman update naturally follows
1071
       the objective function of the POMDP formulation that aims to pursue optimal action in each step
       of the long-term planning. It helps the solver to explore deeper by focusing its search on promising
1072
       parts of the belief tree. The main challenge for the Bellman backup is when facing unexpected
1073
       observations, a lot of deeply explored belief trees will be frequently cut and this case causes poor
1074   planning performance. Hence, the Bellman backup gets a better performance when the good rewards
1075   are sparse in the belief tree, but it is not stable enough for the poor observation prediction. In contrast
1076   to selecting the reward with optimal action, the Monte-Carlo backup computes the average reward
1077   along with different action episodes, which means that the generated belief tree will be more robust
1078   when facing unexpected uncertainty in received observation. In our application for object search, the
1079   real visual observation is not well predictable for the observation model in the POMDP formulation
       and the robot camera will frequently receive unexpected measurements, which may not be deeply
