                                                A BSTRACT
008
009
010           Define an optimizer as having memory k if it stores k dynamically changing vectors
011           in the parameter space. Classical SGD has memory 0, momentum SGD optimizer
012           has 1 and Adam optimizer has 2. We address the following questions: How can
013
              optimizers make use of more memory units? What information should be stored in
              them? How to use them for the learning steps? As an approach to the last question,
014
              we introduce a general method called ”Retrospective Learning Law Correction” or
015
              shortly RLLC. This method is designed to calculate a dynamically varying linear
016           combination (called learning law) of memory units, which themselves may evolve
017           arbitrarily. We demonstrate RLLC on optimizers whose memory units have linear
018           update rules and small memory (≤ 4 memory units). Our experiments show that in
019           a variety of standard problems, these optimizers outperform the above mentioned
020           three classical optimizers. We conclude that RLLC is a promising framework for
021           boosting the performance of known optimizers by adding more memory units and
022           by making them more adaptive.

      1   I NTRODUCTION
027
028   In this paper, we investigate optimizers that store k vectors in the parameter space Rn of a neural
029   network or more generally in the parameter space related to any optimization problem. We call such
030   vectors memory units and we measure the memory usage of an optimizer by the number them.
031   The simplest example for an optimizer with memory is the momentum SGD optimizer which stores a
032   single vector m (momentum vector) in the parameter space Rn . In each step, m is updated according
033   to the rule m ←− βm + ▽θ f (θ) where f is the objective function, θ ∈ Rn is the parameter vector
034   and 0 < β < 1 is a fixed real number. The vector θ is updated according to the rule θ ←− θ − cm
035   where c > 0 is the learning rate.
036   The Adam optimizer operates with two memory units. One of them is the momentum vector and the
037   other one is the momentum of the squares of the gradient vectors. In contrast with the momentum
038   optimizer, the Adam optimizer is not linear in the gradient vectors. Neither the update rule of the
039   memory units, nor the way the memory units are used for the parameter update is linear.
040
      The present paper has two independent contributions. The first contribution is a novel and simple
041
      method that we call RLLC=Retrospective Learning Law Correction. It is an update rule for a vector
042   L (called learning law) that describes a natural way of using a set of dynamically changing memory
043   units for the update of the parameter vector θ. More precisely, L ∈ Rk contains the coefficients
044   of a linear combination of the k memory units which is multiplied by a fixed learning rate c1 and
045   substracted from θ as usual. In each step, before updating θ and the memory units, we update L by
046   the formula L ←− L + c2 M + g where M is the n × k matrix formed by the memory units, M + is
047   the Moore-Penrose inverse of M , g is the newly received gradient and c2 is the meta learning rate.
048   Note that in practice we use the formula M + = B + M T for calculating M + , where B = M T M
049   is the so-called Gram matrix of the memory units. Since B is a k × k matrix, where k is a small
050   number (at most 4 in our examples), the calculation of B + is a negligible part of the computational
051   load. This means that the computational cost of the RLLC step comes mostly from calculating the
052   matrix products M T M and B + M T , which involves only a few elementary operations per parameter
053   if k is small. Our experiments (see Appendix A.4.2) show that RLLC optimizers with small memory
      have a runtime similar to that of more classical optimizers, such as Adam.

054
      The main idea behind the RLLC update rule for L is that the new gradient g contains retrospective
055   information on how the algorithm could have performed better in the previous step. Thus it can be
056   used to compute a corrected version of L which ”thinks more ahead”. Note that our update rule of the
057   learning law can also be regarded as a general framework for associating a k-dimensional adaptive
058   learning rate with an arbitrary set of k evolving memory units.
059
      As the second main contribution, we examine optimizers in which memory units are updated by fixed
060
      linear rules. More precisely, in each step each memory unit is updated to a linear combination of the
061   memory units and the new arriving gradient. The parameter vector is updated by a (possibly changing)
062   liner combination (given by the learning law L) of the memory units. Such optimizers are interesting
063   even if the learning law is fixed. They include SGD, momentum SGD and Nesterov Accelerated
064   Gradient (NAG) Nesterov (2012). Thus, the linear framework provides a useful generalization of
065   these famous optimizers and enables a dynamically changing continuous interpolation between them.
066   The RLLC method turns out to be ideal for this. Our experiments show that linear memory combined
067   with RLLC leads to powerful optimizers. The case of memory 1 is already interesting. A memory 1
068   linear optimizer stores a momentum vector. Applying RLLC in this trivial setting yields a variant of
069
      the momentum SGD optimizer enhanced with a new type of adaptive learning rate. As the number of
      memory units increases, the mathematics becomes more complex, presenting a field of study that
070
      is interesting in its own right. We present some of the fundamental properties of linearly updated
071
      memory units. In particular, we prove a version of basis independence for RLLC combined with
072   linear memory which allows us to apply basis transformations to the update rules without changing
073   the optimization process. This together with a variant of the Jordan normal form over the field R
074   helps to convert these optimizers into a canonical form in which each memory unit is associated with
075   a so-called Jordan block. A Jordan block of size 1 corresponds to a single memory unit (denoted by
076   M (β)) storing a momentum vector of the gradients with parameter β. A Jordan block of size 2 either
077   corresponds to a pair of memory units (denoted by CM (β), β ∈ C) namely the real and imaginary
078   parts of a momentum vector with complex parameter or to a pair of memory units m1 , m2 (denoted
079   by M2 (β)) where m1 is a momentum vector of the gradient and m2 is a momentum vector of m1 ,
080
      both with parameter β. In general, there are two infinite families of Jordan blocks giving rise to
      k-tuples or 2k-tuples of memory units denoted by Mk (β) and CMk (γ). These are the fundamental
081
      building blocks of linearly updated memory. We denote the natural operation by ⊕ which combines
082
      these building blocks into larger memory by the union of the corresponding memory units. By slightly
083   abusing the notation we often identify memory update rules with optimizers where learning is given
084   by the RLLC method. For example, M (β) also denotes the memory 1 optimizer with memory unit
085   M (β) (a momentum vector) and with RLLC. Notice that the M (β) optimizer is a close relative of
086   momentum SGD but it is not equivalent with it.
087
      In our experiments, we identified a number of interesting simple settings involving few (at most 4)
088
      memory units. These include the types of optimizers M (β), M (β) ⊕ M (0), M (β1 ) ⊕ M (β2 ) ⊕
089   M (β3 ), M2 (β), M3 (β) and M (β)⊕M (−β)⊕CM (βi). We observed that these optimizers often
090   surpassed the performance of three commonly used optimizers across a variety of tasks even without
091   carefully optimizing the parameters βi . Notice that M (β) ⊕ M (0) is an adaptively changing linear
092   combination of SGD, momentum SGD and NAG. Thus, it adaptively interpolates between three
093   well known optimizers (for details see appendix). Remarkably, it demonstrated competitive or even
094   superior performance compared to the Adam optimizer in many tasks, which also uses two memory
095   units.
096   This paper primarily aims not to challenge all existing optimizers in the field, but rather to introduce
097   a novel mathematical concept that could spark further research. The experimental results presented
098   here should be interpreted as an illustration of the potential of our approach. We posit that the impli-
099   cations of the RLLC method extend beyond mere enhancements to current optimization techniques,
100   suggesting broader applications and insights in the realm of optimization and machine learning.

106   Similar to our RLLC method and our framework of linear optimizers, various other optimizers
107   (including classical ones such as NAG Nesterov (2012) and Adam Kingma & Ba (2014)) are based
      on storing vectors in the parameter space to enhance performance. A more recent work in this area is

108
      McRae et al. (2022), titled ”Memory Augmented Optimizers for Deep Learning.” The main novelty
109   of this paper is its method for selecting, storing, and using a list of ’critical gradients.’
110
111   Our method has a close connection to adaptive learning rate methods Wang et al. (2022); Keskar
112
      & Socher (2017b), as highlighted by the fact that the RLLC method used for a single memory unit
      is equivalent to a new type of adaptive learning rate. Other notable papers in this area include the
113
      following: AdaBound Luo et al. (2019), which combines the benefits of adaptive methods and SGD
114
      by dynamically bounding the learning rates during training, aiming for fast convergence and improved
115   generalization; AdaBelief Zhuang et al. (2020), which adapts step sizes based on the ”belief” in
116   observed gradients by comparing them to an exponential moving average of past gradients, enhancing
117   both convergence speed and generalization; and DiffGrad Dubey et al. (2020), which adjusts learning
118   rates based on the differences between the current and immediate past gradients, emphasizing updates
119   where the gradient changes rapidly.
120
      Our work is also related to the broader field of metaoptimization, where the optimization methods
121   themselves are optimized, often through learned strategies. In this context, the main approach is
122   training an auxiliary model, often a neural network, to optimize the primary model tasked with
123   solving the original problem. Some of these methods leverage traditional gradient-based optimizers
124   (Andrychowicz et al., 2016; Metz et al., 2022b; Bengio et al., 1991; Wichrowska et al., 2017), while
125   others explore alternative methods such as evolutionary algorithms (Bengio et al., 1991; Metz et al.,
126   2020) or reinforcement learning strategies (Li & Malik, 2017; Bello et al., 2017). The goal of
127   these methods is to design optimizers through learned processes, which can outperform standard,
128   hand-designed optimizers across a wide variety of tasks.
129   Finally we mention a recent and independent paper Pagliardini et al. (2024), which introduces an
130   update to the Adam optimizer by replacing its momentum vector with a (typically fixed) linear
131   combination of two momentum vectors with different decay parameters. The idea of combining
132   multiple momentum vectors with different decay parameters is also an important idea in our paper.
133   However, the main novelty of our work lies in the new methodology, called the RLLC method, which
134   adaptively adjusts the linear combination (learning law) of the momentum vectors (or any other useful
135
      vectors). Our RLLC method is essentially a gradient descent algorithm (with a dynamically changing
      objective function) in the space of potential learning laws, utilizing knowledge from the previous
136
      training step. This meta-learning style adaptive nature is one of the key differences between our work
137
      and the previously mentioned papers.

142   Functional approach to optimizers: The RLLC method is presented through an abstract mathemat-
143   ical framework for optimizers. This framework is somewhat specialized, yet it maintains sufficient
144
      generality to encompass a range of interesting optimizers. We think of optimizers as entities with an
      evolving internal state that updates at each step based on newly received gradients. Additionally, the
145
      optimizer calculates a parameter update vector relevant to the optimization process. A functional
146
      description of such an optimizer is given in the following definition.

204   Explanation of RLLC and remarks:
205   The idea behind the learning rule update is that with the arrival of the new gradient gt the optimizer
206   gains new (retrospective) information on how it could have done better in the previous learning step.
207   Notice that the vector M + gt is the coefficient vector of the orthogonal projection of gt to the space
208   spanned by the memory units when written as a linear combination of the memory units. This means
209   that, if performed with the new law, the outcome of the parameter update in step t − 1 would have
210   been θt − c1 c2 pt instead of θt where pt is the orthogonal projection of gt to the space spanned by the
211   memory units in the t − 1-th step. Notice that (pt , gt ) = (pt , pt ) ≥ 0 and thus the change −c1 c2 pt
212
      points in a direction which improves the objective function.
213   The above heuristics does not take it into account that the objective function ft is also changing. This
214   fact indicates that our update rule is more justified if the second learning rate c2 is small and thus
215   random effects have time to average out leaving only useful directions in the update. Also notice that
      the algorithm does not ”go back in time” to perform the improved learning step. Instead it applies the

219   improvements of the future. For this reason the choice of the memory update rule is a crucial issue
220   which is one of the main topics of the second part of this paper.

531
      Our experiments demonstrate that the RLLC method is capable of boosting the performance of
532
      classical optimizers (such as SGD and momentum SGD) by combining them and making them more
533
      adaptive. Furthermore the case of linearly updated memory units provides a mathematically elegant
534   framework with many new types of promising optimizers such as the ones corresponding to larger
535   Jordan blocks, complex numbers and their combinations. We regard this paper as a starting point for
536   future research in the frame of which the full potential of our approach is explored. One possible
537   research direction is to introduce adaptively changing memory update rules. In particular, in the
538   linear setting the pair B ∈ Rk×k , a ∈ Rk (see Section 4) is fixed for the whole optimization process
539   in the current version. It would be interesting to study a version were B and a are also adaptively
      changing throughout learning.

      learning steps.

      A.2     RLLC INTERPOLATIONS BETWEEN SGD, MOMENTUM SGD AND N ESTEROV
703           ACCELERATED G RADIENT (NAG)
704
705   As we have already explained, momentum SGD method in our interpretation is represented as the
706   propagator M (β) where 0 ≤ β < 1 is the decay parameter. In particular M (0) corresponds to a
707   memory unit which stores the last gradient seen by the optimizer. In this sense, a memory 1 optimizer
708   with memory unit M (0) is basically an SGD optimizer. The learning law in this case is a single real
709   number which manifests as a learning rate.
710   If an optimizer has two memory units M (β) and M (0) then by changing the learning law (described
711   by a pair of real numbers) we can continuously interpolate between momentum SGD, pure SGD and
712   NAG. In the next list we summarize the meanings of special learning laws for M (β), M (0).
713
714         1. (0, c) : SGD
715         2. (c, 0) : momentum SGD with decay parameter β
716
            3. (cβ, c) : NAG with decay parameter β
717
718
      where the learning rate is c ∗ lr. Notice that since β is a prescribed fix number, the above three
719   cases don’t cover all possible learning laws for the pair M (β), M (0). This enables the LM-RLLC
720   optimizer with memory M (β), M (0) to find interesting interpolations between these three classical
721   optimizers.

743   deeper explanation relates this optimizer to another one of the form M (0.9) ⊕ M2 (0.75). More
744
      precisely if we consider O(ϵ) = M (0.9) ⊕ M (0.75 + ϵ) ⊕ M (0.75 − ϵ) we find that as ϵ goes to 0
      the subspace spanned by the abstract rules of the memory units converges to the subspace spanned
745
      by the memory units of M (0.9) ⊕ M2 (0.75). In theory this convergence means that the optimizers
746
      themselves converge. Note that in practice we can not model M (0.9) ⊕ M2 (0.75) by O(ϵ) because
747   numerical instability arises if ϵ is very small.
