009                                                A BSTRACT
010
011            Synthesizing relational data has started to receive more attention from researchers,
012            practitioners, and industry. The task is more difficult than synthesizing a single-
013            table due to the added complexity of relationships between tables. For the same
014            reason, benchmarking methods for synthesizing relational data introduces new
015            challenges. Our work is motivated by a lack of an empirical evaluation of state-
016            of-the-art methods and by gaps in the understanding of how such an evaluation
017            should be done. We review related work on relational data synthesis, common
018            benchmarking datasets, and approaches to measuring the fidelity and utility of
019
               synthetic data. We combine the best practices and a novel robust detection ap-
               proach into a benchmarking tool and use it to compare six methods, including two
020
               commercial tools. While some methods are better than others, no method is able
021
               to synthesize a dataset that is indistinguishable from original data. For utility, we
022            typically observe moderate correlation between real and synthetic data for both
023            model predictive performance and feature importance.

      1   I NTRODUCTION
027
028
      Synthesizing relational data - generating relational data that preserve the characteristics of the orig-
029
      inal data - is an emerging field. It promises several benefits, from protecting privacy to addressing
030   data scarcity while preserving the complexities and inter-dependencies present in the original data.
031   This makes it attractive for domains such as healthcare (Appenzeller et al., 2022), finance (Assefa
032   et al., 2020), and education (Bonnéry et al., 2019), where accessing and utilizing data can be chal-
033   lenging due to privacy concerns, data scarcity, or biases (Ntoutsi et al., 2020; Rajpurkar et al., 2022).
034
      The foundations of synthesizing relational data were laid by the Synthetic Data Vault (Patki et al.,
035
      2016). Recently several deep learning methods have been proposed (Gueye et al., 2023; Li & Tay,
036   2023; Mami et al., 2022; Xu et al., 2023; Canale et al., 2022; Solatorio & Dupriez, 2023; Pang
037   et al., 2024). The field has also received attention from industry, with several commercial tools
038   now available and with Google, Amazon, and Microsoft integrating them into their cloud services
039   (Gretel.ai, 2024).
040
      While there are several packages for evaluating the quality of synthetic data, only the SDMetrics
041
      package (Patki et al., 2016) provides some support for the evaluation of synthetic relational data. As
042   such, the field lacks not only an empirical comparison of available methods but also an understand-
043   ing of how such an evaluation should be done. We address this gap with an evaluation methodology
044   that combines established evaluation metrics (Section 2.2), best practices, sampling procedures for
045   relational data (Section 2.3), and a novel metric that improves on existing approaches and general-
046   izes to relational data (Section 3). We implement the methodology in a benchmarking tool that is
047   available as an open source package and can be easily extended. Finally, we use the benchmark for
048   current state-of-the-art methods (Section 2.1) on several relational data sets (Section 2.3). This is
049   the first comprehensive evaluation and comparison of methods for synthesizing relational data and
050
      provides valuable insights into their ability to synthesize relational aspects of the data (Section 4).
051   Our benchmark reveals that lenient evaluation practices in related work have led to sub-par fidelity in
052   single-table generation. Furthermore, where individual tables are synthesized well, current methods
053   struggle to faithfully model the relationships between them. We highlight these gaps and offer a
      robust evaluation tool to guide and assess future advancements in synthetic relational data.

      2.1     M ETHODS FOR SYNTHESIZING RELATIONAL DATA
057
058   In this work we focus on relational data - a collection of tables connected by foreign keys that form
059   a relational database. We distinguish this from synthesizing tabular data (a single-table), which is a
060   special case and an even more active field (Borisov et al., 2022; Qian et al., 2023b). Here we briefly
061   summarize the methods. A detailed description can be found in Appendix A.
062
      The Synthetic Data Vault (SDV) uses Gaussian copulas and predefined distributions to model rela-
063
      tional data. Row Conditional-TGAN (RC-TGAN) (Gueye et al., 2023) and Incremental Relational
064   Generator (IRG) (Li & Tay, 2023) are based on GANs. The Realistic Relational and Tabular Trans-
065   former (REaLTabFormer) (Solatorio & Dupriez, 2023) and Composite Generative Models (Canale
066   et al., 2022) are based on transformers. The work of Mami et al. (2022) is based on Graph Vari-
067   ational Autoencoders, while Xu et al. (2023) propose a framework for synthesizing many-to-many
068   datasets using random graphs. Recently, Pang et al. (2024) propose ClavaDDPM, a method based
069   on classifier-guided diffusion models.
070
071   2.2     M ETRICS FOR E VALUATING S YNTHETIC DATA
072
073   The two main aspects for evaluating the quality of synthetic tabular and relational data are fidelity
074   and utility. Fidelity measures the degree of similarity between synthetic and real data in terms of its
075   properties, whereas utility measures how well the synthetic data can replace real data when the data
076
      are part of some tasks, for example, for predictive modeling (Hansen et al., 2023). We further divide
      fidelity metrics into statistical, distance-based, and detection-based metrics. Utility of synthetic
077
      data is typically assessed with train-on-synthetic evaluate-on-real methods (Beaulieu-Jones et al.,
078
      2019).
079
080   Another dimension of evaluation metrics for relational data is granularity. The most common are
081   single-column metrics that evaluate the marginal distributions, two-column metrics that evaluate
082
      bivariate distributions, single-table metrics that evaluate tables, and multi-table metrics that evaluate
      the relational aspects.
083
084
085   2.2.1    S TATISTICAL F IDELITY
086
      Statistical fidelity methods are typically used to assess marginal distributions, sometimes bivariate
087   distributions. The most commonly used methods are the Kolmogorov-Smirnov test and the X 2 test
088   for numerical and categorical variables, respectively. For relational data, cardinality shape similarity
089   is used, where for each parent row the number of child rows is calculated. This yields a numerical
090   distribution for both real and synthetic data, on which a Kolmogorov-Smirnov test is performed.
091
092   2.2.2    D ISTANCE - BASED F IDELITY
093
094   Similar to statistical fidelity, distance-based fidelity is typically used to assess the fidelity of marginal
095   distributions. However, some distance metrics also assess entire tables. Commonly used distance-
096   based methods are total variation distance, Kullback-Leibler divergence, Jensen-Shannon distance,
097
      Wasserstein distance, maximum mean discrepancy, and pairwise correlation difference. Unlike sta-
      tistical methods, reports of distance-based fidelity do not include hypothesis testing or any other
098
      quantification of uncertainty. This is an issue both when evaluating a method and when compar-
099
      ing two methods. In the former, a method can achieve a seemingly high distance that is in a high
100   probability region when taking into account the sampling distribution. In the latter, a seemingly
101   large difference between the two methods can be explained away by the variance of the sampling
102   distribution.
103
104   2.2.3    D ETECTION - BASED F IDELITY
105
106   The basic idea of detection-based fidelity is to learn a model that can discriminate between real and
107   synthetic data. If the model can achieve better-than-random predictive performance, this indicates
      that there are some patterns that identify synthetic data. Recent work by Zein & Urvoy (2022)


                                                           2
      Under review as a conference paper at ICLR 2025


108
      shows that using discriminative models can highlight the differences between real and synthetic
109   tabular data.
110
111   The most common detection-based method is logistic detection (LD) (Gueye et al., 2023; Solatorio
112
      & Dupriez, 2023; Li & Tay, 2023; Pang et al., 2024), where a logistic regression model is used for
      discrimination. An extended version of LD known as parent-child logistic detection (P-C LD) is used
113
      to evaluate relational data. P-C LD applies LD to denormalized pairs of synthetic parent and child
114
      tables, assessing the preservation of parent-child relationships. A serious issue with denormalization
115   is that it may introduce correlation between rows, breaking the i.i.d. assumption. This results in an
116   over-performance of the discrimintative model and in underestimating the quality of the method
117   for synthesizing relational data. It also makes it impossible to set a detection threshold for testing
118   fidelity (for example, accuracy would be greater than 50% even if both datasets were from the same
119   data generating process). For these reasons, we do not consider P-C detection.
120
      Note that logistic regression is unable to capture interactions between columns unless these interac-
121   tions are explicitly included as features. Therefore, LD is unable to discriminate between real and
122   synthetic data when the marginal distributions are synthesized well. Furthermore, a mean-preserving
123   transformation can produce synthetic data that LD will not be able to discriminate, although the
124   synthetic data will be very different from the original data. We demonstrate this empirically in
125   Appendix D.1. The popularity of LD implies a lenient evaluation of the state-of-the-art methods.
126   Tree-based ensemble models are a better alternative, which is also suggested by the findings of Zein
127   & Urvoy (2022) for tabular data.
128
129   2.2.4    M ACHINE L EARNING U TILITY
130
131   The utility of synthetic data is most commonly measured with machine learning efficacy (ML-E)
132   - comparing the hold-out performance of a predictive model trained on the original data with a
133   predictive model trained on a synthetic dataset (Canale et al., 2022; Li & Tay, 2023; Mami et al.,
134
      2022; Solatorio & Dupriez, 2023; Pang et al., 2024). Patki et al. (2016) measured utility with a user
      study and Hansen et al. (2023) with the ability to retain model ranking or feature importance ranking
135
      (measured with rank correlation) in the train-on-synthetic evaluate-on-real paradigm. It is important
136
      to highlight that all these studies evaluated utility on a single-table, even those that investigated
137   synthetic relational data.
138
139   Note that the typically used unweighted rank correlation (for example, Spearman or Kendall correla-
140
      tion coefficients) could be misleading. The issue gets worse as we increase the number of models or
      features, and their ordering becomes more susceptible to noise, especially among the models close
141
      to optimal performance and irrelevant features. That is, unweighted ranking will be most affected
142
      by the ranking of models and features in areas where ranking is of little practical utility.
143

145   2.3     R ELATIONAL DATASETS AND S AMPLING P ROCEDURES
146
147
      We organize the datasets used in related work based on the structure of their relational schema, de-
      fined in Section 3. Datasets using only linear relationships (one parent and one child table) include
148
      AirBnB (Montoya et al., 2015) and Rossmann Store Sales (FlorianKnauer, 2015). While this struc-
149
      ture may be sufficient for some practical applications, Gueye et al. (2023) and Xu et al. (2023) high-
150   light the need for methods supporting more complex, multiple-parent relational structures found in
151   datasets like MovieLens (Harper & Konstan, 2015) and World Development Indicators (World Bank,
152   2019). Datasets including multiple child tables include Telstra Network Disruptions (Wendy Kan,
153   2015), Walmart Recruiting - Store Sales Forecasting (Walmart, 2014), and Mutagenesis (Debnath
154   et al., 1991). Datasets with multiple children and parents include Coupon Purchase Prediction (Kato
155   et al., 2015), World Development Indicators (World Bank, 2019), MovieLens (Harper & Konstan,
156   2015) and Biodegradability (Blockeel et al., 1999). An additional possibility in relational databases
157   is the use of composite foreign keys, which only the IRG (Xu et al., 2023) method supports. One
158
      such dataset is the Grants database (Alawini et al., 2018).
159   An important issue with evaluating relational data is that representative sampling is difficult (Buda
160   et al., 2013; Gemulla et al., 2008). If the dataset does not include a time component or if the
161   relationships are non-linear, the sampling becomes non-trivial and directly impacts the performance
      of the generative method. Even if the data have a strict hierarchy between tables, the rows in a child


                                                        3
      Under review as a conference paper at ICLR 2025


162
      table are related via their parent, which breaks the assumption of i.i.d. sampling. Typically, the
163   method for synthesizing relational data is trained using the entire original dataset.
164
165   Note that a benchmark for relational learning based on graphs was recently proposed by Fey et al.
166
      (2023). It includes a collection of relational datasets along with machine learning tasks with defined
      train, evaluation, and test splits. However, these datasets include modalities such as text, which are
167
      not supported by the generative models evaluated in this work.

      3   A G ENERAL A PPROACH TO F IDELITY WITH D ISCRIMINATIVE D ETECTION
171
172
173   In this section, we propose discriminative detection (DD), a generalization of the detection-based
174   approach to fidelity, and its extension to relational data using aggregation (DDA). We are primarily
175
      motivated by the issues of existing approaches to multi-table fidelity, cardinality shape similarity (see
      Section 2.2.1) and P-C LD (see Section 2.2.3), and the subsequent need to strengthen the testing of
176
      this aspect in our benchmark. However, as we show, DD also improves on existing approaches to
177
      single-column and single-table fidelity.
178
179   Fidelity methods are concerned with measuring the similarity between two databases with the same
180   schema but different data. Typically, these will be the real database DREAL and a synthetic database
181
      DSYN , with the goal of detecting whether, to what extent, and where the synthetic data differ from
      the real data.
182
183   Let a relational database be a collection of tables T = {T1 , ..., Tn } and a schema S = (R, A),
184   where R ⊆ T × T are the relations between the tables and ATi = {aT1 i , . . . , aTl i } ∈ A define the
185   tables’ attributes. Each table is a set T = {v1 , ..., vnT } consisting of elements vi called rows. Each
186   row v ∈ T has three components v = (pv , Kv , xv ). A primary key pv that uniquely identifies the
187   row v; the set of foreign keys Kv = {pv′ : v ′ ∈ T ′ and (T, T ′ ) ∈ R}, where pv′ is the primary key
188   of the row v ′ ; and the set of values xv = {(a, x) : a ∈ AT } corresponding to attributes of table T .
189   DD is summarized in Algorithm 1. It can be used for single-table, multi column, or single-column
190   fidelity, which we determine by selecting target table and the subset of target columns. We then
191   combine the two datasets and label the real and synthetic observations. From this point onwards,
192   DD can be interpreted as a classification task of discriminating between real and synthetic observa-
193   tions. First, we use the selected classifier and error estimation procedure to estimate generalization
194   accuracy. Then we use a Binomial test for proportion to test the deviation from baseline accuracy.
195
      Any better than random predictive performance implies a deviation from perfect fidelity. If a devia-
      tion is detected, we can optionally use an interpretability method to provide additional insight into
196
      where the classifier is able to distinguish between real and synthetic data.
197
198   In practice, we have to choose a classifier, an interpretability method for our classifier (optional),
199   and a procedure for estimating accuracy. In our experiments we achieved good results with common
200   choices of XGBoost, built-in feature importance, and cross-validation (see Section 4 for details).
201
      However, we could also consider multiple classifiers and perform model selection.

      3.1   J USTIFICATION AS A T WO S AMPLE T EST
217
218   DD can be interpreted as a null-hypothesis test for comparing two distributions (two sample testing)
219   with classification accuracy as a proxy. The classifier serves as a map from high-dimensional data
220   to a one-dimensional test statistic.
221
      Using ML models is a common approach to two sample testing of high-dimensional data, with
222   methods such as maximum mean discrepancy (Gretton et al., 2012) also used for single-table fidelity
223   of synthetic data. Using predictive performance as a proxy is less common, but it has been receiving
224   more attention (see Lopez-Paz & Oquab (2017), Kim et al. (2021) and Snoke et al. (2018)).
225
      It has been shown that the accuracy-based approach to two-sample testing is consistent and controls
226
      for Type I error and (asymptotically) Type II error (see Kim et al. (2021) for theoretical results
227
      and a summary of empirical results). In practice, we are also interested in finite sample behavior.
228   Experiment-based recommendations show that the approach should have an advantage in power
229   when the data are well-structured or we have a lot of data, or when it is difficult to specify a test
230   statistic, which is very common for high-dimensional data. Therefore, the large, higher-dimensional
231   and structured nature of relational data is a perfect fit for DD.
232
233   3.2   M ULTI - TABLE F IDELITY U SING AGGREGATION
234
235   Discriminative detection with aggregation (DDA) extends DD to multi-table fidelity by augmenting
236   the table with columns that aggregate information from child tables. Aggregation is an established
237
      technique in the field of relational reasoning (Getoor et al., 2007; Džeroski, 2010) and DDA can be
      thought of as a propositionalization (Kramer et al., 2001) approach to the C2ST on relational data.
238
      In DDA we replace the column selection in rows 1 and 2 of Algorithm 1 with calls to the aggregation
239
      algorithm described in Algorithm 2:
240
       1: TREAL ← RelationalAggregation(DREAL , i)
241
       2: TSYN ← RelationalAggregation(DSYN , i)
242
243   For each child table we add CountRows, a count of the the number of child rows corresponding to a
244   parent row. For each attribute in each child table, we compute an aggregation attribute (mean, count,
245   etc.). The aggregation attributes are added to the target table. In practice, different aggregation
246   functions may be applied, as long as they maintain the i.i.d. assumption of the data.
247
248   3.3   A DVANTAGES OVER R ELATED W ORK
249
      DD generalizes LD in two ways: by allowing for any classifier and by wrapping detection into a sta-
250
      tistical testing framework that simplifies decision making. Aggregation (DDA) further generalizes
251
      the approach to multi-table fidelity and addresses the issues of the two methods that are commonly
252   used for multi-table fidelity: cardinality shape similarity, which focuses on a very specific aspect,
253   and Parent-Child LD, which suffers from the issues of denormalization.
254
255
      DD (and DDA) can also be used to detect data copying. For example, if tables A and B are perfect
      copies, some training observations from A will have their corresponding copies in the test set of B
256
      (and vice versa). If any pattern is learned on these, the test data will have the labels reversed, and
257
      accuracy can drop below 21 . Instead of accuracy, LD typically uses 2 · max(AUC, 21 ) − 1, based on
258
      the popular implementation (DataCebo, 2022). Limiting to a minimal value of 12 can therefore mask
259
      data copying (see Appendix D.2 for an example).

262   4     B ENCHMARKING AND R ESULTS
263
264   We combine our findings into a synthetic relational data benchmark, including single-column,
265   single-table and multi-table fidelity metrics, as well as machine learning utility metrics.
266   We compare the following methods for synthesizing relational data: SDV, RC-TGAN, REaLTab-
267   Former, and ClavaDDPM. Other related work does not have an API or available source code or
268   we were not able to run the source code. We also included two of the most popular commercial
269   tools, MostlyAI and GretelAI. The former do not disclose their generative method, while the latter
      provide two generative models TabularLSTM and ACTGAN. As a baseline for single-column and

270
      single-table comparison, we also include state-of-the-art single-table methods (see Appendix D.5
271   for details).
272
273   We include 5 datasets that feature in related work (AirBnB, Rossmann, Walmart, Biodegradabil-
274
      ity, MovieLens) and the Cora dataset by McCallum et al. (2000), a popular dataset in graph rep-
      resentation learning. The datasets vary in types of relationships and number of tables and columns
275
      (see Appendix B.2 for details).
276
277   Our evaluation focuses on all three levels of synthetic relational data generation, with a focus on
278   multi-table evaluation (see Appendix B.1 for details). For statistical metrics, confidence intervals
279   and p-values are readily available, and for detection methods we use a binomial model. For distance-
280
      based metrics we use bootstrapping (1000 bootstrap replications) to approximate the sampling dis-
      tribution. For the purposes of this evaluation, we consider a method failed to achieve fidelity if the
281
      difference between original and synthetic data is significant at level α = 0.05. Most methods are
282
      non-deterministic, so we report results for three different replications. However, all results are stable
283   across replications.
284
285   We use DD with either logistic regression or XGBoost and for DDA we augment the rows with (a)
286
      counts of child rows for each row in each parent table, (b) the mean values of the numeric columns
      in the child table corresponding to the parent row, and (c) the number of unique categories in related
287
      rows. We use 10-fold stratified cross-validation to estimate DD accuracy.

500   5     C ONCLUSION
501
502
503   We surveyed methods for synthesizing relational data and provided a critical review of approaches
504   to evaluating the fidelity and utility of synthetic data. We integrated our findings into the first bench-
505   mark tailored to evaluating relational synthetic data (see Appendix B.3 for a comparison with related
506
      tools). Our work is available as a Python package (URL anonymised and the work included as sup-
      plementary material). that can be easily extended with new methods, metrics, and datasets.
507
508   We introduced DD, a generalization of detection-based approaches to fidelity, based on framing
509   the problem as a classification task. Compared to commonly used statistical and distance-based
510   approaches, we have the additional choices of classifier and, for multi-table fidelity, engineering
511   additional features. However, empirical results show that DD outperforms other approaches even
512
      with basic additional features and XGBoost. The approach can be applied to single-column, multi
      column, single-table, or, with aggregation (DDA), multi-table fidelity. Worse-than-random perfor-
513
      mance of the discriminative model is also a viable diagnostic for data-copying. Finally, we demon-
514
      strate how, by explaining the predictions of the discriminative model, we can gain additional insights
515   into which aspects of the original data were not synthesized well. We argue that DDA is a viable
516   one-size-fits-all approach for investigating the fidelity of synthetic data.
517
518
      We used our benchmark for the first comprehensive evaluation and comparison of the state-of-the-art
      methods for generating synthetic relational data. Methods are not yet able to generate synthetic rela-
519
      tional data that is indistinguishable from original data. Most methods have problems with marginal
520
      distributions at least on some benchmark datasets and with single-tables on most datasets. None of
521   the methods capture the relational properties of the original data, which results in relatively poor
522   fidelity and utility. We highlight this as an important direction for future work on relational data
523   synthesis (see Appendix 5.1 for limitations of the study and directions for future work).
524
525
526   5.1   L IMITATIONS AND F UTURE W ORK
527
528   Our work focused on fidelity and utility, but not privacy. While we do briefly touch upon one as-
529   pect of privacy - data copying - we delegate the research of privacy metrics for synthetic relational
530   data to future work. More work needs to be done in understanding the relationship between model
531   quality and feature importance and practical utility. Unweighted rankings are flawed and it is not
532   clear what weighting should be used or if metrics of this type are even a practically relevant utility
533   measure. Finally, several aspects of synthetic data evaluation are limited by the difficulty of repre-
534   sentative sampling. More work needs to be done in understanding the limitations and preparing new
535   benchmark datasets or dataset generators.
536   Our results reveal significant gaps in multi-table fidelity. However, utility metrics on some datasets
537   show performance comparable to real data, even when fidelity tests fail, highlighting the practical
538   value of the generated data. To improve fidelity, future methods should focus on the relational
539   aspects, with graph representation learning on relational data (Fey et al., 2023) showing promise for
      both generative modeling and a general approach for evaluating multi-table utility.

      A    A S URVEY OF S YNTHETIC R ELATIONAL DATA G ENERATION M ETHODS
706
      The Synthetic Data Vault (SDV) (Patki et al., 2016) introduced the first learning-based method
707
      for generating relational data. The method is based on the Hierarchical Modeling Algorithm (HMA)
708
      synthesizer, which is a multivariate version of the Gaussian Copula method. The method converts all
709   columns to a predefined set of distributions and selects the best-fitting one. To learn dependencies,
710   columns are converted to a standard normal before calculating the covariances. Tables are modeled
711   with a recursive conditional parameter aggregation technique, which incorporates child table covari-
712   ance and column distribution information into the parent table. The method requires the relational
713   structure or metadata, which has since become a common practice.
714
      The work of Mami et al. (2022) leverages the graph representation of relational data using Graph
715   Variational Autoencoders. They focus on the case of one primary table connected by an identi-
716   fier to an arbitrary number of secondary tables. The approach begins by transforming categorical,
717   datetime, and numeric attributes into a normalised numeric format using an invertible function.
718   Subsequently, all tables’ attributes are merged into a single-table, where rows from each table are
719   vertically concatenated. This merged table, along with an adjacency matrix based on foreign key
720   relations, forms a homogeneous graph representation of the dataset. Message passing is then ap-
721   plied to this graph representation using gated recurrent units (GRU). Following the message passing
722   phase, the data is processed through a variational autoencoder, which encodes the joined table and
723
      random samples are taken from its latent space. These samples are then decoded back to the data
      space.
724
725   Composite Generative Models (Canale et al., 2022) propose a generative framework based on
726   codecs for modeling complex data structures, such as relational databases. They define a codec as
727   a quadruplet: C = (E,D,S,L), consisting of an encoder E producing embeddings and intermediate
728   contexts, a decoder D for distribution representation, a sampler S and loss function L. The authors
729
      define the following codecs: Categorical and Numerical Codecs for individual columns, while com-
      posite data types are encoded using Struct and List Codecs, allowing for relational data synthesis.
730
      They also propose a specific implementation using causal transformers as generative models.
731
732   The Row Conditional-TGAN (RC-TGAN) (Gueye et al., 2023) extends the conditional tabular
733   GAN model (Xu et al., 2019) to relational data. RC-TGAN incorporates data from parent rows into
734   the child table GAN model, allowing it to synthesise data conditionally on the connected parent table
735
      rows. The ability for conditional synthesis allows the method to handle various relationship schemas
      without additional processing. They enhance RC-TGAN to capture the influence of grandparent
736
      rows on their grandchild rows, preserving this connection even when the relationship information is
737
      not transferred by the parent table rows. Database synthesis is based on the row conditional generator
738   of RC-TGAN model trained for each table. First, all parent tables are synthesised, followed by
739   sampling the tables for which parents are already sampled. This allows using the synthesised parent
740   rows as features when synthesizing child table rows.
741
      The Incremental Relational Generator (IRG) (Li & Tay, 2023) uses GANs to incrementally fit
742
      and sample the relational dataset. They first define a topologically ordered sequence of tables in the
743
      dataset. Parent tables are modeled individually, while child tables undergo a three-step generation
744   process. First, a potential context table is constructed by combining data from all related tables
745   through join operations and aggregation. Then, the model predicts the number of child rows to be
746   generated for each parent row, which they call its degree. They then extend the context table with
747   corresponding degrees. Taking this table as context, they use a conditional synthetic tabular data
748   generation model to generate the child table.
749
      The Realistic Relational and Tabular Transformer (REaLTabFormer) (Solatorio & Dupriez,
750   2023) focuses on synthesizing single parent relational data and employs a GPT-2 encoder with a
751   causal language model head to independently model the parent table. The encoder is frozen after
752   training and used to conditionally model the child tables. Each child table requires a new conditional
753   model, implemented as a sequence-to-sequence (Seq2Seq) transformer. The GPT-2 decoder with a
754   causal language model head is trained to synthesise observations from the child table, accommodat-
755   ing arbitrary-length synthetic data conditioned on an input. While this method supports conditional
      synthesis of child rows, only one level is supported by this method.


                                                       14
      Under review as a conference paper at ICLR 2025


756
      Xu et al. (2023) propose a method for modeling many-to-many (M2M) datasets via random graph
757   generation. They leverage a heterogeneous graph representation of the relational data and propose
758   a factorization for modeling the graph representation incrementally. First, the edges of the graph
759   are generated unconditionally using a random graph model. Second, one of the tables is generated
760   conditionally on the topology of edges. One way to achieve such conditioning is by using a node
761   embedding. Lastly, the remaining tables are generated using the conditional table model, which
762   requires the generation of each node of the table based on the currently generated tables and all
763   connections. They achieve this by using set embeddings to conditionally generate connected ta-
764   bles. The authors propose two variants using different conditional table models BayesM2M and
765
      NeuralM2M.
766   Privacy-preserving graphical models with latent variables. (PrivLava) (Cai et al., 2023) synthe-
767   sizes relational databases with foreign key dependencies under differential privacy (DP). PrivLava
768   models each foreign key in a relational schema as a separate graphical model, incorporating latent
769   variables to capture inter-relational dependencies. Each entity in a child table associated with a
770   parent table is modeled using a latent variable representing characteristics of the relationship. The
771
      approach handles foreign key relationships by treating them as a directed acyclic graph (DAG). It
      incrementally models the tables following a topological order, beginning with root tables and then
772
      moving on to tables that depend on them. This ensures that each synthetic row in child tables is
773
      conditionally generated based on latent features of related parent rows. Noise is injected at various
774   stages to achieve DP guarantees.
775
776   The Cluster Latent Variable guided Diffusion Probabilistic Models (ClavaDDPM) (Pang et al.,
777
      2024) utilizes classifier-guided diffusion models, integrating clustering labels as intermediaries be-
      tween tables connected by foreign-key relations. The authors first propose a model for generating a
778
      single parent-child relationship. The connection between the tables is modeled by a latent variable
779
      obtained using Gaussian Mixture Model clustering. ClavaDDPM learns a diffusion process on the
780   joint parent and latent variable distribution, followed by training a latent variable classifier on the
781   child table to guide the diffusion model for the child table. Additionally, it includes a model to
782   estimate child group sizes, to preserve relation cardinality. The authors then extend this to more
783   parent-child constraints through bottom-up modeling and address multi-parent scenarios by em-
784   ploying majority voting to mitigate potential clustering inconsistencies. Despite strong performance
785   on our benchmark a key limitation of the method is its inability to generate datasets with multiple
786   relationships between pairs of tables.

850   B.1.2       S EPARABILITY OF S YNTHETIC AND O RIGINAL DATA
851
      Statistical metrics report the underlying statistic and p-value. We decide if the metric was able to
852
      separate synthetic data from the original data if the p-value is less than the significance level α,
853   which in our case is 0.05.
854
855   Distance metrics must report the metric value, the support of the values the metric can obtain and
856
      the goal (minimization or maximization of the metric). Depending on the support and goal, a boot-
      strap confidence interval is constructed, which can be asymmetric depending on the support. The
857
      separability of the original and synthetic data is decided based on the 1 − α confidence interval. If
858
      the metric value falls outside of the confidence interval, the metric is able to differ between real and
859   synthetic data.
860
861   Detection metrics report the classification accuracy, however it can be replaced with any classifica-
862
      tion metric. The separability of the data is determined using a one-sided binomial test for propor-
863   tions, assuming a probability parameter of max(n,m)
                                                     n+m     (where n, m are numbers of rows for real and
      synthetic datasets respectively) for both groups, which indicates complete inseparability of the data.

1113   D.2   D ISCRIMINATIVE D ETECTION AS A DATA C OPYING D IAGNOSTIC
1114
1115   In this section we investigate how discriminative detection can be used to diagnose data copying. We
1116   also demonstrate how the classifier performance commonly reported in LD (2 · max(AUC, 12 ) − 1)
1117   masks this issue. As in the previous experiment we simulate a perfect synthetic generating a dataset
1118   by splitting the original table in half. However, instead of introducing corruption into the second
1119   half, we create an exact copy of the original data (i.e., the first half). The commonly used LD
1120   implementation fails to detect data copying and assigns the copied data a perfect score. In contrast,
1121
