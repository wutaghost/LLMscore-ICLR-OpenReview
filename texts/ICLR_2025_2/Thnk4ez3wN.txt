009                                                        A BSTRACT
010
011              Dataset distillation generates a small set of information-rich instances from a large
012              dataset, resulting in reduced storage requirements, privacy or copyright risks, and
013              computational costs for downstream modeling, though much of the research has
014              focused on the image data modality. We study tabular data distillation, which brings
015              in novel challenges such as the inherent feature heterogeneity and the common
016
                 use of non-differentiable learning models (such as decision tree ensembles and
                 nearest-neighbor predictors). To mitigate these challenges, we present TDColER,
017
                 a tabular data distillation framework via column embeddings-based representation
018
                 learning. To evaluate this framework, we also present a tabular data distillation
019              benchmark, TDBench. Based on an elaborate evaluation on TDBench, resulting in
020              226,200 distilled datasets and 541,980 models trained on them, we demonstrate
021              that TDColER is able to boost the distilled data quality of off-the-shelf distillation
022              schemes by 0.5-143% across 7 different tabular learning models.

027   Dataset distillation or dataset condensation is the process of creating a small set of extremely infor-
028
      mative samples (usually synthetic) from a large dataset such that a model trained on this set will have
      predictive performance comparable to that of a model trained on the original large dataset (Wang et al.,
029
      2020; Yu et al., 2023). First, data distillation reduces data storage costs and can mitigate the privacy
030
      and copyright concerns involved in keeping around (and continuously utilizing) large amounts of raw
031   data. Furthermore, the reduction in the data size implies a lower computational cost of model training,
032   especially when multiple models need to be trained on any given dataset. The above advantages of
033   dataset distillation also facilitate various applications. Continual learning, where we need to learn
034   new tasks while avoiding forgetting older tasks sequentially, often makes use of a “replay buffer”
035   of old task data to be used while learning new tasks to mitigate forgetting of the older tasks (Rol-
036   nick et al., 2019). Dataset distillation reduces the memory overhead of this replay buffer, allowing
037   learning of a larger number of tasks without forgetting (Tiwari et al., 2022; Rosasco et al., 2022).

084   Our contribution. In this paper, we study tabular dataset distillation and present a novel scheme to
085   enhance the distilled data quality of multiple off-the-shelf data distillation schemes across various
086   datasets, models, and distillation sizes. Specifically, we make the following contributions:
087   • We propose Tabular Distillation via Column Embeddings based Representation Learning or
088     TDColER that can utilize modern neural-network architectures such as Transformers and graph
089     neural networks to generate rich compact representations. TDColER improves the quality of
090     distilled data compared to existing distillation schemes. Figure 2a provides an overview of our
091     proposed TDColER.
092   • We present TDBench, a Tabular Distillation Benchmark with 23 tabular datasets, 7 model classes,
093     and 4 distillation schemes. We present an overview of TDBench, an extensible and modular
094     framework for measuring various aspects of data distillation on tabular data, in fig. 1.
095
      • With the elaborate evaluation of our proposed TDColER on TDBench, resulting in over 226,200
        distilled datasets and 541,980 model trainings, we show that, on aggregate across all datasets,
096
        TDColER improves upon direct application of off-the-shelf distillation method on tabular data by
097
        0.5-143% in terms of the distilled data quality across all models at the smallest distillation of 10
098     instances-per-class. Figure 2b presents a snapshot of our results.
099   • Based on our thorough evaluation, we present various insights regarding tabular dataset distillation,
100     such as (i) k-means clustering in the learned representations make for an extremely favorable
101     distillation scheme, (ii) transformer-based tabular data representations obtain the highest distilled
102     data quality on aggregate, while (iii) graph neural network based tabular data representations
103     perform slightly worse than transformers but are significantly more parameter efficient.

107   Dataset distillation was introduced by Wang et al. (2020) as a bilevel optimization problem (Feng
      et al., 2024) and has been widely studied in the context of image data distillation. Most methods can


                                                                         2

108
      be categorized into approaches that match the original data by (i) backbone model performance, (ii)
109   backbone model parameters, or (iii) backbone representation distributions (Yu et al., 2023). Wang
110   et al. (2020) minimized performance differences between the original and distilled data, while Nguyen
111   et al. (2021) introduced kernel-induced points (KIP) using kernel ridge regression with a neural
112   tangent kernel (Jacot et al., 2018). Alternatively, methods have focused on parameter or gradient
113   matching (Zhao et al., 2021; Lee et al., 2022; Jiang et al., 2023; Cazenavette et al., 2022). Gradient
114   matching (Zhao et al., 2021) aligns model gradients between original and synthetic data, while trajec-
115   tory matching (Cazenavette et al., 2022) minimizes discrepancies between entire training trajectories.
116   Other approaches include distribution matching (Zhao & Bilen, 2023), which aligns per-class means,
117
      and cross-layer feature embedding matching (Wang et al., 2022). However, the abovementioned
      methods rely on differentiable backbones, limiting cross-architecture generalization (Cui et al., 2022;
118
      Nguyen et al., 2021). As a result, research has focused primarily on images, leaving tabular data
119
      distillation largely unexplored (Medvedev & Dyakonov, 2021). We address this gap by proposing a
120   more general distillation framework.
121
122   Dataset distillation aligns with coreset selection (Feldman, 2020), which aims to reduce data size,
123
      typically selecting real data instances (potentially risking privacy). In contrast, distillation generates
      synthetic data beyond the real data manifold. Notably, coreset selection is a subset of dataset
124
      distillation, where the synthetic data lies on the real data manifold. Generative modeling (Goodfellow
125
      et al., 2020; Kingma & Welling, 2013) is another related area, usually focused on generating realistic
126   data. In dataset distillation, the goal is to generate informative rather than realistic samples. Recently,
127   Cazenavette et al. (2023) demonstrated how generative modeling can be used to seed the dataset
128   distillation process, arguing that distillation methods should be applied to a latent representation
129   instead of the pixel space. This is aligned with our proposal, in which we demonstrate that distillation
130   in the latent space is critical to obtaining meaningful distilled data quality with tabular datasets.
131   However, the proposed Generative Latent Distillation(GLaD) scheme is very focused on generative
132   vision models, requiring a careful choice of the latent representation from within the model for
133   trade-off in realistic distilled data or expressivity, thus limiting cross-architecture generalization.
134   Cui et al. (2022) benchmarked several distillation methods and found trajectory matching (Cazenavette
135   et al., 2022) to be most effective, followed by KIP (Nguyen et al., 2021). Coreset methods, like k-
136   means clustering, also outperformed many model-based distillation techniques, which we corroborate.
137   We focus on GM and KIP due to the high computational overhead of trajectory matching and omit
138   data augmentation due to its limited applicability to tabular data. As noted before, data augmentation
139   is not standard with tabular data, and we do not consider it in our evaluation with TDBench.

170   2.1    R EPRESENTATION L EARNING VIA C OLUMN E MBEDDING
171
172   A key ingredient in the development of neural networks for tabular data is the use of column
173
      embeddings. First developed for categorical features, the idea is to learn an embedding for each
      of the categories in a categorical feature (Guo & Berkhahn, 2016). This embedding would replace
174
      the one-hot encoded numerical representation of the categories and be used in conjunction with the
175
      (appropriately scaled and imputed) numerical features in standard and custom feed-forward networks
176   (FFNs) (Borisov et al., 2024). Column embeddings for numerical data were developed to use more
177   standard modern architectures such as graph neural networks (GNNs) and Transformers. As with
178   categorical data, each numerical value in a numerical feature of the table would be converted into a
179   learnable embedding. Thus, more precisely, a sample (row) in a table with r numerical features and c
180   categorical features is now represented as a set of (r + c) embeddings in Rm each of size m (where
181   m is a user-specified hyperparameter), thus effectively as the (m × (r + c)) matrix. 2
182
183   Encoder Architectures. Given the Rm×(r+c) representation of a row (sample) using column
184   embeddings, our goal is to learn a more compact yet faithful representation of a row. One simple
185   strategy is to concatenate all the (r + c) column embeddings into a single vector in Rm(r+c) of
186   size m(r + c) and input it into an FFN which projects it down to a lower dimensionality (fig. 8).
187   However, one of our main motivations for using column embeddings is to leverage the capabilities
188
      of more modern architectures. For a given row, the (r + c) column embeddings can be treated as
      initial token embeddings that are progressively updated through multiple Transformer blocks as
189
      described by Gorishniy et al. (2021). Using a dummy [CLS] token, the above process can create a
190
      m-dimensional representation of the row (fig. 11). An alternate procedure is to represent a table as
191   a bipartite graph between columns and rows (with column values and rows as vertices) and utilize
192   the column embeddings as representations for the column vertices (Wu et al., 2021). Then, the row
193   embeddings are obtained by filling in representations for the row vertices via multiple rounds of
194   message passing in a multi-layered GNN (fig. 9). For our purposes, we consider all three architectures
195   – FFN, Transformer and GNN – as encoders that project the Rm×(r+c) representation of row into an
196   embedding in Rm . While categorical column embeddings are standard, there are multiple techniques
197   for numerical column embeddings (Gorishniy et al., 2021; 2022). We discuss and ablate the effect
198   these different schemes have in appendix B.2.
199
200   Learning Objective. Our goal is to retain as much information regarding the original data in the
201   learned representation as possible. The need for high-fidelity learned representations is critical
202
      because we do not assume anything regarding the downstream model, which will be trained with
      the distilled data. Thus, we try to reconstruct the original data from the learned representation as
203
      well as possible. Formally, given column embeddings C : Rr × Cc → Rm×(r+c) , and an encoder
204
      φ : Rm×(r+c) → Rm , we utilize a decoder ψ : Rm → Rr × Cc to reconstruct the original data, and
205
      solve the following optimization problem:

279   Distillation Methods. Given our aforementioned desiderata for model-agnosticity, we have the
280   following existing distillation schemes available, which take as input the set S of N samples and
281
      output a set R of n   N distilled samples (further details regarding implementation of each
      distillation method is provided in appendix A.5.2):
282
283   • k-means Clustering (KM) finds n/L clusters for each of the L classes to produce a total of n
284     distilled samples using Lloyd’s k-means algorithm (Lloyd, 1982). We consider two variations here
285     by (i) using the Euclidean center of each cluster to generate a synthetic sample or (ii) choosing the
286     closest real point to the Euclidean center of each cluster. That is, R comprises n/L cluster centers
287
        (or closest real points) for each of the L classes.
      • Agglomerative Clustering (AG) (Müllner, 2011) again generates n/L clusters for each of the L
288
        classes is similar to k-means. We use the Ward linkage scheme with the Euclidean distance metric.
289
        Similar to k-means, we generate (i) synthetic samples by using the Euclidean center of a cluster or
290     (ii) real samples that are closest to the cluster centers.
291   • Kernel Induced Points (KIP) (Nguyen et al., 2021) uses the neural tangent kernel (NTK) (Jacot
292     et al., 2018) of a wide neural network and kernel ridge regression to produce a distilled set of
293     samples. Given the feature matrix X ∈ RN ×D and the label vector y ∈ Y N , KIP learns the
294     distilled feature matrix X̄ ∈ Rn×D and label vector ȳ ∈ Y n by solving the following problem:

357   Evaluation metric. To have a standardized way to quantify the quality of the distilled data across
358   different models and datasets, we use the notion of relative regret which compares the model’s
359   balanced accuracy score when trained on the full, distilled and randomly sampled data points.
360   Precisely, the relative regret is defined as (AF −A)/(AF −AR10 ), where AF is the balanced accuracy of
361   the model trained on the full training set, AR10 is the balanced accuracy on the same test set when
362   trained on 10 random samples per class averaged over 5 random repetitions, and A is the balanced
363
      accuracy of the model when trained on the distilled dataset over random 5 repetitions. A relative
      regret of 1 matches the performance of random sampling at IPC=10, and a relative regret of 0 matches
364
      the performance of the model trained on the full dataset (which is usually the gold standard) – lower
365
      relative regret implies higher distilled data quality 4 .

        5                     D ISCUSSION
530
531
       This work introduced a tabular data distillation pipeline and evaluated it extensively leveraging various
       distillation methods, with a focus on supporting both non-NN and NN ML classifiers. We introduce
532
       a novel framework, TDColER, that leverages latent representation of tabular data in distillation,
533
       and evaluate it thoroughly in our benchmark, TDBench, which included 23 datasets, 4 distillation
534    algorithms, 3 autoencoder architectures, and 7 downstream classifiers, resulting in over 226,200
535    distilled datasets and 541,980 downstream classifier instances. Our results show that TDColER
536    can induce superior performance in distillation methods on tabular data, improving the quality by
537    0.5-143%. We also show that k-means clustering and transformer autoencoder are a particularly
538    strong combination for tabular data distillation. We hope that this work will serve as a starting point
539    for future research in tabular data distillation and plan to extend this benchmark further to incorporate
       new distillation pipelines.

875   A.3   D ISCUSSION ON PARAMETER SIZE OF AUTOENCODERS
876
      Here we expand on our parameter size of the encoder architectures of the autoencoders. This is worth
877
      noting because if the distilled data is in the latent space, the encoder module is required to project any
878
      new data to the same space. Thus, the encoder is considered to be a part of the distilled output.
879
880   We can characterize the parameter size of each encoder architecture given a D-dimensional binarized
881   dataset with c categorical features and r continuous features that is projected to a d-dimensional
882
      latent space.
883   FFN. We used an FFN architecture with an M -dimensional embedding layer followed by H hidden
884   layers that receive and output W -dimensional vectors. The parameter size of such an FFN is as
885   follows:
886                                O(DM + (c + r)M W + HW 2 + W d)                               (5)
887   The column embeddings are of size O(DM ), the input layer maps the concatenated (c + r)M -
888   dimensional vector to hidden layer dimension W with (c + r)M W size. The hidden layers are of
889   sizes O(W 2 ) each for H hidden layers. The output layer maps the W -dimensional hidden layer
890   output to the desired d-dimensions.
891   GNN. We use a GNN encoder with H consecutive layers. The dimension of the vectors passed
892   between the graph layers are fixed to d, meaning that M = d. Thus, each graph layer maintains a d
893   by d matrix to handle a d-dimensional input vector and output a d-dimensional vector.
894
                                                  O(Dd + Hd2 )                                              (6)
895
896   The column embeddings are of size O(Dd) since M = d. Each of the H GNN layers is of size
897
      O(d2 ).
898   Transformer. We consider an implementation of a transformer autoencoder inspired by the archi-
899   tecture of FT-Transformer Gorishniy et al. (2021). The encoder has an M -dimensional embedding
900   layer followed H transformer blocks. Each transformer block takes in a sequence of M -dimensional
901   embeddings and oututs a single d-dimensional vector. The block is composed of a multihead-attention
902   module with m heads and a FFN module to project the attention scores back to the input space. We
903
      modify the architecture seen in (Gorishniy et al., 2021) by allowing the dimension of the attention
      head to be configurable – i.e. instead of using M/m as the dimension of a single attention head,
904
      we allow the module to compute the attention in dqkv . This choice is motivated by the fact that our
905
      encoders were trained with a latent size of 16, which may not be wide enough for the TF encoder.
906   We then project the resulting embedding in dqkv m-dimension back to M -dimensionals with Wo .

955   A.4.1        O PTIMIZATION FUNCTION
956
957
      For the decoder ψ : Rd → RD , we consider a multi-layered fully-connected feed-forward network.
      Given the encoder φ and the decoder ψ, we use a group-wise softmax operator σ to map the output
958
      of the decoder to a per-input-feature probability simplex: given an initial binary vector b ∈ {0, 1}D
959
      constituting per-input-feature one-hot encodings bi (that is b = [b1 ⊕ . . . ⊕ bc+r ]), and a decoder
960   output B ∈ RD with per-input-feature constituents B i (that is B = [B 1 ⊕ . . . ⊕ B c+r ], we apply the
961
      softmax operation to each per-input-feature constituent to get b̂ = [b̂1 ⊕ . . . ⊕ b̂c+r ] ∈ [0, 1]D , where
962
      b̂i = softmax(B i ). We utilize the following per-sample reconstruction loss:
963                                                 Pc+r
                                                 1           1           i i
964                                  `(b, b̂) = c+r   i=1 log |bi | CE(b , b̂ ),
                                                                           2
                                                                                                               (8)
965
      where CE is the standard cross-entropy loss between a one-hot vector and a softmax output, and
966   |bi | is the length of the i-th constituent one-hot encoding in b, corresponding to the number of
967   categories (or bins) in the i-th categorical (or numerical) feature. This loss is a weighted average
968   of the per-input-feature cross-entropy loss, with weights (1/log2 |bi |) to normalize the loss across all
969   features with varying number of categories or bins.
970
      The encoder and decoder are then learned by optimizing the following unsupervised loss:
971
                           LR (φ, ψ) = N1 (x,y)∈S ` (P (x), σ(ψ(φ(P (x))))) ,

973    the latent representation in such an unsupervised manner makes this distillation pipeline agnostic
974    to the choice of downstream model. Another advantage of this choice is that the decoder allows
975    us to map the distilled artificial samples in the latent space to the original features, which might be

988    A.4.3    E NCODER ARCHITECTURES
989
       Fully-connected feed-forward network (FFN). This encoder first selects the column embeddings
990
       corresponding to nonzero entries in the binary representation b, concatenates them to get a (c + r)M -
991
       dimensional dense vectors (recall that b will only have c + r nonzeros out of the D dimensions),
992
       and inputs them to a fully-connected feed-forward network µ : R(c+r)M → Rd . The encoder
993    φ : {0, 1}D → Rd can be written as:
994
                                z = φ(b) = µ(⊕([wi , i ∈ {1, . . . , D} : b[i] = 1])),                      (11)
995
996    where b[i] is the i-th entry of the D-dimensional vector, and ⊕ is the concatenation operator. The
997    FFN µ and the column embeddings {wi , i ∈ {1, . . . , D}} constitute the parameters of the encoder φ.
998    For a FFN with H hidden layers, each of width W , the total number of parameters in this encoder is
999
       O(DM + (c + r)M W + HW 2 + W d). Figure 8 shows a simplified architecture of the FFN encoder.
1000   Graph neural network (GNN) encoder. We also consider a more recent encoder for tabular data pro-
1001   posed in Wu et al. (2021). A bipartite graph is constructed between the column embeddings {wi , i ∈
1002   {1, . . . , D}} and the (zero-initialized) row (sample) embeddings {zj ∈ Rd , j ∈ {1, . . . , N }}, with a
1003   bidirectional edge between wi and zj if the bj [i] = 1, where bj ∈ {0, 1}D is the binary representation
1004   of the j-th sample. Given the (learned) column embeddings, the row embeddings are obtained via
1005   multiple rounds of message passing through multiple GNN layers. This can be written as:
1006                                   zjh = µh (zjh−1 , Agg(wih−1 , i ∈ Nj )),
1007                                                                                                        (12)
                                       wih = µh (wih−1 , Agg(zjh , j ∈ Ni )),
1008
1009   where µh is the h-th GNN layer, Agg is an aggregation, Ni (or Nj ) is the neighbor set of the
1010   i-th column embedding (or j-th row embedding). We set the initial zj0 = 0 (zero-initialized row
1011   embeddings), wi0 = wi , and utilize zjH as the latent representation for distillation after H GNN
1012   layers. While Wu et al. (2021) only considered Graph Convolutional Networks Kipf & Welling
1013   (2016) as GNN modules, we extend it to GraphSage Hamilton et al. (2017) and Graph Attention
1014   Networks Veličković et al. (2018). An important aspect of the GNN encoder is that the desired row
1015   embedding size d must match the column embedding size M , thus d = M . With H GNN layers,
1016
       the total number of parameters in this encoder is usually O(Dd + Hd2 ), which can be significantly
       smaller than the FFN encoder with moderately sized FFN (large enough M , W ). Figure 9 shows the
1017
       graph formulation (left) and the GNN encoder architeture (right).
1018
1019   Transformer encoder. Finally, we consider a transformer-based autoencoder inspired by the architec-
1020   ture of FT-Transformer Gorishniy et al. (2021). This encoder uses the same embedding layer as the
1021   FFN encoder, which is then followed by transformer blocks. We learn an additional cls embedding,
1022
       which is placed before all other tokens in every sequence. Each block takes in a sequence (one row) of
       d embeddings, and is composed of a multihead-attention (MHA) module and a feed-forward network
1023
       (FFN) module.

1155   A.8   D ETERMINING THE BEST OVERALL PERFORMANCE
1156
       We describe the best overall pipeline in section 4 and table 2. Here, we provide a more detailed
1157
       explanation of how we determined the best overall pipeline. The runs are grouped by their classifier,
1158   dataset and distill size n. Similar to other parts of analysis, the grouping is done in order to ensure
1159   that the comparisons are fair. In this instance, we are interested in only the pipeline components that
1160   lead to the best classifier performance, regardless of the exact classifier kind. Thus, we group every
1161   run by their non-pipeline-specific parameters, which are the classifier, dataset and distill size n. In
1162   each group, we then count the instances the pipeline places on the top 3 in terms of the regret score
1163   and sum up the counts for each pipeline.
1164   Following the previous findings, table 2 shows that k-means based methods have the best performance,
1165   placing in the top 3 with all SFT encoder variants. Surprisingly, we also find pipelines that use KIP and
1166   GM as the 4th and 5th best performers. While we were not able to determine any specific conditions
1167   that cause KIP and GM to place on top, this result shows that there are exist some conditions which
1168   leads the pipelines using gradient-based methods (KIP, GM) to be the top performer. On the other
1169   hand, the consistent rank placement of pipelines that use the autoencoder latent space shows that
1170   fine-tuned autoencoders can indeed boost the performance of distillation methods significantly.

1178   While column embeddings are standard for categorical columns – each category is represented with
1179
       a vector, there are various ways of embedding numerical columns: (i) A numerical feature can be
       binned, and each bin treated as a category with an embedding w ∈ Rm corresponding to each bin.
1180
       (ii) With linearly scaled column embeddings, a single column embedding w ∈ Rm is used for each
1181
       numerical column, and the column embedding for a particular numerical value v ∈ R is obtained
1182   by scaling w to v · w. (iii) Piecewise linear encoding or PLE (Gorishniy et al., 2022) also bin the
1183   numerical feature but use a more sophisticated way of generating the column embeddings for a
1184   given numerical value. We considered binned numerical features in the main paper for a couple of
1185   reasons: (a) Binned numerical features naturally handle missing values (quite prevalent in tabular
1186   data) by maintaining a “missing” bin instead of relying on a heuristic intermediate imputation step;
1187   sometimes, the fact that a value is missing is in itself a signal, and heuristic imputation schemes often
       lose this information. (b) The binned features can be used for all architectures we consider here –
