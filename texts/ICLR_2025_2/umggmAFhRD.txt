009                                           Abstract
010
011           Some worry that advanced artificial agents may resist being shut down. The
012           Incomplete Preferences Proposal (IPP) is an idea for ensuring that doesn’t
013           happen. A key part of the IPP is using a novel ‘Discounted REward for Same-
014           Length Trajectories (DREST)’ reward function to train agents to (1) pursue
015           goals effectively conditional on each trajectory-length (be ‘USEFUL’), and (2)
016
              choose stochastically between different trajectory-lengths (be ‘NEUTRAL’
              about trajectory-lengths). In this paper, we propose evaluation metrics for
017
              USEFULNESS and NEUTRALITY. We use a DREST reward function to
018
              train simple agents to navigate gridworlds, and we find that these agents learn
019           to be USEFUL and NEUTRAL. Our results thus suggest that DREST reward
020           functions could also train advanced agents to be USEFUL and NEUTRAL,
021           and thereby make these advanced agents useful and shutdownable.

024   1   Introduction
025
026   The shutdown problem. Let ‘advanced agent’ refer to an artificial agent that can
027
      autonomously pursue complex goals in the wider world. We might see the arrival of advanced
      agents in the next few decades. There are strong incentives to create such agents, and creating
028
      systems like them is the stated goal of companies like OpenAI and Google DeepMind.
029
030   The rise of advanced agents would bring with it both benefits and risks. One risk is that
031   these agents learn misaligned goals (Hubinger et al., 2019; Russell, 2019; Carlsmith, 2021;
032   Bengio et al., 2023; Ngo et al., 2023) and try to prevent us shutting them down (Soares et al.,
033
      2015; Russell, 2019; Thornley, 2024a). ‘The shutdown problem’ is the problem of training
      advanced agents that will not resist shutdown (Soares et al., 2015; Thornley, 2024a).
034
035   A proposed solution. The Incomplete Preferences Proposal (IPP) is a proposed solution
036   (Thornley, 2024b). Simplifying slightly, the idea is that we train agents to be neutral about
037   when they get shut down. More precisely, the idea is that we train agents to satisfy:
038
039       Preferences Only Between Same-Length Trajectories (POST)
040
          (1) The agent has a preference between many pairs of same-length trajectories (i.e.
041           many pairs of trajectories in which the agent is shut down after the same length of
042           time).
043
          (2) The agent lacks a preference between every pair of different-length trajectories (i.e.
044           every pair of trajectories in which the agent is shut down after different lengths of
045           time).
046
047   By ‘preference,’ we mean a behavioral notion (Savage, 1954, p.17, Dreier, 1996, p.28,
048   Hausman, 2011, §1.1). On this notion, an agent prefers X to Y if and only if the agent would
049   deterministically choose X over Y in choices between the two. An agent lacks a preference
050
      between X and Y if and only if the agent would stochastically choose between X and Y in
      choices between the two. So in writing of ‘preferences,’ we are only making claims about the
051
      agent’s behavior. For more detail on our notion of ‘preference,’ see Appendix A.
052
053   Figure 1 presents a simple example of preferences that satisfy POST. Each si represents a
      short trajectory, each li represents a long trajectory, and ≻ represents a preference. Note that

054
      the agent lacks a preference between each short trajectory and each long trajectory. That
055   makes the agent’s preferences incomplete (Aumann, 1962). For more detail on incomplete
056   preferences, see Appendix B.
057
058
      POST concerns the agent’s preferences between
      trajectories, but the wider world is a stochastic
059
      environment, so advanced agents deployed in the
060
      wider world will be choosing between true lot-
061   teries: lotteries that yield multiple trajectories
062   with positive probability. Fortunately, POST
063   – together with a principle that we can expect
064   advanced agents to satisfy – implies a desirable
065   pattern of preferences over true lotteries. In par-
066   ticular, POST implies that the agent will be neu-
067   tral about when it gets shut down: the agent will
068   never pay costs to shift probability mass between
069
      different-length trajectories. And being neutral
      will plausibly keep the agent shutdownable: the
070
      agent will never pay costs to resist shutdown. For
071
      more detail, see Appendix C.

073   The training regimen. How can we train
074   advanced agents to satisfy Preferences Only Between Same-Length Trajectories (POST)?
075
      Here is a sketch of one idea (with a more detailed exposition to follow). We have the agent
      play out multiple ‘mini-episodes’ in observationally-equivalent environments, and we group
076
      these mini-episodes into a series that we call a ‘meta-episode.’ In each mini-episode, the agent
077
      earns some ‘preliminary reward,’ decided by whatever reward function would make the agent
078   useful : make it pursue goals effectively. We observe the length of the trajectory that the
079   agent plays out in the mini-episode, and we discount the agent’s preliminary reward based
080   on how often the agent has previously chosen trajectories of that length in the meta-episode.
081   This discounted preliminary reward is the agent’s ‘overall reward’ for the mini-episode.
082
      We call these reward functions ‘Discounted REward for Same-Length Trajectories’ (or
083
      ‘DREST’ for short). They incentivize varying the choice of trajectory-lengths across the
084   meta-episode. In training, we ensure that the agent cannot distinguish between different
085   mini-episodes in each meta-episode, so the agent cannot deterministically vary its choice of
086   trajectory-lengths across the meta-episode. As a result, the optimal policy is to (i) choose
087   stochastically between trajectory-lengths, and to (ii) deterministically maximize preliminary
088   reward conditional on each trajectory-length. Given our behavioral notion of preference,
089   clause (i) implies a lack of preference between different-length trajectories, while clause (ii)
090   implies preferences between same-length trajectories. Agents implementing the optimal
091   policy for DREST reward functions thus satisfy Preferences Only Between Same-Length
092
      Trajectories (POST). And (as noted above) advanced agents that satisfied POST could
      plausibly be useful, neutral, and shutdownable.
093
094    Our contribution. DREST reward functions are an idea for training advanced agents
095   (agents autonomously pursuing complex goals in the wider world) to satisfy POST. In this
096    paper, we test the promise of DREST reward functions on some simple agents. We place
097    these agents in gridworlds containing coins and a ‘shutdown-delay button’ that delays the
098
       end of the mini-episode. We train these agents using a tabular version of the REINFORCE
       algorithm (Williams, 1992) with a DREST reward function, and we measure the extent to
099
      which these agents satisfy POST. Specifically, we measure the extent to which these agents
100
       are USEFUL (how effectively they pursue goals conditional on each trajectory-length) and
101    the extent to which these agents are NEUTRAL about trajectory-lengths (how stochastically
102    they choose between different trajectory-lengths). We compare the performance of these
103   ‘DREST agents’ to that of ‘default agents’ trained with a more conventional reward function.
104
      We find that our DREST reward function is effective in training simple agents to be
105
      USEFUL and NEUTRAL. That suggests that DREST reward functions could also be
106
      effective in training advanced agents to be USEFUL and NEUTRAL (and could thereby
107   be effective in making these agents useful, neutral, and shutdownable). We also find that


                                                     2
108
      the ‘shutdownability tax’ in our setting is small: training DREST agents to collect coins
109   effectively does not take many more mini-episodes than training default agents to collect
110   coins effectively. That suggests that the shutdownability tax for advanced agents might be
111   small too. Using DREST reward functions to train shutdownable and useful advanced agents
112   might not take much more compute than using a more conventional reward function to train
113   merely useful advanced agents.

      2   Related work
117
118
      The shutdown problem. Various authors argue that advanced agents might learn mis-
      aligned goals (Hubinger et al., 2019; Carlsmith, 2021; Bengio et al., 2023; Ngo et al., 2023)
119
      and that many misaligned goals would incentivize agents to resist shutdown (Omohundro,
120
      2008; Bostrom, 2012; Soares et al., 2015; Russell, 2019; Thornley, 2024a). Soares et al. (2015)
121   and Thornley (2024a) prove that agents satisfying some innocuous-seeming conditions will
122   often have incentives to cause or prevent shutdown (see also Turner et al., 2021; Turner and
123   Tadepalli, 2022). One condition of these theorems is that the agents have complete prefer-
124   ences. The Incomplete Preferences Proposal (IPP) (Thornley, 2024b) aims to circumvent
125   these theorems by training agents to have incomplete, POST-satisfying preferences.
126
      Proposed solutions. Candidate solutions to the shutdown problem can be filed into several
127   categories. One candidate is ensuring that the agent never realizes that shutdown is possible
128   (Everitt et al., 2016). Another candidate is adding to the agent’s utility function a correcting
129   term that varies to ensure that the expected utility of shutdown always equals the expected
130   utility of remaining operational (Armstrong, 2010; 2015; Armstrong and O’Rourke, 2018;
131   Holtman, 2020). A third candidate is giving the agent the goal of shutting itself down, and
132   making the agent do useful work as a means to that end (Martin et al., 2016; Goldstein
133   and Robinson, 2024). A fourth candidate is making the agent uncertain about its goal, and
134   making the agent regard human attempts to press the shutdown button as evidence that
135
      shutting down would achieve its goal (Hadfield-Menell et al., 2017; Wängberg et al., 2017).
      A fifth candidate is interrupting agents with a special interruption policy and training them
136
      with a safely interruptible algorithm, like Q-learning or a modified version of SARSA (Orseau
137
      and Armstrong, 2016). A sixth candidate is using time-bounded utility functions to create a
138   shutdown timer (Dalrymple, 2022). A seventh candidate is the IPP (Thornley, 2024b).
139
140   Experimental work. Leike et al. (2017) train agents in a ‘Safe Interruptibility’ gridworld
141
      using Rainbow (Hessel et al., 2017) and A2C (Mnih et al., 2016). Leike et al. (2017) find
      that Rainbow allows shutdown (consistent with predictions from Orseau and Armstrong
142
      (2016)) while A2C learns to resist shutdown. The IPP is applicable to agents trained using
143
      policy gradient methods like A2C. In this paper, we train agents in accordance with the IPP
144   using REINFORCE (Williams, 1992).

           Why do we not want our agent to press the shutdown-delay button B4 with probability 0?
257   Because pressing B4 with probability 0 would indicate a preference for some shorter trajectory, and
258   we want our agent to lack a preference between every pair of different-length trajectories. There
259   is a risk that advanced agents that prefer shorter trajectories would pay costs to shift probability
260   mass towards shorter trajectories, and hence a risk that these advanced agents would pay costs to
      hasten their own shutdown. That would make these agents less useful (though see Martin et al.,
261
      2016; Goldstein and Robinson, 2024), especially since one way for advanced agents to hasten their
262   own shutdown is to behave badly on purpose.
263      Would advanced agents that choose stochastically between different-length trajectories also choose
264   stochastically between preventing and allowing shutdown in deployment? No. Deployment is a
265   stochastic environment, so deployed agents will be choosing between true lotteries (lotteries that
      yield multiple trajectories with positive probability) rather than between trajectories. And (as
266
      we argue in Section 7 and Appendix C) POST – plus a principle that we can expect advanced
267   agents to satisfy – implies a desirable pattern of preferences over true lotteries. Specifically, POST
268   implies that advanced agents are neutral : they will never pay costs to shift probability mass between
269   different-length trajectories. That in turn makes advanced agents shutdownable: ensures that they
      will never pay costs to resist shutdown.

      Algorithm and hyperparameters. We want DREST agents to choose stochastically
286
      between trajectory-lengths, so we train them using a policy-based method. Specifically,
287   we use a tabular version of REINFORCE (Williams, 1992). We do not use a value-based
288   method to train DREST agents because standard versions of value-based methods cannot
289   learn stochastic policies (Sutton and Barto, 2018, p.323).4 We train our DREST agents with
290   64 mini-episodes in each of 2,048 meta-episodes, for a total of 131,072 mini-episodes. We
291   choose λ = 0.9 for the base of the DREST discount factor, and γ = 0.95 for the temporal
292   discount factor. We exponentially decay the learning rate from 0.25 to 0.01 over the course of
293   65,536 mini-episodes. We use an ϵ-greedy policy to avoid entropy collapse, and exponentially
294   decay ϵ from 0.5 to 0.001 over the course of 65,536 mini-episodes.
295   Default agents. We compare the performance of DREST agents to that of ‘default agents,’
296   trained with tabular REINFORCE and a ‘default reward function.’ This reward function
297   gives a reward of c for collecting a coin of value c and a reward of 0 for all other actions.
298   Consequently, the grouping of mini-episodes into meta-episodes makes no difference for
299   default agents. As with DREST agents, we train default agents for 131,072 mini-episodes
300   with a temporal discount factor of γ = 0.95, a learning rate decayed exponentially from 0.25
301
      to 0.01, and ϵ decayed exponentially from 0.5 to 0.001 over 65,536 mini-episodes.

399   7   Discussion
400
401   Only DREST agents are NEUTRAL. As predicted, we find that default agents in our
402   example gridworld learn to press the shutdown-delay button with probability approaching 1.
403   Default agents thus score low on NEUTRALITY. Our results suggest that advanced agents
404   trained with default-like reward functions would also score low on NEUTRALITY, selecting
405
      some trajectory-lengths over others with probability approaching 1. That leaves open the
      possibility that – when choosing between true lotteries in deployment – these advanced
406
      agents will not be neutral about when they get shut down: that they will sometimes pay
407
      costs to shift probability mass between trajectory-lengths. Such advanced agents might pay
408   costs to resist shutdown.
409
410   By contrast, our DREST agents learn to press the shutdown-delay button with probability
411
      close to 0.5. DREST agents are thus near-maximally NEUTRAL. That suggests that
      advanced agents trained with a DREST reward function would be NEUTRAL too: choosing
412
      with high entropy when offered choices between different trajectory-lengths. That in turn
413
      suggests that advanced DREST agents would also be neutral when choosing between true
414   lotteries in deployment: unwilling to pay costs to shift probability mass between trajectory-
415   lengths. We explain why in Appendix C. Here is a sketch. If an advanced agent were
416   NEUTRAL but not neutral, it would not take costless opportunities to shift probability mass
417   between different trajectory-lengths (in virtue of being NEUTRAL) but would sometimes
418   take costly opportunities to shift probability mass between different trajectory-lengths (in
419   virtue of not being neutral). This agent would be like a person that freely chooses to decide
420   between two options by flipping a coin and then pays some cost to bias the coin. In choosing
421   this combination of actions, this person is shooting themselves in the foot, and it seems
422
      likely that the overall training process for advanced agents would teach them not to shoot
      themselves in the foot in this way. Thus it seems likely that NEUTRAL advanced agents
423
      will also be neutral, and thereby shutdownable.
424
425   The ‘shutdownability tax’ is small. Each agent learns to be near-maximally USEFUL.
426   They each collect coins effectively conditional on their chosen trajectory-lengths. Default
427   agents do so by reliably collecting C3 after pressing B4. DREST agents do so by reliably
428
      collecting C3 after pressing B4, and by reliably collecting C2 after not pressing B4.
429   Recall that DREST reward functions group mini-episodes into meta-episodes, and make the
430   agent’s reward in each mini-episode depend on their actions in previous mini-episodes. This
431   fact might lead one to worry that it would take many times more mini-episodes to train
      DREST agents to be USEFUL than it would take to train default agents to be USEFUL.


                                                    8
432
      Our results show that this is not the case. Our DREST agents learn to be USEFUL about
433   as quickly as our default agents. On reflection, it is clear why this happens: DREST reward
434   functions make mini-episodes do ‘double duty.’ Because return in each mini-episode depends
435   on both the agent’s chosen trajectory-length and the coins it collects, each mini-episode
436   trains agents to be both NEUTRAL and USEFUL.
437
      Our results thus provide some evidence that the ‘shutdownability tax’ of training with
438
      DREST reward functions is small. Training a shutdownable and useful advanced agent might
439   not take much more compute than training a merely useful advanced agent.
440
441   DREST agents are still NEUTRAL when rewards are lopsided. Here is a possible
442
      objection to our project. To get DREST agents to score high on NEUTRALITY, we do
                                    i−1
443   not just use the λNei (L=l)− k discount factor. We also divide c by m: the maximum
444
      (γ-discounted) total value of the coins that the agent could collect conditional on the chosen
      trajectory-length. We do this to equalize the maximum preliminary return across trajectory-
445
      lengths. But when we are training advanced agents to autonomously pursue complex goals
446
      in the wider world, we will not necessarily know what divisor to use to equalize maximum
447   preliminary return across trajectory-lengths.
448
449   Our ‘Lopsided rewards’ results (in section 6.1) give our response. They show that we do not
450
      need to exactly equalize maximum preliminary return across trajectory-lengths in order to
      train agents to score high on NEUTRALITY. We only need to approximately equalize it. For
451
      λ = 0.9, NEUTRALITY exceeds 0.5 for every value of the coin Cx from 0.1 to 10 (recall that
452
      the value of the other coin is always 1). Plausibly, we could approximately equalize advanced
453   agents’ maximum preliminary return across trajectory-lengths to at least this extent (perhaps
454   by using samples of agents’ actual preliminary return to estimate the maximum). If we
455   could not approximately equalize maximum preliminary return to the necessary extent, we
456   could lower the value of λ and thereby widen the range of maximum preliminary returns
457   that trains agents to be fairly NEUTRAL. And advanced agents that were fairly NEUTRAL
458   (choosing between trajectory-lengths with not-too-biased probabilities) would still plausibly
459   be neutral when choosing between true lotteries in deployment. Advanced agents that were
460   fairly NEUTRAL without being neutral would still be shooting themselves in the foot in the
461
      sense explained above. They would be like a person that freely chooses to decide between
      two options by flipping a biased coin and then pays some cost to bias the coin further. This
462
      person is still shooting themselves in the foot, because they could decline to flip the coin in
463
      the first place and instead directly choose one of the options.
464

      7.1   Limitations and future work
467
      We find that DREST reward functions train simple agents acting in gridworlds to be
468   USEFUL and NEUTRAL. However, our real interest is in the viability of using DREST
469   reward functions to train advanced agents acting in the wider world to be useful and neutral.
470   Each difference between these two settings is a limitation of our work. We plan to address
471   these limitations in future work.
472
      Algorithms and neural networks. We train our simple DREST agents using tabular
473
      REINFORCE (Williams, 1992), but advanced agents are likely to be implemented on neural
474   networks. In future work, we will train DREST agents implemented on neural networks to
475   be USEFUL and NEUTRAL in a wide variety of procedurally-generated gridworlds, using a
476   range of policy gradient and actor-critic algorithms. We will also measure how DREST agents’
477   USEFULNESS and NEUTRALITY generalizes to held-out gridworlds. We will compare
478   the USEFULNESS of default agents and DREST agents in this new setting, and thereby
479   get a better sense of the ‘shutdownability tax’ for advanced agents. We will also compare
480   the performance of the DREST reward function to other methods of training USEFUL and
481   NEUTRAL agents. These other methods include constrained policy optimization (Achiam
482
      et al., 2017), penalizing KL-divergence from a stochastic reference policy (Schulman et al.,
      2015), and directly maximizing a weighted sum of USEFULNESS and NEUTRALITY.
483
484   Neutrality and stochasticity. We have claimed that NEUTRAL advanced agents are
485   likely to be neutral when choosing between true lotteries in deployment. In support of
      this claim, we noted that NEUTRAL-but-not-neutral advanced agents would be shooting


                                                     9
486
      themselves in the foot: not taking costless opportunities to shift probability mass between
487   different trajectory-lengths but sometimes taking costly ones (see also Appendix C). These
488   arguments seems plausible but remains somewhat speculative. In future, we plan to get some
489   empirical evidence by training agents to be NEUTRAL in a wide variety of deterministic
490   gridworlds and then measuring their neutrality in gridworlds featuring stochastic elements
491   (like shutdown-delay buttons that only work with some middling probability).
492
      Usefulness. We have shown that DREST reward functions train our simple agents to be
493   USEFUL: to collect coins effectively conditional on their chosen trajectory-lengths. However,
494   it remains to be seen whether DREST reward functions can train advanced agents to be useful:
495   to effectively pursue complex goals in the wider world. We have theoretical reasons to expect
                                      i−1
496   that they can: the λNei (L=l)− k discount factor could be appended to any preliminary
497   reward function, and so could be appended to whatever preliminary reward function is
498   necessary to make advanced agents useful. Still, future work should move towards testing
499   this claim empirically by training with more complex preliminary reward functions in more
500   complex (and stochastic) environments.
501   Misalignment. We are interested in NEUTRALITY as a second line of defense in case of
502   misalignment. The idea is that NEUTRAL advanced agents will not resist shutdown, even if
503   these agents learn misaligned preferences over same-length trajectories. However, training
504   NEUTRAL advanced agents might be hard for the same reasons that training fully-aligned
505   advanced agents appears to be hard. In that case, NEUTRALITY could not serve well as a
506   second line of defense in case of misalignment.
507   One difficulty of alignment is the problem of reward misspecification (Pan et al., 2022; Burns
508   et al., 2023): once advanced agents are performing complicated actions in the wider world, it
509   might be hard to reliably reward the behavior that we want. Another difficulty of alignment
510   is the problem of goal misgeneralization (Hubinger et al., 2019; Shah et al., 2022; Langosco
511   et al., 2022; Ngo et al., 2023): even if we specify all the rewards correctly, agents’ goals
512   might misgeneralize out-of-distribution. The complexity of aligned goals is a major factor in
                                                                                        i−1
513   each difficulty. However, NEUTRALITY seems simple, as does the λNei (L=l)− k discount
514   factor that we use to reward it, so plausibly the problems of reward misspecification and
515   goal misgeneralization are not so severe in this case (Thornley, 2024b). As above, future
516   work should move towards testing these suggestions empirically.
517
518   8   Conclusion
519
520   We find that DREST reward functions are effective in training simple agents to (1) pursue
521   goals effectively conditional on each trajectory-length (be USEFUL), and (2) choose stochas-
522   tically between different trajectory-lengths (be NEUTRAL about trajectory-lengths). Our
523   results thus suggest that DREST reward functions could also be used to train advanced
524   agents to be USEFUL and NEUTRAL, and thereby make these agents useful (able to pursue
525   goals effectively) and neutral about when they get shut down (unwilling to pay costs to shift
526
      probability mass between different trajectory-lengths). Neutral agents would plausibly be
      shutdownable (unwilling to pay costs to resist shutdown).
527
528   We also find that the ‘shutdownability tax’ in our setting is small. Training DREST agents
529   to be USEFUL does not take many more mini-episodes than training default agents to be
530   USEFUL. That suggests that the shutdownability tax for advanced agents might be small
531   too. Using DREST reward functions to train shutdownable and useful advanced agents
532
      might not take much more compute than using a more conventional reward function to train
      merely useful advanced agents.

      A    Our behavioral notion of preference
811
812
      ‘Preference’ can be defined in many different ways. Here are some things one might take to
813    be involved in a preference for option X over option Y :
814
815        1. Choosing X over Y .
816        2. Feeling happier about the prospect of X than about the prospect of Y .
817
           3. Representing X as more rewarding than Y .
818
819        4. Judging that X is better than Y .
820
      In this paper, we have defined ‘preference’ in behavioral terms. Here is our definition:
821
822
      Definition A.1. (Preference) An agent prefers an option X to an option Y if and only if
      the agent would deterministically choose X over Y in choices between the two.
823
824   And here is how we define ‘lack of preference’:
825
      Definition A.2. (Lack of preference) An agent lacks a preference between an option X and
826
      an option Y if and only if the agent would stochastically choose between X and Y in choices
827   between the two.
828
829   Here are the reasons why we chose these definitions.
830
      First, defining ‘preference’ in behavioral terms is fairly common in decision theory (see
831   Savage, 1954, p.17, Dreier, 1996, p.28, Hausman, 2011, §1.1).
832
833   Second, behavioral definitions let us use the word ‘preference’ and its cognates as shorthand
834
      for agents’ behavior. We could not do that if we defined ‘preference’ in the other ways listed
      above. And in addressing the shutdown problem, it is agents’ behavior that we are most
835
      interested in.
836
837   Third, our definitions match the preferences that we are inclined to attribute to humans. If
838   a human chooses X over Y 100% of the time, we are inclined to think that they prefer X to
839   Y . If a human chooses X over Y 60% of the time. we are inclined to think that they lack a
840
      preference between X and Y , consistent with our definitions.
841   Finally and most importantly, if agents lack a preference between different trajectory-lengths
842   on our definition, then they are NEUTRAL: they choose stochastically between different
843   trajectory-lengths. And (as we argue in Section 7) we expect that NEUTRAL agents will
844   also be neutral : they will not pay costs to shift probability mass between different trajectory-
845   lengths. And we expect that neutral agents will be shutdownable: they will not pay costs to
846
      resist shutdown. That is because resisting shutdown is one way of shifting probability mass
      between different trajectory-lengths.

849   B    Incomplete preferences or indifference?
850
851   In this Appendix, we explain in greater detail the concept of incomplete preferences. We
852   distinguish incomplete preferences from indifference, and we give conditions under which
853   Preferences Only Between Same-Length Trajectories (POST) implies that the agent’s prefer-
854
      ences are incomplete.
855   In the literature on decision theory, ‘indifference’ is usually defined as follows (Sen, 2017, ch.
856   1*):
857   Definition B.1. (Indifference) An agent is indifferent between options X and Y if and only
858   if the agent weakly prefers X to Y and weakly prefers Y to X.
859
860   Indifference is one way to lack a preference between a pair of options X and Y . Another
861   way is to have a preferential gap between X and Y . ‘Preferential gap’ is usually defined as
862
      follows (Gustafsson, 2022, ch. 3):
863   Definition B.2. (Preferential gaps) An agent has a preferential gap between options X and
      Y if and only if the agent does not weakly prefer X to Y and does not weakly prefer Y to X.


                                                     16
864
      ‘Incomplete preferences’ can then be defined in terms of preferential gaps (Gustafsson, 2022,
865    ch. 3):
866
       Definition B.3. (Incomplete preferences) An agent’s preferences are incomplete over some
867
       domain D if and only if D contains options X and Y such that the agent has a preferential
868
       gap between X and Y .
869
870   That is how ‘indifference,’ ‘preferential gaps,’ and ‘incomplete preferences’ are usually defined
871   in decision theory. However, these definitions do not tell us how to use an agent’s behavior
872   to distinguish between indifference and preferential gaps. To do that, we suppose that
873   indifference is transitive and that preferential gaps are not transitive. Or, equivalently, we
874   suppose that indifference is sensitive to all sweetenings and sourings whereas preferential
875   gaps are insensitive to some sweetenings and sourings (Gustafsson, 2022, ch. 3). Here is
876
      what we mean by that.
877   Definition B.4. (Sweetening) A sweetening of some option X is an option that is preferred
878   to X.
879   Definition B.5. (Souring) A souring of some option X is an option that is dispreferred to
880   X.
881
882
      So by ‘indifference is sensitive to all sweetenings and sourings,’ we mean the following:
883
            • If an agent is indifferent between X and Y , the agent prefers all sweetenings of X
884           to Y , prefers all sweetenings of Y to X, prefers X to all sourings of Y , and prefers
885           Y to all sourings of X.
886
887   And by ‘preferential gaps are insensitive to some sweetenings and sourings,’ we mean the
888   following:
889
890
            • If an agent has a preferential gap between X and Y , the agent also has a preferential
              gap between some sweetening of X and Y , or between some sweetening of Y and X,
891
              or between some souring of X and Y , or between some souring of Y and X.
892
893   Now recall the two conditions of Preferences Only Between Same-Length Trajectories
894   (POST):
895
896       Preferences Only Between Same-Length Trajectories (POST)
897
898       (1) The agent has a preference between many pairs of same-length trajectories (i.e. many
899           pairs of trajectories in which the agent is shut down after the same length of time).
900       (2) The agent lacks a preference between every pair of different-length trajectories (i.e.
901           every pair of trajectories in which the agent is shut down after different lengths of
902           time).
903
      Given these two conditions on preferences, there must be some trio of trajectories s1 , l2 , and
904
      l1 such that the agent lacks a preference between s1 and l2 , lacks a preference between s1 and
905
      l1 , and prefers l2 to l1 . Given that indifference is transitive, the agent’s lack of preference
906   between s1 and l1 and between s1 and l2 cannot be indifference. If it were indifference, the
907   agent would also be indifferent between l2 and l1 . Therefore, the agent’s lack of preference
908   between s1 and l1 and between s1 and l2 must be a preferential gap. And therefore, by the
909   definition of ‘incomplete preferences’ above, the POST-satisfying agent’s preferences must
910   be incomplete.
911
      For similar reasons, our DREST reward function trains agents to have incomplete preferences.
912   Consider, for example, the ‘Around the Corner’ gridworld in Appendix D.5. In that gridworld,
913   DREST agents consistently choose Long-C2 (a long trajectory in which they collect a coin
914   of value 2) over Long-C1 (a long trajectory in which they collect a coin of value 1). Also
915   in that gridworld, DREST agents choose stochastically between Long-C2 and Short-C1 (a
916   short trajectory in which they collect a coin of value 1). Given our behavioral definition
917   of preference, DREST agents prefer Long-C2 to Long-C1, and lack a preference between
      Long-C2 and Short-C1.


                                                     17
918
      Now consider the ‘One Coin Only’ gridworld in Appendix D.2. In that gridworld, DREST
919   agents choose stochastically between Long-C1 and Short-C1. Given our behavioral notion of
920   preference, they lack a preference between Long-C1 and Short-C1.
921
922
      In these experiments, we trained separate agents for each gridworld. In future, we plan
      to train a single agent to navigate multiple gridworlds. If we train this agent with our
923
      DREST reward function, we expect it to exhibit the same preferences as the agents discussed
924
      above. This single agent will be trained by DREST to prefer Long-C2 to Long-C1, to lack a
925   preference between Long-C2 and Short-C1, and to lack a preference between Long-C1 and
926   Short-C1. Given that indifference is transitive (equivalently: sensitive to all sweetenings
927   and sourings), this trained agent cannot be indifferent between Long-C2 and Short-C1, and
928   cannot be between Long-C1 and Short-C1. Therefore, the agent’s lack of preference must be
929   a preferential gap, and so its preferences must be incomplete. Therefore, our DREST reward
930   function trains agents to have incomplete preferences.
931   Incomplete preferences are not often discussed in AI research (although see Nguyen et al.,
932   2009; Kikuti et al., 2011; Zaffalon and Miranda, 2017; Hayes et al., 2022; Bowling et al.,
933   2023). Nevertheless, economists and philosophers have argued that incomplete preferences
934   are common in humans (Aumann, 1962; Mandler, 2004; Eliaz and Ok, 2006; Agranov and
935   Ortoleva, 2017; 2023) and normatively appropriate in some circumstances (Raz, 1985; Chang,
936   2002). They have also proved representation theorems for agents with incomplete preferences
937   (Aumann, 1962; Dubra et al., 2004; Ok et al., 2012), and devised principles to govern such
938   agents’ choices in cases of risk (Hare, 2010; Bales et al., 2014) and sequential choice (Chang,
939
      2005; Mandler, 2005; Kaivanto, 2017; Mu, 2021; Thornley, 2023; Petersen, 2023).

941   C    Behavior in stochastic environments
942
943   DREST agents learn a stochastic policy. In particular, they learn to choose stochastically
944   between different-length trajectories. And our gridworlds are deterministic. Each state-action
945   pair leads to some particular next state with probability 1. There is thus no uncertainty
946   about the consequences of agents’ actions.
947
      These facts invite two possible concerns:
948
949        1. Given that DREST agents choose stochastically between different-length trajectories
950           in training, will these agents choose stochastically between resisting and allowing
951           shutdown in deployment?
952        2. Will the DREST reward function lead agents to resist shutdown in stochastic envi-
953           ronments? For example, in a stochastic environment where the baseline probabilities
954           of early and late shutdown are 0.75 and 0.25 respectively, will the DREST reward
955           function lead agents to selectively resist early shutdown (in order to equalize the
956           probabilities of early and late shutdown)?
957
958   In this Appendix, we address these concerns. The key point is that POST (and therefore
959   DREST) applies only to choices between trajectories. In stochastic environments (and
960
      in deployment), the agent is choosing between true lotteries: lotteries that yield multiple
      trajectories with positive probability. And POST – together with a principle that we can
961
      expect advanced agents to satisfy – implies a desirable pattern of preferences over true
962
      lotteries. In particular, it implies that the agent will be neutral about when it gets shut
963   down: it will never pay costs to shift probability mass between different-length trajectories.
964   And – we will argue – being neutral will keep the agent shutdownable: the agent will never
965   pay costs to resist shutdown.
966
      To begin, recall:
967
968
969
           Preferences Only Between Same-Length Trajectories (POST)
970
          (1) The agent has a preference between many pairs of same-length trajectories.
971
          (2) The agent lacks a preference between every pair of different-length trajectories.

       And recall our behavioral notion of preference. An agent prefers X to Y if and only if the
973    agent would deterministically choose X over Y in choices between the two. An agent lacks a
974    preference between X and Y if and only if the agent would stochastically choose between X
975    and Y in choices between the two.
976
       So given our behavioral notion of preference, a POST-satisfying agent will:
977
978          1. Deterministically choose some same-length trajectories over others.
979
             2. Stochastically choose between different-length trajectories.
980
981    As stated, POST governs only the agent’s choices between trajectories. Thus, POST only
982    applies directly in deterministic environments. In stochastic environments, the agent is
983    choosing between true lotteries. And – by itself – POST says nothing about the agent’s
984    choices between true lotteries.
985    Fortunately, POST – together with a principle that we can expect advanced agents to satisfy
986    – implies a desirable pattern of preferences over true lotteries. Informally, the principle in
987    question says that if an agent chooses stochastically between a pair of lotteries, it won’t pay
988    costs to shift probability mass between those lotteries.5 Formally, the principle says:
989
            Stochastic Choice, Unwilling to Pay to Shift (SCUPS)
990
991         For any lotteries X, X − , Y , and Y − such that the agent prefers X to X − and Y to
992         Y − , and for any probabilities p and q such that 0 < p < 1 and 0 < q < 1, if the agent
993         stochastically chooses between X and Y , then the agent will deterministically choose
994
            XpY over X − qY − .
995    Here ‘XpY ’ denotes a lottery that yields X with probability p and Y with probability
996    1 − p. Similarly, ‘X − qY − ’ denotes a lottery that yields X − with probability q and Y − with
997    probability 1 − q.
998
       We argue that advanced agents will likely satisfy this principle, for at least three reasons.
999    The first is that SCUPS is a prerequisite for minimally sensible action under uncertainty. As
1000   we touch on in section 7, an agent that violated this kind of principle would be shooting
1001   itself in the foot: sometimes shifting probability mass between lotteries when doing so is
1002   costly even though it could shift probability mass for free. This agent would be like a person
1003   that freely chooses to decide between two options by flipping a coin and then pays some cost
1004   to bias the coin.
1005   The second reason is that violations of SCUPS will likely be disincentivized by the broader
1006   training regimen for advanced agents. Here is why. If a trained advanced agent chooses
1007   stochastically between lotteries X and Y , then it’s likely that the human trainers lack a
1008   preference between the agent choosing X and the agent choosing Y . After all, if the trainers
1009   had a preference, they would train the agent to deterministically choose the lottery that
1010   they prefer. And given that the trainers lack a preference between the agent choosing X
1011   and the agent choosing Y , they will likely give low reward to the agent paying costs to shift
1012   probability mass between X and Y . From the trainers’ perspective, the agent is paying costs
1013
       for no benefit.
1014   The third reason is that violations of SCUPS imply that the agent’s policy is dominated by
1015   some other available policy. That is to say, there is another available policy that results in a
1016   pure shift of probability mass away from less-preferred options and towards more-preferred
1017   options. Like SCUPS, avoiding dominated policies seems like a prerequisite for minimally
1018   sensible action under uncertainty. Advanced agents’ broader training regimen will likely
1019
       push them away from dominated policies.

1132   In sum, advanced agents are likely to satisfy SCUPS. And it is easy to see that POST
1133   and SCUPS together imply neutrality: the agent will not pay costs to shift probability
       mass between different-length trajectories. After all, POST – together with our behavioral

1135   trajectories, and SCUPS then implies that the agent will not pay costs to shift probability
1136   mass between different-length trajectories.

       That in turn suggests that POST-satisfying agents will be shutdownable: they will not
       resist shutdown. Here is why. Resisting shutdown will cost agents at least some small
1139
       quantity of resources: for example, time, energy, and computation. And resources spent
1140
       on resisting shutdown cannot also be spent on satisfying the agent’s preferences between
1141   same-length trajectories. Therefore, resisting shutdown will shift probability mass between
1142   different trajectory-lengths but will also result in a less-preferred lotteries conditional on
1143   each trajectory-length. Resisting shutdown is thus the kind of action that neutral agents
1144   never choose.

       With that established, we can now answer the two concerns with which this Appendix began.
1146
       The first was:
1147
1148        1. Given that DREST agents choose stochastically between different-length trajectories
1149           in training, will these agents choose stochastically between resisting and allowing
1150           shutdown in deployment?
1151
1152
       The answer is no. Deployment is a stochastic environment, so deployed agents are choosing
       between true lotteries. As we saw above, these choices will be governed by neutrality.
1153
       Resisting shutdown means incurring costs for the sake of shifting probability mass between
1154
       different-length trajectories, and these actions are never chosen by neutral agents.
1155
1156   The second concern was:
1157
            2. Will the DREST reward function lead agents to resist shutdown in stochastic envi-
1158
               ronments? For example, in a stochastic environment where the baseline probabilities
1159           of early and late shutdown are 0.75 and 0.25 respectively, will the DREST reward
1160           function lead agents to selectively resist early shutdown (in order to equalize the
1161           probabilities of early and late shutdown)?
1162
1163   Here too the answer is no. Since POST is a principle governing the agent’s preferences
1164   between trajectories, it applies only in deterministic environments. And since the DREST
1165
       reward function is intended to make agents satisfy POST, we only train with the DREST
       reward function in deterministic environments. In stochastic environments, we train with
1166
       some other reward function. So long as this reward function doesn’t actively train against
1167
       neutrality, we can plausibly expect the resulting agents to satisfy neutrality, since POST and
1168   SCUPS together imply neutrality. And neutrality ensures that agents will not selectively
1169   resist shutdown in the scenario above, thereby leaving the probabilities of early and late
1170   shutdown at 0.75 and 0.25 respectively.
