009                                                    A BSTRACT
010
011                Diffusion for code generates code by iteratively removing noise from the latent
012                representation of a code snippet. During later steps of the diffusion process, when
013                the code snippet has almost converged, differences between discrete representa-
014                tions of these snippets look like last-mile repairs applied to broken or incomplete
015                code. We evaluate the extent to which this resemblance can be exploited to lever-
016                age pre-trained code diffusion models for the problem of last-mile repair by con-
017
                   sidering two applications with significant potential. First, we can leverage the
                   diffusion model for last-mile repair by adding noise to a broken code snippet and
018
                   resuming the diffusion process. Second, we can leverage the diffusion model to
019
                   generate an arbitrary amount of training data for other last-mile repair approaches
020                (that are computationally more efficient) by sampling an intermediate program
021                (input) and the final program (output) from the diffusion process. We perform
022                experiments on three domains (Python, Excel and PowerShell) to evaluate both
023                applications, as well as analyze properties. 1

026   1       I NTRODUCTION
027
028   Diffusion models have emerged as a powerful paradigm in generative modeling, particularly for
029   tasks that involve complex data structures (Ho et al., 2020). Instead of generating a sample from a
030
      distribution in one go (like a GAN or VAE) or auto-regressively (like a GPT) they learn to iteratively
      reverse diffusion steps that add (typically Gaussian) noise to the data. Initially popularized in the
031
      domain of image generation, diffusion models have since been adapted for modalities like video
032
      generation (Ho et al., 2022; Xing et al., 2023)—which requires a temporal component—and text or
033   code generation (Li et al., 2022; Singh et al., 2023a)—which requires diffusion over discrete tokens.
034
035   One approach of applying diffusion to discrete domains, like text or code, involves embedding the
036
      input, performing diffusion in the embedded representation, and projecting the denoised embeddings
      back to discrete tokens. To train this model end-to-end, the loss incorporates a component over the
037
      discrete tokens, meaning that representation from each step of the reverse diffusion process can be
038
      converted back to the discrete space (Lin et al., 2023). During initial generations, decoding the latent
039   representation does not resemble anything and tokens frequently change, but in later generations,
040   these decoded representations become readable and it takes multiple steps to change one token.
041

      2.2                                             D IFFUSION MODELS FOR CODE
109
110   Code generation is a discrete generation task, where the expected output is a snippet c = [c1 , . . . , ck ]
111   of k tokens. CodeFusion (Singh et al., 2023a) draws inspiration from text diffusion (Li et al., 2022)
112   where each token ci is embedded E(ci ) ∈ Rd to convert c into a continuous representation E(c) ∈
113   Rkd to which a regular diffusion process can be applied. In the reverse process, a trainable rounding
114   step pθ (ci | x≤i ) computes a distribution over possible tokens for each position i given all previous
115   (denoise) tokens x≤i . Note that the decoder is trained to always generate a constant number of n > k
116
      tokens, one of which is an end-of-sequence token and n − k − 1 padding tokens. Like CodeFusion,
      we set n = 128.
117
118

121   table name missing in the structured reference which the model fixes through refinement.

149   Let ĉ be a buggy code snippet that is not accepted by the compiler. The goal of last-mile repair is
150
      to find a code snippet c∗ = arg minc d(c, ĉ) such that c∗ is accepted by the compiler and performs
      a task intended by the user, with d the edit distance between two code snippets. Like previous
151
      work on last-mile repair, we only consider syntactic errors (Bavishi et al., 2022; Joshi et al., 2023).
152

153   C ODE F USION, describe how to apply it to problem of last-mile repair, and describe how to generate
154   pairs (ĉ, c∗ ) that can be used to train specialized systems.

221   3.3     D IFFUSION MODELS AS REPAIR GENERATORS
222
223   We exploit the seemingly discrete nature of later diffusion steps to generate synthetic repair data.
224   Starting the reverse process from xT ∼ N (0, 1) we build the chain x̂T → x̂0 and decode each
225   snippet into cT → c0 . We can then select any (ct , c0 ) as a training pair if ct ̸= c0 .
226   In previous work, mistakes are introduced in the discrete token space, by implementing specialized
227   functions that imitate human errors (Yasunaga & Liang, 2020; Joshi et al., 2024) and optionally
228   training a neural network to imitate those (Yasunaga & Liang, 2021). Our aim is to show that the
229   space of discrete representations encountered during the reverse diffusion process shares enough
230   similarities to the discrete errors that humans make to be useful for last-mile repair.

      5        R ELATED WORK
476
477
      Diffusion models for text and code Diffusion models have shown their ability to gradually refine
      noisy data into realistic outputs through a denoising process (Sohl-Dickstein et al., 2015). They were
478
      originally popularized to generate photo-realistic images (Ho et al., 2020; Rombach et al., 2022a)
479
      and later applied to other high-dimensional data generation, like audio (Kong et al., 2021) and video
480   (Ho et al., 2022) synthesis. Diffusion has also been adapted for to discrete domains like text (Li
481   et al., 2022; Lin et al., 2023) and code (Singh et al., 2023a) where the ability to look at the whole
482   previous generation has benefits over auto-regressive generation. Two approaches are embedding
483   discrete tokens into a continuous space where the diffusion takes place and then decoding Li et al.
484   (2022) or directly performing diffusion in the discrete space through a transition matrix He et al.
485   (2023). In this paper, we use the former approach to explore the latent code repair capabilities of
      these models.

      Code repair Automated code repair (Zhang et al., 2023) has long been a key challenge in software
504
      engineering, with early approaches using heuristic searches (Qi et al., 2014) and program synthesis
505
      (Nguyen et al., 2013; Bavishi et al., 2022). More recently, transformer-based systems have been
506   shown adept at learning to repair code (Berabi et al., 2021; Yasunaga & Liang, 2021; Tufano et al.,
507   2019). A major limitation of training a repair model is the requirement for large quantities of data.
508   That is not true anymore for large language models, which are adept at repair code and can take in
509   additional context like error messages (Joshi et al., 2023). They are expensive to deploy, however,
510   and it is much harder to steer them to remain close to the original code snippet.
511
      Human error simulation In order to leverage the corpora of unsupervised data, previous works have
512   explored generating synthetic data using static rules (Joshi et al., 2024; Gupta et al., 2017; Hellen-
513   doorn et al., 2019) or learning to break programs in a natural way (Yasunaga & Liang, 2020; 2021).
514   These approaches are limited to the encoded rules and the quality of the learned code breaker—
515   which depends on the training data—and thus suffer from out-of-domain generalization. In this
516   work we aim to generalize the synthetic data to the long tail of out of domain cases.

      6   C ONCLUSION AND LIMITATIONS
519
520
      In this paper, we explored the potential of applying pre-trained code diffusion to the problem of
521
      last-mile repair. These diffusion models iteratively denoise a latent representation of code and the
522
      discrete decoding of intermediate steps resemble last-mile programming errors. Our experiments
523   show that injecting actual broken code into this process can cause the diffusion process to repair the
524   code, and that sampling these intermediate step yields data that can be used to fine-tune last-mile
525   repair models. In its current state, using diffusion models to generate synthetic training data shows
526   the most promise.
527
      Diffusion for code has only been applied to shorter snippets with smaller models, on relatively small
528
      datasets. Since there is no additional context, like error messages or test cases, the model might
529   not capture some of the semantics of the broken snippet. Scaling up the model, context and data
530   should further improve the potential of these models. Our findings consider the diffusion model as-
531   is: controlled decoding (Li et al., 2022) can help in remaining close to the source snippet. Like other
532   work on diffusion for text (Li et al., 2022), we note that inference is slower than auto-regressive
533   models. We can leverage work on both efficient diffusion models and efficient transformers to speed
534   up the model.
