009                                               A BSTRACT
010
011              This paper presents NT-Java-1.1B, an open-source specialized code language
012              model built on StarCoderBase-1.1B1 , designed for coding tasks in Java program-
013              ming. NT-Java-1.1B achieves state-of-the-art performance, surpassing its base
014              model and majority of other models of similar size on MultiPL-E (Cassano et al.,
015
                 2022) Java code benchmark. While there have been studies on extending large,
                 generic pre-trained models to improve proficiency in specific programming lan-
016
                 guages like Python, similar investigations on small code models for other pro-
017
                 gramming languages are lacking. Large code models require specialized hardware
018              like GPUs for inference, highlighting the need for research into building small
019              code models that can be deployed on developer desktops. This paper addresses
020              this research gap by focusing on the development of a small Java code model, NT-
021              Java-1.1B, and its quantized versions, which performs comparably to open models
022              around 1.1B on MultiPL-E Java code benchmarks, making them ideal for desktop
023              deployment. This paper establishes the foundation for specialized models across
024              languages and sizes for a family of NT Models.

      1       I NTRODUCTION
027
028   The state-of-the-art code models, capable of understanding and generating code in numerous pro-
029   gramming languages, are revolutionizing the way enterprises approach software development. With
030   the ability to understand and generate code across a vast array of programming languages, these
031   code models offer a significant boost in productivity. However, the one-size-fits-all approach of
032   these generic multi-lingual code models often falls short in meeting the nuanced requirements of
033   project-level coding tasks in an enterprise, which tend to be language-specific. This has led to the
034   development of Narrow Transformers (NTs), specialized models further trained on a particular pro-
035   gramming language, offering a more efficient solution for enterprises. These NTs are designed to
036
      optimize performance for a specific programming language, balancing the trade-offs between model
      size, inferencing cost, and operational throughput. As demand for tailored solutions grows, we can
037
      expect a surge in NT development, providing the precision and efficiency required by enterprise
038
      projects.
039
040   However, in practice, the substantial economic cost associated with training and fine-tuning large
041   code models renders language model experiments prohibitively expensive for most researchers and
042
      organizations. Additionally, deploying these massive models in everyday scenarios, such as on per-
      sonal computers, proves either inefficient or unfeasible. These challenges emphasize the importance
043
      of shifting focus to explore Narrow Transformer approach on powerful yet smaller code language
044
      models (code SLMs). Consequently, we developed a Narrow Transformer for Java within a smaller
045   parameter range (i.e., 1.1B), suitable for desktop deployment and democratizing code model exper-
046   iments.

075
      from a specific programming language does not guarantee performance improvement in other pro-
076
      gramming languages, it still guarantees performance improvement in that programming language.
077   Hence, building a model like StarCoder using a specific programming language dataset can improve
078   proficiency in that programming language. Enterprise projects are adopting either these pre-trained
079   generic multi-lingual code models or python-trained multi-lingual code models to augment their
080   project coding tasks. AI-mature enterprises are adopting these models as foundation models to fur-
081   ther train with their project code base for better augmentation. However, if there is a pre-trained
082   code model further trained on enterprise project’s required programming language, then the enter-
083   prise project can use that language-specific model and can further train with their project code base
084   for better augmentation. Due to the widespread adoption of Java in enterprise-level projects, this
085
      paper illustrates the development of such a pre-trained code model specialized on Java.
086   Small Language Models (SLMs) will pivot the focus of AI community in enterprise and consumer
087   solutions. These models stand out for their ability to be deployed on end-user devices, such as per-
088   sonal computers and smartphones, even without a GPU. This enables large-scale deployment while
089   ensuring data privacy and security. Significant examples in the present scenario of code SLMs
090   include SantaCoder-1.1B (Allal et al., 2023), Phi-1 (Gunasekar et al., 2023), DeciCoder-1B2 ,
091
      StarCoderBase-1.1B, WizardCoder-1B-V1.0 (Luo et al., 2023), DeepSeek-Coder-1b-base (Guo
      et al., 2024) and Refact-1.6B3 . All these state-of-the-art models around 1B size are multi-lingual
092
      code models, indicating that no considerable work has been done towards extending training of
093
      multi-lingual code SLMs in building language-specific code SLMs.

      4.1   DATA P REPROCESSING
118
119   For data preprocessing, we employed the Megatron-LM framework. The NT-Java-1.1B uses the
120   StarCoderBase tokenizer of type GPT2BPETokenizer (byte-level Byte-Pair-Encoding) and its vo-
121   cabulary of 49,152 tokens. No additional tokens were added to this vocabulary. The Java dataset
122   comprises 87 parquet files, which were converted into a single file and passed through the Megatron
123   pre-processing module to get the corresponding .bin and .idx files. These files were used for model
124   training. The pre-processing module also performs tokenization and adds an <EOD> token at the
125   end of each Java sample.

146   4.3   T RAINING D ETAILS
147
      NT-Java-1.1B was trained using the Megatron-LM Framework. The training began with
148
      StarCoderBase-1.1B, serving as the initial checkpoint, to build its Java variant. In our experiments,
149
      we utilized a context length of 8192 tokens for tasks involving the Next token prediction and the
150   Fill-in-the-Middle (FIM) (Bavarian et al., 2022) objective. The PyTorch Distributed framework
151   was employed, with data parallelism strategy. We chose bf16 precision and the Adam optimizer
152   (Kingma & Ba, 2015) with β1 = 0.9, β2 = 0.95, and ϵ = 10−8 , along with a weight decay of 0.1.
153
154   E XPERIMENTAL S ETTINGS
155
156   In this study, we delve into the impact of extending pretraining of StarCoderBase-1.1B for Java
157   using two key objectives: Next token prediction and Fill-in-the-Middle.
158   Experiment 1 - Next token prediction objective: We conducted training over 100,000 steps (equiva-
159   lent to 5 epochs) with a batch size of 1 million tokens. The learning rate commenced at 4×10−4 and
160   underwent cosine decay, reaching a minimum of 4×10−6 with 1,000 iterations of linear warmup. A
161   global batch size of 180 facilitated the training process, which spanned 12 days. Model checkpoints
      were saved every 1,000 steps for subsequent evaluation.

      6        C ONCLUSION
291
      In this technical report, we outlined the rationale and training approach used to develop NT-Java-
292
      1.1B, a small language model trained specifically on Java code. We evaluated NT-Java-1.1B across
293
      various coding tasks and compared its performance against models with similar parameters. Our
294   findings indicate that NT-Java-1.1B is competitive with or outperforms other Code SLMs in this
295   parameter range in Java programming tasks.
296
297
      This study demonstrates the successful achievement of its objective of enhancing the efficiency
      of a code SLM for a particular programming language by training it further with a subset of its
298
      dataset for that language. While the research employed the StarCoderBase-1.1B model and its
299
      Java language dataset, other SLMs and their associated programming language datasets can yield
300   comparable experimental outcomes.
301
302   The release of NT-Java-1.1B and its variants aims to democratize code foundation models, making
303
      them accessible for deployment in memory-constrained environments such as developer desktops
      and laptops. By adhering to the principles of the OpenRAIL-M12 and by open-sourcing the cor-
304
      responding scripts on GitHub, we hope to enable both the research and developer communities to
305
      experiment and adopt code SLMs.
