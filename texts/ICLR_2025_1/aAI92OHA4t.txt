011                                                A BSTRACT
012
013            Trained neural networks (NN) are attractive as surrogate models to replace costly
014            calculations in physical simulations, but are often unknowingly applied to states
015            not adequately represented in the training dataset. We present the novel technique
016
               of soft checksums for scientific machine learning, a general-purpose method to
               differentiate between trustworthy predictions with small errors on in-distribution
017
               (ID) data points, and untrustworthy predictions with large errors on out-of-
018
               distribution (OOD) data points. By adding a check node to the existing output
019            layer, we train the model to learn the chosen checksum function encoded within
020            the NN predictions and show that violations of this function correlate with high
021            prediction errors. As the checksum function depends only on the NN predic-
022            tions, we can calculate the checksum error for any prediction with a single for-
023            ward pass, incurring negligible time and memory costs. Additionally, we find that
024            incorporating the checksum function into the loss function and exposing the NN
025            to OOD data points during the training process improves separation between ID
026            and OOD predictions. By applying soft checksums to a physically complex and
027
               high-dimensional non-local thermodynamic equilibrium atomic physics dataset,
               we show that a well-chosen threshold checksum error can effectively separate ID
028
               and OOD predictions.

      1    I NTRODUCTION
032
033
      An ability to detect errors would increase trust in machine learning (ML) surrogate models to make
034
      scientific predictions. In critical applications, unnoticed errors can have severe consequences, lead-
035
      ing to inaccurate conclusions and poor engineering design choices. These errors are often caused by
036   applying a surrogate model on data points where it is not a valid approximation of the true function.
037   For a given surrogate model, there exists a domain of validity where you can reliably characterize
038   how the network behaves due to the use of a validation dataset during training. We aim to develop a
039   metric that excludes predictions when we are not confident that the surrogate model is reliable.
040
      Uncertainty quantification attempts to distinguish between the often unknown domain of validity (or
041
      validation domain) and the domain of intended use when discussing computational physics surrogate
042   models (Oberkampf et al., 2004; Riedmaier et al., 2021; Roy & Oberkampf, 2010). While the
043   domain of validity has boundaries based on the physical experiments or simulations conducted to
044   populate the validation dataset, the domain of intended use refers to all physical states the user may
045   apply the surrogate model to. Ideally there is total overlap, but in reality it is difficult for a user to
046   define the boundaries, and often portions of the domain of intended use are outside of the domain of
047   validity. While physics-based models may have some level of confidence in regions outside of the
048   domain of validity due to a deep understanding of the system, the user can only rely on an estimation
049   of the boundaries and extrapolative capability.
050   This same idea exists in machine learning, which often views data points as sampled from an un-
051   known distribution and correspondingly refers to the domain of validity as the set of in-distribution
052   (ID) data points, and all other data points as out-of-distribution (OOD). While still difficult to detect
053   in practice, the difference between ID and OOD data in classification problems with discrete outputs
      can be simpler to visualize as the OOD data is often from entirely different datasets or classes. How-


                                                          1
054
      ever, when a classification model attempts to identify a blurry image, or a regression model maps
055   inputs to a continuous, potentially high dimensional output space, there is a clearer analogy to com-
056   putational physics and the similarly unknown boundaries between ID and OOD data. Specifically
057   for regression problems, all predictions will have non-zero error, eliminating the binary evaluation
058   of correct or incorrect. Instead, we want to differentiate between ID predictions with acceptably
059   low error and OOD predictions with unacceptably high error, a challenging task due to the complex,
060   application dependent boundary.
061   We can consider ML surrogates as both uncertain physical surrogates (with a domain of validity
062   separate from their domain of use) and as statistical ML models (trained on ID data, but expected
063   to encounter OOD data). An ML surrogate offers the advantage of providing faster results with less
064   computational cost (Almeldein & Van Dam, 2023; Carranza-Abaid et al., 2020; Ganti et al., 2020;
065   Kluth et al., 2020), but this comes with the risk of unnoticed errors propagating and rendering the
066   simulation useless. In particular, it would be helpful if we could detect when the model is being
067   asked to predict outside the domain of validity (i.e. on OOD data). We could use this information
068   to decide whether to trust our final results, or to revert back to detailed and expensive physics sub-
069
      simulations to preserve the reliability of the overall simulation.
070   Our goal is to provide information to help the user by raising a nominal red flag if an ML surrogate
071   model is likely predicting on OOD data and should not be trusted in scientific regression appli-
072   cations. A naive approach might assume that a trained surrogate is able to predict with sufficient
073   accuracy on any data point within a hypercube bounding the training dataset. However, this is likely
074   not valid for physical problems. Collecting data from trusted simulations or experiments likely does
075
      not produce an evenly sampled distribution to populate the training dataset. Especially in high di-
      mensions, this could result in gaps where data points may be physically possible, but are not well
076
      represented.
077
078   The main contribution of this work is a novel checksum based method for indicating untrustworthy
079   predictions due to OOD data. Similar to checksums in message transmissions as described in Sec-
080   tion 2.2, we add an additional output to the surrogate model and encode a checksum function. With
081
      this known relationship between the outputs, we can calculate a checksum error for each prediction.
      We can then differentiate between ID and OOD data as having low and high checksum errors re-
082
      spectively, and flag when the predictions should not be trusted. We refer to the encoded function as
083
      a soft checksum because while it is like a traditional checksum in that it can indicate potential errors,
084   it differs in that it produces a continuous, rather than binary, signal.
085
086   We also propose a modified loss function, combining ideas from Physics Inspired Neural Net-
087
      works (PINNs) (Raissi et al., 2019), fine-tuning (Liu et al., 2020), and Outlier Exposure techniques
      (Hendrycks et al., 2019). We use additional terms in the loss function to shape the checksum er-
088
      ror surface and more consistently produce low values for ID predictions, and high values for OOD
089
      predictions. To achieve this goal, we implement a novel method for exposing a surrogate model to
090   random OOD data during the training process, without biasing it towards a limited OOD region.
091
092   Importantly, using a soft checksum to flag untrustworthy predictions only requires a single model
093
      and forward pass, incurring negligible time and memory costs. This is a general method that makes
      no a priori assumptions about the data, and can be easily added to existing model architectures.

098   2.1   O UT- OF -D ISTRIBUTION D ETECTION
099
100   There are several different methods that have been proposed for OOD detection and uncertainty
101   estimates in regression problems. Bayesian methods are commonly used to approximate a poste-
102   rior distribution with an uncertainty estimate, but can be limited by inaccurate priors and difficulty
103   scaling to large datasets (Blundell et al., 2015; Fortuin et al., 2022; Neal, 1996; Wilson & Izmailov,
104
      2022; Yang et al., 2019). Monte Carlo dropout has been shown to approximate Bayesian methods,
      but retains some of the same limitations with a slower training process (Gal & Ghahramani, 2016;
105
      Wang & Manning, 2013).
106
107   The most common alternatives to these methods are deep ensembles, which produce comparable,
      if not better, results than Bayesian methods by training multiple models and using the variance of


                                                         2
108
      the predictions as a measure of uncertainty (Lakshminarayanan et al., 2017). However, this method
109   involves training and evaluating multiple models, incurring larger computational costs. To avoid the
110   need for multiple models and the complications of training a Bayesian neural network, recent work
111   has focused on anchor based training which can produce an uncertainty with multiple evaluations of
112   a single model (Thiagarajan et al., 2022; 2024).
113
      Predating regression applications, there is considerable work developing OOD detection for classi-
114
      fication problems (Yang et al., 2024). Most relevant for this paper are approaches which improve
115   detection by including OOD data points in the training process. Both Outlier Exposure (Hendrycks
116   et al., 2019) and an energy-based method (Liu et al., 2020) include explicitly OOD inputs in the
117   training process by adding a term in the loss function which trains the model how to flag these data
118   points.
119
120   2.2   C HECKSUMS
121
122   Checksums have been around for decades to verify data integrity, with Fletcher’s Checksum being
123   proposed in 1982 (Fletcher, 1982). The sender adds check bytes to the end of a transmission such
124   that the calculated checksum should be zero. If the receiver does not calculate the same value, this
125   indicates a transmission error and the message is resent.
126   However, the strict requirement of zero errors to satisfy a checksum is not always necessary. It is
127   often sufficient to transmit images and videos with limited errors, or it may be more important to
128   deliver the message and avoid the cost of retransmission rather than fixing corrupt bits. For those
129   cases, it is more useful to estimate the fraction of corrupted bits with error estimating codes (Chen
130   et al., 2010; Zhang & Kumar, 2017) or a soft checksum (Lee & Bahk, 2021), and allow the receiver
131   to set a maximum threshold error, below which there is no need for retransmission.
132   In this paper, we bring these checksum concepts into machine learning for identifying prediction
133   errors. As regression machine learning models will have non-zero error on predictions outside of
134   the training dataset, a binary checksum method would flag all predictions. Instead, we borrow the
135   term soft checksum and error estimation ideas to describe our method for differentiating between ID
136   and OOD data based a continuous checksum and user defined threshold error.

148   3.1   C HECKSUM O UTPUT
149
150   Leveraging the framework of checksums for message transmission errors, we propose a new strategy
151   to detect model prediction errors with a single forward pass by adding one additional output node
152   to the neural network as shown in Figure 1. Adding a check node means that the neural network
153   predicts (ŷ, Ĉy ), where Ĉy attempts to match the checksum function C(ŷ). The user can choose the
154   checksum function for the particular application and dataset. Given that we can always calculate a
155   checksum error L(Ĉy , C(ŷ)) without needing to know the true y values, we make a similar argument
156   as energy based OOD detection (Liu et al., 2020). If the model is unable to produce a small enough
157   checksum error, then the model is likely predicting on OOD data and should not be trusted.
158   In practice, the simulation workflow should answer a binary question, do we trust the prediction or
159   not? The requirements for trusting a prediction will differ for each specific problem, as well as the
160   cost of being wrong. In some cases, incorrectly trusting any OOD prediction (false negative) may
161   be detrimental to the simulation, while in others there may be a level of tolerance depending on the
      error magnitude or number of false negatives. Similarly, some calculations may be so costly that

240   Choosing an optimal method to sample DOOD to calculate LOOD is not always an easy problem
241   due to complexities in delineating ID datasets. The user may bias the model towards specific OOD
242
      data points by only including data from a limited region in the input space, or including data points
      in the OOD dataset that are actually ID. We avoid these issues by randomly sampling data points
243
      outside of the hypercube bounding Dtraining . These data points are not sampled from simulations or
244
      experiments and are well outside the maximum possible bounds of the training data by construction.
245   This procedure seeks to suppress biases or invalid inclusions when calculating LOOD .

249   4   N UMERICAL E XPERIMENT
250
251
      We demonstrate the effectiveness of checksum errors as an OOD detector for a surrogate model
252   of Non-Local Thermodynamic Equilibrium (NLTE) calculations, a key step in atomic kinetics and
253   radiation transport calculations. Applications span a variety of fields, including inertial confinement
254   fusion (ICF), magnetic fusion, X-ray lasers and laser-produced plasmas. With respect to ICF specif-
255   ically, the atomic physics code Cretin (Scott, 2001) carries out the NLTE calculations, taking ten to
256   ninety percent of the total simulations wall clock time (Kluth et al., 2020).
257
      The machine learning surrogate model takes in the electron density, temperature, and radiation spec-
258   trum, and predicts the absorption spectrum. For our example, each spectrum is each defined by 85
259   frequency bins, resulting in an 87 dimensional input space and 85 dimensional output space. We
260   generate the dataset by running many ICF simulations in Cretin, and manually divide ID data for
261   training and validation, and OOD data for evaluating the soft checksum metric, as shown in Figure
262   2. In this way, we are only considering if a soft checksum can flag the more difficult and relevant
263   subset of DOOD that is realistic for the surrogate model to encounter in a simulation.
264   For this study, we conducted a limited parameter sweep to determine the optimal hyperparameters
265   for the given experiment. Importantly, this was not a general method of selecting the hyperparam-
266   eters and depended on the chosen OOD dataset. Specifically, we set λID and λOOD to 0.01. To
267   calculate LOOD , we sample a subset of DOOD with values between 20% to 25% outside of the hyper-
268   cube bounding Dtraining . When encoding (2) as the checksum function, we set w = 0.0001 to achieve
269   a nonlinear relationship while also maintaining a low enough frequency that C(ŷ) is not effectively
      random noise.

301   5   D ISCUSSION
302
303   As introduced in Section 3.1, we measure OOD detection effectiveness using a False Negative at
304   99% True Negative rate (FNR99). This represents the percentage of OOD predictions that have a
305   checksum error less than the 99% True Negative value and our method would incorrectly not flag.
306   Table 1 reports FNR99 rates for soft checksums implemented with four different loss functions. As
307   shown, simply adding the check node to encode a checksum function without including LID or LOOD
308   achieves strong separation between ID and OOD data points. We can further improve the separation
309   by including LOOD in the loss function, effectively training the model to increase checksum errors
310   on OOD predictions. As shown in Figure 3, in addition to separation between ID and OOD data,
311   there is also a linear correlation between checksum error and prediction error for OOD data. The
312
      correlation between errors potentially allows for soft checksums to not only flag OOD predictions,
      but also serve as a proxy for prediction error.
313
314   On the other hand, including LID in the loss function surprisingly has a negative effect on the sep-
315   aration. When including both LID and Lchecksum and assuming there is non-zero error, we con-
316   currently train the surrogate to predict two different values of Ĉy . Lchecksum pushes the prediction
317   towards C(y), while LID pushes the prediction towards C(ŷ). This could explain the decrease in
318   performance and presents an opportunity to better understand how the surrogate model learns the
319   checksum function and the conflict between the two terms.
320   Our proposed method of applying soft checksums to flag untrustworthy predictions shows promis-
321   ing results and deserves more in-depth study. While considerably cheaper and simpler to implement
322   than many current state-of-the-art OOD detection methods, we must also conduct benchmark com-
323   parisons to establish the relative effectiveness. As part of these comparisons, there are areas for
      improvement to investigate.

           1. The checksum function is currently not optimized. An ideal checksum function is complex
345
              enough that the ML surrogate model cannot memorize it for any given inputs, but is not too
346
              complex that it cannot learn the function for ID inputs. For the sinusoid checksum function
347           (2), varying the frequency hyperparameter w spans both of these conditions. If it is set too
348           high, then the ML model sees effectively random noise, but if set too low, it reduces to a
349           simpler linear or constant function.
350
           2. Adding multiple check nodes to encode multiple checksum functions should better reveal
351           the edges of the domain of validity for a given surrogate model. While it is possible that
352           the model learns to memorize one checksum function and produce a low checksum error
353           on OOD data, it is less likely that this is the case for multiple checksum functions. Adding
354           redundancies will reduce the possibility of coincidentally low checksum errors.
355        3. Incorporating OOD data points in the training process improves OOD detection, but has
356           limitations. Optimizing the distance outside of the hypercube to sample DOOD requires a
357           balance between being close enough to the boundary to improve ID and OOD separation,
358           but far enough away to avoid difficulty training the surrogate model. Additionally, while
359           sampling outside of a bounding hypercube guarantees there is no overlap with Dtraining ,
360           it also misses potential OOD regions within the hypercube and holes within the training
361           dataset. There is an opportunity to better capture these regions and improve OOD detection.
