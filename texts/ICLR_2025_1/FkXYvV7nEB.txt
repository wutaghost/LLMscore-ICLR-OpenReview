009                                                A BSTRACT
010
011            Transformer has become the de facto architecture for Large Language Models
012            (LLMs), yet its substantial memory required for long contexts makes it costly to
013            deploy. Managing the memory usage of the key-value (KV) cache during infer-
014            ence has become a pressing challenge, as the cache grows with both model size
015            and input length, consuming significant GPU memory.
016            We introduce a novel post-training KV cache compression method using KV-Dict,
017            a universal dictionary that can accurately decompose and reconstruct key-value
018
               states. Unlike traditional quantization methods, KV-Dict leverages sparse dictio-
               nary learning, allowing for flexible memory usage with minimal performance loss
019
               through fine-grained controls of sparsity levels. Moreover, we retain competitive
020
               performance in the low memory regimes that 2-bit compression struggles to offer.
021
               KV-Dict is remarkably universal, as it uses a small, input-agnostic dictionary that
022
               is shared across tasks and batches without scaling memory. This universality,
023            combined with the ability to control sparsity for different memory requirements,
024            offers a flexible and efficient solution to the KV cache bottleneck, maintaining
025            strong performance on complex reasoning tasks, such as LongBench and GSM8K.

028   1   I NTRODUCTION
029
030   Transformers (Vaswani et al., 2017) have become the backbone of frontier Large Language Models
031
      (LLMs), driving progress in domains beyond natural language processing. However, Transformers
      are often limited by their significant memory demands. This stems not only from the large num-
032
      ber of model parameters, but also from the need to maintain the key-value (KV) cache that grows
033
      proportional to the model size and length of the input. Additionally, each input requires its own
034   KV cache, limiting opportunities for reuse across different user inputs. This creates a bottleneck in
035   generation speed for GPUs with limited memory (Yu et al., 2022) and thus, it has become crucial to
036   alleviate KV cache memory usage while preserving its original performance across domains.
037
      KV cache optimization research has explored both training-stage optimizations (Shazeer, 2019; Dai
038
      et al., 2024; Sun et al., 2024) and deployment-focused strategies (Kwon et al., 2023; Lin et al.,
039
      2024; Ye et al., 2024) to improve the efficiency of serving LLMs. Architectural approaches such as
040   Grouped Query Attention (GQA) (Ainslie et al., 2023) reduce the number of KV heads, effectively
041   reducing KV cache. However, these methods are not applicable as off-the-shelf methods to reduce
042   KV cache for pretrained LLMs, leading to post-training compression efforts.
043
      Post-training approaches include selectively retaining certain tokens (Beltagy et al., 2020; Xiao et al.,
044
      2023; Zhang et al., 2024) and quantization methods, which have had empirical success when quan-
045
      tizing KV cache into 2 or 4 bits (Liu et al., 2024b; He et al., 2024; Kang et al., 2024). However,
046   eviction strategies have limitations on long-context tasks that require retaining a majority of previous
047   tokens, while quantizations to 2 or 4 bits have clear upper bounds on compression rates.
048
049
      In this paper, we focus on utilizing low-dimensional structures for efficient KV cache compression.
      Prior work reports that each key state lies in a low-rank subspace (Singhania et al., 2024; Wang
050
      et al., 2024b; Yu et al., 2024). Yet, it is unclear if all states lie in the same subspace; if so, such
051
      redundancy remains to be taken advantage of. Thus, we naturally ask the following questions:
052
053           Can we find substantial redundancy among keys and values, universal of all inputs?
                        How can we leverage this for efficient KV cache compression?

074   Towards this end, we propose KV-Dict, a universal dictionary that serves as an overcomplete basis,
075   which can sparsely decompose and reconstruct KV cache with sufficiently small reconstruction er-
076   ror. As shown in Section 2.2, we observe that a subset of key states cluster near each other, even
077
      though the keys are from different inputs, while some cluster on different subspaces. To accommo-
      date for such low-dimensional structures, we use sparse dictionary learning, which has developed
078
      algorithms for information compression across various domains, such as signal processing and med-
079
      ical imaging (Candès et al., 2006; Donoho, 2006; Dong et al., 2014; Metzler et al., 2016).
080
081   KV-Dict is simple to learn, can be applied off-the-shelf for KV cache compression, and only occupies
082   small constant memory regardless of input or batch size. Methodologically, KV-Dict utilizes both
083
      sparsity-based compression (steps 1 and 2) and quantization (step 3) in three straightforward steps:
084
            1. Dictionary pretraining: As in Figure 3, we train a dictionary on WikiText-103 (Merity,
085
               2016) for each model. This dictionary is only trained once and used universally across all
086            tasks. It only occupies constant memory and does not increase with batch size.
087
088         2. Sparse decomposition: During prefilling and decoding (Figure 1), KV-Dict decomposes
089            key-value into a sparse linear combination, which consists of s pairs of sparse coefficient
090            and dictionary index. This step by itself provides high compression rates.
091         3. Lightweight sparse coefficients: We obtain higher KV cache compression rates by rep-
092            resenting the sparse coefficients in 8 bits instead of FP16. Lowering precision to 8 bits
093            yields minimal degradation. KV-Dict theoretically allows us to compress more than 2-bit
094            quantization (1/8 of FP16 KV cache size) if s ≤ 10 when head dimension is 128.
095
096   Overall, we make the following contributions:
097
098          • Near-lossless performance: Given similar memory requirements, KV-Dict performs on
099            par with or better than baseline quantization methods on challenging language tasks, such
100
               as LongBench (Bai et al., 2023) and GSM8K (Cobbe et al., 2021).
101          • Compression rates beyond 2-bits: KV-Dict’s sparsity parameter enables both wider and
102            more fine-grained control over desired memory usage. This allows us to explore perfor-
103            mance when using under 15-20% of the original KV cache size, a low-memory regime
104            previous compression methods could not explore.
105
             • Universality: Instead of an input-dependent dictionary, we find a sufficiently small univer-
106
               sal dictionary (per model) that can be used for all tasks and across multiple users. Advan-
107            tageously, such dictionary does not scale with batch size and can be used off-the-shelf.

133   2.2   S PARSE A PPROXIMATION
134
135   Given a dictionary, our goal is to efficiently decompose and represent KV cache, i.e., approximate
136   a state k ∈ Rm as a linear combination of a few vectors (atoms) from an overcomplete dictionary
137   D ∈ Rm×N . This reconstruction is given by k = Dy, where y ∈ RN is the sparse representation
138   vector such that s = ||y||0 . For implementation, y only requires space proportional to s, not N .
139   We hypothesize that the KV cache, like other domains where sparse approximation is effective,
140   contains inherent redundancy that can be leveraged for efficient compression. For instance, Figure 2
141   presents pairwise cosine similarity plots for keys generated during inference on a random subset
142   of the WikiText dataset. Here, we observe that key states cluster in multiple different subspaces.
143   Dictionary learning can take advantage of such redundancy, enabling KV vectors to be represented
144   by a compact set of atoms with only a few active coefficients.

      In this work, we adopt Orthogonal Matching Pursuit (OMP) as the sparse approximation algorithm.
188   Given an input key or value vector k, a dictionary D, and a target sparsity s, OMP iteratively se-
189   lects dictionary atoms to minimize the ℓ2 -reconstruction error, with the process continuing until the
190   specified sparsity s is reached. Our implementation of OMP builds on advanced methods that utilize
191   properties of the Cholesky inverse (Zhu et al., 2020) to optimize performance. Additionally, we
192   incorporate implementation details from Lubonja et al. (2024) for efficient batched GPU execution
193   and extend it to include an extra batch dimension, allowing for parallel processing across multiple
194   dictionaries. The full algorithm is detailed in Appendix A.

196   2.3   L EARNING L AYER - SPECIFIC KV-D ICT
197
      Layer-specific Dictionaries. While the sparse approximation algorithm is crucial, achieving a
198
      high compression ratio relies heavily on well-constructed dictionaries. In this section, we describe
199
      the process for training the dictionaries used in KV-Dict. We adopt distinct dictionaries for the key
200   and value vectors in each transformer layer due to their different functionalities. We denote the key
201   and value dictionaries at each layer as Dk and Dv ∈ Rm×N , where N is the fixed dictionary size.
202   With N = 1024, the dictionaries add an additional 16.8MB to the model’s storage requirements for
203   7B/8B models.
204
      As shown in Figure 3, we train layer-specific KV dictionaries through direct gradient-based opti-
205
      mization. For a given key or value vector, denoted as k ∈ Rm and a dictionary D ∈ Rm×N ,
206   the OMP algorithm approximates the sparse representation y ∈ RN . This process is parallelized
207   across multiple dictionaries, but for simplicity, we present the notation for a single dictionary. The
208   dictionary training objective minimizes the ℓ2 norm of the reconstruction error, with the loss func-
209                        2
      tion L = ∥k − Dy∥2 . We enforce unit norm constraints on the dictionary atoms by removing any
210   gradient components parallel to the atoms before applying updates.
211
212   Training. The dictionaries are trained on KV pairs generated from the WikiText-103 dataset using
213   Adam (Kingma & Ba, 2014) with a learning rate of 0.0001 and a cosine decay schedule over 20
214   epochs. The dictionaries are initialized with a uniform distribution, following the default initializa-
215   tion method for linear layers in PyTorch. For Llama-3.1-8B-Instruct, with a sparsity of s = 32 and
      a dictionary size of N = 1024, the training process takes about 2 hours on an NVIDIA A100 GPU.

232   layer-specific key and value dictionaries, Dk and Dv .

      When the buffer reaches capacity, OMP compresses the KV vectors for the oldest na tokens in
271   the buffer. This process is independent of the attention computation for the newest token and can
272   therefore be performed in parallel.
273
274   Time and Space Complexity. The sparse representations are stored in CSR format, with values
275   encoded in FP8(E4M3), and all indices, including offsets, are stored as int16. Each row in CSR
276   corresponds to a single key or value vector. For a given sparsity level s, the memory usage includes:
277   nonzero values (s bytes), dictionary indices (2s bytes), and the offset array (2 bytes), resulting in a
278   total size of 3s + 2 bytes. For a head dimension of 128, a fully uncompressed vector using FP16
279   takes 256 bytes, yielding a memory usage of 3s+2
                                                     256 × 100 ≈ 1.17s% (e.g., 37.5% for s = 32).

281                                 ⊤

282
      tion is particularly well-suited for long-context tasks when lseq > m where m is anywhere between
283
      1024 and 4096. For short contexts when lseq < m, our method only adds a small overhead to
284   attention computation in actuality.

289   Setup. We evaluate our method on various models (Llama-3-8B, Llama-3.1-8B-Instruct,
290   Llama-3.2-1B-Instruct, Llama-3.2-3B-Instruct, Mistral-7B-Instruct), using dictionaries trained on
291   WikiText-103, as done in Section 2.3. To assess KV-Dict’s effectiveness in memory reduction while
292
      maintaining long-context understanding, we conduct experiments on selected tasks from Long-
      Bench (Bai et al., 2023), following the setup of Liu et al. (2024b). See Appendix B for task details.
293
294   Additionally, we evaluate generative performance on complex reasoning tasks, such as
295   GSM8K (Cobbe et al., 2021) with 5-shot prompting and MMLU-Pro Engineering/Law (Wang et al.,
296   2024a) with zero-shot chain-of-thought. We compare our method against two kinds of KV cache
297   compression methods: namely, quantization-based compression and eviction-based compression.
298
      For quantization-based methods, we evaluate KIVI (Liu et al., 2024b), ZipCache (He et al., 2024),
      and the Hugging Face implementation for per-token quantization. For eviction-based methods, we
299
      evaluate PyramidKV (Cai et al., 2024) and SnapKV (Li et al., 2024). We refer to the 4-bit and 2-bit
300
      versions of KIVI as KIVI-4 and KIVI-2, respectively, and denote its quantization group size as g.
301
302   We report KV size as the average percentage of the compressed cache relative to the full cache at
303   the end of generation. KV-Dict’s sparsity s is set to match the KV size of the baseline.
304
305   Hyperparameter Settings. For both experiments, KV-Dict uses a dictionary size of N = 4096, a
306   buffer size of nb = 128, and an approximation window size na = 1, compressing the oldest token
307
      with each new token generated. For KIVI-4 and KIVI-2, we use a quantization group size of g = 32
      and a buffer size of nb = 128 , as is tested and recommended in Liu et al. (2024b), for LongBench.
308
      For GSM8K and MMLU-Pro, we test for stronger memory savings, so we use g = 64 and nb = 64
309
      for KIVI.

499   4    R ELATED W ORK
500
501   Prior work on KV cache optimization has explored both training-stage and deployment-focused
502
      strategies to improve the efficiency of LLMs. On the deployment side, Kwon et al. (2023) intro-
      duce a Paged Attention mechanism and the popular vLLM framework, which adapt CPU-style page
503
      memory to map KV caches onto GPU memory using a mapping table, thereby minimizing memory
504
      fragmentation and leveraging custom CUDA kernels for efficient inference. While there is a signifi-
505   cant and important line of research in this direction (Lin et al., 2024; Qin et al., 2024), this direction
506   is orthogonal to our work and can often be used in tandem with quantization.
507
508
      Current post-training KV cache compression methods can broadly be categorized into eviction,
      quantization, and merging. Zhang et al. (2024) introduced H2O, which uses attention scores to se-
509
      lectively retain tokens while preserving recent ones that are strongly correlated with current tokens.
510
      Multiple works discuss various heuristics and algorithms to find which tokens can be discarded,
511   while some works find how to complement evictions methods (Ge et al., 2023; Li et al., 2024; Liu
512   et al., 2024a; Devoto et al., 2024; Dong et al., 2024). For this line of work, there is a chance that
513   evictions can work well together with KV-Dict, as Liu et al. (2024a) have successfully combined
514   quantization and eviction.
515
      Quantization methods have also played a crucial role in reducing KV cache size without compro-
516
      mising model performance. Although there is a flurry of work, we only mention those that are most
517   relevant to our discussion and methodology. Hooper et al. (2024) identified outlier channels in key
518   matrices and developed KVQuant, while Liu et al. (2024b) pursue a similar per-channel strategy
519   in KIVI. Further extending these ideas, Yue et al. (2024) presented WKVQuant, which quantizes
520   model weights as well as KV cache using two-dimensional quantization. Kang et al. (2024) follow
521   similar per-channel key and per-token value quantization as KIVI, but with additional low-rank and
522   sparse structures to manage quantization errors.

      5    C ONCLUSION
525
526
      In conclusion, our proposed method, KV-Dict, offers a novel approach to compressing KV cache
527
      for transformers by leveraging low-dimensional structures and sparse dictionary learning. Through
528   this method, we demonstrate that substantial redundancy exists among key states across various in-
529   puts, allowing us to compress the KV cache efficiently while maintaining near-lossless performance.
530   Furthermore, KV-Dict enables compression rates that surpass traditional quantization techniques, of-
531   fering fine-grained and wide control over memory usage. Importantly, our universal dictionary is
532   both compact and scalable, making it applicable across tasks and user inputs without increasing
533   memory demands. This approach provides strong memory savings, particularly for long-context
534   tasks, by alleviating the memory bottlenecks associated with KV cache storage without dropping
535   any previous tokens.
536   Future research directions based on our work include optimizing CSR tensors through customized
537   quantizations and improving latency tradeoffs that occur due to the use of OMP during prefilling
538   and decoding. It would also be interesting to apply “soft-eviction strategies” for sparse tensors in
539   which sparsity level s is determined or dropped later on based on the estimated importance of the
      token. A dynamic allocation of sparsity can further improve our compression method.
