009                                              A BSTRACT
010
011
               To train Large Language Models (LLMs) having a large number of parameters, the
012
               Parameter-Efficient Fine Tuning (PEFT) method based on LoRA, which allows
013
               fine-tuning with fewer parameters, is widely employed. However, these meth-
014            ods are primarily designed for application to Transformer architectures, which
015            presents challenges when attempting to apply them to models such as Mamba.
016            To address this limitation, this work proposes Universal LoRA (ULoRA), which
017            applies a Low-Rank Adapter to all deep learning models at the level of univer-
018            sally common blocks. ULoRA achieves generalizability by applying Low-Rank
019            Adapters to blocks, making it applicable to models that do not utilize Transformer
020            architectures. Furthermore, by grouping multiple blocks and applying a single
021            Low-Rank Adapter, ULoRA provides structural flexibility that allows a further
022
               reduction in the number of parameters. This significantly reduces resource usage
               and inference time, making it well-suited for on-device environments with lim-
023
               ited resources, while only incurring a slight performance loss. Additionally, if all
024
               blocks are grouped to use a single Low-Rank Adapter, task switching during in-
025            ference is enabled by computing only the adapter. Experimental results show that,
026            for LLaMA-3-8B, ULoRA achieves comparable performance to LoRA with only
027            about 60% of the parameters, while delivering up to 8% higher throughput. For
028            Mamba-2.8B, ULoRA outperforms LoRA with only about 20% of the parame-
029            ters. In scenarios with limited available resources, ULoRA can be applied using
030            just 4% of the parameters of LoRA, with only a 10% reduction in performance.

      1   I NTRODUCTION
035
036   Large Language Models (LLMs), which perform tasks such as summarization, translation, predic-
037   tion, and generation based on knowledge obtained from large datasets, have seen rapid advance-
038   ments recently in the field of Natural Language Processing (NLP). LLMs such as GPT-3 (175B)
039
      (Brown et al. (2020)), PaLM (540B) (Chowdhery et al. (2022)), and LLaMA (70B, 400B) (Touvron
      et al. (2023a), Touvron et al. (2023b), AI@Meta (2024)) have demonstrated impressive performance
040
      across a wide range of tasks. Moreover, models like VisionTransformer (Dosovitskiy et al. (2021))
041
      have achieved high performance in the field of computer vision. However, as the parameter size of
042   these models continues to grow, the economic costs associated with their training and deployment
043   have also increased dramatically.
044
045
      To address this issue, Parameter-Efficient Fine-Tuning (PEFT) (Mangrulkar et al. (2022)) methods
      have been proposed. PEFT allows fine-tuning by updating only a portion of the model parame-
046
      ters, thus significantly reducing computational costs while still achieving comparable or equivalent
047
      performance. A representative PEFT method is Low-Rank Adaptation (LoRA) (Hu et al. (2021)),
048   which adds Low-Rank Adapters to some parameters of the model and trains only the adapters, with-
049   out directly training the model itself. This approach can reduce the number of parameters required
050   for training by up to 10,000 times. Moreover, during inference, LoRA merges the adapter weights
051   with the model parameters, thereby eliminating any structural overhead. While LoRA significantly
052   reduces the number of trainable parameters by adding a simple structure, it is not applicable to all
053   models. Since LoRA mainly supports linear layers, it is challenging to apply it to models like Mamba
      (Gu & Dao (2024)) and ResNet (He et al. (2015)).

054
      To find a way to apply Low-Rank Adapters to all deep learning models, we analyzed the common
055   structural components of these models. Most deep learning models, including LLMs, use an internal
056   structure divided into blocks (layers). In this paper, we define the largest common structural unit
057   within models as the ”Outer Block” (BLO ) and propose Universal LoRA (ULoRA), which applies
058   Low-Rank Adapters to these BLO . By dividing models into BLO and applying Low-Rank Adapters
059   at this level, ULoRA achieves general applicability across different models. Furthermore, ULoRA
060   offers structural flexibility, as it is also possible to group multiple BLO into a larger block and apply
061   a single Low-Rank Adapter. For instance, by grouping two BLO together, the number of BLO , and
062   consequently the number of adapters, is reduced by half. This reduction in the number of adapters
063
      allows for the construction of adapters with fewer parameters compared to LoRA.
064   In resource-constrained environments, such as on-device models for mobile devices, ULoRA pro-
065   vides a significant advantage by allowing extreme reductions in parameter count, leading to consider-
066   able decreases in resource usage and inference time. If all BLO are grouped into a single Low-Rank
067   Adapter, the last hidden state remains unchanged during inference. Therefore, for task switching,
068   only the replaced adapter needs to be computed, resulting in substantial reductions in inference
069
      time.
070   We proposed in this work, ULoRA, which applies Low-Rank Adapters—traditionally limited to
071   linear layers—to the largest common structure of a model, termed the BLO , making it applicable to
072   diverse deep learning models. The key contributions of the proposed approach are as follows:
073
074         1. Generality: ULoRA can be applied to any neural network structure that can be divided
075
               into blocks. It is applicable not only to Transformer architectures like LLaMA-3 (AI@Meta
               (2024)), but also to other architectures such as Mamba (Gu & Dao (2024)) and ResNet (He
076
               et al. (2015)), demonstrating its broad applicability.
077
078         2. Structural Flexibility and Efficiency: ULoRA allows combining multiple BLO to apply
079
               a single Low-Rank Adapter. This significantly reduces the number of parameters, thereby
               decreasing resource usage and inference time, providing structural flexibility and efficiency.
080
               This makes ULoRA well-suited for on-device models with limited resources.
081
082         3. Ease and Efficiency of Task Switching: FullStep, which combines all BLO into a single
083
               Low-Rank Adapter, eliminates the need to compute the model multiple times during in-
               ference. If the last hidden state of the model is known, only the new adapter needs to be
084
               computed, significantly reducing inference time, thereby facilitating efficient task switch-
085
               ing. This is particularly useful in on-device environments.
086
087   We performed various experiments to verify the generalizability, structural flexibility, and task-
088   switching ease of ULoRA. First, to evaluate its generalizability, we compare the performance of
089   various adapters on two models, LLaMA-3-8B and Mamba-2.8B. LLaMA-3-8B achieves compa-
090   rable performance to LoRA while using approximately 60% of the parameters. On Mamba-2.8B,
091   ULoRA achieves higher performance than LoRA with only about 20% of the parameters. Second,
092   to assess the structural flexibility and efficiency, we apply ULoRA to large blocks formed by group-
093   ing multiple smaller blocks. In LLaMA-3-8B, ULoRA shows a performance drop of about 10%
094   compared to LoRA but uses only 4% of the parameters, with up to an 8% improvement in through-
095
      put. Finally, to confirm the ease of task switching, we compare the inference time of ULoRA and
      LoRA by applying ULoRA with a single adapter across all blocks.

098   2   R ELATED W ORK
099
100   LoRA (Hu et al. (2021)) is one of the Parameter-Efficient Fine-Tuning (PEFT) methods that ad-
101   dresses computational challenges during the fine-tuning of pre-trained models. LoRA performs low-
102   rank decomposition on the weight matrix W ∈ Rd×d by representing it as W ∈ Rd×r × Rr×d ,
103   allowing fine-tuning to be conducted using only a small subset of parameters. The model parameters
104
      are frozen, and Low-Rank Adapters are attached to some of the parameters, with only the adapters
      being trained. During inference, the adapter and parameter weights are simply merged, effectively
105
      eliminating the adapter structure, and thereby preventing the introduction of any additional overhead.
106
107   Since the introduction of LoRA, various modifications and improved architectures have been pro-
      posed. VeRA (Kopiczko et al. (2024)) is a vector-based random matrix adaptation method that sig-


                                                         2
      Under review as a conference paper at ICLR 2025


108
      nificantly reduces the number of trainable parameters while maintaining performance comparable
109   to LoRA. This method uses a pair of low-rank matrices common across all layers, while instead
110   training a small scaling vector. AdaLoRA (Zhang et al. (2023)) assigns greater parameter budgets
111   to adapters deemed more important in the original LoRA’s Low-Rank Adapter, thus making LoRA
112   more adaptive by reducing the parameters allocated to less important weights. DoRA (Liu et al.
113   (2024a)) decomposes the pre-trained weights into two components: magnitude and direction, and
114   applies LoRA to update only the directional component, minimizing the number of trainable param-
115   eters during fine-tuning.
116   However, all of these LoRA variations are primarily focused on application to Transformer archi-
117   tectures. In contrast, Mamba (Gu & Dao (2024)) either cannot utilize LoRA or demonstrates sub-
118   par performance when applied. On the other hand, the Universal LoRA (ULoRA) proposed in this
119   paper applies Low-Rank Adapters to the largest common block structure found in deep learning
120   models, enabling it to be used with models that do not utilize Transformer architectures. Addition-
121   ally, ULoRA allows multiple blocks to be grouped and a single Low-Rank Adapter to be applied
122   across them; if all blocks are grouped into a single Low-Rank Adapter, only the adapter needs to be
123
      computed during inference, thereby facilitating efficient task switching.

      3.1   UL O RA A RCHITECTURE
129
      ULoRA is an improved architecture over LoRA (Hu et al. (2021)), which applies Low-Rank
130
      Adapters to the largest common block in a model. Since it is applied to block structures, ULoRA
131   offers general applicability even beyond Transformer architectures. It also provides structural flex-
132   ibility by allowing multiple blocks to be grouped together and a single Low-Rank Adapter BA to
133   be applied. If all blocks are grouped to use only one Low-Rank Adapter, task switching during
134   inference becomes more efficient since only the adapter computations are needed.
135
      To provide a more detailed explanation, we employ LLaMA-3 (AI@Meta (2024)), which can ac-
136
      commodate both ULoRA and LoRA, as a representative architecture. Fig. 1 shows the application
137   of LoRA and ULoRA to the Decoder Block (BLD ) of LLaMA-3-8B. Since both the BLD and the
138   Attention Block (BLA ) can be considered block structures, which may lead to confusion, we refer to
139   the largest outermost structural unit, such as the BLD , as the BLO . Blocks within the BLO , such as
140   the BLA , are referred to as Inner Blocks. The key difference between ULoRA and LoRA lies in the
141   structure to which the Low-Rank Adapter is applied. As shown in Fig. 1a, Low-Rank Adapters in
142   methods like LoRA, DoRA, and AdaLoRA are typically applied to components such as the Q and
143   V weights in BLA in the form of BQ AQ and BV AV . In contrast, ULoRA applies the Low-Rank
144   Adapter to the BLO , specifically the BLD , as shown in Fig. 1b.
145   Because ULoRA is applied at the BLO level, it can add Low-Rank Adapters without needing access
146   to the Inner Blocks, making it generally applicable to any deep learning model that can be divided
147   into blocks. Furthermore, as detailed in Section 3.2, ULoRA can also be applied by considering
148   multiple blocks as a single large block.
149   Since ULoRA and LoRA apply Low-Rank Adapters to different structural levels, their computations
150   within the BLD also differ. The Low-Rank Adapter in LoRA is computed within the BLA , whereas
151   the Low-Rank Adapter in ULoRA is computed outside the BLD . Therefore, we present two equa-
152   tions to show the computations for both the BLA and the BLD . In the BLA as shown in Eq. 1 of
153   LoRA, each Low-Rank Adapter, BQ AQ and BV AV , is computed and added to the target layer. In
154   the BLD as shown in Eq. 2, no Low-Rank Adapter is present. Since the Low-Rank Adapter compu-
155   tations are performed within the BLA , LoRA may face challenges if Transformer architecture is not
156   employed.

      3.3    TASK S WITCHING WITH F ULL S TEP
261
      Fig. 4 illustrates an example of task switching using three FullStep ULoRAs. In the forward process
262
      of a model using FullStep ULoRA, the adapter is applied after the computation of the last hidden
263
      layer is completed. Therefore, if the model’s last hidden state is stored during inference, task switch-
264   ing can be achieved by computing only the adapter. For instance, if the value obtained from applying
265   ULoRA1 Adapter is desired after the model’s inference, it is sufficient to add the value of ULoRA1
266   to the model’s last hidden state, requiring only the additional computation of ULoRA1. Similarly,
267   ULoRA2 and ULoRA3 can be applied by performing only their respective computations and adding
268   them to the last hidden state. This method is challenging to apply to autoregressive models that
269   predict the next token but can be a powerful method in autoencoder models or vision tasks where
      different adapters must be used for each task.

287   4.1   E NVIRONMENT S ETTINGS
288
289   The overall experimental environment is described including used models and datasets in this sec-
290   tion. Experiments were conducted using the LLaMA-3-8B and Mamba-2.8B models. LLaMA-3
291   (AI@Meta (2024), Dubey et al. (2024)) is an autoregressive language model by Meta AI, based
292
      on the Transformer architecture, and supports a variety of model sizes such as 8B, 70B, and 400B.
      Mamba (Gu & Dao (2024)) is a novel state space model (SSM) architecture that demonstrates ex-
293
      cellent performance on data with high information density, such as language modeling. It is based
294
      on a structured state space model (SSMs) and features an efficient hardware-aware design and im-
295   plementation.
296
297   The dataset used includes two types selected from those employed in the experiments of LoRA (Hu
298
      et al. (2021)). The E2E Dataset (Novikova et al. (2017)) is a new dataset for training end-to-end,
      data-driven natural language generation systems in the restaurant domain, which is ten times larger
299
      than existing frequently used datasets in this area. Evaluation for E2E was conducted using five
300
      metrics: BLEU, NIST, METEOR, ROUGE-L, and CIDEr. Detailed descriptions of the metrics used
301   for evaluating the models trained on these datasets are provided in Appendix 5.
302
303   The models were trained using TRL’s (von Werra et al. (2020)) SFTTrainer, and the hyperparameters
304
      and hardware used for training and inference are summarized in the Table 6 and Table 7 in Appendix
      5, respectively. The models were trained for five epochs, and the learning rate was set to 1e-5. The
305
      batch size was set to 8, and the warmup steps were set to 500. The optimizer used was AdamW, and
306
      the weight decay was set to 0.01. The label smoothing factor was set to 0.1, and the models were
307   compiled using Torch. The seed was set to 42 to ensure reproducibility.

      4.2   C OMPARISON WITH VARIOUS A DAPTERS
310
311
      This section evaluates the general applicability of ULoRA by measuring performance using various
312   adapters on two models: LLaMA-3-8B and Mamba-2.8B (which does not utilize a Transformer
313   architecture).
314
315
      The adapters compared with ULoRA includes LoRA (Hu et al. (2021)), DoRA (Liu et al. (2024a)),
      and AdaLoRA (Zhang et al. (2023)). LoRA is a Low-Rank Adaptation method that significantly
316
      reduces the number of trainable parameters for downstream tasks by freezing pre-trained model
317
      weights and injecting learnable rank decomposition matrices into each layer of the Transformer
318   architecture. DoRA is a Weight-Decomposed Low-Rank Adaptation that decomposes pre-trained
319   weights into two components: magnitude and direction, and fine-tunes them using LoRA for the
320   directional component, minimizing the number of learnable parameters efficiently. AdaLoRA is an
321   adapter that adaptively allocates parameter budgets across weight matrices. It parameterizes the
322   incremental updates in the form of singular value decomposition, effectively pruning the singular
323   values of less important updates to reduce the parameter budget while avoiding intensive exact SVD
      computations.

325   applied only to Q and V in the LoRA case. ULoRA applied the adapter to the Decoder Layer using
326   1Step, with r = 4 and α = 32, the same as other adapters. Since Mamba is not composed of a
327   Transformer architecture and does not have an Attention mechanism, adapters were applied to all
328   linear layers within the Mamba Block, excluding the Embedding Layer.

522   5   C ONCLUSION
523
524   We introduced Universal LoRA (ULoRA) in this paper, a comprehensive extension of the Low-
525   Rank Adapter (LoRA) approach to enhance its applicability across diverse deep learning mod-
526   els. Traditional LoRA mainly targets linear layers in Transformer architectures, limiting its scope.
527   ULoRA, however, applies to the largest universally common structural unit—termed the Outer
528   Block—enabling its use in a variety of architectures, including Mamba and ResNet. ULoRA’s de-
529   sign allows multiple Outer Blocks to be managed by a single adapter, reducing trainable parameters,
530
      resource consumption, and inference time. This makes it particularly effective for resource-limited
      on-device models. The FullStep configuration, where a single adapter controls all Outer Blocks,
531
      enables efficient task switching at inference by simply updating the adapter. Experimental results
532
      demonstrated that ULoRA matches or exceeds the performance of methods like LoRA, AdaLoRA,
533   and DoRA with fewer parameters. For LLaMA-3-8B, ULoRA achieved similar performance using
534   60% of the parameters and provided up to 8% higher throughput. In Mamba-2.8B, ULoRA outper-
535   formed LoRA using just 20% of the parameters. Its adaptability was also evident across different
536   Step configurations, maintaining strong performance with fewer parameters. In conclusion, ULoRA
537   expands the applicability of parameter-efficient fine-tuning while reducing resource demands, mak-
538   ing it suitable for efficient inference and quick task adaptation. Future work will focus on optimiz-
539   ing ULoRA for autoregressive models and exploring new integrations with emerging deep learning
      paradigms.

      A PPENDIX : E XPERIMENTAL M ETRICS
811
812
           1. Bilingual Evaluation Understudy Score (BLEU) (Papineni et al. (2002)) is a method
813           for evaluating the quality of machine translation by comparing the similarity between the
814           machine-generated translation and a reference human translation, based on n-grams.
815
           2. National Institute of Standards and Technology(NIST) is a method for evaluating the
816
              quality of text which has been translated using machine translation.
817
818
           3. Metric for Evaluation of Translation with Explicit ORdering(METEOR)(Lavie &
              Agarwal (2007)) is a metric for the evaluation of machine translation output. The met-
819
              ric is based on the harmonic mean of unigram precision and recall, with recall weighted
820
              higher than precision. It also has several features that are not found in other metrics, such
821           as stemming and synonymy matching, along with the standard exact word matching.
822
           4. Recall-Oriented Understudy for Gisting Evaluation-Longest(ROUGE)(Lin (2004)) is
823
              a set of metrics and a software package used for evaluating automatic summarization and
824
              machine translation software in natural language processing. ROUGE-L is Longest Com-
825           mon Subsequence(LCS) based statistics. Longest common subsequence problem takes into
826           account sentence-level structure similarity naturally and identifies longest co-occurring in
827           sequence n-grams automatically.
828
           5. Consensus-based Image Description Evaluation(CIDEr)(Vedantam et al. (2015)) is for
829           evaluating image descriptions that uses human consensus.
