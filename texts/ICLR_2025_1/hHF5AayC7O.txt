009                                                  A BSTRACT
010
011            We introduce NNetscape Navigator (NNetnav), a method for training web agents
012            entirely through synthetic demonstrations. These demonstrations are collected
013            by first interacting with a browser to generate trajectory rollouts, which are then
014            retroactively labeled into instructions using a language model. Most work on
015            training browser agents has relied on expensive human supervision, and the limited
016
               previous work on such interaction-first synthetic data techniques has failed to
               provide effective search through the exponential space of exploration. In contrast,
017
               NNetnav exploits the hierarchical structure of language instructions to make this
018
               search more tractable: complex instructions are typically decomposable into sim-
019            pler subtasks, allowing NNetnav to automatically prune interaction episodes when
020            an intermediate trajectory cannot be annotated with a meaningful sub-task. We use
021            NNetnav demonstrations from a language model for supervised fine-tuning of a
022            smaller language model policy, and find improvements of 6 points on WebArena
023            and over 20 points on MiniWoB++, two popular environments for web-agents.
024            Notably, on WebArena, we observe that language model policies can be further
025            enhanced when fine-tuned with NNetnav demonstrations derived from the same
026            language model. Finally, we collect and release a dataset of over 6k NNetnav
027
               demonstrations on WebArena, spanning a diverse and complex set of instructions.

029   1    I NTRODUCTION
030
031   Building grounded language agents that map human language instructions to a sequence of executable
032   actions is a long-standing goal of artificial intelligence (Winograd, 1972), with the ultimate goal of
033
      automating mundane web tasks like flight booking. A promising new approach for building such
      agents is to use large language models to control policies in digital environments like browsers (Yao
034
      et al., 2022; Shinn et al., 2023; Murty et al., 2024; Wang et al., 2024, among others).
035
036   Unfortunately, such grounded instruction following without any training examples is challenging
037   because LMs do not know about the myriad and ever changing interaction possibilities of different
038   websites. For instance, for a new online shopping website, a zero-shot LM agent may not know how
039
      to make a return or change order details, without expensive test-time exploration. Even simple tasks
      like selecting a flight might involve typing in airport codes or selecting from a drop-down menu, and
040
      zero-shot agents cannot know this a-priori.
041
042   One way to provide LM web-agents with knowledge about new web interfaces is via expert demon-
043   strations, that can either be used for in-context learning (Yao et al., 2022) or supervised fine-tuning
044   (Lai et al., 2024). These demonstrations are either fully provided by human experts (Sodhi et al., 2023;
045
      Yao et al., 2022) or consist of human-generated trajectories paired with model-generated instructions
      (Lai et al., 2024). Of course, collecting human demonstrations that cover each possible use case for
046
      every web-site is an unattractively large, never-ending task. Thus, recent work uses entirely synthetic
047
      demonstrations by sampling a synthetic instruction, and then mapping it into a trajectory using a base
048   LLM agent (Patel et al., 2024; Murty et al., 2024).
049
050   Such instruction-first methods for data collection face several challenges. First, synthetic instructions
051
      in these demonstrations are sampled from an ungrounded LM prior that generates only plausible1

089   r/callofdutyfans for reddit is plausible, but not always feasible. Second, generated instructions are
090
      limited to those that reference visible features of the website; e.g., given the landing page of a

095   Instead of starting with a sampled instruction, we start by sampling an interaction first, and then
096
      retroactively labeling it into an instruction that is feasible by design. At a high-level, our approach
      NNetscape Navigator (NNetnav, Fig 1), uses a language model exploration policy to perform
097
      extended interactions with an environment, and another language model trajectory labeler to annotate
098
      trajectories with instructions2 . To effectively control the exponential space of meaningful interactions,
099   NNetnav uses the hierarchical structure of language instructions as a pruning heuristic: for exploration
100   to discover a meaningfully complex task, trajectory prefixes must correspond to meaningful sub-tasks.
101   Thus, during an exploration episode, if a language model cannot label trajectory prefixes (at set
102   time-steps) with a sub-task, further exploration is automatically terminated. Imposing such a structure
103   over search not only enhances efficiency, but also results in complex and hierarchical instructions (See
104   Table 7 for examples). NNetnav prompts the same base language model for exploration, relabeling
105   and inferring sub-tasks, and effectively addresses all limitations of instruction-first data collection.

      Using GPT-4o-mini (Achiam et al., 2023) as our base language model, we use demonstrations
109   collected via NNetnav to fine-tune a smaller Llama-3-8B-Instruct (Dubey et al., 2024) based
110   agent on two benchmarks for web navigation, MiniWoB++ (Shi et al., 2017; Liu et al., 2018) and
111   WebArena (Zhou et al., 2023). Compared to the base agent, performance of the fine-tuned agent
112   improves from 28% to 48% on MiniWoB++, and from 1% to 7% on WebArena. Crucially, these
113   improvements are higher than those from a model that’s fine-tuned with an instruction-first data
114   collection method. Finally, we find that NNetnav can be used for self-training—fine-tuning a small
115   LM agent with NNetnav demonstrations from the same LM leads to an improvement of 4% points
116   absolute (1% to 5%) on WebArena. Further analysis reveals the benefits of retroactive labeling
117
      beyond performance improvements: When using a model-based evaluator, similar to Pan et al. (2024),
      hindsight trajectories from NNetnav have a higher mean reward than trajectories from an LLM agent
118
      based on the same underlying language model. Finally, we collect and release NNetnav-6k, a dataset
119
      of over 6k demonstrations covering a wide and complex range of use cases on WebArena.

      2    BACKGROUND
122
123   Following instructions on web-browsers is a deterministic sequential decision making problem.
124   Given an instruction g, an instruction following agent interacts with the browser by issuing a
125   sequence of actions ⟨a1 , a2 , . . . , aT ⟩ where each ai ∈ A is drawn in response to an observation oi .
126   Executing an action causes a state transition based on some unknown but deterministic environment
127   dynamics, leading to a new observation oi+1 . The entire episode can be summarized as a trajectory
128   τ := ⟨o1 , a1 , o2 , a2 , . . . oT , aT , oT +1 ⟩. We formalize the instruction following agent as a mapping
129   π(at | τ<t ; g) where τ<t := ⟨o1 , a1 , . . . ot ⟩ is the trajectory so far. In this framework, the action
130   space A consists of a finite set of strings, while observations are represented as either flattened DOM
131
      trees or website accessibility trees.
132   LLMs as Instruction Following Agents. Recent work explores prompted large language models
133   (LLMs) to directly parameterize π. These methods typically work in settings with textual observations
134   and action spaces, and many output a reasoning string ri before predicting the action string ai .
135   Concretely, we formalize an LM agent (omitting prompts) as πLM (at | τ<t ; g) := pLM (at , | τ<t , rt ; g)
136   where rt ∼ pLM (r | τ<t ; g) is a reasoning step drawn as a sample from the LM.
137   Given expert demonstrations {g i , τ i } where τ i := ⟨oi1 , r1i , ai1 , oi2 , r2i , ai2 . . . oiT ⟩, prior work adapts
138   LM agents using demonstrations either as in-context examples (Yao et al., 2022; Shinn et al., 2023;
139   Sun et al., 2023; Kim et al., 2023, among others) or as training data for supervised fine-tuning (Furuta
140   et al., 2023; Lai et al., 2024; Lù et al., 2024; Patel et al., 2024). For supervised fine-tuning of πLM on
141   a dataset of demonstrations, we construct training instances {(g i , τ<t       i
                                                                                         ), (rti , ait )} where rti , ait serves
                                                                                        i i
142   as the target reasoning step and action for an intermediate context (g , τ<t ).
143
      Data collection with instruction-first methods. Collecting human demonstrations for training web-
144
      agents is time consuming and costly. Thus, recent work proposes methods for generating synthetic
145   data for web-agents using language model components (Lai et al., 2024; Furuta et al., 2023; Murty
146   et al., 2024). These methods start by sampling synthetic instructions from an instruction generator (a
147   prompted LM that takes the website landing page and a persona as input), and then use a zero-shot
148   LM policy to convert these instructions into trajectories. Resulting demonstrations are filtered using
149   either the ground truth reward function (Furuta et al., 2023), or using another language model based
150   reward function (Lai et al., 2024; Murty et al., 2024). Most of these methods use bigger and better
151   language models for collecting demonstrations, and then use this data to adapt smaller models.
152
153   3    O UR A PPROACH
154
155   NNetnav (Fig 1) is an interaction-first method for constructing demonstrations: An exploration policy
156
      interacts with a browser in a structured manner to sample long trajectories which are retroactively
      labeled into instructions (§3.2). We then post-process each trajectory to add post-hoc reasoning steps
157
      corresponding to the generated instructions, and then use this data for supervised fine-tuning (§3.3).
158

      3.1     LM C OMPONENTS
190
191
      We start by describing various components in NNetnav. All of these components are implemented
192   by zero-shot prompting a language model, with different prompts (see Appendix A for details).
193
194   Exploration Policy. To interact with the environment, we use an exploration policy πexplore , im-
195   plemented using a chain-of-thought prompted language model (Wei et al., 2022). Additionally, to
196
      simulate a diverse set of behaviors from users and improve the diversity of resulting trajectories,
      we seed each episode with a string description of a plausible user persona for the given website
197
      (Shanahan et al., 2023; Argyle et al., 2023).
198
199   Summarizing Trajectory changes. Actions issued by πexplore result in a new observation in the
200   environment. We summarize this change as a short string description via another module ∆LM ,
201   implemented using a language model. In particular, for any state ot , action at and the resulting
202   next state ot+1 , δt = ∆LM (ot , at , ot+1 ) produces a string-valued description of the changes in the
203
      observation as a result of the action. For a trajectory τ , we denote the sequence of state changes as δτ
204   Trajectory Labeling Function. Given state changes δτ , a labeling function Lf LM produces a plausi-
205   ble instruction ĝ = Lf LM (δτ ) that the agent could have followed to produce the given interaction.
206
207
      Reward Function. Given ĝ and δτ , the reward function module assigns a reward sLM (ĝ, δτ ) ∈ {0, 1},
      based on how well the state changes correspond to the given instruction ĝ.

219   3.3   G ENERATING P OST- HOC R EASONING STEPS
220
221   The exploration policy in this work is implemented using a language model that generates a reasoning
222   step, before choosing an action (§2). Since actions in our demonstration set are a result of exploration,
223   corresponding reasoning steps are not generally related to the retroactively generated instruction.
224   Thus, for each demonstration in our synthetic demonstration set, we post-hoc annotate every action
225
      with a new reasoning step that directly corresponds to the generated instruction. Concretely, given
      every (ĝ, oi , ai ) tuple in our synthetic demonstration set, we prompt a language model to output a
226
      suitable reasoning step for choosing action ai given the instruction ĝ and current observation oi . We
227
      note that such a post-hoc reasoning procedure is similar to Yang et al. (2024).

      4.1   DATASETS
232
233   We fine-tune language model policies with NNetnav demonstrations on two web navigation environ-
234   ments, MiniWoB++ (Shi et al., 2017; Liu et al., 2018) and WebArena (Zhou et al., 2023).
235
236         1. MiniWoB++: A dataset of diverse synthetic web-interfaces with a shared action space.
237            Tasks on MiniWoB++ range from clicking on buttons to complex tasks like making a
238            booking on a website. We use a subset of 8 complex tasks from MiniWoB++ as a toy
239            benchmark to evaluate our method. We use the bid-based action space from BrowserGym
240            (Drouin et al., 2024), consisting of 12 actions, and a DOM based observation space. Due to
241            its synthetic nature, MiniWoB++ comes with an automatic reward function. We report the
242
               mean reward over 20 random seeds for each task, similar to (Drouin et al., 2024).
243         2. WebArena: A dataset of realistic web navigation tasks over 5 websites covering domains
244            such as e-commerce, discussion forums, maps and software development. We use the default
245            action space from WebArena (typing, clicking, hovering, tab management) and the default
246
               accessibility tree based observation space. WebArena consists of 812 Web navigation tasks
               across these websites, and provides an evaluator that measures success rate (SR) in terms of
247
               functional correctness. We report the average SR across these tasks.
248
249
      4.2   M ODEL S ETTINGS
250
251   All inference evaluations are conducted using the same base language model, with data collection
252   typically performed using a larger language model unless stated otherwise. We evaluate under the
253   following settings:
254
255         1. Zero-Shot: A baseline zero-shot LM policy πLM , prompted using chain-of-thought prompt-
256            ing (Wei et al., 2022). Next, we consider various fine-tuned models.
257         2. SFT (Instruction-First): Supervised fine-tuning (SFT) of the base policy using data
258            collected via instruction-first sampling. Here, we use the same reward model for filtering
259            demonstrations as NNetnav, and also sample the same number of demonstrations for fair
260            comparison.
261         3. SFT (NNetnav): Supervised fine-tuning of πLM with demonstrations collected via NNetnav.
262
            4. SFT (NNetnav + Distil.): Ablation, where we only retain instructions found via NNetnav
263
               and re-generate trajectories by prompting the same large LM as an agent. We use this setting
264            to isolate performance improvements attributable to NNetnav trajectories.

479   Language Conditioned Digital Assistants. Mapping instructions to action sequences in digital
480   environments has been a long-standing goal in natural language understanding (Allen et al., 2007;
481   Branavan et al., 2009). Most pre-LLM approaches for this rely on expert demonstrations for behavioral
482   cloning (Chen & Mooney, 2011; Humphreys et al., 2022), along with an appropriately shaped reward

485   dataset, as it produced demonstrations of comparable quality to GPT-4o-mini while offering a more permissive
      license for downstream applications.

      function (Branavan et al., 2009; Liu et al., 2018; Misra et al., 2017, among others). Here, learning is
487   driven purely by synthetic demonstrations derived via (language model) exploration of websites.
488
489   Linguistic Priors for Exploration. Several prior works have used natural language priors to inform
490   exploration for sequential decision making. Harrison et al. (2017) use a trained model of associations
491
      between language and state/action pairs to guide exploration during policy learning. Mu et al.
      (2022) use language annotations of states to train a goal generator module that provides intrinsic
492
      rewards for achieving generated goals. Similarly, Du et al. (2023) constrain exploration towards goals
493
      generated by a pre-trained LLM at each intermediate state of an agent. In constrast, NNetnav biases
494   exploration through two news ways of using language priors. First, we use natural language as a way
495   to filter meaningful interactions. Second, we use it as a pruning heuristic to navigate the potentially
496   exponential search space of these interactions.
497
498
      Training Data for LLM Web-Agents. LLMs have shown strong performance over a wide range
      of language understanding tasks, and are increasingly being used to interpret language in grounded
499
      contexts such as browsers (Yao et al., 2022; Lai et al., 2024; Wang et al., 2024; Patel et al., 2024;
500
      Lù et al., 2024, among others). Many of these approaches rely on human demonstrations, either
501   for in-context learning (Yao et al., 2022; Sodhi et al., 2023; Kim et al., 2023) or for finetuning (Lù
502   et al., 2024). However, because human demonstrations are costly, recent work trains LLM agents
503   through synthetic demonstrations generated using instruction-first methods (Lai et al., 2024; Patel
504   et al., 2024). One exception is Murty et al. (2024), which introduces an interaction-first method
505   for generating synthetic demonstrations for in-context learning. Despite its novelty, their approach
506   does not scale well to real websites due to the lack of a mechanism for effective exploration in
507   environments with many possible interactions. In contrast, NNetnav also follows an interaction-first
508   approach but improves efficiency by leveraging linguistically motivated pruning to navigate the space
509
      of meaningful interactions.

511   8    C ONCLUSION
512
      We propose NNetnav, a method for training web-agents with synthetic demonstrations. NNetnav
513
      flips the standard paradigm of synthetic data generation by first interacting with a website to generate
514
      trajectories and then relabeling trajectories into instructions. Real websites have a prohibitively large
515   set of possible interactions; NNetnav searches over this space efficiently using a pruning function
516   inspired by the hierarchical structure of language instructions: any complex instruction consists of
517   language describable sub-tasks and so, if during an interaction a relabeling module cannot infer a
518   meaningful sub-task for the trajectory-so-far, further exploration is pruned. We apply NNetnav to
519   collect demonstrations on MiniWoB++ and WebArena, which are then used to fine-tune a zero-shot
520   base LM agent. This yields significant improvements over the zero-shot baseline and outperforms
521   standard synthetic data generation methods. In addition, we show that NNetnav enables self-training,
522   as demonstrations collected using a base language model can improve the performance of an agent
523
      built on the same model. We find that NNetnav significantly enhances exploration efficiency due to
      the pruning heuristic and generates complex, realistic instructions. Lastly, we release NNetnav-6k,
524
      the largest dataset of demonstrations on WebArena to date, with over 6000 demonstrations covering
525
      broad range of intents and phenomena in WebArena.
