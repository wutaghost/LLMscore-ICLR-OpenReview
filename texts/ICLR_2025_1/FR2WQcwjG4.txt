009                                                A BSTRACT
010
011            There have been several efforts to improve Novelty Detection (ND) performance.
012            However, ND methods often suffer significant performance drops under minor
013            distribution shifts caused by changes in the environment, known as style shifts. This
014            challenge arises from the ND setup, where the absence of out-of-distribution (OOD)
015            samples during training causes the detector to be biased toward the dominant style
016
               features in the in-distribution (ID) data. As a result, the model mistakenly learns to
               correlate style with core features, using this shortcut for detection. Robust ND is
017
               crucial for real-world applications like autonomous driving and medical imaging,
018
               where test samples may have different styles than the training data. Motivated
019            by this, we propose a robust ND method that crafts an auxiliary OOD set with
020            style features similar to the ID set but with different core features. Then, a task-
021            based knowledge distillation strategy is utilized to distinguish core features from
022            style features and help our model rely on core features for discriminating crafted
023            OOD and ID sets. We verified the effectiveness of our method through extensive
024            experimental evaluations on several datasets, including synthetic and real-world
025            benchmarks, against nine different ND methods.

      1    I NTRODUCTION
028
029
      Novelty detection (ND) has emerged as a critical component in developing reliable real-world machine
030   learning models. The primary task of ND is to distinguish Out-of-distribution (OOD) samples from
031   the in-distribution (ID) samples during inference, using only unlabeled ID samples for training
032   (5; 67; 74; 96). This task is essential across various computer vision applications, including industrial
033   defect detection, medical disease screening, and video surveillance (43; 72; 94; 74). However, these
034   methods often experience significant performance drops when confronted with test data exhibiting
035   minor distribution shifts in their style, such as changes in the test sets due to environmental variations
036   (See Figure 1) (13; 11; 85; 86; 20).
037   A robust detector should be invariant to changes in the style features, as variations in these features
038   do not change a sample’s label (ID or OOD). Instead, it should be expected to learn the core features
039   which determine the label (3; 58; 13). Robustness against style shifts is a crucial aspect of ND
040   methods since variations in style are common in real-world applications. For instance, an ND method
041   for autonomous driving tasks trained on images from Germany streets (21) should also perform
042   effectively on the streets of Los Angeles (71), despite variations in style features caused by different
043   lighting and atmospheric conditions. A similar challenge exists in medical imaging, where shifts can
044
      occur due to different imaging equipment, patient positioning, and variations in tissue properties (95).
045   The vulnerability of existing ND methods stems from their implicit assumption that the training data
046   should strongly mirror the test data, even in stylistic features. This leads to the misprediction of an
047   ID test sample with a different style feature as OOD. Furthermore, training data in the ND setup is
048   limited to ID samples. By relying solely on ID samples, the detector learns a correlation between the
049   dominant style features present in ID samples and the label. Consequently, the detector mistakenly
050
      uses these style features for discrimination instead of focusing on core features. As a result, the
      detector incorrectly predicts an ID test sample with a different style as OOD and an OOD sample
051
      with a similar style as ID (13).
052
053   Notably, current domain generalization and domain adaptation methods cannot be applied to develop
      robust ND methods against distribution shifts, as they require access to labeled training data or extra


                                                          1
      Under review as a conference paper at ICLR 2025


054
      data from different environments, which are not available in the ND setup (40; 68; 99; 103; 63; 109;
055   105). Furthermore, our study distinguishes itself from recent works such as RedPanda (20) and PCIR
056   (13), which leverage different environmental annotations as additional information to improve the
057   ND robustness. In many real-world scenarios, ID training samples are collected from unknown
058   environments, and hence such metadata is often missing (11; 77).
059
      Motivated by these challenges, we propose crafting an auxiliary OOD set by identifying the core
060
      features of the ID samples and distorting them. To identify the core features, we employ a feature
061   attribution method (Grad-CAM (80)) applied on the output of a pre-trained network when fed with the
062   ID samples. We apply light augmentations (e.g., color jitter (16; 35; 36)) to the input, and compute
063   saliency maps for both the original and augmented versions. By taking the element-wise product
064   of these saliency maps, we derive a final saliency map where higher values correspond to the core
065   features of the assumed ID sample. These light augmentations facilitate producing a final saliency
066   map agnostic to style shifts. Subsequently, hard transformations (52; 90; 90; 87; 65; 22) (e.g., elastic
067   transformation) are applied to regions of the assumed ID sample with higher saliency values, ensuring
068   robustness against style shifts. Given the crafted OOD set and ID set, we apply light augmentation to
069
      each set while maintaining the labels to provide various style shifts to each set.
070   To effectively leverage information from the created sets and develop a robust ND pipeline, we
071   introduce a task-based knowledge distillation strategy (30). Specifically, we use a pre-trained encoder
072   concatenated with a trainable binary classification layer as the teacher and a model trained from
073   scratch as the student. We train the teacher to classify the created ID and OOD sets while only
074   updating the binary layer. Then, using a novel objective function, we force the student to align its
075
      output with the teacher when the input is an ID sample and to diverge from the teacher when the input
      is an OOD sample. The discrepancy between the student and teacher outputs will be utilized as the
076
      OOD score at inference time. Our approach is inspired by knowledge distillation, which has proven
077
      effective for ND tasks compared to other strategies (19; 78; 23; 32; 93; 11). Notably, our method
078   achieves superior performance compared to both previous knowledge distillation-based and other ND
079   methods, underscoring the effectiveness of our pipeline.
080
081
      Contributions: In this study, we propose a novel data-centric approach along with a new pipeline
      to achieve a robust and meta-data free ND method. Our strategy, by providing augmented samples
082
      obtained through applying style shifts while retaining labels, achieves a more robust representation
083
      of distribution shifts. Moreover, through intervening ID samples by identifying and distorting their
084   core regions, we reach synthesized OOD samples. Such samples are then leveraged to make our
085   model more sensitive to the core features. From a causal viewpoint (Refer to Section 4), by sample
086   intervention, as mentioned above, the unwanted correlation between style features and labels is
087   weakened. We note that the general strategy of some previous work (27; 59; 90; 97) that apply
088   hard augmentations on the entire image to generate OOD samples, do not necessarily weaken the
089   mentioned unwanted spurious correlation. In addition, our augmentation strategy facilitates the
090   generation of OOD samples whose distribution is potentially closer to that of the real OODs. As
091   well as providing theoretical support to our claims, We evaluate our method on real-world datasets
092
      such as autonomous driving and large medical imaging datasets, as well as common datasets such
      as Waterbird. For comparison, we considered representative and recent ND methods. Our pipeline
093
      demonstrates superior results, improving robust and standard performance by up to 12.7% and 6.7%
094
      in terms of AUROC, respectively. We further verify our method through a comprehensive ablation
095   study on its different components.

      Previous Works on Robust ND. Recent studies have proposed ND methods for improving robustness
150   under style shifts, including efforts by GNL (11), RedPanda (20), PCIR (13), Stylist (86), and Env-
151   AD (85). These methods, inspired by invariance-inducing approaches such as IRM (2), assume
152   that ID samples are drawn from multiple environments with known styles. Their effectiveness is
153   contingent upon accurately labeled styles in the training data, which can be a significant limitation in
154   datasets where such labels are mostly unavailable or hard to define. Recently, GLAD (98) has shown
155   impressive results on industrial datasets, but still faces severe performance drop on real-world ones.
156   Moreover, GNL proposes to craft different styles by applying minor shifts to ID samples. However,
157   GNL and other models still suffer from performance drops in real-world datasets, as shown in Table
158
      1, which is extensively considered in this study. Importantly, all mentioned methods lack information
      about potential OOD samples during training, leading to their models struggling with effectively
159
      learning core features.
160
161   Transfer Learning for ND. Several studies (70), including MSAD (69) and UniAD (100), have
      proposed using ImageNet pre-trained networks. These networks could be useful for ND across


                                                                                                 3
      Under review as a conference paper at ICLR 2025


162
      different datasets, such as medical imaging. Among the methods explored, the teacher-student
163   paradigm shows promising results. This approach involves using a pre-trained model as the ’teacher’
164   and a newly trained network from scratch as the ‘student’. The main objective is to train the student
165   model while the teacher remains frozen, aiming to mimic the teacher’s output on ID samples. The
166   rationale is that the student model, trained exclusively on ID samples, will produce discrepant outputs
167   on OOD samples during the inference phase. Methods such as RD4AD (23), Transformaly (19), and
168   ReContrast (32) are based on this paradigm. More details can be found in Appendix B.
169   Auxilary OOD for ND task. It has been demonstrated that using auxiliary OOD samples during
170   the training step can be beneficial for ND tasks by incorporating an extra dataset (37; 91). Recent
171   works have shown that the effectiveness of this technique largely depends on the diversity and the
172   distance of the distribution of the auxiliary OOD set used during training. In response to this, methods
173   including MIXUP (39), CutPaste (52), and VOS (26) have been proposed. More recently, GOE (48),
174   Dream-OOD (27), and FITYMI (59) address this issue by using large generative models (e.g., Stable
175   Diffusion (73)) for OOD crafting. Interestingly, our crafted auxiliary method does not rely on any
176   extra dataset or generative model. More details about these methods can be found in Appendix B.

      Causal Viewpoint. From the perspective of causality, the data-generating process can be modeled as
182
      the Structural Causal Model (SCM) (66) shown in Fig. 2. In this SCM, C and E denote unobservable
183   causal and non-causal (i.e., domain, environment, or style) variables, from which the observable
184   causal and non-causal components XC and XE for an image are obtained. The final image X is the
185   output of ψ(XC , XE ), where ψ(., .) is a combining function. The label Y of the image is caused
186   by XC . In the case of spurious correlation, a hidden confounder U , would be present such that
187   E ← U → C. This creates the path XE ← E ← U → C → XC → Y , which introduces an
188   unwanted correlation between E and Y . While there are solutions for when the environment variable
189   E is observable, they are not feasible when domain annotation of samples is not provided. Our
190   approach is effective even in the absence of domain annotation of samples. More precisely, we
191
      remove or at least weaken the edge E → XE by intervening on some components of XE in order
      to break or loosen the path between E and Y , as shown in Fig. 2b. Another orthogonal way of
192
      weakening this unwanted correlation is intervening XC by altering some core features of the ID
193
      samples (and correspondingly changing their label to Y = “OOD”).
194
195   In other words, we want to learn representations that are invariant to changes in XE and also sensitive
196   to altering XC . By augmenting samples via natural distribution shifts without changing the label,
197
      we reduce the correlation of XE and Y . On the other hand, to make our model more sensitive to the
      causal variables, we synthesize A-OOD samples by altering the core regions of ID images (changing
198
      XC variables and creating samples with Y = “OOD”).

269   Motivation. We propose a task-based knowledge distillation method with a novel contrastive-based
      loss function (16; 35; 42), where the defined task is the classification of ID and crafted OOD samples.

      The teacher model aims to update its knowledge by completing the defined task while concurrently
271   encouraging the student model to mimic its behavior closely for ID samples and diverge for OOD
272   samples. To generate informative OOD samples, we propose a simple yet effective method that
273   relies on estimating core regions and distorting them with hard transformations. In the following
274   subsections, we will explain each component of our method, detailing its functionality and benefits.
275
      Generating Style-Related OOD Samples. Style-related OOD samples, also referred to as near
276
      OOD samples in this study, are those that share stylistic similarities with ID samples but do not
277   belong to the ID set due to differences in core features (90; 59). To generate these style-related OOD
278   samples, we propose a guided strategy that transforms ID samples into OOD by altering the core
279   regions of the ID samples, which contain the primary semantics, while leaving the other regions
280   unchanged.
281
      At first, we define two families of transformations denoted as \protect \mathcal  {T}^+ (light transformations) and \protect \mathcal  {T}^-
282
      (hard transformations). \protect \mathcal  {T}^+ are those that have been shown to preserve semantics in ongoing literature
283   on self-supervised learning (16; 35; 31; 12; 17), while \protect \mathcal  {T}^- has been shown to be harmful to preserving
284   semantics in previous studies (39; 1; 29; 90; 87; 65; 22; 45; 52; 84; 44; 61; 104; 14; 25; 101). For
285   crafting OOD samples, we leverage Grad-CAM (81), which provides a saliency map for an input
286   sample using a common pre-trained model (e.g., ResNet18 (34)). Formally, for an ID sample x, we
287   randomly choose a light transformation \tau ^+_1 \sim \mathcal {T}^+ . We then compute the saliency map for both x and
288   \tau ^+_1(x) and take their element-wise product to ensure the exploited saliency map is style-agnostic. We
289   denote the normalized exploited saliency map as SM_x , where higher values correspond to the core
290   features of the assumed ID sample.
291   For the distortion step, we randomly sample two transformation of harsh transformations
292         \tau ^-_1, \tau ^-_2 \sim \mathcal {T}^-.
            The rationale behind choosing two transformations is to ensure that the distortion shifts the ID
293   sample to OOD. Specifically, for an image x with area A_x and exploited saliency map SM_x , we
294   design a mask m that covers an area \alpha A_x . We set \alpha  randomly between [0.20, 0.50] for each sample
295   to increase the diversity of crafted OOD samples. The mask is then slid over the saliency map, and for
296   each region, the region’s weight is determined by summing the pixel values from SM_x . Subsequently,
297   we choose x_{\text {ID}}^{\text {masked}} as the core region to distort based on these computed scores. The OOD sample
298   is then created as follows: xOOD = τ1− (τ2− (xmasked                  ID    ))) + (1 − m) ⊙ xID . We denote our proposed
299   OOD crafting strategy as G(\cdot ), where x_{\text {OOD}} = G(x_{\text {ID}}). More details about our generation strategy,
300
      including hard transformations and masking approach, can be found in Appendix C. Moreover,
      samples of the crafted OOD data are presented in Figures 10 and 11. Notably, we conduct extensive
301
      ablation studies on various hyperparameters, including \alpha  and the backbones, in Appendix D and E.
302
      Moreover, the motivation behind using local distortions instead of global ones is more thoroughly

306

      OOD set by distorting core features of ID samples. (B) Architecture of the proposed pipeline featuring
      a pre-trained encoder (teacher) and a from-scratch encoder (student), both concatenated to a linear
347
      layer. (C) Training step aims to align the output of the student f_s(\cdot ) closely with the teacher’s output
348
      f_t(\cdot ) for x_{\text {ID}}^1 and x_{\text {ID}}^2 , and to differentiate them for x_{\text {OOD}}^1 and x_{\text {OOD}}^2 . (D) Green circles indicate pairs
349   where the student’s output is intended to be close to the teacher’s output, red circles indicate pairs that
350   are meant to diverge, and gray squares represent pairs that have been omitted from the loss function.
351
      Training Step. Previous T-S works aimed to define LTS , which was generally associated with
352
      increasing sim(fs (x), ft (x)), where x belongs to the ID set. In contrast, we propose an OOD-aware
353   contrastive-based loss, denoted as LOCL . Specifically, considering a batch of ID training samples,
354   BID = {xi }ni=1 , we define BA-OOD = {xi }2n i=n+1 and B = BID ∪ BA-OOD , where BA-OOD is created
355   using our proposed crafting strategy, i.e., BA-OOD = G(BID ).
356
      For a sample x, using τ1 , τ2 ∼ T + , we define x1 = τ1 (x) and x2 = τ2 (x), and define them as positive
357
      pairs, i.e., P (x1 ) = x2 and P (x2 ) = x1 . Then, for each ID sample in B we define
358

366   Meanwhile, the teacher is updated using the classification task with cross-entropy loss LCE (τ1 (B) ∪
367   τ2 (B)), which is defined on ID and augmented OOD samples. It trains its binary layer while keeping

      7      A BLATION S TUDY
450
451
      Pipeline Components. To verify the impact of the proposed elements, we conduct comprehensive
      ablation studies using various datasets. The results are reported in Table 3. In each scenario, we
452
      replace certain components with alternative ones while keeping the remaining elements fixed. Setup
453
      A refers to a scenario where we ignore using auxiliary OOD samples for training and drop the binary
454   classification layers. Instead, we augment ID samples with light transformations and use the common
455   teacher-student based loss function, LTS , for training. Notably, this scenario is similar to the GNL
456   method. Setup B highlights the effect of the defined classification task by modifying the training
457   process. Specifically, it excludes the classification task that updates the binary layer of the teacher
458   model. Both the teacher and student models are trained without binary layers. Instead, we train the
459   student model with LOCL using the created ID and OOD sets. In Setup C, we replaced our defined
460   LOCL with LTS . This tests the efficacy of our proposed loss function in our framework. Setup D
461   specifically targets our OOD crafting strategy. Rather than estimating core regions of an ID sample
462
      for manipulation, this setup randomly distorts regions of ID samples. This OOD crafting approach is
      similar to the CutPaste (52) method in terms of finding the region of modification. Results show that
463
      Setup E, which refers to our proposed (default) framework, achieves superior performance compared
464
      to other setups.
465
466   OOD crafting strategy. In this ablation study, we substituted our OOD crafting strategy with
467   alternative strategies, while keeping other components unchanged. The results, presented in Table
468
      4, demonstrate that our efficient crafting strategy—which does not require an additional dataset
      or generative model—outperforms other methods. This superiority is based on the fact that other
469
      strategies, including MIXUP (39), FITYMI (59), Dream-OOD (27), and GOE (48), fail to preserve
470
      the relationship between the style features of created OOD samples and ID samples. Moreover, these
471   methods tend to generate OOD samples biased towards the datasets their backbones are trained on
472   (e.g., Dream-OOD’s bias towards LAION (79)), resulting in the creation of distant and unrelated OOD
473   samples (see Figure 10). VOS (26), crafting OOD samples in the embedding space, is ineffective in
474   preserving image style features. CutPaste (52), despite being better than other alternatives, distorts
475   random regions and may alter background features instead of core regions. More details on these
476   methods are in Appendix Q with their implementation details in Appendix R. Experiments on global
477   distortions, rather than local ones, are available in Appendix F.

479   8      C ONCLUSION
480
481   In this paper, we presented a robust novelty detection method that handles style shifts without
482   requiring metadata. By crafting an auxiliary OOD set and using a task-based knowledge distillation
483   strategy, our approach focuses on core features, reducing the impact of style variations. Evaluations
484   on real-world and benchmark datasets demonstrated significant performance improvements, achieving
485   up to 12.7% higher AUROC compared to existing methods. Our method proves effective in diverse
      scenarios, offering a robust solution for ND tasks.

996    Previous Works on Robust ND
997    Teacher-student based methods for ND. Efforts to adapt the teacher-student paradigm for ND tasks
998    have involved using a pre-trained model as the teacher and a from-scratch network as the student. The
999    main objective is to train the student model to mimic the teacher’s features on ID samples, with the
1000   rationale that the student model, trained exclusively on OOD-free samples, will generate discrepant
1001   features on OOD samples in inference phase (90). US ensembles several models trained on IDs at
1002   different scales to capture a broader spectrum of ID behavior, enhancing the detection of OOD data.
1003   Multiresolution Knowledge Distillation (MKD) (78) proposes using multi-level feature alignment to
1004   fine-tune the sensitivity to discrepancies between ID and OOD samples. RD4AD (23) advances these
1005
       methods by using a teacher-student setup with the teacher as an encoder and the student as a decoder
       focused on feature reconstruction, enhancing detection capabilities. ReContrast (32) introduces a
1006
       global paradigm for reconstructing teacher features by the student, rather than a regional approach. It
1007
       also incorporates a stop-gradient operation to stabilize the optimization process.
1008
1009   Auxiliary OOD Sample Crafting. CSI (90) and CutePaste (52) propose using fixed hard aug-
1010   mentation to create auxiliary samples. Specifically, CSI relies on Rotation, while CPAD considers
1011
       CutPaste as a pseudo-OOD. The GOE (48) method employs a pretrained GAN on ImageNet-1K to
       craft anomalies by targeting low-density areas. FITYM (59) employed an underdeveloped diffusion
1012
       as a generator. Dream-OOD (27) uses both image and text domains to learn visual representations
1013
       of normal instances in an embedding space of a pretrained stable diffusion (73) model trained on 5
1014   billion data (e.g. LAION (79)). On the other hand, VOS (26) generates OOD embeddings instead of
1015   image data. Notably, we adapt Dream-OOD for generation by using ID sample labels as prompts, as
1016   this generative method requires text for generation.
1017
       Previous Robust ND methods. RED PANDA model propose a robust ND method by focusing on
1018
       the removal of nuisance attributes by leverageing a domain-supervised disentanglement strategy to
1019
       learn representations that are invariant to specified nuisance attributes the model shows promise in
1020   controlled settings, the effectiveness of RED PANDA is contingent upon the accurate labeling of
1021   nuisance attributes in the training data, which can be a significant limitation in datasets where such
1022   labels are mostly unavailable or hard to define. calling a method to work without such anotaions.
1023   PCIR explores robust Unsupervised ND by aiming to identify invariant causal features across various
1024   environments. Specifically, the method assumes that the training data is drawn from multiple known
1025   environments, while the test data may come from different, potentially unseen environments. The
       known environments of each training sample facilitate the development of a regularization term

       Under review as a conference paper at ICLR 2025


1026
       designed to enhance the model’s ability to generalize across diverse environments. Despite the
1027   improved robustness demonstrated by their proposed method on specific datasets, a significant
1028   limitation is its reliance on the strong presupposition that the environment of each training sample is
1029   known. This assumption may not hold in real-world ND scenarios, where datasets often comprise a
1030   vast number of samples with unlabelled or unknown environmental contexts.

1037
1038   Following our method explanation in Section 5, we wish to find the optimal region of the image to
1039   distort. After getting the final normalized saliency map SMx , we use the fact that saliency maps
1040   possess spatial coherence, as stated in (81), and look for regions with higher values. The mentioned
1041   fact ensures that the selected region’s values are continuous, as well as having the core areas covered,
1042   resulting in an area of the image that encloses most of the core parts, rather than just including
1043
       minor and edge areas in it. Noteworthy is that when multiplying the mask by the image, the hard
       transformation might still get applied to regions with a zero pixel value, i.e., the unmasked area. To
1044
       tackle this, we crop the region and apply the transformation on the cropped part. Then, we paste the
1045
       new patch on the original image.
1046

1048   respect to the ID sample, is set between 0.2 and 0.5. This subsection presents an ablation study on

1067   AUGMENTATION D ETAILS
1068
1069   We apply two types of augmentations to each input x ∈ Dtrain , two of which are positive augmentations
1070   and two are negative augmentations. The intuition behind this is that with positive augmentations,
1071   we seek to make the model understand that light augmentations, which simulate environmental
1072   change in actual data, are not decisive in the final decision of the label. Meanwhile, with negative
1073   augmentations, we seek to destroy the core of the image, resulting in a new image with different core
1074   properties, representing OOD data. We also apply light augmentations to the newly crafted OOD
1075   data, to make the model understand environmental changes to this data should not be decisive in the
1076
       final decision, the same as with ID data.
1077   The exact details on transformations \protect \mathcal  {T}^+ and \protect \mathcal  {T}^- are provided in the main text. For each data, we
1078   sample a hard transformation τ + ∈ T + . We then attempt to find the core of the image using the
1079   procedure explained in our method in Section 5. All transformations are applied using official Python
       libraries of Albumenations (9) and ImageCorruptions (57).

       An additional component in our work that adds to the time complexity, in comparison with previous
1545
       ND works, is the saliency map extraction from GradCAM. Using the resources explained in the
1546   previous subsection, we generate saliency maps for one hundred 224×224 images in ∼ 2.7 ± 0.04
1547   seconds over all datasets in our setup. Notably, we compute these maps for each sample before

       Moreover, we observe that our method usually converges after ∼ 150 epochs on average, which
1551
       should be taken into consideration when estimating total time. For the batch size and backbone
1552   specified in Appendix N, each epoch should take less than one minute. Further, evaluation time
1553   is proportional to dataset size, but for an average-sized dataset, e.g. One-class MVTecAD, should
1554   be less than a minute. Formally speaking, calculating the LOCL loss takes O(β 2 ) time, giving
1555   O(GradCAM ) + (total iters) · (O(β 2 ) + O(LCE )). On eval time, we have (|D′test |) · O(f ), where
1556   f is the output of the model.
