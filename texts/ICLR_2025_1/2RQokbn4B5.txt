                                                         Abstract
009
                 Model inversion and membership inference attacks aim to reconstruct and verify the data on
010
                 which a model was trained. However, these methods cannot guarantee to find all training
011
                 samples, as they do not know the training set size. In this paper, we introduce a new task:
012              dataset size recovery, which seeks to identify the number of samples a given model was
013              fine-tuned on. Our core finding is that both the norm and the spectrum of the fine-tuning
014              weight matrices are closely linked to the fine-tuning dataset size. Leveraging this insight, we
015              propose DSiRe, an algorithm that accepts fine-tuned model weights, extract their spectral
016              features, and then employs a nearest neighbor classifier on top, to predict the dataset size.
017              Although it is training-free, simple, and very easy to implement, DSiRe is broadly applicable
018              across various fine-tuning paradigms and modalities (e.g., DSiRe can predict the number of
019              fine-tuning images with a mean absolute error of 0.36 images). To this end, we develop and
020
                 release LoRA-WiSE, a new benchmark consisting of over 25k weight snapshots from more
                 than 2k diverse LoRA fine-tuned models.

024   1     Introduction
025
026   Data is the top factor for the success of machine learning models. Model inversion (Fredrikson et al., 2015;
027   Yang et al., 2019; Haim et al., 2022) and membership inference attacks (Carlini et al., 2022; Shafran et al.,
028   2021; Jagielski et al., 2024) aim to reconstruct and verify the training data of a model, using its weights (Haim
029   et al., 2022; Duan et al., 2023; Nguyen et al., 2023). While these methods may discover some of the training
030   data, they are not guaranteed to recover all training samples. One fundamental limitation that prevents them
031   from discovering the entire training data is that they do not have a halting condition, as they do not know
032   the size of the training set (Haim et al., 2022). E.g., in membership inference, the attacker sequentially tests
033
      samples for membership in the training set, but without knowing the dataset size, it’s difficult to establish the
      halting condition or how many samples should be classified as "in" the training set. Knowing the training set
034
      size provides a crucial halting condition and provides a principled way to set thresholds. Discovering the size
035
      of a training dataset given the model weights is important, even without explicit reconstruction of the images
036   themselves. Understanding the number of images used to train or fine-tune models is also of great interest to
037   researchers, who wish to understand the number of data needed to replicate a model. We therefore propose a
038   new task: Dataset Size Recovery, which aims to recover the number of training samples a given model was
039   fine-tuned on.
040
      To tackle this challenge, we begin by analyzing the relationship between dataset sizes and the corresponding
041
      weights of fine-tuned models. As demonstrated in Fig. 2a, the Frobenius norm of the fine-tuned weights
042   is highly correlative with the corresponding dataset size. Specifically, the norm decreases as the dataset
043   size increases. However, since representing the weights matrix using only the Frobenius norm has limited
044   expressivity (as it’s only a single scalar), we proceed to extract singular values from the weights, providing
045   more expressive features. In Fig. 2b we can see that this approach shares the same phenomenon that we
046   previously described.

068
069   We therefore introduce DSiRe (Dataset Size Recovery), an algorithm for recovering the dataset size across a
070   wide range of fine-tuned models. Given a fine-tuned model, DSiRe represents it by extracting the singular
071   values from each of its layers’ weights matrices. This representation yields a feature space for exploring
072   various classification techniques. In particular, the best version of DSiRe uses the very simple baseline
073
      of nearest neighbors classification to label each model layer independently. The model-level prediction
      is then determined by the majority vote of its layers (See Fig. 1 for an overview). With merely weights
074
      information, it is effective for models fine-tuned on different modalities (e.g., vision and language), different
075
      tasks (e.g., generative and discriminative), and with various architectures (e.g., transformers, CNNs, and
076   MLPs). Moreover, it is not limited to a single fine-tuning paradigm, working well whether the model is fully
077   fine-tuned or partially adapted (e.g. LoRA (Hu et al., 2021), DoRA (Liu et al., 2024)).
078
079
      To evaluate DSiRe and encourage future research, we introduce LoRA-WiSE, a new, large-scale, and diverse
      dataset. LoRA-WiSE comprises over 25k weights checkpoints drawn from more than 2k independent LoRA
080
      models, spanning different dataset sizes, backbones, ranks, and personalization sets. On LoRA-WiSE, DSiRe
081
      recovers the dataset sizes with a Mean Absolute Error (MAE) of 0.36, demonstrating that our method is highly
082   effective in realistic settings.
083
084   To summarize, our main contributions are:
085
            1. Introducing the task of dataset size recovery.
086
            2. Presenting a method for recovering dataset size for fine-tuned models.
087
088         3. Releasing LoRA-WiSE, the first dataset size recovery evaluation suite.

174   3   Motivation
175
176   Frobenius Norm Analysis. Our hypothesis is that the difference between pre-fine-tuning and post-fine-tuning
177   weights, denoted as ∆W , encodes valuable information about the size of the fine-tuning dataset. To investigate
178
      this, we first consider a simple statistic of each fine-tuning matrix; its Frobenius norm, sF , defined as:
179
                                                             X
                                                      sF =       |∆Wij |2                                         (1)
180
                                                              ij
181
182    The norm of a weight matrix is known to correlate with the expressivity of the network. For example,
183   weight decay, a common regularization technique, effectively constrains this norm. To analyze the correlation
184
      between the Frobenius norm and the fine-tuning dataset size, We conducted an experiment using Stable
      Diffusion (SD) 1.5. We fine-tuned the model on 50 micro-datasets of sizes [1, 10, 20, 30, 40, 50] images,
185
      while keeping all other hyper-parameters fixed. Fig. 2a shows the range of values for the Frobenius norm
186
      statistic sF across different dataset sizes. The results clearly demonstrate a negative correlation between sF
187

188
      and the dataset size. We motivate this correlation by over-fitting, i.e., models tend to overfit faster on smaller
189
      dataset sizes, leading to larger values of sF .
190
191   Analyzing the Singular Value Spectrum. To gain a deeper understanding, we extended our analysis to
192   the singular value spectrum of the fine-tuning matrix. Fig. 2b visualize the mth singular value (denoted as
193
      σm ) for different dataset sizes. We note there is a better separation between different dataset sizes for the
      largest singular values. This suggests that the spectrum is more discriminative than the scalar Frobenius norm.
194
      Overall, both sF and the spectrum indicate larger values for smaller dataset sizes.
195
196   Layer-specific Analysis. Finally, we analyzed how discriminative different layers are for predicting fine-tuning
197   dataset size. We plot the spectra of layers in two distinct blocks of the UNet architecture: the first down block
198   and the last up block. For each block, we calculated the mean and standard deviation of singular values across
199
      all layers and present the results in Fig. 3. We can see that the up layer is more discriminative than the down
      one, perhaps suggesting that the UNet decoder is more prone to over-fitting than the encoder. However, it’s
200
      important to note that our experiments showed no single layer is universally discriminative across all models.
201
      Therefore, we conclude that combining results from all layers yields the most robust prediction of dataset
202   size.

      4     Method
206
      4.1   Task Definition: Dataset Size Recovery
207
208   We introduce the task of Dataset Size Recovery for fine-tuned dataset, a new attack vector against fine-
209   tuned models. Formally, given the fine-tuning weights of all layers of a model denoted as ∆W =
210   [∆W1 , ∆W2 , ...∆WL ], our task is to recover the number of images n that the model was fine-tuned on. More
211   formally, we wish to find a function f , such that:
212                                                    n = f (∆W)                                             (2)
213   The effectiveness of this attack was measured by the MAE between f (∆W) and n across a set of models.
214
215   4.2   DSiRe
216
217   We propose DSiRe (Dataset Size Recovery), a supervised method for recovering dataset size from fine-
218   tuned weights. Our approach first constructs a training dataset by fine-tuning multiple models on concept
219   personalization sets across a range of dataset sizes. It then trains a predictor function f that operates on a set
220   of fine-tuning weights of each model and outputs the predicted dataset size n. At test time, it generalizes to
221
      unseen models trained with different concepts. The method can be seen in Fig. 1
222   Training set synthesis. We first synthesize a training set by fine-tuning our model on each of Ntrain datasets,
223   each containing a set of training images. The datasets span a range of sizes; in this paper, we tested the ranges
224   1 − 6, 1 − 50, and 1 − 1000. The result is a set of Ntrain models ∆Wm , each with a corresponding label of
225   the dataset size nm .
226   Constructing DSiRE. Given the set of Ntrain labeled models, we wish to train a predictor that maps the
227   fine-tuning weights Wm to dataset size nm . Motivated by the results of our analysis (see Sec. 3), we represent
228   a given model as the set of spectra of all of its L layers:
229                                                Ψ = [Σ1 , Σ2 , . . . , ΣL ]
230   where Σi denotes the singular values of the weight matrix of the fine-tuning layeri . During inference, given
231   a new model, we label each layeri using the label of the nearest i’th layer in the train set. Then, we get the
232   overall prediction by ensembling the results of the model layers using the majority vote. In practice, we tested
233   many different methods for labeling each layer in Ψ and ablate them in Fig. 4. Overall, the simple Nearest
234   Neighbor (NN) method performed the best.

      5.1   Experimental Setup
238
239   Dataset. Constructing a dataset of real-world foundation models is challenging due to computational and
240   storage limitations. Therefore, we utilize LoRA fine-tuning, which is emerging as the most popular fine-tuning
241   paradigm for foundation models, and propose the LoRA Weight Size Evaluation (LoRA-WiSE) benchmark, a
242   comprehensive dataset designed to evaluate dataset size recovery methods for generative models. LoRA-WiSE
243   comprises 2,350 Stable Diffusion models (versions 1.5 and 2) fine-tuned using LoRA across various dataset
244   sizes, ranging from 1 to 1000 images. The benchmark includes multiple data ranges, LoRA ranks, and
245
      backbones to ensure diverse evaluation scenarios. Unless otherwise stated, we use Stable Diffusion (SD) 1.5
      as the pre-trained backbone and a LoRA rank of 32.
246
247   Settings.      LoRA-WiSE comprises of 3 data regimes: low ({1, 2, 3, 4, 5, 6} samples), medium
248   ({1, 10, 20, 30, 40, 50} samples), and high ({1, 50, 100, 500, 1000} sample). For each regime, we use
249   50 micro-datasets of different concepts (e.g., toys dataset, dogs dataset) to fine-tune SD on. Specifically, for
250   each dataset size s, we sample s images from each dataset, then fine-tune SD 1.5 on these resulted samples.
251
      This yields 300 fine-tuned models for the low and medium settings, and 250 models for the high setting. We
      then split each setting into train and test splits, with 15 models (fine-tuned on 15 micro-datasets) of each size
252
      to train, and the rest left for test. This results in (90, 210) split for the low and medium settings and (75, 175)
253
      split for the high setting. For a detailed description of the LoRA-WiSE benchmark and how these models are
254   trained, see App. C.
255
256   In addition to LoRA-WiSE, we train a set of models to evaluate full fine-tuning. we focus on the difference
257
      between the original and fine-tuned weight matrices. Due to computational resources, we use the medium
      data regime, resulting in 90 models for training and 210 for testing.
258
259   To ensure robust evaluation and test generalizability across different data distributions, we repeat each
260   experiment 10 times. In each time, we use subset sampling from models with varying object classes. We
261   report the average and standard deviation across these experiments.
262   Baseline. We compare DSiRe to a baseline, denoted as Frobenius-NN, which predicts the dataset size using
263   a nearest neighbor classifier on top of the Frobenius norms of the layers’ LoRA weights. Similar to DSiRe, the
264   Frobenius-NN is fitted separately to each layer, and then a majority vote rule is applied to select the prediction
265   from all layer-wise predictions. The analysis in Sec 3 provides motivation for this baseline.
266
      Evaluation metrics. As described in Sec. 4.1, our main evaluation metric is Mean Absolute Error (MAE).
267   For completeness, we choose to report two complementary metrics as well: (i) Accuracy. (ii) Mean Absolute
268   Percentage Error (MAPE). Since DSiRe predicts dataset sizes, simple accuracy does not adequately measure
269   its effectiveness, e.g., predicting 4 when the true value is 5 is not as bad as predicting 1. We therefore provide
270   MAPE scores as well, which compute the percentile from the ground truth that is equal to the absolute error.

      Number of Micro-Datasets. While our attack is data driven and requires access to the pre-trained model,

      we find that only a few examples are needed for DSiRe to perform well. E.g., in our medium data size range,

      our model can reach 86.4% accuracy using only 5 micro-datasets for training. The full results, presented in

394   7   Discussion
395
396   Performance at Low Data Ranges. While our approach shows promising results, there is room for
397   improvement in lower data regimes, where DSiRe reaches less than 80% accuracy. Improving these results
398   will provide tighter upper bounds for membership inference and model inversion attacks.
399   Data Driven Solution. Our method is data driven as it requires training multiple models from each dataset
400   size. However, our analysis shows there is correlation between the Frobenius norm and dataset size (see Fig.
401   2a). This insight could be a stepping stone in developing a data-free solution.
402
      Pre-training dataset size recovery. Another interesting application of dataset size recovery is for pre-
403   training cases. Lower bounding the required number of training set samples for foundation models will have a
404   substantial impact on the research community. Answering this question would require scaling up our method
405   to much larger dataset sizes and weight matrix dimensions.
406

      8   Social impact
408
409
      Research on our new task can positively impact both the research and digital arts communities. Establishing
410   an upper bound for membership inference attacks can promote privacy aware deployment of fine-tuned models
411   across different architectures and modalities. Determining the training dataset needed to train models with
412   poor documentation can help inform researchers that need to collect expensive datasets for new fine-tuning
413   tasks e.g., (Winter et al., 2024) and (Dai et al., 2023).

      9   Conclusion
416
417
      We introduced the novel task of dataset size recovery and proposed DSiRe, a method for learning a predictor
418   for this task in fine-tuned models. Our extensive experiments demonstrate DSiRe’s broad applicability across
419   various modalities, network architectures, fine-tuning paradigms, and dataset sizes, including continuous
420   ranges. We believe our work not only introduces a new capability but also provides valuable insights for
421   research in model privacy and security, potentially serving as an upper bound for model inversion and
422   membership inference attacks.

780   We present the LoRA Weight Size Evaluation (LoRA-WiSE) benchmark, a comprehensive benchmark
781
      specifically designed to evaluate LoRA dataset size recovery methods, for generative models. More
      specifically, it features the weights of 2350 Stable Diffusion (Rombach et al., 2022) models, which were LoRA
782
      fine-tuned by a standard, popular protocol (Ruiz et al., 2023; dre). Our benchmark includes versions 1.5 and 2
783
      of Stable Diffusion, having 2050 and 300 trained models for each version respectively.
784
785   We fine-tune the models using three different ranges of dataset size: (i) Low data range: 1 − 6 images. (ii)
786   Medium data range: 1 − 50 images. (iii) High data range: 1 − 1000. For each range, we use a discrete set of
787
      fine-tuning dataset sizes. In the low and medium ranges, we also provide other versions of these benchmarks
      with different LoRA ranks and backbones. See Tab.9 for the precise benchmark details.
788
789   For our low data range set, we choose Concept101 (Kumari et al., 2023), a previously collected set of
790   micro-datasets (3 − 15 images) designed for personalization research. For our medium and high data ranges
791   we use different classes of ImageNet (Deng et al., 2009) as the data source. This selection of datasets aims to
792   ensure that the fine-tuned models are drawn from a diverse set of concepts, spanning various categories.
793   Each micro-dataset is used to fine-tune the models for each dataset size. The images are randomly selected
794   from the micro-dataset. Each Stable Diffusion model consists of 132 adapted layers (pairs of Ai , Bi ), including
795   various layer types, such as self-attention, cross-attention, and MLPs. We save Ai , Bi separately, i.e., each
796   model provides a total of 264 unique weight matrices. We then split each range of this new benchmark
797   (low, medium, and high ranges) into train and test sets based on the micro-datasets. for more details see
798   appendix C.1.

      Layer weigh matrices In line with our analysis see Sec.3, given n example weights for each Ai , Bi we wish
878
      to build a separate classifier for each one. Knowing the relation between the singular values and the dataset
879
      size, we decompose each matrix using the singular value decomposition (SVD), and use the ordered set of
880   singular values as features for our classifiers. Formally, we note the singular values of Aij as ΣAij and the
881   singular values of Bij as ΣBij . We include the singular values of Bij · Aij denoted as ΣBij ·Aij . Additionally,
882   our observations indicate that the product Bi · Ai also provides useful information for data size recovery.
883   Thus, for each LoRA matrix, we obtain a dataset with n samples, where each sample is a vector of singular
884   values Σij , paired with a corresponding label yj . Our method then trains three separate kNN-classifiers with
885   K = 1 for each layer over (i) Ai (ii) Bi and (iii) Bi Ai . At inference time, the predictions from all classifiers
886   are merged by majority voting.
