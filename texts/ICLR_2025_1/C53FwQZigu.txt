009                                               A BSTRACT
010
011
               Class incremental learning aims to enable models to learn from sequential, non-
012            stationary data streams across different tasks without catastrophic forgetting. In
013            class incremental semantic segmentation (CISS), the semantic content of the back-
014            ground class changes across incremental phases, which is known as semantic
015            drift. Our research identifies two severe issues within semantic drift: separate
016            optimization and noisy semantics, which significantly degrade CISS performance.
017            Based on this insight, we propose a simple yet effective method, Image Posterior
018            and Semantics Decoupling for Segmentation (IPSeg), designed to address these
019            challenges through two specific mechanisms. First, IPSeg leverages image poste-
020
               rior probabilities as guidance to resolve the separate optimization issue. Second,
               IPSeg utilizes semantics decoupling to effectively handle noisy semantics and tailor
021
               the learning strategies for different types of knowledge. Experiment results on
022
               the Pascal VOC 2012 and ADE20K datasets demonstrate superior performance
023            compared to previous state-of-the-art approaches, particularly in more realistic and
024            challenging long-term scenarios. Furthermore, IPSeg exhibits excellent properties
025            in terms of both learning plasticity and memory stability.

      1   I NTRODUCTION
030
031   Deep learning methods have achieved significant success in vision (Qu et al., 2021) and language (Ke
032   & Liu, 2022) tasks with fixed or stationary data distributions. However, real-world scenarios are
033
      characterized by dynamic and non-stationary data distributions, posing the challenge of catastrophic
      forgetting (McCloskey & Cohen, 1989; McClelland et al., 1995). Incremental learning, a.k.a. contin-
034
      ual learning or lifelong learning (Silver et al., 2013), has been proposed to enable models to adapt to
035
      new data distributions without forgetting previous knowledge (Kudithipudi et al., 2022). Within this
036   domain, Class Incremental Learning (CIL) methods (Serra et al., 2018; Li & Hoiem, 2017; Rebuffi
037   et al., 2017; Mai et al., 2022; Wang et al., 2024a) have shown great potential in learning new classes
038   from incoming data, particularly for classification tasks (De Lange et al., 2021).
039
      Class Incremental Semantic Segmentation (CISS) extends the principles of CIL to pixel-wise tasks. In
040
      addition to catastrophic forgetting in CIL, CISS encounters an even more critical challenge: semantic
041
      drift (Yuan & Zhao, 2023) or background shift (Cermelli et al., 2020), which describes the incremental
042   change in the semantic meaning of pixel labels. Several studies (Douillard et al., 2021; Cha et al.,
043   2021; Zhang et al., 2022b; 2023) attribute semantic drift to the dynamic semantic content of the
044   background across incremental stages. Subsequent works (Cermelli et al., 2020; Douillard et al.,
045   2021) early pioneer this investigation using knowledge distillation and pseudo-labeling. More recent
046   works (Cha et al., 2021; Zhang et al., 2022b; 2023) further use saliency maps and segment proposals
047   to differentiate between the foreground and background regions. However, these works predominantly
048   target the separation of noisy semantics, still leaving room for further optimization.
049   Furthermore, we delve into semantic drift challenge and identify an additional critical issue, separate
050   optimization, as being of significant importance. Separate optimization refers to the CIL methods that
051   update the task heads for each target class independently and sequentially. This leads to a scenario
052   where task heads trained earlier may produce higher scores than those trained later for similar-looking
053   categories. Figure 1(a) directly presents the impact of separate optimization, where the SSUL-M
      model mistakenly classifies a horse as a “cow” class with a higher logit score after learning “horse”

093   2      R ELATED W ORK
094
095   Class Incremental Learning (CIL) Class-incremental learning is a method that continuously
096   acquires knowledge in the order of classes, aiming to address catastrophic forgetting (McCloskey &
097   Cohen, 1989) while continually learning new classes. Existing work (Wang et al., 2024b) broadly
098   categorizes these approaches into three main types: Replay-based methods involve storing data or
099
      features of old classes or generating data that includes old classes to mitigate catastrophic forgetting.
      This class can be further divided into Experience Replay (Rebuffi et al., 2017; Bang et al., 2021),
100
      Generative Replay (Liu et al., 2020; Shin et al., 2017), and Feature Replay (Belouadah & Popescu,
101
      2019). Regularization-based methods focus on designing loss functions that incorporate second-
102   order penalties based on the contribution of parameters to different tasks (Kirkpatrick et al., 2017;
103   Jung et al., 2020). They also rely on knowledge distillation, typically using the model from a previous
104   phase as a teacher to constrain the current phase model (Li & Hoiem, 2017; Rebuffi et al., 2017;
105   Douillard et al., 2020; Buzzega et al., 2020). Architecture-based methods dynamically adjust model
106   parameters based on new data, including assigning specific parameters for different data (Gurbuz &
107   Dovrolis, 2022; Serra et al., 2018) and breaking down model parameters into task-specific or shared
      parts (Douillard et al., 2022).

108
      Class Incremental Semantic Segmentation (CISS) CISS is similar to class-incremental learning
109   (CIL) but extends the task to pixel-level predictions (Phan et al., 2022; Camuffo & Milani, 2023;
110   Zhang et al., 2022a; Xiao et al., 2023). MiB (Cermelli et al., 2020) first introduces the concept of
111   semantic shift unique to CISS, employing distillation strategies to mitigate this issue. PLOP (Douillard
112   et al., 2021) utilizes pseudo-labeling techniques for incremental segmentation to address background
113   shift, while SSUL (Cha et al., 2021) further incorporates salient information, introducing the concept
114   of “unknown classes” into each learning phase and using a memory pool to store old data to prevent
115   catastrophic forgetting. RECALL (Maracani et al., 2021) and DiffusePast (Chen et al., 2023) extend
116   traditional replay methods by incorporating synthetic samples of previous classes generated using
117
      Diffusion (Ho et al., 2020) or GAN (Goodfellow et al., 2020) models. MicroSeg (Zhang et al., 2022b)
      employs a proposal generator to simulate unseen classes. CoinSeg (Zhang et al., 2023) highlights
118
      differences within and between classes, designing a contrastive loss to adjust the feature distribution
119
      of classes. PFCSS (Lin et al., 2023) emphasizes the preemptive learning of future knowledge to
120   enhance the model’s discrimination ability between new and old classes.

      Semantic Drift Previous works (Kirkpatrick et al., 2017) mainly attribute the semantic drift to noisy
149
      semantics within the background class cb . They attempt to mitigate this challenge by decoupling
150   the background class cb into subclasses c′b and cu , where c′b denotes the “pure” background and cu
151   denotes the unknown class. The most advanced methods (Zhang et al., 2022b; Cha et al., 2021)
152   further decouple the unknown classes cu into past seen classes C1:t−1 and dummy unknown class c′u
153   using pseudo labeling. However, semantic drift remains unresolved as the decoupled classes are still
154   changing across incremental phases and models are always hard to learn these chaotic classes.
155
      Additionally, another challenge caused by separate optimization of incremental learning exacerbates
156   semantic drift but attracts few attention. Previous work (Kim et al., 2024) points out that freezing
157   parameters from the old stage can preserve the model’s prior knowledge, leading to error propagation
158   and confusion between similar classes. While in CISS, the task head ϕt is exclusively trained by
159   supervision from the current classes and will be frozen to resist catastrophic forgetting in the following
160   incremental phases. In the following task t1 , t1 > t, ϕt may produce high scores on objects from
161   other appearance-similar classes, without any penalty and optimization. Meanwhile, the new task
      head ϕt1 just predicts moderate scores which might be slightly lower than error predictions from ϕt .

      3.2     OVERVIEW
186
      As illustrated in Figure 2, we propose Image Posterior and Semantics Decoupling for Class-
187   incremental Semantic Segmentation (IPSeg) to mitigate semantic drift through two main strategies:
188   image posterior guidance and permanent-temporary semantics decoupling. In section 3.3, we describe
189   how the IPSeg model uses image posterior guidance to overcome separate optimization. To address
190   noisy semantics, IPSeg employs two branches to decouple the learning of noisy semantics. Detailed
191   explanations of this approach are provided in section 3.4.

193   3.3     I MAGE P OSTERIOR G UIDANCE
194
195   As previously discussed, the separate optimization leads to inconsistent output scales across different
196
      incremental task heads and error predictions. We propose leveraging the image-level posterior as the
      global guidance to correct the probability distributions of different task heads. The rationale for using
197
      the image posterior probabilities is based on the following fact:
198
199   Fact: For any image, if its image-level class domain is CI and its pixel-level class domain is CP , the
200   class domains CI and CP are the same, i.e., CI = CP .
201   Inspired by this fact, we propose to use an extra image posterior branch ψ to predict image classifica-
202   tion labels and train it in an incremental learning manner. As illustrated in Figure 2, ψ is composed of
203   Pooling, Fully connected (FC) layers, and Multi-Layer Perceptrons (MLPs) with the input dimension
204   of 4096 and the output dimension of |C1:T |, where the FC layers serve as shared intermediate feature
205   processors, and the MLPs serve as incremental classification heads for incremental classes.
206   In task t (t > 1), the model is only able to access data from both the memory buffer M and the
207
      current task training data Dt . The mixed data samples xm,ti  from these two sources are processed by
208   the network backbone hθ into the image feature hθ (xti ), and further processed by image posterior
209
      branch ψ into the image classification prediction Ŷit . The objective function for training ψ is:

      3.4   P ERMANENT-T EMPORARY S EMANTICS D ECOUPLING
229
230   To further address semantic drift caused by the coupled learning of complex and noisy dummy
231   labels cb and cu along with incomplete yet accurate label Ct , we propose a decoupling strategy that
232   segregates the learning process for different semantics. Here is our empirical observation:
233
      Observation: Given an image in incremental task t, the semantic contents of it include four parts:
234
      the region of past classes C1:t−1 , target classes Ct , unknown foreground c′u and pure background c′b .
235
236   Based on this observation, we first introduce dummy label cf = C1:t−1 ∪ c′u to represent the
237   foreground regions that encompass both past seen classes and unknown classes, which are not the
238
      primary targets in the current task. Subsequently, we decouple the regions of a training image into
      two sets: Ct ∪ cf and c′b ∪ c′u . The former set Ct ∪ cf are current target classes and other foreground
239
      objects, which are temporary concepts varying across different phases. While c′b ∪ c′u are dummy
240
      labels representing “pure” background and unknown objects, which are permanent concepts existing
241   throughout all incremental phases. For instance, “cat” and “horse” are target classes in the current
242   task but change into the region of past seen foreground classes in subsequent tasks, while the concepts
243   of background and unknown objects remain consistent.

284   3.5   I MPROVING M EMORY B UFFER
285
286   The memory buffer M plays a crucial role in our implementation and we implement the memory
287   buffer based on unbiased learning and storage efficiency. IPSeg employs a class-balanced sampling
288   strategy, ensuring the image posterior branch can adequately access samples from all classes. Specifi-
289   cally, given the memory size |M| and the number of already seen classes |C1:t |, the sampling strategy
290
      ensures there are at least |M| // |C1:t | samples for each class. IPSeg also optimizes the storage cost
      of M by only storing image-level labels and object salient masks for samples. Image-level labels
291
      are required for the image posterior branch for unbiased classification. While the salient masks split
292
      images into background and foreground objects, labeled with 0 and 1 respectively. This simplification
293   mechanism requires less storage cost compared to previous methods that store the whole pixel-wise
294   annotations on all classes. More details can be found in the appendix.

298   4.1   E XPERIMENTAL S ETUPS
299
300   Dataset Following previous works (Zhang et al., 2023; Cha et al., 2021), we evaluate our method
301   using the Pascal VOC 2012 (Everingham et al., 2010) and ADE20K (Zhou et al., 2017) datasets.
302   Pascal VOC 2012 includes 20 foreground classes and one background class, with 10,582 training
303   images and 1,449 validation images. ADE20K, a larger-scale dataset, comprises 150 classes of stuff
304   and objects, with 20,210 training images and 2,000 validation images.
305
306   Protocols We primarily use the overlap setting to evaluate our method. This setting is more
307   challenging and realistic than the disjoint setting (Qiu et al., 2023), as the images may contain
308   both seen and unseen classes across different incremental steps. We evaluate IPSeg under several
309   incremental scenarios, denoted as M -N , where M is the number of classes learned initially, and N
310
      is the number of classes learned in each incremental step. For example, VOC 15-1 (6 steps) means
      learning 15 classes initially and one new class in each subsequent step until all 20 classes are learned.
311
      We use the mean Intersection over Union (mIoU) as the evaluation metric.
312
313
314   Implementation details Following previous works (Lin et al., 2022; Baek et al., 2022; Xiao et al.,
315
      2023), IPSeg utilizes DeepLab V3 (Chen et al., 2017) as the segmentation model with ResNet-
      101 (He et al., 2016) and Swin Transformer-base (Swin-B) (Liu et al., 2021) pre-trained on ImageNet-
316
      1K (Deng et al., 2009) as the backbones. The training batch size is 16 for Pascal VOC 2012 and 8 for
317
      ADE20K. IPSeg uses the SGD optimizer with a momentum of 0.9 and a weight decay of 1e-4. The
318   learning rates for both datasets are set to 0.01, with learning rate policies of poly for Pascal VOC
319   2012 and warm poly for ADE20K. All experiments are conducted with 2 NVIDIA GeForce RTX
320   3090 GPUs. For a fair comparison, the memory size is set as the same as SSUL (Cha et al., 2021)
321   that |M| = 100 for Pascal VOC 2012 and |M| = 300 for ADE20K. Pseudo-label (Zhang et al.,
322   2023) and saliency information (Hou et al., 2017) are adopted as previous methods (Cha et al., 2021;
323   Zhang et al., 2022b). To avoid information leaking, the ground truth of the training data in the image
      posterior branch only consists of annotations and pseudo-labels from the corresponding steps.

360   2012, including standard incremental scenarios (15-5 and 15-1) and long-term incremental scenarios

362   on Pascal VOC 2012 with both ResNet-101 and Swin-B backbones. Notably, in the long-term
363   incremental scenarios 10-1 and 2-2, IPSeg achieves performance gains of 4.9% and 6.4% over the
364   second-best method, CoinSeg-M, with the same Swin-B backbone. Meanwhile, the data-free version
365   of IPSeg (denotes as “IPSeg w/o M” ) also demonstrates competitive performance.
366   The superior and robust performance of IPSeg is mainly attributed to the reliable role of guidance
367   provided by the image posterior branch. The image posterior design effectively helps IPSeg avoid
368   catastrophic forgetting and achieve excellent performance on new classes, which often suffer from
369   semantic drift due to separate optimization in new steps. Additionally, the semantics decoupling
370   design enables IPSeg to better learn foreground classes within each incremental step. This design
371   brings the improvement of 12.3% and 6.7% over CoinSeg-M on new classes in the 10-1 and 2-2
372   scenarios, respectively.

525   5   C ONCLUSIONS
526
527   In this paper, we propose IPSeg, a simple yet effective method designed to address the issue of
528   semantic drift in class incremental semantic segmentation. We begin by analyzing the details of
529   semantic drift, identifying two key issues: separate optimization and noisy semantics. To mitigate
530   these issues, IPSeg introduces two specific designs: image posterior guidance and semantics de-
531   coupling. Experimental results on the Pascal VOC 2012 and ADE20K datasets demonstrate the
532   superior performance of our method, particularly in long-term incremental scenarios. Moreover,
533   IPSeg exhibits excellent properties in terms of both learning plasticity and memory stability.
534
535   Limitations and social impact Though IPSeg exhibits superior performance and properties of
536   learning plasticity and memory stability, we acknowledge that the usage of memory buffers leaves a
537
      lot of room for discussion. On the one hand, the use of memory buffers brings additional storage
      costs and the risk of information leakage. On the other hand, the use of memory buffers is related to
538
      privacy issues, such as storing private information without approval. These issues need to be treated
539
      with caution in artificial intelligence applications.

989    The details of efficient data storage in memory buffer For raw data, IPSeg directly stores the
990    image paths in a JSON file, as done in previous works (Cha et al., 2021; Zhang et al., 2022b; 2023).
991    For image-level labels, IPSeg stores the class labels of the images as arrays in the same JSON file with
992    multi-hot encoding, where 1 indicates the presence of a class and 0 indicates absence. The memory
993    cost for this is negligible. For pixel-level labels, instead of storing full-class annotations (with a data
994    type of uint8 ) as prior approaches, IPSeg only stores the salient mask, where the background and
995    foreground are labeled as 0 and 1, respectively (with a data type of bool ). Theoretically, the storage
996
       space could be reduced to 1/8.
