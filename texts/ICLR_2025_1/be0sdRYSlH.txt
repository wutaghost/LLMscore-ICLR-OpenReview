011                                                A BSTRACT
012
013            One-Shot Federated Learning, where a central server learns a global model over
014            a network of federated devices in a single round of communication, has recently
015            emerged as a promising approach. For extremely Non-IID data, training models
016
               separately on each client results in poor performance, with low-quality generated
               data that are poorly matched with ground-truth labels. To overcome these issues,
017
               we propose a novel Federated Model Inversion and Token Relabel (FedMITR)
018
               framework, which trains the global model by better utilizing all patches of the
019            synthetic images. FedMITR employs model inversion during the data generation
020            process, selectively inverting semantic foregrounds while gradually halting the in-
021            version process of uninformative backgrounds. Due to the presence of semantically
022            meaningless tokens that do not positively contribute to ViT predictions, some of
023            the generated pseudo-labels can be utilized to train the global model using patches
024            with high information density, while patches with low information density can be
025            relabeled using ensemble models. Extensive experimental results demonstrate that
026            FedMITR can substantially outperform existing baselines under various settings.

029   1    I NTRODUCTION
030
031   Federated learning (FL) (McMahan et al., 2017) is a machine learning framework where multiple
032   clients collaborate to solve machine learning problems under the coordination of a central server or
033
      service provider. And the raw data of each client is stored locally and not exchanged or transferred
      (Konecnỳ et al., 2016). Recent years, FL has shown its potential to facilitate real-world applications
034
      in many fields, including recommender systems (Liang et al., 2021; Liu et al., 2021b), medical image
035
      analysis (Liu et al., 2021a; Chen et al., 2021), computer vision (Lu et al., 2023; Zhang et al., 2023),
036   and natural language processing (Zhu et al., 2020; Deng et al., 2022). However, FL poses significant
037   challenges in terms of communication cost and data heterogeneity across clients. Communication
038   cost is a major bottleneck in FL systems, as clients need to communicate frequently with the server
039   over multiple rounds during the training process. This paradigm brings forth significant challenges:
040   1) heavy communication burden (Li et al., 2020), 2) the risk of connection drop errors between clients
041   and the server (Kairouz et al., 2021; Dai et al., 2022), and 3) potential risk for man-in-the-middle
042   attacks (Wang et al., 2021) and various other privacy or security concerns (Mothukuri et al., 2021;
043   Yin et al., 2021).
044   One-shot FL (Guha et al., 2019) has emerged as a solution to these issues by restricting communication
045   rounds to a single iteration, thereby mitigating errors arising from multi-round communication
046   and concurrently diminishing the vulnerability to malicious interception. Furthermore, one-shot
047   FL framework is particularly within contemporary model market scenarios (Vartak et al., 2016)
048   where clients predominantly offer pre-trained models. The drawbacks of this method stem from
049   its challenges when dealing with strongly non-iid data (Beitollahi et al., 2024). Due to its single
050
      communication session, it is unable to indirectly gather information from other clients through
      multiple interactions. This leads to a significantly low and unstable accuracy in the single aggregation,
051
      as each client’s acquired knowledge is extremely limited.
052
053   Existing methods often address this challenge by employing federated distillation to acquire knowl-
      edge. The server model is aggregated by distilling knowledge from all client models, commonly using


                                                         1

054
      the ensemble, while the ensemble is also responsible for synthesizing data samples for Data-Free
055   Knowledge Distillation (DFKD) (Zhang et al., 2022a;b). We conducted an in-depth analysis and
056   rethink of existing methods, and found that in the setting of highly heterogeneous data in FL, the
057   knowledge obtained from training various local client models is extremely disparate. As a result, the
058   quality of data generated by simple generators is poor, and there are many cases where labels do not
059   match the data.
060
      To address these challenges, we propose a novel one-shot FL framework named FedMITR which
061   trains the global model by better utilizing all patches of the generated images. FedMITR is based on
062   the model inversion framework for data synthesis on the server side, utilizing the ViT model. We start
063   by synthesizing input images from random noise, without utilizing any additional information from
064   the training data, making it suitable for FL where data privacy is crucial. After a single communication
065   round, we obtain only the model without any data. In a data-free scenario, our approach involves
066   recovering training data from pre-trained client models in some manner and utilizing it for knowledge
067   transfer. Furthermore, due to the dispersed training of client models, the knowledge from each client
068   is limited, resulting in poor quality of synthesized data. Therefore we need to select sparse patches
069
      with different information densities for subsequent processing. For patches with high information
      density, they are likely to match pseudo-labels and can be directly used for training the global model.
070
      For patches with low information density, we also reuse them by relabeling through ensemble models
071
      for knowledge distillation. The experimental results indicate that FedMITR significantly improves
072   accuracy compared to existing one-shot FL methods across various heterogeneous data scenarios.
073   For example, FedMITR surpasses the best baselines with 3.20%, 8.92%, and 7.93% on CIFAR10,
074   OfficeHome, and Mini-Imagenet under Dir(0.1) heterogeneous setting, respectively.
075
      In summary, our main contributions are summarized as follows:
076
077          • We rethink the limitations of existing DFKD methods in FL and first explore the role of
078            vision transformers and model inversion in one-shot FL.
079
             • We propose a novel federated model inversion and token relabel framework named FedMITR.
080            In the model inversion stage, we invert well-trained local models to synthesize images with
081            sparse tokens starting from random noise. And in the token relabel stage , we also utilize
082            the role of other tokens encoding some information on all generated image patches.
083
             • Our proposed method FedMITR is only improved for the server side, requiring no additional
084            training on local clients, making it suitable for contemporary model market scenarios without
085            the need for extra data or model transmissions.
086
             • Extensive analytical and empirical studies on various datasets verify the effectiveness of our
087            proposed FedMITR, consistently outperforming other baselines.

090   2    R ELATED W ORK
091
092   One-Shot Federated Learning. Guha et al. (Guha et al., 2019) first propose the concept of One-
093
      Shot Federated Learning, which treats local models as an ensemble for final prediction and further
      introduced the use of knowledge distillation along with public data for this ensemble in a single
094
      round of communication. Zhou et al. (Zhou et al., 2020) refrain from using public data and instead
095
      propose transmitting refined local datasets to the server. Li et al. (Li et al., 2021) propose a method
096   utilizing a two-tier knowledge transfer structure FedKT for distillation on public datasets. Instead
097   of using public data, Zhang et al. (Zhang et al., 2022a) propose a data-free method for knowledge
098   distillation by synthesizing data directly from ensemble models on the server-side. Diao et al. (Diao
099   et al., 2023) and Heinbaugh et al. (Heinbaugh et al., 2023) modify the local training phase and by
100   introducing placeholders or conditional variational autoencoders require additional transmissions.
101   Yang et al. (Yang et al., 2024) suggest using auxiliary pre-trained diffusion models. Previous works
102   either required additional transmission of information from clients or trained simple generators to
103   synthesize low-quality data, which cannot cope with one-shot FL settings in extremely heterogeneous
104
      environments.
105   Model Inversion. Fredrikson et al. (Fredrikson et al., 2015) introduce model inversion attack to
106   reconstruct private inputs. Subsequent works broaden this approach to new attack scenarios (He et al.,
107   2019; Yang et al., 2019) . Morerecently, model inversion has been used in data-inaccessible scenarios
      for tasks like data-free knowledge transfer (Yu et al., 2023; Braun et al., 2024; Patel et al., 2023).


                                                         2
      Under review as a conference paper at ICLR 2025


108
      DeepInversion (Yin et al., 2020b) improves synthetic data with batch norm distribution regularization
109   for visual interpretability. However, previous studies don’t utilize model inversion in FL and it’s
110   merely used as a tool for synthesizing surrogate data. Our work is the first to apply it to one-shot FL
111   and enhance it to obtain generated data that can be better utilized.
112
      Vision Transformer. ViT (Dosovitskiy et al., 2020) is one of the earlier attempts that achieved
113
      state-of-the-art performance on ImageNet classification, using pure transformers as basic building
114
      blocks (Vaswani et al., 2017). DeiT (Touvron et al., 2021) manages to tackle the data-inefficiency
115   problem by simply adjusting the network architecture and adding an additional token along with the
116   class token for Knowledge Distillation to improve model performance. In this paper, we focus on
117   using existing ViT models to distinguish high and low information density tokens, aiming for better
118   knowledge transfer from ensemble models to the global model.
119
      The most related works to our is the DENSE (Zhang et al., 2022a) and DeepInversion (Yin et al.,
120
      2020b). In one-shot FL, we applied a new model inversion method for data synthesis, which
121   diverges from traditional DFKD approaches. Traditional methods, such as DENSE (Zhang et al.,
122   2022a), generate synthetic data for distillation by training generators. In contrast, we do not require
123   training generators but instead directly use local models to invert and synthesize data. Unlike (Yin
124   et al., 2020b), which uses entire images generated by inversion for knowledge transfer, our method
125   distinguishes tokens into high information density tokens and low information density tokens and
126   relabels the latter for better utilization of the synthetic data.

      3.2   R ETHINKING THE WAY OF S YNTHESIZING DATA IN FL
163
164   How do traditional synthetic data methods work? To tackle the problems in one-shot FL as
165   mentioned in Section 3.1, many previous works (Zhang et al., 2022a;b; Dai et al., 2024) utilize
166   server-side knowledge distillation to improve the global model without the need to share additional
167   information or rely on any auxiliary dataset. They primarily focus on designing loss functions and
168   training a generator on the server side to generate data, which is then used for knowledge transfer
169   with ensemble models. More detailed information is provided in the Appendix.
170   What are the challenges of traditional synthetic data methods? Although the performance of
171   the server-side model has been improved through traditional methods, as seen from Figure 1(b),
172   the synthesized data distribution shows no clear boundaries. The use of such low-quality data for
173   federated distillation limits the scope for performance enhancement. This is because, on one hand,
174   previous efforts mainly involved training a generator to synthesize data using relatively simple
175   generator model structure. On the other hand, in the face of the highly heterogeneous challenges in
176
      federated learning data, many generated data labels do not match the data. This results in a large
      number of errors being learned by the global model during subsequent distillation, thereby limiting
177
      performance improvement.

202   4     M ETHODOLOGY
203
204   To overcome the shortcomings of traditional synthetic data methods in one-shot FL mentioned in
205
      Section 3.2, we propose a novel federated framework named FedMITR and the illustration of the
      training process in the server is demonstrated in Figure 2. After clients upload their well-trained
206
      local models to the server, we first invert well-trained networks (local models) to synthesize class-
207
      conditional images starting from random noise without using any additional information on the
208   training dataset due to privacy concernsn in the model inversion stage. Next, we use patches with
209   high information density, accompanied by generated pseudo-labels, to assist in training the global
210   model, while relabeling patches with low information density for knowledge distillation. These two
211   stages iterate multiple times on the server side.

298   4.2   T OKEN R ELABEL
299
300   Due to the heterogeneity of data in federated learning, many clients’ data may not even contain
301   certain classes in extreme cases. Therefore, many synthesized data labels mismatch with the data
302   itself, requiring the relabeling of certain tokens. In such scenarios, the knowledge learned by each
303   client’s model is extremely limited. First, we begin by utilizing the overall image, following the
304   approach outlined in (Lin et al., 2020). However, relying solely on distillation loss is insufficient for
305
      achieving good results. Not all tokens output by ViTs can be directly and simply utilized (Jiang et al.,
      2021) and examples include that tokens containing semantically meaningless or distractive image
306
      backgrounds do not positively contribute to the ViT predictions (Liang et al., 2022). Therefore, we
307
      utilize tokens of varying information densities generated during the model inversion stage. We train
308   the global model with pseudo-labels for tokens with high information density and conduct distillation
309   for tokens with low information density using an ensemble model for relabeling. The overall loss
310   function for the entire second stage is as follows:

331   Datasets and partitions. Our experiments are conducted on the following four popular real-world
332   datasets: CIFAR10 (Krizhevsky et al., 2009), CIFAR100 (Krizhevsky et al., 2009), OfficeHome
333   (Venkateswara et al., 2017) and Mini-ImageNet (Vinyals et al., 2016). To simulate real-world
334   applications, we adopt two different kinds of partition: 1) pk ∼ Dir(α): for each class, we allocate
335   a pik proportion of the data of class i to client k. The parameter α controls the level of statistical
336   imbalance, with a smaller α inducing more skewed label distributions among local clients. 2)
337   #C = k: each client only has data from k classes and we assign k random classes for each client.
338   Baselines. We compare the performance of FedMITR against four existing FL methods: Fe-
339   dAvg(McMahan et al., 2017), FedFTG (Zhang et al., 2022b), DENSE (Zhang et al., 2022a) and
340   Co-Boosting (Dai et al., 2024). FedAvg(McMahan et al., 2017) and FedFTG (Zhang et al., 2022b) are
341   not proposed for the field of one-shot FL, so they are set to communicate only for a single round in
342   the experiments. Furthermore, since FedMITR is a DFKD method, we also use the data-free method
343   DeepInversion (Yin et al., 2020a) in model inversion as a comparison method, applying it to the
344
      one-shot FL setting with ViTs.

      optimizer with a learning rate ηS = 0.001. The number of total epochs is set to 50. The distillation

481   In this paper, we first present a comprehensive critique of existing methods using synthetic data in
482   federated learning, emphasizing their drawbacks and limitations. Then, we propose a novel Federated
483   Model Inversion and Token Relabel framework named FedMITR. This framework generates synthetic
484   images through ViTs and efficiently utilizes all tokens of the generated images to train the global
485   model. Extensive analytical and empirical studies on various datasets verify the effectiveness of our
      method, consistently outperforming other baseline methods under diverse heterogeneous settings.

651
652   The first step is to train the auxiliary generator during the data generation phase. When aggregating
653   pre-trained models {θi }N  i=1 into one server model θS , we aim to train a generator to generate synthetic
654   data DS with the data distribution DS based on the Ensemble output. In particular, giving a random
655   noise z generated from a standard Gaussian distribution and a random uniformly sampled one-hot
656   label ŷ, the generator G(·) with parameter θG is responsible for generating the data x̂ = G(z),
657   forming the synthetic dataset DS . Since we are unable to access the training data of clients, we cannot
658
      compute the similarity between the synthetic data and the training data directly. Typically, to make
      sure the synthetic data can be classified correctly with a high probability by the Ensemble ES (·), as
659
      in:

663
      often incorporate additional loss functions to train generators to ensure the quality of generated data.

673   A.2    M ORE DETAILS ABOUT THE E XPERIMENT
674
675   Dataset. We use four popular real-world datasets in our experiments. CIFAR10 (Krizhevsky et al.,
676
      2009) consists of 60,000 color images in 10 classes, 50,000 for train, and 10,000 for test, while
      CIFAR100 (Krizhevsky et al., 2009) dataset is similar to the CIFAR10 dataset but it has 100 classes
677
      containing 600 images each. OfficeHome (Venkateswara et al., 2017) is a image recognition dataset
678
      that includes 15,588 images of 65 classes from four different domains (art, clipart, product, and
679   real-world). Mini-ImageNet (Vinyals et al., 2016) is a small subset extracted from the ImageNet-1K
680   (Russakovsky et al., 2015) dataset, consisting of 100 categories with 600 images per category, totaling
681   60,000 images. We split 80% data as the training set and 20% of that as the testing set. Each image is
682   resized to a 224×224 color image. All available test data is used to evaluate the final server model.
683
      Baselines. To ensure a fair comparison, we disregarded methods that require downloading auxiliary
684
      models or additional datasets. Furthermore, due to the single round of communication, regularization-
685   based aggregation methods or similar approaches that rely on multiple iterations are ineffective.
686   Therefore, against four existing FL methods: FedAvg(McMahan et al., 2017), FedFTG (Zhang et al.,
687   2022b), DENSE (Zhang et al., 2022a) and Co-Boosting (Dai et al., 2024). FedAvg(McMahan et al.,
688   2017) learns a shared model by aggregating locally-computed updates and iteratively updating through
689   multiple rounds of communication between clients and the server. FedFTG explores the input space of
690   local models through a generator and uses it to transfer knowledge from the local models to the global
691   model. Additionally, FedFTG proposes a hard sample mining scheme to achieve effective knowledge
692   distillation throughout the training process. Furthermore, FedFTG also develops customized label
693
      sampling and class-level ensemble techniques to maximize knowledge utilization, which implicitly
      mitigates distribution differences among clients. The two methods mentioned above are based on
694
      traditional multi-round communication federated learning, so they are set to communicate only once
695
      in this paper. DENSE (Zhang et al., 2022a) train a generator that considers similarity, stability, and
696   transferability and performe federated distillation on the server side. Co-Boosting (Dai et al., 2024)
697   uses the current Ensemble to synthesize higher-quality samples in an adversarial manner. These
698   hard samples are then employed to promote the quality of the Ensemble by adjusting the ensembling
699   weights for each client model. Since FedMITR is a DFKD method, we also use the data-free method
700   DeepInversion (Yin et al., 2020a) in model inversion as a comparison method, applying it to the
701   one-shot FL setting. The DeepInversion (Yin et al., 2020a) optimizes the input data while keeping the
      teacher model fixed during data synthesis, and it regularizes the distribution of intermediate feature

716   More details about the ablation experiments. In our method, the components of the loss function in
717   Eq.(6) collectively form the overall tokens being processed. Therefore, instead of directly removing
718   some components while retaining others for ablation experiments, we employed alternative methods
719   for experimental validation in Table 3. Here, Inversion + KD refers to knowledge distillation based
720   only on model inversion, while Inversion + PL refers to knowledge distillation using only pseudo-
721   labels without employing token relabel. Below is our additional analysis of the hyperparameters. In
722   the face of the highly heterogeneous challenges in Federated Learning, many generated data labels
723
      do not match the data. This results in a large number of errors being learned by the global model

724
