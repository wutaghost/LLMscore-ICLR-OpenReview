011                                              A BSTRACT
012
013            Transformer-based large language models (LLMs) use the key-value (KV) cache
014            to significantly accelerate inference by storing the key and value embeddings of
015            past tokens. However, this cache consumes significant GPU memory. In this work,
016
               we introduce LSH-E, an algorithm that uses locality-sensitive hashing (LSH) to
               compress the KV cache. LSH-E quickly locates tokens in the cache that are co-
017
               sine dissimilar to the current query token. This is achieved by computing the Ham-
018
               ming distance between binarized Gaussian projections of the current token query
019            and cached token keys, with a projection length much smaller than the embedding
020            dimension. We maintain a lightweight binary structure in GPU memory to facil-
021            itate these calculations. Unlike existing compression strategies that compute at-
022            tention to determine token retention, LSH-E makes these decisions pre-attention,
023            thereby reducing computational costs. Additionally, LSH-E is dynamic – at every
024            decoding step, the key and value of the current token replace the embeddings of a
025            token expected to produce the lowest attention score. We demonstrate that LSH-
026            E can compress the KV cache by 30%-70% while maintaining high performance
027
               across reasoning, multiple-choice, long-context retrieval and summarization tasks.

      1   I NTRODUCTION
031
032   The advent of large language models (LLMs) has enabled sharp improvements over innumerable
033
      downstream natural language processing (NLP) tasks, such as summarization and dialogue gener-
      ation (Zhao et al., 2023; Wei et al., 2022). The hallmark feature of LLMs, the attention module
034
      (Bahdanau, 2014; Luong, 2015; Vaswani, 2017), enables contextual processing over sequences of
035
      tokens. To avoid repeated dot products over key and value embeddings of tokens, a key-value (KV)
036   cache is maintained in VRAM to maintain these calculations. This technique is particularly popular
037   with decoder LLMs.
038
039
      However, the size of the KV cache scales quadratically with sequence length n and linearly with
      the number of attention layers and heads. Assuming the size of the KV cache is n tokens, for
040
      each new decoded token, n attention scores need to be added which requires a total of O(dn2 )
041
      computation, where d is the projection dimension, and O(n2 ) storage. For example, maintaining the
042   KV cache for a sequence of 4K tokens in half-precision (FP16) can require approximately ∼16GB
043   of memory for most models within the Llama 3 family (Dubey et al., 2024). These memory costs are
044   exacerbated with batched inference and result in high decoding latency (Fu, 2024). Consequently,
045   there is significant interest in compressing the size of the KV cache to enable longer context windows
046   and low-resource, on-device deployment.
047
      An emerging strategy for reducing the size of the KV cache is token eviction. This approach drops
048   the key and value embeddings for past tokens in the cache, skipping future attention calculations
049   involving these tokens. Various token eviction/retention policies have been explored in recent liter-
050   ature, including the profiling of token type preferences (Ge et al., 2023), retention of heavy-hitter
051   tokens (Zhang et al., 2024b;a), and dropping tokens based on the high L2 norms of their key em-
052   beddings (Devoto et al., 2024). The latter approach (Devoto et al., 2024) is intriguing as eviction
053   decisions are performed pre-attention. However, this L2 dropout strategy in inclined towards long-
      context retrieval tasks. It developed based on an empirical observation that smaller norm of key


                                                        1
      Under review as a conference paper at ICLR 2025


054
      embedding correlates with higher attention score. For long-context retrieval tasks, high-attention
055   score tokens are the most important tokens since the question’s text will overlap with the piece of
056   context that needs to be retrieved. Thus, it is specialized to retain only those tokens with the highest
057   attention, which we find unsuitable for free response reasoning tasks. Existing literature suggests
058   that retaining tokens with a diverse spectrum of attention scores (skewing high) is necessary (Guo
059   et al., 2024; Zhang et al., 2024b; Long et al., 2023).
060
      Is there a non-attentive KV cache compression strategy that is performant over a wide variety of
061   tasks, including multiple-choice, summarization, long-context retrieval, and free response question-
062   answering? This work answers this question positively by introducing a novel strategy, LSH-E, that
063   dynamically determines token eviction pre-attention via locality-sensitive hashing (LSH) (Goemans
064   & Williamson, 1995; Charikar, 2002). LSH-E evicts a past token from the cache whose key em-
065   bedding is highly cosine dissimilar to the current query token embedding. The intuition behind this
066   strategy is that high cosine dissimilarity indicates a low dot-product attention score. To efficiently
067   scan for cosine (dis)similar tokens without performing attention, LSH-E leverages the SimHash
068   (Charikar, 2002; Goemans & Williamson, 1995) to instead compare Hamming distances between
069
      c-length binary hashes of cached key embeddings and the current query embedding. We depict a
      high-level visualization of this strategy in Figure 1.
070
071   LSH-E requires minimal overhead: for a total sequence length of ℓ tokens with embedding dimen-
072   sion d, LSH-E maintains a constant-size, low-cost binary array in GPU memory of size c × k bytes,
073   where c ≪ d is the hash dimension and k ≪ ℓ. Cached tokens with key embeddings that register
074   low Hamming similarity measurements to decoded query embeddings are gradually replaced.

117
      We aim to capture tokens whose query embeddings will form a large sum of dot products (i.e.,
118
      attention scores) with other key embeddings, but without explicitly calculating attention. We will
119
      leverage locality-sensitive hashing (LSH) to quickly determine cosine similarities since the angle
120   is equivalent to the dot product (for unit vectors). In this section, we review technical concepts
121   crucial to attention and locality-sensitive hashing. We assume some base level of similarity with
122   transformers, but we refer the reader to precise formalism (Phuong & Hutter, 2022).

145   in the lower-dimensional space with high-probability. For a rigorous treatment of LSH functions,
146   see (Andoni et al., 2018; Charikar, 2002).

      2.1   R ELATED W ORKS
174
175   KV Cache Compression Many popular compression strategies adopt an eviction approach, which
176   removes embeddings from the KV cache. H2 O (Zhang et al., 2024b) and Scissorhands (Liu et al.,
177   2024b) calculate token importance by their accumulated attention scores and keep the “heavy hit-
178   ters” in the cache. FastGen (Ge et al., 2023) performs a profiling pass before the generation stage
179   that assigns to each head, according to the head’s attention patterns, a pruning policy which only
180   retains categories of tokens (punctuation, special, etc.) favored by the head. These eviction strate-
181   gies depend on the computation of attention scores for their policy. An attention-free L2 dropout
182
      method (Devoto et al., 2024), which we compare ourselves to in this work, uses the observation
      that high-attention tokens tend to have low L2 key norms to approximately keep important tokens
183
      in cache. Other methods seek to merge KV caches across heads, such as grouped query attention
184
      (GQA) (Ainslie et al., 2023; Dubey et al., 2024). KVMerger (Wang et al., 2024) and MiniCache
185   (Liu et al., 2024a), which searches for similarity between tokens in consecutive attention layers and
186   subsequently merges KV cache entries across these layers. While these consolidation approaches
187   prevent memory complexity associated with KV caches from scaling with depth or multi-head at-
188   tention, the size of any singular cache still tends to scale with sequence length.
189
190   LSH Based Attention Similar to our work, Reformer (Kitaev et al., 2020) employs LSH to find
191   similar tokens, but as a way to replace the softmax attention as opposed to token eviction. It creates
192   hash buckets of tokens that form local attention groups and only attends to tokens in the same and
193   neighboring buckets. However, this makes Reformer vulnerable to missing important tokens due
194   to hash collision or boundary issues, and therefore, it must use multiple hash tables to mitigate this
195   issue. In a similar vein, KDEFormer (Zandieh et al., 2023), HyperAttention (Han et al., 2023), and
196   Zandieh et al. (2024a), use LSH to stably approximate and compressing the attention module thus
197
      accelerating the computation, but without token eviction. SubGen (Zandieh et al., 2024b) uses LSH
      to cluster key embeddings and samples representatives from each cluster to reduce the size of the
198
      KV Cache and consequently speed up attention, though it must initially view all queries and keys to
199
      perform this clustering which could result in VRAM blowup, which our method avoids.

      Complexity Our strategy assumes a fixed memory budget, and therefore, uses constant memory.
241   The computation overhead per time step is also constant, because Fs core is calculated for a constant
242   C number of key vectors in the cache. The extra memory overhead that LSH-E introduces to each
243   attention head is the hash table H, which only uses C ∗ b bits of space and is independent of the
244   sequence length. The hash table is stored on GPU memory and does not introduce any latency
245   bottlenecks associated with CPU-to-GPU streaming (Strati et al., 2024).

      4   E XPERIMENTS
248
249   Tasks We evaluated our LSH eviction strategy across various tasks to demonstrate its effectiveness
250   in reducing the memory cost of the KV cache while preserving the language quality of the generated
251   text. Our experiments are split into four main categories: free response question answering, multiple
252   choice, long-context retrieval and long-context summarization. Our long context retrieval tasks
253   include the multi-key needle-in-a-haystack task and the common words task from (Hsieh et al.,
254   2024). Question answering tasks include GSM8K (Cobbe et al., 2021) and MedQA (Jin et al.,
255   2021). Summarizaiton tasks include GovReport and MultiNews from Bai et al. (2023).
256
257   Metrics The question-answering tasks were evaluated using BERTScore (which includes preci-
258   sion, recall, and F1 scores), ROUGE (ROUGE-1, ROUGE-2 and ROUGE-L and ROUGE-Lsum),
259
      and GPT4-Judge. GPT-4 was prompted to look at both the model prediction and the ground truth an-
      swer, then provide a score from 1 - 5 on the coherence, faithfulness, and helpfulness of the answer in
260
      addition to similarity between the prediction and ground truth (we named this metric GPT4-Rouge).
261
      In this section, we report the average of these four scores. For details on individual scores, please
262   see Appendix B. For the system prompts given to GPT-4, refer to Appendix G.2. For multiple-
263   choice tasks, we use accuracy as our metric. The metric used to evaluate long context retrieval tasks
264   is the string matching score from Hsieh et al. (2024), whose definition is in Appendix G.1. For
265   summarization tasks, we use Rouge as the metric as per direction from Bai et al. (2023).
266
267   Configuration and Setup We conducted most experiments using Meta’s Llama3 8B-Instruct
268   model (Dubey et al., 2024) with the exception of long text summarization tasks which were tested
269   using the Llama3.1 8B-Instruct model. Our method is agnostic to grouped-query attention, so we
      used the default group size of 4. The maximum sequence length was set to the sum of the maximum

      5   D ISCUSSION & C ONCLUSION
522
523
524   In this paper, we introduce LSH-E, a novel attention-free eviction strategy for KV cache compres-
525   sion in transformer-based LLMs. By leveraging locality-sensitive hashing (LSH) to approximate
526   cosine similarity, LSH-E dynamically determines which tokens to evict from the cache without
527   performing costly attention calculations. Our experiments demonstrate that LSH-E can achieve
528   30-70% compression of the KV cache while maintaining strong performance across various tasks,
529   including free-response Q&A, multiple-choice Q&A, and long-context retrieval.
530   The key advantage of LSH-E lies in its ability to efficiently compress the KV cache pre-attention,
531   enabling significant memory savings and faster inference times. Compared to traditional strategies
532   like L2 norm-based eviction (Devoto et al., 2024), LSH-E excels particularly in reasoning and
533   multiple-choice tasks, where maintaining a diverse set of tokens in the cache is crucial for generating
534   accurate and coherent responses.
535   There are several potential areas for future work. Investigating hybrid approaches that combine
536   LSH-based eviction with attention-based mechanisms such as (Zhang et al., 2024b; Ge et al., 2023)
537   could offer a middle ground between computational efficiency and retention of high-importance
538   tokens. Further, reducing the overhead associated with maintaining binary hash codes (e.g., by
539   optimizing bit precision) could further enhance the applicability of LSH-E to memory-constrained
      environments.
