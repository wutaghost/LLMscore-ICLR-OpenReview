009                                               A BSTRACT
010
011            Recent advances in large language models (LLMs) have transformed software
012            development by automatically generating code based on users’ requests in natu-
013            ral language. Despite these advancements, challenges remain in generating fully
014            correct code and aligning with user intent. Our empirical study reveals LLMs
015            tend to dilute their self-attentions on the initial prompt as more code tokens are
016            generated. We hypothesize this self-attention dilution issue is one of the root
017            causes of inaccuracies in LLM-generated code. To mitigate this issue, we propose
018
               Selective Prompt Anchoring (S PA) to amplify the influence of the selected parts in
               the initial prompt, which we refer to as “anchored text”, during code generation.
019
               Specifically, S PA calculates the logit distribution difference with and without the
020
               anchored text. We prove this logit difference approximates the anchored text’s
021            contextual contribution to the output logits. S PA creates an augmented logit distri-
022            bution by linearly combining the original logit distribution and the logit difference.
023            We evaluate S PA with five LLMs on four benchmarks. Our results show that after
024            tuning on a few dozen tasks, S PA consistently improves Pass@1 on new tasks by up
025            to 7.6% across all settings. Notably, with selective text anchoring, a small version
026            of DeepSeek-Coder (6.7B) can achieve better performance than an original much
027            larger version (33B). Our code is available at https://anonymous.4open.
028            science/r/Selective-Prompt-Anchoring-74E7.

      1    I NTRODUCTION
032
033
      Large language models (LLMs) have emerged as powerful programming assistants. They have
      demonstrated unprecedented capabilities in interpreting natural language descriptions and generating
034
      source code. Despite this great progress, LLMs still produce incorrect solutions to some tasks or
035
      generate code that does not fully meet user expectations. The prevalence of such generation errors
036   undermines their reliability and limits their utility in real-world software development.
037
038   To improve the performance of LLMs on coding tasks, many efforts have been made to develop
039
      high-quality training data (Li et al., 2023c; Guo et al., 2024; Wei et al., 2023) and design new domain-
      specific training objectives (Niu et al., 2022; Chakraborty et al., 2022). However, these approaches
040
      require tremendous computational resources. Training-free approaches have been explored to address
041
      this challenge by enhancing the prompting method or incorporating external knowledge, such as
042   retrieval-augmented generation (Du et al., 2024), chain-of-thoughts (Le et al., 2024; Suzgun et al.,
043   2022), self-planning and debugging (Jiang et al., 2023; Chen et al., 2023), etc. While they have been
044   proven to be effective in improving performance, there exist limitations such as being sensitive to the
045   quality of prompt design and retrieved data (Zhao et al., 2021). Compared with existing methods,
046   this work aims to study and improve LLMs in an orthogonal direction through attention adjustment.
047
      One key component of existing LLMs is the self-attention mechanism in the transformer architec-
048   ture (Vaswani et al., 2017), which enables models to focus on crucial parts of the given prompt.
049   Despite the success of the self-attention mechanism, prior works found language models exhibit sim-
050   ple attention patterns (Raganato & Tiedemann, 2018; Voita et al., 2019). Furthermore, an empirical
051   study (Kou et al., 2024) found that given a coding task, there often exists a misalignment between
052   LLM attention and human attention. Compared with human programmers, LLMs often focus on
053   different parts of a natural language description when generating code. Inspired by this finding, we
      hypothesize that a root cause of inaccuracy in LLM-generated code stems from the suboptimal model

054
      attention. To verify our hypothesis, we conduct an empirical study that analyzes the shift in LLMs’
055   attention distribution during code generation. We observe that LLMs’ attention to the initial prompt
056   gradually dilutes as generating more code. We call this phenomenon “attention dilution” in code
057   generation tasks.
058
      In standard decoding algorithms, LLMs calculate a conditional probability for the next token based
059
      on the preceding context. However, the autoregressive nature of LLMs considers both the initial
060
      prompt and possibly wrong self-generated tokens together as the correct context and pays comparable
061   attention to them. We argue LLMs should pay more attention to the absolutely correct prompt and
062   less attention to the following self-generated content that could potentially be wrong.

091   To mitigate this limitation, we propose Selective Prompt Anchoring (S PA), a model-agnostic approach
092   that optimizes LLMs’ attention by amplifying the contextual influence of selective prompt, towards
093   each generated token. S PA is inspired by the anchoring effect (Furnham & Boo, 2011) in psychology,
094   which refers to people being influenced by specific information given before decision-making. In
095   S PA, we refer to this information as anchored text, a group of selected tokens within the prompt that
096   should receive higher attention from the model than others.
097   Figure 1 illustrates the pipeline of S PA. Given the anchored text, S PA creates an original embedding
098   matrix (⃝ 1 ) as well as a masked embedding matrix by replacing the embeddings corresponding
099   to anchored text with mask embeddings (⃝      2 ). We mathematically show that the anchored text’s
100   contextual influence can be approximately measured by the difference between the logit distribution
101   generated from the original prompt and the prompt with the anchored text masked (⃝        3 ). To amplify
102   the influence of anchored text in the model output, S PA multiplies this logit distribution difference by
103   a hyperparameter called anchoring strength (⃝    4 ), and then adds it to the original logit distribution
104
      (⃝5 ). We find while the optimal anchoring strength varies across different models and tasks, it can be
      easily tuned through dozens of tasks.
105
106   We evaluate S PA on four benchmarks with five LLMs. The result shows S PA can significantly
107   and consistently boost Pass@1 across all models and benchmarks, highlighting a new direction for
      controlling LLMs’ high-level attention and effectively improving performance.

      2       A N E MPIRICAL A NALYSIS OF ATTENTION D ILUTION
109
110
      We first conduct an empirical study to analyze the attention dilution phenomenon in large language
111   models (LLMs) during code generation. To improve the generalizability of our findings, we exper-
112   imented with two different methods to compute the attention scores over input tokens. First, we
113   used a self-attention-based method (Zhang et al., 2022; Galassi et al., 2021) to obtain self-attention
114   scores from the last layer in LLMs, which has been shown to represent the most accurate attention
115   distribution (Kou et al., 2024; Wan et al., 2022a). Second, we used a gradient-based method (Selvaraju
116   et al., 2016; Shrikumar et al., 2017) that treats the entire LLM as a function and measures to what
117   extent each input token contributes to the output. Based on these two methods, we calculate the
118   percentage of attention on the initial prompt. Calculation details are provided in Appendix A.1.

143   influenced by tokens generated in recent time steps, rather than the prompt from users. This can
144   be problematic in two ways. First, generation errors in the previous time steps are very likely to
145   propagate to the following steps as the model pays more attention to the preceding code tokens.
146   Second, for complex tasks that require the generation of a long code sequence (e.g., multiple if
147   statements), the model is likely to miss critical descriptions as it pays little attention to the user
148   prompt deep in the code generation process.

168   However, autoregressive decoding assumes all prior tokens are correct, giving them equal opportunity
169   to compete for the model’s attention—even when self-generated tokens may be wrong. As the number
170   of self-generated tokens increases, the model’s attention to the initial prompt (which describes the task

172   that the model attends to incorrect information, increasing the likelihood of generating errors.

261   3.4   T UNING A NCHORING S TRENGTH
262
263   The anchoring strength ω serves as a hyperparameter in S PA. Our experiments demonstrate an
264   unimodal relationship between ω and the performance. As the anchoring strength ω increases, the
265   performance first improves, reaching an optimum, and then declines with further increases of ω. It is
266   simple to tune this single hyperparameter through a few dozen instances. More details are discussed
267   in Section 5.3.

      3.5    S ELECTION OF A NCHORED T EXT
271
272   While S PA can easily anchor the entire prompt, the initial prompt can significantly vary due to task
273   differences. In some scenarios, the prompts can be lengthy and not all information is persistently
274   important throughout the generation. Our goal is to identify and anchor the most informative tokens,
275   which LLMs should persistently focus on, while excluding trivial details in the prompt. For code
276   generation tasks, the prompt commonly comprises four possible components: (1) Natural language
277   instruction or docstring; (2) Starting code snippet; (3) A list of test cases; (4) Few-shot examples.
278
      Intuitively, natural language instruction provides high-level guidance that LLM should continually
      consider. This is confirmed by our experiment in Section 5.4.

284       RQ1 Can S PA effectively and consistently mitigate attention dilution and improve performance?
285       RQ2 Can the anchoring strength tuned on one code generation setting be transferred to another?
286       RQ3 How does the anchoring strength ω of S PA affect code generation performance?
287       RQ4 How does the selection of anchored text affect code generation performance?

289   4.1    C OMPARISON BASELINES
290
291   S PA requires access to the full logits generated by the large language models (LLMs), so we are
292   unable to evaluate closed-source models, such as GPT-4o and Claude-3.5-Sonnet. We select five
293   representative open-source code LLMs: CodeGen-Mono-350M (Nijkamp et al., 2023), CodeLlama-
294
      7B (Rozière et al., 2024), and DeepSeek-Coder-Instruct-1.3B, 6.7B, and 33B (Guo et al., 2024).
      These models have been fine-tuned for code generation tasks. Notably, the DeepSeek-Coder-Instruct
295
      models have been fine-tuned by instruction-tuning (Wei et al., 2022), while CodeGen-Mono and
296
      CodeLlama are standard text completion models. This selection aims to cover diverse SOTA code
297   LLMs of different types and sizes.

300
301   HumanEval (et al., 2021c). It includes 164 Python tasks designed by OpenAI developers. It was

      HumanEval+ and MBPP+. Although HumanEval and MBPP are considered de facto standards
308
      for assessing code LLMs, a recent study (Liu et al., 2023a) found they lack sufficient test cases and

      precise problem descriptions. This has been demonstrated as an issue that can lead to an unreliable

314   4.3    E VALUATION M ETRICS AND E XPERIMENT S ETUP
315
316   Evaluation Metric. Following prior work (et al., 2021b; Kulal et al., 2019; et al., 2021a), we measure
317   model performance using the Pass@k metric, which measures whether any of the top k candidates
318   can pass all the test cases. In our experiments, we calculate Pass@1 and Pass@10. For Pass@1,
319   LLMs generate a single code snippet using greedy sampling. The task is considered successful only
320
      if this generated code passes all test cases. For Pass@10, LLMs generate top 10 most probable code
      snippets using beam search. The task is deemed successful if any of these candidates pass all test
321
      cases.
322
323   Model Deployment. We downloaded and deployed LLMs from Huggingface. To expedite evalua-
      tions, we apply 8-bit quantization (Frantar et al., 2023; Dettmers et al., 2022) to all models. Prior


                                                        6
      Under review as a conference paper at ICLR 2025


324
      studies (Li et al., 2024; Huang et al., 2024) have demonstrated that this approach has very little
325   impact on LLM performance. All experiments were conducted on a 64-bit Ubuntu 22.04 LTS system,
326   equipped with an AMD EPYC 7313 CPU, eight NVIDIA A5500 GPUs, and 512GB of memory. The
327   experiments ran for approximately seven weeks.
328
      Prompt Design. We use the original task descriptions from the datasets as prompts for the text-
329
      completion models, CodeLlama and CodeGen-Mono. For the three DeepSeek-Coder-Instruct models,
330
      we format the prompts using the official chat template from HuggingFace.
331
332   Hyperparameter Tuning. For each model and dataset, we use grid search to tune the anchoring
333   strength ω on 1/5 tasks in dataset to get S PAtuned . We also get the optimal anchoring strength by
334
      tuning on the entire dataset (S PAoptimal ). We evaluate both of them on the remaining 4/5 dataset and
      compute Pass@1 via greedy search as well as Pass@10 via beam search (elaborated in Appendix A.6).

368   LLM’s attention on the initial prompt, we include two code generation examples in Appendix A.4.
369
      Note that the performance improvement is achieved only by amplifying the original prompt’s influence
370
      without introducing new knowledge or fine-tuning model parameters. We attribute S PA’s effectiveness
371
      to two reasons. First, when generating a new token, each prior token carries a risk of being incorrectly
372   attended to by the model. As the model generates more tokens that compete for attention, the
373   likelihood of attending to irrelevant tokens increases, thereby leading to errors. In contrast, the
374   original prompt represents the high-level user intent that is persistently relevant to generated tokens.
375   Anchoring the model’s attention on the original prompt via S PA essentially enlarges the reliable
376   portion of the model’s attention, thereby generating more accurate next tokens. Second, while each
377   self-generated token carries a probability to be error, autoregressive decoding assumes all prior
      tokens are correct. This allows for error propagation as more tokens are generated. By downplaying

378
      self-generated tokens, S PA essentially provides a fairer attention distribution by measuring the
379   trustworthiness of prior tokens. We discuss more in Appendix A.7.

      6    R ELATED WORK
464
465
      Code Generation. In recent years, there has been rapid progress in the development of code
      generation approaches (Dong & Lapata, 2016; Iyer et al., 2018) and benchmarks (et al., 2021c; Austin
466
      et al., 2021; Liu et al., 2023a; Hendrycks et al., 2021). With the advent of large language models
467
      (LLMs), such as GPT-4 (OpenAI & et al., 2024) and Gemini (Team & et al., 2024), code generation
468   has become a standard capability. Subsequent research has focused on fine-tuning these pre-trained
469   LLMs to achieve state-of-the-art performance.
470
471
      Despite their remarkable ability to follow natural language instructions, LLMs still face challenges
      when generating long and complex code. To enhance the code generation capabilities of LLMs,
472
      recent studies have explored train-free approaches such as prompt engineering (Denny et al., 2023;
473
      White et al., 2023), in-context learning (Dong et al., 2023; Li et al., 2023a;b), and retrieval-augmented
474   generation (Lewis et al., 2020; Du et al., 2024). Additionally, self-debugging techniques (Chen
475   et al., 2023) enable LLMs to debug code based on error messages and execution results, while
476   self-planning (Jiang et al., 2023) allows LLMs to decompose tasks into subtasks and implement
477   solutions step-by-step. The chain-of-thought approach (Le et al., 2024; Suzgun et al., 2022; Ma et al.,
478   2023) facilitates a step-by-step reasoning process in LLMs. Complementing these approaches, S PA
479   introduces an orthogonal approach particularly suitable for code generation. It can be integrated with
480   existing methods to further improve performance.
481   Controllable Generation. Compared to fine-tuning a language model (LM) at the decoding time,
482   controllable generation aims to steer the pre-trained LMs to match a sentence-level attribute (e.g.,
483   a topic on sports). Existing approaches usually require additional models or training, such as fine-
484   tuning a smaller LM (Liu et al., 2024a; 2021; Yang & Klein, 2021; Dathathri et al., 2020), a reward
485   model (Deng & Raffel, 2023; Lu et al., 2023), or a fine-tuned model with controlling codes (Krause
      et al., 2021; Li & Liang, 2021; Keskar et al., 2019). The mechanism used in S PA can also be used


                                                          9
      Under review as a conference paper at ICLR 2025


486
      to control the generation by adjusting anchoring strength over the input text. Compared to the
487   aforementioned works, S PA does not require any additional models or training.
488
489   Logit Arithmetic. There has been a growing body of methods that perform arithmetic on multiple
490
      logit distributions to enhance text generation. These methods include contrasting logits from multiple
      LMs (Liu et al., 2024a; 2021; Dou et al., 2019; Zhao et al., 2024), logits of LMs of different sizes (Li
491
      et al., 2023d), logits from different layers of a model (Chuang et al., 2024; Gera et al., 2023), and
492
      logits from the same model given different inputs (Pei et al., 2023; Shi et al., 2023; Malkin et al.,
493   2022; Sennrich et al., 2024; Leng et al., 2023). Similar ideas have also been explored in diffusion
494   models (Han et al., 2024; Ho & Salimans, 2022).
495
496
      S PA can be considered analogous to contrasting logits from the same model when given different
      inputs. However, we delve deeper by modeling a mathematical approximation of semantic adjustment
497
      over arbitrary groups of embeddings. Furthermore, S PA is specifically designed to address the
498
      attention dilution issue in LLMs during code generation—a phenomenon first observed in our work.
499   By contrast, none of existing works explored code generation tasks. They primarily focus on reducing
500   hallucinations (Shi et al., 2023; Sennrich et al., 2024; Leng et al., 2023), enhancing coherence (Malkin
501   et al., 2022), factuality (Chuang et al., 2024), and controllable text generation (Liu et al., 2021; Pei
502   et al., 2023; Zhao et al., 2024). Besides, S PA focuses on perturbation of the original prompt through
503   masking rather than providing additional context (Pei et al., 2023; Shi et al., 2023; Malkin et al.,
504   2022) or changing to a completely new prompt (Sennrich et al., 2024).
505
506
507   7    L IMITATIONS & F UTURE W ORK
508
509
510   We employed 8-bit quantized LLMs to expedite all experiments. Although this method has been
511   shown to have minimal impact on performance, we did notice some degradation. Furthermore, we
512   did not evaluate very large LLMs (e.g., more than 100B) due to computational constraints. Despite
513
      the unimodal feature, it is infeasible to enumerate all the anchoring strength ω on the continuous
      distribution. The real optimal ω should perform slightly better than the values reported in Section 4.
514
515   While S PA achieved a consistent improvement on LLMs with different sizes and types (i.e., instruction-
516   tuned & text completion), we do not observe a monotonic relationship between model attributes
517   and the improvement. Furthermore, there is no obvious correlation between the original model
518   performance and the improvement. It is an interesting future direction to investigate how different
519
      model attributes affect the improvement achieved by S PA. Given the performance improvements, the
      computational overhead of S PA is acceptable. We elaborate on this in Appendix A.5.
520
521   The effectiveness of S PA highlights its potential in other domains, particularly for generation tasks.
522   However, we believe rigorous experiments are necessary to confirm whether attention dilution exists
523   in other tasks, as different tasks may have unique input and output patterns. Investigating the existence
524   of attention dilution and determining which text to anchor in other tasks presents an interesting avenue
525
      for future research. In this work, we pre-define the method for selecting anchored tokens and use a
      fixed anchoring strength when generating code. We consider this approach a baseline. Future work
526
      could explore dynamically determining both the anchored text and the anchoring strength based on
527
      different contexts and sampling stages. Furthermore, the underlying principle of S PA is not confined
528   to transformer-based LLMs and could be adapted for use in other model architectures (e.g., RNNs).
529
530
531
532   8    C ONCLUSION
533
534   In this paper, we propose S PA, a model-agnostic approach designed to enhance the quality of
535   code generated by large language models (LLMs) by mitigating the attention dilution issue. S PA
536   employs a novel technique to adjust the influence of selected groups of input tokens, based on a
537   mathematical approximation. Our empirical study indicates that LLMs may overlook the initial
538   prompt as generating more new tokens. By amplifying the initial prompt’s influence throughout code
539   generation, S PA consistently and significantly improves performance across models of various sizes
      on multiple benchmarks.

911
912   To calculate Pass@10 in Section 5, we employ beam search to generate 10 candidate code snippets.
913   When running beam search with S PA, however, we found that directly sampling top beams based on
914
      the augmented logits produced by S PA led to performance degradation.
915   We hypothesize that this phenomenon occurs because while S PA successfully amplifies the influence

966   A.7   H YPOTHESIZED EXPLANATION FOR ATTENTION DILUTION AND S PA ’ S EFFECTIVENESS
967
968   S PA is motivated by a recent study Kou et al. (2024) and our empirical observations demonstrating
969   the attention dilution issue. Our experiment results in Section 5 echo our observation and confirm the
970   existence of attention dilution during code generation. Here we propose a detailed explanation for
971   this phenomenon based on our knowledge and hypotheses. We believe it stems from two limitations
      in regular decoding: (1) Distraction and (2) Error propagation.


                                                       18
       Under review as a conference paper at ICLR 2025


972
       Distraction. When a transformer generates a token, its correctness depends on two abilities: (1)
973    whether the model attends to the correct context, and (2) whether the model can derive the correct
974    token based on this context. S PA aims to improve the first ability. Suppose we have a perfect
975    transformer. For each generated token, it should only attend to relevant prior tokens and ignore
976    irrelevant ones. However, no model is perfect. For each prior token, there is a chance the model
977    incorrectly identifies and attends to it. More tokens mean a higher probability that the attention
978    contains an error, thereby leading to distraction.
979    While self-generated tokens are also important context, they are less persistently related than task
980    description in code generation. Amplifying the task description via S PA can improve attention
981    reliability, thereby mitigating distraction.
982
       Error propagation. Compared to reliable task description tokens, the self-generated code tokens
983
       may be wrong. However, autoregressive decoding assumes all prior tokens are correct, and all the
984
       tokens have an equal opportunity to compete for the model’s attention. As a result, the later a token
985    is generated, the higher the probability it is wrong as errors propagate. S PA adds extra attention to
986    earlier tokens that are less likely to be incorrect, creating a fairer attention distribution.
