009                                               A BSTRACT
010
011            Large language models (LLMs) are empowering decision-making in open-world
012            agents in several applications, including tool or API usage and answering mul-
013            tiple choice questions (MCQs). However, they often make overconfident, incor-
014            rect predictions, which can be risky in high-stakes settings like healthcare and fi-
015            nance. To mitigate these risks, recent works have used conformal prediction (CP),
016
               a model-agnostic framework for distribution-free uncertainty quantification. CP
               transforms a score function into prediction sets that contain the true answer with
017
               high probability. While CP provides this coverage guarantee for arbitrary scores,
018
               the score quality significantly impacts prediction set sizes. Prior works have relied
019            on LLM logits or other heuristic scores, lacking quality guarantees. We address
020            this limitation by introducing CP-OPT, an optimization framework to learn scores
021            that minimize set sizes while maintaining coverage. Furthermore, inspired by the
022            Monty Hall problem, we extend CP’s utility beyond uncertainty quantification to
023            improve accuracy. We propose a method called conformal revision of questions
024            (CROQ) to revise the problem by narrowing down the available choices to those
025            in the prediction set. The coverage guarantee of CP ensures that the correct choice
026            is in the revised question prompt with high probability, while the smaller number
027
               of choices increases the LLM’s chances of answering it correctly. Experiments on
               the MMLU, ToolAlpaca, and TruthfulQA datasets with Llama-3 and Phi-3 models
028
               show that optimized CP scores reduce set sizes while maintaining coverage guar-
029
               antee, and CROQ shows significant improvement in accuracy over the standard
030            inference procedure.

033   1   I NTRODUCTION
034
035   Large language models (LLMs) (Touvron et al., 2023; Databricks, 2024; Abdin et al., 2024) have
036   demonstrated remarkable capabilities in various decision-making tasks, including multi-choice
037   question answering (MCQ) and tool usage, where the model must select the correct tool or API
038   to complete a task (Qu et al., 2024; Tang et al., 2023; Hendrycks et al., 2021). However, quantifying
039
      the uncertainty of an LLM’s predictions remains challenging, with LLMs themselves often exhibit-
      ing overconfidence in wrong answers (Krause et al., 2023; Groot and Valdenegro Toro, 2024). Given
040
      a reliable uncertainty measure, the LLM could, in highly uncertain instances, take measures such as
041
      deferring to a human or invoking a secondary model, thereby increasing overall accuracy. Without
042   this, it may be difficult to prevent the LLM from making serious errors in these settings. To address
043   these challenges, we focus on improving uncertainty quantification (UQ) and its utility in MCQ and
044   tool usage tasks.
045
      Conformal prediction (CP) is one framework that can be used to quantify uncertainty in LLMs
046
      (Kumar et al., 2023; Su et al., 2024; Mohri and Hashimoto, 2024; Quach et al., 2024). Conformal
047
      prediction is a model-agnostic and distribution-free technique for producing prediction sets that
048   contain the true outcome with a user-specified probability (e.g. 95%). The set sizes provide a natural
049   measure of uncertainty, with small (large) sets representing low (high) uncertainty. The construction
050   of sets in CP depends on a score function, which measures how well a candidate output “conforms”
051   to a given input. For example, in classification settings, it is common to use the classifier logits
052   corresponding to each class for a given input as the score function (Angelopoulos and Bates, 2022).
053   While conformal prediction gives a coverage guarantee for any score function, the size of the output
      sets depends on the score function that is used. As an example, a random score function will yield


                                                        1
      Under review as a conference paper at ICLR 2025


054
      output sets that constitute random subsets of the label space that are large enough to satisfy the
055   coverage guarantee (Angelopoulos and Bates, 2022).
056
057   Previous works that apply conformal prediction for MCQ type settings have used readily available
058
      scores such as the logits (softmax) output from the LLM (Kumar et al., 2023) or have designed
      heuristic scores based for example on repeated querying of the LLM (Su et al., 2024). Logits can be
059
      overconfident and may show biases for some options (Zheng et al., 2024), and heuristic scores are
060
      not guaranteed to produce small sets. Thus we need a principled solution to obtain scores that are
061   designed to minimize set sizes (uncertainty) while preserving the coverage guarantee.
062
063   In addition to providing a measure of uncertainty, we believe that the sets produced by conformal
064
      prediction can be leveraged for other downstream purposes. Specifically, inspired by the Monty
      Hall problem (Selvin, 1975; Rosenthal, 2008), we hypothesize that by revising a multiple-choice
065
      question or tool usage prompt to include only the options within a conformal prediction set, the
066
      LLM is more likely to provide the correct answer due to the reduced number of choices, leading to
067   improved accuracy.
068
069   To summarize, we focus on (1) obtaining optimal scores for conformal prediction for MCQ and
070
      tool usage tasks in LLMs, and (2) investigating whether the prediction sets generated by conformal
      prediction can be used to improve downstream accuracy. Our main contributions are as follows:
071
072
      1. We design a score function optimization framework (CP-OPT) that can be applied to any pre-
073
         trained LLM. Moving away from the potentially unreliable LLM logits and heuristic scores, our
074      framework provides a principled way to learn the scores for conformal prediction. Empirically,
075      we show that our procedure leads to reduction in average set sizes in contrast to the baseline
076      procedure that uses the LLM logits as the scores, at the same level (95%) of marginal coverage.
077
      2. Extending the utility of conformal prediction beyond uncertainty quantification, we propose the
078
         conformal revision of questions (CROQs), in which we revise the question by narrowing down
079      the choices to those in the prediction sets output by conformal prediction. Then the LLM is
080      re-prompted with the revised question. We show that this procedure can boost overall accuracy
081      by upto 5% (Figure 2) and upto 14% on part of ToolAlpaca dataset with 10 options (Table 2).

090   2.1   S OLVING M ULTIPLE C HOICE Q UESTIONS (MCQ S ) U SING LLM S
091
092   MCQ Setup. MCQs are a general abstraction for expressing problems in which the correct choice(s)
093
      must be selected from a given set of choices. These encompass conventional question-answering
      tasks such as MMLU (Hendrycks et al., 2021) as well as other tasks such as tool learning, in which
094
      the LLM must select the correct tool or API to complete a task (Tang et al., 2023; Qu et al., 2024).
095
      An MCQ consists of the question text Q, i.e. a sequence of tokens, and a set of answer choices
096   O = {(Y1 , V1 ), (Y2 , V2 ), . . . , (Ym , Vm )}. Here, each Yj is a unique character from the English
097   alphabet, and we assume that the number of choices m is less than or equal to the size of the alphabet.
098   Each Vj is the option text for the jth option. Denote the whole MCQ instance as x = (Q, O). Let
099   Xm denote the space of MCQs with m choices and PXm denote a distribution over Xm , from which
100   samples for training, calibration, and testing are drawn independently. Here, we assume that for
101   each question Q there is only one correct answer key y ⋆ ∈ {Y1 , Y2 , . . . Ym } = Ym .
102   MCQ Prompt. We concatenate the question text Q and the answer choices O, all separated by a new
103   line character, and append to the end the text “The correct answer is: ”. The expectation
104   is that given this input prompt, the next token predicted by the LLM will be one of the option keys.
105   See Appendix D for a prompt example. We consider zero-shot prompts and do not include example
106   questions and answers in the prompt. We also add the prefix and suffix tokens to the prompt as
107   recommended by the language model providers. Since these are fixed modifications to x, we will
      use x to denote the final prompt and the MCQ instance analogously.

      Under review as a conference paper at ICLR 2025


108
      LLM Inference. We run the forward pass of the auto-regressive LLM (Touvron et al., 2023; Dubey
109   et al., 2024; Abdin et al., 2024) on the input prompt to obtain the logit scores for each possible next
110   token given the prompt, restricting attention to the tokens that correspond to the available answer
111   keys (e.g. “a”, “b”, “c”, “d” if there are four answer options). We take the softmax to convert the
112   logits to probabilities, and then we take as the LLM’s answer the option with the highest probability.
113   This approach ensures that the LLM’s answer will be one of the available answer options, which
114   would not be guaranteed if instead we asked the LLM to simply generate an answer token given the
115   prompt. This approach mirrors what has been done in other works that use LLMs to solve MCQs
116   (Kumar et al., 2023; Su et al., 2024). Formal details are given in Appendix A.

118   2.2   C ONFORMAL P REDICTION
119
      Conformal prediction (CP) (Vovk et al., 2005; Angelopoulos et al., 2022) is a framework for quan-
120
      tifying uncertainty in machine learning models. It provides a flexible and user-friendly approach to
121   output prediction sets (which may be finite sets or intervals) which contain the true output or label
122   with a probability that is specified by the user, e.g. 95%. The key strength of conformal prediction
123   lies in its distribution-free guarantees: it ensures that the constructed prediction sets are valid re-
124   gardless of the underlying data distribution and model. This property is particularly desirable in the
125   context of language models, as it is hard to characterize language data distributions or put specific
126   distributional assumptions/restrictions on the LLMs.
127   Score Function. Let g : Xm × Ym 7→ R be a conformal score function, where larger scores
128   indicate better agreement (“conformity”) between x and y. Intuitively, large scores are intended to
129   indicate that y is a plausible output given x, while smaller scores indicate less plausibility. (Note that
130   some authors prefer to have larger scores indicate greater disagreement, e.g. Clarkson et al. (2024).)
131   A common choice of score function is the softmax scores from the given model. For closed-source
132   LLMs, where these scores are not available, other authors have devised self-consistency scores based
133   on repeated querying of the model (Su et al., 2024).
134   Prediction Sets. Given a threshold τ on the scores, the prediction set for any x ∈ Xm is given by
135
                                    C(x;g, τ ) := {y ∈ Ym : g(x, y) ≥ τ }.                            (1)
136
137   Intuitively, larger sets represent greater uncertainty, while smaller sets represent less uncertainty.
138   This can be used for example to compare two different score functions given a fixed confidence
139
      level: a score function that produces larger sets can be said to result in greater uncertainty. Note
      that because the coverage guarantee is marginal over the data distribution (Proposition 2.1), it does
140
      not immediately follow that set sizes can be compared across different parts of the input space:
141
      coverage conditional on specific set sizes is not necessarily equal to the overall coverage level.
142   However, previous work has found that the size of sets correlates with LLM accuracy in an MCQ
143   setting (Kumar et al., 2023), indicating the set size as a measure of confidence in the LLM’s output.
144
145
      Split Conformal Prediction. Similar to prior works (Kumar et al., 2023; Su et al., 2024), we use
      Split Conformal Prediction (Papadopoulos et al., 2002; Lei et al., 2018) due to its popularity, ease
146

174

175   MCQs with LLMs. Similar ideas have been incorporated in the training objective of classifiers
176   (Stutz et al., 2022) so that the classifiers’ softmax output is better suited for CP. However, the LLMs
177   are not trained with this objective, and we want to apply CP to any given LLM; therefore, we design
178   a post-hoc method to optimize the scores. We first characterize the optimal scores and then describe
179   how we can estimate them in practice.

239   3.2   C ONFORMAL R EVISION OF Q UESTIONS (CROQ)
240
241   Here we consider how conformal prediction sets can be used for downstream purposes other than
242
      uncertainty quantification, namely for improving the final accuracy in MCQ type tasks. Our proce-
      dure involves re-prompting the LLM with the reduced answer options from a conformal prediction
243
      set. We describe our procedure and then discuss the connection to the Monty Hall problem. The
244
      steps are also illustrated with an example in Figure 1.
245
246   Scores and Threshold for Conformal Prediction. We first fix a score function g, which could
247   be any arbitrary function but which here we restrict to either logits from the LLM or our CP-OPT
248
      scores (Section 3.1). We then run the split conformal procedure with coverage level 1 − α for some
      α ∈ [0, 1] to estimate the threshold τ̂ . CROQ then proceeds as follows.
249
250   Step 1: Get Conformal Prediction Set. Given a test instance x, we generate a first stage prediction
251   set, C(x ; g, τ̂ ). Per the coverage guarantee (Proposition 2.1), we expect that the true answer y ⋆ ∈
252   C(x ; g, τ̂ ) with probability at least 1 − α.
253   Step 2: Revise and Ask the LLM Again. If the first stage prediction set C(x ; g, τ̂ ) is empty or is
254   of size 1 or size m (the number of answer options), then we simply utilize the LLM’s initial answer,
255   as described in section 2.1, since the conformal procedure has yielded no additional information.
256   Otherwise, we modify the prompt x to x′ = (Q, O′ ), where O′ = {(Kj , Vj ) : Kj ∈ C(x ; g, τ̂ )}.
257   The keys in O′ are changed so that they start with the first letter of the alphabet and go to the letter
258   corresponding to the number of choices available. For example, if there were initially four answer
259   options {a, b, c, d}, and the conformal prediction set was {c, d}, then the two options in the set
260
      would receive new keys {a, b}. Then x′ is transformed into a prompt format and input to the LLM
      and the standard inference procedure (section 2.1) is run to extract the predicted answer key ŷ ′ .
261
262   Next, we discuss an interesting analogy between CROQ and the Monty Hall problem to provide
263   insights into how CROQ can improve LLM accuracy and show its flexibility to accommodate oracles
264   such as other LLMs.
265   Connection to the Monty Hall Problem. Monty Hall is a probability puzzle (Selvin, 1975;
266   Granberg, 1999; Rosenthal, 2008) based on a popular game show, where a contestant chooses one of
267   three doors (choices), behind one of which is a car and the others, goats. The contestant chooses an
268   initial door, which remains closed. After the host reveals a goat behind another door, the contestant
269   is asked if they want to switch doors. Under typical assumptions, switching offers a higher chance
      of winning, i.e. selecting the door with a car behind it.

      In CROQ, we leverage conformal inference to eliminate “goats” (incorrect answers). The conformal
293   set produced in the first stage of CROQ contains the correct answer with some user-specified proba-
294   bility, say 95%. In that sense, we imagine that an (imperfect or probabilistic) oracle is opening some
295   number of doors (answer options) that with high probability reveal only goats (incorrect answers).
296   Those answer options are eliminated, and then the LLM is queried again with the remaining set
297   of answers. Thus, similar to the Monty Hall, there is a reduction in uncertainty after some doors
298   (choices) are opened (eliminated), which we hypothesize leads to improved chances of winning
299   (answering correctly). Hence, CROQ can enable an LLM agent to “self-correct”.
300   Here the oracle’s “knowledge” comes from the distribution of the scores of correct answers, so
301   it is not contained within the LLM’s representation of any given query. In that sense, it is extra
302   information that gets added to the given query. Additionally, the score function and the resulting
303   distribution of scores can come from any source. For example, from another LLM, from embeddings
304   that measure semantic similarity between questions and answers, etc. The extra information that the
305   conformal procedure represents can therefore come entirely from an external source.
306   Coverage-Accuracy Trade-offs. Conformal prediction has a small risk, namely a probability α, of
307   eliminating the door with the “car” (true choice). If α is too small, we will have high coverage but
308   may have relatively large sets i.e. little reduction in the uncertainty. If α is too large, we will too
309   frequently exclude the correct answer from the revised question. Since our interest is in improving
310   final accuracy, we treat α as a tuning parameter that can be optimized for accuracy (Section 4.2).

472
      Conformal Prediction for Uncertainty Quantification with LLMs Recently there has been
473
      growing interest in using conformal prediction to quantify and control uncertainty in LLM-related
474   tasks. In the context of multi-choice question answering (MCQ), previous works have investigated
475   a variety of conformal score functions, including (the softmax of) the LLM logits corresponding to
476   the response options (Kumar et al., 2023; Ren et al., 2023) or functions thereof (Ye et al., 2024),
477   confidence scores generated by the LLM itself, or “self-consistency” scores derived by repeated
478   querying of the LLM (Su et al., 2024). We build on this work by aiming to learn a conformal score
479   function that yields small conformal sets, rather than taking the score function as given.
480   In addition to the MCQ setting, there has been recent work utilizing conformal prediction in the con-
481   text of open-ended response generation (Quach et al., 2024; Mohri and Hashimoto, 2024; Cherian
482   et al., 2024). This setting differs in that there is not necessarily a unique correct response, so the no-
483   tion of coverage must be redefined around acceptability or factuality rather than correctness. When
484   factuality is the target, the goal is to calibrate a pruning procedure that removes a minimal number
485   of claims from an LLM-generated open response, such that the remaining claims are all factual with
      high probability; that is, the goal is to retain as large a set as possible, rather than to generate a set


                                                          9
      Under review as a conference paper at ICLR 2025


486
      with the smallest number of responses possible as in MCQ. Conformal prediction has also been used
487   to capture token-level uncertainty (Deutschmann et al., 2024; Ulmer et al., 2024).
488
489   Optimizing conformal prediction procedures Several recent works have considered how to learn
490   good conformal score functions from data, primarily in the context of supervised learning models
491   (Bai et al., 2022; Stutz et al., 2022; Yang and Kuchibhotla, 2024; Xie et al., 2024). With LLMs,
492   Cherian et al. (2024) consider how to learn a good score function to achieve factuality guarantees;
493   their optimization problem differs from ours due to the difference in setting as well as the addition
494   of conditional coverage constraints (ensuring that coverage holds in different parts of the feature
495   space). Kiyani et al. (2024) design a framework to minimize the size (“length,” in their terminology)
496   of conformal sets, which they apply to MCQ as well as to supervised learning problems. However,
497
      their framework is concerned with how to generate sets given a model and a conformity score, rather
      than how to learn a conformity score.
498
499   The works mentioned above all aim to produce (small) conformal sets that satisfy coverage guaran-
500   tees. Among these, only Ren et al. (2023) consider how conformal sets may be used downstream, in
501   their case to improve the efficiency and autonomy of robot behavior. To our knowledge, our work is
502   the first to investigate whether conformal prediction can be used to increase the accuracy of LLMs
503
      on MCQ type tasks.
504

      6   C ONCLUSION AND F UTURE W ORK
506   We investigated how conformal prediction can be used to quantify and reduce output uncertainty for
507   decision-making problems such as tool selection and multi-choice question answering (MCQ) with
508   LLMs. We defined an optimal conformal score function (P1) that minimizes average set size subject
509   to a coverage constraint, and we showed how to estimate it using a differentiable loss function
510   that can be optimized via stochastic gradient descent (P2). We called this procedure CP-OPT. In
511   experiments with a variety of models, datasets, and answer option cardinalities, we showed that CP-
512   OPT results in smaller average set sizes than the baseline score function consisting of the LLM logits
513
      corresponding to the MCQ answer options, although these smaller set sizes sometimes resulted in
      reduced coverage. We emphasize that CP-OPT is extremely general and can be applied with different
514
      models, features sets, etc. If the LLM logits are already highly informative with regard to model
515
      uncertainty, then there may be little to gain (but nothing to lose) from applying CP-OPT, but in other
516   settings there may be substantial benefit from optimizing the conformal scores.
517
518   We further investigated whether re-prompting an LLM with the answer options contained in the
519
      conformal set would result in higher final accuracy on MCQ tasks. We called this procedure CROQ.
      The intuition is that conformal sets will contain the true answer with high probability while poten-
520
      tially substantially reducing the number of answer options the LLM has to consider. We found that
521
      CROQ increases accuracy in most of the cases and that there is an interplay between the coverage
522   level and the accuracy improvement which can be optimized. As an additional consideration of how
523   prediction sets can be used for purposes other than uncertainty quantification, we illustrated how to
524   use them to decide when to defer to human judgment, with both the accuracy and the deferral cost
525   increasing as the set size cutoff shrinks.
526
      A particularly interesting extension to the CROQ procedure is performing multiple rounds of ques-
527
      tion revisions. While we have demonstrated that a reduction in the number of available response
528   options increases LLM accuracy, we conjecture that a further reduction in conformal set sizes could
529   occur as well. Repeated conformal inference would come at higher computational costs, as perform-
530   ing it at each round would require repeated calibration, and may require fresh data to calibrate on.
531   Additionally, without adjusting the coverage level used at each round, the final coverage rate will be
532   lower than the nominal rate. Developing methodology to make a multi-round CROQ procedure both
533   efficient and technically sound is a promising line of future research. Different models could also be
534   used to generate the initial prediction set and the final answer, instead of a single LLM for both.
535   Other future lines of investigation include considering how to calibrate conformal score thresholds
536   when the number of answer options that the LLM may encounter in MCQ may vary. This is relevant
537   for example in tool usage problems, where an LLM may be asked to consider very different numbers
538   of APIs for different types of queries. One possible approach involves quantile regression against
539   the numbers of answer options, which may enable reasonable estimation of the quantiles in cases
      where some numbers of answer options are not well represented in the calibration data.

674   B.1   D ETAILS OF FEATURES AND G USED IN EXPERIMENTS

676
      Let z ∈ Rd+m be the concatenation of the LLM’s penultimate layer’s representation (d-
      dimensional) and logits (m-dimensional) for the last token. Our choice of G for the experiments
677
      is defined as follows,
678
679             G := {g : Rd0 → ∆m−1 | g(z) := softmax(W3 tanh(W2 tanh(W1 (z)))),
680                                         W1 ∈ Rd0 ×d1 , W2 ∈ Rd1 ×d2 , W3 ∈ Rd2 ×m }
681
682   Here, d0 = d + m, d1 = (d + m)/2, and d3 = (d + m)/4 and ∆m−1 is the m − 1 dimensional

685   B.2   R ESULTS
686
687   Figure 5 shows accuracy after the CROQ procedure as a function of α for Phi-3. The results are
688   qualitatively similar to the results for Llama-3 in the main text (Section 4.2).
689
      Table 3 illustrates how often CROQ causes the LLM’s answer to change relative to the baseline infer-
690   ence procedure. In Monty Hall terms (Section 3.2), these represent how often the LLM “switches”
691   its answer in response to the elimination of some answer options. We see that the LLM switches
692   some initially correct answers to wrong answers and vice versa but that the wrong-to-right switch
693   frequency generally outweighs the right-to-wrong switch frequency, resulting in higher accuracy
694   after CROQ.
695
      Figure 4 provides motivation for CROQ, demonstrating that a reduction of set size leads to improved
696   accuracy. Using the Truthful QA dataset with 15 response options, we first construct conformal pre-
697   diction sets using logits. With these prediction sets, we then leverage oracle knowledge to reduce the
698   prediction set size by 0 to 10 options - with 0 eliminations being the regular CROQ method. Incor-
699   rect answers are randomly eliminated, ensuring that coverage remains constant. As more incorrect
700   answers are eliminated, the accuracy of the LLM after requerying increases as more answers are re-
701   moved (smaller prediction set). These results motivate the use of a score function which minimizes
      set size while controlling coverage.

       C     C ALCULATION OF S TATISTICAL S IGNIFICANCE
1189
1190
       All our statistical significance results are based on paired sample t-tests at level α = 0.05 of the null
1191
       hypothesis that the difference under consideration is 0. The relevant differences are the differences
1192   in set sizes or coverage values using logits vs. our CP-OPT scores (Table 10), and the differences
1193   in accuracy before and after applying the CROQ procedure (all other tables except for Tables 3 and
1194   22). This is equivalent to constructing 95% confidence intervals for the differences and marking
1195   results as significant whenever the corresponding confidence intervals exclude 0. We used paired
1196   rather than unpaired tests to account for the fact that each pair of values was measured on the same
1197   test set item.
1198   Note that paired t-tests, like paired z-tests, assume that sample means are approximately normally
1199   distributed, which holds in our setting due to the central limit theorem and the relatively large sizes
1200   of the test sets. (The central limit theorem is often invoked to justify approximate normality when
1201   sample sizes are larger than 30.) At our sample sizes, t-tests are almost identical to z-tests, but they
1202   are very slightly more conservative.
1203
       For the CROQ results, hypothesis tests were conducted to compare overall accuracy before and after
1204   the CROQ procedure. Tests were not conducted to compare accuracy conditional on each possible
1205   set size, since many set sizes have small associated samples which results in little power to detect
1206   differences.

       Dataset Description
1213
1214   MMLU (Hendrycks et al., 2021) is a popular benchmark dataset for multiple choice questions
1215   (MCQs) from 57 domains including humanities, math, medicine, etc. In the standard version, each
1216
       question has 4 options, we create two augmented versions with 10 and 15 options for each question
       by adding options from other questions on the same topic. We ensure there is no duplication in
1217
       options. The standard dataset has very little training points, so we randomly draw 30%, and 10%
1218
       of the points from the test split and include them in the training set and validation set respectively.
1219   Note, that we remove these points from the test set. The resulting splits have 4.5k, 2.9k, and 8.4k
1220   points in the train, validation, and test splits.

1366   Dataset Details
1367   The TruthfulQA dataset (Lin et al., 2022) contains 817 questions designed to evaluate truthfulness
1368   of LLM responses. Although the dataset is primarily use to evaluate open responses generated by
1369   LLMs, the dataset is also prepared in an MCQA format. We perform evaluation of MCQA on the
1370   “MC 1 Targets”, and resample questions using additional correct responses from “MC 2 Targets”,
1371   so that a single truthful answer is included in the response set. The dataset was split randomly by
1372   question, so that there was no overlap between splits. After resampling using the “MC 2 Targets”,
1373   the train split contains 1,745 questions, the calibration split contains 695 questions, and the test
1374
       split contains 395 questions. Since we consider questions with a fixed number of response options,
       answers were either downselected, or additional responses from different, random questions from
1375
       the same split were inserted.

1405
1406   Dataset Details
1407
1408   ToolAlpaca (Tang et al., 2023) contains 3.9k tool-use instances from a multi-agent simulation en-
1409
       vironment. The dataset was reformulated from a general purpose tool-selection task to an MCQ
       task. The LLM is prompted with an instruction and an API description and must select the correct
1410
       function based on the function name and a brief description.
1411
1412   We filter out APIs that had an error in generating documentation, instances where a ground truth
1413   label was missing, and instances that required multiple, sequential function calls. After filtering,
1414   2,703 MCQ examples remain. The train split contains 856 synthetic examples, the calibration split
1415
       contains 774 synthetic validation examples, and the test split contains 1040 real and synthetic API
       examples. Splits are created to ensure no overlap in APIs occur. We follow a similar resampling
1416
       procedure as used for TruthfulQA, so that the number of response options is fixed. Arguments are
1417
       stripped from the provided function call so that the MCQ task was focuses towards tool selection, a
1418   critical task in the more general tool usage problem.
