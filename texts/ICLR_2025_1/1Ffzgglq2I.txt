011                                                A BSTRACT
012
013            Offline reinforcement learning has become one of the most practical RL settings.
014            However, most existing works on offline RL focus on the standard setting with
015            scalar reward feedback. It remains unknown how to universally transfer the ex-
016
               isting rich understanding of offline RL from the reward-based to the preference-
               based setting. In this work, we propose a general framework to bridge this gap.
017
               Our key insight is transforming preference feedback to scalar rewards via binary
018
               reward labeling (BRL), and then any reward-based offline RL algorithms can be
019            applied to the dataset with the reward labels. The information loss during the
020            feedback signal transition is minimized with binary reward labeling in the prac-
021            tical learning scenarios. We theoretically show the connection between several
022            recent PBRL techniques and our framework combined with specific offline RL al-
023            gorithms. By combining reward labeling with different algorithms, our framework
024            can lead to new and potentially more efficient offline PBRL algorithms. We em-
025            pirically test our framework on preference datasets based on the standard D4RL
026            benchmark. When combined with a variety of efficient reward-based offline RL
027
               algorithms, the learning result achieved under our framework is comparable to
               training the same algorithm on the dataset with actual rewards in many cases and
028
               better than the recent PBRL baselines in most cases.

      1   I NTRODUCTION
032
033
      Reinforcement learning (RL) is an important learning paradigm for solving sequential decision-
034
      making problems (Sutton & Barto, 2018). Compared to the standard (reward-based) RL that requires
035
      access to reward feedback (Schulman et al., 2017), preference-based RL (PBRL) (Wirth et al., 2017)
036   only requires preference feedback over a pair of trajectories, making it more accessible in practice.
037   In the offline learning setting, the agent only needs a pre-collected dataset of preference labels before
038   training, making it even more convenient (Zhu et al., 2023). When humans provide preference labels,
039   PBRL is known as reinforcement learning from human feedback (RLHF) (Stiennon et al., 2020), a
040   popular framework for aligning large language models.
041
      Despite the success in offline PBRL (Kim et al., 2023; Dai et al., 2023; Hejna & Sadigh, 2024),
042   the topic is less studied compared to the offline RL in the standard reward feedback setting (Li
043   et al., 2024; Levine et al., 2020; Fujimoto & Gu, 2021; Kidambi et al., 2020; Wu et al., 2019;
044   Cheng et al., 2022; Tarasov et al., 2024). The critical feature in offline learning is ‘pessimism’ (Li
045   et al., 2024). The learning agent should be pessimistic about the policies whose behaviors are not
046   included in the dataset. Many pessimistic learning algorithms with theoretical insights have been
047   developed for the standard offline RL setting. However, most empirical works for offline PBRL only
048   adopt limited specific pessimistic learning approaches, such as applying certain regularization to the
049   learning policy (Rafailov et al., 2024; Stiennon et al., 2020), which are shown to be less efficient
050
      compared to recent SOTA methods in the standard RL setting (Cheng et al., 2022; Tarasov et al.,
      2024).
051
052   The difference between the current PBRL and standard RL only comes from the feedback signal’s
053   format. Both settings assume the environment as a Markov decision process (MDP) and aim to find
      the policy that achieves a high cumulative reward (i.e., is aligned with the reward function). The

      to solve the PBRL problem. Wang et al. (2024) has theoretically demonstrated this feasibility in
073
      online learning under certain conditions. This work aims to develop a framework that bridges the
074
      gap between PBRL and standard RL so that one can solve a PBRL problem with a standard offline
075   RL algorithm. To construct such a framework, one faces the following difficulties:
076
077   • Standard offline RL algorithms are very different from each other. Model-based algorithms (Ki-
078     dambi et al., 2020; Yu et al., 2020) learn an environment model and then run online learning
079     algorithms on the model. Model-free algorithms (Cheng et al., 2022; Fujimoto & Gu, 2021) learn
080     a pair of actor-critic models directly to infer the optimal policy without modeling the environment.
081     The framework needs to apply to all kinds of learning algorithms.
082   • Preference feedback contains arguably less information than reward feedback. It is impractical to
083     recover the actual rewards from limited preference feedback. The framework needs to make the
084     standard offline RL algorithms work with incomplete information from preference feedback.
085
086   In this work, we build a novel framework called ‘BRL’ based on a reward labeling technique that
087   labels the preference dataset with scalar rewards. The critical insight is that we only need to consider
088   transforming feedback signals from preference to reward format while minimizing information loss.
089   The standard offline RL algorithm can handle the pessimistic learning afterward. Note that reward
090   labeling fundamentally differs from reward modeling, which is widely considered in recent PBRL
091   studies (Kim et al., 2023; Stiennon et al., 2020). Reward modeling aims to infer rewards at state ac-
092
      tions that are not in the dataset. In contrast, reward labeling focuses on interpreting the rewards for
      the state actions inside the preference dataset. To minimize the information loss in the transforma-
093
      tion, we want to find the optimal reward labels that best explain the state actions in the dataset. We
094
      show that a simple binary labeling technique already gives the optimal reward labels in a practical
095   case where the trajectories in the dataset have no overlap. We further empirically show that in the
096   general case where no matter whether there is an overlap between trajectories or not, the learning
097   algorithms achieve better performance on the reward labels given by the binary labeling technique
098   than on those given by the reward modeling technique. In Fig 1, we illustrate how our framework is
099   used. In summary, our contributions are as follows.
100
101   • We propose a general framework to bridge the gap between offline PBRL and standard offline RL.
102     Our framework involves labeling the dataset with the reward that maintains most information in
103     the preference signals. Afterward, one can train on the reward-labeled dataset with any standard
104
        offline RL algorithms. Our framework can be easily implemented without modifying the original
        RL algorithms. One can possibly construct more efficient PBRL algorithms by applying SOTA
105
        standard RL algorithms to our framework.
106
107   • We mathematically analyze the combination of our framework with standard offline RL algo-
        rithms. We show that current state-of-the-art PBRL techniques are closely related to the combi-

116
117   2.1    O FFLINE R EWARD -BASED R EINFORCEMENT L EARNING
118
119
      Offline reward-based reinforcement learning, or standard offline RL, is the most popular offline RL
      setting. The fundamental challenge for offline RL is known as ‘distribution mismatch’ (Levine et al.,
120
      2020). It refers to the phenomenon that the data distribution in the dataset may not match the dis-
121
      tribution induced by the optimal policy. Due to distribution mismatch, an efficient learner must be
122   ‘pessimistic’ (Li et al., 2024). In general, it says that an agent should rely more on the policies
123   whose distributions match the dataset distribution better, as the agent cannot correctly evaluate other
124   policies not covered by the dataset. Many algorithms have been proposed to solve the problem fol-
125   lowing very different pessimistic learning techniques. Kidambi et al. (2020); Yu et al. (2020; 2021)
126   proposed model-based learning algorithms that learn a world model first and then apply online RL
127   to learn from the world model. Wu et al. (2019) proposes a behavior regularization approach closely
128   related to the regularization considered in the RLHF studies. Fujimoto & Gu (2021); Kostrikov et al.
129   (2021); Kumar et al. (2020); Levine et al. (2020) proposed model-free learning algorithms that don’t
130
      need to learn the world model. Cheng et al. (2022) provide theoretical guarantees on the efficiency
      of their model-free learning methods that the learned policy is always better than the behavior pol-
131
      icy used to collect the dataset and can compete with the best policy covered by the dataset. Li et al.
132
      (2024) mathematically interprets the essence of pessimistic learning algorithms.
133
134
      2.2    O FFLINE P REFERENCE -BASED R EINFORCEMENT L EARNING
135
136   Zhan et al. (2023); Zhu et al. (2023) theoretically investigate the problem of pessimistic learning
137   in the standard offline PBRL setting, the same as the one considered in this work. They propose
138   a method that is guaranteed to learn near-optimal policy depending on the dataset, but it remains
139   unknown how to implement these algorithms in practice. Stiennon et al. (2020); Ouyang et al.
140   (2022) studies the problem of language model fine-tuning, where humans provide the preference
141   labels. These works, also known as RLHF, do not apply to the general RL setting. These works also
142   only consider behavior regularization for pessimistic learning, which is shown to be less efficient
143
      than other advanced pessimistic learning techniques.
144   A line of research studies a variant of the offline learning setting. Here, the learning agent can access
145   a preference dataset consisting of preference labels over trajectory pairs and a demonstration dataset
146   consisting of only trajectories. Kim et al. (2023) proposes to use a transformer architecture to ap-
147   proximate the reward model. Hejna & Sadigh (2024) proposes to adapt the IQL learning framework
148   to the preference-based learning setting. Zhang et al. (2023) proposes to learn a preference model
149
      instead of a reward model and then learn a policy aligned with the preference model.
150   Another line of research Sadigh et al. (2017); Shin et al. (2021; 2023) studies the PBRL problem in
151   the active learning setting. In this case, the learning starts from a pre-collected behavior dataset with
152   no preference label. During training, the agent can query an expert to provide preference feedback
153   on a pair of trajectories sampled from the behavior dataset by the agent in an online manner. Then,
154   the agent learns a reward model from the preference feedback and applies RL algorithms to learn
155
      from the reward model.

      3.2   P REFERENCE BASED R EINFORCEMENT L EARNING (PBRL)
173
174
175
      PBRL is a variant of RL where the feedback signals consist of preferences rather than scalar rewards.
      Given a pair of sequences of states and actions, also known as trajectories, a preference model P
176
      gives a preference for the trajectories. Formally, let T = (S, A)T be the space of trajectories
177
      with length T , the preference model is a mapping P : T × T → [0, 1]. P (τ1 , τ2 ) represents
178   the probability of the preference model preferring the first trajectory τ1 over the second one τ2 .
179   Following recent PBRL studies, we assume the preference model is related to the reward function.
180   More specifically, there exists P  a monotonically increasing
                                                             P      link function f : R → [0, 1] bounded in
181   [0, 1] such that P (τ1 , τ2 ) = f ( (s,a)∈τ1 R(s, a) − (s,a)∈τ2 R(s, a)). The popular Bradley-Terry
182   model, considered in many recent works, uses the sigmoid function as the link function. Our method
183   requires no specific knowledge of the link function in this work.
184
185
186   3.3   O FFLINE PBRL S ETTING
187
188   This work studies PBRL in the standard offline setting Zhan et al. (2023). Here, the environment is
189   still characterized by an MDP M = {S, A, R, D}, but the learner cannot directly interact with the
190   environment or observe any reward signals from R. The learner knows the state and action spaces
191
      S, A and can access a pre-collected dataset D of preference feedback. The dataset D contains N
      tuples of data, and each tuple consists of a pair of trajectories and a preference label. The preference
192
      labels in the dataset are generated by a preference model P unknown to the learning agent. For each
193
      pair of trajectory (τ1 , τ2 ), the preference model randomly generates a preference signals σ ∈ {1, 2}
194   through a Bernoulli trial with a probability p = P (τ1 , τ2 ). The preference signal σ represents
195   the index of the preferred trajectory. For convenience, we call the preferred trajectory ‘the chosen
196   trajectory’ and the other trajectory ‘the rejected trajectory.’ Wang et al. (2023) considers a different
197   setting where multiple preference labels are generated for each pair of trajectories, and we will
198   extend our main method to this setting in the experiments. Note that the preference model P is
199   determined by a link function f and the reward function R of the environment, and both should
200   be unknown to the learner in principle. It is common among recent works that assume f to be a
201   sigmoid function, but in this work, our method does not require the knowledge of f . At a high level,
202
      the agent’s goal is to reliably learn a policy from the preference dataset D with a high cumulative
      reward based on the underlying reward function R.

      To utilize existing efficient reward-based RL algorithms for offline PBRL, we consider an informa-
210   tion translation approach that is applicable to universal reward-based RL algorithms. By assigning
211   a reward label for each state-action in the dataset, any reward-based RL algorithm can be trained on
212   the new dataset with the reward labels. Therefore, our task is to find the reward labels that contain
213   the closest information to the preference labels. Note that it is a common approach to train a reward
214   model (also known as reward modeling) and then use the reward model to generate the reward labels
215   Christiano et al. (2017), but this is not necessary in our case as we only need to assign reward labels
      to the state-actions in the dataset.

      Next, we discuss the general case where the same state action appears in different chosen and re-
      jected trajectories. As mentioned earlier, a common approach is to approximate the solution through
280
      reward modeling with a deep neural network. Alternatively, one can still apply the binary labeling
281
      technique from Lemma 4.2, and the labels for the repeated state actions in effect are the mean of the
282   binary rewards. This is only a sub-optimal solution. However, in Section 5, our empirical results
283   show that such a simple binary labeling method is efficient in both overlapped and no overlap cases
284   and is generally more efficient than the reward modeling method. Note that another advantage of
285   BRL compared to reward modeling is that it does not require learning a reward model.

353   For the construction of the preference dataset, we sample pairs of trajectories from the offline RL

      the process as follows.

         is set to be 20 steps, which is similar to previous studies (Christiano et al., 2017).

534
535   In this work, we propose a framework BRL that bridges the gap between offline PBRL and standard
536   RL. We show that one can easily achieve high learning efficiency on PBRL problems by combining
537   BRL with any efficient standard offline RL algorithm. Our framework is limited to the typical
538   offline learning setting and does not answer the question of which trajectories are more worthy of
539   receiving preference labels. Our experimental evaluation is limited to continuous control problems,
      the standard benchmark in RL studies.
