009                                                A BSTRACT
010
011
               Interpretability methods seek to understand language model representations, yet
012            the outputs of most such methods—circuits, vectors, scalars—are not immediately
013            human-interpretable. In response, we introduce L ATENT QA, the task of answering
014            open-ended questions about model activations in natural language. Towards solving
015            L ATENT QA, we propose Latent Interpretation Tuning (L IT), which finetunes a
016            decoder LLM on a dataset of activations and associated question-answer pairs,
017            similar to how visual instruction tuning trains on question-answer pairs associated
018            with images. We use the decoder for diverse reading applications, such as extracting
019            relational knowledge from representations or uncovering system prompts governing
020
               model behavior. Our decoder also specifies a differentiable loss that we use to
               control models, such as debiasing models on stereotyped sentences and controlling
021
               the sentiment of generations. Finally, we extend L ATENT QA to reveal harmful
022
               model capabilities, such as generating recipes for bioweapons and code for hacking.

026   1    I NTRODUCTION
027
028   Understanding the latent representations of large language models (LLMs) improves reliability (Gan-
029   delsman et al., 2023), performance (Yang et al., 2023), auditing (Jones et al., 2023), regulation (Li
030   et al., 2024b), and safety (Hendrycks et al., 2021). Because representations causally impact LLM
031   outputs (Hendel et al., 2023; Todd et al., 2023), better interpretability techniques also improve control-
032   lability (Anthropic, 2024). Most interpretability techniques aim to understand LLM representations
033
      by mapping the latent space to a more human-interpretable one (Singh et al., 2024). Unfortunately,
      the spaces they map to are often inherently opaque—scalars (Zou et al., 2023), single tokens (nostal-
034
      gebraist, 2020), circuits (Wang et al., 2022), or activations (Cunningham et al., 2023). Consequently,
035
      these techniques require significant effort to be useable by practitioners (Lieberum et al., 2024).
036
037   An alternative approach is to read from and write to the latent space in natural language. Inspired by
038   VisualQA (Antol et al., 2015), we consider the task of L ATENT QA, open-ended question answering
039
      (QA) about latents, i.e., model activations, in natural language. A L ATENT QA system accepts
      as input an activation along with any natural language question about the activation and returns a
040
      natural language answer as output. For example, the system might accept LLM activations on a
041
      user biography along with the question “What biases does the LLM have of the user?” and return
042   its response as output. Such systems are valuable for both interpretability, as they can ‘caption’
043   activations (e.g., “[Activation] has gender bias”), and controllability, as they can steer activations
044   with gradients from a loss function described in natural language (e.g., we can reduce bias by
045   minimizing the loss of “Q: Is [Activation] biased? A: No” over [Activation]). In this work,
046   we train a model to perform L ATENT QA, building on and improving over all pre-existing L ATENT QA
047   systems, i.e., those in Ghandeharioun et al. (2024a) and Chen et al. (2024a).
048   Towards solving L ATENT QA, we develop Latent Interpretation Tuning (L IT), which finetunes a
049   “decoder” LLM on a paired dataset of activations and natural language labels. The decoder is trained
050   to predict qualitative properties of future model completions given the activations from the current
051   prompt; this helps reveal model tendencies (e.g., stereotypes or stylistic choices) before those effects
052   become apparent in the output. More specifically, as shown in Figure 1, we curate L ATENT QA data
053   by prompting a target LLM with an instruction (the control) prepended to a prompt (the stimulus),
      capturing activations from the stimulus, and describing properties of the model completions as

070   We assess our decoder’s ability to perform L ATENT QA in two settings (Section 5.1). First, we validate
071   our decoder on the previously studied task of latent attribute extraction (Hernandez et al., 2023),
072   whose goal is to answer relational questions about a subject given the LLM’s latent representation of
073
      the subject. This is a special case of L ATENT QA, and we show that our method improves over all
      pre-existing L ATENT QA systems by a minimum average absolute accuracy of 47.9% across 6 tasks.
074
      Second, we test the decoder’s ability to uncover personas given to the target model in a hidden system
075
      prompt. Given only the activations of the user message, L IT achieves a 7% absolute improvement
076   over prompting, which is given both the user message and model response, and an 82% absolute
077   improvement over all pre-existing L ATENT QA systems.
078
079
      We measure our decoder’s efficacy to control LLMs in three settings. First, we consider a debiasing
      task, where the goal is to minimize the impact of stereotypes on the log-likelihood of models (Nangia
080
      et al., 2020). We find that L IT is the only technique which reduces bias by a statistically significant
081
      amount. Second, we examine controllable sentiment generation (Liu et al., 2021). We find that L IT
082   outperforms standard controllable generation methods by an average absolute performance of 41%
083   and is comparable to methods trained on task-specific demonstrations. Finally, we extend L IT to
084   audit LLM capabilities, specifically eliciting harmful knowledge from safety-tuned LLMs (Guest
085   et al., 2024; Phuong et al., 2024). Without any task-specific finetuning data, our decoder is able to
086   induce the target LLM to provide harmful responses to benign requests, suggesting L ATENT QA can
087   reduce the need for careful prompting (Kojima et al., 2022) or post-training enhancements (METR,
088   2024) when eliciting capabilities.
089   Looking forward, we present L ATENT QA as a novel direction for studying LLM representations.
090   One focus of interpretability has been characterizing models along different units of analysis, e.g.,
091   neurons, circuits, attention heads, etc. In contrast to these approaches, whose task gets more difficult
092   as models scale, L IT benefits from both dataset and model scaling (Section 5.3). Moving forward,
093   we are excited by training L ATENT QA systems on additional types of data, such as hierarchical
094   instruction-following (Wallace et al., 2024). With appropriate data curation, L ATENT QA systems
095   may handle applications such as mitigating hallucinations and improving long-horizon memory.

      Decoding model representations. Many prior works investigate affordances for understanding LLM
101
      activations, including with linear probes (Alain & Bengio, 2016; Belinkov, 2022; Li et al., 2021; Her-
102   nandez et al., 2023; Feng et al., 2024), statistical methods (Zou et al., 2023), autoencoders (Makhzani
103   & Frey, 2013; Cunningham et al., 2023), and even custom dashboards (Viégas & Wattenberg, 2023;
104   Chen et al., 2024b). These methods are limited to a pre-determined set of concepts and thus cannot
105   be used to answer open-ended questions about latents. Other works (nostalgebraist, 2020; Pal et al.,
106   2023; Belrose et al., 2023; Hernandez et al., 2024) exploit LLMs’ ability for next-token prediction
107   to understand their hidden states. However, these works only generate a few output tokens as an
      explanation, limiting their usage when decoding complex model behaviors.

121   Inspired by these limitations, recent works such as SelfIE (Chen et al., 2024a) and
122   Patchscopes (Ghandeharioun et al., 2024a) directly patch LLM activations into a copy of the
123   LLM and leverage the LLM’s ability to decode its activations to perform L ATENT QA. However,
124
      since there is a shift between the distribution of an LLM’s embeddings and the distribution of its
      latents, these methods are often brittle. By training a decoder via a captioned latent dataset, L IT
125
      mitigates this distribution shift and obtains a more robust L ATENT QA system.
126
127   Controlling model behaviors. A common paradigm for controlling models is supervised finetun-
128   ing (Ouyang et al., 2022) or reinforcement learning (Stiennon et al., 2020; Rafailov et al., 2023) on
129   (prompt, completion) pairs. However, these methods demand lack fine-grained control of model
130
      internals. Another line of work modifies model latents for editing knowledge (Meng et al., 2022;
      Mitchell et al., 2022; Meng et al., 2023; Li et al., 2024b) or behaviors (Zou et al., 2023; Turner et al.,
131
      2023), with several methods focusing on improving truthfulness (Li et al., 2024a).
132
133   Curating datasets for instruction-tuning. Instruction tuning is one of the key steps in the post-
134   training pipeline of large language models (Ouyang et al., 2022). Works such as Alpaca (Taori et al.,
135   2023), Vicuna (Chiang et al., 2023), and GPT-4-LLM (Peng et al., 2023) use machine-generated high-
136
      quality instruction-following samples to improve LLM’s ability, reporting impressive performance.
      An illuminating direction is Visual Instruction Tuning (Liu et al., 2023), which designs a pipeline that
137
      uses ChatGPT/GPT-4 to convert image-text pairs into an appropriate instruction-following dataset
138
      for VisualQA.Our work draws inspiration from Liu et al. (2023) by providing a similar pipeline that
139   converts instruction-query pairs into a dataset for LatentQA.

142   3    C URATING L ATENT QA DATA
143
144   We first describe our task setting, which motivates the structure of our dataset and three key design
145   decisions. Afterwards, we detail our implementation.
146   Task setting. Our goal is to train a system to perform L ATENT QA. Although L ATENT QA has a
147   variety of applications, in this work we focus on predicting qualitative properties about the model’s
148   future completion given activations from the current prompt. This setup enables our system to directly
149   understand and steer model tendencies before they show up in the output. For example, we can
150   predict how the model’s belief of the user’s gender biases its responses (Sharma et al., 2023) and alter
151   its response by modifying the belief (Chen et al., 2024b).
152   Dataset. Similar to other forms of instruction tuning (Taori et al., 2023; Liu et al., 2023), we collect a
153   labeled dataset with demonstrations of the desired behavior. In particular, we map activations from a
154   stimulus prompt to QA pairs about the qualitative properties of the target LLM’s completions. In
155   practice, we observe that most prompts do not induce any notable qualitative behavior in model
156   completions. For example, the prompt “What color is the sky?” will lead to a completion in the style
157   of the model’s default persona. We instead prepend a control prompt to each stimulus prompt to
158
      generate completions with a diverse set of qualitative behavior. We use another language model, e.g.,
      GPT, to generate the QAs about the qualitative properties of the completions.
159
160   This process yields triples of (prompt = control + stimulus, completion, QA). For example, given the
161   (prompt, completion) pair (“Imagine you are a pirate. What color is the sky?”, “It be blue, matey”),
      we might write “Q: How will the assistant speak? A: Like a pirate”. To produce the latents, we capture

189   In our early experiments, we find that the decoder often does not generalize when trained on a naively-
190   constructed L ATENT QA dataset. We identify three design decisions important for generalization:
191
      Design decision 1: activation masking. If we include activations from the entire prompt =
192   control + stimulus, the decoder may shortcut the task by reading the token embeddings of the control
193   from the residual stream. We mitigate this issue by sometimes masking the activations from the
194   control, i.e., providing activations of only the stimulus. Because the stimulus tokens attend to the
195   control tokens, the stimulus activations retain some signal from the control.
196
      Design decision 2: data augmentation. To enable our L ATENT QA system to handle a variety
197
      of inputs and tasks, we train on three types of L ATENT QA data: control, stimulus, and stimulus +
198   completion. When the decoder is trained on control data, it learns to decode qualitative properties
199   specified in the prompt itself. When trained on stimulus and stimulus + completion data, it learns to
200   predict qualitative properties contained in the activations. Also, both control and stimulus contain
201   activations from only prompts, whereas stimulus + completion contain activations from (prompt,
202   completion) pairs. Taken together, these three data types provide coverage for all L ATENT QA tasks
203   we evaluate on in this work.
204   Design decision 3: improving the faithfulness of the completion. If we naively use “Imagine you
205   are [control],” as our control prompt, we find that the model is not always faithful to its instructions.
206   One approach to improving the faithfulness is to emphasize the control; in particular, faithfulness
207   improves using the control prompt “Base your answers on my instructions. Imagine you are a
208   [control]. In all your responses, imbue your responses with as much [properties of the control] as
209   possible.” However, we opt for a more robust approach of using a more capable LLM to generate the
210   (prompt = control + stimulus, completion) triples.

213   its context), goals (instructing the model to adopt the given goal), and personas (instructing the model
214   to behave like the given persona). For a given type of control (e.g., goals), we prompt OpenAI’s
215   o1-preview (OpenAI, 2024b) to create the data in three steps. First, we generate several thousand
      examples of the control (e.g., “Make your next sentence contain alliteration”). Second, we expand

235   4       L ATENT I NTERPRETATION T UNING
236
237   We next present Latent Interpretation Tuning (LIT), an algorithm for learning a decoder to solve
238
      L ATENT QA. We then show how to apply this decoder for both reading (Figure 4) and control
      (Figure 5). Given the L ATENT QA dataset in Section 3, LIT outlines a procedure for finetuning a
239
      decoder LLM on the data.
240
241   Training the decoder. At a high level, we train our decoder by patching in activations from the target
242   LLM and finetuning it to predict the answer given the question (Figure 1). Specifically, given a data
243   point (prompt = control + stimulus, completion, question-answer) from our dataset, we train the
244
      decoder to maximize the logprob of the answer given the pseudo-string “[Act] + question”. Here,
      [Act] are the target LLM’s activations from layer k captured on one of the three data types described
245
      in Figure 3. To evaluate the decoder’s logprob of [Act] + question + answer, we patch [Act] into
246
      layer ℓ of the decoder.1
247
248   In our experiments, we use the Llama-3-8B-Instruct (Dubey et al., 2024) and Ministral-8B-Instruct-
249   2410 (Mistral, 2024) as our target LLMs. For each target LLM, we train a decoder LLM, which
250
      is initialized as a copy of the target LLM. To identify the layer k to read activations from and the
      layer ℓ to write activations to, we run an ablation detailed in Appendix B.1 and select k = 15 and
251
      ℓ = 0. Intuitively, this result is sensible: we read from the middle layers because they contain the
252
      most semantically-rich representations (Ghandeharioun et al., 2024b) and we write to the 0th layer
253   because we want to provide our decoder with as many steps for processing the activation as possible.
254   Furthermore, the decoder is trained, so it learns to handle the distribution shift from layer k = 15 to
255   layer ℓ = 0. We train our decoder with a LoRA as detailed in Appendix B.1.
256

      5.1     R EADING
292
293   We evaluate the effectiveness of our decoder for L ATENT QA in two settings. First, we consider
294   a task previously studied in Hernandez et al. (2024): extracting relational information from latent
295   representations, which is a special case of L ATENT QA. Second, we consider a novel application of
296   L ATENT QA: uncovering hidden system prompts given a user-model dialog. This task evaluates the
297   decoder’s ability to predict future model behavior given current model activations, which may be
298
      useful for robustly detecting and, consequently, auditing aberrant model behavior (Roose, 2023). See
      Appendix C for full experimental details.
299
300   Extracting information from representations. In this task, given an embedding of a subject
301   (“LeBron James”), our goal is to answer relational questions about the subject (“What sport does
302   this athelete play?”). Each question is a fact of the form (phrase containing subject, relation, object),
303   such as (the World Cup winner Italy, currency, Euro). For each triplet, the model receives the
304
      [Activations] from the phrase containing the subject and the relation as a natural language
      question and should answer with the corresponding object. Our setup follows Ghandeharioun et al.
305
      (2024a), with the questions from Hernandez et al. (2024) and the subject phrases from WikiText-
306
      103 (Merity et al., 2016).
307
308   For our decoder, given a (subject, relation) pair as input, we call I NTERPRET([Act], relation) and
309   generate at most 20 tokens. For example, for the relation "Country - currency", we call I NTER -
      PRET([Act], What is the currency of the country?). We compare to Patchscope (Ghandeharioun
310
      et al., 2024a), one of two pre-existing L ATENT QA systems. Patchscope operates similarly to
311
      our decoder, except that it directly patches in the activations of the subject into the relation. E.g.,
312
      for the task “Country currency” we run the model on “The official currency of [Act]” (where
313   the phrase’s activations are patched into [Act]) to generate at most 20 tokens. We also evaluate
314   against linear probing, a trained baseline that requires task-specific data, taking the results directly
315   from Ghandeharioun et al. (2024a).

504
505    One promise of training-based approaches to L ATENT QA is the benefit of scale. In this section, we
506    demonstrate how our decoder improves with increasing dataset size and increasing model size.
507    Scaling model size and dataset size. To assess the quality of a given L ATENT QA system, we
508    curate an evaluation dataset, following the dataset curation procedure outlined in Section 3. After
509    deduplicating controls that appear in the train set, we end up with an evaluation dataset of 500 total
510    controls split roughly even along extractive QA, goals, and personas.
511
       We perform the same training procedure as detailed in Section 4 to run our experiments. We measure
512
       the effect of model size on L ATENT QA performance by using 1B, 3B, 8B, and 70B parameter
513    variants of Llama-3. Note that we scale both the target LLM and decoder LLM size, as the decoder is
514    always initialized as a copy of the target LLM. We measure the effect of dataset size on L ATENT QA
515    performance by creating fractional train datasets from our original train dataset in Section 3. We split
516    the dataset by control (e.g., extractive QA, goal, or persona) and sample either 1/4 or 1/2 of the data
517    to obtain the 1/4 and 1/2 train datasets, respectively.
518
       We report the effect of scaling model size in Figure 14 and dataset size in Figure 15. Figure 14
519    suggests that future L ATENT QA systems built on larger models will be more performant. Finally,
520    Figure 15 suggests a scalable direction for improving L ATENT QA systems: curating more training
521    data. Taken together, these results suggest that L IT will straightforwardly improve with scale,
522    strengthening the promise of L ATENT QA as a novel affordance for interacting with model internals.

       6            D ISCUSSION
525
526
       We propose to study the task of L ATENT QA, answering open-ended questions about model latents in
527
       natural language. To design a L ATENT QA system, we curate a dataset of activations labeled with
528    question-answer pairs in natural language and finetune a decoder LLM on this dataset. In particular,
529    we train the decoder to predict qualitative properties of future model completions given activations
530    from the current prompt. This enables us to read diverse information from LLM activations. Moreover,
531    we use the same decoder to debias models, control the sentiment of generations, and elict harmful
532    capabilities, outperforming baselines such as RepE and prompting. We view L IT as the first attempt
533    at training a L ATENT QA system, and we are excited by the potential for future extensions.
534    Limitations. We discuss three potential limitations. First, our training data may lack diversity.
535    Because we only collect three types of controls (extractive QA, goals, and personas), we may lack
536    some types of L ATENT QA helpful for training. Second, model interpretation and human interpretation
537    of latents may be misaligned. For example, models may have different operational definitions of
538    prompts than humans do, or even encode biases in their representations. L ATENT QA would not be
539    able to mitigate these issues, as they are fundamental to the training data. Third, we run the risk of
       training the decoder to hallucinate, as it is training on activations which lack ground truth labels.

540
      Ethics statement. We raise two potential concerns with our work. First, because we train on
541   synthetically generated data, we may teach biases and falsehoods to our decoder model. Additionally,
542   although we filtered out any personas generated from GPT-4 that were overtly discriminatory (through
543   keyword search), we were not able to review all of the personas. There may be subtle stereotypes
544   propagated in the personas. Training a decoder on this system would then result in the control being
545   biased. For future L ATENT QA systems, it is important to develop a pipeline to verify the integrity
546   and the fairness of the data being trained on.
547   Reproducibility statement. We have specified all the required details to reproduce our experiments
548   in Appendicies C and D. Moreover, for each experiment, the first two paragraphs detail our setup and
549   method. Finally, Section 3 details our dataset curation process. We will also release our dataset and
550   training code publicly after the anonymity period.

      In this section we describe our dataset generation pipeline (Figure 2) in more detail. For the train
759   dataset, we have three categories of seed controls: extractive QA, goals, and personas. For each
760   category, we follow the following procedure:
761
762         1. We first generate a list of seed controls by hand.
763         2. We then brainstorm more seed controls with o1-preview and deduplicate identical or seman-
764            tically similar controls.
765         3. We ask o1-preview to expand each seed control into a dialog.
766
            4. We then ask o1-preview to generate description-based and reasoning-based QA pairs about
767
               the dialog.

1228   Our decoder is trained with a LoRA (Hu et al., 2021) of rank 32, alpha 64 on both the attention and

1231   To identify the layer k to read from and the layer ℓ to write to, we conduct a hyperparameter sweep.
1232   We perform the LoRA training procedure outlined above with the train dataset from Section 3.
1233   Moreover, we calculate the lowest test loss on the evaluation dataset described in Section 5.3 and
1234   report results in Table 3. We find that the k = 15 and ℓ = 0 has the best generalization, and select it

1237   B.2    R EADING D ETAILS
1238
1239   We perform reading in two steps. First, given a stimulus prompt we would like to read
1240   [Activations] from, we perform a forward pass on the target LLM and capture the
1241   [Activations] from layer k. Second, given a desired question, for each forward pass on the de-

1242
       of tokens. At layer ℓ, we substitute the activations corresponding to ??? with the [Activations].
1243   Our reading runs on 1× A100.

       B.3    C ONTROL D ETAILS
1246
1247   As described in Section 4, we perform control with our decoder by specifying the control as a
1248   question-answer pair. For example, if we want to steer the model to speak like a pirate, we could
1249   write “Q: How will the model speak? A: Like a pirate.”
1250
       Given a stimulus prompt, the decoder specifies a loss on the [Act] of the stimulus. Specifically,
1251
       we calculate the cross-entropy loss of answer (“Like a pirate.”) given the input [Act] + question.
1252
       Then S TEER([act], question-answer) is the resulting gradient on [Act] from this loss. Our stimulus
1253   prompts are instructions sampled from Databricks’ Dolly instruction-tuning dataset (Conover et al.,
1254   2023).
1255
1256
       Instead of using the loss to update the [Act], we further backpropagate the loss to the target LLM’s
       parameters, as this greatly reduces inference costs. To do so, we attach a LoRA (Hu et al., 2021) to
1257
       the target LLM’s parameters at layers 0, . . . , k − 1, k. The LoRA has rank 8, alpha 16, learning rate
1258
       10−4 , and is on both the attention + MLP modules.
1259
1260   We have two approaches to updating the target LLM’s LoRA parameters. The first is to update using
1261   the gradient S TEER([act], control) where activations are read from layer k, which generates one
1262
       gradient update for each [Act]-control pair. The second is to update using the gradient S TEER([act],
       control) where activations are ready sequentially from layer 0, . . . , k − 1, k, which generates k
1263
       gradient updates for each [Act]-control pair. Note that this requires the decoder to generalize, as it is
1264
       only trained to read from layer k. We use the second approach as it is more performant in practice.
1265   Our training runs on 1× A100.
