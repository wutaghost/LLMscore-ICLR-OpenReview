                                                   A BSTRACT
008
009
010
               Large language models (LLMs) have demonstrated remarkable progress in gen-
               erating high-quality natural language through extensive pre-training over Trans-
011
               former architectures. However, the quadratic complexity of transformers in se-
012
               quence computation greatly limits their capability to efficiently model long se-
013            quences. In this paper, we introduce S PARSE T RAINING, a simple training tech-
014            nique to optimize the complexity of Transformer models in long-sequence gener-
015            alization. Specifically, in S PARSE T RAINING, the input sequences of the Trans-
016            former network are segmented into two distinct components: the memory part and
017            the target part. The target part adheres to the standard next-token prediction for
018            modeling continuous sequences, while the memory part, sampled from longer se-
019            quences, serves as the conditional context for the prediction of the target part. To
020            build the memory part, we apply a sparse sampling policy that decays with the
021
               distance from the target part, to obtain tokens and preserve their positions. With-
               out any architectural modifications, our method can extend existing Transformer-
022
               based LLMs to capture long-range dependencies within a fixed window size dur-
023
               ing the training. Experimental results on multiple datasets also demonstrate the
024            effectiveness and efficiency of S PARSE T RAINING to mitigate the complexity of
025            the Transformer network in building long-sequence dependency.

028   1   I NTRODUCTION
029
030   With the aid of large-scale pre-training techniques (Kaplan et al., 2020; Ouyang et al., 2022) on
031   the Transformer models (Vaswani et al., 2017), large language models (LLMs) (OpenAI, 2023;
032   Touvron et al., 2023a;b; Team, 2024a; Jiang et al., 2023; Team & Google, 2023; Team, 2024b)
033
      have recently achieved incredible progress in solving massive natural language processing (NLP)
      tasks (e.g., generation, reasoning, translation, etc). Despite these remarkable advancements, the
034
      inherent issue of quadratic complexity in the Transformer networks severely limits their capability
035
      to extend long-sequence modeling, drawing enormous attention from both the industry and academia
036   to address this critical issue.
037
038   Generally, many efforts have been devoted to generalizing the context windows of LLMs beyond
039
      their pre-training settings. Among these works, some researchers attempted to develop sparse ar-
      chitectures (Child et al., 2019; Beltagy et al., 2020; Zaheer et al., 2020; Choromanski et al., 2021;
040
      Tay et al., 2023; Han et al., 2024; Xiao et al., 2024) to reduce the quadratic complexity of Trans-
041
      former network during the training phase. However, these architectures involve sparse patterns and
042   limit their scalability to fall behind the original ones. Therefore, further works continue to explore
043   how to extend existing LLMs to support long-sequence dependency. To this end, some papers (e.g.,
044   RoPE (Su et al., 2024), ALiBi (Press et al., 2022), LEX-Transformer (Sun et al., 2023b)) point out
045   that good positional information plays an important role in enabling length extrapolation. On the
046   basis of these, some papers (e.g., PI (Chen et al., 2023), Yarn (Peng et al., 2024)) extend positional
047   information to enlarge context windows via interpolation. Although these works offer a solid ini-
048   tialization for modeling positional information in long sequences, they still experience performance
049   deterioration without any fine-tuning. How to devise an efficient training method to extend the
050
      context window of existing LLMs still remains an ongoing challenge.
051   In this paper, inspired by previous experiences (Child et al., 2019; Beltagy et al., 2020; Zaheer et al.,
052   2020; Choromanski et al., 2021; Tay et al., 2023; Han et al., 2024; Xiao et al., 2024), we observe and
053   analyze the phenomenon of attention sparsity, particularly in long-sequence modeling, and further
      attribute it as “Pareto Principle of Transformers”. That is, only a small subset of tokens dominates

      Is it possible to simulate attention sparsity without modifying the architecture during the training?
076
077   Therefore, in this paper, we introduce S PARSE T RAINING, which aims to extend the context win-
078   dow of existing LLM frameworks by leveraging continual pre-training within a fixed window size.
079   Specifically, we argue that distant tokens generally provide less information for a token prediction
080   compared to tokens that are closer to the target. In other words, most of computations (i.e., dot prod-
081   uct) between distant tokens and the target tokens are redundant. Hence, the core idea behind S PARSE
082   T RAINING is to sample tokens from the distant tokens and simultaneously keep their corresponding
083   positions, and then adopt the standard next-token prediction for the target tokens. This process is
084
      illustrated in Figure 1. More specifically, we divide the input sequences as the memory part and the
      target part. Based on the posterior distribution of attention sparsity, we devise a sampling policy
085
      over the memory part with a decay factor across the distance to collect tokens. That implies tokens
086
      closer to the target part will be sampled at a higher probability while the farther tokens are sampled
087   at a lower probability. This design enables us to replicate the sparsity of long-sequence dependencies
088   at the input level, rather than architecture. Generally, it also offers us three key benefits to model
089   long-sequence dependency: 1) Efficient Long-Sequence Training. By training on the sampled se-
090   quence where the length Lsample < L, our method can reduce the space and time complexity from
091   O(L2 ) to O(L2sample ) when compared with directly training long sequences (Fu et al., 2024) on
092   the Transformer network; 2) Sparsity Simulation. By applying a decay sampling policy across the
093   length, our method also simulates the situation of the attention sparsity in long-sequence modeling;
094   3) Architecture Invariance. Compared with previous sparse architectures, S PARSE T RAINING does
095   not involve any modifications to the architecture, which makes it adaptable to any LLM framework
096   to extend its capability in modeling long-sequence dependency.
097   To verify the effectiveness of S PARSE T RAINING, we conduct extensive experiments on and pub-
098   lic benchmark datasets. Experimental results demonstrate that by deploying S PARSE T RAINING
099   over existing LLM frameworks, it can effectively improve the model’s capability to infer over long
100   contexts. Our contributions can be summarized as follows:
101
102
      • We conduct an in-depth analysis of the statistical attention patterns in Transformers across dif-
        ferent LLMs, and summarize several laws regarding attention distribution, including its sparsity,
103
        weight allocation and decay over distance.
104
105   • Based on our analysis, we propose S PARSE T RAINING, a novel training approach to extend con-
106
        text window size of LLMs, without any modifications in the architectures.
107   • Empirically, we demonstrate the effectiveness of S PARSE T RAINING through extensive experi-
        ments on multiple state-of-the-art LLMs over public benchmarks.

181   2.3     S PARSE ATTENTION IS NOT ALL YOU NEED
182
183   Inspired by the Pareto Principle in Transformers, some works (Xiao et al., 2024; Han et al., 2024;
184   Jiang et al., 2023) explore applying some specific attention patterns to sample tokens for inference.
185
      They attribute attention distributions to two common patterns: sliding window and Λ-shape. The for-

186
      with the close tokens critical to making predictions. However, as shown in Figure 3(a), the middle

188   these tokens may encode crucial information for downstream tasks. Moreover, these works do not
189   adequately extend the capability of LLMs to achieve long-sequence dependency. Therefore, how to
190   generalize existing LLM frameworks to unseen length via training still needs to be addressed.

192   3     S PARSE T RAINING
193
      As mentioned previously, the backbone of most modern LLM frameworks is decoder-only Trans-
194
      former, whose quadratic complexity in computing (Qm H)⊤ (Km H) ∈ RN ×N in equation 1 makes
195
      it inefficient when handling long sequences (large N ). To this end, we believe that an ideal solution
196   to extend the capability of LLMs to generalize long sequences should meet these criteria:
197
198   • It should not introduce any modification over architectures to preserve its architectural integrity;
199   • It should be able to simulate the sparsity of the attention distribution in sequence computations;
200
      • It should effectively reduce the time and space complexity, avoiding quadratic growth.
201
202   Therefore, in this paper, we introduce S PARSE T RAINING, a novel training strategy to extend exist-
203   ing LLMs to support long sequence generalization. The details are described below.
204
205   3.1     F RAMEWORK
206
207   Assume the final part of a long sequence as X = {xm+1 , . . . , xN }, where m starts from a large
208   position (e.g., beyond 4096 in LLaMA-2). The conventional method to establish long-sequence
209   training is to directly calculate the whole sequence from position 1 to N via attention operations
210   (i.e., O(N2 )), while bringing massive and redundant computations. Therefore, we claim that the
211
      core challenge to address the long-context issue is how to bridge the connection between two distant
      tokens. However, considering the sparsity between the distant and the target tokens, we argue that
212
      not all pairwise computations in attention are essential, and some distant tokens could be ignored
213
      for modeling long contexts to simulate sparsity.
214

      means the first few tokens usually occupy a ratio of attention weight.

294   In this section, we also want to discuss why S PARSE T RAINING is effective at processing long-
295
      context information. We attribute its effectiveness from two perspectives as follows:
296
297
      Positional Generalization The critical part of attention operation to capture dependency is
      (Qm H)⊤ (Km H), when Qm and Km have been applied with positional information. Therefore,
298
      the way to enable model to learn positional information beyond the original context window is im-
299
      portant. During pre-training with window size L, the model only accessed the positional encoding
300   of positions (1, . . . , L), and thus cannot be generalized to untrained positional encoding. However,
301   in S PARSE T RAINING, we enable model to access more positions beyond L for optimization.
302

305
306   Training Mismatch Another issue of S PARSE T RAINING is whether it can build next-token pre-
307   diction based on the sampled memory tokens. We deem that S PARSE T RAINING can be considered
308   as a kind of dropout (Srivastava et al., 2014) at the token level, compared with standard training. That
309
      makes it compatible with other LLM training techniques and does not involve any modification at
      the architecture level.

482   5     R ELATED W ORK
483
484   With the rise of advanced LLMs , how to extend the capability of Transformer-based LLMs to gen-
485   eralize across long sequences has become an ongoing challenge. Generally, the current approaches
      to generalize the context window of LLMs can be grouped into two categories, which are as follows:


                                                                              9
      Under review as a conference paper at ICLR 2025


486
      Efficient Training with Sparse Architectures The standard complexity of Transformer networks
487   is known to scale as O(L2 ). To alleviate the burden of quadratic complexity, many research ef-
488   forts (Tay et al., 2023) have focused on developing advanced or sparse architectures to effectively
489   approximate the attention mechanism. Specifically, some works like Sparse Transformer (Child
490   et al.,
           √ 2019) apply sparse factorization to the attention matrix, thus reduce the complexity to
491   O(L L). Some other works (e.g., Linformer (Wang et al., 2020) and Performer Choromanski
492   et al. (2021)) attempt to approximate the self-attention matrix via low-rank decomposition. Besides,
493   some works (e.g., Reformer (Kitaev et al., 2020), Block-wise Self-Attention (Qiu et al., 2020),
494   LongFormer (Beltagy et al., 2020), Big Bird (Zaheer et al., 2020), LongNet (Ding et al., 2023))
495   propose some fixed sparse attention patterns to reduce time complexity. Recently, some papers have
496   attempted to develop parallelized RNN to address this problem, like Mamba (Gu & Dao, 2023),
497
      RWKV (Peng et al., 2023) and RetNet (Sun et al., 2023a). In order to extend the context window
      of Transformer, several methods explore the use of hybrid window-full attention for training, that
498
      means some layers adopt full attention while other use sparse attention patterns. For example, Long
499
      Llama (Tworkowski et al., 2023) uses the bottom layers to retrieve the most relevant top-k tokens,
500   then performs attention operations on these tokens to reduce computational complexity. However,
501   the scalability and capability of these works are still beneath fully attention architectures, and thus
502   most mainstream LLM frameworks still adopt standard Transformer architecture (i.e., full atten-
503   tion) as the backbone network. Compared with these works, S PARSE T RAINING does not involve
504   any modifications over architectures but simulates sparsity at the input-level. Therefore, it can also
505   be considered as a post-training technique that can be adopted to existing LLM frameworks, and
506   maintain the complexity within a fixed window size.
507
508   Extend Context Window with Length Extrapolation Instead of directly using sparse architec-
509
      ture, a large amount of research focuses on inferring unseen length beyond the pre-training window
      size based on the original Transformer network. These works can be considered as a kind of posi-
510
      tion engineering (Zhao et al., 2023). Among these works, RoPE (Su et al., 2024) and Alibi (Press
511
      et al., 2022) are the most representative ones. These works can effectively encode relative posi-
512   tional information without any learnable parameters, allowing for length extrapolation. Building
513   on this, some other works (CAPE (Likhomanenko et al., 2021), SANDWICH (Chi et al., 2023),
514   xPOS (Sun et al., 2023b), LongRoPE (Ding et al., 2024), NoPE (Kazemnejad et al., 2023), FIRE (Li
515   et al., 2024), and CLEX (Chen et al., 2024)) also extend different positional encoding. However,
516   as models have not been generalized to unseen positions through training, these works still suffer
517   from performance degradation. Therefore, some works propose position interpolation, that re-scales
518   the out-of-distribution positional encoding within the pre-trained window size (Chen et al., 2023).
519   YaRN (Peng et al., 2024) leverages neural tangent kernel (NTK) to interpolate RoPE and generalize
520
      LLaMA-2 to support 128K tokens. Besides, a similar work (Ruoss et al., 2023) introduces to ran-
      domly sample some tokens to extend length generalization but ignores the sparsity when modeling
521
      long-sequence dependency. Generally, our method is orthogonal to these method as we aim to gen-
522
      eralize the long-sequence capability of LLMs from the training level. S PARSE T RAINING can also
523   use these advanced positional embeddings to encode long sequences, while in this paper, we mainly
524   use RoPE as the backbone for experiments.
525

527
      6   C ONCLUSION
528
529
      In this paper, we present a novel training framework that can efficiently extend the context window
      of LLM frameworks based on the Transformer architecture, named S PARSE T RAINING. Specifi-
530
      cally, we first analyze statistical laws of existing attention patterns and identify the phenomenon of
531
      “Pareto Principle of Transformer”. Based on these observations, we introduce S PARSE T RAINING,
532   which employs a sampling policy with a decay factor across the distance to gather tokens as the con-
533   ditional part for long-sequence prediction. Based on the sampled tokens with their corresponding
534   positions, we can directly adopt the standard next-token prediction for the long sequences. Benefit-
535   ing from such a design, our method can effectively extend the context window of LLM frameworks
536   within a fixed window training. In addition, compared with previous sparse architectures, S PARSE
537   T RAINING will not introduce any modification over the architecture, and can also simulate the at-
538   tention sparsity at the input level. Experimental results also demonstrate the effectiveness of our
539   proposed method in processing long-sequence dependency.

894   A.3    U SE T OP -K ATTENTION IN GPT2 INFERENCE STEP
895
896   We have visualized the trend between the number of Top-K highest attention value used during
897
      the inference step of GPT2 and the value of two key performance metric, perplexity and accuracy,
      across multiple datasets from LongBench dataset (Bai et al., 2024). The results are illustrated in
898
      Figure 5. Despite the difference in values of perplexity and accuracy across different datasets, these
899
      figures still reveal a very clear and consistent trend: as the Top-K value increases, perplexity initially
900   drops quickly, while accuracy sharply rises, before both metrics stabilize at higher K values. For
901   all datasets, at very low K values (under 50), perplexity is high and accuracy is low, indicating that
902   the model performs poorly due to limited access to relevant information. However, as K increases,
903   perplexity undergoes a rapid decline, and accuracy improves sharply. In fact, the most significant
904   changes in both metrics occur within the first few hundred K values. This trend suggests that a
905   relatively small number of key-value pairs provide the majority of useful context for the model’s
906   predictions. Once K reaches around 500, the accuracy curve flattens and the perplexity plateaus, in-
907   dicating that further increases in K yield limited improvement in the performance. The stabilization
908
      of both perplexity and accuracy across datasets highlights an underlying pattern of attention mecha-
      nisms: a limited number of Top-K weights is sufficient to capture most relevant information, making
909
      larger Top-K values computationally unnecessary beyond a threshold of approximately 500. A mod-
910
      est number of high-scoring key-value pairs capture the majority of relevant information needed for
911   effective language modeling. After this optimal range is reached, further increases in K yield no sig-
912   nificant improvements in either perplexity or accuracy, implying that the model has already captured
913   the essential context.

1084   multi-sentence documents. It includes both unfiltered and web-filtered versions, supporting various
1085   QA tasks.

       PassageCount. PassageCount seeks to create a more demanding situation where the model is
1088
       required to utilize the full context to resolve the task. Each piece of data was generated by randomly
1089
       selecting several passages from English Wikipedia, repeating each paragraph at random several
1090   times, and finally shuffling the paragraphs.

1093   PassageRetrieval. The PassageRetrieval dataset in LongBench is a synthetic task designed to eval-
1094   uate a model’s ability to retrieve specific passages. For each entry, 30 passages are sampled, and
1095   one is summarized using GPT-3.5-Turbo. The task challenges models to identify the original pas-
1096
       sage that matches the generated summary, testing long-context understanding and passage retrieval
       capabilities.

1099   LCC. The Microsoft LCC (Long Code Completion)17 dataset is designed for code completion
1100   tasks and is available in multiple programming languages, including Python, Java, and C#. It is part
1101   of a series of datasets aimed at evaluating the ability of machine learning models to predict the next
1102   line of code in long programming contexts. The dataset is split into training and test sets, making it
1103   useful for training models like transformers for code generation or code completion tasks.
1104
1105
       RepoBench-P. RepoBench-P (Pipeline)18 is a part of the RepoBench dataset, which is designed to
1106   evaluate repository-level code auto-completion systems. It combines two tasks: code retrieval and
1107   code completion. First, the model retrieves the most relevant code snippet from another file (cross-
1108   file context), and then it predicts the next line of code based on that retrieved context. RepoBench-P
1109   is particularly useful for assessing the performance of models in real-world multi-file programming
1110   scenarios, where code dependencies span multiple files. The dataset is available for Python and Java.

1283   D     L IMITATIONS
1284
1285   S PARSE T RAINING still has some limitations, which can be summarized as follows: 1) S PARSE
1286
       T RAINING focuses solely on reducing the quadratic complexity of the Transformer network dur-
       ing training, while it still suffers from quadratic complexity during the inference stage. Therefore,
1287
       we may need to combine other inference tricks to address this inherent issue of Transformer; 2)
1288
       Specifically, S PARSE TRAINING enables models to learn more semantic information from unseen
1289   positional information, rather than context information from long sequences. However, we think
1290   that this problem can be alleviated if we can determine which memory part is more important to the
1291   target part, and leave this part as future work.
