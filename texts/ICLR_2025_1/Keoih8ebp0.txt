009                                                A BSTRACT
010
011            The growing demand for real-time applications on edge devices underscores the
012            need for faster inference of complex deep neural network (DNN) models. Al-
013            though mobile devices increasingly incorporate specialized processors like GPUs
014            and TPUs, modern DNN models such as Whisper and Vision Transformers often
015            involve dynamic control flows and tensor operations that are incompatible and un-
016            supported on current frameworks with these mobile accelerators. CPU presents
017            the most viable option to improve inference latency on mobile devices due to their
018
               widespread availability, substantial memory caches, and ability to support all types
               of tensor operations. However, existing CPU optimization techniques focus on se-
019
               quential execution, overlooking potential parallelization within Automatic Speech
020
               Recognition (ASR) and transformer-based models, leading to inefficiencies. This
021            work introduces a novel runtime model analysis pipeline that extracts layer and
022            branch structures from DNN model graphs to identify parallelizable branches. We
023            propose BAP, a branch-aware memory allocation strategy that isolates memory
024            arenas for parallel branches, reducing contention and optimizing memory reuse
025            within each branch. Additionally, we leverage CPU multithreading to execute
026            these branches concurrently, optimizing thread management and memory access
027            to minimize overhead. Evaluated on ASR models and transformer-based models,
028            our approach reduces inference latency by up to 38.5%, decreases memory allo-
029
               cation requirements by up to 15.6× and saves up to 20.2% energy cost compared
               to the TFLite naive memory allocation.

      1   I NTRODUCTION
033
034
      The growing demand for real-time machine learning (ML) applications, such as voice assistants,
035
      live translation, and augmented reality, has intensified the need for faster deep learning inference on
036
      edge devices like smartphones and single-board computers (Yang et al., 2021; Dong et al., 2022; Xu
037   et al., 2023; Rekesh et al., 2023). On-device inference offers significant benefits over cloud-based
038   alternatives, eliminating the need to send data to external servers, which introduces privacy risks and
039   delays. To address the demands of efficient edge inference, frameworks such as TensorFlow Lite
040   (TFLite) have been widely adopted (Abadi et al., 2015). TFLite, for instance, employs quantiza-
041   tion and operator fusion to optimize model size and computational efficiency (Nguyen et al., 2020;
042   Orăşan et al., 2022; Xu et al., 2021; Prasad et al., 2020), while leveraging hardware accelerators such
043   as GPUs, TPUs, and NPUs for static models (Park & Kim, 2023; Lee et al., 2019; Xu et al., 2022).
044   Similarly, MNN enhances inference speed by optimizing kernel execution and aggressively reusing
045
      memory (Jiang et al., 2020). These frameworks have achieved notable success in tasks such as image
      classification (e.g., MobileNet) and object detection (e.g., YOLO), where model structures are static
046
      and predictable. Beyond these frameworks, many studies have further optimized edge inference.
047
      Techniques such as co-execution of CPU and GPU tasks, memory-efficient transformations, and dy-
048   namic model partitioning across heterogeneous processors have made strides toward more efficient
049   inference (Kim et al., 2019; Jeong et al., 2022; Jia et al., 2022; Wang et al., 2018; Wei et al., 2023).
050   However, despite these optimizations, significant challenges arise when deploying models with dy-
051   namic control flows and tensor operations, such as ASR models and transformer-based architectures.
052   These models are incompatible with hardware accelerators optimized for static workloads on exist-
053   ing mobile inference frameworks, making traditional optimization techniques ineffective (Cordesius
      et al., 2021). Furthermore, existing solutions often require extensive model refactoring or retraining,

054
      which is impractical for large pre-trained models and can degrade performance during deployment
055   (Kim et al., 2019; Wei et al., 2023). This highlights the need for new approaches that can handle the
056   unique challenges posed by these models on off-the-shelf edge devices.

058
      CPUs remain the most practical option for executing ML models on edge devices due to their flexi-
      bility in handling dynamic workloads and widespread availability (Zeng et al., 2023). Studies show
059
      minimal performance differences between CPUs and GPUs on mobile devices (Zhang et al., 2023;
060
      Wei et al., 2023; Bordawekar et al., 2010), reinforcing the viability of CPUs in this setting. ASR
061   and transformer-based models also bring additional advantages when paired with CPUs. These
062   models naturally lend themselves to parallel execution due to internal structures like multi-head at-
063   tention (Cao et al., 2012; Gulati et al., 2020; Vaswani, 2017). This built-in balance across different
064   computational branches reduces the synchronization overhead typically associated with parallel pro-
065   cessing. CPUs, with their robust multithreading capabilities, are well-equipped to take advantage
066   of this structural balance, distributing tasks efficiently across cores. This not only enhances the per-
067   formance of parallel tasks but also allows CPUs to manage both parallel and sequential operations
068   without the need for significant model modifications. Nevertheless, fully leveraging CPU-based
069
      parallelism presents several challenges: (1) accurately analyzing model graphs to identify opera-
      tions and branches that can be executed in parallel; (2) managing memory allocation in a way that
070
      avoids data dependency conflicts, such as Read-After-Write (RAW) dependencies, where one oper-
071
      ation might overwrite data that another operation still needs; (3) minimizing the overhead of parallel
072   execution, as unbalanced workloads or inefficient thread synchronization can negate the benefits of
073   parallelism; and (4) avoiding significant model refactoring, as this adds complexity and could hinder
074   the deployment process.
075
      To address these challenges, we propose BAP, a novel optimization approach for ASR and trans-
076
      former models on mobile CPUs. BAP dynamically identifies parallelizable branches in model com-
077
      putational graphs via a runtime analysis pipeline. We introduce a branch-aware memory allocation
078   strategy that isolates memory arenas for parallel branches, reducing data conflicts and optimizing
079   cache locality. Leveraging CPU multithreading with optimized thread management and memory ac-
080   cess, our method ensures efficient parallel execution without modifying model structures, meeting
081   the critical need for real-time performance on mobile devices. Evaluated on devices like Dimen-
082   sity, Google Tensor, Kirin-powered Android devices, and Raspberry Pi 4B, BAP achieved a 38.5%
083   reduction in inference latency, up to 15.6× memory allocation improvements over TFLite’s naive
084   allocation, and up to 20.2% energy savings. Our contributions are as follows:

             • We introduce BAP, a CPU-specific optimization system for ASR and transformer-based
086
               models with dynamic control flows and tensor operations. BAP combines branch-aware
087
               memory allocation and multithreading to optimize parallel execution and reduce latency.
088
089          • We develop a branch-aware memory allocation strategy that reduces data contention by iso-
090
               lating memory arenas for parallel branches, enabling safe and efficient parallel execution.
091          • BAP delivers substantial improvements in latency, memory efficiency, and energy con-
092            sumption without requiring model refactoring, supporting practical real-time inference on

095   2   R ELATED W ORK
096
097   Model Optimization Strategies: The demand for optimizing ML inference on edge devices has led
098   to strategies like quantization, operator fusion, and pruning (Yao et al., 2020; Kim et al., 2022; Jiang
099   et al., 2022; Niu et al., 2021). Quantization reduces precision to shrink model size and computational
100   cost, lowering latency and energy consumption for static models. Operator fusion combines multiple
101   operations into a single step to minimize memory access overhead, while pruning removes less
102   essential components to reduce computational load. However, techniques like quantization-aware
103   training and post-training quantization often require access to validation datasets or retraining, which
104
      may not always be feasible (Nguyen et al., 2020). Additionally, these methods can reduce accuracy
      and depend on model-specific tuning, limiting their applicability when models must be deployed
105
      without retraining or extensive infrastructure.
106
107   Another strategy optimizes model structures by replacing computationally expensive operations
      with more efficient alternatives. Models like MobileNet (Howard, 2017) and MobileBERT (Sun


                                                         2
      Under review as a conference paper at ICLR 2025


108
      et al., 2020) use techniques such as depthwise separable convolutions and attention-based pruning
109   to reduce parameters while maintaining accuracy. For example, MobileNet lowers FLOP counts by
110   splitting convolutions into depthwise and pointwise operations, and MobileBERT simplifies BERT’s
111   architecture with minimal performance loss. Similarly, LookupFFN replaces compute-heavy matrix
112   multiplications in transformers with memory-efficient lookup operations, improving suitability for
113   CPU inference (Zeng et al., 2023). However, these methods require careful design to balance effi-
114   ciency and accuracy, may be scenario-specific, and focus on single-threaded efficiency without fully
115   exploring parallelism within the computation graph on edge devices.
116   Hardware Acceleration and Heterogeneous Computing: As edge devices adopt heterogeneous
117   processors, hardware acceleration is key to improving inference speed (Symons et al., 2022). Frame-
118   works like TFLite and MNN offload tasks to accelerators like GPUs, TPUs, and NPUs, enhancing
119   performance for static models. Advanced techniques such as co-execution dynamically partition
120   tasks between CPUs and GPUs for better utilization. For instance, CoDL enables intra-operator
121   parallelism to optimize latency and energy efficiency (Jia et al., 2022), while NN-Stretch transforms
122   sequential models into parallel branches for independent execution across processors (Wei et al.,
123
      2023). BAND manages concurrent DNN inference on heterogeneous processors (Jeong et al., 2022),
      and methods like uLayer (Kim et al., 2019) and OPTiC (Wang et al., 2018) enhance CPU-GPU co-
124
      execution. However, these approaches struggle with dynamic models like ASR and transformers due
125
      to inefficient offloading, leading to CPU fallback. Additionally, co-execution methods often require
126   separate memory allocations to avoid conflicts, increasing memory usage (Wang et al., 2023). While
127   improving hardware utilization, these methods complicate memory management and often require
128   model modifications, hindering deployment and scalability.
129
      Sequential Execution and Memory Management: SOTA frameworks like TFLite and MNN use
130
      sequential execution strategies, traversing computation graphs node by node based on data depen-
131
      dencies (Lee & Pisarchyk, 2020). While this simplifies memory reuse and ensures correctness, it
132   limits parallel execution, especially in models with branching structures like ASR and transform-
133   ers. These frameworks aggressively reuse memory to reduce the footprint, but this leads to data
134   dependency conflicts that hinder parallelism. As a result, despite being optimized for memory ef-
135   ficiency, they fail to fully exploit the parallelism in the computation graph, resulting in suboptimal
136   performance. To address these limitations and accelerate ASR models, we propose a CPU-based
137   parallelism framework with branch-aware memory allocation, optimizing inference on edge devices
138   without modifying the model.

      3.1     G RAPH A NALYSIS P IPELINE
150
      Identifying parallelizable branches is essential for enabling efficient parallel. The main challenge
151
      arises from the varied structures of model graphs G = (V, E), where V is the set of nodes and
152   E is the set of edges representing data dependencies. This complexity makes it difficult to ex-
153   tract parallelism without disrupting the data flow. Even when branches can be executed in parallel,
154   unbalanced workloads can reduce the benefits of parallelism, requiring careful criteria to identify
155   balanced branches. To address this, we design a graph analysis algorithm that splits the model
156   into layer-branch structures, isolating independent branches for parallel execution. This approach is
157   adaptable to various models, ensuring generalizability while assessing workload balance.
158
159   3.1.1    D EFINITIONS : L AYER AND B RANCH
160
161   In our graph analysis, we define two core concepts that are fundamental for isolating parallelizable
      segments: branch and layer.

180   Branch: A branch b, or subgraph S, is a set of sequentially connected nodes {v1 , v2 , . . . , vk } within
181   a layer. A subgraph may have multiple inputs or outputs, but within the subgraph, the nodes are
182   strictly connected in a sequential manner. This ensures that each subgraph can be executed indepen-
183   dently of others within the same layer, provided there are no inter-branch dependencies.
184   Layer: A layer ℓ in the computation graph is a set of branches that can be executed concurrently,
185   with all dependencies resolved. Subgraphs in the same layer have no unmet dependencies, allowing
186   parallel execution, while outputs from one layer feed into the next, ensuring proper data flow and
187   execution order.
188   The motivation for this definition is to minimize interference between branches and layers and the
189   rest of the graph. This structure forms the foundation for enabling parallel execution, ensuring that
190   parallel branches do not disrupt data flow in other parts of the model.

192   3.1.2   N ODE C LASSIFICATION AND S UBGRAPH PARTITIONING
193
194   Our algorithm, Algorithm 1, begins by classifying each node vi ∈ V in the graph into one of four
195   categories: (1) Sequential nodes have a single input and output, representing linear connection; (2)
196   Branching nodes have a single input but multiple outputs vi → {vj , vk , . . . }, indicating points where
197   the flow diverges into parallel paths; (3) Merging nodes have multiple inputs converging into a single
198
      output {vj , vk , . . . } → vi , requiring synchronization of parallel paths; (4) Branching-merging nodes
      combine both branching and merging behaviours, representing more complex graph structures that
199
      require careful handling.
200
201   After classifying the nodes, we traverse the graph to group nodes into S. Starting from each unvisited
202   node, sequential nodes are added to the current S until a branching or merging node is encountered.
203   When a branching node is reached, the current group is finalized, and the branching node starts a new
204
      S, allowing independent parallel execution. Similarly, encountering a merge node finalizes the group
      to ensure proper synchronization. For branching-merging nodes, the current group is immediately
205
      finalized, treating the node as an isolated branch due to its complexity. After forming each S, we
206
      eliminate duplicates by comparing their structures, ensuring only unique execution groups remain.
207   This results in a set of independent Ss that can be executed sequentially or in parallel, depending on
208   their structure and dependencies.

253   3.2     B RANCH -AWARE M EMORY A LLOCATION
254
      Efficient memory management is essential for running DNNs on edge devices. SOTA frameworks
255
      typically manage allocation via a memory arena in sequential execution, freeing intermediate tensors
256
      after use. However, parallel execution complicates this due to uncertainties in tensor lifecycles.
257
258
      3.2.1    A DDRESSING RAW DATA D EPENDENCIES
259
260   Challenge: In frameworks like TFLite, tensor lifecycles are typically tracked by identifying the first
261   and last nodes that use a given tensor (Lee & Pisarchyk, 2020). This approach assumes a sequential
262   execution process, where the last node in the lifecycle completes after all prior operations, allow-
263   ing for safe memory reuse. However, parallel execution disrupts this assumption, as the last node
264
      may execute earlier than expected due to concurrent branch execution. This can lead to premature
      memory reclamation, causing RAW conflicts when other nodes in the graph still require access to
265
      the tensor.
266
267   Proposed Solution: To address these challenges, we propose a branch-aware memory allocation
268   strategy that isolates memory management at the granularity of layers and branches. Each branch-
269   specific memory arena is indexed by both layer and branch identifiers, ASℓii . Within each branch, we
      ensure memory alignment to optimize access speed.

      3.2.2    H ANDLING DYNAMIC R EALLOCATION
285
286   Challenge: Dynamic operations lack predefined tensor shapes until runtime, requiring temporary
287   memory allocation followed by reallocation when actual sizes are determined. In parallel execution,
288   this reallocation introduces synchronization bottlenecks, as threads must wait for memory availabil-
289   ity, negating the benefits of parallelism. Additionally, reallocating memory prematurely can cause
290   RAW conflicts by overwriting tensors still in use by other branches.
291   Proposed Solution: Our approach focuses on restricting reallocation to the current branch and its
292   subsequent layers. When a dynamic tensor’s size is determined, only the directly affected tensors
293   undergo reallocation, following these conditions:
294
295              Reallocate(Tj ) ∀ j ∈ {current branch ∪ future layers}      if size(Tj ) ̸= known
296
      This branch-specific reallocation prevents interference with other concurrently executing branches.
297
      Unlike conventional frameworks, where reallocations in one part of the graph can disrupt other
298   branches, our method ensures independent branch execution. By minimizing synchronization bot-
299   tlenecks, other branches can continue uninterrupted, improving parallel execution efficiency.

301
      3.3     M ULTITHREADING E XECUTION
302
303
      To efficiently execute parallel branches, we maintain a fixed thread pool to reduce overhead from
304   thread management. This allows threads to be dynamically assigned to tasks without additional
305   setup costs. To optimize CPU utilization, we implement task stealing, enabling idle threads to han-
306   dle tasks from busy ones, ensuring balanced workload distribution. By maintaining branch isolation
307   and minimizing synchronization points, we reduce contention between threads, particularly during
308   dynamic tensor reallocations. This approach maximizes parallelism while minimizing multithread-
309   ing overhead across devices.

      5     C ONCLUSION AND F UTURE W ORK
526
527
      We introduce BAP, a system that accelerates inference on mobile devices for ASR and transformer
528   models. By employing CPU parallel execution, branch-aware memory allocation, and efficient mul-
529   tithreading management, BAP fully leverages CPU resources to reduce latency. BAP achieved la-
530   tency reductions of 18% to 38% across various models, averaging 27% to 30%, with maximum
531   improvements up to 38.5%. Memory allocation was reduced by an average of 4.1× to 8.6×, with
532   maximum savings up to 15.6× compared to TFLite’s naive plan. BAP also reduced energy con-
533   sumption by up to 24.64% on high-end devices due to faster inference. These results highlight
534   BAP’s effectiveness in enhancing real-time inference on mobile devices.
535   While BAP demonstrates strong performance improvements in inference latency and energy effi-
536   ciency, there are areas for further exploration. Our method focuses on CPU-based optimization, but
537   future work could extend this approach to heterogeneous computing environments, such as GPUs
538   and NPUs, as support for dynamic tensors becomes more widely available. Additionally, future
539   research could investigate adaptive task scheduling and dynamic workload balancing to further op-
      timize both energy consumption and performance across different edge devices.
