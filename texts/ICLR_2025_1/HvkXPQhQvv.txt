009                                                 A BSTRACT
010
011
               It remains difficult to evaluate machine learning classifiers in the absence of a
012            large, labeled dataset. While labeled data can be prohibitively expensive or impos-
013            sible to obtain, unlabeled data is plentiful. Here, we introduce Semi-Supervised
014            Model Evaluation (SSME), a method that uses both labeled and unlabeled data
015            to evaluate machine learning classifiers. SSME is the first evaluation method to
016            take advantage of the fact that: (i) there are frequently multiple classifiers for the
017            same task, (ii) continuous classifier scores are often available for all classes, and
018            (iii) unlabeled data is often far more plentiful than labeled data. The key idea is to
019            use a semi-supervised mixture model to estimate the joint distribution of ground
020
               truth labels and classifier predictions. We can then use this model to estimate any
               metric that is a function of classifier scores and ground truth labels (e.g., accuracy
021
               or expected calibration error). We present experiments in four domains where
022
               obtaining large labeled datasets is often impractical: (1) healthcare, (2) content
023            moderation, (3) molecular property prediction, and (4) image annotation. Our
024            results demonstrate that SSME estimates performance more accurately than do
025            competing methods, reducing error by 5.1× relative to using labeled data alone
026            and 2.4× relative to the next best competing method. SSME also improves ac-
027            curacy when evaluating performance across subsets of the test distribution (e.g.,
028            specific demographic subgroups) and when evaluating the performance of large
029            language models.

032   1    I NTRODUCTION
033
034
      Rigorous evaluation is essential to the safe deployment of machine learning classifiers. The standard
035
      approach is to measure classifier performance using a large labeled dataset. In practice, however,
036   labeled data is often scarce (Culotta & McCallum, 2005; Dutta & Das, 2023). Exacerbating the
037   challenge of evaluation, the number of off-the-shelf classifiers has increased dramatically through
038   the widespread usage of model hubs. The modern machine learning practitioner thus has a myriad
039   of trained models, but little labeled data with which to evaluate them.
040
      In many domains, unlabeled data is much more abundant than labeled data (Bepler et al., 2019;
041
      Sagawa et al., 2022; Movva et al., 2024). To take advantage of this, we introduce Semi-Supervised
042   Model Evaluation (SSME), a method that can be used to evaluate multiple classifiers using both la-
043   beled and unlabeled data. Our key idea is to estimate the joint distribution of ground truth classes y
044   and continuous classifier scores s using a mixture model, where different components of the mixture
045   model correspond to different classes. The joint distribution allows us to evaluate performance on
046   examples where we have access only to each classifier’s scores, i.e., no labeled data. i.e. unlabeled
047   examples. SSME can estimate any metric which is a function of class labels and probabilistic pre-
048   dictions, which includes widely-used metrics like accuracy, expected calibration error, AUC, and
049   AUPRC.
050   SSME is the first evaluation method to learn from three key facets of modern machine learning set-
051   tings: (i) multiple machine learning classifiers, (ii) probabilistic predictions over all classes, and (iii)
052   unlabeled data. Simultaneously using all three is difficult because it requires accurately estimating
053   the (potentially high-dimensional) joint distribution P (y, s) with primarily unlabeled data. While
      prior work captures subsets of these properties (Welinder et al., 2013; Platanios et al., 2017; Ji et al.,


                                                           1
      Under review as a conference paper at ICLR 2025


054
      2020; Chouldechova et al., 2022; Boyeau et al., 2024) — for example, augmenting labeled data with
055   unlabeled data to evaluate a single classifier — no existing approach accommodates all three.
056
057   We show that using all available data — multiple classifiers, continuous scores over all classes,
058
      and unlabeled data — enables SSME to produce more accurate performance estimates compared to
      prior work. We test SSME’s ability to estimate the absolute performance of each classifier across
059
      eight tasks, four metrics, and dozens of classifiers, where SSME accepts a set of classifiers, little
060
      labeled data (i.e. between 20 and 100 labeled examples) and more abundant unlabeled data (i.e.
061   1000 unlabeled examples). Concretely, we make four contributions:
062
063         1. We propose SSME, a method to evaluate multiple classifiers using labeled and unlabeled
064            data. SSME extends readily to any number of classifiers and classes, and is able to estimate
065            any metric that compares predicted probabilities to ground truth labels.
066         2. We conduct semi-synthetic experiments to characterize factors affecting SSME’s perfor-
067            mance: the accuracy, calibration, and cardinality of the classifier set being evaluated.
068         3. We show, in experiments spanning multiple modalities, domains, and classifier architec-
069            tures, that SSME achieves the lowest metric estimation error compared to using labeled
070            data alone and compared to prior work, across all considered metrics.
071         4. We demonstrate two broadly useful applications of SSME: evaluating subgroup-specific
072            performance, a critical step in assessing algorithmic fairness, and evaluating fine-tuned
073            large language models.

075   2    R ELATED W ORK
076
077   Our work builds on two areas of literature: methods that use a combination of labeled and unlabeled
078   data to 1) evaluate a single classifier, or 2) evaluate the accuracy of multiple discrete annotations. For
079   a discussion of connections to prediction-powered inference and unsupervised classifier evaluation,
080   see Appendix A.
081   Semi-supervised evaluation of single classifiers involves the evaluation of a single classifier using
082   both labeled and unlabeled data. There are two types of assumptions common in this literature. The
083   first places parametric constraints on the distribution of classifier scores. Several works attempt
084   to fit a mixture model to the distribution of classifier scores (Welinder et al., 2013; Chouldechova
085   et al., 2022; Miller et al., 2018), as we do, while others apply techniques from Bayesian calibration
086   (Ji et al., 2020; 2021). Our work differs in that the proposed framework naturally capitalizes on
087   multiple classifiers, and as our results show, doing so results in improved estimates of performance.
088
      The second type of assumption relates to the structure of the shift between the labeled and unlabeled
      data; as Garg et al. (2022) establish, estimating accuracy on the unlabeled data is impossible absent
089
      assumptions about the nature of the distribution shift. Examples of these assumptions include
090
      covariate shift (Chen et al., 2021b; 2022; Lu et al., 2023), conditional independence of features
091   (Steinhardt & Liang, 2016), and calibration on the unlabeled data (Guillory et al., 2021; Jiang et al.,
092   2022). Here too, all work focuses on evaluating individual classifiers and often relies on larger
093   amounts of labeled data than we assume (on the order of hundreds of labeled examples). In contrast,
094   our focus is on the evaluation of multiple classifiers, when the amount of labeled data is too small
095   to reliably learn any model of distribution shift between the labeled and unlabeled data.
096
097   Semi-supervised evaluation of discrete annotators was first introduced by Dawid & Skene
098   (1979), who proposed a method to estimate ground truth in the presence of multiple potentially
099
      noisy discrete annotations. Many follow-on works inherit Dawid-Skene’s strong assumption of
      class-conditional independence of annotator errors (Parisi et al., 2014; Platanios et al., 2017),
100
      including popular approaches in weak supervision (Ratner et al., 2017; Bach et al., 2017; Fu
101
      et al., 2020), where annotators are instead user-provided labeling functions. Such an assumption is
102   plausible in certain contexts, but does not naturally translate to sets of candidate classifiers, whose
103   predictions are likely to be correlated. Subsequent work has made an effort to relax the assumption
104   of class-conditional independence, replacing it with independence conditional on a latent notion
105   of example difficulty (Paun et al., 2018) or adjusting for dependencies between annotators (Ratner
106   et al., 2017; Bach et al., 2017; Fu et al., 2020). However, these methods are designed to estimate
107   the accuracy of binary annotations; they do not exploit the continuous probabilities available
      in multi-classifier evaluation. While some work has made progress towards accommodating

160   Our aim is to develop an approach that captures three common properties of modern classification
161   settings: (i) multiple available trained models, (ii) an abundance of unlabeled data, relative to labeled
      data, and (iii) access to each classifier’s predicted class probabilities on every input.

      The core idea underlying SSME is that if we can estimate the joint distribution of ground truth labels
163   y and classifier scores s — i.e., P (y, s) — then we can estimate any metric that is a function of
164   classifier scores and ground truth labels. Notably, we use the classifier scores s (i.e. the probability
165   outputs over all classes) and not the raw inputs x to estimate this joint density, consistent with
166   prior semi-supervised evaluation work (Ji et al., 2020; Boyeau et al., 2024). There are two reasons
167   for this: (1) the output distribution over classes s and the corresponding labels y are sufficient to
168   characterize most standard metrics (calibration, accuracy, etc.), and (2) modeling s directly is a
169   standard technique to avoid estimating densities over x, which can be difficult due to the frequently
170   high-dimensional nature of the input distribution.
171   SSME makes two additional assumptions on the (true) mixture density p(y, s). First, we assume that
172   the unlabeled samples s(i) are drawn from the same distribution as the labeled samples (y (i) , s(i) ).
173   Second, given the challenges of density estimation in high-dimensional settings (Wang & Scott,
174   2019; Rippel & Adams, 2013), we primarily focus on settings with a limited (≤ 50) number of
175   classes. Our latent variable model can support higher-dimensional evaluation problems but would
176   require robust semi-supervised density estimation procedures.
177   To estimate the accuracy of the first model, for instance, one could (repeatedly) draw labels ŷ from
178   P (y|s) for each datapoint s and compute agreement between the drawn label and the first model’s
179   predicted label. By modeling P (y, s) using a mixture model, we can capture class-specific variation
180   in model scores — for instance, classifiers in our set may agree in their predictions on y = 0 but
181   disagree in their predictions on y = 1.

      with unit-sum constraints (Aitchison, 1982; Pawlowsky-Glahn & Buccianti, 2011). To implement
217   this, we transform probabilistic predictions over K classes to scores in RK−1 using the additive log
218   ratio transform, which produces a one-to-one mapping between the two spaces. Additive log-ratio
219   transforms are just a reparameterization trick; they simply transform the classifier scores from a
220   hard-to-model space — the probability simplex — to an easier-to-model space — unbounded reals.
221   For details on the additive log-ratio transform, see Appendix E.1. Our approach helps us overcome
222   well-documented issues with density estimation over bounded spaces.
223
224   Model estimation For all experiments, we fit each kernel density estimator with a Gaussian kernel.
225   The kernel bandwidth is estimated using the improved Sheather-Jones algorithm (Botev et al., 2010).
226   In accordance with previous semi-supervised mixture models, we train our mixture model with
227   expectation-maximization (EM) (Dempster et al., 1977; Zhao et al., 2023), which alternates between
228   (1) the E-step, which estimates which mixture component each datapoint belongs to, and (2) the M-
229   step, which estimates the parameters of each component based on the soft component assignments,
230   including the overall class prior P (y). We take the same approach, where we alternate between
231   estimating the true label for a given example, and estimating the class-conditional distribution of
232   model predictions based on these estimated labels. We optimize the parameters using EM over 50
233   epochs. We initialize component assignments by drawing a label for a given example according to
234
      the mean classifier score across the set of classifiers.
235   Evaluation
236
       After fitting the mixture model, we sample the true label for each unlabeled point using our fitted
237
      posterior distribution for P (y (i) |s(i) ) multiple times for each unlabeled example s(i) in our dataset.
238
      Alternatively, one could compute an expectation over the the label of each individual s(i) and sum
239
      over the entire dataset; we discuss this approach in Appendix E.2. For labeled examples, we use
240
      the true y (i) . The estimated metrics are computed by averaging across all realizations (sampled
241
      for unlabeled, fixed for labeled) of y (i) . Once we have fit the parameters θ for our mixture model,
242
      we can use our fitted density Pθ (s, y) to estimate metrics of interest, using the procedure described
243   in panel 3 of Figure 1. In particular, given an estimate of the full joint density Pθ (s, y), we can
244   infer a distribution over the label y (i) for every unlabeled example s(i) and repeatedly draw ground
245   truth labels from the distribution. To estimate the accuracy of classifier j, we would then compute, in
246                        (i)
      expectation, how sj agrees with y (i) , and average this over all examples i. For unlabeled examples,
247
      we use the inferred distribution over y (i) , and for labeled examples we use the true y (i) . Details on
248
      our metric evaluation procedure are available in Appendix E.2. Finally, it may be the case that not all
249
      classifiers a practitioner wishes to evaluate are simultaneously available (for example, during active
250   model development). In this case, one can refit SSME to the expanded classifier set.
251
252
      Alternative parameterizations SSME can also accommodate alternate parameterizations of
253
      P (s(i) |y (i) ) provided they can (1) accommodate both labeled and unlabeled data and (2) be fit
254   using the mixture model framework described above. One alternative that we explore is using a
255   normalizing flow to model each mixture component; we detail our approach and investigate when
256   this improves over the KDE parameterization in Appendix C. While normalizing flows can also be
257   learned in a semi-supervised setting, they often struggle to model multimodal distributions (Stimper
258   et al., 2022; Cornish et al., 2020). We use the KDE to generate our results in binary datasets. We
259   additionally test the normalizing flow in multiclass settings and show it performs competitively.

294   5.2   BASELINES
295
296   We compare against fiveseven baselines that (with the exception of Labeled) make use of both
297   labeled and unlabeled data to arrive at performance estimates:
298
            1. Labeled represents the standard approach to classifiers evaluation and compares classifier
299            scores to ground truth labels, only using examples for which labels are available.
300         2. Pseudo-Labeled trains a logistic regression classifier to predict the true label from the clas-
301
               sifier scores—directly estimating P (y (i) |s(i) )—and then labels the unlabeled examples us-
302            ing this classifier.
303         3. Majority-Vote ensembles the classifier predictions by performing an accuracy-weighted ag-
304            gregation of each classifier’s prediction. We assign weights to each classifier based on the
305            accuracy achieved on the small sample of labeled data.
306         4. Dawid-Skene (Dawid & Skene, 1979), uses multiple noisy discrete annotations to estimate
307            the latent true label of each example. Dawid-Skene assumes class-conditional errors are
308            independent across annotators (i.e. P (ŷ ̸= y|y = k) is independent).
309         5. Bayesian Calibration (Ji et al., 2020) re-calibrates a classifier’s predicted probabilities
310            based on the observation that if a classifier’s scores are calibrated, estimating performance
311            from the predictions alone is possible. Bayesian Calibration does not extend to multiclass
312            settings, so we can only use this baseline in our binary experiments.
313         6. AutoEval (Boyeau et al., 2024) learns a rectifier on limited labeled data to debias classifier
314            predictions. This rectifier is then applied to the unlabeled examples to estimate metrics of
315            interest. We compare to AutoEval on accuracy; additional metrics are not directly sup-
316
               ported by the public implementation.
317
            7. Active-Testing (Kossen et al., 2021) reframes semi-supervised evaluation as an active learn-
               ing problem, in which a method selects which examples to label out of a large pool of un-
318
               labeled examples. The performance metric is then computed according to the sample of
319
               labeled data. We compare to Active-Testing on accuracy, a metric for which an acquisition
320            strategy is available.
321

323   ensembling (i.e., drawing the “ground truth” label based on the average classifier score) in Appendix

      5.3   E VALUATION
340
341   We evaluate SSME’s ability to estimate four continuous performance metrics for each binary classi-
342   fier: accuracy, area under the receiver operating characteristic curve (AUC), area under the precision-
343   recall curve (AUPRC), and the expected calibration error (ECE). For multi-class problems, we eval-
344   uate accuracy and top-label calibration error (Gupta & Ramdas, 2022); evaluations of multiclass
345   AUC and AUPRC are also possible, but less standard.
346   We partition each dataset into three splits: the classifier training split (which we use to train the
347   classifiers whose performance we will estimate), the estimation split (which we use to fit SSME and
348   estimate classifier performance), and the evaluation split (which we use for a held-out, ground-truth
349   measure of classifier performance). All splits are sampled from the same distribution, except when
350   estimating subgroup-specific performance, where the evaluation split pertains to a single subset of
351   the test distribution.
352   To evaluate metric estimates, we measure the absolute error of the estimated metric, computed
353   using the estimation split, compared to the true metric, computed on the held-out evaluation split
354   (averaging over classifiers in the set). The estimation split consists of either 20, 50, or 100 labeled
355   examples and 1000 unlabeled examples across all experiments. The size of the evaluation split is
356   on the order of thousands of labeled examples and varies by task (see Appendix B.1 for exact split
357   sizes). For each task, we report results over 50 random samples of the splits. In line with prior work,
358   we report rescaled estimation error for each metric (where all errors are relative to using labeled data
359
      alone), allowing us to standardize the scale of errors across datasets and metrics (Garg et al., 2022).

403   SSME estimates performance more accurately because it makes use of all available information:
404   multiple classifiers, continuous scores over all classes, and unlabeled data. Dawid-Skene1 and Auto-
405   Eval discretize classifier scores (although AutoEval does make use of classifier confidence associated
406
      with each discrete prediction). Pseudo-Labeling, Bayesian-Calibration, and AutoEval each learn a
      mapping from s to y using only the labeled data and apply that mapping to the unlabeled data (rather
407
      than learning from labeled and unlabeled data together). By jointly learning across both labeled and
408
      unlabeled data, SSME is able to generalize much better in cases where there aren’t enough labels
409   to estimate the joint distribution of classifier scores and labels from labeled examples alone. Fi-
410   nally, Bayesian-Calibration learns from a single classifier’s scores, and does not learn from multiple
411   models at once.
412
413   Comparison across metrics SSME provides the greatest benefits relative to labeled data alone
414   when measuring expected calibration error (ECE), with a reduction in estimation error of 7.2× (av-
415   eraging across tasks). ECE is harder to estimate with few labeled examples because it requires
416   binning and then averaging calibration error across bins. This process tends to yield greater vari-
417   ability when the number of labeled points per bin is small. We observe the smallest benefits relative
418   to labeled data alone when measuring AUPRC (a reduction in estimation error of 2.2×, relative to
419
      labeled data).

      to labeled data alone, there are diminishing but positive returns to adding labeled data to SSME’s

      6.2   C LASSIFIER EVALUATION ON MULTI - CLASS TASKS
453
      We now validate SSME’s performance on multi-class tasks; Figure 3 reports our results. Because
454   the utility of kernel density estimators is known to degrade in higher dimensional problems (such
455   as here with multi-class outputs) (Jiang, 2017), we provide an instantiation of SSME using a nor-
456   malizing flow, which we term SSME-NF (for additional implementation details, see Appendix C).
457   Normalizing flows have been shown to effectively model mixtures of high-dimensional distributions
458   (Izmailov et al., 2020); our results align with these findings. SSME-NF improves over labeled data
459   by 2.9× (averaging over tasks and metrics); SSME-KDE improves over labeled data by 2.7×. In
460   contrast, the next best baseline (Dawid-Skene) improves over labeled data by 1.5×. On each dataset
461   and metric, SSME-NF is consistently one of the top two methods.

      Applications to LLM evaluation Our results are encouraging in the context of evaluating large
      language models as annotators, an emerging application (Ziems et al., 2023). The classifier sets we
464

465

      6.3   A SSESSING SUBGROUP - SPECIFIC PERFORMANCE
474
475   SSME can be applied to measure performance within demographic groups of interest, a task central
476   to assessments of algorithmic fairness (Chen et al., 2021a). These groups may be based on gender,
477   age, race, or other groups who face disparities. Unlabeled data has been used in multiple ways to
478   improve subgroup evaluation and algorithmic fairness (Ji et al., 2020; Sagawa et al., 2021; Ktena
479   et al., 2024; Movva et al., 2024): in particular, it can be used to estimate gaps in performance
480   between groups, such as disparities in accuracy across race (Ji et al., 2020).
481   We conduct our analysis in the context of critical outcome prediction on MIMIC-IV, as prior studies
482   have established that predictive models often display disparities in error rates for this task (Movva
483   et al., 2023). The first two steps involved in SSME remain the same: we (1) acquire classifier scores
484   on all patients using each classifier, and (2) fit the mixture model to classifier scores over the entire
485   sample of labeled and unlabeled data. We then produce subgroup-specific performance estimates by
      using the empirical distribution of s within each subgroup; that is, we sample ground truth labels y

      6.4   I MPACT OF CLASSIFIER SET CHARACTERISTICS ON SSME’ S PERFORMANCE
511
512   We now characterize SSME’s performance in a semi-synthetic setting, in which we can assess how
513   characteristics like classifier accuracy and calibration affect the performance of the proposed ap-
514   proach. To do so, we create sets of three classifiers based on the widely-used Adult dataset (Becker
515   & Kohavi, 1996), where the task is to predict whether a person’s income is above $50K. To create
516   differences between the three classifiers in a set, we train them on random fixed-size samples of 100
517   labeled examples from different portions of the dataset, partitioned based on age. In doing so, our
518   semi-synthetic classifier sets mimic how training data for different real-world classifiers can differ in
519
      demographically meaningful ways. We repeat this procedure to produce 500 sets of three classifiers,
      where sets differ in the training data provided to each classifier. Our procedure naturally produces
520
      random variation in classifier properties, like accuracy and calibration, which we can use to study
521
      how well SSME performs. For additional experimental details, refer to Appendix B.3.
522
523
      Classifier accuracy We find that as the average accuracy among classifiers in the set increases,
524
      SSME’s performance estimation error decreases (Fig. 4, left). This trend holds across each of the
525
      four metrics we consider (see Figures S1 and S2 for additional plots) and can be attributed to how
526   more accurate classifiers produce more separable components of the mixture model we aim to esti-
527   mate. More accurate classifiers allow SSME to better estimate the ground truth label y for unlabeled
528   examples. Classifier accuracy directly impacts the separation of mixture model components, which
529   existing work has shown to result in more accurate parameter estimation (Redner & Walker, 1984).
530
531   Classifier calibration Similarly, when the classifier set grows more calibrated on average,
532   SSME’s estimation error decreases. We can attribute this behavior to how poorly calibrated clas-
533   sifiers result in worse initialization for the mixture model, because we initialize component assign-
534   ments for the unlabeled examples based on the mean prediction across classifiers.
535
536   Classifier quantity Conditional on each of the classifier set characteristics discussed (accuracy
537   and calibration), increasing the number of classifiers reduces performance estimation error. Our
538   results suggest that increasing the number of classifiers can sometimes be more beneficial than
539   acquiring a set of more accurate or better calibrated classifiers. Given the widespread availability of
      pretrained classifiers, this is a promising path towards more accurate evaluations.

561   7    D ISCUSSION
562
563   In this paper, we presented Semi-Supervised Model Evaluation (SSME), a method which supple-
564   ments sparse labeled data with unlabeled data to more accurately estimate classifier performance.
565   SSME exploits three aspects of the current machine learning landscape: (i) there are frequently mul-
566   tiple classifiers for the same task, (ii) continuous classifier scores are often available for all classes,
567
      and (iii) unlabeled data is often far more plentiful than labeled data. We show that across multiple
      tasks, architectures, and modalities SSME substantially outperforms using labeled data alone and
568
      standard baselines.
569
570   These results suggest several directions for future work. First, each of the metrics we examined cen-
571   ter on evaluating a single classifier. But because SSME estimates the full joint distribution P (y, s),
572   it could also be used to to measure properties of the classifiers as a set. For instance, recent work
573
      has highlighted the importance of measuring systemic failures (Kleinberg & Raghavan, 2021) where
      all classifiers produce errors on the same instances. Second, our experiments assess settings in
574
      which the unlabeled data is sampled from the same distribution as the labeled data. Although this
575
      is common — for example, when a random subset of examples is annotated — there are other set-
576   tings where the available unlabeled data systematically differs from the labeled data (Sagawa et al.,
577   2021). Applying SSME to those settings represents a natural direction for future work. Finally,
578   future work could also extend SSME beyond classification to estimate the joint distribution P (y, s)
579   for continuous y (e.g, using a mixture density network) or structured y (e.g. graphs) (Vishwakarma
580   & Sala, 2022). More generally, our results strongly indicate that when large amounts of labeled data
581   are unavailable, semi-supervised evaluation can be valuable. SSME is one way to do this, but other
582   approaches are worth exploring.

866
867   A     R ELATED WORK
868
869   We detail related work in unsupervised performance estimation here. Works below assume access
870   to only unlabeled data; in contrast, SSME learns from both labeled and unlabeled data.
871
872   Unsupervised performance estimation involves estimating the performance of a model given
873   only unlabeled data. Methods designed to address this problem often focus on out-of-distribution
874
      samples, where labeled data is scarce and model performance is known to degrade. Several works
      have illustrated strong empirical relationships between out-of-distribution generalization and thresh-
875
      olded classifier confidence (Garg et al., 2022), dataset characteristics (Deng & Zheng, 2021; Guillory
876
      et al., 2021), in-distribution classifier accuracy (Miller et al., 2021), and classifier agreement (Parisi
877   et al., 2014; Platanios et al., 2017; Baek et al., 2022).
878
879   Several works have formalized when unsupervised model evaluation is possible (Donmez et al.,
880
      2010; Chen et al., 2022; Garg et al., 2022; Lu et al., 2023), and propose assumptions under which es-
      timates of performance are recoverable. Donmez et al. (2010) and Balasubramanian et al. (2011) as-
881
      sume knowledge of p(y) in the unlabeled sample. Steinhardt & Liang (2016) assume conditionally-
882
      independent subsets of the observed features, inspired by conditional-independence assumptions
883   made in works such as Dawid & Skene (1979). Guillory et al. (2021) assume classifier calibration
884   on unlabeled samples. Chen et al. (2022) assume a sparse covariate shift model, in which a subset
885   of the features’ class-conditional distribution remains constant. Lu et al. (2023) illustrate misesti-
886   mation of p(y) in the unlabeled example, and assume that p(y) out-of-distribution is close to p(y)
887   in-distribution. As Garg et al. (2022) highlight, assumptions are necessary to make any claim about
888   the nature of unsupervised model evaluation, and the above methods are a representative sample
889   of assumptions made by prior works. Finally, there has been a surge of interest in unsupervised
890   performance estimation in the context of large language models (Zheng et al., 2023; Huang et al.,
891
      2024). A standard approach here is to use a large language model to adjudicate the quality of text
      generated by other language models. Methods in this literature are often specific to large language
892
      models, while SSME is not.
893
894   Our work is also similar, in spirit, to methods that learn to debias classifier predictions on a small
895   set of labeled data and then apply that debiasing procedure to classifier predictions on unlabeled
896   examples. Prediction-powered inference (Angelopoulos et al., 2023) and double machine learning
897
      (Chernozhukov et al., 2018) both learn a debiasing procedure to ensure that unlabeled metric es-
      timates (e.g., accuracy) are statistically unbiased. One of the baselines we compare to, AutoEval
898
      (Boyeau et al., 2024), is built atop prediction-powered inference.

908
      as training data), an estimation split (provided to each performance estimation method), and an

909

910

913   experiment, and the evaluation split for each task is fixed across runs (exact sample sizes reported
914   below).

1104   C    N ORMALIZING F LOW
1105
1106   One alternative parameterization is to use a normalizing flow to model our mixture of distributions.
1107   Normalizing flows learn and apply an invertible transform fθ to a random variable z ∼ D1 to obtain
1108   fθ (z) ∼ D2 . Here, we set z ∼ D1 to a Gaussian mixture model and learn a transformation such
1109              dist.
       that fθ (z) ≈ s, i.e., the transformed distribution roughly matches our classifier score distribution.
1110   By modeling z explicitly as a Gaussian mixture model, one can move back and forth between the
1111   two distributions, as fθ−1 (fθ (z)) = z. Specifically, we set the distribution of Z to follow a Gaussian
1112   mixture:
1113
1114                                         Z|(Y = k) ∼ N (µk , Σk )
1115                                                   PK
1116   Thus, the marginal distribution of Z is pZ (z) = k=1 N (z|µk , Σk ) · p(y = k) is the overall density
1117   of z. We apply our invertible transformation fθ to obtain s = fθ (z). To find p(s|y = k), we follow
1118
       the approach of Izmailov et al. (2020):
1119
                                                                            δf
1120                       pS (s|y = k) = N (fθ−1 (s)|µk , Σk ) · det                · p(y = k)
1121                                                                        δx
1122
1123   Intuitively, we transform (s, y) into a distribution (z, y) which follows a Gaussian mixture model.
1124
       By enforcing the constraint that this transform is invertible, the joint distribution on (z, y) captures
       all the information in (s, y).
1125
1126   We use the RealNVP architecture (Dinh et al., 2016) to parameterize fθ using 10 coupling layers,
1127   3 fully-connected layers, and a hidden dimension of 128 between the fully connected layers. Our
1128   normalizing flow is lightweight and trains in less than a minute for each dataset in our experiments
1129   section using 1 80GB NVIDIA A100 GPU.
1130   Note there are two optimizations here: (1) the normalizing flow transformation fθ which maps
1131   s into our latent Gaussian mixture space and (2) the Gaussian mixture model parameters µk , Σk
1132   themselves. We begin by fixing the GMM parameters µk , Σk to values estimated from our classifier
1133   scores s and learning only the flow fθ for 300 epochs. Afterwards, we optimize the GMM parameters
       µk , Σk with EM for another 700 epochs.

1261   D.3       C OMPARISON TO BASELINES DRAWN FROM WEAK SUPERVISION
1262
1263   Popular approaches to weak supervision including Snorkel (Ratner et al., 2017) and FlyingSquid
1264   (Fu et al., 2020) implement a latent variable model equivalent to Dawid-Skene. Both works build
1265   on Dawid-Skene to incorporate information about pairwise correlations between labeling functions;
1266   (Ratner et al., 2017) employs a technique to infer dependencies, while (Fu et al., 2020) assume these
1267
       dependencies to be user-provided. When we applied a standard approach to dependency inference
       (Bach et al., 2017) in our setting, we observed that (1) all classifiers are inferred to be dependent
1268
       on one another, and (2) the number of dependencies raised issues with convergence. It is thus not
1269
       feasible to incorporate dependency inference, and the resulting latent variable model is equivalent
1270   to Dawid-Skene.

       E.3   T HEORETICAL INSIGHTS INTO SSME
1420
1421   Since SSME is a semi-supervised learning method, we can gain theoretical insights into its per-
1422   formance by drawing from results in semi-supervised learning theory. We summarize the data and
1423   modeling assumptions, backed by prior theoretical work, under which SSME is likely to succeed (or
1424   fail). For a full survey of the theory of semi-supervised learning, see Mey & Loog (2022).
1425
       Data assumptions: SSME will perform better when two common semi-supervised learning as-
1426
       sumptions are met in the data:
1427
1428         1. Smoothness: examples i, j with similar classifier scores s(i) , s(j) (for a suitable notion of
1429            similar) are likely to share the same true label y (i) , y (j) .
1430
             2. Clusters and low-density separation: classifier scores s cluster according to their true
1431
                classes y, and high-density cluster centers are separated by low-density regions. These
1432
                low-density regions can help identify decision boundaries, even in the absence of large,
1433            labeled datasets.
1434
1435   In our setting, assumption (2) can be particularly helpful. If, for instance, in a K = 3 class clas-
1436   sification problem, we observe that most classifier scores s cluster into one of three corners of the
1437   3-simplex (with low density in between the corners), it is likely that each of these clusters corre-
1438   sponds to a different class.
1439
       Prior work has shown that when the above two assumptions are met, semi-supervised learning works
1440   well (Singh et al., 2008). Singh et al. (2008) formalizes these assumptions and shows that if mixture
1441   component densities and boundaries are discernable from m unlabeled examples but not from n <
1442   m labeled examples, then semi-supervised learning can improve performance relative to labeled data
1443   alone. They characterize this relationship and show that the error between the estimated density and
1444   true density is reduced by a function of n, m, and the data dimension d when using semi-supervised
1445   learning compared to labeled data alone. Our empirical results also substantiate this result, as we
1446   show that SSME outperforms metric estimation using labeled data alone.
1447   Modeling assumptions: SSME is more likely to succeed when the mixture densities are well-
1448   specified. In other words, if the true underlying density is from the same class of distributions as
1449   the parameterized mixture densities (e.g., Gaussian density), then adding unlabeled data enables
1450   more accurate estimates of the components relative to using labeled data alone (Cozman et al.). If,
1451   on the other hand, the mixture distribution is not well-specified, unlabeled data may hurt. Cozman
1452   et al. formalize this analysis, including under what conditions semi-supervised mixtures are robust
1453   to mis-specification. Our default parameterization, a KDE, is flexible and can accommodate a wide
1454
       variety of distributions.
1455   Even when the data assumptions are met, the model complexity and performance will likely follow
1456   the classic bias-variance tradeoff. Simpler models with fewer parameters (e.g., Dawid-Skene) are
1457   likely to yield biased estimates of the true density (and thus calculated metrics like accuracy) while
       more complex models will yield less bias at the cost of greater variance. Our parameterization,


                                                           27
       Under review as a conference paper at ICLR 2025


1458
       KDEs, have few parameters (other than the bandwidth), unlike approaches which explicitly model
1459   the joint density from a particular distribution (e.g., Gaussian mixture model).
