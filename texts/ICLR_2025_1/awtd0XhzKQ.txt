009                                              A BSTRACT
010
011            Modern Question Answering (QA) and Reasoning approaches based on Large
012            Language Models (LLMs) commonly use prompting techniques, such as Chain-
013            of-Thought (CoT), assuming the resulting generation will have a more granular
014            exploration and reasoning over the question space and scope. However, such
015            methods struggle with generating outputs that are faithful to the intermediate
016
               chain of reasoning produced by the model. On the other end of the spectrum,
               neuro-symbolic methods such as Faithful CoT (F-CoT) and Logic-LM propose to
017
               combine LLMs with external symbolic solvers. While such approaches boast a
018
               high degree of faithfulness, they usually require a model trained for code genera-
019            tion and struggle with tasks that are ambiguous or hard to formalise strictly. We
020            introduce Faithful Logic-Aided Reasoning and Exploration (FLARE), a novel in-
021            terpretable approach for traversing the problem space using task decompositions.
022            We use the LLM to plan a solution, formalise the query into facts and predicates,
023            which form the problem space, using a logic programming code and simulate that
024            code execution using an exhaustive multi-hop search over the defined space. Our
025            method allows us to compute the faithfulness of the reasoning process w.r.t. the
026            generated code and explicitly trace the steps of the multi-hop search without rely-
027
               ing on external solvers. Our methods achieve SOTA results on 7 out of 9 diverse
               reasoning benchmarks. We also show that model faithfulness positively correlates
028
               with overall performance and further demonstrate that FLARE allows pinpointing
029
               the decisive factors sufficient for and leading to the correct answer with optimal
030            reasoning during the multi-hop search. Our findings reveal that successful traces
031            exhibit, on average, a 18.1% increase in unique emergent facts, a 8.6% higher
032            overlap between code-defined and execution-trace relations, and a 3.6% reduction
033            in unused code relations.

037
038   Complex Reasoning in natural Question Answering (QA) tasks assumes the capability to explore
039
      the problem space of the designated query with a formalised set of facts, relations, commonsense
      knowledge and logical implications. In line with this, LLMs have been enhanced with CoT (Wei
040
      et al., 2022) prompting, which supplements the QA process by generating intermediate reasoning
041
      chains given a set of in-context examples (Brown et al., 2020a), as shown in Figure 1. This allowed
042   for advancement in commonsense (Madaan et al., 2022), symbolic (Wang et al., 2022; Sprague
043   et al., 2024) and mathematical (Jie et al., 2023) reasoning. Although CoT allows for a problem ex-
044   ploration in natural language steps, such an approach has been shown to cause performance degrada-
045   tion for reasoning tasks involving multi-step planning (Valmeekam et al., 2022; Suzgun et al., 2023),
046   problem exploration (Yao et al., 2022), and arithmetic tasks (Hendrycks et al., 2021b; Madaan &
047   Yazdanbakhsh, 2022b). These discrepancies arise as CoT suffers from a limited ability to decom-
048   pose, search, verify and backtrack using intermediate rationale chains (Yao et al., 2022), cascading
049   hallucinations and errors (Ling et al., 2023) and that natural language might not be an optimal rep-
050
      resentation for describing the reasoning process (Li et al., 2024). Simultaneously, LLM output has
      been shown to be unfaithful and inconsistent w.r.t. the intermediate CoT rationale (Jacovi et al.,
051
      2024; Lanham et al., 2023b; Turpin et al., 2023).

072   further executed using an external symbolic solver. Producing and executing code enables the gen-
073   eration of outputs guided by external solvers, leveraging search with backtracking to explore the
074   problem space effectively. However, strict translations of natural language queries into code, such as
075
      autoformalisation (Szegedy, 2020; Wang et al., 2018), is a non-trivial task involving direct inference
      of implicit commonsense and domain-specific knowledge and the ability to align abstract and infor-
076
      mal concepts directly to constrained formal definitions for further execution (Wu et al., 2022). An
077
      example query, “Do all parts of the aloe vera plant taste good?”, is challenging to formalize or ad-
078   dress with a strict algorithmic solution, as it requires interpretative, deductive and context-dependent
079   reasoning, referred to as soft or fuzzy reasoning. Using external solvers makes such fuzzy reasoning
080   impossible and requires consistently generating syntactically correct executable code. While some
081   LLMs have coding capabilities stemming from their pretraining (Jiang et al., 2024; Aryabumi et al.,
082   2024), relative code consistency is more probable with models explicitly trained for coding (Chen
083   et al., 2021).
084   To overcome these problems, we propose Faithful Logic-Aided Reasoning and Exploration
085   (FLARE), an interpretable method that allows for planning, fuzzy reasoning, and traversing the
086   problem space with backtracking, exact task decomposition, and measuring faithfulness. In FLARE,
087   given a natural language query, we prompt an LLM to sequentially generate a plan that includes an
088   analysis and the logical steps necessary for formalising and answering the question, a logic program-
089   ming (Wielemaker et al., 2012) code that allows formalising the query into a set of facts, relations
090   and their composition forming the space for exploring that query and the search, which is an LLM-
091   generated code execution simulation. An illustration of FLARE can be seen in Figure 1. In our
092
      framework, the generated code must not be consistently executable by an external solver, allowing
      for the soft-formalisation of natural language. Although we see that even generalist LLMs are able
093
      to produce executable code in ≥ 50% of cases. FLARE allows us to measure the faithfulness of
094
      the outcome w.r.t. the simulated code execution by directly comparing the search paths produced
095   by the external solver to that LLM generation. This comparison also allows for pinpointing model
096   hallucinations and inconsistencies. We systematically study the effectiveness of our method using 4
097   general-purpose LLMs of varying scales across 9 diverse QA and 3 logical inference benchmarks,
098   covering Math World Problems, Multi-hop QA, Relation inference, deductive and analytical reason-
099   ing and show that our method achieves state-of-the-art results in 7 out of 9 QA datasets and 2 out
100   of 3 logic datasets in comparison to CoT, F-CoT and Logic-LM. We also show that the method is
101   competitive for models tuned for coding, with an average overall increase of 16% over F-Cot and
102   9% over CoT. Our findings show that model accuracy strongly correlates with the faithfulness of
103
      the reasoning process towards search traces from the simulated code execution. We also provide ab-
      lations showing that the model can interpretably pinpoint hallucinations, underutilized knowledge,
104
      and the limitations of the search over the problem space. Our key contributions are the following:
105

107   • We introduce FLARE a novel paradigm for logic-aided and interpretable formalisation and search
        over the problem space in QA and logic reasoning tasks.

      2   R ELATED W ORK
119
120   Reasoning in Natural Language Few-shot prompting (Brown et al., 2020b) has been shown to
121
      be an effective approach for increasing the reasoning capabilities of LLMs in natural language gen-
      eration (Gehrmann et al., 2021; Reif et al., 2022; Sanh et al., 2022). LLM reasoning can be further
122
      enhanced with prompting techniques such as CoT (Wei et al., 2022), which attempts to segment
123
      reasoning into explicitly written intermediate steps. Concurrent work has also proposed that models
124   “think step by step” (Kojima et al., 2022), or divide the problem into subtasks before the solu-
125   tion (Zhou et al., 2023, Least-to-Most). These approaches have been shown to suffer from arith-
126   metic inaccuracies (Lewkowycz et al., 2022; Hendrycks et al., 2021a) and reasoning inconsistencies
127   (Madaan & Yazdanbakhsh, 2022a). Further attempts have been made to add a planning stage before
128   reasoning by dividing the process into recursive plan formulation and execution steps (Yao et al.,
129   2023b; Wang et al., 2023a). The plan generation step in FLARE is a hybrid technique inspired by
130   these methods but focused on generating a natural language strategy for formalising the query into
131   code.
132
133   Reasoning with Search Several lines of work propose using techniques to expand the reasoning
134   paths over the problem space. Self-consistency decoding (Wang et al., 2023b) is an approach used
135   to sample many natural language reasoning paths and take a majority vote for an answer. Another
136   popular approach is Tree-of-Thoughts (ToT; Yao et al., 2023a), which proposes to explore the query
137
      with reasoning similar to a tree traversal, where each state is created and evaluated using an LLM.
      Similar techniques try to adapt symbolic search approaches akin to DFS, BFS (Besta et al., 2024),
138
      A∗ (Lehnert et al., 2024) or other combinations (Gandhi et al., 2024) with direct tuning (Lehnert
139
      et al., 2024), imitation training (Yang et al., 2022) or few-shot prompting (Zhang et al., 2024). It
140   must be noted that all of these techniques have only been tested in constrained mathematical puzzle-
141   solving and algorithmic domains like the 24 Game (Yang et al., 2022), Countdown (Wikipedia,
142   2024), Sorting (Besta et al., 2024), maze solving (Yang et al., 2022), Sokoban (Lehnert et al., 2024),
143   and others. Although the search component of FLARE has some similarities to these techniques, we
144   argue that our method allows for generalistic reasoning with interpretable multi-hop search through
145   simulated code execution.
146
147   Reasoning with Formalisation Another line of research has tried formalising natural language
148   queries into code (Gao et al., 2023; Li et al., 2024) or pseudo-code (Chae et al., 2024; Gandhi et al.,
149   2024). This allows the translation of the query into a strict structure and offloads the reasoning and
150   search components to deterministic solvers like Python Chen et al. (2023), PDDL Lyu et al. (2023);
151   Liu et al. (2023), DataLog Lyu et al. (2023) and others. While models are capable of synthesising
152   programs (Austin et al., 2021; Nijkamp et al., 2023) and benefit from the use of code in numerical
153
      and algorithmic reasoning settings (Chen et al., 2023; Gao et al., 2023), the usage of code for general
      QA has not been rigorously explored. The reasons are that formalisation from natural language into
154
      a strict and executable code is challenging (Wu et al., 2022), following the exact syntactic constraints
155
      of the programming language not abundantly used during pre-training is onerous (Liu et al., 2024)
156   and can require models explicitly tuned for coding (Chen et al., 2021). Using an external solver for
157   reasoning also limits the capability for soft reasoning in commonsense knowledge and implications.
158   Although we formalise the natural language query into a logic programming Prolog program during
159   the code generation part of FLARE, we do not explicitly require the code to be executable and
160   do not use external solvers during inference. This allows for the further use of the LLM for soft-
161   reasoning to simulate code execution in a logic-based problem space traversal similar to Prolog
      while circumventing the need for code tuning a generalist model.

180   Reasoning Faithfulness An explanation is considered faithful if it explicitly and accurately de-
181   scribes the reasoning process of the model during inference (Gilpin et al., 2018; Jacovi & Goldberg,
182   2020). In the context of prompting techniques such as CoT, we are interested in the faithfulness of
183   the intermediate reasoning chains towards the final output. Faithful intermediate reasoning chains
184   should not just look plausible (Herman, 2017) but have exact reflections of the problem exploration
185   and reasoning used to arrive at the final answer. Natural language reasoning chains prevalent in
186   CoT and similar methods are shown to be unfaithful, either masking the reasoning biases (Turpin
187   et al., 2023) of the model or outright ignoring the intermediate reasoning (Lanham et al., 2023a). In
188
      FLARE, we introduce a method to seamlessly measure the faithfulness of the final outcome w.r.t.
      completed search.

237    Benefits of Prolog Prolog is a symbolic logic-programming engine (Bowen, 1979) designed
238    for heuristic search over Horn Clauses (Chandra & Harel, 1985). As a declarative programming
239
       paradigm (Lloyd, 1994), the code is expressed as the logic of computation, expressed as a set of
       facts F and relations R defining the problem space, with the goal G being a first-order logic combi-
240
       nation of them. Prolog employs depth-first search (DFS) (Bowen, 1979) for sub-goal decomposition
241
       and traversal of the problem space, satisfying G through a sequence of steps known as the trace.
242    Each step either confirms/invalidates a sub-goal, expands the search tree, or retries failed sub-goals
243    with new combinations. An example of such a search is shown in Figure 1. Prolog supports exhaus-
244    tive search by exploring all paths that satisfy or fail the goal. This explicit segmentation of facts,
245    relations, and search strategies simplifies query formalization. As a declarative language, Prolog
246    enables segmentation using a simple regexp heuristic, referred to as EXTRACT in Equation (2) and
247    Equation (3). Including exhaustive traces in-context allows an LLM to simulate sub-goal decompo-
248    sition, backtracking, and intermediate goal invalidation, discussed further in the next paragraph.

292   3.3   M EASURING FAITHFULNESS
293
294   We propose a method to measure the faithfulness of the LLM reasoning process when using FLARE.
295
      As mentioned in Section 3.1, for each query in a dataset D = [Q1 , . . . , Q|D| ], we generate a set of
296
      codes Φ = [C1 , . . . , C|Φ| ] and simulated problem space searches Ψ = [S1 , . . . , S|Ψ| ]. We use the
297
      Prolog engine to execute all of the codes Φ and obtain a set of correctly written programs Φ′ and
      exact search paths Ψ′ . As we do not require explicit programmatic correctness during inference in
298
      FLARE for any code Ci , some Prolog executions resulting in an error are filtered out in Ψ′ . To assess
299
      model reasoning faithfulness towards code formalisations, we compare the search paths Φ′ obtained
300   from Prolog execution with their designated counterparts Φ′gen generated by the LLM from the same
301   code. We use ROUGE (Lin, 2004) to compute the matching score for each executed and simulated
302   search path. In particular, we use ROUGE-Lsum, which uses the longest common subsequence
303   (LCS) over each line to obtain the final score. This method fits our cause as a line in a Prolog search
304   execution represents a single logic step within the traversal. This allows us to measure the similarity
305   of the reasoning contents and structure in exact and simulated searches.

417   5.2     I S SIMULATING SEARCH USEFUL ?
418
419   To understand if simulating a search over the problem space is useful, we compare the performance
420   of FLARE where we only generate the plan without the subsequent code or search components. We
421   refer to this framework setup as plan-only, which can be seen in Figure 1 if we were to use only the
422   plan for answer generation. We completed this ablation using CmDR, CmDR+, and GPT-3.5, and
423
      we used GSM8K, AQuA, and StrategyQA as our baselines. The results in Table 3 confirm that all
      of the models suffer massive performance degradation from 61.1 → 49.9 when omitting the code
424
      and the search components of FLARE. We hypothesise that this is caused by insufficient problem
425
      space exploration when using the plan-only setting. Furthermore, we have already seen in Table 1
426   that in methods, like F-CoT, that do not use simulated problem space exploration for soft-reasoning
427   and only rely on plan and code, the performance also deteriorates even resulting in a complete
428   breakdown of reasoning over the designated datasets. This can be viewed as a constrained version
429   of FLARE with code-only execution. Consequently, our results show that simulating problem space
430   traversal is highly beneficial as it avoids the pitfalls posed by plan-only and code-only modes by
431   exploring the problem space more rigorously and soft-reasoning during that traversal instead of
      using external solvers.

531
      This work introduces FLARE, a novel approach for logic-aided interpretable formalisation and rea-
532
      soning with simulated search over the problem space. We show that models of varying scales obtain
533
      state-of-the-art results compared to prompting paradigms like CoT and F-CoT. We further pinpoint
534   that using FLARE allows us to perform soft-reasoning with simulated search, making it flexible
535   for diverse reasoning benchmarks. We introduce a method to measure model reasoning faithfulness
536   w.r.t. the problem formalization ingrained within FLARE. Our results show that model performance
537   is positively correlated with the faithfulness of the reasoning process. The systematic studies of
538   the method show the benefits of using simulated search compared to natural language reasoning
539   and external symbolic solvers. We further show that using FLARE allows us to interpretably and
      rigorously detect hallucinations and sub-optimal and inconsistent reasoning patterns.

       A.3   FLARE P SEUDO - CODE
1027
1028
       Below, we present the pseudo-code for the execution of the plan, code, and search procedures in
1029   FLARE. The pseudo-code describes the modular pipeline in FLARE for tackling natural language
1030   queries with faithful simulated search.
1031
1032
             • Plan Generation: This stage creates a structured natural language outline of the reasoning
1033
               process, breaking down the query into logical steps and analysis. The plan serves as the
1034
               foundation for formalization into a logic-based representation.
1035
1036         • Code Generation: Based on the generated plan, a logic programming code (e.g., in Prolog)
1037           is synthesized. This code formalizes the query into a set of facts, relations, and goals, which
1038           collectively define the problem space for reasoning.
1039
1040         • Search Simulation: The generated code is utilized to simulate a search trace over the
1041           problem space. This includes iterative reasoning, backtracking when goals are unmet, and
1042           extracting emergent facts or relations during the process.
1043
1044   Each of these stages is implemented as a modular component. The generation from each of the
1045   stages feeds into the next, allowing seamless integration and incremental improvement in reasoning
1046   accuracy. A detailed pseudo-code is provided below in Appendix A.3.
1047
1048
       Algorithm 1 FLARE Methodology: Faithful Logic-Aided Reasoning and Exploration
1049
1050   Require: Query Q, Language Model M
1051   Ensure: Answer A

        1: Initialization: Load few-shot examples for plans (EP ), code (EC ), and search traces (ES )
        2: Input: Natural language query Q

        3: procedure G ENERATE P LAN
