033                                                    A BSTRACT
034                 We introduce Gaga, a framework that reconstructs and segments open-world 3D
035                 scenes reconstructed with 3D Gaussians by leveraging inconsistent 2D masks pre-
036
                    dicted by zero-shot class-agnostic segmentation models. Contrasted to prior 3D
                    scene segmentation approaches that heavily rely on video object tracking, Gaga
037
                    utilizes spatial information provided by 3D Gaussians and effectively associates
038
                    object masks across diverse camera poses through a novel 3D-aware memory
039                 bank. By eliminating the assumption of continuous view changes in training im-
040                 ages, Gaga demonstrates robustness to variations in camera poses, particularly
041                 beneficial for sparsely sampled images, ensuring precise mask label consistency.
042                 Furthermore, Gaga accommodates 2D segmentation masks from diverse sources
043                 and demonstrates robust performance with different open-world zero-shot class-
044                 agnostic segmentation models, significantly enhancing its versatility. Extensive
045                 qualitative and quantitative evaluations demonstrate that Gaga performs favorably
046                 against state-of-the-art methods, emphasizing its potential for real-world applica-
047
                    tions such as scene understanding and manipulation. The source codes will be
                    made available to the public.

073   different mask label IDs, as the 2D class-agnostic segmentation model independently processes the
074   multi-view images. Naively lifting these inconsistent 2D masks to 3D introduces ambiguity and
075   leads to inferior results in 3D scene segmentation. Hence, we show that assigning each mask a
076   multi-view consistent universal mask ID is crucial before lifting them to 3D. We refer to this task as
077   mask association (Ye et al., 2024).
078   Prior research efforts (Ye et al., 2024; Dou et al., 2024) leverage the recent advance in 3D reconstruc-
079   tion, e.g., Gaussian Splatting (Kerbl et al., 2023), attempt to solve this task by treating multi-view
080   images as video sequences and adopting an off-the-shelf video object tracking method (Cheng et al.,
081   2023a). Nevertheless, this design relies on the assumption of minimal view changes between multi-
082   view images. This condition may not consistently hold in real-world 3D scenes, especially when the
083   input views are sparse. Consequently, these approaches struggle with similar objects or occluded
084   objects that intermittently disappear and reappear in the sequence, as shown in Fig. 2.
085   As such, we analyze the fundamental disparity between the 3D mask association and video object-
086   tracking tasks: the latter does not leverage 3D information naturally provided by the 3D scene.
087   Specifically, masks of the same object across different views shall correspond to the same group of
088   3D Gaussians. Hence, we can assign two masks from different views with the same universal mask
089   ID if there is a large overlap between two groups of 3D Gaussians splatted onto the masks.
090   In this work, we propose Gaga, which groups any 3D Gaussians and renders consistent 3D class-
091   agnostic segmentation across different views. Given a collection of posed RGB images, we first
092   employ Gaussian Splatting (Kerbl et al., 2023) to reconstruct the 3D scene and extract 2D masks
093   using an open-world segmentation method. Subsequently, we iteratively build a 3D-aware memory
094   bank that collects and stores the Gaussians grouped by category. Specifically, for each input view,
095   we project each 2D mask into 3D space using camera parameters and search the memory bank
096   for the category with the largest overlapping with the unprojected mask. Depending on the degree
097   of overlapping, we either assign the mask to an existing category or create a new one. Finally,
098
      following the mask association process described above, we can get a set of multi-view consistent
      2D segmentation masks.
099
100   We use a 16-dimension feature for each Gaussian, i.e., Identity Encoding (Ye et al., 2024). Lever-
101   aging the consistent 2D segmentation masks as supervision, identity encoding is trained via differ-
102   entiable Gaussian rendering to render 2D segmentation feature maps. A linear layer is employed to
103   classify the splatted 2D features into semantic labels for segmentation rendering.
104   Our approach, Gaga, is capable of: 1) synthesizing novel view images of RGB and segmentation
105   with inherent 3D consistency; 2) grouping 3D Gaussians based on their 2D segmentation masks
106   and providing accurate 3D object segmentation for scene manipulation; 3) accommodating any 2D
107   segmentation methods without additional mask pre-processing.

109
110          • We propose a framework that reconstructs and segments 3D scenes using inconsistent 2D
111            masks generated by open-world segmentation models.
112
             • To resolve the inconsistency of 2D masks across views, we design a 3D-aware memory

120   2   R ELATED W ORK
121
122   Segment and Tracking Anything in 2D. Segment Anything (SAM) (Kirillov et al., 2023) and
123
      EntitySeg (Qi et al., 2023) demonstrate the effectiveness of large-scale training in image segmen-
      tation, thus establishing a pivotal foundation for open-world segmentation methods. Subsequent
124
      studies (Yang et al., 2023; Cheng et al., 2023b;a) further extend the applicability of SAM to video
125
      data by leveraging video object segmentation algorithms to propagate SAM masks. Conversely, ac-
126   quiring data for training their 3D counterparts poses a challenge, given that existing 3D datasets with
127   annotated segmentation (Straub et al., 2019; Dai et al., 2017) primarily focus on indoor scenarios.
128
129
      NeRF-based 3D Segmentation. Neural Radiance Fields (NeRF) (Mildenhall et al., 2020) model
      scenes as continuous volumetric functions, learned through neural networks that map 3D coordi-
130
      nates to scene radiance. This approach facilitates the capture of intricate geometric details and the
131
      generation of photorealistic renderings, offering novel view synthesis capabilities.
132
133   Semantic-NeRF (Zhi et al., 2021) initiates the incorporation of semantic information into NeRFs and
134   enables the generation of semantic masks for novel views. Note that semantic segmentation masks
135
      do not face the challenge of ambiguous mask ID across views. Numerous methods expand the scope
      by introducing instance modeling and matching instance masks relying on existing 3D bounding
136
      boxes (Liu et al., 2023b; Fu et al., 2022), resorting to the cost-based linear assignment during train-
137
      ing (Siddiqui et al., 2023; Wang et al., 2023) or directly training instance-specific MLPs (Kundu
138   et al., 2022). However, most of these methods are developed based on ground truth segmentation
139   and tailored for scene modeling within specific domains. They often entail high computational costs
140   and lack substantial evidence of their performance in open-world scenarios.
141
      Leveraging SAM’s open-world segmentation capability, SA3D (Cen et al., 2023b) endeavors to
142
      recover a 3D consistent mask by tracing 2D masks across adjacent views with user guidance. Simi-
143
      larly, Chen et al. (Chen et al., 2023) distill SAM encoder features into 3D and query the decoder. In
144   contrast, Gaga achieves multi-view consistency without user intervention, offering segmentation for
145   all objects rather than an instance. Garfield (Kim et al., 2024) focuses more on clustering than seg-
146   mentation. They densely sample SAM masks and train a scale-conditioned affinity field supervised
147   on the scale of each mask deprojected to 3D.
148
      Gaussian-based 3D Segmentation. As an alternative to NeRF and its variants (Mildenhall et al.,
149
      2020; Chen et al., 2022; Müller et al., 2022; Tancik et al., 2023), Gaussian Splatting (Kerbl et al.,
150   2023; Chen & Wang, 2024; Wu et al., 2024; Yu et al., 2024) has recently emerged as a powerful
151   approach to reconstruct 3D scenes via real-time radiance field rendering. By representing the scene
152   as 3D Gaussians from posed images, it achieves photorealistic novel view synthesis with high re-
153   construction quality and efficiency. Additionally, manipulating 3D Gaussians for scene editing is
154   more straightforward compared to NeRF’s representation.
155
      SAGA (Cen et al., 2023a) renders a 2D SAM feature map and uses a SAM guidance loss to learn 3D
156   segmentation from the ambiguous 2D masks. Similar to (Cen et al., 2023b), this method requires
157   user input and only provides segmentation for one object at a time. Feature 3DGS (Zhou et al., 2024)
158   distills LSeg (Li et al., 2022) and SAM features to 3D Gaussians and decodes rendered features to
159   obtain segmentation. However, it fails to provide consistent segmentation across views. Gaussian
160   Grouping (Ye et al., 2024) and CoSSegGaussians (Dou et al., 2024) use a video object tracker (Cheng
161   et al., 2023a) to associate masks across different views. However, in scenarios with significant
      changes in camera poses between frames, such approaches struggle to maintain accuracy.

192   Gaussian Splatting. Recently, Gaussian Splatting (Kerbl et al., 2023) has significantly advanced
193   the 3D representation field by combining the benefits of implicit and explicit 3D representations.
194   Specifically, a 3D scene is parameterized as a set of 3D Gaussians {Gi }. Each Gaussian Gi =
195   {pi , si , qi , αi , ci } is defined by its position pi = {x, y, z} ∈ R3 , scale si ∈ R3 , orientation q ∈ R4 ,
196   opacity α ∈ R and color features c encoded by spherical harmonics (SH) coefficients.
197   Gaussian Splatting employs the splatting rendering pipeline, wherein 3D Gaussians are projected
198   onto the 2D image space using the world-to-frame transformation matrix corresponding to each
199   camera pose. Gaussians projected to the same coordinates (x, y) (represented as i ∈ N ) are blended
200   in depth order and weighted by their opacity α to produce the color cx,y of each pixel:

237    However, because the segmentation model processes each input image independently, the resulting
238    masks are not naturally multi-view consistent. To resolve this issue, (Ye et al., 2024; Dou et al.,
239    2024) assume that nearby input views are similar and apply a video tracker to associate inconsistent
240
       2D masks of different views. Yet, this assumption may not hold for all 3D scenes, especially when
       the input views are sparse, as demonstrated in Fig. 2.
241
242    Gaga is inspired by the fundamental disparity between the task of mask association across multiple
243    views and tracking objects in a video: the incorporation of 3D information. To reliably generate
244    consistent masks across different views, we propose a method that leverages 3D information with-
245    out relying on any assumptions about the input images. Our key insight is that masks belonging to
246
       the same object in different views shall correspond to the same Gaussians in the 3D space. Conse-
       quently, these Gaussians should be grouped together and assigned an identical group ID.
247
248    Corresponding Gaussians of Mask. Based on this intuition, we first associate each 2D segmen-
249    tation mask with its corresponding 3D Gaussians. Specifically, we splat all 3D Gaussians onto the
250    camera frame, given the camera pose of each input image. Subsequently, for each mask within the
251    image, we identify 3D Gaussians whose centers are projected within that mask. Those Gaussians
252
       should be identified as representatives of the mask in 3D and can be used as guidance for associating
       masks from different views.
253
254    Notably, segmentation masks typically describe the shape of foreground objects under the current
255    camera pose. However, as Fig. 4 (a) shows, a significant proportion of Gaussians do not contribute
256    to the pixels in the 2D segmentation mask, as they represent objects in the back of the 3D scene. To
257    address this, we experiment with different solutions and find that selecting the front x percentage of
258
       3D Gaussians closest to the camera frame as the corresponding Gaussians of the mask works best.
       Here, x serves as a hyperparameter that can be adjusted based on the nature of the current 3D scene.
259
260    As shown in Fig. 4 (b), some masks in 2D represent a large object in 3D, i.e., the blue screen in
261    the image. Selecting corresponding Gaussians based on the entire mask will inaccurately represent
262    the shape of the 3D object, as the selected front Gaussians often concentrate in a confined area
263    (see the yellow Gaussians in Front x% Gaussians of a Mask). Subsequently, when the
264
       camera pose changes, the front x% of Gaussians of the same mask may land in a different confined
       area (see Fig. 4 (b) View 2 row 1 3D Gaussian Group) and have no or little overlap with the
265
       Gaussian Group in the memory bank, fail to associate masks across different camera poses. To
266
       resolve this issue, we propose a strategy wherein we partition an input image into patches and
267    calculate corresponding Gaussians within each patch, as shown in Fig. 4 (b) row 2. Specifically, we
268    begin by dividing the image into N × N patches. Subsequently, we identify the collection of top x%
269    of 3D Gaussians closest to the camera frame within each patch to be the corresponding Gaussians of

273   3D-aware Memory Bank. Next, to collect and categorize 3D Gaussians into groups and use them to
274
      associate masks across different views, we introduce a 3D-aware Memory Bank (see Fig. 3). Given

275

276

      3.3 3D S EGMENTATION R ENDERING AND D OWNSTREAM A PPLICATION .
299
300   After the group ID assigning process, masks projected by the same group of Gaussians are supposed
301   to have the same group ID across different views. Similar to (Ye et al., 2024), we learn a 16-
302
      dimension identity encoding feature for each Gaussian. Like RGB rendering, identity encoding
      renders a 2D identity encoding feature map given a camera pose. Then, a linear layer and a SoftMax
303
      function are employed to predict a semantic label for each pixel in the rendered identity encoding
304
      feature map, resulting in segmentation rendering. We use those associated masks as pseudo-labels
305   to train the identity encoding.
306
307   Our segmentation-aware 3D Gaussians can be readily used for various downstream applications.
308
      For instance, we can render segmentation masks of novel views with consistent semantic labels
      for the same object across different camera poses. Gaussians can also be selected by their identity
309
      encoding for scene editing tasks, including removal, color-changing, position translation, etc., as
310
      demonstrated in Sec. 4.6.

533   5     C ONCLUSIONS
534   We introduce Gaga, a framework that reconstructs and segments open-world 3D scenes using incon-
535   sistent 2D masks predicted by zero-shot segmentation models. Gaga employs a 3D-aware memory
536   bank to store the indices of pre-trained 3D Gaussians and establishes mask association across differ-
537   ent views by identifying the overlap between Gaussians projected to each mask. Results on various
538   datasets demonstrate that Gaga outperforms previous methods with superior segmentation accu-
539   racy, multi-view consistency, and reduced artifacts. Additionally, application in scene manipulation
      highlights Gaga’s high segmentation accuracy and practical utility.

       D.3    OVERLAP T HRESHOLD
1052
1053   During the group ID assigning process, if none of the existing groups in the memory bank has a
1054   larger overlap with the current mask than the threshold, we incorporate this mask into the memory
1055   bank as a new group, signifying the discovery of a new 3D object. We present the ablation study on
1056   overlap threshold in Tab. 8. When the threshold is set to 0.01, we rarely establish a new group and
1057   prefer to associate the mask with an existing group. It provides the best precision but at the expense
1058   of inferior IoU performance. Conversely, setting the threshold to 0.2 results in a frequent declaration
1059   of new group IDs, yielding the best IoU but a significant decrease in precision. Therefore, we set
1060   the threshold to 0.1 to strike a balance in performance across all three metrics.

1072   E     L IMITATIONS
1073
1074   Though Gaga achieves SOTA performance compared to existing works, there are a few limitations
1075   and future works. First, the optimization process of identity encoding and the rest of the Gaussian
1076   parameters are independent, this is because we need to first train 3D Gaussians to acquire their spatial
1077   location for mask association. While this pipeline allows for the utilization of any pre-trained 3D
1078   Gaussians as input without the need to re-train the entire scene, it does require additional training
1079   steps. We aim to enable the joint processing of mask association and identity encoding training in
       future works.
