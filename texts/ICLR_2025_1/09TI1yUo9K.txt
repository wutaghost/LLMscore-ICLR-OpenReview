011                                               A BSTRACT
012
013            3D anomaly detection is a crucial task in computer vision, aiming to identify
014            anomalous points or regions from point cloud data. However, existing methods
015            may encounter challenges when handling point clouds with high intra-class vari-
016
               ance, especially for methods that rely on registration techniques. In this study, we
               propose a novel 3D anomaly detection method, termed Information Gain Block-
017
               based Anomaly Detection (IGB-AD), to address the challenges of insufficient
018
               anomaly detection information and high intra-class variance. To extract ordered
019            features from 3D point clouds, the technique of Rotation-Invariant Farthest Point
020            Sampling (RIFPS) is first introduced. Then, an Information Perfusion (IP) module
021            composed of stacked Information Gain Blocks (IGB) is proposed to utilize prior
022            noise to provide more distinguishing information for the features, where IGB is
023            designed to utilize noise in a reverse-thinking manner to enhance anomaly detec-
024            tion. Finally, a Packet Downsampling (PD) technique is developed to preserve key
025            information between multiple clusters to solve the complex downsampling situa-
026            tion. The main purpose of the framework is to utilize the effective information
027
               within prior noise to provide more detection criteria for anomaly detection. In
               addition, an Intra-Class Diversity (ICD) 3D dataset is constructed, which contains
028
               multiple categories with high class-variance. Experimental results show that the
029
               proposed IGB-AD method achieves the State-Of-The-Arts (SOTA) performance
030            on the Anomaly ShapeNet dataset, with an P-AUROC of 81.5% and I-AUROC of
031            80.9%, and also gains the best performance on the ICD dataset, with an P-AUROC
032            of 57.4% and I-AUROC of 60.2%. Our dataset will be released after acceptance.

035   1   I NTRODUCTION
036
037   3D anomaly detection has emerged as one of the most pivotal topics in computer vision and graph-
038   ics processing, and extracting distinctive features is crucial for distinguishing between normal and
039   anomalous point clouds. Existing 3D anomaly detection methods can be broadly classified into
040   traditional approaches and deep learning-based methods.
041
      Traditional methods, such as Back to the Feature (BTF), primarily focus on individual 3D struc-
042   tures and use mathematical techniques to design specific point or local descriptive features Horwitz
043   & Hoshen (2022). Additionally, some researchers have attempted to enhance feature descriptors
044   through teacher-student networks Bergmann & Sattlegger (2022). Although these methods have
045   demonstrated promising results, they may face limitations when extracting features with similar
046   structures. Moreover, relying solely on handcrafted features may fail to fully leverage prior knowl-
047   edge across samples.
048   Deep learning methods, by incorporating prior knowledge from large datasets, have gradually be-
049   come a mainstream approach to addressing these limitations. Current deep learning methods can
050   be primarily categorized into embedding-based and reconstruction-based Zhou et al. (2024). The
051   embedding-based methods involve mapping features extracted using transfer learning onto a specific
052   interval for learning. Distributions that do not fall within this interval are classified as anomalies.
053   RegAD Liu et al. (2023) utilizes the PointMAE model trained on large-scale datasets to capture
      prior information for anomaly detection. However, efficiently extracting useful information from


                                                        1
      Under review as a conference paper at ICLR 2025


054
      individual points or feature matrices poses great challenges. When using a pre-trained model for
055   transfer learning, describing each point in a patch is challenging, increasing memory requirements
056   and complexity, while the model’s weights may also affect patch descriptions and overall feature
057   extraction accuracy Wang et al. (2023); Zhao et al. (2024). Reconstruction-based methods primar-
058   ily focus on the network’s performance gap between reconstructing normal and anomalous point
059   clouds. To leverage the prior knowledge of transfer learning for improved reconstruction, IMR-
060   Net Li et al. (2023) was proposed, aiming to utilize the capabilities of the pre-trained PointMAE for
061   more detailed reconstructions.
062   When employing prior transfer learning for feature embedding and point cloud reconstruction, noise
063   plays a crucial role in the data flow. Traditionally, researchers have viewed noise as an element to
064   be removed from the feature matrix, leading to the development of denoising encoders. Vincent
065   et al. (2008). Based on this concept, many anomaly detection networks have been proposed, such
066   as teacher-student distillation networks, which utilize pre-trained teacher models to further mitigate
067   the effects of noise and obtain more accurate 3D anomaly detection features Rudolph et al. (2022).
068   The R3D-AD method attempts to iteratively remove noise using a distillation model, ultimately
069
      obtaining the original point cloud Zhou et al. (2024). However, complete noise elimination is often
      difficult, and an excessive focus on noise reduction can even degrade performance. Building on this,
070
      noise may require a more meaningful interpretation in the context of anomaly detection.
071
072   Anomaly detection often requires additional information to differentiate between normal and anoma-
073   lous features, with large language models serving as an effective source of supplementary informa-
074   tion Cheng et al. (2024). However, the training cost may present challenges. Noise, as a prior source
075
      of information, consists of a combination of various types of data. By removing irrelevant informa-
      tion, the remaining useful data can enhance anomaly detection. This introduces a new perspective
076
      on the role of noise in 3D anomaly detection: To fully harness the valuable information in the
077
      prior noise, the key lies in developing a method that separates useful information from the
078   complex noise. When useful information is extracted, it can be injected into the feature matrix as
079   supplementary data, thereby improving detection accuracy.
080
081
      To address these challenges, we propose leveraging noise as prior information to improve the gen-
      eralization of traditional feature descriptors by introducing diversity. Incorporating noise into self-
082
      supervised learning expands the inherent feature expressions of normal samples, thereby enhancing
083
      anomaly detection and rotational invariance. We introduce the Information Gain Block Anomaly
084   Detection (IGB-AD), which includes an Information Gain Block (IGB) that retains valuable in-
085   formation in the FPFH feature matrix while assigning error ranges to overcome its generalization
086   limitations. The IGB operates in a self-supervised manner, extracting features independent of reg-
087   istration. Additionally, we propose Packet Downsampling (PD), a memory-efficient method for
088   managing diverse point-level features. Our contributions are summarized as follows:
089
090          • We propose the IGB module, which uses noise as a prior dependency to extend the range
091            of available information for the feature matrix. Based on IGB, we propose IGB-AD frame-
092            work. We propose RIFPS to be responsible for the ordered initial feature matrix extraction.
093
               Furthermore, we propose an IP module based on IGB to inject information into the initial
               feature matrix. Finally, PD is used to perform better downsampling.
094
095          • We introduce the ICD dataset, the first 3D anomaly detection dataset with multiple sub-
096            classes, offering new possibilities and challenges for 3D anomaly detection.
097
             • Our IGB-AD framework leverages noise to overcome the limitations of traditional descrip-
098
               tors, compensating for insufficient information in the feature matrix and alleviating the
099            challenges posed by high intra-class variance. It achieves State-Of-The-Arts (SOTA) per-
100            formance on the Anomaly-ShapeNet dataset with I-AUROC 80.9% and P-AUROC 81.5%,
101            and on our custom ICD dataset, it achieves I-AUROC 60.2% and P-AUROC 57.4%, sig-
102            nificantly improving anomaly detection.

      tion and industrial inspection. Deep learning approaches have gained prominence, using neural

      2.1   E MBEDDING - BASED A PPROACH
112
113   In order to obtain features that can distinguish anomalies, a embedding-based method extracts plaque
114   or point-level features from normal samples and stores them in the memory bank. In the inference
115   stage, extracted features of test samples and compared them with those in the memory bank to
116   obtain anomalies. Embedding-based methods are often based on traditional methods of local feature
117   extraction directly or feature extraction methods from transfer learning.
118   Traditional methods for point cloud feature extraction, such as Fast Point Feature Histograms
119   (FPFH) Szalai-Gindl & Varga (2024), have played a significant role in providing essential feature
120   descriptors for point clouds. While these approaches are effective in capturing local geometric prop-
121   erties, the limited ability to transfer learned features between different point clouds undermines their
122   effectiveness, especially in tasks requiring high generalization, such as anomaly detection.
123   In contrast, embedding-based methods, grounded in deep learning, aim to address these limitations
124   by focusing on learning distinctive representations of normal point clouds directly from raw data.
125   These methods prioritize efficient feature representation in memory banks for anomaly detection
126   and leverage pre-trained models, such as PointNet++, PointMLP, and Point Transformer Qi et al.
127   (2017); Ma et al. (2022); Zhao et al. (2021), which significantly enhance the feature extraction pro-
128   cess. PointMAE Pang et al. (2022) further improves point cloud understanding through masked
129   autoencoders, offering more refined representations. Furthermore, contrastive learning has been
130   proposed as a method to enhance the representation in memory banks Zhu et al. (2024). By in-
131
      corporating this technique, the ability to distinguish between normal and anomalous point clouds
      is substantially improved, providing a more robust solution for anomaly detection tasks. This shift
132
      from traditional methods to deep learning-based approaches represents a significant advancement in
133
      achieving both efficient and generalizable feature representations in point cloud analysis. However,
134   deep learning methods often have large memory consumption, inaccurate description at the point or
135   patch level, and weak differentiation, and still need to be further improved.
136
137   2.2   R ECONSTRUCTION - BASED APPROACH
138
139   Reconstruction-based methods detect anomalies by focusing on the reconstruction error between
140   normal and anomalous point clouds. The surface of point clouds is characterized by indirect and
141   direct representation at different feature levels.
142   Initially, emphasis was placed on reconstructing point clouds in representational space, and accurate
143   original point cloud reconstruction was often not the goal. For instance, methods based on predicting
144   signed distance functions (SDF) were used to indirectly reconstruct surfaces, offering a way to
145   approximate point cloud geometry without requiring exact reconstructions Chu et al. (2023).
146
      More recently, the focus has shifted toward achieving more accurate reconstructions. IMRNet, an
147   extension of PointMAE, reconstructs anomalous clouds into their normal counterparts and detects
148   anomalies by comparing the differences between the original and reconstructed models Li et al.
149   (2023). Due to the thermal diffusion process in the evolving thermodynamic and kinetic system, the
150   researchers proposed a more accurate R3D-AD reconstruction method, which can be more focused
151   on the more subtle reconstruction process of the 3D surface. Zhou et al. (2024). Despite these ad-
152   vances, achieving precise reconstruction remains challenging, as difficulties in accurately capturing
153   complex details often result in suboptimal detection performance. Researchers need a more accurate
154   characterization of normal and abnormal structures.

274
275   In order to get more refined and rich features, we try to use more stacks for continuous perfusion.

277   IP infuses the FPFH feature matrix with prior information from noise via multi-layer IGB. The IP
278   process is formulated as:
279
280
                             F(i) = F(i−1) + IGBi (Z, F(i−1) ),            i = 1, 2, . . . , k.          (8)

282
      where F(0) is the initial FPFH feature matrix, Z is the noise input, and IGBi is the i-th IGB module.
283
      The output F(i) is obtained by adding the output of the previous layer to the current IGB module,
284
      iterating from i = 1 to layer k, yielding the final enhanced feature matrix F(k) . Each IGB layer
285
      enhances F by adding gain information from noise Z, iteratively enriching its representation for
286   anomaly detection.

      4.1   DATASETS
345
346   Comparative experiments were performed on two mainstream datasets: Intra-Class Diversity and
347   Anomaly-ShapeNet datasets.
348   Anomaly-ShapeNet comprises 40 categories, with 1,600+ positive and negative samples. Each cat-
349   egory’s training set contains 4 normal samples, while test sets include both normal and anomalous
350   samples exhibiting various defects.
351   Ours: Intra-Class Diversity (ICD) datasets introduce new challenges for anomaly detection, the
352   training set consists of four morphologically distinct subspecies per class, and the test set is derived
353
      from each of these subspecies. This poses a unique challenge for models, requiring them to effec-
      tively extract features from individual samples while simultaneously capturing the variations across
354
      different subspecies. The dataset consists of four subspecies, with the test set containing between
355
      41 and 64 samples per class, covering both normal and anomalous variations of each subspecies. A
356   partial visualization is shown in Figure 1, and a full description is presented in the appendix A.

457   The selection of Gaussian distribution as the noise model is based on its independent randomness.
458   Under the independent identically distributed (IID) noise hypothesis, each sample is independent
459
      of the others and follows the same probability distribution. This independence ensures that the
      noise does not exhibit spatio-temporal dependence, allowing each noise sample to be modeled in-
460
      dividually. This not only simplifies the calculation, but also meets our expectation that the added
461
      information for each eigenvalue ensures that the sum of multiple independent noise sources will
462   approximate a normal distribution by CLT. Thus, Gaussian noise effectively captures uncertainties
463   in independent random processes, making it a robust and computationally tractable choice for noise
464   modeling. In contrast, other types of noise, such as autoregressive or Poisson noise, introduce de-
465   pendencies or discreteness and lack the simplicity and universality of Gaussian noise.

      5   C ONCLUSION
469
470
471   To address the challenges of insufficient information for anomaly detection and high intra-class
472   variance, we propose the Information Gain Block Anomaly Detection (IGB-AD) framework. The
473   framework first proposes Rotation-Invariant Farthest Point Sam- pling (RIFPS) to ensure the order
474   of the feature matrix, and then proposes an Information Perfusion (IP) composed of multiple In-
475   formation Gain Block (IGB). The IGB layer is a module that can learn noise as prior information.
476   Finally, Packet Downsampling (PD) is proposed to further reduce the influence of intra-class vari-
477
      ance. In order to further verify the ability of the model to face large cases of intra-class variance, we
      also propose an Intra-Class Diversity dataset (ICD). Experimental results demonstrate that the pro-
478
      posed IGB-AD method achieves State-Of-The-Arts (SOTA) performance on the Anomaly ShapeNet
479
      dataset, with P-AUROC of 81.5% and I-AUROC of 80.9%, and outperforms on the ICD dataset, with
480   P-AUROC of 57.4% and I-AUROC of 60.2%. We incorporate noise as prior information into the
481   features through a self-supervised approach to obtain more informative and discriminative repre-
482   sentations. This presents a novel avenue for addressing the challenges of insufficient information
483   and high intra-class variance in anomaly detection. Limitations: Given the inherent randomness
484   and complexity of noise, quantitatively assessing the precise amount of information injected into the
485   feature matrix remains challenging. Establishing a mechanism for controlled information injection
      will thus be a key objective in our future research endeavors.

743   Algorithm 1 Packet Downsampling (PD)
744
      Input: Feature matrix F , number of clusters C, downsampling ratio r
745
      Output: Downsampled feature matrix Fdown
746
       1: Step 1: Group features into C clusters
747
       2: Apply clustering (e.g., k-means) on F to form C clusters: {C1 , C2 , . . . , CC }
748    3: Step 2: Select representative feature from each cluster
749    4: for each cluster Cj do
750    5:     Compute importance score S(fi ) for each feature fi ∈ Cj
751    6:     Select the feature frep with the highest score in cluster Cj
752    7:     Add frep to the downsampled set Fdown
753    8: end for
754    9: Step 3: Return the downsampled feature matrix Fdown
755   10: return Fdown

757   Algorithm 2 Rotation-Invariant Farthest Point Sampling (RIFPS)
758   Input: Point cloud P = {p1 , p2 , . . . , pN }, number of points N , desired sample size M
759   Output: Sampled point set F
760    1: Step 1: Compute the geometric center C of the point cloud
                  1
                    PN
761    2: C ← N        i=1 pi
762    3: Step 2: Calculate the Euclidean distance from each point pi to the center C
763    4: for i = 1 to N do
764    5:     Di ← ∥pi − C∥2
765    6: end for
766    7: Step 3: Select the farthest point pf from the center
       8: pf ← arg maxi Di
767
       9: Step 4: Perform iterative Farthest Point Sampling (FPS)
768
      10: Initialize the sampled set F with the farthest point pf
769
      11: F ← {pf }
770   12: for j = 2 to M do
771   13:     Find the point pnext that maximizes the minimum distance to any point in F
772   14:     pnext ← arg maxp∈P minf ∈F ∥p − f ∥2
773   15:     Add pnext to the sampled set F
774   16:     F ← F ∪ {pnext }
775   17: end for
776   18: Return the sampled point set F

779   Algorithm 3 Information Gain Block (IGB)
780   Input: Noise matrix Z, feature matrix F (i−1)
781   Output: Extracted useful information X
782    1: Step 1: Decompose noise matrix Z
783    2: Decompose Z into useful information X and irrelevant noise Y :
784    3: Z ← X + Y
785    4: Model Y as Gaussian noise: Y ∼ N (0, σ 2 )
786    5: Step 2: Extract useful information X using MLP
787    6: X ← fMLP (Z, F (i−1) )
       7: Step 3: Apply loss functions to ensure relevance of extracted information
788
       8: Compute smoothness loss:
789
       9: Lsmooth ← SmoothL1Loss(F (i−1) , F (i−1) + X)
790
      10: Compute richness loss:
791
      11: Lrichness ← sigmoid(α · mean(Var(F (i−1) + X)))
792
      12: Total loss:
793   13: Ltotal ← β · Lsmooth + λ · (1 − Lrichness )
794   14: Step 4: Return the extracted useful information
795   15: return X

799   Input: Initial feature matrix F (0) , noise matrix Z, number of layers k
800   Output: Enhanced feature matrix F (k)
801    1: Step 1: Initialize feature matrix F (0) ← F initial
802    2: Step 2: Perform stacked IGBs for k layers
803    3: for i = 1 to k do
804    4:     Apply Information Gain Block (IGB) on F (i−1)
805    5:     X (i) ← IGB(Z, F (i−1) )
806    6:     F (i) ← F (i−1) + X (i)
807    7: end for
808    8: Step 3: Return the enhanced feature matrix F (k)
809    9: return F (k)
