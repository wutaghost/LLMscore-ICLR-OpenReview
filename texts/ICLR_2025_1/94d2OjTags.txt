009                                                    A BSTRACT
010
011                Recently, when dealing with high-resolution images, dominant large multimodal
012                models (LMMs) usually divide them into multiple local images and one global im-
013                age, which will lead to a large number of visual tokens. In this work, we introduce
014                AVG-LLaVA, an LMM that can adaptively select the appropriate visual granular-
015                ity based on the input image and instruction. This approach not only reduces the
016                number of visual tokens and speeds up inference, but also improves the overall
017
                   model performance. Specifically, we introduce the following modules based on
                   LLaVA-NeXT: (a) a visual granularity scaler that includes multiple pooling layers
018
                   to obtain visual tokens with different granularities; (b) a visual granularity router,
019
                   which includes a Transformer layer, an MLP layer, and a voter layer, used to
020                select the appropriate visual granularity based on the image and instruction. Fur-
021                thermore, we propose RGLF, a novel training paradigm that aims at aligning the
022                granularity predicted by the router with the preferences of the LMM, without the
023                need for additional manually annotated data. Extensive experiments and analysis
024                show that AVG-LLaVA achieves superior performance across 11 benchmarks, as
025                well as significantly reduces the number of visual tokens and speeds up inference
026                (e.g., an 85.3% reduction in visual tokens and a 2.53× increase in inference speed
027                on the AI2D benchmark). 1

      1       I NTRODUCTION
031
      Recently, the field of artificial intelligence (AI) has witnessed a significant advancement in large
032
      multimodal models (LMMs) (OpenAI, 2023b; Zhu et al., 2023; Dai et al., 2023; Liu et al., 2023a;
033
      2024a), marking a further step toward artificial general intelligence (AGI). Most existing LMMs
034   follow the structure of LLaVA (Liu et al., 2023a; 2024a), which includes a vision encoder to embed
035   images into visual tokens and a connector to map them into the word embedding space. Subse-
036   quently, these visual tokens are fed into a large language model (LLM) (Touvron et al., 2023; Ope-
037   nAI, 2023a; Chiang et al., 2023) for multimodal understanding and reasoning, alongside the word
038   embeddings.
039

040   the fixed aspect ratio (e.g., 1:1) and
041   low resolution (e.g., 336×336) used
042   by visual encoders (e.g., CLIP-ViT
043   (Radford et al., 2021)), earlier LMMs
044   face challenges in processing high-
045   resolution images with different as-
046   pect ratios. To deal with this lim-
047   itation, dominant models, such as

      visual token sequences, such as 2880 visual tokens for a 672×672 image. However, in practice, such
055   fine-grained visual information is not always necessary, and in some cases, coarse-grained visual
056   information can even be more beneficial for model predictions. For instance, as shown in Figure 1,
057   when the model is asked to recognize the number on the jersey, it requires relatively fine-grained
058   visual information. In contrast, determining the color of the jersey only necessitates coarse-grained
059   visual information.
060
      In this paper, we propose Adaptive Visual Granularity LLaVA (AVG-LLaVA), an LMM that can adap-
061   tively select the appropriate visual granularity based on the input image and instruction. The basic
062   intuition behind our model is that humans only scrutinize images carefully when answering difficult
063   questions; otherwise, a brief glance is sufficient. As displayed in Figure 2, AVG-LLaVA extends
064   LLaVA-NeXT with a visual granularity scaler and a visual granularity router. The visual granular-
065   ity scaler performs multiple rounds of pooling on visual tokens without training, each time halving
066   the number of visual tokens, thus obtaining a series of visual features with different granularities.
067   The visual granularity router adaptively selects the appropriate visual granularity features based on
068   the input multi-granularity visual features and text features. By doing so, for images and instructions
069
      that do not require fine-grained details, the number of visual tokens can be reduced, which not only
      speeds up inference but also improves performance. This performance enhancement likely stems
070
      from the reduction of redundant information, as selecting the appropriate visual granularity makes
071
      it easier for the model to answer questions based on images effectively.
072
073   Besides, we observe that it is challenging to train the visual granularity router directly through visual
074   instruction tuning (Liu et al., 2023a). This may be because the router cannot learn the distinctions
075
      between different visual granularities from visual instruction tuning, making it difficult to learn how
      to select the most appropriate visual granularity based on the image and instruction. To deal with
076
      this issue, we propose a novel training paradigm called Ranking Granularity to align LMM Feedback
077
      (RGLF). This paradigm aligns router probabilities of multiple granularities with LMM preferences
078   by a ranking loss (Hadsell et al., 2006; Hopkins & May, 2011; Liu et al., 2022), effectively aiding
079   the router in distinguishing between different visual granularities and selecting the appropriate one.
080
081
      We further evaluate AVG-LLaVA on 11 benchmarks including tasks from various types (e.g., general
      VQA and text-oriented VQA, etc.). Extensive experimental results show that AVG-LLaVA can
082
      effectively reduce the number of visual tokens and improve inference speed (e.g., an 85.3% reduction
083
      in visual tokens and a 2.53× increase in inference speed on the AI2D benchmark) while achieving
084   better performance under the same base LLM.

087   2   R ELATED W ORK
088
089   Large Multimodal Models LLMs such as GPT-4 (OpenAI, 2023a), LLaMA (Touvron et al.,
090   2023), and Gemini (Team et al., 2023) have achieved significant success in language understanding
091   and generation. Benefiting from this, multimodal large models (LMMs) have garnered widespread
092   attention. Flamingo (Alayrac et al., 2022), BLIP-2 (Li et al., 2023b), and LLaMA-adapter (Zhang
093   et al., 2023) integrate a frozen visual encoder and trainable modules into a LLM, extending it into
094
      a LMMs. These models are then fine-tuned using plain image-text pairs, enabling them to process
      and perceive visual content. To further improve instruction-following abilities and response quality,
095
      LLaVA (Liu et al., 2023a) fine-tunes the entire model using visual instruction data generated by
096
      GPT-4. However, since these LMMs rely on CLIP-ViT to process images at a fixed resolution (e.g.
097   336x336), it hinders the LMMs from perceiving image details at higher resolutions.
098
099
      High-Resolution LMMs To perceive images with higher resolutions, Qwen-VL (Bai et al., 2023)
100
      increases the input resolution of the visual encoder to 448×448 and introduces an additional training
101
      stage. Along this line, both Vary (Wei et al., 2023) and Mini-Gemini (Li et al., 2024a) include two
102   vision encoders: one is an additional introduced high-resolution vision encoder, and the other is the
103   original low-resolution vision encoder. Unlike the methods mentioned above, SPHINX (Lin et al.,
104   2023) and Monkey (Li et al., 2024b) enlarge the input image to a high resolution, and then divide
105   it into a fixed number of local images, which are individually encoded using an image encoder to
106   obtain local image tokens. Subsequently, the original global image tokens are concatenated with all
107   local image tokens to feed into the LLM. Furthermore, LLaVA-NeXT (Liu et al., 2024b) enumer-
      ates various resolutions and adaptively selects the one that most closely matches the input image

133
      Vision Token Reduction for LMMs Recently, several methods are proposed to reduce the visual
134
      tokens for LMMs, including visual token pruning and merging. For example, CrossGET (Shi et al.,
135   2023) introduces a cross-modal token for leveraging cross-modal information to make decisions on
136   token selection and merging. LLaVA-PruMerge (Shang et al., 2024) employs the similarity between
137   the class token and other tokens as a key criterion for pruning and merging vision tokens. Unlike the
138   aforementioned methods, FastV (Chen et al., 2024b) finds that most image tokens receive inefficient
139   attention after the second decoder layer, and thus prunes half of the image tokens. Similarly, VTW
140   (Lin et al., 2024b) adopts a more aggressive strategy to prune all visual tokens at a certain layer.
141   Unfortunately, despite the above methods effectively reducing the number of visual tokens, they
142   often lead to a certain degree of decline in model performance. More recently, LLaVA-M 3 (Cai et al.,
143
      2024) obtains multi-granularity visual features by merging visual tokens through pooling, enabling
      manual control of the tradeoff between inference cost and performance. Significantly different from
144
      these methods, our model can adaptively select the appropriate visual granularity based on the input
145
      image and instruction, improving model performance while reducing visual tokens.

150   3.1   M ODEL A RCHITECTURE
151
      As shown in Figure 2, in addition to the visual encoder, visual-language connector, and LLM, AVG-
152
      LLaVA introduces two additional modules on top of LLaVA-NeXT: the visual granularity scaler and
153
      the visual granularity router. The key components will be elaborated in the following.
154
155
      High-Resolution Image Encoding Given an input image I ∈ RH×W ×3 , we follow common prac-
156   tice (Liu et al., 2024b) to divide it into multiple smaller local images Ilocal ∈ RHv ×Wv ×3 , where Hv
157   and Wv are the resolution that the vision encoder is originally trained for. Then, these local images
158   are individually encoded into a Hp × Wp grid of visual tokens Xlocal ∈ RHp ×Wp ×C by the image
159   encoder, where C is the dimension of the visual encoder. To preserve the global context information
160   of the input image, we resize the original image to (Hv , Wv ) and encode it as global visual tokens.
161   Finally, we map both global visual tokens and local visual tokens to the word embedding space
      through an MLP-based vision-language connector.

175   Visual Granularity Scaler This module sequentially stacks 1×2 and 2×1 average pooling layers,
176   thereby obtaining visual features at multiple granularities and preserving the spatial information. In
177   this work, we consider CLIP-ViT-L-336 (Radford et al., 2021) as the visual encoder, and thus each
178   image is encoded into 24×24 grid of visual tokens. Then, these visual tokens are fed into the
179   visual granularity scaler, obtaining visual tokens with a grid of 24×12, 12×12, 12×6 and 6×6,
180   respectively. In this way, we can obtain visual tokens of different granularities in a fine-to-coarse
181   manner without training.
182
183   Visual Granularity Router Different visual granularity features can be considered as different
184   experts, so the Mixture of Experts (MoE) (Shazeer et al., 2017; Komatsuzaki et al., 2023; Lin et al.,
185   2024a) structure is particularly well-suited for selecting the appropriate visual granularity. Unlike
186
      the previous MoE studies that use linear layers as routers, we propose a multi-layer structure as
      illustrated in Figure 2 to select the appropriate visual granularity based on the input image and the
187
      instruction. Specifically, when dealing with an image, we first flatten and concatenate its visual
188
      tokens of all granularities to form multi-granularity visual tokens Xv = [X1v ; X2v ; ...; XN
                                                                                                  v ], where
189
      Xiv represents the visual tokens of the i-th granularity, and N is the number of visual granularities2 .
190
      Then, these visual tokens are concatenated with the filtered instruction tokens Xinstruct to serve
191   as the input for the visual granularity router. Here, Xinstruct is obtained by calculating the co-
192   sine similarity between the original instruction tokens Xinstruct and the visual tokens with original
193   granularity Xv , retaining the top-k most relevant ones. Afterwards, we apply a single Transformer
194   (Vaswani et al., 2017) layer to facilitate the fusion of visual tokens at different granularities with
195   instruction tokens. Subsequently, an MLP is applied to each token to predict the appropriate vi-
196   sual granularity, resulting in the logits Zout ∈ RL×N , where L is the number of both visual and
197   instruction tokens. To vote for the most appropriate visual granularity, we use a learnable weight
198   matrix (Voter) W ∈ R1×L to aggregate the logits predicted by all tokens, yielding the final logits
199
      Zf inal ∈ R1×N . Finally, we use softmax to calculate the probability distribution of each visual
      granularity, where the visual tokens corresponding to the granularity with the highest probability are
200
      fed into the LLM.

304   Training Datasets Note that in this work, we mainly focus on investigating the effectiveness of
305   automatic visual granularity selection in reducing the number of visual tokens and improving model
306   performance. Therefore, during the first stage, we also use CC-595K image-text pairs for model
307   training, as implemented in LLaVA-NeXT (Liu et al., 2024b). In the subsequent training stages,
308   we also hope to use the same data as LLaVA-NeXT. However, the real user interaction data used
309
      for visual instruction fine-tuning in LLaVA-NeXT are not open-sourced, so we opt to extract 200K
      samples from ALLaVA (Chen et al., 2024a) dataset as a substitute. Although LLaVA-NeXT re-
310
      places TextVQA (Singh et al., 2019) with DocVQA (Mathew et al., 2021) and SynDog-EN (Kim
311
      et al., 2022), the TextVQA has already been included in the training data of most existing LMMs.
312   Consequently, we choose to retain it to ensure a fair comparison with other models.3 In total, the
313   visual instruction fine-tuning data we use contains 1M image-text pairs.
314
315   Implementation Details Following Liu et al. (2024b), we use CLIP ViT-L/14 as the visual en-
316   coder, Vicuna-7B (Chiang et al., 2023) as the LLM. We set the filtered instruction token number k
317   to 32 and the cross-entropy loss weight α to 0.1. The impact of these two hyperparameters on model
318   performance is discussed in Section 4.6. In our first two stages, we adopt the same training settings
319   as LLaVA-NeXT for pre-training and visual instruction fine-tuning. In the third stage, the learning
320   rates for the visual encoder and other modules are set to 2×10−5 and 1×10−5 , respectively, with
321
      a batch size of 128. In the fourth stage, the learning rate for the visual granularity router is set to
      1×10−3 , with a batch size of 128. More details of the training process are provided in Appendix

      5     C ONCLUSION
524
525
      In this work, we propose AVG-LLaVA, an LMM that can adaptively select appropriate visual granu-
      larity based on input image and instruction. AVG-LLaVA builds upon LLaVA-NeXT by introducing
526
      a visual granularity scaler and a visual granularity router, which are used to obtain multi-granularity
527
      visual features and select the appropriate visual granularity based on image and instruction, re-
528   spectively. Besides, we introduce RGLF, which aligns router-predicted probabilities of multiple
529   granularities with LMM preferences by a ranking loss, effectively helping the model learn to distin-
530   guish between different granularities based on image and instruction. Experimental results show that
531   AVG-LLaVA not only exhibits superior performance across 11 benchmarks, but also significantly
532   reduce the number of visual tokens and speed up inference in tasks that do not require fine-grained
533   information. Particularly, on the AI2D benchmark, it reduces the number of visual tokens by 85.3%
534   and speeds the inference by 2.53×. We hope our work can inspire more attention to the visual
535   granularity in LMMs.
536   While AVG-LLaVA has achieved good results, there is still considerable potential to be further ex-
537   plored. On text-intensive benchmarks, the model tends to select the finest-grained visual tokens,
538   which may be due to the pooling directly reducing half of the tokens, resulting in significant differ-
539   ences in granularity size. In the future, we plan to design a more suitable granularity scaling network
      to provide richer visual granularities.
