011                                               A BSTRACT
012
013              We provide an efficient O(n2 ) implementation for solving the all pairs minimax
014              path problem or widest path problem in an undirected dense graph. It is a code
015              implementation of the Algorithm 4 (MMJ distance by Calculation and Copy) in
016
                 a previous paper. The distance matrix is also called the all points path distance
                 (APPD). We conducted experiments to test the implementation and algorithm,
017
                 compared it with several other algorithms for solving the APPD matrix. Result
018
                 shows Algorithm 4 works good for solving the widest path or minimax path APPD
019              matrix. It can drastically improve the efficiency for computing the APPD ma-
020              trix. There are several theoretical outcomes which claim the APPD matrix can
021              be solved accurately in O(n2 ) . However, they are impractical because there is
022              no code implementation of these algorithms. It seems Algorithm 4 is the first al-
023              gorithm that has an actual code implementation for solving the APPD matrix of
024              minimax path or widest path problem in O(n2 ), in an undirected dense graph.

      1       I NTRODUCTION
028
029
      The minimax path problem is a classic problem in graph theory and optimization. It involves finding
      a path between two nodes in a weighted graph such that the maximum weight of the edges in the
030
      path is minimized. 1
031
032   Given a graph G = (V, E) where V is the set of vertices and E is the set of edges, each edge
033   e ∈ E has a weight ew . For an undirected graph with n vertices, the maximum number of edges is
      n(n−1)
034      2   . A dense graph has close to n(n−1)
                                              2    edges. We can say a dense graph has O(n2 ) edges. In
035   an undirected graph, each edge is bidirectional, meaning it connects two vertices in both directions.
036   The objective of the minimax path problem is to find a path P from a starting node i to a destination
037   node j such that the maximum weight of the edges in the path P is minimized. A minimax path
038   distance between a pair of points is the maximum weight in a minimax path between the points
039   (Equation 2).
040
041                                   Φ = {max weight(p) | p ∈ Θ(i,j,G) }                               (1)
042
043
044                                       M (i, j | G) = min(Φ)                                       (2)
045   where G is the undirected dense graph. Θ(i,j,G) is the set of all paths from node i to node j. p is
046   a path from node i to node j, max weight(p) is the maximum weight in path p. Φ is the set of all
047   maximum weights. min(Φ) is the minimum of Set Φ Liu (2023).
048
      The distance can also be called the longest-leg path distance (LLPD) Little et al. (2020) or Min-
049
      Max-Jump distance (MMJ distance) Liu (2023). The all pairs minimax path distances calculate
050
      the distance between each pair of points in a dataset X or graph G . It is also called all points
051   path distance (APPD) Little et al. (2020). It is a matrix of shape n × n. A dataset X can be
052   straightforwardly converted to a complete graph.

087   We can use a modified version of the Floyd–Warshall algorithm to solve the APPD in both directed
088   and undirected dense graphs Weisstein (2008), or use the Algorithm 1 (MMJ distance by recursion)
089   in Liu (2023), both of them take O(n3 ) time. However, in an undirected dense graph, we have a

091   oretical outcomes which claim the APPD matrix can be solved accurately in O(n2 ) Sibson (1973);
092
      Demaine et al. (2009; 2014); Alon and Schieber (2024). However, there is no code implementation
      of these algorithms, which implies they are impractical.
093
094   Code implementation is the process of translating a design or algorithm into a programming lan-
095   guage. It is critical in algorithm design where ideas are turned into practical, executable code that
096   performs specific tasks.
097   In section 4.3 (MMJ distance by calculation and copy) of Liu (2023), Liu proposes an algorithm
098   which also claims to solve the APPD matrix accurately in O(n2 ), in an undirected dense graph. The
099   algorithm is referred to as Algorithm 4 (MMJ distance by Calculation and Copy). In the paper, the
100   algorithm is left unimplemented and untested. In this paper, we introduce a code implementation of
101   Algorithm 4, and test it.
102   The widest path problem is a closely related topic to minimax path problem. In contrary, The
103   objective of the widest path problem is to find a path P from a starting node s to a destination
104   node t such that the minimum weight of the edges in the path P is maximized. Any algorithm
105   for the widest path problem can be easily transformed into an algorithm for solving the minimax
106   path problem, or vice versa, by reversing the sense of all the weight comparisons performed by
107   the algorithm. Therefore, we can roughly say that the widest path problem and the minimax path
      problem are equivalent.

      2     RELATED WORK
126
127
      Numerous distance measures have been proposed in the literature, including Euclidean distance,
128
      Manhattan Distance, Chebyshev Distance, Minkowski Distance, Hamming Distance, and cosine
129   similarity. These measures are frequently used in algorithms like k-NN, UMAP, and HDBSCAN.
130   Euclidean distance is the most commonly used metric, while cosine similarity is often employed
131   to address Euclidean distance’s issues in high-dimensional spaces. Although Euclidean distance is
132   widely used and universal, it does not adapt to the geometry of the data, as it is data-independent.
133   Consequently, various data-dependent metrics have been developed, such as diffusion distances
134   Coifman and Lafon (2006); Coifman et al. (2005), which arise from diffusion processes within a
135   dataset, and path-based distances Fischer and Buhmann (2003); Chang and Yeung (2008).
136   Minimax path distance has been used in various machine learning models, such as unsupervised
137   clustering analysis Little et al. (2020); Fischer et al. (2001; 2003); Fischer and Buhmann (2003),
138   and supervised classification Chehreghani (2017); Liu (2023). The distance typically performs well
139   with non-convex and highly elongated clusters, even when noise is present Little et al. (2020).
140
141   2.1   C ALCULATION OF MINIMAX PATH DISTANCE
142
143   The challenge of computing the minimax path distance is known by several names in the literature,
144   such as the maximum capacity path problem, the widest path problem, the bottleneck edge query
145   problem Pollack (1960); Hu (1961); Camerini (1978); Gabow and Tarjan (1988), the longest-leg
146   path distance (LLPD) Little et al. (2020), and the Min-Max-Jump distance (MMJ distance) Liu
147
      (2023).
148   A straightforward computation of minimax path distance is computationally expensive due to the
149   large search space Little et al. (2020). However, for a fixed pair of points x and y connected in a
150   graph G = G(V, E), the distance can be calculated in O(|E|) time Punnen (1991).
151   A well-known fact about minimax path distance is: “the path between any two nodes in a mini-
152   mum spanning tree (MST) is a minimax path.”Hu (1961) With this conclusion, we can simplify an
153   undirected dense graph into a minimum spanning tree, when calculating the minimax path distance.
154
155

160   Tarjan (1988). The resulting APPD may not be accurate when calculating with bottleneck span-
161   ning tree, because a MST (minimum spanning tree) is necessarily a MBST (minimum bottleneck
      spanning tree), but a MBST is not necessarily a MST. A variant of the Floyd-Warshall algorithm

      3       I MPLEMENTATION OF THE ALGORITHM
217
218
      As described in Section 1, the Algorithm 4 (MMJ distance by Calculation and Copy) in Liu (2023)
219   also claims to solve the APPD matrix accurately in O(n2 ), in an undirected dense graph. But it is left
220   unimplemented and untested. Figure 1a is Algorithm 4 (MMJ distance by Calculation and Copy) in
221   Liu (2023), for convenience of reading, we re-post it here. Figure 1b is its python implementation.
222
      Note the three embedded for-loops make it look like an O(n3 ) algorithm, but it is actually an O(n2 )
223
      algorithm. Because when the variable i in Line 21 is small, both tree1 and tree2 are of size O(n);
224
      but when the variable i is large, both tree1 and tree2 are of size O(1). The final net effect is that the
225   three embedded for-loops only access each cell of the APPD matrix only once. Therefore, it is an
226   O(n2 ) algorithm.
227
228
      In the implementation, we first construct a minimum spanning tree (MST) of the undirected dense
      graph. The complexity of constructing a MST with prim’s algorithm is O(n2 ). Then, we sort the
229
      edges of the MST in descending order. It is critical to remove the edges from the MST one-by-one,
230
      from large to small. Only by this we can get the two sub-trees, tree1 and tree2. By traversing each
231   sub-tree, nodes of the two sub-trees can be obtained, respectively.

235
236   In an experiment, we tested the Algorithm 4 (MMJ distance by Calculation and Copy) on seven
237   datasets with different number of data points, note a dataset can be easily converted to a complete
238
      graph. The performance of Algorithm 4 is compared with three other algorithms that can calculate
      the APPD matrix.

      shortest path between each pair of nodes, then compute the maximum weight on the shortest path.
      Its complexity is O(n3 log(n)). Figure 3 is its python implementation. The implementation is based

276
277   5.1      M ERIT OF A LGORITHM 1
278
279
      Algorithm 1 (MMJ distance by recursion) has a merit of warm-start. Suppose we have calculated the
      APPD matrix MG of a large graph G, then we got a new point (or node) p, where p ∈     / G. The new
280
      graph is noted G + p. To calculate the APPD matrix of graph G + p, if we use other algorithms, we
281
      may need to start from zero. Algorithm 1 has the merit of utilizing the calculated MG for computing
282   the new APPD matrix, with the conclusions of Theorem 3.3., 3.5., 6.1., and Corollary 3.4. in Liu
283   (2023). This is especially useful when the graph is a directed dense graph, where starting from
284   zero needs O(n3 ) complexity, but a warm-start of Algorithm 1 (MMJ distance by recursion) only
285   needs O(n2 ) complexity. We can say Algorithm 1 supports online machine learning3 , in which data
286   becomes available in a sequential order.
287
288   5.2      U SING PARALLEL PROGRAMMING
289
290   If speed is the main concern of calculating the APPD matrix, we can use parallel programming to
291   accelerate Algorithm 4. Firstly, we can use different processors for traversing the tree1 and tree2 in
292   Line 25 and 26 of Figure 1b. Secondly, we can copy the minimum spanning tree (MST) to many
293   processors. For the nth processor, we just remove the n largest edges, obtaining the nth tree1 and
294
      tree2, traversing them, then fill in the corresponding positions of the APPD matrix that are decided
      by the nth tree1 and tree2.

297   6       C ONCLUSION
298
299   We implemented the Algorithm 4 (MMJ distance by Calculation and Copy) that was introduced
300   by Liu in a previous paper. Then tested the implementation and compared it with several other
301   algorithms that can calculate the all pairs minimax path distances, or also called the all points path
302   distance (APPD). Experiment shows Algorithm 4 works good for solving the widest path or minimax
303   path APPD matrix. As an algorithm of O(n2 ) complexity, it can drastically improve the efficiency
304   of calculating the APPD matrix. Note algorithms for solving the APPD matrix are at least in O(n2 )
305   complexity, because the matrix is an n × n matrix.
306   In Section 2.3.3. of the paper “Path-Based Spectral Clustering: Guarantees, Robustness to Outliers,
307   and Fast Algorithms,” Little et al. (2020) Dr. Murphy and his collaborators write:

309
      of O(min{n2 log(n)+n|E|, n|E|log(n)}). However the APPD distance matrix can be computed in
310   O(n2 ), for example with a modified SLINK algorithm (Sibson, 1973), or with Cartesian trees (Alon
311   and Schieber, 1987; Demaine et al., 2009, 2014). ”
