009                                                A BSTRACT
010
011
               As the demand for intelligent robots and cognitive agents rises, the ability to re-
012            tain and utilize past experiences through episodic memory has become crucial,
013            especially for social companion robots that rely on previous interactions for task
014            execution. To address this, we introduce Episodic Memory for Cognitive Agents
015            (EMCA), a novel framework that advances knowledge representation by integrat-
016            ing real-world interactions. EMCA enables agents to adapt to complex environ-
017            ments by learning from tasks, interacting with humans, and processing multimodal
018            data—such as speech, vision, and non-verbal cues—without pretraining on spe-
019            cific scenarios. EMCA models episodic memory through a graph-based structure ,
020
               allowing for incremental storage and retrieval of experiences. Each interaction or
               event enriches the memory graph, supporting continuous learning and adaptation
021
               without extensive retraining. This human-like memory formation optimizes the
022
               agent’s ability to retrieve relevant information for tasks like localization, planning,
023            and reasoning based on prior experiences. Unlike conventional models relying on
024            temporal markers or recurrent patterns, EMCA encodes data like human mem-
025            ory, allowing reasoning across diverse scenarios regardless of temporal patterns.
026            The framework dynamically builds a memory graph with semantic and temporal
027            connections based on the agent’s experiences, promoting flexible temporal reason-
028            ing. It also introduces mechanisms for clustering new memories and a dynamic
029            retrieval policy that adjusts based on context or query type, ensuring robustness
030            even in unpredictable scenarios. Empirical tests show EMCA adapts effectively
031
               to real-world data, offering reliability and flexibility in dynamic environments.

121   psychology, specifically the ’what’, ’where’, and ’when’ (WWW) components of episodic mem-
122   ory. This approach highlights the agent’s ability to independently capture multimodal data—visual
123   and auditory—to build a comprehensive understanding of its environment. As depicted in Figure
124   2, the system comprises two primary stages: Experience Memory Collection, where the agent au-
125   tonomously compiles and stores experiences in a knowledge base without pretraining, and Memory
126
      Retrieval, where it recalls relevant experiences for question answering and reasoning. This method
      enhances adaptability by utilizing prior experiences to inform decision-making. The episodic mem-
127
      ory framework is applicable in diverse contexts, such as memory localization and experience-based
128
      recommendation. Detailed methodology is given in Appendix B

      3.3.3   E DGE C ONNECTION
258
259   Contextual edges between episodes are formed by analyzing shared clusters based on common
260   characters, locations, or events. The similarity between two episodes E1 and E2 is deter-
261   mined by the number of overlapping clusters, with an edge created if two or more clusters are
262   shared. The edge weight is computed by combining the location, character, and event similari-
263   ties: Weight(E(N1 , N2 )) = L(N1 , N2 )∥C(N1 , N2 )∥E(N1 , N2 ), where L(N1 , N2 ), C(N1 , N2 ),
264   and E(N1 , N2 ) represent location, character, and event similarities, respectively. The primary
265   connection between episodes is established by the maximum similarity across these features:
266
      Link(E(N1 , N2 )) = arg max ({L(N1 , N2 ), C(N1 , N2 ), E(N1 , N2 )}).
267   Temporal Edges are defined by the time relationship between consecutive episodes. The temporal
268   connection between episodes Et−1 and Et is represented as Tedge (Et−1 , Et ) = 1. Temporal edges
269   help handle missing timestamps, as no traditional statistical methods are used to estimate them.
      Instead, each episode is treated as representing a distinct day, and the temporal indexer is updated

304   3.4     I MPACT OF N ODE R EMOVAL ON C ONNECTIONS
305
306
      When an episode, such as E2 , is removed from the dynamic graph as part of the forgetting mech-
      anism, its associated semantic links are eliminated, disrupting connections. However, the temporal
307
      edges are readjusted, allowing E3 to connect to the next relevant node, preserving the chronological
308
      structure. Consequently, E3 updates its temporal link to E1 , maintaining event sequence continuity.
309   This selective removal of semantic associations, while preserving temporal coherence, eliminates
310   the need for a full graph reevaluation. The remaining nodes retain contextual relevance, enabling
311   the agent to reason based on prior experiences. Thus, implementing a forgetting mechanism does
312   not compromise the integrity of the episodic memory graph. We have tested node removal mech-
313   anism by using frequency based weight decay to prune nodes as described in Appendix C. Results
314   after node pruning are given in E.2.2

      5   DATASET
340
      We propose a comprehensive dataset framework designed to evaluate and enhance episodic memory
341
      systems in artificial agents. This framework integrates multiple datasets, including a custom set of
342
      episodic questions based on the TV series The Big Bang Theory, spanning all nine seasons (181
343   episodes). The aim is to assess memory recall and narrative understanding in complex scenarios.
344
345   We introduce the Agent Dataset, a 10-episode time-series dataset created in Unity3D, where a virtual
346
      agent performs tasks and interacts with characters in realistic environments, simulating the role
      of companion robots. This dataset emphasizes the importance of multi-sensory inputs and task
347
      execution, challenging the agent to process and integrate information from dialogues and visual
348
      cues to maintain task order and achieve context-driven objectives.
349
350   Additionally, we adapted the Ego4D dataset, restructuring its activity sequences into simulated
351   chronological episodes to address the original absence of time-series data—portraying an agent
352
      performing a series of activities over 30 days. We also combined group activity videos designed
      for active speaker recognition. This transformation enables episodic queries such as ”Where did I
353
      place the agricultural tool on the last day of farming?”, enhancing the ability to localize and retrieve
354
      temporal experiences effectively.
355
356   Together with the PerLTQA Du et al. (2024) and LLQA Dolan & Brockett (2005) datasets,
357   which test essential episodic memory dimensions—”what” (context), ”when” (time), and ”where”
358
      (place)—this framework forms a robust benchmark for evaluating advanced episodic memory capa-
      bilities in AI systems.
359
360   Data Annotation: The data was carefully annotated to tag scene information and identify charac-
361   ters in dialogues, ensuring that the model could recognize character presence and understand related
362   events. This included explicitly tagging scene details for location identification and differentiating
363   characters present in the scene versus those mentioned. Events within dialogues were also meticu-
364
      lously annotated to capture key details, facilitating effective memory representation beyond simple
      summaries. Capturing these essential details is crucial for episodic memory tasks, as it allows the
365
      agent to recall past experiences accurately. Each episode was annotated with 10 what, when, and
366
      where questions.
367
368   Data Statistics: The dataset includes a distribution of question types: temporal questions make up
369   24%, spatial questions 38%, contextual questions 18%, multimodal questions (integrating visual
370
      and auditory information) 10%, and dialogue-based questions 10%. These detailed annotations
      enable the model to handle temporal, spatial, and contextual elements, as well as multimodal inputs,
371
      ensuring comprehensive event recognition and effective interaction.

526
      The EMCA system is designed to understand and process temporal timescales in a manner similar
527
      to human cognition. It dynamically organizes events along subjective timescales, allowing it to track
528   and retrieve memories based on the relative importance of events rather than fixed timestamps. This
529   enables the system to adapt to varying time-frames, understanding how past experiences may influ-
530   ence present contexts. By mimicking human-like temporal reasoning, EMCA can handle queries
531   about ”when” events occurred in both short-term and long-term memory, adjusting its responses
532   based on the perceived significance of past events, much like human memory recalls important mo-
533   ments more vividly than routine occurrences.
534   Social Impact: The Episodic Memory Cognitive Agent has the potential to serve as a valuable so-
535   cial agent, particularly for individuals with memory-related disorders. However, without robust data
536
      protection mechanisms, there is a risk of data breaches, which could have serious privacy implica-
      tions.
537
538   Limitations: Currently, EMCA lacks a forgetting mechanism and the ability to identify key events,
539   as humans do, based on factors such as surprise, novelty, or emotional significance. These aspects
      will be addressed in future work.

661   B     D ETAILED M ETHODOLOGY
662
663   EMCA’s methodology for collecting episodic experiences in robotic cognition draws inspiration
664   from the human brain’s mechanisms for encoding sensory information, particularly the distinct roles
665   of the occipital and temporal lobes. In human cognition, the occipital lobe processes visual stimuli,
666   while the temporal lobe is responsible for auditory information. Despite these processes occurring
667   in specialized regions, the brain synchronizes these sensory inputs within a unified temporal frame-
668   work, enabling the formation of cohesive and contextually rich memories. EMCA replicates this
669   principle by employing separate pipelines for processing visual data (e.g., spatial and object recog-
670
      nition) and auditory data (e.g., speech and environmental sounds), which are then temporally aligned
      to construct a coherent representation of the agent’s environment.
671
672   The system comprises two core stages: Experience Memory Collection and Memory Retrieval.
673   During Experience Memory Collection, the agent autonomously gathers and stores multimodal ex-
674   periences in a structured knowledge base without requiring prior training. In the Memory Retrieval
675   phase, the agent leverages these stored experiences to answer queries, contextualize new events, and
676
      reason effectively based on past interactions. By integrating visual and auditory modalities within
      the same temporal window, EMCA achieves a sophisticated level of synchronization and contextual
677
      understanding, akin to human episodic memory.
678
679   This approach enhances the adaptability and decision-making capabilities of the agent by enabling
680   it to draw upon prior experiences. Furthermore, the episodic memory system facilitates advanced
681   applications such as memory localization associating specific memories with spatial and temporal
682
      contexts—and personalized recommendations based on historical data. The integration of biolog-
      ically inspired memory encoding principles with robotic cognition underscores EMCA’s potential
683
      for advancing human-like reasoning in artificial systems.
