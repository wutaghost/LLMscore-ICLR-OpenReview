009                                                A BSTRACT
010
011            Intelligent agents based on large language models have demonstrated certain pro-
012            gramming abilities, but there is still significant room for improvement in com-
013            plex project-level debugging tasks. Previous work has utilized general multi-agent
014            workflows to enhance performance but has the following issues: 1) excessive re-
015            liance on the reasoning capabilities of large language models without debugging
016
               and detailed analysis of the code; 2) lack of intrinsic code information, such as
               call relationships and dependencies; 3) insufficient analysis and optimization of
017
               critical stages, especially the code search capability in fault localization, which
018
               directly affects the effectiveness of subsequent stages. Based on the SWE-bench
019            dataset, we first isolate the fault localization capability for separate analysis and
020            experiments, and introduce program call graphs to demonstrate the effectiveness
021            of this information for debugging. Furthermore, during the debugging phase, we
022            propose a simulated debugging mode that enables large language models to sim-
023            ulate program debugging without relying on other debugging tools. Compared
024            to the real machine debugging mode, our experiments prove the effectiveness and
025            generality of the simulated debugging mode. We conducted experiments on SWE-
026            bench and improved the resolution rate by approximately 27.3%, demonstrating
027
               the potential of this method.

029   1   I NTRODUCTION
030
031   In recent years, the development of large language models (LLMs) has revolutionized the field
032   of artificial intelligence, enabling intelligent agents with remarkable language understanding and
033   generation capabilities. These LLM-based agents have shown promising results in programming
034   tasks, including code generation, comprehension, and completion (Brown et al., 2020; Chen et al.,
035   2021). However, when it comes to complex project-level debugging tasks, the performance of these
036   agents still falls short of human expert level (Feng et al., 2020).
037   Previous work has explored the use of generic multi-agent workflows to enhance the performance
038   of LLM-based programming agents. For example, OpenAI’s Codex (Chen et al., 2021) and Deep-
039   Mind’s AlphaCode (Li et al., 2022) have demonstrated impressive code generation capabilities by
040   leveraging large-scale pre-training on code repositories. However, these approaches heavily rely
041   on the reasoning capabilities of LLMs without conducting in-depth code analysis and debugging.
042   Moreover, they often overlook intrinsic code information, such as call relationships and dependen-
043   cies, which can provide valuable insights for debugging (Allamanis et al., 2014).
044   Another key limitation of existing approaches is the lack of focused analysis and optimization of
045   critical stages in the debugging process. In particular, the code search capability in fault localization
046   plays a crucial role in determining the effectiveness of subsequent debugging stages (Wong et al.,
047   2016). Previous research on automated software debugging has emphasized the importance of fault
048   localization techniques, such as spectrum-based fault localization (Abreu et al., 2007) and learning-
049   to-rank methods (Xuan & Monperrus, 2014), in improving debugging efficiency. However, the
050
      integration of these techniques with LLM-based programming agents remains largely unexplored.
051   To address these limitations, we propose a novel approach that leverages program semantic informa-
052   tion to enhance the performance of LLM-based programming agents in complex debugging scenar-
053   ios. Our work builds upon the growing body of research on intelligent code analysis and automated
      software debugging (Pradel & Sen, 2018; Dinella et al., 2020), aiming to investigate the potential

141
142   Our proposed method as shown in figure 2 aims to enhance the performance of large language
143   models based agents in complex project-level debugging tasks. The method consists of three main
144   stages: Fault Localization, Patch Generation. Below, we detail the approaches and algorithms used
145   in each stage.
146
147
      2.1     FAULT L OCALIZATION
148
149
      Fault localization is the first critical stage in our debugging framework, where we aim to identify the
150
      parts of the codebase that are most likely to contain faults. We achieve this through a multi-faceted
151
      approach that incorporates Abstract Syntax Trees (ASTs), Call Graphs, and Spectrum-Based Fault
152   Localization (SBFL).
153
154
155   2.1.1     S YMBOL I NDEXING C ONSTRUCTION
156
157   We construct an AST for the given codebase to understand its syntactic structure. Each node in
158
      the AST represents a symbol or a construct in the code. We create an index for these symbols to
      facilitate quick lookup and retrieval of relevant code segments.
159
160   The use of ASTs and symbol indexing allows for efficient code analysis and navigation, enabling
161   the agent to quickly identify relevant code segments based on the issue description and test case
      information.


                                                         3
      Under review as a conference paper at ICLR 2025


162
      2.1.2    C ALL G RAPH C ONSTRUCTION
163
164   We generate a call graph to capture the reference relationships between different parts of the code.
165   This graph helps in understanding the flow of execution and the dependencies between functions
166   and methods.
167   The call graph provides crucial information for fault propagation analysis and helps the agent in
168   tracing the root cause of the issue. By understanding the dependencies between code components,
169   the agent can efficiently navigate through the codebase and identify potential fault locations.
170
171   2.1.3    S PECTRUM -BASED FAULT L OCALIZATION (SBFL)
172
173   Using the test cases, we apply SBFL to pinpoint the fault locations. SBFL assigns a suspiciousness
174   score to each line of code based on its association with failing test cases.
175

180   all test cases, and wi,t is the weight of line i with respect to test case t.
181
182
      SBFL leverages the execution information from test cases to guide the fault localization process. By
      prioritizing code segments that are more likely to be associated with failing test cases, SBFL helps
183
      the agent focus on the most suspicious parts of the codebase.
184
185
      2.1.4    R EFINEMENT OF FL P ERFORMANCE
186
187   In order to enhance the accuracy of fault localization, we have researched how to better understand
188   and utilize the information recalled by SBFL. Our goal is to improve the precision and recall of the
189   fault localization process.
190
      To improve recall rate, we focus on refining symbol indexing and reference analysis techniques, by
191   accurately mapping the problem description to relevant code segments and understanding the depen-
192   dency relationships between code components, we can reduce false positives in fault localization.
193
194
      To improve precision rate, our goal is to score and rank candidate code segments through SBFL,
      giving priority to code segments that are most associated with failing test cases.
195
196
197   2.2     PATCH G ENERATION
198
      In the patch generation phase, once the faulty code blocks have been identified, we proceed to the
199
      stage of generating patches. Previous approaches at this stage primarily involved direct generation
200   based on context by Large Language Models (LLMs), which, however, fall short in conducting a
201   detailed internal analysis of the program. Therefore, we simulate a debugging process to create fixes
202   for the identified issues.
203
204
      2.3     V IRTUAL E XECUTION D EBUGGING
205
206   For each identified code block, we locate the corresponding test cases and determine the entry points
207   of the blocks. We then carry out virtual execution debugging to analyze the behavior of the code.
208   Virtual execution debugging allows the agent to step through the code and analyze the program state
209   at each step. By simulating the execution flow and observing key variable values and control flow,
210   the agent can gain a deeper understanding of the code behavior and identify the root cause of the
211   issue.
212   Based on the debugging process, we identify logical errors and regenerate the code blocks to fix these
213   errors. The logical patch generation process involves comprehending the intended behavior of the
214   code and producing a fix that aligns with the specifications. By leveraging the knowledge obtained
215   from virtual execution debugging, along with the issue description and test case information, the
      agent can propose patches that address the identified logical errors.

      2.4   F EEDBACK -D RIVEN I MPROVEMENT
217
218
      Algorithm 1 Iterative Debugging
219
        Input: Codebase C, Issue Description I, Test Cases T
220
        Output: Fixed Codebase C ′
221     while not fixed do
222       F ← FaultLocalization(C, I, T )
223       P ← PatchGeneration(F )
224       C ′ ← ApplyPatch(C, P )
225       R ← RunTests(C ′ , T )
226       if R passes then
227          fixed ← True
228       else
229
             T ← UpdateTests(R)
          end if
230
        end while
231
        return C ′

233
      To further enhance the debugging capabilities of LLM-based agents, we introduce a continuous
234
      improvement phase. In this stage, we utilize feedback from the generated patches and the outcomes
235
      of the repaired code to refine the fault localization and patch generation processes.
236
237   We collect feedback on the generated patches, including their effectiveness in fixing the issues and
238   any additional test cases that the patches might trigger. Employing an iterative debugging approach,
239
      the agent repeatedly applies the fault localization and patch generation stages until a satisfactory fix
      is achieved. Each iteration builds upon the knowledge gained from the previous one, allowing the
240
      agent to refine its understanding of the issue and produce more accurate patches.
241
242   The iterative debugging process enables the agent to gradually improve the quality of the generated
243   patches. By integrating the results of the repaired code and updating the test cases, the agent can
244   identify any remaining issues and generate more comprehensive fixes.
245   By integrating these stages and adopting continuous improvement techniques, our method aims to
246   significantly enhance the debugging capabilities of LLM-based agents, leading to higher resolution
247   rates in complex software engineering tasks.

249   3     E XPERIMENTS
250
251   To evaluate the effectiveness of our proposed method, we conduct experiments on the SWE-bench
252   dataset. The experiments are designed to assess the performance of our LLM-based agent in resolv-
253   ing real-world software engineering issues.
254
255   3.1   DATASET
256
257   To evaluate the effectiveness of our proposed methods, we conduct experiments using the SWE-
258   bench and SWE-bench lite datasets Jimenez et al. (2024). SWE-bench is a comprehensive bench-
259
      mark consisting of 2,294 real-life software engineering task instances collected from the repositories
      of 12 popular large Python projects. Each task instance contains a pair of GitHub issue and corre-
260
      sponding pull request, where the issue either reports a bug to be fixed or requests a new feature to
261
      be implemented.
262

      3.2   E XPERIMENTAL S ETUP

266
267          • SWE-agent: A generic multi-agent workflow that utilizes an agent-computer tool interop-
268
               eration interface for debugging.
269          • AutoCodeRover: An approach that introduces new code search tools and incorporates
               AST for enhanced fault localization.

      3.4   D ISCUSSION
332
333   The experimental results demonstrate the effectiveness of our proposed method in enhancing the de-
334   bugging capabilities of LLM-based agents. The integration of program call graphs provides valuable
335   information about code execution flow and dependencies, enabling more accurate fault localization.
336   The simulated debugging mode allows the agent to analyze code behavior and generate logical
337   patches without relying on external debugging tools.
338   The ablation study highlights the importance of both the program call graphs and the simulated
339   debugging mode in our method. The call graphs help in understanding the relationships between
340   code components and tracing the root cause of issues, while the simulated debugging mode enables
341   the agent to reason about the code behavior and generate effective patches.
342
      Our method achieves significant improvements over the baselines in terms of resolution rate, fault
343
      localization accuracy, and repair rate. The continuous improvement techniques employed in our
344   method, such as feedback-driven refinement and iterative debugging, contribute to the agent’s ability
345   to learn from previous iterations and generate more accurate and comprehensive fixes.
346
347
      However, there are still challenges that need to be addressed in future work. One limitation of our
      method is the reliance on test cases for fault localization and patch evaluation. In real-world sce-
348
      narios, test cases may not always be available or may not cover all possible program behaviors.
349
      Developing techniques to generate meaningful test cases or leverage alternative sources of informa-
350   tion for debugging could further enhance the applicability of our method.
351
352   Another challenge is the scalability of our method to larger codebases and more complex software
353
      engineering tasks. As the size and complexity of the codebase increase, the fault localization and
      patch generation stages may become more computationally expensive. Investigating techniques to
354
      efficiently navigate and analyze large codebases while maintaining the accuracy of debugging is an
355
      important direction for future research.
356
357   Despite these challenges, our method represents a significant step towards enabling LLM-based
358   agents to autonomously debug and improve software. The integration of program semantic informa-
359
      tion and the simulated debugging mode opens up new possibilities for intelligent code analysis and
      automated software engineering.

362   4     C ONCLUSION
363
364   In this paper, we proposed a novel method for enhancing the debugging capabilities of large lan-
365   guage model (LLM)-based agents in complex software engineering tasks. Our method integrates
366   program semantic information, such as AST and call graphs, and introduces a simulated debugging
367   mode to enable LLMs to effectively localize faults and generate accurate patches. We conducted
368
      experiments on the SWE-bench dataset, demonstrating significant improvements in resolution rate,
      fault localization precision, recall, and repair rate compared to state-of-the-art baselines. Our work
369
      represents a significant advancement in LLM-based programming assistance and paves the way for
370
      more effective and efficient automated software debugging. By addressing the limitations of existing
371   approaches and introducing novel techniques, we believe our method is a step towards realizing the
372   vision of autonomous software engineering, where LLMs can actively assist developers in resolving
373   complex software issues.
