                                                   A BSTRACT
008
009
010
               In the few-shot learning problem, a model trained on a disjoint meta-train dataset
               is required to address novel tasks with limited novel examples. A key challenge in
011
               few-shot learning is the model’s propensity to learn biased shortcut features(e.g.,
012
               background, noise, shape, color), which are sufficient to distinguish the few ex-
013            amples during fast adaptation but lead to poor generalization. In our work, we
014            observed when the model learns with higher consistency, the model tends to be
015            less influenced by shortcut features, resulting in better generalization. Based on
016            the observation, we propose a simple yet effective meta-learning method named
017            Meta Self-Distillation. By maximizing the consistency of the learned knowledge
018            during the meta-train phase, the model initialized by our method shows better
019            generalization in the meta-test phase. Extensive experiments demonstrate that our
020            method improves the model’s generalization across various few-shot classification
021
               scenarios and enhances the model’s ability to learn consistently.

      1   I NTRODUCTION
025
026   Few-shot learning aims to address novel tasks with a limited number of examples, typically through
027
      rapid adaptation of a model trained on a dataset with disjoint labels. Many approaches tackle this
      issue from the perspective of meta-learning(Finn et al., 2017; Lee et al., 2019; Ravi & Larochelle,
028
      2016; Lake & Baroni, 2023). Methods such as Model-Agnostic meta-Learning (MAML) (Finn
029
      et al., 2017) and its variants (Raghu et al., 2019; Ye & Chao, 2021; Antoniou et al., 2018; Kao et al.,
030   2021; Nichol et al., 2018) aim to learn initialized parameters for a model with prior knowledge for
031   fast adaptation. Recent research has explored more challenging scenarios, such as cross-domain
032   few-shot learning (Ullah et al., 2022; Triantafillou et al., 2019; Tseng et al., 2020; Guo et al., 2020),
033   where the novel task belongs to a different domain and label set than the training dataset.
034
      A key challenge in various few-shot learning problems is the model’s tendency to learn biased short-
035
      cut features (e.g., background, noise, shape, color) from limited examples (Shah et al., 2020; Teney
036   et al., 2022; Lyu et al., 2021; Le et al., 2021). These shortcut features may suffice to distinguish the
037   few classes during rapid adaptation but result in poor generalization. Several solutions have been
038   proposed to address these issues. Although these approaches partially mitigate the problem, they of-
039   ten require additional resources or learn generalized features only within the meta-train dataset (Le
040   et al., 2021; Zhou et al., 2023; Liu et al., 2020; Dvornik et al., 2020; Snell et al., 2017). From the
041   perspective of meta-learning, we ask the following question: Can we make the initialized model
042   more inclined to learn generalization features rather than shortcut features when addressing novel
043   tasks?
044   This problem is challenging to address directly, as identifying generalized versus shortcut features
045   in the data is difficult. In our study, we generate different views of the same data through data
046   augmentation, which makes these views have different shortcut features but similar generalized
047   features. We use these views to update the model and observe that when model learning with better
048   consistency tends to exhibit better generalization. This implies that when tasks are learned with
049   higher consistency by the model, the model is less influenced by the shortcut features and reaches
050
      higher accuracy. At this point, if we can enhance the model’s consistency of learning across all
      tasks, we can make the model less influenced by the shortcut feature and more inclined to learn
051
      generalized features.
052
053   Based on this observation and inspired by the idea of self-distillation (Caron et al., 2020; Chen
      & He, 2021; Caron et al., 2021), we proposed meta self-distillation, which aims to maximize the

104   2.1     F EW- SHOT LEARNING
105
106   Few-shot learning aims to address novel tasks with a limited number of examples, typically through
107   the rapid adaptation of a model trained on base classes, which are disjoint from the classes in novel
      tasks. Solutions to few-shot learning are primarily categorized into meta-learning and transfer learn-


                                                                     2
      Under review as a conference paper at ICLR 2025


108
      ing approaches. meta-learning (Antoniou et al., 2018; Finn et al., 2017; Ye & Chao, 2021) aims to
109   train a model with prior knowledge that can fast adapt to novel tasks. Transfer learning (Tian et al.,
110   2020; Mangla et al., 2020; Liu et al., 2021) focuses on developing a generalized feature extractor
111   from base classes that can generalize to novel tasks. Traditionally, few-shot learning assumes that
112   base and novel classes originate from the same domain but differ in categories. Recent studies have
113   extended this to cross-domain few-shot learning, where base and novel classes belong to different
114   domains (Ullah et al., 2022; Triantafillou et al., 2019; Tseng et al., 2020; Guo et al., 2020). A critical
115   challenge in few-shot learning is that during fast adaptation, models tend to learn shortcut features,
116   leading to overfitting on novel tasks (Shah et al., 2020; Teney et al., 2022; Lyu et al., 2021; Le et al.,
117
      2021). Various methods have been proposed to address this issue. For instance, Poodle (Le et al.,
      2021) suggests using additional data to penalize out-of-distribution samples, while LDP-net (Zhou
118
      et al., 2023) employs local and global knowledge distillation to enable the model to learn more
119
      diverse features from the meta-training dataset. Although these methods mitigate the problem to
120   some extent, they often require additional data or parameters to adapt to unseen tasks and domains.
121   Alternatively, some approaches train a powerful feature extractor solely on the meta-train dataset,
122   which may limit the model’s ability to recognize unseen features in novel tasks (Le et al., 2021;
123   Zhou et al., 2023; Liu et al., 2020; Dvornik et al., 2020; Snell et al., 2017). Our method aims to
124   make the initialized model inclined to learn generalized features, thereby avoiding such limitations.
125
126   2.2   META -L EARNING
127
128   Meta-learning, also known as learning to learn, aims to learn initialized parameters with prior knowl-
129   edge for fast adaptation. It is mainly divided into metric-based meta-learning, represented by Pro-
130   toNet (Snell et al., 2017), and optimize-based meta-learning, represented by MAML (Finn et al.,
131   2017). Metric-based meta-learning improves model representation by bringing the representation
132   between the support data and the query data that belong to the same category closer, typically not
133   requiring fine-tuning during the meta-test phase. Optimize-based meta-learning aims to provide the
134
      initial parameters with prior knowledge, offering better generalization performance when fine-tuning
      on novel category samples. This category includes algorithms like MAML (Finn et al., 2017) and its
135
      variants, such as (Ye & Chao, 2021), which utilizes a single vector to replace the network’s classi-
136
      fication head weight, thus preventing the permutation in the meta-test phase. MAML++ (Antoniou
137   et al., 2018) enhances MAML’s performance by addressing multiple optimization issues encoun-
138   tered by MAML, while ANIL (Raghu et al., 2019) improves MAML’s performance by freezing the
139   backbone during the inner loop. In our work, we mainly focus on the optimize-based meta-learning.
140   From the perspective of meta-learning, our goal is to train initialized parameters that incline to learn
141   generalized features rather than shortcut features, thereby enhancing accuracy in the few-shot learn-
142   ing problems.
143
144   2.3   SELF - DISTILLATION
145
146   Self-distillation is a variant of contrastive learning (Caron et al., 2020; Chen & He, 2021), which
147   is trained by bringing the representations of positive instance pairs closer without using negative
148   pairs. BYOL (Grill et al., 2020)utilizes the exponential moving average of the network to produce
149   the target of an online network. SimSiam (Chen & He, 2021) further explored how self-distillation
150
      avoids collapse in a self-supervised setting. (Allen-Zhu & Li, 2020) suggests that self-distillation
      can serve as an implicit ensemble distillation, allowing the model to distinguish more view features.
151
      Self-distillation is an effective method to enhance the model’s feature extraction capabilities and can
152
      be combined with meta-learning (Li et al., 2022; Ni et al., 2021). Typically, self-distillation aims
153   to maximize the similarity of the representations across different views. Different from the typical
154   self-distillation that directly aligns the representation, we propose to use meta self-distillation to
155   maximize the consistency of the different updated models’ outputs. In this way, we can make the
156   initialized model less influenced by shortcut features when addressing a new task.

      3.1   P ROBLEM D EFINITION F OR F EW- SHOT C LASSIFICATION
163
164   Following (Vinyals et al., 2016; Chen et al., 2019; Wang et al., 2020), We define the few-shot clas-
165   sification problem(FSL) as an N -way K-shot task, where there are N classes, each containing K-
166   labeled support samples. Typically, K is small, such as 1 or 5. The data used to attempt to update the
167   model is defined as the support data S = {xs , ys }, where each xs represents the model’s input, and
168   ys denotes the corresponding label for xs . The data used to evaluate the effectiveness of the model
169   updates is defined as the query data Q = {xq , yq }, which has the same class as the support data,
170
      but the samples contained in the query set are different from those in the support set. The FSL task
      is defined as the problem of learning to correctly classify the query data Q with the support data S,
171
      which can be written as follows:
172
173
174                                  arg min ES,Q∼Dmeta-test [LFSL (θ, S, Q)]                            (1)
                                            θ
175
176   If the model is randomly initialized and directly fine-tuned on the limited support data, the model
177   will overfit. To address that, we need to transfer knowledge from seen data to the unseen data. The
178   seen data used in FSL is referred to as the meta-train set, and the unseen data is referred to as the
179   meta-test set. The labels in the two sets are disjoint, and in the cross-domain few-shot learning, the
180   domains of the two sets are also different. The goal of FSL is to pretrain or initialize the parameters
181
      by using the meta-train set and generalize to the unseen task sampled from the meta-test set.

257   To validate the point, we evaluate the consistency and accuracy of the parameters initialized by
258   MAML. During the meta-test phase, we sample one task and augment its support data, updating the
259   initialized parameters respectively. To evaluate the knowledge acquired from these support data, we
260   tested the updated models using the same query data and recorded the average prediction accuracy
261
      and output consistency across the tasks. For each task group, the support data are augmented from

262
      fore, the inconsistency of the differently updated models is mainly caused by different shortcut
263
      features. When the model is more inclined to learn generalized features, its outputs for the same
264   query data should be similar. In contrast, if the model tends to learn shortcut features, which results
265   in overfitting, the augmented inconsistencies in these features lead to greater output variance for the
266   same query data. The results are illustrated in Figure 2: lower consistency corresponds to lower
267   average prediction accuracy. Additionally, as the amount of support data increases, the model is
268   less influenced by shortcut features, and both consistency and accuracy are improved consistently.

      6     C ONCLUSION
510
511
      The tendency to learn shortcut features is the key challenge to few-shot learning. In our work, we
512   observe that the model learned with higher consistency tends to be less influenced by the short-
513   cut features. Building on this foundation, we introduce a meta-learning method named meta self-
514   distillation(MSD). MSD updates the model respectively by utilizing different augmented views of
515   support data in the inner loop, then maximizing the consistency of the outputs of the same query
516   produced by different updated models. We evaluate MSD across three few-shot learning problems.
517   MSD significantly enhances the performance of algorithms across various settings.
518   Learning to learn consistently is a new perspective for meta-learning. We believe our proposed algo-
519   rithm represents a step forward in enhancing models’ learning ability. Future research could extend
520   such a framework to the domain of self-supervised learning and apply it to larger-scale models.
