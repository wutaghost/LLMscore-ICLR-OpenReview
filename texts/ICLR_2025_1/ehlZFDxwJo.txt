009                                                A BSTRACT
010
011            Partial Label Learning (PLL) is known as a valuable learning technique that trains
012            Machine Learning (ML) models on partial label datasets, where the ground truth
013            label is concealed within the candidate label set of each data instance. It learns
014            label correlation based on a single centralized dataset to predict the latent true la-
015            bel. When data is non-independent and identically distributed (non-i.i.d.) among
016            workers in Federated Learning (FL), the label correlation interference problem
017
               occurs. To address the issue, in this paper, we propose pFedPLL, a personalized
               federated partial label learning algorithm with two new designs. In Label Correla-
018
               tion Isolation (LCI), we first develop a twin-module architecture, where a feature-
019
               level correlation matrix layer for each worker is isolated locally to prevent it from
020            being interfered with by others. In Label Correlation Personalization (LCP), we
021            then propose a bi-directional calibration loss to identify a more accurate learning
022            direction, where the positive calibration aligns the prediction result with the latent
023            true label, and the negative calibration pushes away the prediction result that falls
024            into the non-candidate
                                   q label
                                           set. We provide a convergence analysis of pFedPLL
025            with a rate of O      1
                                         for smooth non-convex problems. Experiment results
                                     T
026
               demonstrate that pFedPLL outperforms SOTA federated PLL algorithms and the
027
               federated version of centralized PLL algorithms across nine datasets.

030   1    I NTRODUCTION
031
032   Federated Learning (FL) (McMahan et al., 2017) has become an emerging topic in recent years
033   due to its advantages in efficient parallel training processes and private data isolation across work-
034   ers/clients. The performance of FL is highly related to the quantity, quality, and heterogeneity of
035   data. However, due to different data collection environments (non-expert users, limited computa-
036   tional power, and varying geo-locations of edge devices, etc.), it is costly to collect a large amount
037   (high quantity) of high quality (with ground truth label) data instances. One possible solution is
038   to first assign a set of potential true labels to each specific data instance to quickly collect a large
039
      quantity of data. Then, the Machine Learning (ML) model learns and analyzes the label correlation
      to mitigate the negative effect of noisy labels during training. Such a learning method is within the
040
      scope of Partial Label Learning (PLL) (Cour et al., 2011), where the candidate label set (containing
041
      potential true labels) consists of one ground truth label and several correlated false positive labels
042   (noisy labels). Since the ground truth label is concealed in the candidate label set, PLL aims to
043   train a classifier to predict the latent true label (the most likely true label) for each data instance by
044   analyzing the correlations among the labels within the candidate label set. The model performance
045   is highly dependent on the accurate label correlation learned during training.
046
      The label correlation in PLL has been proven to be a valuable component in centralized ML. It
047
      finds a way of training a model with relatively large quantity but low quality data while maintaining
048   the model performance. However, when data is non-independent and identically distributed (non-
049   i.i.d.) among workers in FL, the centralized label correlation may not work in such heterogeneous
050   data scenario. Due to non-i.i.d. data, the label correlation learned from the local dataset is only
051   applicable to each worker. When aggregation occurs, it will cause the label correlation interference
052   problem. For example, in worker “A”, the image data instance of digit “2” is very similar to digit
053   “3” (high label correlation between digit “2” and “3”), while in worker “B”, the digit “2” has a high
      label correlation with digit “5”. During the aggregation phase, the global model aggregates each


                                                          1

054
      worker’s label correlation and learns that the digit “2” has a high label correlation with both “3” and
055   “5”. When the global model is distributed back to workers, such label correlation interference will
056   mislead each worker’s unique label correlation information, resulting in the degradation of model
057   performance.
058
      To address this issue, we develop an all-in-one solution, pFedPLL, a personalized federated par-
059
      tial label learning algorithm that efficiently trains FL models on heterogeneous partial label data.
060
      pFedPLL generally consists of two components: 1 Label Correlation Isolation (LCI) and 2 Label
061   Correlation Personalization (LCP).
062
063   LCI. We develop a new correlation matrix layer that is only updated on the local/worker dataset and
064
      is not aggregated to the global model, preventing each worker’s unique label correlation information
      from being interfered with by others. The training layers before the correlation matrix layer are
065
      aggregated to obtain the global representative feature information. Please note that we insert the
066
      correlation matrix layer in the second-to-last layer of the DNN model instead of the last layer. This is
067   because the last layer learns the exact label correlation (the same dimensions as the number of classes
068   of labels) with coarse granularity. To learn more correlation information, we push the correlation
069   matrix layer to the second-to-last layer by learning feature correlation with fine granularity, where
070   the second-to-last layer usually has more dimensions than the number of classes of labels.
071
      LCP. Since the correlation matrix in LCI is isolated locally, the matrix itself is personalized. In order
072
      to better utilize the personalized correlation matrix information to help identify a more accurate
073   learning direction, we further develop a bi-directional calibration loss. Apart from the basic well-
074   known summarization PLL loss (Feng et al., 2020), we propose positive calibration to align the
075   prediction result with the latent true label, and negative calibration to push away the prediction
076   result that falls into the non-candidate label set (labels that are not within the candidate label set).
077
      We evaluate our pFedPLL algorithm both theoretically and experimentally. We prove that pFedPLL
078                                                           q

079   converges for smooth non-convex problems at a rate of O       T over T global iterations. In the ex-
080   periments, we compare pFedPLL with mainstream federated PLL algorithms and federated version
081   of centralized PLL algorithms across nine datasets. The ablation study of LCI and LCP is also eval-
082   uated. The experiment results demonstrate that pFedPLL consistently outperforms all benchmark
083   algorithms with up to 49.93% accuracy increase in various ML settings.

085   2   R ELATED W ORK
086
087   Partial label learning. Current approaches to PLL disambiguation can be divided into two cate-
088   gories (Tian et al., 2023): averaging and identification methods. Averaging methods (Cour et al.,
089   2011; Hüllermeier & Beringer, 2005; Zhou & Gu, 2018) treat all candidate labels equally as ground
090   truth. Further advancing this, Gong et al. (2021) propose a discriminative metric learning approach
091   using a Mahalanobis distance metric to assess similarity between neighbors with similar labels.
092   However, such predictions are easily misled by false positive labels. On the other hand, identification
093
      methods (Liu & Dietterich, 2012) aim to discover latent true labels during training, using techniques
      such as Expectation Maximization (EM) algorithms (Jin & Ghahramani, 2002) and maximum mar-
094
      gin methods (Nguyen & Caruana, 2008; Yu & Zhang, 2016) to resolve ambiguity. However, Both
095
      approaches, however, rely on global information, such as label distribution in EM and distance met-
096   rics, which conflict with privacy requirements in federated learning (FL). As a result, traditional PLL
097   methods encounter significant challenges when applied directly in FL environments. Recently, DL-
098   based methodologies(Wen et al., 2021; Feng et al., 2020) have emerged to address the PLL problem.
099   Zhang et al. (2021) introduced Class Activation Value (CAV) to transform PLL into a supervised
100   learning problem. Although these methods perform well in centralized settings, simply migrating
101   them to an FL environment may still not work. Our proposed pFedPLL algorithm provides a series
102   of methods (LCI & LCP) to enable training models in federated partial label datasets.
103   Federated learning. FL (McMahan et al., 2017) allows local models to collaboratively train a
104   global model while keeping data private. However, the non-i.i.d. nature of local data can degrade
105   global model performance (Zhu et al., 2021). To address this, Personalized Federated Learning
106   (PFL) (Tan et al., 2022) was developed to adapt the global model to individual workers. Methods
107   like Split Learning (SL) (Vepakomma et al., 2018) and the “base layers + personalized layers”
      design (Arivazhagan et al., 2019) help manage data heterogeneity by decoupling shared and local


                                                         2
      Under review as a conference paper at ICLR 2025


108
      model components. However, these approaches have mostly been tested on supervised datasets and
109   have yet to be adapted for PLL scenarios. Our pFedPLL algorithm personalizes part of the worker’s
110   model (LCP) to address the issue of non-i.i.d. partial label datasets.
111
112
      Weakly supervised learning in FL. Weakly supervised learning includes Semi-supervised Learn-
      ing (SSL), Noise Label Learning (NLL), Multi-instance Learning (MIL), and PLL, but studies on
113
      PLL in FL are limited. Recently, Yan & Guo (2024) introduced FedPLL LAAR, which reduces
114
      client drift via adaptive gradient alignment regularization but neglects label correlations. The method
115   assumes a class-dependent generation process (Lv et al., 2020), but in realistic scenarios, partial label
116   datasets should follow an instance-dependent generation process (Xu et al., 2021), where feature-
117   related false labels are more likely to enter the candidate label set, leading to stronger label cor-
118   relations. Besides PLL, other weakly supervised learning methods like SSL, NLL, and MIL are
119   well-studied in FL. Unlike PLL, these methods deal with single or unlabeled data and don’t involve
120   selecting the correct label from a candidate set. For example, FedMatch (Jeong et al., 2020) in SSL
121   uses labeled data as anchors to improve performance, while PLL faces greater challenges due to
122   uncertain true labels. NLL in FL, studied by (Song et al., 2022), focuses on mitigating the effect
123
      of noisy data (incorrectly labeled), while PLL must identify the correct one from a set. FedMIL
      (Bastola et al., 2024) focuses on bag-level predictions in MIL rather than identifying the latent true
124
      label. By developing a feature-level correlation matrix layer in LCI, pFedPLL utilizes knowledge
125
      from label correlations to effectively identify the latent true label from the candidate label set.
126
127   Considering the growing concerns about data privacy and the high costs of data labeling, exploring
128   PLL in an FL environment is essential. This paper investigates the PLL problem in FL, synergisti-
129
      cally utilizing the advantages of DL-based PLL methods and FL.

151   3.2    LCI: L ABEL C ORRELATION I SOLATION
152
153   The typical label correlation mechanism works well in centralized PLL (Xu et al., 2020). It learns
154   the label correlation from a single centralized dataset. When training FL models on decentralized
155   heterogeneous (non-i.i.d.) datasets, each worker can only learn its own label correlation from its own
156   dataset. When aggregation occurs, one worker’s label correlation might mislead others due to non-
157   i.i.d. data. To this end, we develop the Label Correlation Isolation (LCI) mechanism, which keeps
158
      the label correlation information local and thus prevents it from being interfered with by others.

160   a label correlation matrix layer with the dimension of p × p. Then, we insert the layer into the

      3.3   LCP: L ABEL C ORRELATION P ERSONALIZATION
203   We have utilized LCI in Section 3.2 to prevent each worker’s correlation information from being in-
204   terfered with by others. In this circumstance, the worker’s unique correlation matrix is personalized.
205   The next step is to further utilize this information to help identify a more accurate learning direc-
206   tion for each worker. On the basis of the summarization PLL loss (Feng et al., 2020), we propose

      Datasets. We utilize four benchmark datasets (MNIST (LeCun et al., 1998), Fashion-MNIST (F-

390   of these datasets can be found in Appendix B.1.
391   Data generation. To convert benchmark datasets into partial label datasets, we follow the instance-
392   dependent generation process (Xu et al., 2021), where the flipping probability of each incorrect
393   label is determined by the confidence prediction of a clean neural network trained on a supervised
394   dataset, and ρ ∈ [0, 1] is the temperature hyperparameter used to control the size of the candidate
395   label set. Please refer to Appendix B.2 for the detailed process. For non-i.i.d. data generation, we
396   use a Dirichlet distribution Dir(β) (Minka, 2000) to generate local data for each worker, where
397   β ∈ (0, +∞) is a hyperparameter that controls the level of data heterogeneity. A smaller β indicates
398   a higher level of non-i.i.d. In all experiments, we set β = 0.5. For accuracy assessment, since
399
      the aggregator lacks a complete model (only a global representation module exists), we assess the
      final accuracy as the averaged accuracy from all workers. Following the same methods (Tan et al.,
400
      2023; Lu et al., 2022), we split each worker’s dataset into training (80%) and testing (20%) datasets
401
      without any data instance overlap and then test each worker’s accuracy to obtain the final accuracy.

404   equipped with 4 NVIDIA GeForce RTX 3090 GPUs. In all experiments, models are updated by
405
      mini-batch SGD with learning rate η = 0.01, momentum factor = 0.9, and λ1 = λ2 = λ3 = 1.
      Other hyperparameter settings are specified in Appendix B.3 Table 4.

421   Benchmark datasets. We evaluate our pFedPLL algorithm using four benchmark datasets: MNIST,
422   F-MNIST, K-MNIST, and CIFAR-10, all adapted to partial label datasets. Table 1 demonstrates that
423
      pFedPLL outperforms all benchmarks, with a 1.1% accuracy improvement on MNIST, 7.38% on K-
      MNIST, and 1.17% on F-MNIST compared to Fed LW, the second-best algorithm. On CIFAR-10,
424
      pFedPLL exceeds Fed CC, the second-best algorithm, by 11.19%. Federated version of centralized
425
      PLL methods (Fed LW, Fed CC, Fed RC, Fed CVAL) lack mechanisms to mitigate the non-i.i.d.
426   issue in FL, leading to suboptimal performance. In contrast, pFedPLL’s LCI design effectively ad-
427   dresses this issue by preventing interference from other workers’ models. We also observe that
428   FedPLL LAAR does not perform well. This is because it utilizes a class-dependent generation pro-
429   cess, where only labels with large differences are included in the candidate label set (e.g., horse
430   vs. cat). Nevertheless, in our experiment, we implement an instance-dependent generation process,
431   where similar labels are included in the candidate label set (e.g., horse vs. donkey), making disam-
      biguation more difficult. In pFedPLL, we implement a fine-grained feature-level correlation matrix

      and bi-directional calibration loss to distinguish similar labels, leading to superior performance.
433   Overall, pFedPLL consistently outperforms all benchmarks, with training accuracy improvements
434   ranging from 1.1–49.93%.

491   the cornerstone of the subsequent bi-directional calibration (Lpc &Lnc ). 3 Ls + Lpc > Ls . This
492   demonstrates that Lpc plays a crucial role in effectively identifying the latent true label. Ls first
493   helps point a correct update direction (falling into the candidate label set) and Lpc then distinguishes

      update direction, causing unstable loss calculations. Once the model finds the correct direction, Lnc

      enhances the performance. When only applying Lnc alone, the model may fail to converge, because
499   it merely prevents predictions from falling into the non-candidate label set without considering the

533   In this paper, we propose pFedPLL, a personalized federated partial label learning algorithm. We de-
534
      velop label correlation isolation and label correlation personalization to prevent the workers’ unique
      label correlation information from being interfered with while helping identify more accurate learn-
535
      ing direction for better performance.
                                      q   We provide a convergence analysis for pFedPLL, demonstrat-
536
                                         1

538   both benchmark and real-world datasets illustrate that pFedPLL consistently outperforms SOTA al-

      benchmark datasets and 0.37–17.22% on real-world datasets.
