025   1   I NTRODUCTION
026
027   We consider the problem of selecting the “best” k-out-of-n elements, e.g. selecting k locations to
028   place sensors, selecting k features to include in data analysis, or selecting k samples for training a
029   model. We are particularly interested in the case where:
030
031   Expensive query access We are not given an explicit function to compute what makes one subset
032          “better” than another: rather we have expensive query access to an oracle that tells us that
033          evaluates different candidate subsets (e.g. estimating the quality of prediction from a subset
034
             of features or samples by training a smaller model on them).
035   Sublinear query complexity Motivated by the expensive query constraint and large problem sizes
036           in machine learning applications, we ask whether it is possible to obtain approximately
037           optimal subsets with query complexity that scales sublinearly with n.
038
039
      In full  generality, it is clearly hopeless to find an approximately optimal k-subset without querying
      all nk subsets (not to mention sublinear query complexity), so as is common in the literature we
040
      restrict our attention to monotone submodular objective functions, i.e. we assume that the marginal
041
      contribution from each additional element is diminishing, yet it is always non-negative (see Section 2
042   for a formal definition). Since our result for worst-case functions in this class is quite negative, we

      P consider the most important special case: (monotone) additive objective functions, i.e. f (S) =

046

047

052   rithms whose total work (specifically, total query complexity) is sublinear. Indeed, this is motivated
053   in part by the increasing monetary and environmental cost of total work of state-of-the-art machine
      learning algorithms that already achieve significant parallelization. (Of course, beyond the important

      1.1   T ECHNIQUE OVERVIEW
132
133   The linear lower bound for submodular maximization The key idea driving our approach for
134   proving lower bounds is to relate the query complexity of submodular maximization to the commu-
135   nication complexity of distributed set detection problem.
136
      We first describe the distributed set detection task, which is inspired by Braverman et al. (2016).2
137
      The distributed set detection is a multi-party communication problem, where each party observes the
138
      outcome of n coins. Among these n coins, k coins are fair and have mean 1/2, while the rest n − k
139   coins are biased and have small mean vallue. The goal of these parties is to collectively identify a
140
      small fraction (e.g. 0.1k) of fair coins. We prove that the distributed set detection requires Ω(n)
                                                                                                     e
141   communication cost using the distributed strong data process inequality (SDPI) and a direct-sum
142   argument.
143
144
      Our key observation is that the linear communication lower bound for distributed set detection yields
      a linear query lower bound of submodular maximization. To this end, consider a linear function
145
      whose weight on the i-th element equals the summation of the outcome of the i-th coin. The op-
146
      timal k-subset is exactly the k fair coins, given the number of parties is roughly Θ(log(n)). The
147   crucial observation is one could simulate the query algorithm in the communication model, with
148   only polylog(n) overheads: If the query algorithm asks for the value of f (S) for some set S, then
149   all parties just locally compute the summation of coins in S and broadcast their results, it takes at
150   most O((log(n))2 ) communication bits (O(log(n)) parties and log(n) bits per party). This proves
151   that finding the optimal k-subset requires Ω(n)
                                                 e    queries due to the Ω(n)
                                                                         e    communication lower bound
152   of distributed set detection.
153
      When the goal is to estimate the value of the optimal k-subset, the above hard instance fails be-
154
      cause the query algorithm could easily find one fair coin using O(n/k)
                                                                          e       queries. To this end, we
155
      construct a monotone submodular function by applying two levels of truncation to the above linear
156
      function. Roughly speaking, the optimal k-subset still corresponds to the k fair coins in distributed
157   set detection, but the two-level truncation over the linear function (see Eq. (2)(3)) masks off useful
158   information on large sets and requires the detection of a non-trivial fraction of the fair coins (see
159   Section 3.2 for detailed description).

173   Sublinear algorithm for linear function Our algorithm is a multi-scale combination of two base
174   subroutines. The first subroutine is to randomly select a set of size n/m and estimate the value
175
      of each individual element. This subroutine yields a good estimate on the top m-th quantile of
      the ground set, but fails to give an accurate estimation on top o(m) elements (to see this, imagine
176
      there are o(m) elements that have super large value, then one can observe their value only after
177
      sampling ≫ n/m elements). The second subroutine is to partition the ground set into m subsets
178   and estimate the value of each subset. Intuitively, this subroutine alleviates the weakness of the first
179   subroutine – if there are o(m) elements that have super large value, then most of them would fall
180   into different subsets and we can have a good sense of their value after querying the value of m
181   subsets. Nevertheless, the second subroutine could fail if the top o(m) elements are not significantly
182   larger than rest elements (say they have value 2 and the rest have value 0 or 1, then querying the
183   value of m subsets gives negligible hints on the value of top o(m) elements).
184   Intuitively, both subroutine have their own weakness, but they are somewhat complementary to each
185   other. We wish to combine them in a careful manner to get an optimal sublinear algorithm. The
186   real situation gets complicated due to the possible intricate choice of the top k elements, so we only
187   sketch our final solution here. Indeed, we compose the two base subroutines in multiple scales. Let
188   kr = (1 + ϵ)r , at each level r, the algorithm randomly partitions the ground set into nkr /k subsets
189   and estimates the kr -th quantile of these subsets (hence each level takes O(n/k)
                                                                                   e       queries). Roughly
190   speaking, we expect the top kr -th subsets to be as valuable as the top kr -th element + an average
191   bucket. Our final estimate is a weighted average over the output at each scale. See Section 4 for
192   details.

      Notation We write [n] = {1, 2, . . . , n}. For random variables X, Y , we use I(X; Y ) to denote
200
      the mutual information of X, Y , h(X; Y ) to denote the Hellinger distance, TV(X; Y ) to denote the
201
      total variation distance. For any value p ∈ [0, 1], let Bp the Bernoulli distributions with mean p.
202
203   Submodular maximization Let f : {0, 1}n → R+ be a nonnegative set function. For any sets
204   A, B ⊆ [n], let fA (B) := f (A ∪ B) − f (A) be the marginal value of a set B w.r.t. a set A. The
205   function is monotone if f (B) ≥ f (A) for any A ⊆ B. The function is said to be submodular if
206   fA (u) ≥ fB (u) holds for every sets A ⊆ B ⊆ [n] and every element u ∈ [n] \ B. In a con-
207   strained monotone submodular maximization problem, there is a budget k ∈ [n] and the goal is
208   find a subset S ⊆ [n] of size at most k that (approximately) maximizes the function value, i.e.,
209   maxS⊆[n],|S|≤k f (S). The problem is studied in the query oracle model, where each time the algo-
210   rithm submit a query V ⊆ [n] and the oracle returns the function value f (V ). We assume the oracle
211   returns the exact value of f (V ) and leave the study of noisy query to future work.
212   Let S ∗ := argmaxS⊆[n],|S|≤k f (S) be the optimum solution set (breaking ties arbitrarily). In the
213   search problem, the goal is to find an (approximately) optimal solution, and we say the algorithm
214   finds an α-approximate solution if it returns a set S (|S| ≤ k) such f (S) ≥ αf (S ∗ ). In the decision
215   problem, the goal is to determine the optimal value, given a value OPT, we say an algorithm is
      α-approximate if it can distinguish between f (S ∗ ) ≥ OPT vs. f (S ∗ ) ≤ α OPT.
