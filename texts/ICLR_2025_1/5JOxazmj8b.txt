011                                               A BSTRACT
012
013            Dynamic link prediction is an important problem considered by many recent works
014            proposing various approaches for learning temporal edge patterns. To assess their
015            efficacy, models are evaluated on publicly available benchmark datasets involving
016
               continuous-time and discrete-time temporal graphs. However, as we show in this
               work, the suitability of common batch-oriented evaluation depends on the datasetsвҖҷ
017
               characteristics, which can cause multiple issues: For continuous-time temporal
018
               graphs, fixed-size batches create time windows with different durations, resulting in
019            an inconsistent dynamic link prediction task. For discrete-time temporal graphs, the
020            sequence of batches can additionally introduce temporal dependencies that are not
021            present in the data. In this work, we empirically show that this common evaluation
022            approach leads to skewed model performance and hinders the fair comparison of
023            methods. We mitigate this problem by reformulating dynamic link prediction as a
024            link forecasting task that better accounts for temporal information present in the
025            data. We provide implementations of our new evaluation method for commonly
026            used graph learning frameworks.

029   1   I NTRODUCTION
030
031   Many scientific fields study data that can be modeled as graphs, where nodes represent entities that
032   are connected by edges. Examples include social (Lazer et al., 2009), financial (Bardoscia et al.,
033
      2021), biological (Davidson et al., 2002) as well as molecular networks (David et al., 2020). Apart
      from the mere topology of interactions, i.e., who is connected to whom, such data increasingly
034
      include information on when these interactions occur. Depending on the temporal resolution, the
035
      resulting temporal graphs are often categorized as continuous-time or discrete-time (Longa et al.,
036   2023): State-of-the-art data collection technology provides high-resolution continuous-time temporal
037   graphs, which capture the exact (and possibly unique) occurrence time of each interaction. Examples
038   include time-stamped online interactions (Kumar et al., 2019) or social networks captured via high-
039   resolution proximity sensing technologies (Vanhems et al., 2013). In contrast, discrete-time temporal
040   graphs give rise to a temporally ordered sequence of static snapshots, where each snapshot contains
041   interactions recorded within a (typically coarse-grained) time interval. Examples include scholarly
042   collaboration or citation graphs, which frequently include monthly or yearly snapshots.
043   Building on the growing importance of temporal data and the success of graph neural networks
044   (GNNs) for static graphs (Bronstein et al., 2017; Corso et al., 2024), deep graph learning has recently
045   been extended to temporal (or dynamic) graphs (Feng et al., 2024). To this end, several temporal
046   graph neural network (TGNN) architectures have been proposed that are able to simultaneously learn
047   temporal and topological patterns. These architectures are often evaluated in dynamic link prediction,
048   where the task is to predict the existence of edges during a future time window of length вҲҶt, e.g., to
049   provide recommendations to users (Kumar et al., 2019).
050   For dynamic link prediction, TGNNs commonly utilize temporal batches to speed up training (Su
051   et al., 2024). To construct these temporal batches, the sequence of temporally ordered edges is divided
052   into a sequence of equally large chunks that contain the same number of edges. Within each batch,
053   edges are typically treated as if they occurred simultaneously, thus discarding temporal information
      within a batch. For continuous-time temporal graphs, such fixed-size batches are likely associated

      While batch-oriented processing is a technical necessity for efficient model training, our work shows
109   that tuning the batch size essentially tunes the link prediction task, thus fitting the task to the model
110   and undermining a fair comparison of temporal graph learning techniques. Proposing a time-window-
111   oriented evaluation of dynamic link forecasting, our work provides a simple yet effective solution,
112   facilitating a fairer and more realistic evaluation approach that better reflects real-world scenarios.

127   Dynamic link prediction. Given time-stamped edges up to time t, the goal of dynamic link predic-
128

130

131   fixed size of b edges. Edges within the same batch are typically processed in parallel (Su et al., 2024;
132   Rossi et al., 2020), thereby discarding temporal information within each batch. In addition to the
133   existing (positive) edges (u, v) вҲҲ Bi+ , non-existing (negative) edges (uвҲ’ , v вҲ’ ) вҲҲ BiвҲ’ are sampled
134   and used for training and evaluation. This is done since real-world graphs are typically sparse and
135   using all possible edges between all node pairs would lead to a large class imbalance and longer
136   runtime.

140

158   CAWN learns temporal motifs based on causal anonymous walks (CAW) (Wang et al., 2021b).
159   GraphMixer takes an attention-free and transformer-free approach, using an MLP-based link encoder,

      Several further approaches for discrete-time dynamic link prediction exist, including DyGEM (Taheri
163   et al., 2019), DySAT (Sankar et al., 2020), and EvolveGCN (Pareja et al., 2020). For a recent survey
164   of deep-learning-based dynamic link prediction, we refer to Feng et al. (2024).

166   Temporal graph training. Recent works (Su et al., 2024; Zhou et al., 2022; 2023) identified
167   issues in the training setup for memory-based TGNNs with large batch sizes: Processing edges
168   that belong to the same batch in parallel ignores their temporal dependencies, resulting in varying
169   performance depending on the chosen batch size. This issue has been termed temporal discontinuity.
170   Su et al. (2024) propose PRES which accounts for intra-batch temporal dependencies through a
171   prediction-correction scheme. Zhou et al. (2023) propose a distributed framework using smaller

      discontinuity in evaluation.
174
      Temporal graph evaluation. Recent progress in terms of TGNN evaluation includes the temporal
175
      graph benchmark (TGB) (Huang et al., 2023) similar to the static open graph benchmark (OGB) (Hu
176
      et al., 2020). Poursafaei et al. (2022) identify problems with random negative sampling for dynamic
177   link prediction and propose new negative sampling techniques dependent on time to improve the
178   evaluation of TGNNs. Gastinger et al. (2023) identify issues in the evaluation of temporal knowledge
179   graph forecasting. Although none of the models used for this task overlap with regular TGNNs for
180   dynamic link prediction, some of the problems can be related, e.g., differences in forecasting horizons
181   leading to incomparable results.

191   mitigate these issues, we then formulate the link forecasting task based on fixed-length time windows.

      3.2    L INK FORECASTING : TASK DEFINITION

361   The study of temporal information is at the center of time series forecasting (Benidis et al., 2023) and,
362   therefore, we relate our task definition to a fixed temporal quantity to solve the identified problems.
363   We can interpret the temporal edges E as n2 Boolean time series, each of which takes the value 1
364   at the times an edge occurs. Standard multivariate models output a value for each timestamp over
365   a forecasting horizon h. In large-scale temporal graphs, it is computationally infeasible to forecast
366   the existence of all n2 possible links, thus, only a sample of negative edges is considered instead. In
367   continuous-time dynamic graphs, observations are available at high resolution, e.g. seconds, however,
368
      for many practical applications, predicting at lower granularity suffices. For example, it is typically
      enough to predict whether a customer purchases a certain product within the next day or week.
369
      Therefore, we consider forecasting for all timestamps [t + 1, t + h] during a time window at once
370
      instead of for each of them individually, and define the link forecasting task as follows:
371
372   Definition (Dynamic link forecasting). Let G = (V, E) be a temporal graph with node features HV
373   and edge features HE . Let h be the time horizon and Wi+ := {(u, v) | вҲғ(u, v, t) вҲҲ E with iВ·h вүӨ t <
374   (i + 1) В· h} the set of edges in the i-th time window. We use WiвҲ’ to denote a sample of |Wi+ | negative
375
      edges that are sampled using one of the negative sampling approaches described in appendix A and
      do not occur as positive edges in time window [i В· h, (i + 1) В· h), i.e. WiвҲ’ вҲ© Wi+ = вҲ…. We further
376
377   use EМӮi = {(u, v, t) | вҲғ(u, v, t) вҲҲ E : t < i В· h} to denote the set of past edges for time window i.
      The goal of dynamic link forecasting is to find a model fОё (u, v|EМӮi , HV , HEМӮi ) with parameters Оё

378
      that, for all edges (u, v) вҲҲ Wi+ вҲӘ WiвҲ’ in each time window i, forecasts whether (u, v) вҲҲ Wi+ or
379
      (u, v) вҲҲ WiвҲ’ .
380
381   Crucially, this definition makes the evaluation independent of the batch size b and instead introduces
382   a time horizon h that defines the forecasting time window. Essentially, this forecasting time window
383   aggregates edges into snapshots, discarding temporal information inside each time window since
384   we are forecasting whether a link exists in any of the timestamps t вҲҲ [t + 1, t + h]. Similar
385   to batch-based dynamic link prediction, adopting such a time-window-based perspective enables
386   parallel processing of edges, but does not preclude information loss entirely. Instead, time windows
387   require deliberately choosing the temporal resolution based on the characteristics of each dataset and
388   application (see appendix C for examples), resulting in a trade-off between evaluation runtime and
389
      granularity of temporal information available to the TGNN models. To summarize, link forecasting
      requires choosing a time horizon that affects and controls the information loss, but ensures that this
390
      information loss is consistent for different models, facilitating fair performance comparisons.
391
392
393   Computational cost Let m be the total number of links, where typically m вүӘ n2 since real-world
394
      graphs are sparse, then we can assign each link to its corresponding time window in O(m) by
      checking each linkвҖҷs interaction time. After each link is assigned, the time complexity during model
395
      evaluation is the same as for the batch-based approach. Since the number of links per time window
396
      varies and windows can become large during periods when many temporal edges occur, we cannot
397   preclude memory overflows entirely. To mitigate this issue, one can split large time windows into
398   smaller batches for GPU-based computations. Since information leakage instead of information loss
399   needs to be prevented between batches of the same time window, steps such as negative sampling or
400   memory updates need to be done based on the whole time window.
401
402   Implementation We provide implementations for our evaluation procedure in commonly used
403   PyTorch libraries to simplify the adoption of our approach. Specifically, we implement a new
404   DataLoader called SnapshotLoader that replaces the widely used TemporalDataLoader
405   in PyTorch Geometric (Fey & Lenssen, 2019). We extend DyGLib (Yu et al., 2023) with a command
406   line argument horizon that can be used in the evaluation pipeline. The latter was used for the
407   experiments in this work and can be used to reproduce our results. The implementations are added as
408   supplementary material to ensure anonymity and will be made publicly available after acceptance of
409
      the paper.

501   5   C ONCLUSION
502
503   In this work, we considered issues associated with current evaluation practices for dynamic link
504   prediction in temporal graphs. To address computational limitations, edges in the test set are split
505
      into fixed-size batches, making the task вҖңtoo easyвҖқ but at the same time вҖңtoo hardвҖқ: вҖңToo easyвҖқ
      because state-of-the-art approaches for dynamic link prediction have treated the batch size as a
506
      tunable parameter. Changing the batch size, however, changes the prediction task, resulting in
507
      incomparable results between different batch sizes. For discrete-time temporal graphs, multiple
508   batches that include edges from the same timestamp further leak information that can be utilized by
509   memory-based TGNNs. вҖңToo hardвҖқ since in continuous-time temporal graphs, fixed-size batches
510   create varying-length time windows that essentially lead to a different task for each duration, requiring
511   the model to capture multiple task definitions at the same time. Furthermore, for edges within a
512   batch, information regarding their temporal ordering is lost. In discrete-time temporal graphs where
513   snapshots are typically larger than the batch size, batches additionally impose an ordering of edges
514   not present in the data.
515   We solve these issues by formulating the dynamic link forecasting task. Dynamic link forecasting
516   acknowledges the resolution at which temporal interaction data is recorded and explicitly considers a
517   forecasting horizon corresponding to a prediction time window of a fixed duration. Depending on the
518   dataset and problem setting, the horizon may span seconds, minutes, hours, or longer, but crucially,
519   time windows always span the same length. We evaluated dynamic link forecasting performance
520   of nine state-of-the-art temporal graph learning approaches on 14 real-world datasets, comparing
521   it to the common dynamic link prediction evaluation. We find substantial differences, especially
522
      for memory-based TGNNs. We provide data loader implementations for PyTorch Geometric and
      DyGLib to facilitate practical applications of our evaluation approach.
523
524
525
      Limitations and Open Issues Limitations of our work include that our reformulation of the
      dynamic link prediction task suggests time-window-based approaches for model training, which
526
      however goes beyond the scope of our paper. Furthermore, the metrics used in this work consider
527
      the problem of dynamic link prediction or forecasting as binary classification, using one negative
528   sample for each positive edge. Other approaches (Huang et al., 2023; You et al., 2022) consider the
529   problem as a ranking task (e.g. using the mean reciprocal rank (MRR)) and compare each positive
530   edge against a large number of negative samples. These approaches typically use only one positive
531   sample per batch which alleviates the information loss and provides a better estimate of the modelsвҖҷ
532   precision due to the large number of negative samples. In contrast, our window-based approach allows
533   the parallel processing of many positive samples in each time window, leading to a considerably
534   faster evaluation by making reasonable simplifications of the task. Specifically, it is enough to make
535   predictions at a lower resolution than the data is available in for many tasks, e.g. it may not be
536
      required to predict whether a customer will purchase a certain product within the next second; making
      such a prediction for the next day or week may be sufficient. Combining the advantages of both, our
537
      time-window-based evaluation and the ranking-based tasks, is left for future work.
538
539   Lastly, there are no expected negative societal impacts that go beyond those of other foundational
      works in machine learning research.

      A    N EGATIVE SAMPLING APPROACHES
757
758
      Dynamic link prediction is typically framed as a binary classification problem to predict class 1 for
759   existing links during a certain time window and 0 otherwise. Due to the sparsity of most real-world
760   graphs, it usually suffices to train and evaluate using all existing (positive) edges and a sample of
761   non-existing (negative) edges out of all possible edges V 2 . In static link prediction, negative edges
762   are typically sampled randomly from V 2 without replacement but Poursafaei et al. (2022) showed
763   that this technique is ill-suited for dynamic link prediction. One reason is rooted in the characteristics
764   of temporal graphs where already-seen interactions tend to repeat several times during the observation
765   period. To address this issue, Poursafaei et al. (2022) introduced negative sampling which we cover
766   in the following.

       C    E VALUATION BASED ON REALISTIC TIME HORIZONS
973
974
       In our main experiments, we determined the time horizon based on the average number of links per
975    time window to enable a fair comparison to the batch-based approach. In section 3.1, we discussed
976    that the time horizon for a realistic evaluation should instead be chosen carefully for each individual
977    dataset. To provide an example of such a realistic evaluation, we will assign a reasonable horizon for
978    each dataset used in this work. Note that this enables a fair model comparison since we use the same
979    horizon for all models across each dataset. The temporal resolution for most discrete-time datasets
980    is limited due to the data collection process. The US Legislators dataset for example contains one
981    snapshot for each congress which provides a natural horizon of one snapshot. Thus, we will only
982    consider continuous-time datasets and discrete-time datasets where the duration of a snapshot is not
983
       yet a natural time horizon for evaluation, i.e. Contacts.
984    We list the considered datasets and the chosen horizon with the reasoning behind it in the following:
985
986           вҖў Enron is a network of users with edges representing emails sent between them. We choose
987
                24 hours as a reasonable horizon since 90% of all email replies are typically sent within a
                day Kooti et al. (2015).
988
989           вҖў UCI is a social network based on student communications. We use 30 min as the horizon
990             since users receiving a text message typically feel pressured to reply between the next 20
991
                minutes and the end of the day Aranda & Baig (2018).
992           вҖў MOOC connects students to units of an online course based on their interactions. We set
993             h = 6 min since Guo et al. (2014) recommend keeping the learning video of a unit shorter
994             than this time frame.
995           вҖў Wikipedia represents the editing behaviour of users in a graph. Since editing Wikipedia
996             articles is unpaid, we do not expect frequent interactions from each user. We assume that
997             users come from different time zones and consider that the total duration of the dataset is
998
                only one month. Therefore, we see the typical working time of 8 hours as an appropriate
                time horizon to take into account that interactions wonвҖҷt appear very frequently but there are
999
                still enough time windows for evaluation.
1000
1001          вҖў LastFM connects users to the songs they listen to. We select 24 hours as the horizon to
1002
                evaluate this dataset on a task where the goal is to predict the songs that users will listen to
                tomorrow based on their listening behaviour during the last days.
1003
1004          вҖў Myket represents users and Android applications that are connected when an application is
1005            installed. Considering that, similar to the Wikipedia dataset, no frequent interactions are
1006
                expected, we choose 24 hours as the time horizon since the total duration of the dataset is
                longer than the total duration of the Wikipedia dataset.
1007
1008          вҖў Social Evolution is a proximity network gathered from students in a dormitory. Thus, we
1009            select 2 hours вҖ“ a typical duration of a lecture including breaks вҖ“ as the time horizon.
1010          вҖў Reddit contains the posting behaviour of Reddit users. Since dynamics in a social network
1011            are typically fast, we select a time horizon of 15 minutes.
1012          вҖў Contacts: Similar to Social Evolution, we select 2 hours as the time horizon since the
1013            network also captures the proximity of students.
1014

1016
       specified datasets with the selected time horizons. The results are presented in Table 5 and 6. For

1017

1081
1082
       For reproducibility, we provide a Python package extending the dynamic graph learning library
1083   DyGLib1 Yu et al. (2023) as a supplement, including a bash script to run the experiments. The code
1084   will be made publicly available on GitHub after acceptance of the paper.
1085
1086
       We use the best hyperparameters reported by Yu et al. (2023) and, for completeness, list these
       hyperparameters for the 13 datasets used by Yu et al. (2023) below. However, the Myket dataset
1087
       Loghmani & Fazli (2023) was not included in the study. Therefore, for Myket, we use each methodвҖҷs
1088
       default parameters as suggested by the respective authors.
1089
1090   We use 9 state-of-the-art dynamic graph learning models and baselines (JODIE Kumar et al. (2019),
1091   DyRep Trivedi et al. (2019), TGAT Xu et al. (2020), TGN Rossi et al. (2020), CAWN Wang et al.
1092
       (2021b), EdgeBank Poursafaei et al. (2022), TCL Wang et al. (2021a), GraphMixer Cong et al. (2023)
       and DyGFormer Yu et al. (2023)). The neural-network-based approaches (all except EdgeBank)
1093
       are trained five times for 100 epochs using the Adam optimizer with a learning rate of 0.0001.
1094
       An early-stopping strategy with a patience of 5 is employed to avoid overfitting. For training and
1095   validation, a batch size of 200 is used. The training, validation and test sets of each dataset contain
1096   70%, 15% and 15% of the edges, respectively. The sets are split based on time, i.e., the training set
1097   contains the edges that occurred first while the test set comprises the most recent edges.
1098
       The experiments were conducted on a variety of machines with different CPUs and GPUs. A list of
