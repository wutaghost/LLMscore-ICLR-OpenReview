009                                                A BSTRACT
010
011            In this paper, we show that reasoning/proving issues of GPT/LLM are an inherent
012            logical consequence of the architecture. Namely, they are due to a schema of its
013            prediction mechanism of the next token in a sequence, and randomization involved
014            into the process. After natural formalization of the problem into a domain of finite
015            graphs, G(ω), we prove the following general theorem:
016            For almost all proofs, any learning algorithm of inference, that uses randomiza-
017            tion in G(ω), and necessitates veracity of inference, is almost surely literal learn-
018            ing.
019            In the context, ”literal learning” stands for one which is either vacuous, i.e.
020            ∀x [P (x) =⇒ Q(x)] where P (x) is false for every x, or create a random
021            inference from a false assumption (hallucination), or it essentially memorizes the
022            inferences from training/synthetic data.

065

066

067   imitations are fundamental and innate.

069   A crucial observation is a scheme of transformer prediction mechanism of next token in a sequence.

071
072   For almost all proofs, any learning algorithm of inference, that uses randomization in G(ω) 1 , and
073   necessitates veracity of inference, is almost surely literal learning.
074   We provide a few proofs for this statement. For a quick look at a formal proof, refer to C.2. To
075   develop an intuition for the phenomenon, there is an informal proof (1) where consideration is

078
      Namely, we work in the first-order model G(ω) (appendix B.1), where connectivity between nodes a
079

080   are going to use a → b for the implication as well, interchangeably). For our purposes, the graph

082   any algorithmic randomization on the nodes allows us to view G(ω) as a suitable probability space
083   (C.2). Finally, literal (or vacuous) learning is defined as such that, almost surely, the proof chain,
084   generated by an algorithm, is equivalent (in G(ω)) to one in a training dataset.

      3    M OTIVATION /P RELIMINARIES
156
157   To demonstrate the model that proves the inherent limitation phenomena, we need to formalize
158   logically the mechanism employed by GPT/LLM to predict the next token in a sequence. It turns out
159   that randomization used by GPT architecture (namely ”temperature”) is the main reason. However,
160   it isn’t easy to devise an alternative when dealing with training on a large text corpus. Note that the
161   architecture becomes too ”predictable/plain” if we choose the most probable pattern in the list of
      candidates for the next token. There, we would need a certain randomization to become ”creative”.

      As we will see, that necessary hack is sufficient to preclude the architecture from ever succeeding in
163   a rigorous context, in a formal setting when we need to infer our next supposition with strict regard to
164   its veracity. A good example would be generating proof for a theorem. Note that, despite infamous
165   issues with Generative Learning with rigor in this context, there have been a few feasible attempts to

171   Namely, we present a simple first-order theory on the language of (random) graphs where one can
172   state that the generative inference that admits randomization on implications will almost necessarily
173   lead to logical faults (i.e., with probability 1). This result is based on a 0-1 law (and its variations)

      Discussion-Datasets In (Gendron et al., 2023), is shown that the baseline dataset construction for
325   rigorous learning needs to be a formal exercise. Consider the task of equation completion in which
326   one has to predict a missing symbol. Since this is perfectly aligned with the main premise of LLM
327   based on transformers, one can expect that the success rate for this task will be quite high. As is
328   shown in the paper, this is not the case. An associated (and well-known) phenomenon of a plateau
329   of performance and subsequent degradation in an exponential fashion manifests in the same way as
330   for the generic sequence case. Similarly, few authors summarize a few problems in the answers in
331   contemporary systems associated with a low P/R w.r.t citation usage from the underlying sources.
332   These experimental results are not for the rigorous context.
333   There is a widespread belief that because the training set contains ”everything”, any result, including
334   novel ones, can be proven using symbolic inference from the corpus. However, it is just not the case.
335   It is well-known that any mathematical problem of significance requires one or multiple critical
336   insights that are just not to be found. These are not combinations of known results (or tactics), but
337   rather completely new, albeit inevitable, ideas. For instance, for some long-standing problems, new
338   fields of mathematics had to be created, representing a new body of knowledge. Thus, generalizing
339
      the LLM solution for these targets is a task of yet another level of complexity for which the method
      is not suited. Moreover, as we show below, it is guaranteed to fail. The inevitable conclusion is that
340
      the apt inference model has to be more deferential to logic.

      Let L be a language (an extension of a basic formal logical language, L0 ).
543
544
      Definition and base notations The set of L−terms is the smallest set Lt such that contain all
545   constant symbols of L, all variables, and if t1 , t2 , ..., tn are in Lt then for any n-ary function symbol
546   f , f (t1 , t2 , ..., tn ) is also in Lt . Set La of atomic formulas are represented by the properties:
547   (1) if t1 and t2 are terms then t1 = t2 is in La , and
548   (2) the corresponding n-ary function symbols are also in La .
549

553   sentences in first-order logic over L. We also assume first-order logic with equality; in other words,
554   only normal models are employed. Thus, the models, considered herein (e.g., Erdős–Rényi or finite
