009                                               A BSTRACT
010
011            In this work, we focus on training Graph Foundation Models (GFMs) for graph-
012            level tasks like protein classification. Effective GFM training requires capturing
013            information consistent across different domains. We have discovered that graph
014            structures provide more consistent cross-domain information compared to node
015            features and graph labels. However, traditional in-context learning methods pri-
016
               marily focus on transferring node features from various domains into a unified rep-
               resentation space but often lack structural cross-domain generalization. To address
017
               this, we introduce a method called GraphProp, which emphasizes structural gener-
018
               alization. The GraphProp training process consists of two main phases: initially,
019            it trains a structural GFM through the supervised prediction of graph structural
020            properties. It then uses the structural representation from this GFM as positional
021            encoding to train a comprehensive GFM. This phase of training utilizes in-context
022            learning with domain-specific node features and graph labels to improve cross-
023            domain node feature generalization. Additionally, employing data augmentation
024            in training the structural GFM helps address the scarcity of labeled graph data and
025            facilitates explicit cross-domain structural generalization. Our experimental re-
026            sults demonstrate that GraphProp significantly outperforms traditional in-context
027
               learning methods, especially in handling graphs without node features.
028
029   1   I NTRODUCTION
030
031   Graph Foundation Models (GFMs) are gaining more and more attention in research related to graph
032   data, as they aim to leverage diverse data to improve effectiveness across various tasks and domains.
033   Recent advancements (Galkin et al., 2023; Zheng et al., 2023) demonstrate that GFMs can generalize
034   well to new, unseen graphs. These models are typically classified into three types based on their
035   adaptability: domain-specific, task-specific, and primitive (Mao et al.). Domain-specific GFMs are
036   designed to learn universal features within a particular domain, allowing a single model to efficiently
037   handle multiple tasks, often outperforming specialized models. Examples include Mole-BERT (Xia
038   et al., 2023), DPA-2 (Zhang et al., 2023), and DiG (Zheng et al., 2023), which are developed to
039
      handle multiple molecular tasks within the chemical domain. Task-specific GFMs are trained on
      rich data sources to perform specific tasks, making them suitable for fields with less abundant data.
040
      For example, ULTRAQUERY (Galkin et al., 2024) and ULTRA (Galkin et al., 2023) are designed for
041
      knowledge graph reasoning and can be trained on extensive data sources like Wikipedia, enabling
042   their application in less data-rich domains. Similarly, GraphFM (Lachi et al., 2024), GraphAny
043   (Zhao et al., 2024), and GRAPHTEXT (Zhao et al., 2023b) are trained for node classification tasks
044   using graphs from various domains, including chemical and social networks. Primitive GFMs are
045   more versatile but limited in the range of datasets and tasks they can handle. For instance, UniAug
046   (Tang et al., 2024b), a universal graph structure augmentor based on a diffusion model, captures
047   diverse graph data patterns and can be used to adaptively assist downstream tasks.
048   The main challenge in developing GFMs is capturing consistent information from graph data that
049   varies across different domains (Galkin et al., 2023). For instance, in molecular data (Yang et al.,
050   2016), graph structures represent 3-D spatial relationships and atomic bonds, while node features
051   capture chemical properties. In social networks (Dwivedi et al., 2023), structures represent user
052   connections, and node features reflect user attributes. Due to these differing distributions, it is
053   challenging for a single model to learn unified representations across domains. Traditionally, one
      approach is to convert graph structures and node features into text and leverage large language mod-

      els (LLMs) to create unified representations for graphs from different domains. Although no direct
055   method currently exists for learning unified graph structure representations, existing graph reasoning
056   LLMs can be adapted for this purpose. For example, GraphQA (Fatemi et al., 2023) describes graph
057   connectivity in text and then poses graph reasoning questions to LLMs. By incorporating domain
058   descriptions into these prompts, GraphQA can be adapted to learn unified representations of graph
059   structures, effectively functioning as a structural GFM. Similarly, other graph reasoning LLMs,
060   like NLGraph (Wang et al., 2024), which focuses on tasks such as shortest paths and connectivity
061   by converting graphs into text, can also be used to learn unified graph structure representations. For
062   learning unified node feature representations, the One For All (OFA) method proposed by (Liu et al.,
063
      2023a) employs text-attributed graphs (TAGs) to consolidate graph datasets from various domains
      into a single, large TAG dataset, and then utilizes LLMs to learn unified node feature representa-
064
      tions across all domains jointly. However, these methods have limitations. Graph reasoning GFMs
065
      primarily focus on reasoning abilities rather than comprehensive structural representations, and in-
066   context GFMs may struggle with structural generalization, especially when dealing with graph data
067   lacking node features.
068
069
      To improve GFMs, we aim to capture information that remains consistent across different domains.
      We have observed that the structure of graphs contains invariant information (properties depending
070
      on the abstract structure only) that is shared across domains. For example, whether dealing with
071
      molecular data or social networks, their abstract graph structures exhibit common properties like the
072   fractional chromatic number (Scheinerman & Ullman, 2013) and Lovász number (Lovász, 1979),
073   even if their specific values differ. On the other hand, node features and graph labels are highly
074   domain-specific and lack this cross-domain consistency. For instance, node features in molecular
075   data describe chemical properties, while in social networks, they represent user attributes, without
076   overlap between them. Similarly, graph labels, such as the class of a molecule or the type of social
077   community, are tied to domain-specific knowledge, making them unique to each domain.
078   Given these distinctions, we introduce GraphProp, a GFM training method that separates the use of
079   structural information from domain-specific node features and graph labels. GraphProp begins by
080   training a cross-domain structural GFM through the prediction of graph properties in a supervised
081   manner. To achieve comprehensive cross-domain structural representations, GraphProp incorporates
082   a wide range of graph properties, including novel ones like the fractional chromatic number. This
083   approach enables explicit unified graph structure learning through graph data augmentation, extend-
084   ing traditional methods like G-Mixup (Han et al., 2022) for cross-domain GFM training. GraphProp
085   also addresses the scarcity of graph data by leveraging unlabeled and synthetic graphs, ensuring
086
      sufficient data for effective GFM training. After training the structural GFM, we use its structural
      representation as positional encoding to train a comprehensive GFM. This training phase employs
087
      in-context learning with domain-specific node features and graph labels to enhance cross-domain
088
      node feature generalization. Overall, the GraphProp framework achieves both structural and node
089   feature generalization across domains.
090
091   Our contributions are as follows:
092
             • We introduce GraphProp, a GFM training method that first trains a structural GFM by
093
               predicting graph properties. Then, it uses these structural representations to train a com-
094            prehensive GFM through in-context learning with domain-specific features and labels.
095
096
             • To the best of our knowledge, GraphProp is the first GFM that achieves both structural and
               node feature generalization across domains for graph-level tasks. Many existing GFMs
097
               merely use in-context learning with text-attributed graphs, lacking structural generalization.
098
099          • We bridge the use of graph theory in GFM training through graph property prediction. This
100            approach addresses the scarcity of labeled data by effectively utilizing unlabeled and even
101
               synthetic graphs for scalable GFM training.

164   In-context learning is a widely used method for creating unified graph representations across dif-
165   ferent domains. It works by describing graph structures or node features in text, incorporating

334   3.4   A DVANTAGES
335          • Structural and Node Feature Cross-Domain Generalization: To the best of our knowl-
336            edge, GraphProp is the first GFM that achieves both structural and node feature generaliza-
337            tion across domains, designed for graph-level tasks. It learns unified structural represen-
338            tations Z through property prediction and unified node features via in-context learning on
339            TAG, enabling it to handle various graph types, including those without node features.
340            Many existing GFMs prioritize node feature generalization through in-context learning but
341            often overlook structural generalization. For instance, OFA (Liu et al., 2023a) performs
342            well with node features but struggles with graphs that lack them. Models like GraphQA
343            Fatemi et al. (2023) try to achieve structural generalization by reasoning with TSGs and
344
               posing simple questions to LLMs, yet they mainly focus on enhancing reasoning rather than
               developing comprehensive structural representations. Converting complex graph structures
345
               into text can result in the loss of essential information about the graph’s overall proper-
346
               ties. In contrast, our structural GFM directly regresses graph properties without relying
347            on TSGs, enabling effective cross-domain structural generalization, as these properties are
348            topological characteristics present across various domains.
349
             • Bridging GFMs and Graph Theory: GraphProp is a self-learning method that leverages
350
               a wide array of graph properties from graph theory, enabling the structural GFM to learn
351
               comprehensive representations Z. Beyond graph-level properties, this method can also pre-
352            dict node-level properties (e.g., degree and centrality in Table 8) and node-pair properties
353            (e.g., shortest path (Schrijver, 2012)). More details are available in Appendix B.1.
354
             • Addressing Data Scarcity: Training foundation models usually requires large amounts of
355
               labeled data, which can be hard to find. In contrast, unlabeled data is much more plenti-
356            ful. Similar to how LLMs like GPT (Radford, 2018) are pre-trained on unlabeled data by
357            predicting the next word in a sequence, GraphProp uses unlabeled graph data for training
358            structural GFMs through property prediction. Additionally, for large structural GFMs, data
359            augmentation can create synthetic graphs to support scalable GFM training.
360
361   3.5   L IMITATIONS
362
             • Limited Scalability for Large Node-Level Tasks: GraphProp is primarily designed for
363
               graph-level tasks and may struggle with large-scale node-level tasks, such as those involv-
364
               ing graphs with many nodes (e.g., ogbn-arxiv (Wang et al., 2020)). Some graph property
365            computations have polynomial complexity, while others are NP-hard, making them ineffi-
366            cient for very large graphs.
367
             • Graph Property Requirements: Certain graph properties, like graph diameter, apply only
368
               to specific types of graphs, such as connected graphs. This means that graphs must be
369            checked before certain properties can be used.
370
             • Addressing Domain-Specific Node Features: While GraphProp can generate synthetic
371
               graphs to alleviate the scarcity of structural data, it does not address the lack of domain-
372
               specific node features, which requires specialized knowledge.

479   6     C ONCLUSION
480
481   This paper introduces a new method called GraphProp for training GFMs by predicting graph prop-
482   erties. The idea comes from the observation that graph structures share common properties across
483   different domains, offering more consistent cross-domain information than domain-specific node
484   features. By training a structural GFM, we can improve the structural generalization of other GFMs.
485   Finally, we combine the structural GFM with widely used in-context GFMs to achieve better gener-
      alization in both graph structure and node features.

1067   set of graph properties that can be computed in polynomial time, including some properties like the

1074   converting graphs into text for LLMs. These benchmarks are designed to train reasoning abilities
1075   rather than to provide comprehensive graph property learning.
