009                                               A BSTRACT
010
011            Transformer-based large language models (LLMs) excel in modeling complex
012            language patterns but face significant computational costs during inference, es-
013            pecially with long inputs due to the attention mechanism’s memory overhead.
014            We observe that certain attention heads exhibit a distribution where the attention
015            weights concentrate on tokens near the query token, termed as recency aware,
016
               which focuses on local and short-range dependencies. Leveraging this insight,
               we propose RecurFormer, a novel architecture that replaces these attention heads
017
               with linear recurrent neural networks (RNNs), specifically the Mamba architec-
018
               ture. This replacement reduces the cache size without evicting tokens, thus main-
019            taining generation quality. RecurFormer retains the ability to model long-range
020            dependencies through the remaining attention heads and allows for reusing pre-
021            trained Transformer-based LLMs weights with continual training. Experiments
022            demonstrate that RecurFormer matches the original model’s performance while
023            significantly enhancing inference efficiency. Our approach provides a practical
024            solution to the computational challenges of Transformer-based LLMs inference,
025            making it highly attractive for tasks involving long inputs.

      1   I NTRODUCTION
028
029
      Transformer-based LLMs (OpenAI, 2023; Touvron et al., 2023) excel at modeling complex language
030   patterns but come with significant computational costs. During inference, the prefill phase processes
031   the input in parallel, generating the first token and initializing the key-value cache (KV-Cache),
032   while the generation phase recursively produces each token, adding new keys and values to the KV-
033   Cache (Radford et al., 2019; Brown et al., 2020). In long input tasks like document-based dialogue
034   generation, the attention mechanism’s overhead in the prefill phase leads to memory issues, making
035   optimization crucial (Yang et al., 2024).
036   Currently, one of the optimization methods applicable to the prefill phase is PyramidInfer (Yang
037   et al., 2024), which reduces the KV-Cache size by progressively removing tokens with lower at-
038   tention weights. However, the intrinsic drawback is that token removal can decrease generation
039   quality as the sequence length increases. Moreover, in modern Transformer-based inference frame-
040   works, such as vLLM (Kwon et al., 2023), the prefix caching strategy is employed, which caches
041   the KV-Cache from long documents or dialogue history to reduce computational complexity during
042   the prefill phase in long document retrieval and multi-turn dialogue scenarios. Since it is difficult to
043   predict which tokens may be needed in future token generation, such scenarios are not suitable for
044
      compression through token eviction.
045   Inspired by the dependency length minimization (DLM) phenomenon (Futrell et al., 2015) observed
046   in quantitative linguistics and the principles of attention mechanisms, we identified a local, short-
047   range token attention distribution pattern, referred to as the recency aware, where attention is con-
048   centrated on tokens close to the query token. Additionally, we discovered an attention distribution
049   pattern where the attention weights are independent of relative position, termed contextual retrieval,
050
      characterized by attention being distributed across tokens at arbitrary positions in the sequence,
      reflecting a global focus.
051
052   While the attention mechanism excels at modeling dependencies over varying distances (Vaswani
053   et al., 2017), we discover that using a linear RNNs is a more efficient option for fitting the recency
      aware so that it reduces cache size requirements. On one side, limited cache size makes linear

107   Optimization Strategies for Efficient Inference. Improving the computational and memory effi-
      ciency of Transformer-based LLMs has become a key research focus, driven by the need for scalable


                                                        2

108
      inference in complex natural language processing applications. Optimization methods can be cate-
109   gorized into token eviction-based and non-eviction-based approaches. Token eviction-based strate-
110   gies reduce storage and computation by dynamically removing less important tokens, functioning
111   as a sparse attention mechanism. BigBird (Zaheer et al., 2020) pioneered this approach by retaining
112   specific positional and random tokens, while H2O (Zhang et al., 2023) introduced attention-weight-
113   based eviction to prioritize key information. Further optimizations, such as Scissorhands (Liu et al.,
114   2024) and PyramidInfer (Yang et al., 2024), evict different tokens at each layer, with PyramidIn-
115   fer optimizing the prefill phase by expelling tokens with low attention at each layer progressively.
116   However, these methods become less efficient as the decoded sequence grows, especially in tasks
117
      requiring retention of dialogue history or external knowledge.
118   Non-eviction-based methods, such as DMC (Nawrot et al., 2024), maintain higher generation qual-
119   ity by merging the key and value of the current token with the last one in the KV-Cache. While
120   effective at controlling KV-Cache growth, non-eviction methods face limitations in the prefill phase
121   due to their sequential nature, which limits parallelization. This study aims to develop a non-eviction
122   token optimization method applicable to both the prefill and generation phases, with the goal of re-
123
      ducing cache size in Transformer-based LLMs while maintaining generation quality comparable to
      the original model.
124
125
126
      Linear RNNs. Non-linear RNNs, as the typical form of RNNs, is fundamentally constrained by
      non-linear dependencies in state transitions, which limit their ability to perform parallel computa-
127
      tions across the sequence dimension. Inspired by physical systems, T-LSTM (Balduzzi & Ghifary,
128
      2016) introduced a pioneering approach to linearize hidden state transitions by decoupling input
129   and hidden states. This linearity enables cell states to be updated through associative operations.
130   Building on this property, GILR-LSTM (Martin & Cundy, 2018) employs a parallel scan strategy to
131   accelerate computation across sequence elements.
132
      Recently, Linear RNNs based on state space models (SSM) have garnered considerable attention. By
133
      discretizing SSM, these models become applicable to discrete sequences, and when combined with
134
      the Hippo (Gu et al., 2020) memory update mechanism, they yield the structured state space (S4)
135   model (Gu et al., 2022). Expanding upon S4, the selective structured state space (S6) model (Gu
136   & Dao, 2023) further enhances sequence modeling capacity by increasing the dimensionality. By
137   integrating S6 with gating architecture, the Mamba (Gu & Dao, 2023) was introduced, showing
138   strong potential in sequence modeling tasks. However, due to limited memory capacity, Mamba
139   still falls short in tasks such as sequence copying and long-range dependency modeling compared
140   to attention-based architectures like Transformer (Jelassi et al., 2024). In RecurFormer, we identify
141   scenarios that circumvent Mamba’s limitations in modeling long-range dependencies, by focusing
142   on cases where short-term dependencies are predominant.

144   3     M ETHODOLOGY
145
146   Overview. The process begins by identifying attention heads in the model that exhibit a strong
147   short-range focus, characterized by the recency aware. Once identified, we replace the attention
148   mechanism in these heads with a more efficient linear RNNs, which is better suited for modeling
149   local dependencies. This replacement strategy allows us to maintain the model’s generation ability
150   while reducing cache size in both the prefill and generation phase. Our approach involves two steps:
151   • Select replaced head(Sec. 3.1): We first calculate the recency ratio (RR) for each attention head
152     in the pre-trained LLMs and assign a recency aware index (RA-I). Attention mechanisms in heads
153     with higher RA-I are prioritized for replacement by a linear RNNs.
154   • Replace and continual training(Sec. 3.2): We replace the attention mechanism in selected heads
155     with a linear RNNs, enabling RecurFormer to capture local dependencies more efficiently. contin-
156     ual training is then conducted to restore performance, ensuring it matches the original Transformer
157     while reducing computational costs during prefill and generation phases.

      Furthermore, when replacing the attention mechanism with a linear RNNs, the input gate in the
229   linear RNNs provides an effective solution to this issue. It projects the value vector vt for each token
230   in the sequence, adjusting the value distribution across all tokens so that the value vector of the first
231   token can become similar to those of the other tokens. Specifically, this transformation is expressed
232   as vi′ = Bi vi , where Bi is the weight vector of the input gate applied to vi , the original value vector

234   in the attention mechanism, without requiring additional sequence-level modeling operations.

236   3.2   R EPLACING S ELECTED H EADS AND CONTINUAL TRAINING
237
238   Employing attention mechanisms in recency aware dominant heads results in inefficiencies. To
239   address this, we propose substituting these heads with a linear RNNs structure, termed Mamba,
240
      within Transformer models. Mamba (Gu & Dao, 2023), based on the S6 architecture, is designed
      to model sequence information more efficiently. Although its performance in modeling long-term
241
      dependencies is constrained by finite memory compared to Transformer (Jelassi et al., 2024), it
242
      mitigates this limitation in scenarios where capturing local dependencies is critical. Additionally,
243   since Mamba’s state transitions do not rely on nonlinear activation functions, the operators within
244   its state transitions are associative, allowing for efficient parallel computation via a parallel scan
245   mechanism. This significantly enhances computational efficiency, particularly in the prefill phase,
246   where parallel computation across multiple tokens is required.
247

      5   C ONCLUSION
523
524
      We explored the necessity of self-attention in all heads of Transformer-based LLMs and demon-
      strated the potential of replacing some self-attention mechanisms with a Linear RNNs. By iden-
525
      tifying the recency aware phenomenon, where certain attention heads focus on recent tokens, we
526
      introduced RecurFormer. In RecurFormer, we replace self-attention with a linear recurrent neural
527   network in these specific heads. This substitution reduces computational costs and cache size dur-
528   ing both the prefill and generation phases without compromising performance. Our experiments
529   showed that RecurFormer effectively reuses pretrained Transformer weights with continual training,
530   maintaining generation quality while enhancing efficiency. However, RecurFormer faces a key lim-
531   itation in efficiently parallelizing the computation of Mamba blocks and self-attention heads within
532   the same layer. This challenge is particularly evident under small batch sizes, affecting hardware
533   resource utilization, which we aim to improve in future work. By integrating linear recurrence
534   into Transformer architectures, RecurFormer provides a promising foundation for future large-scale
535
      model architectures, potentially leading to more efficient and scalable neural networks.

652   HashHop provides an objective evaluation of how the quality of generated content changes with
653   sequence length. In this task, the model must sequentially reconstruct a linked list, starting from the
654   first element. Each element in the linked list is represented as a random string of length he , with the
655   target list containing hp connections, and the total input consisting of hl characters.
656   We use the hgq metric to evaluate the model’s performance, defined as the ratio of the longest correct
657   sequence starting from the first element to the total length of the target list. This metric objectively
658   reflects the model’s ability to preserve generation quality as the sequence grows.
659
      For example:
660
661                         10 → 17,      04 → 05,     01 → 02,      62 → 23,        02 → 03,
662                         99 → 85,      21 → 34,     03 → 04,      42 → 73,        05 → 06.
663
664   In this case, the valid linked list is:
665
666
                                          01 ⇒ 02 → 03 → 04 → 05 → 06.
667
      Here, the elements in light gray represent random pairs that serve as distractors, and the symbol ⇒
668   indicates the starting point of the linked list. The task challenges the model to connect pairs where
669   the second element of one pair matches the first of another, forming a valid linked list.
