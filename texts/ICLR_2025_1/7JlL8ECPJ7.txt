007                                              A BSTRACT
008
009            Banzhaf values offer a simple and interpretable alternative to the widely-used
010            Shapley values. We introduce Kernel Banzhaf, a novel algorithm inspired by
011
               KernelSHAP, that leverages an elegant connection between Banzhaf values and
               linear regression. Through extensive experiments on feature attribution tasks, we
012
               demonstrate that Kernel Banzhaf substantially outperforms other algorithms for
013
               estimating Banzhaf values in both sample efficiency and robustness to noise. Fur-
014            thermore, we prove theoretical guarantees on the algorithm’s performance, estab-
015            lishing Kernel Banzhaf as a valuable tool for interpretable machine learning.

018   1   I NTRODUCTION
019
020   The increasing complexity of AI models has intensified the challenges associated with model inter-
021   pretability. Modern machine learning models, such as deep neural networks and complex ensemble
022   methods, often operate as “opaque boxes.” This opacity makes it difficult for users to understand
023   and trust model predictions, especially in decision-making scenarios like healthcare, finance, and
024   legal applications, which require rigorous justifications. Thus, there is a pressing need for reliable
025   explainability tools to bridge the gap between complex model behaviors and human understanding.
026   Among the various methods employed within explainable AI, game-theoretic approaches have
027   gained prominence for quantifying the contribution of features in predictive modeling and enhanc-
028   ing model interpretability. While primarily associated with feature attribution (Lundberg & Lee,
029   2017; Karczmarz et al., 2022), these methods also contribute to broader machine learning tasks such
030   as feature selection (Covert et al., 2020) and data valuation (Ghorbani & Zou, 2019; Wang & Jia,
031   2023). Such applications extend the utility of explainable AI, fostering greater trust in AI systems
032
      by providing insights beyond traditional explanations.
033   Shapley values, rooted in cooperative game theory, provide a principled way to attribute the contri-
034   bution of n individual players to the overall outcome of a game (Shapley, 1953). In the context of
035   feature attribution, each “player” is a feature and the “game” is defined by a set function that maps
036   a subset of features to the prediction score of an AI model on that subset. For a given feature, the
037   Shapley value quantifies the average marginal contribution of the feature on the model’s prediction,
038
      computed as the weighted average over all possible combinations of features included in the model
      (see e.g., Equation 1) (Lundberg & Lee, 2017).
039
040   An alternative to Shapley values are Banzhaf values, which also compute each individual’s contribu-
041   tion to the system’s overall outcome (Banzhaf, 1965). While Shapley values are more widely used,
042   Banzhaf values are often considered more intuitive for AI applications since they treat each subset
043   of players as equally important, directly measuring the impact of each player across all possible
044
      combinations (see e.g., Equation 2). Additionally, Banzhaf values can be computed more efficiently
      and tend to be more numerically robust (Karczmarz et al., 2022; Wang & Jia, 2023).
045
046   For general set functions, the exact computation of Shapley and Banzhaf values is an NP-hard prob-
047   lem (Deng & Papadimitriou, 1994), so they are often approximated in practice. The problem of esti-
048   mating Shapley values, especially for model explanation, has been well-studied. A leading method
049   is KernelSHAP (Lundberg & Lee, 2017), a model-agnostic technique that leverages a connection
050
      to linear regression, approximating Shapley values by solving a subsampled weighted least squares
      problem (Charnes et al., 1988; Lundberg & Lee, 2017). KernelSHAP has been further improved
051
      with paired sampling (Covert & Lee, 2020) and leverage score sampling Musco & Witter (2024).
052
053   In contrast to Shapley values, only a few algorithms have been proposed to compute Banzhaf values
      for arbitrary set functions. For tree-structured set functions, such as decision tree based models,

077   In this work, we apply ideas that have been proven effective for estimating Shapley values to Banzhaf
078   values (Lundberg & Lee, 2017; Covert & Lee, 2020; Musco & Witter, 2024). Our starting point
079
      is a formulation of Banzhaf values in terms of a specific linear regression problem (Section 3.1).
      While a connection to linear regression was known for ‘simple’ set functions (i.e., set functions with
080
      binary outputs that satisfy a monotonicity property) in Hammer & Holzman (1992), to the best of
081
      our knowledge, we are the first to establish the connection for general set functions. We leverage
082   this connection to design a new algorithm for approximating Banzhaf values: we propose Kernel
083   Banzhaf, inspired by KernelSHAP, to solve a subsampled instance of the Banzhaf linear regression
084   problem (Section 3.2). Similar to Leverage SHAP (Musco & Witter, 2024), we exploit leverage
085   score sampling to give approximation guarantees on the ℓ2 -norm error.

233   We now consider the time complexity of Algorithm 1. The time to randomly sample with replace-
234   ment is O(n) per sample, along with the time to compute the complement and store both vectors.
235
      Let Tm be the time complexity to evaluate the set function m times, possibly in parallel. This cost
      is unavoidable for any algorithm that evaluates the set function m times. Finally, the least squares
236
      regression problem requires O(mn2 ) time because the matrix Ã has m rows and n columns and the
237
238
      vector b̃ has m entries. In total, the time complexity of Algorithm 1 is O(Tm + mn2 ). For most set
      functions, we expect the O(Tm ) term to dominate. For example, even a forward pass on a shallow
239
      fully connected neural network with dimension at most n will take O(n2 ) time per sample.
240
241   The state-of-the-art methods for Shapley estimation, KernelSHAP (Lundberg & Lee, 2017; Covert

244
      approach (until m = 2n , in which case we can exactly compute Banzhaf values). Moreover, sam-

      exactly m samples. Further details are provided in Appendix G.

326
      We conduct a detailed experimental evaluation and compare Kernel Banzhaf against state-of-the-art
327   estimators across eight popular datasets with varying size (having between 8 and 241 features) These
328   datasets have been used in prior research for semi-value-based model explanation (Lundberg & Lee,
329   2017; Covert & Lee, 2020; Lundberg et al., 2020; Karczmarz et al., 2022), and they are described
330   in detail in Appendix C. To assess the accuracy of the estimators, we employ the ℓ2 -norm error as
331   our primary error metric for evaluation. While prior work used convergence ratio as a measure of
332   effectiveness (Wang & Jia, 2023; Covert & Lee, 2020), convergence is insufficient to measure how
333   close the estimators are to exact Banzhaf values. We use XGBoost (Chen & Guestrin, 2016) for
334   our models in the main experiments, which makes it possible to apply the tree-based algorithm of
335
      (Karczmarz et al., 2022) to compute exact Banzhaf values. Implementation details are given in Ap-
      pendix B. We also assess the effectiveness of our approach on neural network models using smaller
336
      datasets for which Equation 2 can be used to calculate exact Banzhaf values. These experiments are
337
      presented in Appendix D and their results are consistent with those for the tree models. In addition
338   to ℓ2 -norm, we use the objective value naturally suggested by the linear regression formulation to
339   measure the estimation error; the results in Appendix E corroborate the superior performance of
340   Kernel Banzhaf.

460
      Game-Theoretic Explainable AI Semivalues like Shapley and Banzhaf values, derived from
461
      game theory, are widely used for machine learning tasks. Feature attribution involves determin-
462
      ing the impact of individual features on model predictions, with Shapley values notably used in
463   Lundberg & Lee (2017). This method also facilitates feature selection by modifying set functions to
464   measure the feature’s impact on model performance via loss metrics (Covert et al., 2020). Data val-
465   uation extends to treat individual data points as players, assessing their influence on model training
466   outcomes like accuracy or AUC (Ghorbani & Zou, 2019).
467
      A parallel trend in utilizing Banzhaf values has been noted, with studies indicating their robustness
468
      compared to Shapley values, particularly under conditions of low numerical precision (Karczmarz
469   et al., 2022). Furthermore, Wang & Jia (2023) established the superiority of Banzhaf values among
470   semivalues regarding safety margin, the threshold of set function perturbations that the value order
471   can withstand without changing. These properties have led to extensive application of Banzhaf
472   values for feature attribution (Datta et al., 2015; Kulynych & Troncoso, 2017; Sliwinski et al., 2018;
473   Patel et al., 2021; Karczmarz et al., 2022) and, more recently, in data valuation, where Wang & Jia
474   (2023) and Li & Yu (2024) have shown their superior performance compared to Shapley values.
475
476   Shapley Estimators KernelSHAP, introduced by Lundberg & Lee (2017), employs game-
477   theoretic Shapley values in a linear regression framework to explain any machine learning model’s
478   predictions. The method’s efficiency and applicability are enhanced through paired sampling and
479   sampling without replacement, which balances the number of samples used to update each Shapley
480
      value, as refined by Covert & Lee (2020); Jethani et al. (2021). In recent work, Musco & Witter
      (2024) proposed Leverage SHAP, a variant of KernelSHAP that uses leverage score sampling to
481
      build the subsampled linear regression problem. Leverage scores are a statistical quantity that intu-
482
      itively measure the importance of a data point (Woodruff et al., 2014; Drineas & Mahoney, 2018).
483   Leverage SHAP outperforms KernelSHAP and, because of the mathematical properties of leverage
484   scores, provably returns accurate approximations. Meanwhile, TreeSHAP, developed by Lundberg
485   et al. (2020), specifically tailors the SHAP framework for tree-based models, ensuring faster, exact
      computations by directly incorporating tree structures into Shapley value calculations.

501
502   Banzhaf Estimators Merrill III (1982) initially introduced approximating the Banzhaf power in-
503   dex through Monte Carlo sampling across subsets. Building on this, Bachrach et al. (2010) applied
504   Hoeffding bounds to analyze the sample complexity required for these approximations. Further
505   advancing the method, Wang & Jia (2023) developed the Maximum Sample Reuse (MSR) Monte
506   Carlo estimator. This approach samples sets individually for each term in the marginal utility calcu-
507   lation, facilitating the reuse of samples across different computations of marginal utility. However,
508   due to this variation, the MSR estimates of Banzhaf values are correlated among different players
509
      and exhibit large variance. Moreover, for structured set functions such as decision trees, where
      predictions are aggregated from root to leaf, Banzhaf values can be computed with exact precision
510
      (Karczmarz et al., 2022). This computational efficiency is attributable to the ability to independently
511
      isolate and assess the impact of each feature along the decision path. Despite its effectiveness within
512   this context, this method lacks generalizability to other models and machine learning tasks.
513
514   Banzhaf Values and Regression Hammer & Holzman (1992) shows how least squares regression
515   can approximate the Banzhaf power index, which only requires a ‘simple’ set function v : 2N →
516   {0, 1}. Their regression formulation differs from what we propose in Theorem 3.2 and may not yield
517   exact Banzhaf values even when no subsampling is done. Alonso-Meijide et al. (2015) proposed
518   modifications of Banzhaf values to enable them to satisfy the Efficiency property, via additive and
519   multiplicative normalization, and formulated a least squares regression to obtain them.
520

      6                    C ONCLUSION
522
523   In this paper, we presented Kernel Banzhaf, an innovative estimator for Banzhaf values that lever-
524   ages linear regression to improve the efficiency and robustness of Banzhaf value estimation for ex-
525
      plainable AI. Our empirical and theoretical analysis reveal that Kernel Banzhaf outperforms existing
      methods for Banzhaf estimation. Additionally, our findings highlight its superior robustness com-
526
      pared to Shapley estimators, establishing Kernel Banzhaf value as a viable and effective alternative
527
      to other semivalue-based methods for feature explanation in machine learning models.
528
529   Future research can focus on extending Kernel Banzhaf to diverse set functions and examine its
530   potential for integration into real-time systems. This will help us to further understand the scalability
531
      and applicability of our approach in diverse machine learning contexts. Additionally, a concrete
      direction for future work is to develop adaptations of Kernel Banzhaf for weighted Banzhaf values
532
      (Li & Yu, 2024).

      Feature perturbation and exact Banzhaf value calculation There are generally two approaches
813   to handling removed features in feature perturbation for general set functions, as discussed in Chen
814   et al. (2020) and Kumar et al. (2020). Given an explicand x and a subset of features S, define xS as
815   the observation where xSi = xei if feature i ∈ S and, otherwise, xS̄ is sampled from a distribution.
816   The first method involves sampling from the conditional distribution of the removed features, where
817   replacement values for the absent features are sampled according to xS̄ ∼ p(xS̄ |xS ). This approach,
818   while precise, is computationally expensive. Alternatively, the marginal distribution can be used
819   where the observed features xS are ignored, and replacement values are sampled according to xS̄ ∼
820   p(xS̄ ). Due to its lower computational complexity, we adopt the latter approach.
821
822
      For implementation, distinct feature perturbation methods are applied depending on the set function,
      i.e. model type. For tree-based models, as we need to use the tree-based algorithm for calculating the
823
      ground truth Banzhaf values to measure errors, we utilize a method aligned with Algorithm 1 from
824
      Karczmarz et al. (2022), which computes predictions using partial features. Specifically, during tree
825   traversal, if feature i ∈ S, we proceed according to the threshold to select the child node; if i ∈
                                                                                                        / S,
826   we traverse both children and compute a weighted average of the predictions, effectively nullifying
827   the influence of features not in S without any feature value replacement.
828
      For neural network models, instead of using fixed baseline values for the removed features, we
829
      compute the average of the model’s predictions using replacement values randomly sampled from
830
      50 baseline points, different from the explicand. The explicand xe is repeated 50 times, with the
831   non-selected features replaced by values from these baseline points, and the average of M (xS ) is
832   taken to estimate the impact of marginalizing out the non-selected features. To calculate ground
833   truth Banzhaf values, we evaluate all 2n subsets of features in this way.
834
835   Sample size for Monte Carlo Given a sample size of n, the MSR and Kernel Banzhaf estimators
836   effectively use n samples to estimate Banzhaf values for all features concurrently. In contrast, the
837   Monte Carlo method estimates each feature independently and requires a fair allocation of the total
838   m samples. To achieve this, we implement a loop where for each iteration i = 1, . . . , m, we use
839   i mod n + 1 samples when i is divisible by n, and i mod n samples otherwise. This approach
840
      ensures that the Monte Carlo method also utilizes exactly m samples, maintaining consistency in
      sample usage across different estimators.

1050   H                   P ROPERTIES OF S HAPLEY VALUES AND BANZHAF VALUES
1051
1052   Shapley values satisfy four desirable properties: Null Player ensures that a player who does not
1053   contribute to any coalition, meaning their inclusion in any subset of players does not affect the
1054   overall outcome, is assigned a value of zero. Symmetry requires that two players who contribute
1055
       equally to all possible coalitions receive the same value. Linearity requires that the Shapley value
       of a player in a combined game (formed by adding two games together) is equal to the sum of that
1056
       player’s Shapley values in the two individual games. Efficiency requires that the total value assigned
1057
       to all players must sum to the value generated by the full set of players. (Shapley, 1953).
1058
1059   Instead of the Efficiency property, the Banzhaf index satisfies 2-Efficiency, which requires that the
1060   sum of the values of any two players equals the value of these two players when considered jointly
1061
       in a reduced game setting (Banzhaf, 1965; Lehrer, 1988). The necessity of the efficiency property
       has been debated in the context of machine learning. Sundararajan et al. (2017) suggest that the
1062
       Efficiency property is only essential in contexts where semi-values, such as those in voting games,
1063
       are interpreted numerically; Kwon & Zou (2022) argues that the utility function in machine learning
1064   applications often does not correspond directly to monetary value, so aligning the sum of data values
1065   with total utility is unnecessary. In applications where the primary goal involves ranking features
1066   according to their importance or evaluating data, the exact numerical contribution of each feature is
1067   less critical. Both Banzhaf and Shapley values, despite their theoretical disparities, often yield the
1068   same ordering of players as shown in Karczmarz et al. (2022), which suffices for these applications.
1069   Therefore, given their efficiency and robustness properties, Banzhaf values serve as particularly
1070   effective tools in machine learning tasks (Wang & Jia, 2023).
