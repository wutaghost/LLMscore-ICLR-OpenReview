009                                               A BSTRACT
010
011
               The limited context window of contemporary large language models (LLMs) re-
012
               mains a huge barrier to their broader application across various domains. While con-
013
               tinual pre-training on long-context data is a straightforward and effective solution,
014            it incurs substantial costs in terms of data acquisition and computational resources.
015            To alleviate this issue, we propose SharedLLM, a novel approach grounded in the
016            design philosophy of multi-grained context compression and query-aware infor-
017            mation retrieval. SharedLLM is composed of two short-context LLMs such as
018            LLaMA-2, termed upper model and lower model. The lower model functions as a
019            compressor while the upper model acts as a decoder. The upper model receives
020            compressed, multi-grained context information from the lower model and performs
021            context-aware modeling on the running text. Information transfer between the
022
               compressor and decoder occurs only at the lowest layers to refrain from long
               forward paths in the lower model and redundant cross-attention modules in the
023
               upper model. Based on this architecture, we introduce a specialized tree-style data
024
               structure to efficiently encode, store and retrieve multi-grained contextual infor-
025            mation for text chunks. This structure, combined with a search algorithm, enables
026            rapid encoding and retrieval of relevant information from various levels of the tree
027            based on the input query. This entire process, wherein the sender and receiver are
028            derived from the same LLM layer, is referred to as self-injection. In our evaluation
029            on long-context modeling and understanding tasks, SharedLLM achieves superior
030            or comparable results to several strong baselines, striking an effective balance
031            between efficiency and performance. Meanwhile, with the aforementioned design
032            choices, SharedLLM can greatly reduce memory consumption, and demonstrates
033
               substantial speed-ups over other advanced baselines (2× over streaming, 3× over
               encoder-decoder architectures). The core code of our implementation along with
034
               training and evaluation is available in appendix and supplementary.

038   1   I NTRODUCTION
039
040
      Since the release of GPT-3 (Brown, 2020), the rapid advancement of large language models
041
      (LLMs) (Chowdhery et al., 2022; Achiam et al., 2023; Touvron et al., 2023a;b; Dubey et al., 2024)
042   has revolutionized the NLP research community and transformed various workflows. Pretrained on
043   trillions of tokens, LLMs exhibit remarkable abilities, such as completing unfinished text or code and
044   following human instructions to perform designated tasks after minimal supervised fine-tuning (Wei
045   et al., 2021; Chung et al., 2024).
046
      Despite their impressive capabilities, several factors limit their broader application. One major
047
      constraint is the context window size (Hsieh et al., 2024), which refers to the maximum number of
048   tokens an LLM can process smoothly in a single input. The length of context window is typically set
049   during pretraining—for example, LLaMA and LLaMA-2 have context windows of 2,048 and 4,096
050   tokens, respectively. When input text exceeds this limit, LLMs may exhibit erratic behavior during
051   inference. Unfortunately, due to GPU memory constraints, high training costs, and the scarcity of
052   long-context training data, LLMs are often pretrained with relatively short context windows. This
053   limitation severely restricts their use in many daily tasks of large context lengths, such as long
      document summarization and information retrieval, where much longer windows are needed.

054
      Many researchers endeavour to extending the context window of LLMs while minimizing the time,
055   memory, and training costs during both training and inference. One approach involves post-pretraining
056   LLMs on long-context corpora using hundreds of GPUs (TogetherAI, 2023; Xiong et al., 2024).
057   Another line of work explores position interpolation (Chen et al., 2023; Peng et al., 2023), which
058   rescales the RoPE (Rotary Position Embedding) frequency and attention scores. While this method
059   is canonical, it still requires long-context continual pretraining. For example, YaRN (Peng et al.,
060   2023) extends LLaMA’s context length to 128K tokens by continuing pretraining on 64K-token
061   sequences using full attention. The combination of parameter-efficient fine-tuning (PEFT) and
062   sparse attention (Chen et al., 2024) accelerates tuning but faces challenges with extrapolation. Other
063
      approaches like streaming-style architectures (Xiao et al., 2024b; Zhang et al., 2024a; Yen et al.,
      2024), maintain a constant-sized memory that operates as a sliding window. While this design
064
      significantly reduces memory usage, its specialized attention pattern causes incompatibility with high-
065
      performance attention implementations like FlashAttention (Dao et al., 2022; Dao, 2023), potentially
066   leading to slower inference speeds. Context compression techniques are also widely explored (Zhang
067   et al., 2024a; Yen et al., 2024). Although they offer high parallelism, improving speed, they tend to
068   consume significant memory, greatly limiting their real applications.
069
      To strike a balance between efficiency and performance, we propose SharedLLM in this pa-
070
      per. SharedLLM features a lightweight hierarchical architecture that consists of one upper model and
071
      one lower model. Both models are initialized from the same off-the-shelf checkpoint of a short-
072   context LLM, either in full or in part. Since there is no disparity in the hidden space between the
073   two submodules, SharedLLM can be trained from scratch without extra stages for hidden-space
074   alignment. The lower model compresses past context information into multi-grained representations.
075   Through layer-wise connections between the lower and upper model, the compressed information
076   can be passed to the upper model for context-aware language modeling.
077
      For better organization and utilization of this multi-grained information, we further propose a
078   dedicated data structure, dubbed context tree. The trees operate in parallel, with each tree handling
079   an independent text chunk split from the longer raw input. The tree nodes contain token sequences
080   of varying lengths, which are encoded and downsampled by the lower model to obtain a set of
081   meaningful representations. The sequence corresponding to each node is split from its parent, making
082   it a subsequence of all its ancestors. Meanwhile, we set the compression ratio proportional to the
083   sequence length. As a result, the nodes at higher levels have longer sequence and larger compression
084   ratio, which express more coarse-grained information after encoding, as opposed to nodes at lower
085   levels. We further develop an algorithm for dynamic construction and search on the tree given the
086
      task-specific query. The algorithm accelerates the information gathering process by identifying the
      most relevant text segments and encoding them as fine-grained representations, while converting less
087
      relevant parts into coarse-grained representations in a depth-first manner. In the information injection
088
      stage, since the obtained context trees are ordered, we define a rule to assign chunk-level position ids
089   for these encoded keys from the lower model, as well as the queries from the upper model. With these
090   unique chunk-level ids, queries can perceive the relative position of trees to generate high-quality
091   output.
092
      With these design principles, SharedLLM presents extraordinary performance on down-stream tasks
093
      with high efficiency. Specifically, on language modeling tasks, trained on text of up to 8K tokens, our
094
      model demonstrates excellent extrapolation capabilities when tackling sequences up to 128K-token
095   length. On other long-context instruction-following tasks with input lengths ranging from thousands
096   to millions of tokens, SharedLLM delivers promising results comparable to several strong baselines.
097   In terms of system indicators, all experiments with a maximum length of over 200K can be conducted
098   on a single A800 80G GPU. SharedLLM delivers several times the speed of all baselines while
099   maintaining relatively low memory consumption.

103
104   Long-context Language Models. There are two prevalent routines to build LLMs that are capable
105   of processing extremely long text: directly pretraining on large corpus of targeted context length from
106   scratch (Touvron et al., 2023a; Dubey et al., 2024; Jiang et al., 2023; GLM et al., 2024) or adapting
107   short context-window LLMs to longer context lengths via combined various techniques (Tworkowski
      et al., 2024). The former approach consumes tremendous data and computational resources, while


                                                         2
      Under review as a conference paper at ICLR 2025


108
      the latter allows for more convenience and flexibility for researchers and developers to explore
109   potential optimization to the default settings (Fu et al., 2024). The core idea behind these adaptations
110   is to mimic short input scenarios (i.e., length within the model’s text window) when the input
111   length exceeds window size. Attention map manipulation is the most common approach for this
112   goal, which can be realized via positional encoding (PE) rescaling, such as ALiBi (Press et al.,
113   2021), positional interpolation (PI) (Chen et al., 2023) and YaRN (Peng et al., 2023), or positional
114   index rearranging (Xiao et al., 2024b; Ding et al., 2023; An et al., 2024; He et al., 2024). Both
115   directly or indirectly adjust attention scores to be similar as the short-input scenarios so that the
116   model can handily deal with. Another line of works compress past tokens sequentially into dense
117
      representations (Chevalier et al., 2023; Zhang et al., 2024a) as input at the next step or store them in
      an external retrievable memory (Wu et al., 2022; Xiao et al., 2024a) to reduce the input lengths. Yen
118
      et al. (2024) utilizes small model such as RoBERTa (Liu, 2019) for context encoding to boost speed
119
      and enable higher parallelism. However, this heterogeneous architecture necessitates meticulous
120   task design for the extra pretraining and warmup stages to stabilize the fine-tuning process. In
121   contrast to these works, our method directly tunes off-the-shelf models to compress context into
122   structural representations for query-aware retrieval. Powered by efficient architecture design and a
123   fast-forwarding mechanism, the whole procedure can be fully paralleled online without excessive
124   memory usage, which greatly cuts down the latency during inference time.
125
126   Efficient Methods for Long-context Modeling. In vanilla self-attention, the space and time
127   complexity grows quadratically (O(L2 )) with the input sequence length L, which can cause out-of-
128   memory (OOM) issues on GPU clusters. A straightforward solution is to add parameter efficient
129   fine-tuning (PEFT) modules (Chen et al., 2024; Zhang et al., 2024a;b) to shrink the size of gradient
130   tensors during backward propagation. Many works strive to reduce the memory footprint of attention
131
      computation to enhance computational efficiency. Longformer (Beltagy et al., 2020) introduces a
      hybrid attention pattern to capture local and global semantic features concurrently. Katharopoulos
132
      et al. (2020) designs linearized attention that merely demands O(L) space to accomplish attention
133
      computation. FlashAttention (Dao et al., 2022; Dao, 2023) and PagedAttention (Kwon et al.,
134   2023) maximize the memory efficiency from system’s perspective. More recently, Xiao et al.
135   (2024b) discovers the “attention sink” phenomenon and proposes streaming-llm to address high
136   perplexity issue in generation under window-attention. Our work basically follows the efficient
137   design principle in three aspects: 1) lightweight architecture through lower layer self-injection; 2)
138   compact structural representations via structural information extraction and compression; 3) efficient
139   construction and retrieval algorithm based on the proposed context tree structure.

      3.1   OVERVIEW
148
      As illustrated in Figure 1, SharedLLM adopts a hierarchical architecture, akin but not identical to
149
      classical encoder-decoder models. The lower model, or the “compressor”, breaks down the long input
150   context XC into smaller chunks that can be processed within limited GPU memory. It then uses the
151   same LLM model to compress each context chunk into compact and structured representations in
152   parallel. The upper model, or the “decoder”, takes the rear part of the input text (the running context,
153   such as questions) as input, then integrates the compressed information from the lower model, and
154   finally predicts future tokens in an auto-regressive manner.
155
      The lower and upper models are connected via shared key-value (KV) states and cross-attention
156   modules between corresponding layers. To enable efficient and effective information retrieval and
157   integration, the context information processed by the lower model is organized into a binary tree,
158   referred to as the context tree, which stores multi-grained information at different levels. This
159   structure allows the upper model to leverage its processed running text to efficiently retrieve relevant
160   information from the binary tree based on a depth-first search algorithm. The retrieved information
161   is then integrated with the input through cross-attention, enabling the model to answer questions or
      perform language modeling.

234   Query-Dependent Dynamic Tree Construction and Search. A task-specific query is typically
235   highly relevant to certain tree nodes while being less relevant to others. For highly relevant nodes,
236   further expansion is necessary to extract fine-grained information. However, for less relevant nodes,
237   expansion is unnecessary. Thus, instead of constructing an entire static context tree as aforementioned,
238
      we build a query-dependent dynamic tree that expands only the relevant nodes, as shown in Figure 2,
      significantly saving both GPU memory and time.

368   Training We train SharedLLM on an 8× A800 GPU machine. The batch size is set to 1 per GPU
369
      with gradient accumulation of 16 steps (global batch size is 128) for language modeling and 1 step
      (global batch size is 8) for SFT. Zero Redundancy Optimizer (ZeRO) stage 3 from DeepSpeed without
370
      offload is enabled in both training to distribute the memory allocation among GPUs. The cross-
371
      attention layers remain fully tunable, while we opt to train upper model’s top N − M self-attention
372   layers in language modeling as post-injection aggregation for faster convergence. No parameter
373   efficient fine-tuning (PEFT) techniques, such as LoRA, are applied during both training, as PEFT
374   seriously slows down model’s convergence (Chen et al., 2024), which actually requires longer tuning
375   time than partial parameter fine-tuning to reach the optimum. We adopt AdamW optimizer with the
376   starting learning rate 1e−5 and cosine scheduler during training. The chunk size is set to 1,024 for
377   langauge modeling or 512 in SFT, with tree height h = 3 and compression ratio β = 8. For other
      configurations and hyperparameters, please refer to Appendix A.1 for more details.

      5    C ONCLUSION
526
527
      In this work, we present SharedLLM, which leverages a self-injection mechanism to adapt a pair of
528   short-context LLMs for efficient long-context modeling. By integrating the operations of context
529   compression and key information retrieval into a dedicated binary-tree structure, SharedLLM excels in
530   language modeling and various downstream instruction-following tasks, while maintaining excellent
531   memory and time efficiency. Besides, SharedLLM is directly trained from off-the-shelf LLMs,
532   eliminating the need for additional feature alignment steps and making implementation easier. We
533   hope this learning paradigm can be generalized to other short-context LLMs, offering a scalable
534   approach for a context-window extension to an arbitrary length.
535   Limitations. While SharedLLM demonstrates superior performance on both language modeling
536   and long-context benchmarks, as well as high efficiency in terms of time and memory, there are still
537   some limitations. First, although this work strikes a relatively good balance between efficiency and
538   performance at the model architecture level, further improvements could be achieved by optimizing
539   at the system and hardware levels. Second, while a simple and effective retrieval mechanism is
      implemented in this work, more advanced retrieval techniques, such as BM25 (Robertson et al.,


                                                         10

      2009) and Graph-RAG (Edge et al., 2024), were not explored and may further enhance performance.
541   We aim to pursue these improvements in future research.

723
724
      We provide the pseudo code for the online split-and-search algorithm introduced in Section 3.2, from
725   the splitting of root node till collecting all key-value states for all preserved nodes and all M layers.
726   The code snippet in the entire model.py file can be found in the supplementary material.
727
728
729   Algorithm 1: Pseudocode of dynamic Construction-and-Search.
730
      # N: number of trees; L: chunk size
731
      # depth: tree depth; chunk_ids: the entire input ids for chunk in shape (N, L)
732   # gamma: a hyper-parameter to adjust the variance of the gaussian sampling
733
      selected_input_ids = chunk_ids
734   selected_length = chunk_ids.shape[-1]
      all_kvs = []
735
736   for i in range(depth):
         # sample lengths of left and right child
737      if i < depth - 1:
            half_length = last_length // 2
738         sigma = half_length / gamma
739         delta = random.randn(1) * sigma
            l_left, l_right = half_length - int(delta), half_length + int(delta)
740
              # split the node into two children
741           left_input_ids, right_input_ids = input_ids[:l_left], input_ids[-l_right:]
742           # query_aware is a flag indicating if the selected nodes are determined on query
              if query_aware:
743              # short forward (1-layer) to get representation vectors for the query and two nodes
                 h_q = upper_model(query, 1)
744              h_left, h_right = lower_model(left_input_ids, 1), lower_model(right_input_ids, 1)
745              selected = argmax(sim(h_q, h_left), sim(h_q, h_right)
              else:
746              selected = 1 # deterministic example, can change to 0 or random selection
747           selected_input_ids = [left_input_ids, right_input_ids][selected]
748           selected_length = [l_left, l_right][selected]
749          preserved_input_ids = [left_input_ids, right_input_ids][1 - selected]
          else:
750          preserved_input_ids = cat(last_input_ids.chunk(2, -1), 0)
751
          cur_level_kvs = lower_model(preserved_input_ids).past_key_values
752       cur_level_kvs = downsample(cur_level_kvs)
          all_kvs.append(cur_level_kvs)
