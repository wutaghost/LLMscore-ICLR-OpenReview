011                                              A BSTRACT
012
013            Text-to-graph generation, aiming for controlled graph generation based on nat-
014            ural language instructions, holds significant application potentials in real-world
015
               scenarios such as drug discoveries. However, existing generative models fail to
               achieve text-to-graph generation in the following two aspects: i) language model-
016
               based generative models struggle with generating complex graph structures, and
017
               ii) graph-based generative models mainly focus on unconditional graph generation
018            or conditional generation with simple conditions, falling short in understanding as
019            well as following human instructions. In this paper, we tackle the text-to-graph
020            generation problem by employing graph diffusion models with guidance from
021            large language models (LLMs) for the first time, to the best of our knowledge.
022            The problem is highly non-trivial with the following challenges: 1) How to align
023            LLMs for understanding the irregular graph structures and the graph properties
024            hidden in human instructions, 2) How to align graph diffusion models for follow-
025            ing natural language instructions in order to generate graphs with expected rela-
026
               tional semantics from human. To address these challenges, we propose a novel
               LLM-aligned Graph Diffusion Model (LLM-GDM), which is able to generate
027
               graphs based on natural language instructions. In particular, we first propose the
028
               self-supervised text-graph alignment to empower LLMs with the ability to accu-
029            rately understand graph structures and properties by finetuning LLMs with several
030            specially designed alignment tasks involving various graph components such as
031            nodes, edges, and subgraphs. Then, we propose a structure-aware cross-attention
032            mechanism guiding the diffusion model to follow human instructions through in-
033            herently capturing the relational semantics among texts and structures. Extensive
034            experiments on both synthetic and real-world molecular datasets demonstrate the
035            effectiveness of our proposed LLM-GDM model over existing baseline methods.

      Graph generation is widely adopted in many real-world applications, such as molecule design (Xu
040
      et al., 2023; Gruver et al., 2023), social network analysis (Grover et al., 2019), code comple-
041
      tion (Brockschmidt et al., 2019), neural architecture search (NAS) (Lee et al., 2021), etc., yet its
042   accessibility remains limited for users unfamiliar with graph concepts or coding skills since the
043   models require expert knowledge to interact. Text-to-graph generation, that is to generate graphs
044   following natural language instructions, holds significant application potentials in real-world sce-
045   narios. In molecular design, for instance, a researcher might specify a molecule as ”soluble in water,
046   stable at high temperatures, and effective against a specific enzyme.” A text-to-graph generation
047   model would interpret these instructions to create a graph, with nodes representing atoms and edges
048   symbolizing bonds, thereby constructing a molecule that potentially fulfills all specified conditions.
049   The development of such text-to-graph generation technologies can significantly accelerate the dis-
050
      covery and optimization of new molecules (Bhowmik et al., 2024; Moret et al., 2023; Flam-Shepherd
      et al., 2022; Hsu et al., 2022).
051

      eration. As illustrated in Figure 1, these models generate graphs by describing nodes and edges in

      2.1   G RAPH DIFFUSION MODEL
114
115   Diffusion models are a class of generative models that recently gained popularity for their excel-
116   lent performance in computer vision. Recently, several works have successfully applied diffusion
117   models to graph generation. In this paper, we focus on graph diffusion models based on stochastic
118   differential equations (SDEs), and provide a brief description of them below.
119
      A graph diffusion model consists of a forward process and a reverse process, both of which are
120
      defined as SDEs that operate on graph data. Consider a graph G = (X, E), where X represents the
121   node features and E represents the edge features. The forward process introduces Gaussian noise
122   into G as the time variable t increases from t = 0 to t = T :
123
                                           dG = f (G, t)dt + g(t)dw,                              (1)
124
      where f (G, t) and g(t) are predetermined functions, and dw is a standard Wiener process. We
125
      denote the value of G at time t as G(t). With appropriate choices of f and g, G(T ) becomes
126
      indistinguishable from Gaussian-distributed values and contains nearly no information about the
127   original graph G.
128
129   The reverse process goes backwards in time and describes the reverse of the forward SDE. It takes
130
      the following form as demonstrated in earlier works (Song et al., 2021).
131                          dG = [f (G, t)dt − g(t)2 ∇G log pt (G)]dt + g(t)dw,                    (2)
132   where pt (G) is the marginal distribution of G(t), and ∇G pt (G) is called its score.
133   Since the score is intractable, diffusion models use neural networks to approximate it as sθ (G, t) ≈
134   ∇G pt (G), which can be trained using denoising score matching as follows:
135
               Lscore = Et EG(0) EG(t)|G(0) λ(t)∥sθ (G(t), t) − ∇G(t) log p0t (G(t) | G(0))∥2 ,

                                                                                                         (3)
136
137   where p0t (G(t) | G(0)) is the transition probability of the forward process, and λ(t) is a weighting
138
      function. After training, new graphs can be generated with graph diffusion models by starting with
      G(T ) sampled from Gaussian distributions and solving the reverse process using methods like Euler-
139
      Maruyama or Predictor-Corrector methods (Song et al., 2021).

      2.2   C LASSIFIER - FREE DIFFUSION GUIDANCE
142
143   Classifier-free guidance (Ho & Salimans, 2021) is a widely used technique for conditional genera-
144   tion using diffusion models. It augments the score prediction network in diffusion models with the
145   conditional information as an additional input. Let c be the condition, the conditional score network
146   sθ (G, t, c) is trained to approximate the conditional score ∇G pt (G | c).
147
      To improve the quality of conditional generation, classifier-free guidance introduces a scale factor
148   that controls the influence of the condition. Let k be the scale factor, classifier-free guidance modifies
149   the estimated score function in the reverse process by scaling the difference between the predicted
150   conditional and unconditional scores:
151                            scfg (G, t, c) = sθ (G, t) + k(sθ (G, t, c) − sθ (G, t)).                     (4)
152
153   In practice, the scale factor is large than 1, so the influence of conditional data is amplified. Data
154   generated with higher scale factors tend to be more consistent with the provided condition, while
155   lower scale factors can result in increased generation diversity.

192   3.1   F RAMEWORK
193
194   We use a diffusion model for graph generation in our method. We represent a graph G by its node
195   types X = {xi } and adjacency matrix E = {eij }. Here, xi ∈ {1, 2, . . . , Cnode } is the type of
196   the i-th node in G, and eij ∈ {0, 12, . . . , Cedge } is the type of edge between nodes i and j, with
197   eij = 0 indicating no edge. Our method aims to generate corresponding graph data G for a given
198
      text description T by learning the conditional distribution pθ (G|T ).
199   The framework of our method is illustrated in Figure 2. In the first stage of our method, we use the
200   self-supervised text-to-graph alignment task to finetune the Llama-3-8B model, obtaining a graph-
201   aligned LLM. In the second stage, we construct a conditional graph diffusion model to generate
202   graphs according to text description, and use a graph transformer (Yun et al., 2019) with structure-
203   aware cross-attention as its conditional score predictor.
204   For a given text description T , the conditional score predictor uses a feature extractor with a graph-
205   aligned LLM to extract text features from T , as introduced in Section 3.2. Then, the score predictor
206   uses structure-aware cross-attention to modify the prediction results based on the extracted text
207   features, as detailed in Section 3.3. During the generation process, we use classifier-free guidance
208   to generate graphs according to text description T .
209
210   3.2   S ELF - SUPERVISED TEXT- GRAPH ALIGNMENT
211
212   To incorporate text descriptions into the generation process, we need to extract features from the
213   text. Pretrained language models are known to produce high-quality text features for downstream
214   tasks. With billions of parameters, LLMs demonstrate strong performance in various graph tasks (Li
215   et al., 2023), indicating their ability to understand the graph structure contained in text. This makes
      it feasible to use LLMs to extract text features relevant for graph tasks.

      3.3   S TRUCTURE - AWARE CROSS - ATTENTION MECHANISM
239
240   To allow text-to-graph generation using diffusion models, it is necessary to incorporating text fea-
241   tures into the conditional score predictor. While there are methods like affine conditioning or cross-
242   attention for constructing the score predictors of conditional diffusion models, these approaches do
243   not account for the complex nature of graph data. Directly applying them to nodes and edges in
244   graph diffusion models can lead to suboptimal results. Since the edges in graphs represent the re-
245   lationship between nodes, an effective conditioning method for graphs should respect this property
246   and ensure the edge conditioning accounts for the nodes. Additionally, the size of the adjacency ma-
247
      trix is quadratic in the number of nodes. For more computationally expensive conditioning methods
      like cross-attention, calculating edge conditioning for each edge independently is costly.
248
249   In this section, we propose a structure-aware cross-attention method for graph diffusion models,
250   which integrates a sequence of conditional text features into the score predictor network by comput-
251   ing attention between the graph data and the text features.
252   Specifically, we use the structure-aware cross-attention mechanism in the score predictor module
253   of the graph diffusion model, where the results of node attention and edge attention are added into
254   the network using residual connections. Let X and E be the node and edge features in some layer
255   of the score predictor, and C be the sequence of text features. Structure-aware cross-attention first
256   computes the cross-attention between node features and text features as follows, and use the attention
257   results for node conditioning:
258                                  Q = XWQ ,        K = CWK ,          V = CWV ,                           (7)
259                                                                  T
                                                            QK
260                                             A = softmax( √ ),                                            (8)
261                                                            d
262                                                Xcond = AV,                                               (9)
263   where Q, K, and V are weight matrices for queries, keys, and values, d is the dimensionality of
264   keys, A is the attention score for nodes, and Xcond is the node conditioning result.
265
      Then, structure-aware cross-attention computes the attention scores for edges based on the node
266
      attention results. For each edge (u, v), its attention score should be related to the attention scores of
267
      node u and v. We compute two scalar values G1,uv using a gating mechanism for each edge based
268   on its features Euv , which represents the influence of two endpoints u and v in the edge:
269

279                                           Econd,uv = Aedge,uv V.                                 (12)

281   By deriving the edge conditioning from the node conditioning, structure-aware cross-attention can
282   utilize the text features in the conditional score predictor with relatively low computational costs, and
283   captures the relational semantics among texts and structures more efficiently. In our experiments,
284
      we apply structure-aware cross-attention in each layer of the score predictor.

337
      We conducted experiments on the text to molecule dataset and demonstrated the effectiveness of our
      method in generating molecular graphs based on text descriptions through comparison with the text
338
      to molecule text model.
339
340
      Datasets Generating molecules according to text is an emerging task, and currently the only
341
      widely used and public dataset in this task is the ChEBI-20 dataset (Edwards et al., 2021). This
342   dataset contains 33010 molecules and their corresponding text descriptions. We follow the approach
343   of MolT5 (Edwards et al., 2022) for dataset splits and preprocessing.
344
345   Baselines We compare with the following methods.
346
347    • RNN: A 4-layer GRU recurrent neural network with bidirectional encoder trained on the ChEBI-
348      20 dataset.
349    • Transformer: A vanilla transformer model consisting of six encoder and decoder layers trained
350      on the ChEBI-20 dataset.
351    • MolT5: An encoder-decoder Transformer model initialized with a public checkpoint of T5, then
352      pretrained on the combined dataset of C4 and ZINC, finally finetuned on the ChEBI-20 dataset.
353    • Llama-3.1: An decoder-only Transformer model published by Meta. Llama-3.1-8B has 8B pa-
354      rameters, while Llama-3.1-70B has 70B parameters. We use the model in a zero-shot way, ask-
355      ing the model to generate the SMILES representation of molecules according to text descriptions
356      with a prompt.
357
358   Metrics We use the following metrics to evaluate our method and compare it with the baseline.
359
       • Fingerprint-based molecule similarity metrics: We measure the fingerprint Tanimoto similarity
360
         (FTS) between each generated molecule and the corresponding ground truth molecule. MACCS,
361      RDK, and Morgan represent the three different extraction methods for molecular fingerprints.
362      We consider these to be the primary metrics measuring the quality of generated graphs in our
363      experiments.
364    • FCD: We measure the Fréchet ChemNet Distance (FCD) (Preuer et al., 2018) between the gen-
365      erated molecules and ground truth molecules. It reflects the distance of chemical and biological
366      information between the two sets of molecules.
367    • Exact: We measure the proportion of generated molecules that are exact matches of their corre-
368      sponding ground truth molecules.
369    • Validity: We report the validity of the generated molecules as measured by RDKit sanitization.
370    • Diversity: We measure the diversity of generated molecules, defined as the average pairwise
371      differences between multiple molecules generated by the model guided by the same text descrip-
372      tion. The pairwise differences are calculated by subtracting their FTS from 1. In the experiments,
373      we select the first 100 text descriptions in the test set and generate 10 graphs for each description
374      to calculate the diversity metrics.

492   5     R ELATED WORK
493
494   5.1   D IFFUSION - BASED GRAPH GENERATION
495
496   Diffusion model has achieved great success in the field of computer vision. Recently, some re-
497   searchers have used diffusion models to solve graph generation task. For example, EDP-GNN (Niu
498
      et al., 2020) is the first work using Score Matching with Langevin Dynamics (SMLD) (Song & Er-
      mon, 2019) diffusion model to generate graphs, which learns the score function of the adjacency
499
      matrices distributions of the graphs. GDSS (Jo et al., 2022) proposes a graph generation method
500
      using continuous-time diffusion models (Song et al., 2021), which models the joint distribution of
501   the nodes and edges through stochastic differential equations (SDEs). DiGress (Vignac et al., 2023)
502   uses a diffusion model over discrete data space for graph generation, and additionally preserves the
503   marginal distribution of node and edge types and incorporates auxiliary graph-theoretic features.
504   These methods have demonstrated excellent performance on the task of graph generation.
505
      In order to generate graphs that match specific requirements, conditional graph generation has re-
506
      ceived attention in recent years. For example, DiGress (Vignac et al., 2023) uses classifier guidance
507   to perform graph generation guided by several graph-level properties, like the dipole moment and
508   highest occupied molecular orbit of molecular graphs. However, existing methods have not ex-
509   plored the task of text-guided graph generation, which is necessary for the popularization of graph
510   generation methods in various fields.
511
512
      5.2   L ARGE LANGUAGE MODELS FOR GRAPHS
513
514   LLMs have achieved good results on various natural language process tasks. Recently, many works
515   explored the application of LLMs in graph tasks. For example, Chen et al. (2023) uses LLMs to
516   predict node categories on text attribute graphs. Wang et al. (2023) proposes a benchmark framework
517   to evaluate the performance of LLMs with several graph algorithmic tasks, including topological
518   sort, maximum flows, etc. Zhang et al. (2023) evaluates the abilities of LLMs to handle spatial-
519   temporal information on dynamic graphs. Yao et al. (2024) designs a series of tasks to evaluate
520   the graph generation ability of LLMs. These and other studies have demonstrated the potential of
521   LLMs for processing graph tasks, showing the possibility to extract graph structure features from
522
      text description using LLMs.
523   Christofidellis et al. (2023) and Fang et al. (2024) are text-to-molecule methods that leverage lan-
524   guage models for generation, where molecules are represented in the SMILES format. Gong et al.
525   (2024) employs a text diffusion model to generate the SMILES representation of molecules. While
526   these methods focus on generating molecules from text representations, they do not extend to other
527   graph generation tasks. Zhu et al. (2024) utilizes a latent space diffusion model to generate latent
528
      features, which are then decoded into molecule graphs using a graph decoder. In contrast, this pa-
      per explores for the first time the use of graph-aligned LLMs to extract text features for guiding
529
      conditional graph diffusion models.
530
531
532   6     C ONCLUSION
533
534   This paper addresses the critical gap in text-to-graph generation by proposing the LLM-aligned
535   Graph Diffusion Model (LLM-GDM), which integrates large language models with graph diffusion
536   model to generate graphs from natural language instructions. By developing a self-supervised text-
537   graph alignment process and introducing a structure-aware cross-attention mechanism, our approach
538   enhances the model’s understanding of graph structures and properties and ensures that generated
539   graphs adhere to the specified relational semantics in the text. Extensive experimental results on
      synthetic and molecular datasets confirm the efficiency of our method.

703
704
      Our code is based on GDSS-Transformer (https://github.com/DongkiKim95/GDSS-Transformer/).
705   We modified the code to allow text-conditional graph generation. We initialized the parameters of
706   the base architecture with the checkpoint of GDSS-Transformers’s model trained on ZINC250k, and

708   of 2e-4 on ChEBI-20. For LLM finetuning, we use LoRA with rank set to 8 and alpha set to 64, and
709   trained for 6 epochs.
710
      All experiments are performed with a single NVIDIA A100-SXM4-40GB. Finetuning the LLM
711   takes about 5 hours, and training our diffusion model on ChEBI-20 takes about 1 day per run.
712   Evaluating the diffusion model on ChEBI-20 takes about 4 hours per run.

722
723    • Wheel: A graph formed by connecting a single node to all nodes of a cycle. An example of text

725    • Bipartite: A complete bipartite graph with the specified number of nodes. An example of text
726      description in the dataset is “A complete bipartite graph with 6 nodes and 8 nodes in each split”.
727    • K-regular graphs: A graph whose every node have the same number of neighbors. An example
728      of text description in the dataset is “A 2-regular graph with 12 nodes”.
729    • Component: A graph with the specified number of nodes and connected components. An ex-
730      ample of text description in the dataset is “A graph with 12 nodes and 3 connected components”.
731
       • Mix: A graph with the specified number of nodes and connected components, and every com-
732      ponent is a graph that satisfies a certain rule. An example of text description in the dataset is “A
733      graph with 12 nodes and 3 components. 2 components are trees. 1 component is a cycle.”.
