                                                   A BSTRACT
010
011            Quantization is one of the leading techniques to reduce the memory usage of ma-
012            chine learning models. It works by approximating the weights of a model by some
013            function with a smaller domain (e.g., replace 32-bit floats with 8-bit integers that
014            are coefficients in some function that maps back to 32-bit floats).
015            Although most quantization methods approximate weights with a linear or affine
016            function, the weights of current machine learning models often exhibit non-linear
017            behavior at the extremities. Moreover, some studies suggest that the extremities
018            are important for the end-to-end accuracy.
019            In this paper, we introduce PTNQ, a novel post-training quantization technique
020            that approximates weights by searching through a pool of non-linear functions.
021            We show that PTNQ provides significant advantages over affine functions, achiev-
022
               ing similar accuracy while requiring 2 to 4 fewer bits per coefficient.
023
024   1    I NTRODUCTION
025
026   Quantization is widely used today to reduce the cost of inference in machine learning models. While
027   models are usually trained with 16- or 32-bit floating-point numbers, they are typically deployed
028   with smaller data types. This has two important benefits: 1) reduces memory consumption so the
029   model fits in smaller devices, and 2) may reduce the computation cost (depending on the quantization
030   method).
031   The most straightforward quantization method consists
032   in approximating a weight (or parts of it) using a linear
033   or affine function. A post-training algorithm takes the
034   weights of a model after training, computes the param-
035   eters for a linear function so it interpolates the weights in
036   the best way, and then replaces the weights with coeffi-
037   cients for that function.

      2       PTNQ: P OST-T RAINING N ON -L INEAR Q UANTIZATION
074
      PTNQ is a technique for quantizing weights after the training process. Weights are then dequantized
075
      during inference, and the model is executed with its original data type (e.g,, 16-bit floats).
076
077   Figure 2 gives an overview of PTNQ. At high-level, PTNQ consists of two phases:
078
079            1. Enumeration of candidate quantization methods (quantization functions and parameter se-
080
                  lection methods).
081            2. Evaluation of each quantization method to select the best.
082
083
      The key motivation for this pipeline is that there is no one-size-fits-all quantization method. There-
      fore, PTNQ searches through a list of quantization methods and selects the best one. It is often a
084
      good tradeoff to have a slower quantization process if we get to fit a model in a smaller device. The
085
      cost of quantization is largely outweigh by the cost of training and inference.
086
087   In the next sections we detail the process of generating candidates and evaluating them.
088
089   2.1      G ENERATION OF Q UANTIZATION M ETHODS
090
091
      The first stage involves generating quantization methods. Each method has three components: a non-
      linear function, a method for initializing the function’s parameters, and a parameter optimization
092
      algorithm. It is crucial to have a diverse pool of each of the three components so we can find the
093
      best combination for each model.
094
095
      2.1.1      N ON -L INEAR F UNCTION G ENERATION
096
097   We first assume that there is a predefined list of primitive non-linear functions. These are combined
098   up to a user-defined depth k using basic arithmetic operations (e.g., addition and multiplication) and
099   by composing the functions with each other. The process is illustrated in Figure 3.
100   The set of generated functions is then filtered so that non-invertible functions are dropped. This
101   restriction has two motivations. The first is that we need to dequantize the weights during inference.
102   Having the inverse function guarantees that we can do it efficiently and accurately. The second is
103   that it simplifies the optimization of the function parameters (next section).
104
      In our implementation we use SymPy1 to generate the inverse functions. The final set of functions
105   and their inverses is then translated into PyTorch code, implementing both the quantization and
106   dequantization functions.

      2.1.2   I NITIALIZATION AND A PPROXIMATION OF Q UANTIZATION PARAMETERS
129
130   Each quantization function has a set of parameters that needs to be selected somehow. For example,
131   for the candidate function cos(a · x) · s, x is a value of a weight to be quantized, and a and s are
132   parameters. We use a different set of parameters per channel.
133
      For each function, we try different algorithms for initializing the parameters and then approximating
134
      them. We will show later that a good initialization strategy is critical to achieve good performance.
135
      We implemented three initialization strategies: 1) initialize all parameters as ones, 2) randomly draw
136   from a standard normal distribution with range [−1, 1], and 3) a technique we call space search. The
137   motivation behind space search stems from the observation that certain functions require initial
138   values outside the typical range of [−1, 1] supported by the other two methods.
139
      Space search approximates parameters through an iterative process. The algorithm begins by gener-
140
      ating random parameters with a large range of values as initial candidates. It evaluates these by cal-
141
      culating the MSE between the original weights and their quantized-dequantized versions, after which
142   the top 10% are kept. From this group, the algorithm computes the average and maximum values for
143   each channel to generate new parameters within the narrower range [−(max+avg), max+avg]. The
144   whole process is repeated 50 times and, in the final step, the best parameters from the last iteration
145   are chosen as the optimal solution.
146
      Each function has a special, and important, scale parameter s. This parameter scales the output
147
      of functions to ensure they range over the full domain of the chosen quantization bitwidth. The
148   initialization of s follows the standard procedure for affine functions, i.e., it is the ratio between the
149   input data range and the quantized output data range.
150
151
      Optionally, after initialization, we can further refine the parameters by using non-linear regression.
      We sort the weight values and try to to fit the candidate function better by adjusting the parameters.
152
153
154   2.1.3   T RAINING Q UANTIZATION PARAMETERS
155
156   Following initialization of the parameters, we then train them. The training proceess aims
157   to minimize the layer-wise mean squared error (MSE) between the original weights and their
158
      quantized-dequantized counterparts, potentially improving the end-to-end model performance post-
      quantization.
159
160   We support three different training strategies, each with a different learning rate (LR) scheduler.
161   The rationale behind focusing on LR schedulers stems from our initial setup, where the parameters
      are initially well-approximated. A poorly-tuned LR, especially if set too high, can destabilize the

178   maintain stability and prevent the model from deviating from its favorable starting point.
179   The three learning rate schedulers supported are:
180
181         1. Linear learning rate scheduler, which gradually decreases the learning rate over time.
182            Specifically, the LR of each parameter group is decayed by a small multiplicative factor
183            linearly until a predefined milestone epoch is reached. The idea behind this approach is
184
               to provide the training process with enough initial momentum to quickly approach an op-
               timal solution, while progressively reducing the learning rate to prevent overshooting or
185
               divergence.
186
187         2. Cosine annealing with warm restarts (Loshchilov & Hutter, 2017), in which the learning
188
               rate follows a cosine function, gradually decreasing to a minimum before restarting at a
               higher value. The warm restarts allow the model to escape local minima and explore the
189
               solution space more thoroughly. This technique is particularly useful in scenarios where
190
               the model might benefit from periodic boosts to the learning rate to refine the optimization
191            process.
192
            3. No learning rate scheduler, in which the learning rate is kept constant throughout the train-
193
               ing process, using the initial value from the setup.
194
195   The initial learning rate was tuned to avoid disrupting the starting loss value, especially when the
196   initial parameters were already a good approximation. The initial learning rate we use is 10E/2−1 ,
197   where E is exponent of the initial loss. This approach preserves good initializations by using a
198   smaller learning rate, while allowing poor approximations to start with a higher learning rate so they
199   converge faster.

201   2.2   E VALUATION AND S ELECTION OF Q UANTIZATION M ETHODS
202
203
      The last step consists in evaluating each of the quantization methods and selecting the best one,
      Each method is benchmarked against commonly used domain-specific metrics, such as accuracy,
204
      perplexity, or word error rate (WER), depending on the task.
205
206   We test quantization methods by starting with the highest bitwidth (e.g., 8 bits) and reducing it
207   one-by-one. To reduce the resources used by the process, PTNQ employs a pruning strategy: if
208   a quantization method fails at a higher bitwidth (e.g., achieving zero accuracy), it is not evaluated
209
      again at lower bitwidths, as it is unlikely to yield good results.

453   4   R ELATED W ORK

      The field of machine learning model quantization is vast, encompassing a wide array of techniques.
456   For a comprehensive overview, we direct readers to recent surveys (Gholami et al., 2022; Rokh et al.,
457   2023; Li et al., 2024) that provide a thorough exploration of the broader landscape. Here, we focus
458   specifically on non-linear post-training quantization, addressing limitations of traditional methods
459   in handling complex distributions.
460
      Earlier approaches, such as those proposed by Dettmers et al. (2022); Kim et al. (2024b), attempted
461
      to handle outliers by isolating them and processing them separately using sparse algebra. How-
462   ever, more recent research has shifted towards methods that aim to reduce the impact of outliers
463   without the need for separate processing. Techniques like QuaRot (Ashkboos et al., 2024) and Spin-
464   Quant (Liu et al., 2024) propose rotating matrices to effectively eliminate outliers. While these
465   methods show promise in maintaining model accuracy, they come at the cost of increased computa-
466   tional overhead due to the additional transformations required during inference.
467
      Lookup table (LUT) quantization has emerged as a flexible alternative to the uniform quantization
468   techniques, offering the ability to map intervals to arbitrary values. This approach, explored by Wang
469   et al. (2022) and further developed by FLUTE (Guo et al., 2024), allows for better preservation of
470   outlier information compared to uniform quantization. LUTs can potentially capture the distribution
471   of weights more accurately, leading to improved model performance. However, the learning process
472   for these lookup tables is challenging since LUTs are not differentiable, thus requiring sophisticated
473   optimization techniques. Additionally, models quantized with LUTs consume more memory when
474   compared to PTNQ due to the need to store the whole function mapping.
475   In this work we focus on the quantization of linear layers, which differs from the quantization of
476   non-linear layers such as Softmax, GELU, or LayerNorm, which has also been explored in recent
477   years. For example, Kim et al. (2024a) propose an approach that leverages layer-wise sensitivity
478   analysis through SQNR to determine the optimal quantization method for each of these layers from
479   a predefined pool of sub methods (Lin et al., 2022; Kim et al., 2021; Li & Gu, 2023).
480   Quantization with non-linear functions remains relatively unexplored, with logarithmic representa-
481   tions being the primary focus. Miyashita et al. (2016); Cai et al. (2018) proposed a base-two loga-
482   rithmic quantization method for low-resolution representation of weights and activations. Building
483   on this concept, Yan et al. (2024) introduced an integer-only scalar Power-of-Two (PoT) quantiza-
484   tion scheme that quantizes both weights and activations into PoT representations at low precision. In
485   this paper, we show that using a broad pool of functions provides increased accuracy and performs
      better in a wider range of scenarios.


                                                        9
486
      Jiang et al. (2024) explore efficient hardware implementations in FPGAs for logarithm-based quan-
487   tization functions. We believe similar techniques can be be used to optimize inference with the
488   majority of the functions used in our pool.

491   5   C ONCLUSION AND F UTURE W ORK
492
493   We presented PTNQ, a post-training quantization technique that searches for the best non-linear
494
      quantization function from a pool. Our results show that PTNQ achieves similar accuracy to affine
      functions while using up to 50% fewer bits.
495
496   For future work, we plan to explore the use of different functions per layer or even per channel
497   and assess the impact on accuracy and quantization time. Additionally, we will investigate other
498   non-linear functions, such as more complex polynomials and combinations of functions from our
499   test pool. We also aim to develop custom kernels to accelerate inference and study whether further
500
      hardware extensions could enhance inference performance.
