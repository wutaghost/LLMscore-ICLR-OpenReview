009                                               A BSTRACT
010
011            In many machine learning applications, the most relevant items for a particular
012            query should be efficiently extracted. The relevance function is usually an expensive
013            similarity model making the exhaustive search infeasible. A typical solution to this
014            problem is to train another model that separately embeds queries and items to a
015            vector space, where similarity is defined via the dot product or cosine similarity.
016            This allows one to search the most relevant objects through fast approximate nearest
017            neighbors search at the cost of some reduction in quality. To compensate for this
018
               reduction, the found candidates are re-ranked by the expensive similarity model.
               In this paper, we investigate an alternative approach that utilizes the relevances of
019
               the expensive model to make relevance-based embeddings (RBE). The idea is to
020
               describe each query (item) by its relevance for a set of support items (queries) and
021            use these new representations to obtain query (item) embeddings. We theoretically
022            prove that relevance-based embeddings are powerful enough to approximate any
023            complex similarity model (under mild conditions). An important ingredient of RBE
024            is the choice of support items. We investigate several strategies and demonstrate
025            that significant improvements can be obtained compared to random choice. Our
026            experiments on diverse datasets illustrate the power of relevance-based embeddings.

029   1   I NTRODUCTION
030
031   Finding the most relevant element (item) i to a query q among a large set of candidates I is a key task
032   for a wide range of machine learning problems, for example, information retrieval, recommender
033   systems, question-answering systems, or search engines. In such problems, the final score (relevance)
034   is often predicted by a pairwise function R : I × Q → R, where Q is a query space and R
035
      approximates some ground truth relevance such as click probability, time spent or something else.
      Depending on the task, the relevance function R can utilize query attributes (e.g., the text of the query
036
      or a set of numerical features describing the user, such as age, time spent on the service, etc.), item
037
      attributes, or attributes describing the query-item pair (e.g., statistics based on counts of each query
038   term in the document in information retrieval tasks).
039
040   The problem of relevance retrieval for a query q can be written as arg maxi∈I R(i, q). For practical
041
      applications, it is usually required to return not one but K best items (for directly displaying to the
      user or further re-ranking). Most recommender systems are characterized by a large size of the item
042
      space I (millions to hundreds of millions), so an exhaustive search is not feasible. This problem is
043
      often solved by training an auxiliary model R̃, called a Siamese, two-tower, or dual encoder (DE), in
044
      which late binding is used: R̃(i, q) = S(FI (i), FQ (q)), where FI : I → Rd , FQ : Q → Rd , and S
045
      is some lightweight similarity measure, usually dot product or cosine similarity.
046
047   While a lot of effort has been put into developing dual-encoder models, the cross-encoder (CE) ones
048   are generally more powerful (Wu et al., 2019; Yadav et al., 2022). Moreover, as mentioned above,
049   in practice it is typical to also have features that describe a query-item pair: e.g., counts of query
050
      terms in the document (information retrieval), information about previous user-item interactions
      (recommender systems), and so on. Such features cannot be used by dual encoders.
051
052   In a recent paper, Yadav et al. (2022) suggested an alternative approach: to approximate the relevance
053   of a given query to all the items using the relevance of this query to a fixed set of randomly chosen
      support items. In more detail, the authors apply the matrix factorization to the query-item relevance


                                                         1

054
      matrix to represent it as a product of its submatrix containing only a few columns (relevances to
055   support items) and some other, explicitly computable.
056
057   Motivated by the above-mentioned work, we propose and analyze the concept of relevance-based
058
      embeddings (RBE). The main idea is to describe users by their relevance to some pre-selected support
      items and describe items by their relevance to some support users. Then, such representations can be
059
      used in various ways: as in Yadav et al. (2022), they can be multiplied by a certain matrix to obtain
060
      relevance approximations, or they can be passed into a neural network to potentially obtain better
061   approximations (trained for a desired loss), or they can be additionally combined with the original
062   node features to get even more powerful embeddings.
063
064
      We theoretically prove the power of relevance-based embeddings. Namely, we show that (under mild
      conditions) they are informative enough to approximate any continuous similarity function. In partic-
065
      ular, when the similarity function utilizes pairwise features, dual encoders based on the individual
066
      user and item features inevitably lose this information; however, relevance-based representations
067   contain this information and thus can approximate the desired function.
068
069   From a practical perspective, an important aspect of our approach is how to properly choose support
070
      elements. Previous studies (Morozov & Babenko, 2019; Yadav et al., 2022) sampled them uniformly
      at random, while we show that there is significant room for improvement. We investigate different
071
      options: from simple heuristics (e.g., popular or diverse elements) to methods directly optimizing
072
      the relevance approximation quality. Surprisingly, even very simple strategies like clustering the
073   elements and choosing the cluster centers as support items already give significant improvements that
074   can be further improved by more advanced and theoretically justified strategies.
075
076
      To evaluate the performance of RBE, we conduct experiments on textual and recommendation
      datasets. We compare our approach with dual encoders and with Yadav et al. (2022) and get an
077
      average improvement of 33% over this baseline for various datasets (from 8% to 69%, see Table 2).

081   2   R ELATED WORK
082
083
084   In this section, we discuss research areas and representative papers related to our study.
085   Relevance retrieval problem is widespread in the context of building information retrieval sys-
086   tems (Kowalski, 2007), such as text search engines (Huang et al., 2013), image search (Gordo
087   et al., 2016), entertainment recommender systems (Covington et al., 2016), question answering sys-
088   tems (Karpukhin et al., 2020), e-commerce systems (Yu et al., 2018), and other practical applications.
089
      Usually, such problems are solved by learning query and item embeddings into a certain space and
090   then searching for approximate nearest elements in this space, followed by rearrangement using a
091   heavier ranker. In particular, Covington et al. (2016); Huang et al. (2013) explicitly use this approach,
092   offering two-tower models (a.k.a. dual encoders). Note that there are simple alternatives to dual
093   encoders that use, e.g., BM25 scores applicable to texts (Logeswaran et al., 2019; Zhang & Stratos,
094   2021) or other cheaper or more expensive alternatives (Humeau et al., 2019; Luan et al., 2021).
095   However, there is usually trading-off complexity for quality.
096   It is also worth mentioning the works trying to facilitate the training of the dual encoder through
097   distilling a heavier ranker model (Wu et al., 2019; Hofstätter et al., 2020; Lu et al., 2020; Qu et al.,
098   2020; Liu et al., 2021). Although these works aim at simplifying the learning of light ranking using
099   the heavy one, they differ from our approach in two aspects. First, distillation means that there are
100   still two heavy (comparable in orders of magnitude of trainable parameters) models. Second, this
101   approach requires two different complex model architectures that can be a significant disadvantage
102   for, e.g., recommender systems with a wide variety of image/textual/statistical/sequential features.
103   As for the nearest neighbors search in a common query-item space, a wide variety of algorithms
104   exist, including locality-sensitive hashing (LSH) (Indyk & Motwani, 1998; Andoni & Indyk, 2008),
105   partition trees (Bentley, 1975; Dasgupta & Freund, 2008; Dasgupta & Sinha, 2013), and similarity
106   graphs (Navarro, 2002). LSH-based and tree-based methods provide strong theoretical guarantees,
107   however, it has been shown that graph-based methods usually perform better (Malkov & Yashunin,
      2018; Aumüller et al., 2020), which explains their widespread use in practical applications.


                                                         2
      Under review as a conference paper at ICLR 2025


108
      Another research direction is methods that combine nearest neighbors search with heavy ranker
109   calls (Morozov & Babenko, 2019; Chen et al., 2022) instead of separately embedding queries and
110   items in a common space where the search for the nearest items can be efficiently performed. Such
111   methods show better quality in comparison with separate embeddings, however, their practical
112   application may be limited due to a significant change in the structure of the search index. In
113   particular, in practice, microservices with neural networks and microservices with document indexes
114   are different services, which allows for increasing GPU utilization on the one hand and using
115   specialized (including sharded) solutions with a large amount of memory on the other. Therefore, in
116   this paper, we focus on the basic scenario with a separate investment in space and a separate search
117
      for the nearest elements in it.
118   A recent paper by Yadav et al. (2022) is the most relevant for our research. The idea is to apply
119   the matrix factorization to the query-item relevance matrix in order to represent it as a product
120   of its submatrix containing only a few columns (relevances for random support items) and some
121   other, explicitly computable. Despite the simplicity of the idea and implementation, the authors
122   have shown in detail the superiority of their algorithm over more complex approaches, such as dual
123
      encoders. Our work is motivated by this study: we show that the approach of Yadav et al. (2022) has
      theoretical guarantees and suggest several improvements that significantly boost the performance.
124
      As an enhancement of the original approach, in another paper, Yadav et al. (2023) proposed the
125
      idea of selecting support items per each query independently, but the time complexity of each query
126   processing becomes linear in the number of elements, which makes the approach infeasible in most
127   practical applications. On the contrary, our support items selection is performed in the pre-processing
128   stage and does not increase the query time.
129
      Finally, the most recent article (Yadav et al., 2024) proposes an alternative solution to the problem.
130
      Their AXN algorithm dynamically learns the difference between the DE and CE (or other query-
131
      document embeddings) predictions for each query independently (at the query time). Learning the
132   difference is carried out through iteratively choosing a set of anchor (supporting) elements, calculating
133   CE scores for them, and learning linear regression with embeddings of these elements as features
134   and CE scores as targets. Some disadvantages of this method are: 1) it requires previously trained
135   embeddings of queries and documents; 2) it has increased query processing time due to the need for
136   iterative refining of all item relevances for each query. For completeness of our study, we use the
137   AXN algorithm as one of our baselines.

140   3     R ELEVANCE - BASED EMBEDDINGS
141
142   In general, the information (attributes) used to calculate the ground-truth item-to-query relevances
143   R(i, q) can be divided into three types: depending only on the query q, only on the item i, and on
144   both of them. The key problem when constructing separate embeddings of items and queries in
145   the common space (that can be used for searching for the nearest elements) is the inability to use
146   information that depends on both query and item, which lowers the quality of relevance search.
147
      To address the above-mentioned issue, we introduce relevance-based representations that describe
148   each query by its relevance to a pre-selected set of items and, vice-versa, each item by its relevance to
149   a pre-selected set of queries. We prove that, under certain conditions, any relevance function can be
150   well estimated using only such individual vectors.
151
152
153   3.1   P RELIMINARIES
154
155   Let Q and I be compact topological spaces of queries and items, respectively. Assume that we are
156   given a relevance function R : I × Q → R. In practice, R is our relevance model which may be
157   computationally expensive and rely on pairwise features.
158   Let SI ⊂ I and SQ ⊂ Q be some finite ordered sets of support items and support queries:
159   SI = {i1 , . . . , im }, SQ = {q1 , . . . , qn }. Let R(i, SQ ) be a relevance vector of the item i w.r.t. the
160   set of support queries SQ : R(i, SQ ) = (R(i, q1 ), . . . , R(i, qn )). Similarly, R(SI , q) is a relevance
161   vector of the query q w.r.t. the set of support items SI : R(SI , q) = (R(i1 , q), . . . , R(im , q)). By
      R(SI , SQ ) we denote a relevance matrix composed in a similar way.

      3.2   CUR APPROXIMATION
163
164   Relevance vectors can be utilized in different ways and one of the possible approaches is to use
165   the CUR decomposition (this approach was used by Yadav et al. (2022)). Using our notation, the
166   relevance for a query q is approximated as:
167
                             R̃(I, q) := ⟨ R(I, SQ ) × pinv(R(SI , SQ )), R(SI , q) ⟩,                      (1)
168
169   where pinv(X) is the pseudo-inverse matrix of X. As support queries, Yadav et al. (2022) take the
170   set of train queries: SQ = Qtrain ⊂ Q.
171   Regarding the computational complexity, we note that the CUR approximation requires computing
172   the matrix R(I, Qtrain ) which takes O(M · |SQ |) = (M · |Qtrain |) CE calls, where M is the total
173   number of items in a database, which can be infeasible for large databases.
174
      Our first theoretical result provides guarantees for the CUR approximation (1). Namely, we show
175
      that its regularized version approximates the true relevance arbitrarily precisely in L2 . Formally,
176
      let pinvλ (A) = (AT A + λE)−1 AT with E being the identity matrix. Then, the regularized CUR
177   approximation CURλ is defined by (1) with pinvλ instead of pinv. For CURλ , we prove the following
178   result (see Appendix A.3 for the proof).
179
      Theorem 3.1. Suppose that I and Q are equipped with the structure of a measure space and the
180
      integral of R4 (i, q) over I ×Q is finite. Then, the CURλ approximation of R can be chosen arbitrarily
181   close to the true R in L2 (I × Q) provided enough uniformly sampled support items and queries and
182   sufficiently small λ.
183
184   Our result gives theoretical support for the good performance of the CUR decomposition demonstrated
185   in Yadav et al. (2022). In the next section, we show that stronger theoretical guarantees can be provided
186   if we allow transforming query and item vectors into more powerful embeddings.

188   3.3   R ELEVANCE - BASED EMBEDDINGS
189
      In this section, we propose extending the CUR-based approximation by allowing transformations of
190
      relevance vectors, e.g., with a neural architecture. With such embeddings, we prove a stronger result:
191
      that any continuous relevance function can be uniformly approximated.
192
193   We say that a function on I is a relevance-based embedding if it has a representation of the form
194   eI (i) = fI (R(i, SQ ), θI ) where SQ is a set of support queries and fI is some ML-architecture with
195   parameters θI which parametrizes a mapping Rn → Rd . Analogously, eQ (q) = fQ (R(SI , q), θQ ) is
196
      a relevance-based embedding of the query q.
197   The following theorem holds (the proof can be found in Appendix A.1 and the guarantees for the
198   RBE approach on a sphere are discussed in Appendix A.2).
199   Theorem 3.2. Let I and Q be compact topological spaces, and R : I × Q → R be a continuous
200   function. Then, R can be uniformly approximated up to an arbitrarily small absolute error by a
201   function R̃(i, q):

229   arbitrary (and, in particular, can rely on pairwise features only available for query-item pairs), it can
230   be well approximated by relevance-based embeddings of individual users and items. This is a clear
231   advantage of RBE over conventional dual-encoder models.

233   3.4   S ELECTING SUPPORT ITEMS
234
235   Let us revise the statement of Theorem 3.2. The theorem states that there exist such sets SQ , SI that
236   the relevance function R could be effectively approximated by separated embedders fI , fQ . In the
237   related works (Morozov & Babenko, 2019; Yadav et al., 2022), this selection is random, implicitly
238
      assuming that the elements are in some sense equivalent. However, for example, when building a
      recommender service, the popularity of different objects has a strongly skewed distribution, which is
239
      why more information is known about a small set of highly popular items than about a large set of
240
      unpopular ones. Thus, it is natural to assume that the choice of support items may have a significant
241   effect on performance. We investigate this direction and compare several approaches from simple
242   heuristics to more complex ones optimizing the approximation quality.
243
244   Simple heuristics Our heuristic strategies include:
245
246          • Random Support items are chosen uniformly at random (Morozov & Babenko, 2019;
247            Yadav et al., 2022). For better reproducibility, we also present the results of using the first
248
               |SI | items as support ones, assuming that the order of queries is pseudorandom.
249          • Popular As mentioned above, a recommender service usually has a small set of very
250            popular elements that many users interact with. As a result, a lot of information can be
251
               collected from these interactions thus making the popular elements more informative. Since
               it is not always possible to get popularity explicitly, we consider the following surrogate:
252
               choose the objects with the highest average relevance for the training set.
253
254
             • Clusters centers When it comes to the allocation of a representative subset of vectors, it is
               reasonable to consider the allocation of clusters. We consider various clustering algorithms
255
               and select the cluster centers as support elements. The number of clusters is set to the
256
               number of required support elements.
257
             • Most diverse This strategy is a greedy algorithm maximizing the minimum distance
258
               between the support elements. We first choose the element furthest from the center (by
259
               Euclidean distance) and then, at each step, an element is selected whose minimum distance
260            to the current support elements is maximal.
261
262   The approaches that require item representations can be applied to their relevance vectors.
263
264   l2 -greedy approach Let us now discuss a more theoretically justified approach that we call l2 -
265   greedy. It aims at selecting the key elements that allow for a better approximation of the relevance
266   matrix R(i, q). In this strategy, we greedily select items so that the MSE error of the CUR approxi-
267
      mation (Mahoney & Drineas, 2009) is minimized for the train queries.1

292   3.5   A DDITIONAL PRACTICAL CONSIDERATIONS
293
294   Dynamic set of items We note that relevance-based embeddings can naturally handle scenarios
295   where the set of items frequently changes. The embeddings fI (R(i, SQ ), θI ) can be easily calculated
296   for the new items without the need of re-training the embedding model fI (similarly to feature-based
297
      dual encoders). On the other hand, if the set of items I is finite or changes not so frequently (e.g.,
      the set of movies currently available in a recommender service), the transformation fI (R(i, SQ ), θI )
298
      can be replaced with trainable embeddings θI (i) ∈ Rd . In contrast to items, the query set Q cannot
299
      be assumed to be finite: queries can be represented by texts of unlimited length or characterized by
300
      real-valued features.
301
302
303   Improving approximation quality As discussed above, it is natural to assume the heavy ranker R
304   to be the most expensive part in terms of computational complexity. Thus, during the calculation
305   of R̃, we are mainly limited by the sizes of the support sets SQ and SI . In contrast, calculating fQ ,
306
      fI , or the dot product between them is assumed to be significantly cheaper. Thus, fQ and fI may
      embed the relevance vectors in a higher-dimensional space. In particular, this may help to eliminate
307
      the disadvantages of the dot product, in comparison with other (Shevkunov & Prokhorenkova, 2021)
308
      ways of measuring the distances or similarities between objects.
309
310   While relevance-based embeddings have theoretical guarantees, they hold in the limit, when the
311   sets of support queries and items are sufficiently large. In practice, to improve the approximation
312
      performance and reduce the number of CE calls per query, the mapping fQ (or fI ) could be extended
      by enriching it with the features of the original query (or item).
313
314
315   Scalability The cost of the pre-processing and the ways to scale it are discussed in Section 3. At the
316   inference, we need to compute the query representation, which requires m relevance computations.
317   Then, the inference is similar to dual encoders: the item representations are pre-calculated and placed
318   in the Approximate Nearest Neighbours index like HNSW (Malkov & Yashunin, 2018), which takes
319
      the embedding of the query as input. These m additional computations are taken into account in our
      experiments: when comparing with dual encoders, we reduce the final re-ranking budget by m.
320
321
322   Other applications Although the goal of RBEs is to approximate the relevance function, relevance
323   vectors can also be considered good general representations. Thus, their application is not limited to
      relevance predictions, as shown below in Appendix D.2.

415   4.3      RBE IMPLEMENTATION
416
417   Following Theorem 3.2, we train lightweight neural networks fI and fQ , which are significantly
418
      faster than the heavy ranker R, and independently transform the relevance vectors R(i, SQ ), R(SI , q)
      into embeddings. The training is performed by the Adam algorithm on sampled batches with a
419
      listwise loss function (see Appendix C), similar to the training of various DEs.
420
421   As follows from Section 3.2, the CUR decomposition gives a reasonably good approximation of the
422   relevance function. Hence, we split the RBE representation into the CUR representation and the
423   trainable prediction of its error. In the experiments, such decomposition improves the convergence
424
      and training stability. Technical implementation details are placed in Appendix C.

      6    C ONCLUSION
515
516
      In this paper, we present the concept of relevance-based embeddings. We justify our approach
      theoretically and show its practical effectiveness on various datasets. We demonstrate that RBE
517
      allows one to obtain better quality compared with existing approaches. An important contribution of
518
      our work is the study of different strategies for choosing the support elements for RBE: we show that
519   a proper choice of the support elements allows one to significantly boost performance.
520
521   Promising directions for future research include a deeper investigation of support element selection
522
      strategies as well as applying the proposed RBE to other algorithms, e.g., based on using a heavy
      ranker during the nearest neighbor search (Morozov & Babenko, 2019) or adaptive nearest neighbor
523
      search (Yadav et al., 2024). Regarding the latter approach, we note that AXN can be naturally
524
      combined with any query-item representations, e.g., RBE. The obtained AXNRBE model has to
525   perform at least as well as RBE due to the presence of the regularization that balances between the
526   query-item representation and the proposed modification.

      A.1    P ROOF OF T HEOREM 3.2
651
652   Queries as functions on items and vice versa Each query q defines a function rq on items:
653   rq (i) = R(i, q). Let us call two queries q and q ′ R-equivalent if rq = rq′ and write q ∼R q ′ to denote
654   this relation. R-equivalent queries are interchangeable when it comes to measuring their relevance to
655   any item. Let QR be the set of R-equivalence classes. QR may be considered as an image of Q in C(I)
656   under the mapping RQ which maps query q to rq . This point of view suggests a natural metric dQR on
657   QR induced by the uniform norm on C(I): dQR (q, q ′ ) = ∥rq − rq′ ∥ = supi∈I |R(i, q) − R(i, q ′ )|.
658   Now let us note that the map RQ : Q → C(I) is continuous since R is continuous and I is compact.
659   Therefore, QR is compact as an image of a compact space Q under a continuous mapping. And
660   there is an (injective) embedding of C(QR ) to C(Q) under which the function f ∈ C(QR ) goes to
661   f ◦ RQ ∈ C(Q). Simply speaking, a continuous function on the equivalence classes of queries is
662   also a continuous function on the queries themselves.
663   Analogously, we define:

665                    RI : I → C(Q),        RI (i) = ri ,        ri (q) = R(i, q),    RI (I) = IR ,

667                             dIR (i, i′ ) = ∥ri − ri′ ∥ = sup |R(i, q) − R(i′ , q)|.
                                                              q∈Q

688   Claim 1. ri ∈ C(QR ) and, similarly, rq ∈ C(IR ).

690   Proof. We note that ∥ri (q) − ri (q ′ )∥ = ∥rq (i) − rq′ (i)∥ ≤ ∥rq − rq′ ∥ = dQR (q, q ′ ). So, ri (q) −
691   ri (q ′ ) = 0 if rq = rq′ and the value ri (q) does not change if a query is replaced with an equivalent
692   one. It means that ri is a correctly defined function on the classes of equivalent queries, i.e., on
693   QR . And finally the same inequality ∥ri (q) − ri (q ′ )∥ ≤ dQR (q, q ′ ) implies that the function ri is
694   1-Lipschitz with respect to the metric dQR .

696   Claim 2. R ∈ C(IR , QR ).
697
698   Proof. We have |R(i, q) − R(i, q ′ )| = |rq (i) − rq′ (i)| ≤ ∥rq − rq′ ∥ = dQR (q, q ′ ) and |R(i, q) −
699   R(i′ , q)| ≤ dIR (i, i′ ). It follows that the value R(i, q) does not change after replacement of a
700   query-item pair (i, q) with some equivalent pair (i′ , q ′ ). So, R can be considered as a function on
701   IR × QR . And by the same inequality R is 1-Lipschitz with respect to the metric dR ((i, q), (i′ , q ′ )) =
      dIR (i, i′ ) + dQR (q, q ′ ) and hence is continuous.
