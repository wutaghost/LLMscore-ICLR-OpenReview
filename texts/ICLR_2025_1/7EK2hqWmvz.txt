009                                                A BSTRACT
010
011
               Deploying large language model inference remains challenging due to their high
012
               computational overhead. Early exit optimizes model inference by adaptively re-
013
               ducing the number of inference layers. Current methods typically train internal
014            classifiers to determine whether to exit at intermediate layers. However, such
015            classifier-based early exit frameworks require significant effort to train the clas-
016            sifiers while can only achieve comparable performance at best. To address these
017            limitations, this paper proposes RAEE, a robust Retrieval-Augmented Early Exit
018            framework for efficient inference. This paper first demonstrates that the early exit
019            problem can be effectively modeled as a distribution prediction problem, in which
020            the distribution is approximated through the exit information of similar data. Subse-
021            quently, it outlines the methodology for collecting exit information to construct the
022
               retrieval database. Finally, leveraging the pre-constructed retrieval database, RAEE
               utilizes the exit information from retrieved similar data to guide the backbone
023
               model’s exit at the layer. Experimental results demonstrate that RAEE significantly
024
               accelerates inference while achieving robust zero-shot performance across eight
025            downstream tasks.

029   1    I NTRODUCTION
030
031   Large language models have been widely used in various application scenarios due to their excellent
032   performance (Thoppilan et al., 2022; Touvron et al., 2023; Scao et al., 2022). However, improving the
033   efficiency of model inference remains a critical and challenging task due to the high computational
034   overhead involved in the process (Dao et al., 2022; Liu et al., 2023). As an advanced technique,
035   model pruning provides a new direction for efficient inference (Valicenti et al., 2023; Ma et al., 2023).
036   It selectively removes less important weights or connections from the neural network to reduce
037   complexity and computational requirements without significantly degrading performance. One
038   popular model pruning method is the early exit technique, which speeds up inference by adaptively
039
      reducing the number of inference layers.
040   Most early exit frameworks (Liu et al., 2020; Zhu, 2021; Xin et al., 2020; Fan et al., 2024) leverage
041   classifiers to predict the exit layer and stop the inference at the predicted exit layer. Those early exit
042   frameworks can be categorized into three branches according to the training strategy. The first one is
043   training-based early exit frameworks (Zhu, 2021; Zhou et al., 2020; Zhu et al., 2023; Bae et al.,
044   2023; Schuster et al., 2022), which requires training the classifiers along with the backbone models
045
      and updating all parameters of backbone models as well as classifiers. These methods introduce
      significant training overheads, particularly when applied to large language models. The second one
046
      is semi-training-based early exit frameworks (Fan et al., 2024). The backbone models in those
047
      works would not be updated, and only classifiers would be fitted to predict the exit layer. These
048   methods may not capture the patterns between inputs and exit layers well, requiring significant human
049   effort in feature engineering. The last one is training-free early exit frameworks (Sun et al., 2022),
050   which requires no parameter updates and uses heuristics to determine the exit layer. These methods
051   lack the generalization ability to predict the exit layer and often fail to achieve good performance.
052   Moreover, most early exit frameworks sacrifice the model performance for acceleration (Fan et al.,
053   2024; Sun et al., 2022; Schuster et al., 2022; Bae et al., 2023). This paper mainly focuses on improving
      training-free early exit methods to avoid introducing too many training overheads.

079   To address the above limitations, this paper first demonstrates that the exit layer can be predicted
080   from an exit distribution, which can be approximated by similar data’s exit information. Based on
081   the observations, this paper proposes RAEE, a robust retrieval-augmented early exit framework for
082   efficient inference. First, RAEE collects exit information from data. Next, RAEE builds the indexing
083   and database to retrieve exit information from similar data. Finally, RAEE predicts the exit layer
084   based on the top-k nearest neighbors’ exit information during the inference and stops the model
085   inference at the predicted exit layer.
086   We conduct comprehensive experiments to evaluate the proposed RAEE and various comparison
087   methods on eight downstream tasks. Experimental results demonstrate that RAEE can accelerate the
088   model inference while achieving robust model performance. Codes are available at 1 .

      The main contributions of this paper are:
090
091                • We model the early exit problem as a distribution prediction problem and demonstrate that
092                  the exit information of similar data can approximate the exit distribution;
093                • We propose a robust retrieval-augmented early exit framework, RAEE, which leverages an
094
                     external database to guide the early exit;
                   • Experimental results show that the proposed RAEE can accelerate the model inference and
095
                     achieve robust performance.

      Motivations of Retrieval-Augmented Early Exit. This paper aims to leverage retrieval-based
132
      techniques to guide the early exit, which behaves like training-free-based early exit frameworks
133   requiring no parameters to update. Existing retrieval databases use clustering and product quantization
134   to build the retrieval indexing over millions of embeddings, which can efficiently approximate nearest-
135   neighbor searching. The indexing building process is not resource-constrained and can run on either
136   GPUs or CPUs. Besides, since the retrieval database stores the original data, every retrieval in the
137   retrieval-augmented early exit can be regarded as a generalization over several semantically similar
138   data. In contrast, those original data may be used to train classifiers or build hashing functions in
139   other early exit frameworks. The retrieval-augmented techniques also exhibit strong adaptability to
140   new data. The index can retrieve up-to-date information by adding new data to the retrieval database.
141   To further demonstrate the efficacy of retrieval-augmented early exit, this paper conducts some
142   analysis experiments in Figure 2. It shows the distribution of probability values for predicting
143   the correct answers across all model layers during inference, including two SST-2 test samples
144   (represented by the blue line with dots) and their respective top-8 nearest neighbors (depicted as blue
145   box plots at each layer) retrieved from the SST-2 training dataset. The probability values in Figure 2
146   are calculated as the normalized logits of the answer label tokens. Note that the reason why we can
147
      collect the probability values of SST-2 data for predicting the correct answers across all model layers
      is because the SST-2 task has labels that allow us to extract the relevant probability data for predicting
148
      the label token.
149
150   The experimental results in Figure 2 (a) and (b) indicate that the layer-wise probability trends for
151   correctly predicting the answers during inference exhibit a similar pattern between the test samples
152   and their respective nearest neighbors. At the same time, different inputs exhibit diverse behaviors
153
      in these layer-wise predictive probabilities. Notably, as observed in Figure 2 (a), the probability
      values for predicting the correct answers of the SST-2 test sample and its top-8 nearest neighbors
154
      at the 24th layer are lower than those at layers 20 to 23. In such cases, leveraging the probability
155
      distribution of the top-8 nearest neighbors to guide the determination of the test sample’s exit layer
156   can allow it to exit at a layer with higher probability values for correct predictions. Thus this
157   approach not only enhances inference speed by reducing the number of inference layers but also
158   improves the accuracy of predictions. Consequently, the experimental results demonstrate that the
159   probabilities of top-k nearest neighbors from the pre-collected database can approximate the
160   layer-wise exit probabilities of new input data. Additionally, different inputs exhibit distinct
161   probability distributions across the model’s all exit layers. These observations motivate us to propose
      a retrieval-augmented early exit framework.

234   3.2   T HE R ETRIEVAL -AUGMENTED E ARLY E XIT F RAMEWORK
235
      In this section, this paper then presents a retrieval-augmented early exit framework named RAEE to
236
      optimize the model inference. RAEE regards the exit layer as a random variable z, taking values in the
237   set of {1, . . . , m}, where m is the total number of layers in the backbone model M. The probability
238   mass function P (z = l) represents the probability of the case that the backbone model exits at the
239   layer l. With the gold label, we can observe that the random variable z follows an unknown discrete
240   distribution F . Then, this paper shows how to leverage the retrieval database to approximate the
241   distribution F .
242

243   vi has mi possible exit layers. Naturally, we can approximate the distribution F by estimating the
244   probability function P (z = l),

416   4.3   R EASONS FOR S IGNIFICANT P ERFORMANCE I MPROVEMENT
417
418
      The main results demonstrate that the proposed RAEE can significantly outperform backbone models,
419
      which is not an intuitive result compared to previous early exit methods. The reasons for this lie in
420   that the collected exit information guides the RAEE as an error corrector. This means that RAEE
421   can learn from the exit information of examples that are correctly predicted by intermediate layers,
422   but backbone models without early exit fail to predict.
423
      To better support the above claims, we also conducted an analysis experiment using the retrieval
424
      database that only contains the exit information of data that Llama-3-8B correctly predicted without
425
      early exit. As shown in Table 2, RAEE w/o refers to the one built on only correctly predicted
426   examples. As expected, RAEE w/o achieves comparable performance to baselines but accelerates
427   the inference process. This is because the test data that is correctly predicted by RAEE w/o can
428   also be correctly predicted by backbone models. However, due to a lack of exit information on
429   examples where backbone models fail to predict, RAEE w/o also fails to predict on the test data
430   where backbone models fail. Therefore, when providing the exit information based on examples
431   where backbone models fail to predict but intermediate outputs succeed in predicting, RAEE can
      make correct predictions and exit earlier.

461   Impact of Top-k: Table 3 illustrates the impact of varying the number of retrievals on the distribution
462   approximation. As k increases, the proposed RAEE with the backbone model RoBERTa-Large

      improves the overall performance from 60.73 to 63.41. This suggests that more retrieved exiting

467   approximate the exit distribution, thereby saving time in the retrieval process.

      performance of RAEE with the backbone model RoBERTa-Large increases significantly from 61.16

473   RAEE, thus approximating the exit distribution more accurately.

      5     R ELATED W ORK
487
488
489
      5.1   E ARLY E XIT F RAMEWORK
490
      Model inference with early exit has been a popular pruning method to reduce both computation and
491
      memory overhead on text classification or generation tasks. Most current works (Bae et al., 2023;
492   Kong et al., 2022; Ji et al., 2023; Wolczyk et al., 2021; Hooper et al., 2023) introduce classifiers in
493   each layer to determine whether the inference should continue. (Xin et al., 2020; Liu et al., 2020;
494   Zhou et al., 2020; He et al., 2021) train the classifier by minimizing the differences between each
495   layer’s outputs and final outputs, then perform early-exit-based inference according to a threshold.
496   (Liao et al., 2021) incorporates the past and future information to predict the early exit layer. Instead
497   of training a neural network as classifiers, (Fan et al., 2024) only fit the machine learning classifiers on
498   the extracted features for the early exit. (Zhu, 2021; Zhu et al., 2021; 2023; Zhang et al., 2023a) focus
499   on designing novel loss functions for training a more robust classifier. (Li et al., 2021) incorporates
500
      sentence-level features as well as token-level features to predict the early exit. Different from
      those works, our method does not require training the classifier. (Sun et al., 2022) proposes a
501
      hash-based early exit method that uses the hashing functions to map tokens to exit layers. (Bajpai &
502
      Hanawal, 2024) introduces an online learning algorithm for early exits in BERT models, dynamically
503   determining exit points based on confidence thresholds. (Regol et al., 2023) proposes a jointly-learned
504   framework for early exiting and inference in dynamic neural networks, integrating gating mechanisms
505   and intermediate inference modules. (Balagansky & Gavrilov, 2022) introduces a deterministic
506   Q-exit criterion and revising the model architecture. Our method predicts exit layers using pre-built
507   databases, resulting in better generalization. Other works (Jazbec et al., 2023; Li et al., 2023; Huang
508   et al., 2018) focus on designing early exit frameworks for image classification tasks. They are
509   optimizations in the different domains compared to our works.
510
511   5.2   R ETRIEVAL - BASED AUGMENTATIONS
512
513   Retrieval-based augmentations (Li et al., 2022; Wang et al., 2023; Xiong et al., 2023; Cui et al., 2023;
514   Wu et al., 2024a;b) have been widely used in various natural language processing (NLP) tasks and
515   achieved remarkable performance. Current works mostly leverage external knowledge databases to
516   augment generator models on various text-generation tasks, such as language modeling (Khandelwal
517   et al., 2020; Lewis et al., 2020; Borgeaud et al., 2022), question-answering (Guu et al., 2020;
518   Izacard & Grave, 2021), machine translation (Khandelwal et al., 2021; Wang et al., 2022), dialogue
519   system (Cheng et al., 2023). Those works focus on improving the model’s generation quality, while
520
      our work aims to use the retrieval knowledge to accelerate the model’s inference. Additionally, other
      work like (Zhang et al., 2023b) aims to accelerate the inference process by retrieving precomputed
521
      trajectories from a knowledge base, which is specifically designed for diffusion models that differ
522
      from our target models. These works are out of the scope of the research problems in this paper.
523
524
525   6     L IMITATIONS
526
527
      Although the proposed RAEE can improve both model performance and model efficiency, several
528   limitations warrant discussion. The effectiveness of RAEE depends on the pre-built in-domain
529   retrieval databases, which can well approximate the exit distribution for predictions. The framework
530   is primarily designed for in-domain training and testing scenarios, which represent the mainstream
531   tasks. Consequently, the out-of-domain performance of RAEE may be constrained; however, this
532   aspect is not the primary focus of this paper and will be the subject of future research.
533
534
535   7     C ONCLUSION
536
537   This paper models the early exit problem as a distribution prediction problem and observes that similar
538   data’s exit information can be used to approximate the distribution. Based on the observations, this
539   paper proposes a retrieval-augmented early exit framework named RAEE. Experimental results show
      that RAEE can accelerate the model inference while significantly improving the model performance.

      C    I MPLEMENTATION D ETAILS
898
899
900   This section lists the implementation details.
901
902          • For DeeBERT(Xin et al., 2020), we use RoBERTa-Large as its backbone model. Since
903            DeeBERT(Xin et al., 2020) is a classical entropy-thresholding-based early-exit method, it
904            requires first fine-tuning the backbone model on the downstream task and then updating
905            all but the last off-ramp, for a fair comparison, we only update the off-ramp in DeeBERT
906
               on each downstream task. We also use RoBERTa-large as the backbone model and train
               all off-ramps for 50 epochs (much larger than the default setting of 10 epochs). Other
907
               experimental settings for DeeBERT(Xin et al., 2020) remain as default.
908
909
             • For CALM (Schuster et al., 2022), we use T5-Large (Raffel et al., 2020) as its backbone
910            model. CALM (Schuster et al., 2022) is also a classical entropy-thresholding-based early-
911            exit method, and we evaluate it under the zero-shot setting.
912
913          • For SLEB(Song et al., 2024), we use Llama-3-8b (Dubey et al., 2024) and Gemma-7B (Mes-
914            nard et al., 2024) as its backbone model. SLEB(Song et al., 2024) tackles the limitation
915            of early exit methods by eliminating redundant transformer blocks. Since the proposed
916            RAEE exits at about 40% layers, for a fair comparison, we also set the hyper-parameter
917            num_remove_blocks of SLEB(Song et al., 2024) as int(60% · num_layers) for compa-
               rable efficiency.
