011                                               A BSTRACT
012
013            This paper introduces VALL-E 2, the latest advancement in neural codec language
014            models that marks a milestone in zero-shot text-to-speech synthesis (TTS), achiev-
015            ing human parity for the first time. Based on its predecessor, VALL-E, this work
016
               introduces two significant enhancements: Repetition Aware Sampling refines
               the original nucleus sampling process by accounting for token repetition in the
017
               decoding history. It not only stabilizes the decoding but also circumvents the
018
               infinite loop issue. Grouped Code Modeling organizes codec codes into groups
019            to effectively shorten the sequence length, which not only boosts inference speed
020            but also addresses the challenges of long sequence modeling. Our experiments
021            on the LibriSpeech and VCTK datasets show that VALL-E 2 surpasses previous
022            systems in speech robustness, naturalness, and speaker similarity. It is the first
023            of its kind to reach human parity on these benchmarks. Moreover, VALL-E 2
024            consistently synthesizes high-quality speech, even for sentences that are tradition-
025            ally challenging due to their complexity or repetitive phrases. The advantages
026            of this work could contribute to valuable endeavors, such as generating speech
027
               for individuals with aphasia or people with amyotrophic lateral sclerosis. See
               https://anonymous/valle2 for demos of VALL-E 2.

      1   I NTRODUCTION
049
050   Text-to-speech synthesis (TTS) aims to generate high-quality speech from text input with a high degree
051   of clarity and intelligibility. Along with the progress of deep learning, significant improvements have
052   been made in TTS research in recent years (Shen et al., 2018; Li et al., 2019; Ren et al., 2019). Some
053   systems, trained with clean single-speaker speech data recorded in sound-recording studios, have
      even achieved human-level quality for single-speaker speech generation (Tan et al., 2024). However,


                                                        1
      Under review as a conference paper at ICLR 2025


054
      zero-shot TTS, which requires the model to synthesize speech for unseen speakers using a short
055   enrolled speech sample during inference, remains a challenging problem.
056
057   The previous work, VALL-E (Wang et al., 2023a), marked a significant breakthrough in this area. It
058
      is capable of synthesizing personalized speech using only a 3-second recording, while preserving
      the speaker’s voice, emotion, and acoustic environment. VALL-E is a neural codec language model
059
      that represents speech signals as discrete codec codes with a neural audio codec model. Specifically,
060
      it trains an autoregressive language model to generate the coarse codec codes and another non-
061   autoregressive model to generate the remaining fine codec codes. Instead of using greedy search,
062   which continually generates silence codec codes, VALL-E uses random sampling for model inference.
063   However, VALL-E has two key limitations: 1) Stability: The random sampling used during inference
064   can lead to instability in output, while nucleus sampling with a small top-p value may cause an
065   infinite loop issue. This can be mitigated by multiple-time sampling and subsequent sorting, but
066   this approach increases the computational cost. 2) Efficiency: The autoregressive architecture of
067   VALL-E is bound to the same high frame rate as the off-the-shelf audio codec model, which cannot
068   be adjusted, resulting in a slower inference speed.
069   Several follow-up works have been proposed to address these problems (Song et al., 2024; Xin et al.,
070   2024; Borsos et al., 2023; Le et al., 2024; Ju et al., 2024). To improve stability, some works leverage
071   text-speech alignment information in model training and inference (Song et al., 2024; Xin et al.,
072   2024). These methods, relying on a forced-alignment model, inevitably introduces errors in the
073   alignment result, which could affect the final performance. It also complicates the overall architecture
074   and increases the burden for data scaling up. To improve modeling efficiency, some works explore
075
      fully non-autoregressive methods for zero-shot TTS (Borsos et al., 2023; Le et al., 2024; Ju et al.,
      2024). However, these methods require frame-aligned text-speech data for model training, facing the
076
      same problem as discussed before. Additionally, the non-autoregressive model generates the tokens
077
      with a pre-determined duration result, which constrains the search space of the generated speech and
078   sacrifices the prosody and naturalness.
079
080   In this work, we propose VALL-E 2, the first human parity zero-shot text-to-speech synthesis system.
081
      Building upon its predecessor VALL-E, VALL-E 2 employs a neural codec language modeling
      method for speech synthesis and incorporates two key modifications: repetition aware sampling and
082
      grouped code modeling. Repetition aware sampling, an improvement over the random sampling
083
      used in VALL-E, adaptively employs either random or nucleus sampling for each time step token
084   prediction. This selection is based on the token repetition in the decoding history, enhancing the
085   stability of the decoding process and circumventing the infinite loop issue encountered in VALL-E.
086   Grouped code modeling, on the other hand, partitions the codec codes into groups, each of which is
087   modeled in a single frame in the AR modeling process. This approach not only accelerates inference
088   by reducing the sequence length but also improves performance by mitigating the long context
089   modeling problem. Notably, VALL-E 2 requires only simple utterance-wise speech-transcription
090   pair data for training, greatly simplifying the process of collecting and processing training data and
091   facilitating potential scalability.
092   VALL-E 2 is trained on the large-scale Libriheavy dataset (Kang et al., 2024). Subsequent evaluations
093   demonstrate that it achieves performance on par with human capabilities on both the in-domain
094   LibriSpeech dataset (Panayotov et al., 2015) and the out-of-domain VCTK datasets (Veaux et al.,
095   2016). As illustrated in Figure 1, VALL-E 2 significantly outperforms VALL-E and other prior works
096   on the LibriSpeech dataset in terms of robustness, naturalness, and similarity score, even achieving
097   human parity performance. The numbers in Figure 1 are relative numbers (△ Score(Model) =
098
      Score(Model) − Score(GroundTruth)) based on the results reported in the paper. In this context,
      human parity indicates that the robustness, naturalness, and similarity metrics of VALL-E 2 surpass
099
      those of the ground truth samples (meaning that △ WERR(VALL-E 2) > 0, △ CMOS(VALL-E 2) >
100
      0, and △ SMOS(VALL-E 2) > 0), meaning that VALL-E 2 can generate accurate, natural speech
101   in the exact voice of the original speaker, comparable to human performance. It is important to
102   note that this conclusion is drawn solely from experimental results on the LibriSpeech and VCTK
103   datasets. Moreover, VALL-E 2 can accelerate the decoding process by multiple times with almost
104   no performance degradation. To specifically evaluate the stability of VALL-E 2, we synthesize
105   speech for complex sentences that are hard to read or contain many repeated phrases, and found
106   that VALL-E 2 can always stably generate high-quality speech. The benefits of this work could
107   support meaningful initiatives, such as generating speech for individuals with aphasia or people with
      amyotrophic lateral sclerosis.

      2.1   Z ERO -S HOT TTS
111
112   Early work in zero-shot TTS typically employed speaker adaptation and speaker encoding methods,
113   which often required additional fine-tuning, complex pre-designed features, or heavy structure
114   engineering (Chen et al., 2019; Wang et al., 2020; Arik et al., 2018; Casanova et al., 2022). Inspired
115   by the success of Large Language Models (LLMs) in natural language processing, VALL-E (Wang
116   et al., 2023a; Zhang et al., 2023b) represented speech as discrete codec codes with an off-the-shelf
117   neural codec model, and approached TTS as a conditional codec language modeling task. This
118   approach allowed VALL-E to train a codec language model on large-scale training data and perform
119
      zero-shot TTS via prompting, achieving significant zero-shot TTS capability.
120   This breakthrough inspired subsequent research works to address zero-shot TTS through a language
121   modeling approach. For instance, VALL-E X (Zhang et al., 2023b) extended VALL-E to cross-lingual
122   TTS tasks with an additional language ID token. SPEAR-TTS (Kharitonov et al., 2023) and Make-
123   a-voice (Huang et al., 2023a) leveraged semantic units from a speech self-supervised model as an
124   intermediate interface between text and acoustic codec codes, enabling better training data efficiency.
125
      Mega-TTS (Jiang et al., 2023b) and Mega-TTS 2 (Jiang et al., 2023a) proposed to first disentangle the
      multiple attributes in speech, then only model partial attributes with a language modeling approach.
126
      ELLA-V (Song et al., 2024) and RALL-E (Xin et al., 2024) improved VALL-E’s robustness and
127
      stability by including speech-text alignment prediction into the decoding process. UniAudio (Yang
128   et al., 2023b) and BASE TTS (Łajszczak et al., 2024) further explored scaling the codec language
129   model to 1b parameters and 100k hours of training data.
130
131
      Meanwhile, other works explored fully non-autoregressive modeling methods to accelerate the
      inference speed. For example, Soundstorm (Borsos et al., 2023) leveraged the confidence-based
132
      parallel decoding scheme (Chang et al., 2022) to generate the acoustic codec codes with a non-
133
      autoregressive model. StyleTTS 2 (Li et al., 2024), UniCATS (Du et al., 2024a), NaturalSpeech 2
134   (Shen et al., 2023) and NaturalSpeech 3 (Ju et al., 2024) used diffusion model (Ho et al., 2020) for
135   the prompt-conditioned text to speech synthesis. Voicebox (Le et al., 2024) and Audiobox (Vyas
136   et al., 2023) used flow-matching method (Lipman et al., 2022) and achieved better speech modeling
137   capability. In this work, VALL-E 2 follows the codec language modeling method of VALL-E,
138   and enables a stable decoding process without the need for complex speech data processing and
139   preparation, such as duration or pitch information used in previous methods. Notably, VALL-E 2
140   is the first to successfully achieve human parity in zero-shot TTS on both LibriSpeech and VCTK
141   datasets.
142
143   2.2   C ODEC - BASED S PEECH M ODELS
144
145   Inspired by the promising performance of neural codec codes in zero-shot TTS, many subsequent
146   research works have started to explore its effectiveness on more speech tasks. For instance, PolyVoice
147
      (Dong et al., 2023) adopted VALL-E and built a codec-based language model for speech-to-speech
      translation. SpeechX (Wang et al., 2023c) extended VALL-E with multi-task learning, demonstrating
148
      efficacy in zero-shot TTS, noise suppression, target speaker extraction, speech removal, and speech
149
      editing tasks. In addition to speech generation, VioLA (Wang et al., 2023b) further explored codec-
150   based speech models for speech understanding tasks, unifying codec language models for speech
151   recognition, synthesis, and translation tasks. AudioPaLM (Rubenstein et al., 2023) fused the codec
152   tokens into the LLM PaLM 2 (Anil et al., 2023), and demonstrated promising results on speech
153   recognition and translation tasks.
154
      These works typically employ SoundStream (Zeghidour et al., 2021) and Encodec (Défossez et al.,
155
      2022), initially designed for speech compression, as the neural codec model. Inspired by these
156   successes, several works have proposed more novel neural codecs specifically for speech processing
157   tasks. These include Vocos (Siuzdak, 2023), SpeechTokenizer (Zhang et al., 2023a), AudioDec (Wu
158   et al., 2023), AcademiCodec (Yang et al., 2023a), Descript-audio-codec (DAC) (Kumar et al., 2024),
159   FunCodec (Du et al., 2024b), and RepCodec (Huang et al., 2023b). The Codec-SUPERB challenge
160   (Wu et al., 2024) was announced to benchmark various codec codes across a wide range of speech
161   tasks. In this work, we utilize the Encodec model to tokenize speech signals and the Vocos decoder
      to generate target high-quality speech signals.

      3.1   P ROBLEM F ORMULATION : G ROUPED C ODEC L ANGUAGE M ODELING
165
166   Following VALL-E, we use an off-the-shelf neural audio codec model to represent speech signals as
167   discrete codec code sequence, and regard TTS as a conditional codec language modeling task. To
168   improve the efficiency, VALL-E 2 introduce a grouped codec language modeling method, where we
169   partition the codec code sequence into groups of a certain size, and model each group of codec codes
170   as one frame. In this way, we can get rid of the frame rate constraint of the off-the-shelf neural audio
171
      codec model, and reduce the frame rate by integer multiples. It is not only beneficial for the inference
      efficiency but also the overall speech quality by mitigating the long context modeling problem.
172
173   With TTS training objective, VALL-E 2 is optimized to maximize the likelihood of the grouped
174   code sequence given the text condition. Specifically, given an audio sample y and its corresponding
175   tokenized text transcription x = [x0 , x1 , . . . , x(L−1) ], where L is the text sequence length, we first
176   use a pre-trained neural audio codec model to convert the audio sample y into a codec code sequence
177
      CT ×J = [c0 , c1 , . . . , c(T −1) ], where T is the code sequence length, J (here J = 8) is the number
      of the quantizers in the codec model, and each ct represents the 8 codes for each time step. Then
178
      we partition it into the grouped code sequence CG = [C0:G , CG:2G , . . . , C(T −G):T ] with the group
179
      size G, and C0:G stands for the group [c0 , c1 , . . . , c(G−1) ]. Due to the typical short silence at the
180
      start of an utterance, we can clip a few codes from the start of the code sequence to let the code
181
      sequence length T be the integer multiple of the group size without removing any speech information.
182   Finally, we train the VALL-E 2 model θ to minimize the negative log-likelihood of the grouped code
183   sequence CG conditioned on the text sequence x:

237   3.3            VALL-E 2 T RAINING
238
239   Figure 2a shows the overview of VALL-E 2 model training. It is noteworthy that the training of
240   VALL-E 2 requires only simple utterance-wise speech-transcription pair data, without any complex
241   data such as force-alignment result or additional audio clips of the same speaker for reference. This
242
      greatly simplifies the process of collecting and processing training data.
243   Specifically, for each audio and corresponding transcription in the training dataset, we initially utilize
244   the audio codec encoder and text tokenizer to obtain the codec codes C = [c0 , c1 , . . . , c(T −1) ] and
245   the text sequence x = [x0 , x1 , . . . , x(L−1) ], respectively. These are then used for the AR model and
246   the NAR model training.
247
248   3.3.1                   AUTOREGRESSIVE M ODEL T RAINING
249
      The AR model is trained to predict the first codec code sequence c:,0 = [c0,0 , c1,0 , . . . , c(T −1),0 ]
250   conditioned on the text sequence x in an autoregressive manner.
251

362   Different from the random sampling method used in VALL-E, in this work, we propose a repetition

533   5     C ONCLUSION
534
535   We introduce VALL-E 2, a language modeling approach that achieves human parity zero-shot text
536   to speech synthesis for the first time. Based on the success of VALL-E, VALL-E 2 introduce two
537   simple but effective methods: repetition aware sampling for better decoding stability and grouped
538   code modeling for better modeling efficiency. Furthermore, our observations reveal that VALL-E 2
539   is capable of reliably synthesizing speech for complex sentences, including those that are challenging
      to read or contain numerous repeated phrase.

719   A.1       E XPERIMENTAL S ETUPS
720
721   A.1.1        M ODEL T RAINING
722
723
      We use Libriheavy corpus (Kang et al., 2024) as the training data. This corpus is a labeled version
      of the Librilight corpus (Kahn et al., 2020) that contains 50k hours of speech with around 7000
724
      distinct speakers derived from open-source English audiobooks that are part of the LibriVox project1 .
725
      We use Byte-Pair Encoding (BPE) for text tokenization, and the pre-trained open-sourced EnCodec
726   model (Défossez et al., 2022) at 6K bitrates for 24kHz audio reconstruction for speech tokenization.
727   Additionally, we use the open-sourced pre-trained Vocos model (Siuzdak, 2023) as the audio codec
728   decoder for speech generation.
729
      Following VALL-E, both the AR model and the NAR models employ the same Transformer archi-
730
      tecture in VALL-E 2. In our experiments, we mainly evaluate 4 VALL-E 2 models, which share
731
      the same NAR model but different AR models. The 4 AR models corresponds to the group size of
732   1, 2, 4 and 8. Among these models, the AR model with group size of 1 is implemented without the
733   group embedding layer and group prediction layer, and the baseline model VALL-E employs the
734   same NAR model and AR model with group size of 12 .
735
      Both the AR and NAR models are trained using 16 NVIDIA TESLA V100 32GB GPUs. The models
736
      are optimized with the AdamW optimizer, with the learning rate warmed up for the first 32k updates
737
      to a peak of learning rate, then linearly decayed. For NAR model training, the length of the acoustic
738   condition is randomly sampled to be the maximum of half of the current utterance with a random
739   value from 3s to 30s.
740
741   A.1.2        E VALUATION M ETRICS
742
743   We employ subjective evaluation metrics, including SMOS and CMOS, to assess the speaker similarity
744   and comparative naturalness of synthesized speech, respectively. We invite 20 external native speakers
745   of American English to participate as contributors in a crowdsourcing effort to evaluate each speech
746
      from various perspectives.
747   SMOS (Similarity Mean Opinion Score) is used to evaluate the speaker similarity of the speech to
748   the original prompt. The SMOS scale ranges from 1 to 5, with increments of 0.5 points.
749
      CMOS (Comparative Mean Opinion Score) is used to evaluate the comparative naturalness of the
750   synthesized speech against a given reference speech. The CMOS scale ranges from -3 (indicating
751   the synthesized speech of the new system is much worse than the reference) to 3 (indicating the new
752   system is much better than the reference), with intervals of 1. In our study, we use the ground truth
753   speech as the comparison reference.
754

756
      We also employ objective evaluation metrics including SIM, WER, and DNSMOS to assess the
757   speaker similarity, robustness, and overall perceived quality of each synthesized speech. For a better
758   comparison in speech continuation, we evaluate the entire utterance instead of focusing solely on the
759   continuation segment.
760
      SIM is used to evaluate the speaker similarity between the original prompt and synthesized speech,
761
      leveraging the SOTA speaker verification model, WavLM-TDNN 3 (Chen et al., 2022). The similarity
762
      score predicted by WavLM-TDNN is in the range of [−1, 1], with a larger value indicating higher
763   speaker similarity.
764
765   WER (Word Error Rate) is used to evaluate the robustness of synthesized speech. Neural TTS
766
      systems sometimes experience deletion, insertion, and replacement errors due to incorrect attention
      alignments, which can affect their robustness. We perform ASR on the generated audio and calculate
767
      the WER with respect to the original transcriptions. In this experiment, we employ the open-sourced
768
      Conformer-Transducer model4 (Gulati et al., 2020) as the ASR model.
769
770   DNSMOS (Deep Noise Suppression Mean Opinion Score) is used to assess the overall perceived
771   quality of the generated speech (Reddy et al., 2021). Specifically, we use a model trained with ground
772
      truth human ratings obtained using ITU-T P.808 (ITU, 2018)5 to predict the DNSMOS score, which
      is in the range of [1, 5], with a larger value indicating better quality.

      A.1.3    E VALUATION S ETTINGS
775
776   We use LibriSpeech test-clean (Panayotov et al., 2015) and VCTK (Veaux et al., 2016) for zero-shot
777   TTS evaluation, ensuring none of the speakers from these corpora are included in the training data.
778
      LibriSpeech test-clean is an official test split from the LibriSpeech corpus, containing English speech
779
      sampled at 16kHz. It originates from the same domain of the LibriVox project as the training data
780   but features different speaker IDs. Following Borsos et al. (2022) and Wang et al. (2023a), we use
781   samples from LibriSpeech test-clean with lengths between 4 and 10 seconds, resulting in a 2.2 hours
782   subset and 40 unique speakers. We evaluate each sample synthesis under two settings: 3s Prefix as
783   Prompt and Ref Utterance as Prompt. For the first setting, we perform speech continuation and utilize
784   the 3-second prefix of the speech as the prompt. In the second setting, we use a reference utterance
785   from the same speaker as the prompt. Specifically, we begin by filtering the official speech list of
786   LibriSpeech test-clean based on length. For the ordered speech list of each speaker, in the first setting,
787   we synthesize the i-th speech sample using the first 3 seconds of the ground-truth i-th speech sample
788
      as the prompt. In the second setting, we synthesize the i-th speech sample using the (i − 1)-th sample
      as the prompt and synthesize the first speech sample using the last sample as the prompt.
789
790   VCTK is a reading corpus with speech sampled at 48kHz by 108 English speakers. Compared to
791   LibriSpeech, VCTK presents a greater challenge as it encompasses speakers with a wide range of
792   accents. We evaluate each sample synthesis under three settings: using prompts of 3s, 5s, and 10s
793   in length. Specifically, for each speaker, we select an utterance whose length is closest to but less
794
      than 3s/5s/10s to serve as the prompts. We then randomly sample another utterance and use the
      corresponding transcription as the text input for speech synthesis.
795
796   For each sample synthesis, we first perform inference with the AR model to generate the first code
797   sequence using the repetition aware sampling method (Section 3.4.1), where we set the hyperparameter
798   K = 10, tr = 0.1, and select the top-p value v from 0.0 to 0.8 with the intervals of 0.1. Next, we
799   perform inference on the NAR model seven times to generate the remaining seven code sequences
800
      using a greedy decoding method. The sampling-based decoding method of the AR model allows us
      to generate diverse samples from the same input.
