009                                               A BSTRACT
010
011            Recent advances have demonstrated the powerful reasoning capabilities of large
012            language models (LLMs), and accurately measuring the confidence of reasoning
013            paths is crucial for improving the performance and trustworthy of AI systems.
014            Benefiting from consistency function for reasoning, the self-consistency method
015            often provides an effective confidence estimation. However, it suffers from the
016            variance issue, which extremely constrains the performance when the sampling
017            is insufficient. Existing methods such as the temperature sampling cannot well
018
               resolve this problem as it not only necessitates a calibration set but also tends to
               sacrifice the reasoning capability of LLMs. In this paper, we propose a data-free,
019
               and highly sampling efficient method to control the variance. The merit of our
020
               approach lies in a reasonable integration of the LLM’s probability estimation and
021            the self-consistency confidence. Our theoretical analysis confirms the efficacy of
022            our method by achieving a lower estimation error and a higher error reduction
023            rate. Furthermore, an in-depth analysis of the error decomposition reveals an im-
024            proved technique, which can significantly improve error reduction rate with only a
025            small scale of bias induced. Experimental results across seven benchmark datasets
026            demonstrate that our proposed approaches achieve superior confidence estimation,
027            boosting the accuracy on both mathematical reasoning tasks and code generation
028            tasks. Our code is provided in the supplementary material.

      1   I NTRODUCTION
032
033
      Recently, large language models (LLMs) have made significant progress in various applications,
      including problem solving (Lewkowycz et al., 2022b; Li et al., 2024), planning (Valmeekam et al.,
034
      2023; Deng et al., 2024), and decision making (Ouyang & Li, 2023; Sblendorio et al., 2024), show-
035
      casing their strong reasoning capabilities. The confidence in reasoning, i.e., the likelihood of the
036   reasoning answer being correct, can help refine the answers from reasoning paths (Wang et al.,
037   2022), enhance the interpretability of reasoning results (Stengel-Eskin & Durme, 2023), and ulti-
038   mately contribute to building trustworthy artificial intelligence systems (Guo et al., 2017; Felicioni
039   et al., 2024). However, recent studies (Shen et al., 2024; Geng et al., 2024; Zhao et al., 2024) have
040   shown that LLMs often fail to provide reliable confidence estimates, underscoring the importance
041   of accurate confidence estimation for LLM reasoning.
042   There are several typical confidence estimation methods for LLM reasoning, i.e., perplexity (Chen
043   et al., 1998), verbalized confidence (Xiong et al., 2023; Tian et al., 2023), and self-consistency con-
044   fidence (Wang et al., 2022; Chen et al., 2023). Among them, our empirical studies show that, on
045   math reasoning tasks, self-consistency confidence with a proper consistency function can consis-
046   tently provide superior and satisfactory performance in both accuracy and calibration error metrics.
047   However, the self-consistency confidence suffers from the large variance issue when the sampling is
048   insufficient. This is because an accurate self-consistency confidence is computationally infeasible,
049   and existing methods tend to adopt the Monte-Carlo sampling as an estimation, which suffer from
050
      low variance reduction efficiency of a linear rate.
051   A straightforward method to reduce the variance could be controlling the sampling temperature,
052   since the decrease of temperature can narrow down the sampling space and alleviate the requirement
053   of sample size. Nevertheless, the tuning of temperature often necessitates an additional calibration
      set, and a low temperature also limits the reasoning capability, degrading the LLM performance.

054
      In addition, our empirical results show this issue is further enlarged, as the increase of the reason-
055   ing difficulty. To this end, we explore a new problem, namely, Resource-Constrained Confidence
056   Estimation for LLM Reasoning, where the sample size is limited and additional calibration set is un-
057   available. The constraints in sampling size and additional dataset is very practical in LLM reasoning
058   tasks, especially considering huge computational overhead (Zhou et al., 2024) and the expensive
059   labeling cost (Lightman et al., 2023). Hence, our goal is to estimate accurate self-consistency confi-
060   dence using given samples to achieve better reasoning performance.
061   To address this problem, we propose to integrate the prediction probability of LLMs into the self-
062   consistency confidence estimation, which forms our Perplexity Consistency confidence estimation
063   approach (P C). The rationale of this approach lies in that using the accurate LLM prediction prob-
064   ability with zero variance instead of the crude Monte-Carlo sampling can significantly reduce the
065   variance. In addition, theoretical analysis also confirms that the integration could indeed reduce the
066   estimation error and achieves a quadratic O(1/n2 ) decreasing rate. Moreover, the decomposition
067   of estimation error guides us to further boost the convergence rate to be exponential in n, through
068   pruning the low probability reasoning paths. To achieve this, we propose Reasoning Pruning to
069
      model the confidence distribution and automatically remove the reasoning paths with low probabil-
      ity. Combining P C approach and Reasoning Pruning, we build our Reasoning-pruning Perplexity
070
      Consistency confidence estimation approach (R PC). Our experimental results on seven benchmark
071
      datasets, including mathematical reasoning and code generation tasks, demonstrate that our pro-
072   posed P C and R PC approaches deliver superior performance compared to existing methods.
073
074   The contributions of this paper are summarized as follows:
075
            (1) We introduce and highlight a new problem setting, namely, Resource-Constrained Confi-
076
                dence Estimation for LLM Reasoning, which aims to accurately estimate self-consistency
077
                confidence with a constrained sample size and without additional calibration set in order to
078             achieve improved reasoning performance.
079
080         (2) We propose the P C and R PC approaches, which leverage accurate LLM prediction prob-
081             abilities to reduce the variance in crude Monte-Carlo sampling process and prune low-
082
                probability reasoning paths to achieve faster convergence, respectively.
083         (3) Our theoretical analysis demonstrates that P C achieves a quadratic error decreasing rate of
084             O(1/n2 ), which is faster than the linear rate of standard self-consistency method. Further-
085             more, the decomposition of the estimation error guides us in designing our approach to
086             boost the convergence rate to be exponential in n.
087         (4) We conducted experiments on four mathematical reasoning tasks, and the results demon-
088             strate that our P C and R PC approaches achieve significant improvements in both accuracy
089             and calibration error. Moreover, the results from code generation tasks further confirm the
090             generalizability of our approaches.

      Challenges. Although promising, S C’s performance faces a dilemma. On one hand, its performance
217   could be significantly affected by the sampling sufficiency of reference answer. For example, as
218   shown in Figure 2a, the requirements of sample size dramatically grows as the reasoning problems
219   become much harder (e.g., MATH→AIME). On the other hand, constructing a large collection of
220   reference answers is often computationally expensive and time-consuming. For example, generating
221   a 100-sized reference answers for all problems in MATH, which contains 5,000 problems, using a
222   single A800 GPU would take about 18 hours. The total emissions are estimated to be 4.14 kg CO2
223   when using Google Cloud Platform in the asia-south1 region (Lacoste et al., 2019). This dilemma
224   raises a key question: Is it possible to accurately estimate self-consistency confidence, even when
225
      the sample size is limited?
226   Since the sampling estimation in self-consistency confidence is unbiased, the error from insufficient
227   sampling is mainly caused by variance. The above question then becomes: How to effectively control
228   the variance of self-consistency confidence, particularly when the sample size is limited?
229   To answer the above question, a straightforward method to control the variance could be choosing
230   a conservative temperature, which tends to narrow the sampling space of LLMs. To illustrate this,
231   we present the perplexity distribution at different temperatures in Figure 2b. However, as evident
232   in Figure 2c, although a low sampling temperature successfully decreases the induced variance,
233   it sacrifices model performance. In other words, a high temperature T offers better performance
234   potential but requires a larger sample size n, creating another dilemma between accuracy and effi-
235   ciency. Moreover, selecting an appropriate temperature T requires an additional calibration set and
236   computational resources (Guo et al., 2017), which is expensive in practice.
237   To this end, our analysis motivates us to study a novel and challenging problem setting for LLM rea-
238   soning, namely, Resource-Constrained Confidence Estimation for LLM Reasoning. In this setting,
239   we aim to achieve better confidence estimation even when the sampled reference collection is of
240   small size, thereby improving both accuracy and reducing calibration error in LLM reasoning tasks.

472       6                   C ONCLUSION
473
474      In this paper, we explore Resource-Constrained Confidence Estimation for LLM Reasoning, where
475      the goal is to estimate accurate self-consistency confidence with a limited sample size to achieve
476      better reasoning performance. We integrate the probability directly obtained from LLMs to reduce
477      the variance of crude Monte-Carlo sampling estimation method, forming the P C approach. Our
478      theoretical analysis guarantees that the estimation error decreases at a quadratic rate of O(1/n2 ).
479      Additionally, we introduce a Reasoning Pruning strategy, motivated by our decomposed estimation
480      error, to further boost the convergence rate to exponential in n. Experiments on four mathematical
481      reasoning tasks demonstrate the effectiveness of the proposed P C and R PC approaches. Furthermore,
482
         results from code generation tasks highlight the generalizability of our methods.
483      One limitation of this work is that we were unable to explore additional mathematical models with
484      varying parameter scales due to resource constraints. However, we believe that our current exper-
485      iments are robust enough to validate our claims and demonstrate the superior performance of our
         proposed approaches.

840   problems and we use its 5,000 testing data for evaluation. The MathOdyssey dataset (Fang et al.,

      2024) contains 387 problems, covering advanced high-school level, university-level, and Olympiad-
      level mathematics. The OlympiadBench dataset (He et al., 2024) contains 8,476 Olympiad-level
842
      mathematics and physics problems. We select the English problems without images, resulting in
843
      a testing dataset of 1,284 problems. The AIME dataset (Zamil & Rabby, 2024) contains 993 test
844   problems collected from the American Invitational Mathematics Examination, spanning from 1983
845   to 2024.

      HumanEval (Chen et al., 2021) contains 164 hand-written Python programming problems.

      B.2         D ETAILES OF C ODE R EASONING TASK
853
854   Code generation. On the code reasoning task, we let LLM generate a code snippet to solve a given
855   programming problem, and then evaluate its functional correctness based on the ground-truth test
856   cases provided by the dataset. In detail, we set the top p to 0.95, the max generation length to 1024.
857   For code snippet post-processing, we first extract the code text from code block surrounded by triple-
858   backticks(‘‘‘), and then we follow Chen et al. (2021) to truncate the generated code snippet before
859   the following stop sequences: ”\nclass”, ”\ndef”, ”\n#”, ”\nif”, ”\nprint”. At the same time, we
860
      also obtain the log-probability of each token from the LLM response. For ”verbalization” setting, the
      verbalized confidence is also extract from the text generated by LLM along with the code snippet.
861
862   Self-consistency on code. We follow Chen et al. (2022) to sample 100 test cases for each pro-
863   gramming problem from the same model. And then we achieved self-consistency on code at the
      semantic equivalence level, which based on the execution behavior of any two codes on this set of


                                                                                                       16
      Under review as a conference paper at ICLR 2025


864
      test cases. More formally, we implemented the consistency function IC (·, ·) as an indicator function
865   that indicates whether two codes are semantically equivalent, i.e., IC (x, y) = 1 if and only if code
866   x and y execute the same result on this set of test cases.
