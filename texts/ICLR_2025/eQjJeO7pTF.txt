009                                               A BSTRACT
010
011            Generative models have become a powerful tool for image editing tasks, including
012            object insertion. However, these methods often lack spatial awareness, generating
013            objects with unrealistic locations and scales, or unintentionally altering the scene
014            background. A key challenge lies in maintaining visual coherence, which requires
015            both a geometrically suitable object location and a high-quality image edit. In
016
               this paper, we focus on the former, creating a location model dedicated to iden-
               tifying realistic object locations. Specifically, we train an autoregressive model
017
               that generates bounding box coordinates, conditioned on the background image
018
               and the desired object class. This formulation allows to effectively handle sparse
019            placement annotations and to incorporate implausible locations into a preference
020            dataset by performing direct preference optimization. Our extensive experiments
021            demonstrate that our generative location model, when paired with an inpainting
022            method, substantially outperforms state-of-the-art instruction-tuned models and
023            location modeling baselines in object insertion tasks, delivering accurate and vi-
024            sually coherent results.

      1   I NTRODUCTION
027
028
      Explicit modeling of object locations has recently proven to be an effective strategy for generating
029
      complex scenes. Methods that use this strategy often separate the tasks of determining where objects
030   should be placed and what those objects should look like (Feng et al., 2024a; Phung et al., 2024;
031   Cho et al., 2024; Lian et al., 2023). This two-step approach typically involves generating a spatial
032   layout and then conditioning an image generation model on this layout to create the final image. The
033   success of these approaches has brought renewed attention to location modeling as a key component
034   for achieving realistic and coherent scene generation.
035
      We explore explicit location modeling for object insertion, which involves determining where new
036   objects can be placed in a given scene. This is an important task with applications in generative data
037   augmentation (Zhao et al., 2023; Kupyn & Rupprecht, 2024; Fang et al., 2024), virtual reality (Park
038   et al., 2005) and robotics (Cheong et al., 2020). Given a scene image and an instruction specifying
039   an object to be added, object insertion aims to place a new object into the scene with a realistic
040   appearance and geometry, while preserving the background and other objects intact. Unlike gener-
041   ating a full scene layout (Gupta et al., 2021), object insertion must account for the rich contextual
042   information provided by the image, which also imposes strong constraints on where objects can and
043   cannot be placed.
044   Current state-of-the-art object insertion methods rely on instruction-tuned image editing (Brooks
045   et al., 2023; Zhao et al., 2024; Zhang et al., 2024b; Wasserman et al., 2024), which requires models to
046   jointly learn the realism in both appearance (what) and spatial placement (where). This is a difficult
047   task that requires large-scale object insertion datasets that are inherently hard to construct. Despite
048   training on such datasets, existing approaches often favor realism in appearance over placement,
049   generating objects in unrealistic locations, replace or destroy existing objects, and create unintended
050
      changes in unrelated areas, as seen in Figure 1 (a).
051   In this work, we propose a two-stage object insertion pipeline that first identifies the object location
052   using a dedicated location model, then generates the object locally in the appointed location using an
053   inpainting model, as illustrated in Figure 1 (b). Object placement datasets (Liu et al., 2021; Wasser-
      man et al., 2024) suitable for training such a model have recently been released. However, as it is

078

079   their scalability across datasets.
080
081   To overcome these challenges, we approach the problem from a generative perspective, as training
082
      a generative model only requires access to samples from the target distribution. Specifically, we
      represent the input image and the object class as a sequence of tokens and use an autoregressive
083
      transformer model (Vaswani et al., 2017; Radford et al., 2019) to iteratively decode bounding box
084
      coordinates of plausible object locations. Furthermore, we can also fully leverage any additional
085   negatively labeled locations by treating pairs of positive and negative labels as a preference dataset,
086   where positive locations are preferred over negative locations. This perspective allows for direct
087   preference optimization (Rafailov et al., 2024) on the location model, which further enhances the
088   accuracy of the locations.
089
      We empirically observe that given a precise location of the object to be generated, off-the-shelf in-
090
      painting models outperform state-of-the-art instruction-tuned object insertion models. Moreover,
091   we observe that any inaccuracy in estimating the object location can significantly degrade the image
092   quality, highlighting the role of location modeling as the critical component of the object inser-
093   tion pipeline. Our location model produces high-quality locations, allowing it to outperform both
094   instruction-finetuned models in object insertion and existing location models in terms of positional
095   accuracy. We further validate the effectiveness of our approach in a user study.
096
      Summarizing our main contributions are as follows:
097
098          • We propose a two-stage object insertion approach, that overcomes the limitations of
099            instruction-tuned editing with an explicit location model.
100
101          • We effectively handle sparsity in location annotations with a generative approach: an
102            image-conditioned autoregressive transformer that models bounding box locations.
103
             • We can leverage available negative annotations following our generative formulation using
104            direct preference optimization, further improving the accuracy of our location model.
105
106          • Through extensive experiments and a user study, we demonstrate that our approach
107            achieves state-of-the-art performance in location modeling, and significantly outperforms
               instruction-tuned image editing models in object insertion tasks.

      2     R ELATED W ORK
109
110
      Instruction-based Image Editing. InstructPix2Pix (Brooks et al., 2023) introduced an approach
111   that finetunes Stable Diffusion (Rombach et al., 2021) to interpret text instructions, trained with a
112   dataset of paired images before and after specific edits. Subsequently, other methods (Zhang et al.,
113   2024b; Hui et al., 2024; Zhang et al., 2024a) have released similar datasets for instruction-tuning,
114   capable of changing the style and content of a given image.
115
      Only recently has there been an emphasis on adding objects using text instructions, supported by
116
      datasets that provide paired images where specific objects have been artificially removed by inpaint-
117   ing (Wasserman et al., 2024; Zhao et al., 2024). One downside of these datasets is that they contain
118   inpainting artifacts, which can cause models trained on them to replace existing objects or alter
119   backgrounds. Additionally, since these models are typically trained to regenerate the entire image,
120   they often introduce unintended changes to the scene. In contrast, our approach decouples the object
121   insertion process by using a dedicated location model for determining placement and an inpainting
122   model for rendering. This factorization allows for more control and precision in object insertion.
123   Object Placement. Traditionally, object placement has relied on copy-pasting an object segment
124   by simply determining its location and scale (Zhang et al., 2020; Zhu et al., 2023; Zhou et al., 2022;
125   Tripathi et al., 2019; Niu et al., 2022). However, this approach is not ideal for inserting objects into
126   background images, as it requires the user to prepare an image of the object that fits seamlessly into
127   the scene purely by placement.
128
      To avoid relying on object segments, some approaches predict locations from class labels by query-
129   ing a classifier (Dvornik et al., 2018) on random bounding boxes, categorizing them as either plausi-
130   ble or implausible locations for the given class. To enable training a discriminative model, unlabeled
131   locations are typically treated as negatives or implausible locations. Such an assumption is not al-
132   ways valid in object placement, as the lack of annotations for specific locations does not necessarily
133   indicate they are implausible. Therefore, penalizing them may result in an inaccurate location model.
134   To overcome this limitation, we instead propose a generative approach that requires only samples
135   from the target distribution, and does not make any assumption about unlabeled locations and only
136   requires samples of the target distribution (i.e., positive locations).
137   Layout Generation. Another related category of work is focused on scene layout generation, usu-
138   ally via a generative model of bounding box locations (Jyothi et al., 2019; Gupta et al., 2021; Chai
139   et al., 2023; Inoue et al., 2023) or segmentation maps (Lee et al., 2018). Such layouts are often
140   used as a condition for image generation models such as GLIGEN (Li et al., 2023) to generate com-
141   plex scenes (Feng et al., 2024a; Phung et al., 2024; Cho et al., 2024; Lian et al., 2023; Gani et al.,
142   2024; Feng et al., 2024b). Unlike these full-layout generation approaches, our goal is to insert an
143
      object into an existing scene. Having access to the background image fundamentally changes the
      nature of the task. On one hand, the background image provides valuable context for realistic ob-
144
      ject placement, but on the other hand, it imposes strong constraints on where objects can be placed.
145
      We therefore design a location model that integrates these contextual cues and avoids placement in
146   unrealistic locations.

201   While the training objective in Equation 2 allows training the location model on sparse positive
202   annotations, training solely on positive feedback can lead to predictions in implausible locations.
203   Incorporating negative annotations, when available, into the training objective can be beneficial for
204   refining the model, encouraging it to assign lower likelihoods to undesirable locations and thereby
205   improving overall accuracy.
206   Our generative formulation allows us to use any negative labels in the dataset as well, so that the
207   model can learn to avoid predicting bounding boxes for implausible locations. Specifically, we treat
208   the positive and negative labels as a preference dataset, where positive locations are implicitly pre-
209   ferred over negative ones, even though annotators were not explicitly asked to rank them. Using this
210   preference structure, we fine-tune the model with direct preference optimization (DPO) (Rafailov
211   et al., 2024), penalizing high logits assigned to negative labels. We repeat the training objective

242   4.1       DATASETS AND ARCHITECTURE
243
244   PIPE Dataset. The PIPE dataset (Wasserman et al., 2024) was created by removing objects from
245   object detection datasets (Lin et al., 2014; Kuznetsova et al., 2020; Gupta et al., 2019) by inpainting.
246   This process results in pairs of images, one including the object and the other without it. To train our
247
      location model we need positive bounding box locations, that we derive for the missing object by
      thresholding the pixel-wise difference between the two images. An example is reported in Figure 3
248
      (left). While the dataset offers a large number of 888,000 samples, many images contain inpainting
249
      artifacts, potentially introducing noise in the bounding box extraction process.
250
251   OPA Dataset. The OPA dataset (Liu et al., 2021) was created by asking human annotators to judge
252   the plausibility of object placement locations for a subset of COCO images (Lin et al., 2014). This
253
      dataset includes on average 41.5 annotations per image, and can be used to encourage diversity in
      location model predictions (see Figure 3 right for an example). The train set includes 1022 images,
254
      and the test set include 130 images. As OPA provides negative labels, we also use OPA as the
255
      preference dataset for DPO training. Note that instruction-tuned editing models cannot leverage this
256   location dataset, as they require fully rendered images for training.
257
258   Implementation details. We use a small GPT-2 (Radford et al., 2019) architecture as our autore-
259
      gressive location model. For tokenizing the image and the object class, we use a ViT (Dosovitskiy,
      2021) model pre-trained on ImageNet-21K (Ridnik et al., 2021) as our image encoder and the CLIP
260
      text encoder (Radford et al., 2021) as our object class encoder. We pre-train on PIPE for 30K itera-
261
      tions and finetune on OPA for 3K iterations. For batch size 64, the model can be trained on a single
262   Nvidia V100 GPU. We quantize each box coordinate into one of 512 bins (i.e., one bin per pixel).
263   For DPO training, we train using the OPA dataset for 4K iterations. Please refer to Section D of the
264   Appendix for further details.
265
      We evaluate the performance of our generative location model in an object insertion pipeline in
266
      Section 4.2, where we rely on PowerPaint (Zhuang et al., 2023) as the inpainting approach. Specif-
267
      ically, we utilize the V2 version1 , built on top of BrushNet (Ju et al., 2024). We remark that our
268   pipeline has the flexibility to incorporate any inpainting method for performing localized edits (see
269   later in Section 4.4).

      Baselines. We compare our object insertion pipeline against recent instruction-tuned image edit-
      ing models, which represent some of the most advanced techniques for object insertion to date.
291
      Specifically, we evaluate against general-purpose image editing models InstructPix2Pix (Brooks
292
      et al., 2023), HIVE (Zhang et al., 2024b), and MagicBrush (Zhang et al., 2024a), as well as object
293   insertion-specific models, Paint-by-Inpaint (Wasserman et al., 2024) and Diffree (Zhao et al., 2024).
294
295   Moreover, we experiment with three additional explicit location models that we also pair with the
296
      PowerPaint inpainter. First, we test random locations as a simple baseline. Then, we train Con-
      textCNN (Dvornik et al., 2018), a classifier designed to assess masked regions of an image, deter-
297
      mining their suitability for object placement. To be able to use it with the OPA class space, we
298
      retrain the model on COCO (whose classes comprehend all the ones in OPA). Furthermore, we
299   train a Faster-RCNN object detector (Ren et al., 2015) using positive OPA annotations to serve as a
300   high-performing discriminative model.
301
302
      Evaluation Metrics. Similar to existing image editing benchmarks (Wasserman et al., 2024), we
      evaluate the success of object insertion by considering both how well the original scene is preserved
303
      and how accurately the target object is inserted. Using an object detector (Zhu et al., 2021) we
304
      separate the background, which ideally should remain untouched, and the foreground, where the
305   newly inserted object appears.
306
307   To measure background preservation, we compute the Structural Similarity Index Measure
308
      (SSIM) (Wang et al., 2004) on the background region. To assess the accuracy of the object, we
      measure the CLIP similarity (Radford et al., 2021) between the cropped object and the text “an
309
      {object class}”. If no new object is detected, we assign a CLIP score of 0 to reflect the failure to
310
      properly insert the object.
311
312   For the PIPE dataset, we also evaluate the diversity of edits by calculating the average LPIPS dis-
313   tance (Zhang et al., 2018) across 10 different edits per background-object pair. High LPIPS indicates
314
      that a model can generate diverse results from the same instruction, highlighting approaches that are
      limited to producing a single edit.
315
316   Results. We compare instruction-finetuned image editing models, location models paired with
317   strong inpainting models, and our approach in Figure 4. Our approach substantially outperforms
318   all baselines by leveraging a dedicated location model for inpainting, which effectively reduces
319   background distortions while maintaining high-quality object generation.
320   By design, localized inpainting is an effective strategy for preserving the background, but it is no-
321   table that even a random location model outperforms the best instruction-finetuned method on back-
322   ground SSIM. Since the key difference between our approach and other location modeling baselines
323   lies in the plausibility of the predicted locations, the figure suggests that an accurate placement
      directly impacts the quality of inpainting results.

336   The instruction-tuned models often face a trade-off: they either preserve the background well but
337   fail to generate the object convincingly, or they successfully generate the object but significantly
338   alter the surrounding scene. This inconsistency may stem from the fact that these models address

353   participants, each of which ranks 40 image pairs. For further details, see Appendix Section F.

      Results are shown in Figure 5. Participants preferred edits generated by our approach over those

355
      edit quality can be achieved through precise location modeling. Additional qualitative examples can
356
      be found in Figure 7, Figure 8, and Appendix Section G.

      4.3   L OCATION M ODELING
359
360   Baselines. We compare our generative location model against two discriminative approaches that
361   classify locations as plausible or implausible, namely ContextCNN (Dvornik et al., 2018) and Faster-
362   RCNN (Ren et al., 2015), as described in Section 4.2. Additionally, we compare to two object
363   placement baselines (Zhou et al., 2022; Niu et al., 2022) that perform placement of specific object
364   segments rather than generic class labels. We use the official implementations relying on foreground
365   segments available within the OPA dataset. Note that a direct comparison to these two methods is
366   challenging, as our method does not use the object segment, and it is hard to say whether access to
367   one makes the task harder or easier. Still, we include these baselines for completeness.
368   Evaluation Metrics. The OPA test set provides plausible (positive) and implausible (negative)
369   locations for objects given an image. However, due to the sparse nature of these annotations, it is
370   impossible to sample locations until every ground-truth bounding box is matched with a prediction.
371   Therefore, we evaluate location models based on a “hit rate” metric, which compares the rate of each
372   predicted box being a plausible or implausible location. Specifically, we measure the True Positive

      where the cost function is the inverse of the intersection over union (IoU).

377   while false positive predictions are those assigned to negative labels under the same IoU threshold.

      4.4   A BLATION S TUDY
409
410   DPO Training. Training a generative location model exclusively on positive locations already
411   demonstrates strong performance, as shown in Figure 6 (right). However, incorporating negative
412   labels further enhances accuracy by explicitly guiding the model on where not to predict object lo-
413   cations. Notably, unlike existing location modeling techniques, which assume non-labeled locations
414   as negative even when negative labels are present, our approach leverages only the negative labels
415   provided by annotators. This ensures more precise predictions, as it avoids the potential inaccuracies
416   introduced by assuming non-labeled locations are negative.
417   Alternative Inpainting Methods. Our location model is not tied to a single inpainting method and
418   can incorporate various inpainting techniques (Xie et al., 2023; Ju et al., 2024; Cao et al., 2024).
419   To illustrate this, we use the same predicted locations to render objects with the inpainting model
420   of GLIGEN (Li et al., 2023). The resulting images rendered achieve a background SSIM of 0.6511
421   and foreground CLIP of 0.2833, compared to a performance of 0.7184 background SSIM, 0.2849
422   foreground CLIP, when using PowerPaint. Obtaining similar foreground CLIP score indicates that
423   inpainting is often successful even with the older GLIGEN model. We therefore expect that our
424
      approach benefits from future advancements in inpainting techniques as well.

524   inpainting techniques tailored specifically for object insertion, which we leave as future work.

526   6   C ONCLUSION
527
528   In this paper, we present a location model that identifies plausible locations for objects to be inserted
529   within an image. By taking a generative approach, we are able to work with sparsely annotated
530   location datasets. We also show that it is beneficial to use negative labels via direct preference
531   optimization. Our experiments suggest that in the task of object insertion, separating the problems
532   of where to place an object and what to place in the given location, is currently much more reliable
533   than instruction-tuned insertion approaches.
534   More generally, we believe that building spatial awareness is a key factor for building reliable models
535   interacting with the real world. This is true whether a model operates in the image editing setting, as
536   in this work, or in more complex domains, such as robotics or virtual reality. Our work shows that
537   a model is able to learn such awareness from example positive and negative annotations. Although
538   we focus on locations for 2D bounding boxes conditioned on images, we hope to see similar models
539   scale to handle other scene representations (e.g., depth or semantic layouts) and precise locations
      (e.g., 3D bounding boxes or object masks) in the future.

      Broader Impact Statement Spatial awareness is crucial for real-world applications where pre-
541   cise and context-aware object placement is necessary. Improving the reliability of object insertion
542   methods can benefit numerous fields, such as augmented reality, robotics, and generative data aug-
543   mentation, by enabling more realistic and practical scene manipulations. However, image editing
544   methods can also be misused, for example, to manipulate images by inserting individuals into scenes
545   to damage their reputation. Our method, which specifies only the desired object class rather com-

547   use cases. Nevertheless, one should consider malicious use cases of image editing methods before

759       A                            DATASET S TATISTICS AND P REPROCESSING
760
761
762       A.1                          PIPE DATASET
763
764      The PIPE dataset (Wasserman et al., 2024) was created by removing objects from object detection
765      datasets (Lin et al., 2014; Kuznetsova et al., 2020; Gupta et al., 2019) using an inpainting model,
766      resulting in over 600 object classes for insertion and more than 888,000 training pairs of images,
767      showing scenes before and after object removal. To preprocess this dataset, we pair each background
768      image with the original location of the removed object. The locations are identified by computing
769
         the pixel-wise difference between the before-and-after images, and extracting coordinates where the
         difference exceeds a certain threshold. Despite this thresholding, the resulting bounding boxes can
770
         be noisy, and the background images generated by the inpainting model often contain artifacts. Also,
771
         the PIPE dataset only provides a single positive location for an object for each background image
772      and does not provide negative labels.
773
774
775
          A.2                          OPA DATASET
776
777
778
         The OPA dataset was created by manually annotating samples from the COCO dataset (Lin et al.,
         2014), resulting in 47 object categories for object placement. The dataset is intended for the task of
779
         finding locations to copy-paste images of objects, and therefore includes background images, object
780
         images with transparent backgrounds, and labeled plausible/implausible locations for placing the
781      objects. Since our focus is on object location modeling rather than the insertion of object images,
782      we ignore the object images and restructure the dataset to include pairs of background images and
783      their corresponding object locations. This restructuring yields 1,496 training samples and 184 test
784      samples. While the number of images is relatively small, each sample contains around 40 annota-
785      tions, making it a richly annotated dataset. In total, the OPA train set contains 21,376 positive labels
786      and 40,698 negative labels.
787      Despite having an average 40 annotations for each sample, this accounts for fewer than 1% of the
788      typical number of anchor boxes used in object detectors, leaving most locations unlabeled. Further-
789      more, as illustrated in Figure 11, not only are the number of bounding boxes across samples highly
790      imbalanced, but the distribution of positive and negative labels for each sample is extremely incon-
791      sistent. This sparsity and imbalance make it extremely challenging to train discriminative models
792      that classify locations as plausible or implausible. Our generative approach bypasses this issue by
793      modeling the distribution of plausible locations, using negative labels only for DPO.

      D     A RCHITECTURE AND I MPLEMENTATION D ETAILS
885
886   D.1   A RCHITECTURE OF THE L OCATION M ODEL
887
888
      Our location model is based on a GPT-2 small architecture (Radford et al., 2019), consisting of 12
      layers of Transformer blocks (Vaswani et al., 2017). For image and object class encoding, we use
889
      a pre-trained ViT-B model (Wightman, 2019; Dosovitskiy, 2021) for the image encoder and a ViT-
890
      B CLIP text encoder (Ilharco et al., 2021; Radford et al., 2021) for the object class. The images
891   are converted into 196 embeddings, while the text is transformed into a single embedding, creating
892   a sequence length of 197 when combined. These 197 embeddings are prepended to our location
893   model before predicting the coordinates. To quantize the coordinates, we use 512 bins for both
894   height and width, resulting in a vocabulary size of 514, including the start-of-sequence (SOS) and
895   end-of-sequence (EOS) tokens.
896
      The location model, including the ViT-B backbone, comprises a total of 411 million parameters,
897   and loading the weights in float16 precision requires just 2.05 GB of VRAM. Running the model
898   with batch size 1 in float16 precision requires an additional 1.93 GB of VRAM for the activations,
899   meaning the model can easily be run on consumer-grade hardware. On a Nvidia Tesla V100 GPU,
900   inference runs in 0.03 seconds. For editing images, we first sample locations, unload the model
901   from the GPU, and perform inpainting, ensuring no additional memory overhead beyond what the
902   inpainting model requires.
903   In comparison, the PowerPaint inpainting model has three components: a VAE with 83.7 million
904   parameters, a text encoder with 123.1 million parameters, and a UNet with 859.5 million parameters.
905   Although the location model is nearly half the size of the UNet in terms of parameters, it is used
906   only once in advance (similar to the VAE and text encoder), whereas the UNet is used at every
907   reverse step. On the same GPU, the it takes 7.10 seconds to perform 50 reverse steps, meaning
908   the inference overhead of the location model is 0.4%, an acceptable cost for the improvement in
909   generation quality.
910
911   D.2   T RAINING D ETAILS
912
913   We train the model using the Adam optimizer (Kingma, 2015) for training the location model and
914   Stochastic Gradient Descent (SGD) for DPO training. The model is trained on the PIPE dataset for
915   30,000 iterations with a learning rate of 1e-4, incorporating a linear warmup over the first 1,000
916   steps. Subsequently, we fine-tune the model on the OPA dataset for 3,600 steps and perform DPO
917   for an additional 4,600 steps. For batch size 64, the model can be trained on a single Nvidia V100
      GPU.

      An essential aspect of location models is their ability to identify diverse plausible locations, as
      multiple potential placements often exist for a given object. To quantify this diversity, we apply
949
      Non-maximum Suppression (NMS) to the predicted locations and count the remaining bounding
950
      boxes. Specifically, we perform a standard NMS with a threshold of 0.7 on a set of 100 predicted
951   locations for the same object in the same scene. We observe that 82.5 bounding boxes remain after
952   performing NMS, which highlights the diversity of our location model.

957   We use four baselines in our user study: three instruction-finetuned models, and the strongest loca-

959   Participants were presented with the original image, an instruction (“Add a {object class}”), and

961   model or location modeling baseline. For each image, we randomly swap which method is shown
962   on the left or right side. Each participant was asked to evaluate which of the two edited images
963   better adhered to the editing instruction and maintained the overall coherence of the scene. In total,
964
