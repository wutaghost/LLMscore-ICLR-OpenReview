009                                                A BSTRACT
010
011
               Inference in large language models (LLMs) is often slow due to their autoregres-
012            sive nature. In this work, we formulate a semi-autoregressive decoding paradigm
013            for LLMs that delegates part of the expensive computation from the original large
014            model to a smaller, more efficient autoregressive model. The core of our design
015            lies in the separate modeling of token dependencies, where the large model han-
016            dles long-term dependencies on distant tokens, while the smaller model addresses
017            short-term dependencies on recent tokens. When employed as a draft model in
018            speculative decoding, our method allows for substantial reuse of computation in
019            the LLM without missing any token dependencies, thereby striking a good balance
020
               between draft quality and drafting speed. Experiments on text summarization,
               medical QA, code generation, and mathematical reasoning tasks demonstrates the
021
               efficacy of our method.

025   1   I NTRODUCTION
026
027   Large Language Models (LLMs) have demonstrated remarkable capabilities across diverse natural
028   language processing tasks (OpenAI, 2024; Dubey et al., 2024). However, their deployment in real-
029   world applications is often hindered by the substantial inference latency. The autoregressive nature
030   of LLMs exacerbates this issue, as generating n tokens requires n sequential passes through the
031   model, each of which involves expensive computation of stacked attention layers (Vaswani, 2017).
032   To address the latency challenges, several approaches (Ding et al., 2024; Jiang et al., 2024; Leviathan
033   et al., 2023) have been introduced to improve LLM inference efficiency by delegating parts of the
034   computation to smaller models. Among these innovations, speculative decoding (Leviathan et al.,
035   2023) has emerged as a particularly promising technique. This method leverages the observation that
036   certain tokens within the same inference run are easier to predict and can be handled by a smaller
037   model. Speculative decoding employs a smaller draft model to generate draft tokens, which are then
038   verified and refined by the larger target model. Compared to other approaches, speculative decoding
039
      offers a distinct advantage by guaranteeing generation quality, as it always falls back to the target
      model if necessary. Since its introduction, speculative decoding has proven effective across a wide
040
      range of generation tasks and has become a widely adopted tool for efficient LLM inference.
041
042   The acceleration achieved by speculative decoding, however, hinges on two factors: (i) the accep-
043   tance rate of the draft tokens, as well as (ii) the latency of the draft model itself. As such, a key in-
044   gredient in speculative decoding has been the design of the draft model. Existing work has explored
045
      several designs of the draft model, including the use of a separate transformer model (Leviathan
      et al., 2023; Chen et al., 2023; Kim et al., 2024), training additional modules to predict multiple
046
      tokens simultaneously (Cai et al., 2024; Luk et al., 2024; Bhendawade et al., 2024), or leveraging
047
      selected layers of the target model itself as the draft model (Elhoushi et al., 2024; Bae et al., 2023;
048   Zhang et al., 2024b). Despite reported successes, these designs often involve a trade-off between
049   acceptance rate and latency: some designs achieve a high acceptance rate but with compromises on
050   latency (Elhoushi et al., 2024; Bae et al., 2023; Zhang et al., 2024b; Leviathan et al., 2023), whereas
051   others may offer low latency but at the expense of draft quality (Cai et al., 2024; Luk et al., 2024;
052   Bhendawade et al., 2024; Fu et al., 2024; Gloeckle et al., 2024). The reason for this trade-off stems
053   from the fact that proposing good draft tokens requires the draft model to be complex enough to
      competently process token dependencies, which in turn leads to considerable drafting latency.

064   In this work, we propose a new draft model design that achieves both a high acceptance rate as well
065   as low inference latency. We begin by introducing a unified probabilistic modeling framework to
066   analyze the trade-off between the two metrics both conceptually and empirically — a key piece that
067   is missing in existing literature. Through this analysis, we identify that the optimal trade-off hinges
068
      on the efficient and effective modeling of the dependencies over recent and distant tokens. Moti-
      vated by this insight, we develop a new draft design, semi-autoregressive drafting, which separately
069
      addresses the dependencies over distant and recent tokens of different-sized models. This approach
070
      effectively balances draft quality and latency, resulting in considerable acceleration. In summary,
071   our main contributions are as follows.
072
073   • We propose a unified probabilistic framework for draft model design in speculative decoding,
074     where we systematically compare their trade-off between acceptance rate and inference latency;
075
      • Building upon this framework, we develop a new draft model that works in a semi-autoregressive
076
        fashion, serving as a high-quality yet computationally cheap draft model for speculative decoding.
077     A systematic study in the trade-offs in different implementations of this model is also conducted;
078
      • Focusing on downstream LLM applications (realized via fine-tuning after model pretraining), we
079
        validate our method across four text generation tasks with three models in two training modes,
080
        demonstrating its advantages in terms of acceleration, memory cost, and training convenience.

085   2.1   L ARGE LANGUAGE MODEL
086
087
      A large language model assigns a probability p(w) to a sequence of words w = (w1 , ..., wT ). This
      joint probability is usually factorized using the chain rule:
088
089                                               T
                                                  Y
090                                      p(w) =         p(wi |w1 , ..., wi−1 )                           (1)
091                                               i=1
092
      which reduces language modeling to the problem of estimating the conditional probability of the
093   next word given the history of all preceding words, hence autoregressive. This autoregressive prop-
094   erty imposes a sequential constraint on the decoding process, requiring n sequential passes through
095   the model to generate n tokens, which limits the speed of generation.
096
097   2.2   S PECULATIVE DECODING
098
099   To accelerate inference in autoregressive language models, speculative decoding (Leviathan et al.,
100   2023) leverages a more efficient approximation model, i.e. the draft model Mq , to generate K draft
101   tokens {d1 , ..., dK } from an approximate distribution q(w) ≈ p(w). The larger target model Mp is
102   then used to validate or override the draft tokens.
103   Specifically, a draft token wi ∼ q(wi |w<i ) is accepted if q provides a sufficiently close approx-
104   imation to the target distribution p. In this work, we enforce a greedy acceptance criterion: the
105   draft token is accepted if wi = arg maxv p(v|w<i ). Otherwise, the draft token is rejected, and the
106   larger model overrides the draft by generating a new token from wi ∼ p(wi |w<i ). If a draft token
107   is rejected, all subsequent draft tokens are also discarded, and the generation process reverts to the
      newest token generated by Mp .

122
123   In this section, we present a unified view incorporating various draft model designs in speculative
124   decoding as special cases, focusing on the different considerations for token dependency modeling.

126   represent the hidden states of the last layer of the target model Mp and the draft model Mq computed
127   for the input tokens, respectively.

135   To improve decoding efficiency, we usually consider three desirable properties when designing Mq :
136   (i) high acceptance rate: distribution q should well approximate the target distribution p of the
137   LLM; and (ii) fast drafting: sampling from q should be faster than sampling from p; (iii) q is a
138   low-dimensional distribution, meaning that K is small. The last property is important in our design.
139   One way to meet the aforementioned requirements is to focus on a powerful yet computationally
140   cheap mechanism for modeling dependencies among the tokens d<j , w≤i . Specifically, there are
141   two distinct types of dependencies: (i) the dependencies on distant tokens w≤i , i.e., the tokens
142   already accepted and verified by p; and (ii) the dependencies on recent tokens d<j , i.e., the draft
143   tokens sampled from q. In fact, many previous works can be categorized by how they handle these
144   dependencies. For example:
145
146   • Block autoregressive model (Cai et al., 2024; Luk et al., 2024; Gloeckle et al., 2024) focuses on
147
        maximizing fast drafting by ignoring the dependencies of recent tokens and parallelizing draft
        token inference. In this case, the draft tokens dk and dl are conditionally independent given w≤i ,
148
        i.e., dk ⊥ dl |w≤i (Fig. 2a), leading to:
149
150                                        qj (dj |d<j , w≤i ) = qj (dj |hp (w≤i ))                                  (3)
151     where each qj (·) is a different LM head implemented as an independent multilayer perceptron
152     (MLP) and hp (·) represents the hidden states of the last layer of the target model.
153   • Pruned autoregressive model (Elhoushi et al., 2024; Zhang et al., 2024b) aims to achieve a balance
154     between fast drafting and high acceptance rate. Unlike the block autoregressive model, it proposes
155     to model all token dependencies with a pruned version of the target model Mp , denoted as Mp′
156     (Fig. 2b):
157                                 qj (dj |d<j , w≤i ) = q0 (dj |hp′ (d<j , w≤i ))                      (4)
158     where q0 (·) is the LM head for the draft model and hp′ (·) is the hidden representation of a ‘trun-
159     cated’ LLM, which can be either the first few attention layers of the target model Mp (Elhoushi
160     et al., 2024) or replacement of full attention with approximate attention (You et al., 2024). This
161     pruned model can capture token dependencies and can be directly accelerated on commodity hard-
        ware depending on the type of pruning scheme employed.

      Summary. The above two dependency modeling designs differ in terms of (a) acceptance rate and
163   (b) drafting speed. In particular, block autoregressive model ignores the dependencies among recent
164   tokens d<j , which enables the reuse of hidden states hp to achieve highly efficient parallel decoding.
165   However, this simplification harms the acceptance rate. In contrast, pruned autoregressive model
166   captures all token dependencies through a pruned LLM, aiming to balance fast sampling (through
167   pruning) and high draft quality (through dependency modeling). However, its accuracy is highly
168   dependent on the pruned model. Furthermore, it processes both recent and distant tokens together
169   without distinguishing them, leading to a sub-optimal design. To address these issues, we propose
170   a new design based on the unified view of the draft model. This approach improves the balance
171
      between drafting speed and accuracy by separately modeling the two types of token dependencies.

175   4.1       H IGH - LEVEL DESIGN
176
177   The core intuition of our proposal is to combine the advantages of the two previous models, i.e.,
178
      maximizing hidden state re-use while effectively modeling token dependencies. Specifically, we
      propose to model distant token and recent token dependencies separately, using the target model Mp
179
180
      for distant tokens, and a tiny language model1 (TLM) Mq for recent tokens.
181
      • Distant token dependency. These tokens determine the context and the semantics for drafting sub-
182     sequent tokens. For these tokens, we process them with the original LLM Mp , where we reuse the
183     hidden state hp (w1 , ...wi ) computed by the LLM. This hidden state represents the LLM’s under-
184     standing of the text context. Similar to the block autoregressive model, this context representation
185     only needs to be computed once and is reused throughout the drafting process.
186   • Recent token dependency. These tokens correspond to the K-gram generated by the draft model.
187     To reduce the cost, we use a small model to compute the hidden states hq (d1 , ..., dj−1 ). These
188     hidden states encode the local status of the draft. Since K is relatively small, a very simple model
189     for Mq suffices, which we refer to as a TLM.
190
191   By combining these two design choices, we can model recent token dependencies, as opposed to
192   the block autoregressive model, while incurring small or negligible additional computation cost
193   compared to the pruned autoregressive model, through the use of a TLM. This approach unifies the
194
      strengths of LLMs and TLMs for draft generation. While LLMs are powerful in understanding text
      semantics, they are known to be expensive to run. We thereby use them sparingly, calling them only
195
      when the context of the text is likely to have changed. In contrast, TLMs are much cheaper to run,
196
      though they struggle to parse complex semantics. We thereby only use them to handle short phrases
197   and K-grams, conditioned on the semantic understanding provided by the LLM. In this way, we
198   achieve both high efficiency and good draft quality.
199

      5    R ELATED WORKS
285
286
      Efficient inference schemes in LLMs (Bai et al., 2024; Xu et al., 2024; Zhou et al., 2024) have
287
      received significant attention recently due to the ever-increasing size of such models. Approaches to
288
      improving efficiency range from static methods, such as pruning (Men et al., 2024) and quantization
289   (Ashkboos et al., 2024), which reduce overall model size and hence computational requirements, to
290   dynamic methods, such as early-exiting (Schuster et al., 2022; Bae et al., 2023) and hybrid models
291   (Kag et al., 2022; Ding et al., 2022; 2024), which accelerate inference by adjusting the amount of
292   computation based on the (estimated) difficulty of a given token or prompt. Our work is complemen-
293   tary to these general efficiency efforts (e.g., our semi-autoregressive scheme can be easily applied
294   on top a quantized model). Note, however, that unlike our work and other speculative decoding
295   frameworks, such classic approaches cannot guarantee that LLM’s outputs remain identical.
296   Non-autoregressive decoding (NAR) accelerates inference by eliminating or relaxing the sequen-
297   tial dependencies between tokens (Gu et al., 2018). However, NAR often suffers from reduced
298   accuracy compared to its autoregressive counterparts. Current efforts to improve performance fo-
299   cus on reintroducing some degree of conditional dependence between tokens, for instance, through
300   generative flows (Ma et al., 2019) and conditional random fields (Sun et al., 2019). Additional re-
301   finements include iterative decoding strategies (Lee et al., 2018; Ghazvininejad et al., 2019), as well
302   as improvements to training data and loss functions (Ding et al., 2021; Du et al., 2021).
303   Speculative Decoding (Leviathan et al., 2023) is a widely used framework for accelerating LLM
304   inference. It employs a (smaller) draft model to propose multiple tokens at once, which are then
305   verified by a (larger) target model. Initially, a separate transformer-based language model was used
306   as the draft model (Leviathan et al., 2023; Chen et al., 2023; Kim et al., 2024). More recent work has
307   shifted toward using (part of) the target model itself for drafting (i.e., self-speculative decoding). For
308   example, Medusa Decoding (Cai et al., 2024; Gloeckle et al., 2024) combines non-autoregressive
309
      (NAR) techniques with speculative decoding by training multiple LM heads conditioned on the
      target model’s final layer to generate draft tokens in parallel. Techniques like LayerSkip (Elhoushi
310
      et al., 2024) and FREE (Bae et al., 2023) integrate early-exit strategies by drafting with earlier layers
311
      and verifying with later layers, while Zhang et al. (2024b) adaptively skips intermediate layers for
312   drafting. Recent works have explored the use of more light-weighted model as draft models, such
313   as a n-gram model (Fu et al., 2024) and a small RNN (Zhang et al., 2024a), as discussed below.
314
315
      Light-weighted draft model design. Like our work, concurrent works have also explored taking
      the draft model as a small autoregressive model conditioned on the hidden states of the original
316
      LLM (Ankner et al., 2024; Zhang et al., 2024a; Li et al., 2024; Nair et al., 2024). Among these
317
      works, Zhang et al. (2024a) is similar to one of the implementations in our design. Apart from
318   differences in practical implementation details, the major differences to these works are (a) unlike
319   these works which focus on specific implementations, our works focuses on high-level design, which
320   naturally connects different implementations. The disentanglement of specific implementations and
321   high-level design allows us to explore different architecture choices with various trade-offs in draft-
322   ing efficiency and draft quality; (b) Unlike existing works which primarily focus on learning the
323   draft model on pre-trained dataset e.g. ShareGPT (2023), our work focus on fine-tuning scenarios,
      where the draft model is trained on specific domain data jointly or separately with the original LLM.

528   7       C ONCLUSION
529
530   In this work, we proposed a unified probabilistic framework for token dependency modeling and
531   classified existing literature based on how they attempted to model the recent and distant token de-
532
      pendencies. We then analyzed the trade-off regarding drafting speed and acceptance rate explored
      in prior works. Based on these insights, we proposed an improved semi-autoregressive draft model,
533
      that processes the distant and recent tokens by the original LLM and TLM respectively for fast
534
      drafting while retaining a high acceptance rate. We proposed two variants of our scheme based on
535   (i) a simplified transformer, and (ii) an LSTM. We validated the design of our draft model via exper-
536   iments on four distinct applications, realized via model finetuning, where our model outperformed
537   other competing methods in 3 out of 4 settings, and performed on par in the structured prediction
538   task where the modeling of recent token dependencies carry negligible value. We further analyzed
539   the reasons for these improvements, and highlighted several interesting avenues for future work.

      In section 6.1, we compare several implementations of the network hq for processing draft tokens
759   d1 , ...dj in the proposed semi-autoregressive draft model. One implementation we considered is a
760   transformer with varying number of layers. We discuss here how this implementation is related to
761   wo state-of-the-art draft model designs: Hydra (Ankner et al., 2024) and EAGLE (Li et al., 2024).
762
      Connection to Hydra (Ankner et al., 2024). Given previously accepted tokens w≤i and previous
763
      draft tokens d≤j = {d1 , ...dj }, Hydra predicts the next draft token by jointly sending the embed-
764
      dings of all current draft tokens {e(d1 ), ...e(dj )} and the hidden states hp (w≤i ) from the original
765   LLM to a LM head (with a total of K LM heads). The mentioned implementation of hq as a standard
766   transformer with zero decoding layers can be viewed as a simplified version of Hydra, where the
767   model only uses the embedding of the most recent draft token e(dj ) and the hidden states hp (w≤i )
768   from the original LLM in prediction, ignoring e(d1 ), ...e(dj−1 ). This setup, while being slightly
769   less flexible due to the lost in draft token dependence, is more memory efficient as it only requires
770   storing one LM head instead of K different LM heads.
771   Connection to EAGLE (Li et al., 2024). Given previously accepted tokens w≤i and previous draft
772   tokens d≤j = {d1 , ...dj }, EAGLE predict the next draft token dj+1 by using a transformer decoder
773   layer which takes inputs as both the hidden states of the current sentence and also the draft tokens.
774   The output of this decoding layer, combined with the original hidden states, is then passed to an
775   LM head to predict the next draft token. When we implement hq as a standard transformer with
776   one decoding layer, it can be seen as a simplified and more streamlined version of EAGLE, where
777   attention is only applied among the draft tokens.
