009                                                A BSTRACT
010
011              The scaling of neural networks with increasing data and model sizes necessitates
012              the development of more efﬁcient deep learning algorithms. A signiﬁcant chal-
013              lenge in neural network training is the memory footprint associated with activa-
014              tion tensors, particularly in pointwise nonlinearity layers that traditionally save
015              the entire input tensor for the backward pass, leading to substantial memory con-
016              sumption. In this paper, we propose a modiﬁcation to the handling of activa-
017              tion tensors in pointwise nonlinearity layers. Our method involves saving the
018              output tensor instead of the input tensor during the forward pass. Since the sub-
019
                 sequent layer typically also saves its input tensor, this approach reduces the to-
                 tal memory required by storing only one tensor between layers instead of two.
020
                 This optimization is especially beneﬁcial for transformer-based architectures like
021
                 GPT, BERT, Mistral, and Llama. To enable this approach, we utilize the inverse
022              function of the nonlinearity during the backward pass. As the inverse cannot be
023              computed analytically for most nonlinearities, we construct accurate approxima-
024              tions using simpler functions. Experimental results demonstrate that our method
025              signiﬁcantly reduces memory usage without affecting training accuracy or com-
026              putational performance. Our implementation is provided as a drop-in replace-
027              ment for standard nonlinearity layers in the PyTorch framework, facilitating easy
028              adoption without requiring architectural modiﬁcations. The code is available at
029              https://github.com/removed/for/anonimity.

064

065
               performance remains unaffected.

078   systems.

           2.1   A LTERNATIVES TO B IT-C OMPRESSED B OOLEAN I NDICATOR
224
225
           An essential component for inverting nonlinearities such as GELU or SiLU is an indicator function
226
           that identiﬁes which monotonic part the original x belongs to. Currently, we store this information
227        in a space-efﬁcient, bit-compressed Boolean tensor, where each Boolean value occupies only 1 bit
228        of memory. In this section, we will discuss two alternative methods for storing this information
229        without requiring even 1 bit per element.
230
           Sign-bit Inverted Activation. Both GELU and SiLU functions are bounded, which means there
231
           exists some constant C (see Fig. 1) such that for all x, f (x) ≥ C, where f represents either the
232
           GELU or SiLU function. This implies that the value f (x) − C is always greater than zero. Instead
233        of storing f (x) for the backward pass, we can store f (x) − C with its sign bit modiﬁed according
234        to the indicator function s (see Eq. (4)).
235
236
           For most popular ﬂoating-point formats used in deep learning, such as ﬂoat32, ﬂoat16, and bﬂoat16,
           the sign bit is the last bit of a ﬂoating-point number. This approach allows us to eliminate the need
237
           for additional memory to store the 1 bit per element for the indicator function, making the entire
238
           nonlinearity layer memory-free in terms of footprint. The modiﬁed value y = f (x) does not pose
239        an issue for subsequent computations, as C is a predeﬁned global constant for a given activation
240        function. For example, the computation for an activation function followed by a fully-connected
241        linear layer can be expressed as follows:
242
243    1 N # number of bits in floating point data type
244    2 C # minimum of activation function f
       3 def forward(X, W_linear):
245
       4   S = X < T
246    5   Y = f(X) - C
247    6   Y = (Y & (~(1 << N))) | (S << N) # sets sign bit of tensor Y
248    7                                    # to indicator function S
249    8   save_for_backward(Y)
       9
250
      10     return (Y + C) @ W_linear
251   11
252   12 def backward(dY, W_linear):
253   13   Y = saved_for_backward()
254   14   S = Y & (1 << N)# extracts indicator function from last bit of tensor Y
      15   Y = Y ^ S        # reverts sign bit of tensor Y back to zero
255
      16   dW = (Y + C).T @ dY
256
      17   dX = dY * f‘(f−1(Y + C, S))
257   18   return dY, dW
258

260
           Here, the forward pass takes the input tensor X and the matrix Wlinear of the fully-connected linear
           layer and returns f (X) · Wlinear .
261
262        Unfortunately, to the best of our knowledge, it is not possible to implement such a layer in PyTorch
263        that works independently of what comes after the nonlinearity function, meaning that you would
264        have to write a custom implementation for each new architectural block. As a result, the sign-bit
265        approach does not satisfy condition 3 outlined in the Introduction (see Section 1), and thus we do
266
           not consider it a viable candidate for broad adoption.
267        Precision Bit Inverted Activation. The second approach to store the indicator function s (see
268        Eq. (4)) is to use the lowest precision bit of a ﬂoating-point number (for most popular ﬂoating-point
269        data types, this is the ﬁrst bit in a base-2 representation). During the forward pass, we compute
           y = f (x) and modify this bit of y to be equal to s (see Eq. (4)).


                                                             5
270
           Compared to the previous approach, this method does not violate condition 3, as it allows for a conve-
271        nient drop-in replacement for classical activation functions on all modern deep learning frameworks.
272        However, it is more challenging to evaluate the accuracy of this approach. Both the standard inverted
273        activation and the sign-bit inverted activation perform the forward pass exactly and introduce only a
274        minimal approximation error during the backward pass. This method, on the other hand, sacriﬁces
275        one precision bit for the pointwise nonlinearity computation during the forward pass. Moreover,
276        with the ongoing trend of reducing ﬂoating-point precision, loss of accuracy may become larger in
277        the future. For this reason, we do not consider it a viable candidate for broad adoption at this time,
278        as it requires extensive experimentation to validate its accuracy. However, we still regard it as an
279
           interesting approach that may ﬁnd applications in the future.

281    1 def forward(X):
282    2     S = X < T
283    3     Y = f(X)
284    4     Y = (Y & (~1)) | S
       5     save_for_backward(Y)

      The efﬁcient scaling of neural, has necessitated innovations in low-precision training, quantization
      techniques, and various compression methods. These approaches address computational and mem-
460
      ory bottlenecks, enabling the training of ever-larger models on hardware-limited resources.
461
462   Quantization methods aim to compress neural network weights and activations by representing
463   them in lower-precision formats, such as 4-bit integers. One can quantize model weights Jacob et al.
464   (2018); Zafrir et al. (2019); Gholami et al. (2021) or activations that are saved for backward Chen
465
      et al. (2021); Novikov et al. (2023). Former are used more in during inference, but approaches
      to utilize it during training are also exist Liu et al. (2024); Shao et al. (2024); Xu et al. (2024).
466
      These methods are especially popular for the ﬁne-tuning stage of training, where reducing memory
467
      usage without extensive retraining is crucial. While quantization effectively decreases the memory
468   footprint, it can introduce computational inaccuracies. As a result, these errors lead to degradation
469   in model performance, making the balance between memory savings and maintaining model quality
470   a key consideration when applying quantization.
471
      Low-precision training techniques have emerged as another powerful approach to reducing both
472
      the memory and computational requirements of large-scale models Micikevicius et al. (2018). One
473
      widely adopted format is bﬂoat16 Kalamkar et al. (2019), which retains a wider range of values
474   compared to traditional FP16 while providing faster computation and reduced memory consump-
475   tion. More recently work towards even lower precision training in ﬂoat8 formats have also been
476   explored Wang et al. (2018) for their potential to further reduce memory and computational de-
477   mands. These formats are particularly attractive for their extreme compression capability, though
478   their usage requires careful consideration to avoid signiﬁcant accuracy loss. Moreover, widespread
479   adoption of these lower precision formats relies heavily on GPU manufacturers integrating support
480   for them directly into hardware. Until this support is in place, these formats remain experimental
481   and difﬁcult to implement at scale.
482   Custom CUDA and Triton implementations Unsloth (2023); Hsu et al. (2024) have become
483   essential for optimizing critical building blocks in popular architectures, signiﬁcantly improving
484   memory efﬁciency and computational speed. For instance, highly fused implementations of MLP
485   blocks Müller et al. (2021) or optimized GeGLU layers are widely used to reduce the overhead of
      separate operations by merging them into a single, streamlined kernel. Another prominent example


                                                                                           9
486
      is FlashAttention Dao (2024), which signiﬁcantly improves the efﬁciency of attention mechanisms
487   by reducing memory bottlenecks while maintaining performance. These custom kernels, often de-
488   veloped in CUDA or Triton, enable ﬁne-grained optimizations tailored to speciﬁc hardware architec-
489   tures, providing substantial performance improvements. However, the development of such highly
490   optimized components demands signiﬁcant engineering effort and a deep understanding of both the
491   underlying hardware and neural network operations. Each optimization, whether in matrix multipli-
492   cations, element-wise operations, or memory management, requires meticulous custom work, and
493   the resulting implementations are typically neural netowrk architecture-speciﬁc. Despite the com-
494   plexity, continuous innovation in these low-level building blocks is critical, as it provides massive
495
      improvements in memory usage and training speed, making it possible to push the boundaries of
      model scaling and performance.

498   5   L IMITATIONS
499
500   It is important to note that inverted nonlinearities save memory only when the next layer after the
501   nonlinearity also saves its input for the backward pass. Fortunately, this behavior is common in most
502   popular neural network architectures, such as transformers.
503   Additionally, our method reduces memory usage only for nonlinearity layers. The percentage of
504   memory savings depends on the architecture in use. Furthermore, the presence of complemen-
505   tary techniques may impact these gains. For instance, when training models with checkpoint-
506   ing Griewank & Walther (2000), the memory bottleneck could shift between activations saved dur-
507   ing the forward pass and activations recomputed during the backward pass, potentially affecting the
508   beneﬁts of our approach.
509   Despite these limitations, our highly efﬁcient implementation of inverted activations within the Py-
510   Torch framework using Triton kernels makes us conﬁdent that our method will see broad adoption.
511   We believe that it will prove invaluable to deep learning practitioners who seek to reduce memory
512   usage without compromising speed order and model accuracy.

515
516
      In this work, we presented an innovative method to reduce the memory footprint during neural
517   network training by modifying how activation tensors are saved for backward passes in pointwise
518   nonlinearity layers. By saving the output tensor instead of the input tensor, we signiﬁcantly de-
519   creased the memory requirements for transformer-based architectures such as GPT, BERT, Mistral,
520   and Llama.
521
      Our proposed method includes the development of a space-efﬁcient Boolean indicator to handle the
522
      non-invertible nature of these functions, enabling a practical implementation within the PyTorch
523   framework. Additionally, we introduced accurate approximations for the derivatives of the inverse
524   functions of GELU and SiLU, ensuring that training performance remains unaffected while achiev-
525   ing substantial memory savings.
526
      The experiments and approximations demonstrated the feasibility and effectiveness of our approach,
527
      with results showing that the training process using our modiﬁed nonlinearity layers is indistinguish-
528
      able from using the original layers in terms of accuracy.
529
530   We set out to meet three key criteria for a successful replacement of standard nonlinearity layers:
531   maintaining model speed, preserving model accuracy, and ensuring ease of use. Our proposed
532
      method satisﬁes all three conditions. It does not slow down the neural network, it does not degrade
      model quality, and it is extremely easy to integrate, making memory savings essentially "free."
533
534   We have implemented this method for the PyTorch framework, and the code is available at https:
