009                                               A BSTRACT
010
011            The combination of transformers and masked image modeling (MIM) pre-training
012            framework has shown remarkable potential in various vision tasks. However, the
013            high computational cost of pre-training hinders the practical application of MIM.
014            This paper introduces FastMIM, a simple and versatile framework that expedites
015            masked image modeling through two steps: (i) pre-training vision backbones us-
016
               ing low-resolution input images and (ii) reconstructing Histograms of Oriented
               Gradients (HOG) feature instead of original RGB values of the input images. Fur-
017
               thermore, we propose FastMIM-P, which progressively increases the input reso-
018
               lution during the pre-training stage to improve the transfer learning performance
019            of models with high capacity. We point out that: (i) a wide range of input reso-
020            lutions during pre-training can result in similar performances in fine-tuning and
021            downstream tasks such as detection and segmentation; (ii) the shallow layers of
022            encoder are more important during pre-training, and discarding the last few layers
023            can speed up the training process without affecting fine-tuning performance; and
024            (iii) HOG is more stable than RGB values when transferring resolution. Equipped
025            with FastMIM, any type of vision backbone can be efficiently pre-trained. For ex-
026            ample, using ViT-B/Swin-B as backbones, we achieve 83.8%/84.1% top-1 accu-
027
               racy on ImageNet-1K. Compared to previous approaches, our method can achieve
               better top-1 accuracy while accelerating the training procedure by ∼5×.

030   1   I NTRODUCTION
031
032   Self-supervised learning is a promising paradigm that aims to learn feature representations from
033   scalable unlabeled data, and has achieved significant results in natural language processing (NLP)
034   through masked language modeling (MLM) (Radford et al., 2018; Devlin et al., 2018; Brown et al.,
035   2020; Chen et al., 2020b). Recently, it has also attracted increasing attention in vision community,
036   where masked image modeling (MIM) has emerged as a self-supervised pre-training framework.
037   Different from previous contrastive learning based approaches (Wu et al., 2018; Chen et al., 2020c;
038   He et al., 2020; Caron et al., 2021), MIM learns representations through a mask-then-predict manner,
039
      e.g., predicting the raw pixels (He et al., 2021; Xie et al., 2022) or other tokenizations (Bao et al.,
      2021; Zhou et al., 2021; Chen et al., 2022b) of randomly masked input images.

043   tasks, the pre-training stage of self-supervised
044   learning-based approaches is extremely com-

      ple, contrastive learning based SimCLR (Chen

077   To reduce the pre-training costs of self-supervised learning and make MIM a more efficient and
078   practicable framework for vision, we devise a simple and straightforward framework (Figure 5),
079   viz, FastMIM, for faster training speed and easier deployment of AI applications. Inspired by Sim-
080   MIM (Xie et al., 2022), which retains all input tokens during pre-training stage, we directly mask
081   the raw RGB input and keep the illuminated input the same as in the supervised learning producer.
082   This presents a fresh opportunity for FastMIM to serve as a generic framework because no modifi-
083
      cation is made to the architecture and the input shape. Yet, standard input images of size 224×224

084
      example, the encoder of ViT-B (Dosovitskiy et al., 2020) needs to tackle 196 input patches in Sim-
085
      MIM (Xie et al., 2022). To alleviate memory consumption, we propose a straightforward approach
086   of reducing the input resolution, e.g., from 224×224 to 128×128, and the number of input patches
087   is reduced to 64 accordingly, as shown in Figure 2 and 5. We further leverage the HOG target (Dalal
088   & Triggs, 2005; Wei et al., 2022) to compensate for the loss of texture information resulting from
089   the reduction of image resolution. Our main contributions can be summarized as:
090   • We investigate various configurations of MIM, identify the key design to expedite the pre-training
091      stage and reduce the memory consumption: directly reducing the input resolution for MIM.
092   • We elaborate the characteristic of the HOG feature, which is almost invariant to the geometric
093      changes in images. Compared with pixel target, reconstructing HOG target can better compensate
094      for the loss of texture information resulting from the reduction of image resolution.
095   • Based on the above observations, we propose FastMIM, which can expedite the overall pre-
096      training speed by 5× and reduce the memory consumption simultaneously. Extensive experiments
097      demonstrate the effectiveness and efficiency of our proposed framework.
098
099   Overall, the heavy memory consumption of previous self-supervised learning frameworks erects an
100   unfortunate barrier for more researchers to dive into this filed. We hope our findings and FastMIM
101
      can provide avenues and insights for making MIM more accessible to the vision community.

254
255   Our FastMIM pre-trains vision backbones through masked image modeling, and is also a reinforced
256
      version of SimMIM (Xie et al., 2022), as illustrated in Figure 5. In principle, it is straightforward
      and convenient to replace the encoder with other vision backbones in aforementioned MIM pre-
257
      training framework. We choose the representative isotropic and hierarchical vision transformers,
258
      i.e., ViT (Dosovitskiy et al., 2020) and Swin Transformer (Liu et al., 2021) as our baselines. We
259   directly mask the input image (e.g., X ∈ R128×128×3 ) with the mask token (e.g., learnable vector
260   [MASK] ∈ R1×1×3 ). The ViT encoder embeds patches by a linear projection added with positional
261   embeddings (PE), while there is no extra PE for the decoder. As for Swin, the window size for
262   Swin-B and Swin-L is set to 7 and 14 following (Xie et al., 2022; Huang et al., 2022), respectively.
263

      4.2      FastMIM-P: P ROGRESSIVELY E NLARGE THE I NPUT
306
307   To further improve the scalability of FastMIM, we propose to progressively enlarge the input res-
308   olution during the pre-training stage, viz, FastMIM-P. Although HOG can preserve the texture in-
309
      formation when reducing the input resolution to some extent, the performance of model with high
      capacity, e.g., Swin-L, still has small gap (-0.3% in Table 3) compared to the counterpart trained with
310
      high-resolution input images. More specifically, in contrast to FastMIM that trains Swin-L in fixed
311
      1282 inputs, FastMIM-P trains Swin-L in 1282 /1602 /1922 for 200/100/100 epochs (400 epochs in
312   total), and achieve better trade-off between accuracy and training time, as shown in the last row of
313   Table 3. Additionally, we report the results of FastMIM-P on ViT-L, trained with input resolutions
314   of 1282 /1602 /1922 for 500/200/100 epochs respectively. Implementation details can be found in the
315   supplementary material.
316
      Discussion. As shown in Table 3, FastMIM-P achieves better performance compared to FastMIM
317
      with less pre-training time. However, as the input resolution continually increases, the GPU memory
318   consumption will inevitably increase. The resolution and training schedule need to be carefully de-
319   signed to achieve a better space-time trade-off. In addition, we find that reducing the input resolution
320   results in slightly more pronounced accuracy degradation as the model scales up from ViT-B/Swin-B
321   to ViT-L/Swin-L. This validates that larger models indeed tend to be more data-hungry, and a smaller
322   number of input tokens may lead to insufficient pretraining. However, by leveraging the progres-
323   sively enlarged input resolution strategy, we effectively mitigate this issue and achieve significant
      improvements in accuracy for larger models.

532
533
      This paper presents a simple yet effective FastMIM to expedite the self-supervised MIM pre-training
534   for various vision backbones. As a generic framework, we directly mask input images, allowing all
535   encoders to be trained in the same way as supervised learning. Besides, simply reducing the image
536   resolution and reconstructing HOG target can train both isotropic and hierarchical architectures 5×
537   faster and save the GPU memory consumption by up to ∼70% compared with previous approaches,
538   while obtaining a comparable performance on classification and other downstream vision tasks. We
539   hope our observations and the simple framework can make MIM more practicable and demolish the
      barrier so that more researchers can dive into this field.

      7   D ISCUSSION ON P OTENTIAL L IMITATIONS OF FAST MIM
541
542
      While our proposed FastMIM approach demonstrates significant improvements in training efficiency
543   and resource utilization, there are inherent limitations to the method. Specifically, while using HOG
544   features effectively compensates for the loss of texture information caused by reduced image reso-
545   lution, HOG’s reduced sensitivity to color and texture variations may make it less suitable for tasks
546   requiring precise appearance modeling. For vision classification and related tasks, where global
547   structure and high-level semantic features are more critical, this approach can yield performance
548   improvements, as demonstrated in our experimental results. However, for tasks that heavily depend
549   on fine-grained visual details, such as image generation or super-resolution, the loss of detailed color
550   information could hinder performance. This limitation underscores the need for caution when apply-
551
      ing FastMIM to such specialized tasks. Future work could explore hybrid approaches that integrate
      HOG with complementary representations to enhance adaptability across a broader range of tasks.

741   A.1    P RELIMINARY OF MIM
742
      Notations. Following commonly used configurations (He et al., 2021; Bao et al., 2021; Xie et al.,
743
      2022), given an input image X ∈ RH×W ×C , where H, W , and C are the height, width, and number
744   of channels, some of the pixels in X are randomly masked out by being replaced with a mask token,
745   denoted as [M]. Let S ∈ {0, 1}H×W ×C denotes the spatial mask, where 0 indicates a pixel1 is
746   invisible for the encoder, and 1 indicates a pixel is visible.
747
      Framework. MIM learns representations by predicting the masked area of an input X. Existing
748
      MIM methods can be roughly classified into two categories: (i) MAE (He et al., 2021; Huang et al.,
749
      2022) discards the masked area and only the visible part is sent to the encoder for latent feature
750   extracting, then the decoder reconstructs the masked part from latent representation and mask token;
751   (ii) SimMIM (Xie et al., 2022; Bao et al., 2021; Wei et al., 2022; Dong et al., 2021; Zhou et al.,
752   2021) retains the masked part, the new input can be formulated as X̂ = X ⊙ S + [M] ⊙ (1 − S),
753   where ⊙ denotes the Hadamard product.
754
         1
755        Here we directly mask on the input RGB images, which is different with previous methods (Xie et al.,
      2022; Bao et al., 2021; Huang et al., 2022) which mask on the image patches (tokens).

786   A.2     A BLATION S TUDY ON MIM
787
      Mask strategy. We analyze two typical mask strategies of the encoder, i.e., MAE (He et al., 2021)
788
      and MIM (Xie et al., 2022). The former only operates on visible patches without [MASK] to-
789
      kens, while the latter operates on the entire image patches. As shown in the top two rows of Ta-
790   ble 10a, MIM achieves slightly better transfer performance compared to MAE, but operating on
791   whole patches results in a heavier computational burden (as demonstrated in Figure 2 of our main
792   paper). Moreover, the third row illustrates that masking on image patches (e.g., 14×14×768 in
793   ViT-B (Dosovitskiy et al., 2020)) has almost the same effect as masking on the original image (e.g.,
794   224×224×3). We further examine the impact of the [MASK]. We study two kinds of mask token,
795   one with a learnable vector and the other set to zeros. We find that filling mask tokens with zeros
796   degrades MAE performance by 0.4%, but has little impact on MIM. One primary reason is that the
797   encoder in MIM can process the [MASK] earlier and more comprehensive than in MAE. In general,
798
      excluding the masked regions will not affect the final fine-tuning (transfer) result, except for the pre-
      training computational cost. Mask size and mask ratio. We study how different mask sizes and
799
      ratios affect the effectiveness of MIM in Table 10b. We observe that both isotropic and hierarchical
800
      architectures achieve their best results only when the mask size is equivalent to the patch size of
801   the last stage of encoder. Notably, when the mask size is smaller than the patch size, MIM can still
802   obtain comparable results, demonstrating its ability for representation learning. With an appropriate
803   mask size, MIM remains stable with ratios varying from 0.5 to 0.85.

845
      Following common practice (He et al., 2021; Xie et al., 2022; Bao et al., 2021), we first conduct
846   self-supervised pre-training on ImageNet-1K (Deng et al., 2009) training set without label, and then
847   validate the proposed FastMIM by conducting end-to-end fine-tuning on downstream tasks including
848   classification, object detection, instance segmentation, and semantic segmentation. All experiments
849   are conducted on 8 V100 GPUs with PyTorch (Paszke et al., 2019).
850
      ViT architecture. We follow the standard ViT architecture (Dosovitskiy et al., 2020). The encoder
851
      ends with an extra Layer Normalization (LN) (Ba et al., 2016). To match the different widths
852   between encoder and decoder, we adopt a linear projection layer after the encoder following (He
853   et al., 2021). Our FastMIM only adds absolute positional embeddings (the sine-cosine version) to
854   the encoder inputs. And we retain the class token (Dosovitskiy et al., 2020) during our pre-training
855   stage.
856
      Swin architecture. We follow the standard Swin-B architecture (Liu et al., 2021). When pre-
857
      training with input images of size 128×128, the window size is set to 4 accordingly. When fine-
858   tuning with input images of size 224×224, the window size is set to 7. And we simply leverage
859   the “bicubic” interpolation to remap “relative position table” (Liu et al., 2021) when pre-trained
860   window size mismatches with fine-tuned window size. As for Swin-L, we set the window size to 14
861   during fine-tuning stage following (Xie et al., 2022; Huang et al., 2022). FastMIM pre-trains Swin-L
862   with 128×128 inputs and the window size is set to 8 accordingly. FastMIM-P gradually increases
863   the input resolution during pre-training stage. We first initialize the window size to 14, and then
      interpolate the corresponding “relative position table” for different input resolutions.

1238   Pre-raining loss of the HOG target. Figure 7 shows the pre-training losses of different input
1239   resolutions. We can find that HOG target can reduce the gap of loss values between different input

       Visualization of ground truth target. Figure 8 and Figure 9 show more visualization results of
1243   pixel and HOG target on ImageNet-1K and COCO, respectively. Although reducing the image res-

1245   and edges, will be discarded when using pixel target. However, HOG is more invariant to the reso-

1249   target with HOG target as shown in Figure 10 and Figure 11. We can find that both pixel and

1251   errors caused by ambiguous problems (Wei et al., 2022), while HOG is more robust to ambiguity.

       as dark blue, which is in fact red in the top area. This wrong prediction can result in a high loss

1254
