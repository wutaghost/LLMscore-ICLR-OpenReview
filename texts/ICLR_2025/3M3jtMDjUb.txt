009                                                A BSTRACT
010
011            There is an ongoing effort to develop feature selection algorithms to improve inter-
012            pretability, reduce computational resources, and minimize overfitting in predictive
013            models. Neural networks stand out as architectures on which to build feature selec-
014            tion methods, and recently, neuron pruning and regrowth have emerged from the
015            sparse neural network literature as promising new tools. We introduce RelChaNet,
016            a novel and lightweight supervised feature selection algorithm that uses neuron
017
               pruning and regrowth in the input layer of a dense neural network. For neuron
               pruning, a gradient sum metric measures the relative change induced in a network
018
               after a feature enters, while neurons are randomly regrown. We also propose an
019
               extension that adapts the size of the input layer at runtime. Extensive experiments
020            on nine different datasets show that our approach generally outperforms the current
021            state-of-the-art methods, and in particular improves the average accuracy by 2% on
022            the MNIST dataset. Our code is available in the supplementary material.

025   1    I NTRODUCTION
026
027   Feature selection is an elemental task in predictive modelling. It can serve to reduce computational
028
      resources, improve interpretability by highlighting important features, or improve predictive perfor-
      mance by reducing overfitting (Li et al., 2018). To further these goals has been the driving motivation
029
      of large recent efforts to improve existing and develop new feature selection algorithms. Feature
030
      selection algorithms can be categorized into embedded, wrapper, and filter approaches. Embedded
031   methods select features during training of a predictive model, such as linear regression (Tibshirani,
032   1996) or neural networks (Lemhadri et al., 2021). Wrapper approaches also work around a specific
033   predictive model, but treat it as a black box with the feature set as a hyperparameter, e.g., via particle
034   swarm optimization (Rostami et al., 2021). Filter approaches select feature sets without being tailored
035   around a predictive model, but using information-theoretic measures. They include, for example,
036   statistical tests of the relationship between the feature and the outcome (Bommert et al., 2020).
037   Neural networks have a great ability to capture nonlinear relationships and offer many entry points
038   for slightly modifying their architecture or training algorithm to build successful embedded feature
039   selection methods. To decide on the utility of an input neuron, approaches added gates in the input
040   layer (Yamada et al., 2020), added residual connections to the output (Lemhadri et al., 2021), or
041   added gradients with respect to data changes to the loss (Cherepanova et al., 2023).
042
      Feature selection in neural networks translates to aiming for a sparse input layer and is therefore
043   a special case of sparse neural networks (Hoefler et al., 2021). Recently, it was shown that sparse
044   neural network training (Mocanu et al., 2018; Evci et al., 2020) can be adapted to achieve a dominant
045   feature selection performance (Liu et al., 2024; Atashgahi et al., 2024; Sokar et al., 2024). However,
046   we have identified potential improvements to enhance the network’s ability to detect important
047   features and make it easier for regrown neurons to compete with established neurons during training.
048
      In this paper, we introduce RelChaNet, a novel neural network feature selection algorithm using
049   relative change scores. It applies neuron pruning and regrowth in the input layer of a dense neural
050   network based on a relative change metric shown in Figure 1. Our main contributions are:
051
052         1. The RelChaNet feature selection algorithm, which has two key hyperparameters that allow
053            it to adapt to the characteristics of the dataset used. It addresses two identified drawbacks
               by giving candidates multiple mini-batches of time to show their potential relevance in the

      2    BACKGROUND AND RELATED WORK
089
090   In this section, we introduce the feature selection problem within the framework of neural networks
091   and review previous solution approaches. Most approaches slightly modify a dense neural network
092   architecture or the loss function. Recently, successful approaches have been taken from the framework
093   of sparse neural networks.
094
095   Feature selection in neural networks. We consider the task of selecting a set of K features that
096   are most valuable for making accurate predictions in a supervised learning setting. Specifically for
097   neural networks, we can express this task using L0 regularization of the first layer network weights.
098   Accordingly, we want to optimize the network under the condition that only K input neurons are
099   active, i.e., have any non-zero adjacent weights. If we consider a neural network with one input
100
      neuron for each feature i ∈ {1, . . . , N }, N > K, we can express the feature selection task as finding
      a specific set of network weights W that fulfills

                                                               (1)
102                              arg min L(W) | #{i | ||Wi. ||1 > 0} = K                                    (1)

104   where L(W) represents evaluating the loss function L using the data and network weights, and Wi.
105   is the vector of outgoing first layer weights from input neuron i. The key challenge in solving this
106   task is to implement an effective L0 regularization. Exact solutions are computationally prohibitive
107   and become intractable in high-dimensional settings (Yamada et al., 2020). Consequently, the related
      work discussed below uses various approximations to address this challenge.

108
      Dense neural networks. There are several methods embedded in dense neural networks for feature
109   selection. A common property is that the number of active neurons is not strictly enforced before
110   model convergence. Instead, selection is gradual, starting with a full input layer of N neurons
111   and reducing active neurons during training. This approach makes it easier to identify complex
112   interactions between features, at the cost of increased computational complexity. Stochastic gates
113   (Yamada et al., 2020) approach the L0 regularization by adding a gate to each input layer neuron.
114   For each gate, a trainable parameter controls the probability of a feature being active. The LassoNet
115   (Lemhadri et al., 2021) adds a residual connection from each input layer neuron to the network output.
116   The absolute sizes of these N residual weights are added to the loss function and for each feature i
                                                                                                (1)
117   individually represent a bound on the size of the corresponding first layer weights, ||Wi. ||1 . A less
118   invasive approach is DeepLasso (Cherepanova et al., 2023), which adds the gradient with respect to
119   changes in the input data to the loss function. This encourages the network not to use some features
120
      during training, rendering the corresponding input neuron inactive.
121
122   Sparse neural networks. Sparse neural networks keep a large fraction of the weights throughout
123   the network at 0 to reduce memory requirements or training time (Hoefler et al., 2021). One method
124
      to achieve this is structured sparsity, such as neuron pruning, where all of a neuron’s outgoing weights
      are set to 0. Metrics for deciding which neurons to prune include the magnitude of the outgoing
125
      weights or the sensitivity of the output to the neuron. Neurons can also be periodically regrown,
126
      based on criteria such as the size of gradients or adjacent weights. Molchanov et al. (2019) propose
127   a neuron/filter pruning method that calculates a score across mini-batches, similar to our approach.
128   However, their method is not specific to the input layer, calculates the product of weight and gradient,
129   and does not involve regrowing neurons or reusing a score later in training. GradEnFS (Liu et al.,
130   2024) uses sparse neural networks for feature selection. Similar to DeepLasso, it measures the
131   importance of neurons based on how sensitive the loss is to changes in the input neurons. After the
132   model converges, it selects the top K features based on neuron importance. We see this selection
133   procedure as a disadvantage because no specific sets of K features are assessed during training.
134   Since pruning the input layer reduces the number of active neurons, as required in Equation 1, methods
135   that do so are promising for feature selection. NeuroFS (Atashgahi et al., 2023) extends adaptive
136   sparse neural network training, which utilizes weight pruning (Mocanu et al., 2018; Evci et al., 2020)
137   , by incorporating input layer neuron pruning. Input neurons are pruned after each epoch based on the
                                                      (1)
138   magnitude of their outgoing connections, ||Wi. ||1 . To regrow an input neuron, NeuroFS calculates
139   the absolute gradients of all currently pruned first layer weights. Neurons are then regrown based on
140   the largest absolute gradient among their adjacent weights. During training, the number of active
141   neurons in the input layer is continuously reduced. After training, the input neurons with the largest
142   outgoing connections among the remaining active neurons are selected.
143   We generally observe two drawbacks in gradient-based regrowing and absolute weight-based pruning
144   for feature selection. Firstly, in the regrowing procedure, features need to signal their importance
145   through high adjacent gradients before the network makes any adjustments for them. However, the
146   network might take longer, e.g., multiple mini-batches, to recognize the importance of a feature,
147   especially if it is involved in complex interactions with other features. Secondly, in later training
148   epochs, the absolute weights of regrown neurons are compared to those of longer established neurons.
149   In consequence, features are compared while being given different times to grow their weights. To
150
      mitigate both of these drawbacks, we propose to regrow features randomly and to use a metric of the
      change a feature induces in the network over the first few mini-batches after it enters the network for
151
      pruning.

155
156   We propose the RelChaNet algorithm for supervised feature selection using neural networks.
157   RelChaNet computes a score for each input neuron by aggregating gradients over mini-batches.
158   These scores are normalized and used to update a high score vector, which guides feature selection
159   (see Figure 1 for an illustration). This section walks through the pseudocode in Algorithm 1 and
160   explains its rationale. RelChaNet is implemented using PyTorch (Paszke et al., 2019) and is available
161   as a Python package in the supplementary material.

186   Architecture and initialization. The algorithm uses a multi-layer perceptron (MLP) with a feed-
187
      forward architecture and is integrated into the backpropagation training using the Adam optimizer
      (Goodfellow et al., 2016; Kingma & Ba, 2015). This implies the adoption of the hyperparameters of
188
      learning rate, batch size, and number of hidden layers and their sizes. The size of the input layer is
189
      based on the desired number of selected features K plus a percentage cratio of the remaining features,
190   Kc , which will be referred to as candidates.
191
192   Relative change scores. Steps 5-10 calculate the relative change scores s, where gradient sums for
193   each input neuron are aggregated and normalized to reflect their relative contribution across the last
194   nmb mini-batches (see also Figure 1). Instead of gradient sums, one could also use weight changes as
195   a relative change metric in Steps 5-8, which we compare in an ablation study in Section 4.2.
196
197   Input layer rotation. Steps 11–15 dynamically update the input layer by selecting a combination
198   of top features and new candidates. The relative change scores computed earlier are used to identify
199   the top K features (Step 12), ensuring they remain in the input layer. Additional candidate features
200   are randomly sampled from the remaining features (Step 13). Together, these form the input
201   layer (Step 14). To avoid symmetry issues during training, the weights of candidate features are
202
      reinitialized to small random values (Step 15), following best practices in neural network initialization
      (Goodfellow et al., 2016). This rotation ensures that feature selection is iteratively refined based on
203
      relevance.
204
205
      Key mechanism. Our algorithm approaches the L0 regularization task laid out in Equation 1 by
206
      stabilizing the high score vector h. At the time of each input layer rotation, the network is forced to
207   adhere to the criterion of only K active features, after which it gets to assess additional candidates
208   again for a few mini-batches. The high scores h, since they preserve information over time, allow
209   a comparison of the entry performance of candidates with the entry performance of features that
210   entered epochs ago. Specifically, in later epochs of training, good candidates do not need to surpass
211   the absolute first layer weights of the more established neurons.
212
213   Random Regrowth. Random regrowth offers a key advantage over metric-based methods, such as
214   gradient-based selection, by giving candidates multiple mini-batches to demonstrate their relevance.
215   It facilitates the inclusion of features that do not have a straightforward relationship with the output
      but contribute to complex patterns that only emerge over time. Additionally, by combining random

242   3.1   A DAPTIVE NETWORK SIZES
243
      To address sensitivity to the cratio hyperparameter, we introduce RelChaNet flex, which dynamically
244
      adjusts the input layer size during training based on the behavior of the loss function. It extends
245
      Algorithm 1 between Steps 12 and 13, i.e., prior to selecting new candidate features, and is detailed
246   in pseudocode in Algorithm 2, RelChaNet flex.
247
248   Key mechanism. RelChaNet flex monitors the running loss, l, and compares it with the loss
249   recorded at the time of the last input layer size change, lchange . If the loss stagnates (i.e., does not
250   decrease for a fixed number of rotations), the algorithm adjusts the input layer size. Specifically:
251
252         1. Direction adjustment: If the loss increases compared to lchange , the direction of change
253            (shrink or grow) is reversed
254         2. Size adjustment: Depending on the direction, cratio is halved or doubled, bounded by
255            predefined limits. The upper limit of cratio = 1 represents using the maximum number of
256            candidates, N − K, while the lower limit ensures a minimum input layer size of 56 K.
257
258   Rationale. A well-balanced input layer size allows the network to explore a sufficient pool of
259   candidate features in the presence of random regrowth. Shrinking the input layer promotes stability,
260   while growing it enables exploration of additional candidates. The dynamic adjustment ensures that
261
      the network can escape suboptimal configurations.
262
263
      Practical considerations. The running loss l as well as the loss at the time of input layer change,
      lchange , can be either a training or validation loss, depending on whether the algorithm is used with a
264
      validation set. In our experiments, we use a validation set, which is detailed in Appendix A.2.

390   4.2     A DDITIONAL ANALYSES
391
392   In this section, we highlight some additional aspects to give a more complete picture of RelChaNet.
393   We include a comparison of the computational efficiency with similar methods, an ablation study of
394   the impact of the chosen change metric, and an investigation of the impact and feasible ranges of
395   hyperparameters.
396
397   Computational efficiency. We examine the comparative computational costs with two other
398
      approaches, NeuroFS and LassoNet. Both are well-performing sparse and dense neural network
      based methods, respectively. One drawback of our approach is that, since candidate features are
399
      chosen randomly, it generally requires more training epochs than other approaches to ensure that all

473   5    D ISCUSSION
474
475   In this paper, we introduce a novel feature selection algorithm aimed at enhancing the predictive
476   performance and interpretability of predictive models. Our approach incorporates neuron pruning
477   and regrowth from the sparse neural network literature into a dense neural network framework.
478   RelChaNet uses a relative change metric for pruning, which measures the relative change induced
479   in a network after a feature enters, while neurons are randomly regrown. Extensive experiments
480   demonstrate that our method, along with an extension featuring an adaptive input layer, consistently
481   outperforms state-of-the-art techniques on datasets with more cases than features. For datasets with
482
      more features than cases, its performance is comparable to previous approaches. While the adaptive
      version has theoretical advantages and performs better on one dataset, the base algorithm stands out
483
      for its simplicity and competitive performance in most scenarios.
484
485   The primary limitation of our approach lies in its theoretical disadvantage in computational efficiency.
      This is due in part to the reliance on a dense network, which typically has higher computational


                                                                                                9
      Under review as a conference paper at ICLR 2025


486
      training costs than sparse networks with the same number of layers and neurons. Additionally,
487   regrowing neurons randomly necessitates either a large input layer or longer training. However, our
488   experiment demonstrates that these challenges can be mitigated by employing a small neural network
489   architecture without compromising feature selection performance. Furthermore, the efficiency was
490   found to be competitive with another dense approach. It is important to note, however, that this may
491   not generalize to scenarios beyond those studied.
492
      We see many potential directions for future research. One avenue is to integrate our pruning and
493   regrowth protocol into sparse neural networks. This could be applied to the input layer for feature
494   selection, or extended to other layers for general sparse neural network training. Another direction is
495   to explore the utility of our approach for interpretable machine learning. For instance, the values in
496   the high score vector h could be evaluated as a measure of variable importance.

      A     E XPERIMENTAL SETUP
649
650
      A.1    BASELINES
651
652   The methods compared against our approach are as follows. Their specific implementations are
653   detailed in Atashgahi et al. (2023):
654
655          • Fisher Score (Gu et al., 2011): A classic filter method that selects feature sets based on their
656
               ability to separate data points.
657          • CIFE (Conditional Infomax Feature Extraction, Lin & Tang, 2006): A filter method that
658            aims to maximize the class-relevant information of the feature set.
659          • ICAP (Interaction Capping Criterion, Jakulin, 2005): A filter method that considers the
660            complementary relationship between features.
661          • RFS (Robust Feature Selection, Nie et al., 2010): A method embedded in regression that
662            uses joint L1 and L2 regularization of the weights.
663
             • QS (Quick Selection, Atashgahi et al., 2022): A method embedded in sparse neural networks
664            that combines denoising autoencoders and the L1 norm of first layer neuron weights.
665
             • STG (Stochastic Gates, Yamada et al., 2020): A method embedded in neural networks that
666
               controls the input layer neurons using a trainable probabilistic gate.
667
668          • LassoNet (Lemhadri et al., 2021): A method embedded in neural networks that adds a
669
               regularized residual connection from the input layer to the output. The residual connection
               controls the sizes of first layer weights.
670
671          • RigL (Evci et al., 2020): A method embedded in sparse neural networks that rotates features
672            by pruning based on parameter weights and regrowing based on gradients. Feature selection
673
               can be performed by investigating first layer weights after training (Atashgahi et al., 2023).
674          • NeuroFS (Atashgahi et al., 2023): A method embedded in sparse neural networks that
675            extends the ideas used in RigL to input neurons.
676
677   A.2    R EL C HA N ET SETUP
678
679
      The parameters used for RelChaNet in the main experiment are as follows. We employ a single
      hidden layer neural network with 100 neurons and a ReLU activation function. For training, we use a
680
      batch size of 1024 and a learning rate of 0.001 for the Adam optimizer. If there are fewer cases in the
681
      dataset, full batches are used instead. The hyperparameters specific to our method are: cratio = 0.2
682   and nmb = 100 for long datasets, and cratio = 0.5 and nmb = 5 for wide datasets. Stopping is based
683   on a combination of validation loss and the identified feature set. For this, the training data is split
684   again into a training and a validation set. Training continues on the training set until the validation
685   loss does not decrease for 100 input layer rotations or the set of K features with the highest values in
686   h remains unchanged for 100 rotations. Afterwards, the training is again performed on the complete
687   training data for the determined number of rotations. For the flex algorithm, during this final training
688   phase, the input layer is scaled from its initial size to the final size using a total of ten size change
689   steps.
