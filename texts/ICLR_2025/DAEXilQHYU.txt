                                                  A BSTRACT
008
009
               Link prediction with knowledge graphs has been thoroughly studied in graph ma-
010            chine learning, leading to a rich landscape of graph neural network architectures
011            with successful applications. Nonetheless, it remains challenging to transfer the
012            success of these architectures to inductive link prediction with relational hyper-
013            graphs, where the task is over k-ary relations, substantially harder than link pre-
014            diction on knowledge graphs with binary relations only. In this paper, we propose
015            a framework for link prediction with relational hypergraphs, empowering appli-
016            cations of graph neural networks on fully relational structures. Theoretically, we
017            conduct a thorough analysis of the expressive power of the resulting model ar-
018
               chitectures via corresponding relational Weisfeiler-Leman algorithms and also via
               logical expressiveness. Empirically, we validate the power of the proposed model
019
               architectures on various relational hypergraph benchmarks. The resulting model
020
               architectures substantially outperform every baseline for inductive link prediction,
021            and also lead to state-of-the-art results for transductive link prediction.

079   Approach. We first investigate the expressive power of existing graph neural networks proposed for
080
      relational hypergraphs — such as G-MPNNs (Yadati, 2020) and RD-MPNNs (Zhou et al., 2023) —
      to rigorously identify their limitations. This is achieved by studying the framework of hypergraph
081
      relational message passing neural networks (HR-MPNNs) which subsumes these architectures. To
082
      address the limitations of HR-MPNNs, we introduce hypergraph conditional message passing neu-
083   ral networks (HC-MPNNs) as a framework for inductive link prediction inspired by the conditional
084   message passing paradigm studied for knowledge graphs (Zhu et al., 2021; Huang et al., 2023). We
085   conduct a systematic expressiveness study showing that HC-MPNNs can compute richer properties
086   of nodes — dependent on k other nodes — when compared to HR-MPNNs. Specifically, our study
087   for expressive power answers the following questions:
088
      1. Which nodes can be distinguished by an architecture? To answer this question, we generalize ex-
089      isting results given for graph neural networks on knowledge graphs (Barceló et al., 2022; Huang
090      et al., 2023) using Weisfeler-Leman algorithms designated for relational hypergraphs.
091   2. What properties of nodes can be uniformly expressed by an architecture? To answer this ques-
092      tion, we investigate logical expressiveness which situates the class of node properties that can be
093      expressed by an architecture within an appropriate logical fragment.
094

095
096   • We rigorously identify the expressive power and limitations of HR-MPNNs that encompass most

      2    R ELATED WORK
109
110
      Knowledge graphs. Link prediction with knowledge graphs has been studied extensively in the
111   literature. Early literature is dominated by knowledge graph embedding models including TransE
112   (Bordes et al., 2013), RotatE (Sun et al., 2019), ComplEx (Trouillon et al., 2016), TuckER (Balaze-
113   vic et al., 2019), and BoxE (Abboud et al., 2020), which are all restricted to the transductive regime.
114   In the space of graph neural networks, RGCN (Schlichtkrull et al., 2018) and CompGCN (Vashishth
115   et al., 2020) emerged as architectures extending standard message passing neural networks (Gilmer
116   et al., 2017) to knowledge graphs using a relational message passing scheme. GraIL (Teru et al.,
117   2020) is the first architecture explicitly designed to operate in the inductive regime, but it suffers
118   from a high computational complexity. Zhu et al. (2021) proposed NBFNets as an architecture that
119
      subsumes previous methods such as NeuralLP (Yang et al., 2017) and DRUM (Sadeghian et al.,
      2019). NBFNets perform strongly and have better computational complexity thanks to their high
120
      parallelizability Zhu et al. (2021). Recently, A*Net (Zhu et al., 2023) is proposed to scale NBFNets
121
      further with the usage of a neural priority function. NBFNets are shown to fall under the frame-
122   work of conditional message passing neural networks (C-MPNNs) (Huang et al., 2023), as they
123   compute node representations “conditioned” pairwise on other node representations, making these
124   architectures suitable for binary link prediction tasks and explaining their superior performance. The
125   success of conditional message passing on knowledge graphs serves as a motivation for our work on
126   relational hypergraphs.
127
      Relational hypergraphs. Link prediction with relational hypergraphs has been widely studied in
128   the context of shallow embedding models (Wen et al., 2016; Abboud et al., 2020; Fatemi et al.,
129   2020; 2023). To score facts of the form r(u1 , · · · , uk ), some methods extend the scoring function
130   (i.e., decoder) of existing knowledge graph embedding methods to consider multiple entities. For
131   example, m-TransH (Wen et al., 2016) is an extension of TransH (Wang et al., 2014) designed to
132   handle multiple entities jointly. Similarly, GETD (Liu et al., 2020) builds on the bilinear embed-
133   ding method TuckER (Balazevic et al., 2019) to handle higher-arity relations. Fatemi et al. (2020)
134   proposed HSimplE and HypE that disentangle the position and relation embedding. BoxE (Ab-
135   boud et al., 2020) is an embedding model that encodes each relation using box embeddings, and
136
      naturally applies to k-ary relations (using k boxes) while achieving strong results on transductive
      benchmarks. Fatemi et al. (2023) explores the connection between relational algebra and relational
137
      hypergraph embeddings and proposes ReAlE. In the space of graph neural networks, Feng et al.
138
      (2018) and Yadati et al. (2019) leverage message-passing methods on undirected hypergraphs. The
139   first approach that is tailored to relational hypergraphs is G-MPNN (Yadati, 2020), which operates
140   by relational message passing. RD-MPNNs (Zhou et al., 2023) builds on this approach and addition-
141   ally incorporates the positional information of entities in their respective relations during message
142   passing, which is critical for relational facts since the order of nodes in each edge clearly matters.
143   G-MPNN and RD-MPNNs represent closest related works to the present study and we show that
144   these architectures are instances of HR-MPNNs and hence are subject to the same limitations.
145   Hyper-relational knowledge graphs and beyond. We carefully distinguish between link prediction
146   with relational hypergraphs with hyper-relational knowledge graphs (Galkin et al., 2020), which are
147   knowledge graphs where each edge is additionally augmented with additional information: a mul-
148   tiset of “qualifier-value” pairs, and n-ary relational graphs (Guan et al., 2019) relying on unordered
149   hypergraphs. We focus on link prediction with relational hypergraphs in this work but note that
150   in practice, we can convert one form of hypergraphs to another with certain transformations. See
151   detailed discussion of in Appendix B.

      Relational hypergraphs. A relational hypergraph G = (V, E, R, c) consists of a set V of nodes, a
156   set E of hyperedges (or simply edges or facts) of the form e = r(u1 , . . . , uk ) ∈ E where r ∈ R is
157   a relation type, u1 , . . . , uk ∈ V are nodes, and k = ar(r) is the arity of the relation r. We consider
158   labeled hypergraphs, where the labels are given by a coloring function on nodes c : V → D. If the
159   range of this coloring satisfies D = Rd , we say c is a d-dimensional feature map and use the notation
160   x. We write ρ(e) as the relation r ∈ R of the hyperedge e ∈ E, and e(i) to refer to the node in the i-
161   th arity position of the hyperedge e. We define E(v) = {(e, i) | e(i) = v, e ∈ E, 1 ≤ i ≤ ar(ρ(e))}
      as the set of edge-position pairs of a node v. Intuitively, this set captures all occurrences of node v in


                                                          3
      Under review as a conference paper at ICLR 2025


162
      different hyperedges and arity positions. We also define the positional neighborhood of a hyperedge
163   e with respect to a position i as Ni (e) = {(e(j), j) | j ̸= i, 1 ≤ j ≤ ar(ρ(e))}. This set represents
164   all nodes that co-occur with the node at position i in a hyperedge e, along with their positions. A
165   knowledge graph is a relational hypergraph where for all r ∈ R, ar(r) = 2.
166
      Link prediction on hyperedges. Given a relational hypergraph G = (V, E, R, c), and a query
167
      q(u1 , ..., ut−1 , ?, ut+1 ..., uk ), where q ∈ R is the query relation and “?” is the querying position,
168
      link prediction is the problem of scoring all the hyperedges obtained by substituting nodes v ∈ V in
169   place of “?”. We denote a k-tuple (u1 , . . . , uk ) by u and the tuple (u1 , . . . , ut−1 , ut+1 , . . . , uk ) by
170   ũ. For convenience, we commonly write a query as a tuple q = (q, ũ, t).
171
172
      Isomorphisms. An isomorphism from a relational hypergraph G = (V, E, R, c) to a relational hy-
      pergraph G′ = (V ′ , E ′ , R, c′ ) is a bijection f : V → V ′ such that c(v) = c′ (f (v)) for all v ∈ V ,
173
      and r(u1 , · · · , uk ) ∈ E if and only if r(f (u1 ), · · · , f (uk )) ∈ E ′ , for all r ∈ R and u1 , · · · , uk ∈ V .
174
175   Invariants. For k ≥ 1, we define a k-ary relational hypergraph invariant as a function ξ associating
176   with each relational hypergraph G = (V, E, R, c) a function ξ(G) with domain V k such that for
177   all relational hypergraphs G, G′ , all isomorphisms f from G to G′ , and for all k-tuples of nodes
178
      u ∈ V k , we have ξ(G)(u) = ξ(G′ )(f (u)).
179   Refinements. Given two relational hypergraph invariants ξ and ξ ′ , we say a function ξ(G) : V k →
180   D refines a function ξ ′ (G) : V k → D, denoted as ξ(G) ⪯ ξ ′ (G), if for all u, u′ ∈ V k , ξ(G)(u) =
181   ξ(G)(u′ ) implies ξ ′ (G)(u) = ξ ′ (G)(u′ ). In addition, we call such functions equivalent, denoted
182   as ξ(G) ≡ ξ ′ (G), if ξ(G) ⪯ ξ ′ (G) and ξ ′ (G) ⪯ ξ(G). A k-ary relational hypergraph invariant ξ
183   refines a k-ary relational hypergraph invariant ξ ′ , if ξ(G) refines ξ ′ (G) for all relational hypergraphs

      G. Similarly for equivalence.

      4.2     L OGICAL EXPRESSIVENESS OF HR-MPNN S
242
243   The previous WL characterization of HR-MPNNs is non-uniform in the sense that it holds for a
244   given relational hypergraph G. We now turn our attention to a uniform analysis of the power of
245   HR-MPNNs and study the problem of which (node) properties can be expressed as HR-MPNNs,
246   which is well-suited for the inductive setup. Following Barceló et al. (2020), we investigate logical
247   classifiers, i.e., those that can be defined in the formalism of first-order logic (FO). Briefly, a first-
248
      order formula ϕ(x) with one free variable x defines a logical classifier that assigns value true to
      node u in relational hypergraph G whenever G |= ϕ(u). A logical classifier ϕ(x) is captured by a
249
      HR-MPNN A if for every relational hypergraph G the nodes u that are classified as true by ϕ and
250
      A are the same.

505   6.3     A BLATION STUDIES ON THE IMPACT OF INITIALIZATION AND POSITIONAL ENCODING
506
507   To assess the contribution of each model component, we conduct ablation studies mainly on different
508
      choices of positional encodings and initialization functions on WP-IND and JF-IND datasets with
      the same empirical setup described in Section 6.1. Complete results are reported in Appendix Q.
509
510   Initialization. We conduct experiments to validate the impact of different initialization by evalu-
511   ating all combinations of whether including positional encoding pi or learnable query vectors zq .
512   From Table 3, we observe that both positional encoding pi and the relation zq are essential in the
513   initialization, as removing either of them worsens the overall performance of HCNet. A closer look
514
      reveals that the removal of the positional encoding is more detrimental compared to removing re-
      lational embedding since the model could deduce the relation types based on implicit information
515
      such as the arity of the query relation.
516
517   Positional encoding. We also examine the importance of the choice of positional encodings, which
518   serves as an indicator of which position the given entities lie in a hyperedge. We provide experi-
519   ments on multiple choices of positional encodings and report the results in Table 4. Empirically, we
520
      notice that the sinusoidal positional encoding produces the best results due to its ability to measure
      sequential dependency between neighboring entities, compared with one-hot positional encoding
521
      which assumes orthogonality among each position. We also notice that learnable embeddings do
522
      not produce better results since it is generally hard to learn a suitable embedding that respects the
523   order of the nodes in a relation based on random initialization. Finally, constant embedding evi-
524   dently performs the worst as it pays no respect to position information and treats all hyperedges
525   with the same set of nodes in the same way regardless of the order of the nodes in these edges.

529
530
531
      We investigated two frameworks of relational message-passing neural networks on the task of link
      prediction with relational hypergraphs, namely HR-MPNNs and HC-MPNNs. Furthermore, we
532
      studied the expressive power of these two frameworks in terms of relational WL and logical expres-
533
      siveness. We then proposed a simple yet powerful model instance of HC-MPNNs called HCNet and
534   presented its superior performance on inductive link prediction tasks, which is further supported by
535   additional transductive link prediction and synthetic experiments. One limitation lies in the poten-
536   tially high computational complexity of our approach when applied to large relational hypergraphs.
537   Our approach is also limited to link prediction and a potential future avenue is to investigate com-
538   plex query answering on fully relational data. Our study extends the success of link prediction with
539   knowledge graphs to relational hypergraphs where higher arity relations can be effectively modeled
      with GNNs, advancing applications of graph neural networks to fully relational structures.

      B    O N REPRESENTATIONS OF HIGH - ARITY FACTS
740
741
      We carefully distinguish between the task setting of relational hypergraphs (also known as knowl-
742   edge hypergraphs in Fatemi et al. (2020; 2023), or multi-relational ordered hypergraphs in Yadati
743   (2020)), hyper-relational knowledge graphs, and n-ary relational graphs.
744
745
      • Relational hypergraphs. A relational hypergraph is G = (V, E, R), where each facts in E is
        represented as k-ary tuple r(u1 , · · · , uk ) for u1 , · · · , uk ∈ V and r ∈ R. As of this work,
746
        many works (Fatemi et al., 2020; Yadati, 2020; Zhou et al., 2023) have considered this form of
747
        representation.
748
      • Hyper-relational knowledge graphs. A hyper-relational knowledge graph G = (V, E, R),
749     where R is a set of relation and each fact in E is represented as a tuple (u, r, v, {{(qri , qvi ) | 1 ≤
750     i ≤ n}}) where u, v ∈ V with r ∈ R is the main triplet, and {{(qri , qvi ) | 1 ≤ i ≤ n}} ∈ P(R×V )
751     is a set of qualifier-value pairs. Note that qualifiers are also chosen from the set of relation R and
752     are used to describe the entities as the additional information stored in the knowledge graph for
753     each triplet. Earlier research (Galkin et al., 2020; Xiong et al., 2023) mainly investigated this form
754     of representation.
755   • n-ary relational graphs. A n-ary relational graph G = (V, E, C), where each fact in E is
        represented by a set of role-value tuple {{(ri , vi ) | 1 ≤ i ≤ n}} ∈ P(C × V ), and C is the set of


                                                         14
      Under review as a conference paper at ICLR 2025


756
          roles, which are unary relation defined over entity for each fact, acting as additional information.
757       Earlier study (Guan et al., 2019) adopt this form of representation.
758
759   To further clarify the difference, we show an example with the following visualization for each type
760
      of graph in Figure 4. Given a high-arity fact “Hawking went to Oxford to study Physics and received
      a BA degree” to be captured:
761
762   • In relational hypergraphs, each fact is represented by a tuple (thus the ordering is fixed):
763
                                     StudyDegree(Hawking, Oxford, Physics, BA).
764
765   • In hyper-relational knowledge graphs, each fact is represented by a tuple of main triplet together
766       with a set of qualifier-value pairs:
767
                         (Hawking, Received, BA, {{University(Oxford), Subject(Physics)}}).
768
769   • In a n-ary relational graphs, each fact is represented as a set (thus unordered):
770
771                   {{Person(Hawking), University(Oxford), Subject(Physics), Degree(BA)}}.
772   Thus, notice that link prediction with relational hypergraphs is a more general problem setup, where
773   no roles and qualifiers are provided as extra information. This differs from the problem setup of
774   link prediction with hyper-relational knowledge graphs or n-ary relational graphs, where the unary
775   relations (qualifiers/roles) are also within the relation vocabulary.
776
777   On transformation between relational hypergraphs and other forms. Note that relational hy-
778   pergraphs can transformed into hyper-relational knowledge graphs and n-ary relational graphs by
779   generating a brand new qualifier per relation per position. However, note that the hyper-relational
780   knowledge graphs (n-ary relational graphs) generated this way are restricted: they must satisfy
781   the property that qualifiers can only appear together with their corresponding relation in the main
782
      triplet, i.e., transforming r(u1 , u2 , u3 , u4 ) to (u1 , r, u4 , {r2 : u2 , r3 : u3 }) will enforce the newly
      introduced qualifiers r2 and r3 to appear together with each other and with the main relation r. They
783
      are a very general form of representation of high-arity facts.
784
785   On the other hand, hyper-relational knowledge graphs and n-ary relational graphs can be trans-
786   formed into relational hypergraphs injectively by hashing the relation and qualifiers as a new rela-
787   tion type. However, empirically it is difficult to view these datasets as relational hypergraphs due to
788
      the explosion in the number of relations: any combination of existing relations and qualifiers would
      result in a brand new relation type in a relational hypergraph, which is impractical. Another type of
789
      transformation can be applied by directly dropping qualifiers and treating the relation on the main
790
      triplet as high-arity relations. Such transformation will lose essential qualifier information and is not
791   injective, which is a significantly difficult and different task.
792
793   We also highlight the evaluation differences as experiments on hyper-relational knowledge graphs
794
      only corrupt entities in the main triplets, whereas, in link prediction with relational hypergraphs
      setting, all entities mentioned at all positions are corrupted. We thus opt out datasets of hyper-
795
      relational knowledge graphs such as WD50K (Galkin et al., 2024) and focus only on the datasets
796
      designed for relational hypergraphs to verify our theoretical expressiveness results.

1114
       Fix a set of relation types R and a set of node colors C. The hypergraph graded modal logic (HGML)
1115
       is the fragment of FO containing the following unary formulas. Firstly, a(x) for a ∈ C is a formula.
1116
       Secondly, if φ(x) and φ′ (x) are HGML formulas, then ¬φ(x) and φ(x) ∧ φ′ (x) also are. Thirdly,
1117   for r ∈ R, 1 ≤ i ≤ ar(r) and N ≥ 1:
1118
1119                            ∃≥N ỹ r(y1 , . . . , yi−1 , x, yi+1 , . . . , yar(r) ) ∧ Ψ(ỹ)
1120
       is a HGML formula, where ỹ = (y1 , . . . , yi−1 , yi+1 , . . . , yar(r) ) and Ψ(ỹ) is a boolean combina-
1121
       tion of HGML formulas having free variables from ỹ. Intuitively, the formula expresses that x
1122   participates in at least N edges e at position i, such that the remaining nodes in e satisfies Ψ.
1123
1124   Let G = (V, E, R, c) be a relational hypergraph where the range of the node coloring c is C. Next,
1125
       we define the semantics of HGML. We define when a node v of G satisfies a HGML formula φ(x),
       denoted by G |= φ(v), recursively as follows:
1126
1127       • if φ(x) = a(x) for a ∈ C, then G |= φ(v) iff a is the color of v in G, i.e., c(v) = a.
1128
           • if φ(x) = ¬φ′ (x), then G |= φ(v) iff G ̸|= φ′ (v).
1129
           • if φ(x) = φ′ (x) ∧ φ′′ (x), then G |= φ(v) iff G |= φ′ (v) and G |= φ′′ (v).

1131       • if φ(x) = ∃≥N ỹ (r(y1 , . . . , yi−1 , x, yi+1 , . . . , yar(r) ) ∧ Ψ(ỹ)) then G |= φ(v)
1132         iff there exists at least N tuples (w1 , . . . wi−1 , wi+1 , . . . , war(r) ) of nodes of G
1133         such that r(w1 , . . . , wi−1 , v, wi+1 , . . . , war(r) ) holds in G and the boolean combination
             Ψ(w1 , . . . wi−1 , wi+1 , . . . , war(r) ) evaluates to true.

1142   Observe that HGML formulas have a restricted form and hence they are not able to represent all
1143   logical queries, which hints at the fundamental limitations of our studied models. For instance,
1144   formulas in HGML can only express local properties of nodes. That is, properties of the form “a node
1145   is connected (via hyper-edges) to other nodes satisfying other (local) properties”. This is illustrated
1146   in the example above as the variables y, z, m are forced to appear together with x in the hyper-edge
1147   StudyDegree(x, y, z, m). Another limitation of HGML is that once we quantify over the neighboring
1148   variables for x (in the example y, z, m), we can only check (local) HGML properties separately for
1149
       the neighboring variables and combine them via Boolean combinations. In the example above, we
       check the property “m has been awarded less than two prices p of some subject w” for university
1150
       m via the HGML formula α(m) = ¬∃≥2 p, w (Awarded(w, p, m)). In particular, we cannot check
1151
       properties that involve simultaneously two or more neighboring variables, as these properties would
1152   not be HGML properties (they would not even be unary). As an example, consider the property “x
1153   is a person who obtained a degree y of a subject z at a university m that has been awarded less than
1154   two prices p in subject z.” Now we do not impose that m has less than two prices in any subject,
1155   but less than two prices in the particular subject z (the same related with person x). This can be
1156   expressed as:
1157
1158      ϕ(x) = Person(x) ∧ ∃y, z, m StudyDegree(x, y, z, m) ∧ ¬∃≥2 p (Awarded(z, p, m))
1159
1160   Note that this is not an HGML formula as β(m, z) = ¬∃≥2 p (Awarded(z, p, m)) checks a condition
1161   that involves two neighboring variables (m and z). This violates exactly the requirement discussed
1162   above.

1698   I     C OMPUTING k- ARY INVARIANTS
1699
1700   In this section, we present a canonical way to construct a valid k-ary invariants. We start by intro-
1701   ducing a construction of a valid k-ary invariants termed as atomic types, following the convention
1702   by Grohe (2021).
1703
1704   I.1    ATOMIC TYPES
1705
1706   Given a relational hypergraph G = (V, E, R, c) with l labels and a tuple u = (u1 , ..., uk ) ∈ V k ,
1707
       where k > 1, we define the atomic type of u in G as a vector:
1708                                                                               k      2        m
                                            atpk (G)(u) ∈ {0, 1}lk+(2)+m +|R|k ,
1709
1710   where l is the number of colors and m is the arity of the relation with maximum arity. We use the
1711   first lk bits to represent the color of the k nodes in u, another k2 bits to indicate whether node ui
1712   is identical to uj . We then represent the order of these nodes using m2 bits and finally represent the
1713   relation with additional |R|k m bits.
1714
       Atomic types are k-ary relational hypergraph invariants as they satisfy the property that
1715
       atpk (G)(u) = atpk (G′ )(u′ ) if and only if the mapping u1 7→ u′1 , . . ., uk 7→ u′k is an isomor-
1716   phism from the induced subgraph G[{u1 , · · · , uk }] to G′ [{u′1 , · · · , u′k }].

2033   HC-MPNNs are more expressive than HR-MPNNs in the task of link prediction with relational
2034   hypergraphs.
2035   Dataset. We construct HyperCycle, a synthetic dataset that consists of multiple relational hyper-
2036   graphs with relation R = {r0 , r1 , r2 }. Each relational hypergraph G is parameterized by 2 hyperpa-
2037   rameters: the number of nodes n which is always a multiple of 4, and the arity of each edge k. Given
2038   such (n, k) pair, we generate the relational hypergraph G(n, k) = (V (n, k), E(n, k), R(n, k))
2039   where
2040                    V (n, k) = {x1 , · · · , xn }
2041                    E(n, k) = {r(i mod 2)+1 (x(i+j) mod n | 0 ≤ j < k) | 1 ≤ i ≤ n}
2042
                         R(n, k) = {r0 , r1 , r2 }
2043
2044   In short, there is a directed hyper-edge of arity k with alternating relations between r1 and r2 for
2045   all k consecutive nodes in this cycle. We present one example of such relational hypergraph in
2046   Figure 5, where n = 8 and k = 3. We generate the dataset by choosing n = {8, 12, 16, 20} and
2047   k = {3, 4, 5, 6, 7}. We then randomly pick 70% of the generated graphs as the training set and the
2048
       remaining 30% as the testing set.
2049   Objective. The objective of this task is for each node to identify the node that is located at the
2050   “opposite point” in the cycle of the given node as true. Formally speaking, for a relational hyper-
2051   graph G(n, k), we want to predict a 2-ary (hyper-)edge of relation r0 between any node xi and its
       “opposite point” x(i+n/2 mod n) for all 1 ≤ i ≤ n, i.e., classify r0 (xi , x(i+n/2) mod n ) as true. The

2074
       This is exactly due to the lack of expressiveness of HR-MPNNs by relying on a k-ary decoder for
2075
       link prediction. Theoretically, all nodes of the relational hypergraphs in HyperCycle, due to their
2076   rotational symmetry introduced by alternating relation types r1 , r2 , can be partitioned into two sets.
2077   Since the nodes within each set are isomorphic to each other, it is impossible for any HR-MPNNs
2078   to distinguish between these nodes by only computing its unary invariant. Thus HR-MPNNs cannot
2079   possibly solve this task, as whenever they classify the target “opposite point” node to be true, they
2080   also have to classify the “2-hop” node to be true, and vice versa.
2081
       However, HCNet can bridge this gap by introducing the relevant notion of “distance”. As HCNet
2082   carries out message-passing after identifying the source node, the relative distance between the
2083   source node and the target “opposite point” node will be different than the one with the “2-hop”
2084   node. Thus, by keeping track of the distance from the source node, HCNet will compute a different
2085   embedding for the positive triplet and the negative triplet, effectively solving this task.

2095
       pair of nodes relies heavily on the structural properties of these nodes (due to the lack of node fea-
2096

2097   for relational hypergraphs since the prediction now relies on the structural properties of k nodes and
2098   any model will suffer from scalability issues if k becomes large. With that being said, our approach
2099   remains feasible for the benchmark datasets, but we think it is important for future work to scale
2100   up these models for larger datasets, much like it has been done for classical GNNs (Hamilton et al.,
2101   2017; Zhu et al., 2023).

2121   On the surface, it seems that HC-MPNNs does not directly take node features into account. This
2122   is because in the task of link prediction on relational hypergraphs, no node features are explicitly
2123   provided to begin with, and thus we did not assume the presence of node features in this partic-
2124   ular task setting. However, it is relatively straightforward to account for node features by simply
2125   concatenating the node feature xv on top of the current representation hv to obtain h∗v = [hv ∥xv ].
2126   Indeed, the only requirement for HC-MPNNs in the initialization is to satisfy generalized target node
2127   distinguishability, and thus concatenating node features will preserve this property. As a result, all
2128
       theoretical results can be directly applied to HC-MPNNs with node features. It is worth noting that
       this concatenating technique has already been applied in Zhang et al. (2021) on knowledge graphs
2129
       with node features and has proven to be successful. Additionally, this technique is also mentioned in
