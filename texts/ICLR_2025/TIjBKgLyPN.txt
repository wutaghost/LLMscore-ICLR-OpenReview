009                                                        A BSTRACT
010
011
               Sparse autoencoders (SAEs) are a promising approach to extracting features from neural networks,
012            enabling model interpretability as well as causal interventions on model internals. SAEs generate
013            sparse feature representations using a sparsifying activation function that implicitly defines a set
014            of token-feature matches. We frame the token-feature matching as a resource allocation problem
015            constrained by a total sparsity upper bound. For example, TopK SAEs solve this allocation problem
016            with the additional constraint that each token matches with at most k features. In TopK SAEs,
017            the k active features per token constraint is the same across tokens, despite some tokens being
018            more difficult to reconstruct than others. To address this limitation, we propose two novel SAE
019            variants, Feature Choice SAEs and Mutual Choice SAEs, which each allow for a variable number
020
               of active features per token. Feature Choice SAEs solve the sparsity allocation problem under the
               additional constraint that each feature matches with at most m tokens. Mutual Choice SAEs solve
021
               the unrestricted allocation problem where the total sparsity budget can be allocated freely between
022
               tokens and features. Additionally, we introduce a new auxiliary loss function, aux_zipf_loss,
023            which generalises the aux_k_loss to mitigate dead and underutilised features. Our methods result
024            in SAEs with fewer dead features and improved reconstruction loss at equivalent sparsity levels as a
025            result of the inherent adaptive computation. More accurate and scalable feature extraction methods
026            provide a path towards better understanding and more precise control of foundation models.

029   1   I NTRODUCTION
030
031
      Understanding the internal mechanisms of neural networks is a core challenge in Mechanistic Interpretability. Increased
032   mechanistic understanding of foundation models could provide model developers with tools to identify and debug
033   undesirable model behaviour.
034
035
      Dictionary learning with sparse autoencoders (SAEs) has recently emerged as a promising approach for extracting
      sparse, meaningful, and interpretable features from neural networks, particularly language models (Huben et al., 2024;
036
      Sharkey et al., 2022).
037
038   One problem with wide SAEs for foundation models is that there are often many dead features (Rajamanoharan
039   et al., 2024a; Templeton et al., 2024; Gao et al., 2024). Dead features are features which are remain inactive across
040   inputs, effectively wasting model capacity and hampering efficient training. Another problem is that approaches like
041
      TopK SAEs (Gao et al., 2024) don’t have a natural way to take advantage of Adaptive Computation: spending more
      computation, and crucially more features, to reconstruct more difficult tokens.
042
043   We frame the problem of generating sparse feature activations corresponding to some given neural activations as a
044   resource allocation problem, allocating the scarce total sparsity budget between token-feature matches to maximise
045   the reconstruction accuracy. Within this framing, we naturally motivate two novel SAE variants which can provide
046   Adaptive Computation: Feature Choice SAEs and Mutual Choice SAEs (FC and MC SAEs respectively). Feature
047
      Choice SAEs solve the sparsity allocation problem under the additional constraint that each feature matches with at
      most m tokens. Mutual Choice SAEs solve the unrestricted allocation problem where the total sparsity budget can be
048
      allocated freely between tokens and features. These approaches combine the Adaptive Computation of Standard SAEs
049
      with the simple optimisation and improved performance of TopK SAEs.
050
051   Our contributions are as follows:
052
053         • We provide a framing for sparsifying activation functions in SAEs as a solution to a resource allocation
              problem.

101   2.1   S PARSE AUTOENCODERS
102
103   Sparse Autoencoders (SAEs) (Lee et al., 2007; Le, 2013; Mairal et al., 2014) learn an over-complete basis, or
104
      dictionary, of sparsely activating features. The feature activations, z, correspond to their associated neural activations,
      x, via the feature dictionary. In particular, we can write an SAE as:
105
106                                                    z = σs (Enc(x)) ∈ RF                                                   (1)
107                                                                      N
                                                       x̂ = Dec(z) ∈ R                                                        (2)

108
      where σs is a sparsifying activation function (e.g. ReLU), Dec is an affine map and x ∈ RN 1 . z′ = Enc(x) are the
109   pre-activation features, which we will call the token-feature affinities.
110
111   SAEs are trained to minimize the Reconstruction Error (Mean Squared Error) between x and x̂. This reconstruction
112
      error term is combined with an optional Sparsity Loss term (for example, an L1 penalty to induce sparsity) and an
      optional Auxiliary Loss term to reduce dead features:
113
114                                    L(x) = |x − x̂|22 + λ1 Lsparsity (z) + λ2 Laux (x, z, x̂)                     (3)
115
      Rajamanoharan et al. (2024a); Templeton et al. (2024); Gao et al. (2024) have shown that decomposing neural activations
116
      using the SAE feature dictionary allows for increased human interpretability of models even at model sizes comparable
117
      to frontier foundation models.

      2.2     A DAPTIVE C OMPUTATION
120
121   In Adaptive Computation, neural networks decide how much compute (and/or which parameters) to apply to a given
122   input example (Graves, 2017; Xue et al., 2023). Ideally, the model should learn to apply less compute to easier examples
123   and more compute to more difficult examples in order to maximise performance within a compute budget. In our setting,
124   we consider the token-feature matches to be the scarce quantity to allocate, where we say that a token matches with a
125   feature if the feature is activated on that token.
126
127   2.3     T OP K SAE S
128
      TopK SAEs (Gao et al., 2024) use a TopK activation function instead of the L1 penalty to induce sparsity, as in
129
      Makhzani & Frey (2014). Though in the standard L1 SAE formulation, the number of features-per-token is variable, in
130
      the TopK formulation the features-per-token is fixed at the same k for all tokens. We hypothesize that having a fixed k
131   is a key drawback of the TopK method. Variable k values introduce Adaptive Computation which can focus more of the
132   token-feature matching budget on more difficult tokens.
133
134
      In concurrent work, Bussmann et al. (2024) introduce BatchTopK which is closely analogous to our Mutual Choice
      SAEs and also provides adaptive computation. However, they do not deal with the problem of underutilised features.
135
136
137
      2.4     D EAD F EATURES
138   SAE features which remain inactive across many inputs are known as dead features. Bricken et al. (2023) declare a
139   feature to be dead when it hasn’t fired for at least 1e7 tokens. Dead features present a challenge especially when scaling
140   to larger models and wider autoencoders. For example, Templeton et al. (2024) find 64.7% of features are dead for their
141   autoencoders with 34M features. Our Feature Choice approach naturally results in zero dead features by ensuring that
142   each feature activates for every batch.
143
144   2.5     AUXILIARY K LOSS FUNCTION
145
146   Gao et al. (2024) propose the auxiliary loss function, aux_k_loss to reduce the proportion of dead features. Given the
147
      SAE residual e = x − x̂, they define the auxiliary loss Laux = |e − ê|2 , where ê = Dec(zdead ) is the reconstruction using
      the top kaux dead features. Gao et al. (2024) find fewer dead features when using the aux_k_loss for SAE training.
148
149   However, the aux_k_loss is only applied to features which qualify as dead. We apply a similar auxiliary loss, the
150   aux_zipf_loss, to underutilised, but not yet dead, features.

      3.1.1      A NALOGY T O M IXTURE OF E XPERTS ROUTING
242   We choose our naming here to align with the study of token-expert matching in the Mixture of Experts paradigm,
243   where there is a close analogy. Token Choice MoE routing strategies (Shazeer et al., 2017; Fedus et al., 2022) have the
244   constraint that each token can be routed to at most k experts allowing for expert imbalances. On the other hand, Expert
245   Choice routing strategies (Zhou et al., 2022) have the constraint that each expert processes at most m tokens, which
246   eliminates the possibility of underutilised experts and allows tokens to be routed to a variable number of experts.
247
      Here the intuitions are “each token picks the k experts to be routed to” and “each expert picks the m tokens to be routed
248   to that expert” for Token Choice and Expert Choice respectively. The variety of MoE routing algorithms is explored in
249   Liu et al. (2024). The Feature Choice approach we propose is directly analogous to the Expert Choice approach in
250   MoEs and TopK SAEs are directly analogous to Token Choice MoEs. For this reason, we will call TopK SAEs, Token
251   Choice SAEs to unify our notation.

253   3.2      D ISTRIBUTION OF F EATURE D ENSITIES
254
255   When analysing the distribution of feature densities in open-source SAEs from Gao et al. (2024), we find (as shown in
256   Figure 3) that the distributions typically follow a power law described by the Zipf distribution with R2 = 0.982.
257   We note that although the Zipf distribution well fits most of the distribution, there is a considerable residual at the lower
258   end of the distribution (the 20k+ rank tokens). We suggest that these features are underutilised. Underutilised features
259   see fewer gradient updates than other features leading to a self-reinforcing cycle of these features being less useful and
260   hence further underutilised until they die.
261   We will refer to these underutilised features as dying features. We define a dying feature as a feature which is one of
262   the 25% least prevalent features which is also <60% as prevalent as we might predict from the prevalence rank of the
263   feature using the fitted Zipf curve. More formally:

287   4.2     R EDUCING THE P ROPORTION OF DYING F EATURES
288
289   To address the problem of dying features discussed in Section 3.2, we add an additional auxiliary loss for dying features,
290   which is a natural generalisation of the aux_k_loss. Given the SAE residual e = x − x̂, we define the auxiliary loss
291   Laux_zipf = |e − ê|2 , where ê = Dec(zdying ) is the reconstruction using the top kaux dying features. We can think of this
292   aux_zipf_loss acting preventatively on features which could be at risk of becoming dead and acting rehabilitatively
293   on features which have been recently revived. In this way, we reduce the proportion of both dead and dying features.
294
295   4.3     C HOOSING THE F EATURE C HOICE C ONSTRAINT
296
      In the Feature Choice approach, there remains the question of how to distribute the sparse feature activations across the
297
      feature dimension.
298
299   The simplest approach to this is to take mi = M/F for all i, where F is the number of features. In other words, each
300   feature can pick exactly m tokens to process. We call this approach Uniform Feature Choice.
301   Uniform prevalence is a natural way to organize the features so that a feature firing provides maximal information
302   about the token, under the assumption that all features provide approximately equal information. However, we have
303   seen that in existing open-source SAEs, all features are not equivalently prevalent. Instead, they are approximately
304   Zipf-law distributed. To maintain this distribution of feature density we choose mi ∼ Zipf(α, β), where Zipf represents
305   a truncated Zipf distribution and i is the rank of a given feature in terms of feature density.
306                                                                                 1
307                                                         mi = Zipf(i) ∝                                                                    (4)
                                                                                (i + β)α

      We call the Feature Choice approach where the mi are Zipf-distributed, Zipf Feature Choice, henceforth simply Feature

      Choice.

448   We believe that conceiving of the role of SAE encoders as defining matching/routing algorithms (similar to other
449   Adaptive Computation work, for example, within the Mixture of Experts literature) could be a valuable intuition pump
450   for further improvements to SAE architectures.
451   Our approaches can also be combined with the MDL-SAE (Ayonrinde et al., 2024) formulation which treats conciseness
452   (Description Length) as the relevant quantity for evaluation and model selection rather than sparsity (L0 ). The
453   Description Length of a set of feature activations is a function of the sparsity, the distribution of activation patterns for
454   each feature and the SAE width.

      The computational efficiencies of the Token Choice (TopK), Mutual Choice and Feature Choice approaches are
456   theoretically equivalent in terms of Floating Point Operations (FLOps) and in our experiments we find empirically that
457   the Mutual Choice and Feature Choice activation functions do not add significant computational requirements (<1%
458   difference in wall clock time) despite converging more quickly.

      7.1   Z IPF D ISTRIBUTED F EATURES
462
      We find that constraining the number of tokens per feature with the Zipf distribution outperforms using the uniform
463
      distribution by >10% model loss recovered. The large drop in performance using the uniform distribution compared to
464   the Zipf distribution gives additional evidence that naturally occurring features are not uniformly distributed.
465
466   We hypothesise that the reason that features appear to be Zipf distributed may be strongly analogous to the reason that
467
      the frequency of words in natural languages like English are also Zipf distributed. Words are the semantic units of
      sentences. Since features are the semantic units of computation within language models, a similar mechanism could
468
      explain the empirical tendency for the densities of SAE features to tend towards the Zipf distribution.
469
470   In the Computational Linguistics literature, it is well known that the distribution of words in natural languages
471   approximately follows a Zipf distribution (Zipf, 1949). For example, in written English text, the empirical distribution
472   over words (treated as a categorical variable) can be modelled as Zipf(α, β) = Zipf(1, 2.7).
473   We speculate that the Preferential Attachment Theory explanation (Zhu et al., 2018; Chen, 2012) for the tendency
474   of words to be Zipf-distributed, which states that frequently used words (features) tend to be used more often, may
475   be analogously applicable here. At initialisation, there is some variance in feature prevalence. The tokens which are
476   initially most highly activated early in training receive the most gradient signal and are most refined, leading to a
477   virtuous cycle where they are more effective and useful for a larger part of the feature density spectrum. Figure 6
478   illustrates this dynamic over time. We would be excited about further work detailing an explicit mechanism for why
479
      features in Neural Networks tend towards being Zipf distributed.
480
481   7.2   A DAPTIVE C OMPUTATION FOR VARYING T OKEN D IFFICULTY
482
483   One benefit of the Mutual Choice and Feature Choice approaches is that they allow for Adaptive Computation - difficult
484   to reconstruct tokens (which may represent complex or rare concepts) can be reconstructed using more features, whereas
485   other tokens which are more straightforward can use fewer features. Where Token Choice (TopK) SAEs suggest that all
      tokens are equal; MC and FC SAEs suggest that, in fact, some tokens are more equal than others Singer (2017).


                                                          9
      Preprint. Under review.


486
      As an extreme example, we might expect that the activations resulting from the < BOS > token are relatively easy to
487   reconstruct, considering they are both very common and have exactly the same value every time. We might expect that
488   an effective SAE could learn to productively reallocate the sparsity budget that would have been spent on the < BOS >
489   token to more difficult tokens, thus increasing the SAE’s effective capacity. We provide an example of the features per
490   token distribution in Appendix D.

492   7.3      D EAD F EATURES
493
494   Our methods, especially Feature Choice SAEs, have many fewer dead features than other approaches (typically zero)
495   without complex and compute-intensive resampling procedures (Bricken et al., 2023). This is especially important for
496   large SAEs where the problem of dead features is typically more significant. We hypothesize that this might be an
497
      additional reason for the improved performance: all features receive some gradient signal at every step.
498   We note that this simplified approach eliminates the need for complex resampling procedures. This approach suggests a
499   simplified procedure for SAE training and model selection than previous SAEs with fixed sparsity budget.

501   7.4      L IMITATIONS
502
503   Appeals to the Monotonic Importance Heuristic: In Section 3.1 we defined the Monotonic Importance Heuristic
504   (MIH) - the assumption that the importance of a feature is monotonically increasing in feature activation magnitude.
505
      We use this assumption when we choose the feature activations with the largest magnitudes with our TopK-style
      activation functions. TopK SAEs also implicitly assume the MIH (Monotonic Importance Heuristic). We can think of
506
      Jump-ReLU SAEs and Gated SAEs as relaxing this assumption slightly to a weak MIH. For Jump-ReLU and Gated
507
      SAEs, the importance of activations is still related to their magnitude and they still filter out any low magnitude feature
508   activations; however, the filtering threshold varies for each feature. So Jump-ReLU SAEs do not have to make magnitude
509   comparisons across features which may have different natural scales. It may be, however, that there are low magnitude
510   activations which, within a certain context, are nonetheless critically important in capturing information which is useful
511   for reconstruction and/or downstream model performance. These important but low magnitude activations are difficult
512   to capture with our current SAE approaches 10 .
513
      Though the weight-sharing form of Gated SAEs (Rajamanoharan et al., 2024a) implicitly encodes the weak MIH
514   prior, the non-sharing form does not. Weight-sharing Gated SAEs, however, tend to perform better. The improved
515   performance of approaches which encode the MIH prior could be considered as evidence for the truth of the claim.
516   Alternatively, we might note that the Monotonic Importance Heuristic acts as an inductive bias for our models. Good
517   inductive biases often allow models to perform better at first; however, with increased scale we may not need such
518   inductive biases and may prefer allowing the model to learn more of the solution independently (Xiao, 2024), (Sutton,
519   2019).
520   Generalisation Across Modalities: We tested our SAEs within the domain of language. We currently don’t know
521   to what extent our results generalise across modalities, especially to inherently continuous modalities like audio and
522   images. We would be excited about future work applying similar techniques to Interpretability problems in a wider
523   range of modalities.
524
      Evaluation: The Mechanistic Interpretability field doesn’t currently have widely agreed upon metrics for evaluating
525
      Sparse Autoencoders. Disentanglement benchmarks like Huang et al. (2024) have been proposed as well as evaluation on
526   tasks where the ground truth is known (Karvonen et al., 2024). Developing a more comprehensive suite of benchmarks
527   for SAEs would help us to have higher confidence in our comparisons between SAE variants.

911   F    M ONOTONIC I MPORTANCE H EURISTIC
912
913   In Section 3 we appeal to the Monotonic Importance Heuristic (MIH), as in TopK SAEs, in order to simplify our
914
      allocation problem. Empirically we find that this works well though we discuss the case for not using the MIH in
      Section 7.4.
915
916   We can formally write the Monotonic Importance Heuristic as the hypothesis that given some token-feature affinities z’
917   and corresponding feature activations z1 and z2 , if ∥z1 ∥1 > ∥z2 ∥1 , z1 is likely to result in lower reconstruction error
      under the decoder map.


                                                        17
      Preprint. Under review.


918
      One theoretical (though informal) motivation for the MIH is as follows. Since the decoder dictionary is fixed to unit
919   norm, the norm of any feature’s contribution to the output is exactly the magnitude of the feature activation to which it
920   corresponds. Hence if there’s limited cancellation between features (which is likely in an N dimensional space where
921   the per-token sparsity is much less than N) then we might expect the component of x̂ in any feature direction to be very
922   close to the feature activation for that feature. In particular, consider a feature with a small magnitude of ε. This feature
923   can only possibly influence the reconstruction loss by at most ε|e| where e = x − x̂. Features corresponding to larger
924   activations can plausibly influence the reconstruction loss by a greater amount.
