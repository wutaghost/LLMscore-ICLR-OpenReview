009                                                A BSTRACT
010
011            Large language models, such as ChatGPT (Josh Achiam, 2023), Claude (An-
012            thropic, 2024), or LLaMA (Touvron et al., 2023), are gigantic, monolithic, and
013            possess the superpower to simultaneously support thousands of tasks. However,
014            high-throughput applications often prefer smaller task-specific models because of
015            their lower latency and cost. One challenge of using task-specific models is the
016
               incremental need for solving newer tasks after the model is already deployed for
               existing tasks. A straightforward solution requires fine-tuning the model again
017
               for both existing and new tasks, which is computationally expensive and time-
018
               consuming. To address this issue, we propose a model merging based approach
019            called S UPER M ERGE. S UPER M ERGE is a gradient-based method to systemati-
020            cally merge several fine-tuned models trained on existing and new tasks. S UPER -
021            M ERGE is designed to be lightweight and fast, and the merged model achieves
022            similar performance to fully fine-tuned models on all tasks. Furthermore, we pro-
023            posed a hierarchical model merging strategy to reduce the peak space requirement
024            without sacrificing the performance of the merged model. We experimentally
025            demonstrate that S UPER M ERGE outperforms existing model merging methods
026            (Ilharco et al., 2022; Yu et al., 2024; Yadav et al., 2024; Yang et al., 2024) on
027
               common natural language processing and computer vision tasks.
028
029   1   I NTRODUCTION
030
031   The rapid development of foundational models has led to numerous challenges and opportunities.
032   Majority of the foundational models are generative, wherein they are capable of “generating” the
033   output given an input. The input and output both can be free-form text, images, or multi-modal.
034   This is a paradigm shift from the earlier predictive models that had rigid and task-specific inputs
035   and outputs and were unable to generate free-form text or images. Generative capabilities enable
036   foundational models to learn several tasks simultaneously and generalize to tasks unseen during
037   training. This has lead to research in the topics of multi-task learning for several sub-disciplines of
038   machine learning, such as computer vision (Chen et al., 2018; Misra et al., 2016), natural language
039
      processing (Collobert & Weston, 2008; Dong et al., 2015), and recommendation systems (Song
      et al., 2024; Ma et al., 2018).
040
041   In this context, it has become common to train large foundational models on thousands of tasks that
042   could be performed with modest accuracy. During training these models gain the ability to generate
043   a response based on a vast repository of public data and commonsense knowledge. However, there
044   are situations where the model is not adequately trained on organization-specific tasks, whose data
045
      is not available in the public domain. In such cases, fine-tuning these models to perform a specific
      task (or subset of tasks) is often required. For example, a general model could be outstanding at
046
      summarizing text, but could be mediocre at summarizing organization-specific technical jargon.
047
048   Fine-Tuning. There are a few major classes of fine-tuning approaches. The first approach is called
049   end-to-end fine-tuning. In this approach all existing model parameters are adjusted to minimize a
050   loss function using mini-batch gradient descent and back-propagation over a fine-tuning data set.
051   For tasks like classification, a new classification head is added to the model, such that it learns to
052   classify on the fine-tuning data set. The parameters in the classification head are initialized randomly.
053   However, for fine-tuning larger models the compute needed for end-to-end fine-tuning becomes
      prohibitively large and therefore methods known as parameter efficient fine-tuning (or PEFT) were


                                                         1

054
      introduced. These methods only tune a sub-set of the parameters, while others are held fixed. This
055   dramatically reduces the memory and compute required for fine-tuning and achieves acceptable
056   performance on the fine-tuning tasks (Hu et al., 2021; Zhang et al., 2023; Liu et al., 2022; Dettmers
057   et al., 2024).
058
      One such popular PEFT approach is LoRA (Hu et al., 2022). In LoRA, each weight matrix in the
059
      model W of size n-by-m is represented as W + U V ⊤ where U and V are low-rank and of size n-
060
      by-r and m-by-r and respectively. The original parameters W do not participate in the fine-tuning
061   process and only weights in U and V are tuned using gradient descent. The number of additional
062   parameters that LoRA introduces are therefore r(n + m). r controls the number of new fine-tuning
063   parameters, where typically r < 5. Thus, a tiny fraction of tunable parameters are used for learning
064   from the fine-tuning data set. This drastically reduces memory and compute requirements.
065
      Deployed fine-tuned models often require updates to extend their capabilities to newer tasks. For
066
      instance, a model fine-tuned for the text-to-SQL task could be extended to handle data processing
067   in NumPy or Pandas for providing comprehensive tooling experience for data scientists. However,
068   these extension efforts are non-trivial. One alternative is to start from the pre-trained model and fine-
069   tune the model again on existing and new tasks. Even with PEFT, such repetitive fine-tuning process
070   can be computationally expensive, especially when multiple rounds of task updates are required.
071
072   Limitations of Ensemble Learning Techniques. Utilizing ensemble learning is another approach
073   for enabling new model capabilities (Caruana et al., 2004). However, foundational models are
074   large and therefore ensembling them is often challenging. Additionally, creating an model ensemble
075
      entails obtaining predictions from all ensemble components. Given that foundational models have
      high inference cost, obtaining predictions from all ensemble components becomes slow and costly.
076
      Another approach here is to learn a ”router” model that simply routes the input to the appropriate
077
      model for inference. However, such ”routers” are models themselves and they have to be retrained
078   frequently for maintaining performance. Alternatively, some approaches propose ensembling differ-
079   ent checkpoints of the same model that are saved while training or fine-tuning (Garipov et al., 2018;
080   Wortsman et al., 2022; Fort et al., 2019). These approaches often work when models are trained for
081   the same task, but are relatively ineffective in combining models fine-tuned on different tasks.
082
083   Model Merging. Given these challenges, there has been an emergence of approaches that propose
084   combining model trained on disparate tasks —instead of the predictions— to form a single model.
085   These techniques are collectively referred to as model merging methods. As model merging methods
086
      output a single model the number of inference parameters remain unchanged, plus the new model
      is capable of performing existing and new tasks. Secondly, merging techniques are very efficient as
087
      compared to fine-tuning or even parameter efficient fine-tuning, making them an attractive alterna-
088
      tive for enhancing task generalization performance of fine-tuned models.
089
090   Model merging techniques start from two or more models whose weights have been tuned on dif-
091   ferent data sets (or tasks). Their main objective is to merge these models with minimal impact to
092
      performance and with lower cost as compared to fine-tuning on all data sets from scratch. Model
      merging techniques typically work with models from identical architectures, but with either same or
093
      different initialization or starting points. Examples of the model merging techniques with the same
094
      initialization are DARE (Yu et al., 2024), Model Soups (Wortsman et al., 2022), and TIES (Yadav
095   et al., 2024). Similarly, examples of techniques that work on different initialization are OT Fusion
096   (Singh & Jaggi, 2020), Git-Rebasin (Ainsworth et al., 2023), and ZipIT (Stoica et al., 2023).
097
098
      As we show in Section 4 and Fig. 2, existing model merging techniques such as DARE (Yu et al.,
      2024), TIES (Yadav et al., 2024) and Task-Arithmetic (Ilharco et al., 2022) are sensitive to scaling
099
      hyperparameters. We show that every model layer does not participate equally in fine-tuning: some
100
      layers receive larger weight updates than other layers. Plus, every model layer does not participate
101   equally across tasks: some tasks generate larger updates for a layer than others. In an attempt to
102   address these challenges, we propose a principled merging method called S UPER M ERGE. S UPER -
103   M ERGE is a supervised model merging approach that learns the contribution of each layer using a
104   tiny amount of validation data, instead of relying of uniform layer-agnostic scaling hyperparameters.
105   S UPER M ERGE adaptively merges at an appropriate granularity to deliver models that achieve state-
106   of-the-art performance. S UPER M ERGE achieves this performance while introducing a minuscule
107   number of tunable merging parameters. We demonstrate that S UPER M ERGE achieves an average
      accuracy improvement of 5.8% across all tasks, while the per-task improvement is upto 49.4% over

      LLM Fine-Tuning Techniques. The core idea of fine-tuning is to extend the knowledge acquired
132
      from pre-training and adapt it to a specific target domain through additional training using a task-
133
      specific data set. In the natural language processing domain, instruction fine-tuning (Wei et al.,
134   2022; Chung et al., 2024) serves as a widely adopted approach to enhance the model’s ability to com-
135   prehend and accurately execute desired tasks. To enhance fine-tuning efficiency, parameter-efficient
136   fine-tuning (PEFT) methods have been proposed. PEFT methods typically introduce lightweight
137   task-specific adaptations to the pre-trained model. One approach is to add adapters (Houlsby et al.,
138   2019) as sub-modules to the pre-trained model, which enables efficient learning of new knowledge.
139   Low-Rank Adaptation(LoRA) (Hu et al., 2022) factorizes each weight matrix into a low-rank de-
140   composition that has minimal number of tunable parameters. Infused Adapter by Inhibiting and
141   Amplifying Inner Activations (IA3 ) (Liu et al., 2022) proposes a method of learning a subset of
142
      additional parameters to re-scale inner activations in the attention and feed-forward modules of
      transformer-based language models. These methods aim to achieve comparable performance to
143
      traditional full fine-tuning, while achieving significant reduction of tunable parameters.
144
145   Multi-Task Learning via Model Merging. Model merging methods aim to combine two or more
146   fine-tuned models into a single model. Most existing methods focus on merging models that are
147   derived from the same base architecture and initialization. The most intuitive approach is calcu-
148   lating a smooth average of the model parameters across different tasks, such as Task Arithmetic
149   (Ilharco et al., 2022) and Fisher-Weighted averaging (Matena & Raffel, 2024). TIES-Merging (Ya-
150   dav et al., 2024) takes a rule-based approach to resolve sign conflicts and merges with least feature
151   redundancy. DARE (Yu et al., 2024) performs sparsification on each fine-tuned model by randomly
152   dropping and ensembles them with rescaling. AdaMerging (Yang et al., 2024) is the most relevant
153
      related work to S UPER M ERGE. Unlike S UPER M ERGE, AdaMerging is an unsupervised approach
      that learns the layer contributions using an entropy-based loss function. This also restricts the ability
154
      of AdaMerging to only merging predictive models (classification and regression), while S UPER M E -
155
      RGE can merge predictive and generative models. Another direction explores merging models with
156   different initializations. The Git Re-Basin (Ainsworth et al., 2023) method permutes the units of
157   one model to match them with the reference model, enabling the merging of two models in the
158   aligned space. Optimal Transport Model Fusion (Singh & Jaggi, 2020) (or OT Fusion) leverages
159   the Wasserstein barycenter to align weights or activations across different models, performing opti-
160   mal transport computation in each layer to enable model fusion. Zipit (Stoica et al., 2023) merges
161   models layer-by-layer by using a manually constructed computational graph. It uses this graph to
      examine the latent feature connections and redundancy that is identified through the similarity of

      The models that we consider for merging are deep learning models composed of numerous layers.
183   Particularly, we assume that we have k fine-tuned models with identical structure each having n
184   layers. We also assume that all the parameters of model i are materialized into a single vector. We
185   denote such a vector by θ(i), where i = (1 . . . k). At times we need to refer to a particular layer
186   j = (1 . . . n) of a fine-tuned model this is denoted as θ f (i, j). Next, we denote by θ p the weights
187   of the pre-trained model and θ m denotes the weight of the final merged multi-task model. The
188   fine-tuned and merged models have identical architecture as the pre-trained model.
189   A particular layer j for pre-trained and merged model is denoted as θ p (j) and θ m (j) respectively1 .
190   We also assume that each of the fine-tuned models uses its own training, validation and test data sets.
191   These data sets are denoted as D1train . . . Dktrain , D1val . . . Dkval , and D1test . . . Dktest respectively. We
192   use Dtrain , to indicate the union of all the training data sets. Similarly, Dval and Dtest indicate the
193   union of all the validation and test data sets. We assume that a fine-tuned model’s parameters θ f (i)
194   have been already tuned using their corresponding training data set Ditrain .

197   4       BACKGROUND AND M OTIVATION
198
199   A majority of recent model merging approaches that follow the “identical architecture, identical
200   initialization” merging style claim to be training-free2 . In this category, there are three important
201   and well-known model merging techniques relevant to S UPER M ERGE: (1) Task-Arithmetic (Ilharco
202
      et al., 2022), 2) TIES (Yadav et al., 2024), and 3) DARE (Yu et al., 2024). Let us understand these
      methods with a toy setup of merging only two models, i.e. k = 2. The Task-Arithmetic approach
203
      begins by computing task vectors that are defined as τ (1) = θ f (1) − θ p and τ (2) = θ f (2) − θ p .
204
      The weights of the merged multitask model are given as θ m = θ p + λ(τ (1) + τ (2)). In the
205   TIES approach, the final weights are still computed similar to Task Arithmetic, but the task vectors
206   τ (1) and τ (2) are computed differently. The TIES task vectors are sparsified by first keeping only
207   the top-k entries. Then, positive and negative values of a particular parameter across models are
208   independently averaged. The final parameter value is the one with larger average magnitude. In
209   the DARE approach, the task vectors of each model τ (1) and τ (2) are first sparsified by randomly
210   setting the parameters in τ (i) to zero with probability p. The remaining parameters are then re-
211   scaled by multiplying them with 1/(1 − p) to form the modified task vectors τ̃ (i) and τ̃ (2). Re-
212   scaling is performed to regain the scale that existed before sparsification. The modified task vectors
213
      τ̃ (i) are combined using λ to form the merged parameter vector θ m = θ p + λ(τ̃ (1) + τ̃ (2)).

      Observe that the existing so-called “training-free” model merging methods are not entirely devoid
242   of training procedures. Crucially, the weighting parameter λ involved in the Task Arithmetic, TIES-
243   Merging, and DARE can be characterized as a form of hyperparameter, whose optimal value is
244   found using the validation data set Dval . These methods typically employ a single hyperparameter
245   λ to scale the task vectors and incorporate it into the pre-trained weight. λ is determined through
246   an extensive grid search process spanning values between 0 and 1, a non-trivial amount of computa-
247   tional resources are still required to identify this optimal value. As shown in Fig. 2, when we merge
248   models across 11 different NLP tasks, the performance of DARE drop significantly on both sides
249   of the optimal λ = 0.3. Similar behaviour is observed for TIES. Therefore, although the merging
250
      algorithm itself does not require any data, performing grid-search to the best λ requires data and
      setting λ without grid-search is detrimental to the merged model’s performance (refer Fig. 2).
251
252   Next, we analyse the layer-wise distribution of the task vectors. For this, we PEFT tuned the T5
253   model Raffel et al. (2020) on two tasks, computed the layer-wise task vectors, and plot them in
254   Fig. 3. Observe that the layer-wise distribution of task vectors could vary significantly across differ-
255   ent layers and tasks. The distribution of the exhibits distinct patterns across different blocks of the
256
      T5 model. Both encoder and decoder demonstrate a general trend of increasing variance from initial
      to deeper layers. Additionally, the pattern of variation is task-specific: the same layer can receive
257
      different magnitude of updates on two different tasks. This observation demonstrates the limitations
258
      of employing a universal hyperparameter (such as λ) to scale the merged task vector, as different
259   blocks and weight matrices exhibit distinct magnitudes of variations.

264   Given the challenges for selecting the optimal value for λ and the large layer-specific variations in
265   the task vectors, we propose a simple supervised approach for model merging called S UPER M ERGE.
266   S UPER M ERGE introduces a tiny number of trainable parameters that can be tuned using a small
267   amount of validation data Dval . The number of parameters we introduce is proportional to the
268   number of model layers. Typically, the number of layers is in the few hundreds, while the number
269   of model parameters could be in the billions. As a result, we end up introducing only a minuscule
      number of new parameters.

356   Baselines. We use two types of baseline methods: non-gradient based methods and gradient based
357   methods. Non-gradient based methods do not use gradient descent to adjust the tunable parameters.
358   Here we use Task Arithmetic (Ilharco et al., 2022), DARE (Yu et al., 2024), and TIES (Yadav
359   et al., 2024). These methods use the hyper-parameter λ to add the merged task vector to pre-trained
360   weight. Gradient based methods use gradient descent to tune the weights. Here we consider: In-
361   dividual fine-tuned models that are fine-tuned using Ditrain . A single Multitask model fine-tuned
362   using Dtrain . Finally, we consider AdaMerging3 (Yang et al.), which is a gradient-based model
363   merging methods. It uses an unsupervised loss to learn the merging weight. We omit older baselines
364   such as Fisher Merging (Matena & Raffel, 2024) and RegMean (Jin et al.) as they are known to be
365
      sub-optimal as compared to the recent baselines, such as TIES and DARE that are included in our
      comparison.

530   7      C ONCLUSION
531
532   We introduced a gradient-based model merging method called S UPER M ERGE. S UPER M ERGE only
533   a handful of training parameters and runs significantly faster than full fine-tuning while maintain-
534   ing similar performance. Our experiments demonstrate S UPER M ERGE outperforms existing model
535   merging methods on numerous tasks. Furthermore, merging several models at once could require
536   substantial memory that increases with the number of tasks and the size of the base model. We pro-
537   posed the hierarchical model merging strategy to alleviate this problem by merging a small subset
538   of base models at a time. Hierarchical merging significantly reduces the peak memory requirement
539   without compromising the merged model’s performance.

784   where k is the number of task. The peak memory requirement consists of four components. Memory
785   required for all the weights of the pre-trained model. Each parameter needs 4 bytes space. Opti-
786   mization requires extra space to save the gradients and optimizer states of trainable parameters. We
787   chose AdamW as the optimizer and it needs 8 bytes per parameters to maintain 2 states. The space
788   for task vectors of all k task-specific fine-tuned models. Note that only model merging methods need

      these methods use the pre-trained model as the starting point. Thus, we added a indicator function
      to the forth term in Eq. (3).
