009                                               A BSTRACT
010
011            The size of the key-value (KV) cache plays a critical role in determining both the
012            maximum context length and the number of concurrent requests supported during
013            inference in modern language models. The KV cache size grows proportionally
014            with the number of attention heads and the tokens processed, leading to increased
015            memory consumption and slower inference for long inputs. In this work, we ex-
016            plore the use of MixAttention, a model architecture modification closely related to
017            a blog published by Character.AI (Character.AI, 2024). MixAttention combines
018
               sliding window attention, where only a small subset of recent tokens is stored in
               the KV cache, with KV cache sharing across layers. Our experiments demonstrate
019
               that MixAttention significantly reduces memory usage and improves inference
020
               speed without sacrificing model performance in both short and long-context tasks.
021            We also explore various configurations of this architecture, identifying those that
022            maintain quality across evaluation metrics while optimizing resource efficiency.

025   1   I NTRODUCTION
026
027   Transformer-based language models are getting increasing popular in consumer usage as well as
028   industrial workloads. A general trend seen so far has been that bigger models are better at tasks than
029   smaller models, but that comes at the cost of increased inference cost and slower speed (Hoffmann
030   et al., 2022; Sardana et al., 2024). Further, the memory consumption and latency during inference
031   for causal attention transformer models like Llama (Touvron et al., 2023; Dubey et al., 2024), GPT
032   (Radford et al., 2019), and Gemini (Team et al., 2023) increases linearly with the input length. This
033
      causes problems for use cases such as Retrieval Augmented Generation (RAG) (Lewis et al., 2020),
      where the input to the models can become very long (Leng et al., 2024).
034
035   An important component of the Transformer architecture whose memory footprint grows with model
036   size and input length is its KV cache. When generating the next output token, the transformer model
037   processes all the tokens in its context through the attention mechanism. For causal attention models,
038   the internal representation of the previous tokens in the context is unaffected by the newer tokens,
039
      and hence it can be cached. This is stored in the KV cache, and its size increase with context
      length (since it caches information for each token seen so far) and with the size of the model (since
040
      there is a separate KV cache for each KV head in the model). Larger KV cache not only means
041
      more memory consumption by the model, but it also slows down inference because for long inputs,
042   LLM inference can be dominated by the I/O cost of moving the KV cache from HBM to the GPU’s
043   shared memory. Thus, it has become imperative to reduce the size of the KV cache for faster and
044   cost-effective inference with modern LLMs.
045
      Several methods have been proposed for reducing the KV cache size including sparse attention meth-
046
      ods (Beltagy et al., 2020), reducing the number of KV heads (Ainslie et al., 2023; Shazeer, 2019),
047
      KV quantization (Hooper et al., 2024), inference-time cache sparsification through token eviction
048   (Zhang et al., 2024), or even replacing some of the attention layers with State Space Machine (SSM)
049   layers (Lieber et al., 2024). Most of these methods are compatible with others, for example using
050   GQA with Sliding Window Attention (Jiang et al., 2023), using GQA with quantization (Hooper
051   et al., 2024; Lin et al., 2024), or interleaving SSM layers with Sliding Window Attention layers
052   (Lieber et al., 2024; Ren et al., 2024). In this paper, we explore such a combination proposed by
053   Character.AI where they combine Sliding Window Attention with KV cache sharing across layers
      (Character.AI, 2024). We train and evaluate several variants of this architecture, and find that differ-

      1.1           C ONTRIBUTIONS
076
077   We find that KV cache sharing between layers and adding sliding window layers speeds up inference
078   and reduces inference memory usage while maintaining model quality, although some eval metrics
079   show some degradation (Figure 1). In addition, our ablation experiments show the following:
080
081                 • Having a few standard attention layers is crucial for the model’s long context abilities. In
082
                      particular, having the standard KV cache computed in the deeper layers is more important
                      for long context abilities than the standard KV cache of the first few layers.
083
084                 • KV cache of standard attention layers can be shared between non-consecutive layers with-
085
                      out any observed degradation in long context abilities.
086                 • Increasing the KV-cache sharing between sliding window layers too much also hurts the
087                   long-context abilities.

089   2       R ELATED W ORK
090
091   Reducing the KV cache size has been an area of active research, with many different approaches. In
092   this section we talk about some of them.
093
094   Linear Attention and SSM Models. Transformer models (Vaswani, 2017) differ from traditional
095   Recurrent Neural Networks (RNNs) (Sherstinsky, 2020) and modern State Space Models (Gu et al.,
096   2022; Ren et al., 2024) in that Transformer models have an internal representation (the KV cache)
097
      that grows linearly with the length of the input. This allows RNNs and SSMs to be faster and more
      memory efficient during inference. However, it has been seen that while such models are competitive
098
      with Transformer models on certain tasks, Transformer models still outperform equally-sized pure
099
      RNN or pure SSM models on other tasks, especially some long context tasks (Waleffe et al., 2024).
100   Thus, hybrid architectures which interleave attention layers and SSM layers have been proposed,
101   that show that such hybrid architectures exhibit good long context abilities (Lieber et al., 2024;
102   Ren et al., 2024). Other works have linearized the attention mechanism by replacing the softmax
103   operation with kernelized similarity computation, showing both speed and memory improvements
104   for inference (Katharopoulos et al., 2020).
105
106   KV Quantization. KV quantization works by reducing the precision of the cached key-value
107   (KV) pairs which reduces the overall storage requirements and improves the data movement effi-
      ciency during inference (Lin et al., 2024). Hooper et al. (2024) combined several novel methods for


                                                                                                     2
      Under review as a conference paper at ICLR 2025


108
      quantizing the KV cache to achieve significant improvements in long-context language model per-
109   formance, achieving context lengths up to 10 million tokens while maintaining performance metrics
110   close to those of unquantized models.
111
112   KV Eviction. KV eviction dynamically remove less relevant or older tokens from the KV cache
113   during inference to reduce its size. This approach ensures that only the most pertinent tokens are
114   retained, helping alleviate memory bottlenecks in long-context tasks. Zhang et al. (2024) proposed
115   Heavy-Hitter Oracle (H2 O), an efficient generative inference approach that selectively evicts tokens
116   from the cache based on their relevance, significantly improving the performance of models operat-
117   ing on large contexts. Chen et al. (2024) introduced NaCl which generalizes H2 O and adds random
118
      token eviction to retain long context performance while evicting tokens.
119
      KV Head Reduction. Architectures like Multi-Query Attention (MQA) (Shazeer, 2019) and
120
      Grouped Query Attention (GQA) (Ainslie et al., 2023) show that the number of KV heads in the
121
      attention layer can be decreased without significantly impacting model performance. Multi-Query
122   Attention (MQA) simplifies the standard Multi-Head Attention mechanism (Vaswani, 2017) by shar-
123   ing the key and value projections across all attention heads in a layer while retaining independent
124   queries. This approach drastically reduces the size of the KV cache, as fewer unique key-value
125   pairs are stored during inference. However, when serving models on multiple GPUs using Tensor
126   Parallelism (TP) (Shoeybi et al., 2019), the single key and value cache must be replicated across
127   the tensor parallel ranks, thus essentially losing a significant fraction of the memory savings gained
128   from using MQA. Hence, Grouped Query Attention (GQA) extends the same idea of query sharing
129   but instead of having one set of keys and values for all the queries, this architecture partitions queries
130
      into multiple sets and shares keys and values within each set, where the number of sets (and hence
      the number of keys and values) often matches the TP rank.
131
132
      Sparse and Local Attention. Sparse and Local Attention mechanisms have been extensively ex-
133
      plored as a means to improve the efficiency of Transformer models by reducing the quadratic com-
134
      plexity of traditional global attention. Since these methods focus on attending to only a subset of
135   tokens, they reduce the computational and memory costs during both training and inference. One of
136   the most prominent methods in this category is Longformer (Beltagy et al., 2020), which introduces
137   several variants of local attention mechanism including Sliding Window Attention. Sliding Window
138   Attention and its variants restrict the attention of each token to a fixed window of neighboring to-
139   kens, rather than all tokens in the sequence, drastically reducing the number of key-value pairs that
140   need to be stored and processed during inference. This method has been shown to work well but
141   often fails on tasks with long-context dependencies due to the fundamental lack of global attention.
142   Sparse Attention mechanisms further optimize the attention computation by introducing sparsity
143
      patterns, where only certain key-value pairs are attended to based on predefined criteria (Beltagy
      et al., 2020). Notably, GPT-3 used interleaving global and local attention layers in its architecture
144
      (Brown, 2020).
145
146
      KV Sharing. KV Sharing (Brandon et al., 2024; Wu and Tu, 2024) is a key technique used to
147   reduce the memory footprint of Transformer models during inference by allowing multiple layers to
148   reuse the same key-value (KV) instead of having separate KV pairs for each layer. Brandon et al.
149   (2024) demonstrate that cross-layer attention, where KV caches are shared across different layers,
150   leads to substantial memory savings without degrading accuracy.

154   Standard transformer models use global attention in each layer. To create inference-friendly model
155   architectures, we use a combination of sliding window attention layers, standard attention, and KV
156   cache reuse layers. Below is a brief discussion on each component:
157
158   Sliding Window Attention Layers (Beltagy et al., 2020): In Sliding Window Attention (or Local
159   Attention) with window size s, the query only pays attention to the last s keys instead of all the keys
160   preceding it. This means that during inference, the KV cache size needs to only store the KV
161   tensors for the past s tokens instead of storing the KV tensors for all the preceding tokens. In our
      experiments, we set a window size of s = 1024 tokens.

187   4     E XPERIMENTS
188
189   4.1   T RAINING
190
191   We used LLM Foundry (Mosaic, 2023b) to train MixAttention models. Similar to prior work on
192
      training long context models [5, 6], we followed a multi-stage training procedure to impart long
      context abilities to the models.
193
194         1. We pretrained the models with a RoPE theta of 0.5M on 101B tokens, where each sequence
195            sequence has been truncated to 4k token length.
196
            2. To increase the context length, we then trained the model on 9B tokens on a mix of natu-
197
               ral language and code data, where the sequences have been truncated to 32k tokens. We
198            increased the RoPE theta to 8M for this stage. When training at 32k context length (i.e.,
199            this step and the next step), we trained only the attention weights and froze the rest of the
200            network. We found that this delivered better results than full network training.
201
            3. Finally, we trained the model on a 32K-length, synthetic, long-context QA dataset [5, 8].
202
203
                  • To create the dataset, we took natural language documents and chunked them into
                     1k-token chunks. Each chunk was then fed to a pretrained instruction model and the
204
                     model was prompted to generate a question-answer pair based on the chunk. Then, we
205
                     concatenated chunks from different documents together to serve as the “long context.”
206                  At the end of this long context, the question-answer pairs for each of the chunks were
207                  added. The loss gradients were computed only on the answer parts of these sequences.
208               • This phase of training was conducted on 500M tokens (this number includes the tokens
209                  from the context, questions, and answers). The RoPE theta was kept at 8M for this
210                  stage.

212   4.2   E VALUATION
213
214   The models were evaluated on the Mosaic Evaluation Gauntlet v 0.3.0 (Mosaic, 2023a) to measure
215   model quality across various metrics including reading comprehension, commonsense reasoning,
      world knowledge, symbolic problem solving, and language understanding. To evaluate the models’


                                                        4
      Under review as a conference paper at ICLR 2025


216
      long context abilities, we used RULER (Hsieh et al., 2024) at a context length of 32000 tokens.
217   RULER is a composite benchmark consisting of 13 individual evals of the following types:
218
219          • Needle-in-a-haystack (NIAH): These types of evals hide a single or multiple keys and val-
220            ues in a long text, and the model is evaluated on its ability to retrieve the correct value(s)
221            from the long context for a given key(s).
222          • Variable Tracking (VT): This eval provides the model with a long context containing vari-
223            able assignment statements, and the model is tasked to figure out which variables have a
224            particular value by the end of all the variable assignments.
225
             • Common and Frequent Word Extraction (CWE and FWE): These tasks ask the model to
226            extract the most common or frequent words from the text.
227
             • Question Answering (QA): Given a long context, the model is asked a question from some-
228
               where in the context and the model is evaluated on whether it can correctly answer that
229
               question.
230
231   We used SGLang (Zheng et al., 2023) to deploy our models on 1 NVIDIA H100 GPU to run RULER
232   and get inference speed and memory consumption metrics.

398   6    C ONCLUSION
399
400   We find that MixAttention models are competitive with standard attention models on both long-
401   and short-context abilities while being faster during inference and supporting larger batch sizes. We
402   note that on some long context tasks like Variable Tracking and Common Word Extraction, neither
403   MixAttention nor standard attention models perform well. We believe this was because our models
404   weren’t trained long enough or the models need a different kind of long context data to be trained
405
      for such tasks. More research needs to be done to measure the impact of MixAttention architectures
      on such metrics.
406
407   We encourage others to explore more MixAttention architectures to learn more about them. Below
408   are a few observations to help with further research:
409
410
             • Adding a standard attention layer in the initial layers by itself does not seem to help long
               context abilities (for example, see MA-NoShare-1 in the appendix), even if the KV cache
411
               from that layer is reused in layers deeper into the network (MA and MA-EndSlide). Hence
412
               we recommend placing the first standard attention layer deeper in the network (like MA-
413            Offset) or having multiple standard attention layers, at least one of which is computed at a
414            deeper layer (like MA-Pairs).
415
             • Sliding window layers also contribute to the model’s long context abilities. Increasing the
416
               KV cache sharing amongst the sliding window layers worsened long context abilities (MA-
417            Offset-SlideShare and MA-Pairs-SlideShare). For that reason, we think that the 2-3 sharing
418            pattern in sliding window layers (Character.AI, 2024) seems to strike a good balance.
419
             • Sharing standard attention KV caches between consecutive layers gave mixed results, with
420
               slightly worse accuracy on long context QA tasks (see the appendix).
421
             • In our experiments, MA-Offset and MA-Pair showed great speedup and memory savings
422
               during inference, while also maintaining long and short context abilities. Hence, MA-
423
               Offset and MA-Pairs might be good configurations for further research.
424
425   In general, there is a large hyperparameter space to explore, and we look forward to seeing a variety
426   of new strategies for reducing the cost of inference via combinations of sliding window attention
427   and KV cache reuse.
