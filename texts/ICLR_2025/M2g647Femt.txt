009                                               A BSTRACT
010
011            Inverse design tasks are an important category of problem in which we want to
012            identify some input vector x satisfying some desirable properties. In this paper we
013            propose a mechanism for representing inequality constraints as Signed Distance
014            Functions (SDFs). SDFs permit efficient projection of points into the solution re-
015            gion as well as providing a mechanism for composing constraints via boolean set
016            operations. In this paper, we provide theoretical motivation for Signed Distance
017            Functions (SDFs) as an implicit representation of inequality constraints. Next, we
018
               provide analysis demonstrating that SDFs can be used to efficiently project points
               into solution regions. Additionally, we propose two novel algorithms for comput-
019
               ing SDFs for wide families of machine learning models. Finally, we demonstrate
020
               practical utility by performing conditional image generation using MNIST and
021            CelebA datasets, and computational drug design using the ZINC-250K dataset.
022            From the experimental results, we note that the composable constraints can reli-
023            ably and efficiently compute solutions to complex inverse design tasks with deep
024            learning models.

033   tion f (x) : RN → R and constraint functions C(x) : RN → {False, True}M , inverse design and
034
      constrained optimization tasks correspond to finding solutions to equations 1 and 2 respectively,

      By imposing limitations on the forms that C(x) and f (x) can take, methods such as Linear Pro-
      gramming can provide strong bounds in terms of computational performance. However, the simple
040
      models allowed by such algorithms are not suitable for complex settings that require more sophis-
041
      ticated modelling. Conversely, machine learning models have shown success on a wide variety of
042   complex inverse design tasks, such as drug design and conditional image generation. For exam-
043   ple, machine learning use learned emulator models for target properties, and learned feature spaces
044   for parameterizing designs when solving inverse design tasks. However, while the introduction of
045   complex machine learning models provides avenues for more sophisticated modelling and design,
046   it comes at a cost. In particular, the non-linear nature of machine learning models, coupled with
047   high-dimensional input spaces, presents a significant challenge to efficiently solving inverse design
048   tasks and constrained optimization problems in these settings. As a result, current methodologies
049   suffer from a variety of limitations such as poor computational performance, limitations on the types
050
      of constraints, or requiring expensive retraining when the constraints are changed.

052   we propose a mechanism for representing and efficiently solving systems of composable constraints.
053   In particular, by representing inequality constraints as Signed Distance Functions (SDFs), we can

      Constrained Optimization Methods such as linear and quadratic programming impose strict re-
102   strictions on the form of C(x) and f (x) in order to efficiently compute solutions to equation 2. Both
103   linear and quadratic programming restrict the constraints to linear equalities C(X) : Ax = B and in-
104   equalities C(X) : Ax ≥ B. Furthermore, linear programming requires a linear objective f (x) : c⊤ x
105   (Karloff (2008)) while quadratic programming requires a quadratic objective f (x) : 21 x⊤ Qx + c⊤ x
106   (Floudas & Visweswaran (1995)). Several algorithms and corresponding computational complexity
107   bounds exist for solving linear and quadratic programs (Vaidya (1989); Ye & Tse (1989)), allowing
      us to reason about performance in various settings.

      Conversely, non-linear optimization makes very few assumptions on the structure of either the ob-
109   jective or constraint functions. While such methods are powerful and flexible, performance and
110   convergence remains a challenge depending on the algorithm chosen (Yuan, 1991; Moritz et al.,
111   2016). In the context of this paper, augmented Lagrangian constrained optimization algorithms are
112   of particular interest, as they perform well in high-dimensional spaces (Nocedal & Wright, 1999).
113   In particular, we use augmented Lagragians in order to construct a linear-time algorithm for com-
114   puting SDFs in high dimensional spaces. Subsequently, we use linear and quadratic programming
115   to construct an SDF algorithm for certain neural network architectures.

117   Guided Gradient Descent Guided gradient descent (GGD) aims to solve inverse design tasks by
118
      explicitly inverting a predictive model. To perform GGD, we first encode the inverse design task into
      a differentiable loss function. A typical loss function L(x) might be the L2 norm between the model
119
      output M (x) and some desired outputs y ∗ : L(x) = ||M (x) − y ∗ ||2 . By performing gradient descent
120
      through the model, we optimize the objective in the input space to identify solutions to equation 1.
121   Specifically, we apply the update rule xt+1 = xt + ∇xt L(xt ) until convergence.
122
123   GGD can be used for flexible post-hoc generation and can accommodate a wide variety of mod-
124
      els and objectives. Additionally GGD has shown success in tasks such as drug design (Gómez-
      Bombarelli et al., 2018). However, GGD frequently produces adversarial attacks when applied
125
      naively (Goodfellow et al., 2014). In this paper, we use GGD as a baseline to which composable
126
      constraints are compared.

146   We now can introduce the concept of composable constraints as a principled approach for solving
147   inverse design tasks in deep-learning settings. In this section, we describe expressing and solving

151   3.1   E XPRESSING C ONSTRAINTS AS S IGNED D ISTANCE F UNCTIONS
152
153
      First, let us recall the definition of a Signed Distance Function (SDF) from equation 4. In addition to
      the definition, we note the intuition that an SDF is an implicit representation of a given solid volume.
154
      Given a constraint of the form Mi (x) ≥ ki , the region in which the constraint is satisfied is in fact
155
      a solid volume. Consequently, it is possible to represent the solution region volume implicitly using
156   an SDF. From a more formal point of view, the level set Mi (x) = ki represents the solution region
157   boundary and corresponds exactly to ∂S from equation 4. From there, finding the closest point
158   x′ ∈ ∂S, ∂S = {x : Mi (x) = ki } yields the signed distance. The SDF representing the solution
159   to a constraint is referred to as the constraint SDF (cSDF). See figure 1 for a visualization of the
160   cSDF. Next, given multiple cSDFs, we can combine them algebraically using the boolean SDF CSG
161   operations defined in equations 5, 6 and 7. Combining multiple SDFs via boolean operations will
      yield yet another SDF, thus allowing for arbitrary constraints to be composed.

208   4   A LGORITHMS FOR C OMPUTING S IGNED D ISTANCE F UNCTIONS
209
210   As described in section 3, we can express and solve complex systems of constraints using SDFs
211   as an implicit representation of the solution region. However, this approach requires that the SDF
212   can be computed for non-linear models in high-dimensional spaces. Unfortunately, current methods
213   for computing SDFs are either overly restrictive in the types of models which can be used, or are
214   intractable in high-dimensional spaces (Molchanov et al., 2010; Lu et al., 2018; Fuhrmann et al.,
215   2015; Wu & Kobbelt, 2003; Huang & Wang, 2010; Ottaviani & Sodomaco, 2020). Consequently,
      novel SDF algorithms are required. In this section, we show that by imposing mild requirements on


                                                         4

216
      the model form, that it is possible to derive efficient algorithms for computing the cSDFs. Notably,
217   we present a linear-time algorithm for computing cSDFs for a wide class of smooth functions, as
218   well as a heuristic-search based algorithm for computing cSDFs of neural networks with arbitrary
219   continuous piece-wise linear activation functions.
220
221   4.1   S IGNED D ISTANCE F UNCTIONS S MOOTH A SYMPTOTIC FUNCTIONS
222
223   To compute the SDF, we need to solve equation 9. To this end, if we can efficiently enumerate all
224   potential solutions regions, then we can then identify the nearest point on the solution boundary, thus
225   computing the SDF. Unfortunately, enumerating the solution regions remains a difficult problem.
226   However, if a function is continuously differentiable and asymptotic such that lim M (x) =
                                                                                          ||x||2 →∞
227   c, |c| < ∞, then each solution region to the constraint Mi (x) ≥ k, k > c or Mi (x) ≤ k, k < c
228   must necessarily contain at least one extrema. Consequently, by enumerating the extrema of such
229   a function, we can enumerate the single-constraint solution regions and thus compute the SDF. We
230   formalize this intuition in Theorem 1 and appendix C.
231
232   Theorem 1 Let M : Rn → R, such that the image of M is continuous, continuously differentiable
233   and asymptotic such that lim M (x) = c and a ≤ M (x) ≤ b, a, b ∈ R, ∀x ∈ RN . Let k ∈ R
                                ||x||2 →∞
234   where c ̸= k and a ≤ k ≤ b. Then under the constraint M (x) ≤ k or M (x) ≥ k, a search algorithm
235   need only search among the critical points and local extrema of M to compute the Signed Distance
236   Function.

251   Theorem 1 provides an avenue for implementing an efficient cSDF algorithm for a wide family of
252   models. In fact, the requirement that the function is continuously differentiable and asymptotic is
253
      quite permissive and can be easily satisfied by a variety of machine learning models. However,
      the theorem also requires that the extrema of the function can be efficiently enumerated. For most
254
      models, enumerating the extrema poses a difficult challenge. To address this challenge, we make
255
      use of Shepard Interpolation Neural Networks (SINNs) as defined in section 2. First, we show that
256   SINNs are continuously differentiable and asymptotic (see appendix C). Next, we note that SINNs
257   permit a convenient mechanism for enumerating the extrema. In fact, the entries of the learnable
258   parameter matrix B represent the coordinates of the extrema of the output of the corresponding
259   SINN model (see appendix C).
260
      Consequently, SINNs satisfy the requirements of theorem 1, as well as providing an efficient mech-
261
      anism for enumerating the extrema. Thus, we can apply algorithm 1 to SINNs in order to efficiently
262   compute the cSDF. The full details are given in algorithm 1 and appendix C, with computational
263   complexity analysis in M.

      Given a neural network with a continuous piece-wise linear activation function f (x), the output of
290
      the neural network is itself piece-wise linear and continuous. Thus, the ReLU network is separable
291
      into discrete linear domains (Zhang et al. (2018)). Subsequently, given a linear domain, it is possible
292   to solve a quadratic programming sub-problem to find the closest solution within the linear domain.
293   By enumerating all domains, we could then compute the cSDF of the neural network. However,
294   recent results from tropical geometry indicate that the number of linear domains grows combinato-
295   rially with the network depth, and as such it is intractable to enumerate all the domains (Zhang et al.
296   (2018)). However, in order to compute the SDF we need only find the nearest solution, rather than
297   enumerating all possible solutions. Consequently, performing a local search would allow the search
298   space to be greatly reduced and thus improve the performance of a cSDF algorithm. Thus, if we
299   can enumerate neighbouring linear domains of a given starting point, we could attempt to perform a
300
      local search such as Breadth-First Search (BFS).
301   In particular, we need two operations for a piece-wise linear SDF algorithm: 1) given a starting
302   point x0 , efficiently identify the linear domain to which it belongs and 2) given a linear domain, find
303   adjacent linear domains. Using these two operations, we can discretize the piece-wise linear neural
304   network into a graph of connected linear domains, which we then traverse to find the closest solution
305   satisfying the given constraint. For example, 2 demonstrates a 2D example of a linear-domain based
306
      local search. Furthermore, see appendix D for the derivation of both operations as well as the full
      ReLU SDF algorithm, and computational complexity analysis in M.

505   molecular property not available in the original training set, GGD would require jointly retraining
506   the VAE and regressor on the new property, while composable constraints would allow us to simply
507   add a new predictive model for the property of interest, and use the current models as-is.

      6   R ELATED W ORKS
510
511
      In recent works, a variety of generative models, such as Variational Auto-encoders (VAEs), Gen-
512
      erative Adversarial Networks (GANs) and diffusion models have achieve impressive results in a
513   variety of inverse design tasks such as conditional image generation Karras (2017); Rombach et al.
514   (2022) and drug discovery Gómez-Bombarelli et al. (2018); Szymczak et al. (2023). Notably, VAEs
515   can perform unconditional (Blei et al. (2017)) or conditional generation (Lim et al. (2018)). While
516   not fundamental to composable constraints, we make use of VAEs for dimensionality reduction to
517   improve computational performance and sample quality. Similarly, GANs are typically used for con-
518   ditional (Mirza & Osindero (2014)) or unconditional (Karras (2017)) image generation. However,
519   unlike VAEs, GANs do not produce a semantic latent space and thus are not applicable to com-
520   posable constraints. Alternatively, diffusion models are a powerful class of models used in image
521
      generation and chemical design (Rombach et al. (2022); Bohan (2024); Ho et al. (2020); Ingraham
      et al. (2023)). Unfortunately, diffusion models do not produce semantic latent spaces, and thus are
522
      not easily combined with composable constraints.
523
524
525   7   C ONCLUSION
526
527   In this paper, we propose a novel method for constructing and solving composable constraints on
528   deep learning models using Signed Distance Functions (SDFs). We provide theoretical motivation
529
      for the approach, two novel algorithms for computing SDFs in high-dimensional spaces, and empir-
      ically validate the proposed methodology on conditional image generation and computational drug
530
      design tasks. Overall, composable constraints provide a principled framework for flexible, post-hoc
531
      conditional generation.
532
533
      Limitations First, we note that composable constraints require a computable SDF function, lim-
534   iting the applicability of this method to model families with known SDF functions. Additionally,
535   the algorithms for computing the SDF are more complex and computationally intensive than GGD,
536   particularily for the ReLU SDF algorithm, and thus may not be suitable for very large models. See
537   appendices G, H and I for algorithm runtime results. However, solutions are typically computed in
538   a few seconds, and thus are fast enough to be useful in practice. Finally, composable constraints are
539   novel, and thus are poorly characterized. In particular, the impact of the predictive model form on
      the agreement rate is not well understood and would require further work to fully characterize.

      The proof in this appendix aims to support Theorem 1 as well as provide confidence in the theoretical

      Proof of Theorem 1 Let M : Rn → R, such that the image of M is continuous, continuously
926   differentiable and asymptotic such that lim M (x) = c and a ≤ M (x) ≤ b, a, b ∈ R, ∀x ∈ RN .
927                                            ||x||2 →∞
928   Let k ∈ R where c ̸= k and a ≤ k ≤ b. Then under the constraint M (x) ≤ k or M (x) ≥ k, a
929   search algorithm need only search among the critical points and local extrema of M to compute the
930
      Signed Distance Function. See appendix F for proof.
931   For some constraint threshold k such that a ≤ k ≤ b ∈ R where M (x) ≤ k s.t. c > k or
932   M (x) ≥ k s.t. c < k, then the boundary M (x) = k is bounded by k − ϵ ≤ M (x) ≤ k + ϵ for
933   0 < ϵ. Consequently, since M (x) is bounded by two planes k−ϵ, k+ϵ and M (x) is asymptotic, then
934   the boundary M (x) = k must enclose a finite region. Given that the solution region of the constraint
935   is thus bounded and finite, we can conclude by Rolle’s theorem that each finite and bounded region

978    Proof of Algorithm 1 The intuition for algorithm 1 is that we enumerate the extrema of the model
979    which satisfy the given the constraint M (x) ≥ k or M (x) ≤ k, and then perform constrained
980    optimization to find the closest point on the boundary. Then, given the minima on each boundary,
981
       we can select the global minima and thus compute the cSDF.
982    In algorithm 1, augmented Lagrangians are used to perform the constrained optimization to com-
983    pute the SDF. In fact, augmented Lagrangians can be used to solve constrained optimization under
984    equality constraints of the form M (x) = k or inequality constraints of the form M (x) ≥ k, using
985    only the Jacobian, allowing for linear scaling.
986
987    Assumption It is not known in general if the global minima of the solutions to equation 9 is
988    always found among the local minima p1 of solutions generated by constrained optimization among
989
       the extrema e. However, a SINN of a single node yields a flat surface, and any additional local
       curvature of the hyper surface is the result of the additional nodes. As such, it may be possible to
990
       construct a proof by induction that all local curvature is due to a combination of nodes, and thus
991
       any local minima would occur in the neighbourhood of a p1 . On the other hand, in the case that
992    the statement does not hold true, the SDF algorithm will produce an upper bound pseudo-SDF.
993    Computing an upper bound is still acceptable in practice, since a solution to the qSDF can still be
994    found via gradient descent as evidenced by the results of the case studies.
995
996    Corollary For the case that M (x) ≥ k s.t. c ≥ k or M (x) ≤ k s.t. c ≤ k, the constraint crosses
997    the asymptote of M (x) and thus the solution region will be infinite. However, given the definition
998    of the SDFs, the constraints M (x) ≤ k and M (x) ≥ k share the same boundary ∂S. Thus, if
999    the constraint crosses the asymptote, we can substitute with M (x) ≤ k = ¬(M (x) ≥ k). The
1000   substitution produces the same SDF, while avoiding issues with infinite solution regions, allowing
1001   theorem 1 to still apply.
1002   For any point p that does not satisfy the constraint, all solution regions can be found by searching
1003   among the critical points of the SINN. However, for initial points p that are within the solution
1004   region, we must search points outside of the solution region to identify the boundary. In order to
1005   identify the boundary from within a region, we can construct an additional constraint to identify
1006   a complement set of solutions from the original constraint, and prove that there must exist critical
1007   points outside of the solution region and would thus be able to identify the solution region boundary
1008
       from within via the same algorithm. For a constraint M (x) ≤ k s.t. c < k, we can construct an
       alternative constraint M (x) ≥ k + ϵ s.t. 0 < ϵ. Similarly, for the case M (x) ≥ k s.t. k ≤ c we can
1009
       construct the alternative constraint M (x) ≤ k + ϵ s.t. 0 < ϵ. From these alternative constraints, we
1010
       can demonstrate that for any constraint, that there exists a complement solution region, such that at
1011   least one critical point will exist outside of the initial solution region. Thus the algorithm must only
1012   search among the critical points of the function to identify the boundary and thus the cSDF, even if
1013   the starting point is initially within the solution region.
1014
       In conclusion, all the solution regions are guaranteed to contain at least one critical point and thus
1015
       all solution regions will be found when searching among the critical points of a model meeting the
1016
       requisite properties. Next, the local solutions p1 are computed by augmented Lagrangian for each
1017   extrema ei in order to produce the set of all of the solutions to equation 9. Thus, all solutions will
1018   be found and algorithm 1 will correctly compute the cSDF.

1144   point x0 . Consequently, we need to perform a local search of nearby cells to find solutions. In
1145   order to perform the local search, we need to efficiently enumerate neighbouring cells. From the

       M     C OMPUTATIONAL C OMPLEXITY OF SDF A LGORITHMS
1837
1838
       M.1    SINN SDF C OMPLEXITY
1839
1840   As can be seen from algorithm 1, computing the SDF requires enumerating the extrema of the func-
1841   tion, as well as performing constrained optimization on each extrema. As shown in algorithm 1,
1842   augmented Lagrangians are used to compute the SDF. Augmented Lagrangians are a family of non-
1843   linear constrained optimization techniques which only require computing the Jacobian, and thus
1844   scale linearly in the number of variables Nocedal & Wright (1999). Additionally, augmented La-
1845
       grangians scale linearly with the number of neurones in an SINN model. Thus, given M neurons and
       N dimensions, the computational complexity of Augmented Lagrangian is O(M N ). Additionally,
1846
       we repeat the process for each of the M neurons, for a computational complexity of O(M 2 N ) for
1847
       algorithm 1.
1848
1849
       M.2    R E LU SDF C OMPLEXITY
1850
1851   As described in Algorithm 2, there are three major components of the computational complexity of
1852   the ReLU SDF: 1) the complexity of the local search algorithm (such as A*), 2) the complexity of
1853   solving linear programs for enumerating the regions and 3) the complexity of solving a quadratic
1854   program to obtain the actual SDF value.
1855
       In the context of signed distance functions, we want to minimize the L2 distance from a point
1856   x0 ∈ RN to the solution set of some given constraints. Since the L2 norm is quadratic, we can
1857   rewrite the SDF of a linear model with linear constraints as a quadratic objective. Given a linear
1858   model M (x) = Ax + b, we impose the solid boundary ∂S as an equality constraint of the form
1859   Ax + b = k, and derive the SDF quadratic objective in equation 32.
1860
1861                    ||x − x0 ||2 = (x − x0 )T (x − x0 )

                                        T

1865                                  1

1868
1869   In general, solving non-convex quadratic programs is NP-Complete Vavasis (1990). However, if
1870   the problem is convex, it can be solved in polynomial time. Given N variables, m constraints
1871   and L bits, the computational complexity of a quadratic program is O(N 4 L2 ) Ye & Tse (1989).
1872   Since the objective from equation 32 is convex, we can efficiently compute the signed distance
1873   function. Furthermore, several software packages such as CVXOPT and quadprog implement
1874
       quadratic programming solvers, typically using either interior point, active set or augmented lan-
       grangian based algorithmsCaron et al. (2024). Additionally, Vaidya provides worst-case bounds
1875
       for various Linear Programming algorithmsVaidya (1989). Given m constraints, N variables and
1876
       a factor L = log2 (1 + detm ax) + log2 p + log2 (m + N ), three algorithms are provided with
1877   computational complexities of O(m1.5 N L), O((mN 2 + m1.5 N )L) and O(m3 L).
1878
1879   The computational complexity of the A* is O(md ), where d is the depth, and m is the number of
1880
       neighbours of a given cell. For a given neurons, there are at most 2 action domains adjacent to
       the current activation domain. Thus, a given activation configuration and n neurons, there are at
1881
       most 2n neighbouring configurations. Thus, for the search algorithm we obtain a complexity of
1882
       O(nd ). Furthermore, for N input variables, and M constraints, the linear problem is at best solved
1883   in O(M 1.5 N L). The number of constraints is twice the number of neurons, such that M = 2n,
1884   thus the linear sub-problem is solvable in O(n1.5 N L). Finally, the quadratic sub-problem must
1885   only be solved once, in O(N 4 L2 ) time. Thus, the overall complexity of the ReLU SDF algorithm is
1886   O(nd n1.5 N L+N 4 L2 ). Notably, the complexity is dominated by the nd term. However, with proper
1887   hyper-parameters, the depth d required to find a solution is quite small. This is the key advantage of
1888   using local search to compute SDFs in this context, as finding the solution minimizing the distance
1889   necessarily limits the depth to a small local neighbourhood.
