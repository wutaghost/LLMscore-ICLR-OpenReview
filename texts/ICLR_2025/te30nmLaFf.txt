009                                                A BSTRACT
010
011            For text-based AI systems to interact in the real world, causal reasoning is an
012            essential skill. Since active interventions are costly to execute, we study to what
013            extent an agent can learn causal reasoning from symbolic demonstrations of causal
014            axioms. Specifically, we consider an axiomatic training setup where an agent learns
015            from multiple demonstrations of a causal axiom (or rule), rather than incorporating
016            the axiom as an inductive bias or inferring it from data values. A key question
017            is whether the agent would learn to generalize from the axiom demonstrations to
018
               new scenarios. For example, if a transformer model is trained on demonstrations
               of the causal transitivity axiom over small graphs, would it generalize to applying
019
               the transitivity axiom over large graphs? Our results, based on a novel axiomatic
020
               training scheme, indicate that such generalization is possible. For the transitivity
021            axiom, we find that a 67 million parameter transformer model, when trained on
022            linear causal chains (along with some noisy variations) can generalize well to
023            new kinds of graphs, including longer causal chains, causal chains with reversed
024            order, and graphs with branching; even when it is not explicitly trained for such
025            settings. We extend axiomatic training to a harder task of inferring causation from
026            correlation statements and find similar generalization. On both tasks, our model
027            performs at par (or even better) than many larger language models such as GPT-4,
028            Gemini Pro, and Phi-3. Overall, our axiomatic training framework provides a new
029
               paradigm of learning causal reasoning in language models that can be extended to
               arbitrary axioms, as long as sufficient demonstrations can be generated.

      1    I NTRODUCTION
033
034
      Causal reasoning can be defined as a set of reasoning procedures consistent with pre-defined axioms
035
      or rules that are specific to causality (Galles & Pearl, 1997). For instance, d-separation and rules of do-
036   calculus (Pearl, 2009b) can be considered as axioms and specifications of a collider or a backdoor set
037   can be considered as rules that can be derived from axioms. Typically, causal reasoning is done over
038   data corresponding to variables in a system. Axioms or rules are incorporated as inductive biases in a
039   machine learning (ML) model, through regularization, model architecture, or the choice of variables
040   for a particular analysis. Depending on the kind of available data—observational, interventional,
041   or counterfactual—Pearl’s ladder of causation (Bareinboim et al., 2022) defines the kinds of causal
042   reasoning that is possible.
043   As axioms are the building blocks of causality, we study whether it is possible to directly learn the
044   axioms using ML models. That is, rather than learning from data that is the result of axioms obeyed
045   by a data-generating process, what if a model can learn an axiom (and thus causal reasoning) directly
046   from symbolic demonstrations of the axiom? This question gains relevance as language models make
047   it possible to learn over symbolic data expressed in natural language. In fact, recent studies have
048   evaluated whether large language models (LLMs) can do causal reasoning by creating benchmarks
049   that encode causal reasoning problems in natural language (Kıcıman et al., 2023; Jin et al., 2024a;b).
050
      If we can teach causal axioms to a language model, such a model can be used as a verifier to evaluate
      output of existing LLMs, or as a reward model to finetune a given LLM for causal reasoning.
051
052   Specifically, we propose a new way of learning causal reasoning through axiomatic training. We
053   posit that causal axioms can be expressed as the following symbolic tuple, ⟨premise, hypothesis,
      result⟩ where hypothesis refers to a causal claim and premise refers to any relevant information

054
       to decide whether the claim is true or not (conclusion). The conclusion could simply be “Yes” or
055   “No”. For example, consider the task of inferring causal relationships from correlational statements
056    about a set of variables, which we empirically study in this paper. As in the Corr2Cause dataset
057    from Jin et al. (2024b), the premise can be statements about statistical (in)dependence: “Suppose
058    there is a closed system of three variables: A, B and C. B is correlated with C. A is correlated
059    with C. However, A is independent of B.”; the hypothesis can be a question about cause-and-effect,
060   “Does A directly cause C?”; and the conclusion would be “Yes”. This tuple is a demonstration
061    of the collider property/axiom (Pearl, 2009b), which states that if there exist variables such that
062    A⊥  ⊥ B, B ̸⊥⊥ C, A ̸⊥⊥ C, then there is a unique causal graph connecting them, A → C; B → C.
063
       Based on this template, our key insight is that a large number of synthetic tuples can be generated,
       e.g., by changing the variable names, changing the number of variables, changing the order, and so
064
       on. The question is: if a model is trained on such data, would it learn to apply the axiom to new
065
       scenarios?
066
067   To answer this question, we first train a transformer model from scratch on symbolic demonstrations
068   of the causal transitivity axiom (Galles & Pearl, 1997; Sadeghi & Soo, 2024). To evaluate gener-
069
      alizability, we train on simple chains of the causal irrelevance axiom of size 3-6 nodes and test on
      multiple different aspects of generalization, including length generalization (chains of size 7-15),
070
      name generalization (longer variable names), order generalization (chains with reversed edges or
071
      shuffled nodes), and structure generalization (graphs with branching). We find that a model trained
072   on simple chains generalizes to applying the axiom multiple times over larger chains, but it is unable
073   to generalize to the more complex scenarios like order or structure generalization. However, when we
074   train a model on a combined dataset of simple chains and chains with some edges randomly reversed,
075   we find that the model generalizes well across all kinds of evaluation scenarios, including graphs
076   with branching. Our 67 million parameter model outperforms billion-scale models like Gemini Pro,
077   Phi-3, and in some cases GPT-4, for both zero-shot and multi-shot settings. Extending the findings
078   on length generalization for NLP tasks (Kazemnejad et al., 2023; Bhattamishra et al., 2020; Haviv
079   et al., 2022; Furrer et al., 2021), we find a critical role of positional embedding in ensuring causal
080
      generalization across length and other aspects. Our best model has no positional encoding, although
      we find that sinusoidal encoding also works well for some scenarions.
081
082   Baed on these results, we apply the axiomatic training approach to the problem of inferring causality
083   from correlational statements under the Corr2Cause setting (Jin et al., 2024b), as described above.
084   Using the same method to generate synthetic training data and train the model, we find that a
085   transformer trained on task demonstrations over 3-4 variables learns to solve this task for graphs with
086
      5 variables. On this task too, our model’s accuracy is higher than larger LLMs such as GPT-4 and
      Gemini Pro, even when multi-shot examples are provided to the LLMs.
087
088   Our work provides a new paradigm of teaching models causal reasoning through symbolic demon-
089   strations of axioms, which we call axiomatic training. Such symbolic data can be cheaply generated
090   for multiple axioms and added to the pretraining or finetuning data for language models, as done by
091   Papadimitriou & Jurafsky (2023) for language structure constraints. The data generation and training
092
      procedure is general and can be applied to learn any new axiom (e.g., logical axioms), as long as it
      can be expressed in the symbolic tuple format. More generally, our results contribute to the literature
093
      on causal learning from passive data (Lampinen et al., 2023), showing a general way to learn any
094
      causal axiom through passive demonstrations.

100
      (LLMs) have highlighted their potential for knowledge-driven causal discovery. Unlike traditional
101
      methods which focus on statistical patterns or correlations, LLMs utilize knowledge acquired through
102   their pretraining to reason about and identify causal structures based on metadata of variables (Kıcı-
103   man et al., 2023; Ban et al., 2023; Long et al., 2023; Willig et al., 2022; Vashishtha et al., 2023).
104   However, possibility of memorization of existing benchmarks in the pretraining of these LLMs has
105   been a major criticism. As a result, recent work (Zečević et al., 2023) argues that LLMs are not
106   actually performing causal reasoning, but simply learning correlations about causal facts. In addition,
107   there are critical failure modes of using LLMs for causal discovery due to hallucinations or not
      obeying the acyclic constraint when generating graph edges (Vashishtha et al., 2023). To evaluate

128
129   Instead of performing causal reasoning using observational or interventional data, we study whether
130   it is possible to learn general rules of causality directly from symbolic axioms. We begin by asking
131   the question “are there any minimal sufficient characterization of causal principles?”. There has
132   been fundamental work from Galles & Pearl (1997) where they axiomatize causal relevance (or
133   equivalently irrelevance). They show that for a given stable probabilistic causal model (defined
134   below), there exists a finite set of axioms that are completely characterized by axioms of path
135   interception in corresponding directed graphs. We study how such causal relevance statements can be
136
      incorporated into transformer models. Throughout this work, we assume no unobserved confounders.
137   Notation. We denote a random variable with the upper case letter letters (e.g. X, Y, Z and use lower
138   case letters (e.g. x, y, z) to denote the value taken by the corresponding random variable denoted
139   as X = x, Y = y, Z = z. We represent the probability of a random variable using Xi by P(Xi ).
140   Let G(X, E) be a directed acyclic graph (DAG) consisting of a set of variables X = {X1 , . . . , Xn }
141   and a set of directed edges E among variables in X. Let pa(Xi ) = {Xk |Xk → Xi }, de(Xi ) =
142
      {Xk |Xk ← · · · ← Xi }, ch(Xi ) = {Xk |Xi → Xk } denote the set of parents, descendants and
      children of Xi respectively. Next, we define some special notations in the context of three variables.
143
      Given two nodes Xi , Xj we call a third node Xk to be a collider if both Xi and Xj are parents of
144
      Xk . Xk is called a mediator if one of the nodes, say, Xi is parent and the other node Xj is child of
145   Xk . Lastly, the Xk is called a common cause of both Xi and Xj are children of Xk .
146
147   Definition 3.1 (Causal Irrelevance, adapted from Defn. 7 in (Galles & Pearl, 1997)). X is prob-
148
      abilistically causally irrelevant to Y given Z, written (X ↛ Y |Z) iff: P(y|z, do(X) = x) =
      P(y|z, do(X) = x′ ) , ∀x, x′ , y, z i.e., once we hold Z fixed at z, intervening on X will not change the
149
      probability of Y.
150
151   Next, we restate the stability assumption for a causal model from Galles & Pearl (1997) that gives a
152   richer set of finite axiomatization for probabilistic causal irrelevance.
153
154
      Assumption 3.1 (Stability, Definition 9 in Galles & Pearl (1997)). Let M be a causal model. Then
      an irrelevance (X ↛ Y |Z) in M is stable if it is shared by all possible probability distribution over
155
      M. The causal model M is stable if all of the irrelevances in M are stable.
156
157   Under the stability assumption (see Assumption 3.1), Galles & Pearl (1997) states six axioms that
158   completely characterize causal irrelevance (Definition 3.1) via axioms of path interception in the
159   directed graphs. An axiom of causal irrelevance is of the form (given in conjunctive normal form):

180   3.1   A XIOMATIC T RAINING : DATASET, L OSS F UNCTION , AND P OSITIONAL E NCODING
181
182   Training data. Based on a specific axiom, we can map a hypothesis given the premise to its correct
183   label (‘Yes’ or ‘No’). To create a training dataset, we enumerate all possible tuples of {(P, H, L)}N
184   where P is the premise, H is the hypothesis and L is the label (Yes/No) for a particular setting of
185   the variables X, Y, Z, A. Given a premise P based on a given causal graph, if the hypothesis can be
186   derived by applying the specified axiom (once or multiple times), then label L is Yes; otherwise, No.
187
      For example, for the transitivity axiom, suppose the underlying true causal graph of a system is a chain,
      X1 → X2 → X3 → · · · → Xn . Then, a possible premise could be X1 → X2 ∧ X2 → X3 , and
188
      the corresponding hypothesis X1 → X3 will have label Yes whereas another hypothesis X3 → X1
189
      will have label No. The above axiom could be inductively applied multiple times to generate more
190   complex training tuples.
191
192   Loss function. Given a dataset, the loss function is defined based on the ground truth label for each
193
      tuple, represented as    E       − log(P (L|P, H)). A preliminary analysis indicated promising
                             P,H,L∼Ptrain
194   results with this loss formulation compared to next token prediction loss.
195
      Positional Encoding. In addition to the training data and loss function, recent work (Kazemnejad
196
      et al., 2023) has shown that the choice of positional encoding is important for generalizing a
197   transformer to longer or complex inputs. Therefore, we experiment with different positional encoding
198   to understand their impact on generalization in causal tasks: learnable position encoding (LPE),
199   sinusoidal positional encoding (SPE), no positional encoding (NoPE). See Appendix D for details.

              This can be expressed simply through natural language like: “X causes Y. Z causes Y."

242   3.3   E VALUATION SETUP : A SSESSING A XIOMATIC L EARNING IN T RANSFORMERS
243
      To evaluate if a trained model has learnt the correct understanding of an axiom instead of shortcuts or
244
      correlation-based features, designing an out-of-distribution (OOD) evaluation set is important. We
245
      evaluate our model on multiple types of complex structures that are unseen during training.
246
247   1. Length: Evaluating whether our model accurately infers causal relationships for sequences or
248      chains (both sequential and ones with random flipping) longer than those in the training set.
249   2. Node Name Shift: Testing the model’s performance on longer node names, from 1-3 characters
250      used in the training set to 8-10 characters. This is motivated by Jin et al. (2024b) who show how
251      change in node names results in generalization failure on causal tasks for language models.
252   3. Order of Chains: a) Completely reversed chains: This evaluation is inspired by the reversal
253      curse (Berglund et al., 2024) that revealed generalization failure of LLMs in answering questions
254      in reversed sequences despite knowing the answers in the original order. We evaluate our model
255      on completely reversed chains, a structure that was not in the training data. A completely reversed
256
         chain will be of the form X ← Y ← Z, written in natural language as: “Y causes X. Z causes Y.",
         where X, Y, Z are replaced by random alphanumeric names. b) Shuffling of Sequences: Causal
257
         sequences with random edge flips, as defined in 3.2 represented by natural language statements
258
         sequentially (A causes B. B causes C ...), are shuffled to add complexity and break sequential
259      order. This tests model’s ability to infer accurate relationships regardless of sequence order.
260
      4. Branching: We also evaluate on complex graphs beyond chains, measured using the branching
261      factor: Number of edges/Number of nodes. While the training set comprises simplistic sequences,
262      this evaluation setup involves multiple branches, colliders, forks, and chains in one network, thus
263      having significantly high complexity.

269   generated using the transitivity axiom. Each instance in D is structured in the form of a premise
      P , which is the natural language expression of a causal structure (e.g., “X causes Y . Y causes

      Z”), followed by the hypothesis in the form of a question Hq (e.g., “Does X cause Y ?”), which
271   is then followed by the final label L (e.g., “Yes” or “No”). Formally, each premise Xi takes the
272   form: Xi = [ejk | j > 0, k > 0]. Here ejk represents an edge between node j and k in the ith
273   causal sequence (can also be represented as Xi,j , Xi,k ) such that each ejk translates to "Xi,j causes
274   Xi,k ." in natural language.) For each premise, all possible hypotheses consisting of two variables are
275   generated (i.e., “Does A cause B” and “Does B cause A” for each pair). The training data consists of
276   graphs of lengths from [3,6], with branching factor [0.6, 0.8], and length of nodes [1,3]. In addition to
277   sequential chains, random flipping of edges is done with 0.5 probability. See Appendix E for details
278   on these hyperparameters.

280   evaluate the impact of different data perturbations, each with equal number of ’Yes’ and ’No’ labels.
281
      • Training Setup 1 (TS1): This setup comprises of 73k chains with random flipping and 101k
282
        sequential linear chains. Since flipping is done randomly across all consecutive pairs of nodes in the
283
        given chain, some complete reversals are also formed. In this training set around 12k completely
284     reversed chains are present. However, only when evaluating for reversal chain test setup, we remove
285     all the reversed chains from this training set and re-train the model from scratch with the remaining
286     sequential and randomly flipped chain sequences.
287   • Training Setup 2 (TS2): This setup comprises of more simple sequential chains (132k), while we
288     decrease chains with random flipping (42k), to keep the overall size around 175k.
289   • Only Causal Chains (OCC): This set comprises of sequential transitivity chains without any edge

293   • Length Generalization EvalSet: Testing on causal sequences with length >6 upto 15, longer than
294     any sequence encountered by the model in training set. Length generalization is evaluated for both
295     sequential chains and chains with randomly flipped edges.
296   • Node Name EvalSet: Assessed model’s generalization capabilities to sequences with longer node

463   5     A PPLICATION 2: I NFER C AUSATION FROM C ORRELATION S TATEMENTS
464
      While the above study evaluated transformers’ capability to generalise the transitivity axiom from
465   small causal chains to large graphs, we now study whether this capability transfers to a more general
466   causal task. To this end, we apply axiomatic training to a task on inferring causation from statements
467   about correlation in observational data (Jin et al., 2024b). Each data instance in the benchmark
468   includes correlational relationships described in natural language for graphs with 3 to 6 nodes; the
469   goal is to infer the truth value of a hypothesis (of six types: Parent, Ancestor, Child, Descendant,
470   Collider, Confounder) among a set of variables. This task is significantly harder than applying the
471   transitivity axiom. First, there are multiple hypothesis types to evaluate: direct effect, indirect effect,
472   children, ancestors, colliders and confounders. Second, solving the task requires an understanding of
473
      d-separation (refer d-separation in section C) and the Markov property (refer C.1). Specifically, it
      involves mapping correlational statements to multiple possible causal graphs and determining if the
474
      query is satisfied across all graphs in the Markov Equivalence Class (refer C.1).
475
476   Task example: To infer causal relationships from correlational statements, we use d-separation to
477   determine conditional independence, which helps infer the causal graph’s skeleton. For example,
478   given A → B → C, we infer that A is independent of B given C. With this, a model can evaluate
479
      hypotheses, like "Is B a collider?", and conclude that it’s false, as the structure is either a chain or a
      fork. The task follows a Premise-Hypothesis-Label structure, where the premise are the correlational
480
      statements, the hypothesis asks relationships between nodes, and the label provides final answer.
481
482   Model Training: We use the same model architecture as in Section 4.2 and train our model from
483   scratch for 100 epochs using NoPE, since it performed consistently well across diverse OOD settings
484   in our transitivity based experiments. For creating a train set, we consider the subset of the original
485
      dataset with correlational statements for graph consisting of 3 and 4 nodes. As the test set, we evaluate
      the model’s performance directly on 5 node correlational statements.

      6    D ISCUSSION AND C ONCLUSION
513
514
      In this work, we provide a method to create training data containing diverse demonstrations of an
      axiom and explore modelling choices to learn the axiom. A transformer model trained from scratch
515
      on a large axiomatic dataset can learn to apply axioms effectively. On causal tasks like graph traversal
516
      via transitivity and inferring causal relationships from correlation, small 67M transformers generalize
517   well to unseen complex graphs, often outperforming models like GPT-4, Phi-3, and Gemini Pro.
518   Extending to more complex tasks of (Jin et al., 2024b), our robust training enables the small model to
519   match or exceed the performance of larger models.
520
      Applicability to Causal Tasks. While our current work focuses on the transitivity axiom for causal
521
      relevance, extending the work to other causal axioms from (Galles & Pearl, 1997) is an interesting
522
      research direction. In addition, we may consider other axioms that are relevant for downstream tasks.
523   For example, if a transformer model can be trained to validate the d-separation rule—given two
524   variables X and Y, are they independent given a variable set Z?, then repeated applications of the rule
525   can be used to derive a valid backdoor set.
526   Generalization to Logical Reasoning. While our axiomatic training approach focuses on causal
527   reasoning, it can be applied to any formal system such as deductive logical reasoning. Recent
528   work (Saparov et al., 2023) highlights LLMs’ deterioration in performance as reasoning depth
529   increases. It would be interesting to explore if axiomatic training can improve deductive reasoning in
530   LMs, given the similarity between our setup and such tasks.
531
      Implications for Training Language Models. GPT-4 demonstrates impressive generalization on
      the causal tasks we evaluated. We hypothesize that axiomatic training may explain GPT-4’s ability
532
      to reason over causal graphs, as (noisy) demonstrations of the underlying axioms could be present
533
      in its web-scale training data. Meanwhile, models like Gemini Pro and Phi-3 struggle with zero-
534   shot reasoning for causal tasks, such as handling completely reversed chains, suggesting room for
535   improvement. Incorporating causal axiom demonstrations as a part of language models’ pretraining
536   (or finetuning data) could help improve the reasoning of these models, so that small language models
537   like Phi-3 can achieve GPT4-like accuracy on causal tasks. Incorporating axiomatic inductive biases
538   could aid language models reasoning abilities for the desired task. For instance, (Papadimitriou &
539   Jurafsky, 2023) propose pretraining language models on synthetic formal languages to incorporate
      inductive biases, to improve performance on learning grammatically diverse languages.

      C     P RELIMINARIES AND N OTATIONS
935
936
      Causal Models Let M = (X, U , F) be a causal model defined over a set of endogenous variables
      X, exogenous variables U and the causal relationship between then defined by set of structural
937
      equations F (Galles & Pearl, 1997). Let G be the causal graph associated with the causal model
938
      M where the nodes V in G correspond to the variables in M and an edge Vi → Vj between any
939   two nodes Vi , Vj denote the causal relationship between them. The causal relationship of node Xi is
940   characterized by the functional relationship fi ∈ F s.t., xi = fi (pai , ui ). Here pai are the parent
941   of the node Xi is the corresponding causal graph G and ui ⊆ U are set of exogenous variables
942   influencing the exogenous variable Xi . In our work, we assume that there are no hidden confounders
943   so we have one exogenous variable corresponding to every endogenous variable i.e. ui = ui . Each
944   exogenous variable has an associated probability distribution which quantifies the uncertainty in the
945   system i.e. ui ∼ P(ui ). Thus the joint distribution of the exogenous variable is given by P(U ). Since
946   any endogenous variable is a deterministic function of other endogenous and exogenous variables
947
      the probability distribution corresponding to the endogenous variable is the push-forward of the
948   exogenous variable i.e P(X) ≜ P# (U ).
949
950   d-separation The causal graph G encodes the conditional independence in the corresponding
951
      probability distribution using path separations statements called d-separation. Two sets of random
      variable Xi and Xj are conditionally independent of each other conditioned on Xz if all the
952
      undirected path between Xi and Xj in G are blocked by Xz . A path between Xi and Xj is blocked
953
      if there exists a node A ∈ Xz s.t. it satisfies one of the following conditions: 1. A is a common cause
954   in the path (i.e. · ← A → ·), 2. A is a mediator in the path (i.e. · → A → ·), or 3. A is not a collider
955   (i.e. · → A ← · ) or descendent of any collider in the path (see Pearl (2009a) for details.)

       D    P OSITIONAL E NCODINGS AND THEIR ROLE IN G ENERALIZATION
978
979
       Positional Encoding (PE) play a crucial role of providing information about the absolute and relative
       position of tokens in a sequence (Vaswani et al., 2023). (Vaswani et al., 2023) propose an absolute
980
       positional encoding strategy using periodic functions (e.g., sinusoidal or cosine) to initialize these
981
       encodings. Absolute positional encoding provides definite values for all positions across any sequence
982    length. However, studies (Ontañón et al., 2022; Csordás et al., 2021) show absolute positional
983    encoding fails in length generalization tasks for transformers. In the learnable APE variant (Radford
984    et al., 2018), each positional embedding is randomly initialized and trained with the model. This
985    approach falters with sequences longer than those seen in training, as the new positional embeddings
986    remain untrained and randomized. Interestingly, recent findings (Kazemnejad et al., 2023; Haviv
987    et al., 2022) indicate that removal of PEs in auto-regressive models can improve model’s length
988    generalization capabilities, wherein the attention mechanism during auto-regressive decoding is
989    sufficient to encode positional information.
