                                                  A BSTRACT
010
011
               In this paper, we propose a method to automatically select hyperparameters to
012
               compress 3D Gaussians to a target file size while maximizing visual quality. We
013
               iteratively search for a hyperparameter configuration until the file size meets the
014            specified budget. However, existing compression frameworks require complet-
015            ing the entire compression process to determine the compressed file size, which
016            is time-consuming. To accelerate this, we design a tailored size estimator for
017            frameworks that can determine hyperparameters without requiring fine-tuning.
018            Although the finetuning-free frameworks are more predictable, they typically un-
019            derperform compared to fine-tuning-based approaches, which utilize end-to-end
020            differentiable structures to achieve superior results. To close this performance gap,
021            we propose a mixed-precision quantization strategy that exploits the heterogene-
022
               ity of attribute channels by compressing each channel with different bit-widths.
               The resulting combinatorial optimization problem is efficiently solved using 0-1
023
               integer linear programming. Additionally, we partition each attribute channel into
024
               blocks of vectors, quantizing each vector based on the optimal bit-width deter-
025            mined in the previous step. The block length is then determined via dynamic pro-
026            gramming. Our method identifies hyperparameter settings that meet the target file
027            size within 70 seconds, outperforming state-of-the-art methods in both efficiency
028            and quality. Extensive experiments demonstrate that our approach significantly
029            enhances the performance of fine-tuning-free methods, with its upper-bound per-
030            formance comparable to that of fine-tuning-required techniques.

033   1   I NTRODUCTION
034
035   In recent years, 3D Gaussian Splatting (3DGS) (Kerbl et al., 2023) has emerged as a popular research
036   topic due to its excellent quality and real-time rendering speed in novel view synthesis. 3DGS rep-
037   resents a 3D scene using a set of neural Gaussians initiated from Structure-from-Motion (SfM) with
038   learnable attributes such as color, shape, and opacity. The 2D images can be effectively rendered
039
      using differentiable rasterization and end-to-end training is enabled. Meanwhile, benefiting from
      efficient CUDA implementation, real-time rendering is achieved. Despite its success, 3DGS still en-
040
      counters limitations in storage efficiency. For example, 5.27106 Gaussians are required to represent
041
      the bicycle scene in the Mip-NeRF 360 dataset (Barron et al., 2022), occupying 1.3 GB of storage
042   under 32-bit float precision. This sizable file poses challenges in transmission and storage. Hence, a
043   tailored codec for 3D Gaussians is required.
044
045
      Current 3D Gaussian compression frameworks can be categorized into two types: finetuning-
      free (Xie et al., 2024; Niedermayr et al., 2024; Fan et al., 2023; Papantonakis et al., 2024) and
046
      finetuning-required (Lee et al., 2024; Chen et al., 2024; Morgenstern et al., 2024; Papantonakis et al.,
047
      2024; Wang et al., 2024). The finetuning-free approach allows direct compression of a fully-trained
048   3DGS using given hyperparameters, without any additional training. In contrast, the finetuning-
049   required approach involves training a module from scratch as part of the compression process, such
050   as learning a pruning mask (Lee et al., 2024) or a context model (Chen et al., 2024). To compress
051   the 3DGS to meet a specific size budget while achieving optimal visual quality, we have to itera-
052   tively search for a hyperparameter configuration until the file size meets the size budget. However,
053   neither of these frameworks cannot support such an operation because they require completing the
      entire compression process to determine the final file size. For example, finetuning-free methods

      Under review as a conference paper at ICLR 2025


054
      like MesonGS typically take about one minute to complete compression, while finetuning-required
055   methods can take up to 20 minutes, as they need to learn a pruning mask from scratch to remove
056   Gaussian points. In addition, the file size fluctuates during the finetuning process, making the final
057   size harder to predict. As a result, we chose MesonGS (Xie et al., 2024), a finetuning-free method,
058   as the foundation of our approach and designed a tailored size estimator for it.
059
      As ScaffoldGS (Lu et al., 2024) became a mainstream basemodel for 3D Gaussian compression, we
060
      first migrate MesonGS to ScaffoldGS. Then, we design a size estimator tailored to the ScaffoldGS-
061   based MesonGS, which can predict the final size with negligible latency under a given hyperparam-
062   eter setting. Based on the size estimator and the observation of heterogeneity across channels of
063   attributes, we formulate the size-aware mixed-precision quantization as an Integer Linear Program-
064   ming (ILP) problem to find the best bit-precision setting. Compared to the contemporary work (Chen
065   et al., 2024) - HAC, our approach leverages the differences between different channels of attributes,
066   while HAC cheats the different channels of attributes as the same. Furthermore, given the number
067   of blocks, we have to slice each channel of attributes into multiple blocks, we propose a dynamic
068   programming to determine the length of each block. Our main contributions can be listed as follows:
069
             • We propose an estimator to predict the final size of compressed 3D Gaussians before com-
070
               pression. This Size Estimator can help quickly search for hyperparameter settings that meet
071            the size budget while maximizing visual quality.
072
073
             • Based on the estimator, we propose a size-aware hierarchical mixed precision quantization
               scheme. On the inter-attribute level, we formulate mixed bit-width selection as a 0-1 ILP
074
               problem subject to the size. On the intra-attribute level, we propose a high-speed dynamic
075
               programming algorithm to solve the mixed block-length setting inside a channel.
076
077
             • Our method can find hyperparameter settings that meet the size budget and maximize visual
               quality within 70 seconds, which is 100× faster than baselines. Furthermore, our mixed-
078
               precision method addresses the shortcomings of fine-tuning-free approaches, achieving per-
079
               formance comparable to or even better than state-of-the-art results across multiple datasets.

084   2.1   3D G AUSSIAN S PLATTING
085
      3D Gaussian Splatting (Kerbl et al., 2023) is a novel method for reconstructing 3D scenes from 2D
086
      images. It represents a scene using a set of 3D Gaussian distributions, which are small, fuzzy blobs.
087
      These Gaussians capture the density, color, and opacity of the scene. It is faster and less memory-
088   intensive than previous methods like NeRF (Mildenhall et al., 2021) and can produce high-quality,
089   smooth results with fewer artifacts. Recently, a lot of work has been proposed for compressing
090   3D Gaussians. At first, they focused on compressing 3DGS model (Niedermayr et al., 2024; Lee
091   et al., 2024; Girish et al., 2024; Morgenstern et al., 2024; Navaneet et al., 2024; Papantonakis et al.,
092   2024; Fan et al., 2023; Xie et al., 2024). But now, many works pay attention to compressing a more
093   efficient GS model (Fang & Wang, 2024; Lu et al., 2024), in which ScaffoldGS (Lu et al., 2024)
094   became the hot spot. It proposed to divide anchors into voxels and introduce an anchor feature
095   for each voxel to grasp the common attributes of neural Gaussians in the voxel, i.e., the neural
096
      Gaussians are predicted by the anchor features. HAC (Chen et al., 2024) recognized the advantages
      of ScaffoldGS and proposed a tailored compression method for it, extracting a context from the 3D
097
      coordinates to guide the quantization steps and entropy encoding parameters. ContextGS (Wang
098
      et al., 2024) divides anchors into hierarchical levels and encodes them progressively. However,
099   existing compression frameworks for ScaffoldGS often require training from scratch, and thus the
100   size varies greatly over time, making size estimation under given hyperparameters challenging. This
101   paper transfers the relatively easy-to-estimate MesonGS to ScaffoldGS and conducts an in-depth
102   analysis of the compression components of MesonGS to build a nearly delay-free size estimator.
103
104   2.2   M IXED - PRECISION Q UANTIZATION
105
106   Significant efforts have recently been made to improve the trade-off between the accuracy and ef-
107   ficiency of neural networks. A promising direction is to use mixed-precision quantization (Wang
      et al., 2019; Dong et al., 2019; 2020; Yao et al., 2021; Tang et al., 2022). However, the challenge


                                                         2
      Under review as a conference paper at ICLR 2025


108
      with this approach is to find the right mixed-precision setting for the different layers of neural net-
109   works. A brute force approach is not feasible since the search space is exponentially large in the
110   number of layers. HAQ (Wang et al., 2019) proposes to employ reinforcement learning (RL) to
111   search this space. However, this RL-based solution requires tremendous computational resources.
112   HAWQ (Dong et al., 2019; 2020; Yao et al., 2021) proposes first to assign each layer a sensitiv-
113   ity score with the Hessian spectrum and then formulate an ILP solution that can generate mixed-
114   precision settings with various constraints (such as model size, BOPS, and latency). Unlike the
115   challenges faced by neural networks, all attributes of 3DGS are equivalent to the weights in neural
116   networks, and two levels of mixed precision need to be selected: inter-attributes and intra-attributes.
117
      Moreover, 3DGS demands a higher compression speed than neural networks. We identify an oppor-
      tunity to apply mixed precision quantization to attributes of the GS model and propose a hierarchical
118
      scheme to quickly determine the optimal mixed-precision setting for the attribute. Besides, 3DGS
119
      compression has 16 bit options available, far exceeding 2 bit options of HAWQ (INT4 and INT8).
120   Directly using the integer programming formulation from HAWQ3 makes it difficult to achieve good
121   results. To address this, we establish a more general and fast 0-1 integer programming formulation
122   to determine the optimal bit-width for each attribute channel.

      4.1      S IZE E STIMATOR FOR C OMPRESSED 3D G AUSSIAN F ILE
187
188   Estimating the file size for compressed 3DGS is nontrivial. First, the difficulty of size estimation
189   varies across different frameworks. We observe that the more modules that require training, the
190   harder it becomes to estimate the final file size. The learnable masking module proposed by (Lee
191   et al., 2024) requires training; in HAC (Chen et al., 2024), context information extracted from the
192   coordinates (represented using a Hash Grid) needs to be trained from scratch, as well as the hyper-
193   parameters used for entropy coding. These representations, which need to be learned from scratch,
194   result in a compressed file size that is difficult to predict. In contrast, frameworks like MesonGS,
195
      which allow for offline compression, are easier to predict since none of the modules require training.
      As shown in Fig. 2a, the final file size of HAC fluctuates continuously during training, while the
196
      size of MesonGS remains very stable with almost no variation. Therefore, we adopt the MesonGS
197
      framework for compression.
198
199   Since ScaffoldGS is currently the base model of the state-of-the-art 3D Gaussian compression
200   works (Chen et al., 2024; Wang et al., 2024; Bagdasarian et al., 2024), we also adopt it as the
201
      base model for our approach. Hence, we migrate MesonGS onto ScaffoldGS and the compression
      process can be summarized as follows. As illustrated in the middle of Fig. 2, we first calculate the
202
      importance of each anchor point by averaging the importance of Gaussian splats generated by it.
203
      Then, we prune the anchor points based on a hyperparameter pruning percentage τ . The coordinates
204   of the anchor points are voxelized to form an Octree. For anchor points within the same voxel, we
205   average the attributes to ensure that each voxel corresponds to only one anchor point. We denote
206   the number of points that after pruning and voxelization as N . Based on the voxelized coordinates,
207   we apply region-adaptive hierarchical transform (RAHT) to all attributes, which produces the DC
208   and AC coefficients. DC coefficients are stored in float format while the AC coefficients are then
209   quantized in a block-wise manner. We denote the channels of attributes as M . Here, block-wise
210   quantization means dividing each channel’s attributes into K blocks and then quantizing each block
211   into the bit width of 8. Here, elements in a block share the same scale and zero-point parameters. Fi-
212
      nally, all components are compressed by LZ77 (Gailly & Adler, 2003; Ziv & Lempel, 1977; 1978)
      codec as shown in the right side of the Fig. 1. The metadata includes the octree depth d and the
213
      number of blocks K.
214

      model size are pruning percentage τ , octree depth d, and the number of blocks K. First, to simplify

241   4.2                          I NTER -ATTRIBUTE M IXED P RECISION Q UANTIZATION
242
243   Although post-training compression techniques like MesonGS are more convenient for estimating
244   the final size, pre-fixing hyperparameters hinder the model from achieving optimal performance.
245   The core reason lies in the fact that MesonGS uses a uniform quantization precision, which prevents
246   it from obtaining better compression results. As shown in Fig. 2, we observe that different attributes
247   are better suited to varying levels of compression granularity. Moreover, a block strategy with mixed
248   granularity yields better results for the same attribute. Besides, attributes occupy over 90% of the
249
      final storage, further compressing them offers the greatest potential benefit.
250   Uniformly quantizing all the attributes to low-bitwidth (e.g. INT4) could lead to significant quality
251   degradation. However, it is possible to benefit from low-precision quantization by keeping a sub-
252   set of sensitive attributes at high precision. The basic idea is to keep sensitive channels at higher
253   precision and insensitive layers at lower precision. An important component of MPGS is that we
254   directly consider the size metric, to select the bit-precision configuration. Previous methods have
255
      been unable to control the file size of the GS model. For example, given a desired file size, it has
      been difficult to identify the bit configuration that is closest to this size while achieving the optimal
256
      quality. In this work, we formalize the problem as an Integer Linear Programming (ILP) problem.
257
258   Assume that for each attribute channel, there are B quantization options (e.g., 2 options for INT4 or
259   INT8). For the AC coefficients with H attribute channels, the search space of the ILP is B H . The
260   objective of solving the ILP is to find the best bit configuration among these B H possibilities that
261
      optimally balances information loss Ω and the user-specified file size. Each bit-precision setting can
      lead to a different model perturbation. To simplify the problem, we assume that the perturbations of
262
      each channel are independent of one another. This allows us to precompute the information loss of
263
      each channel separately, and it only requires BH computations. For the information loss metric, we
264   use the mean square error between the original attributes and the restored attributes1 . Formally, we
265   can precompute the information loss matrix Ω ∈ RH×B with:

      4.4    S EARCHING AND F INETUNING
331
332   Given a target file size, we first randomly select two τ values and perform the compression process
333   under 8-bit precision to obtain the final file sizes. Then, we substitute these results into the equation
334   size = ατ + β to solve for the parameters α and β, as illustrated in Fig 2b. Afterward, we can
335   solve for the τ corresponding to the target file size. Finally, we employ ILP and DP to determine the
336   bit-width and block length settings.
337
      During fine-tuning, we fix the pruning mask and the 3D coordinate. We noticed that the fine-tuning
338   speed of the MesonGS is relatively slow and found this is due to MesonGS applying quantization
339   block by block. To accelerate this process, we implemented a CUDA kernel to parallelly run the
340   block-wise quantization.

      to worse fine-tuning results compared to keeping the coordinates fixed.

476   In this paper, we propose a fine-grained mixed precision compression work to solve the problem
477   of automatically selecting hyper-parameters to compress 3D Gaussians to a target file size while
478   maximizing the visual quality. We propose several key components, including the selection of base
479   model - ScaffoldGS, the selection of compression framework - MesonGS, the size estimator, the
480   inter-attribute mix-precision quantization, and the intra-attribute mix-precision quantization. Ex-
481
      tensive experimental results validate the effectiveness of our size-aware 3D Gaussians compression
      methodology, showcasing a remarkable improvement in the size control of the 3D Gaussian com-
482
      pression and better compression quality compared to SOTA methods. Our method lays the founda-
483
      tion for providing a more controllable solution for the following transmission or streaming tasks.
