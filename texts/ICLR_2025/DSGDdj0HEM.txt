009                                                      A BSTRACT
010
011                Class Incremental Learning (CIL) presents a major challenge due to the phe-
012                nomenon of catastrophic forgetting. Recent studies on Linear Mode Connectivity
013                (LMC) reveal that Naive-SGD oracle, trained with all historical data, connects to
014                previous task minima through low-loss linear paths—a property generally absent
015                in current CIL methods. In this paper, we explore whether LMC holds for the
016                CIL oracle. Our empirical results confirm the presence of LMC in the CIL oracle,
017                showing that models can retain performance on earlier tasks by following the
018
                   discovered low-loss linear paths. Motivated by this finding, we propose Increment
                   Vector Transformation (IVT), which leverages the diagonal of the Fisher Informa-
019
                   tion Matrix to approximate Hessian-based transformation, uncovering low-loss
020
                   linear paths for incremental updates. Our method is orthogonal to existing CIL
021                approaches, serving as a plug-in with minor extra computational costs. Extensive
022                experiments on CIFAR-100, ImageNet-Subset, and ImageNet-Full demonstrate
023                significant performance improvements when integrating IVT with representative
024                CIL methods.

027   1       I NTRODUCTION
028
029   Class Incremental Learning (CIL) poses a significant challenge in machine learning, requiring models
030   to learn sequentially without access to previous training data. A notorious phenomenon in this
031   paradigm is catastrophic forgetting (McCloskey & Cohen, 1989), where models overwrite previously
032   acquired knowledge when adapting to new tasks. To mitigate this, various approaches have been
033   proposed. Regularization methods (Kirkpatrick et al., 2016; Zenke et al., 2017) constrain updates
034
      to crucial parameters for past tasks or transfer knowledge from previous tasks through intermediate
      features and outputs (Kirkpatrick et al., 2016; Hou et al., 2019; Douillard et al., 2020). Memory
035
      replay methods (Rebuffi et al., 2016; Liu et al., 2020; Luo et al., 2023) retain a subset of exemplars
036
      from previous tasks for rehearsal, selecting representative samples to optimize memory efficiency.
037   Dynamic architecture methods (Liu et al., 2021; Zhou et al., 2022) introduce new network components
038   to accommodate new tasks. However, despite these advancements, incremental models still fall short
039   compared to oracles trained incrementally with access to all historical data.
040
      Recently, key insights into this performance gap have emerged from the studies on mode connectivity
041
      in neural networks (Draxler et al., 2018; Garipov et al., 2018; Frankle et al., 2020). Mode connectivity
042
      refers to the existence of low-loss paths that connect different minima in the loss landscape. In CIL,
043   Mirzadeh et al. (2021) demonstrated that the Naive-SGD oracle exhibits more favorable linear mode
044   connectivity (LMC), meaning that a simple linear manifold of low error connects the Naive-SGD
045   oracle and the minima of past tasks. Following this linear path results in minimal degradation of
046   performance on past tasks. In contrast, this property generally does not hold for incremental solutions.
047   Beyond the Naive-SGD, Wen et al. (2023) explored mode connectivity for recent advanced CIL
048   approaches, and empirically found that LMC is still absent in these methods.
049   In this paper, we further investigate the connection between LMC and CIL by addressing a crucial
050   question: “Does LMC hold for the oracle of a CIL approach?”. The significance of this question
051   lies in its implications: If the CIL oracle1 exhibits LMC, then there must be a transformation to
052   uncover this low-loss linear path for the CIL models. Surprisingly, we empirically demonstrate

107   et al., 2020; Verwimp et al., 2021; Wen et al., 2023). Mirzadeh et al. (2021) empirically demonstrate

238   3.2   INCREMENT VECTOR TRANSFORMATION FOR CIL
239
240   In neural networks with numerous parameters, explicitly computing the full Hessian matrix is
241   often impractical. The Fisher Information Matrix (FIM) (Fisher, 1922; Amari, 1996) is an efficient
242   alternative for Hessian estimation, as it can be directly derived from first-order derivatives. Building
243
      on Proposition 1, we propose a novel method for CIL named Increment Vector Transformation (IVT),
      which utilizes the diagonal of the FIM.
244
245   As is common in existing approaches (Kirkpatrick et al., 2016; Matena & Raffel, 2022; Daheim et al.,
246   2023), we can reduce the computation cost by using the diagonal of the FIM, bringing it to a level
247   comparable to training on N samples. The diagonal of the FIM is computed as follows:
248                                                                     2

249
250   In our implementation, we compute the diagonal of FIM in an online manner by accumulating the
251   backpropagated gradients from each batch during training, leading to negligible computational cost.
252   By replacing the Hessian in Eq. 9 with Eq. 10, we formally define IVT as follows:

425   5     R ELATED WORK
426
427   Class Incremental Learning. Existing CIL methods can be broadly categorized into three main
428   approaches. Regularization methods mitigate catastrophic forgetting by imposing constraints on
429   model parameters or outputs. Approaches like EWC (Kirkpatrick et al., 2016) calculate the importance
430   of parameters for previous tasks and penalize changes to crucial parameters, while knowledge
431   distillation techniques such as LUCIR (Hou et al., 2019), PODNet (Douillard et al., 2020), and GeoDL
      (Simon et al., 2021) use output logits or intermediate features to preserve learned representations.


                                                                         8
      Under review as a conference paper at ICLR 2025


432
      To address class imbalance, methods like BiC (Wu et al., 2019) and FOSTER (Wang et al., 2022a)
433   apply post-hoc corrections and classifier adjustments to reduce bias toward newly introduced classes.
434   Memory replay methods store a subset of exemplars and replay them during new task learning. For
435   instance, iCaRL (Rebuffi et al., 2016) selects samples that best approximate class means, while
436   Mnemonics (Liu et al., 2020) and CIM (Luo et al., 2023) optimize exemplar selection or compression
437   to maximize memory efficiency. When storing real data is infeasible due to privacy or memory
438   constraints, prompt-based methods (Wang et al., 2022c;b), prototype-based approaches (Zhu et al.,
439   2021; 2022), and synthetic data techniques (Choi et al., 2021; Qiu et al., 2024) simulate replay
440   without violating these constraints. Dynamic architecture methods adapt the network structure to
441
      accommodate new tasks by expanding or modifying network components. Approaches like AANet
      (Liu et al., 2021) and MEMO (Zhou et al., 2022) dynamically allocate resources, effectively isolating
442
      new knowledge from previously acquired information. This adaptability balances stability and
443
      plasticity, allowing the model to learn new information flexibly while preserving existing knowledge.
444
445   Mode Connectivity. Mode connectivity is a phenomenon where different minima in the loss
446   landscape of deep neural networks are connected by low-loss paths in the parameter space (Draxler
447   et al., 2018; Garipov et al., 2018). It offers a novel perspective on optimization, suggesting that optima
448
      obtained through gradient-based methods are points on a connected, low-loss manifold. Various
      methods, such as polygonal chains, Bézier curves, elastic bands, and simplicial complexes, have been
449
      used to model these low-loss paths (Draxler et al., 2018; Garipov et al., 2018; Benton et al., 2021).
450
      The initialization of minima plays a crucial role: high-loss ridge often exists along the linear path
451   between minima trained from different initializations, but linear connectivity can be achieved when
452   minima share the same initialization and are stable to SGD noise (Frankle et al., 2020; Neyshabur
453   et al., 2020). Mode connectivity advances our understanding of neural network optimization and
454   facilitates applications in loss landscape analysis, weight pruning, and model ensembling (Draxler
455   et al., 2018; Frankle et al., 2020; Fort & Jastrzebski, 2019).
456
457   6    C ONCLUSION
458
459   In this paper, we investigate whether LMC holds in the CIL oracle and confirm that models can retain
460   performance on earlier tasks by following these low-loss linear paths. Inspired by this finding, we
461   introduce Increment Vector Transformation (IVT), a method that uses the diagonal of the Fisher
462   Information Matrix to approximate a Hessian-based transformation, allowing the discovery of low-
463   loss linear paths for incremental updates. IVT is compatible with existing CIL methods and requires
464   minimal additional computational overhead. Extensive experiments on CIFAR-100, ImageNet-
465   Subset, and ImageNet-Full demonstrate that integrating IVT with state-of-the-art CIL methods leads
466
      to substantial performance improvements.
467
      Limitations. Since IVT is a transformation method based on Hessian information, the accuracy
468
      of Hessian estimation is critical. Our use of the diagonal Fisher Information Matrix approximation
469
      may not achieve high accuracy. Furthermore, as tasks progress, the effectiveness of the accumulated
470   diagonal Fisher Information Matrix stored by IVT may decrease. Updating the Hessian information
471   for past tasks is likely to improve performance. We leave these considerations for future work.

656          • CIFAR-100 is a widely-used image classification dataset, consisting of 60,000 color images

658            designed to represent a distinct object category (e.g., animals, vehicles, everyday objects).
659
               The dataset is split into a training set of 50,000 images, with 500 images per class, and a
               validation (or test) set of 10,000 images, with 100 images per class.
660
661          • ImageNet-subset (ImageNet-100) is a smaller, 100-class subset derived from the larger
662            ImageNet dataset. It is frequently used for tasks like transfer learning and incremental
663            learning, offering a balance between dataset size and complexity. Each class in ImageNet-
664
               Subset contains approximately 1,300 training images and 50 validation images, making it a
               more computationally manageable version of the full ImageNet dataset while still providing
665
               substantial class diversity and variability in visual content.
666
667          • ImageNet-Full (ImageNet-1000) refers to the subset of ImageNet containing 1,000 classes.
668            It is the most commonly used version of ImageNet for tasks such as image classification,
669
               pretraining, and benchmarking deep learning models. This dataset includes around 1.2
               million training images and 50,000 validation images, with approximately 50 images per
670
               class in the validation set. Each class in ImageNet-full represents a distinct object category,
671
               ranging from animals to everyday objects.
