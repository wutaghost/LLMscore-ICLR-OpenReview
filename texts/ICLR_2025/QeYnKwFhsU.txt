                                                  A BSTRACT
008
009
010            This study introduces dataset distillation (DD) tailored for 3D data, particularly
011            point clouds. DD aims to substitute large-scale real datasets with a small set of
012            synthetic samples while preserving model performance. Existing methods mainly
013            focus on structured data such as images. However, adapting DD for unstructured
014
               point clouds poses challenges due to their diverse orientations and resolutions in 3D
               space. To address these challenges, we theoretically demonstrate the importance
015
               of matching rotation-invariant features between real and synthetic data for 3D
016
               distillation. We further propose a plug-and-play point cloud rotator to align the
017            point cloud to a canonical orientation, facilitating the learning of rotation-invariant
018            features by all point cloud models. Furthermore, instead of optimizing fixed-
019            size synthetic data directly, we devise a point-wise generator to produce point
020            clouds at various resolutions based on the sampled noise amount. Compared to
021            conventional DD methods, the proposed approach, termed DD3D, enables efficient
022            training on low-resolution point clouds while generating high-resolution data for
023            evaluation, thereby significantly reducing memory requirements and enhancing
024            model scalability. Extensive experiments validate the effectiveness of DD3D in
025
               shape classification and part segmentation tasks across diverse scenarios, such as
               cross-architecture and cross-resolution settings.

      1   I NTRODUCTION
030
031
032   Dataset distillation (DD) (Wang et al., 2018) aims to distill the knowledge of a large-scale dataset
033
      into a few synthetic samples, where the models trained on the real and synthetic data will have
      comparable performance. By doing so, DD significantly reduces the computational cost of training
034
      neural networks from scratch. Due to its remarkable efficiency and effectiveness, DD has been used
035
      in a variety of domains, such as image (Zhao et al., 2021; Zhao & Bilen, 2023; Cazenavette et al.,
036   2022), video (Wang et al., 2024), text (Maekawa et al., 2023) etc. Despite great progress, existing
037   DD methods only succeed on structured 1D and 2D data, while the distillation of unstructured 3D
038   data, e.g., point cloud, is still under-explored.
039
      Point cloud data exists in large quantities in various fields. For example, MVPNet (Yu et al., 2023)
040
      scans more than 87K point clouds from real-world videos for machine vision, Objaverse-XL (Deitke
041
      et al., 2023) provides more than 10M high-quality 3D assets, and Qu et al. (2022) constructs a 100M
042   dataset for high-energy physics, where particles are modeled as point clouds. Training on these
043   datasets from scratch is time-consuming and resource-intensive, requiring more efficient approaches.
044   However, several reasons prevent existing DD frameworks from generalizing to 3D point clouds.
045
      First, point clouds with different orientations represent the same semantic information, e.g., shapes.
046
      However, existing DD methods do not take the symmetry of data into account, which cannot handle
047
      the randomly rotated data and result in sub-optimal performance. As shown in Figure 1a, directly
048   applying DD to the point clouds with different orientations cannot obtain meaningful synthetic data.
049   Second, point clouds have flexible resolutions, i.e., the number of points, depending on specific
050   models and applications. Generally, a larger resolution encodes more fine-grain information but
051   also increases the computational costs (Huang et al., 2024; Qiu et al., 2021). However, existing DD
052   methods initialize the synthetic data as a fixed-size tensor, which cannot be customized for different
053   point cloud models. Moreover, the memory budget for fixed-size tensors will increase rapidly when
      dealing with dense-resolution scenes, e.g., segmentation (Chang et al., 2015; Ren et al., 2022).

067
068   Once the weaknesses of existing methods are identified, it is natural to ask: How can we build a
069   distillation framework that overcomes the orientation and resolution issues of 3D point clouds?
070
      To answer this question, we first theoretically prove that random rotations weaken the principle
      components of real data, thereby degenerating the distillation performance. Based on this discovery,
071
      we propose DD3D, the first DD framework for 3D point clouds, illustrated in Figure 1b. Specifically,
072
      DD3D first uses a rotator to convert the point cloud into a canonical orientation by learning a rotation-
073   equivariant projection matrix to offset random rotation. Then, the knowledge of rotation-invariant
074   data is distilled into a point-wise generator to predict the point coordinates from noise, where the
075   resolution is based on the number of sampled noises. Finally, the rotator and generator are jointly
076   optimized by minimizing the gradient differences between the real and synthetic data.
077
      The contributions are summarized as follows. (1) We propose the first 3D distillation framework,
078
      DD3D, which can eliminate the influence of random rotations and synthesize point clouds at arbitrary
079   resolutions. (2) We theoretically prove that matching the rotation-invariant features can preserve
080   the principal components of real data and prevent data degeneration. (3) DD3D can be trained with
081   low-resolution point clouds and generates high-resolution data for evaluation, significantly reducing
082   memory usage and enhancing model scalability. (4) Extensive experiments on shape classification
083   and part segmentation tasks validate the effectiveness of DD3D over baselines.

      2       R ELATED W ORK
086
087   Dataset Distillation. Research on DD can be roughly divided into two directions. The first is
088   to explore advanced matching objectives to improve the distillation performance. For example,
089   performance matching (Wang et al., 2018), gradient matching (Zhao et al., 2021; Zhao & Bilen, 2021),
090   distribution matching (Zhao & Bilen, 2023; Wang et al., 2022), trajectory matching (Cazenavette et al.,
091   2022; Guo et al., 2024; Du et al., 2023) and feature regression (Zhou et al., 2022; Loo et al., 2022;
092   Nguyen et al., 2021). On the other hand, some methods innovate efficient data parameterizations
093   to avoid directly optimizing the synthetic data. For example, neural networks (Liu et al., 2022),
094   spectral representation (Shin et al., 2023), linear transformation (Deng & Russakovsky, 2022), and
095
      up-sampling (Kim et al., 2022). Among them, a special parameterization technique is to distill the
      knowledge into a generative model (Zhao & Bilen, 2022; Wang et al., 2023; Zhang et al., 2023;
096
      Cazenavette et al., 2023; Zhang et al., 2024), which can generate diverse synthetic data with unlimited
097
      samples. Although valid, these methods rely on the prior knowledge of generative models pre-trained
098   on large-scale datasets, which is not feasible for point clouds. A recent work1 also applies GM to
099   point cloud data. However, neither of them considers the orientation and resolution issues. For a
100   more comprehensive introduction to DD, please refer to the recent surveys (Yu et al., 2024; Lei &
101   Tao, 2024; Geng et al., 2023; Sachdeva & McAuley, 2023).
102
      Point Cloud Analysis. Deep learning on point clouds plays a vital role in 3D data analysis (Guo
103
      et al., 2021b). Traditional methods can be classified into three categories: Point-based methods,
104   e.g., PointNet (Qi et al., 2017a) and PointNet++ (Qi et al., 2017b), convolution-based methods,
105   e.g., PointCNN (Li et al., 2018) and PointConv (Wu et al., 2019), and relation-based methods, e.g.,
106   DGCNN (Wang et al., 2019) and Point Transformer (Guo et al., 2021a). However, these methods
107

      are rotation-sensitive and cannot handle point clouds with different orientations. Some advanced
109   methods are designed to learn rotation-equivariant or invariant features, such as vector neuron (Deng
110   et al., 2021), spherical harmonic (Poulenard et al., 2019), tensor field (Thomas et al., 2018), and
111   graph features (Kim et al., 2020; Zhao et al., 2019). However, these methods introduce additional

114   leverages the eigenvectors of the covariance matrix to transform point clouds into the direction with
115   maximum variance. But this approach suffers from the sign-ambiguity issue (Xiao et al., 2020; Yu
116   et al., 2020; Li et al., 2021).

166   The above analysis demonstrates that learning rotation-invariant representations is crucial for point

168   solve this problem, we propose a plug-and-play point cloud rotator to transform the point clouds into
169   their canonical view, enabling all methods to learn rotation-invariant representations.
170   Rotation-equivariant. We can leverage the orthogonality of the rotation matrix to eliminate its
171   influence, i.e., RR⊤ = I, where PCA is a typical method:

      4.2    P OINT- WISE G ENERATOR
204
205   In addition to the orientation, point clouds often have different resolutions, which also need to be
206   considered in the distillation process. Traditional DD methods update the synthetic data in an explicit
207   way, i.e., directly optimizing the fixed-size tensors, which is unsuitable for point cloud synthesizing.
208   On the other hand, the implicit neural representation (INR) methods (Sitzmann et al., 2020; Park
209   et al., 2019) show great potential in generating data with arbitrary resolutions (Chen et al., 2021;
210   Singh et al., 2023). Generally, INR predicts the signals of given coordinates, but the coordinates of
211   the synthetic point clouds are unknown.
212   Point Denoising. Our solution is to use INR as a generator g : R → R3 , whose input is random noise
213   and output is the coordinates of a point. This means that we treat the noise as a special continuous
214   coordinate, and the generator is used to obtain “3D signals” by denoising the noise. It is also worth
215   noting that the generator adopts a point-wise paradigm rather than an instance-wise generation.
      Therefore, the amount of points is the same as the sampled noise, which allows us to generate point

306   4.4   D ISCUSSION
307   Complexity. The complexity of DD contains three parts: data parameterization, model forward, and
308   data alignment. For a point cloud with n points, DD3D has an additional time complexity O(Lnd) to
309   generate synthetic data, which makes the time overhead higher than that of vanilla DD methods. But
310   if we consider down-sampling the points, the time complexity of all three parts can be reduced. See
311
      Section 5.5 for a comprehensive comparison of the time and space overhead between DD and DD3D.
312   Limitations. Unlike parameterizing data as an explicit matrix, DD3D leverages a generator to
313   synthesize data, significantly reducing the computational costs and memory budget. However, a
314   major drawback is that the generator cannot take original data as initialization, which may affect the
315   convergence of the model. A comparison can be found in Section 5.4. Moreover, there are still some
316   issues that remain unsolved. For example, existing methods cannot applied to tasks with continuous
317
      labels, such as detection, which limits their applications. Besides, how to make the synthetic datasets
      learn fine-grained details beyond shapes remains a challenge.

      Datasets. We employ three datasets of different scales for the shape classification task: (i) ScanOb-
325   jectNN (OBJ BG) (Uy et al., 2019) is the smallest dataset but consists of real-world data, which is
326   challenging to distillate. (ii) ModelNet40 (Wu et al., 2015) is a larger synthetic dataset generated
327   from CAD models. (iii) MVPNet (Yu et al., 2023) is the largest dataset, containing 87K point clouds
328   scanned from real-world videos. We use its subset MVPNet100, which includes data from the 100
329   most populous categories, to alleviate the influence of long-tail distribution, similar to the CAFIR-100
330   dataset2 . For the part segmentation task, we follow Qi et al. (2017a) and choose ShapeNet-part (Yi
331   et al., 2016) dataset for evaluation. All the datasets use the standard data splits, and their detailed
332   statistic information can be found in Appendix C.
333   Data Preparation and Metrics. Each cloud contains 1,024 points and is normalized into a unit
334   sphere. We consider two settings: Aligned and Rotated. In the Aligned setting, both training and test
335   point clouds have the same orientation, while in the Rotated setting, both training and test data are
336   rotated randomly. For the rotated data, we project them along the direction of maximum variance
337   during pre-processing. Note that the point clouds in MVPNet only have 180◦ views, so we do not
338   randomly rotate them. The details of pre-processing can be found in Appendix C. We report the
339
      Overall Accuracy (OA, %) of each method in the shape classification task and the average class
      intersection of union (IoU, %) in the part segmentation task.
340
341   Baselines. To demonstrate the effectiveness of our method, we choose two types of baselines: (1)
342   Coreset-based methods, including Random, Herding (Welling, 2009) and K-Center (Sener & Savarese,
343   2018). (2) Distillation-based methods, including Gradient Matching (GM) (Zhao et al., 2021),
344   Distribution Matching (DM) (Zhao & Bilen, 2023), and Trajectory Matching (TM) (Cazenavette
345
      et al., 2022). We choose GM as the distillation objection for DD3D as it makes a trade-off between
      time and memory consumption. See Appendix D for the detailed hyperparameters.
346
347   Backbones. We provide a lightweight PointNet as the distillation backbone, which abandons the
348   transformation network because previous literature (Yu et al., 2024) pointed out that complex network
349   architecture may lead to degraded distillation performance. See Appendix E for more details. In
350   the evaluation stage, we adopt various advanced backbones to evaluate the generalization ability
351
      of distilled datasets, including PointNet++ (Qi et al., 2017b), DGCNN (Wang et al., 2019), Point
      Transformer (Guo et al., 2021a), PointMLP (Ma et al., 2022), and PointNext (Qian et al., 2022).

455   5.4                       A BLATION S TUDIES
456
457   Point Cloud Rotator. We first verify the effectiveness of the proposed point cloud rotator on the
458   rotated ModelNet40 dataset. Specifically, we consider three different models: (1) PointNet, which is
459   rotation-sensitive; (2) PointNet + PCA, which is rotation-invariant but sign-variant; (3) PointNet +
460   Rotator, which is rotation- and sign-invariant. It can be observed from Table 5 that the performance
461
      of all methods drops rapidly when the data is randomly rotated. On the other hand, leveraging PCA
      to transform the point clouds into a canonical orientation can significantly improve the distillation
462
      performance. However, the results are still far from the model with the point cloud rotator, which
463
      reflects that sign ambiguity will seriously prevent the distillation model from learning meaningful
464   synthetic data. Finally, it can be observed that the proposed rotator can help point cloud models to
465   rotation-invariant representations, thus benefiting the learning of synthetic data.
466
467
      Point-wise Generator. In Section 4.4, we have discussed the pros and cons of DD and DD3D. Here,
      we make a further attempt to show the advantages of the proposed point-wise generator. Firstly, in
468
      Table 6, we report the results of DD with and without initialization. It is noticeable that initializing
469
      DD with real data is important for the distillation performance. However, its performance is still
470   not as good as DD3D, which does not rely on any initialization. Moreover, the performance of
471   DD3D can still be improved if we use sampling during the evaluation, i.e., generating different point
472   clouds at each epoch, because the generated data serves as data augmentation to improve the model
473   performance. This strategy is more useful when the value of CPC is small. Additionally, in Figure 4,
474   we visualize the matching loss of DD and DD3D. It is observable that DD without initialization has a
475   higher loss value and converges more slowly than DD3D, reflecting the advantages of the proposed
476   point-wise generator in synthetic point clouds.

        6                   C ONCLUSION
535     This paper introduces DD3D for 3D point cloud distillation, which matches the rotation-invariant data
536     distribution between real and synthetic data by transforming point clouds into a canonical orientation.
537     Once trained, DD3D can synthesize point clouds at arbitrary resolutions, reducing memory budget
538     and improving scalability. Extensive experiments on both classification and segmentation tasks
539     validate the superiority of DD3D over traditional DD methods. A promising direction is to initialize
        DD3D with real data to improve its performance.

750   B    I MPLEMENTATION D ETAILS OF DD3D
751
752   Here, we explain some details of DD3D, consisting of two important components: a point cloud
753   rotator and a point-wise generator. Both components are built based on the SIREN (Sitzmann et al.,
754   2020) model, which stacks multiple fully connected layers with sin(·) activation to capture the
755   high-frequency information. The PyTorch code is shown in Algorithm 4, where some details are
      highlighted.

           Under review as a conference paper at ICLR 2025


756
           Algorithm 4 PyTorch code of DD3D
757
758    1   import torch
       2   import torch.nn as nn
759    3   import SIREN
       4
760    5   class Rotator(nn.Module):
761    6      def __init__(self, hidden_dim, w0):
       7         super().__init__()
762    8
       9          # w0 is to adjust the frequency of sine function
763   10          self.sign_encoder = SIREN(1, hidden_dim, w0=w0)
764   11          self.sign_decoder = SIREN(hidden_dim, 1, w0=1.)
      12
765   13       def forward(self, x):
      14          x = x.unsqueeze(-1) # x: [B, N, 3, 1]
766   15
767   16          feat = self.sign_encoder(x).mean(dim=1, keepdim=True) # [B, N, 3, 1] -> [B, 1, 3, d]
      17          feat = self.sign_decoder(feat) # [B, 1, 3, d] -> [B, 1, 3, 1]
768   18          sign = torch.sign(feat) # sign-equivariant
      19
769   20          x = x * sign # [B, N, 3, 1] * [B, 1, 3, 1] -> [B, N, 3, 1]
770   21          return x.squeeze(-1)
