011                                               A BSTRACT
012
013            The incorporation of memory into agents is essential for numerous tasks within the
014            domain of Reinforcement Learning (RL). In particular, memory is paramount for
015            tasks that require the utilization of past information, adaptation to novel environ-
016            ments, and improved sample efficiency. However, the term “memory” encompasses
017
               a wide range of concepts, which, coupled with the lack of a unified methodology
               for validating an agent’s memory, leads to erroneous judgments about agents’ mem-
018
               ory capabilities and prevents objective comparison with other memory-enhanced
019
               agents. This paper aims to streamline the concept of memory by providing precise
020            definitions of agent memory types, such as long-term versus short-term memory
021            and declarative versus procedural memory, inspired by cognitive science. Using
022            these definitions, we categorize different classes of agent memory, propose a robust
023            experimental methodology for evaluating the memory capabilities of RL agents,
024            and standardize evaluations. Furthermore, we empirically demonstrate the impor-
025            tance of adhering to the proposed methodology when evaluating different types of
026            agent memory by conducting experiments with different RL agents and what its
027            violation leads to.
028

030   1    I NTRODUCTION
031
032   Reinforcement Learning (RL) effectively addresses various problems within the Markov Decision
033   Process (MDP) framework, where agents make decisions based on immediately available informa-
034   tion (Mnih et al., 2015; Badia et al., 2020). However, there are still challenges in applying RL to
035   more complex tasks with partial observability.
036   To successfully address such challenges, it is essential that an agent is able to efficiently store and
037   process the history of its interactions with the environment (Ni et al., 2021). Sequence processing
038   methods originally developed for natural language processing (NLP) can be effectively applied
039   to these tasks because the history of interactions with the environment can be represented as a
040   sequence (Hausknecht & Stone, 2015; Esslinger et al., 2022; Samsami et al., 2024).
041
      However, in many tasks, due to the complexity or noisiness of observations, the sparsity of events, the
042   difficulty of designing the reward function, and the long duration of episodes, storing and retrieving
043   important information becomes extremely challenging, and the need for memory mechanisms
044   arises (Graves et al., 2016; Wayne et al., 2018; Goyal et al., 2022). Nevertheless, in the existing
045   literature on RL, where the concept of “memory” is discussed, the definitions of memory are only
046   defined in terms of the specific problem under consideration.
047
      For example, in some works, memory is defined as the ability of an agent to effectively establish
048   and use dependencies between events within a fixed-size sequence of tokens (context) in decision
049   making (Esslinger et al., 2022; Ni et al., 2023; Grigsby et al., 2024). In other works, the term
050   “memory” refers to the agent’s ability to use out-of-context information through the use of various
051   memory mechanisms (Parisotto et al., 2020; Lampinen et al., 2021; Cherepanov et al., 2024). In
052   the context of Meta-Reinforcement Learning (Meta-RL), however, the term “memory” is used to
053   describe an agent’s ability to use experience from other tasks or episodes to adapt to a new, previously
      unknown environment (Team et al., 2023; Kang et al., 2024a; Grigsby et al., 2024).

      To avoid possible cases of ambiguous interpretation of the concept of memory in RL and, as a
055   consequence, the conduct of incorrect experiments, we introduce a formal definition of agent memory
056   in RL, distinguish classes of tasks that require an agent to have memory, and introduce a classification
057   of agent memory types. Furthermore, we propose a formal methodology that can be used to
058   unambiguously test different agent memory capabilities according to our proposed classification.
059
      In summary, our contribution can be described as follows:
060
061         1. We formalize the definition of “memory” depending on the problem to be solved: long-term
062            memory and short-term memory, declarative memory and procedural memory (section 5).
063         2. We introduce a decoupling of tasks that require an agent to have memory into two classes:
064            Memory Decision-Making (Memory DM) and Meta-Reinforcement Learning (Meta-RL)
065            (section 5).
066
            3. We propose a generic experimental methodology for testing the memory capabilities of
067            agents in Memory DM tasks (subsection 5.2).
068
            4. We show that if the proposed experimental methodology is not followed, judgments about
069
               the agent’s memory type can become extremely incorrect (section 6).

072   2    PARTIALLY O BSERVABLE M ARKOV D ECISION P ROCESS
073
074   The Partially Observable Markov Decision Process (POMDP) is a generalization of the Markov
075
      Decision Process (MDP) that models sequential decision-making problems where the agent has
      incomplete information about the environment’s state. POMDP can be represented as a tuple
076
      MP = ⟨S, A, O, P, R, Z⟩, where S denotes the set of states, A is the set of actions, O is the set of
077
      observations and Z = P(ot+1 | st+1 , at ) is an observation function such that ot+1 ∼ Z(st+1 , at ).
078
      An agent takes an action at ∈ A based on the observed history h0:t−1 = {(oi , ai , ri )}t−1       i=0 and
079   receives a reward rt = R(st , at ). It is important to note that state st is not available to the agent at
080   time t. In the case of POMDPs, a policy is a function π(at | ot , h0:t−1 ) that uses the agent history
081   h0:t−1 to obtain the probability of the action at . Thus, in order to operate effectively in a POMDPs,
082   an agent must have memory mechanisms to retrieve a history h0:t−1 . Partial observability arises in a
083   variety of real-world situations, including robotic navigation and manipulation tasks, autonomous
084   vehicle tasks, and complex decision-making problems.
085
086   3    R ELATED W ORKS
087
088   Researchers’ interest in memory-enhanced RL agents is evident in the abundance of works proposing
089   architectures with memory mechanisms and benchmarks (Osband et al., 2019; Morad et al., 2023;
090   Pleines et al., 2023) for their validation (see Appendix B and Appendix D for details). However,
091   despite the rather large number of works devoted to this topic, the term “memory” has multiple senses,
092   and the selection of benchmarks and experiments is not always done correctly.
093
      Thus, for instance, in Oh et al. (2016), memory is understood as the ability of an agent to store
094   recent observations into an external buffer and then retrieve relevant information based on temporal
095   context. In Lampinen et al. (2021), memory is the ability to store and recall desired information at
096   long intervals. In Fortunato et al. (2020), memory refers to working and episodic memory (with
097   short-term and long-term nature, respectively) from cognitive psychology and neuroscience, which
098   allows an intelligent agent to use information from past events to make decisions in the present and
099   future. Ni et al. (2023) describes two distinct forms of temporal reasoning: (working) memory and
100   (temporal) credit assignment, where memory refers to the ability to recall a distant past event at the
101   current time. In Kang et al. (2024b) authors use the concept of reconstructive memory Bartlett &
102
      Kintsch (1995) discovered in psychology, which establishes a reflection process based on interaction.
103

106   Most works related to the concept of memory in RL use various principles from cognitive psychology
107   and neuroscience such as long-term memory (Lampinen et al., 2021; Ni et al., 2023; Grigsby et al.,
      2024), working memory (Graves et al., 2014; Fortunato et al., 2020), episodic memory (Pritzel et al.,

      2017; Fortunato et al., 2020), associative memory (Parisotto & Salakhutdinov, 2017; Zhu et al.,
109   2020), and others to introduce it. Despite the fundamental differences in these concepts, works on
110   memory in RL often simplify these concepts to their inherent temporal scales (short-term memory
111   and long-term memory). Regardless, the temporal scales are often presented qualitatively without
112   clearly defining the boundaries between them. For example, many studies assume that remembering
113   a few steps within an environment represents short-term memory, while remembering hundreds of
114   steps represents long-term memory, without considering the relative nature of these concepts. This
115   ambiguity between short-term and long-term memory can lead to a misattribution of an agent’s
116   memory capabilities and to an incorrect estimation of them when conducting experiments. To address
117
      this ambiguity, in this section we introduce formal definitions of agent memory in RL and its types,
      and propose an algorithm for designing an experiment to test agent memory in a correct way.

      4.1   M EMORY IN C OGNITIVE S CIENCE
121
122
      Human cognitive abilities that ensure adaptive survival depend largely on memory, which determines
123   the accumulation, preservation, and reproduction of knowledge and skills (Parr et al., 2020; 2022).
124   Memory exists in many forms, each of which relies on different neural mechanisms. Neuroscience
125   and cognitive psychology distinguish memory by the temporal scales at which information is stored
126   and accessed, and by the type of information that is stored. Abstracting from this distinction, a
127   high-level definition of human memory is as follows: “memory – is the ability to retain information
128   and recall it at a later time”.
129   The definition aligns with the common understanding of memory in RL. Thus, we will use it to create
130   terminology for various types of agent memory. In neuroscience, memory is categorized by temporal
131   scale and behavioral manifestation. Typically, this leads to a distinction between short-term memory,
132   which retains information for seconds, and long-term memory, which can last a lifetime (Davis &
133   Squire, 1984). Additionally, memory is divided by behavioral manifestations into declarative memory
134   (explicit) and procedural memory (implicit) (Graf & Schacter, 1985). Declarative memories can be
135   consciously recalled, encompassing events and facts, while procedural memories are unconscious
136   and relate to skills like skiing or driving.
137   In the following section, we introduce formal definitions of the above types of memory from
138   neuroscience for RL tasks. Using these definitions, which are written in quantitative terms, we can
139   uniquely classify the type of memory an agent has when using past information in decision making.

142   4.2   M EMORY IN RL
143
144   The interpretation of memory in RL varies across studies. In some POMDPs, agents need to retain
145   crucial information to make future decisions within a single environment. Here, memory typically
146
      encompasses two aspects: 1) the efficiency of establishing dependencies between events within a
      fixed time interval (e.g., transformer context (Esslinger et al., 2022; Ni et al., 2023)); and 2) the
147
      efficiency of establishing dependencies between events outside a fixed time interval (Parisotto et al.,
148
      2020; Cherepanov et al., 2024).
149
150   Based on the neuroscience definitions outlined in subsection 4.1, the first interpretation aligns with
151   short-term memory, while the second corresponds to long-term memory. Both interpretations are also
152
      closely related to declarative memory. In Meta-RL, memory typically refers to an agent’s ability to
      leverage skills from different environments/episodes Team et al. (2023); Kang et al. (2024a), akin to
153
      procedural memory.
154
155   However, many studies fail to differentiate between agents with declarative and procedural memory,
156   often treating Meta-RL tasks as a whole rather than focusing on decision-making based on past
157   information. For instance, when a paper asserts that an agent possesses long-term memory, it may
158
      only be tested on Meta-RL tasks based on MDPs. To clarify the concept of agent memory in RL, we
      provide formal definitions in this section.
159
160   In this paper, we primarily study an agent’s memory, which is used to make current decisions based
161   on past information within the same environment. Accordingly, our focus will be on declarative
      memory, specifically its short-term and long-term forms.

162
      Memory and Credit Assignment. Papers exploring agent memory, particularly declarative mem-
163   ory, often distinguish between two concepts based on the temporal dependencies the agent must
164   handle: memory and credit assignment (Osband et al., 2019; Mesnard et al., 2020; Ni et al., 2023).
165   In Ni et al. (2023), the authors formally differentiate between two forms of temporal reasoning in
166   RL: (working) memory and (temporal) credit assignment: “memory refers to the ability to recall a
167   distant past event at the current time, while credit assignment refers to the ability to determine when
168   the actions that merit current credit occurred” (Ni et al., 2023).
169   While distinct, these concepts both establish different temporal dependencies between related events.
170   In this work, we focus on the agent’s ability to form these dependencies, treating “memory” and
171   “credit assignment” as a single entity. We will use the definition from subsection 4.1 to define memory
172   generally. Notably, the definitions for “memory” also apply to “credit assignment”, as they pertain
173   solely to temporal dependencies rather than their essence.

343   5.2   L ONG - TERM MEMORY IN M EMORY DM
344
345   As defined in Definition 4, Memory DM tasks with short-term memory occur when event-recall
346   pairs in the memory-intensive environment M̃P are within the agent’s context (ξ ≤ K). Here,
347   memory involves the agent’s ability to connect information within a context, regardless of how large
348   K is. Examples include works like Esslinger et al. (2022); Ni et al. (2023); Grigsby et al. (2024).
349
      Validating short-term memory is straightforward by simply setting a sufficiently large context length
      K. However, validating long-term memory capabilities is more complex and of greater interest.
350
351   Memory DM tasks requiring long-term memory occur when event-recall pairs in the memory-
352   intensive environment M̃P are outside the agent’s context (ξ > K). In this case, memory involves
353   the agent’s ability to connect information beyond its context, necessitating memory mechanisms
354   (Definition 6) that can manage interaction histories h longer than the agent’s base model can handle.
355   Definition 6 (Memory mechanisms). Let the agent process histories ht−K+1:t of length K at the
356   current time t, where K ∈ N is agents context length. Then, a memory mechanism µ(K) : N → N
357   is defined as a function that, for a fixed K, allows the agent to process sequences of length Kef f ≥ K,
358   i.e., to establish global correlations out of context, where Kef f is the effective context.
359                                               µ(K) = Kef f ≥ K                                          (5)
360
      Memory mechanisms are essential for addressing long-term memory challenges (processing out-of-
361   context information) in the Memory DM framework.
362
363   Example of memory mechanism. Consider an agent based on an RNN architecture that can
364   process K = 1 triplets of tokens (observations, actions, and rewards) at all times t. By using memory
365   mechanisms µ(K) (e.g., as in Hausknecht & Stone (2015)), the agent can increase the number
366   of tokens processed in a single step without expanding the context size of its RNN architecture.
367   Therefore, if initially in a memory-intensive environment M̃P : ξ > K = 1, it can now be
368   represented as M̃P : ξ ≤ Kef f = µ(K). Here, the memory mechanism µ(K) refers to the RNNs
369   recurrent updates to its hidden state.
370
      Thus, validating an agent’s ability to solve long-term memory problems in the Memory DM framework
371   reduces to validating the agent’s memory mechanisms µ(K). To design correct experiments in such
372   a case, the following condition must be met:

516   7     C ONCLUSION
517
518   In this study, we propose formal definitions of memory tasks arising in the RL domain, inspired by
519   neuroscience: long-term memory (LTM) vs. short-term memory (STM), declarative memory vs. pro-
520   cedural memory. We also distinguish explicitly two classes of POMDPs: Memory Decision-Making
521   (Memory DM) and Meta-Reinforcement Learning (Meta-RL), survey the main memory mechanisms
522   used in such tasks, partition existing Memory DM algorithms according to our classification, and
523
      propose an algorithm for setting up an experiment to test explicitly and correctly an agent LTM and
      STM capabilities within the Memory DM framework. We have also shown that without following
524
      the proposed methodology for conducting an experiment to test LTM and STM, one can obtain
525
      ambiguous results that do not allow us to separate these types of agent’s memory. We hope that our
526   work will bring clarity to the understanding of the concept of memory in RL and advance progress in
527   the field.

889   B    A PPENDIX – M EMORY M ECHANISMS
890
891   In RL, memory has several meanings, each of which is related to a specific class of different tasks. To
892   solve these tasks, the authors use various memory mechanisms. The most prevalent approach to incor-
893   porating memory into an agent is through the use of Recurrent Neural Networks (RNNs) (Rumelhart
894   et al., 1986), which are capable of handling sequential dependencies by maintaining a hidden state
895   that captures information about previous time steps (Wierstra et al., 2010; Hausknecht & Stone, 2015;
896   Sorokin et al., 2015; Duan et al., 2016; Song et al., 2018; Zintgraf et al., 2020). Another popular
897
      way to implement memory is to use Transformers (Vaswani et al., 2017), which use self-attention
      mechanisms to capture dependencies inside the context window (Parisotto et al., 2020; Lampinen
898
      et al., 2021; Esslinger et al., 2022; Melo, 2022b; Team et al., 2023; Pramanik et al., 2023; Robine
899
      et al., 2023; Ni et al., 2023; Grigsby et al., 2024; Shala et al., 2024). State-space models (SSMs) (Gu
900   et al., 2021; Smith et al., 2023; Gu & Dao, 2023) combine the strengths of RNNs and Transformers
901   and can also serve to implement memory through preservation of system state (Hafner et al., 2019;
902   Lu et al., 2023; Becker et al., 2024; Samsami et al., 2024). Temporal convolutions may be regarded
903   as an effective memory mechanism, whereby information is stored implicitly through the application
904   of learnable filters across the time axis (YuXuan Liu & Hsieh, 2016; Mishra et al., 2018). A world
905   model (Ha & Schmidhuber, 2018) which builds an internal environment representation can also be
906   considered as a form of memory. One method for organizing this internal representation is through
907   the use of a graph, where nodes represent observations within the environment and edges represent
908
      actions (Morad et al., 2021; Zhu et al., 2023; Kang et al., 2024b).
909   A distinct natural realization of memory is the utilization of an external memory buffer, which enables
910   the agent to retrieve pertinent information. This approach can be classified into two categories:
911   read-only (writeless) (Oh et al., 2016; Lampinen et al., 2021; Goyal et al., 2022; Cherepanov
912   et al., 2024) and read/write access (Graves et al., 2016; Zaremba & Sutskever, 2016; Parisotto &
913   Salakhutdinov, 2017). Detailed information about each memory mechanism can be found in the
914
      Appendix, Appendix B.
915   Using these memory mechanisms, both decision-making tasks based on information from the past
916   within a single episode and tasks of fast adaptation to new tasks are solved. However, even in works
917   using the same underlying base architectures to solve the same class of problems, the concepts of
      memory may differ.

      B.1   R ECURRENT N EURAL N ETWORKS
919
920
921   Recurrent Neural Networks (RNNs) (Rumelhart et al., 1986) are a class of neural networks designed
922   to process sequential data by maintaining a hidden state that captures information from previous
923   time steps. This hidden state, which is essentially a form of internal memory, allows RNNs to
924   model temporal dependencies and long-range relationships within the input sequence. Variations of
925   RNNs, such as Gated Recurrent Units (GRUs) (Chung et al., 2014) and Long Short-Term Memory
926
      (LSTM) (Hochreiter & Schmidhuber, 1997) networks, have been developed to address the vanishing
      gradient problem and improve the ability to retain long-term information. LSTMs introduce an
927
      additional cell state and output gate to better manage the flow of information through time, while
928
      GRUs combine the update and reset gates to simplify the LSTM architecture. These enhancements
929   enable RNNs and their variants to effectively handle complex sequential tasks in RL.
930
931   RNNs are able to maintain and utilize information from previous states or experiences, which is
932
      especially valuable in POMDPs. This capacity is integral for tasks where the current state does not
      provide sufficient information to make optimal decisions. Prominent RL models that incorporate
933
      RNNs include Deep Recurrent Q-Networks (DRQN) (Hausknecht & Stone, 2015) and Deep Attention
934
      Recurrent Q-Networks (DARQN) (Sorokin et al., 2015), which extend the capabilities of Deep Q-
935   Networks (DQN) (Mnih et al., 2013) by integrating LSTM layers, and Recurrent Policy Gradient
936   (RPG) (Wierstra et al., 2010) methods, which adapt policy gradient techniques to sequential data.
937   Additionally, Asynchronous Advantage Actor-Critic (A3C) (Mnih et al., 2016) with RNNs and
938   Recurrent Deterministic Policy Gradient (RDPG) (Song et al., 2018) further illustrate the effectiveness
939   of RNNs in complex RL tasks.
940
      In the context of Meta-RL, RNNs are integral to enhancing an agent ability to learn and adapt
941   from temporally correlated data. By maintaining a hidden state that evolves over time, RNNs can
942   effectively encode information about past experiences, which is critical for Meta-RL tasks that
943   require fast adaptation to new environments. The Fast RL via Slow RL (RL2 ) (Duan et al., 2016)
944   algorithm exemplifies this, utilizing an LSTM network to embed the history of interactions, thereby
945   enabling the agent to quickly infer the optimal policy for a new task. Additionally, the Variational
946   Bayes-Adaptive Deep RL (VariBAD) (Zintgraf et al., 2020) algorithm employs a RNN to maintain
947   a latent representation of the task, which is updated as new information becomes available, thus
948   facilitating rapid adaptation.
949
950

952   B.2   T RANSFORMERS
953
954
955   Transformers (Vaswani et al., 2017) are a type of neural network architecture designed to process
956   sequential data. Unlike RNNs, which process input sequences sequentially and use recurrence to
957
      capture long-range dependencies, Transformers process input sequences in parallel and use self-
      attention mechanisms to capture dependencies inside the context window. This parallelization allows
958
      Transformers to be much faster and more efficient than RNNs, especially for longer sequences.
959
960   Transformers have found application in various areas of RL (Li et al., 2023; Agarwal et al., 2023):
961   Online RL (Parisotto et al., 2020; Lampinen et al., 2021; Esslinger et al., 2022; Zheng et al., 2022;
962   Melo, 2022a; Team et al., 2023; Pramanik et al., 2023), Offline RL (Cherepanov et al., 2024;
963
      Janner et al., 2021; Lee et al., 2022; Jiang et al., 2023), and model-based RL (Chen et al., 2022;
      Micheli et al., 2023; Robine et al., 2023), including for solving credit assignment problems and
964
      working in memory-intensive environments (Chen et al., 2021; Ni et al., 2023; Grigsby et al.,
965
      2024), provided that the entire trajectory fits within the model context. Transformers for Meta-
966   Reinforcement Learning (TrMRL) (Melo, 2022b) is a Meta-RL agent that mimics the memory
967   reinstatement mechanism using the transformer architecture. Hierarchical Transformers for Meta-
968   Reinforcement Learning (HTrMRL) (Shala et al., 2024) encode the same intra-episodic memory as
969   proposed in TrMRL and allow us to understand the transition dynamics within episodes and accesses
970   the intra-episodic memories to encode how the transition dynamics across episodes relate to each
971   other. Gated Transfrormer-XL (GTrXL) (Parisotto et al., 2020) uses Transformer-XL (Dai et al.,
      2019) with GRU-like (Chung et al., 2014) gating mechanism to solve Meta-RL problems.

       B.3   S TATE - SPACE MODELS
973
974    State-Space Models (SSMs) are a fundamental concept in control theory, used to describe and analyze
975    dynamic systems by representing the system’s behavior through a set of state variables. In the
976    context of deep learning, a subset of SSMs, specifically Linear-Time Invariant (LTI) systems, has
977    gained significant attention due to their remarkable performance and potential as an alternative to
978    traditional transformer-based models. These LTI-based SSMs, exemplified by models like Structured
979    State Space sequence model (S4) (Gu et al., 2021) and implified Structured State Space Sequence
980    Models (S5) (Smith et al., 2023), as well as SSMd with input-dependent matricesMamba (Gu &
981
       Dao, 2023), have showcased not only remarkable performance but also computational efficiency,
       making them a compelling alternative to transformers. To deal with the vanishing/exploding gradients
982
       problem (Pascanu et al., 2013) and improve memorization ability these models use high-order
983
       polynomial projection operators (HiPPO) theory (Gu et al., 2020) of continuous-time memorization.
984    Their high inference speed and parallelizable training making them potentially useful in RL.
985
986    Deep Planning Network (PlaNet) (Hafner et al., 2019) is a Recurrent State Space Model (RSSM)
987
       model-based agent that learns the environment dynamics from images and chooses actions through
       fast online planning in latent space. Modified S5 model (Lu et al., 2023) enables us to initialize and
988
       reset the hidden state in parallels used for modeling long-term decision-making tasks and solving
989
       Meta-RL problems. KalMamba (Becker et al., 2024) leverages Mamba to learn the dynamics
990    parameters of a linear Gaussian SSM in a latent space. Recall to Image (R2I) (Samsami et al., 2024)
991    combines S5 and Dreamer (Hafner et al., 2020) models.

       B.4   G RAPHS
994
995
       A world model is a learned internal representation that captures the agent understanding of the
996    environment. This model acts as a form of memory, enabling the agent to recall past experiences. One
997    way to structure this internal representation is through a graph, where nodes represent observations in
998    the environment and edges represent actions.
999
       In the Graph-Based Memory Reconstruction (GBMR) (Kang et al., 2024b) the memory is imple-
1000
       mented as a graph storing states, connections between states, and policy. This graph imitates Bartlett’s
1001
       reconstruction mechanism of human memory. Value Memory Graph (VMG) (Zhu et al., 2023) is a
1002   graph-structured world model in Offline-RL setting that represents the original environments as a
1003   graph-based MDP. Graph actions are sampled via Value Iteration and then converted to environment
1004   actions via the actions translator. Graph Convolutional Memory (GCM) (Morad et al., 2021) receives
1005   an observation as input and places it in a graph, determines the neighbors of that vertex and adds
1006   edges, and then through graph neural network receives belief state and forms a policy

1008   B.5   T EMPORAL C ONVOLUTIONS
1009
1010   In the context of neural networks, temporal convolutions function as an efficient memory mechanism
1011   for handling sequential data. Unlike RNNs, which use explicit memory cells, Temporal Convolutions
1012   implicitly remember information by applying learnable filters across the time axis. This approach
1013   enables the network to capture both short-term and long-term dependencies in the data. The con-
1014   volution operation aggregates features from previous time steps, effectively summarizing historical
1015   context and using it to inform predictions about future time steps.
1016   Simple Neural Attentive Meta-Learner (SNAIL) (Mishra et al., 2018) combines Temporal Convolu-
1017   tions to aggregate information from past experience and soft attention to focus on specific pieces of
1018   information. A3CTConv is a A3C model (Mnih et al., 2016) augmented with a Temporal Convolution
1019   layer after the last convolution layer and A3CTConvRNN is a A3CTConv modification with the
1020   recurrent layer (YuXuan Liu & Hsieh, 2016).
1021
1022   B.6   E XTERNAL STORAGE WITH READ / WRITE OPERATORS
1023
1024   Another promising approach to enhancing an agent’s memory is to utilize an external information
1025   storage with read-only or read/write access, as is common in computer architectures (tapes, RAMs,
       GPUs, etc.).


                                                         19
       Under review as a conference paper at ICLR 2025


1026
       External storage with read-only access allows information to be stored sequentially in a buffer,
1027   enabling the agent to read data without the ability to modify it. Models employing this type of
1028   memory include MemNNs, such as Memory Q-Network (MQN), Recurrent Memory Q-Network
1029   (RMQN), and Feedback Recurrent Memory Q-Network (FRMQN) (Oh et al., 2016). Additionally,
1030   Retrieval-Augmented Agent (R2A) (Goyal et al., 2022), Hierarchical Chunk Attention Memory
1031   (HCAM) (Lampinen et al., 2021), and Episodic Memory Deep Q-Networks (EMDQN) (Lin et al.,
1032   2018) exemplify systems that leverage read-only external storage.
1033   In contrast, external storage with read/write access prevents the loss of important information from
1034   the buffer and allows for the addition of new data in an addressable manner. This capability enables
1035   models to dynamically update their memory based on ongoing experiences. Notable examples of
1036   models utilizing this type of memory include Neural Map (Parisotto & Salakhutdinov, 2017), which
1037   is designed for structured memory in RL, and Differentiable Neural Computer (DNC) (Graves et al.,
1038   2016), which mimics the functionality of traditional computers by separating memory from computa-
1039   tion. Reinforcement Learning Neural Turing Machines (RL-NTM) (Zaremba & Sutskever, 2016)
1040   also exemplify this approach by incorporating neural networks with external memory capabilities.
1041   Moreover, replay buffers and their variations can also be classified under this type of memory
1042   mechanism. Replay buffers are crucial in RL as they store past experiences, allowing agents to learn
1043   from a diverse set of previously encountered states. This mechanism helps stabilize learning by
1044   breaking the correlation between consecutive experiences, which can lead to more robust training
1045   outcomes.
1046
1047   B.7   M EMORY IN PARAMETERS
1048
1049
       When RL agents are parameterized by neural networks, the optimization process itself can be seen
       as a form of memorization process. the simplest example is, feedforward networks that learn to
1050
       associate outputs with their corresponding inputs. This process updates parameters (or “weights”) of
1051
       neural networks so that each input-output pair is memorized. This type of memory is often reffered
1052   to as in-weight memory as a counterpart of in-context memory studied in this work.
1053
1054   From the optimization perspective, certain phenomena or best practices in training neural networks
1055
       may favour or harm memory when RL agents are implemented by neural networks. These include:
       the loss of plasticity, exponentially moving average (EMA) teacher in RL, catastrophic forgetting.
1056
       Loss of plasticity is a phenomenon observed in continual learning when neural network lose their
1057
       learnability over the sequence of tasks. Catastrophic forgetting is a vulnerability of all modern deep
1058   learning models - when faced with a new task, neural network might forget the first task, while it can
1059   remember both if it had to solve them simultaneously. Exponentially moving average is a technique,
1060   initially proposed to stabilize deep q-networks. It can also be seen as a form of memory since the
1061   neural network becomes an average of multiple past versions of it.
1062
       Unfortunately, the in-weight memory (as opposed to in-context memory studied in this work) is much
1063
       less reliable. When agents are tasked to be trained continually, catastrophic forgetting might erase
1064   past knowledge and skills learned by RL agent in the series on RL tasks.

       B.8   M EMORY AS WORLD MODEL
1067
1068   Model-based reinforcement learning (MBRL) is another framework which is related to Memory DM.
1069   The central idea of MBRL is to train a world model, i.e. the distribution of the form pθ (st+1 | st , at )
1070   parameterized by trainable parameters θ. In POMDPs , real states are unknown, therefore, world
1071   model must learn it from the sequences of actions and observations in an unsupervised or self-
1072   supervised fashion. This typically happens through learning of latent states and modelling them
1073
       sequentially (Hafner et al., 2020; 2022; 2023; Bruce et al., 2024). This mechanism can also be seen
       as a form of memory – since the world model learns a compressed representation of the environment,
1074
       it needs to remember events that are relevant for decision making.

1079   In this section, we explore the concept of Meta-Reinforcement Learning (Meta-RL), a specialized
       domain within POMDPs that focuses on equipping agents with the ability to learn from their past

       experiences across multiple tasks. This capability is particularly crucial in dynamic environments
1081   where agents must adapt quickly to new challenges. By recognizing and memorizing common patterns
1082   and structures from previous interactions, agents can enhance their efficiency and effectiveness when
1083   facing unseen tasks.
1084
       Meta-RL is characterized by the principle of “learning to learn”, where agents are trained not only to
1085
       excel at specific tasks but also to generalize their knowledge and rapidly adjust to new tasks with
1086
       minimal additional training. This adaptability is achieved through a structured approach that involves
1087   mapping data collected from various tasks to policies that guide the agent’s behavior.
1088
1089   Meta-RL algorithm is a function fθ parameterized with meta-parameters that maps the data D,
1090
       obtained during the process of training of RL agent in MDPs (tasks) Mi ∼ p(M), to a policy
       πϕ : ϕ = fθ (D). The process of learning the function f is typically referred to as the outer-loop,
1091
       while the resulting function f is called the inner-loop. In this context, the parameters θ are associated
1092
       with the outer-loop, while the parameters ϕ are associated with the inner-loop. Meta-training proceeds
1093   by sampling a task from the task distribution, running the inner-loop on it, and optimizing the inner-
1094   loop to improve the policies it produces. The interaction of the inner-loop with the task, during which
1095   the adaptation happens, is called a lifetime or a trial. In Meta-RL, it is common for S and A to
1096   be shared between all of the tasks and the tasks to only differ in the reward R(s, a) function, the

1097   dynamics P(s | s, a), and initial state distributions P0 (s0 ) (Beck et al., 2024). The formal definition

1183
1184   This section provides an extended description of the environments used in this work.
1185
1186   Passive-T-Maze (Ni et al., 2023). In this T-shaped maze environment, the agent’s goal is to move
1187   from the starting point to the junction and make the correct turn based on an initial signal. The
       agent can select from four possible actions: a ∈ lef t, up, right, down. The signal, denoted by the


                                                                                                                                                                               22
       Under review as a conference paper at ICLR 2025


1188
       variable clue, is provided only at the beginning of the trajectory and indicates whether the agent
1189   should turn up (clue = 1) or down (clue = −1). The episode duration is constrained to T = L + 1,
1190   where L is the length of the corridor leading to the junction, which adds complexity to the task.
1191   To facilitate navigation, a binary variable called f lag is included in the observation vector. This
1192   variable equals 1 one step before reaching the junction and 0 at all other times, indicating the agent’s
1193   proximity to the junction. Additionally, a noise channel introduces random integer values from the
1194   set −1, 0, +1 into the observation vector, further complicating the task. The observation vector is
1195   defined as o = [y, clue, f lag, noise], where y represents the vertical coordinate.
1196   The agent receives a reward only at the end of the episode, which depends on whether it makes a
1197   correct turn at the junction. A correct turn yields a reward of 1, while an incorrect turn results in a
1198   reward of 0. This configuration differs from the conventional Passive T-Maze environment (Ni et al.,
1199   2023) by featuring distinct observations and reward structures, thereby presenting a more intricate set
1200   of conditions for the agent to navigate and learn within a defined time constraint. To transition from a
1201   sparse reward function to a dense reward function, the environment is parameterized using a penalty
                                   1
1202   defined as penalty = − T −1    , which imposes a penalty on the agent for each step taken within the
1203   environment. Thus, this environment has a 1D vector space of observations, a discrete action space,
1204
       and sparse and dense configurations of the reward function.
1205
1206
       Minigrid-Memory (Chevalier-Boisvert et al., 2023). Minigrid-Memory is a two-dimensional
       grid-based environment specifically crafted to evaluate an agent’s long-term memory and credit
1207
       assignment capabilities. The layout consists of a T-shaped maze featuring a small room at the
1208
       corridor’s outset, which contains an object. The agent is instantiated at a random position within the
1209   corridor. Its objective is to navigate to the chamber, observe and memorize the object, then proceed to
1210   the junction at the maze’s terminus and turn towards the direction where the object, identical to that
1211   in the initial chamber, is situated. A reward function defined as r = 1 − 0.9 × Tt is awarded upon
1212   successful completion, while failure results in a reward of zero. The episode concludes when the
1213   agent either makes a turn at a junction or exhausts a predefined time limit of 95 steps. To implement
1214   partial observability, observational constraints are imposed on the agent, limiting its view to a 3 × 3
1215   frame size. Thus, this environment has a 2D space of image observations, a discrete action space, and
1216   sparse reward function.
