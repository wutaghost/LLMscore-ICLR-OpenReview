009                                              A BSTRACT
010
011
               A recent line of work has shown promise in using sparse autoencoders (SAEs)
012            to uncover interpretable features in neural network representations. However, the
013            simple linear-nonlinear encoding mechanism in SAEs limits their ability to per-
014            form accurate sparse inference. In this paper, we investigate sparse inference and
015            learning in SAEs through the lens of sparse coding. Specifically, we show that
016            SAEs perform amortised sparse inference with a computationally restricted en-
017            coder and, using compressed sensing theory, we prove that this mapping is inher-
018            ently insufficient for accurate sparse inference, even in solvable cases. Building
019            on this theory, we empirically explore conditions where more sophisticated sparse
020
               inference methods outperform traditional SAE encoders. Our key contribution
               is the decoupling of the encoding and decoding processes, which allows for a
021
               comparison of various sparse encoding strategies. We evaluate these strategies
022
               on two dimensions: alignment with true underlying sparse features and correct
023            inference of sparse codes, while also accounting for computational costs during
024            training and inference. Our results reveal that substantial performance gains can
025            be achieved with minimal increases in compute cost. We demonstrate that this
026            generalises to SAEs applied to large language models (LLMs), where advanced
027            encoders achieve similar interpretability. This work opens new avenues for un-
028            derstanding neural network representations and offers important implications for
029            improving the tools we use to analyse the activations of large language models.

032   1   I NTRODUCTION
033
034
      Understanding the inner workings of neural networks has become a critical task since these models
035
      are increasingly employed in high-stakes decision-making scenarios (Fan et al., 2021; Shahroudne-
036   jad, 2021; Räuker et al., 2023). As the complexity and scale of neural networks continue to grow,
037   so does the importance of developing robust methods for interpreting their internal representations.
038   This paper explores the synergy between sparse autoencoders (SAEs) and sparse coding techniques,
039   aiming to advance our ability to extract interpretable features from neural network activations.
040
      Recent work has investigated the “superposition hypothesis” (Elhage et al., 2022), which posits that
041
      neural networks represent interpretable features in a linear manner using non-orthogonal directions
042   in their latent spaces. Building on this idea, researchers have shown that individual features can be
043   recovered from these superposed representations using sparse autoencoders (Bricken et al., 2023;
044   Cunningham et al., 2023). These models learn sparse and overcomplete representations of neural
045   activations, with the resulting sparse codes often proving to be more interpretable than the original
046   dense representations (Cunningham et al., 2023; Elhage et al., 2022; Gao et al., 2024).
047
      The mathematical foundation of SAEs aligns closely with that of sparse coding. Both approaches
048   assume that a large number of sparse codes are linearly projected into a lower-dimensional space,
049   forming the neural representation. However, while sparse coding typically involves solving an opti-
050   misation problem for each input, SAEs learn an efficient encoding function through gradient descent,
051   potentially sacrificing optimal sparsity for computational efficiency. This trade-off introduces what
052   we term the “amortisation gap” – the disparity between the best sparse code predicted by an SAE en-
053   coder and the optimal sparse codes that an unconstrained sparse inference algorithm might produce
      (Marino et al., 2018).

      In this paper, we explore this amortisation gap and investigate whether more sophisticated sparse
055   inference methods can outperform traditional SAE encoders. Our key contribution is decoupling the
056   encoding and decoding processes, allowing for a comparison of various sparse encoding strategies.
057   We evaluate four types of encoding methods on synthetic datasets with known ground-truth features.
058   We evaluate these methods on two dimensions: alignment with true underlying sparse features and
059   inference of the correct sparse codes, while accounting for computational costs during both training
060   and inference. To demonstrate real-world applicability, we also train models on GPT-2 activations
061   (Radford et al., 2019), showing that more complex methods can yield interpretable features in large
062   language models. This approach aims to identify optimal strategies for extracting interpretable
063
      features from neural representations across different computational regimes.

067   2.1   S PARSE N EURAL R EPRESENTATIONS
068
069   Sparse representations in neural networks specifically refer to activation patterns where only a small
070   subset of neurons are active for any given input (Olshausen & Field, 1996). These representations
071   have gained attention due to their potential for improved interpretability and efficiency (Lee et al.,
072   2007). Sparse autoencoders (SAEs) are neural network architectures designed to learn sparse repre-
073   sentations of input data (Ng et al., 2011; Makhzani & Frey, 2013). An SAE consists of an encoder
074   that maps input data to a sparse latent space and a decoder that reconstructs the input from this latent
075
      representation. Sparse coding, on the other hand, is a technique that aims to represent input data
      as a sparse linear combination of basis vectors (Olshausen & Field, 1997). The objective of sparse
076
      coding is to find both the optimal basis (dictionary) and the sparse coefficients that minimise recon-
077
      struction error while maintaining sparsity. While both SAEs and sparse coding seek to find sparse
078   representations, they differ in their approach. SAEs learn an efficient encoding function through
079   gradient descent, allowing for fast inference but potentially sacrificing optimal sparsity. Sparse cod-
080   ing, in contrast, solves an optimisation problem for each input, potentially achieving better sparsity
081   at the cost of increased computational complexity during inference.
082
083   2.2   S UPERPOSITION IN N EURAL R EPRESENTATIONS
084
085   The superposition hypothesis suggests that neural networks can represent more features than they
086   have dimensions, particularly when these features are sparse (Elhage et al., 2022). Formally, let us
087   consider a neural representation y ∈ RM and a set of N features, where typically N > M . In a linear
088   representation framework, each feature fi is associated with a direction wi ∈ RM . The presence
                                                   PN
089   of multiple features is represented by y =      i=1 xi wi where xi ∈ R represents the activation
090   or intensity of feature i. Features are often defined as interpretable properties of the input that
091   a sufficiently large neural network would reliably dedicate a neuron to representing (Olah et al.,
092   2020).
093   In an M -dimensional vector space, only M orthogonal vectors can fit. However, the Johnson-
094   Lindenstrauss Lemma states that if we permit small deviations from orthogonality, we can fit expo-
095   nentially more vectors into that space. More formally, for any set of N points in a high-dimensional
096   space, there exists a linear map to a lower-dimensional space of O(log N/ϵ2 ) dimensions that pre-
097   serves pairwise distances up to a factor of (1 ± ϵ). This lemma supports the hypothesis that LLMs
098   might be leveraging a similar principle in superposition.
099   Superposition occurs when the matrix W = [w1 , ..., wN ] ∈ RM ×N has more columns than rows
100   (i.e., N > M ), making W T W non-invertible. Superposition relies on the sparsity of feature activa-
101   tions. Let s = |x|0 be the number of non-zero elements in x = [x1 , ..., xN ]T . When s ≪ N , the
102   model can tolerate some level of interference between features, as the probability of many features
103   being active simultaneously (and thus interfering) is low.

128   2.4   S PARSE AUTOENCODERS
129
      Sparse autoencoders (SAEs) offer an alternative approach to extracting sparse representations, us-
130
      ing amortised inference instead of the iterative optimisation used in sparse coding. SAEs learn to
131

133   SAE consists of an encoder network that maps the input to a hidden, sparse representation of latent
134   coefficients, and a decoder network that reconstructs the input as a linear combination of vectors,
135   with the coefficients defined by the sparse representation. Let xi ∈ Rk be an input vector (as in
136   our sparse coding formulation), and αi ∈ Rm be the hidden representation (analogous to the sparse
137   codes in sparse coding), where typically m > k. The encoder and decoder functions are defined as:

      2.5    A PPLICATIONS IN N EURAL N ETWORK M ODELS
188
189
      Sparse autoencoders (SAEs) have emerged as a promising tool for enhancing the interpretability
190
      of large language models (LLMs) by extracting interpretable features from their dense representa-
191
      tions. Early work by Cunningham et al. (2023) and Bricken et al. (2023) demonstrated the potential
192   of sparse dictionary learning to untangle features, lifting them out of superposition in transformer
193   MLPs. This approach was extended to attention heads by Kissane et al. (2024), who scaled it to
194   GPT-2 (Radford et al., 2019). These studies have shown that SAEs can extract highly abstract, mul-
195   tilingual, and multimodal features from LLMs, including potentially safety-relevant features related
196   to deception, bias, and dangerous content (Templeton, 2024). In vision models, Gorton (2024) and
197   Klindt et al. (2023) trained SAEs on convolutional neural network activations. The latter found that
198   K-means (which is equivalent to one-hot sparse coding) outperformed SAEs (Fig.12) in quantitative
199   interpretability metrics (Zimmermann et al., 2024).
200   The scaling of SAEs to larger models has been a focus of recent research, with significant progress
201   made in applying them to state-of-the-art LLMs. Gao et al. (2024) proposed using k-sparse autoen-
202   coders (Makhzani & Frey, 2013) to simplify tuning and improve the reconstruction-sparsity frontier,
203   demonstrating clean scaling laws with respect to autoencoder size and sparsity. They successfully
204   trained a 16 million latent autoencoder on GPT-4 activations. Similarly, Templeton (2024) reported
205   extracting high-quality features from Claude 3 Sonnet, while Lieberum et al. (2024) released a com-
206
      prehensive suite of SAEs trained on all layers of Gemma 2 models. These advancements underscore
      the importance of developing efficient and accurate SAE techniques to reduce the amortisation gap,
207
      especially as applications to larger models become more prevalent. The growing body of work on
208
      SAEs in LLMs suggests that they may play a crucial role in future interpretability research.

      3.1   T HEORY: P ROVABLE S UBOPTIMALITY OF SAE S
217
218   Theorem 1 (SAE Amortisation Gap). Let S = RN be N sources following a sparse distribution
219   PS such that any sample has at most K ≥ 2 non-zero entries, i.e., ∥s∥0 ≤ K, ∀s ∈ supp(PS ).
220   The sources are linearly projected into an M -dimensional space, satisfying the restricted isometry
                             N
221   property, where K log K   ≤ M < N . A sparse autoencoder (SAE) with a linear-nonlinear (L-NL)
222   encoder must have a non-zero amortisation gap.
223
224
      The complete proof of Theorem 1 is provided in Appendix A. The theorem considers a setting where
      sparse signals s ∈ RN with at most K non-zero entries are projected into an M -dimensional space
225
      (M < N ). Compressed sensing theory guarantees that unique recovery of these sparse signals is
226
      possible when M ≥ K log(N/K), up to sign ambiguities (Donoho, 2006). However, we prove
227   that SAEs fail to achieve this optimal recovery, resulting in a non-zero amortisation gap. The core
228   of this limitation lies in the architectural constraints of the SAE’s encoder. The linear-nonlinear
229   (L-NL) structure of the encoder lacks the computational complexity required to fully recover the
230   high-dimensional (N ) sparse representation from its lower-dimensional (M ) projection. Figure 1
231   illustrates this concept geometrically.

      3.5   D ISENTANGLING D ICTIONARY L EARNING AND S PARSE I NFERENCE
271
272   Our study decomposes the sparse coding problem into two interrelated tasks: dictionary learning
273   and sparse inference. Dictionary learning involves finding an appropriate sparse dictionary D ∈
274   RM ×N from data, while sparse inference focuses on reconstructing a signal x ∈ RM using a sparse
275   combination of dictionary elements, solving for s ∈ RN in x ≈ Ds where s is sparse. These tasks
276   are intrinsically linked: dictionary learning often involves sparse inference in its inner loop, while
277   sparse inference requires a dictionary.
278
279
      Known Sparse Codes. In this scenario, we assume knowledge of the true sparse codes s∗ and
280
      focus solely on the encoder’s ability to predict these latents, effectively reducing the problem to la-
281   tent regression. We define the objective as minimising L(f (x), s∗ ) = 1 − cos(f (x), s∗ ), where f is
282   the encoding function and cos denotes cosine similarity.1 In this setting, only the SAE encoder and
283   MLP are applicable, as they directly learn mappings from input to latent space. The SAE encoder
284   learns an amortised inference function, while the MLP learns a similar but more complex mapping.
285   Conversely, SAE+ITO and sparse coding are not suitable for this task. SAE+ITO focuses on op-
286   timising reconstruction using a fixed dictionary, which is irrelevant when true latents are known.
287   Similarly, sparse coding alternates between latent and dictionary optimisation, which reduces to
288   encoder training when the dictionary is disregarded.
289
290   Known Dictionary. When the true dictionary D∗ is known, we focus on optimising the encoder
291   or inference process while keeping the dictionary fixed. This scenario is applicable to SAE, MLP,
292   and SAE+ITO methods. For SAE and MLP, we optimise minθ Ex [|x − D∗ fθ (x)|22 ], where fθ
293   represents the encoder function with parameters θ. SAE+ITO, in contrast, performs gradient-based
294   optimisation at inference time: mins |x − D∗ s|22 + λ|z|1 for each input x, incurring zero training
295   FLOPs but higher inference-time costs. This differs from SAE and MLP by directly optimising
296   latent coefficients rather than learning an encoding function. Sparse coding is not applicable in this
297
      scenario, as it reduces to SAE+ITO when the dictionary is known and fixed.
298
299   Unknown Sparse Codes and Dictionary. This scenario represents the standard setup in sparse
300   coding, where both the sparse codes s and the dictionary D are unknown and must be learned
301   simultaneously. All four methods—SAE, MLP, SAE+ITO, and sparse coding—are applicable in this
302   context, each approaching the problem differently. SAE and MLP learn both an encoder function
303   fθ (x) and a dictionary D simultaneously. SAE+ITO and sparse coding learn a dictionary during
304   training and optimises latents at inference time.

450     5          I NTERPRETABILITY OF S PARSE C ODING S CHEMES
451
452     A common criticism of more powerful encoding techniques is that they can find concepts that are
453     more difficult to interpret, or that are not actually used by the model. To investigate the interpretabil-
454     ity of more complex encoding techniques, we trained both an SAE and MLP on 406 million tokens
455     from OpenWebText. The MLP used a hidden width of 4224 and both models had 16, 896 sparse
456     codes. The models were trained on the residual-stream pre-activations of Layer 9 in GPT-2 Small,
457     which have a dimension of 768. We used a learning rate of 3 · 10−4 and an L1 penalty of 1 · 10−4 .
458     Throughout training, we tracked normalised mean squared error (MSE), dividing it by the error
459
        when predicting the mean activations, as well as L0 sparsity. The final SAE achieved a normalised
        MSE of 0.062 with an average L0 of 39.55, while the MLP reached a normalised MSE of 0.051
460
        with an average L0 of 40.20. The final models had a significant number of dead neurons (measured
461
        as not having fired in the last 50, 000 training steps): 71% for the SAE and 66% for the MLP.
462
463     To assess the interpretability of the learned features, we selected 200 random features that activated
464     on our test set of 13.1 million OpenWebText tokens. For each feature, we employed an automated
465
        interpretability classification approach. We identified the top 10 activating examples for the feature
        in our test dataset and labelled the token with the highest activation. We also computed logit ef-
466
        fects for each feature through the path expansion WU · f , where WU is the model’s unembedding
467
        matrix and f is the feature vector. The top 10 and bottom 10 tokens resulting from this logit effect
468     calculation were noted.
469
470     We provided both the activating examples and the top and bottom promoted logits to GPT-4, which
471
        was instructed to construct a precise explanation of the feature’s function (prompt in Appendix H).
        To evaluate the accuracy of these interpretations, we presented them to another instance of GPT-4
472
        along with 5 new activating examples and 5 non-activating examples, labeling the token on which
473
        the feature potentially activates. The model was then asked to predict which of these examples the
474     feature would activate on, and we calculated the F1-score compared to the ground truth.
475
476     Figure 7 displays the distributions of the F1 scores for the 200 SAE and MLP characteristics. The
477
        results indicate that the MLP features demonstrate interpretability comparable to the SAE features.

482     sation gap in sparse autoencoders (SAEs) when applied to neural network interpretability tasks. We
483     prove that SAEs with linear-nonlinear encoders cannot achieve optimal sparse recovery in settings
484     where such recovery is theoretically possible. This limitation is corroborated by our experimental
485     results, which demonstrate superior performance of more complex encoding methods, such as mul-
        tilayer perceptrons (MLPs) and sparse coding, across various synthetic data scenarios. Notably, our

503   ble to, and in some cases exceeding, that of SAE features. These findings challenge the prevailing
504
      assumption that simpler encoders are necessary for maintaining interpretability. The results of our
      study have significant implications for the field of neural network interpretability, particularly in the
505
      context of large language models. They suggest that more sophisticated encoding techniques can
506
      potentially improve feature extraction and interpretability without compromising the validity of the
507   extracted features. However, this potential improvement comes with increased computational costs.
508   Our work provides a framework for exploring this trade-off.
509
510
      The use of simple linear-nonlinear encoders in SAEs for language model interpretability has been
      primarily motivated by concerns that more powerful methods might extract features not actually
511
      utilised by the transformer (Bricken et al., 2023). However, this approach may be overly conservative
512
      given the complexity of representations in transformer layers, which result from multiple rounds of
513   attention and feed-forward computations. Better encoders aligns with recent work on inference-
514   time optimisation (Nanda et al., 2024), and will be validated as we improve encoding evaluation
515   (Makelov et al., 2024). Regardless, SAEs are sensitive to hyperparameters and fragile (Cunningham
516   et al., 2023), so exploring more powerful encoders is warranted.
517
      The computational cost of more complex encoders should be weighed against potential benefits in
518
      feature extraction and interpretability. Given the significant resources already invested in projects
519   like Gemma Scope (Lieberum et al., 2024), allocating additional compute to enhance representation
520   quality before decoding may be justified. Importantly, more sophisticated encoders can still maintain
521   the linear decoder necessary for downstream tasks such as steering. Empirical studies comparing
522   feature quality across encoder complexities will be important, as will addressing concerns about
523   non-zero-centered representations raised by Hobbhahn (2023).
524
525   Limitations Our study has several limitations that warrant consideration. We primarily explored
526   scenarios with constant sparsity and uncorrelated channels in the sparse representation, which may
527   not fully capture the complexity of real-world data. Additionally, our analysis focused on vanilla
528   implementations of the models, which are susceptible to issues like shrinkage due to the L1 penalty.
529
      To comprehensively understand the current amortisation gap, future work should incorporate recent
      SAE variants such as top-k SAEs (Makhzani & Frey, 2013; Gao et al., 2024) and JumpReLU SAEs
530
      (Rajamanoharan et al., 2024b), as discussed in Appendix F.1. This extension would allow us to anal-
531
      yse how large the amortisation gap is with the new SAE architectures. Similarly, our implementation
532   of SAE+ITO did not use advanced techniques like matched pursuit or gradient pursuit, potentially
533   underestimating its performance. The traditional dictionary learning approaches explored in Ap-
534   pendix F suggest room for improvement in this area. Lastly, our synthetic data generation process
535   did not account for varying feature importance, a key aspect of Elhage et al. (2022)’s framework.
536   Addressing these limitations in future research would provide a more comprehensive understanding
537   of sparse encoding strategies and their applicability to complex neural representations.

771   A    A MORTISATION GAP PROOF
772
773   Theorem 1 (SAE Amortisation Gap). Let S = RN be N sources following a sparse distribution
774   PS such that any sample has at most K ≥ 2 non-zero entries, i.e., ∥s∥0 ≤ K, ∀s ∈ supp(PS ).

841   B    C ONCEPTUAL M ODELLING F RAMEWORK
842
843   The concept of distributed representations in neural networks originated with the Parallel Distributed
844   Processing (PDP) movement (Rumelhart et al., 1986). This work explored how information could
845   be encoded across multiple units in a network, rather than in localised, symbolic representations
846   (Thorpe, 1989). A distributed representation of information means that no single processing unit
847   in a network performs a syntactically or semantically determinable subtask alone. Instead, an as-
848   sembly of processing units generates a “distributed pattern of activation” to represent information
849   (Smolensky et al., 2006; Van Gelder, 1990; Gelder, 1992).
850   We show in Figure 8 the general modelling framework we are studying for uncovering these dis-
851   tributed representations. Inputs pass through a neural network and generate some internal neural
852   representation, that is often distributed and in superposition. We use an encoding process to de-
853   termine active latents, or features, inherent in that process, and a learned decoder to specify what
854   those feature directions should be. This lies at the heart of all methods studied: inference-based
855   methods (i.e., sparse coding), amortised methods (i.e., sparse autoencoders and autoencoders with
856
      a more powerful encoder, such as an MLP) and hybrid approaches (i.e., sparse autoencoder with
      inference-time optimisation).

1066   F.1.1    A DVANCED S PARSE AUTOENCODER T ECHNIQUES
1067
       Sparse autoencoders trained with L1 regularisation are susceptible to the shrinkage problem. Wright
1068
       & Sharkey (2024) identified feature suppression in SAEs, analogous to the activation shrinkage first
1069
       described by Tibshirani (1996) as a property of L1 penalties. The shrinkage problem occurs when
1070   L1 regularisation reduces the magnitude of non-zero coefficients to achieve a lower loss, potentially
1071   underestimating the true effect sizes of important features.
1072
1073
       Several techniques have been proposed to mitigate this issue:
1074
1075
               • ProLU Activation: Taggart (2024) introduced the ProLU activation function to maintain
                 scale consistency in feature activations.
1076
1077           • Gated SAEs: Rajamanoharan et al. (2024a) developed Gated Sparse Autoencoders, which
1078             separate the process of determining active directions from estimating their magnitudes.
1079             This approach limits the undesirable side effects of L1 penalties and achieves a Pareto
                 improvement over standard methods.

1155   F.2     T OP -k SPARSE CODING
1156
1157   Building on this exploration, we introduced a top-k sparse coding approach. We aimed to determine
1158   whether (1) setting very small active latents to zero would improve performance and (2) optimising
1159
       with a differentiable top-k function, rather than using exponential or ReLU functions, could yield
       further benefits.
1160
1161   Figure 15 presents the results of these experiments. We first trained the sparse coding model for
1162   20,000 steps on the training data and optimised for an additional 1,000 steps on the test data. During
1163   this process, we measured mean squared error (MSE) loss, latent MCC, and the L0 norm of the
1164   latent codes. Due to the presence of very small active latents, all initial setups led to an L0 value of
1165
       1.0, indicating that all latents were active, as shown by the blue star in the figure. We also show a
       sparse autoencoder trained with different L1 penalties as a comparison.
1166
1167   Next, we applied a top-k operation to enforce sparsity by setting all but the top-k largest activations
1168   to zero. This process resulted in improved L0 values, but the MSE loss and MCC results indicated
1169   that the top-k optimisation itself was hampered by an insufficient learning rate. We hypothesise that
1170   with proper tuning of hyperparameters, we could achieve Pareto improvements by using the top-k
1171
       function directly, rather than applying it to exponentiated codes.
1172   We believe that further adjustments to the optimisation process, including a higher learning rate
1173   for top-k functions, could result in better performance. Additionally, applying the top-k function
1174   directly, without exponentiating the codes, may offer further gains in performance and sparsity.

       H     AUTOMATED INTERPRETABILITY
1297
1298
       In this section, we describe the automated interpretability pipeline used to understand and evaluate
1299   the features learned by sparse autoencoders (SAEs) and other models in the context of neuron acti-
1300   vations within large language models (LLMs). The pipeline consists of two distinct tasks: feature
1301   interpretation and feature scoring. These tasks allow us to generate hypotheses about individual
1302   feature activations and to determine whether specific features are likely to activate given particular
1303   token contexts.
1304
1305   H.1   F EATURE I NTERPRETER P ROMPT
1306
1307   We use a feature interpreter prompt to provide an explanation for a neuron’s activation. The in-
1308
       terpreter is tasked with analysing a neuron’s behaviour, given both text examples and the logits
       predicted by the neuron. Below is a summary of how the interpreter prompt works:
1309
1310   You are a meticulous AI researcher conducting an investigation into a specific neuron in a language
1311   model. Your goal is to provide an explanation that encapsulates the behavior of this neuron. You
1312   will be given a list of text examples on which the neuron activates. The specific tokens that cause
1313   the neuron to activate will appear between delimiters like <<this>>. If a sequence of consecu-
1314
       tive tokens causes the neuron to activate, the entire sequence of tokens will be contained between
       delimiters <<just like this>>. Each example will also display the activation value in paren-
1315
       theses following the text. Your task is to produce a concise description of the neuron’s behavior by
1316
       describing the text features that activate it and suggesting what the neuron’s role might be based
1317   on the tokens it predicts. If the text features or predicted tokens are uninformative, you can omit
1318   them from the explanation. The explanation should include an analysis of both the activating to-
1319   kens and contextual patterns. You will be presented with tokens that the neuron boosts in the next
1320   token prediction, referred to as Top logits, which may refine your understanding of the neu-
1321   ron’s behavior. You should note the relationship between the tokens that activate the neuron and
1322   the tokens that appear in the Top logits list. Your final response should provide a formatted
1323   explanation of what features of text cause the neuron to activate, written as: [EXPLANATION]:
1324   <your explanation>.
1325
1326   H.2   F EATURE S CORER P ROMPT
1327
       After generating feature interpretations, we implemented a scoring prompt to predict whether a
1328
       specific feature is likely to activate on a given token. This ensures that the explanations generated
1329
       by the interpreter align with actual activations. The scoring prompt tasks the model with evaluating
1330   if the tokens marked in the examples are representative of the feature in question.
1331
1332   You are provided with text examples where portions of the sentence strongly represent the feature,
1333
       with these portions enclosed by << and >>. Some of these examples might be mislabeled. Your
       job is to evaluate each example and return a binary response (1 if the tokens are correctly labeled,
1334
       and 0 if they are mislabeled). The output must be a valid Python list with 1s and 0s, corresponding
1335
       to the correct or incorrect labeling of each example.
1336
1337
       H.3   E VALUATION OF AUTOMATED I NTERPRETABILITY
1338
1339   To evaluate the accuracy of the interpretations generated by the feature interpreter and feature scorer,
1340   we compared model-generated explanations against held-out examples. The evaluation involved
1341   calculating the F1-score, which was done by presenting the model with a mix of correctly labeled
1342   and falsely labeled examples. The model was then tasked with predicting whether each token in
1343   the example represented a feature or not, based on the previously generated interpretation. By
1344   comparing the model’s predictions with ground truth labels, we can assess how accurately the feature
1345   interpretation aligns with actual neuron activations. This process helps validate the interpretability
1346
       of the features learned by SAEs, MLPs, and other models.
1347   This pipeline is based on the work of Juang et al. (2024), which itself builds on the work of others.
1348   Bills et al. (2023) used GPT-4 to generate and simulate neuron explanations by analyzing text that
1349   strongly activated the neuron. Bricken et al. (2023) and Templeton (2024) applied similar techniques
       to analyze sparse autoencoder features. Templeton (2024) also introduced a specificity analysis to
