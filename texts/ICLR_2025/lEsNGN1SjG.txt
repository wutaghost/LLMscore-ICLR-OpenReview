009                                                A BSTRACT
010
011
               Deep learning has become the cornerstone of recent advances in artificial in-
012
               telligence. However, the presence of adversarial samples makes deep learning
013            vulnerable in safety-critical applications. Moreover, adversarial examples have
014            been shown to be unavoidable in some degree. So how to keep the network secure
015            under adversarial attacks? To address this issue, we propose the bias classifier.
016            This approach employs the bias component of a neural network, using ReLU as
017            its activation function, as a classifier. The bias classifier has been shown to uni-
018            versally approximate any classification problem with a high degree of probability.
019            Moreover, it can be made information-theoretically safe against the original model
020            gradient-based attack in the sense that any such attack produces a completely
021
               random attacking direction for any given input. Thus, the bias classifier provably
               achieves the maximum possible robust accuracy under specified attacks. Experi-
022
               ments are used to validate our theoretical results and to show that the bias classifier
023
               is accurate and robust for simple models.

027   1   I NTRODUCTION
028
029
      Deep learning has become the most powerful machine learning method, which has been successfully
030   applied in computer vision, natural language processing, and many other fields. However, the
031   existence of adversarial samples (Szegedy, 2013; Biggio et al., 2013) makes deep learning vulnerable
032   in safety-critical applications such as autonomous driving (Cao et al., 2019), face recognition (Dong
033   et al., 2019), and medical image processing (Ma et al., 2021). Although many effective methods to
034   defend adversaries have been proposed (Xu et al., 2020), adversarial examples have been shown to be
035   inevitable in certain sense (Azulay & Weiss, 2019; Shafahi et al., 2019a; Bastounis et al., 2021; Yu
036   et al., 2023; Gao et al., 2022).
037   In (Shafahi et al., 2019a), it is shown that for certain data distributions and network F, if F gives the
038   correct result for a benign sample x, then x must have an adversarial example. In (Bastounis et al.,
039   2021), it is proven that for any network F with a fixed structure, there exists a dataset D such that
040   if F is accurate on D then F has adversarial examples. In (Yu et al., 2023), it is proven that if the
041   network is large enough, then small perturbation of the network parameters will lead to adversarial
042   examples. On the other hand, for a given dataset D, it is shown that there exists a network which is
043   robust for D with a given budget (Li et al., 2022a; Yu et al., 2024), but the network is too large to be
044
      practical. Furthermore, in order to be robustly generalizable, the network must have an exponential
      number of parameters (Li et al., 2022a).
045
046   There are many methods to improve the robustness of networks (Xu et al., 2020). Adversarial
047   training (Madry, 2017) is one of the most used defenses and has been shown to be optimal in certain
048   sense (Gao et al., 2022). However, the robust accuracy for networks developed through adversarial
049   training is not very high for moderately complex dataset such as CIFAR-10. Besides, the application
050
      of diffusion models for input purification achieves good results to some extent (Nie et al., 2022). But
      all these defenses cannot give the provable highest robust accuracy.
051
052   Since theoretical and experimental results show that adversarial examples cannot be totally eliminated
053   for the commonly used network structures and the practical robust accuracies do not achieve provable
      highest values, we may ask the following basic question about the neural network security.


                                                         1

054
      Question. For neural networks with a given structure, for instance VGG-19 or ResNet-18, is it
055   possible to provably achieve the highest possible robust accuracy under certain attacks?
056
057   In this paper, we present a partial answer to the above question by introducing the bias classifier,
058
      that is, using the bias part of a deep neural network (DNN) as the classifier. We show that the bias
      classifier can be made information-theoretically safe for certain attacks in the sense that the success
059
      rate of adversarial attack is close to that of a random direction attack. Since random direction attack
060
      can be considered the weakest attack, robust accuracy under this attack is the highest possible, and in
061   this sense, the information-theoretically safe bias classifier gives the highest robust accuracy.
062
063   Let I = [0, 1] and F : In → Rm be a classification DNN and m the number of labels, using ReLU as
064
      the activation function. For any x ∈ In , there exist Wx ∈ Rm×n and Bx ∈ Rm such that
065
                                                F(x) = Wx x + Bx .
066
067   By the bias classifier, we mean to use F’s bias part BF (x) = Bx : In → Rm as a classifier. The
068   contributions of this paper are summarized below.
069
070
      First, the universal classification power of the bias classifier is proved. Precisely, it is shown that for
      any classification problem G : In → [m], there exists a DNN whose bias part gives the correct label
071
      G(x) for x ∈ In with arbitrarily high probability.
072
073   Second, the bias classifier can be provably safe against certain gradient-based attacks of the form:
074   A(x, y) = x + ρDA (x, y) : In → In , where ρ ∈ R+ is the attack coefficient and DA (x, y) ∈
075   {−1, 1}n is the attack direction. A neural network F : In → Rm is called information-theoretically
076
      safe against an attack A, if a random attack direction DA (x, y) is generated for any input sample x.
      The robust accuracy under such an attack A is equal to the robust accuracy under an attack using
077
      the random attack direction. It is clear that such an attack gives the highest possible robust accuracy,
078
      which can be achieved practically. This seems to be the first time that the concept of information-
079   theoretically safe classifier is proposed for adversarial attacks. The notion of information-theoretically
080   safe is borrowed from cryptography (Goldreich, 2001), which means that the ciphertext yields no
081   information regarding the plaintext for cyphers that are perfectly random.
082
      Third, an effective training method for the bias classifier is proposed. It is observed that the adversarial
083
      training introduced in (Madry, 2017) significantly increases the classification power of the bias part.
084
      Using the adversarial training to the loss function LCE (BF (x), y) + γLCE (F(x), y) further increases
085   the classification power of the bias part, and hence is used to train the bias classifier.
086
087   Finally, experimental results are used to validate the theoretical results and to show that the bias
088
      classifier is quite accurate and robust on some simple tasks. We first use experiments to show that
      the bias classifier is indeed informtion-theoretically safe by achieving the highest possible robust
089
      accuracy. We then use simple networks such as Lenet-5 for MNIST and VGG-19 for CIFAR-10 to
090
      illustrate the practical performance of the bias classifier. Following (Carlini et al., 2019), we show
091   that the bias classifier is quite robust against major attacks, such as gradient-based white box attack,
092   black box attack, AutoAttack, and BDPA attack (Athalye et al., 2018).

095   2    R ELATED WORK
096
097   Adversarial Defenses. There exist two main approaches to obtain more robust DNNs: using a better
098   training method or a better structured DNN. Of course, the two approaches can be combined.
099
      Many new defensive structures for DNNs were proposed (Xu et al., 2020). Closely related to this
100
      paper, gradient masking or obfuscation defenses try to make the gradient information of the DNN
101
      unuseful, so that attacks based on gradient cannot work directly (Athalye et al., 2018). Gradient
102   masking methods include the shattered gradients (Buckman et al., 2018; Guo et al., 2017), the
103   stochastic gradients (Dhillon et al., 2018; Xie et al., 2017), the exploding gradients (Song et al., 2017;
104   Samangouei, 2018), and the quantization methods (Bernhard et al., 2019; Khalid et al., 2019; Song
105   et al., 2021). In (Athalye et al., 2018), it was shown that most of these defences can be circumvented
106   by properly learning the gradients. Other models were also introduced to defend adversarial samples,
107   including the ensemble adversarial training (Tramèr et al., 2017), denoising layer (Xie et al., 2019),
      and difference-privacy noise layer (Lecuyer et al., 2019),


                                                           2
      Under review as a conference paper at ICLR 2025


108
      The bias classifier introduced in this paper is similar to gradient obfuscation defense, but is eseentially
109   different and has the following novel properties. First, the bias classifier has zero gradient, unlike
110   other methods, which try to make the gradients approximately zero. Second, we show that BPDA
111   attack cannot break the bias classifier in our experimental settings, while the BPDA attack can
112   break most gradient obfuscated defences (Athalye et al., 2018). Finally, the bias classifier can be
113   made provably robust against the original model gradient-based attack. Previous random gradient
114   obfuscation models do not claim provably robustness.
115   Many effective methods have been proposed to train more robust DNNs to defend adversarial
116   attacks (Xu et al., 2020; Shafahi et al., 2019b; Zhang et al., 2019; Wu et al., 2020; Nie et al., 2022).
117   Adversarial training (Madry, 2017) is one of the best practical defenses and can achieve optimal
118   adversarial accuracy in certain sense (Gao et al., 2022). In this paper, adversarial training (Madry,
119   2017) is used for a new loss function to train the bias classifier.
120
      Provable Robustness. Provable or certified robustness for DNNs can be achieved by verification or
121   by deriving explicit robust radius bounds or boundaries. Certified verification aims to decide whether
122   a trained DNN is robust by computing the robust boundaries (Li et al., 2023; Wang et al., 2021;
123   Li et al., 2022b). Explicit lower bounds for robust radius are given for DNNs trained with various
124   approaches such as randomized smoothing (Cohen et al., 2019) or L2 regulations (Yu & Gao, 2023).
125   In (Hein & Andriushchenko, 2017; Raghunathan et al., 2018), some security boundaries were given.
126   Compared to the above approaches, our approach is more practical in that our models are trained like
127   the usual adversarial training, while certified verification usually requires heavy computation, such as
128   SMT solving (Katz et al., 2017) and only works for small DNNs, and the explicit robust radius is
129
      usually very small (Yu & Gao, 2023) to be useful for commonly used DNN models.

166   result of BF (x). Specifically, we define classification functions as follows (Cybenko, 1989).
167   Definition 2. G : In → [m] is called a classification function, if In = ∪m    i=1 Pi is a partition of I
                                                                                                              n
168   into m disjoint and measurable subsets and G(x) = i if and only if x ∈ Pi .
169   Remark 3. Real-world classification problems such as MNIST or CIFAR-10 can be treated as
170   classification functions as follows. Let O ⊂ In be the objects to be classified. For x ∈ O and r ∈ R+ ,
171   when r is small enough, it is reasonable to treat all objects in
172                                   B(x, r) = {x + η | η ∈ Rn , ||η||∞ < r}
173   as images that have the same label as x. Furthermore, for all elements in In \ ∪x∈O B(x, r), we may
174   assign a new label. In this way, all elements of In have a label.

176   The following existence theorem shows that the bias classifier has the power to interpolate any
177
      classification functions with arbvitray high probability.
178   Theorem 4. Let G : In → [m] be a classification function. Then for any ϵ ∈ (0, 1/2), there
179
      exists a neural network F : In → Rm and an open set D ⊂ In with volume V (D) < ϵ, such that

184   network F : In → Rm with ReLU as the activation function can be obtained from G, which satisfies

      6   C ONCLUSION
493
      In this paper, we show that the bias part of a DNN can be effectively trained as a classifier. The bias
494
      classifier is shown to have universal power to approximate classification problems. Furthermore, the
495   bias classifier can be made provably robust in certain sense. This answers the important question of
496   how to design neural networks that provably achieve the highest possible adversarial accuracy for
497   certain attacks. Finally, experimental results are used to show that the bias classifier is comparable to
498   other defensive DNNs of similar sizes against major types of adversarial attack.
499
      Limitations and Future Research. The biggest problem is how to design a network suitable for
500
      bias classifier, in this paper, we still use the classic network structure V GG or ResN et, but they
501   may not suitable for bias classifier. For further research, the estimations in Theorems 15 and 18 are
502   not optimal, and better estimations are desirable. Moreover, how to design a more suitable network
503   structure for bias classifier is also an interesting question.
