009                                                      A BSTRACT
010
011                Comparing two (large) language models (LMs) side-by-side and pinpointing their
012                prediction similarities and differences on the same set of inputs are crucial in many
013                real-world scenarios, e.g., one can test if a licensed model was potentially pla-
014                giarized by another. Traditional analysis compares the LMs’ outputs on some
015                benchmark datasets, which only cover a limited number of inputs of designed
016                perspectives for the intended applications. The benchmark datasets cannot pre-
017
                   pare data to cover the test cases from unforeseen perspectives which can help us
                   understand differences between models unbiasedly. In this paper, we propose a
018
                   new model comparative analysis setting that considers a large input space where
019
                   brute-force enumeration would be infeasible. The input space can be simply de-
020                fined as all token sequences that a LM would produce low perplexity on — we
021                follow this definition in the paper as it would produce the most human-readable
022                inputs. We propose a novel framework Model-diff that uses text generation by
023                sampling and deweights the histogram of sampling statistics to estimate predic-
024                tion differences between two LMs in this input space efficiently and unbiasedly.
025                Model-diff achieves this by drawing and counting the inputs at each prediction dif-
026                ference value in negative log-likelihood. Experiments reveal for the first time the
027                quantitative prediction differences between LMs in a large input space, potentially
028
                   facilitating the model analysis for applications such as model plagiarism.

      1       I NTRODUCTION
031
032
      It is crucial in many real-world scenarios to compare two (large) language models (LMs) side-by-
033
      side and pinpoint their prediction differences. For example, the prediction differences may help
034   identify which model agrees more with human annotations (Liu et al., 2020; Hendrycks & Gim-
035   pel, 2016; Hendrycks et al., 2019; Hsu et al., 2020; Lee et al., 2017; 2018; Liang et al., 2018;
036   Mohseni et al., 2020; Ren et al., 2019; Szegedy et al., 2013; Rozsa et al., 2016; Miyato et al.,
037   2018; Kurakin et al., 2016; Xie et al., 2019; Madry et al., 2017); it can also identify model plagia-
038   rism between a licensed open-sourced model and its small variate version whose weights are added
039   small noise (PrimerYang). Traditional analysis compares the LMs’ outputs on the same benchmark
040   datasets which only cover a limited number of inputs from their designed perspectives. However,
041   (large) LMs have recently been deployed online and widely accessible to the public. As users in
042
      principle can input any types of data that could potentially cause the models to behave unexpectedly,
      it could be beneficial if we can compare models by a large amount of data without favoring the
043
      designed perspectives. The challenge of using benchmark datasets in this case is twofold: (1) the
044
      tested perspectives are limited by the types of test sets, and (2) the variety of inputs from the same
045   perspective is limited by the dataset size.
046
047   In this paper, we propose to compare models with a large number of data, or more generally, in a
048
      (discrete) input space that is finite but computationally impossible to enumerate all inputs. While
      the combination of all token sequences is a straightforward space, it is not ideal as most of them
049
      are sequences with random tokens, hence not beneficial for analysis. One reasonable input space
050
      is the collection of human-understandable inputs. It can be defined as all token sequences that an
051   LM produces negative log-likelihood (NLL)1 within a range of small values. Here we do not treat
052   models as language generator but as language modeler, where a (good) language model is expected
053
          1
              NLL is log-probability or log-perplexity which is the loss to train next token prediction for LMs.

      To tackle this new evaluation of a large input space, we propose a sampling-based comparative
      analysis framework, Model-diff, that can efficiently estimate the prediction difference for the types
081
      and count of the inputs at each level of the prediction difference value. Consider two hypothetical
082
      models that optimize the NLL loss. They are fine-tuned to domain tasks as code-LM (model MA )
083   and math-LM (model MB ). In Fig 1(a), the types of inputs that code-LM predicts with a low NLL
084   could be the variable assignment code such as “i=i+1”, whereas math-LM thinks this text is a wrong
085   math equation and thus assigns a high NLL (Fig 1(b)). If two models predict similar NLLs for each
086   input in an input space, the count of prediction difference (e.g. D = NLLA − NLLB ) concentrates
087   around 0, indicating the models are similar. If not, we can quantify the difference by the number of
088   inputs at each D. Because the input space is not enumerable, we sample the input space to assess the
089   prediction differences between the two models by scrutinizing the inputs and counting the number
090   of inputs given different output values (e.g., there are 10 math equations when D = −5 and 23 code
091
      inputs when output is D = 10 etc), similar to Fig 1(d).
092   Model-diff samples the models whose predictions are within a range of low NLL. The outputs of
093   sampling, including the sampled inputs and the count histogram that is processed to get the output
094   distribution, can be used to compare the types and counts of the inputs. Output distribution (Liu
095   et al., 2023b) is a distribution of the count of the inputs given each output value, the exact quantity for
096   count comparison. Comparing the summations of two models over this quantity for differentvalues
097
      can help compare the total count of inputs in an input space even though the input space is usually too
      large to be enumerated. Scrutinizing the sampled inputs for different outputs can help understand the
098
      types of the inputs. Model-diff leverages them to quantify the agreed/disagreed predictions between
099
      the two models. Moreover, to ensure a fair comparison, each model will propose its own input space
100   containing inputs of low NLLs and compare predictions of the other model. Model-diff involves a
101   novel normalization strategy to normalize the two output distributions sampled from the input spaces
102   proposed by each model.

117   2     T HE M ODEL - DIFF F RAMEWORK
118
119   Model-diff leverages the output distribution at each prediction difference D and the corresponding
120   inputs mapped to D to analyze the types and count of the inputs at each D value. We first introduce
121   the concept of output distribution and then how we use it for prediction difference analysis in an
122   ideal case when enumeration is available. Next section we will discuss how to derive the quantities
123   needed for this analysis when enumeration is replaced by sampling.
124   Background: Output distribution. Given the entire discrete input space Ω = {0, ..., M }N and a
125   training set ΩT ⊆ Ω, a model f (x) learns to map inputs x ∈ ΩT to output z ∈ R. As the current
126   language models (LMs) are trained to predict the next token, we choose the loss function, negative
127   log-likelihood (NLL), as the output. Later we also define output distribution for the parameter of
128   prediction difference D. Each input x is a sequence of N tokens. M + 1 is the vocabulary size.
129   Each of the N tokens takes one of the M + 1 words. The output distribution in an input space
130   Ω∗ is the distribution of the count for each z. Ω∗ can be Ω or some other space ΩM specified by a
131   generative model M. As every input in Ω∗ holds equal importance for analysis, the inputs within Ω∗
132
      should follow the principle of equal a priori probabilities – each input within Ω∗ follows a uniform
      distribution. Mathematically, the output distribution ρ(z) is defined as:
133
                                                    X
134                                         ρ(z) =       δ(z − f (x)),
135                                                  x∈Ω∗
136
137   where δ(·) is 1 if the input z − f (x) = 0, or δ(·) is 0 otherwise. In practice, a histogram is used
138
      to collect the statistics (the y-axis is the count and the x-axis is the output values z). The sampled
      inputs with similar output values in a small range [z − ∆z, z + ∆z) are called representative inputs
139
      at z and are mapped to the same z bin. ∆z is a small positive constant.

      In reality, enumeration is impossible because of computation inefficiency. We need to estimate the
219   key distributions ρA→B (D) and ρB→A (D) by sampling. We first introduce the background of text
220   generation by sampling, and the method(s) to sample the output distribution. We then discuss how
221   to acquire comparable ρA→B (D) and ρB→A (D) through output distribution and normalization.
222
223   3.1   BACKGROUND AND TERMINOLOGY
224
225    Text Generation by Sampling. Besides generating the next token in an autoregressive manner,
226   sampling methods are common in text generation in language models (Kumar et al., 2022; Qin et al.,
227   2022), by Markov Chain Monte-Carlo (MCMC). It starts with a sequence of random tokens and by
228   tweaking the tokens randomly to lower the NLLs, a sequence of understandable text is generated.
229   MCMC sampling is employed because enumeration of the input space in general is not possible.
230
      As pointed out (Du et al., 2023), text generation by sampling in principle should employ samplers
      of discrete input space (Goshvadi et al., 2024; Grathwohl et al., 2021; Zhang et al., 2022). These
231
      samplers sample the target distribution
232
233                                            p(x) ∝ exp(g(x)/T ),                                        (6)
234
      where g(·) is called (negative) “energy” and T is a predefined parameter (temperature). When T is
235
      1, g(·) is the log-probability which is popular in many machine learning problems when they learn
236
      to model log-probability (LeCun et al., 2006). p(x) in Equ. 6 is a common target distribution for
237   sampling in machine learning. Importantly, it biases the inputs with high g(x).
238
239   Model-diff adopts the exact same sampling setting of discrete inputs and this is the major bottleneck
240
      of Model-diff. The time complexity of Model-diff is therefore similar to text generation by sampling.
      Post-processing of Model-diff after text generation by sampling only takes a few hours.
241
242   Sampling the output distribution.                Parallel Tempering and Histogram Reweighting
243   (PTHR) (Hukushima & Nemoto, 1996; Swendsen & Wang, 1986) is commonly used to sample
244   output distribution. It starts with the results of text generation by sampling for target distribution of
245   Equ. 6. Because the MCMC samplers sample x more often whose output g(x) is larger, it needs
246
      reweights the sampled distributions by exp(·) to acquire the output distribution. Therefore, sam-
      pling output distribution can generate the same statistics as if we were sampling uniformly the input
247
      space without biasing the inputs with large g(x). Moreover, PTHR is a downstream task of text
248
      generation by sampling and it is compatible with MCMC samplers. Therefore it can take advantage
249   of the development of MCMC samplers that follow the same target distribution of Equ. 6.

      3.2   S AMPLING WITH PROBABILITY WEIGHTS OF ρ̃A (z) OR ρ̃B (z)
252
253    For very large input space, exact values of ρA→B (D) and ρB→A (D) cannot be estimated because
254   XA and XB (Sec. 2.1 Introductory example and goal) are not available as enumeration of the input
255   space is infeasible. Thus, we need to estimate them by sampling. We denote the sampled quan-
256   tity used for practical analysis with “Tilde” (e.g. ρ̃) in contrast to the quantity from ground truth
257   enumeration without “Tilde” (e.g. ρ) for conceptual discussion purposes.
258   As mentioned in Sec. 2.1, we focus on the case where every input whose outputs within Z as equally
259   important; therefore the outputs that contain more inputs should be sampled more often. Text gen-
260   eration by sampling is not directly applicable because it favors low NLL. We instead leverage the
261   output distribution ρA (z) (or ρB (z)) that describes the various numbers of inputs mapped to each
262   output value by a model. For example, as shown in Fig. 1 (b), one output value of model B has
263   4 inputs, and should be selected 2 times more frequently than the other output value with only 2
264   inputs. Output distribution ρA (z) (or ρB (z)) ensures the sampled representative inputs follow the
265   frequency of appearance for the different output values in Z for model A (or B) result in a sampling
266
      process as if we were uniformly extracting inputs from XA and XB .
267   In practice, we apply the well-established algorithms of text generation by sampling and PTHR.
268   After text generation by sampling, we compute ρ̃A (z) and ρ̃B (z) that approximate ρA (z) and ρB (z)
269   through PTHR. We then sample an output value z with probability weights ρ̃A (z) (or ρ̃B (z)) so
      that the output z with more inputs will be sampled more often. Afterward, we uniformly choose an

      Our sampling target is the negative-log-likelihood (NLL), the training loss used for next-token pre-
337   dictions. Though a low NLL generally indicates the model (strongly) believes the input is close to
338   the training distribution, the inputs with very low NLLs are repeating words that are incomprehensi-
339   ble by humans (Appendix G and Holtzman et al. (2019)). Therefore, we generally set a reasonable
340   range of Z and only consider inputs whose NLL ∈ Z. We choose the range Z by ensuring the
341   bins have human-understandable inputs, but our method works in any range (input space) selected
342   for the specific task. Fig. 2 shows the sampling results of D = NLLA − NLLB with one unit of
343   standard deviation for three runs (two for Toy) after we have the PTHR results. Tab. 1 shows the
344   detailed statistics about Fig. 2 for Model-diff analysis. More details of experimental settings are in
345
      Appendix F.

453   4.4      A PPLICATIONS
454
455    We demonstrate a few real-world examples of applications using the types and counts Model-diff
456   focuses, as we have confirmed the applicability of Model-diff.
457
      Deciding which model is better. We define our task of which model is better in terms of which
458
      model’s prediction agrees more with human annotation. We achieve this by annotating the in-
459
      puts. We choose to annotate the inputs from -1 to -0.6 and from 1 to 0.6 where the dominant
460   number of inputs concentrates and |D| is not too small when the two models do not show signif-
461   icant prediction differences. We sum from the -1 to -0.6 for PDsmall→medium and from 1 to 0.6
462   for PDmedium→small . Using Equ. 4 and Equ. 5, we compute 0.56 precision and recall is 0.32 for
463   GPT2-small-25. For GPT2-medium-25, we compute 0.58 precision and recall is 0.57. This shows
464   while the two models disagree with the prediction of the other model’s representative inputs, GPT2-
465   medium-25’s disagreement aligns more closely to human annotation. Therefore, GPT2-medium is
466   a better model as its prediction agrees more with human annotation. Without introducing extra bi-
467   ases from datasets, we can use Model-diff to attain a better understanding of the models’ prediction
468
      agreement and disagreement.
469   Model-plagiarism. Nowadays, open-sourced LMs are easily accessed for commercial and research
470   purposes. It remains an open question whether the new models are sufficiently distinct from their
471   original counterparts or if they are merely altered by adding noise to the weights (PrimerYang). We
472   offer a different angle to approach this problem than watermarking. We test Model-diff by compar-
473   ing GPT2-small-25 and GPT2-small-0.001-noise-25 where we add Gaussian noise to each weight
474
      with zero-mean and standard deviation = 0.001 (Fig. 3(a)). It shows that GPT2-small-noise-25
      almost always predicts a higher NLL on GPT2-small-25’s representative inputs. This is reason-
475
      able since GPT2-small-noise-25 with noisy weights predicts inputs with higher NLL in general.
476
      However, it is noteworthy that GPT2-small-25 predicts a lower NLL on almost all GPT2-small-
477   noise-25’s representative inputs. This is in contrast with the output distributions in Fig. 2 where two
478   different models disagree on each other’s representative inputs. We further compare GPT2-small-
479   25 and GPT2-small-0.00001-noise-25 a smaller noise with standard deviation 0.00001 to weights.
480   Fig. 3(b) shows consistent results, though the area of overlap is larger because the two models are
481   more similar. This shows the output distributions that Model-diff focuses on can produce potentially
482   useful signal to detect if a model is sufficiently different from its original counterpart. This signal
483   can be an indicator for further confirmation of plagiarism.

499   5       R ELATED W ORKS AND D ISCUSSIONS
500
501   Model understanding and analysis. Recent works (Booth et al., 2021; Liu et al., 2023a) propose
502   to understand models (Zeiler & Fergus, 2014; Ribeiro et al., 2016; Lundberg & Lee, 2017; Ghorbani
503   et al., 2019) beyond the datasets by sampling the model itself, which can also avoid being biased even
504   if the dataset is generated by (external) models (Luo et al., 2023; Prabhu et al., 2023; Shu et al., 2020;
505
      Leclerc et al., 2022). Model-diff follows the recent methods of estimating (Liu et al., 2023b) and
      leveraging the output distribution for analysis (Liu et al., 2023a). Its new normalization algorithms
506
      facilitate the analysis of model prediction differences without the need to sample accurately all the
507
      output values. Strobelt et al. (2021) proposes a microscopic view of how each token is predicted
508   differently by the two models on the same input. It can serve as a microscopic analysis tool for
509   Model-diff once the representative inputs are sampled. Model-diff, on the other hand, examines two
510   important macroscopic properties: the types and count of inputs.
511
      Open-world Model Evaluation is a unique challenge including in out-of-distribution detection (Liu
512
      et al., 2020; Hendrycks & Gimpel, 2016; Hendrycks et al., 2019; Hsu et al., 2020; Lee et al., 2017;
513
      2018; Liang et al., 2018; Mohseni et al., 2020; Ren et al., 2019), adversarial sets (Szegedy et al.,
514   2013; Rozsa et al., 2016; Miyato et al., 2018; Kurakin et al., 2016; Xie et al., 2019; Madry et al.,
515   2017) etc. Instead of targeting specific types of inputs, Model-diff addresses the model comparison
516   in an input space through output distribution. The two models first efficiently map the inputs in the
517   input space to different output values. Human inspection of the mapping follows after computing
518   prediction difference D.
519
      Samplers for output distribution were known in physics as sampling the density of states (Wang
520   & Landau, 2001). The connection between the two has been discovered recently (Liu et al.,
521   2023b). Parallel tempering and histogram reweighting algorithms can also sample output distri-
522   bution (Hukushima & Nemoto, 1996; Swendsen & Wang, 1986), which are more compatible with
523   the machine learning samplers (Grathwohl et al., 2021; Zhang et al., 2022) for energy function for
524   discrete input space.
525
526
527   6       C ONCLUSION AND F UTURE W ORKS
528
529   We propose a novel framework, Model-diff, for comparative analysis between two models with-
530   out introducing external models or datasets. Model-diff leverages the output distributions and the
531   corresponding representative inputs of the two models to understand the types and quantity of the
532
      agreed/disagreed predictions in each model’s meaningful input space. In future work, more efficient
      samplers could speed up the sampling procedure for Model-diff. Moreover, better normalization can
533
      be developed for comparing more than two models.

674   B    L IMITATION
675
676   Our framework is designed to be a general framework, but it is not preferable for all settings. First,
677   our analysis depends on the sampler(s). As sampling the output distribution is a relatively new topic
678
      in the machine learning community, more advanced samplers with more computation resources can
      scale our experiments. Although our proof-of-concept method depends on the samplers’ results, the
679
      analysis method itself is parallel to the development of the sampler, meaning that the method of how
680
      to use output distributions to analyze the models will be consistent, even though the sampled results
681   may improve with better samplers.
682
683   Another is our analysis focuses on NLL. While it is the training loss for many next-token-
684
      predictions, it does not cover other interesting problems in NLP that do not use the loss. Our method
      in general targets a set of problems that uses log-probability as output. This problem is covered as
685
      energy-based models LeCun et al. (2006), where the “energy function” (log-probability) is a mea-
686
      surement of the compatibility between the (input) variables. Therefore, our method can also choose
687   these measurements as output to be sampled. Moreover, it is also important to scale our method to
688   multi-dimensional output, such as feature embedding analysis. Concrete examples of applications
689   for problems beyond NLL are left as future work.

785    GPT2-Toy is a simple experiment with dataset of sequences {x(i) } with length 8. Each token xj
786   for an input x(i) is an integer from
                                        P 0 to 9 (vocabulary size is 10). The modulo of the sum of the
787   sequences is required to be 0: ( xj ) mod 30 = 0. The entire input space for this setting is 108
788   which is enumerable. There are around 3.8 million sequences that satisfy the modulo requirement,
789   and we pick 500K to build theP    training set. We use two GPT2 models to learn to generate the
790   sequences whose sum satisfies ( xj ) mod 30 = 0. The GPT2-small-Toy has 4 heads and 6 layers.
791
      The GPT2-large-Toy has 8 heads and 8 layers. The number of embeddings for both models is
      64. After training, both models can generate sequences that satisfy the modulo requirement with
792
      100.0%.

795   the two models to be compared using a PTHR. We then sample D through the representative inputs
796   within Z with 100000 steps for GPT2 experiments or 50000 steps for other experiments.
797   GPT2-small-25 samples 25 tokens with the GPT2-small and GPT2-medium-25 samples 25
798   tokens with GPT2-medium. Both models are sampled NLL in [2.0, 4.0] with temperature

800   with the GPT2-small and GPT2-medium-100 samples 100 tokens with GPT2-medium. The output
