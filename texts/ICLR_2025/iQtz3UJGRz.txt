009                                                      A BSTRACT
010
011                We propose a new “bi-metric” framework for designing nearest neighbor data
012                structures. Our framework assumes two dissimilarity functions: a ground-truth
013                metric that is accurate but expensive to compute, and a proxy metric that is cheaper
014                but less accurate. In both theory and practice, we show how to construct data
015                structures using only the proxy metric such that the query procedure achieves the
016                accuracy of the expensive metric, while only using a limited number of calls to
017                both metrics. Our theoretical results instantiate this framework for two popular
018
                   nearest neighbor search algorithms: DiskANN and Cover Tree. In both cases
                   we show that, as long as the proxy metric used to construct the data structure
019
                   approximates the ground-truth metric up to a bounded factor, our data structure
020
                   achieves arbitrarily good approximation guarantees with respect to the ground-
021                truth metric. On the empirical side, we apply the framework to the text retrieval
022                problem with two dissimilarity functions evaluated by ML models with vastly
023                different computational costs. We observe that for almost all data sets in the
024                MTEB benchmark, our approach achieves a considerably better accuracy-efficiency
025                tradeoff than the alternatives, such as re-ranking.

028   1       I NTRODUCTION
029
030   Similarity search is a versatile and popular approach to data retrieval. It assumes that the data items
031   of interest (text passages, images, etc.) are equipped with a distance function, which for any pair of
032   items estimates their similarity or dissimilarity1 . Then, given a “query” item, the goal is to return
033   the data item that is most similar to the query. From the algorithmic perspective, this approach is
034
      formalized as the nearest neighbor search (NN) problem: given a set of n points P in a metric space
      (X, D), build a data structure that, given any query point q ∈ X, returns p ∈ P that minimizes
035
      D(p, q) . In many cases, the items are represented by high-dimensional feature vectors and D is
036
      induced by the Euclidean distance between the vectors. In other cases, D(p, q) is computed by a
037   dedicated procedure given p and q (e.g., by a cross-encoder).
038
039   Over the last decade, mapping data items to feature vectors, or estimation of similarity between
040
      pairs of data items, is often done using ML models. (In the context of text retrieval, the first task
      is achieved by constructing bi-encoders Karpukhin et al. (2020); Neelakantan et al. (2022); Gao
041
      et al. (2021b); Wang et al. (2024), while the second task uses cross-encoders Gao et al. (2021a);
042
      Nogueira et al. (2020); Nogueira & Cho (2020)). This creates efficiency bottlenecks, as high-accuracy
043   models are often larger and slower, while cheaper models do not achieve the state-of-the-art accuracy.
044   Furthermore, high-accuracy models are often proprietary and accessible only through a limited
045   interface at a monetary cost. This motivates studying “the best of both worlds” solutions which utilize
046   many types of models to achieve favorable tradeoffs between efficiency, accuracy and flexibility.
047
      One popular method for combining multiple models is based on “re-ranking” Liu et al. (2009).
048
      It assumes two models: one model evaluating the metric D, which has high accuracy but is less
049   efficient; and another model computing a “proxy” metric d, which is cheap but less accurate. The
050   algorithm uses the second model (d) to retrieve a large (say, k = 1000) number of data items with
051   the highest similarity to the query, and then uses the first model (D) to select the most similar items.
052   The hyperparameter k controls the tradeoff between the accuracy and efficiency. To improve the
053

055   neighbor data structures. Such data structures are constructed for the proxy metric d, so they remain
056   stable even if the high-accuracy metric D undergoes frequent updates.

058
      Despite its popularity, the re-ranking approach suffers from several issues:
059
      1. The overall accuracy is limited by the accuracy of the cheaper model. To illustrate this phenomenon,
060
         suppose that D defines the “true” distance, while d only provides a “C-approximate” distance,
061
         i.e., that the values of d and D for the same pairs of items differ by at most a factor of C > 1.
062      Then the re-ranking approach can only guarantee that the top reported item is a C-approximation,
063      namely that its distance to the query is at most C times the distance from the query to its true
064      nearest neighbor according to D. This occurs because the first stage of the process, using the
065      proxy d, might not retain the most relevant items.
066
      2. Since the set of the top-k items with respect to the more accurate model depends on the query,
067
         one needs to perform at least a linear scan over all k data items retrieved using the proxy metric d.
068      This computational cost can be reduced by decreasing k, but at the price of reducing the accuracy.
069
070
      Our results We show that, in both theory and practice, it is possible to combine cheap and expensive
071   models to achieve approximate nearest neighbor data structures that inherit the accuracy of expensive
072   models while significantly reducing the overall computational cost. Specifically, we propose a
073   bi-metric framework for designing nearest neighbor data structures with the following properties:
074
075   • The algorithm for creating the data structure uses only the proxy metric d, making it efficient to
076     construct,
077   • The algorithm for answering the nearest neighbor query leverages both models, but performs only
078     a sub-linear number of evaluations of d and D,
079
080
      • The data structure achieves the accuracy of the expensive model.
081
      For a more formal description of the framework, see Preliminaries (Section 2).
082
083   The simplest approach to constructing algorithms that conform to our framework is to construct the
084   data structure using the proxy metric d, but answer queries using the accurate metric D; we also
085   propose more complex solutions that have better practical performance. We note our approach is
086   quite general, and is applicable to any approximate nearest neighbor data structure that works for
087
      general metrics. Our theoretical study analyzes the simple approach when applied to two popular
      algorithms: DiskANN Jayaram Subramanya et al. (2019) and Cover Tree Beygelzimer et al. (2006),
088
      under natural assumptions about the intrinsic dimensionality of the data, as in Indyk & Xu (2023).
089

090   the query answering procedure essentially retains the accuracy of the ground truth metric D.

      The proof of the theorem crucially uses the properties of the underlying data structures. It is an
109   interesting open direction to determine if our bi-metric framework can be theoretically instantiated
110   for other popular nearest neighbor algorithms, such as those based on locality sensitive hashing.

112
      To demonstrate the practical applicability of the bi-metric framework, we apply it to the text retrieval
      problem. Here, the data items are text passages, and the goal is to retrieve a passage from a large
113
      collection that is most relevant to a query passage. We instantiated our framework with the DiskANN
114
      algorithm, using a high-quality “SFR-Embedding-Mistral” model Meng et al. (2024) to define D, and
115   a lower-quality “bge-micro-v2” model AI (2023) to define d. Both metrics d(p, q) and D(p, q) are
116   induced by the Euclidean distance between the embeddings of p and q using the respective models.
117   The sizes of the two models differ by 3 orders of magnitude, making D much more expensive to
118   evaluate than d. Specifically, in our experiments, embedding a single passage takes 0.00043 seconds
119   when using bge-micro-v2 and 0.13 seconds when using SFR-Embedding-Mistral, making the second
120   model >300 times slower.
121   We evaluated the retrieval quality of our approach on a benchmark collection of 15 MTEB retrieval
122   data sets Thakur et al. (2021), comparing it to the re-ranking approach, which retrieves the closest data
123   items to the query with respect to d and re-ranks using D. We observe that for almost all data sets, our
124   approach achieves a considerably better accuracy-efficiency tradeoff than re-ranking. In particular,
125   for several data sets, such at HotpotQA Yang et al. (2018), our approach achieves state-of-the-art
126   retrieval accuracy using up to 4x fewer evaluations of the expensive model.

128   1.1   R ELATED W ORK
129
130   As described in the introduction, a popular method for utilizing a cheap metric d and expensive
131   metric D in similarity search is based on "filtering" or “re-ranking”. The idea is to use d to construct
132   a (long) list of candidate answers, which is then filtered using D. It is a popular approach in many
133   applications, including recommendation systems Liu et al. (2022) and computer vision Zhong et al.
134   (2017). Due to the popularity of this method, we use it as a baseline in our experiments.
135   In addition to the re-ranking method, multiple other papers proposed different methods for combining
136   accurate and cheap metrics to improve similarity search and related problems. We discuss those
137   papers in more detail below. We note that, with the exception of Moseley et al. (2021); Silwal et al.
138   (2023); Bateni et al. (2024), those methods do not appear to come with provable correctness or
139   efficiency guarantees, or generally applicable frameworks (in contrast to the proposal in this paper).
140   Furthermore, the three aforementioned papers Moseley et al. (2021); Silwal et al. (2023); Bateni et al.
141
      (2024) focus on various forms of clustering, not on similarity search. The paper Moseley et al. (2021)
      is closest to our work, as it uses approximate nearest neighbor as a subroutine when computing the
142
      clustering. However, their algorithm only achieves the (lower) accuracy of the cheaper model, while
143
      our algorithms retains the (higher) accuracy of the expensive one.
144
145   There are also several other empirical works on similarity search that combine cheap and expensive
146   metrics, none of which fully capture our framework to the best of our knowledge. The aforementioned
147
      paper Jayaram Subramanya et al. (2023) describes (in section 3.1) an optimization which uses the
      ground truth metric D during the indexing phase, and proxy metric d during the search phase. In
148
      contrast, our framework uses D during the search phase and d during indexing. This difference
149
      seems crucial to our ability of providing strong approximation guarantees for the reported points,
150   which are not limited by the distortion C between d and D. In another paper Chen et al. (2023), the
151   authors use the proxy metric d obtained by “sketching” D during the query answering phase, in order
152   to prune some points from the search queue without resorting to computing D. However, the data
153   structure index is still constructed using the expensive metric D, as opposed the proxy metric d as
154   in our framework, which makes preprocessing more expensive in terms of space and time. Finally,
155   Morozov & Babenko (2019) present a method for constructing a similarity graph with respect to
156   an approximate distance function derived from a complex one; during the query phase the graph is
157   explored using a more complex relevance function. However, their algorithm uses specific proxy
158
      metric derived from the expensive one; in contrast, our framework allows arbitrary distance functions
      d and D, as long as the distortion C between them is bounded.
159

      graph-based data structures for (approximate) nearest neighbor search. Such data structures work

169   2   P RELIMINARIES
170
171   Nearest neighbor search We first consider the standard formulation of exact nearest neighbor
172   search. Here, we are given a set of points P , which is a subset of the set of all points X (e.g.,
173   X = Rd ). In addition, we are given access to a metric function D that, for any pair of points p, q ∈ X
174   returns the dissimilarity between p and q. The goal of the problem is to build an index structure that,
175   given a query point q ∈ X, returns p∗ ∈ P such that
176                                         p∗ = arg minp∈P D(q, p).
177
178   The formulation is naturally extended to more general settings, such as:
179
      • (1 + ε)-approximate nearest neighbor search, where the goal is to find any p∗ ∈ P such that
180
181
                                       D(q, p∗ ) ≤ (1 + ε) min D(q, p).
                                                               p∈P
182
      • k-nearest neighbor search, where the goal is to find the set of k nearest neighbors of q in P with
183     respect to D. If the algorithm returns a set S ′ of k points that is different than the set S of true k
184     nearest neighbor, the quality of the answer is measured by computing the Recall rate or NDCG
185     score Järvelin & Kekäläinen (2002).
186
187   Bi-metric framework: In our framework, we assume that we are given two metrics over X:
188
189   • The ground truth metric D, which for any pair of points p, q ∈ X returns the “true” dissimilarity
190
        between p and q. The metric D plays the same role as in the standard nearest neighbor search
        problem.
191
192   • The proxy metric d, which provides a cheap approximation to the ground truth metric.
193
      Throughout the paper, we think of D as being ‘expensive’ to evaluate, while d as the cheaper, but
194   noisy, proxy.
195
196   We assume that the algorithm for constructing the data structure has access to the proxy metric d, but
197
      not to the ground truth metric D. The algorithm for answering a query q has access to both metrics.
      However, the complexity of the query-answering procedure is measured by counting only the number
198
      of evaluations of the expensive metric D.
199
200   As described in the introduction, the above formulation is motivated by the following considerations:
201
      • Computing all embeddings using the expensive model D (e.g. SFR-Embedding-Mistral (Meng
202
        et al., 2024)) requires lots of time and space. For example, it takes an A100 gpu around 196 hours
203     to compute all embeddings of 5 million passages from the HotpotQA dataset and these embeddings
204     occupy 83GB of disk storage. Meanwhile, using the cheap model d (e.g. bge-micro (AI, 2023)),
205     computing these embeddings only takes 0.62 hours and 7GB of disk storage. As a comparison, the
206     graph index size of 5 million points occupies roughly 1GB of disk storage.
207   • Evaluating ground truth metric D during query answering time is also very expensive, due to factors
208     such as model size or monetary costs associated with querying proprietary models. Therefore, our
209     cost model for the query answering procedure only accounts for the number of such evaluations.
210   • In applications that use similarity search data structures in model training, the metric D can change
211     after each model update, necessitating re-computing embeddings and the search index over the
212     entire database. Since this is expensive, some works (e.g., (Borgeaud et al., 2022)) freeze the
213     parts of the model that compute embeddings to avoid the computational cost of updating the data
214     structure. Our framework offers an alternative approach, where one constructs a stable index for
215     a proxy d using frozen embeddings, but uses the up-to-date model to compute the ground-truth
        metric D when answering nearest neighbor queries.

      Assumptions about metrics: Clearly, if the metrics d and D are not related to each other, the data
217   structure constructed using d alone does not help with the query retrieval. Therefore, we assume that
218   the two metrics are related through the following definition.
219
      Definition 2.1. Given a set of n points P in a metric space X and its distance function D, we say the
220
      distance function d is a C-approximation3 of D if for all x, y ∈ X,
221
222                                          d(x, y) ≤ D(x, y) ≤ C · d(x, y).                                (1)
223
224
      For a fixed metric d and any point p ∈ X, radius r > 0, we use B(p, r) to denote the ball with radius
      r centered at p, i.e. B(p, r) = {q ∈ X : d(p, q) ≤ r}. In our paper, the notion of doubling-dimension
225
      is central. It is a measure of intrinsic dimensionality of datasets which is popular in analyzing high
226
      dimensional datasets, especially in the context of nearest neighbor search algorithms Gupta et al.
227   (2003); Krauthgamer & Lee (2004); Beygelzimer et al. (2006); Indyk & Naor (2007); Har-Peled &
228   Kumar (2013); Narayanan et al. (2021); Indyk & Xu (2023).
229
      Definition 2.2 (Doubling Dimension). X has doubling dimension λd with respect to metric d if for
230
      any p ∈ X, and radius r > 0, X ∩ B(p, 2r) can be covered by at most 2λd balls with radius r.
231
232   Finally, for a metric d, ∆d is the aspect ratio of the input set X, i.e., the ratio between the diameter
233   and the distance of the closest pair.

      3       T HEORETICAL ANALYSIS
236
237
      We instantiate our bi-metric framework for two popular nearest neighbor search algorithms: DiskANN
238   and Cover Tree. The goal of our bi-metric framework is to first create a data structure using the
239   proxy (cheap) metric d, but solve nearest neighbor to 1 + ε accuracy for the expensive metric D.
240   Furthermore, the query step should invoke the metric D judiciously, as the number of calls to D is
241   the measure of efficiency. Our theoretical query answering algorithms do not use calls to d at all.
242
      We note that, if we treat the proxy data structure as a black box, we can only guarantee that it returns
243
      a C-approximate nearest neighbor with respect to D. Our theoretical analysis overcomes this, and
244   shows that calling D a sublinear number of times in the query phase (for DiskANN and Cover Tree)
245   allows us to obtain arbitrarily accurate neighbors for D.
246
247
      At a high level, the unifying theme of the algorithms that we analyze (DiskANN and Cover Tree)
      is that they both crucially use the concept of a net: given a parameter r, a r-net is a small subset
248
      of the dataset guaranteeing that every data point is within distance r to the subset in the net. Both
249
      algorithms (implicitly or explicitly), construct nets of various scales r which help route queries to
250   their nearest neighbors in the dataset. The key insight is that a net of scale r for metric d is also a net
251   under metric D, but with the larger scale Cr. Thus, if we construct smaller nets for metric d, they
252   can also function as nets for the expensive metric D (which we don’t access during our data structure
253   construction). Care must be taken to formalize this intuition and we present the details below.
254
      We remark that the intuition we gave clearly does not generalize for nearest neighbor algorithms
255
      which are fundamentally different, such as locality sensitive hashing. For such algorithms, it is not
256   clear if any semblance of a bi-metric framework is algorithmically possible, and this is an interesting
257   open direction.
258
259
      In the main body, we present the (simpler) analysis of DiskANN and defer the analysis of Cover Tree
      to Appendix C.

      As described earlier, both metrics d(p, q) and D(p, q) are induced by the Euclidean distance between
365   the embeddings of p and q using the respective models. The embeddings defining the proxy metric
366   d are pre-computed and stored during the pre-processing, and then used to construct the data
367   structure. The embeddings defining the accurate metric D are computed on the fly during the query
368   processing stage. Specifically, to answer a query q, the algorithm first computes the embedding f (q)
369   of q. Then, whenever the value of D(q, p) is needed, the algorithm computes f (p) and evaluates
370   D(p, q) = ∥f (q) − f (p)∥. Thus, the cost of evaluating D(p, q) is equal to the cost of embedding p.
371   (In other scenarios where D(p, q) is evaluated using a proprietary system over the Internet, the cost is
372   determined by the vendor’s prices and/or the network speed.).

394
395   Please refer to Figure 1 for our results with d distance function set to “bge-micro-v2” and D set
396   to “SFR-Embedding-Mistral”, with the underlying graph index being DiskANN. To better focus on
397   the convergence speed of different methods, we cut off the y-axis at a relatively high accuracy, so
398   some curves may not start from x equals 0 if their accuracy doesn’t reach the threshold. We observe
399   that our method converges to the optimal accuracy much faster than bi-metric (baseline) and single
400   metric in most cases. For example for HotpotQA, the NDCG@10 score achieved by the baselines for
401
      8000 calls to D is comparable to our method, using less than 2000 calls to D, leading to >4x fewer
      evaluations of the expensive model. This leads to substantial time savings. For example, consider our
402
      largest data set HotpotQA. The first stage of the query answering procedure (using d) takes only 0.37s
403
      per query q, while each evaluation of D(p, q) during the second stage takes 0.13s; this translates
404   into roughly 260s per query when 2000 evaluations of D are used. In contrast, the baseline method
405   requires 8000 calls to D, which translates into a cost of roughly 1040s per query.
406
407
      This means that utilizing the graph index built for the distance function proxy to perform a greedy
      search using D is more efficient than naively iterating the returned vertex list to re-rank using D
408
      (baseline). It is also noteworthy to see that our method converges faster than “Single metric” in all
409
      the datasets except FiQA2018 and TRECCOVID, especially in the earlier stages. This phenomenon
410   happens even if “Single metric” is allowed infinite expensive distance function calls in its indexing
411   phase to build the ground truth graph index. This suggests that the quality of the underlying graph
412   index is not as important, and the early routing steps in the searching algorithm can be guided with a
413   cheap distance proxy functions to save expensive distance function calls.
414
      Similar conclusion holds for the recall plot (see Figure 4) as well, where our method has an even
415
      larger advantage over Bi-metric (baseline) and is also better than the Single metric in most cases,
416   except for FEVER, FiQA2018, and HotpotQA. We report the results of using different model pairs,
417   using the NSG algorithm as our graph index, and measuring Recall@10 in Appendix D. Please see
418   their ablation studies in Section 4.3.
419
      Lastly, we measure the empirical value of C (the relationship between d&D from (1)). For simplicity,
420
      we assumed that d ≤ D ≤ C · d for C ≥ 1 in (1) in our theoretical bounds. This is without loss of
421
      generality by scaling, and we could have alternatively written our theorem statements by substituting
422   (maxx,y D(x, y)/d(x, y))/(minx,y D(x, y)/d(x, y)) for C. In practice, we observe that the ratio
423   of distances C := D/d is always clustered around one. For example, if we use “SFR-Embedding-
424   Mistral” to provide the distance D, and “bge-micro-v2” to provide the distance d, then for HotpotQA,
425   we empirically found that 99.9% of 105 randomly sampled pairs satisfy 0.6 ≤ C ≤ 1.5. We observed
426   the same qualitative behaviour for our other datasets; see Figure 11 in the appendix for more details.

514   Impact of the first stage search In the second-stage search of our method, we start from multiple
515   points returned by the first-stage search via the cheap distance metric. We investigate how varying
516
      the starting points for the second-stage search impact the final results. We try four different setups:
517
      • Default: We start a standard nearest neighbor search using metric D from the default entry point of
518     the graph index, which means that we don’t use the first stage search.
519
      • Top-K points retrieved by the first stage search: Suppose our expensive distance calls quota is Q.
520
        We start our second search from the top K points retrieved by the first stage search. We experiment
521
        with the following different choices of K: K1 = 1, K100 = 100, KQ/2 = max(100, Q/2) (note
522     KQ/2 is the choice we use in Figure 1).
523
524   From Figure 3, we observe that utilizing results from the first-stage search helps the second-stage
525   search to find the nearest neighbor quicker. For comparison, we experiment with initializing the
526   second-stage search from the default starting point (green), which means that we don’t need the
527   first-stage search and only use the graph index built from d (cheap distance function). The DiskANN
528   algorithm still manages to improve as the allowed number of D distance calls increases, but it
529   converges the slowest compared to all the other methods.
530   Using multiple starting points further speeds up the second stage search. If we only start with the
531   top-1 point from the first stage search (brown), its NDCG curve is still worse than Bi-metric (baseline,
532   red) and Single metric (orange). As we switch to top-100 (purple) or top-Q/2 (blue) starting points,
533   the NDCG curves increase evidently.
534   We provide two intuitive explanations for these phenomena. First, the approximation error of the
535   cheap distance function doesn’t matter that much in the earlier stage of the search, so the first stage
536   search with the cheap distance function can quickly get to the true ‘local’ neighborhood without
537   any expensive distance calls, thus saving resource for the second stage search. Second, the ranking
538   provided by the cheap distance function is not accurate because of its approximation error, so starting
539   from multiple points should give better results than solely starting from the top few, which also
      justifies the advantage of our second-stage search over re-ranking.

795
796   The notion of a cover is central.
797
798   Definition C.1 (Cover). A r-cover C of a set X given a metric d is defined as follows. Initially C = ∅.
799   Run the following two steps until X is empty.
800
801         1. Pick an arbitrary point x ∈ X and remove B(x, r) ∩ X from X.
802
803         2. Add x to C.
804
805
806
      Note that a cover with radius r satisfies the following two properties: every point in X is within
      distance r to some point in C (under the same metric d′ ), and all points in C are at least distance r
807
      apart from each other.
808

      create a sequence of covers C−1 , C0 , . . .. Every Ci is a layer in the final Cover Tree T .
