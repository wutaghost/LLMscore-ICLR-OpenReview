011                                               A BSTRACT
012
013            Serving LLM inference for long sequences poses significant challenge due to the
014
               enormous memory footprint from key-value (KV) cache. In this paper, we propose
               a novel KV cache merging algorithm, KVMerger, to achieve adaptive KV cache
015
               compression for long sequence tasks without significant performance degradation.
016
               Our approach is inspired by the intriguing observation that key states exhibit high
017            similarity at the token level within a single sequence. Based on the observation, we
018            develop an effective merging set identification algorithm to identify suitable KV
019            states for merging and a Gaussian kernel weighted merging algorithm to selec-
020            tively merge all states within each merging set. Our merging set identification al-
021            gorithm stimulates the second observation that KV cache sparsity, from similarity
022            perspective, is independent of the dataset and remains persistent at the model level.
023            We conduct extensive experiments to demonstrate the effectiveness of KVMerger
024            for long-context tasks under constrained memory budgets, on Llama2-7B/13B-
025
               chat and Mistral-7B-instruct. Our results show that KVMerger achieves superior
               performance across tasks compared to other KV cache compression techniques,
026
               including H2O and CaM on LongBench, ZeroScroll, and Needle-in-a-Haystack
027
               benchmarks, with both 50% and 35% KV cache budgets.

030   1   I NTRODUCTION
031
032   Large Language Models (LLMs) have demonstrated exceptional performance across a variety of ap-
033
      plications, particularly excelling in long-context scenarios that are increasingly relevant in everyday
      life. However, as LLMs process larger volumes of data over extended contexts, KV cache starts to
034
      pose a substantial obstacle to LLM’s performance and scalability. For example, a 175-billion pa-
035
      rameter GPT-3 model, with a batch size of 64 and a sequence length of 4,096 tokens (including both
036   prefilled and generated tokens), necessitates approximately 1,208 GB of GPU memory (Liu et al.,
037   2024), which exceeds the memory capacity of most advanced GPUs. Therefore, compressing KV
038   cache while maintaining LLM accuracy, especially for long-context tasks, becomes quite essential.
039
      Current efforts for KV cache compression can be broadly categorized into three types: quantization,
040
      eviction, and merging, as illustrated in Figure 1. Quantization replaces floating point KV states
041
      (e.g., FP16) with low-bit values to decrease memory usage. Recent advancements, such as Coupled
042   Quantization (Zhang et al., 2024b) and KIVI (Zirui Liu et al., 2023), have demonstrated that KV
043   cache can be quantized to 1-bit or 2-bit precision while preserving performance. In contrast, KV
044   cache eviction methods selectively remove unimportant states based on certain signals from the
045   model, thereby reducing the memory footprint by limiting the number of key and value states in the
046   KV cache (Xiao et al., 2024; Liu et al., 2023b; Zhang et al., 2023; Ge et al., 2024). While eviction-
047   based methods have demonstrated promising results on short context tasks with simple perplexity
048   metrics, a significant drawback of eviction methods is their potential to accidentally and permanently
049   remove important tokens, leading to context damage and adversely affecting their effectiveness in
050
      long-context tasks that heavily rely on context information.
051   Very recently, KV cache merging has emerged a complementary method of eviction (Zhang et al.).
052   Unlike eviction-based methods, KV cache merging technique does not strictly remove key and value
053   states. Instead, it involves merging states that are otherwise to be dropped by eviction method into
      single token state. By amalgamating states rather than directly evicting them, this method ensures

077
      To address the aforementioned challenges associated with KV cache merging, we propose an effec-
      tive KV cache merging method for improving LLMs’ performance on long-context tasks. We start
078
      by introducing and analyzing an intriguing observation: key states exhibit high cosine similarity at
079
      the token level within a single sequence across different attention heads and model layers. our ob-
080   servation also opens opportunities for effective merging of key and value states based on their cosine
081   similarity. Subsequently, we formulate the KV cache merging as a constrained clustering problem,
082   and propose an effective merging set identification method for KV cache merging. Based on that,
083   we define KV cache sparsity from the perspective of states similarity. Our finding indicates that KV
084   cache sparsity is independent of the dataset and remains persistent at the model level. Building on
085   top of this, we propose a Gaussian kernel weighted merging function to merge states within each
086   identified merging set. Our contributions can be summarized as follows:
087   • As one of the pioneering researches concerning KV cache merging for LLMs, we developed
088     KVMerger, an effective KV cache merging algorithm especially designed for long-context tasks,
089     including merging set identification and Gaussian kernel weighted merging function.
090
      • We introduce an intriguing observation that key states share a high similarity at the token level
091     within a single sequence, as an important complementary to the previous observations concerning
092     high query states similarity (Dai et al., 2024) and intra-layer KV cache similarity (Liu et al., 2024).
093     We also investigate the root cause of why such phenomenon appears.
094
      • We evaluate KVMerger on diverse long-context tasks including LongBench, ZeroScroll, and
095
        Needle-in-a-Haystack across various models under different KV cache budgets, and the results
096     show comparable performance or much smaller performance gap compared to full cache budgets.

099
100
      KV Cache Quantization. Quantization methods involve converting high-precision numerical val-
101
      ues of key and value states into lower-precision formats, thereby decreasing the storage requirements
102   within the cache (Hooper et al., 2024; Sheng et al., 2023; Liu et al., 2023a; Zhang et al., 2024c).
103   Due to the presence of outliers in key and value states, recent works such as KIVI (Zirui Liu et al.,
104   2023) and Gear (Kang et al., 2024) employ fine-grained group-wise quantization, which quantize
105   small channel groups within each token. MiKV (Yang et al., 2024) addresses the information loss
106   introduced by KV cache eviction methods by preserving those KVs in lower precision rather than
107   directly dropping them. ZipCache (He et al., 2024) proposes an efficient channel-separable quantiza-
      tion scheme, disentangling the channel and token dimensions without excessive memory overhead.

108
      Different from quantization, this work studies compression of KV cache via states merging, which
109   is complementary to quantization.
110
111   KV Cache Eviction. KV cache eviction methods focus on retaining those important key-value
112
      pairs and discard those unimportant ones permanently. One of the common selection policies of
      key-value pairs is to exploit signals from the attention mechanism of LLMs. For example, H2O
113
      (Zhang et al., 2023), Scissorhands (Liu et al., 2023b), and RoCo (Ren & Zhu, 2024) compress
114
      KV cache by maintaining a small set of KV states whose corresponding tokens are determined by
115   the ranking of attention scores. StreamingLLM (Xiao et al., 2024) finds that keeping the initial
116   tokens, called attention sink, together with the recent window tokens is pivotal to maintain LLM’s
117   performance. More recently, Ge et al. (2024) and Yu et al. (2024) find that attention sinks also occurs
118   in the middle of the sentences, and Ge et al. (2024) introduces FastGen which can choose the most
119   appropriate compression strategy for each heads with different attention distribution patterns.
120
      KV Cache Merging. Instead of permanently discarding key and value states, KV cache merging
121   offers a promising direction for KV cache compression while maintaining the performance of LLMs.
122   As the first work concerning cache merging, CaM (Zhang et al.) adaptively merges to-be-evicted
123   value states into the remaining conserved value states, resulting in minimal output perturbation due
124   to the merging operation. Recently, several concurrent work also propose to merge KV cache. For
125   example, MiniCache (Liu et al., 2024) finds that KV states of some consecutive layers have high
126   similarity and proposes an effective intra-layer KV cache merging and restoration algorithms to
127   reduce memory usage by KV cache. Similarly, D2O Wan et al. (2024) selectively merges both value
128   and key states to be evicted with those to be conserved using an Exponential Moving Average (EMA)
129
      threshold, and uses weighted merging based on cosine similarity. However, these methods rely
      heavily on prior eviction strategies, defining effective KV cache merging algorithms remain open
130
      challenges. This paper is the first one that formally formulates the KV cache merging problem and
131
      conducts in-depth study of this problem via effective merging set identification policy and Gaussian
132   weighted merging algorithm while exploiting the high similarities among key states.

191   of this token level similarity in key states and then analyze the potential reasons behind it.

257

      tasks are also compared with existing eviction method H2O and merging method CaM.

417   Implementation details We test KVMerger in two compression scenarios. The first one is 50%
418   KV cache budget, where the proportion of recent tokens to be reserved is set to 0.17%, and the
419   proportion of states not selected for the merging process in terms of aggregated attention scores is
420   set to 0.12%. The remaining key states and value states participate in the merging process. The
421
      second compression scenario is 35% KV cache budget, where the proportion of recent tokens is set
      to 0.08%, and the proportion of states not selected for the merging process is set to 0.02%. The
422
      cosine similarity threshold for both two scenarios is set to 0.75. We conducted our experiments on
423
      a cluster with A100 40GB GPUs and a cluster with A100 80GB GPUs. The evaluation process
424   for LongBench and ZeroScrolls follows THUDM (2024) and Lab (2024). The implementation of
425   Needle-in-a-Haystack test follows Kamradt.

531
      In this paper, we propose KVMerger, an adaptive KV cache merging method inspired by the ob-
532
      servation that key states exhibit high and persistent similarity within each sequence, allowing for
533
      layer-wise KV cache compression. We initially abstract the merging set identification problem as
534   a constrained clustering problem and introduce a specialized greedy clustering algorithm to iden-
535   tify merging sets based on cosine similarities between key states. Furthermore, we implement a
536   Gaussian Kernel weighted merging method to merge key and value states within each merging set.
537   Compared to other KV cache eviction and merging methods, our approach achieves superior re-
538   sults on the LongBench and ZeroScrolls benchmark under the same cache budget. Additionally, our
539   method effectively recovers the model’s long-context retrieval capabilities, as demonstrated by the
      needle-in-a-haystack tests.
