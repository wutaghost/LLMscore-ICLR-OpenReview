009                                                A BSTRACT
010
011            Graph Neural Networks (GNNs) have shown remarkable performance in various
012            applications. Recently, graph prompt learning has emerged as a powerful GNN
013            training paradigm, inspired by advances in language and vision models. Here,
014            a GNN is pre-trained on public data and then adapted to sensitive tasks using
015            lightweight graph prompts. However, using prompts from sensitive data poses
016
               privacy risks. In this work, we are the first to investigate these risks in graph
               prompts by instantiating a membership inference attack that reveals significant
017
               privacy leakage. We also find that the standard privacy method, DP-SGD, fails to
018
               provide practical privacy-utility trade-offs in graph prompt learning, likely due to
019            the small number of sensitive data points used to learn the prompts. As a solution,
020            we propose two algorithms, DP-GPL and DP-GPL+W, for differentially private
021            graph prompt learning based on the PATE framework, that generate a graph prompt
022            with differential privacy guarantees. Our evaluation across various graph prompt
023            learning methods, GNN architectures, and pre-training strategies demonstrates that
024            our algorithms achieve high utility at strong privacy, effectively mitigating privacy
025            concerns while preserving the powerful capabilities of prompted GNNs.

      1    I NTRODUCTION
028
029
      Graph Neural Networks (GNNs) have emerged as a powerful tool for learning representations of
030   graph-structured data and have shown significant advancements across various applications, such
031   as drug design (Al-Rabeah & Lakizadeh, 2022; Qian et al., 2023), anomaly detection (Sun et al.,
032   2022b; Tang et al., 2022), and social network analysis (Chen et al., 2020). Recently, graph prompt
033   learning (Sun et al., 2023d; Zi et al., 2024; Sun et al., 2023b; Fang et al., 2024; Sun et al., 2022a;
034   2023a) has emerged as a promising GNN training paradigm. Graph prompt learning first pre-trains a
035   GNN model on general public graph data and then tunes a graph prompt (Sun et al., 2023b; Huang
036   et al., 2024; Ge et al., 2023) or tokens (Fang et al., 2024; Sun et al., 2022a; Liu et al., 2023b) on
037   some sensitive downstream data. By reformulating the downstream task into the pretext task used in
038
      pre-training, it then enables predictions for the downstream task.
039   The fact that graph prompts are tuned on sensitive downstream data can raise significant privacy
040   concerns. In fact, in the language and vision domains, it has been shown that private information
041   from downstream data can leak through predictions of prompted models (Duan et al., 2023b; Wu
042   et al., 2023). To the best of our knowledge, no such insights exist for the graph domain, and no prior
043   work has explored the privacy risks of graph prompt learning.
044   In this work, we set out to close this gap. We first assess the privacy risks of graph prompts by
045   adapting a state-of-the-art membership inference attack (Shokri et al., 2017; Carlini et al., 2022) to
046   graph prompt learning and measuring the empirical leakage. Our evaluation demonstrates significant
047   privacy risks for the downstream data when used to tune graph prompts. For example, we show that
048   the membership inference attack can achieve an AUC score as high as 0.91 on the PubMed dataset.
049   We also investigate the relationship between the number of data points used to tune the prompt and the
050
      attack success and find that with less data, the privacy risk grows, posing a significant risk to standard
      graph prompt learning that usually relies on a small number of data points (Sun et al., 2023a).
051
052   As a naive solution to mitigate this privacy risk, we first turn to the Differential Privacy-Stochastic
053   Gradient Descent (DP-SGD) algorithm (Abadi et al., 2016)—a gold standard in privacy-preserving
      machine learning. However, we find that this approach significantly degrades the downstream

105   2.1   P ROMPT L EARNING
106
107   Prompt learning is a new machine learning paradigm that has been recently proposed to improve the
      performance of large models while addressing the limitations of fine-tuning (Li & Liang, 2021; Lester

108
      et al., 2021; Liu et al., 2023a). The idea is to learn a task-specific prompt that can be added to the
109   input data while freezing the pre-trained model’s parameters. In addition to many effective prompt
110   methods in the language domain, such as hand-crafted textual prompts (Brown, 2020), automated
111   discrete prompts (Gao et al., 2020; Shin et al., 2020), and trainable prompts in the continuous space
112   (Li & Liang, 2021; Liu et al., 2021), also in the vision domain (Jia et al., 2022; Sohn et al., 2023) and
113   for multi-modal models (Zhou et al., 2022), prompt tuning has become a prevalent paradigm.
114
115   2.2   GNN S AND G RAPH P ROMPT L EARNING
116
117   GNNs achieve strong performance on numerous applications (Sun et al., 2023c; Tang et al., 2022;
118   Chen et al., 2020). Therefore, they rely on various effective architectures, such as Graph Convolutional
119
      Network (GCN) (Kipf & Welling, 2022), Graph Attention Network (GAT) (Veličković et al., 2018a),
      and Graph Transformer (Shi et al., 2020)—usually trained in a supervised manner. To make graph
120
      learning more adaptive, many graph pre-training approaches have been proposed (Veličković et al.,
121
      2018b; Hou et al., 2022; Sun et al., 2022a; Xia et al., 2022) that first learn some general knowledge
122   for the graph model with easily accessible data, and then fine-tune the model on new tasks. This is
123   often referred to as ”pre-train & fine-tune” paradigm. However, the large diversity between graph
124   tasks with node level, edge level, and graph level may cause a ”negative transfer” results where the
125   knowledge learned during the pre-training phase hurts performance when fine-tuning on a specific
126   downstream task, rather than improving it (Sun et al., 2023b). As a solution, graph prompt learning
127   was proposed. The goal of graph prompt learning is to learn transformation operations for graphs to
128   reformulate the downstream task to the pre-training task. It can be formulated as follows:

      For instance, Graph Pre-training and Prompt Tuning (GPPT) (Sun et al., 2022a) applies prompt-
141
      based tuning methods to models pre-trained by edge prediction. It introduces virtual class-prototype
142
      nodes/graphs with learnable links into the original graph, making the adaptation process more akin to
143   edge prediction. Fang et al. (2024) proposed a universal prompt-based tuning method, called Graph

145   learnable vector to all node features in the graph while its variant GPF-plus incorporates different
146   prompted features for different nodes in the graph. Sun et al. (2023b) proposed All-in-one, a graph
147   prompt that unifies the prompt format in the language area and graph area with the prompt token,
148   token structure, and inserting pattern. They reformulate the downstream problems to the graph-level
149   task to further narrow the gap between various graph tasks and pre-training strategies. Graph prompt
150   learning has superior performance compared to traditional fine-tuning methods and is especially
151
      effective in few-shot settings, i.e., when only a small number of data points are sampled to tune the
      prompt. While graph prompt learning benefits various graph applications, in this work, we focus on
152
      node classification tasks and three state-of-the-art graph prompt learning methods, namely GPPT,
153
      All-in-one, and GPF-plus.

      2.3   P RIVACY R ISKS IN GNN S AND G RAPH P ROMPT L EARNING
156
157   GNNs have been shown to be vulnerable to various privacy risks, such as membership inference
158   attacks (MIAs) (Olatunji et al., 2021; He et al., 2021; Conti et al., 2022), model inversion attacks
159   (Zhang et al., 2022a), and property inference attacks (Wang & Wang, 2022; Zhang et al., 2022b).
160   Specifically, MIAs against GNNs aim to infer whether a given node or graph was used to train the
161   GNN model, model inversion attacks aim to recover the model’s training data from the model’s
      output, and property inference attacks aim to infer the sensitive properties of the training data through


                                                         3
      Under review as a conference paper at ICLR 2025


162
      the access to the target GNN model. Regarding graph prompt learning, some prior work explores
163   backdoor attacks in graph prompt learning, which utilize prompts to insert backdoor triggers into the
164   GNN model (Lyu et al., 2024) to impact output integrity. To the best of our knowledge, there is no
165   prior work on assessing and mitigating the privacy risks in graph prompt learning.

      2.4   D IFFERENTIAL P RIVACY
168
169   Differential privacy (DP) (Dwork, 2006) is a mathematical framework that provides privacy
170   guarantees for randomized mechanisms M : I → S. Therefore, it upper-bounds the probability
171   that M, when executed on two neighboring datasets D, D′ , i.e., dataset that differ in only one data
172   point, output a different result by formalizing that Pr[M(D) ∈ S] ≤ eϵ · Pr[M(D′ ) ∈ S] + δ. The
173   privacy parameter ε specifies by how much the output is allowed to differ, and δ is the probability
174   of failure to meet that guarantee. There are two main algorithms to implement DP guarantees for
175
      traditional machine learning. The differentially private stochastic gradient descent algorithm
      (DP-SGD) (Abadi et al., 2016) extends standard stochastic gradient descent with two additional
176
      operations, first, gradient clipping that limits the impact of each individual training data point (often
177
      called ”sensitivity”) on the model update, and then the addition of calibrated amounts of stochastic
178   noise to provide formal privacy guarantees. The second private aggregation of teacher ensembles
179   algorithm (PATE) (Papernot et al., 2017; 2018) trains an ensemble of teacher models on disjoint
180   subsets of the private data. Then, through a noisy labeling process, the ensemble privately transfers
181   its knowledge to an unlabeled public dataset. Finally, a separate student model is trained on this
182   labeled public dataset for release.
183
184   DP for Graphs. As the classical DP guarantee makes no assumptions about potential correlations
185   between data points, there are existing works that extend DP on graph data (Mueller et al., 2024;
186   Sajadmanesh et al., 2023; Kasiviswanathan et al., 2013; Olatunji et al., 2023; Sajadmanesh & Gatica-
187   Perez, 2024; Xiang et al., 2024). There are three variants of DP on graph data: node-level DP,
188   edge-level DP, and graph-level DP, depending on what the data owner requires to protect. Specifically,
189   node-level DP aims to protect the privacy of individual nodes in the graph data, including its attributes
190   and associated edges (Sajadmanesh et al., 2023; Kasiviswanathan et al., 2013; Daigavane et al., 2021;
191   Olatunji et al., 2023). Edge-level DP aims to protect the relationships between nodes, which can be
192
      applied to social network graphs (Hay et al., 2009) or location graphs (Xie et al., 2016), where the
      edges contain sensitive information, but the data represented in the nodes of the graph are assumed
193
      to be non-sensitive. Graph-level DP aims to protect the entire graph data, including the structure of
194
      the graph, node attributes, and edge relationships (Mueller et al., 2022). However, graph-level DP
195   has not been thoroughly investigated in the literature (Mueller et al., 2024). In this work, we focus
196   on node-level DP as we aim to protect the privacy of individual nodes in the graph data. Different
197   from the existing node-level DP guarantees (Sajadmanesh et al., 2023; Kasiviswanathan et al., 2013;
198   Olatunji et al., 2023; Sajadmanesh & Gatica-Perez, 2024; Xiang et al., 2024), which often results in
199   large ϵ values, limiting their practical utility, we aim to achieve meaningful privacy guarantees for
200   graph prompt learning with small and manageable ϵ values (ϵ <= 2).
201
202   2.5   P RIVATE P ROMPT L EARNING IN THE V ISION AND L ANGUAGE D OMAIN
203
204   In the vision domain, Li et al. (2023) leverage the PATE algorithm for private prompt tuning to vision
205   encoders. Therefore, they have to tune a prompt and train an additional label mapping for each
206   teacher. In contrast, our method instantiates different teachers only through graph input prompts.
207
      In the language domain, multiple approaches have been proposed to privatize prompts. Chen et al.
      (2023) rely on named entity recognition to identify and hide private information in text prompts.
208
      This approach is not easily transferable to the graph domain and additionally does not yield formal
209
      privacy guarantees. The DP-OPT (Hong et al., 2024) framework relies on a local large language
210   model (LLM) to derive a discrete, i.e., text, prompt with DP, and then transfers this prompt to a
211   central LLM. The framework is tightly coupled to the language domain and derives plain language
212   prompt templates that are not applicable to GNNs. Panda et al. (2023) rely on a PATE-style teacher
213   ensemble implemented through different prompts, and generate noisy output predictions for the LLM.
214   Yet, due to the absence of a student model in their framework, each query to the ensemble consumes
215   additional privacy budget, making the approach impractical. Duan et al. (2023a) solve this limitation
      by generating a student prompt from the teacher ensemble, similar to our work. Yet, they treat all

      teachers in the ensemble equally, which can yield sub-optimal privacy-utility trade-offs. In contrast,
217   we leverage the inherent structure of the graph data to identify more important data points and weight
218   their teachers’ votes higher, improving privacy-utility trade-offs.

        The standard approach for privacy-preserving machine learning is based on the DP-SGD algo-
296
        rithm (Abadi et al., 2016). The DP-SGD algorithm can be applied in gradient-based learning
297     approaches to limit the impact of individual training data points on the final model and add calibrated
298     noise to implement the privacy guarantees. We explore this naive way of implementing privacy
299     guarantees into graph prompt learning and show that it fails to yield reasonable utility even at low
300     privacy regimes, i.e., with very high ε’s. Motivated by this insight, we propose two non-gradient
301     based algorithms for private graph prompt learning based on the PATE framework.
302
303      4.1                        NAIVE I MPLEMENTATIONS OF P RIVACY IN G RAPH P ROMPT L EARNING FAIL
304
305     As a naive solution to yield private graph prompt learning, we rely on the DP-SGD algorithm.
306     Therefore, we keep the GNN frozen, calculate the gradients only with respect to the graph prompts,
307
        clip and noise them according to the desired privacy protection, and update the prompt iteratively
        to minimize the loss on the downstream task. Our evaluation of this naive approach in Table 6 in
308
        Appendix A.4.3 highlights that DP-SGD yields inadequate privacy-utility trade-offs for private graph
309
        prompt learning. While our results show the general trend that with increasing privacy budgets, the
310     performance of the downstream task increases, DP-SGD still significantly degrades the downstream
311     task performance even at high privacy budgets. For instance, with a privacy budget as high as ε = 64
312     in the 5-shot setting, the accuracy of the downstream task on the Cora dataset still drops from 48.70%
313     to 18.47%, which is close to random guessing.
314
315      4.2                        T WO D IFFERENTIALLY P RIVATE G RAPH P ROMPT L EARNING F RAMEWORKS
316
317      Motivated by the failure of the naive DP-SGD approach, we propose two non-gradient based DP graph
318      prompt learning frameworks, DP-GPL and its variant DP-GPL+W. We detail the general workflow
319      of DP-GPL+W in Figure 1. Note that DP-GPL follows a similar structure but differs in the private
320
         data partitioning and aggregation blocks as detailed below.
321     Following PATE (Papernot et al., 2017; 2018), our algorithms contain the broader stages of training
322     the teacher models, performing a private knowledge transfer, and obtaining the student. In contrast to
323     standard PATE, we do not train teachers from scratch, but using the same frozen pre-trained GNN,
        we tune teacher prompts. Additionally, our student is again not a trained model like in PATE, but a


                                                                                                              6
      Under review as a conference paper at ICLR 2025


324
      prompt tuned on the public data labeled during the knowledge transfer. As an additional difference,

      Datasets. We use ogbn-arxiv (Hu et al., 2020), which is a large-scale graph dataset, as the pre-training
437
      dataset. For the downstream tasks, we use Cora (Yang et al., 2016), CiteSeer (Yang et al., 2016), and
438
      PubMed (Yang et al., 2016). Since the pre-trained dataset (i.e., ogbn-arxiv) and downstream dataset
439   (i.e., Cora, CiteSeer, and PubMed) have various input feature dimensions, we here use SVD (Singular
440   Value Decomposition) to unify input features from all dimensions as 100 dimensions, following the
441   process in Sun et al. (2023b). We provide more details about these datasets in Appendix A.1. For
442   each dataset, We randomly select 50% of the nodes as the private data and the remaining 50% as
443   the public data. Within the public data, we randomly select 50 nodes as the query nodes and the
444   remaining nodes as the testing data.
445   Models. We use three widely-used GNN models, i.e., GCN (Kipf & Welling, 2022), GAT (Veličković
446   et al., 2018a), and Graph Transformer (GT) (Shi et al., 2020) as the backbone for both ”pre-train &
447   fine-tune” and graph prompt learning paradigms. The default hyperparameters used for pre-training
448   GNN models are presented in Table 3. For pre-training strategies, we select four mostly used methods
449   covering node-level, edge-level, and graph-level strategies, i.e., DGI (Veličković et al., 2018b),
450   GraphMAE (Hou et al., 2022), EdgePreGPPT (Sun et al., 2022a), and SimGRACE (Xia et al., 2022).
451   Graph Prompt Learning Methods. Current popular graph prompt learning methods can be classified
452   into two types, ’Prompt as graph’ and ’Prompt as token’ (Zi et al., 2024). For ’Prompt as graph’ type,
453   we select All-in-one (Sun et al., 2023b), and for ’Prompt as token’ type, we use GPPT (Sun et al.,
454   2022a), and GPF-plus (Fang et al., 2024). These graph prompt methods are all state-of-the-art. Also,
455   we focus on the 5-shot graph prompt learning setting as it has high performance on downstream tasks
456   (as shown in Table 5 in Appendix A.4.1) and also high MIA risk (as shown in Figure 3).
457   Privacy Parameters and Accounting. We set the privacy parameters for DP-GPL according to
458   Table 4 in Appendix A.2. Note that, since we scale the weights in DP-GPL+W to match the number of
459   teachers (i.e., the original sum of votes, see lines 6-8 in Algorithm 2), we can use the same parameters
460   over both methods. To empirically account for the per-teacher privacy loss during our experiments,
461   we build on the code-based from Boenisch et al. (2023).
462
      DP-GPL and DP-GPL+W. We use an ensemble of 200 teacher prompts, and each teacher prompt is
463   trained with disjoint 5 shots of data from the private downstream task. For query dataset, we select
464   50 public samples from the downstream distribution. Both methods are implemented to immediately
465   stop querying once a teacher has reached their privacy limit, which we set to ε = 2. We repeat each
466   experiment three times and report the average and standard deviation of the public student prompt’s
467   accuracy on the testing dataset.
468
      Baselines. We compare against three baselines. (1) Lower Bound (LB): (ε = 0). Given a pre-
469   trained GNN model, we directly evaluate its performance on the downstream test data. (2) Ensemble
470   Accuracy (Ens. Acc.):(ε = ∞). We use the histogram of the private teacher ensemble votes and
471   return the clean argmax. (3) Upper Bound (UB): (ε = ∞). i.e., we select the teacher prompt which
472   has the best testing accuracy.

515   6   C ONCLUSIONS
516
517   In this work, we are the first to highlight the privacy risks that arise from graph prompt learning.
518   By running a membership inference attack, we showed that private information from the private
519   dataset used to tune the graph prompts can leak to external parties who query the prompted GNN.
520   To mitigate the resulting risk for the downstream data, we set out to design a private graph prompt
521   learning algorithm. Motivated by our finding that the naive application of the DP-SGD algorithm,
522   the standard to implement DP guarantees in machine learning, fails to yield good privacy-utility
523   trade-offs, we designed DP-GPL and DP-GPL+W, which build on the PATE algorithm and perform a
524   noisy knowledge transfer from teachers to a student prompt. Leveraging the natural structure of graph
525
      data, in contrast to standard PATE, in DP-GPL+W, we do not weight every teacher’s vote equally
      during the knowledge transfer. Instead, we weight teachers who hold nodes with higher centrality
526
      (more influential nodes) higher. We thoroughly analyzed the resulting utility and privacy implications
527
      and highlighted that our DP-GPL and DP-GPL+W are able to yield strong utility at high privacy
528   guarantees. Thereby, our work contributes towards leveraging the computational and utility benefits
529   from graph prompt learning but without additional privacy risks for the downstream data.
