009                                              A BSTRACT
010
011
               LoRA has become the prevailing technique for finetuning large neural networks
012            with limited computational resources. Historically, activations have been regarded
013            as small and computationally inexpensive to manipulate—a view reflected by
014            LoRA, which leverages this assumption and adds a low-rank term to intermediate
015            activations. However, in the era of modern large language models (LLMs) and dif-
016            fusion models, this notion has been challenged by the desire for increasing context
017            lengths and smaller models, a trend which inevitably leads activations to consume
018            more memory than the model weights themselves. Surprisingly, when finetuning
019            a 1B model with a context length greater than 2048, we find that LoRA finetun-
020
               ing uses more memory than full-parameter finetuning. This study demonstrates
               that manipulating additional model weight representations within the computation
021
               graph in parameter-efficient finetuning techniques can often be more memory-
022
               efficient than operating on the activations. We provide a semantically-equivalent
023            computation graph reformulation for LoRA, and other popular PeFT techniques,
024            which saves memory and trains faster, advancing the Pareto-frontier for finetun-
025            ing tasks that can be achieved on consumer hardware. Under practical conditions,
026            this reformulation provides up to a 1.4x reduction in max memory usage and
027            latency for LoRA finetuning across various language and diffusion transformers
028            without affecting the predictive performance of the technique.

031   1   I NTRODUCTION
032
033
      The release of the transformer architecture (Vaswani et al., 2017) inspired the widespread scaling
034
      of language models with billions or trillions of parameters. This unprecedented scale ushered in a
035
      fundamental paradigm shift in language modeling: it was no longer necessary to train models from
036   scratch when generalist pre-trained models could be finetuned for higher performance on down-
037   stream tasks. It is no coincidence that as these models continued to scale, the field has taken a
038   significant effort to understand how to finetune them efficiently, as doing so otherwise often meant
039   consuming hundreds of gigabytes of GPU memory.
040
      The existence of pretrained LLMs led to the creation of LoRA (Hu et al., 2021), a finetuning tech-
041
      nique that focuses not on optimizing the parameters of the target model but instead on a small set
042   of new parameters that can be merged into the original model without extra cost during inference.
043   LoRA has now become a bedrock technique for efficiently finetuning LLMs. It introduces a pair of
044   low-rank matrices that are forward-passed in parallel to a given model layer; these activations are
045   summed and passed forward to the rest of the network. This approach can offer a significant mem-
046   ory saving over full-parameter finetuning, in which all original network parameters are optimized, as
047   well as regularizes the model to forget less pre-training information (Biderman et al., 2024). LoRA
048   achieves memory savings by eliminating the need to store the optimizer’s state, including the first-
049   order and second-order moments required by optimizers like ADAM. LoRA achieves 3x memory
050
      saving when fine-tuned on GPT-3 with 175B parameters (Hu et al., 2021).
051   The field of language modeling has seen substantial growth in recent years. New architectures,
052   larger corpuses of multimodal data, and increased FLOP scaling have led to smarter, more capable
053   models. Irrespective of these trends, the use of LoRA has largely persisted, and it continues to be a
      popular technique for finetuning the latest available models.

111   Will LoRA and other activation-based approaches still be usable at the current trend rate? In this

124
      designing future parameter-efficient finetuning techniques while practically enabling broader access
      to LLM finetuning.

129   2.1   P RELIMINARIES
130
131   Low-rank adaptation (LoRA). LoRA (Hu et al., 2021) inserts two trainable low-rank matrices
132   A ∈ Rm×r and B ∈ Rr×n for every targeted weight W ∈ Rm×n in a pretrained model. During
133
      a forward pass of the model, they are used to construct an additional projection that is added to the
      hidden state in the form:
134
                                               y = W x + BAx
135
      where BAx term would be scaled by a constant factor c before added to W x. During finetuning,
136
      only the parameters A and B are updated, while all weights from the pretrained model are frozen.
137
      During inference, A and B’s product is merged back to W , and thus does not increase the inference
138   latency.
139
140   (IA)3 . (IA)3 (Liu et al., 2022) is a PEFT method that learns rescaling factors for activations in the
141   network. Given a frozen weight W ∈ Rm×n , bias b ∈ Rm×n , input x ∈ Rn×1 , (IA)3 would insert
142   a trainable vector l ∈ Rm×1 to directly rescale the activations:
143                                            y = (W x + b) ⊙ l
144   where ⊙ means the hadamard product which performs column-wise scaling.
145
146
      2.2   ACTIVATIONS
147
148   Activations are the tensors passed forward between layers in a neural network. They are retained
149   during training for backpropagation. In a transformer-based (Vaswani et al., 2017) model, an inter-
150   mediate activation is typically a three-dimensional tensor of shape (B, S, D), where B is batch size,
151   S is sequence length, and D is hidden size.
152                                    Size of Activation = B × S × D                                    (1)
153
      D would remain fixed for the same pretrained model. In contrast, batch size and sequence length
154
      could differ for different training runs.
155
156   Batch size. Larger batch size produces comparatively more reliable gradient direction, providing a
157   more accurate approximation for the whole-dataset gradient (Liu et al., 2023). The linear-scaling
158
      rule (Goyal et al., 2017) between batch size and learning rate also suggests that larger batch size
      allows for larger steps in training and stabilizes training. In the context of LLM training and fine-
159
      tuning, a common practice is to use the maximum batch size that GPU memory allows.
160
161   Sequence length. Pretrained LLM’s context length grows over time. For example, LLaMA (Tou-
      vron et al., 2023a) is pretrained using a context length of 2048, and LLaMA2 (Touvron et al., 2023b)

      4096. LLaMA3 series (Dubey et al., 2024) progressively increases the context length from 8K to
163   128K during pretraining. With pretrained models supporting longer context length, long context
164   finetuning is playing an increased role in adapting pretrained LLMs; for example with datasets
165   related to instruction finetuning and chain-of-thought. Furthermore, finetuning models are not
166   bounded by their pretrained context lengths. Works have been done to adapt a pretrained LLM
167   to a context length longer than that of the pretraining stage (Jin et al., 2024).
168
      Transformer-based diffusion models also benefit from longer sequence lengths. DiT (Peebles & Xie,
169   2023) demonstrated that reducing the patch size, thereby increasing the sequence length of patches,
170   leads to a significant improvement in image generation quality.
171
172   Growth of activations. With both batch size B and sequence length S being pushed up by the
173   current trend, from formula 1 it is immediately clear that the size of activations would continue to
174
      grow over time.

176   2.3   ACTIVATIONS V. S . W EIGHTS
177
      Activation-to-weight ratio. Unlike activations, the dimension D of a model’s weights remains fixed
178
      across different finetuning runs. This static nature of weights contrasts with the dynamic growth of
179
      activations, which scale with batch size B and sequence length S.
180
181   Consider the size of the intermediate activations, which can be represented as B × S × D. For
182   simplicity, suppose this corresponds to a linear layer of size D × D producing these activations.
183
      Then the ratio between the size of the activations and the size of its corresponding weights can be
      expressed as:
184
                                              B×S×D            B×S
185                                                         =                                            (2)
                                                D×D              D
186
      As illustrated in the previous section, batch size B and sequence length S would continue to be
187   scaled up in the forseeable future. Thus for any existing model, future finetuning runs would possibly
188   have a larger activation-to-weight ratio.
189
190   Smaller models. Smaller models have been designed to handle increasingly longer context lengths.
191   For example, models like LLaMA-3.2-1B are capable of handling sequence lengths of 128K tokens,
192
      far longer than its larger predecessors. This trend demonstrates that improvements in model archi-
      tecture and training techniques have allowed for smaller models (and thus smaller D) to manage
193
      longer contexts, further driving up the activation-to-weight ratio in formula 2.

      2.4   O PERATING ON WEIGHTS INSTEAD OF ACTIVATIONS
196
197
      Activation-based paradigm. Currently, PyTorch is based on activations. Weights are implemented
198   as functions that operates on activations. The implementation paradigm of a neural network in
199   PyTorch is to iteratively apply different operations to the intermediate activation x.
200
201   Activation-based PEFT implementation. The current PEFT implementation follows the
202   activation-based paradigm during training. For example, in the standard LoRA implementation,
203   LoRA operates at the level of the activations as it injects a low-rank update into the activation flow
204
      rather than directly manipulating the weight matrices. This LoRA implementation falls into the
      category where a reformulation to weights is possible, which will be illustrated in the next section.
205
206   Weight-based operations. During the forward pass during training, if an operation on activations
207   could be reformulated to an operation on its corresponding weights, then the intermediate represen-
208   tation that needs to be stored shifts from an activation to the weight. The size of the intermediate
209   representation would decrease when activations are larger than weights.

      6   R ELATED W ORKS
487
488
      PEFT + Compression. Methods such as QLoRA Dettmers et al. (2024) and LoftQ (Li et al., 2024)
489   reduce model size in memory through low-bit quantization of model parameters, while reducing
490   information with LoRA. Similarly, Wanda (Sun et al., 2024) uses pruning to reduce the number
491   of active weights in the model and uses LoRA to regain lost model performance. While these
492   techniques reduce memory footprints via model compression, our approach further complements
493   these by decreasing the memory consumption during post-compression training. Our contribution
494   furthers the memory efficiency of finetuning with compression without sacrificing performance.
495   Finetuning Frameworks. Finetuning frameworks like DeepSpeed (Rasley et al., 2020) and ZeRO
496   (Rajbhandari et al., 2020) focus on infrastructure-level optimizations to improve parallelism and
497   latency efficiency by sharding optimizer states across accelerators. Our method is designed to inte-
498   grate with these frameworks. These frameworks take a fixed computation graph and optimize the
499   training system over this graph. Our technique aims to build on top of these methods by providing a
500   more efficient computation graph to the frameworks.
501   Compilers. Other methods have approached efficient training through compilation. PyTorch2
502   (Ansel et al., 2024) introduced TorchDynamo, a just-in-time compiler for computation graphs.
503   While these works focus on speeding up training time, our work complements it by addressing
504   memory consumption. As a byproduct of our work, we also improve on latency, as shown in 4.3.

      7   L IMITATIONS AND C ONCLUSION
507
508
      We acknowledge that our work is not without its limitations. It is important to recognize that the
509
      benefits of our reformulation can diminish, particularly in situations where the batch size and se-
510   quence length is small or when the hidden dimension size is large. We believe the field is moving
511   away from this paradigm as smaller models become more capable and contexts become longer. Ad-
512   ditionally, although not being actively used, the current dropout implementation in LoRA is only
513   applied to activations that pass through the LoRA branch, while not being applied to the input to the
514   frozen weight. Therefore, the activation-based LoRA dropout implementation would make weight-
515   based reformulation not exactly equivalent to the LoRA when using dropout. Nonetheless, Lin et al.
516   (2024) proposes to apply dropout to LoRA weights, which is compatible with our weight-based
517   reformulation.
518   Parameter-efficient finetuning techniques have prevailed for their ability to reduce memory usage
519   during finetuning as well as overfitting (Biderman et al., 2024). In this study, we offer an equivalent
520   reformulation for these these techniques that further reduces their memory usage and latency. Our
521   main insight is the trend that capable, state-of-the-art models are getting smaller while their con-
522   text lengths are getting larger. This insight means that activations are trending towards becoming
523   larger than the size of the weights of the model. We leverage this information to have these PEFT
524
      techniques not produce activations, but rather a delta to the weights they are being applied to. Our
      experiments demonstrate that this leads to memory and latency savings as sequence lengths and
525
      batch sizes increase. We hope that that this research provides greater accessibility to finetuning and
526
      inspires future research in optimizing PEFT methods.

      A.2   SINGLE - LAYER ANALYSIS
649
650   Settings. We study the memory usage when applying LoRA finetuning on a linear layer with
651   W ∈ R2048×2048 , which is of the same dimension as the query projection layer Wq in LLaMA-
652   3.2-1B. We also used a LoRA rank of 8, batch size of 32, and sequence length of 1024. This setup
653   allowed us to isolate the memory behaviors of the two LoRA variants while running on a CPU to
654   avoid GPU-specific optimizations. We use the PyTorch profiler to track peak memory usage during
655   forward and backward passes, and the script can be found in our anonymous repo in a file called
656   single layer analysis.py.
657
      Memory Profile. In the weight-based formulation, the extra operations primarily involve matrix
658   multiplication BA and addition W + BA, which each consume 16 Mb. Additionally, the linear
659   transformation (W + AB)x takes 64 Mb of memory. In contrast, the activation-based formulation
660   introduces additional activations of much larger size, including W x of size 64 mb, and B(Ax) and
661   the subsequent addition W x + B(Ax), each of size 128 Mb.
662
      There is a significant difference in peak memory consumption between the two formulations: the
663
      weight-based approach reaches a maximum of 160.125 Mb, while the activation-based method peaks
664   at 256.63 Mb of CPU memory. This 60% increase in memory usage for the activation-based ap-
665   proach can be primarily attributed to the storage requirements for intermediate activations such as
666   Wx and B(Ax).
667
      It is worth noting, however, that the practical implementation of these approaches may be influenced
668
      by CUDA optimizations and PyTorch optimizations, which can introduce additional nuances to the
669
      memory usage patterns. Nonetheless, this simplified single-layer analysis motivates the potential
670   benefits of the weight-based LoRA formulation.
