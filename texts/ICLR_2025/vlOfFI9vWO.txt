011                                                A BSTRACT
012
013            Vision Transformers (ViT) have revolutionized the field of computer vision by
014            leveraging self-attention mechanisms to process images. However, the computa-
015            tional cost of ViT increases quadratically with the number of tokens. Dynamic
016            token selection methods which aims to reduce computational cost by discard re-
017            dundant tokens during inference, are primarily based on non-differentiable binary
018
               decisions methods and relaxations methods. However, Reinforcement Learning(
               (RL) based methods, which have astonishing decision-making ability, is consid-
019
               ered to have high variance and high bias, not adopted for dynamic token selection
020
               task in previous work. Yet, RL-based methods have been successfully applied to
021            many binary decision problems such as neural pruning, routing, path selection. In
022            this paper, we propose Reinforcement Learning for Dynamic Vision Transformer
023            (RL4DViT), a novel framework for the dynamic token selection task in ViT using
024            RL. By harnessing the powerfull decision-making capabilities of Multi-Agent Re-
025            inforcement Learning(MARL) algorithms, our method dynamically prunes redun-
026            dant tokens based on input complexity, significantly reducing the computational
027            cost while maintaining high accuracy. Extensive experiments on the ImageNet
028            dataset indicate that our approach reduces the computational cost by up to 39%,
029
               with only a 0.17% decrease in accuracy. To the best of our knowledge, this is
               the first RL-based token selection method for efficient ViT. All our source code is
030
               publicly available at [link].

034   1   I NTRODUCTION
035
036   Vision Transformers (ViT) have achieved state-of-the-art performance in many computer vision tasks
037   while negating the need for convolution operations. However, they utilize a fixed token sequence
038   independent of the input during both training and inference. This fail to take into account the fact that
039
      the complexity of a given computer vision task often varies depending on the input. For instance,
      distinguishing a flower from other categories is generally easier than differentiating between two
040
      similar species of flowers. Even within a single image, tokens containing detailed object features
041
      are far more informative than those from the background. Previous work Chefer et al. (2021) has
042   shown that the final prediction of ViT is often based on a subset of the most informative tokens,
043   suggesting that many tokens can be removed without significantly affecting performance. Motivated
044   by this insight, we propose a framework that dynamically prunes redundant tokens based on input
045   complexity, with minimal impact on task performance. By discarding redundant tokens in simpler
046   cases and retaining more tokens for more complex ones, our approach significantly reduces the
047   computational cost of ViT.
048   Several works have explored dynamic token selection in the ViT. For instance, DynamicViT Rao
049   et al. (2021) introduces decision layers after each transformer block, trained with a loss function
050   that considers both accuracy and computational cost. A-ViT Yin et al. (2021) employs a halting
051   mechanism to discard less informative tokens without additional parameters. Moreover, A-ViT
052   Wang et al. (2021) adds decision layers that determine not only which tokens to discard but also
053   which attention heads and Transformer blocks to activate. Since dynamic inference in ViT is often
      based on binary decisions, which are non-differentiable, these works employ the Gumbel-Softmax

      In this paper, we introduce Reinforcement Learning for Dynamic ViT (RL4DViT), an RL-based dy-
      namic token selection framework for the ViT. The overall structure of this framework is illustrated
091
      in Figure 1. RL4DViT consists of two parts: The ViT for inference and the MAPPO for token selec-
092
      tion. When the input tokens in the ViT reach a specified block, we employ the MAPPO algorithm to
093   determine whether a subset of these tokens should be discarded. The decision is then communicated
094   back to the ViT. Based on this feedback, masked attention is applied as the tokens traverse the block,
095   cutting off the connections between the tokens discarded by MAPPO and the remaining tokens. To
096   the best of our knowledge, this is the first work to incorporate reinforcement learning for efficient
097   Vision Transformer architectures with dynamic token selection.

120   2.1   DYNAMIC T OKEN S ELECTION IN V ISION T RANSFORMER
121
122   Inference speed is crucial for deploying deep models on edge devices. Numerous techniques can
123   accelerate the inference speed of deep models, including quantization Osmanović (2021); Jacob
124   et al. (2018), pruning Frankle & Carbin (2018), and knowledge distillation Gou et al. (2021); Pham
125   et al. (2022). Several works also focus on accelerating the inference speed of transformer models.
126
      For instance, TinyBERT Jiao et al. (2019) introduces a distillation method to speed up transformer
      inference, while Swin-Transformer Liu et al. (2021) reduces the quadratic space and time complexity
127
      to linear by replacing the fully connected structure with a star-shaped topology. However, most of
128
      these works focus on natural language processing (NLP) tasks.
129
130   Our method, on the other hand, dynamically prunes less important tokens by exploiting the sparsity
131   of informative image patches in Vision Transformers. The key to implementing dynamic inference in
132
      Vision Transformers is to prune tokens with less information as much as possible while maintaining
      model performance. Serveral techniques can be employed for this task, such as the Gumbel-Softmax
133
      Jang et al. (2016) trick, halting mechanisms Yin et al. (2021), and RL-based methods. For example,
134
      DynamicViT addresses this issue by adding decision layers after each transformer block, training
135   these layers using a loss function designed to balance both accuracy and computational cost Rao
136   et al. (2021). Similarly, A-ViT Wang et al. (2021) implements a halting mechanism to discard less
137   informative tokens without adding extra parameters, modifying the Vision Transformer architecture
138   in the process. Moreover, AdaViT introduces decision layers to not only determine which tokens to
139   discard but also which attention heads and transformer blocks to activate. Unlike Gamble-Softmax
140   trick and halting mechanisms, RL-based methods has not yet been applied to this task, even though
141   it has strong decision-making capabilities.

      3     T HE P ROPOSED F RAMEWORK
144
145
      We propose RL4DViT, a MARL-based adaptive computation framework designed to reduce the
146   computational cost of ViT with minimal impact on performance. Given an input image, RL4DViT
147   learns to adaptively derive policies for selecting which tokens to discard or retain in the Vision
148   Transformer backbone, conditioned on the input image. An overview of our method is illustrated
149   in Figure 1. In this section, we present how to apply MARL techniques, such as MAPPO, to the
150   dynamic token selection task in Section 3.1. Then, we elaborate on reward engineering and attention
151   masks in Sections 3.2 and 3.3.

      3.3   T OKEN S ELECTION WITH ATTENTION M ASK
271
272   To execute action A in the Vision Transformer, we need to mask out the tokens that have been dis-
273   carded. We can simply prune the tokens that the MARL algorithm has decided to discard. However,
274   the action A is usually unstructured, and the actions for different samples are not the same. There-
275   fore, for samples within a batch, simply pruning tokens where At = 0 would result in a non-uniform
276   number of tokens, making it difficult to parallelize the computation. To maintain a consistent num-
277   ber of tokens across samples within a batch, we opt to zero out the values of the discarded tokens,
278
      which is akin to padding operations in NLP tasks. However, merely zeroing out those discarded to-
      kens is insufficient, as these zeroed tokens can still affect other tokens through the softmax function
279
      in the calculation of the self-attention matrix Vaswani et al. (2017):
280
281
282
                                                           QK T
                                           Attn = Sof tmax( √ )
283                                                          dk
284   One way to address this problem is to set the corresponding elements in the attention matrix to
285   a negative value, such as -1000, after calculating the scores and before applying softmax. This
286   approach ensures that the corresponding values after passing through softmax are almost 0 Yin
287   et al. (2021). Alternatively, after applying softmax, we can set the corresponding elements in the
288   attention matrix to 0 Rao et al. (2021). By doing so, we can effectively cuts the connection between
289   the discarded tokens and the other tokens.
290   The masking strategy described above keeps the computational cost of our training iterations similar
291   to that of the original Vision Transformer’s training cost. During inference, we simply prune the
292   discarded tokens from computation to measure the actual computational efficiency gained by our
293   reinforcement learning algorithms.

      4     E XPERIMENT
296
297
      In this section, we will demonstrate the superiority of reinforcement learning-based dynamic token
298   selection through extensive experiments. In all of our experiments, we fix the number of transformer
299   blocks that adopt token selection to L = 3. Furthermore, inspired by previous work, the token
300   selection is performed hierarchically in three stages: at block 3, 6, and 9. We train the MARL agents
301   with ViT weights fixed for one epoch and fine-tune the ViT with MARL agents’ weights fixed for
302   another epoch. All the experiments are carried out on a single NVIDIA RTX 3090 GPU. More
303   implement details can be found in the supplementary material.
304   RL4DViT can be trained very fast. While previous work had to train the entire network from scratch
305   or train the selection network for many epochs, RL4DViT can achieve comparable performance
306   within one epoch. Another advantage of RL4DViT is that the total number of parameters for token
307   selection is much smaller. In DynamicViT, it was necessary to introduce MLP into three decision
308   blocks. In A-ViT, it was necessary to implement an h-gate in each transformer block, while in
309   RL4DViT, the network for token selection consists solely of the Actor (which is one MLP) with a
310   total parameter count of 0.1M, reducing the number of parameters utilized for decision-making by
311   a factor of 9 compared to DynamicViT.

      In this paper, we proposed the dynamic token selection framework RL4DViT, which significantly
489   enhances the efficiency of ViT in computer vision tasks. By modeling the token selection task as a

572   A.1   PARAMETER S HARING
573
574   Agents in MAPPO are homogeneous, meaning they possess identical observation and action spaces.
575   We utilize parameter sharing to reduce the number of parameters in our RL4DViT framework. Ad-
576
      ditionally, previous works have demonstrated that this approach improves learning efficiency Chris-
      tianos et al. (2021) Terry et al. (2020). In our implementation of MAPPO for dynamic Vision
577
      Transformer, the agents share both the policy and value function parameters.
578
579

      A.2   F EATURE C ONSTRUTION
581   As mentioned earlier, MAPPO operates within the centralized training with decentralized execution
582   (CTDE) framework, enabling individual PPO agents to communicate through a global value func-
583                                                                                             (t)
      tion. The input to the value function typically consists of the global state information si . This
584   global state transforms a partially observable Markov decision process (POMDP) into a Markov
585                                                                           (t)
      decision process (MDP). Proper design of the global state information si is crucial.
586
587   One approach is to concatenate the local observation information of all agents as the input for the
588   critic. While this can be effective in most cases, in our scenario, where the number of agents is
589   significant (in our case, t = 196), this may lead to an excessively high input dimension for the value
590
      function compared to the policy function, complicating the learning of value functions, as noted in
      Lowe et al. (2017).
591
592   An alternative is to use an Environment Provided global state (EP), which contains general global
593   information about the environment state. However, the EP state typically encompasses information
      common to all agents and may overlook critical local agent-specific details Foerster et al. (2018). A


                                                       11
      Under review as a conference paper at ICLR 2025


594
      third approach is to construct an Agent-Specific Global State (AS), which generates a global state
595   for agent t by concatenating the EP state with the local observation of agent t, as discussed in the
596   original MAPPO paper Yu et al. (2022).
597
598
      In our case, we construct the Agent-Specific Global State by utilizing the class token in the Vision
      Transformer as the Environment Provided global state (EP). By concatenating tokencls and tokent ,
599
      we create the global state for agent t. Thus, the Agent-Specific Global State for agent t is given by:
