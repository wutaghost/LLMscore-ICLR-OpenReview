011                                                A BSTRACT
012
013            We propose a novel in-context learning algorithm for building autonomous
014            decision-making language agents. The language agent continuously attempts to
015            solve the same task by reasoning, acting, observing and then self-correcting each
016
               time the task fails. Our selected language agent demonstrates the ability to solve
               tasks in a text-based game environment. Our results show that the gemma-2-9b-it
017
               language model, using our proposed method, can successfully complete two of
018
               six tasks that failed in the first attempt. This highlights the effectiveness of our ap-
019            proach in enhancing the problem-solving capabilities of a single language model
020            through self-correction, paving the way for more advanced autonomous agents.
021            The code is publicly available at https://anonymous.4open.science/
022            r/AutonomousLLMAgentwithAdaptingPlanning-D613/.

025   1   I NTRODUCTION
026
027   Large language models (LLMs) are large statistical models that predict the next word, phrase, sen-
028   tence, or paragraph based on a given input (Demszky et al., 2023). The quality of the output from
029   a language model can be heavily influenced by the input prompt it receives (Arvidsson & Axell,
030   2023). One of the capabilities of LLMs is in-context learning, where they learn a new task from
031   a small set of exemplars provided in the prompt during inference (Minaee et al., 2024). Prompt
032   engineering is the process of designing and refining input prompts to elicit desired responses from
033
      LLMs (Ekin, 2023).
034   In Chain-of-Thought (CoT) (Wei et al., 2022), given a prompt with exemplars that include an input
035   part and an output part, a chain of thought consists of a series of intermediate natural language rea-
036   soning steps added between the input and output parts in each exemplar to produce the final output.
037   However, the CoT prompting doesn’t have the ability to update its knowledge from the external
038   world. ReAct (Yao et al., 2023) prompting addresses the problem by providing the language model
039
      with a prior language description to guide its reasoning about solving diverse language reasoning
      and decision making tasks and adapting this reasoning by acting on and receiving the feedback from
040
      the external world. In Reflexion (Shinn et al., 2023), they proposed autonomous decision-making
041
      LLM agents by adding a reflection step to the CoT or ReAct prompt to adjust the reasoning, facilitat-
042   ing language agents’ learning from prior failings through verbal reinforcement. VOYAGER (Wang
043   et al., 2023) is a LLM based agent designed to explore an open-ended world and attain diverse
044   skills through the integration of automatic curriculum, skill library management, and an iterative
045   prompting mechanism incorporating environmental feedback, execution errors, and self-verification
046   to enhance program performance. In Motif (Klissarov et al., 2024), an agent is trained through re-
047   inforcement learning to maximize rewards from a parameterized model, which is trained based on
048   preferences selected by a language model over pairs of actions, aimed at achieving a specific goal in
049   a given environment.
050   In this study, we propose Self-Adaptive Language Agent (SALA), which is an adaptive decision-
051   making language agent that self-adjusts the reasoning process of ReAct with a correction mechanism
052   from Reflexion. Here, we let the language model agent adapt its own policy by correcting its previ-
053   ous failure using its internal knowledge. The proposed SALA differs from Reflexion (Shinn et al.,
      2023), which uses two LLMs: one for action generation and another for reflection whereas SALA

083   Let a text game be denoted as a function f that maps the state s ∈ V and the action a ∈ V to an
084   observation o ∈ V, where V is a set of vocabulary. Let πθ be an LLM agent over a pre-trained set
085   of parameters θ. Let s0 be the initial state of the environment f , we aim to produce a sequence of
086   actions (a0 , a1 , a2 , . . . ), where ai ∈ V for i ∈ Z, to change the state to a terminal state that indicates
087   the game is cleared.
088   In ReAct prompting (Yao et al., 2023), they propose to use an LLM to produce the thought
089   (t0 , t1 , t2 , . . . ), where ti ∈ V for i ∈ Z, by ti ∼ πθ (ti |si ), where si = {si−1 , ti−1 , ai−1 , oi }
090   for i ∈ Z+ , ai ∼ πθ (ai |si , ti ) for i ∈ Z, and oi+1 = f (si , ai ) for i ∈ Z.
091
      The limitation of ReAct prompting (Yao et al., 2023) is that complex tasks with a large action space
092
      require more demonstrations to learn effectively. The LLM may produce incorrect actions that do not
093
      lead to task completion. Reflexion (Shinn et al., 2023) addresses this problem by using an additional
094   LLM to iteratively provide reflection text that will be added to the ReAct prompt for improvement.
095   More specifically, for each trial ep ∈ Z+ , if ep ≥ N , where N is a maximum number for each
096   trail, a self-reflection rep is generated and a new state sep+1
                                                                 0    = {sep+1

097   as the initial state in the next trial ep + 1. However, the method in Reflexion (Shinn et al., 2023)
098   necessitates two LLMs, where one LLM is used to generate the thought or action, and another LLM
099   is used to generate the reflection. We will modify this by using a single LLM to generate thought,
100   action, and self-adaptation, which is the correction from the previous failed trial.
101
      The architecture of the main idea of our work is shown in Figure 1. A desire is provided to an
102   agent to motivate it to solve a specific task in a given environment. The agent can perform an action
103   to interact with the environment, causing the state of the environment to change. The agent then
104   receives an observation that describes the status of the environment and a reward signal. The action
105   may be proposed from three different processes: the reasoning process determines the next action
106   based on the current progress; the planning process proposes a series of actions that can be used to
107   solve a specific task; and the adaptation process summarizes previous progress to provide a better
      plan towards maximizing the reward.

      3     T HE ALFW ORLD ENVIRONMENT
163
164
      There are six types of tasks in the ALFWorld environment (Shridhar et al., 2020): Pick and Place,
165   Examine in Light, Clean and Place, Heat and Place, Cool and Place, and Pick Two and Place. For
166   each task, a description of available receptacles is given in the first part of the instruction as follows:
167   You are in the middle of a room. Looking quickly around you, you see a {recep1 id}, a {recep2
168   id}, . . . , and a {recepN id}, where recepN refers to the Nth receptacles like drawers or cabinet and
169   id∈ Z+ . An example is shown in the text in Figure 2 with a green background.
170
      After the description of the available receptacles, the goal instructions are provided based on the six
171   different task types. For a Pick and Place task, the instruction will be either ”put a obj in recap” or
172   ”put some obj on recap”. For an Examine in Light task, it will be either ”look at obj under the lamp”
173   or ”examine the obj with the lamp”. For a Clean and Place task, it will be either ”put a clean obj in
174   recap” or ”clean some obj and put it in recap”. For a Heat and Place task, it will be either ”put a hot
175   obj in recap” or ”heat some obj and put it in recap”. For a Cool and Place task, it will be either ”put
176   a cool obj in recap” or ”cool some obj and put it in recap”. For a Pick Two and Place task, it will
177   be either ”put two obj in recap” or ”find two obj and put them in recap”. Inside the curly brackets,
178   {obj}, {recep} and {lamp} refer to object, receptacle, and lamp classes, respectively. An example
179
      is shown in the text in Figure 2 with a red background.
180   After the goal instruction of the task, the agent or the user can interact with the game environment
181   using the following nine different text actions: go to recap id, open recap id, clean obj id with recap
182   id, take obj id from recap id, close recap id, close recap id, heat obj id with recap id, put obj id in/on
183   recap id, and use recap id. Given the action, the game environment will return text observations
184   accordingly. An example is shown in the text in Figure 2 with a magenta background.

190   In ReAct (Yao et al., 2023), they randomly annotate the trajectories for each task type. Each tra-
191   jectory includes sparse thoughts that decompose the goal, track subgoal completion, determine the
192   next subgoal, and use commonsense reasoning to find an object and determine what to do with it.
193   An exemplar used in ReAct (Yao et al., 2023) is shown in Figure 3. The instruction text, “Interact
194   with a household to solve a task. Here are two examples.”, is added above the exemplars to indicate
195
      that the general goal is to complete a household task, and the texts below it include two exemplars.
      Following the examples, the text ’Here is the task.’ is concatenated, followed by the description of
196
      available receptacles and the description of the goal instruction from the ALFWorld environment
197
      (Shridhar et al., 2020). Finally, the greater-than symbol (’>’) is added to indicate that the language
198   model should generate text after it, starting with actions or thoughts.
199
200   ReAct (Yao et al., 2023) evaluated their method on 134 tasks in the ALFWorld environment (Shrid-
201
      har et al., 2020), achieving a success rate of 70.9% using PaLM-540B (Chowdhery et al., 2023) and
      78.4% using the GPT-3 text-davinci-002 model (Brown et al., 2020), where each output token is se-
202
      lected with the highest probability. However, PaLM-540B (Chowdhery et al., 2023) is not publicly
203
      available, and the text-davinci-002 model (Brown et al., 2020) was shut down by OpenAI on January
204   4th, 2024. Due to the difficulty in reproducing the results from (Yao et al., 2023), we tested various
205   open-source LLMs and selected the one with the highest success rate to develop our method. The

313
314   In Reflexion (Shinn et al., 2023), a status text indicating whether a task is successful or not, along
315   with a reflection text that guides the next trial toward success, are concatenated after the ReAct
316   exemplars. Reflexion (Shinn et al., 2023) used two exemplars, as shown in Figure 4 in Appendix
317   A.1, to guide the LLM in generating the reflection.
318   Unlike Reflexion (Shinn et al., 2023), which uses one LLM to generate thoughts and actions and
319   another LLM to generate reflections, SALA uses a single LLM to generate thoughts, actions, and
320   self-adaptations by concatenating the two Reflexion exemplars (Shinn et al., 2023) after the two Re-
321   Act exemplars (Yao et al., 2023) for each task. This approach allows the adaptation to be generated
322   after the LLM receives the string ”STATUS: FAIL”. We experiment with this single-LLM setup
323   using the exemplars from ReAct (Yao et al., 2023) and Reflexion (Shinn et al., 2023) as depicted in
      Figure 5 in Appendix A.1.

      Initially, the input to the LLM is a prompt with the instruction, exemplars, and the description of
336
      the environment and the task to be completed. The output of the LLM will be the input prompt
337   concatenated with a series of thoughts, actions, and observations. We extract the first thought or
338   action from the output by extracting the line immediately following the input prompt content. The
339   line immediately after the greater than symbol (>) is used as input to the environment. If the input
340   to the environment contains the string ”think:”, we overwrite the output with ”OK.”, following the
341   process described in (Yao et al., 2023; Shinn et al., 2023).
342
      If the task is completed, the environment will return a variable with a value of 1, and the process is
343   finished. Otherwise, we concatenate the extracted thought or action with the observation and append
344   this new text to the input prompt to form the next input prompt. After 50 steps, if the task remains
345   unfinished, we reinitialize the ALFWorld environment (Shridhar et al., 2020) and append ”STATUS:
346   FAIL” and ”New plan:” after the last observation. This updated text becomes the next input prompt
347   to the LLM. The output of the LLM will include the adaptation text following ”New plan:”, which
348   is extracted and concatenated above the initial prompt, sep
                                                               0 . This new prompt is then used as the next
349   input to the LLM to obtain corrected thoughts and actions. We set a maximum of 9 trails; if ep ≥ 10,
350   we terminate the trial and proceed to the next task. The results of this experiment, conducted for 14
351
      different tasks using gemma-2-9b-it, are shown in Table 2.
352   Among the fourteen tasks, ten were completed successfully, while four were not finished after reach-
353   ing 10 adaptation steps. Of the ten successful tasks, eight were completed without reaching the
354   adaptation step. Failures in tasks 7 and 9 for both methods were due to issues within the ALF-
355   World environment (Shridhar et al., 2020). Although both agents successfully finished the tasks for
356   tasks 7 and 9, the environment failed to indicate the task were completed. Consequently, tasks 7
357
      and 9 are excluded from the evaluation of different methods. Excluding these tasks, the proposed
      SALA achieves a success rate of approximately 83%, outperforming the ReAct-based agent, which
358
      achieved a success rate of 67%.

      5   C ONCLUSION
393
394
      We present a novel in-context learning algorithm designed for a single language model to complete
395
      tasks in a text-based game by correcting its previous failures. This approach, by introducing self-
396
      adaptation reduces the number of models used in previous work (Shinn et al., 2023) from two LLMs
397   to one LLM, gaining in autonomy. Our findings indicate that the gemma-2-9b-it model achieves
398   the highest success rate of 62% for completing tasks in the ALFWorld environment (Shridhar et al.,
399   2020) using ReAct prompting (Yao et al., 2023), compared to other selected open-source language
400   models. In the twelve selected tasks from the ALFWorld environment (Shridhar et al., 2020), the
401   proposed SALA, utilizing the gemma-2-9b-it model, achieved a success rate of 83%, outperforming
402   the ReAct-based agent (Yao et al., 2023) with the same model, which attained a 67% success rate.
403   In addition, we show that using SALA with gemma-2-9b-it, two of the six tasks that could not be
404   completed in one trial can be completed in the second attempt by appending the adaptation from the
405
      previous trial. Future work will involve further experimentation with different in-context learning
      algorithms to complete decision-making tasks using a single language model without Reflexion
406
      exemplars for reducing the number of tokens in the input prompt.
