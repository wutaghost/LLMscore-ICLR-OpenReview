011                                                A BSTRACT
012
013            Graph neural networks (GNNs) have achieved dominant performance in various
014            prediction tasks on graphs. When deploying GNNs in the real world, estimating
015            the possibility of out-of-distribution (OOD) testing samples becomes a crucial
016            safety concern. Although some research has investigated the graph OOD detec-
017            tion problem, most have concentrated on single-label classification scenarios, a
018            specific case of the more general multi-label classification, which has broader ap-
019
               plications, such as in social networks where nodes can represent users with mul-
               tiple interests or attributes. In this paper, we first introduce and define the multi-
020
               label graph OOD detection problem and propose a simple yet effective pattern
021
               matching-based OOD detection method to address it. In particular, our method
022            utilizes feature pattern matching and label pattern matching to obtain two match-
023            ing scores. By incorporating topological structure adjustment, we ultimately de-
024            rive confidence scores, serving as indicators of the likelihood that a test sample
025            is an OOD instances. We conduct extensive comparisons with existing OOD de-
026            tection methods in the context of multi-label graphs. The results show that our
027            method achieves an impressive 7.61% reduction in FPR95 compared to the lead-
028            ing baselines, setting a new state-of-the-art. Furthermore, our approach can serve
029            as a benchmark for OOD detection on multi-label graphs.

032   1    I NTRODUCTION

034
      Graphs serve as a fundamental data structure that plays a pivotal role in modeling intricate relation-
035
      ships between entities, including social, biological, and internet networks. With the rapid advance-
036   ment of deep learning, the emergence of graph neural networks (GNNs) (Scarselli et al., 2008) has
037   revolutionized the way of analyzing and leveraging graph-structured data (Taghibakhshi et al., 2023;
038   Brilliantova et al., 2023; Gao et al., 2021). GNNs are proven to be exceptionally adept at uncovering
039   complex patterns and making predictions based on the graphical inter-connections (between nodes
040   and edges) along with their associated features, enabling graph learning tasks, like node classifica-
041   tion, link prediction, and more.
042   However, when deploying a trained GNN involving graph-structured data in practice, it is inevitable
043   to encounter previously unseen out-of-distribution (OOD) test inputs, which do not conform to the
044   distribution of the training data. While graph OOD detection is underscored by recent research
045   (Zhao et al., 2020; Stadler et al., 2021; Wu et al., 2023), it is worth highlighting that existing studies
046   primarily focus on single-label classification, where each node is assigned to a single label. Notably,
047   the more general and practical scenario of multi-label graphs remains largely unexplored.
048   Basically, in single-label classification tasks on graphs, each node is assigned an exclusive label from
049   a predefined set of classes. For example, as shown in Figure 1 (a), in the analysis of occupational
050   categories of users in social networks, nodes might be categorized as teachers, engineers, or doctors,
051   with each node belonging to only one of these categories. In contrast, in a multi-label graph, nodes
052   can simultaneously belong to multiple categories. As shown in Figure 1 (b), when analyzing users’
053   interests and hobbies, each user can have multiple attributes, such as music, sports, and movies. In
      this case, a user can be associated with multiple labels simultaneously. Compared to the former,

      2.1   M ULTI - LABEL N ODE C LASSIFICATION
126
127   Multi-label node classification refers to the task of assigning multiple labels to each node in a graph.
128   This has widespread applications in the real world, such as protein function prediction (Zhao et al.,
129   2023), social networks (Wu et al., 2018), and more (Xu et al., 2013). The methods for addressing
130   the task of multi-label node classification can be categorized into three main types.
131   The first category involves methods based on node embedding. For example, (Perozzi et al., 2014;
132   Khosla et al., 2020) employs a lookup table to generate embeddings where similar nodes are closely
133   located. The learned representations are then applied as input features to various downstream pre-
134   diction modules.
135
      The second category involves methods based on convolutional neural networks, such as (Shi et al.,
136
      2020; Zhou et al., 2021). These methods commence by gathering node representations through the
137   aggregation of feature information within their local neighborhoods. Then, the extracted feature
138   vectors are fused with label embeddings to form ultimate node embeddings. Subsequently, these
139   node embeddings are fed into a classification model for the generation of node labels.
140
      The last category is based on graph neural networks, with the differences primarily in how the aggre-
141
      gation layers are implemented. The classical Graph Convolutional Network (GCN) (Kipf & Welling,
142
      2017) performs degree-weighted aggregation over neighborhood features. Since GCN is limited in
143   learning general neighborhood mixing relationships, MixHop (Abu-El-Haija et al., 2019) proposes
144   learning these relationships by repetitively mixing features of neighbors at varying distances. Addi-
145   tionally, some methods combine GNNs with label propagation (LPA) algorithm. For instance, GCN-
146   LPA (Wang & Leskovec, 2020) utilizes LPA as regularization to assist GCN in learning appropriate
147   edge weights, thereby enhancing classification performance.
148
      Instead, in this paper, we focus on the OOD detection problem in the context of multi-label graph
149   node classification, which is underexplored.
150
151
152   2.2   O UT- OF - DISTRIBUTION D ETECTION
153
      In recent years, the problem of OOD detection has garnered widespread attention, particularly in the
154
      context of vision tasks. A category of methods (Hendrycks & Gimpel, 2017; Liang et al., 2018) relies
155
      on computing uncertainty scores based on the output of trained neural network, to measure OOD
156   samples. For example, (Hendrycks & Gimpel, 2017) proposes that samples with lower maximum
157   softmax probability values in model predictions are more likely to be OOD. Also, there are OOD
158   detection methods based on distances between samples in the feature space (Sehwag et al., 2021;
159   Sun et al., 2022), as well as methods involving training a generative model for OOD detection (Ren
160   et al., 2019). Given that previous methods primarily focused on detecting OOD samples in the setting
161   of multi-class classification, where each sample is assigned to a single label, some methods(Wang
      et al., 2021; Hendrycks et al., 2022) have been proposed to address the challenge of multi-label OOD


                                                         3
      Under review as a conference paper at ICLR 2025


162
      detection. For example, (Wang et al., 2021) estimates an OOD uncertainty score by aggregating label
163   energy scores from multiple labels.
164
165   However, as the aforementioned methods are primarily designed for vision tasks, most of them rely
166
      on the assumption that input samples are independently and identically distributed, which is evi-
      dently not applicable to the scenario of node classification in graph (Wu et al., 2023). Consequently,
167
      some OOD detection methods (Zhao et al., 2020; Stadler et al., 2021; Wu et al., 2023) tailored
168
      for graphs have been proposed. For example, (Zhao et al., 2020) proposes a Graph-based Kernel
169   Dirichlet Distribution Estimation(GKDE) method, aiming to accurately predict node-level Dirichlet
170   distributions and detect OOD nodes. (Wu et al., 2023) utilizes an energy function directly extracted
171   from a GCN to discriminate whether an input sample belong to ID or OOD.
172
      The existing graph-based OOD detection methods mainly focus on single-label node classification
173
      task where each node is assigned with only one label, overlooking the crucial application scenario
174
      of multi-label node classification in graph.

      3.1   M ULTI - LABEL G RAPHS
181
      Consider an undirected graph G = {V, E}, where V = {v1 , . . . , vN } represents the set of ver-
182
      tices (or nodes), with N denoting the number of vertices and E = {e1 , . . . , eM } represents the
183
      set of edges connecting these vertices. Let A ∈ {ai,j }i≤N,j≤N be the adjacency matrix of the
184   graph, where each element aij = 1 indicates the existence of an edge connecting vi and vj , and
185   0 otherwise. In a multi-label setting, each vertex vi is associated with a label vector denoted as
186   yi = {y1 , y2 , . . . , yKl }, where Kl is the total number of classes.
187
188
189   3.2   M ULTI - LABEL N ODE C LASSIFICATION (MLNC)
190
191   Given a graph G, following the semi-supervised learning paradigm, we assume that there are Nl
192
      labeled nodes forming the training set Dtr , and Nu unlabeled nodes forming the testing set Dte ,
      where N = Nl + Nu . The goal of multi-label graph node classification is to learn a mapping
193
      f : (V, A) → Ŷ, where Ŷ = {ŷ1 , . . . , ŷN } is the predicted label matrix, and f (·) is a graph neural
194
      network.

197   3.3   OOD D ETECTION FOR MLNC
198
199   The task of multi-label graph node classification aims to achieve high classification performance
200   on test nodes, under the assumption that test inputs have identical distribution with training data,
201   referred to as the in-distribution (ID). A robust classifier for multi-label graph node classification
202   should not only accurately classify ID inputs (from known categories) but also effectively identify
203   OOD test inputs (from unknow categories).
204   To formalize the problem, we consider two distributions Pin and Pout as ID and OOD data spaces,
205   respectively. We operate under the assumption that training data Dtr originate from Pin . Given a
206   set of unlabeled test inputs Dte , we assume it consists of both ID and OOD nodes, sampled from
207   Pin × Pout . Then, the goal of MLNC OOD detection is essentially to find a decision boundary H
208   for any given test input x, such that,

214   graphical topology to facilitate OOD detection during the testing phase, which contrasts with the
215   OOD detection setups commonly used in vision tasks (Hendrycks & Gimpel, 2017), where test
      inputs are generally treated as independent instances.

307   In this section, we introduce the another pattern matching strategy, called the label pattern matching.

310

311

312

366   is commonly employed for error correction and subsequent refinement of final predictions (Huang
367

368

369

532   6     C ONCLUSION
533
534   In this paper, we formulate the problem of OOD detection for multi-label graph node classification,
535   and propose a pattern matching-based OOD detection method. Our method first utilizes the logit
536   and feature information of the data to match with the patterns in the training data, resulting in
537   a matching score. Then, we employ a propagation algorithm to further amplify the difference in
538   matching scores between ID and OOD samples. By normalization and summing two scores, we
539   obtain the final confidence score, which is used to determine whether the sample belongs to the ID
      or OOD category. Extensive experiment results demonstrate the superiority of our proposal.
