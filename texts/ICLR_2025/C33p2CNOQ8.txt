009                                               A BSTRACT
010
011            We demonstrate that architectures which traditionally are considered to be ill-
012            suited for a task can be trained using inductive biases from another architecture.
013            Networks are considered untrainable when they overfit, underfit, or converge to
014            poor results even when tuning their hyperparameters. For example, plain fully
015            connected networks overfit on object recognition while deep convolutional net-
016
               works without residual connections underfit. The traditional answer is to change
               the architecture to impose some inductive bias, although what that bias is, is un-
017
               known. We introduce guidance, where a guide network guides a target network
018
               using a neural distance function. The target is optimized to perform well and to
019            match its internal representations, layer-by-layer, to those of the guide; the guide
020            is unchanged. If the guide is trained, this transfers over part of the architectural
021            prior and knowledge of the guide to the target. If the guide is untrained, this trans-
022            fers over only part of the architectural prior of the guide. In this manner, we can
023            investigate what kinds of priors different architectures place on a fully connected
024            network. We demonstrate that this method overcomes the immediate overfitting
025            of fully connected networks on vision tasks, makes plain CNNs competitive to
026            ResNets, closes much of the gap between plain vanilla RNNs and Transformers,
027
               and can even help Transformers learn tasks which RNNs can perform more easily.
               We also discover evidence that better initializations of fully connected networks
028
               likely exist to avoid overfitting. Our method provides a mathematical tool to in-
029
               vestigate priors and architectures, and in the long term, may demystify the dark
030            art of architecture creation, even perhaps turning architectures into a continuous
031            optimizable parameter of the network.

034   1   I NTRODUCTION
035
036   When creating neural networks, as a community we follow recipes that select among a few architec-
037   tures known to work for particular tasks (Ren et al., 2021; Cong & Zhou, 2023; Goodfellow et al.,
038   2014). Architecture is critical. The gains made on tasks like object recognition had to do with
039
      imposing an inductive bias, i.e., a prior, by designing new architectures (Goyal & Bengio, 2022;
      Bachmann et al., 2024). Convolutional networks unlocked many vision problems (Krizhevsky et al.,
040
      2012; He et al., 2016a) and the recent advent of the Transformer (Vaswani, 2017; Devlin, 2018;
041
      Achiam et al., 2023) did the same for language. Despite this, finding new architectures and over-
042   coming the limitations of existing architectures is a dark art. While an architecture imposes some
043   prior, we often do not fully understand what that prior is. For example, there is discussion about
044   exactly what the role of residual connections is (Jastrzebski et al., 2017). If we fully understood the
045   priors that architectures imposed, we could translate between priors and architectures freely – either
046   by creating the prior we want and then deriving the appropriate architecture or by dispensing with
047   architectures and imposing the prior directly.
048   Recent theorems (Poggio & Fraser, 2024) state that for each function which is efficiently Turing
049   computable, there exists a deep network that can approximate it well. Furthermore, a graph rep-
050   resenting such a function is compositionally sparse, that is the nodes of the associated Directed
051   Acyclic graph (DAG) represent constituent functions with a small effective dimensionality. A rea-
052   sonable conjecture is that neural networks with an architecture which is similar to the DAG of the
053   unknown target function are especially successful in learning it, as it is the case for convolutional
      networks for image recognition and similar tasks. Because we do not understand the relationship

      between the kinds of priors on the target functions that different architectures impose, even simple
083
      questions have no known answer, such as, does there exist an initialization of an FCN that makes it
084   behave like a CNN, though the graphs of the function they represent are fundamentally different?
085
086   We introduce a new tool which we term guidance to study this problem. Given a target network,
087
      we guide it with a guide network. In addition to the target’s original loss, the target attempts to
      match the representation of its intermediate layers to those of the guide. We use a measure of
088
      representational similarity (Kornblith et al., 2019; Cristianini et al., 2001; Cortes et al., 2012), also
089
      termed a neural distance function, to compute the distance between representations of two arbitrary
090   layers. Neural distance functions are often used in neuroscience to compare activity in networks and
091   brains (Schrimpf et al., 2018; Conwell et al., 2021a; Subramaniam et al., 2024). In light of recent
092   work that shows that networks of very different architectures have internal activity that is extremely
093   similar to one another (Han et al., 2023; Conwell et al., 2021a;b), we repurpose this distance function
094   as a means to transfer priors between networks layer by layer.
095
      We make the following contributions:
096
097         1. We develop guidance to transfer priors between networks using representational alignment,
098            investigate one representational alignment method, centered kernel alignment, CKA (Ko-
099
               rnblith et al., 2019), and apply it to several classical problems.
            2. Image classification improvements: Deep or wide fully connected networks stop overfitting
100
               when guided by a ResNet. No-skip CNNs close much of the gap with ResNets when guided
101
               by a ResNet.
102         3. Sequence modeling improvements: RNNs significantly improve their copy and paste ac-
103            curacy when guided by a Transformer. Transformers increase their parity accuracy when
104            guided by an RNN. RNNs close much of the gap to a Transformer when guided by one.
105         4. Many of these gains exist when the guide network only transfers over its priors, not its
106            knowledge, i.e., the guide is untrained and is never updated.
107         5. By disconnecting the guide early, we provide evidence for a better initialization regime for
               fully connected networks that avoids immediate overfitting.

      Through the transfer of priors across architectures, guidance can serve as a new tool for investigat-
109   ing the role of architecture and the relationship between priors and architectures. It can also help
110   discover new initialization methods, for which no known tool exists.
111
112
      Our work has a number of limitations. We aimed for coverage of many tasks instead of maximal
      performance on any one task. This would have required us to carefully tune the hyperparameters
113
      involved. We preferred to show how guidance works in general rather than in cherry picked and
114
      carefully tuned settings. To that end, we also did not optimize networks to convergence, nor did
115   we attempt to experiment with other optimizers. Once we reproduced a well-known problematic
116   training phenomenon, we showed that it could be overcome. We consider a network trainable and a
117   problem to be overcome, when the original problem disappears. For example, successfully training
118   fully connected networks for object recognition was hopeless because they immediately overfit;
119   using our guidance method they no longer do so. This does not mean that they are necessarily useful
120   as object recognizers at present. In the case of fully connected networks, their present performance
121   with guidance training is too low, but with additional work we believe their performance could be
122   substantially increased now that their train and test loss are moving in the right direction. In some
123
      cases, by applying guidance, we do see large useful improvements, such as with deep CNNs, RNNs
      and Transformers, although much more remains to be exploited there too.

      Representational Distance: Our method builds on several metrics that measure distance between
130
      high-dimensional activations extracted from neural networks or activity in the brain (Klabunde et al.,
131
      2023). Some of these distance metrics make comparisons based on kernel matrices (Kornblith et al.,
132   2019; Cristianini et al., 2001; Cortes et al., 2012) or relative distances (Kriegeskorte et al., 2008;
133   Moschella et al., 2022) between sample representations in a set. Others compute linear (Wehbe
134   et al., 2014; Schrimpf et al., 2018) or orthogonal projections (Beauducel, 2018) from one set of
135   representations to another. These metrics are designed based on a set of desired invariant properties
136   such as permutation invariance or invariance to linear transformations.
137
      Such approaches have been commonly applied in neuroscience for measuring representational dis-
138   tance of activations from networks and activity in the brain to understand which neural networks are
139   architecturally most similar to the brain (Wehbe et al., 2014; Conwell et al., 2021a; Subramaniam
140   et al., 2024; Goldstein et al., 2020). Under this context, Han et al. (2023) has shown the inability of
141   current representational distance metrics – specifically the metric used here, centered kernel align-
142   ment – to distinguish representations based on architecture. This paper gives the foundation for our
143   intuition that networks may have similar representations that allow for transferring inductive biases
144   from one network to another.
145   Untrainable Networks: This work examines FCNs and plain CNNs for image classification and
146   RNNs for sequence modeling. Prior work explored similar approaches but performed poorly com-
147   pared to the guide networks we leverage for improved training.
148
      For image classification, small feed-forward networks with 3-5 hidden layers and fewer than 100
149
      units per layer were trained on object recognition datasets (Ma & Khorasani, 2004; Bebis & Pa-
150   padourakis, 1992; Khasnobish et al., 2012; Oh & Suen, 2002). These efforts prioritized training
151   fit over generalization performance and achieved low results (Ma & Khorasani, 2004; Bebis & Pa-
152   padourakis, 1992). Strategies to reduce overfitting, such as topological structure (Schittenkopf et al.,
153   1997) or early stopping (Caruana et al., 2000), were hindered by complex designs and hyperparam-
154   eter tuning, leading to poor training fits. Further methods used alignment between thin deep FCNs
155   and wide shallow FCNs to prevent overfitting, an approach similar to our paper (Romero et al.,
156   2014). Deep convolutional networks were also applied to image classification (Krizhevsky et al.,
157   2012; Bengio et al., 1993) but struggled with vanishing gradients, limiting their depth.
158   In sequence modeling, classical RNNs (Schuster & Paliwal, 1997; Pearlmutter, 1990; Connor et al.,
159   1994; Hammer, 2000) were constrained by vanishing and exploding gradients (Hochreiter, 1998),
160   making them unsuitable for long sequence tasks requiring memorization (Graves, 2014). Gradi-
161   ent flow techniques were developed, but significant progress came from architectures like LSTMs
      (Hochreiter, 1997) and transformers (Vaswani, 2017). Transformers, however, have been found un-

162
      trainable on formal language tasks requiring full-sequence reasoning, where RNNs succeed (Bhat-
163   tamishra et al., 2020).

165   Model Distillation: Guidance shares a resemblance with model distillation (Hinton, 2015; Gou
166
      et al., 2021; Sanh, 2019; Hsieh et al., 2023). Distillation transfers knowledge from a teacher model
      to a student model by introducing a new component to the loss function that enforces the student
167
      model to behave like the teacher model (Kim et al., 2021; Zhou et al., 2021). This usually consists
168
      of penalizing the KL-divergence between the logit predictions of the student and teacher model.
169
170   Representation-based distillation (Tian et al., 2019; Chen et al., 2021; Lin et al., 2020) and alignment
171   techniques have been proposed to improve alignment between two networks. Certain works have
172
      proposed correlation congruence or similarity preserving metrics (Ramos et al., 2023) for aligning
      two networks, particularly as a way to do architecture search between CNNs (Bashivan et al., 2019).
173
      Methods have been proposed that use CKA as an alignment approach between representations of
174
      two networks or with representations in the brain with notable improvement in network performance
175   (Saha et al., 2022; Dapello et al., 2022).
176
177   We distinguish guidance from distillation. Guidance can use a smaller untrained guide instead of a
178
      larger trained teacher. This is due to guidance operating over intermediate activations of the network
      instead the output of the network probabilities or output features, like distillation does. Guidance
179
      also operates at many levels at the same time, aligning many layers at once. This helps address the
180
      credit assignment problem that gradient descent has when tuning weights early in a network. We
181   also consider many more networks for guidance than is traditional for distillation including networks
182   which have very different architectures like Transformers to RNNs. Distillation is usually carried
183   out between two closely related architectures. We apply guidance to do the opposite.
184

189   ment with a guide network, N G . Only the parameters, θT , of the target are updated — the guide’s
190   parameters, θG , remain fixed. Per minibatch, representational similarity is computed between the
191   activations of iG th layer of the guide, AG     G                                                T
                                                 iG (θ ), and activations from a corresponding layer i of

192   the target, AiT (θ ). We refer to the correspondence between layers of the guide {i } and layers of
193   the target {iT } as I. While this correspondence, I, could be complex as any two architectures can
194   form a guide/target pair, here we choose architectures that make the correspondence obvious as is
195
      discussed later. For example, the stacked RNNs and Transformers have the same number of layers
      in our experiments.
196
197   The target and guide receive the same input. Per minibatch, we collect activations from intermediate
198   layers of both networks. Layers of guide network are mapped to layers of the target network; see
199   Figure 1. We formulate the loss in terms of minimizing the representational dissimilarity, M̄, i.e.,
200   the complement of a representational similarity metric, between guide and target activations layer
201
      by layer, summing the results. We only consider the centered kernel alignment, CKA, between
      the activations in this publication. Many other possibilities exist. Any representational similarity
202
      function which is differentiable could be used. Efficiency or incremental computation is much more
203
      important than it is in traditional applications since this operation happens for every minibatch. We
204   discuss details in Appendix C.1.

220   Layerwise Mapping We design a simple method for mapping guide layers to target layers as part
221   of providing supervision. The goal of this method is to make guide and target networks architec-
222   turally agnostic i.e. we can supervise any target network with any guide network.
223   As a simple approach, we evenly spread layer computations of our guide network over our target
224   network. For example, if we consider ResNet-18 and a 50-layer FCN, we would map every convolu-
225   tional ResNet layer to every second or third linear layer of the FCN. The intuition for this approach
226   follows from the aim of discovering the same function of the guide network using a target network.
227   Through the design of evenly spreading layers of our ResNet-18, we are guiding the FCN to find a
228   function similar to the guide network.
229   For our mapping, we consider activations from all layers with tunable weights i.e. convolutional,
230   linear, or LSTM/RNN based layers as well as activations from layer normalizations. For multiple
231   stacked RNNs, LSTMs, or transformers, we extract feature representations from intermediate layers
232   in the stack as well. Using all layers is useful for guidance as it provides a strong signal to induce
233   alignment between the target and guide networks during training. We empirically find that more lay-
234   ers leads to stronger results. Skipping layers based on non-linear transformations reduces memory
235   overhead associated with storing representations per batch.

240
      We design several settings with different target and guide networks to thoroughly test our approach.
      We include a range of image and sequence modeling tasks. In choosing target networks, we con-
241
      sider a broad range of designs for networks that are not traditionally applied (e.g., a FCN in image
242
      classification).
243
244   To systematically evaluate our approach, we
245         1. Untrainable Architectures: Experiments where the target networks are difficult to train
246            due to architectural limitations, irrespective of the task. For example, overfitting in deep
247            FCNs or memory incorporation in RNNs.
248         2. Untrainable Tasks: Experiments where certain tasks are inherently challenging for spe-
249            cific architectures, making them untrainable without additional supervision. For example,
250            sequence classification with transformers.
251
252
      Tasks We describe the task settings. For an image-based task, we focus on image classification
253
      and use the ImageNet-1K dataset (Deng et al., 2009) for training and testing. We use the splits
254   defined by the dataset. We report accuracy on the validation set for all experiments.
255
256   We consider three sequence modeling tasks to allow for a broader range of architectural settings.
257
      We first consider a task called copy-paste (Graves, 2014). In this task, we generate a sequence of
      numbers in the range of 1 − 10. The model is trained to recover the same sequence in the output.
258
      In our setting, we consider sequence lengths that range from 20 to 40 values total (internal sequence
259
      and padding). We generate a copy-paste dataset, sampling sequences containing numbers between
260   1 and 10. We generate a total of 100, 000 examples, training on 80, 000 examples, validating on
261   10, 000 examples, and testing on 10, 000 examples.
262
263
      We also include the parity task, a binary classification task where a model is fed a bitstring and
      outputs 1 when there is an even number of ones in the bitstring and 0 otherwise. We generate a series
264
      of bitstrings with sequence lengths that range from 2 and 50 as done in prior work (Bhattamishra
265
      et al., 2020).
266
267   Finally, we consider a language modeling task using the WikiText-103 dataset (Merity et al., 2016)
268   where models must predict the next token given some context. This uses the train, validation and
269
      testing splits defined by the WikiText dataset and for all experiments, we use a context length of 50.
      We tokenize the text data using the GPT-2 (Radford et al., 2019) tokenizer.

297   Sequence Modeling: For our copy-paste and language modeling tasks, we use a vanilla, 4-layer
298   RNN as our target network. In copy-paste, architectural and algorithmic limitations make RNNs an
299   untrainable architecture. For language modeling, vanishing gradients and limited context incorpora-
300
      tion make RNNs an untrainable architecture as the training loss saturates. For the parity task, we use
      a 1-layer transformer encoder architecture, similar to prior work (Bhattamishra et al., 2020; Hahn &
301
      Rofin, 2024). For the copy-paste task, we train a guide network, 4-layer transformer decoder model
302
      which achieves 96.90% accuracy. Similarly, for language modeling, we train a 4-layer transformer
303   decoder guide network with a context window of 50. Our final perplexity is 29.69. For the parity
304   task, we train a 1-layer vanilla RNN as a guide network which achieves 100% accuracy as reported
305   by (Bhattamishra et al., 2020).

308   Training For each setting, we train the base target network and perform an experiment where both
309   a trained and untrained guide network supervises the base target network. All networks are trained
310   with either cross-entropy loss or binary cross-entropy loss, although other loss functions could be
311   applied as well.
312   When training networks for object detection using ImageNet-1K, we use the Adam (Kingma, 2014)
313   optimizer. For all sequence modeling tasks, i.e. copy-paste, parity, and language modeling we use
314   AdamW (Loshchilov, 2017). For language modeling, we also incorporate gradient clipping due to
315   unstable training with long sequences.
316
      To ensure consistency of comparisons across learning curves, we use a consistent batch size of
317
      256. In general, representational similarity metrics are affected by the number of samples in the
318   calculation. The more samples, the better the metric approximates representational distance. We
319   use 256 as a proxy, dependent on GPU memory, although more memory would allow for bigger
320   batch sizes with potentially better results. Due to the large number of training settings, we employ
321   several different learning rates. We tune the learning rate carefully for baseline training to ensure
322   maximal performance of base training. We sweep the parameter across 5 different values and choose
323   the results with the lowest validation loss. This ensures we are choosing the training with the best
      performance.

      Image Classification Across all our networks, we observe significant improvement from using a
362
      guide network to provide representational guidance; see table 2. We see significant accuracy gains
363   of 5-10% on validation performance. We also observe significantly better loss curves from a better

479
      guided network to confirm whether networks adopt priors from their guide networks. Using Deep
480   FCN as our target model, we guide it with a ResNet-18 or a ViT-B-16 (Dosovitskiy, 2020). We
481   then measure the error consistency (Geirhos et al., 2020) between all of the networks; see fig. 3.
482   The error consistency between the initial FCNs is entirely unlike the ResNet-18 or ViT-B. Guidance
483   creates two FCNs which have the same relationship to one another. The ResNet-18-guided FCN
484   and ViT-B-guided FCN have the same error consistency with respect to one another as ResNet-18
485   and ViT-B do. It’s not just that the FCN gets generically better, it adopts a prior from the original
      architecture. We provide further details of the error consistency metric in appendix G.

      Network Initialization Is guidance needed throughout training, or is the effect of the guide to
487   move the target into a regime where the two are aligned and the target can be optimized further
488   without reference to the guide? The answer to this question can shed light on whether the guide
489   is telling us that better initializations are likely to exist for the target. To answer this question, we
490   disconnect the guide from the target after an arbitrary and nominal number of training steps, 150.
491   We did not investigate what the minimal number of steps needed was.
492
      For FCNs, even this early disconnect avoids overfitting; see fig. 4. Furthermore, while preventing
493   overfitting, we have lower training loss from guidance, indicating a better fit. This implies that there
494   likely exists a better initialization for FCNs. In appendix K, we further explore how guidance can be
495   used to find new initialization schemes in FCNs. In future work, we intend to investigate what the
496   untrained guide is doing to the FCN weights to prevent validation loss from increasing dramatically.

500   6   C ONCLUSION
501
502
503
      We demonstrated that guidance eliminates the failure modes of networks previously considered
504   unsuitable or ineffective for specific tasks. Aligning with another network overcomes these short-
505   comings by transferring inductive biases—either architectural and knowledge-based, or solely ar-
506   chitectural when using an untrained guide. This opens the door to many applications.
507
      Our method can be used to study representational and functional design of neural networks in new
508
      ways to reinforce or reanalyze prior theory of neural network optimization. For example, we can
509
      understand distances between architectural components based on which target networks are easier
510   to guide with a particular guide network. We also refine this notion to include a narrow channel
511   through which guidance can occur, the representational similarity. This can serve as a kind of probe.
512   Different representational similarity methods enforce alignment between different properties of the
513   activations of networks.
514
      Characterizing architectural priors and untrainable networks can be made more precise using our
515
      methodology. For example, guidance clearly distinguishes between at least three cases. One where
516   the guide does not help. Another where the architecture of the guide helps, but not the knowledge.
517   And one where both the architecture and the knowledge of the guide help. These are different
518   phenomena whose root causes are not understood at present but that could elucidate the relationship
519   between priors and architectures.
520
      Guidance also proved to be a tool with which to discover the possibility of new initializations. At
521
      the moment, no known method exists to find better initializations for networks. In some cases as
522   with the FCNs for vision, guidance can be disconnected after a nominal number of steps, but still
523   goes on to regularize the target network. This strongly implies that an initialization regime for that
524   target with the same regularization exists. This is all that guidance could do in that case. We now
525   need tools to go backwards, given networks which are correctly initialized and networks which are
526   not, discover what that initialization is. This is a much better place to be in. A systematic sweep of
527   targets and guides to look for better initializations should be carried out.
528   For some networks, like fully connected ones, we only overcame the initial obstacle. Further re-
529   search is needed to refine these into effective vision models, as they avoid immediate overfitting.
530   This may be a matter of scale, optimizers, learning rates, etc. Our results suggest practical appli-
531   cations by significantly narrowing the performance gap between vanilla stacked RNNs and Trans-
532   formers, albeit in small-scale experiments. Given that stacked RNNs are equivalent to single-layer
533   RNNs, most directly to delayed RNNs (Turek et al., 2020), this implies that complex modifications
534   to RNNs may be unnecessary for language modeling. In the future, we hope to find methods for
535   making these networks competitive with their respective guide networks.
536   Looking into the long-term future, it may be that architecture can be turned into a prior that we add
537   to the loss of a generic fully-connected network, or any convenient and capable-enough network.
538   Architecture would lose its prominence and the computational substrate could be disconnected from
539   the priors we need for learning and inference. Architecture could potentially even be turned into a
      parameter to be optimized, rather than a hyperparameter.

771   B     M ETHODOLOGY L IMITATIONS
772
773   Our guide network supervision through representational alignment has one primary limitation due
774   to increased memory usage during training. Due to saving activations across several layers of the
775   two networks, GPU memory usage increases dramatically. Moreoever, our methodology works
776
      better as batch size increases since this allows for better approximation of representational similarity,
      increasing memory usage even more. Furthermore, including more layers for supervision leads to
777
      improved results.
778
779   In this paper, we introduce simple techniques to handle memory constraints such as gradient ac-
780   cumulation and gradient checkpointing (Lin et al., 2017). In practice, more memory optimization
781   techniques may become necessary to consider larger untrainable networks. Further work could con-
782
      sider using stronger representational alignment strategies to reduce the number of samples necessary
      to achieve a strong fit.

1107
1108
       We measure error consistency (κ) (Geirhos et al., 2020) between the guided target networks which
1109   indicates the error overlap between two networks based on the accuracy of the networks, i.e. do
1110   the two networks make similar class predictions? The measure first calculates the expected er-
1111   ror overlap. Suppose a1 is the accuracy of the first guided network and a2 is the accuracy of
1112   the second. The expected error overlap is given by cexp = a1 ∗ a2 + (1 − a1 ) ∗ (1 − a2 ).
1113   Next, we measure the observed error overlap across each sample in the validation set as cobs =
1114   # of samples where both models agree / total trials. Finally, we can write κ as:
1115
1116
                                                                          cobs − cexp
1117                                                                 κ=                                                (7)
                                                                           1 − cexp
1118
1119
1120   κ ranges from −1 to 1, where 1 is perfect agreement, −1 is perfect disagreement and 0 is change
1121   agreement. When κ > 0, this implies that models make consistent error patterns, κ < 0 implies that
1122
       models make inverse error patterns, and κ ≈ 0 implies independent error patterns.
1123
1124   G.1    G UIDE N ETWORK R EPRESENTATION C OMPARISON
1125
1126   We contextualize the findings in error consistency by comparing the representations of the guide net-
1127   works, in this case ResNet-18 and ViT-B-16. We apply a layer mapping between layers of ResNet-
1128   18 and ViT-B-16 and compute the representational similarity over 1000 input images. Results are
1129   shown in fig. 9.
1130   Our findings are useful for error consistency. If models are inheriting inductive biases from their

1354   We use the RSA formulation as described in Kriegeskorte et al. (2008). Specifically, RSA constructs
1355   representational dissimilarity matrices (RDMs) for two sets of representations and compares them
1356   using an outer similarity function.
1357

1358
       set of representations using a distance function d. Formally, we define D ∈ Rb×b as

1367   vectorize the RDM matrices using a function v (since the RDMs are symmetric, we only need to
1368   compare the lower triangles), and compute the similarity between the two vectorized RDMs using a
1369   similarity function s.
