                                                  A BSTRACT
008
009
               In the standard transformer architecture, increasing model parameters leads to lin-
010
               ear growth in computational cost and activation memory. To address this issue, we
011            propose a novel Infinite Parameter Large Language Model (IP-LLM) architecture
012            that decouples model size from computational cost and device memory.
013            Existing large language models are all fixed-parameter models, while human
014            knowledge is infinite and expands daily. Finite parameters are inherently lim-
015            ited in their capacity to accommodate this boundless knowledge. Our IP-LLM
016            architecture can potentially accommodate infinite knowledge, resolving this issue
017            and laying the foundation for realizing a truly omniscient and omnipotent artificial
018            general intelligence in the future.
019            Our architecture achieves performance comparable to MOE (Clark et al., 2022)
020            while requiring significantly less memory.

023   1   I NTRODUCTION
024
025   The emergence of Large Language Models (LLMs) has fundamentally transformed the landscape
026
      of natural language processing, demonstrating exceptional performance across a wide range of real-
      world applications (OpenAI, 2023; Chowdhery et al., 2023).
027
028   However, contemporary Large Language Models possess fixed parameter sizes, resulting in static
029   knowledge upon completion of training. In contrast, human knowledge continuously expands, ren-
030   dering models that require millions of dollars for training obsolete in a relatively short period, which
031   is a considerable waste of resources.
032   Updating knowledge in fixed-parameter Large Language Models poses significant challenges due
033   to catastrophic forgetting (De Lange et al., 2021), often resulting in a decline in their original
034   general capabilities (Cheng et al., 2023; Dong et al., 2023). This necessitates a methodology that
035   enables large models to retain previously acquired knowledge while simultaneously facilitating the
036   continuous learning of new information.
037   Scaling laws for neural language models show the power of scaling (Kaplan et al., 2020; Hoffmann
038   et al., 2022): increasing the number of parameters, amount of training data, or the computational
039   budget has proven to be a reliable way to improve model performance. However, there is a linear
040   relationship between computational footprint, as measured by FLOPs and device memory consump-
041   tion, and parameter count.
042   To address these challenges, we propose a novel Infinite-Parameter Large Language Model (IP-
043   LLM) architecture. Unlike traditional fixed-parameter models, IP-LLM cleverly decouples the rela-
044   tionship between the model parameter scale and the linear cost of computation and device memory.
045   Our core strategy is to divide the model parameters into multiple independent experts, with each
046   expert responsible for storing knowledge in a specific category. During inference, IP-LLM only
047   loads the experts relevant to the current task, significantly reducing computation costs and memory
048   consumption.
049   Specifically, IP-LLM employs a routing-based dynamic parameter selection mechanism. This mech-
050   anism first determines the category of the input text based on its content and context. Then, IP-LLM
051   loads only the expert parameters corresponding to the relevant category for computation, while
052   experts unrelated to the current task remain unloaded. This dynamic parameter loading approach
053   enables the model to effectively handle an infinite amount of knowledge without being constrained
      by memory and computational resources. This is conceptually similar to the Mixture of Experts

055   derstanding capabilities to perform routing, rather than relying on only a subset of parameters as in

115   2.1   C ATASTROPHIC F ORGETTING
116
117   Preventing catastrophic forgetting during training remains a classic challenge in deep learning. Since
118
      2018, numerous studies have explored strategies to mitigate this issue for Large Language Models
      (LLMs). For instance, (Yang et al., 2024) introduced a self-distillation method that addresses the
119
      distribution gap between task datasets and LLMs, effectively reducing catastrophic forgetting while
120
      maintaining general capabilities.
121
122   In empirical investigation, (Luo et al., 2024) revealed that as the scale of Large Language Models
123   (LLMs) increases during continual instruction tuning, the incidence of catastrophic forgetting be-
124
      comes more pronounced. They suggested that employing general instruction tuning could alleviate
      this challenge.
125
126   (Hsieh et al., 2023) introduced a novel approach for training smaller models, demonstrating that
127   these models can outperform LLMs with less training data by utilizing rationales extracted from
128   LLMs as additional supervision. Furthermore, (Wang et al., 2023b) demonstrated O-LoRA, a
129   method designed to mitigate catastrophic forgetting by learning new tasks in orthogonal subspaces,
130
      thereby minimizing interference with previously acquired knowledge.
131
132   2.2   M IXTURE OF E XPERTS
133
134   Several recent works (Shazeer et al., 2017; Lepikhin et al., 2020; Fedus et al., 2022; Zhou et al.,
135
      2022) have adopted the Mixture-of-Experts (MoE) architecture to decouple computational cost from
      parameter count.
136
137   (Geva et al., 2021; Dai et al., 2022) argue that feedforward (FFW) layers store factual knowledge
138   .These layers constitute approximately two-thirds of the total parameters in a transformer architec-
139   ture. The Mixture-of-Experts (MOE) architecture deviates from the traditional single dense feedfor-
140   ward network (FFW) by utilizing a set of sparsely activated expert modules, frequently implemented
141
      as FFWs.
142   (Clark et al., 2022) investigated the scaling properties of MoE language models, demonstrating that
143   increasing the number of experts can effectively enhance performance without incurring additional
144   inference costs. However, their experiments revealed that the efficiency gains offered by MoEs
145   plateau after reaching a particular model size.
146   More recently, (Krajewski et al., 2024) identified that this plateauing phenomenon was a conse-
147   quence of using a fixed number of training tokens. Their findings demonstrate that when the number
148   of training tokens is optimized for computational efficiency, MoEs consistently outperform dense
149   models in terms of FLOPs (floating-point operations) per parameter. Furthermore, they introduced
150   granularity, the number of active experts, as a novel scaling dimension. Their empirical studies
151   revealed that employing higher granularity leads to improved performance.
152
153   2.3   P ROGRESSIVE L EARNING
154
155   The concept of progressive training has attracted significant attention for its potential to expedite the
156   training of large-scale models in both computer vision (Zhang et al., 2023) and natural language
157   processing (NLP) (Yao et al., 2023; Li et al., 2023).
158   A stacking technique that successively doubles model depth was proposed by (Gong et al., 2019).
159   Enhancing this approach, CompoundGrow (Gu et al., 2020) integrates Feed-Forward Network ex-
160   pansion into its scheduling framework. Moreover, (Shen et al., 2022) introduced a method that
161   supports the expansion of hidden sizes in a staged manner. Notably, both Bert2BERT (Chen et al.,
      2021) and LiGO (Wang et al., 2023a) are designed to accommodate all dimensions of growth.

      2.4   LIFELONG LEARNING / CONTINUAL LEARNING USING MOE
163
164   (Chen et al., 2023) proposed using MoE (Mixture of Experts) for lifelong learning in large language
165   models (LLMs), referred to as ”Lifelong MoE.” However, it has limitations: it can only mitigate
166   catastrophic forgetting but cannot prevent it entirely. Moreover, adding new experts may lead to
167   performance degradation in routing for old tasks. The innovations of ”IP-LLM” lie in the following
168   aspects:
169
170   Infinite Parameter Model Architecture : IP-LLM introduces the concept of infinite parameters
171   by grouping parameters and using on-demand loading. During inference, only the required param-
172   eters are loaded, theoretically surpassing the limitations of model size. While Lifelong MoE also
173   expands the model by adding experts, its total parameters remain finite.
174
175   On-Demand Parameter Loading : A core innovation of IP-LLM is its on-demand parameter
176   loading mechanism. The model’s parameters are divided into multiple groups, with each group
177   corresponding to a specific type of knowledge or domain. During inference, only the parameter
178   groups relevant to the current task are loaded, significantly reducing memory usage. Although
179   Lifelong MoE also utilizes a subset of experts during inference, it requires all experts to be loaded
180   into GPU memory, limiting parameter scalability.
181
182   Improvements in Pretraining : IP-LLM introduces a staged pretraining framework. It first learns
183   fundamental language knowledge (e.g., vocabulary, grammar), then trains parameters on different
184   data categories separately, and finally integrates them. This approach helps reduce the parameter
185   size of individual experts.
186
187   Avoiding catastrophic forgetting : Adding new experts can be done without modifying the pa-
188   rameters of existing experts, thereby preventing catastrophic forgetting.
189

      2.5   OTHER RESEARCH
191
192   The Branch-Train-Merge (BTM) (Li et al., 2022) method decomposes large language models
193   (LLMs) into multiple independent expert models, which are trained in parallel and then merged
194   for inference. The main goal is to accelerate training. There are two merging approaches: **En-
195   semble**, where all models are run and their results are weighted and averaged, leading to high
196   inference costs; and **Parameter Averaging**, where the parameters of all expert models (ELMs)
197   are weighted and averaged before running, reducing inference costs but leading to worse perfor-
198   mance.
199   The downside is that the merging process often leads to performance degradation. Using different
200   weights for different ELM models can alleviate this issue, but selecting the optimal weights still
201   depends on the specific task and dataset, which is cumbersome.
202
      Branch-Train-MiX (BTX) (Sukhbaatar et al., 2024) is an improvement of BTM, where the merging
203
      process is changed to turn ELMs into experts in an MoE model, adding a token-level routing mech-
204   anism and then fine-tuning. However, the drawback is that averaging parameters across different
205   layers of the ELMs can lead to performance degradation, making it unstable. After merging into
206   MoE, the parameter size increases significantly, requiring much more memory.
207
      BTM, BTX, and IP-LLM differ significantly, mainly in the following aspects:
208
209
210   Difference in Objectives : The primary aim of BTM and BTX is to improve training speed.
211   In contrast, the main goal of IP-LLM is to achieve an infinitely large model, dynamically adding
212
      parameters, avoiding catastrophic forgetting when learning new domain knowledge, and requiring
      minimal retraining.
213
214
215   Architectural Differences : In BTM and BTX, the parameters of different ELM models are inde-
      pendent, with no shared parameters, leading to inefficient use of parameters.

      Under review as a conference paper at ICLR 2025


216
      In IP-LLM, different experts share most parameters, with common parameters for general language
217   skills and basic knowledge, and only a small set of domain-specific parameters differing, making it
218   highly efficient.
219
220
      After training, BTM and BTX require merging, which can cause unpredictable performance losses.
      In contrast, IP-LLM does not require merging and avoids such performance loss.
221
222
      Routing Differences : IP-LLM uses the language understanding ability of large language models
223
      for routing, rather than relying on just a few simple layers of neural networks, resulting in much
224
      higher accuracy.

      5.3     A BLATION STUDY
447
448
      5.3.1    ROUTING S TRATEGY
449
450   Routing at each Transformer layer : In MOE, the routing strategy selects a route at each Trans-
451   former layer, which makes it impossible to predict which expert should be loaded before inference.
452   As a result, all experts need to be loaded into memory. Given the small memory size of current
453   devices, this clearly cannot support an infinitely large model.
454
455
      Routing at each token prediction : Before inference, it is possible to predict which expert to
456   load. This allows only the specified expert to be loaded into memory, without the need to load all
457   parameters, enabling models with parameters far larger than the available memory. However, the
458   performance is significantly slowed down due to switching experts for each token.
459
460
461
      Routing at the start of each inference task : Suppose a task requires predicting 100 tokens.
      Routing is only performed once before the first token, after which the expert is switched, and the
462
      same expert is used for predicting the following 100 tokens. This reduces the routing overhead and
463
      expert switching costs to a minimum, making it suitable for infinitely large models.
464
465
466   5.3.2    N UMBER OF L AYERS
467
468
      The number of layers for each category’s parameters depends on the amount of knowledge contained
      in that category. The more categories there are, the less knowledge each category contains, and thus,
469
      fewer parameters are required for each category.
470
471
472   5.3.3    BASE M ODEL
473
474   The more knowledge the base model contains, the fewer parameters the experts need, but the infer-
475
      ence cost increases. In the extreme case, where the base model contains all knowledge, the experts’
      parameters are zero, resulting in the highest inference cost.
476
477   The base model should aim to contain shared knowledge required across all domains. Reducing the
478   amount of knowledge in the base model increases the number of parameters required by the experts,
479   but decreases the inference cost.
