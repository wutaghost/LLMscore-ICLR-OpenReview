009                                                A BSTRACT
010
011            Recent CLIP-like Vision-Language Models (VLMs), pre-trained on large amounts
012            of image-text pairs to align both modalities with a simple contrastive objective,
013            have paved the way to open-vocabulary semantic segmentation. Given an arbitrary
014            set of textual queries, image pixels are assigned the closest query in feature space.
015            However, this works well when a user exhaustively lists all possible visual concepts
016
               in an image, which contrast against each other for the assignment. This corresponds
               to the current evaluation setup in the literature which relies on having access to
017
               a list of in-domain relevant concepts, typically classes of a benchmark dataset.
018
               Here, we consider the more challenging (and realistic) scenario of segmenting a
019            single concept, given a textual prompt and nothing else. To achieve good results,
020            besides contrasting with the generic “background” text, we propose two different
021            approaches to automatically generate, at test time, textual contrastive concepts that
022            are query-specific. We do so by leveraging the distribution of text in the VLM’s
023            training set or crafted LLM prompts. We also propose a metric designed to evaluate
024            this scenario and show the relevance of our approach on commonly used datasets.

      1    I NTRODUCTION
027
028
      Vision-language models (VLMs) such as CLIP (Radford et al., 2021) are trained to align text and
029
      global image representations. Recently, VLMs have been proposed for denser tasks (Zhou et al., 2022;
030   Ghiasi et al., 2022; Li et al., 2022). This includes the challenging pixel-level task of open-vocabulary
031   semantic segmentation (OVSS), which consists of segmenting arbitrary visual concepts in images, i.e.,
032   visual entities such as objects, stuff (e.g., grass), or visual phenomena (e.g., sky). To that end, several
033   methods exploit a frozen CLIP model with additional operations (Zhou et al., 2022; Bousselham
034   et al., 2024; Wysoczańska et al., 2024b;a), or fine-tune the model with specific losses (Xu et al., 2022;
035   Ranasinghe et al., 2023; Cha et al., 2023; Luo et al., 2023; Mukhoti et al., 2023).
036   Most OVSS methods label each pixel with the most probable prompt (or query) among a finite set of
037   prompts provided as input, contrasting concepts with each other. This works well for benchmarks
038   that provide a large and nearly exhaustive list of things that can be found in the dataset images, such
039   as ADE20K (Zhou et al., 2019) or COCO-Stuff (Caesar et al., 2018). However, when given a limited
040   list of queries, these methods are bound to occasionally suffer from hallucinations (Wysoczańska
041   et al., 2024b; Miller et al., 2024). In particular, common setups do not handle the case where only a
042   single concept is queried (Cha et al., 2023; Xu et al., 2022), which results in classifying all pixels
043   using the same concept.
044   To catch such hallucinations, a common strategy consists in using an extra class labeled ‘background’,
045   intended to capture pixels that do not correspond to any visual concept being queried. This extra
046   class is already present in object-centric datasets, such as Pascal VOC (Everingham et al., 2012). It
047   provides an easy, generic concept to be used as a negative query, i.e., to be used to contrast with
048   actual (positive) queries, but to be discarded from the final segmentation. However, the notion of
049   background is not well defined as it is context-dependent, therefore providing suboptimal contrasts.
050
      This strategy also fails when a queried concept (e.g., “tree”) falls in the learned background (which
      commonly encompasses trees).
051
052   In this work, we consider the practical and realistic OVSS task in which only one or a few arbitrary
053   concepts are to be segmented, leaving out the remaining pixels without any prior knowledge of other
      concepts that may occur in an image. We name this setup open-world Given a query, instead of

      2                   R ELATED WORK
104   Open-vocabulary semantic segmentation. VLMs trained on web-collected data to produce aligned
105   image-text representations (Radford et al., 2021; Jia et al., 2021; Zhai et al., 2023) had a major
106   impact on open-vocabulary perception tasks and opened up new avenues for research and practical
107   applications. While CLIP can be used off-the-shelf for image classification in different settings, it
      does not produce dense pixel-level features and predictions, due to its final global attentive-pooling


                                                         2
      Under review as a conference paper at ICLR 2025


108
      (Zhou et al., 2022; Jatavallabhula et al., 2023). To mitigate this and produce dense image-text features,
109   several methods finetune CLIP with dense supervision. Other approaches devise new CLIP-like
110   models trained from scratch using a pooling compatible with segmentation. Their supervision comes
111   from large datasets annotated with coarse captions (Ghiasi et al., 2022; Ranasinghe et al., 2023;
112   Liang et al., 2023; Xu et al., 2022; Liu et al., 2022; Xu et al., 2023; Mukhoti et al., 2023; Cha et al.,
113   2023), object masks (Rao et al., 2022; Ghiasi et al., 2022; Ding et al., 2022) or pixel labels (Li
114   et al., 2022; Liang et al., 2023). However, when models are finetuned, they face feature degradation
115   (Jatavallabhula et al., 2023), or require long training cycles on huge amounts of images when trained
116   from scratch.
117   CLIP densification methods have emerged as a low-cost alternative to produce pixel-level image-text
118   features while keeping CLIP frozen (Zhou et al., 2022; Wysoczańska et al., 2024a; Jatavallabhula
119   et al., 2023; Abdelreheem et al., 2023; Wysoczańska et al., 2024b; Bousselham et al., 2024). The
120   seminal MaskCLIP (Zhou et al., 2022) mimics the global pooling layer of CLIP with a 1 × 1 conv
121   layer. The aggregation of features from multiple views and crops (Abdelreheem et al., 2023; Kerr
122   et al., 2023; Wysoczańska et al., 2024a; Jatavallabhula et al., 2023) also leads to dense features,
123
      yet with the additional cost of multiple forward passes. Some methods (Shin et al., 2022; 2023;
      Karazija et al., 2023) rely on codebooks of visual prototypes per concept, including per-dataset
124
      negative prototypes (Karazija et al., 2023), or leverage self-self attention to create groups of similar
125
      tokens (Bousselham et al., 2024). The recent CLIP-DINOiser (Wysoczańska et al., 2024b) improves
126   MaskCLIP features with limited computational overhead thanks to a guided pooling strategy that
127   leverages the correlation information from DINO features (Caron et al., 2021).
128
129
      Prompt augmentation. Prompt engineering is a common practice for adapting Large Language
130   Models (LLMs) to different language tasks (Kojima et al., 2022) without updating parameters. This
131   strategy of carefully selecting task-specific prompts also improves the performance of VLMs. For
132   instance, in the original CLIP work (Radford et al., 2021), dataset-specific prompt templates, e.g., “a
133   photo of the nice {· · ·}” were devised towards improving zero-shot prediction performance. Although
134   effective, manual prompting can be a laborious task, as templates must be adapted per dataset
135   and sufficiently general to apply to all classes. Afterwards, different automated strategies were
136   subsequently explored, e.g., scoring and ensembling predictions from multiple prompts (Allingham
137   et al., 2023). Prompts can also be augmented by exploiting semantic relations between concepts
138
      defined in WordNet (Fellbaum, 1998) to generate new coarse/fine-grained (Ge et al., 2023) or
      synonym (Lin et al., 2023) prompts. LLMs can be used as a knowledge base to produce rich visual
139
      descriptions adapted for each class starting from simple class names (Pratt et al., 2023; Menon &
140
      Vondrick, 2023). Prompt features can be learned by considering visual co-occurrences (Gupta et al.,
141   2019), a connection between training and test distributions (Xiao et al., 2024), mining important
142   features for the VLM (Esfandiarpoor et al., 2024) or by test-time tuning on a sample (Shu et al.,
143   2022). Most of these strategies have been designed and evaluated for the image classification task,
144   and their generalization and scalability for semantic segmentation are not always trivial. Here, we
145   aim to obtain better prompts for semantic segmentation to separate queried object pixels from their
146   background. We do this automatically without supervision and without changing the parameters
147   of either the text encoder or the image encoder, leveraging statistics from VLM training data or
148   LLM-based knowledge.
149
150   Dealing with contrastive concepts in OVSS. Our contrastive concept discovery is tightly related
151   to background handling in the context of open-vocabulary semantic segmentation, since the standard
152   benchmark datasets for this task, originally designed for supervised learning, use background to
153
      describe unlabeled pixels, for example, to cover concepts outside of the dataset vocabulary. There
      are three main types of approaches to address this problem. The first one is to threshold uncertain
154
      predictions (Cha et al., 2023; Bousselham et al., 2024; Xu et al., 2022) with a given probability value
155
      (Xu et al., 2022; Bousselham et al., 2024) or clip similarities (Cha et al., 2023). The second group
156   of methods leverages the object-centric nature of certain datasets by defining background through
157   visual saliency (Wysoczańska et al., 2024a;b). Finally, a significant body of work addresses the
158   same issue by defining dataset-level concepts either by adding handcrafted names of concepts to the
159   background definition (Lin et al., 2024; Yu et al., 2023; Ranasinghe et al., 2023; Cho et al., 2024)
160   or by extracting visual negative prototypes with a large diffusion model (Karazija et al., 2023). In
161   contrast, in this work, we aim for automatic discovery of contrastive concepts without any prior
      access to the vocabulary used for the annotation of the dataset.

      Visual grounding is the task of localizing within images specific objects from text descriptions.
163   The major instances of visual grounding tasks are referring segmentation that produce pixel-level
164   predictions for one (Hu et al., 2016; Ding et al., 2023; Wang et al., 2022) or multiple target objects
165   (Liu et al., 2023) given a text description, and referring expression comprehension (Chen et al., 2018;
166   Deng et al., 2021; Liao et al., 2020; Liu et al., 2024) that detects objects. Similarly to referring
167   segmentation, we aim to segment specific user-defined objects. In contrast, we do not use supervision
168   to align textual descriptions with object masks and do not focus on text-described relations between
169   objects and mine contrastive concepts to disentangle target objects from the background.

171   3     O PEN - WORLD SEGMENTATION WITH TEST- TIME CONTRASTIVE CONCEPTS
172
173   We consider the following segmentation task: given an image and a set of textual queries characteriz-
174   ing different visual concepts, the goal is to label all pixels in the image corresponding to each concept,
175   leaving out unrelated pixels, if any. Moreover, we want to do so without any prior knowledge of what
176   concepts could be prompted at the test time. That is, not only do we want to be open-vocabulary in
177   terms of the choice of words for querying, but we also want to be open-world, that is, not specialized
178   in a given domain or set of categories. For evaluation purposes, segmenting a specific dataset thus
179
      shall not assume anything about the dataset, such as knowledge of represented classes.
180
181   3.1   I NTRODUCING THE USE OF TEST- TIME CONTRASTIVE CONCEPTS
182
      Closed-world vs open-world open-vocabulary semantic segmentation. Even when it is open-
183
      vocabulary, traditional semantic segmentation is closed-world in the following sense. Given an
184   RGB image I ∈ RH×W ×3 and a set of textual queries q ∈ Q, semantic segmentation yields a map
185   Sclosew : {1...H} × {1...W } 7→ Q, where each image pixel has to be assigned one of the queries
186   as a label. In contrast, open-world segmentation considers an extra dummy label ‘⊥’ to represent
187   any visual concept that is different from the queries. The segmentation map, in this case, is then
188   Sopenw : {1...H} × {1...W } 7→ Q ∪ {⊥} . For instance, to label a boat, it is enough to ask for the
189   “boat” segment; other pixels (sky, sea, sand, rocks, trees, swimmers, etc.) are expected to be labeled ⊥
190   and thus ignored.
191   Following, we show how to use any open-vocabulary segmenter in an open-world fashion. We only
192   assume that the segmenter uses a CLIP-like architecture with a text encoder, noted ϕT (·), used to
193   extract textual features ϕT (q) ∈ Rd for any query q, where d is the feature dimension. Patch-level
194   features ϕV (I) ∈ Rh×w×d are generated using the visual encoder, noted ϕV (·), where h = H/P ,
195   w = W/P , and P is the patch size. The cosine similarities between each query feature and a
196   patch feature are then used as logits when upsampling to obtain pixel-level predictions. It yields a
197   closed-world segmentation, given our definition above.
198   From such segmentation, open-world segmentation could be derived by assigning a pixel (or patch)
199   to a query if the cosine similarity between the visual embedding and the query embedding is above a
200   given threshold. However, in practice, it has been commonly observed that the CLIP space is not
201   easily separable (Miller et al., 2024), thus making the definition of such a threshold difficult without
202   overfitting the query or datasets (Bousselham et al., 2024; Cha et al., 2023).
203
204   Train-time contrastive concepts. Cues to separate visual concepts without supervision primarily
205
      come from data where these concepts occur separately and are described in their captions. If some
      concepts always co-occur, they are harder to be told apart. This applies in particular to OVSS models
206
      that are trained only from captioned images, rather than from dense information. Sharing a caption
207
      pushes their embedding to align on a common textual feature, which in turn tends to bring the visual
208   embeddings closer together. Still, such frequently co-occurring visual concepts can often be separated
209   in a closed-world setting: pixels (or patches) are then just mapped to the query with which they align
210   the most. However, a problem arises if a visual concept of a query q can be mistaken for another
211   visual concept present in the image but not queried (e.g., querying “boat” but not “water” as in Fig. 1).
212
213   Test-time contrastive concepts. To address this problem, we propose to use one or more additional
214   textual queries of visual concepts that are likely to contrast well with q. For instance, when querying
215   “boat”, we want to add the query “water”. We name such queries test-time contrastive concepts
      and note them CC q . We further propose different solutions to automatically generate CC q , and such

216
      without assuming prior access to the image domain. Given prompt queries {q} ∪ CC q , we perform
217   closed-world segmentation and assign to the dummy label ⊥ any patches which are labeled as CC q .
218
219   Multi-query segmentation. This principle can be generalized to several S    simultaneous queries Q,
220   with |Q| > 1, considering the union of their contrastive concepts CC Q = q∈Q CC q . Open-world
221   multi-query segmentation consists in segmenting Q ∪ CC Q , and ignoring pixels not assigned to the
222   queries in Q, as in the single-query case. However, some queries in Q may already contrast each
223   other, which puts them in competition with the set of contrastive concepts CC Q and could lead to
224   their elimination when pixels labeled in CC Q are discarded. To prevent it, we propose to exclude
225   contrastive concepts CC Q that are too similar

226   features above some threshold β: CC Q = q∈Q {q ′ ∈ CC q | ϕT (q ′ ) · ϕT (q) ≤ β}. In the following,
227   for simplicity, we only consider the single-query scenario, where |Q| = 1.
228   Moreover, to the best of our knowledge, none of the currently used evaluation benchmarks for OVSS
229   allows us to measure the effectiveness of such CCs. We, therefore, propose a variant of the traditional
230   evaluation metric for semantic segmentation and discuss it in detail in Sec. 4.1.

      3.2     C ONTRASTING WITH “ BACKGROUND ” (CC BG )
233
234   In recent work (Ranasinghe et al., 2023; Wysoczańska et al., 2024a;b), the word “background” has
235   been used to try to capture a generic visual concept to help segment foreground objects, separating
236   them from their background. In our framework, it amounts to defining “background” as a test-time
237   contrastive concept to any query q. In other words, it defines CC BG
                                                                        q  = {“background”}.
238
      However, if the word “background” feels natural to us, it is not obvious why it should also make
239   sense in the CLIP space. In fact, this formulation is not contextual, meaning that the contrastive
240   concept is not specific to the query, which might be suboptimal. Worse, the “background” samples
241   that CLIP learned from could accidentally include the visual concept of the query, which could make
242   the query representation close to the background representation and defeat the contrast mechanism.
243
      To sort it out, we investigate the occurrence of “background” in VLM training data. First, we use the
244
      metadata provided by Udandarao et al. (2024), which describes the representation of four thousand
245   common concepts in LAION-400M (Schuhmann et al., 2021), which is a subset of the web-crawled
246   LAION-2B dataset (Schuhmann et al., 2022) used to train CLIP. In Fig. 2a, we plot the frequency
247   of occurrence of “background” among other VOC class names. We observe that “background” is
248   significantly more frequent than all other words, hinting that it is widely available not only in CLIP
249   training data but also in web-crawled data in general.

283   3.3     AUTOMATIC CONTRASTIVE CONCEPTS (CC) GENERATION
284
285   To generate contrastive concepts that are query-specific but also domain-agnostic, the only data we
286   can then leverage are (i) the VLM’s training data, or (ii) unspecific external data. As we focus on
287   text-based contrasts, we can (i) exploit the large vocabulary of concepts used for VLM training,
288
      or (ii) generate prompts via an LLM. Finally, as we want good contrasts, we have to find hard
      negatives. These are concepts that surround queries in images. To gather them, we can (i) look for
289
      word co-occurrences in training data, or (ii) ask an LLM to list such concepts. Sec. 3.3.1 investigates
290
      option (i), and Sec. 3.3.2, option (ii).
291
292
      3.3.1    M INING CO - OCCURRENCE - BASED CONTRASTIVE CONCEPTS (CC D )
293
294   As discussed above, ambiguity in segmentation for unsupervised approaches arises from co-
295   occurrences in training data. Yet, OVSS does a better job when being prompted to create segments
296   simultaneously for co-occurring concepts. To list contrastive concepts specific to a given query q, we
297   thus propose to use information of co-occurrence in the VLM training captions. For efficiency, we
298   construct offline a co-occurrence dictionary, built for a large lexicon of textual concepts extracted
299   from the captions. We note CC D  q the co-occurrence-based contrastive concepts we extract for a
300   query q based on this lexicon.
301
302   Co-occurrence extraction. We consider as lexicon a set of textual concepts T extracted from
303   captions of the VLM training dataset and construct the co-occurrence matrix X ∈ N|T |×|T | . Con-
304   cretely, two concepts {i, j} ⊂ T co-occur if they appear simultaneously in the caption of an image.
305
      Xi,j counts the number of times concepts {i, j} co-occur in some images. Next, we normalize the
      symmetric matrix X row-wise by the number of occurrences of concept i in the dataset, producing the
306
307
      frequency matrix X̂. We then consider only concepts with frequent co-occurrences: for each i ∈ T ,
308
      we select concepts Ti = {j ∈ T | X̂i,j > γ}, for some frequency threshold γ . Selecting only a few
      contrastive concepts in this way is also consistent with the fact that we target online segmentation:
309
      we need to be mindful of the computational costs.
310
311
      Concept filtering. To improve the quality of selected contrastive concepts Ti , we design a simple
312
      filtering pipeline. For each target concept i ∈ T (which can be thought of as a future query), we
313
      remove from Ti any concept that might interfere with i and induce false negatives. First, we discard
314   uninformative words that appear in captions: {“image”, “photo”, “picture”, “view”}. Then, we
315   remove abstract concepts, such as “liberty”. To do so, we ask an LLM whether a given word can
316   be visible or not in an image (more details in Appendix C.2). We also filter out concepts that are
317   too semantically similar to target concept i, e.g., such that their cosine similarity with ϕT (i) is more
318   than a threshold δ . We also consider an alternative approach to filtering, which uses the structured
319   ontology WordNet (Fellbaum, 1998) to remove the CCs that possibly interfere with q. However, our
320   experiments, which are discussed in Appendix B.2, show that our proposed filtering mechanisms
321   based on dataset statistics are more effective.
322
323   Generalization to arbitrary concepts. So far, we discussed how to select contrastive concepts

345   4.1     E VALUATING OPEN - WORLD SEGMENTATION
346
347
      We discuss here our evaluation protocols and present our new metric IoU-single specifically designed
      to evaluate open-world segmentation.
348
349
      Evaluation datasets. We conduct our experiments on six datasets widely used for the task of
350
      zero-shot semantic segmentation (Cha et al., 2023), fully-annotated COCO-Stuff (Caesar et al.,
351
      2018), Cityscapes (Cordts et al., 2016) and ADE20K (Zhou et al., 2019) and object-centric VOC
352   (Everingham et al., 2012), COCO-Object (Caesar et al., 2018) and Context (Mottaghi et al., 2014),
353   when considering “background” pixels. We treat the input images following the protocol of (Cha
354   et al., 2023), which we detail in Appendix A.
355
356   Our IoU-single metric. To better evaluate the ability of a method to localize a visual concept
357   when given no other information, we propose the IoU-single metric. It modifies the classic IoU by
358   considering each concept independently and then averaging. Concretely, we individually segment
359   each class annotated in the dataset for the considered image, thus with |Q| = 1. The IoU-single is
360   then the average of each IoU with the corresponding ground-truth class segment. In Appendix A.3,
361   we give a more intuitive illustration of our metric in Fig. 6 and a pseudo-code in Algorithm 1. If a
362   dataset contains a background class, we do not consider it in the mIoU calculation.
363
364   Classic mIoU evaluation. We also evaluate the impact of using our CC in the classic mIoU scenario
365
      on the datasets that consider “background” as a class, i.e., VOC and COCO-Object. We prompt at
      once all dataset classes together with their CCs, using our multiple-query strategy discussed in 3.1.
366
      We then assign pixels that fall into any of the CCs to “background”, ensuring that none of the concepts
367
      competes with the dataset queries. It allows us to verify if our CCs can act as background without
368   hurting the performance on foreground classes.

531   5               C ONCLUSION
532
533
      In this work, we identify limitations of the current evaluation setup for open-vocabulary semantic
534   segmentation tasks, which are inherited from close-world evaluation benchmarks. To bridge the
535   gap between close- and open-world setups, we propose the single-class segmentation scenario. We
536   study the limitations of current state-of-the-art models when we assume no prior access to in-domain
537   classes and propose to automatically discover contrastive concepts CC that are useful to better
538   localize any queried concept. To do so, we propose two methods leveraging either the distribution
539   of co-occurrences in the VLM’s training set or an LLM to generate such CC. Our results show the
      generalizability of our proposed method across several setups.

      A.2     BACKGROUND HANDLING OF BASELINES
786
787
      We detail here the different strategies employed in the methods that we evaluate to handle the
788   background.
789
790         TCL (Cha et al., 2023) applies thresholding and considers pixels with maximal logit ≤ 0.5 to
791            be in the background, where the logits are the cosine similarities of the visual embedding
792            with the embedding of queries.
793
            GEM (Bousselham et al., 2024) applies a background handling strategy only for Pascal VOC.
794
              It only predicts the foreground classes. The background is obtained by thresholding the
795
              softmax-normalized similarity between the patch tokens and the text embedding of each
796           class name. The threshold is fixed (set to 0.85). In our experiments with VOC, we explore
797           the performance of GEM both with and without background handling and report each time
798           the better score. For other datasets than VOC, we apply only our methods.
799
            MaskCLIP (Zhou et al., 2022) does not use any dedicated mechanism for background. There-
800
               fore, we do not report the original setup for it.
801
802         CLIP-DINOiser (Wysoczańska et al., 2024b) leverages a foreground/background saliency
803            strategy which focuses on foreground pixels. In that case, the foreground/background is
804            defined following FOUND (Siméoni et al., 2023), which focuses on objectness and mainly
805
               discards pixels corresponding to stuff-like classes, which might also be of interest.
806         CAT-Seg (Cho et al., 2024) does not apply any background handling strategy. Instead, for VOC
807            they create a list of potential background classes and use them as "dummy" classes. This
808            approach is the closest to what we propose in our work. In practice, for the VOC dataset,
809            the authors use class names from the Context dataset, an extension of VOC with +40 class
               names.

813             Y – ground-truth annotations of I: gt ∈ NH×W ×1

817
      procedure IoUsingle(I, Y ):
818      // Get unique classes from Y
819      gtcls ← unique(Y)
820      scores ← ∅
821      for i ∈ gtcls do
822         q ← Ti
823         // Text prompts include query q and contrastive concepts of q
824         tq ← q ∪ CCq
825         // Get model predictions for given prompt set
826
            ŷ ← model(I, tq )
            // Get binarized version of predicted mask
827
            ŷ ← binarize(ŷ, i)
828
            // Get ground-truth binary mask for gt class i

830         // Record corresponding IoU
831         scores ← scores ∪ IoU(ŷ, y)
832      end for

1047   B.5              L IMITATIONS
1048
1049   In many images, the objects of interest are surrounded by unrelated other objects or stuff. Contrastive
1050   concepts are then of little help, and the objects of interest are reasonably well segmented by only
1051   contrasting with “background”. This is, in particular, the case for object-centric datasets such as VOC.
1052
       Differently, in some images, the objects of interest are surrounded by other objects or stuff that are
       frequently co-occurring in the training set. Using specific contrastive objects helps the segmenter
1053
       to better separate the objects of interest from the rest. Consequently, the average gain of using
1054
       CC is good but possibly moderate. However, in some images and queries, the specific gain can be
1055   particularly high.

1127   query concepts. Note that in CC L , we explicitly do it in the prompt itself (Fig. 10). Fig. 11 presents
1128   one of such examples when removing “wheel” from the CC D of query “bicycle” gives a slight
1129   improvement for MaskCLIP segmentation. However, we do not notice a particular improvement in
1130   the case of other segmentation methods since, typically, they refine the masks or feature maps to
1131   include localization priors. For example, in Fig. 11, the second row presents the same example for
1132   CLIP-DINOiser (DINOiser), where the improvement is marginal. Finally, we observe little or no
1133   quantitative improvement when applying part removal filtering on entire datasets. Therefore, we do
       not include it in our final method.
