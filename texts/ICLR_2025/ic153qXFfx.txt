009                                               A BSTRACT
010
011
               Evaluating instruction following capabilities for multimodal, multi-turn dialogue
012
               is challenging. With potentially multiple instructions in the input model context,
013
               the task is time-consuming for human raters and we show LLM based judges are
014            biased towards answers from the same model. We propose MMMT-IF, an image
015            based multi-turn Q&A evaluation set with added global instructions between ques-
016            tions, constraining the answer format. This challenges models to retrieve instruc-
017            tions dispersed across long dialogues and reason under instruction constraints. All
018            instructions are objectively verifiable through code execution. We introduce the
019            Programmatic Instruction Following (PIF) metric to measure the fraction of the
020            instructions that are correctly followed while performing a reasoning task. The
021            PIF-N-K set of metrics further evaluates robustness by measuring the fraction of
022
               samples in a corpus where, for each sample, at least K out of N generated model
               responses achieve a PIF score of one. The PIF metric aligns with human instruc-
023
               tion following ratings, showing 60 percent correlation. Experiments show Gemini
024
               1.5 Pro, GPT-4o, and Claude 3.5 Sonnet, have a PIF metric that drops from 0.81
025            on average at turn 1 across the models, to 0.64 at turn 20. Across all turns, when
026            each response is repeated 4 times (PIF-4-4), GPT-4o and Gemini successfully
027            follow all instructions only 11% of the time. When all the instructions are also
028            appended to the end of the model input context, the PIF metric improves by 22.3
029            points on average, showing that the challenge with the task lies not only in follow-
030            ing the instructions, but also in retrieving the instructions spread out in the model
031            context. We plan to open source the MMMT-IF dataset and metric computation
032            code.

      1   I NTRODUCTION
036
037
038   Despite the significant success of Large Foundation Models (LFMs) (Team et al., 2024), (Open
039
      AI, 2024), (Anthropic, 2024), (OpenAI et al., 2024) instruction following is still a challenging task
      (Zhou et al., 2023a). This challenge becomes more pronounced when there are multiple instructions
040
      spread out over several turns in a chat setting between a user and a LFM, where the model needs to
041
      reason over various turns of the conversation. While there are several instruction following evalua-
042   tion datasets, for example (Zhou et al., 2023a), (Zhang et al., 2024), these evaluations are usually
043   single turn and most often use text input. Another key challenge is developing objective evaluation
044   criteria for instruction following. In collecting human annotated reference answers for our evalua-
045   tion dataset, annotators reported that, at each answer turn, rewriting the answer to follow all given
046   instructions took 10 minutes on average, highlighting that human evaluation is time intensive. Re-
047   cent developments have suggested using LLMs as judges of answer quality, but we found that there
048   was a bias in the LLM judge to favor responses coming from the same model.
049   A new development has been to create tasks where model answers can be programmatically checked,
050   in the domains of coding (Yang et al., 2023), data science (Huang et al., 2022), and text (Dong et al.,
051   2024), ensuring an objective evaluation. Among these, (Dong et al., 2024) also focus on instruction
052   following, but only in the single turn, single modality setting. Current chat use cases are often multi-
053   modal and multi-turn, showing the need for objective instruction following evaluation datasets in this
      domain.

054
      To address some of these limitations, we propose an instruction following benchmark, MMMT-IF,
055   along with new metrics for multi-modal multi-turn dialogue. Our proposal extends the MMDU
056   evaluation dataset (Liu et al., 2024c), a multi-modal, multi turn chat task with independent question
057   turns. The extension adds instructions to constrain the answer format in between questions in the di-
058   alogue. All instructions are chosen so that the correctness of a response can be verified through code
059   execution, enabling an unbiased and automated evaluation of instruction adherence. The instruc-
060   tions are global within a chat, meaning that all instructions from previous turns needs to be followed
061   for future turns. Each instruction is chosen from separate categories (for example, one category
062   dictates the start character for answer sentences, and another category dictates the end character
063
      for answer sentences), all the categories are independent from each other. Each category has either
064   2 or 3 instruction options. Before each question, with probability 1 − # Instruction6 given so far another
065
      instruction is added, uniformly at random chosen from a category (in total there are 6 categories)
      not yet added. This challenges the models as the task requires long context reasoning and retrieval
066
      of the instructions from different chat turns, creating a dataset that not only measures single turn
067
      instruction following performance, but also how well a model can follow multiple instructions given
068   throughout a conversation, a common chat use case. The task is not particularly challenging for
069   human raters, who follow on average 94% of given instructions at each turn when writing reference
070   answers for the MMMT-IF evaluation dataset.
071
      We develop two metrics to measure instruction following capabilities of different models: Program-
072
      matic Instruction Following (PIF), the fraction of given instructions in the chat that are followed
073
      at a certain turn, and (PIF-N-K), to stress test the ability of the models to consistently generate
074   responses that follow all the instructions. To compute the PIF-N-K metric at a turn, we generate N
075   responses, and the PIF-N-K metric is the fraction of the responses where at least K of the response
076   candidates at a given turn follow all instructions, i.e. has PIF metric of 1.
077
      We show that the evaluation suite is challenging for the models with evaluate it on: Gemini 1.5
078
      Pro (Team et al., 2024), Claude 3.5 Sonnet (Anthropic, 2024) and GPT-4o (Open AI, 2024), with
079
      a significant loss in performance both over multiple turns and over multiple given instructions, as
080   measured by the PIF metric. The average PIF across the models at turn 1 is high at 0.81, while
081   at turn 20, it declines to 0.64. We develop a more nuanced measure for the model performance by
082   considering the empirical CDF of the PIF score at each question turn. Interestingly, the empirical
083   CDF for Sonnet 3.5 is stochastically dominated by the empirical CDF for Gemini 1.5 Pro for all
084   question turns. This means that not only is the average PIF score better for Sonnet at each question
085   turn, it’s also true that P (PIFSonnet (X, Y ) > x|turn = i) ≥ P (PIFGemini (X, Y ) ≥ x|turn = i)
086   for all x ∈ [0, 1], and for all turns i, for any model input context X and model response Y in the
087   evaluation set at turn i, and P is the empirical measure from all samples in the MMMT-IF evaluation
088
      set.
089   A similar pattern is seen when conditioned on the number of given instructions. Conditional on hav-
090   ing given 6 instructions, the best model in our benchmark, Sonnet 3.5 has a PIF score of 0.74, and
091   Gemini 1.5 Pro has a PIF score of only 0.4. This is in stark contrast to the PIF metric conditional
092   on 1 instruction given, where Gemini 1.5 Pro has an average PIF score of 0.68 and Sonnet 3.5 has
093   an average PIF score of 1 on the evaluation dataset.
094   For the PIF-4-K metric, the PIF-4-4 metric is only 11% for both Gemini 1.5 Pro and GPT-4o, and
095   28% for Claude 3.5 Sonnet, showing that all models fail to robustly follow all given instructions
096   correctly.
097
      We show that a significant part of the challenge with the evaluation set is not following the in-
098   structions, but rather retrieving the instructions from the model context and then reasoning over
099   the instructions. When all instructions are added in the end of the model input context in addition
100   to the model context, the average PIF increased 22.3 points across all models, with Gemini 1.5
101   Pro improving from 0.473 to 0.739, GPT-4o from 0.647 to 0.856, and Sonnet from 0.771 to 0.974,
102   highlighting that in addition to following the instructions, retrieving the instructions from the input
103   model context remains challenging. This highlights similarities with tasks such as multiple needles
104   in a haystack, where the needles are instructions that needs to be reasoned over. Furthermore, our
105   most challenging metric, the PIF-4-4 metric, showed an average improvement of 27 points, from
106
      an average of 0.16 across all models to an average of 0.43 when all given instructions were added in
      the end of the input model context.

524   6    C ONCLUSION
525
526
      In this work we proposed the MMMT-IF instruction following evaluation set for multi-modal, multi-
527
      turn dialogue, along with several challenging metrics for the current LFMs. All the metrics are
528   objective and verifiable by code execution, ensuring an unbiased evaluation. Our analysis shows
529   that the main difficulty of the task lies not within the instruction following, but rather to retrieve the
530   instructions from different parts of the input context and then reason over them. We find that Gemini
531   1.5 Pro consistently, and to a lesser extent also GPT-4o and Claude 3.5 Sonnet, have a significant
532   PIF metric degradation on the turns in the evaluation set with many instructions. We also find that
533   all examined models have low PIF-N-K scores, indicating that they fail to robustly follow all given
534   instructions correctly. We hope that the PIF and the PIF-N-K metric can serve more broadly for
535   practitioners wishing to create other evaluation benchmarks for multimodal, multi-turn instruction
536
      following.
537   Possible directions for future work include creating training datasets for Reinforcement Learning
538   from Execution Feedback (RLEF) with the PIF and PIF-N-K as reward signals. Another possible
539   extension include creating dependent instructions, such as having instructions that modify previously
      given instructions. This will make the task further more challenging.

687   A     L ITTERATURE R EVIEW
688
689   A.1   L ONG CONTEXT R ETRIEVAL
690
691
      There have been several works focusing on the effect of long input context on model performance
      on downstream tasks, including (Liu et al., 2023), (Levy et al., 2024), and (An et al., 2023). Similar
692
      to the Lost-in-the-middle paper Liu et al. (2023), our paper examines the effect of where in the input
693
      context information is located. The results in (Levy et al., 2024) are also complementary, as both
694   observe performance degradation with input context length increases. Our evaluation set can also
695   be viewed as a task similar to multiple needles in the haystack (a task where several tokens needs to
696   be retrieved from a long input context), where each needle is an instruction that the model needs to
697   reason over.
698
699   A.2   I NSTRUCTION F OLLOWING E VALUATION
700
701   There a many instruction following evaluation benchmarks, several are, like our work, focusing on
      instructions related to answer constrains on a Q&A task, (Xia et al., 2024), (Zhou et al., 2023a),


                                                       13
      Under review as a conference paper at ICLR 2025


702
      (Zhang et al., 2024), (Tam et al., 2024) and (Sun et al., 2024a). Compared with these works, we
703   focus on multiple instructions spread out over a long context, testing not only instruction following
704   but also retrieval and complex reasoning from the input context. There have been a long range of
705   other instruction following evaluation sets (Chen et al., 2024c), (Zhou et al., 2023b), (Adlakha
706   et al., 2024), (Sun et al., 2024b), (Yan et al., 2024), (Jiang et al., 2024), (Chia et al., 2023),
707   (Skopek et al., 2023), and (Qin et al., 2024), but their focus in not on multiple instructions spread
708   out in the input context for multi-modal multi-turn chat.
709
710   A.3   P ROGRAMMATIC I NSTRUCTION F OLLOWING
711
712   There have been several previous papers that use program execution to determine instruction follow-
713   ing capability, for code (Yang et al., 2023), Data science (Huang et al., 2022), and text (Dong et al.,
714   2024). Our work is most related to (Dong et al., 2024), but we fix a set of instructions, and instead of
715   a single instruction use case, we focus on multiple instructions, over multiple turns of multi-modal
716   question answering.
717
718   A.4   M ULTI - MODAL E VALUATION DATASETS
719
720   There have been several benchmarks suggested for multi-modal models, for the multi-turn chat use
721   case, we have (Liu et al., 2024c) and (Liu et al., 2024a). However, while the datasets are multi-turn,
722
      the chat turns can be independently answered, thus making it less relevant for long context models.
      By introducing given at several locations throughout the chat, we introduce long range dependencies
723
      in the data needed to answer questions. Our evaluation set is an extension of the MMDU dataset (Liu
724
      et al., 2024c). Other work for evaluating multi-modal models include (Yue et al., 2024), (Liu et al.,
725   2024b), (Srinivasan et al., 2021), (Yu et al., 2023), (Xu et al., 2023), (Chen et al., 2024b) and (Wang
726   et al., 2024). None of these focus on multi-turn instruction following.
727
728
      A.5   LLM JUDGES
729
730   There have been several previous works on using LLMs to judge quality of other LLM responses,
731   including (Dubois et al., 2024b), (Zheng et al., 2023), (Chen et al., 2024a), (Dubois et al., 2024a),
732   (Zeng et al., 2024), and (Liu et al., 2024c). Compared with these works, our LLM judge approach is
733   different in that we combine an LLM evaluation with a programmatic evaluation through a weighed
734   sum to create final model evaluations.
