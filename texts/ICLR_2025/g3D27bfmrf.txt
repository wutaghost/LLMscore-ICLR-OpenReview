009                                               A BSTRACT
010
011
               With recent advancements in long-context model variants, Large language mod-
012
               els (LLMs) can conveniently process different types of task-related information
013
               by simply converting them into an input sequence, even consisting of over 100K
014            tokens. Though with a simple and unified form, there is still considerable room
015            in leveraging input context effectively and efficiently. In this paper, we propose
016            a simple yet effective CASD (Context-Aware Speculative Decoding) method to
017            boost context usage. CASD is a decoding algorithm that requires no extra train-
018            ing or draft models. It improves not only generation performance but also in-
019            ference efficiency. Experiments on 8 datasets (including question answering,
020            summarization and code completion tasks in LongBench) show that CASD in-
021            creases the average generation score by 3.3 points. CASD achieves a mean ac-
022
               ceptance length of 3.10 and a speed-up ratio of 1.99. Moreover, CASD integrates
               effectively with context compression technology, addressing the issue of exces-
023
               sive memory overhead caused by long contexts. Since CASD directly retrieves
024
               token-level content from the input context to boost the generation accuracy, it
025            can effectively mitigate the possible side-effects of context compression methods
026            when crucial context information is dropped. Our anonymous code is available at
027            https://anonymous.4open.science/r/CASD.

031   1   I NTRODUCTION
032
033   Directly prompting large language models (LLMs) with all relevant task information (e.g., task
034   description, query, relevant documents) has become a common setting when serving LLMs for real-
035   world applications, especially with recent long-context models (Anthropic, 2024; Meta-AI, 2024).
036   However, overly long input context might pose great challenges to LLMs in accurately capturing
037   salient context information and achieving decent generation performance, possibly making LLMs
038   unfollow instruction and output hallucinations (Belyi et al., 2024). Besides, since prefilling long
039
      context input significantly increases memory overhead, many recent works explore prompt com-
      pression methods (Jiang et al., 2023; Xu et al., 2024; Pan et al., 2024) to lower inference costs.
040
      These compression techniques will inevitably drop part of critical context information, leading to
041
      inferior generation performance.
042
043   Speculative decoding (Stern et al., 2018; Leviathan et al., 2023; Cai et al., 2024; Li et al., 2024)
044   algorithms have recently shown excellent performance in lossless decoding acceleration for autore-
045
      gressive models. They generate multiple tokens in a single step by drafting with a small model
      and verifying with the target model. The draft process can act as an interface to access external
046
      information. REST (He et al., 2024) retrieves drafts from a general database. However, the input
047
      context can provide drafts that are more relevant to the query and of higher quality. Inspired by this,
048   we propose a context-aware speculative decoding algorithm with conditional verification to enhance
049   the generation accuracy and efficiency of LMs, which is abbreviated as CASD. The overall design
050   is demonstrated in Figure 1. CASD uses the original context as the draft source and replaces the
051   strict verification mode in conventional speculative decoding with conditional verification, allowing
052   the model to access and utilize the retrieved information for accurate generation. In each decoding
053   step, pieces of the relevant context are verified and accepted at the token level, thus making better
      use of the contextual information.

084   Experimental results in Section 4.2 show that CASD outperforms the baseline for 8 datasets in ques-
085   tion answering, summarization and code completion tasks. With a manually adapted verification
086   condition, the performance of CASD can be further improved, demonstrating great potential for
087   reusing the context at the decoding stage. Experiments in Section 4.4 indicate that CASD can be
088
      well adapted to prompt compression techniques and can make up for the performance loss caused by
      the compression to some degree. CASD improves the average scores by 2.09, 1.00 and 1.39 under
089
      a compression ratio of 3×, 5× and 10× when conducting prompt compression by LLMLingua-2
090
      (Pan et al., 2024). As an external component, the draft pool does not need to be input into the
091   model, so it has no strict length limit. For context within 1M tokens, the draft retrieval time is less
092   than 0.01 seconds. As a result, we can construct the draft pool with full context while inputting the
093   compressed one. Even if some crucial passages were cut out, the draft pool retains the possibility
094   that the LM considers them. Besides, CASD achieves a speed-up ratio of up to 2 times compared to
095   autoregressive decoding, which is detailed in Section 5.1.
096
      To sum up, we introduce a simple yet effective approach called CASD in this paper, which makes
097   better use of the relevant context during generation. Our contributions are as follows:
098
099   • We propose a novel decoding paradigm CASD for LMs, which does not require additional train-
100     ing and draft models. It applies to most scenarios where each request contains a question and
101     the relevant context. CASD reuses the input context during the decoding process. It enhances
102     generation accuracy from the token level, thus making better use of the relevant context.
103
      • The proposed method can be effectively combined with prompt compression methods. While
104
        dealing with the problems caused by long context input, it also alleviates the loss in performance
105     caused by the compression of vital information.
106
107   • CASD improves generation accuracy while reducing inference latency. It increases the average
        score on 8 datasets by 3.3 points and increases throughput by up to 2 times.

      2   R ELATED W ORK
109
110   Long Context Generation Using context to improve generation performance is typical in retrieval-
111   augmented generation (RAG) scenarios. RAG methods (Zheng et al., 2023; Dai et al., 2023; Gao
112   et al., 2023; Fan et al., 2024; Zhao et al., 2024) retrieves relevant documents based on the given input
113   for model reference, thus improving the quality of generation. The naive approach (Ma et al., 2023)
114   applies the search engine as the retriever and directly combines the retrieved documents with the
115   user query as the input for frozen LLMs (e.g., GPT-4 (Achiam et al., 2023)). Most RAG methods
116   (Yoran et al., 2023; Luo et al., 2023; Asai et al., 2023; Melz, 2023; Yan et al., 2024; Wang et al.,
117   2024b) leveraging context on the input side to enhance generation. Self-RAG (Asai et al., 2023)
118
      introduces generating reflection tokens to enable customizing models’ behaviors for different tasks.
      Speculative RAG (Wang et al., 2024b) adopts instruct-tuned draft models to drafting according
119
      to different retrieved documents and uses the target model to pick out the best draft as the final
120
      response. Apart from them, CoG Lan et al. (2023) proposes a encoder-based model architecture to
121   seek suitable text spans from the context during generation. Cao et al. (2024) improves CoG through
122   linguistic heuristics initialization and iterative self-reinforcement.
123
124
      Prompt compression (Li et al., 2023; Jiang et al., 2023; Xu et al., 2024; Pan et al., 2024) methods
      have been proposed to extract crucial information from context and deal with the burden of long
125
      text on models and hardware resources. LLM-Lingua (Jiang et al., 2023) employs a small language
126
      model to conduct iterative token-level prompt compression, which takes into account the conditional
127   dependencies between tokens. LLM-Lingua2 (Pan et al., 2024) treats this problem as a binary
128   classification problem and trains an encoder model as a compressor.
129
130
      Speculative decoding (SD) (Stern et al., 2018; Leviathan et al., 2023; Xia et al., 2023; 2024) adopts
      a ”drafting-verification” pattern to lossless accelerates autoregressive decoding. At each decoding
131
      step, a small model is used to draft the following few tokens based on the current input. Then, the
132
      target model verifies the draft in parallel and accepts tokens that are consistent with the original
133   output. In this way, multiple tokens can be generated in a single step. Some works (Leviathan
134   et al., 2023; Cai et al., 2024; Li et al., 2024) employ independent small models or additional trained
135   modules as the draft models, while others (Saxena, 2023; Fu et al., 2024; He et al., 2024) retrieve
136   drafts from a draft pool. The draft structure has evolved from n-grams (Leviathan et al., 2023; Fu
137   et al., 2024) to draft trees (Cai et al., 2024; Li et al., 2024; Wang et al., 2024a). Among them,
138   REST (He et al., 2024) uses a common public data source to build a draft pool and retrieves the tree
139   structure draft according to the last several tokens of the current sequence at each decoding step.
140
141   3   CASD
142
143   In this section, we introduce CASD, a context-aware speculative decoding algorithm based on con-
144   ditional verification that improves generation performance and efficiency. Our approach applies to
145   scenarios where the input can be separated into a query and a supporting context.
146
      Given a query q and the relevant context dq ((q, dq ) ∈ D) , where D is the whole dataset, our goal
147
      is to make the language model M generate an answer Âq that are as close to the ground truth Aq
148
      as possible based on q and dq . Query q and dq are usually highly correlated. In addition, Aq often
149
      contains some the original phrases in dq .
150
151   Based on the above observations, we argue that the dependence of the content generated by the
152   model on the relevant context can be enhanced to achieve better generation accuracy. We design
153
      CASD, a context-aware speculative decoding algorithm with conditional verification. By retrieving
      high-quality drafts from the relevant context according to the current prompt, CASD enables the
154
      LM to call the original fragment in the context directly under certain conditions. It enhances the
155
      generation performance at a token level.
156

187   layer of T. m is the depth of T. We use the exact-match-based retrieval algorithm proposed by He
188   et al. (2024) to retrieve and construct the draft tree from the draft pool. The tree size changes as the

192
      Conditional verification For standard speculative decoding, tokens in T are verified by position
193
194
      order. For draft token ŷij , it is accepted if it matches the token sampled from p(yi |F(ŷij ), M ). If
195   ŷij is rejected, all its child nodes will be rejected. Through this rigorous verification, the output of
196
      the model at any temperature will be consistent with the vanilla autoregressive decoding. However,
      our goal is to use the relevant context to enhance the reliability of the generated content. An overly
197
      strict verification mode limits model utilization of key information in the relevant context, while
198
      over-reliance on the context may reduce the quality of the output. Therefore, we set a probability
199   threshold δ to balance the model’s confidence in the context and the quality of generation. Draft
200
      token ŷij will be accepted under the following condition:
201
202                             p(ŷij |F(ŷij ), M ) > max(δ, p(⟨eos⟩ |F(ŷij ), M )).                         (3)
203   The probability of ⟨eos⟩ token is taken into consideration to avoid duplicate answers generation.
204   Note that its parent nodes should still have been accepted. Otherwise, it will be rejected uncondi-
205   tionally. In addition, to further ensure the quality of generation, we also consider a mixed condition
206   verification mode, which accepts ŷij if:
207
            p(ŷij |F(ŷij ), M ) > max(δ, p(⟨eos⟩ |F(ŷij ), M )) and     ŷij ∈ T opk(p(yi |F(ŷij ), M )),   (4)
208
209   where T opk(p) return the set of k tokens with the highest probability in distribution p.
210   We argue that using the probability threshold condition is superior to using the top-k condition alone
211   since the output probability directly reflects the model’s confidence in each draft token. Moreover,
212   the verification condition of top-k is not applicable for some extreme cases. For example, suppose
213   the probability of a certain token is close to 1, and the probability of all other words is close to 0,
214   except for the token with the highest probability. In that case, no other token should be accepted. If
215   all tokens have the same probability, then they should all be accepted or rejected at the same time.
      However, the top-k verification condition will only pass individual tokens and reject others.

      Combination with prompt compression Excessively long inputs have been a challenge for com-
217   puting resources and a maximum length of some LLMs, which also affect inference efficiency.
218   However, this is not a problem for the plug-in draft pool. For context with less than 1M tokens,
219   changes in context length have little impact on the retrieval efficiency of the draft pool, which is
220   elaborated in Section 5.1. To cope with the challenges posed by long context issue, we construct the
221   draft pool with full context and input the compressed prompt by LLMLingua-2 Pan et al. (2024).
222   Note that we only compress the context and leave the original question or instruction intact. In
223   this way, we retain the potential for missing critical information caused by prompt compression to
224   influence the model’s output.

      We show the performance of CASD with two different verification strategies: 1) CASD (Fixed):
240
      We set the threshold δ to 0.1 for all experiments, 2) CASD (Mixed): With the threshold set to 0.1,

      6     D ISCUSSION
470
471
      In this paper, we proposed CASD, a context-aware speculative decoding algorithm with conditional
      verification, which enhances the generation accuracy of LMs at the token level. We explored the
472
      potential for leveraging the input context in the decoding stage. CASD increases the probability that
473
      the model adopts the tokens in the relevant context, thus making better use of the given information.
474   It does not introduce additional parameters and training, thus can be conveniently applied to pre-
475   trained LMs. CASD outperforms the baseline in experiments on 8 datasets in terms of generation
476   performance and efficiency. Implementation of CASD in prompt compression scenarios also bene-
477   fits the performance. CASD currently uses heuristic verification rules. If the verification strictness
478   can be adaptively set according to different inputs, the versatility of this method would be further
479   improved. We would like to explore adaptive verification conditions at the request level or token
480   level in future work.
481   Limitations CASD accepts the longest verified draft in each decoding step to pursue inference
482   efficiency. However, this generation paradigm conflicts with the sampling methods commonly used
483   in autoregressive models, such as greedy sampling and temperature sampling. Besides, if the input
484   context contain harmful information, using CASD may lead the model to incorporate it into its
485   responses. Therefore, in practical applications, CASD requires additional safety alignment to ensure
      the security of the language model.
