                                                 A BSTRACT
008
009
010            We present Worldcraft, a hybrid implicit method for generating vast, interactive
011            3D worlds at unprecedented scale and speed by modeling them as exchangeable
012            sequences of latent 3D objects. In contrast to existing methods that produce lim-
013            ited scenes, Worldcraft’s novel approach constructs expansive environments com-
014            prising thousands of elements, extending to over a million objects in seconds, on a
015            single GPU. The resulting created worlds are defined in terms of possessing cer-
016            tain essential properties: Object Individuality, Collective Semantics, and Expand-
017
               ability. To achieve this with both speed and scale, we conceptualize world gener-
               ation as a set generation problem, introducing three key technical innovations: (i)
018
               Hierarchical and Exchangeable Sequence Modeling ensures Object Individuality
019
               while capturing Collective Semantics; (ii) Hybrid Implicit Generation Method en-
020            ables rapid creation of vast worlds, supporting both Scale and Expandability; and
021            (iii) Multi-level Indexing Functions allow efficient manipulation across scales,
022            reinforcing Collective Semantics and enabling on-demand generation for Speed
023            and Expandability. We demonstrate Worldcraft’s capabilities using Minecraft as
024            a test-bed, generating complex, interactive environments that users can explore.
025            However, this approach is applicable to any suitable platform, potentially revolu-
026            tionizing various applications in 3D environment generation.

      1   I NTRODUCTION
055
056   Digital media and simulators often require large scale interactive 3D environments comprised of
057   thousands of individual assets. While there has been recent progress in generating these assets
058
      by combining NeRF-like methods (Mildenhall et al., 2021; Yariv et al., 2021; Poole et al., 2022;
      Jain et al., 2022), generating the world they populate remains an open problem. Current methods
059
      for world generation rely on explicitly labeled graphs (Li et al., 2019), autoregressive approaches
060
      (Ritchie et al., 2019; Paschalidou et al., 2021) or operate on scenes of a fixed size (Tang et al., 2024)
061   which limit their applicabilty to large scale and variably sized environments. More recent frame-by-
062   frame modeling techniques (Bruce et al., 2024; Valevski et al., 2024) sidestep the issue of scale but
063   fail to capture the persistent, explorable nature of a real world.
064
      In this paper, we take the concept of world seriously, drawing a clear distinction between a world and
065
      the scenes generated by existing methods. We define a world as having three essential properties,
066
      largely (and usefully) similar to Minecraft’s world model: (i) Object Individuality: Each element
067   possesses a unique identity (analogous to a single block in Minecraft); (ii) Collective Semantics:
068   Multiple objects aggregate to form larger, semantically meaningful structures (comparable to a "re-
069   gion" in Minecraft); (iii) Expandability: The generation process is dynamic, capable of creating
070   new areas as they are explored. In contrast to existing methods, our approach can generate worlds
071   comprising over a million elements on a single, consumer GPU. Thus richly complex worlds rather
072   than mere scenes are achievable both quickly and efficiently. These requirements, combined with
073   our achievements in scale and speed, preclude all existing generative methods (to the best of our
074   knowledge) in the field of 3D environment generation, without significant modifications.
075   At the core of our technical contribution is casting world generation as a set generation problem –
076   this paradigm shift allows us to create entire worlds at scale and speed, while fulfilling the unique
077   properties in our design of a world. Inspired by Ashcroft et al. (2023) generating complex vector
078   drawings with an implicit set space, Worldcraft models the distribution of the top-level parameter
079   of a hierarchy, rather than directly learning the log-likelihood of a permutation-invariant sequence.
080   We then transform a sampled parameter through a series of index functions into a set representing
081
      the world. This innovative approach enables us to overcome the limitations of existing methods and
      achieve our ambitious goals in world generation.
082
083   It follows that our key technical innovations all directly tackle the challenges of scale, speed, and
084   our defined world properties:
085
            1. Assumption of Exchangeability: We represent worlds as an exchangeable sequences of la-
086
               tent variables, leveraging the General Representation Theorem (De Finetti, 1929; 1970) to
087            implicitly to represent complex joint distributions through top-level parameters of a hierar-
088            chy. This enables us to maintain Object Individuality while simultaneously capturing col-
089            lective semantics, as the hierarchical structure allows for individual elements to be unique
090            yet form coherent, larger structures and the use of latent variables allows for flexibility in
091            the 3D asset representation.
092         2. Hybrid Implicit Generation Method: We combine a prescribed probabilistic model (Diggle
093            & Gratton, 1984; Mohamed & Lakshminarayanan, 2016), specifically a Denoising Diffu-
094            sion Probabilistic Model (Ho et al., 2020), with deterministic subjective mapping functions
095            to efficiently generate large, variable-sized worlds. This hybrid approach is key to achiev-
096            ing both the Scale and Speed requirements, allowing us to generate thousands of elements
097            rapidly on a single GPU.
098         3. Multi-level Indexing Functions: We introduce a system that allows reconstruction of the
099            world hierarchy at multiple levels, enabling flexible generation and manipulation of world
100            subsets. This directly supports Expandability, allowing for dynamic creation of new areas
101
               as they are explored.
102   These contributions collectively ensure that we can generate vast, coherent worlds that meet our
103   key requirements. Our set-based approach facilitates the creation of large-scale environments with
104   Object Individuality, the exchangeable sequence modeling captures Collective Semantics across dif-
105   ferent scales, and the multi-level indexing enables the Expandability needed for truly interactive and
106   potentially infinite worlds. Finally it is important to emphasize that Worldcraft is not a tailored solu-
107   tion to generating Minecraft environments but a flexible framework with a range of design choices
      that enable its application to a wide range of platforms.

      2   3D SCENES : REPRESENTATION AND GENERATION
109
110   Implicit functions represent 3D scenes by capturing their underlying volumetric fields from multi-
111   view images through Neural Radiance Fields (NeRFs) (Mildenhall et al., 2021) and SDFs (Yariv
112   et al., 2021; Zhu et al., 2023). This allows for reconstruction of novel views (Mildenhall et al.,
113   2021) and scene geometry (Yariv et al., 2021), optimized (Zhang et al., 2020a) through voxelisation
114   (Sun et al., 2022) and hashgrids (Müller et al., 2022). Flexibility in implicit representations further
115
      allows extension to dynamic scenes through time-specific deformation in 3D space (Cai et al., 2022;
      Liu et al., 2022a) or by adding a new time dimension altogether (Xian et al., 2021; Shao et al.,
116
      2023) for 4D representation. Despite offering high-fidelity novel view generation (Mildenhall et al.,
117
      2021), implicit radiance fields often suffer from poor surface and geometry reconstruction (Darmon
118   et al., 2022). NeRF like methods have proven powerful building blocks in larger 3D generative sys-
119   tems (Chan et al., 2021; Schwarz et al., 2020; Gu et al., 2021; Liu et al., 2022b; Poole et al., 2022;
120   Chung et al., 2023). Explicit representations for 3D objects popularly as tangible meshes (Liu et al.,
121   2023) accurately capture surface and geometry (Liu et al., 2021). However, extending meshes for
122   3D scenes is non-trivial as complexity quickly accumulates from connected objects and scene hier-
123   archies. Recent works suggest using explicit 3D gaussians (3DGS) (Kerbl et al., 2023) as a flexible
124   representation for complex scenes through anisotropic splatting. 3D Gaussian Splatting offers much
125   faster rendering and higher fidelity novel view estimations than corresponding implicit NeRF-based
126
      approaches. Surface reconstruction from 3DGS scenes is possible with underlying scene SDFs
      (Chen et al., 2023; Lyu et al., 2024) and constrained gaussian formation (Guédon & Lepetit, 2024).
127
      Both explicit and implicit representation of 3D objects allow for a latent representation of the object.
128
129   Scene synthesis is the task of generating realistic 3D scenes of distinct objects often from an exist-
130   ing catalogue of 3D assets. Recent approaches have sought to employ a procedural techniques (Qi
131   et al., 2018; Prakash et al., 2019; Devaranjan et al., 2020; Kar et al., 2019). These require specific
132
      pre-defined rule sets which can be expensive and time consuming to create. An alternative approach
      is to model a scene as a graph(Li et al., 2019; Wang et al., 2019; Zhou et al., 2019; Luo et al., 2020;
133
      Purkait et al., 2020; Zhang et al., 2020c;b; Keshavarzi et al., 2020; Di et al., 2020; Gao et al., 2024),
134
      explicitly modelling the relationship between objects in the scene. Paschalidou et al. (2021) formu-
135   lated this problem as a set generation problem and employed a permutation autoregressive approach
136   to generate small scale scenes. Diffusion models have also been employed to this effect (Tang et al.,
137   2024; Yang et al., 2024). What differentiates Worldcraft from existing approaches is the scale of the
138   set generation problem, the use of a hybrid implicit generation method and a hierarchy that instead
139   of modelling explicit relationships between objects uses parameters to represent semantic concepts.

317   4.2   A SSET ENCODING
318   The first step in our method is encode the 3D asset, in this case a chunk, into a latent representation.
319   To do so we employ a 3D convolutional variational autoencoder. We represent each chunk as two
320   channel cube, with the first channel corresponding to the block ID and the second channel the block
321   state. We train the model with a loss function comprised of three components, a weighted KL
322   regularization term, a cross entropy loss for the block-id channel and a cross entropy loss for the
323   block state channel. Our model was trained for 800k steps with a batch size of 128 and constant
      learning rate of 0.0002.

      4.3     R ECONSTRUCTION
369   Our goal in reconstruction is that given some parameter θk ∈ Θ is to reconstruct the exchangeable
370   sequence that parameter implicitly represents. For a controllable reconstruction process we include
371   the use of an an index set I such that we want to learn some function fα (i, θk ) → xi that maps from
372   i and θk onto x.
373   Indexing: While a Minecraft region is a 32 × 16 × 32 3D grid of chunks, above 8 chunks in y
374   it is almost exclusively empty space. As such we define an index set I = {(x, y, z)| x ∈ X, y ∈
375   Y and z ∈ Z} resulting in a set with a maximum cardinality of 8192. We scale each index to between
376   zero and one. Each value is then passed through a fourier transform to obtain an n-dimensional
377   positional embedding. In this case we are using the indexing of each element in the sequence to
      assign it a position in the region.

399   Alternative Indexing: Minecraft is a dense 3D grid of chunks, for which we know all possible
400   asset positions. This enables us to efficiently explicitly index the set. However in 3D spaces that are
401   continuous this option is less effective as it requires us to either explicitly train the model to assign
402   a latent variable to each possible point in the space or to learn continuous representation. There is
403   however an alternative indexing method that does not require us to know the potential position of an
404   object in advance. If we define our index set in a 1D space and add the position (and any other meta
405   information) of the asset as an output of the model then we are able to not only model continuous
406   3D spaces but also we do not need to train our model on any empty space.

408   4.4 W ORLD G ENERATION
409   The final stage in applying the Worldcraft method is to train a generative model over the parameter
410   space of the latent reconstruction model which then acts as an implicit probability measure over the
411   entire world. To do so we use a Denoising Diffusion Probablistic Model (DDPM) and the same MLP
412   based network architecture as Ashcroft et al. (2023). Once the model is trained we can then perform
413
      the full process by sampling from the model, reconstructing the latent sequence, reconstructing the
      sequence of encoded assets and finally decoding these latent variables into observable data. This
414
      process takes 225s for a world comprised of 144 regions. There is an increased amount of time per
415
      region as the data needs to saved before rendering. A full generated world can be seen in Figure 6.

418   5   C ONCLUSION
419
      We have presented Worldcraft, a novel and flexible framework for generating large scale interactive
420   3D environments that satisfies our three conditions for a world: Object Individuality, Collective
421   Semantics and Expandability. Instead of directly learning the distribution of a large set of objects we
422   learn the distribution of the top level parameter of the hierarchy and then reconstruct the set through
423   a series of mapping functions. By prioritizing worlds over scenes, this potentially revolutionary set-
424   based approach enables us to generate large scale interactive 3D environments comprised of over a
425   million distinct objects on a single consumer grade GPU, with possible applications across a range
426   of platforms.
427   A limitation of Worldcraft is that it requires large amounts of training data. Data scarcity is already
428   a challenge in 3D generation tasks but by representing entire worlds of over a million objects as a
429   single sample for generative model learning the distribution of the top level parameters this exacer-
430   bates the problem. The method was only tested on Minecraft however we argue with suitable index
431   functions and asset encoding it may be applied to a range of different platforms. In future work

      A.1   M INECRAFT
730   On this section we provide specific details about Minecraft and its use in this work. Each world in
731   Minecraft is given a name when created. In the game directory a sub directory with that name will
732   contain a regions sub directory. This directory contains .MCA files. Each .MCA file contains all of
733   the chunks in a region. By geenrating an empty world this sub directory will be empty and we can
734   place our generated data in there to render the data.
735
      Game version: We used Minecraft version 1.12.1 to generate our data. The reason for this is that
736   more modern versions of Minecraft swapped from the block state, block ID system for determining
737   blocks and simply assign a block an ID between 0−4095. We rendered all of our results in Minecraft
738   1.2.1
739
      Rendering: All of the results rendered in this paper were done with BSL shaders on. This is not a
740
      requirement.
741
742   Data Generation: Minecraft generates new terrain within a range of the player in the world. To
743   save the potentially hundreds of hours it would take to exhaustively explore the expansive world we
744   used as a data set we used a world pregenerator mod. This causes the world to generate without the
745
      need for manually moving around the world. It took roughly 24 hours to generate the world.
746   Data extracting: To convert the Minecraft data from the .MCA files into usable data we use the
747   anival-parser python library and save the data numpy arrays, where each array contains the entire
748   region.
