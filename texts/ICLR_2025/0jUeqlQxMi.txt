009                                                A BSTRACT
010
011            Given an input image and set of class names, panoptic segmentation aims to label
012            each pixel in an image with class labels and instance labels. In comparison, Open
013            Vocabulary Panoptic Segmentation aims to facilitate the segmentation of arbitrary
014            classes according to user input. The challenge is that a panoptic segmentation
015            system trained on a particular dataset typically does not generalize well to unseen
016            classes beyond the training data. In this work, we propose a retrieval-augmented
017            panoptic segmentation method that improves the performance of unseen classes.
018
               In particular, we construct a masked segment feature database using paired image-
               text data. At inference time, we use masked segment features from the input
019
               image as query keys to retrieve similar features and associated class labels from
020
               the database. Classification scores for the masked segment are assigned based on
021            the similarity between query features and retrieved features. The retrieval-based
022            classification scores are combined with CLIP-based scores to produce the final
023            output. We incorporate our solution with a previous SOTA method (FC-CLIP).
024            When trained on COCO, the proposed method demonstrates 30.9 PQ, 19.3 mAP,
025            44.0 mIoU on the ADE20k dataset, achieving +4.5 PQ, +2.5 mAP, +10.0 mIoU
026            absolute improvement over the baseline.

029   1   I NTRODUCTION
030
031   Panoptic segmentation (Kirillov et al., 2019) is a computer vision task that combines semantic seg-
032   mentation and instance segmentation. Semantic segmentation (Long et al., 2015) labels every pixel
033
      in an image with a class category, such as ”tree” or ”car.” Instance segmentation (Bolya et al., 2019)
      differentiates between individual objects of the same class (1st car, 2nd car). Panoptic segmentation
034
      unifies these tasks to label every pixel with a class label and identify distinct objects within the same
035
      category with an instance label. This method is valuable in fields like autonomous driving (Feng
036   et al., 2020) and robotics (Milioto & Stachniss, 2019), where detailed scene understanding is cru-
037   cial. A key challenge for traditional panoptic segmentation is the need for highly granular pixel-level
038   data annotation. Lack of data limits the number of possible classes for panoptic segmentation, mak-
039   ing the system closed-vocabulary (Ding et al., 2023).
040
      Open vocabulary panoptic segmentation (Ding et al., 2023; Xu et al., 2023c; Yu et al., 2024) is
041
      an advanced version of the traditional panoptic segmentation task that extends its capabilities to
042   identify and label objects from a potentially unlimited set of classes. Unlike standard panoptic
043   segmentation which relies on a fixed set of known classes, open vocabulary segmentation allows
044   the system to recognize and categorize objects even if they haven’t been specifically included in the
045   training dataset.
046
      Recent methods for open vocabulary segmentation (Ding et al., 2023; Xu et al., 2022b; Liang et al.,
047
      2023; Xu et al., 2023c; Yu et al., 2024) involves a two-stage framework. The first step is to generate a
048   class-agnostic mask proposal and the second step is to leverage pre-trained vision language models
049   (e.g., CLIP (Radford et al., 2021)) to classify masked regions. In this approach, the input class
050   descriptions are encoded with a CLIP text encoder and the masked image region is encoded with a
051   CLIP vision encoder. The masked region is classified based on the cosine similarity of masked image
052   features and class-related text features. CLIP has shown the ability to improve open vocabulary
053   performance because it is pre-trained to learn joint image-text feature representation from large-
      scale internet data. However, the performance of the CLIP vision encoder suffers from a limitation


                                                         1
      Under review as a conference paper at ICLR 2025


054
      when we encode a masked image instead of a natural image. This poor quality of encoded features
055   hurts open vocabulary segmentation performance (Liang et al., 2023).
056
057   In this work, we address the bottleneck mentioned above in the context of open vocabulary panop-
058
      tic segmentation. In order to mitigate the domain shift between the natural image feature and
      the masked image feature, we propose a retrieval-augmented approach for panoptic segmentation.
059
      Specifically, we first use large-scale image-text pairs to construct a feature database with associated
060
      text labels for the masked regions. Then during inference time, the masked region feature extracted
061   from the input image is used as a retrieval key to retrieve similar features and associated class labels
062   from the database. The masked region is classified based on the similarity between the retrieval key
063   and retrieval targets. Since both the retrieval key and retrieval target use a CLIP vision encoder on
064   masked regions, the proposed approach does not suffer from the domain shift between the natural
065   image feature and the masked image feature. We combine this retrieval-based classification mod-
066   ule with the CLIP-based classification module to improve open vocabulary panoptic segmentation
067   performance. Our contributions are as follows:
068
             • We proposed a retrieval-augmented panoptic segmentation approach that tackles the do-
069
               main shift between the natural image feature and masked image feature with respect to the
070
               CLIP vision encoder. The proposed approach can incorporate new classes in the panoptic
071            segmentation system simply by updating the feature database in a fully training-free man-
072            ner. Moreover, the feature database can be constructed from paired image-text data which
073            is widely available for thousands of classes.
074
             • We demonstrate that the proposed system can improve open vocabulary panoptic segmen-
075
               tation performance in both training-free setup (+5.2 PQ) and cross-dataset fine-tuning setup
076            (+ 4.5 PQ, COCO→ADE20k).

081   Fully Supervised Fully supervised methods typically involve training or fine-tuning the system
082   on a dataset with pixel-level annotations (Li et al., 2022; Ghiasi et al., 2022; Xu et al., 2022c; Luo
083
      et al., 2023a). Ding et al. (2023) use a trainable relative mask attention module to produce robust
      masked segment features from a frozen CLIP backbone. Xu et al. (2023a) proposes combining the
084
      internal representation of pretrained text-to-image diffusion models and discriminative image-text
085
      models for open vocabulary panoptic segmentation. Liang et al. (2023) fine-tune a CLIP backbone to
086   improve alignment between text representation and masked image representation. Xu et al. (2023c)
087   use a student-teacher self-training to improve mask generation for unseen classes and fine-tune CLIP
088   to improve query feature representation. Yu et al. (2024) use a frozen CNN-based CLIP backbone
089   for both mask proposal generation as well as classification.
090
091   Weakly Supervised Weakly supervised methods are trained on image-level annotations (Xu et al.,
092   2022a; Liu et al., 2022; Zhou et al., 2022; Xu et al., 2023b). Luo et al. (2023b) train the system on
093   image-text pairs using a semantic group module to aggregate patches with learnable image regions.
094   He et al. (2023) use self-supervised pixel representation learning guided by CLIP image-text align-
095   ment for semantic segmentation. Mukhoti et al. (2023) propose patch-level contrastive learning that
096
      learns alignment between visual patch tokens and text tokens. This approach generalizes to the open
      vocabulary setting without any training on pixel-level annotations. Wang et al. (2024b) combine
097
      the spatial understanding of Segment Anything Model (SAM) (Kirillov et al., 2023) and semantic
098
      understanding of CLIP for open vocabulary semantic segmentation. They use continual learning and
099   knowledge distillation methods to ensure the resulting model retains the capabilities of the original
100   models.
101
102   Training Free Training-free methods typically exploit pretrained models (e.g. CLIP) for open
103   vocabulary segmentation without any fine-tuning on pixel-level or image-level annotations (Wang
104   et al., 2024c; Tang et al., 2024; Wang et al., 2024a). Shin et al. (2022) construct a database of ref-
105   erence image segments using CLIP. During inference, the reference images are used for segmenting
106   relevant segments from the input image. Karazija et al. (2024) generate synthetic reference images
107   using a text-to-image diffusion model and perform segmentation by comparing input images with
      synthetic references. Wysoczańska et al. (2024) encodes small image patches separately to the vi-

147   3.1    C ROSS DATASET PANOPTIC S EGMENTATION
148
149
      In the cross-dataset variant of open vocabulary panoptic segmentation, the system is fine-tuned on
      one dataset (e.g. COCO) and evaluated on another dataset (ADE20k) with some unseen classes. Our
150
      cross-dataset method is based on FC-CLIP (Yu et al., 2024) where a mask proposal generator and
151
      mask decoder are fine-tuned on COCO (Lin et al., 2015). The overview of the system is shown in
152   Figure 1.
153
154   Shared Backbone Similar to FC-CLIP, we use a frozen CNN-based CLIP backbone. The back-
155   bone is shared between the mask generation and segment classification. Yu et al. (2024) have demon-
156   strated that CNN-based CLIP backbone is a more robust variation in image resolution. We use the
157   ConvNeXt-Large variant of CLIP backbones from OpenCLIP (Cherti et al., 2023). The model is
158   trained on the LAION-2B dataset (Schuhmann et al., 2022). The CLIP backbone converts the input
159   image to patch-specific dense features which is used for mask generation and segment classification.
160
161   Mask Proposal Generation The mask proposal generator is based on Mask2former (Cheng et al.,
      2022). A pixel decoder is used for enhancing dense features from the CLIP backbone. The en-


                                                       3
      Under review as a conference paper at ICLR 2025


162
      hanced features and class-related queries are fed to a series of mask decoders. The mask decoders
163   are equipped with self-attention, masked cross-attention, and a feed-forward network. Finally, the
164   segmentation logits are produced via matrix multiplication between class queries and transformed
165   pixel features.
166
167   In Vocabulary Classification The in-vocabulary classification path is shown in green in Figure 1.
168   The dense features are computed from the input image feature and mask proposals using mask
169   pooling. Dense features for masked regions and class name embeddings are projected to the same
170   embedding space using linear projection. The linear projection parameters for in-vocabulary clas-
171   sifiers are fine-tuned on COCO. The classification scores are obtained based on cosine similarity
172   between class embeddings and masked segment features.
173
174   Out-of-vocabulary Classification Via Retrieval The retrieval-based classification path is shown
175   in violet in Figure 1. The retrieval module uses masked segment features as retrieval keys to perform
176   approximate nearest neighbor search in the feature database. The output is a set of distance scores
177   between the retrieval key and retrieval targets and associated class labels. The distance scores are
178
      normalized using min-max normalization and subtracted from one. This step produces retrieval-
      based classification scores. In case any of the user-provided class names are missing in the feature
179
      database, we retrieve image samples for those input classes from a secondary image dataset. The
180
      label matching between datasets is performed with CLIP text embedding of class names with simi-
181   larity score > 0.95.
182
183
      Out-of-vocabulary Classification Via CLIP Similar to FC-CLIP, we have a CLIP-only segment
184   classifier. This is helpful in case the feature database does not have similar features compared to the
185   segment features. The classification is performed using cosine similarity between segment features
186   and class name embeddings. Unlike in-vocabulary classifiers, the features do not go through fine-
187   tuned linear projection layers.
188
189   Ensemble Let’s assume C is the set of classes for prediction and Ctrain is the set of classes in the
190   fine-tuning dataset. Let siclip , siret , siiv be classification scores for class i using CLIP, retrieval and
191   in-vocabulary classifier. The scores from the three classification pipelines are combined as follows,
192   where α, β, γ are hyper-parameters.
193
194
                                 sioov = siret × γ + siclip × (1 − γ)
195                                si = sioov × α + siiv × (1 − α) if i ∈ Ctrain
196                                si = sioov × β + siiv × (1 − β) if i ∈
                                                                        / Ctrain

200   3.2   T RAINING F REE PANOPTIC S EGMENTATION
201
      In training free variant of open vocabulary panoptic segmentation, none of the system components
202
      are fine-tuned on pixel-level panoptic annotations. We use an open vocabulary objection detection
203   model and SAM for mask proposal generation. The segment classification was performed with CLIP
204   and retrieval. The overview of the system is shown in Figure 2.
205
206   Mask Proposal Generation Given an input image and a list of classes, we use Grounding
207   DINO (Liu et al., 2024) to detect bounding boxes associated with each class. All bounding boxes
208   detected with a minimum confidence threshold are retained. The bounding boxes are passed to
209   SAM for generating class-aware masks. The outputs of SAM are used as class-agnostic mask pro-
210   posals. All potential classes for panoptic segmentation are passed to the object detection method
211   and confidence-based filtering is performed to prune absent classes.
212
213   Dense Feature for Masked Regions A CLIP backbone is used to extract dense features from the
214   input image. The mask proposals from the previous step are used to extract masked image regions
215   from the image-level dense features. We use mask pooling to convert image-level dense features to
      region-level dense features.

256   3.3   F EATURE DATABASE C ONSTRUCTION
257
      The objective of the database construction step is to take a paired image-text dataset as input and
258
      convert it into a database of masked segment features and associated class labels. The database
259
      construction has four steps, namely object detection, mask generation, dense feature generation, and
260   mask pooling. The overview of the process is shown in Figure 3.
261
262   Object Detection In this step, an image and class labels present in the image are fed to an open
263   vocabulary object detection method. The output is a bounding box associated with each class present
264   in the image. We use the SOTA open vocabulary object detection method Grounding DINO (Liu
265   et al., 2024).
266
267   Mask Generation In this step, the input image and associated bounding box prompts are fed to
268   SAM (Kirillov et al., 2023) for mask generation. Even though SAM can generate masks without
269   class-aware bounding boxes, the resulting masks often break up a single class (e.g. car) into multiple
      masks (e.g. wheel, car body, window). An example of this phenomenon is shown in Figure 4. The

      5      C ONCLUSIONS
369
      In this work, we exploit a retrieval-based method for improving open vocabulary panoptic segmen-
370
      tation. We construct a visual feature database using paired image-text data. During inference, we
371
      use masked segment features from the input image as query keys to retrieve similar features and
372   associated class labels from the database. Classification scores for the masked segment are assigned
373   based on the similarity between query features and retrieved features. The retrieval-based classifi-
374   cation scores are combined with CLIP-based scores to produce the final prediction. The proposed
375   approach improves PQ from 26.4 to 30.9 on ADE20k when fine-tuned on COCO. Even though the
376   proposed method achieves reasonable performance in an open vocabulary setting, it remains vulner-
377   able to the quality of mask proposal generation. Future work may focus on improving the quality of
      mask proposal generation for unknown classes.
