009                                                A BSTRACT
010
011            Sparse Autoencoders (SAEs) have shown promise in improving the interpretability
012            of neural network activations, but can learn features that are not features of the input,
013            limiting their effectiveness. We propose M UTUAL F EATURE R EGULARIZATION
014            (MFR), a regularization technique for improving feature learning by encouraging
015            SAEs trained in parallel to learn similar features. We motivate MFR by showing
016
               that features learned by multiple SAEs are more likely to correlate with features
               of the input. By training on synthetic data with known features of the input, we
017
               show that MFR can help SAEs learn those features, as we can directly compare
018
               the features learned by the SAE with the input features for the synthetic data.
019            We then scale MFR to SAEs that are trained to denoise electroencephalography
020            (EEG) data and SAEs that are trained to reconstruct GPT-2 Small activations. We
021            show that MFR can improve the reconstruction loss of SAEs by up to 21.21%
022            on GPT-2 Small, and 6.67% on EEG data. Our results suggest that the similarity
023            between features learned by different SAEs can be leveraged to improve SAE
024            training, thereby enhancing performance and the usefulness of SAEs for model
025            interpretability.

      1   I NTRODUCTION
028
029
      Interpretability aims to explain the relationship between neural network internals and neural network
030   outputs. Many interpretability techniques examine raw activations, equating proposed fundamental
031   units of neural networks such as neurons or polytopes to human understandable concepts (Erhan et al.,
032   2009; Nguyen et al., 2016; Bau et al., 2017; Olah et al., 2018; Black et al., 2022). These techniques
033   often benefit from a clean correspondence between those fundamental units and concepts, and may
034   fail if concepts are distributed over many units, or many concepts focused in a single unit, such as
035   in the case of feature superposition (Elhage et al., 2022). We describe features of the input as the
036   atomic, human-understandable concepts represented by input data.
037   To derive a representation of activations with a stronger one-to-one correspondence of features and
038   neurons, sparse autoencoders (SAEs) have been trained on neural network activations. The decoders
039   of SAEs trained on neural network activations have been shown to form dictionaries of features more
040   easily explained than the neurons themselves, making SAEs potentially useful for understanding the
041   internals of neural networks (Bricken et al., 2023; Cunningham et al., 2024; Gao et al., 2024).
042   Despite the recent popularity of SAEs, early results suggest they may learn features that are not
043   features of the input, reducing their usefulness for interpretability (Till, 2024; Huben, 2024; Anders
044   et al., 2024). One failure mode considers transformations on the space of inputs: features of the input
045   that are ‘split’ over multiple decoder weights, or multiple features ‘composed’ in one decoder weight.
046   Conceivably, the representation learned by the SAE could be so varied from the input as to contain
047   features entirely incompatible with the input space. These failures are alarming, as studying an SAE
048   would not be guaranteed to reveal information about the neural network that SAE was trained on
049   activations from. In the worst case, if features were commonly split and composed, it is not obvious
050
      why studying SAEs would be more useful than studying the raw activations directly, although prior
      work has given evidence against this.
051
052   We hypothesize that if a feature is learned by multiple SAEs, that feature is more likely to be a
053   feature of the input, and show that this is true for SAEs trained on synthetic data comprised of known
      features. Based on this result, we encourage multiple SAEs trained on the same data to learn common

      2.1   S PARSE AUTOENCODERS
091
092
      Olshausen and Field (1996) introduced unsupervised learning of sparse representations, capturing
093
      structure in data more efficiently than dense representations. Sparse autoencoders (SAEs) have since
094   found wide application in domains such as representation learning (Coates et al., 2011; Henaff et al.,
095   2011), denoising (Vincent et al., 2010; Duan et al., 2014), and anomaly detection (Sakurada and Yairi,
096   2014; Xu et al., 2015).
097
      SAEs reconstruct an input x ∈ Rd through a hidden representation h ∈ Rh , minimizing the
098                               2
099
      reconstruction loss ∥x − x̂∥2 while maintaining sparsity in h, written as x̂ = W′ σ(Wx + b), where
100
      W ∈ Rh×d is the encoder weight matrix, W′ ∈ Rd×h is the decoder weight matrix, b ∈ Rh is the
      encoder bias, and σ is an activation function on h. Ideally columns of W′ correspond to features
101
      comprising x.
102
103   Recent work has shown that the TopK activation function (Makhzani and Frey, 2013) better approxi-
104   mates the L0 norm in training than alternative techniques such as L1 regularization (Gao et al., 2024),
105   allowing more precise control over the sparsity of h. The TopK activation function on h is defined as:

114   2.2   E VALUATING S PARSE AUTOENCODERS
115
116   Evaluating SAEs is challenging due to the lack of a ground truth for the input features represented
117   by large neural networks. Thus, SAE evaluations act as proxies the extent to which interpretable
118   representation of these input features have been learned in a way that does not require access to them.
119   The reconstruction loss, measured as the Euclidean distance between the SAE input and output, is
120   a widely used metric for the faithfulness of an SAE’s learned representations to the input features
121   represented by a neural network. Although reconstruction loss does not account for the interpretability
122   of the learned features, improved reconstruction loss has previously been accompanied by improved
123   performance on interpretability evaluations (Rajamanoharan et al., 2024a; Gao et al., 2024; Raja-
124   manoharan et al., 2024b). However, the reconstruction loss is not itself sufficient to evaluate SAEs,
125   as there may be solutions to optimizing reconstruction loss that do not preserve the structure of the
126
      input features or convey information about them, such as learning the identity function.
127   The interpretability of SAE features has been evaluated by the ability of humans and language models
128   to accurately describe those features (Bricken et al., 2023; Cunningham et al., 2024; Rajamanoharan
129   et al., 2024b). In this evaluation, humans and language models generate feature descriptions based
130   on token sequences and their corresponding activations. The accuracy of these descriptions is then
131   evaluated by predicting feature activations on unseen tokens. The correlation between predicted and
132
      true activations, typically quantified using the Pearson correlation coefficient, is used as a measure of
      description accuracy. However, recent work has critiqued this method, suggesting that even highly
133
      accurate feature descriptions may not faithfully represent the model being explained (Huang et al.,
134
      2023).
135
136   Alternative SAE evaluations analyze the sparsity of SAE outputs through the L0 norm, the presence of
137   consistently inactive features, and ’loss recovered’. Loss recovered refers to the discrepancy in model
138
      loss between zero ablation of a layer and the insertion of an SAE output as if it were the activations
      of that model. The motivation for loss recovered is that it more directly approximates the information
139
      preserved by the SAE output, as this may not be accurately measured by the reconstruction loss.

143   2.3   S EMI - SUPERVISED LEARNING WITH MULTIPLE MODELS
144
145   Semi-supervised learning with multiple models involves training several models on both human-
146   labeled and model-labeled data. Co-training, a semi-supervised technique, uses two distinct and
147   conditionally independent views of the same data to iteratively improve the performance of two
148   classifiers (Blum and Mitchell, 1998; Zhou and Li, 2005). This approach aims to maximize agreement
149   between the classifiers and has been shown to improve their accuracy (Nigam et al., 2000). Similar
150   techniques have been applied in deep learning for tasks such as machine translation (Xia et al., 2016)
151   and image recognition (Qiao et al., 2018).
152   ‘Mutual learning’ and ‘co-teaching’ have been used to describe techniques where student models
153   trained in parallel teach each other by minimizing the divergence between their predictions. These
154   methods have shown superior performance for the size of the student model compared to distillations
155   of larger models (Zhang et al., 2017; Tarvainen and Valpola, 2017; Han et al., 2018; Ke et al.,
156   2019; Wu and Xia, 2019). Related techniques include temporal ensembling, which improves model
157   robustness by averaging predictions over multiple training epochs (Laine and Aila, 2016), and
158
      fraternal dropout, which encourages models trained in parallel to make similar predictions for the
      same data points, serving as a method of regularization to prevent overfitting (Żołna et al., 2017).
159
160   Our work builds on the semi-supervised learning literature. However, our motivation differs from
161   the motivation for most of these techniques, which often relates to a lack of training data rather than
      learning features not in the input space.

      Following Sharkey et al. (2022), we generate a synthetic dataset of vectors that represent more
165   features than their dimensionality, allowing a direct measurement of how well an SAE learns the
166   features of inputs with superposition. This simplifies our analysis by mitigating the problem of
167   imperfect SAE evaluations (discussed in Section 2.2) as we can then directly compare the SAE and
168   input features, but is only applicable when the input features are known, which is not typical for
169   real-world data.

498   4.3   W EIGHT ANALYSIS
499
500   One concern with MFR may be that in encouraging the SAE decoder features to be similar, the
501   decoder weight matrices end up more similar than without MFR. This could be problematic, as SAEs
502
      with lower values of k for the TopK activation that were trained with MFR alongside SAEs with
      higher values of k could end up less sparse by becoming more similar to the SAEs with higher values
503
      of k. To test this, we measure the L2 distance of the decoder weight matrices for the baseline and
504
      MFR SAEs trained in Section 10.
505
506   We find that SAEs trained with MFR tend to be more different in terms of the L2 distance, but that as
507   k increases they trend toward lower L2 distances (Figure 10). This is in contrast to the baseline SAEs,
508
      which are more similar at smaller values of k, but trend towards larger L2 distances as k increases.
      At the values of k we train at, we do not consider this problematic, as the L2 distances suggest the
509
      decoder weight matrices are more different on average rather than less.

      5     C ONCLUSION
513
514
515   We proposed a method for training SAEs designed to recover more features of the input. We first
516   establish a motivating hypothesis for MFR, that feature similarity across SAEs is correlated with
517
      feature similarity to the input features, showing that this hypothesis is true for SAEs trained on
      synthetic data, and that MFR improves the fraction of features of the input recovered (Section 3). We
518
      then scale MFR to both language model activations and a realistic denoising task, and show that it
519
      improves SAE performance on key metrics at scale (Section 4). We believe our method encourages
520   SAEs to learn more features of the input, increasing their usefulness for interpretability.
521
522
523   L IMITATIONS
524
525   Although we show improved performance of SAEs with MFR, this comes at a relative increase in
526   computational cost, as the auxiliary penalty used in MFR requires training additional SAEs. As
527   all of our experiments are easily completed on a single GPU, this is not problematic in our work.
528   However, larger models can require SAEs with very large hidden dimensions, making this cost
529   unmanageable if SAEs need to be trained for many layers. Training a multiple of the SAEs that
530
      would need to be trained without the auxiliary penalty may not be justifiable depending on the scale
      of the experimentation.
531
532   Despite the increased computational requirements, we believe that the auxiliary penalty is still
533   valuable due to the small computational budget for SAEs relative to the models they are trained on.
534   For smaller models (where the cost of training SAEs is less significant), it may be worth the increase
535   in training compute for more accurate SAEs. For example, in the case of GPT-2 Small, the additional
536
      compute may not be of concern, as training is manageable on a single GPU or a small cluster making
      the additional information about the input features worth the additional compute.
537
538   We hope that future work will investigate efficient mutual learning-based approaches for SAEs
539   that can benefit from the positive relationship between feature similarity across SAEs, and feature
      similarity with the input features without requiring more compute.
