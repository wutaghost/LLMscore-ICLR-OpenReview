009                                                A BSTRACT
010
011
               While conventional Transformers generally operate on sequence data, they can be
012            used in conjunction with structure models, typically SE(3)-invariant or equivariant
013            graph neural networks (GNNs), for 3D applications such as protein structure mod-
014            elling. These hybrids typically involve either (1) preprocessing/tokenizing struc-
015            tural features as input for Transformers or (2) taking Transformer embeddings and
016            processing them within a structural representation. However, there is evidence that
017            Transformers can learn to process structural information on their own, such as the
018            AlphaFold3 structural diffusion model. In this work we show that Transformers
019            can function independently as structure models when passed linear embeddings of
020
               coordinates. We first provide a theoretical explanation for how Transformers can
               learn to filter attention as a 3D Gaussian with learned variance. We then validate
021
               this theory using both simulated 3D points and in the context of masked token
022
               prediction for proteins. Finally, we show that pre-training protein Transformer en-
023            coders with structure improves performance on a downstream task, yielding better
024            performance than custom structural models. Together, this work provides a basis
025            for using standard Transformers as hybrid structure-language models.

      1     I NTRODUCTION
029
030
      1.1   BACKGROUND
031
032
      Transformers typically operate on sequential data, however many applications of Transformers ben-
033   efit from an ability to learn geometric reasoning. For instance, ESM-2 (Lin et al., 2023) demonstrates
034   that in order to effectively predict masked tokens in protein sequences, the model has learned some
035   ability to predict protein structures. Other tasks such as image processing or even natural language
036   processing may benefit from an internal representation of objects in 3D space. However, it is unclear
037   how Transformers can learn to use 3D representations to perform spatial reasoning. To this end, cus-
038   tom structural Transformers have been created which model data as graphs and represent distance
039   between nodes as edge features in order to perform SE(3)-invariant attention (Ingraham et al., 2019;
040   Fuchs et al., 2020; Liao & Smidt, 2023; Liao et al., 2023).
041   SE(3) invariance means that all functions of coordinates reduce to functions of relative distance.
042   That is, for every function f of two coordinates x1 and x2 , there exists an equivalent function g
043   which depends only on their relative distance:
                                               p      f (x⃗1 , x⃗2 ) = g(|x⃗1 − x⃗2 |). Since relative distance
044   in Euclidean space is defined as |⃗x| = x2 + y 2 + z 2 , it may be easier to learn functions of the
045   square of the relative distance — a linear combination of functions of the individual coordinates.
046
      In this manuscript, we investigate how conventional Transformers can learn to approximate func-
047
      tions of the squared distance between points, thereby learning an approximately SE(3)-invariant
048   measure of distance. In short, we show that “out of the box” Transformers can act as 3D structural
049   models. Our main contributions are (1) to provide a theoretical explanation for how standard Trans-
050   formers can learn to measure distance and perform structural reasoning, (2) to show that Transform-
051   ers indeed learn Gaussian functions of distance and investigate efficient data augmentation methods
052   which can be used to learn SE(3), and (3) to train a protein masked token prediction model with
053   coordinates and show that finetuning it for function prediction yields a model which outperforms
      structural GNNs.

      1.2   P RIOR WORK
055
056   There have been prior approaches to merge Transformers with SE(3)-(in/equi)variant models, es-
057   pecially for computational chemistry and 3D point clouds. Some methods add attention blocks to
058   SE(3) GNNs to create SE(3)-invariant GNN Transformers (Fuchs et al., 2020; Liao & Smidt, 2023).
059   These have shown good results on a number of tasks, however tend to be memory-intensive, par-
060   ticularly because attention is performed on edges, which grow as n2 for fully-connected graphs.
061   As a result, the graph connectedness of the GNNs is typically limited to k-nearest neighbours. In
062
      contrast, memory-efficient attention implementations such as FlashAttention (Dao et al., 2022; Dao,
      2023) have enabled linear-memory standard Transformers.
063
064   Previous works have demonstrated that sequence-only protein Transformers can learn attention maps
065   which correlate with physical contacts (Lin et al., 2023; Vig et al., 2020). However, these works do
066   not formally model structure and so are limited by which contacts can be predicted from purely
067   sequential patterns. To overcome this, Transformers are often paired with structural models, for
068
      instance methods such as ProSST (Li et al., 2024), ESM-IF (Hsu et al., 2022), and ESM3 (Hayes
      et al., 2024) use custom graph-based modules to create structural tokens which are fed into standard
069
      Transformers. In contrast, our work shows that standard Transformers are natively capable of using
070
      coordinates to model structure and measure distance.
071
072   Similarly, AlphaFold2 (Jumper et al., 2021) and ESMFold (Lin et al., 2023) use Transformers to
073   preprocess protein sequences for structure prediction. Again, these preprocessed representations
074
      have been shown to correlate with structural contacts. AlphaFold2 makes this explicit during train-
      ing by minimizing a distrogram loss which encourages the EvoFormer to learn structural contacts.
075
      However, it is unclear if these representations are learning to explicitly embed coordinates in 3D
076
      and both models still require an SE(3)-equivariant GNN structure module to actually produce 3D
077   structures.
078
079   In building AlphaFold3, DeepMind replaced AlphaFold2’s SE(3)-equivariant structure module with
080
      linearly embedded coordinates fed into a diffusion transformer (Abramson et al., 2024). Al-
      phaFold3’s structure module uses inner product attention with a pair bias learned from the pair
081
      representation. At present, this still requires quadratic memory, however does indicate that nearly-
082
      standard Transformers with linearly embedded coordinates can learn on structure. Here, we explore
083   how such linearly embedded coordinates can be used by standard Transformer attention modules
084   to measure the Euclidean distance between tokens, and, in contrast to prior work, show that no
085   modifications are necessary for the standard Transformer architecture to learn to perform structural
086   reasoning.

091   Throughout this section, we assume that all coordinates are small. This can be achieved by scal-
092   ing the inputs and outputs manually or via learned weights in the linear input/output maps. For more
093   detail, see Appendix A.1.

      3.1.2                    T RANSFORMERS NEED n + 2 EMBEDDING DIMENSIONS TO LEARN DISTANCE IN Rn
238
      Next, we explored how large a spatial embedding must be to learn a good approximation of distance
239
      for Rn . For the case of 1 spatial dimension, we provide a theoretical 4-dimensional embedding. We
240   trained models to predict Euclidean distance in Rn for n ∈ 1, 2, 3, 4 using head dimensions from 1
241   to 8. Figure 2b shows the relationship between validation loss and head dimension. We found that
242   the model only needs n + 2 head dimensions to learn a good approximation of distance for all spatial
243   dimensions. This means that to learn a good approximation of distance in R3 , a model must reserve
244   at least 5 embedding dimensions for each head. This is a surprisingly compact requirement which
245   should be easily accommodated even in small Transformers.
246
247   3.1.3                    SE(3) TRANSFORMATIONS IMPROVE LEARNED SE(3)- INVARIANCE
248
249   We investigated whether Transformers will learn to overfit training data in a low data regime and if
250   this can be prevented. This could also correspond to a scenario where there is only a strong structural
251   signal in a small number of training examples. We reduced the number of training points to 100 and
252   measured the training and validation loss. To test the importance of data augmentation, we trained
253   models with and without the random rotations (Figure 3a). The raw coordinates clearly demonstrate
254   overfitting while the randomly rotated coordinates show near perfect alignment between training
255
      and validation loss. Importantly, this form of data augmentation does not require creating new data
      points, only rotating the training data each epoch.
256
257   We measured the average ℓ1 distance between predictions of randomly rotated structures for the
258   models trained with and without random rotations, as a measure of SE(3) divergence (Figure 3b). In
259   both cases the SE(3) divergence was almost the same as the validation loss, indicating that randomly
260   rotating training structures reduces overfitting by encouraging models to learn an SE(3)-invariant
261
      measure of distance.

295   3.2.1        P RETRAINING A STRUCTURAL PROTEIN LANGUAGE MODEL
296
297
298   To test a Transformer’s ability to learn useful structural patterns in proteins, we trained an
299   ESM/BERT-style (Rives et al., 2021; Devlin et al., 2019) model to complete masked token pre-
300   diction. We trained two models: one with coordinates (‘coords model’) and the other without (‘non-
301   coords model’). The version with coordinates added a linear embedding of the coordinates to the
302   token embedding in the same way as the simulated experiments. Both models were very similar
303
      to the smallest publicly released ESM1 model, consisting of a 6-layer Transformer encoder with a
      hidden dimension of 768, 12 attention heads, a feedforward dimension of 2048, and GeLU activa-
304
      tion (Hendrycks & Gimpel, 2023). As in ESM, we omitted dropout. To prevent the model from
305
      focusing too much on linear positional information, we used Sinusoidal Positional Encodings rather
306   than Rotary Positional Encodings (Su et al., 2022). As is common in masked token prediction, we
307   masked 15% of tokens. Of the masked tokens, 80% were replaced with a [MASK] token, 10% were
308   replaced with a random amino acid, and 10% were left unchanged. We clustered the data by 50%
309   sequence identity using MMSeqs2 (Steinegger & Söding, 2017) and randomly held out 1% of the
310   clusters to use as a validation set. We trained each model for 100 epochs with a fixed batch size
311   of 24, resulting in approximately 150K updates. We used the Adam optimizer with 4,000 warmup
312   steps to a peak learning rate of 2.3×10−4 , followed by inverse square decay. Each time a structure
313   was loaded, its coordinates were recentred, randomly rotated, and rescaled.
314   As shown in Figure 4a, adding coordinates substantially improved the model, leading to a final
315   training perplexity of 6.5 with coordinates vs 11.9 without. The final training loss for the version
316   without coordinates (after 100 epochs) was surpassed by the version with coordinates after 8 epochs.
317   Additionally, the final validation loss was surpassed after only 4 epochs which may indicate that the
318   structural features learned early in training are more robust to dissimilarity in sequence space.
319   We also investigated the difference in sequence recovery rates between the two models. The total
320   sequence recovery rate was ∼23% for the non-coords model compared to ∼38% for the coords
321   model. Figure 4b shows a breakdown of the recovery rates per amino acid type. The recovery rate
322   for the coords model was greater than or equal to that of the non-coords model for all amino acid
323   types. The difference was particularly stark for glycine and proline, which may be related to their
      distinct backbone conformational preferences (Ho & Brasseur, 2005; Beck et al., 2008).

348
349   In the model trained in the previous paragraph, there are three inputs to the model for each token:
350
      amino acid type, sequential position, and 3D coordinates. To test if the pretrained model with
      coordinates was learning to measure distance as predicted, we plotted the average attention paid by
351
      each pair of tokens across all heads in a layer as a function of distance. We also plotted the average
352
      attention paid to each token as a function of relative sequence distance. To isolate the effect of each
353   feature, we fixed all amino acids to alanine. We also fixed the linear sequence index to a constant
354   value for all tokens while measuring 3D dependence and fixed the 3D position to (0, 0, 0) while
355   measuring linear dependence. For the distance measurements, we rounded each pairwise distance
356   to the nearest Angstrom and computed the average for each distance value.
357
      Figure 5 shows the plots for 3D and linear positional dependence for all layers for both models as
358
      well as the amplitude and standard deviation of the fit Gaussian for each layer. As expected, the
359   model without coordinates shows no correlation with Euclidean distance. Conversely, the model
360   with coordinates shows a strong dependence in the early layers which is well-approximated by a
361   Gaussian. Both models progressively widen their field of view as information passes through the
362   layers. This may correspond to a system of reasoning where the model collects information about
363   the local environment before processing this information alongside global patterns. We provide a
364   similar plot without isolating each of the factors in Appendix A.3.

      4    C ONCLUSIONS
459
460
      In this work we show that standard Transformers are capable of performing structural reasoning by
461
      learning an approximately SE(3)-invariant distance filter on attention. We predict that even linearly
462   embedded positions can produce Gaussian attention filters of distance and validate this prediction
463   using experiments on simulated points and proteins. The protein model naturally learns to use
464   the 3D coordinates to measure distance which substantially improves its ability to predict masked
465   tokens. The structural information also materially improves the model’s ability to inform function
466   prediction, providing even greater benefit than existing custom-built structural models.
467
      We show that Transformers can learn to measure distance and operate as hybrid structure/language
468   models. In contrast to many conventional structure models which are based on GNNs, Transformers
469   do not explicitly model edges. This admits memory-efficient implementations such as FlashAtten-
470   tion (Dao et al., 2022; Dao, 2023) which allow for fast, fully-connected updates in linear memory.
471   Most structure models store distance in edges which use quadratic memory for fully-connected
472   graphs. Practically, this means that Transformers can perform structural reasoning on more highly
473   connected structures, which may allow them to “see” more while making decisions.
474   As shown in Section 3.2.2, the pretrained protein model trained with coordinates showed a strong
475   positional dependence in attention in early layers followed by a weak positional dependence in the
476   last few layers. It is possible that this corresponds to the model identifying structural features such as
477   secondary structure and local physics, before encoding these and performing long-range sequential
478   processing. This corresponds with the contemporary trend of preprocessing structural information
479   to create structural tokens for Transformers compared to the more traditional approach of using
480   language model embeddings as input to structural GNNs.
481   In this work we explore two protein tasks: masked token prediction and function prediction. Vir-
482   tually all protein learning tasks benefit from combined sequence and structure processing and so
483   this work could be applied across areas including inverse folding, structure prediction, and arbi-
484   trary property prediction. As is common in tasks such as inverse folding, the input structures could
485   include more atoms from the backbone. This could be achieved by simply projecting these atom co-
      ordinates to the input representation, unlike GNN-based methods which require explicitly including


                                                          9
      Under review as a conference paper at ICLR 2025


486
      all pairwise distances in the edge features. Additionally, while proteins are a natural fit for struc-
487   tural Transformers due to their combined sequential and spatial data, there are many other possible
488   applications of this type of model. Some of these include tasks with explicit 3D information such as
489   small molecules and 3D objects. However, there are also tasks where learning an approximate rela-
490   tionship between entities in Euclidean space could help with reasoning, such as vision Transformers
491   (Dosovitskiy et al., 2021) or even large language models.
