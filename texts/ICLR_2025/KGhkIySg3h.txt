009                                                   A BSTRACT
010
011                We consider the problem of fine-tuning pre-trained language models with a small
012                amount of trusted data (high-fidelity) and a larger amount of data with noisy labels
013                (low-fidelity). We propose Multi-Fidelity Fine-Tuning (MFFT), a novel approach
014                which implicitly determines for new inputs when we can rely on information
015                from high-fidelity data and when instead we need to fall back on knowledge from
016
                   low-fidelity data. MFFT does not require any architecture changes to the base
                   model and simply provides its fine-tuned version that can be easily deployed for
017
                   inference. We extensively benchmark MFFT on various classification tasks against
018
                   several baselines, with both simulated label noise, and in realistic scenarios with
019                LLM generated data. MFFT consistently improves performance compared to using
020                trusted data alone and outperforms all baselines across experiments with macro
021                F1-score improvements of 2-4%. Finally, it provides substantial improvements in
022                uncertainty calibration with expected calibration error (ECE) reductions of 40-60%
023                compared to the best baselines.1

      1       I NTRODUCTION
027
      Fine-tuning foundation models (Bommasani et al., 2021) using a small amount of clean data and
028
      more abundant noisy data is ubiquitous to many deep learning settings, such as learning from expert
029   and non-expert annotated data (Shapiro et al., 2013; Su et al., 2012; Sylolypavan et al., 2023), training
030   end models with weak supervision (Zhang et al., 2022; Ratner et al., 2017; Rühling Cachay et al.,
031   2021) and learning from humans and large language models (LLMs) in combination (Thapa et al.,
032   2023; Ding et al., 2024; Meng et al., 2023; Zhang et al., 2024; Wang et al., 2023a; Li et al., 2021b).
033   However, combining data sources of varying quality for fine-tuning effectively is not trivial. As
034   shown in Figure 1, simply adding lower quality data to the training set is often detrimental to model
035   performance (Zhou et al., 2024; Wang et al., 2023a). While training neural networks by combining
036   noisy and clean data has been extensively studied (Song et al., 2022; Hendrycks et al., 2018; Patrini
037
      et al., 2017; Veit et al., 2017), how to develop a strategy to fine-tune pre-trained language models
      (Vaswani et al., 2017; Radford et al., 2019; Dubey et al., 2024) has yet to be explored.
038
039   We propose Multi-Fidelity Fine-Tuning (MFFT), a novel language model fine-tuning approach for
040   down-stream tasks which leverages small amounts of data with trusted labels in combination with
041   a larger data set with noisy labels. Our approach uses two fine-tuned versions of the base model to
042   infer pseudo-labels that are used to fine-tune a final model (Figure 2). We first fine-tune a low-fidelity
043
      model using the abundant noisy data. Then, we fine-tune a high-fidelity model with the scarce clean
      data. Based on the expected log likelihood, MFFT determines whether the low or high fidelity
044
      model should be used for inference given a new input. This selective inference is run over the whole
045
      low-fidelity data set to infer soft labels. The pseudo-labeled version of the original low-fidelity
046   data set is used together with the high-fidelity data to fine-tune the base model resulting into a final
047   model. This final fine-tuned model implicitly learns when it can infer using information gained from
048   high-fidelity data and when, conversely, it has to fall back on predictions learned from low-fidelity
049   data, which are less accurate, due to the noise in the labels, but more robust, due to the abundance of
050   examples.
051
      In our extensive evaluation, we use MFFT to fine-tune different language models for a variety of
052   classification tasks with a small amount (50-100 examples) of trusted data and a larger noisy data

      2.1        L EARNING WITH N OISY AND C LEAN L ABELS
081
082   The problem of learning with noisy labels (LNL) has been extensively studied (Song et al., 2022)
083   for exploiting a small amount of data with clean labels in combination with larger amounts of data
084   with noisy labels. In the context of LNL, available clean data is often referred to as trusted data
085   or anchor points (Song et al., 2022; Patrini et al., 2017). Some methods propose to use the anchor
086   points to first learn a label correction model for inferring a clean label jointly from inputs and noisy
087   label (Xiao et al., 2015; Zheng et al., 2021; Veit et al., 2017). Other methods propose instead to use
088   the clean data to design and calibrate a noise-robust cost function (Hendrycks et al., 2018; Patrini
089
      et al., 2017). While proven effective in deep learning, these methods are difficult to apply directly
      to fine-tuning foundational models. Firstly, because of the data regime we target; As fine-tuning
090
      pre-trained language models requires much less data than training neural networks from scratch
091
      (Zhou et al., 2024; Qiu et al., 2020), we aim to push the boundaries of learning from mixed quality
092   data in terms of clean data requirements and use only up to tens of clean examples per class. This
093   causes label correction and cost calibration methods to over-fit. Secondly, many existing approaches
094   require specific model architectures (Song et al., 2022; Zheng et al., 2021), while we wish to maintain
095   our fine-tuning strategy applicable in a plug-and-play fashion to any foundation model.
096
097
098
      2.2        M ULTI -F IDELITY M ODELS
099
      Another significant research area related to learning with data of different quality, is that of Multi-
100
      Fidelity models (MFMs)(Fernández-Godino, 2023; Peherstorfer et al., 2018). MFMs are models
101
      designed to learn from several sets of training data having different levels of quality. These approaches
102   are typically used to learn from different granularity of numerical simulations and real measurements
103   in physical experiments (Christen & Fox, 2005; Meng et al., 2021; Tonolini et al., 2020), although
104   more common learning settings, such as classification have also been explored (Costabal et al., 2019;
105   Chen et al., 2022). MFMs often use uncertainty quantification in order to capture model confidence,
106   in particular for the higher fidelity data, which is often sparse (Meng et al., 2021; Peherstorfer et al.,
107   2018). We draw inspiration from this key feature of MFMs to build our fine-tuning strategy, which
      learns from sparse clean data with a deep ensemble and capture model uncertainty.

      2.3      F INE -T UNING L ANGUAGE M ODELS WITH M IXED Q UALITY DATA
129
130   Fine-tuning pre-trained language models with data of mixed quality is ubiquitous to many settings,
131   including augmenting clean data with weakly supervised data (Li et al., 2021a; Lu & Radha, 2023;
132   Yu et al., 2020b), active learning in noisy labels settings (Zhang et al., 2024; Goel et al., 2022)
133   and fine-tuning with both human and LLM labeled or generated data (Wang et al., 2023a; Zhang
134   et al., 2024; Meng et al., 2023). In some settings, clean and noisy data are simply aggregated in the
135   fine-tuning set (Zhang et al., 2024). Other methods aggregate the two sets, but assigning a different
136   cost weight to clean data (Wang et al., 2023a), while some approaches learn the noise process in
137
      different ways and incorporate it when training on noisy labels Jindal et al. (2019); Zhuang et al.
      (2023). Kim et al. (2024) have recently proposed to exploit the robustness properties of parameter
138
      efficient fine-tuning to learn from mixed quality data. Using LLMs to assist the noise cleaning
139
      process has also been explored Wang et al. (2023b). An effective and relatively simple approach,
140   similar to task adaptive pre-training (Gururangan et al., 2020; Shi et al., 2023), domain adaptation
141   (Chronopoulou et al., 2019) and transfer learning (Chronopoulou et al., 2019; Hedderich et al., 2020),
142   first fine-tunes with the noisy data, and subsequently continues fine-tuning with the clean data (Li
143   et al., 2021b; Tamkin et al., 2020; Zhu et al., 2022; Li et al., 2022). We also adopt this strategy as part
144   of our approach, however, preventing over-fitting to clean data with our multi-fidelity strategy.

      3.2   OVERVIEW
166
167   We start by fine-tuning the pre-trained model pθ0 (y|x) in two stages. First, we fine-tune the model
168   on low-fidelity data DL . Second, we continue fine-tuning with high-fidelity data DH to obtain a
169   high-fidelity model pH (y|x). This sequential fine-tuning approach is common in transfer learning
170   and semi-supervised learning with pre-trained language models (Yu et al., 2020a; Qin et al., 2022;
171   Gururangan et al., 2020; Zhu et al., 2022; Shi et al., 2023). We find that it is as effective in our multi-
172   fidelity scenario, providing a competitive baseline itself. However, with scarce high-fidelity data DH ,
173   pH (y|x) still over-fits, losing information gained during the first fine-tuning stage; a phenomenon
174
      sometimes referred to as catastrophic forgetting (Chen et al., 2020; Kotha et al., 2023). This results
      in pH (y|x) to be accurate for some inputs x that are somewhat similar to high-fidelity examples xH ,
175
      but inaccurate, and especially poorly calibrated, for others.
176
177   To address the aforementioned problem, we introduce a second version of the model by freezing the
178   weights after the first fine-tuning stage. The resulting model is fine-tuned on low-fidelity data DL
179   only and we name it accordingly the low-fidelity model pL (y|x). This model does not over-fit, as it
180
      is fine-tuned on abundant examples, but its accuracy is limited by the data noise in DL . Our strategy
      is to use pL (y|x) as a fall-back for pH (y|x) for those inputs where the latter is expected to be less
181
      accurate than the former. Because we do not want to load and run both models at inference time, we
182
      also distill the resulting system into a single final fine-tuned language classifier pθ∗ (y|x). Our MFFT
183   approach is summarized in four stages, schematically illustrated in Figure 2:
184
185         1. Fine-tune the pre-trained model pθ0 (y|x) with low-fidelity data DL to obtain pL (y|x).
186
            2. Continue fine-tuning pL (y|x) with high-fidelity data DH to obtain pH (y|x).
187
188         3. For each input in the low-fidelity set xL,i ∈ xL , estimate the expected log likelihoods of
189            both pH (y|xL,i ) and pL (y|xL,i ) and infer soft labels pM (y|xL,i ), using the model expected
190
               to perform best.
191         4. Fine-tune the pre-trained model pθ0 (y|x) with all inputs [xL , xH ] and combined inferred
192            soft labels and high-fidelity hard labels as targets [pM (y|xL ), yH ], obtaining the final model
193            pθ∗ (y|x).

195   3.3   L OW- AND H IGH -F IDELITY M ODEL F INE -T UNING
196
197   A key consideration in fine-tuning high and low fidelity models for our strategy is that we need
198
      to estimate their accuracy for new inputs to choose which one to use. As pL (y|x) is trained with
      abundant noisy data DL , its accuracy predominantly depends on data noise and, while it is not
199
      possible to estimate this from model output alone, we can approximately estimate it using clean
200
      examples in DH as evaluation points (Hendrycks et al., 2018; Patrini et al., 2017). Contrarily, the
201   accuracy of pH (y|x) depends predominantly on model mis-specification, or lack of knowledge, as
202   it was fine-tuned on clean, but scarce data DH . This means that, to detect for which inputs x the
203   high-fidelity model is expected to be inaccurate, we need to construct pH (y|x) to obtain accurate
204   uncertainty estimation with respect to its reducible error, i.e., due to lack of data.
205
      Fine-Tune Ensemble: To obtain models that give accurate uncertainty estimation, we adopt a deep
206
      ensemble approach. We repeat the two-stages fine-tuning with different random seeds, obtaining K
207   fine-tuned models p(y|x, θL,k ) and K fine-tuned models p(y|x, θH,k ). At inference time, the high
208   and low fidelity label probabilities are obtained by aggregating their outputs:
209

214   minarayanan et al., 2017; Abdar et al., 2021; Rahaman et al., 2021). However, obtaining effective
215   deep ensembles from pre-trained models is challenging, as the starting weights are fixed and cannot
      be randomly initialized for each model, leading to poor diversification and inaccurate uncertainty

217   fidelity setting, we exploit the abundance of low-fidelity data. We split the low-fidelity data into K

219   second stage, all models are then fine tuned on the entire high fidelity set DH . We found this strategy

      4.1   E XPERIMENTAL S ETTINGS AND BASELINES
293
294   We test MFFT using six benchmark text classification data sets; AGNews (Zhang et al., 2015b),
295   DBPedia 14 (Zhang et al., 2015a), GLUE-SST2, GLUE-QQP (Wang et al., 2019), TREC (Li & Roth,
296   2002) and Yahoo Answers (Adamic et al., 2008). More details about these data sets and tasks are
297   given in Appendix B.1. In all our experiments, we take 5, 000 examples to construct the training set
298   and 1, 000 examples for testing. The training set is split into two sub-sets; a high-fidelity set DH , for
299   which the labels are directly extracted from the original source data, and a low-fidelity set DL , for
300
      which the labels are either artificially corrupted, in the simulated noise experiments, or inferred using
      an LLM, in the LLM experiments. In all experiments, we compare MFFT to six baseline approaches
301
      used in similar settings in related work:
302
303          • High: We discard the low-fidelity data DL and fine-tune the pre-trained language model
304            solely with high-fidelity data DH .
305          • Together: High and Low fidelity data sets are simply aggregated into a single data set which
306            is used to fine-tune the pre-trained language model. This approach has been used for the
307            fine-tuning component of several works involving mixed quality data (Zhang et al., 2024; Li
308            et al., 2021a; Lu & Radha, 2023).
309          • Cost Adjustment: Similarly to the above, the model is trained on all available data together.
310            However, a higher weight is assigned to the cost from high-fidelity data following Wang
311            et al. (2023a).
312          • Low-High: A domain adaptation approach, where the model is initially fine-tuned on
313            low-fidelity data DL and then on DH . This approach is adopted in several multi-fidelity
314            learning works (Aydin et al., 2019; Li et al., 2021b; 2022) and can be considered a special
315            case of task adaptive pre-training (TAPT) (Gururangan et al., 2020; Shi et al., 2023).
316          • High-Low: The language model is first fine-tuned on high-fidelity data DH and then on
317            low-fidelity data DL , with label smoothing and temporal ensembling (Meng et al., 2023).
318
             • Noise Correction: The high-fidelity points are used to learn a linear noise process from
319            clean to noisy labels. This mapping is applied to the cost function for low-fidelity examples,
320            which are then used together with high-fidelity ones to fine-tune the final model (Hendrycks
321            et al., 2018; Jindal et al., 2019).

471   5     C ONCLUSION
472
473   We proposed Multi-Fidelity Fine-Tuning (MFFT), a novel method to fine-tune pre-trained language
474   models with a small amount of clean trusted data (high-fidelity) and a larger amount of noisy
475   data (low-fidelity). MFFT exploits knowledge derived from noisy data and knowledge derived
476   from trusted data differently, implicitly learning when the latter can be used to infer and when,
477   conversely, predicting from the former is expected to be more accurate. This leads to models fine-
478   tuned with MFFT to consistently benefit from additional noisy data, while other methods carry the
479
      risk of degrading performance compared to using trusted data only. In our experiments, MFFT
      consistently outperformed all baselines across experiments, especially in calibration, with macro
480
      F1-score improvements of 2 − 4% and expected calibration error (ECE) reductions of 40 − 60%
481
      compared to the best baselines.

784   models. As we consider scenarios where clean labels are scarce, we set the number of validation

786   learning rate of 5 × 10−5 . The macro f1 score is computed with the validation set every 100 iterations

788   using the standard cross-entropy cost with hard labels, and using equation 7 with soft or mixed labels.

833   B.3   D ETAILS OF N OISE S IMULATION
834
      For our simulated experiments, we introduce noise by altering labels in the source data set with a
835
      stochastic process. In our simulation, we define two controllable parameters; noise level l ∈ [0, 1]
836
      and noise bias b ∈ [0, 1]. Both of these are used in constructing a noise process matrix Mϵ which
837   maps clean labels to probabilities of noisy labels:
838
839                                           p(y ′ |y) = Mϵ p(y ∗ |y),                                   (14)
                ′                                                   ∗
840   where p(y |y) is the probability of corrupted labels and p(y |y) is the probability form of the ground-
841   truth labels, i.e., a vector as long as the number of classes Nc , with one on the class corresponding
842   to the ground-truth label y and zeros everywhere else. The corrupted label y ′ is then obtained by
843   sampling from this distribution y ′ ∼ p(y ′ |y). The transition matrix Mϵ is computed with the noise
844   level parameter l and the bias pareameter b as follows:
845                                                              l
846                           Mϵ = (1 − l)I(1 + bR) +                (1 − I)(1 + bR).                     (15)
                                                              Nc − 1
847
      Here I is the Nc × Nc identity matrix, 1 is a Nc × Nc matrix of ones and R is a Nc × Nc matrix
848
      where the elements are random uniform between zero and one. The matrix Mϵ is first capped so that
849
      all elements are between zeor and one and secondly normalized so that the rows add up to one. With
850   b = 0 (no bias), the matrix results in labels corrupted through equation 14 to be changed to another
851   class with probability l and staying the same with probability 1 − l. The higher the value of b, the
852   higher the randomness in the transition matrix, meaning that not all classes have equal probability to
853   change and a given label y has non-equal probability to change to each one of the other classes. This
854   introduces label bias proportionally to b. In the simulated experiments of section 4.2, we fix b = 0.3
855   and test with two noise levels; l = 0.3 (results in section 4.2) and l = 0.5 (results in section C.2).
