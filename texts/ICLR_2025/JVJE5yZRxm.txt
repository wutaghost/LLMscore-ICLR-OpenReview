009                                               A BSTRACT
010
011            Recent advancements in large language models have demonstrated their effective-
012            ness in various tasks. However, the question of these models’ limitations remains
013            open though. For instance, can a language model learn to perform code execution
014            (i.e., predicting the output of code)? Current research indicates that the perfor-
015            mance of state-of-the-art large language models in code execution is still limited.
016            The reasons for this limitations are unclear though. Is it due to fundamental con-
017
               straints or other factors such as training data and computational resources? Is
               the next-token prediction objective sufficient for learning code execution? How
018
               small can a language model be while still capable of learning code execution?
019
               In this paper, we investigate these questions. More specifically, we investigate
020            whether tiny language models, trained from scratch using the next-token predic-
021            tion objective, can effectively learn to execute code. Our experiments show that,
022            given appropriate data, model size, and computational resources, tiny language
023            models can indeed learn to perform code execution with a 99.13% accuracy for
024            a tiny Turing-complete programming language. We begin by defining a tiny pro-
025            gramming language called TinyPy. Millions of randomly generated codes in this
026            language, along with their outputs, are used to train our tiny language models
027            using the next-token prediction task. We then conduct a series of experiments
028
               to determine the smallest model size, data amount, and computational resources
               necessary to train our language model to achieve near-perfect accuracy in code
029
               execution. Our findings reveal that TEX, our proposed tiny language model with
030
               15M parameters, can successfully learn code execution. This suggests that a task
031            as complex as predicting code output is within the reach of language models.

034   1   I NTRODUCTION
035
      The rise of large language models (LLMs) such as GPT-3 (Brown et al., 2020) has marked a signif-
036   icant advancement in the field of natural language processing (NLP). These models, typically based
037   on autoregressive, decoder-only architectures, have demonstrated superior performance across a
038   broad range of language-related tasks. Building on this success, several studies have extended sim-
039   ilar architectures to the domain of code modeling (Chen et al., 2021; Lu et al., 2021; Zheng et al.,
040   2023), leading to the development of code language models trained on vast corpora of programming
041   languages.
042   While these models have achieved state-of-the-art results in various coding tasks, such as code gen-
043   eration and completion, accurately executing code (predicting the output of a given code) remains
044   a significant challenge for these models. Previous studies have highlighted this gap (Austin et al.,
045   2021), indicating that even the largest code language models often struggle to perform code exe-
046   cution tasks reliably. In addition, our evaluation of state-of-the-art models, including Code Llama
047   (Rozière et al., 2024) and GPT-4o, on our test dataset, reveals limitations in their ability to execute
048   code perfectly (above 99% accuracy).
049   In this paper we investigate whether a language model can learn to execute code. Our investigation
050   is driven by several key questions: Can a language model learn to predict the output of a code (i.e.,
051   learn code execution) accurately? Is the next-token prediction objective, which has been the corner-
052   stone of language model training, sufficient for learning code execution? How can we effectively
053   teach code execution to a language model? And perhaps most importantly, how small can a language
      model be while still maintaining the ability to learn and perform code execution tasks?

054
      We believe that the task of code execution represents a significant challenge that offers valuable
055   insights into the capabilities of language models. While code execution might not be inherently
056   crucial for language models in practice, as dedicated tools exist for this purpose, it serves as an
057   excellent proxy for assessing a model’s ability to handle complex, structured information. The
058   precision required for code execution make it an ideal benchmark for evaluating a language model’s
059   capacity to represent and manipulate abstract concepts, follow instructions and logical sequences,
060   and produce accurate outputs based on specific commands and a formal syntax. By focusing on code
061   execution, we aim to shed light on the capabilities and limitations of language models, potentially
062   uncovering insights that could inform future developments in the field.
063   To answer our questions, we followed a structured experimental approach. We started by defining a
064   tiny, yet turing-complete, programming language to train our tiny language model. We call it TinyPy.
065   This language was specifically designed to be complex enough to maintain Turing-completeness
066   while remaining minimalistic, allowing a tiny language model to learn and execute it. Next, we
067   generated millions of random code snippets in TinyPy, creating a diverse dataset that covers various
068   code structures and patterns. Following this, we trained models with different sizes from scratch
069
      using the next-token prediction objective. Finally, we evaluated these models on their ability to
      accurately execute code.
070
071   Our findings reveal that TEX, our proposed tiny language model with only 15M parameters, can
072   achieve up to 99.13% accuracy in executing TinyPy code. Importantly, we find that the next-token
073   prediction task is sufficient for teaching code execution to language models. By demonstrating that
074   tiny models can learn to execute code, this study lays the groundwork for further exploration into
075
      the capabilities and applications of language models in code execution and beyond.
076   The main contributions of this work are as follows:
077
078          • We demonstrate that tiny language models, with less than 20M parameters, trained from
079            scratch using only the next-token prediction objective, can achieve near-perfect accuracy
080
               (up to 99.13%) in code execution.
081          • We propose a tiny language model, TEX, that achieves 99.13% accuracy in the task of
082            TinyPy code execution.
083          • We conduct a systematic analysis of the key factors —data size, model size, and training
084            duration—that directly influence the performance of a language model in executing code.
085          • We release our dataset, and open-source our random code generator, model, and entire
086            codebase, enabling the replication and extension of our work by the research community.

      2   R ELATED W ORK
089
090   Code execution. Previous research has extensively explored the task of code execution, highlight-
091   ing its importance. Various architectures have been employed for this purpose, including recurrent
092   neural networks (RNNs) (Zaremba & Sutskever, 2015), graph neural networks (GNNs) (Bieber
093
      et al., 2020; Wang et al., 2020), and Transformers (Dehghani et al., 2018; Yan et al., 2020). They all
      take a piece of code as input and predict its output.
094
095   Recent studies have also leveraged pre-trained models to perform the task of code execution. For
096   instance, Austin et al. (2021) evaluated pre-trained models, ranging in size from 2M to 137B pa-
097   rameters, and found that even the largest models struggle to predict program outputs, even with
098   fine-tuning, never exeeding a 29% accuracy. Nye et al. (2021) introduced a “scratchpad” to store
099
      intermediate computation steps, enhancing the ability of models to perform multi-step computa-
      tions. A study done by Liu et al. (2023b) investigates how well pre-trained models can perform code
100
      execution. Their model, CodeExecutor, is an encoder-decoder model that uses a training objective
101
      designed specifically for the task of code execution and which involves predicting both the line order
102   and the intermediate states of the execution trace. With this specifically designed pre-trained task,
103   CodeExecutor could reach an accuracy of 48.06% in predicting outputs. While this result represents
104   a significant milestone, current state-of-the-art work still does not answer the question of whether
105   language models can perfectly learn the task of code execution? Is this task within the reach of cur-
106   rent language models, given enough training data? Or even with a large amount of data and training
107   time, language models cannot perform this task? Is a generic pre-training objective such as next
      token-prediction enough, or we need to use a specialized training objective? Even state-of-the-art


                                                        2
      Under review as a conference paper at ICLR 2025


108
      large language models such as Code Llama (Rozière et al., 2024) and GPT-4o still have limitations
109   in this task as we show in the evaluation. Does this mean that the task itself is fundamentally be-
110   yond the capabilities of language models, or simply it is a matter of lack of data and lack of training
111   time? All of these questions are still open research questions. Our focus in this paper is to address
112   these questions rather than simply building a model for code execution. We belive that the task of
113   code execution is a challenging task that provides insights on the abilities of language models and
114   therefore we use it to study the capabilities of language models.
115   To the best of our knowledge, our work is the first to demonstrate that a language model pre-trained
116   using next-token prediction, can indeed learn code execution with near-perfect accuracy (99.13%).
117   That is, given enough data, and training time, language modeling alone is enough to learn the task
118   of code execution, assuming the output of code is present in the training data. Unlike previous wok,
119   we focus on studying decoder-only language models, and we do not define a specific objective for
120   this task but rather use the generic next-token prediction objective, known as language modeling,
121   which is a widely used objective for language models.
122
123   Small language models. Recent advancements in small language models have revealed their po-
124   tential across various specialized tasks. Lee et al. (2023) demonstrates that small transformers can
125
      efficiently learn to perform arithmetic operations when trained on high-quality data. Eldan & Li
      (2023) illustrate the capability of small models with fewer than 10 million parameters to generate
126
      fluent and coherent short stories in natural language. Liu et al. (2023a) highlight that fine-tuning
127
      small models on high-quality datasets can achieve high accuracy on the GSM8K benchmark. Addi-
128   tionally, Joshi et al. (2024) present a 60M parameter model trained on Excel formulas that outper-
129   forms larger models in tasks related to Excel formulas. Our work extends this line of research by
130   being the first to thoroughly explore the performance of small language models on the task of code
131   execution and whether it can be learned by language models.

      3   OVERVIEW
135   Our study employs an incremental approach to investigate code execution capabilities in tiny lan-
136   guage models. We start by designing a tiny Turing-complete programming language, called TinyPy.
137   This language is tiny because it supports minimal language constructs that allow it to be turing-
138
      complete but does not support more complex language constructs (which are usually designed to
      improve productivity). The language has a vocabulary size of 53 tokens, supports statements that
139
      perform arithmetic operations, and control flow (if-conditionals, for loops and while loops). It sup-
140
      ports 8-bit unsigned integers as a data type (more detail about TinyPy in subsection 4.1). Next,
141   we use the TinyPy Generator (Yamani et al., 2024), an automatic python code generation tool, to
142   generate millions of codes written in this language (more on data generation in subsection 4.2). The
143   dataset consists of the randomly generated codes, each followed by its corresponding output ex-
144   pressed in a comment at the end of the code. We use this dataset to train our tiny language models.
145
      Through iterative refinement, we develop our proposed model, TEX (Tiny Executor), a language
146
      model of 15M parameters that achieves 99.13% accuracy in code execution (details about the model
147   are provided in section 5). This result demonstrates that a language model (as tiny as TEX) can
148   effectively learn to execute code when trained from scratch using the next-token prediction only.
149   Figure 1 provides an overview of our approach.

      4.1 L ANGUAGE D ESCRIPTION
186
187
      The custom language we developed, named TinyPy, is a Turing-complete language derived from a
      carefully selected subset of Python. This subset was chosen to retain the essential language con-
188
      structs of Python that allow for Turing-completeness while minimizing complexity, making it suit-
189
      able for our tiny language model.
190
191   TinyPy has the following language constructs (full grammar is provided in Appendix A) :
192
             • Assignment statements;
193
194          • Arithmetic operations (addition, subtraction, multiplication, and division);
195          • Comparison operators (≤, <, ≥, >, =, ̸=) and the logical operator not
196
             • Control flow:
197
198
                  – Conditional statements (if-else, if-elif-else);
199               – Loops (for loops and while loops).
200
      TinyPy is a Turing-complete language due to its support for 1) sequences of statements; 2) con-
201
      ditionals (if-then-else); and 3) while loops. According to the Böhm-Jacopini theorem (Böhm &
202
      Jacopini, 1966), also known as the structured program theorem, these three language constructs are
203   sufficient to make a given language Turing-complete.
204
205   To further limit the problem space, we’ve restricted the numeric range of values used for initializing
206
      variables to unsigned 8-bit integers. Note that we do not put restrictions on the range or data type of
      the intermediate variables. Such variables can be floating point numbers and can be negative. The
207
      final output can also be a floating point number or a negative number.
208
209   Note also that, the fact that TinyPy supports only 4 arithmetic operations and unsigned 8-bit integers
210   (for data initialization) does not make it Turing-incomplete. It is still Turing-complete, but has a
211   lower level of abstraction. Moreover, complex operators and data types can still be emulated using
212
      the basic operators and data type supported by TinyPy (for example, one can encode a floating point
      number using a sequence of the two numbers 0 and 1).
213
214   When designing our language, we had three goals in mind. First, the language should allow us
215   to train language models on a small vocabulary compared to complete programming languages.
      This would allow us (as well as future researchers in the community) to perform experiments on


                                                                       4
      Under review as a conference paper at ICLR 2025


216
      limited resources. Second, the language should have enough expressive power to allow for important
217   programming patterns. Third, the language should have the same syntax as an existing language,
218   so that we avoid the need to build a while new compiler and software ecosystem. TinyPy satisfied
219   these goals.

462   7   D ISCUSSION
463   Our experiments demonstrate that tiny language models, such as the 15M-parameter TEX model,
464   can achieve high accuracy (up to 99.13%) in the task of code execution on a simple, yet Turning-
465   complete language such as TinyPy. The results also highlight that the task of next-token prediction
466   is sufficient to teach a model the task of code execution, assuming the training data contains enough
467   samples of codes and their outputs.
468
469   Next-Token Prediction for Code Execution. Our research provides a proof-of-concept that the
470   next-token prediction objective, commonly used in language model training, is indeed sufficient for
471   learning the task of code execution. The high accuracy achieved by our model, trained solely using
472   next-token prediction on data containing codes and their output, without any other special type of
473
      training, demonstrates the effectiveness of next-token prediction in capturing the underlying patterns
      and logic of code execution. This finding is significant as it suggests that, unlike the current com-
474
      mon practice in state-of-the-art models for code execution (Liu et al., 2023b), specialized training
475
      objectives may not be necessary for teaching code execution to language models.
476
477
      Scaling Experiments. Our study reveals that teaching code execution to language models requires
478   a careful balance between data size, model size, and training duration. We observed diminishing
479   returns beyond certain thresholds— 15M parameters for model size, and 5 epochs for training.
480   However, these results are specific to our experimental configuration, and different data sizes, model
481   capacities, or training durations may yield different outcomes. Studying what is the best amount of
482   data, model size and training time, is not within the scope of this work. Further exploration is needed
483   to perform a comprehensive exploration across a broader range of configurations.
484
485   Minimum Model Size for Code Execution. Our research provides insights into the question of
      how small a language model can be while still maintaining the ability to learn and perform the task of


                                                                               9
      Under review as a conference paper at ICLR 2025


486
      code execution. The near-perfect performance of our 15M-parameter model suggests that even tiny
487   models can effectively learn code execution. However, our scaling experiments indicate that there is
488   a lower bound to model size, below which performance degrades significantly. For our specific task
489   and language, models with fewer than 1M parameters struggled to achieve high accuracy.
490
491   Language Limitations It is important to note that the limitations of the TinyPy langauge are or-
492   thogonal to our main hypothesis. While our designed language may have a lower level of abstraction
493   and may lack certain features found in programming languages with a higher level of abstraction,
494   these limitations do not impact our core investigation, as the language is complex enough and has
495   the minimum programming language constructs to qualify as Turing-complete (sequences of state-
496   ments, for and while loops and if-conditionals). Our primary focus in this work is on the fundamental
497
      ability of language models to learn and execute a simple form of code (yet complex enough). Now
      that we have demonstrated such an ability, we plan to extend our work to support full programming
498
      languages (e.g., Python).
499
500
501   8   L IMITATIONS
502   While our study demonstrates the potential of tiny language models in code execution tasks, it has
503   some limitations. Our use of the TinyPy, our custom programming language, allowed us to focus
504   on core programming language constructs but does not cover more complex constructs. Further
505   studies are needed to generalize our findings to other, more complex programming languages. Such
506   studies are orthogonal to our goal in this paper though. Our goal is to evaluate whether a language
507   model can learn the task of code execution on a simple, yet turing-complete language, and TinyPy is
508
      sufficient to achieve this goal. Additionally, the restricted numeric range of unsigned 8-bit integers
      limits the scope of numerical computations addressed, potentially overlooking challenges present in
509
      broader numerical ranges. These limitations provide valuable directions for future research, inviting
510
      exploration of the capabilities of tiny language models in more diverse and complex programming
511   environments, and investigation of their performance across a wider range of programming lan-
512   guages and numerical operations.
513
514
515
      9   C ONCLUSION
516   This study provides empirical evidence that tiny language models, with fewer than 20M parameters,
517   can effectively learn to execute simple code with high accuracy. Our proposed model TEX achieves
518   a 99.13% accuracy in executing code from TinyPy, a Turing-complete programming language that
519   performs arithmetic operations, and supports control flow statements such as if-conditionals, for
520
      loops and while loops.
521   The development of TEX, a 15M parameter model, serves as a proof-of-concept that a language
522   model trained from scratch using only the next-token prediction objective on data containing codes
523   and their outputs, and without any other task-specific training objective, can effectively learn to
524   execute code.
525   This research contributes to the understanding of language models’ capabilities in executing code
526   and opens new directions for efficient and accessible AI models in coding applications. All code
527   and datasets used in this study are open-sourced, providing a foundation for further research and
528   development in this area.
