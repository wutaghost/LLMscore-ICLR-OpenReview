                                                 A BSTRACT
008
009
               Marked temporal point processes (MTPPs) are used to model sequences of dif-
010
               ferent types of events with irregular arrival times, with broad applications rang-
011            ing from healthcare and social networks to finance. We address shortcomings
012            in existing point process models by drawing connections between modern deep
013            state-space models (SSMs) and linear Hawkes processes (LHPs), culminating in
014            an MTPP that we call the deep linear Hawkes process (DLHP). The DLHP mod-
015            ifies the linear differential equations in deep SSMs to be stochastic jump differ-
016            ential equations, akin to LHPs. After discretizing, the resulting recurrence can
017            be implemented efficiently using a parallel scan. This brings parallelism and lin-
018            ear scaling to MTPP models. This contrasts with attention-based MTPPs, which
019
               scale quadratically, and RNN-based MTPPs, which do not parallelize across the
               sequence length. We show empirically that DLHPs match or outperform existing
020
               models across a broad range of metrics on eight real-world datasets. Our proposed
021
               DLHP model is the first instance of the unique architectural capabilities of SSMs
022            being leveraged to construct a new class of MTPP models.

158   2.2   S TATE -S PACE M ODELS
159
160   Deep state-space models (SSMs) are a recent innovation in recurrent models that have found success
161   in long-range sequence modeling tasks (Gu et al., 2022b) and language modeling tasks (Gu & Dao,
      2023), while also having favorable computational properties. The backbone of deep SSMs is the

                                                          3
      Under review as a conference paper at ICLR 2025


162
      linear state-space equations, which define a continuous-time dynamical system with input and output
163   signals u(t), y(t) ∈ RH , respectively, through linear differential equations:
164
165                                            d
                                                  x(t) = Ax(t) + Bu(t)                                            (7)
166                                            dt
167                                              y(t) = Cx(t) + Du(t),                                            (8)
168
      where x(t) ∈ RP is the (hidden) state of the system, and A ∈ RP ×P , B ∈ RP ×H , C ∈ RH×P , and
169
      D ∈ RH×H are the parameters that control the system’s dynamics.
170
171   Deep SSMs then stack these recurrences interleaved with non-linear position-wise functions, σ. The
172   function σ can contain activation functions, residual connections and normalization layers, and trans-
173   forms the output y of the previous recurrence into the input u of the next, i.e. u(l) (t) := σ(y(l−1) (t))
174
      for layer l. This combination yields a sequence model where each recurrence is conditionally linear
      in time given the input, but is ultimately non-linear in depth due to the function σ.
175
176   To evaluate the SSM, we first discretize the continuous-time system at the desired times, and then
177   evaluate as though it were a conventional discrete-time RNN architecture. Crucially, the linearity of
178   the resulting discrete-time recurrence allows it to be evaluated using parallel scans (Blelloch, 1990;
179   Smith et al., 2022; Gu & Dao, 2023), leading to linear work scaling (i.e. number of operations),
180
      and, importantly, sublinear scaling of the computation time with respect to sequence length given
      sufficient parallel compute. Note this contrasts with conventional sequential RNNs (e.g. LSTMs),
181
      which process sequences serially; and attention-based methods, which can be parallelized over a
182
      sequence, but have quadratic work scaling with respect to sequence length. This allows SSMs to
183   fully and efficiently leverage modern massively parallel hardware while also a being highly expres-
184   sive and performant model class. Importantly for our purposes, evaluating a linear recurrence with
185   a parallel scan natively admits evaluations with varying observation intervals. We will leverage this
186   to parsimoniously handle the variable inter-event times observed in MTPP settings.

      In this section, we introduce our deep linear Hawkes process (DLHP), a neural MTPP that draws a
191
      novel connection between LHPs and deep SSMs. Stochastic jump differential equations, akin to the
192   LHP intensity, form the basis of the conditionally linear recurrent layer, which we refer to as a latent
193   linear Hawkes (LLH) layer. The LLH layer can be viewed as a modified SSM recurrence, while
194   still admitting parallel computation. Taking further inspiration from deep SSMs, the DLHP is then
195   made up of a stack of LLH layers, each interleaved with non-linear, position-wise transformations
196   to increase the overall expressivity of the model (see Fig. 2a). In this section, we formalize this
197   approach and outline implementation details.

268   Unlike the LHP intensity, the recurrence in the LLH layer does not permit an analytic solution.
269   As such, we must discretize the continuous-time process to compute values of the layer at specified
      time points. If we approximate the input signal by treating it as constant over an update interval, also

273   and then impose the zero-order hold restriction on it. Doing so converts the matrix exponential into
274   an element-wise exponential operation. This is done for all LLH layers that compose the DLHP.

317   3.5   C OMPUTING LLH R ECURRENCE
318
319   Thus far, we have created the LLH layer by diagonalizing and discretizing a modified SSM. As
320
      discussed earlier, we would like to take advantage of the efficient parallel scans leveraged by many
      SSM-based models (Smith et al., 2022; Gu & Dao, 2023; Dao & Gu, 2024). Below we explain how
321
      we can still use the parallel scan, despite the modified recurrence.
322
323   Parallel scans admit efficient inference over linear recurrences of the form zi+1 = Ai zi + bi (Blel-
      loch, 1990). Although we have added an impulse to the recurrence, this is still intrinsically of this


                                                            6
      Under review as a conference paper at ICLR 2025


324
      form, where zi := xti , Ai := exp(Λi (ti+1 − ti )), and bi := (Ai − I)B̃uti+1 − + Ẽαki+1 . As a re-
325   sult, we can leverage efficient parallel scans to compute the sequence of right-limits xt1:N in parallel
326   across the sequence length. The corresponding left-limits xt1:N − can then be efficiently computed
327   after by subtracting off Ẽαk1:N from xt1:N . In Algorithms 1 to 3 we compactly detail how to use a
328   parallel scan to compute the sequence of right limits given events; how to evolve those right limits
329   to compute left limits; and then how to subsequently compute the log-likelihood of the sequence.

      4   R ELATED W ORKS
332
333
      Neural MTPPs Marked temporal point processes (MTPPs) are generative models that jointly
334
      model the time and type of continuous-time sequential events, typically characterized by mark-
335   specific intensity functions (Daley & Vere-Jones, 2003). Early approaches, such as self-exciting
336   Hawkes processes (Hawkes, 1971; Liniger, 2009), used simple parametric forms for the inten-
337   sity. More recently, neural architectures such as RNNs (Du et al., 2016; Mei & Eisner, 2017),
338   CNNs (Zhuzhel et al., 2023), and transformers (Zhang et al., 2020; Zuo et al., 2020; Yang et al.,
339   2022) have been used to more flexibly model the conditional intensity. For intensity-free MTPPs, ap-
340   proaches include normalizing flows (Shchur et al., 2020a; Zagatti et al., 2024), neural processes (Bae
341   et al., 2023), and diffusion models (Zeng et al., 2023; Zhang et al., 2024); however, the most com-
342   mon approach is to model intensities as it requires fewer modeling restrictions.
343   Efficient MTPPs Due to their recurrent nature, RNN-based MTPP models incur O(N ) complex-
344   ity for sequences of length N as events must be processed sequentially. Attention-based MTPP
345   models can be applied in parallel across the sequence, but the computational work scales as O(N 2 ).
346   Türkmen et al. (2020) proposed modeling events as conditionally independent so long as they oc-
347   curred within the same time bin of a specified size. This resulted in parallel computation within
348   bins, but still scales overall as O(N ). Shchur et al. (2020b) proposed an intensity-free TPP which
349
      uses triangular maps and the time-change theorem (Daley & Vere-Jones, 2003). This was extended
      by Zagatti et al. (2024) to handle marks, but in doing so, lost many of the benefits of the origi-
350
      nal model and scales linearly in the mark dimension—which can rapidly become untenable with
351
      O(N K) work. To the best of our knowledge, our proposed model is the first that efficiently scales
352   with sequence length and mark space, as well as the first to fully leverage SSMs and parallel scans.
353
354   SSMs for Sequential Modeling SSMs have found recent success as alternatives to RNNs, CNNs,
355
      and transformers, enjoying reduced training cost and comparable modelling power (Gu et al.,
      2022b). A range of variants have been developed (Gu et al., 2021; Gupta et al., 2022; Gu et al.,
356
      2022a; Smith et al., 2022), and have been applied in language modeling (Gu & Dao, 2023),
357
      speech (Goel et al., 2022), and vision (Wang et al., 2023; Zhu et al., 2024). The linear recurrence
358   allows for parallelism, as well as accessible long contexts which would be prohibitive for transform-
359   ers due to their quadratic scaling. However, SSMs have not previously been applied to MTPPs, in
360   part due to the irregular inter-event times and the input being a stochastic counting process.
361
      [Edited] Concurrent work by Gao et al. (2024) used Mamba (Gu & Dao, 2023), a recent deep SSM
362
      architecture, in an MTPP setting, in what they call the Mamda Hawkes Process (MHP). The MHP
363
      uses a mamba SSM as the encoder in an encoder-decoder architecture, also leveraging the variable
364   interval capabilities. Crucially, however, they use a separate parametric decoder for intermediate
365   intensities (similar to, for instance, the THP). This misses the opportunity to “fully” leverage the
366   SSM architecture, re-using the same variable interval evaluation to evaluate the the intensity.

370
      We now evaluate our deep linear Hawkes process model. Our core objectives in using SSMs for
371
      MTPP modeling were to define an architecture that is both (a) highly performant in its forecasting
372   ability, and (b) able to leverage efficient parallel compute methods to accelerate inference. To this

391   Metrics Daley & Vere-Jones (2003, p. 276) state that “testing the model on the basis of its fore-
392   casting performance amounts to testing the model on the basis of its likelihood” (emphasis added).
393   As such, our primary metric of interest to assess model performance is the per-event log-likelihood,
394   LTotal . We also investigate time- and mark-prediction performance through their own log-likelihood
                          PN               RT                      PN          ki
395   values, LTime =       i=1 log λti − 0 λs ds and LMark =         i=1 log(λti /λti ), respectively, where
396   LTotal = LTime + LMark . The log-likelihood of the arrival time characterizes the ability of the model
397   to predict when the next event will arrive. The log-likelihood of the mark is effectively the negative
398   cross-entropy classification loss and measures the ability of the model to predict what types of event
399   will occur given their arrival times. We discuss additional metrics in Appendices D.1 and D.4.

414   5.1      S YNTHETIC E XPERIMENTS
415
416   We start by performing a simple investigation into the expressivity of the DLHP intensity function
417   and the ability to capture background intensities. We train our model and baselines on 5,000 se-
418   quences over the time period [0, 7.5] drawn from an unmarked, inhomogeneous Poisson process
419   with a square-wave intensity function, λt := 1(t ∈ (1, 2) ∪ (3, 4) ∪ (5, 6)) (see Fig. 1). We plot the
420   estimated intensity functions conditioned on no events occurring, i.e. Ht := ∅ ∀t.
421   Intensity estimates are shown for NHP and DLHP specifically in Fig. 1 (and for all models in Fig. 6
422   in Appendix D.2). We can see that our model successfully captures the true, underlying background
423   intensity process almost perfectly. This is largely attributed to the expressivity of the linear recur-
424   rences and non-linear depth of the model. Other models have various failure modes: struggling to
425   capture the multi-modality (RMTPP, NHP, SAHP, and THP), not matching the square shape (previ-
426   ous four and IFTPP), or not being able to stop the pattern from repeating a fourth time (AttNHP).
427   [Added] We also examine the performance of our DLHP on randomly instantiated parametric
428   Hawkes processes, finding that DLHP is able to successfully recover the ground truth intensity.
429   Furthermore, in this setting DLHP achieves the best held-out log-likelihood scores and competitive
430   next-event time prediction (see Appendix D.5 for more details). These simple experiments confirm
431   that the DLHP is sufficiently expressive to be able to represent more complicated intensity functions
      while other methods break down.

      6               C ONCLUSION
506
507
      We present the deep linear Hawkes process (DLHP)—a novel combination of ideas from LHPs and
508   SSMs. Our DLHP leverages the unique properties of deep SSM architectures to achieve a flexible
509   and performant model, without additional and restrictive intensity decoding heads. We then demon-
510   strated that our DLHP outperforms existing methods across a range of standard and new benchmark
511   tasks over various metrics, such as log-likelihood and runtime across sequence lengths. One lim-
512   itation of our method is the increased complexity of the implementation [Added] (as we require a
513   parallel scan), compared to, for instance, the NHP [Added (A)] (which only requires a basic for
514   loop). Following from this, a second limitation is that we have lost most of the interpretability of the
515   latent dynamics and parameters enjoyed by the LHP. [A] While there is some limited interpretabil-
516
      ity from the mark-specific impulses due to each LLH layer incrementing the residual stream of the
      model, this is largely correlative and imprecise. Future research directions therefore include im-
517
      proving on and strengthening these aspects, as well as developing additional theory around the use
518
      of deep SSMs in this novel MTPP setting, and [Edited] developing heuristics and best-practices for
519   configuring hyperparameters in various modeling settings. [A] Specifically, exploring the relative
520   benefits of the forward and backward discretization is a unique research direction arising from the
521   DLHP. However, we believe the robustness, performance, computational efficiency, and extensibility
522   of DLHPs make them a very competitive model out-of-the-box for a wide range of applications.

      B.2    D ISCRETIZATION AND Z ERO O RDER H OLD
865
866   The linear recurrence is defined in continuous-time. This mirrors the (M)TPP setting, where event
867   times are not on a fixed intervals. We use the zero-order hold (ZOH) discretization method, to
868   convert the continuous-time linear recurrence into a sequence of closed-form updates, given the
869   integration times, that can also be efficiently computed. We refer the reader to Iserles (2009) for a
870   comprehensive introduction to the ZOH transform.
871   The main assumption of the ZOH discretization is that the input signal is held constant over the time
872   period being integrated. Under this assumption, it is possible to solve for the dynamics and input
873   matrices that yield the correct state at the end of the integration period. For the LLH dynamics in
874   Eq. (10), when no events occur in (t, t′ ), this becomes

924   B.4    F ORWARDS AND BACKWARDS Z ERO O RDER H OLD D ISCRETIZATION
925
926   In Section 3.3 we highlighted that the ZOH discretization is exact when ut is held constant over the
927   integration window. This raises a unique design question for DLHPs: what constant value should
928   ut take on when evolving x from time t to t′ ? For the first layer of the model, the input is zero
929   by construction, so there is no choice to be made—in fact, since u is constant for the first layer the
930   updates are exact. However, the input is non-zero at deeper layers, and, crucially, varies over the
931
      integration period.
932   We must therefore decide how to select a u value over the integration period. This should be a value
933   in (or function of) {us | s ∈ [t, t′ )}. Note this is because the value at t′ , ut′ , cannot be incorporated
934   as this would cause a data leakage in our model; while values prior to t would discard the most
935   recent mark. For this work, we explore two natural choices: (i) the input value at the beginning of
936   the interval, ut , and (ii) the left-limit at the end of the interval, ut′ − . We illustrate the backwards
937
      variant in Fig. 2, where in the rightmost panel, we use the ut∗ values at each layer, as opposed to
      ut3 . We refer to these options as forwards and backwards ZOH, respectively. All experiments in the
938
      main paper utilize backwards ZOH.
939
940   It is not obvious a priori which one of these modes is more performant. We therefore conducted an
941   ablation experiment in Table 11. We see that there is little difference between the two methods. We
942   also note that models are learned through this discretization, and so this decision does not mean that
943
      a model is “incorrectly discretized” one way or the other, but instead they define subtlety different
      families of models. Theoretical and empirical investigation of the interpretations of this choice is an
944
      interesting area of investigation going forwards, extending the ablations we present in Table 11.

967            re-applies the whole independently attention mechanism for each integration point. How-
968

       We apply a grid search for all models on all datasets for hyperparameter tuning. We use a default
1031

1032

1033   the models on each dataset. We use the Adam stochastic gradient optimizer (Kingma & Ba, 2015),
1034   with a learning rate of 0.01 and a linear warm-up schedule over the first 1% iterations, followed
1035   by a cosine decay. Initial experiments showed this setting generally worked well across different
1036   models and datasets leads to convergence within 300 epochs. We also clip the gradient norm to
1037   have a max norm of 1 for training stability. We use Monte-Carlo samples to estimate the integral in
1038   log-likelihood, where we use 10 Monte-Carlo points per event during training.

1040   an extended grid based on the architecture reported in the EasyTPP paper. Specifically, we search
1041   over hidden states size h = {16, 32, 64, 128, 256} for RMTPP, h = {32, 64, 128} for NHP, and h =
1042   {16, 32, 64} for IFTPP. For SAHP, THP, and AttNHP, we searched over all combinations of number
1043   of L = {1, 2, 3}, hidden state size = {16, 32, 64, 128}, and number of heads = {1, 2, 4}. Finally,
1044   for DLHP, we considered combinations for number of layers = {1, 2, 3, 4}, p = {16, 32, 64, 128}
1045   and h = {16, 32, 64, 256}. We fixed the activation function as GeLU (Hendrycks & Gimpel, 2016)

       We use the default train/validation/test splits for EasyTPP benchmark datasets. For MIMIC-II, we
       copy Du et al. (2016) and keep the 325 test sequences in the test split, and further split the 2,935 train-
1097
       ing sequences into 2,600 for training and 325 for validation. In our pre-processed datasets, Last.fm
1098
       and EHRShot, we randomly partition into subsets containing 70%, 15%, 15% of all sequences for
1099   training/validation/test respectively. We provide a high-level description of all the datasets we used,
1100   followed by our pre-processing procedure of Last.fm and EHRShot in more detail. Note that for
1101   datasets that contain concurrent events or effectively discrete times, we apply a small amount of
1102   jittering to ensure no modeling assumptions are violated in the MTPP framework.
1103
       Amazon (Ni et al., 2019) contains user product reviews where product categories are considered as
1104
       marks. Retweet (Zhao et al., 2015) predicts the popularity of a retweet cascade, where the event
1105   type is decided by if the retweet comes from users with “small”, “medium”, or “large” influences,
1106   measured by number of followers (Mei & Eisner, 2017). Taxi data (Whong, 2014; Mei et al.,
1107   2019) uses data from the pickups and dropoffs of New York taxi and the marks are defined as the
1108   Cartesian product of five discrete locations and two actions (pickup/dropoff). Taobao (Xue et al.,
1109   2022) describes the viewing patterns of users on an e-commerce site, where item categories are
1110   considered as marks. StackOverflow contains the badges (defined as marks) awarded to users on
1111   a question-answering website. Finally, MIMIC-II (Saeed et al., 2002) records different diseases
1112   (used as marks) during hospital visits of patients. We add a small amount of noise to the MIMIC-II
1113
       event times so that events do not lie on a fixed grid. Both StackOverflow and MIMIC-II datasets
       were first pre-processed by Du et al. (2016).
1114
1115   Last.fm Celma Herrada et al. (2009); McFee et al. (2012) records 992 users’ music listening habits
1116   that has been widely used in MTPP literature (Kumar et al., 2019; Boyd et al., 2020; Bosser &
1117   Taieb, 2023). Mark types are defined as the genres of a song, and each event is a play of a particular
1118   genre. Each sequence represents the monthly listening behavior of each user, with sequence lengths
1119
       between 5 and 500. If the song is associated with multiple genres we select a random one of the
       genres, resulting in a total of 120 different marks.
1120
1121   EHRShot Wornow et al. (2023) is a newly proposed large dataset of longitudinal de-identified pa-
1122   tient medical records, and has rich information such as hospital visits, procedures, and measure-
1123   ments. We introduce an MTPP dataset derived from EHRShot, where medical services and proce-
1124   dures are treated as marks, as identified by Current Procedural Terminology (CPT-4) codes. Each
1125
       patient defines an event sequence, and we retain only CPT-4 codes with at least 100 occurrences in
       the dataset. For the < 1% events of events where there are more than 10 codes at a single times-
1126
       tamp, we retain the top 10 codes with the most frequencies and discard the rest. We then add a
1127
       small amount of random noise to the event time to ensure they are not overlapping. This process
1128   ensures we still satisfy the MTPP framework, and can reasonably instead compute top-10 accuracy
1129   for the next mark prediction. Other work has considered extending the MTPP framework to con-
1130   sider simultaneous event occurrence (Chang et al., 2024). Then we standardize each sequence to
1131   start and end with start and end of a sequence events. Note that we do not score these events. Event
1132   times are normalized to be in hours. We discard sequences that have less than 5 events and a single
1133   timestamp. This leads to the final version of our dataset to have 668 marks, and the sequence lengths
       range from 5 to 3955 events, reflecting patient histories that can span multiple years. We include the

1460   To further probe the models, we evaluate the calibration metrics of MTPPs that are proposed in liter-
1461   ature (Bosser & Taieb, 2023), which has a different focus than log-likelihood-based evaluation. On
1462   a high level, calibration describes how well the uncertainty in the model is reflected in the observed
1463   data. However, a model can achieve perfect calibration by predicting the marginal distribution, so
1464   better calibration does not necessarily transform into better predictive performance. We therefore

1466
