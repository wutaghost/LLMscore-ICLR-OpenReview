                                                   Abstract
008
009
010              Backpropagation is effective for gradient computation but can requires large mem-
011
                 ory, limiting scalability. This work explores forward-mode gradient computation
                 as an alternative in invertible networks and, more generally, ones with surjective
012
                 differentials (submersive networks), showing its potential to reduce the memory
013
                 footprint without substantial drawbacks. We introduce a novel technique based on
014              a vector-inverse-Jacobian product that accelerates the forward computation of gra-
015              dients compared to naı̈ve forward-mode methods while retaining their advantages
016              of memory reduction and preserving the fidelity of true gradients. Our method,
017              Moonwalk, has a time complexity linear in the depth of the network, unlike the
018              quadratic time complexity of naı̈ve forward, and empirically reduces computation
019              time by several orders of magnitude without allocating more memory. We fur-
020              ther accelerate Moonwalk by combining it with reverse-mode differentiation to
021              achieve time complexity comparable with backpropagation while significantly re-
022
                 ducing its memory footprint in some network architectures. Finally, we showcase
                 the robustness of our method across several architecture choices. Moonwalk is
023
                 the first forward-based method to compute true gradients in submersive networks
024
                 in computation time comparable to backpropagation and using significantly less
025              memory.

      1     Introduction
030
031   In recent years, the evolution of deep learning models has been significantly influenced by the
032   advent of automatic differentiation (AD) packages (Paszke et al., 2019; Bradbury et al., 2018; Abadi
033
      et al., 2015), facilitating expedited model construction and research. The most commonly used
      differentiation algorithm is backpropagation (Backprop), which effectively addresses the challenge
034
      of time-efficient gradient computation, but fails to tackle the issue of memory consumption (Gomez
035
      et al., 2017; Chakrabarti & Moseley, 2019). The issue is that, during the construction and reversion of
036   the computation graph, Backprop retains the value of many intermediate computations (“activations”)
037   acquired during the forward execution phase, as needed for the backward phase. For large networks,
038   the memory footprint scales with the number of activations (Novikov et al., 2023) which can result
039   in significant memory overhead limiting the ability to scale neural networks.
040
      An alternative to traditional Backprop is forward-mode gradient computation (Forward), a long-
041
      established concept in training neural networks (Williams & Zipser, 1989) but one that remains less
042   widely adopted in practice due to its very high computation time. The notion of employing forward
043   differentiation has recently gained attention as a promising strategy for alleviating layer activation
044   memory constraints inherent in Backprop (Silver et al., 2021; Baydin et al., 2022; Fournier et al.,
045   2023). However, a primary drawback of forward gradient computation lies in its requirement to com-
046   pute full Jacobian matrices for every layer of the computation graph, proving more computationally
047   expensive than the vector-Jacobian products (vjp) used in Backprop.
048   To address this challenge, one potential approach involves projecting true gradients onto a subspace
049   and computing only these projections in forward-mode (Baydin et al., 2022). Some prior works
050   employ projections onto random subspaces, thereby introducing variance in gradient estimation
051   that limits applicability in large networks (Silver et al., 2021). Other works predict the gradient
052   direction based on auxiliary networks or past gradients and use it as a preferred projection subspace,
053   showing promise in reducing variance but to-date falling short of the end-to-end training accuracy
      of Backprop (Fournier et al., 2023).

054
      In this work, we identify a novel mathematical identity in computing gradients of invertible networks
055   and, more generally, of networks whose layers have differentials that are everywhere surjective,
056   defined in section 3.1 as submersive networks. This identity allows significant savings in memory and
057   time when computing true gradients in forward-mode without projection. Our method, Moonwalk,
058   relies on the observation that, once we obtain the gradient of the objective with respect to just the input
059   of the first layer, we can efficiently compute the remaining gradients using a vector-inverse-Jacobian
060   product (vijp) operator in forward-mode. To our knowledge, Moonwalk is the first forward-mode
061   differentiation method that can outperform Backprop in both time and memory requirements when
062   computing true gradients in submersive networks.
063   Moonwalk computes gradients in two phases. In the first phase, it computes the gradient of the
064   objective (the loss) with respect to the first layer’s input. In the second phase, Moonwalk uses this input
065   gradient in a forward pass to obtain each layer’s parameters gradient through an operator involving
066   vijp with respect to the layer’s input, as well as vjp with respect to its parameters. Computing the input
067   gradient in the first phase can be done in two ways: pure-forward via forward-mode differentiation, by
068   computing the full Jacobian of just the input, which is typically much smaller than the Jacobian of the
069
      entire network; and mixed-mode, by computing the input gradient in reverse-mode. Pure-forward
      Moonwalk is significantly faster than full Forward differentiation, and potentially fast enough to
070
      be worth the immense memory saving over Backprop, particularly when the dimensionality of the
071
      input is small. When the input is high-dimensional, mixed-mode Moonwalk is preferred, greatly
072   accelerating the computation compared to full Forward at the cost of more memory than pure-forward
073   Moonwalk, but still less memory than full Backprop.
074
075
      In summary, this work contributes two novel automatic differentiation methods for invertible and,
      more generally, submersive networks that compute full gradients in forward-mode:
076
077          • Pure-forward Moonwalk, an entirely forward-mode method that significantly reduces the
078            time requirements of naı̈ve forward differentiation and is the first forward-mode method to
079            feasibly address the memory challenge of Backprop, being particularly fast when the input
080            dimension is very small; and
081          • Mixed-mode Moonwalk, a variant implementing the first phase of Moonwalk in reverse-
082            mode for further acceleration to achieve time complexity comparable with Backprop, while
083            maintaining a smaller memory footprint than Backprop.

087
      Reducing memory. Previous studies have used checkpointing to alleviate the memory footprint of
088   neural networks (Martens & Sutskever, 2012; Chen et al., 2016; Gruslys et al., 2016; Kumar et al.,
089   2019; Zhao et al.,                                                                    with L layers

090   by a factor of L through the selective storage of activations at intervals of L layers and their
091   forward-mode recomputation between checkpoints occurs as they become needed during Backprop.
092   This can be viewed as an equivalent network with fewer layers whose Jacobian-vector products are
093   harder to compute, and our result applies equally to these networks as they scale up.
094
      Invertible architectures. Recently, invertible (also known as reversible) architectures have gained
095
      significant attention owing to their diverse applications in reducing memory (Gomez et al., 2017;
096   MacKay et al., 2018; Mangalam et al., 2022), enhancing learned representation (Jacobsen et al.,
097   2018), boosting performance (Kingma & Dhariwal, 2018), and generative modeling (Dinh et al.,
098   2014; Rezende & Mohamed, 2015). One key benefit of these architectures is the ability to avoid
099   storing activations altogether. When the input to each layer can be computed from its output using
100   the inverse function, several methods recompute activations backward during Backprop (Gomez
101   et al., 2017; MacKay et al., 2018; Mangalam et al., 2022). For example, Bulo et al. (2018) replaced
102   ReLU and batch normalization layers with invertible variants, reducing memory usage by up to
103   50%. Additionally, Jaderberg et al. (2017) used synthetic gradients for efficient activation storage,
104
      leveraging pre-trained gradient estimators.
105   Forward propagation. The concept of learning neural network weights in a forward fashion has
106   been originally explored in the real-time recurrent learning (RTRL) algorithm (Williams & Zipser,
107   1989), similar to forward-mode AD. This idea becomes more attractive when directional derivatives
      (effectively, gradient projections) are employed to eliminate the additional computation time of full

111   using local greedy loss functions and Fournier et al. (2023) employed local auxiliary networks

      3.1        Notation
118
119   Consider a neural network fθ : Rn → Rk with L layers and parameters θ = {θi }i=1,...,L . Where
120   |θi | = di is a parameter of a layer. We will denote the output of layer i ∈ {1, ..., L} by xi =
121   fi (xi−1 ; θi ) ∈ Rni , where x0 ∈ Rn is the input to the network. Let Jθ (xL ) = J(fθ (x0 )) be the
122   scalar loss function, whose gradient with respect to θ we wish to compute as part of a gradient-based
123   optimization algorithm.
124   Definition 1 (Submersion). A differentiable function f : Rn → Rk is a submersion if its differential
125   d f (x) is surjective for all x ∈ Rn .
126
127   Smooth submersions are useful in differential topology, where they are defined more generally
128
      for differentiable maps between differentiable manifolds, but we focus on differentiable functions
      between vector spaces, where the differential is simply right-multiplication by the k × n Jacobian
129   ∂f/∂x. A submersion then has k ≤ n and a Jacobian that is surjective (right-invertible) for all inputs.
130
      We call a neural network submersive if all its layers are submersions for any value of their parameters.
131   Note that invertible networks are all submersive, because invertible layers have invertible Jacobians,
132   but not all submersive networks are invertible.
133
134

      4.2   Mixed-Mode Moonwalk
253
254
255
      Alternatively, we can use Backprop to compute h0 . While it may seem counterintuitive that a
      method including this phase could improve on full Backprop, the key insight is that only part of
256
      the computation graph is downstream of x0 . When using Backprop for h0 , we can therefore avoid
257
      storing activations computed from parameters θi independent of any xi . Another consideration, more
258   practical than principled, is that many Backprop implementations store all gradients before applying
259   any of them, even when the parameters are layer-wise disjoint and can be updated immediately during
260   Backprop and their gradients discarded. Here, in contrast, computing only the input gradient avoids
261   parameter gradients altogether. In many network architectures of interest, these distinctions lead to
262   significant memory savings in Backprop when only computing the input gradient. Figure 1b shows
263   the flow of this procedure.
264   Computing h0 via Backprop reduces the time complexity by a factor of up to n, if d = O(n),
265   compared to forward-mode (see Table 1), but may increase the memory footprint, which in the worst
266   case can now be as high as Backprop’s. However, depending on the network architecture, as analyzed
267   in the next section and demonstrated in Section 6.2, real-world architectures are often much better
268   than this worst case, leading to significant memory saving over Backprop. We also note that, when
269   a checkpointed implementation of Backprop can reduce the effective number of layers, mixed-mode
      Moonwalk can use it as well.

      5   Complexity Analysis
271
272
      While estimating the exact time and memory consumption of different methods for computing the
273   gradients hugely depends on the choice of the network’s architecture and the detailed implementation,
274   in this section, we will provide an asymptotic analysis, in terms of the main architectural parameters,
275   of the time and memory complexities of our methods and compare them with related previous
276   works (Table 1). We omit all methods’ linear dependence in time and memory on the mini-batch
277   size. We analyze the computational complexity of the following methods:
278
279
            1. Backprop: Throughout the forward pass, all (checkpointed) activations are cached, and
               subsequently, during a backward pass, gradients for each layer are computed using vjp.
280
281         2. Forward: During the forward pass, complete Jacobians for each layer are computed using
282            jvp. In practice, a separate forward pass is used for each column to reuse memory.
283         3. ProjForward: In Projected Forward (Baydin et al., 2022), parameter gradients projected in
284            a random or predicted direction are obtained using jvp concurrently with the forward pass.
285         4. RevBackprop: In Reversible Backprop (Gomez et al., 2017), no activations are stored
286            during the forward pass. In a subsequent backward pass, the output of each layer is used to
287            compute its input via the inverse function, as well as its parameter gradient via vjp.
288         5. Moonwalk: Initially, the input gradient is computed using Forward. Then parameter
289            gradients are obtained using vijp and vjp in a second forward pass (Section 4.1).
290
            6. Mixed: Similar to Moonwalk, but the input gradient is computed using Backprop, to reduce
291            computation time at the expense of some memory impact (Section 4.2).
292

439

      time, achieving efficiency up to several orders of magnitude compared to Forward. We evaluate all

444
445   The results, illustrated in Figure 4, accentuate the notable efficiency gains achieved by Moonwalk.

455   6.4   Time-Memory trade-off
456
457
      While achieving noteworthy time reduction using Moonwalk, we can further optimize efficiency
      by precomputing the gradients of the input, denoted as h0 , in our method using backpropagation.
458
      Although the memory reduction in this approach is constrained by the memory required for activa-
459
      tions, it significantly decreases overall memory consumption compared to Backprop. Two pivotal
460   components contribute to this reduction:
461
462   Elimination of Activations Storage: We refrain from storing activations utilized in computing the
463
      gradients of the weights Mθ L. However, it is important to note that in this scenario, we are still bound
      by the activations necessary for computing h0 . In instances where input is padded to 32x32x8 (see
464
      Figure 7), we do not incur a memory disadvantage compared to alternative methods. In scenarios
465
      featuring larger input size 32x32x18, as depicted in Figure 7, a noticeable increase in memory usage
466   over Forward becomes apparent.
467
468   Sequential Memory Usage: There is no longer a need to store gradients and activations simultane-
469
      ously, as Backprop does. After precomputing h0 , we discard all activations, freeing up memory for
      gradient computations.
470
471   In Figure 4, Mixed is only about twice as slow as Backprop. The primary advantage of this method
472   lies in its substantial time reduction compared to Forward.

518   7     Conclusion and Future work
519
520   The efficiency of memory utilization in Forward surpasses that of Backprop, thereby alleviating
521   the challenges associated with activation storage constraints. However, this enhanced memory effi-
522   ciency comes at a considerable computational cost, rendering Forward computationally impractical.
523
      To address this limitation, we augmented Forward processing through the incorporation of invert-
      ible networks. Our experimental findings reveal that our proposed method, Moonwalk, exhibits a
524
      marked acceleration, surpassing Forward by several orders of magnitude. Furthermore, we proposed
525
      an approach to mitigate the computational overhead associated with our method. By integrating
526   Backprop for the computation of input gradients, we achieved a substantial acceleration of overall
527   gradient computation.
528
529
      In this paper, we restricted our attention to the class of invertible and submersive layers in order
      to compute the gradients for the next layer given the previous one. Future work would study the
530
      effect of using projected forward-mode gradients instead of the full forward-mode. Also, one could
531
      investigate the class of neural networks composed of locally invertible layers to shed light on the
532   applicability of Moonwalk for a larger family of architectures.

      8.2   Numerical Stability
662
663   It is worth noting that RevBackprop (MacKay et al., 2018), while requiring a bijective network with
664   similar computation time, encounters challenges in terms of numerical stability. As demonstrated
665   by (Gomez et al., 2017) and (Jacobsen et al., 2018), invertible architectures with finite precision
666   encounter numerical instability issues, leading to divergence from true gradients and convergence
667   to a different local minimum than Backprop. In this study, we highlight that RevBackprop fails
668   to converge when an activation function is added to the output of the layer. Employing the same
669   network and dataloader for simultaneous training, Fig 3 illustrates that both Backprop and Mixed
670
      achieve perfect accuracy, while RevBackprop fails to converge. This divergence is attributed to the
      error accumulating with each update.
671
672   Fig 3 displays the gradient error estimation between the algorithm’s gradients and oracle gradients
673   at each step using the same parameters. The experiment reveals the accumulating error causing
674   the network weights to drift away from true gradients. Mixed exhibits more stable convergence
675   due to lower gradient error. Such disparity is a result of different approaches involved in gradient
676
      computation. In order to compute the gradients, RevBackprop requires computing the inverse of
      the function, which might be extremely unstable, as in the case with a tanh activation function. As
677
      opposed to RevBackprop, Moonwalk and Mixed methods only require computing inverse-vector
678
      Jacobian product and avoid computation of the inverse function itself. In some cases, as we have
679   shown with tanh, this tends to be the more stable approach to computing gradients. The impact of
680   ”gradient drifting” in the reversible method becomes noticeable after 30 epochs.

      8.4   Implementation
757
758   We implement all methods using JAX Bradbury et al. (2018). Forward involves applying the
759   jax.jvp operator sequentially at each layer, with the number of applications corresponding to
760   the number of parameters in that layer. For each layer of the network, we define both forward
761   and inverse functions. To enhance computation speed, we flatten all layer parameters and create
762   an identity matrix used for projection. Utilizing a batched function map accelerates computation,
763   although it comes at the cost of an increased overall memory footprint for forward-based methods.
764
      An alternative to mitigate the increased memory footprint is to compute the jax.jvp product one
      by one without creating a large identity matrix. While this approach reduces the memory footprint,
765
      it also significantly increases compilation time and affects overall computation, prohibiting the use
766
      of jax.vmap in such scenarios. The depicted performance of Mixed in the following subsections
767   is not optimized, though it is already only twice as slow compared to Backprop. Some of the factors,
768   including data transfer between compiled and non-compiled environments, significantly affect its
769   performance. In theory, it might be possible to make it faster than Reversible.
770
771   8.5   Stochasticity of ProjForward
772
773   ProjForward demonstrates the most conservative memory usage; however, it stands as the sole
774   method not generating true gradients. Instead, it introduces variance in its estimation owing to the
775   adoption of an arbitrary random distribution for projecting the parameter space. However, it has
776
      been substantiated to be advantageous in specific scenarios, particularly when a reliable estimate for
      the gradient direction is available (Silver et al., 2021; Baydin et al., 2022; Fournier et al., 2023).
