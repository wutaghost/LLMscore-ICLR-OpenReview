009                                               A BSTRACT
010
011            Visual projector plays a crucial role in bridging the visual model and the large
012            language model (LLM) in modern multimodal LLM. Most mllms use a simple
013            MLP to project all visual features into visual tokens, causing a heavy computational
014            burden and redundant visual tokens. In order to solve this problem, some efficient
015            visual projectors, e.g., the resampler or the adaptive pooling, are developed to
016            reduce the visual tokens. However, they only reduce the visual tokens based on
017            the image feature, leading to the feature misalignment between visual tokens and
018
               text tokens. In this paper, we present a novel Language-guidance Visual Projector
               (LVP), where the text feature serves as a guide to selecting the important visual
019
               tokens. Specially, we first adopt a lightweight text encoder to extract the text
020
               feature. Then, a lightweight cross-modal feature enhancement module is proposed
021            to enhance the cross-modal feature alignment. Finally, we select the important
022            visual tokens according to the feature similarity between visual tokens and text
023            tokens and apply a deformable attention module to integrate the visual features
024            from the visual encoder into the selected visual tokens. We further propose a
025            multi-level language-guidance visual projector, which selects the visual tokens
026            from different stages of the visual encoder. Extensive experiments demonstrate that
027            our LVP compresses the visual tokens by more than 75% while achieving the best
028            performance among the existing visual projectors. For instance, LLaVA1.5-LVP
029
               with Qwen2.5-14B obtains 72.4% accuracy on VQAT , realizing the state-of-the-art
               result. The code and the model will be released.

      1   I NTRODUCTION
033
034
      Large Language Models (LLMs) (Touvron et al., 2023b;a; Achiam et al., 2023; Bai et al., 2023a)
035
      have made significant progress in recent years, promoting the rapid development of Multimodal
036   Large Language Models (MLLMs). The main idea for MLLMs is to employ a visual projector to
037   bridge the visual model and the LLM and train the visual projector using multimodel data while
038   keeping the parameters of the visual model and LLM. Such a simple paradigm enables MLLMs to
039   preserve and utilize the pre-training knowledge of visual model and LLM, making MLLMs show
040   a strong capability in vision-language reasoning (Liu et al., 2024b), understanding (Alayrac et al.,
041   2022), and interaction capabilities (You et al., 2023).
042   The efficiency of MLLM gains more and more attention due to the limit of compute resource in
043   the practical use. The recent works (Li et al., 2023c; 2024c) show that LLM dominates the major
044   computational resource and the number of input tokens directly affects the efficiency of LLM.
045   Meanwhile, the number of visual tokens is much more than the number of text tokens in MLLMs.
046   Reducing the number of visual tokens outputted by the visual projector is an effective way to improve
047   efficiency. Besides, the quality of visual tokens affects the overall performance of MLLMs. Therefore,
048   a visual projector, generating fewer but better visual tokens, is important for efficient MLLM.
049   Current research on the visual projector can be summarized into two lines: learnable query-based and
050   linear projector-based. As for the learnable query-based methods, Q-Former (Li et al., 2023c) and
051   resampler (Bai et al., 2023b) are the typical work. Both of them utilize learnable queries to squeeze
052   and extract the visual features. However, DeCo (Yao et al., 2024a) demonstrates the training efficiency
053   of the resampler and Q-former is low when training data is limited. As for the linear projector-based
      methods, such as MLP, they map the visual contexts into visual tokens without squeezing the visual

102   2.1   M ULTIMODAL L ARGE L ANGUAGE M ODELS
103
104   Early efforts (Li et al., 2021; Tan & Bansal, 2019) construct a series of architectures for Multimodal
105   Large Language Models (MLLMs), consisting of a visual encoder and a language model. With the
106   rapid development of LLM (Touvron et al., 2023b;a; Bai et al., 2023a; Achiam et al., 2023; Bi et al.,
107   2024), many studies (Li et al., 2024a; Bai et al., 2023b; Chen et al., 2023a) focus on infusing visual
      features into LLM with a visual projector. LLaVA (Li et al., 2024a) feeds all visual tokens into

108
      LLM and trains the model via visual instruction tuning, enabling the LLM to comprehend the image
109   features and generate the correct response. MobileVLMV2 (Chu et al., 2024) proposes a 1B/3B
110   model to benefit the resource-constrained scenarios. Qwen-VL (Bai et al., 2023b) pretrains the model
111   with a large-scale dataset, effectively scaling up the MLLMs. Recent MLLMs, e.g., InternVL (Chen
112   et al., 2024b) and MiniCPMV (Yao et al., 2024b), adopt an effective visual projector to enhance the
113   model efficiency, indicating the visual projector is a significant topic to be investigated.
114
115   2.2   V ISUAL P ROJECTOR IN MLLM S
116
117   Modern MLLMs adopt the visual projector to connect the visual encoder and LLM. Early works,
118   such as the linear projector in LLaVA (Li et al., 2024a) and MiniGPT2 (Chen et al., 2023a), preserve
119
      all visual features and map them into the language space via the fully connected layer. This approach
      significantly increases the computational burden due to the generation of numerous visual tokens.
120
      To reduce training resources, some efficient visual projectors have been proposed. Q-former (Li
121
      et al., 2023c) and resampler (Bai et al., 2023b) utilize a group of learnable queries to squeeze the
122   visual features. Although such a learnable architecture reduces training resources, it underperforms in
123   scenarios with limited training data. An alternative research direction uses convolution or pooling to
124   reduce visual tokens. Abstractor (Cha et al., 2024) and LDP (Chu et al., 2023) leverage convolution
125   layers to extract visual features and output compressed visual tokens. DeCo (Yao et al., 2024a)
126   demonstrates the adaptive average pooling layer is an efficient way to compress the visual token.
127   However, these methods neglect the fine-grained information, hurting the visual reasoning capabilities
128   of MLLMs. TokenPacker (Li et al., 2024c) and MiniGemini (Li et al., 2024d) address this by
129   employing cross-attention layers to inject fine-grained information from high-resolution images into
130
      compressed visual tokens. Nevertheless, their approaches focus on local regions, overlooking the
      global information, leading to suboptimal performance in learning global semantic features. Other
131
      approaches, such as Pixel-Shuffle (Chen et al., 2024a) and nearby concatenation (Dong et al., 2024b),
132
      directly permute the length dimension and the channel dimension, distorting intrinsic characteristics.
133   In contrast to the existing methodologies, our LVP treats the text feature as an effective guide to select
134   the important visual tokens.

      3.1   OVERVIEW
139
140   The goal of the Multimodal Large Language Model (MLLM) is to generate the response corresponding
141   to the input instruction. In this paper, MLLM receives the image and text (instruction) as the inputs
142   and outputs the text (response) in an autoregressive manner. Formally, the multimodal input token
143   consists of two types: image token Ximg and text token Xtext . Then, the large language model
144   (LLM) generates the response Y = {gi }L    i=1 conditioned on the Ximg and Xtext , where L is the
145   number of tokens in the response. The process of multimodal generation can be formulated by
146                                                    L
                                                       Y
147

148                                                    i=1
149
150   where p denotes the conditional probability.
151
      Model architecture. The architecture of MLLM is composed of three parts: visual encoder, visual
152
      projector, and LLM. The visual encoder outputs a sequence of image features. The visual projector
153
      translates the image features into a sequence of image tokens that LLM can interpret. LLM processes
154   the text token and image token and generates the response autoregressively. In MLLM, the efficiency
155   is mainly affected by the number of visual tokens fed into LLM. To improve the efficiency of MLLM,
156   effective visual projectors are developed to reduce the visual tokens.
157

159
160   The role of the visual projector is to bridge the visual encoder and LLM. As described in (Li et al.,
161   2023c; Cha et al., 2024; Li et al., 2024c), the number of visual tokens affects the overall efficiency
      of MLLM. Considering the scenarios of processing multiple images and large images, numerous

      the features of foreground objects in the image, such as the surfer in this picture, but this causes the
      visual projector to ignore the important background information contained in the text (instruction),
187
      resulting in misalignment between the visual tokens and the text tokens.
188
189   Stemming from the above analysis, we propose a novel Language-guide Visual Projector (LVP)
190   to align the visual tokens and text tokens. LVP follows two key principles: 1) effective alignment
191   between visual tokens and text tokens. 2) flexibility over the number of visual tokens. Our LVP
192
      can not only determine the number of the visual tokens fed into LLM flexibly to improve the
      computational efficiency but also boost the overall performance of MLLM by aligning the visual
193
      tokens and text tokens.

196   3.3   L ANGUAGE - GUIDE VISUAL PROJECTOR
197
198   Architecture. The overall architecture of our MLLM is shown in Figure 3. LVP consists of three
199   parts: text encoder, cross-modal feature enhancement, and language-guide visual token selection.
200   Specifically, given an input text (instruction) and an image, the visual encoder outputs the image
201
      feature XI ∈ RNI ×D , where NI and D denote the number of image tokens and dimension of
      XI . Text encoder is composed of two transformer layers and each transformer layer contains a
202
      self-attention layer and a feed-forward network (FFN). Experimental results (see Table A1 in the
203
      Appendix) demonstrate that such a lightweight text encoder is enough for text feature extraction.
204
205   Cross-modal feature enhancement. Inspired by GLIP (Li et al., 2022), LXMERT (Tan & Bansal,
206   2019), and Grounding-DINO (Liu et al., 2023b), we introduce a lightweight Cross-modal Feature
207
      Enhancement module (CFE) to prompt the efficiency of cross-modal feature learning. CFE includes
      an image-to-text attention module and a text-to-image attention module. As depicted in Figure 3, the
208
      process of the image-to-text attention module can be formulated by Xto = Attention(XI , XT , XT ),
209
      where Xto is the enhanced text feature, XT ∈ RNT ×D is the text feature from the text encoder,
210   NT is the number of text tokens, and Attention(Query, Key, V alue) represents a standard cross-
211   attention module. In the same way, the process of the text-to-image attention module can be expressed
212   by Xio = Attention(Xto , XI , XI ), where Xio ∈ RNI ×D stands for the enhanced image feature.
213   LXMERT adopts a similar cross-modal encoder to enhance cross-modal feature learning. Our CFE
214   differs from it in two aspects: 1) Image-to-text attention and Text-to-image attention in LXMERT
215   are parallel, while our CFE is a sequential structure. 2) The cross-modality encoder in the LXMERT
      structure is much heavier than CFE. Experimental results (see Table A4 in the Appendix) show

261   3.4   M ULTI - LEVEL LANGUAGE - GUIDANCE VISUAL PROJECTOR
262
      To further improve the performance of MLLM, we propose a multi-level language-guide visual
263
      projector. Visual features from different stages of the visual encoder represent different visual
264   information, e.g., visual features from the shallow stage contain rich detailed features while visual
265   features from the deep stage tend to represent the global semantic feature. Specifically, we first divide
266   the layers of the visual encoder into four stages following TokenPacker (Li et al., 2024c). Then, for
267   each stage, we select the top Na visual tokens as Eq. 2. The total number of visual tokens fed into
268   LLM is Na × 4. Finally, all selected visual tokens are concatenated along the feature dimension. In
269   this way, the visual tokens include both detailed features and global semantic features. The overall
      pipeline of the multi-level language-guide visual projector is shown in Figure 4.

      We evaluate our language-guide visual projector under the normal resolution and high resolution

530   5    C ONCLUSION
531
532   We introduce a novel Language-guide Visual Projector (LVP) for efficient MLLM. LVP adopts the
533   text (instruction) feature as the guidance to select the important visual tokens, effectively reducing
534   the visual tokens while aligning the visual tokens fed into LLM with text tokens. To make full use
535   of the features from the different stages of the visual encoder, we further propose a novel multi-
536   level language-guide visual projector. Experimental results show that LVP achieves state-of-the-art
537   performance among existing visual projectors. Notably, InternLM-XC2-LVP establishes the best
538
      performance on MMDU benchmark with much fewer visual tokens.

812   A.1   A DDITIONAL IMPLEMENTATION DETAILS
813
814   Implementation of attention map visualization. In this section, we describe the implementation
815   of attention map visualization in detail. We adopt an approach similar to R-GAE in DeCo (Yao
816   et al., 2024a). Specifically, we first construct a Text-to-Visual map Mt ∈ RNI ×Nq . Mt is initialized
817   to an identity matrix. For each layer in the projector, an attention map is obtained by utilizing the
818   gradients to average across the attention heads for the resampler (Bai et al., 2023b) and LVP. For
819
      Linear projector and LDPv2 (Chu et al., 2024), an attention map is obtained by adopting the gradients
      of each layer. For generation time step t, we can propagate the Mt from the projector’s first layer to
820
      its last layer. Finally, we average the step t and average the Mt to get the final attention map.
821
822   Implementation of high-resolution. We take the high-resolution image processing in LLaVA-
823   HD (Liu et al., 2023a) as our high-resolution image processing method. Given a high-resolution
824   image, LLaVA-HD first splits the image into different patches and each patch is fed into the visual
                                                                                     N
825
      encoder. The visual encoder outputs a sequence of visual tokens. We use Pi=1       to represent the
      sequence of visual tokens and N is the number of patches. Besides, LLaVA-HD resizes the original
826
      high-resolution image to the size the visual encoder can process. Here we use PH the denote the the
827                                                                                  N
      visual token of the high-resolution image. Finally, LLaVA-HD concatenate the Pi=1  and PH . We use
828   PC to stand for the concatenated visual tokens. PC is the visual input of our LVP. We use the text
829   feature as a guide to select the Top Nq (Nq is much smaller than the number of PC ) visual tokens
830   from PC based on the similarity between visual features and text features.

904
      64, 16, 16, and 16, respectively. The visualization results are displayed in Figure A1. We can see that

905
      "surfer". However, from the attention map of the linear projector, we can find that "wave" should be
906
      allocated several visual tokens. As for our LVP, it can be observed that the proportion of the visual
