024                                                  A BSTRACT
025
026             We investigate the internal representations of autoregressive vision-and-language
027             models (VLMs) and how they encode task representations. We consider tasks
028             specified through examples or instructions, using either text or image inputs. Sur-
029             prisingly, we find that conceptually similar tasks are mapped to similar task vector
030             representations, regardless of how they are specified. Our findings suggest that to
031
                output answers, tokens in VLMs undergo three distinct phases: input, task, and an-
                swer, a process which is consistent across different modalities and specifications.
032
                The task vectors we identify in VLMs are general enough to be derived in one
033
                modality (e.g., text) and transferred to another (e.g., image). Additionally, we find
034             that ensembling exemplar and instruction based task vectors produce better task
035             representations. Taken together, these insights shed light on the underlying mech-
036             anisms of VLMs, particularly their ability to represent tasks in a shared manner
037             across different modalities and task specifications.

040   1     I NTRODUCTION
041
042   Many modern vision-and-language models (VLMs) are designed as autoregressive models that
043   tackle various computer vision tasks through text. For example, tasks like image recognition, OCR,
044
      and object detection can be formulated as visual question answering (Antol et al., 2015) and solved
      with textual outputs (Alayrac et al., 2022; Lu et al., 2022; Liu et al., 2023a).
045
046   Despite their success, the underlying structures and inductive biases that drive such VLMs remain a
047   mystery. This urges us to ask what representations enable VLMs to process multi-modal inputs to
048   answer questions. We investigate a specific type of representation known as task vectors, which have
049   been studied in language-only (Hendel et al., 2023; Todd et al., 2024) and vision-only models (Hojel
050
      et al., 2024). These studies observe that models conditioned on in-context learning (ICL) examples
      contain token representations that encode task information.
051
052   In this work, we discover that VLMs encode tasks within a shared embedding space, where similar
053   tasks are clustered together regardless of how they are specified. We examine tasks that can be
      defined through either text or image examples, as well as instructions. For instance, the task of

      More specifically, we investigate task vectors in VLMs and demonstrate that they are cross-modal,
      allowing task representations to transfer between modalities (see Figure 2). Our analysis further
075
      reveals that as VLMs generate answers, token representations evolve across model layers in a con-
076
      sistent pattern: starting with the literal input, transitioning to the task representation, and finally, con-
077   verging to an answer. This suggests that not only are task representations similar across modalities
078   but the entire process of answer generation may be shared, despite differences in task specification.
079
080
      Motivated by this similarity between the token representations regardless of the input modality,
      we quantitatively evaluate the cross-modal transfer performance of task vectors for early-fusion
081
      and late-fusion VLMs on a range of tasks. For text-to-image transfer, cross-modal patching can
082
      improve over text ICL in the same context window by as much as 33%. Ensembling text instructions
083   with examples can improve the sample efficiency of the task vector, with an 18% performance
084   improvement over examples alone in the low-data regime. Surprisingly, we also find that task vectors
085   are transferable between the base LLM and the fine-tuned VLM, meaning that the VLM is able to
086   re-purpose functions learned in a language-only setting on image queries.
087
      Our contributions are threefold. First, we illustrate a taxonomy of task vectors, where they can be
088
      specified not only via examples as studied in prior work but also instructions. Second, we show that
089   VLM representations evolve in a common pattern regardless of the input modality or specification
090   format. Finally, we explore cross-modal transfer, which is a useful measure for the interchangeabil-
091   ity of different task representations and offers greater expressiveness when defining tasks.

      In-context learning can be formulated as follows. For a given task t 2 T , a few-shot prompt can
102   be constructed from N input-output examples pt = [(x1 , y1 ), . . . , (xN , yN )]. The model f has to
103   learn the mapping from input to output from pt and apply it onto xq . Previous work has shown that
104   large transformer models implicitly compress this function into a latent activation, also called the
105   task vector, for both LLMs (Hendel et al., 2023; Todd et al., 2024) and computer vision models (Ho-
106   jel et al., 2024). Specifically, the forward pass f (pt ) produces intermediate latent activations that
107   capture the task information, in some transformer layer l 2 L at the delimiter token between the last
      input and output (xN , yN ). Thus, the original function can be decomposed into the task vector (a

137

138

139   tasks, where we display these analogous specifications in Table 1. In this work, we categorize set-

142

143
      exemplars onto an image query. We run two forward passes: one to extract the task vector from the
144

145   layer output at the delimiter token between the last input-output pair (xN , yN ), and we inject it
146   directly at the corresponding layer and token position of the query. To obtain a good estimate of ht ,
147   we sample and average the activations from multiple task prompts, and we determine the best layer
148   l for each model via average task accuracy on the validation set. We also compare against the few-
149   shot prompting baseline, where the task specification and query are jointly fed to the transformer,
150   see Figure 2b. We explore three main cases of cross modal patching, corresponding to the different

      Identifying Three Phases. We first look at all the phases the token representation undergoes across
      model layers. We analyze Idefics2 (Laurençon et al., 2024), which supports both text and image
237
      ICL. Using logit lens (nostalgebraist, 2020), we leverage the model’s existing vocabulary space to
238
      decode the last token representation. In Figure 3 we visualize the probability the token decodes
239   to these different embedding types (input, task, and answer), where we define the tokens in each
240   category manually per task. In Figure 4 we dive into individual phases, showing the set of top-1
241   decodings for different model layers. The early layer decodes to the token auf, which in Idefics2
242   globally corresponds to the colon, or the input used for the last token. The middle layer decodes to
243   a small set of task summaries similar to those displayed in Table 2. The late layer decodes to tokens
244   that resemble the output space. We limit the visualization in both figures to the Country-Capital task
245   and provide visualizations for all tasks in Sec. A.7 of the Appendix.
246   Decoding the Task Phase. Drilling down to the task phase, we take the token representation at a
247   middle layer and average it across multiple runs, then depict the top-5 decodings in Table 2. We
248   find that task vectors defined in either modality often decode into meta-tokens that summarize the
249   task. The text-only case is consistent with prior work (Hendel et al., 2023; Todd et al., 2024) that
250   investigates such decodings in language models. For example headquarters, currency, and species
251   are the top-1 decodings for both text and image ICL in the first three tasks in the table. In the case
252   of image ICL, this alignment with language is not immediately obvious. Prior work has shown
253
      the input image and text embeddings are quite different, i.e., these embeddings exhibit low cosine
      similarity (Lin et al., 2024) and form distinct PCA clusters (Liang et al., 2024). Even more, the
254
      decodings for image ICL are often noisier than text ICL, which suggests that cross-modal patching
255
      could help convey a cleaner expression of the task.

337   3.1    T EXT ICL T RANSFER
338
339   Quantitative Evaluation. Recall Sec. 2, where we observe that whether the same task is repre-
340   sented via text or image samples, the model compresses these demonstrations into interpretable task
341   vectors. With this in mind, can we provide demonstrations using only text and apply them to an
342   image query? We evaluate this transfer setting in Table 3 and show qualitative results in Figure 5.
343
      We find that cross-modal patching performs the best across all VLMs (Text ICL xPatch). Patching
344
      performs 14-33% better than providing the examples in the same context window (Text ICL xBase).
345   In fact, Text ICL xBase struggles to even execute the task on the image query, which performs at
346   most 4% better than Random. One possible explanation is that mixed-modal examples are relatively
347   out-of-domain whereas decomposed task vectors are more in-domain for the model.
348
      The cross-modal text examples are more helpful than the unimodal image examples, with Text ICL
349
      xPatch outperforming the strongest image ICL baseline (Image ICL Base, Patch) by 8-13%. We
350
      hypothesize that image ICL requires an additional visual recognition step to understand the task
351   compared with text ICL, which may lead to noisier task representations (see Table 2).
352
353   LLM to VLM Transfer. Given that many VLMs are initialized from a pre-trained LLM, we explore
354
      the extent to which the task representations are preserved after fine-tuning. We illustrate the transfer
      setting for the base LLM task vectors in Figure 6 and report quantitative results in Table 4. We limit
355
      this evaluation to the late-fusion models with a corresponding LLM, where LLaVA-v1.5 corresponds
356
      to Vicuna (Chiang et al., 2023) and Idefics2 corresponds to Mistral (Jiang et al., 2023).
357
358   We find that given the same text ICL examples, the base LLM and VLM produce highly similar
359   task vectors. The task vectors have a cosine similarity of 0.89 or more, which is much higher than
360
      the random baseline which averages the cosine similarity between all mismatched pairings of task
      vectors in Idefics2. Motivated by this observation, rather than transferring text ICL task vectors to
361
      image queries in the same model (VLM-VLM xPatch), we evaluate inter-modal transfer (LLM-
362
      VLM xPatch). Surprisingly, the LLM-VLM setting performs 1-5% better than the VLM-VLM
363   setting. This result suggests VLMs can reuse functions learned only in language by LLMs, and that
364   some elements of the base LLM’s task representation space may be retained after fine-tuning.

      4    R ELATED W ORK
487
488
      Mechanistic Interpretability. The goal of mechanistic interpretability in deep learning is to make
489   deep models more transparent and interpretable by understanding how and why model decisions are
490   made (Gilpin et al., 2018; Gurnee & Tegmark; Liu et al., 2022; Geva et al., 2020; Nanda et al., 2023).
491   To uncover the relationships within the model, causal interventions (Pearl, 2022) are often used. For
492   example, Activation Patching (Zhang & Nanda, 2023) is a technique used to modify neural network
493   activations to observe changes in outputs, often with causal insights to correct biased or erroneous
494   behavior (Meng et al., 2022; Bau et al.). Here, we use Activation Patching to demonstrate that task
495   representations transfer across modalities, regardless of being specified by examples or instructions.
496   In Context Learning. With the recent advent of LLMs (Brown et al., 2020), researchers have sought
497   to explain in-context learning (Liu et al., 2023b), the phenomenon in which LLMs can adapt to new
498   tasks with a few input examples in the forward pass. Olsson et al. (2022) hypothesized that ICL
499   is driven by attention heads (“induction heads”), while Xie et al. (2021) interprets ICL as implicit
500   Bayesian Inference process, and Garg et al. (2022) showed that ICL can emerge in the simple case
501   of linear functions. More recently, Hendel et al. (2023) and Todd et al. (2024) hypothesized that ICL
502   creates task (or function) vectors, latent activations that encode the task in LLMs, and Hojel et al.
503
      (2024) demonstrated a similar behavior in computer vision models. Huang et al. (2024) proposed
      to use task vectors in VLMs to compress long prompts that would otherwise not fit in a limited
504
      context length. We study how task information evolves within VLMs, specifically the similarity and
505
      transferability of the representation when the task is expressed in different modalities.
506
507   Vision-and-Language Models. Inspired by the success of LLMs, new vision-and-language models
508   (VLMs) have been proposed (Liu et al., 2023a; Li et al., 2023; Tong et al., 2024; Team, 2024;
509
      Laurençon et al., 2024; Zhou et al., 2024). Recent VLMs can be roughly categorized to modality
      late-fusion (Liu et al., 2023a; 2024) and early-fusion (Bavishi et al., 2023; Lu et al., 2022; 2023;
510
      Team, 2024) approaches. Late-fusion approaches typically combine a pre-trained visual encoder and
511
      LLM by training adapters, potentially with a short end-to-end fine-tuning stage. In contrast, early-
512   fusion approaches focus on end-to-end training without any pre-initialization of the representations.
513   We observe cross-modal task representations for both model categories, suggesting that this property
514   can emerge regardless of the initialization. Several works examine image ICL in VLMs, proposing
515   new models designed for ICL (Alayrac et al., 2022; Laurençon et al., 2024; Doveh et al., 2024; Jiang
516   et al., 2024) and analyzing the impact of in-context example selection on performance (Baldassini
517   et al., 2024). Our work offers a new perspective on image ICL by comparing it with text ICL and
518   demonstrating the similarity between the two processes. We even show VLMs that lack image ICL
519   capabilities (Liu et al., 2023a; Lin et al., 2023; Doveh et al., 2024) can still benefit from task vectors.
520
521   5    L IMITATIONS
522
523   In this work, we demonstrate that VLMs learn cross-modal task representations but we lack a defini-
524   tive explanation for why. Empirical studies offer several hypotheses, such as the existence of isomor-
525   phic structures between language and other perceptual representation spaces (Abdou et al., 2021;
526   Patel & Pavlick, 2022; Pavlick, 2023), or representational convergence from modeling the same
527   underlying reality (Huh et al., 2024). Additionally, we observe quantitative improvements for text-
528   to-image transfer but not image-to-text transfer, possibly because VLM training is more text-centric.
529
      However, we believe that learning task representations from visual data has its advantages, and we
      provide qualitative examples where image-to-text transfer proves beneficial.
530
531
532   6    C ONCLUSION
533
534   Vision-and-language models (VLMs) are generalist models capable of solving a wide range of com-
535   puter vision tasks by framing them as question answering problems in text. Despite their success, we
536   lack a clear understanding of how they work. Our primary observation is that VLMs map inputs into
537
      a shared task representation space, regardless of whether the task is defined by text examples, image
      examples, or explicit instructions. Based on this, we show it is possible to transfer task vectors from
538
      one modality (e.g., text) to another (e.g., images). We hope our work will inspire further exploration
539
      into the inductive biases of VLMs and the reasons behind their success.
