009                                               A BSTRACT
010
011            Query-based models are extensively used in 3D object detection tasks, with a wide
012            range of pre-trained checkpoints readily available online. However, despite their
013            popularity, these models often require an excessive number of object queries, far
014            surpassing the actual number of objects to detect. The redundant queries result in
015            unnecessary computational and memory costs. In this paper, we find that not all
016            queries contribute equally – a significant portion of queries have a much smaller
017            impact compared to others. Based on this observation, we propose an embarrass-
018
               ingly simple approach called Gradually Pruning Queries (GPQ), which prunes
               queries incrementally based on their classification scores. A key advantage of
019
               GPQ is that it requires no additional learnable parameters. It is straightforward
020
               to implement in any query-based method, as it can be seamlessly integrated as a
021            fine-tuning step using an existing checkpoint after training. With GPQ, users can
022            easily generate multiple models with fewer queries, starting from a checkpoint
023            with an excessive number of queries. Experiments on various advanced 3D de-
024            tectors show that GPQ effectively reduces redundant queries while maintaining
025            performance. Using our method, model inference on desktop GPUs can be accel-
026            erated by up to 1.31x. Moreover, after deployment on edge devices, it achieves up
027            to a 67.86% reduction in FLOPs and a 76.38% decrease in inference time. The
028            code will be available soon.

      1   I NTRODUCTION

033
      3D object detection is a key task for autonomous driving. Among various algorithms, DETR-based
      methods (Carion et al., 2020; Wang et al., 2022; Liu et al., 2022; Wang et al., 2023a;b) stand out for
034
      their end-to-end detection capabilities without relying on hand-crafted components, thanks to their
035
      set-prediction pipeline. A key feature of DETR-based models is the use of pre-defined queries in
036   transformer modules, which are generated from pre-defined reference points Zhu et al. (2020); Liu
037   et al. (2022; 2023a); Wang et al. (2023a;b); Jiang et al. (2024); Li et al. (2022); Yang et al. (2023);
038   Liao et al. (2023a;b). These queries are refined in the self-attention module and interact with image
039   features in the cross-attention module. The updated queries are then passed through MLPs to predict
040   classification scores and 3D bounding boxes.
041
      Despite their effectiveness, these methods are computationally intensive due to the large number of
042   object queries required. More precisely, the number of pre-defined queries is typically set to 300
043   for 2D object detection tasks (Meng et al., 2021; Chen et al., 2023; Gao et al., 2022), and this num-
044   ber increases to 900 for 3D detection (Wang et al., 2022; Li et al., 2023; Liu et al., 2022; Wang
045   et al., 2023a;b; Jiang et al., 2024), which significantly exceeds the actual number of objects in both
046   cases. As depicted in DETR3D (Wang et al., 2022), the performance of the model is positively cor-
047   related with the number of queries. We conducted experiments with different query configurations
048   of StreamPETR (Wang et al., 2023b), which further validates this conclusion. Specifically, as shown
049   in Table 1, the model’s performance consistently declines as the number of queries is reduced.
050   Since the number of predictions during model inference is tied to the number of queries, fewer
051   queries lead to fewer predictions. Reducing queries requires additional computation to cover the
052   solution space and may even result in failure to find optimal parameters. Therefore, using fewer
053   queries generally results in poorer performance. For instance, as illustrated in Figure 1 (a), if a
      model were to use only a single query to predict two object types – such as pedestrians and barriers

      1.1     C ONFIGURING AN E XCESSIVE N UMBER OF Q UERIES IS I NEFFICIENT
096
097   As is well known, DETR-based models use the Hungarian algorithm to match predictions with
098   ground truth labels. When the number of queries significantly exceeds the number of objects to
099   be detected, most predictions are treated as negative instances during bipartite matching. In fact, in
100   most non-specialized scenes, the number of objects to be detected is typically fewer than 100 (Caesar
101   et al., 2020; Wilson et al., 2021), implying that during the training process, the ratio of negative to
102   positive instances could reach 8:1. For these negative instances, there are no ground truth labels to
103   serve as supervision. As a result, during loss calculation, zero vectors are matched with negative
104
      predictions, progressively driving these queries to produce lower classification scores.
105   In cases where positive instances far outnumber negative ones, the selection of queries would grad-
106   ually become imbalanced. This not only results in wasted computational resources but also causes
107   the classification scores of queries that are more frequently selected as negative instances to be sig-
      nificantly lower than others. During inference, NMS-free selector directly selects predictions with

139   1.2        P RUNING R EDUNDANT Q UERIES
140
141   To address query redundancy, we propose GPQ, a method that gradually prunes redundant queries.
142   Queries with lower classification scores are considered less contributive to detection and exhibit
143   weaker representation capabilities. As a result, we remove these lower-scoring queries, retaining
144   only those with higher classification scores to ensure better performance.
145   Our method is embarrassingly simple: load a checkpoint and run the model as it should be, then
146   sort the queries by classification scores after each iteration, and remove the bottom-k queries every
147   n iterations. Compared to existing pruning methods that utilize a learnable binary mask (Yu et al.,
148   2022; Kong et al., 2022; Tang et al., 2023; Shi et al., 2023; Chen et al., 2024; Ilhan et al., 2024;
149   Khaki & Plataniotis, 2024; Wei et al., 2023; Yu & Xiang, 2023), GPQ introduces no additional
150   learnable parameters and, therefore, incurs no extra computational cost. Additionally, our method
151   allows for the creation of multiple model versions with varying numbers of queries using an existing
152
      checkpoint that contains a large number of queries. This eliminates the need to re-train the model
      with fewer queries, which would require additional time to restore performance.
153
154   Our contributions can be summarized as follows:
155
156             1. To the best of our knowledge, we are the first to address the issue of redundant queries in
157                commonly used 3D object detectors and to conduct a comprehensive analysis of the role
158
                   of queries in detection transformers. Our findings indicate that the majority of queries in
                   existing query-based methods are redundant and unnecessary.
159
160             2. We propose an embarrassingly simple yet effective strategy that gradually prunes redundant
161                queries in detection transformers, enabling us to better utilize existing pre-trained check-
                   points to reduce model complexity while maintaining detector performance.

170   2.1   DETR-BASED 3D O BJECT D ETECTORS
171
172   Based on transformers (Vaswani et al., 2017), DETR-based methods have been widely used in 3D
173   object detection tasks (Li et al., 2022; Yang et al., 2023; Liu et al., 2022; 2023a; Wang et al., 2023a;
174   Jiang et al., 2024; Li et al., 2023; Liu et al., 2023b; Chen et al., 2022; Wang et al., 2024b; Xie et al.,
175   2023). Liu et al. (2022) develops position embedding transformation to generate position-aware 3D
176   features. Liu et al. (2023a) introduces temporal modeling and generates 3D positional embedding
177   according to features of input images. Wang et al. (2023b) proposes an object-centric temporal
178   modeling method that combines history information with little memory cost. By utilizing high-
179
      quality 2D object prior, Far3D (Jiang et al., 2024) generates 3D adaptive queries to complement 3D
      global queries, which extends the detection range to 150 meters.
180
181   All these methods use similar pre-defined queries, either as learnable parameters or generated from
182   pre-defined reference points. Despite variations in sampling strategies (Li et al., 2022; Zhu et al.,
183   2020), they all share the query, key and value components, which are processed through trans-
184   former layers. While these methods share similar designs and advantages, they also face common
185
      drawbacks: high computational costs and excessive memory usage. This makes it particularly chal-
      lenging to deploy DETR-based methods on edge devices.
186

188   2.2   P RUNING T RANSFORMERS
189
190   Distillation and pruning methods have been developed to reduce the resource requirements of large
191   models (Yang et al., 2022; Li et al., 2024; Liang et al., 2023). As transformers have become increas-
192   ingly prominent in fields like natural language processing and computer vision, numerous pruning
193   methods targeting transformer models have been proposed (Michel et al., 2019; Fan et al., 2019;
194   Chen et al., 2021; Sanh et al., 2020; Liu et al., 2024; Lagunas et al., 2021; Yu et al., 2022; Kwon
195
      et al., 2022; Yu & Xiang, 2023). In particular, Michel et al. (2019) discovers that a large percentage
      of heads can be removed at test time. Fan et al. (2019) randomly drops transformer layers at training
196
      time. Chen et al. (2021) explores sparsity in vision transformers, which proposes an unstructured
197
      and a structured methods to prune the weights. While magnitude pruning methods rely on the ab-
198   solute values of parameters to determine which units to prune, Sanh et al. (2020) uses first-order
199   information rather than zero-order information as the pruning criterion. Lagunas et al. (2021) ex-
200   tends Sanh et al. (2020) to local blocks of any size. Yu et al. (2022) reduces both width and depth of
201   transformer simultaneously. Kwon et al. (2022) proposes a post-training pruning method that does
202   not need retraining. Yu & Xiang (2023) proposes an explainable method by assigning each unit an
203   explainability-aware mask. Liu et al. (2024) prunes tokens in ViT(Dosovitskiy, 2020) model, where
204   pruned tokens in the feature maps are preserved for later use.
205   Most of the aforementioned pruning methods use a binary mask to determine which parameters to
206   prune (Yu & Xiang, 2023; Yu et al., 2022; Sanh et al., 2020; Lagunas et al., 2021), which adds extra
207   training costs. Furthermore, these methods often struggle to adapt effectively to 3D object detection
208   models or show limited performance when applied to such tasks. This limitation arises from several
209   key challenges:
210   Nonexistent Pruning Targets. Many pruning methods for NLP or ViT models focus on “attention
211   head” pruning (Michel et al., 2019; Kwon et al., 2022; Yu et al., 2022). This is feasible because,
212   in these models, “attention heads” are well-defined, trainable structural components that can be
213   partially pruned using techniques like masking. However, in object detection methods, particularly
214   3D object detection, attention heads are essentially implemented as reshaping operations (Wang
215   et al., 2022; Liu et al., 2022; 2023a; Wang et al., 2023a). Modifying their number does not impact the
      computational cost. Consequently, such pruning methods are not applicable to 3D object detection.


                                                          4
      Under review as a conference paper at ICLR 2025


216
      Structural Inconsistencies. In NLP or ViT models, an important assumption is that tokens gen-
217   erated from the input simultaneously act as queries, keys, and values. This ensures the resulting
218   attention map is a square matrix with dimensions Nq × Nk . Several pruning methods depend on
219   this square attention map structureWang et al. (2024a). However, in object detection tasks, pre-
220   defined queries are commonly used, resulting in Nq ̸= Nk . As a result, the attention map becomes
221   a non-square matrix, rendering these methods unsuitable for 3D object detection models.
222
      Massive Data Differences. In ViT models, each batch typically processes a single image, producing
223   fewer than 200 tokens per image. However, in 3D object detection, the need to predict additional
224   indicators and process multi-view images significantly increases the token count. Even with lower
225   resolutions, at least 4,000 tokens are generated, and this number can exceed 16,000 when using
226   larger backbones and higher resolutions. The sheer volume of tokens presents a significant chal-
227   lenge for applying token-pruning methods to 3D object detection, as the computational cost of these
228   methods often scales with the number of tokens(Bolya et al.; Wang et al., 2024a).
229   These challenges highlight the need for dedicated pruning strategies tailored to the unique charac-
230   teristics of 3D object detection models.

      3.4       W HY NOT T RAIN A N EW M ODEL U SING O NLY A F EW Q UERIES ?
328
329   We introduce a query pruning method to reduce the computational load for DETR-based detectors.
330   A natural question arises: why not train the model with fewer queries from the beginning? On one
331   hand, as discussed in Section 1, training with a larger number of queries enhances the model’s capac-
332   ity to adapt to a diverse range of objects. By using GPQ, it becomes possible to prune a checkpoint
333   trained with many queries, producing models with different query counts. This approach offers flex-
334
      ibility for various scenarios without necessitating retraining for each specific query configuration.
335   On the other hand, a key advantage of GPQ is its ability to retain the knowledge gained during
336   training with a higher query count. This leads to more accurate predictions compared to training a
337   model from scratch with fewer queries. As illustrated in Figure 4, the distinction between pruning
338   redundant queries and training with fewer queries becomes more apparent. When redundant queries
339   are pruned, the remaining queries cluster together and continue to occupy the original solution plane.
340
      In contrast, training with fewer queries from the outset results in a more scattered distribution of
      queries, reducing their overall representational effectiveness.

482
483   To our knowledge, this is the first study to explore query redundancy in Transformer-based 3D detec-
484   tion models. As a result, it is challenging to find comparable methods for direct evaluation. However,
485   in this section, we compare our approach with ToMe (Bolya et al.), a method that shares a similar
      goal of improving efficiency. Specifically, ToMe increases the throughput of Vision Transformer


                                                        9
      Under review as a conference paper at ICLR 2025


486
      (ViT) models by dividing N tokens into two equal size groups A and B. These tokens represent
487   image features extracted by the image backbone. To merge similar tokens, ToMe calculates a simi-
488   larity matrix S of size N2 × N2 , where the (i, j)-th entry represents the similarity between token i in
489   group A and token j in group B. It then merges r token pairs with the highest similarity scores.
490
      We apply ToMe to StreamPETR, and compare its performance with GPQ in Table 4. Surprisingly,
491
      instead of accelerating computation, ToMe causes the model to run slower. We believe this is due
492
      to the overhead introduced by calculating the similarity matrix. Unlike image classification tasks,
493   which typically handle only a few hundred tokens, 3D object detection methods often generate a
494   much larger number of tokens (e.g., StreamPETR generates 4224 tokens even with a relatively small
495   image size of 704×256). This increased token size significantly amplifies the computational cost of
496   constructing the similarity matrix, making ToMe inefficient for 3D object detection tasks. Compared
497   to ToMe, GPQ not only preserves the model’s performance but also achieves faster inference speed.

534
535   In this paper, we propose GPQ, an incredibly simple yet effective pruning method that gradually
536   eliminates redundant queries in DETR-based 3D detection models, aiming to reduce computational
537   cost without compromising performance. Results on the nuScenes dataset confirm that our method
538   effectively maintains the detection performance across all evaluated models. To the best of our
539   knowledge, this is the first study to explore query pruning in query-based models, and we hope our
      work will inspire further research into pruning DETR-based models.

833   A.3   I NTEGRATING WITH T RAINING
834
835   A question may arise: for previous models with existing checkpoints, we can directly use GPQ to
836   prune an existing checkpoint. But for future methods, does GPQ still have its value? If we train a
837   model first and then prune it, why not train with fewer queries for more epochs?
838   Fortunately, GPQ can be integrated with training. As show in Table 7, we use PETR(Liu et al.,
839   2022) and StreamPETR(Wang et al., 2023b) as examples, starting with 900 queries, and gradually
840   pruning during training, ultimately reaching a state of 300 queries. With this technique, one can
841   directly train a model using GPQ without the need for an additional pruning and fine-tuning stage
842   or more epochs to achieve the same performance with fewer queries.
