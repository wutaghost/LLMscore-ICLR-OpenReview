009                                               A BSTRACT
010
011
               Understanding class-level hardness is essential for addressing class bias in ma-
012            chine learning. Traditionally, class bias has been explored with two primary ap-
013            proaches: analyzing raw input data to improve preprocessing strategies or examin-
014            ing neural network latent representations to refine model training. In this work, we
015            find that PCA-transformed spaces—despite being produced through linear trans-
016            formations—still contain substantial information about class-level hardness. This
017            suggests that, despite their distinct goals and methodologies, both PCA and neural
018            networks may encode similar features related to class bias, offering new insights
019            into the nature of class bias and how data representations are formed in both PCA
020
               and neural networks.
021            Analyzing class bias commonly involves Pearson Correlation, which assumes sta-
022            ble inputs. However, we find that class bias is a highly unstable phenomenon with
023
               respect to variables such as training time and model initialization, with class-level
               variability often exceeding the differences between classes. Together with in-
024
               creased variability in class accuracies over dataset-level ones, this suggests that
025
               current methods for addressing dataset-level variability may be inadequate for
026            handling class bias.

      1   I NTRODUCTION
031
032   Class bias, where class-level hardness is not uniform across classes, is a significant phenomenon in
033
      machine learning. This issue has traditionally been explored in the context of imbalanced datasets,
      where classes are represented by unequal numbers of samples (Kubat et al., 1997); often where the
034
      minority classes is of critical importance such as healthcare diagnostics (Fotouhi et al., 2019) or
035
      fraud detection (Makki et al., 2019). These imbalances naturally lead to models favouring majority
036   class over the minority class. However, imbalance ratio is just one of the factors responsible for class
037   bias. In fact, recent studies challenge the assumption that imbalance ratios are the primary cause of
038   class bias (Sinha et al., 2020; 2022), revealing that this bias persists even in balanced datasets (Ma
039   et al., 2023b). This insight allows studying class bias in scenarios where imbalance ratios are not a
040   confounding factor.
041
      Investigation of class bias can be performed principally at two stages: the input space and the latent
042   space. In the input space, we scrutinize raw data to discern which classes are likely to be harder to
043   learn. This approach facilitates more effective data preprocessing and reduces the risk of harm from
044   uniform data augmentation (Balestriero et al., 2022; Kirichenko et al., 2023). In the latent space, we
045   focus on the impact of various latent representations on class performance, which allows us to devise
046   specialized regularization techniques that guide neural networks toward better representations (Ma
047   et al., 2023a;b).
048   From a geometrical standpoint (Narayanan & Mitter, 2010; Fefferman et al., 2016), inference in-
049   volves disentangling class manifolds (Olah, 2014; Brahma et al., 2015). The geometric properties of
050   these manifolds influence the difficulty of class disentangling (Griffin et al., 2024), and these prop-
051   erties evolve as data progresses through neural network layers (Naitzat et al., 2020; Magai, 2023;
052   Suresh et al., 2024). As a result, metrics like curvature are ineffective at estimating class bias in
053   early layers but become strong indicators in later layers, while the opposite trend is observed for
      class separation metrics (Ma et al., 2023b).

054
      Principal Component Analysis (PCA), in contrast, is a straightforward dimensionality reduction
055   technique that projects raw data into a lower-dimensional space using linear transformations. Thus,
056   the representations generated by PCA differ significantly from those produced by neural networks
057   due to their distinct objectives. In this work, we discover that PCA-transformed spaces contain
058   valuable information about class-level hardness. For example, the curvature of class manifolds
059   within these spaces serves as a strong indicator of class-level hardness, paralleling findings in neural
060   network latent spaces. This discovery introduces new research directions in class-level hardness,
061   allowing usage of more cheaply obtainable data representations when estimating class hardness,
062   and expanding our understanding of class bias.
063   To effectively analyze class hardness, our approach necessitates categorizing data samples rather
064   than treating hardness as a continuous spectrum. During this process, we found that when we apply
065   various metrics to the data, the resulting distributions of many metrics follow inverse cumulative
066   distribution function reminiscent to that of a Gaussian distribution. This discovery suggests that a
067   three-tier categorization into easy, medium, and hard samples is more representative than the com-
068   monly used binary split into easy and hard (Sorscher et al., 2022; Seedat et al., 2024). This approach
069
      is particularly important for other tasks that require explicit categorization, such as data pruning
      (Sorscher et al., 2022), where this more granular categorization can lead to better performance.
070
071   To assess which properties of class manifolds most influence class hardness, researchers commonly
072   employ Pearson correlation (Ma et al., 2023a;b; 2024; Kaushik et al., 2024). However, this approach
073   assumes that class bias is a consistent phenomenon with minimal variation across model runs. Con-
074   trary to this assumption, our results show that the variability in class-level performance resulting
075
      from different random initializations can be greater than the differences observed between classes
      themselves. This leads to inconsistent rankings of class hardness based solely on initialization, high-
076
      lighting a significant limitation of using correlation-based approaches for understanding class bias.
077
      Our findings suggest that existing methods to account for dataset-level variability might not be ro-
078   bust enough for addressing the nuances of class bias. In summary our work makes the following
079   contributions:
080
081         1. Estimating class bias using PCA-transformed spaces: We find that PCA-transformed
082            spaces can effectively estimate class bias, similar to those obtained from neural network
083            latent representations. This discovery opens new avenues for research on class bias and
084            enhances our understanding of latent representations in machine learning.
085         2. Unravelling complexity of evaluating class-level hardness: We show that variations in
086            class-level performance across ensemble models is often larger than the differences in per-
087            formance between classes themselves. Coupled with higher variability in class-level per-
088            formance over dataset-level, this highlights new challenges in addressing class bias and
089            suggests current methods may lack robustness.
090         3. Refining categorization of hardness: Our results suggest that categorizing samples into
091            easy, medium and hard is a closer representation of the underlying hardness distributions
092            than the commonly used easy and hard categorization. This finding can be significant
093            for fields that by definition cannot consider hardness as a spectrum, and rely on hardness
094            categorization, such as data pruning.

096   2   BACKGROUND
097
098   Evolution of intrinsic dimension (ID) across network layers was explored by Ansuini et al. (2019),
099   showing that ID initially increases before decreasing in the final layers, and found that the ID of the
100   latent dataset manifold in the last hidden layer reliably indicates dataset-level hardness. Extending
101   this, Pope et al. (2021) demonstrated a correlation between the average ID of randomly sampled class
102   pairs in the input space and dataset-level hardness, suggesting that ID remains a valuable metric for
103   estimating class bias even after model bias is incorporated into data representations. Building on
104
      these findings, Ma et al. (2024) further examined the relationship between ID and model fairness,
      highlighting the broader utility of ID for understanding model behavior.
105
106   Naitzat et al. (2020) demonstrated that the topology of class manifolds simplifies through the layers
107   of a well-trained network, as measured by decreasing Betti numbers. Extending this, Magai (2023)
      observed that changes in the persistent homological fractal dimension differ across architectures


                                                         2
      Under review as a conference paper at ICLR 2025


108
      based on convolutions versus attention mechanisms. Building on these insights, Suresh et al. (2024)
109   proposed using the rate of decay in topological complexity, defined as the sum of Betti numbers, to
110   quantify the effect of architectural choices on dataset-level hardness.
111
112
      Kaufman & Azencot (2023) found that trained CNNs exhibit a characteristic curvature profile—an
      initial steep increase, followed by a plateau, and then another rise—and demonstrated that the cur-
113
      vature gap between the last two layers strongly correlates with dataset-level hardness. Meanwhile,
114
      Ma et al. (2023b) showed that the average curvature of latent class manifolds is a reliable indicator
115   of class-level hardness, while curvature in the input space is not.
116
117   Ma et al. (2023a) found a strong positive correlation between the volumes of the learned embeddings
118
      (latent manifolds) and class bias, demonstrating that this correlation holds both with and without
      accounting for inter-class distances. Similarly, Kaushik et al. (2024) demonstrated that the spectrum,
119
      defined as the set of eigenvalues of the covariance matrix of latent manifolds, is an effective indicator
120
      of class-level hardness. Both studies highlight that class dispersion within latent space correlates
121   with class-level hardness.
122
123   Kienitz et al. (2022) found that dataset entanglement, estimated through the entanglement of repre-
124
      sentative pair of similar classes using a Linear Support Vector Classifier, provides a better estimate
      of dataset-level hardness than intrinsic dimension. Additionally, Ma et al. (2023b) showed that the
125
      average separation degree between class manifolds is a good indicator of class-level hardness in the
126
      input space but not in latent space.
127
128   Hardness has been extensively studied at the dataset level, leading to numerous metrics, as outlined
129   in several surveys (Ho & Basu, 2000; Ho et al., 2006; Barella et al., 2021) and supported by open-
130
      source libraries (Komorniczak & Ksieniewicz, 2023; Lorena et al., 2024). While most of these
      metrics focus on dataset-level complexity (Smith et al., 2014), Lorena et al. (2019) suggests that
131
      majority can be adapted for instance-level hardness.

      Hardness identifiers In this work we use fourteen different instance-level metrics that measure:
137
      1) intra-class structure (DCC, ADSC, MDSC); 2) separation from other classes (N3, DNOC, ADOC,
138   MDOC, CP); 3) inter-class comparison (CDR, MDR, ADR); and 5) curvature (MC, GC). However,
139   when modified to perform as class-level hardness identifiers, the role these metrics changes. Be-
140   low we categorize the instance-level metrics we used based on their role as class-level hardness
141   identifiers:
142
143         1. Class Dispersion: Distance to class centroids (DCC).
144
145
            2. Class Density: Average (ADSC), and minimum (MDSC) distance to same-class samples
               within kNN.
146
147         3. Class Separation and Overlap: The adapted N3 metric (N3) (Gøttcke et al., 2023), which
148            checks if the nearest neighbor belongs to another class, along with distances to other-class
149            centroids (DNOC), the average (ADOC) and minimum (MDOC) distances to other-class
150            samples within the 40NN, and class purity (CP) within the 40NN (Xiong et al., 2012).
151            Apart from that, Ratios of within- to between-class distances for centroids (CDR), 40NN
152            minimum distances (MDR), and 40NN average distances (ADR)
153         4. Geometric properties: Mean curvature (MC) and Gaussian curvature (GC), as curvature
154            has been shown to correlate with hardness (Kienitz et al., 2022; Kaufman & Azencot, 2023;
155            Ma et al., 2023b).
156
157   For class dispersion we add three class-level metrics: 1) algorithm from Ma et al. (2023a) (V);
158   and 2) maximum (max λ), and average (avg λ) of eigenvalues of the covariance matrix, used by
159   Kaushik et al. (2024). In Appendix A, we provide precise information on how each of these metrics

192   Experimental design We apply our metrics to MNIST, KMNIST, FashionMNIST, and CIFAR10
193   under two scenarios: full and part information. In the full scenario, metrics are computed on the
194   entire dataset, similar to curriculum learning, active learning, or data pruning. Here, we measure
195   hardness based on all of the available data samples. On the other hand, in part scenario we apply
196   metrics only to the training set. This setting is introduced to assess how data-based methods, many
197   of which rely on kNN, are affected by incomplete access to information.
198   In order to compute the class bias, which is essential when working with class-level hardness, we
199   train ensembles of networks for each dataset. On CIFAR10, we use ResNet56 (He et al., 2016), train-
200   ing 25 networks for hundred epochs with Adam (Kingma & Ba, 2015) (lr=0.01, weight decay=1e−4,
201   cosine scheduler). For MNIST, KMNIST, and FashionMNIST, we use LeNet, training hundred net-
202   works for ten epochs with SGD (lr=0.001). All experiments use a batch size of 32.
203
204   Analysing distributions of metric values After computing the metric values for each datum, we
205   sort the data samples based on these values. This reveals three families of metrics, based on how their
206   values are distributed: 1) logarithmic; 2) inverse cumulative; and 3) exponential. As shown in Fig.
207   1, the majority of metrics belong to the second family. Notably, even when the setting or dataset
208   changes, most metrics consistently remain in the same family. The only exceptions are N3 and
209   Purity, which switch from the logarithmic to the exponential family depending on the dataset. While
210   the first and third families support easy and hard sample divisions, the second family introduces a
211   medium-hardness category.

249
          fication. To address this, we adopt adaptive division points to classify samples as easy, medium, or

          hard based on gradient values. Since most metrics yield an inverse cumulative distribution similar
251       to that of a Gaussian distribution, we set the division points where the gradient consistently falls

260        4                  C OMPUTING AND INVESTIGATING CLASS BIAS
261
262
          Class hardness is is evidenced by the presence of class bias. Hence, to measure the performance of
263
          metrics as class-level hardness identifiers it is essential to firstly compute class bias. Furthermore,
264       it’s important to differentiate between hardness emerging due to different types of errors. Hence, we
265       consider two settings: full and part. In the full setting, we train on the entire dataset and measure
266       class bias based on accuracies across the entire dataset, providing insights into classes that are diffi-
267       cult to learn due to approximation error. In the part setting, we train on the training set and evaluate
268       class bias based on test set accuracies, which accounts for both approximation and generalization er-
269       rors. As far as we know, this is the first work making such distinction when working with class-level
          hardness.

291       Inconsistency of class bias We first measure the average class-level accuracies as a function of
292       the number of models in an ensemble. Intuitively, this function should plateau after a few itera-
293       tions, with high fluctuations indicating inconsistent class-level performance across models. Since
294       the effectiveness of metrics as class-level hardness identifiers is commonly assessed using Pearson
295       correlation (Ma et al., 2023a;b; 2024; Kaushik et al., 2024), it is essential to establish the consistency
296       of class bias. However, our results demonstrate that in MNIST, KMNIST and FashionMNIST class
297       bias is often inconsistent, with variations in class-level accuracies surpassing intra-class differences
298
          (see Fig. 2). This inconsistency makes Pearson correlation extremely poor measure for evaluating
          class-level hardness metrics, as the results depend heavily on the size of the ensemble. Notably,
299
          we observe this phenomenon primarily in simpler datasets, with KMNIST and MNIST showing the
300
          most severe effects, while it is negligible in CIFAR10. This raises an important question: is the
301       inconsistency of class bias a data or model related issue. In other words, as class-level performance
302       improves and the differences between classes shrink, will the class-level variations also decrease
303       proportionally, ensuring stable class bias?
304
          To investigate this, we perform preliminary experiment where we train another ensembles on Fash-
305
          ionMNIST using forty epochs 3. We find that for full setting, the differences in the performance
306
          among classes reduces significantly with the increase of strength of an ensemble. This results in
307       more fluctuating or unstable measurements of class bias, resembling those observed in MNIST and
308       KMNIST in Fig. 2. Meanwhile, in the part setting we observe minimal effects of increased ensem-
309       ble power on the class-level performance and variability. This most likely happens due to limited
310       generalization capabilities of LeNet, and we might observe similar trends to these observed in full
311       setting when using more powerful model. When analyzing the evolution of the average asymptotic
312       class-level accuracies of the ensembles (see Fig. 3c and 3f) the variability of class bias becomes
313       even more apparent. These results clearly suggest, that the order of classes based on hardness is not
314       fixed, and varies depending on multum of factors. To summarize, our observations suggest that the
315
          inconsistency of class bias is primarily a model-driven phenomenon, rather than being an inherent
          property of the data itself.
316
317
318       Intricacies of class-level hardness Our experiments also reveal that the final ranking of class
319       difficulties differs between the part and full setting. For example, in CIFAR10 we notice that the
320       class represented by the brown line is the most difficult in full setting (Fig 2d), but only the third
321       most difficult in part setting (Fig 2h). This discrepancy arises from differences in approximation and
322       generalization errors—some difficult samples may be easier to approximate but harder to generalize
323       on, and vice versa. This highlights that class bias should be distinguished based on whether it results
          from approximation or generalization difficulties.

340   deviation of dataset-level accuracies across models within an ensemble (see Table 1). This is most
341   notable on FashionMNIST, where the standard deviation of dataset-level accuracies of an ensem-

      than dataset-level hardness, due to larger standard deviations. While it is well known that model
      performance varies based on initialization, and ensembles mitigate this at the dataset level, the prob-
347

      After computing class bias, the next step is to evaluate the class-level performance of various hard-
      ness identifiers. This is done by measuring the correlation between class-level hardness identifiers
355
      and class bias. To the best of our knowledge, Ma et al. (2023b) is the only study that uses instance-
356
      level metrics as class-level hardness identifiers, transforming them by averaging instance-level val-
357   ues to obtain class-level metrics. However, this method discards valuable information inherent in
358   the instance-level granularity of the metrics, potentially overlooking important insights related to
359   class-level hardness. As an alternative, we propose to analyze the distribution of hard and easy sam-
360   ples within each class, rather than relying on averages. This allows us to capture the extremes—how
361   many easy or hard samples are concentrated in a given class—and compare this distribution to the
362   observed class bias. In our experiments we use Spearman’s rank correlation. We analyze the class-
363   level distribution of these samples and compare it to class bias, expecting positive correlation for
364   easy samples and negative correlation for hard samples.
365
366   Performance of class-level metrics obtained from raw data Our results reaffirm prior research,
367
      showing that curvature is a poor class-level hardness identifier, while class separation is a strong one
      when measured from raw data. In fact, we discover that for MNIST, KMNIST, and FashionMNIST,
368
      even straightforward metrics like the distance to other-class samples provide a reliable estimate of
369
      class hardness. However, for more complex datasets like CIFAR10, more advanced techniques, such
370   as the class separation proposed by Ma et al. (2023b), become essential for properly capturing inter-
371   class relationships. On the other hand, both type dispersion- and density-based metrics appear to be
372   weak class-level hardness identifiers. While the results vary by dataset, even in the best cases, they
373   are only marginally statistically significant, as indicated by p-values below 0.05 but above 0.1. We
374   also find that both the full and part settings yield similar results, confirming that data- and model-
375   based methods remain effective even when reducing the amount of information 14% − 16%.
