                                                  A BSTRACT
008
009
010
               Embeddings are extensively used in many domains to represent information about
               domain entities in a compressed manner. In recommendation systems, these
011
               embeddings are trained to extract meaningful information about items or users
012
               from behavioral data consisting of users’ ratings or users’ implicit feedback. These
013            behavioral embeddings are usually not trained on data from a language domain,
014            but they encode useful behavioral information which cannot be easily described
015            using language. In contrast, Large Language Models (LLMs) do not have good
016            representations for either behavioral data or behavioral entities(items or users), as
017            these are usually not textual and the data is specific to a recommendation system.
018            Bridging this gap between behavioral understanding and language understanding
019            can enable new item and language interleaved tasks. In our work we show how we
020            can efficiently adapt rich behavioral embeddings for use as input representation
021
               in pre-trained LLMs. To achieve this we adapt a Querying Transformer with a
               new item-item contrastive loss and show improved item-text joint understanding in
022
               PALM-2 and also demonstrate improved capabilities in recommendation domain
023
               compared to using the behavioral embeddings directly as input to PALM-2.

      1   I NTRODUCTION
027
028
      Large Language Models (LLMs), trained on web-scale data using a very large number of parameters,
029
      have shown remarkable emergent capabilities, such as in-context learning, reasoning, coding (Brown
030
      et al., 2020; Chowdhery et al., 2022; Google, 2023). Recently, those abilities have been extended to
031   multimodal domains, including image, audio, and video (OpenAI, 2024; Gemini Team, 2024a;b).
032   On various professional and academic benchmarks, those models achieve or surpass human-level
033   performance (Hosking et al., 2024). By contrast, the capabilities of pre-trained LLMs in recommen-
034   dation domain have not derived similar breakthroughs. Traditional recommenders such as matrix
035   factorization (Koren et al., 2009) and sequential item recommenders (Kang & McAuley, 2018) still
036   outperform pre-trained LLMs like Llama (Touvron et al., 2023) by a large margin in domain specific
037   tasks, even after finetuning. One reason is the difference in characteristics of recommendation data
038   and language data. For example, a video recommender system typically recommends videos to
039
      users based on their past history of watched or skipped videos, with users rarely providing natural
      language feedback. We call this behavioral interaction data. Recommendation is dependent on this
040
      interaction data to train models for recommendation tasks, this data is specifically obtained from the
041
      recommender while users are interacting with it. In its native form this data is not textual and most is
042   not available freely on the web, hence off-the-shelf LLMs do not have sufficient understanding of
043   recommendations items & users. Large scale recommendation systems deal with a varied number
044   of items and users, and the interaction data has a very sparse coverage of this large domain-specific
045   vocabulary. Specifically, each user only interacts with a very small set of the full item vocabulary,
046   making it hard to learn representations for the items & users directly using a language model with
047   fine-tuning. For instance, a user will only watch and rate a smaller set of movies from the full catalog
048   of movies.
049   The sparse nature of this past interaction data makes traditional collaborative filtering (CF) models
050   suitable for tasks in this domain. For example, CF models are good at inferring that if many users have
051   viewed both v1 and v2 , then a user who likes v1 may also like v2 . Traditional recommenders such as
052   matrix factorization and sequential recommenders outperform very large pre-trained LLMs in such
053   tasks. However, these traditional recommenders do not have good natural language understanding.
      We expect that combining the language understanding of LLMs and behavioral understanding of

054
      traditional recommenders can help us learn new tasks that utilize language relevant to the domain and
055   vocabulary of domain entities in a unified manner.
056
057   Our goal is to improve upon language generation tasks using both behavioral and textual information
058
      about an item. We propose Item-Language Model, (ILM, hereafter) a framework that learns new
      input representation that bridge the gap between language domain and recommendation domain to
059
      enable new tasks that can utilize both language and behavioral input representations interchangeably
060
      in an interleaved manner. Our contribution is adapting Querying Transformer to bridge the gap
061   between language modality and behavioral modality and a new item-item contrastive component in
062   the Querying Transformer to extract behavioral understanding.

      2    R ELATED WORK
066
067
      Behavioral representation in LLMs Efficiently representing users and items in recommender sys-
      tems is a rich field with years of work using traditional techniques such as Matrix Factorization (Koren
068
      et al., 2009; Rendle et al., 2022). These learn an embedding representation from past interaction data
069
      and other metadata about the items. The embedding represents meaningful information extracted
070   about the items & users and projects them in an N -dimensional space, with the goal that items &
071   users close together in this space are similar. Let’s look at some existing work on representing items
072   & users in LLMs. Using text representation, such as the title of an item or a random identifier to
073   represent recommender users is a straightforward input representation. ELM (Tennenholtz et al.,
074   2024) shows how to interpret input embedding spaces by feeding semantic embeddings and behavioral
075   embeddings to LLM with a Multi-Layer Perceptron (MLP) projection to adapt it to text token space.
076   Similarly, CoLLM (Zhang et al., 2023) feeds user and item collaborative filtering embeddings to LLM
077   to improve quality in recommendation tasks. OpenP5 (Xu et al., 2023; Hua et al., 2023) introduces
078
      collaborative indexing techniques that use the structure of the assigned identifier to encode some
      preprocessed collaborative information. These identifiers are passed, without modification, to the
079
      text tokenizer of LLM to improve recommendation tasks. Recently, USER-LLM (Ning et al., 2024)
080
      integrates user embeddings within LLMs through a “perceiver” adaptor (Jaegle et al., 2021; Alayrac
081   et al., 2022). This prior work shows that it is hard to improve the pure recommendation capability
082   of LLMs like Llama to match the performance of traditional recommendation-specific models that
083   contain a few transformer layers, trained specifically for recommendation task. Specifically, our work
084   does not tackle the goal of having an LLM beat recommendation task benchmarks. We are interested
085   in enabling new language generation tasks that can use both language and behavioral representations
086   in a unified manner.
087
088   Vision language models Work done in computer vision, and specifically, vision-based representa-
089   tions show an alternate approach. Here, vision and language are two different modalities, and the
090
      foundation models are trained with both modalities for generative and contrastive learning objectives.
      Existing work like BLIP-2 (Li et al., 2023), CoCa (Yu et al., 2022), and MaMMUT (Kuo et al., 2023),
091
      achieve state-of-the-art performance on vision-language tasks. While these approaches are promising,
092
      item representations for recommenders require behavioral data that is usually not public, and thus
093   cannot be used in LLM pretraining. To alleviate this, we adopt a two-phase workflow, similar to the
094   two-phase workflow of BLIP-2, including pretraining a recommender item-adapter in phase-1 and
095   task fine-tuning in phase-2. In addition, we adapt it to include a collaborative item-item contrastive
096   loss.

149   3.1   L ANGUAGE MODEL
150
151   An LLM is trained on large amounts of data, such as billions of words, to learn statistical relationships
152
      between words and phrases. This allows them to perform natural language processing tasks, such
      as generating text, summarizing documents, answering questions, classifying text and learning
153
      meaningful representations for text.
154
155   In an off-the-shelf LLM the input text is usually broken down into a sequence of language tokens,
156   l ∈ V, each token is converted into numerical representations called embeddings, el ∈ Rd . The
157   sequence of input embeddings is passed through a stack of decoder layers that are part of a pre-trained
158
      LLM and the LLM generates one output token Oi ∈ V at a time until a special end-of-sentence token
      is generated.
159
160   In ILM, the input can also be recommendation IDs, r ∈ U ∪ I. er ∈ Rk is also available from the
161   recommendation domain. These IDs and text tokens are mixed and appear interchangeably in the
      input. The spaces Rk and Rd are different, and mapping between them is handled in our QFormer


                                                               3
      Under review as a conference paper at ICLR 2025


162
      adapter. The QFormer maps ID inputs to language inputs, passed to the LLM. The decoder layers
163   and generation of output tokens Oi are unchanged and reused from off-the-shelf LLM. We introduce
164   technique to generate the embeddings er for items and users using behavioral data in next section.

      3.2   B EHAVIORAL EMBEDDINGS (CF EMBEDDING )
167
168   We can swap any embeddings from an external domain. For this paper we utilize collaborative filtering
169   trained using Alternating Least Squares (Rendle, 2022) to generate embedding representations of
170   recommendation items and users. In the recommendation domain, a user u ∈ U interacts with item i
171   from a catalog I. We consider all such pairs of < u, i > as a positive interaction examples and all
172   pairs when the user did not interact with the item as negative interaction examples. This data forms a
173   binary matrix of interactions, A ∈ R|U |×|I|
174
      Formally, collaborative filtering (CF, hereafter) does the following, given a matrix of behavioral
175
      interactions between users and items, A ∈ R|U |×|I| , we seek to find matrices B and C such that:
176
177
178                                                  A ≈ BC,                                                (1)
179                  |U |×k           k×|I|
      where: B ∈ R            ,C ∈R
180
181   Typically, the scale of |U| and |I| varies depending on the domain. The value k is chosen to be
182   much smaller than both |U| and |I|, resulting in a compressed representation of the original matrix A.
183
      Hence, these latent representations of the users and items encode rich behavioral information which
      we will use in our formulation of the ILM to represent recommender items and users.

187   3.3   C O - INTERACTED ITEMS
188
189   We define two items x and y as “co-interacted” if the at least one user has interacted with both items,
190   and hence these items have similar representations in the embedding space Ei . One interpretation of
191   CF is that the dot-product ex .ey represents how similar the two items are. We use this co-interaction
192   signal and the CF embeddings in QFormer as described in the next section.

      4.1   Q UERYING TRANSFORMER
197
198   We adapt the Querying transformer (QFormer, hereafter) of BLIP-2 (Li et al., 2023) as depicted
199   in our Figure 2(a) for the problem of bridging the gap between recommender items modality and
200   text modality. The new component we add to the QFormer is a novel item-item contrastive loss and
201   user-item contrastive loss that preserves the behavioral information in CF embeddings while adapting
202   them to the text modality as depicted in Figure 2(b). The effects of the new component are depicted
203   in Figure 2(c). Our QFormer has 4 training tasks. The first 3 tasks are adapted as-is from BLIP-2 and
204   the fourth task is added to extract behavioral information. The possible inputs are pre-trained item
205
      CF embedding and/or textual metadata about the item. The actual input and output varies for each
      task. Our tasks are:
206
207
            1. Unimodal encoder, which separately encodes an item and text (Unimodal - text is one
208
               modality, item ID is another modality). The input is a set of positive item-text pairs,
209
               generated from metadata about the item. For example, given a movie and its genre, the
210            movie-genre pairs are “positive” item-text pair. Text from other examples in the batch are
211            used to sample “negative” item-text pairs, for example a movie and a genre that does not
212            belong to it. The text encoder is the same as BERT (Devlin et al., 2019), where a [CLS] token
213            is appended to the beginning of the text input to summarize the text. Item-Text Contrastive
214            loss (ITC, hereafter) is a contrastive loss that aligns the feature space of the item transformer
215            and the text transformer by encouraging positive item-text pairs to have similar encoded
               representation in contrast to the negative pairs.

249
              one additional cross-attention layer between the self-attention layer and the Feed Forward
250           Network for each transformer block of the text encoder. A task-specific [Encode] token is

252           the item-text pair. Item-Text Matching loss (ITM, hereafter) aims to learn item-text cross-
253           domain representation that captures the fine-grained alignment between recommendation
254           items and language. ITM is a binary classification task, where the model uses an ITM
255           head (a linear layer) to predict whether an item-text pair is positive (matched) or negative
256           (unmatched) given their input features. For example, this is trained to predict if a movie
257           matches a given genre.
258        3. Item-grounded text decoder, which replaces the bidirectional self-attention layers in the
259           item-grounded text encoder with causal self-attention layers. A [Decode] token is used
260           to signal the beginning of a sequence, and an end-of-sequence token is used to signal its
261           end. The Language Modeling loss, also called Item-Text Generative loss (ITG, hereafter)
262           activates this decoder, which aims to generate textual descriptions given an item. It optimizes
263           a cross entropy loss which trains the model to maximize the likelihood of the text in an
264           auto-regressive manner. For example, given a movie generate the genre tags associated with
265
              it or given a movie generate its title.

267           co-interacted items and provides one output token per item. Item-Item Contrastive (IIC,

269           co-interacted pairs to have similar representations in contrast to the negative pairs. For

      6   C ONCLUSION
433
434
      We presented ILM, a novel item-language unified model. We had traditional representations that
435   encode rich information in the recommendation domain and we had language models that provide
436   language understanding, we have shown how we can unify both and learn new tasks that interpolate
437   between these domains and can utilize items and text in a unified fashion. A pre-training step to
438   generate behavioral embeddings is required to ensure our technique performs best. We have shown
439   that we do better when we combine these two domains using semantic consistency tasks from ELM.
440   In Appendix A.1, we used our model designed for language generation task to evaluate hardcore
441   recommendation tasks and show reasonable performance, however the existing baselines for those
442   tasks use different backbone LLMs and are trained to perform well specifically on recommendation
443
      tasks and not language generation. We also note that our technique is agnostic to the domain and can
      be applied to any new domain that has rich embedding representations of domain entities.

542   A.1   R ECOMMENDATION CAPABILITIES IN LLM
543
544   In addition to the main semantic consistency task, we are interested in knowing the domain capabilities
545   added to the LLM as a result of this Item-Language unified architecture. We use a dataset designed
546   for evaluating traditional recommender models. Especially, we are interested in evaluating which
547   aspects of our work contribute to learning domain specific tasks using LLMs.
548   OpenP5 (Xu et al., 2023; Hua et al., 2023) is a dataset for LLM-based Recommendation development,
549   finetuning, and evaluation. It provides 10 popular preprocessed public datasets, and each dataset
550   contains two kinds of tasks: Sequential Recommendation and Straightforward Recommendation.
551   We select the MovieLens-1M and Beauty datasets for our benchmarks. The training target for each
552   example is the ground truth item ID. For training inputs, we append each item’s random indexing
553   ID with its behavioral embedding on the user sequence training set. We use the provided train,
554   development, and test split in the OpenP5 dataset, which uses the last item in the user sequence
555
      for testing and the second from the last item in the user interaction sequence for development. For
      OpenP5 tasks, we report top-k Hit Rate (HR@K) and Normalized Discounted Cumulative Gain
556
      (NDCG@K) with K = 5, 10 to evaluate the recommendation performance. Since the outputs for the
557
      tasks in this dataset are only from the recommender item vocabulary I, to compute those metrics,
558   we use beam search to generate 10 outputs for each example, and remove invalid outputs that do not
559   match the regular expression “.*item_(\d+)$”.
