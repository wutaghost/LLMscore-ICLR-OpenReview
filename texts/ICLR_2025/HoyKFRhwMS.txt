009                                                 A BSTRACT
010
011            Training a neural network is a monolithic endeavor, akin to carving knowledge
012            into stone: once the process is completed, editing the knowledge in a network
013            is nearly impossible, since all information is distributed across the network’s
014            weights. We here explore a simple, compelling alternative by marrying the rep-
015            resentational power of deep neural networks with the flexibility of a database.
016            Decomposing the task of image classification into image similarity (from a pre-
017            trained embedding) and search (via fast nearest neighbor retrieval from a knowl-
018
               edge database), we build a simple and flexible visual memory that has the fol-
               lowing key capabilities: (1.) The ability to flexibly add data across scales: from
019
               individual samples all the way to entire classes and billion-scale data; (2.) The
020
               ability to remove data through unlearning and memory pruning; (3.) An inter-
021            pretable decision-mechanism on which we can intervene to control its behavior.
022            Taken together, these capabilities comprehensively demonstrate the benefits of an
023            explicit visual memory. We hope that it might contribute to a conversation on
024            how knowledge should be represented in deep vision models—beyond carving it
025            in “stone” weights.

028   1    I NTRODUCTION
029
030   In the pretty diagrams on ”Intro to Machine Learning” slides, an ideal ML workflow looks like
031   this: Data collection, preprocessing, choosing a model, training, evaluation, deployment. Happy
032   ending—the model is deployed, the users love it, and one can finally go on that well-deserved
033
      vacation and catch up on the latest AGI memes.
034   Until, of course, the enemy of any ideal world sets in: reality. The real world constantly keeps
035   changing, and so do data requirements. New data and datasets become available, and existing ones
036   become deprecated for a variety of reasons, including concerns around fairness, biases or unsafe
037   content. Knowledge changes, and concepts drift (Tsymbal, 2004; Lu et al., 2018): Phones and cars
038   look different today than they did a few years ago, and different from how they will look in the
039
      future. When it comes to data, the only constant is change (Cao & Yang, 2015; Bourtoule et al.,
      2021; Nguyen et al., 2022; Zhang et al., 2023). Consequently, from a modeling perspective, in
040
      order to keep up with this change one would ideally want to constantly re-train or fine-tune models,
041
      which is of course not feasible. In short, as anyone who has ever deployed a model has experienced
042   firsthand, one is constantly battling the symptoms of a single underlying cause: the fact that deep
043   learning models have a static knowledge representation entangled in millions or billions of model
044   parameters. We, among many others working on memory (e.g. Weston et al., 2014; Chen et al.,
045   2018; Wu et al., 2021; Iscen et al., 2022; Nakata et al., 2022; Iscen et al., 2023; Prabhu et al., 2023;
046   Gui et al., 2024; Shao et al., 2024; Silva et al., 2024), believe that this is not a great way to represent
047   visual knowledge for deep learning. Instead, we argue that we should build models that cleanly
048   separate representation (how things are represented, e.g. through feature embeddings) from visual
049   memory (what is known). In short, deep learning models need a flexible visual memory: a way to
050
      explicitly utilize and edit knowledge.
051   In this work, we build a simple visual memory for classification and show that it has seven desir-
052   able capabilities, including the ability to flexibly add data across scales (from individual samples
053   to classes and even billion-scale data), the ability to remove data from our model’s classification
      process through machine unlearning and memory pruning, and a simple, interpretable decision-

054
      mechanism on which we can intervene to control its behavior. Our main goal is to provide a com-
055   pelling idea of how beneficial a flexible visual memory for deep learning can be from a variety
056   of perspectives and capabilities. From a technical standpoint, we aim for simplicity: retrieving k
057   nearest neighbors (in an embedding feature space) along with their labels to classify a query image.
058   This approach allows us to investigate where a simple visual memory mechanism helps, where its
059   limitations may be, and where there might be opportunities for improvement through a more com-
060   plex system. We hope that by demonstrating clear benefits from a simple visual memory, this article
061   might contribute to a conversation on how knowledge ought to be represented in deep vision models.
062   Here are some highlights of this article:
063
064       1. Improved aggregation of retrieved samples: we propose using RankVoting, a power-law
065          weighting that surpasses previous SOTA (SoftmaxVoting) for a deep learning based memory.
066       2. Re-ranking samples using a vision-language model achieves 88.5% top-1 ImageNet validation
067          accuracy, improving over both DinoV2 ViT-L14 kNN and linear probing.
068       3. Flexible perception: the visual memory achieves perfect unlearning, scales to billion-scale data
069          without additional training, and enables controlling sample influence via memory pruning.
070
071   We argue that the way current deep learning models represent knowledge (static knowledge repre-
072
      sentation, hard to update, hard to unlearn, hard to understand how a decision is made) is problematic.
      As an alternative, we built a working proof-of-concept: By building on the long history of nearest
073
      neighbor methods, and “marrying” them with a powerful deep learning representation (such as SSL
074
      features from DinoV2) and a billion-scale visual memory.
075
076   Related work. The concept of a visual memory has a long history in ML, neuroscience and psy-
077   chology. In psychology, exemplar theory posits that humans recognize objects by comparing them to
078
      existing examples in visual memory (Medin & Schaffer, 1978; Nosofsky, 1986; Dopkins & Gleason,
      1997; Jäkel et al., 2008; Nosofsky, 2011), like the ALCOVE model (Kruschke, 2020). In ML, prior
079
      to deep learning, instance-based learning (also known as memory-based learning) was a popular
080
      alternative to model-based learning (Aha et al., 1991; Quinlan, 1993). For instance, Turk & Pent-
081   land (1991) used nearest neighbor methods to classify faces, and Sivic & Zisserman (2003) build
082   a visual memory inspired by text retrieval for object retrieval from videos. In recent years, hybrid
083   approaches have started to combine the benefits of both approaches. Deep neural network variants
084   (model-based since they learn generalized abstractions of data) of k-nearest neighbor algorithms
085   (instance-based since they compare new data to existing exemplars in memory) have been proposed
086   with various motivations, including few-shot learning (Wang et al., 2019b; Yang et al., 2020; Bari
087   et al., 2021), improving adversarial robustness (Sitawarin & Wagner, 2019; Papernot & McDaniel,
088   2018; Rajani et al., 2020), medical image classification (Zhuang et al., 2020), confidence calibration
089
      (Papernot & McDaniel, 2018), interpretability (Papernot & McDaniel, 2018; Wallace et al., 2018;
      Lee et al., 2020; Rajani et al., 2020), image denoising (Plötz & Roth, 2018), retrieval-augmented
090
      learning (Khandelwal et al., 2019; Drozdov et al., 2022), anomaly and out-of-distribution detection
091
      (Bergman et al., 2020; Sun et al., 2022). Recently, Nakata et al. (2022) tested a kNN-based vi-
092   sual memory up to ImageNet-scale (1.28M images), and Khandelwal et al. (2019); Wu et al. (2021)
093   applied kNN-based approaches to neural language models.

      2     B UILDING A RETRIEVAL - BASED VISUAL MEMORY FOR CLASSIFICATION
097
      Given a dataset Dtest := {(x̃1 , y1 ), · · · , x̃n , yn }, we want to classify each image x̃i ∈ Dtest . Our
098
      classification approach consists of two steps: (i) building a visual memory, and (ii) fast nearest
099
      neighbor based inference using the visual memory.

102

104   directly retrieving those images that are considered similar to a test image according to a model.
105   The model is a fixed pre-trained image encoder, meaning that no training takes place when adding
106   information to visual memory. No copies of the dataset are stored in the visual memory. Instead,
107   feature maps are extracted from the model based on a set of images related to the downstream
      classification task at hand, such as a standard training set. For our experiments, our visual memory

135           2.2                      R ETRIEVAL - BASED CLASSIFICATION USING VISUAL MEMORY
136
              Given a query image x̃ ∈ Dtest , we extract its feature map, z̃ = Φ(x̃). We then query VisualMemory
137
              to extract k feature vectors, Neighbors(x̃) := {(z [1] , y[1] ), (z [2] , y[2] ((, · · · , (z [k] , y[k] )}, that are
138           closest to the query features z̃ using the cosine distance, which is the default retrieval similarity
139           measure for SSL models like DinoV2. Neighbors(x̃), are ordered by distance i.e.
140                                            dist(z̃, z [i] ) ≤ dist(z̃, z [j] ), ∀i ≤ j.
141
142           We then assign a weight, wi , to each neighbour (z [i] , y[i] ) and aggregate the scores for each neigh-
143           bour with the same label. Finally, we assign that label to the query image with the highest aggregate
144           score. We implemented retrieval based classification using one of the following two approaches:
145           1. Fast inference using matrix multiplication on GPUs/TPUs: For smaller datasets like Ima-
146           geNet, we saved VisualMemory as a matrix of size num images × num dims. During inference, for
147           an encoded query image of size 1 × num dims, we computed the dot product of this encoded image
148           with every entry in VisualMemory getting a matrix of size num images × 1. We then computed the
149           k nearest neighbors using the arg max operation.
150           2. Fast and scalable nearest neighbor search: We used ScaNN (Guo et al., 2020) for accelerating
151           nearest neighbor search at scale. Specifically, we saved the VisualMemory as a database and used
152           ScaNN for fast lookup of nearest neighbors during inference. This method scales easily to billion-
153           scale memory (cf. Section 3.3). Appendix J details latency and storage; storing features requires
154           only about 1–3% of the space of storing the dataset itself and even with a 1B memory.
155
              We mentioned earlier that we retrieve a set of neighbors, Neighbors(x̃) and aggregate information
156           across them to make a classification decision. In order to understand how reliable (i.e., accurate) re-
157           trieved memory samples are from the first to the 100th neighbor, we systematically analyze neighbor
158           reliability in Figure 1. As expected, reliability decreases as the neighbor index k increases, but even
159           at large k the neighbors contain above-chance information about the ground truth class. This sug-
160           gests that aggregating information across different neighbors may be beneficial to decision-making,
161           leading to the question: What is the best aggregation strategy? We empirically study this by testing
              different weighting strategies for aggregation:

182          Plurality voting: Each neighbour in Neighbors(x̃) is assigned an equal weight of 1.0. This is the
183          classic, most simple voting method and used for instance by Nakata et al. (2022).
184
             Distance voting: Each neighbour in Neighbors(x̃) is  assigned a weight based on its Cosine distance
185
             to the query image x̃ i.e. wi = exp − dist(z̃, z [i] ) . This approach has been used by Khandelwal
186          et al. (2019) for nearest neighbor language models.
187
188          Softmax voting: Each    neighbour is assigned a weight based on the softmax function i.e. wi =
189
             softmax dist(z̃, z [i] , τ ) where τ is the temperature. This voting method is considered state-of-
             the-art; for example nearest neighbor accuracies of self-supervised models are reported using this
190
             method. A temperature of τ = 0.07 frequently appears in literature (Wu et al., 2018; Caron et al.,
191
             2021; Oquab et al., 2023) and is reported as a parameter “which we do not tune” in the Dino pa-
192          per (Caron et al., 2021, p. 18). We observe that performance is sensitive to this parameter; other
193          temperatures perform worse. We therefore follow the literature in using τ = 0.07.
194
195
             Rank voting: We propose using a simple aggregation approach wherein each neighbour is assigned
             a power-function weight based on its rank in the ordered set Neighbors(x̃) i.e. wi = 1/(α + ranki )
196
             where ranki is i and α is an offset to avoid division by zero that is set to 2.0. This is similar, though
197
             not identical to, Gou et al. (2011) who used power-law weighting in a different context.
198
199          In Figure 2a, we compare the top-1 ImageNet validation accuracy of different ranking methods as
200          a function of number of neighbours, with the ImageNet-1K training set as the visual memory using
201
             the DinoV2/ViT-L14 model as the featurizer. Paradoxically, existing aggregation methods like plu-
             rality voting, distance-based voting, and softmax voting show decaying performance as the provided
202
             information (number of nearest neighbors) increases. This suggests that the methods are overconfi-
203
             dent in distant neighbors, assigning them too much weight. Our simple, parameter-free rank based
204          voting method, however, leads to an increase in performance with more neighbors until a certain k
205          after which the performance plateaus, which is the ideal scenario (Figure 2b). Furthermore, rank-
206          based voting also outperforms baselines in absolute terms; quantitative comparisons can be found
207          in the Appendix (Tables 4 to 8) where we also study the influence of hyperparameters (Figure 9).
208          This indicates that a simple, power-function based method can reliably integrate information across
209          retrieved memory samples.
210          Gemini re-ranking. Our results above demonstrate that different aggregation strategies have a

      3     C APABILITIES OF A VISUAL MEMORY
224
      Our primary goal is to motivate the concept of a machine visual memory from a variety of different
225
      perspectives. To this end, we investigate how such a memory can benefit the following capabili-
226
      ties: 3.1 Flexible lifelong learning: adding novel OOD classes; 3.2 Flexibly trading off compute
227   and memory; 3.3 Flexibly adding billion-scale data without training; 3.4 Flexible removal of data:
228   machine unlearning; 3.5 Flexible data selection: memory pruning; 3.6 Flexibly increasing dataset
229   granularity; 3.7 Interpretable & attributable decision-making.
230
231
      3.1   F LEXIBLE LIFELONG LEARNING : ADDING NOVEL OOD CLASSES ( DATA AND LABELS )
232
233   Standard classifiers, whether trained end-to-end (supervised models) or with a linear classifier (self-
234   supervised models), are not able to handle new information without re-training. For instance, adding
235   new classes or changing labels in an existing model usually involves either re-training or fine-tuning
236   parts of the model. A retrieval-based visual memory, in contrast, is able to process such information
237   in a natural and flexible way, aligning with the requirements of lifelong learning (Parisi et al., 2019).
238   We tested this by adding data for 64 new classes, along with their new labels, to the visual memory of
239
      a pre-trained DinoV2 ViT-L14 model (in addition to the ImageNet train set, which is in-distribution
      for the model). We took the new classes from the NINCO dataset (Bitterwolf et al., 2023), a dedi-
240
      cated OOD dataset that is designed to have no overlap with existing ImageNet labels and samples.
241
      This requires the model to transfer what it has learned to new, unseen concepts. The new task is
242   therefore harder, as the model has to retrieve images from both in-distribution and OOD classes.
243   The resulting visual memory has 1064 classes (1K from ImageNet and 64 from NINCO). Table 1
244   shows that with a visual memory it is possible to add new classes such that the in-distribution accu-
245   racy is maintained without catastrophic forgetting (the new classes only change ImageNet validation
246   performance by 0.02–0.04% depending on the aggregation method), while at the same time reaching
247   very high accuracy on the new OOD classes (approx. 87% top-1) without any training. Figure 12
248   in the appendix confirms that the samples are indeed OOD for the model, as demonstrated by larger
249   distances to nearest neighbors. This highlights that a visual memory is capable of flexibly adding
250
      new information—an important capability since the world is not static. Furthermore, the memory is
      incredibly robust towards label corruption up to 60% random labels, as shown in Appendix D.

        3.3                       F LEXIBLY ADDING BILLION - SCALE DATA WITHOUT TRAINING
305
306     Billion-scale dataset with pseudo labels. As demonstrated in Section 3.2, performance systemat-
307     ically improves with increased memory size across both small and large models. We here test how
308     far this trend holds beyond relatively small-scale, well-curated settings like ImageNet-1K by scal-
309     ing visual memory to the billion-scale unlabeled data regime. We obtain a large-scale dataset from
310     the union of the ImageNet-1K train set and a subset of the JFT-3B dataset (Zhai et al., 2022). To
311     this end, we treat JFT as an unlabeled dataset by ignoring its original labels and instead obtaining
312     pseudo labels by running them through ViT-22B-224px (Dehghani et al., 2023), a highly performant
313     classifier. We excluded images whose labels do not have a correspondence with the ImageNet labels.
314     Scaling. In Figure 3b, we show the downstream ImageNet validation performance of two DinoV2-
315     ViTs as a function of memory size. The plot demonstrates that even in the billion-scale data regime,
316     validation error decreases when increasing memory size without any training. The gain from more
317     data is most prominent when having fewer samples in memory (e.g., going from 1 to 10 samples per
318     class). In log-log space, a logarithmic function fits the empirical scaling trend well. In the literature,
319     simple scaling trends such as the one we observe are powerful predictors of scaling behaviour for
320
        different model and dataset sizes (Hestness et al., 2017; Kaplan et al., 2020; Hoffmann et al., 2022).
321     Out-of-distribution performance. In order to understand whether the benefits of increased memory
322     size transfer to out-of-distribution (OOD) data, we compared DinoV2 ViT-L14 once with ImageNet-
323     train in memory and once with JFT pseudo-labels in memory. The models are evaluated on the

      3.4     F LEXIBLE REMOVAL OF DATA : MACHINE UNLEARNING
346
347   The world is not static. Thus, in addition to the need to flexibly add novel data, there is often a
348   need to remove the influence of specific training data from a model’s decision-making process after
349   it has been trained (Cao & Yang, 2015; Bourtoule et al., 2021; Nguyen et al., 2022; Zhang et al.,
350   2023). A range of intricate methods are being developed to remove or reduce the influence of certain
351   training samples (Gupta et al., 2021; Sekhari et al., 2021; Ullah et al., 2021; Kurmanji et al., 2024;
352   Sepahvand et al., 2024)—a challenging endeavour if knowledge is embedded in millions or billions
353   of model weights. In contrast, for models with an explicit visual memory, machine unlearning
354
      becomes as simple as removing the dataset sample from the visual memory. For instance, after
      adding the NINCO dataset (Bitterwolf et al., 2023) into visual memory, we can remove any NINCO
355
      sample with outstanding performance on all three key unlearning metrics reported by Liu (2024):
356
      Efficiency: How fast is the algorithm compared to re-training? (Lightning fast.) Model utility: Do
357   we harm performance on the retain data or orthogonal tasks? (Not at all.) Forgetting quality: How
358   much and how well are the ‘forget data’ actually unlearned? (Completely and entirely.) Can machine
359   unlearning therefore be solved with a visual memory? If the embedding model is trained on data
360   that needs to be unlearned, machine unlearning remains challenging. If, however, the embedding
361   model is trained on a safe, generalist dataset (e.g., a publicly available image dataset) and data that
362   may need to be considered for unlearning later is simply put into the visual memory, then machine
363   unlearning indeed becomes as simple as deleting a datapoint from the visual memory. This can be
364   particularly helpful for tasks that may require private or confidential data—a model can be trained
365
      on publicly available datasets to learn general and information features and the private data can be
      added to a visual memory on local devices for downstream tasks to preserve privacy.
366

369
      The ability to flexibly remove the influence of certain datapoints is not just desirable in the unlearn-
370
      ing sense, but also advantageous in the context of dataset pruning, an emerging field that analyzes
371
      the quality of individual data points. The goal of dataset pruning is to retain only useful samples,
372   while removing those that have a neutral or harmful effect on model quality. The key challenge
373   is that in standard black-box models, it is entirely unclear whether any given sample is helpful or
374   harmful. The gold standard is leave-one-out-training (for ImageNet, this would consist of training
375   1.28 million models); current methods seek to approximate this extremely costly approach with
376   various heuristics (Feldman & Zhang, 2020; Chitta et al., 2021; Paul et al., 2021; Sorscher et al.,
377   2022; Abbas et al., 2023a). By contrast, the contribution of a data sample to decision-making in
      a visual memory based system is straightforward. For any given query image x̃, the neighbor set

473         4                    D ISCUSSION
474
475         Summary. Typical neural networks are trained end-to-end: perfect for static worlds, yet cumber-
476         some to update whenever knowledge changes. This is limiting their potential in real-world settings
477
            since the world is constantly evolving. Incorporating a visual memory, in contrast, enables a range of
            flexible capabilities that embrace change: lifelong learning through incorporating novel knowledge,
478
            being able to forget, remove and unlearn obsolete knowledge, flexible data selection through mem-
479
            ory pruning, and an interpretable decision-making paradigm on which one can intervene to control
480         its behavior. We systematically explored a simple visual memory that decomposes the task of image
481         classification into two primitives, image similarity (from a pre-trained embedding representation)
482         and search (via fast, scalable nearest neighbor search from a vector database). Our results demon-
483         strate that technical improvements like RankVoting improve kNN accuracies for both DinoV2 and
484         CLIP over the widely used SoftmaxVoting method that is sensitive to two hyperparameters (tem-
485         perature τ and number of neighbors k). Our approach also narrows the accuracy gap between a
            nearest neighbor memory (best flexibility, perfect unlearning, improved interpretability) and a fixed

520   Limitations and future work. First, we only considered the task of image classification across
521   a broad range of datasets. It will be interesting to extend the approach to other visual tasks, such
522   as object detection, image segmentation, instance recognition and to image generation where a vi-
523   sual memory would be desirable, too (since it is prohibitively expensive to re-train large generative
524   models every time data needs to be removed or added). Secondly, our approach relies on a fixed,
525
      pre-trained embedding model; strong distribution shifts may require updating the embedding. Self-
      supervised models are a particularly flexible choice, but it is an open question whether one could
526
      train smaller models that excel at their task with the help of a larger memory database. Conceptually,
527
      if a model needs to save less information in its weights, it might be possible to reduce the computa-
528   tional footprint of such a model. Furthermore, we sometimes observe a trade-off between flexibility
529   and accuracy. Exploring the use of the memory pruning weights as a data selection criterion in the
530   context of dataset pruning to improve over power-law scaling in deep learning (Sorscher et al., 2022)
531   might be an interesting avenue for future work.
532
      Outlook. Deep learning is increasingly becoming a victim of its own success: the more widely it is
533
      deployed, the stronger its limitations are felt. While the static nature of end-to-end trained networks
534   can easily be forgotten when focusing on fixed academic benchmarks, the real world is anything but
535   static. Data is constantly evolving, leading to the dreaded “model drift” where once-optimal models
536   gradually become less effective (Bayram et al., 2022). Incorporating an explicit visual memory—
537   however it may be instantiated—appears to be a promising way forward for real-world tasks where
538   flexibility is key. While the specific approach we employ here might well be improved through
539   more complex systems, we hope that the flexible capabilities we demonstrated might inspire and
      contribute to a conversation on how knowledge ought to be represented in vision models.

1090   H    M EMORY PRUNING
1091
1092   For memory pruning from Section 3.5, we implemented two pruning methods: removing unreliable
1093   neighbors from memory entirely (“hard memory pruning”), and reducing their weight (“soft memory
1094   pruning”). We report results on the ImageNet validation set with a (potentially pruned) ImageNet-
1095   train set in memory. For hard pruning, we excluded images from memory that contributed to a wrong
1096   decision at least 128 times (this meant excluding 26,257 images for DinoV2 ViT-L14), based on
1097   querying the ImageNet-train set against a memory consisting of the ImageNet-train set and querying
1098   100 neighbors for each sample. In order to obtain a fair comparison, instead of reporting accuracies
1099   for an arbitrary choice of k (the number of neighbors) we instead evaluate accuracy for each k in
1100
       [1, 100] and report the maximum accuracy obtained in Table 3. This ensures that differences in
       observed accuracy can indeed be attributed to memory pruning, as opposed to a choice of k. For soft
1101
       pruning, instead of excluding unreliable neighbors entirely as in hard pruning, the neighbor weights
1102
       (1.0 for PluralityVoting, or a rank-based weight in case of RankVoting) are instead multiplied by a
1103                                   d
       reliability factor γ with γ = c+v  where v is the number of times the image contributed to a wrong
1104   decision on the ImageNet-train set, c = 1 to avoid division by zero, and d = 1.75. This results,
1105   for instance, in γ = 0.88 for images that only contribute to a single wrong decision; in γ = 0.16
1106   for images that contribute to ten wrong decisions, and in γ = 0.02 for images that contribute to
1107   100 wrong decisions on the training set. Images that never contributed to any wrong decision are
1108   assigned γ = 1.0, i.e. their default weight remains unchanged.

1115   a full grid search sweep over three settings (output layers in 1, 4; pooling token concatenation in
1116   yes, no, and 13 different learning rates), resulting in 52 linear probes. Then, the authors evaluated
1117   the ImageNet validation accuracy for all of those 52 probes and only reported the highest one, as
1118   described in Appendix B.3 of the DinoV2 paper. Some may call this test set tuning or double
1119   dipping; the DinoV2 paper describes it as “common practice” (Oquab et al., 2023, p. 31). CLIP
1120   linear probe results are based on a logistic regression classifier learned using scikit-learn’s L-BFGS
1121   implementation, and hyperparameter sweeps are performed on a held-out set not used for evaluation,
1122
       according to Radford et al. (2021).

1289
1290   A flexible visual memory also provides a path to analyze representations of various models, particu-
1291   larly, how different models represent multiple concepts in an image. We study this for an ImageNet-
1292   train visual memory of DinoV2 ViT-L14 and CLIP ViT-L14. We use manually selected query images
1293   from outside the ImageNet dataset that have multiple objects from the ImageNet labels. We query
1294   the visual memory for nearest neighbors of the query image. Subsequently, we obtain the resid-
1295   ual image by subtracting the features of the nearest neighbor from the features of the query image.
       We, then, obtain the nearest neighbors for the residual image from the visual memory. We plot the

       results in Figure 15 which shows that DinoV2 ViT-L14 and CLIP ViT-L14 represent concepts in
1297   their features in a different manner. The nearest neighbors for DinoV2 are mostly images with a
1298   single concept (or object) from the query image. The residual image, subsequently, leads to nearest
1299   neighbors dominated by another single object in the query image. In contrast, CLIP often results in
1300   nearest neighbors that are generally a blend of concepts from the query image. These qualitative ex-
