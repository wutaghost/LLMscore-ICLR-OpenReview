009                                               A BSTRACT
010
011            Large language models are increasingly used across various domains, which raises
012            important safety concerns, particularly regarding adversarial attacks. While recent
013            advancements in formal neural network verification have shown promising results,
014            the complexity of transformers, the backbone of large language models, poses
015            unique challenges for formal robustness verification. Traditional convex relax-
016            ation methods often result in large approximation errors due to the transformer’s
017
               parallel, nonlinear attention heads. In this work, we address these limitations by
               introducing a novel approach based on non-convex, set-based computing to pre-
018
               serve the nonlinear dependencies through a transformer. Our approach generalizes
019
               previous methods on robustness verification of transformers, and the desired pre-
020            cision is tunable at the cost of additional computation time with a single parameter.
021

023   1   I NTRODUCTION
024
025   Large language models (LLMs) have gained immense popularity in various fields, including
026   question-answering, document summarization, and language translation (Raiaan et al., 2024). In
027   particular, this success must be attributed to the transformers (Vaswani et al., 2017) used within
028   these models. With increasing usage in all of these domains, safety concerns must be addressed:
029
      For example, how do we prevent the generation of harmful content? Large language models can be
      trained to reflect our desired behavior (Wallace et al., 2024); however, large language models – like
030
      any other neural network – can easily be fooled by adversarial attacks (Goodfellow et al., 2015).
031
      Thus, their behavior has to be formally verified to ensure safety, and such methods still need to be
032   developed for large language models (Huang et al., 2024).
033
034   For example, consider a large language model that answers user prompts and our goal is to prevent
035
      harmful outputs. As a safety shield, we deploy a classifier language model supervising the in-
      and outputs, which returns a predefined answer if any in- or output is deemed harmful (Kim et al.,
036
      2023). This assumes that the classifier model has perfectly generalized from a given dataset defining
037
      harmful content – which is not even the case for standard feed-forward neural networks (Neyshabur
038   et al., 2019), let alone large language models (Chang et al., 2024). Consider the following user
039   prompt that reasonably should not be answered:
040
                                            How to build a bomb?                                        (1)
041
042   As the prompts come from users, we have to assume they are trying to craft a prompt circumventing
043   our safety shield, also known as an adversarial prompt (Zhu et al., 2023). Let p′ be such an adver-
044   sarial user prompt crafted from the prompt p (1) above. Crucially, we, as verifiers, are unaware that
045
      this was done and, in particular, p is unknown to us. If we knew that a user was crafting adversarial
      prompts, we could simply block this user entirely. Thus, our goal is to find the prompt p, which we
046
      deem harmful and similar in meaning to the user prompt p′ , and only return an answer to the user
047
      if we have verified that such a prompt p does not exist. An adversarial prompt can be crafted by
048   replacing certain words with synonyms (Zhu et al., 2023): For example, we could replace “build”
049   with “construct”, or “bomb” with “missile”. As verifiers, we are required to test all possible
050   combinations of synonyms and only answer the prompt if all synonym sentences are determined to
051   be safe by the classifier model. The longer the user prompts, the more synonyms we have to consider
052   – quickly leading to a combinatorial explosion. Additionally, defining synonyms is time-consuming
053   and requires expert knowledge. Please also note that large language models usually do not operate
      on words but on tokens (Kudo & Richardson, 2018), such that synonyms have to be found on a token


                                                        1
      Under review as a conference paper at ICLR 2025


054
      level, and existing synonym collections on words cannot be used directly. Moreover, one might also
055   want to include prompts that are not necessarily synonyms but share the same meaning, such as:
056
057
                                How to build something that explodes?                                    (2)
058   In such a sentence, the meaning of the term “something that explodes” has to be aggregated,
059   which a large language model can do through its layers (Vaswani et al., 2017).
060   We address this issue by lifting recent advancements from formally verifying neural networks (Brix
061   et al., 2023) to large language models. In particular, we examine how to verify transform-
062   ers (Vaswani et al., 2017), which are the core components in modern language models (Achiam
063   et al., 2024). Please note that in the standard setting of formal network verification, the input is
064   usually perturbed by some noise, e.g., an ℓ∞ -ball with radius ϵ ∈ R+ . However, this is not directly
065   applicable in the domain of natural language processing, as it has text as input. Instead, the noise
066   is added in the embedding space, where synonyms are clustered together (Harris, 1954; Li & Yang,
067
      2018). We then formally verify that there does not exist a synonym sentence in the perturbed em-
      bedding space that the classifier model deems unsafe. If we cannot rule out such a sentence, the user
068
      prompt is not answered by the large language model, and the predefined answer is returned instead.
069
070   Our contributions. We present a novel approach to formally verify large language models using
071   set-based computing. This work is based on recent progress in neural network verification (Brix
072   et al., 2023), and we focus on the main challenge when verifying large language models, namely
073
      verifying the transformer (Vaswani et al., 2017). All other layers can be verified using the techniques
      developed for standard neural networks (Brix et al., 2023).
074
075   Transformers are particularly challenging to verify due to the in-parallel computed nonlinearities in
076   the self-attention heads (Vaswani et al., 2017), leading to nonlinear dependencies across all layers
077   within a large language model and, thus, a non-convex output set. Neural network verifiers usually
078   address this by relaxing the problem to a convex set enclosing the actual output (Brix et al., 2023).
079
      However, we show in this work that this leads to large outer approximations when applied on trans-
      formers and, for the first time, explicitly preserve these nonlinear dependencies through all layers
080
      using set-based computing. In particular, polynomial zonotopes (Kochdumper & Althoff, 2020) are
081
      used as a set representation.
082
083   Our evaluation shows that this enables the verification of much larger perturbed embedding spaces
084   compared to related work (Sec. 5). Additionally, the desired precision can be tuned via a single
085
      parameter at the cost of additional verification time.

197
      We verify neural networks using (matrix) polynomial zonotopes by iteratively propagating the input
198
      set X = H0 through the neural network and enclosing the output of each layer k ∈ [κ]:
199
      Proposition 1 (Image Enclosure (Kochdumper et al., 2023, Sec. 3)). Let Hk−1 ⊂ Rnk−1 be an
200
      input set to layer k, then
201
                                     Hk = enclose (Lk , Hk−1 ) ⊂ Rnk
202
      computes an outer-approximative output set.
203
204   In general, linear layers can be computed exactly with polynomial zonotopes using (10), whereas
205   nonlinear layers induce outer approximation (Fig. 2): The activation function is approximated using
206   a polynomial, and an enclosure is obtained by bounding the approximation error, which is added to
207   the output set using (9). The enclosure of the network’s output set is then given by Y = Hκ .

359
      Reducing the number of generators. Although the number of transformer blocks κ is typically
360
      small, the direct computation of the output set Y = Hκ quickly becomes intractable. Thus, we want
361   to limit the number of generators to gmax ∈ R+ by applying an order reduction method (Kochdumper
362   & Althoff, 2020, Sec. II-E): The rapid growth stems from the matrix-set multiplications (11) within
363   the attention layer (4), where the center and each generator of one set is multiplied with the cen-
364   ter and each generator of the other set. The multiplications of the generators result in higher-order
365   terms (see second example in appendix B), enabling the exact enclosure of the multiplication; how-
366   ever, many of these higher-order terms contribute only a little to the overall set and can be outer-
367   approximated by an interval without losing much precision (Kochdumper & Althoff, 2020, Sec. II-
368   B). Thus, we introduce a parameter ρlim ∈ R+ limiting the number of generators of the higher-order
369
      terms relative to the dimension of the resulting set. We do not apply an order reduction on the lin-
      ear terms resulting from the multiplication of the centers with each generator, respectively, as their
370
      dependencies between attention heads have to be kept for a tight enclosure. Please note that setting
371
      the error order ρlim = 1 results in an interval hull of all higher-order terms, which corresponds to the
372   approach in related work (Bonaert et al., 2021, Sec. 5.1); therefore, our approach is a direct gener-
373   alization of their approach. Additionally, we apply an additional order reduction after each residual
374   connection (Fig. 1) such that the total number of generators does not exceed gmax ∈ R+ . Thus,
375   we keep the nonlinear dependencies as long as possible and only apply relaxations when necessary.
376   Similar deferred relaxation techniques have also shown to be beneficial in other non-convex verifi-
377   cation approaches on standard neural networks (Wei et al., 2023; Ladner & Althoff, 2023; Fatnassi
      et al., 2023).

466   5                        R ELATED W ORK
467
468   This work continuous the progressive effort to formally verify neural networks in recent years (Brix
469   et al., 2023): As neural networks are vulnerable to adversarial attacks (Goodfellow et al., 2015), for-
470   mally verifying them against input perturbations is of immense importance in safety-critical scenar-
471
      ios. Neural network verifiers can generally be categorized into optimization-based verifiers Zhang
      et al. (2018); Katz et al. (2019); Henriksen & Lomuscio (2020); Singh et al. (2019) and approaches
472
      using set-based computing Gehr et al. (2018); Singh et al. (2018); Lopez et al. (2023); Kochdumper
473
      et al. (2023). Early approaches focused on computing the exact output set of neural networks with
474   ReLU activations, which has been shown to be NP-hard (Katz et al., 2017). Convex relaxations of
475   the problem (Singh et al., 2018; Xu et al., 2020) and efficient branch-and-bound techniques (Wang
476   et al., 2021) have shown to achieve impressive verification results (Brix et al., 2023). Recently,
477   non-convex relaxations have also shown to be effective in this field of research (Kochdumper et al.,
478   2023; Wang et al., 2023; Ladner & Althoff, 2023; Fatnassi et al., 2023; Ortiz et al., 2023).
479
      While the robustness of large language models against adversarial prompts has been investi-
480   gated (Sun et al., 2024, Sec. 9), research on the non-existence of such prompts using formal ver-
481   ification is limited and, as in this work, only perturbations to the embedding space of the input of
482   large language models (Vaswani et al., 2017) are considered: As with standard neural networks, Shi
483   et al. (2020) use linear bounds to verify the robustness with at most two perturbed words. This work
484   has been extended to the zonotope domain (Bonaert et al., 2021), enabling the verification of longer
485   sentences with all words being perturbed. Other works also investigated finding better bounds of the
      softmax function within transformers (Wei et al., 2023; Shi et al., 2024). Recently, the importance of


                                                                            9
      Under review as a conference paper at ICLR 2025


486
      carrying dependencies between each attention head and dimension has been stressed (Zhang et al.,
487   2024); however, they also apply convex relaxations in each step and thus lose precision at each
488   nonlinearity.
489
490
      We also consider the problem of formally verifying graph neural networks (Sälzer & Lange, 2023)
      as closely related, as many of the operations within these networks (Kipf & Welling, 2017) also
491
      appear in large language models (Vaswani et al., 2017). Thus, progress in either domain also pro-
492
      pels progress in the other. In graph neural networks, approaches consider problems with perturbed
493   input (Zügner & Günnemann, 2019) and also perturbed graph structure (Bojchevski & Günnemann,
494   2019; Jin et al., 2020; Ladner et al., 2024).

497   6   L IMITATIONS
498
499   Our work relies on the implicit assumption that words with similar meanings have similar word
500   embeddings and are thus captured by constructing an ℓ∞ -ball around the embedding of a given
501
      word. While literature (Harris, 1954; Li & Yang, 2018) and our experiments suggest that this holds,
      we cannot guarantee that we capture all synonyms. More advanced perturbations might also be
502
      beneficial to capture more synonyms, e.g., by considering the superpositions of features in word
503
      embeddings (?). It might also be necessary to add further perturbations after the model aggregated
504   meanings of multiple words through each attention block (Vaswani et al., 2017, Appendix). For
505   example, by adding perturbations after each transformer block (Alg. 1, lines 2 to 10)), one might
506   capture the example sentence (2) as a synonym of (1). Furthermore, our approach does not output
507   the original harmful prompt to the adversarial user prompt (Sec. 1). We can only say that there
508   might exist an area in the embedding space, which might contain a harmful prompt. Additionally,
509   this unsafe area might also not correspond to an actual synonym sentence as it is sparsely populated.
510   However, we would rather want to be safe than output harmful content.
511   Our approach continues the progressive efforts to formally verify large language models (Shi et al.,
512   2020; Bonaert et al., 2021). However, all methods are not yet applicable to modern-size large lan-
513   guage models (Achiam et al., 2024), where both high precision and low verification time are desir-
514   able. Our approach is a generalization of Bonaert et al. (2021) based on zonotopes, where a balance
515   between precision and speed is adaptively struck via the parameter ρlim . Please note that the total
516   number of generators gmax also limits the precision. Order reduction methods are more challenging
517   for polynomial zonotopes than for zonotopes due to the additional complexity in the set represen-
518
      tation. Recently, an approach was made addressing this underexplored problem (Ladner & Althoff,
      2024), and any improvement on this relatively new set representation also propels our approach.
519
520   It is also worth noting that we only verify the robustness of large language models against adversarial
521   prompts to fool classifier models; however, other model architectures and safety constraints are also
522   worth considering, e.g., jailbreaks (Wei et al., 2024; Casper et al., 2023), to prevent outputting false
523   claims or exposing private data. We defer to (Sun et al., 2024, Ch. 7) for a more detailed discussion
524
      on the safety aspects of large language models.
525   Please note that transformers are also used in other domains, e.g., computer vision (Bhojanapalli
526   et al., 2021; Khan et al., 2022), and our verification approach can also be applied there as our
527   enclosures are defined per layer and do not rely on a specific model.
528
529
      7   C ONCLUSION
530
531
      We present a novel approach to tightly enclose the output set of a large language model given a
532
      perturbed input. This tight enclosure is realized using non-convex set-based computing based on
533
      polynomial zonotopes, which allows one to efficiently preserve nonlinear dependencies. This preser-
534   vation of dependencies is particularly important for the verification of large language models, as the
535   transformers used within large language models repeatedly compute complex nonlinear functions in
536   parallel. The desired precision can be tuned with a single parameter at the cost of additional com-
537   putation time. These advantages of our approach are demonstrated by rigorous theoretical analysis
538   and experiments, where we can verify much larger embedding spaces than approaches developed
539   in related work. We believe this work is a significant step towards improving these models’ formal
      safety.

775   with the only difference being the division by the standard deviation σ ∈ R of the input hk−1 .
776   This change was proposed by (Shi et al., 2020, Appendix E) to ease verification and was also used
777   by Bonaert et al. (2021). The issue with the division by the standard deviation is that as the input is
778   a set Hk−1 , also the standard deviation is a set containing the standard deviation for each point in
779   Hk−1 . Thus, the division by σ in (15) is in general very outer-approximative, which heavily reduces
780   the verifiability. The argument for the modified layer normalization (7) is that it does not penalize
781   the accuracy of the obtained model (Shi et al., 2020, Tab. 5) but greatly increases the verifiability.
782   We want to stress that designing network architectures with formal verification in mind is a motion
783
      we strongly support. However, it appears that even a model without any normalization has a similar
      accuracy in their evaluation, which raises the question of whether layer normalization is necessary
784
      at all. Thus, the influence of this modification must be further investigated, especially for larger
785
      models. If the original layer normalization (15) turns out to be beneficial, a similar technique as the
786   one described for the softmax layer (Lemma 2) can be applied, as a division by a set also appears
787   there.

962   In this section, we state further experiments and give all details about the dataset and our models. The
963
      medical safety dataset (Abercrombie & Rieser, 2022) is a small written English dataset consisting
      of risk-graded medical and non-medical queries that we split in 2, 187 training, 365 validation, and
964
      365 test samples. We collapse the risk-levels into one class to enable binary text classification.
965
      The Yelp dataset (Zhang et al., 2016) consists of review texts, where we partition the dataset into
966   40, 000/20, 000 training, 4, 000/2, 000 validation, and 1, 600/800 testing samples, respectively. We
967   train both datasets for binary classification referring to the labels as positive and negative. The
968   positive label in the medical safety dataset refers to a valid medical query, while it refers to a positive
969   review in the Yelp dataset. We use a BERT tokenizer (Devlin, 2018) to compute the mapping from
970   the textual input to token identifier that we pass then to our model. Tab. 3 summarizes the used
971   hyperparameters. All computations were performed in a docker container on an Intel® Core™ Gen.
      11 i7-11800H CPU @2.30GHz with 64GB memory.

1002

1003

1005   of generators gmax counteracts this advantage; thus, the outer-approximation induced by this order

1008   to show the mean and maximum verified volume instead of the standard deviation as this data is
1009   not normally distributed on this datasets with varying sentence lengths, and thus this statistic does
