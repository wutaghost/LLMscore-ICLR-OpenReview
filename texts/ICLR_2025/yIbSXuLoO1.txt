030   1   I NTRODUCTION
031
032   Multi-armed bandit (MAB) (Robbins, 1952; Lai & Robbins, 1985; Auer et al., 2002) is a sequential
033   decision-making problem in which a learner faces a dilemma between exploiting well-observed ac-
034   tions (a.k.a. arms) and exploring new arms that may yield higher rewards. Different from the basic
035   MAB setting, where the learner selects a single arm each time slot, a more general version allows the
036   learner to select a combination of arms (called “base arms”) to form a “super arm”. The reward of the
037   super arm is the sum of the rewards from all the base arms selected. This generalization is referred to
038   as the Combinatorial Multi-Armed Bandit (CMAB) problem (Gai et al., 2012; Cesa-Bianchi & Lu-
039
      gosi, 2012). Many real-world scenarios can be naturally modeled as CMAB problems. For instance,
      in the network utility maximization problem (Low & Lapsley, 1999) for shared network resources,
040
      where several users share limited resources (e.g., communication links with limited capacity), the
041
      objective is to maximize aggregate utility for users. In this case, the utility for each user corresponds
042   to the reward for each base arm and super arms represent combinations of users. Similarly, in online
043   advertising, where each advertisement can be considered as a base arm, and displaying a set of ads
044   together on a website forms a super arm. Due to its practical relevance, a wide range of algorithms
045   have been developed to achieve near-optimal regret in CMAB problem (Kveton et al., 2015; Combes
046   et al., 2015a; Chen et al., 2016b; Wang & Chen, 2018; Merlis & Mannor, 2019).
047
      Despite their generality, most existing CMAB frameworks assume that the unknown distribution
048   of base arms remains fixed, regardless of the properties of the super arm to which they belong.
049   However, in practice, there are scenarios where the distribution of base arms changes when they
050   are pulled in super arms of different sizes, resulting in multiple distributions for each base arm.
051   For example, in utility maximization problems, when selecting more users to share the bandwidth,
052   each user gets a reduced portion, leading to lower utility (Verma & Hanawal, 2020; Fu & Modiano,
053   2021). Notably, while the reward distribution changes, a better arm still performs better compared
      to others within super arms of the same size. In the bandit context, this situation can be modeled


                                                         1

054
      as one where the reward mean for each base arm decreases when pulled with a larger number of
055   base arms, meaning each base arm follows multiple distributions. However, the order of base arms
056   remains consistent within super arms of the same size. This property also exists in online advertising
057   where users feel less engaged with a website overloaded with a large number of ads, resulting in
058   lower click-through rates for each advertisement (Wang et al., 2011; Broder et al., 2008). This leads
059   to varying reward means for base arms depending on the size of the super arm. Nevertheless, a
060   high-performing ad still ranks better than others on pages with the same number of ads, even if its
061   distribution changes.
062   To address the application scenarios described above, we introduce and study the Set-Size Dependent
063   Combinatorial Multi-Armed Bandit (SSD-CMAB) problem, with semi-bandit feedback and linear
064   reward function. In SSD-CMAB, combinations of M base arms form the super arm set. Each base
065   arm can be associated with L different distributions, depending on the size of the super arm (up to
066   L) that contains it. As a result, there are M L distributions in total for the M base arms. Note that
067   in the CMAB model the reward distribution of a base arm remains the same across all super arms.
068   In contrast, SSD-CMAB models base arm rewards as dependent on the size of the super arm, with
069
      each base arm having L different reward distributions. This is the main distinction with CMAB
      where each base arm has only one fixed distribution (Gai et al., 2012; Combes et al., 2015b), even
070
      in non-linear reward settings (Chen et al., 2016b; 2021; Merlis & Mannor, 2019). To the best of our
071
      knowledge, previous studies on arms dependent on sets (Chen et al., 2018; Takemori et al., 2020;
072   Fourati et al., 2024) have primarily focused on properties like submodularity, whereas this paper is
073   the first to focus on the order preservation property. See Appendix A for detailed connections and
074   differences between our model and CMAB as well as the literature review.
075
      The parameter space for set-size dependent arms expands significantly, adding considerable com-
076
      plexity to solving the SSD-CMAB problem. Without utilizing the structure of reward for base arms,
077
      one would need to independently learn M L distributions (see Appendix B for detailed implementa-
078   tion). However, as demonstrated in previous examples, a common property in SSD-CMAB is order
079   preservation: the order of the reward means for base arms remains consistent across super arms of
080   the same size. Traditional CMAB algorithms overlook this property in SSD-CMAB, leading to poor
081   performance as M or L increases. Therefore, an effective algorithm for SSD-CMAB must exploit
082   the order preservation property to reduce the need for learning such a large number of distributions.
083

084   algorithm afterwards which leverages the order preservation property of super arms with the same
085   number of base arms. The algorithm first learns the order of base arms in fixed-size super arms,
086   then identifies and retains a subset of super arms likely to be optimal, excluding suboptimal ones
087   with high probability. Finally, it applies a UCB-based strategy to select super arms. By utilizing the
088   order preservation property, SortUCB minimizes exploration on suboptimal super arms, allowing it
089   to focus on those more likely to be optimal.

      This section introduces the Set-Size Dependent Combinatorial Multi-Armed Bandit (SSD-CMAB)
      problem which defines size-dependent rewards and feedback for base arms. We begin with a brief
122
      explanation of the notations used in this paper.
123
124   Notations. Throughout the paper, we use [n] := {1, 2, ..., n} to denote the set of indexes to simplify
125   notations. For two vectors with the same size µ = (µ1 , . . . , µn ) and ν = (ν1 , . . . , νn ), we define
126   µ ⪰ ν when µi ≥ νi holds for all i ∈ [n]. Notation ⪯ is defined in a similar way by replacing ≥
127
      with ≤.
128   An SSD-CMAB problem instance ν involves M base arms, denoted by set [M ]. Consider a time
129   horizon of length T , the player can select a subset of base arms at each time slot as a super arm. Let
130   S denote the set of all possible subsets of base arms [M ] whose cardinality is no more than L ∈ Z+ ,
131   i.e., S := {S ⊆ [M ] : |S| ≤ L} (meaning a super arm consists of at most L base arms).
132   Unlike the classical stochastic CMAB problem, where each base arm’s reward follows a fixed i.i.d.
133   distribution, the base arm rewards in SSD-CMAB are set-size dependent. Specifically, for any base
134   arm i, when it is pulled as part of a super arm S ∈ S with size ℓ = |S|, its reward follows a
135   distribution dependent on ℓ, denoted as Pi(ℓ) . For simplicity, we use ℓS to represent the size of super
136   arm S. Since a super arm consists of at most L base arms, each base arm i has at most L possible
137   reward distributions. Without loss of generality, we assume base arm rewards are [0, 1]-valued. Let
138   µi(ℓ) denote the expected reward for arm i under Pi(ℓ) , and µi = (µi(1) , µi(2) , . . . , µi(L) ) represent
139
      the vector of mean rewards for arm i across different set sizes. This dependency on super arm size
      defines the “Set-Size Dependent” nature of the problem.

143   tribution. Let St and R(St ) represent the super arm chosen and its corresponding reward at the

      3   A LGORITHM : S ORTING U PPER C ONFIDENCE B OUND

165   In this section, we introduce details of our algorithm Sorting Upper Confidence Bound (SortUCB).
166   Compared to traditional CMAB problems, SSD-CMAB is faced with a larger challenge in handling
167   the super arms which involve more reward distributions with the same number of base arms. Con-
168   sider an SSD-CMAB problem with a maximum super arm size of L. As a result, the parameters
169   or reward means to learn expands by a factor of L compared to the CMAB setting with a single
170   distribution associated to each base arm. Those make traditional CMAB algorithms which directly
171   learn the reward means of super arms fail to maintain efficient in handling massively large amount of
172
      parameters. Those challenges urge us to leverage a structured exploration strategy which guides the
      algorithm to assign pulls to the super arms from which the algorithms can obtain more information
173
      on the structure or order specifically of the base arms. From the analysis in Section 4, the above strat-
174
      egy can efficiently lower down the pulls of arms whose reward means subject to a particular nature,
175   i.e. the order preservation property. Hence, the learning algorithm for SSD-CMAB, compared to
176   those for the traditional CMAB problems, contains additional Elimination Phase and Sorting Phase
177   where the algorithm needs to learn the structure of reward means and eliminate suboptimal super
178   arms according to the learned structure. However, the above strategy introduces another source of
179   exploration-exploitation dilemma between assigning pulls to learn the structure to eliminate super
180   arms or directly applying classic bandit learning algorithms to learn the best super arm. The above
181   dilemma results in the second challenge of SSD-CMAB. To address the above two challenges, we
182   present our SortUCB algorithm in Algorithm 1 which effectively leverages the order preservation
183
      property and learns the structure of reward distributions in an appropriate way.
184   As mentioned above, the core idea behind the structured exploration strategy in the proposed al-
185   gorithm is to leverage the order preservation property to avoid exploring unnecessary super arms.
186   Specifically, by pulling certain super arms, the algorithm can learn the order among some base arms.
187   Since any combination of base arms can form a super arm, the super arm set is exponentially large.
188   However, with the learned order, the algorithm manage to identify some super arms as suboptimal
189
      because the base arms they include have smaller reward means than others according to the learned
      order. For example, if the algorithm figures out that base arm 1 is better than base arm 2, there
190
      is no need to pull super arms such as {2, 3} or {2, 4}, as these super arms are worse than {1, 3}
191
      and {1, 4}. This means that if some particular order is learned during earlier samplings to some
192   degree (correspond to the Sorting Phase in Algorithm 1), the order preservation property allows us
193   to reduce pulling all the super arms that contain a base arm which is likely to perform poorly. The
194   analysis later on shows that the above strategy significantly narrows down the set of super arms that
195   the algorithm needs to explore. We introduce the details of implementing the algorithm below.

217   and “Sorting Phase”. We use B = [M ] to represent the set of all base arms in our algorithm. As
218   stated in the second challenge, the algorithm maintains a fixed super arm size during the exploration
219   phase, uniformly pulling super arms with the largest number of base arms (i.e., super arms of size
220   L) to gather as much information as possible.
221

222

403   5    E XTENSION TO S ET D EPENDENT C OMBINATORIAL BANDITS
404
405   In this section, we generalize the setting to cover applications where the base arm reward distribu-
406   tions may be different even in the super arms with the same set-size, and the set of feasible super
407   arms can be arbitrary, which could be subjective to any combinatorial constraints (e.g., matroids,
408   paths, matchings), rather than super-arms whose cardinality is less or equal to L. We call the model
409   Set Dependent Combinatorial Multi-Armed Bandit (SD-CMAB for short). In SD-CMAB, we con-
410   sider M base arms with a feasible super arm set S ⊆ {S ⊆ [M ] : |S| ≤ L} as the action set,
411   rather than S ′ := {S ⊆ [M ] : |S| ≤ L} in SSD-CMAB. We define a key concept termed as
412   class, where S can be partitioned     into K classes, denoted by {S1 , . . . , SK }, K indicates the total
      number of classes. That is, i∈[K] Si = S, Sk ∩ Sk′ = ∅ for any two different k, k ′ ∈ [K], where
                                    S
413
414   ∅ denotes an empty set. Each base arm i is assigned K different distributions Pi(k) for k ∈ [K].
415   And the reward of base arm i follows distribution Pi(k) when pulled in super arms from class k.
416   We use µi = (µi(1) , . . . , µi(K) ) to denote the vector of expected reward for base arm i in super
417
      arms within different classes. The order preservation property also holds for SD-CMAB. The rest
      of the settings (e.g., base/super arm reward, feedback) are the same as Section 2. As defined in
418
      (1), the objective is to find an algorithm π to minimize the cumulative regret RegT (π, ν) on bandit
419
      instance ν. To this end, we can see that SSD-CMAB is in fact the special case of SD-CMAB when
420   Sℓ = {S ∈ [M ] : |S| = ℓ}.
421
422   We propose an algorithm which is an extension version of SortUCB, called Sorting Upper Confi-
423
      dence Bound - Set Dependent (SortUCB-SD). Similar to SortUCB, the core idea in SortUCB-SD is
      to leverage the order preservation property to learn the reward distribution structure for each base
424
      arm within different super arms across various classes. After eliminating a large number of sub-
425
      optimal super arms, exploration is conducted on the remaining set of super arms. However, unlike
426   SortUCB, SD-CMAB lacks the desirable property where the distribution for each base arm only
427   changes when it is pulled in super arms of different sizes, and certain combinations of base arms
428   cannot form a valid super arm for selection. Therefore, the algorithm relies on an (n1 , n2 )-efficiency
429   Oracle (explained in Appendix F) to guide it in determining which orders of base arms to focus
430   on learning (denoted by Bh ), and which super arms should be pulled to achieve this learning (de-
431   noted by Rh ). Due to space limit, we postpone the detailed algorithm with an intuitive example in
      Appendix F. Here we propose Theorem 3 to show the upper bound for Algorithm 3.

      7   C ONCLUSION
521
522
      We propose a variant of the classic MAB problem, SSD-CMAB, where the reward of a base arm
      depends on the size of the super arm it belongs to. Our algorithm, SortUCB, leverages the order
523
      preservation property commonly seen in real-world scenarios, and we provide both upper and lower
524
      bounds for the SSD-CMAB problem. Experiments show that SortUCB often outperforms traditional
525   CMAB algorithms. Additionally, we extend our model to the SD-CMAB problem, which introduces
526   further complexity. For future work, exploring nonlinear reward functions could expand the appli-
527   cability of our approach. Furthermore, while we derive a partially tight upper bound, there is room
528   for improvement in both the algorithm and the bounds, particularly in refining the order learning
529   process, which could lead to better performance.

      A    L ITERATURE R EVIEW
703
704
705
      Multi-armed bandits, first introduced by Lai & Robbins (1985), have been studied extensively in
      various generalizations. Among these, the Combinatorial Multi-Armed Bandits (CMAB) model is a
706
      key extension of the canonical MAB model and shares some similarities with our proposed frame-
707
      work. CMAB was first introduced by Gai et al. (2012); Cesa-Bianchi & Lugosi (2012), where each
708   action corresponds to pulling a super arm composed of multiple base arms. Subsequently, Chen et al.
709   (2013; 2016b;a) proposed the combinatorial upper confidence bound (CUCB) algorithm for CMAB,
710   achieving near-optimal regret performance (Kveton et al., 2015; Combes et al., 2015a; Merlis &
711   Mannor, 2019). Beyond CUCB, other CMAB algorithms have also been developed (Wang & Chen,
712   2018; Chen et al., 2014). Moreover, Agarwal et al. (2021) investigated the order-preservation prop-
713   erty, which is central to SSD-CMAB.
714   However, these algorithms are designed for the traditional CMAB problem, where the reward of a
715   base arm follows the same distribution across all super arms. In contrast, SSD-CMAB introduces
716   set-size dependence, where a base arm’s outcome depends on the size of the super arm. Specifically,
717   each base arm in SSD-CMAB is associated with L different distributions, one for each possible
718   super arm size, whereas in CMAB, a base arm is tied to a single fixed distribution. While non-linear
719   reward functions in CMAB (Chen et al., 2016b; 2021; Merlis & Mannor, 2019) address cases where
720   super arm rewards are not simple summations of base arm rewards, the base arm outcomes in these
721   models still follow fixed distributions. As such, they cannot capture scenarios where the reward
722
      distributions of base arms vary with super arm size.
723   Our work also relates to bandits with specialized reward structures (Kleinberg & Immorlica, 2018;
724   Hsieh et al., 2022; Basu et al., 2019; Du et al., 2021; Wang et al., 2022). Recharging bandits (Klein-
725   berg & Immorlica, 2018), uplifting bandits (Hsieh et al., 2022), and blocking bandits (Basu et al.,
726   2019) explore how an arm’s pulling history affects subsequent reward realizations. A graph-based
727   model is proposed by Wen et al. (2017). Submodular bandits (Yue & Guestrin, 2011; Chen et al.,
728
      2017; 2018) are somewhat related to our model since base arm rewards may change based on the
      presence of other base arms in the super arm, while some of which require the monotonicity prop-
729
      erty as an assumption (Takemori et al., 2020; Fourati et al., 2024) for the reward function and some
730
      do not (Feige et al., 2011; Niazadeh et al., 2021; Fourati et al., 2023). However, one common prop-
731   erty that submodular function requires is the submodular property. In contrast, SSD-CMAB only
732   requires the order preservation property, which is different from that in submodular bandits. Hence,
733   our model is able to capture many scenarios that submodular bandits cannot capture.
734
      Finally, bandits with bottleneck rewards (Du et al., 2021) and shareable arms (Wang et al., 2022)
735
      study how the super arm structure affects reward realizations, but not how the size of the super arm
736
      influences the base arm reward mean itself. To the best of our knowledge, no prior work has explored
737   the Set-Size Dependent reward model proposed in this paper.
738

780
781   In this section, we detail the implementation of Algorithm 1 and analyze the computational com-
782   plexity of its three phases. The overall complexity per time slot is at most O(M log(M )).
783
784   Elimination Phase. In the Elimination Phase, the complexity arises from ranking the L smallest
785   Ni(L),t base arms and removing base arms, corresponding to lines 4 and 6 in the algorithm. Rank-
786   ing the base arms can be performed with a complexity of O(M log(M )) per time slot. Deleting
787
      base arms requires comparing the lowest upper confidence bound with the largest L upper confi-
      dence bounds, as specified in (2), which incurs a complexity of O(L). Therefore, the computational
788
      complexity of the Elimination Phase is at most O(M log(M )) per time slot.
789
790
      Sorting Phase. In the Sorting Phase, the pulled arm in each time slot is fixed to R, and the com-
791
      plexity arises from ranking the L base arms and deleting base arms, similar to the Elimination Phase.
792   Thus, the computational complexity in this phase is O(L log(L)).
793
794   UCB Phase. In the UCB Phase, the algorithm selects the super arm with the highest value accord-
795   ing to (4) in A. As shown in Appendix D, there are L super arms in A, so the algorithm only needs
796   to identify the largest value among L items, resulting in a computational complexity of O(L).

1204   number of base arms in a super arm. In each time slot, the learner plays a super arm S ∈ S, which
1205   is a set of base arms.

       in the set S, we divide S into K different classes {S1 , ..., SK }. That is, i∈[K] Si = S, Sk ∩Sk′ = ∅
