027                                                                            A BSTRACT
028
029                Can large language models (LLMs) directly serve as powerful world models for model-
030                based agents? While the gaps between the prior knowledge of LLMs and the specified
031                environment’s dynamics do exist, our study reveals that the gaps can be bridged by align-
032                ing an LLM with its deployed environment and such “world alignment” can be efficiently
033                achieved by rule learning on LLMs. Given the rich prior knowledge of LLMs, only a few
034                additional rules suffice to align LLM predictions with the specified environment dynam-
035                ics. To this end, we propose a neurosymbolic approach to learn these rules gradient-free
036
                   through LLMs, by inducing, updating, and pruning rules based on comparisons of agent-
                   explored trajectories and world model predictions. The resulting world model is composed
037
                   of the LLM and the learned rules. Our embodied LLM agent “WALL-E” is built upon
038
                   model-predictive control (MPC). By optimizing look-ahead actions based on the precise
039                world model, MPC significantly improves exploration and learning efficiency. Compared
040                to existing LLM agents, WALL-E’s reasoning only requires a few principal rules rather
041                than verbose buffered trajectories being included in the LLM input. On open-world chal-
042                lenges in Minecraft and ALFWorld, WALL-E achieves higher success rates than existing
043                methods, with lower costs on replanning time and the number of tokens used for reason-
044                ing. In Minecraft, WALL-E exceeds baselines by 15-30% in success rate while costing
045                8–20 fewer replanning rounds and only 60–80% of tokens. In ALFWorld, its success rate
046                surges to a new record high of 95% only after 6 iterations. Code is available here.

072   dynamics is both necessary and crucial to make it a promis- LLM+rules world model’s predictions.
073   ing world model, which enables us to build more powerful embodied agents. In particular, we introduce a
074   neurosymbolic world model that composites a pretrained LLM with a set of newly learned rules from the
075   interaction trajectories with the environment. This specific form of world model combines the strengths of
076   both in modeling the environment dynamics, i.e., (1) the rich prior knowledge, probabilistic, and deduc-
077   tive reasoning capability of LLMs (Hu & Shu, 2023); and (2) the hard constraints and rigorous guarantees
078   enforced by rules (Li et al., 2024a). While creating a rule-only world model for a complex environment is
079   challenging due to the massive amount of rules and uncertainty (Xiao et al., 2021), in our method, only a few
080
      complementary rules suffice to align a pretrained LLM to specific environment dynamics. This is achieved

081
      existing LLM agents usually require expensive finetuning of LLMs via RL/imitation learning on trajectory
082
      data, or memory-heavy inference with a long input context of buffered trajectories (Mu et al., 2023b; Gao
083   et al., 2023a; Yang et al., 2024; Shinn et al., 2024).
084
085   To this end, we propose “World Alignment by ruLe LEarning (WALL-E)”, which builds the neurosymbolic
086
      world model by learning complementary rules with LLMs’ inductive reasoning and code generation capa-
      bility. Specifically, in each iteration, WALL-E interacts with the environment to collect a real trajectory
087
      and compare it with the world model predictions. The comparison results are then analyzed by an LLM,
088
      which extracts new rules or modifies existing ones to improve the consistency between the predicted and
089   real trajectories. To keep the rule set minimal necessarily, at the end of each iteration, we prune the rules by
090   solving a maximum coverage problem, which aims to select a subset of rules with the maximal coverage of
091   the transitions failed being predicted by the LLM in the world model (without applying any rules). Hence,
092   the selected rules are complementary to the LLM predictions. The above rule learning procedure repeats for
093   multiple iterations until the LLM+rules performs as an accurate world model.

      The precise world model achieved by WALL-E enables us to create better model-based LLM agents for
095
      challenging open-world tasks. However, model-based reinforcement learning (RL) of LLM agents in com-
096   plex environments is still hindered by the expensive exploration and finetuning of LLMs. In this paper,
097   we revisit the classical idea of model-predictive control (MPC) (Qin & Badgwell, 2003; Hafner et al., 2019;
098   2020; 2023), compared to RL, which does not require training a policy network but needs to optimize actions
099   for a look-ahead time window in every step. To reduce the optimization cost per step, we instead apply the
100   LLM agent as an optimizer searching for the optimal look-ahead actions by interacting with the WALL-E’s
101   world model. With an aligned world model and an efficient LLM-based optimizer, MPC leads to a more
102   promising and efficient framework of LLM agents in open-world environments.
103   We evaluate WALL-E on challenging tasks in open-world environments such as Minecraft and ALFWorld
104   where the agents can explore freely and target complicated tasks. Our main contributions are threefold.
105
106   • We investigate the underexplored “world alignment” challenge for LLM agents.
107   • We propose a novel class of neurosymbolic world models based on rule learning on LLMs.
108
      • We develop LLM agents based on model-predictive control (MPC) with the neurosymbolic world model.

113   Recent studies have integrated LLMs with rule learning to improve reasoning and generalization capabilities
114   across various tasks, including numerical reasoning, knowledge graph exploration, and adherence to prede-
115   fined rules (Yang et al., 2023a; Zhu et al., 2023c; Mu et al., 2023a; Yang et al., 2023b; Luo et al., 2023).
116
      However, prior work has not focused on aligning LLM-based world models with dynamic environments.
      Our research addresses this gap by applying rule learning to enhance model-based agent performance in
117
      such contexts. Several works have also used LLMs to construct world models for task planning by translat-
118
      ing natural language into representations or combining LLMs with task-specific modules (Wong et al., 2023;
119   Guan et al., 2023; Tang et al., 2024). Unlike these approaches, we directly employ LLMs as world models,
120   leveraging their inherent knowledge for greater flexibility and efficiency. While some works use LLMs as
121   world models, typically relying on fine-tuning or human defined prompts for alignment with environment
122   dynamics (Xiang et al., 2024; Xie et al., 2024; Zhao et al., 2024; Hao et al., 2023; Liu et al., 2023). Our
123   method advances this by automatically learning rules through exploration, reducing human intervention and
124   improving performance. For a more comprehensive discussion of related work, please refer to Appendix A.

      3.3   I NFERENCE ON LLM AGENTS WITH L EARNED RULES
246
247   After completing the rule learning process, we obtain rules in two distinct forms: natural language rules
248   RNL and code-based rules Rcode . Both types of rules enhance the LLM world model’s ability to predict the
249   next state ŝt+1 within the planning framework: For natural language rules, these can be embedded directly
250   into the LLM’s input prompt to guide the model’s predictions, e.g., ŝt+1 = fwm (st , at , RNL ). For code-
251   based rules, these are applied programmatically after the LLM generates its initial prediction, e.g., ŝt+1 =
252
      ApplyRules(fwm (st , at ), Rcode ). Here, the function ApplyRules serves as a verification layer, overriding the
      LLM’s prediction if an active rule contradicts the generated outcome. For further details on rule activation,
253
      refer to Appendix G.
254
255   By integrating learned rules, the aligned LLM world model enhances the agent’s planning process signifi-
256   cantly. This alignment allows the agent to more effectively obtain optimal action sequences at:t+H through
257   two key improvements: First, the alignment leads to more accurate reward evaluations F(ŝt+1 ), increas-
258
      ing the likelihood of selecting optimal action sequences at:t+H within the MPC framework. Second, the
      aligned world model, equipped with learned rules, provides high-quality feedback that helps the agent
259
      refine at:t+H effectively. Along with predicting action results and state information, it offers auxiliary infor-
260
      mation when an action is predicted to fail, including:
261
262          • Feedback: A textual explanation of the failure based on violated rules.
263
             • Suggestion: Recommendations for corrective actions or improvements based on the current state,
264            action taken, and violated rules.
265
266   This information is crucial when an action fails, guiding the agent in revising its strategy by exploring
267   alternatives or adjusting its approach(see Appendix D.2 for examples).
268
      In conclusion, integrating learned rules improves the LLM world model’s prediction accuracy and provides
269   actionable feedback, enabling more efficient and adaptive planning.

      5     C ONCLUSION
458
459
460   We have shown that LLMs can effectively serve as world models for agents when aligned with environment
461   dynamics through rule learning. Our neurosymbolic approach bridges the gap between LLMs’ prior knowl-
462
      edge and specific environments without gradient updates. By integrating a rule-enhanced LLM-based world
      model with MPC, our agent WALL-E demonstrates superior planning and task-solving abilities. Experi-
463
      ments indicate that WALL-E outperforms baselines in Minecraft and ALFWorld, achieving higher success
464
      rates with fewer replanning rounds and reduced token usage. Specifically, WALL-E attains a 15–30% higher
465   success rate in Minecraft, requires 8–20 fewer replanning rounds, and uses only 60–80% of the tokens com-
466   pared to baselines. In ALFWorld, it rapidly reaches a 95% success rate from the 6th iteration onward. The
467   rule learning converges swiftly by the 4th iteration, outperforming buffered trajectory methods in both effi-
468   ciency and effectiveness. These results suggest that minimal additional rules suffice to align LLM predictions
469   with environment dynamics, enhancing model-based agents in complex environments.

      reasoning by generating and validating rules from training data. In the same vein, Mu et al. (2023a) devel-
618
      oped the RuLES framework to evaluate LLM adherence to developer-specified rules, addressing challenges
619
      like rule evasion through adversarial inputs. Furthermore, Yang et al. (2023b) presented the Tuning-free Rule
620   Accumulation (TRAN) framework, allowing LLMs to accumulate rules from incorrect cases to avoid repeat-
621   ing mistakes without additional tuning. Lastly, in knowledge graph reasoning, Luo et al. (2023) introduced
622   ChatRule, a framework that mines logical rules over knowledge graphs using LLMs.
623
      These studies show the potential of combining LLMs with rule learning to improve reasoning and general-
624
      ization. However, none have integrated rule learning with LLM-based world models, which is the focus of
625
      our work. We explore how rule learning can align LLM world models with specific environment dynamics,
626   thereby improving the performance of model-based agents in dynamic environments.
627
628
629   Using LLMs to Build World Models. Many studies have leveraged LLMs to construct world models for
630   planning. For example, Wong et al. (2023) proposed translating natural language instructions into adaptable
631   planning representations via LLMs, enabling flexible and context-aware world modeling. Similarly, Guan
632   et al. (2023) showed that combining pre-trained LLMs with task-specific planning modules improves task
633
      success rates by providing a more detailed understanding of the environment. Another approach, World-
      Coder Tang et al. (2024), exemplified an LLM agent that constructs world models by generating and execut-
634
      ing code to simulate various states and actions, refining its understanding iteratively.
635
636   These studies demonstrate the utility of LLMs in building world models to improve planning and reasoning
637   in complex environments. However, unlike these works, our approach directly employs the LLM as the
638   world model, utilizing its inherent knowledge and reasoning abilities without an explicit model-building
639
      phase. This direct use of LLMs enhances adaptability and computational efficiency.
640
641   Using LLMs as World Models. Several studies have explored using LLMs directly as world models by
642   leveraging their implicit knowledge. Some methods rely on fine-tuning to align the LLM world model
643   with the environment. For example, Xiang et al. (2024) fine-tuned LLMs with embodied experiences in
644   a simulated world to enhance reasoning and planning abilities in embodied environments. Similarly, Xie
645   et al. (2024) transformed LLMs into world models by incorporating knowledge of action preconditions and
646   effects, fine-tuning the models to reason about actions and predict their outcomes accurately.
647   Other approaches align LLMs as world models through prompting. For instance, Zhao et al. (2024) intro-
648   duced the LLM-MCTS algorithm, prompting LLMs to serve as both the policy and world model for large-
649   scale task planning, integrating commonsense priors with guided search. In another approach, Hao et al.
650   (2023) introduced Reasoning via Planning (RAP), where LLMs are prompted to act as reasoning agents and
651   world models by generating reasoning trees to explore solutions. Finally, (Liu et al., 2023) used a Bayesian
652   adaptive Markov Decision Process to guide LLMs in planning future trajectories, prompting them to predict
653   future states.
654   While these approaches demonstrate the potential of using LLMs as world models, they often require exten-
655   sive fine-tuning or rely heavily on human-crafted prompts, making them labor-intensive and inflexible. Our
656   work overcomes these limitations by automatically extracting rules from exploration experiences, reducing
657   human effort and enhancing adaptability across different environments.

1447   Task Details. We used the ’Tech Tree’ series tasks in MineDojo. Minecraft presents a structured progression
1448   system involving various levels of tools and armor, each with unique properties and increasing difficulty to
1449   unlock. To advance through these levels, the agent must develop and apply systematic, compositional skills
1450   to navigate the technology tree. Tasks are structured into six technology tiers: wood, stone, iron, gold,
1451   diamond, and redstone with each level presenting a higher degree of difficulty. And each level contains a
1452   certain number of tasks, which is shown in Table 5. Additionally, we only do tasks in overworld, so tasks
1453   that require materials from Nether and End to complete are disregarded (e.g. redstone observer, redstone
1454
       lamp, redstone comparator).
1455   Method Setup. We utilize GPT-4o as the backend for our method. To rigorously assess the agent’s per-
1456   formance, we initialize it in the ”open-ended” mode—the most challenging and interactive environment

       Under review as a conference paper at ICLR 2025

       available, analogous to survival mode. In this setting, the agent starts with an empty inventory and random-
1458
       ized seeds for both the environment and its starting position, requiring it to strategize effectively. Unlike
1459   creative or adventure modes, the agent must contend with the dynamic generation of hostile mobs, introduc-
1460   ing additional complexity and difficulty. Starting without any resources, the agent is forced to actively mine
1461   materials and craft essential items to progress, testing its planning, adaptability, and problem-solving skills.

       G    C ODE - BASED RULES VERIFICATION LOGIC

1553
       Criteria for Determining Whether a Rule is Active. We apply rules only in scenarios where they are
1554
       relevant to the current transition, defining this relevance as the rule being ”active” or ”effective.” A rule
1555   is considered active if its outcome of the current transition yields the specific outcome it is designed to
1556   address. Specifically, in our framework, transition success is represented as True, and failure is represented
1557   as False.
1558
1559          • Rules Designed to Identify Successes:
1560            A rule intended to detect successes is considered active when it evaluates the current transition and
1561            returns True.
1562          • Rules Designed to Identify Failures:
1563            A rule intended to detect failures is considered active when it evaluates the current transition and
1564            returns False.
1565
       In essence, a rule is active when its outcome aligns with the type of outcome it is meant to assess (either
1566
       success or failure). This ensures that rules are applied appropriately and only influence the LLM world
1567
       model’s predictions when relevant to the specific circumstances of the transition.
1568
1569   Determining Whether a Rule is Correct or Incorrect When a rule is active, if it makes an incorrect
1570   judgment—predicting success when the transition actually fails or vice versa—the rule is considered invalid
1571
       and is removed from the rule set. Transitions where the rule is not applicable—referred to as ”inactive” or
       ”dormant”—are excluded from the evaluation process.

1576   Currently, our rule learning framework generates simple rules that primarily assess whether actions align
1577   with environment dynamics (i.e., rules for transitions). Future research should explore advanced reasoning
1578   methods that enable LLMs to derive more abstract rules, such as those governing entire planning processes.
1579
       Furthermore, many embodied environments exhibit stochastic dynamics, where actions have probabilistic
       outcomes. For example, resource gathering at night in Minecraft often fails due to hostile creatures but can
1580
       sometimes succeed. Our current rule learning process cannot handle such randomness, typically classifying
1581
       these scenarios as failures. Addressing this limitation by enabling rules to account for stochastic dynamics
1582   is a promising research direction, potentially leading to more accurate and reliable world models.
