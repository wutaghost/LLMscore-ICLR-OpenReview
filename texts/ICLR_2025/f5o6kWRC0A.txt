011                                              A BSTRACT
012
013            Source-free Unsupervised Domain Adaptation (SFUDA) aims to adjust a source
014            model trained on a labeled source domain to a related but unlabeled target do-
015
               main without accessing the source data. Many SFUDA methods are studied in
               closed-set scenarios where the target domain and source domain categories are
016
               perfectly aligned. However, a more practical scenario is a partial-set scenario
017
               where the source label space subsumes the target one. In this paper, we prove
018            that reducing the differences between the source and target domains in the partial-
019            set scenario helps to achieve domain adaptation. And we propose a simple yet
020            effective SFUDA framework called the Machine Unlearning Framework to alle-
021            viate the negative transfer problem in the partial-set scenario, thereby allowing
022            the model to focus on the target domain category. Specifically, we first generate
023            noise samples for each category that only exists in the source domain and generate
024            pseudo-labeled samples from the target domain. Then, in the forgetting stage, we
025            use these samples to train the model, making it behave like the model has never
026
               seen the class that only exists in the source domain before. Finally, in the adap-
               tation stage, we use only the pseudo-labeled samples to conduct self-supervised
027
               training on the model, making it more adaptable to the target domain. Our method
028
               is easy to implement and pluggable, suitable for various pre-trained models. Ex-
029            perimental results show that our method can well alleviate the negative transfer
030            problem and improve model performance under various target domain category
031            settings.

034   1   I NTRODUCTION
035
036   Although computer vision tasks have achieved significant success in various fields (Voulodimos
037   et al., 2018), current deep models often experience a decline in performance when tested in new
038   environments that have a considerable domain gap from the training environment (Patel et al., 2015).
039
      Unsupervised Domain Adaptation (UDA) successfully addresses model performance degradation
      by automatically learning transformations of the feature space, allowing knowledge from the source
040
      domain to be transferred to the target domain (Wilson & Cook, 2020). However, traditional UDA
041
      methods require datasets and labels from the source domain (Ganin et al., 2016). Firstly, the dataset
042   from the source domain is likely to be quite large (1∼100 GB) (Lin et al., 2014; Yu et al., 2020),
043   which presents significant challenges in terms of storage and transmission. Secondly, many datasets
044   are not publicly available due to privacy and security considerations (Sun et al., 2017; Tian et al.,
045   2023; Nagrani et al., 2018). In many cases, we can only access models trained in the source domain,
046   which has led to widespread interest in Source-Free Unsupervised Domain Adaptation (SFUDA).
047
      SFUDA aims to transfer a pre-trained model from the source domain to the target domain without
048   source data (Fang et al., 2024). Due to the lack of images and labels from the source domain, SFUDA
049   cannot directly employ the methods used in UDA, such as adversarial learning (Goodfellow et al.,
050   2014; Ganin et al., 2016). In addition, most SFUDA methods focus on closed-set scenarios, where
051   the source and target domain samples come from the same category set (Fang et al., 2024; Liang
052   et al., 2024). But a more realistic scenario is the partial-set shown in Figure 1, where some source
053   domain categories do not exist in the target domain. The mismatch in class labels can lead to the
      problem of negative transfer, making domain adaptation tasks more difficult. Unlike in the UDA

      To this end, we propose a Machine Unlearning framework aimed at solving the negative transfer
      problem caused by the mismatch of category information between the source and target domains,
087
      thereby achieving better performance of the model in the target domain. The framework consists of
088
      two main stages: in the forgetting stage, we use a pre-trained model to make predictions on the target
089   domain data, selecting reliable samples as a pseudo-label dataset based on the model’s confidence
090   in its predictions. Simultaneously, we generate noise samples for each category that only exists
091   in the source domain to create a noise sample dataset. We use them to train the model, allowing
092   it to forget the information of each redundant category in the source domain while minimizing
093   the impact on other categories. In the adaptation stage, we use the reliable pseudo-label dataset
094   for self-supervised learning of the model, allowing the model to adapt more effectively to the target
095   domain. The parameters in our framework are easy to adjust, and training the model does not require
096   high performance hardwares, which makes our approach easy to implement. The experiment prove
097
      that our methods effectively solve the negative transfer problem and improve the performance of
      the model in the target domain under multiple target domain category settings. Overall, the main
098
      contributions of this work are described as follows:
099
100          • We study the partial-set scenario in SFUDA and prove that reducing the differences be-
101            tween the source and target domains in the partial-set scenario helps to achieve domain
102            adaptation.
103
             • We innovatively introduce machine unlearning into SFUDA, and design an efficient and
104
               easy-to-implement framework. Furthermore, our method is pluggable.
105
106          • The experimental results show that our work can effectively alleviate the negative transfer
107            problem in partial-set scenario and improve the accuracy of the model on target data under
               various target domain category settings.

      2   RELATED WORK
109
110
111   Unsupervised Domain Adaptation. Due to the differences between domains, classifiers trained
112   on the source domain may experience performance degradation when tested on the target domain.
113
      UDA specifically addresses the situation where target data are not labeled in domain adaptation
      (Ganin et al., 2016; Yang et al., 2024), eliminating the reliance on potentially expensive target data
114
      labels in domain adaptation. The main UDA methods focus on learning domain-invariant features,
115
      with the aim of aligning the characteristics of the target domain data with those learned during model
116   training (Zhao et al., 2019). In addition, there are many studies that address the partial-set problem
117   in UDA, such as Guo et al. (2022) uses maximum cosine similarity (MoC) to select useful data in
118   the source domain for retraining, in order to reduce domain differences; Wang & Breckon (2021)
119   uses Locally Preserved Projection (SLPP) to better align two domains in the subspace, and detectes
120   the source domain category of non target domain categories, deletes them, and retrains the model.
121   However, UDA relies too heavily on source domain data, whereas SFDUA is more practical.
122   Source-free Unsupervised Domain Adaptation. SFUDA is a domain adaptation method that
123   uses only the source domain model without using the data of the source domain. Recently, an
124   increasing number of SFUDA methods have been applied in fields such as image classification,
125   object recognition, and face anti-spoofing (Liu et al., 2021; Chen et al., 2022; Liu et al., 2022). The
126   main SFUDA research includes methods for generating source domain data and fine-tuning models
127   (Fang et al., 2024), and even studies that use API services of source domain models for knowledge
128   distillation (Yang et al., 2022; Liang et al., 2022). While the aforementioned methods demonstrate
129   strong performance, most studies are constrained to closed-set scenarios. In practice, it is rare for the
130
      source domain and the target domain to completely share the label space, and the labels in the target
      domain usually only comes from a subset class of the source domain. In the partial-set scenario,
131
      directly using traditional transfer methods cannot address the issue of negative transfer, which leads
132
      to a decline in the model’s performance. Thus, we are committed to solving the negative transfer
133   problem caused by mismatched label spaces between source and target domains, and achieving more
134   effective knowledge transfer.
135
136
      Machine Unlearning. The goal of Machine Unlearning is to replicate a model that consumes less
      time and performs consistently with models trained without specific data (Nguyen et al., 2022; Wang
137
      et al., 2023; Li et al., 2024). This is a special requirement arising from privacy, availability, and the
138
      right to be forgotten (Dang, 2021). In fact, removing the influence of abnormal training samples
139   from the model can also lead to higher model performance and robustness (Chien et al., 2023; Wang
140   et al., 2023). The methods of Machine Unlearning can be roughly divided into data reassembly
141   methods and weight model manipulation methods. For example, SISA(Bourtoule et al., 2021) par-
142   titions and sorts data, retraining only the model for the partition containing the forgotten data; Task
143   Arithmetic(Ilharco et al., 2023) modifies pre-trained model behavior via task vectors, enabling sig-
144   nificant performance reduction in the targeted task through task vector subtraction while minimally
145   affecting other tasks. We use the mechanism of Machine Unlearning to eliminate categories that
146   only exist in the source domain, addressing the negative transfer problem in artial-set scenarios and
147
      thereby enhancing the model’s performance in the target domain.

151
152   In this section, we first describe the partial-set scenario task under source-free unsupervised domain
153   adaptation. We then prove that reducing the disparity between the source and target domains in the
154   partial-set scenario facilitates domain adaptation. Next, we elaborate on our proposed method, the
155   Machine Unlearning Framework, which consists of three steps: acquiring the sample set needed
156   for model updating, the forgetting stage and the adaptation stage. Specifically, the framework uti-
157   lizes a filtering mechanism to filter the pseudo-labels in the target domain. It then generates noise
158
      samples for each category unique to the source domain to facilitate model forgetting. Finally, it
      employs pseudolabels from the target domain for simple self-supervised learning. In particular, our
159
      framework is capable of iterative updates, enabling the model to achieve stronger performance.
160

187
188   In SFUDA settings, we have a labeled source domain dataset Ds = {(xis , ysi )}ni=1s
                                                                                           and an unlabeled
                                 i nt
189   target dataset Dt = {(xs )}i=1 , where ns and nt represent the number of samples in the source
190   and target domains, respectively. The source domain model trained in Ds is defined as hs . SFUDA
191   aims to predict the labels of the target domain using the target model trained from the source domain
192   model. Similarly, we define the target domain model as ht . We define the sets of source domain
193
      and target domain categories as Cs and Ct , respectively. The current deep models are trained on
      large-scale datasets such as ImageNet-1K (Deng et al., 2009). We consider partial set scenarios that
194
      are more realistic than closed set scenarios, and we have Ct ⊆ Cs .
195
196   In classification tasks, the model not only outputs the predicted class for the image but also provides
197   the score associated with the predicted class. We define the scoring function of the source domain
198   model hs in the hypothesis space F as fs :x → RCs where the output on each dimension represents
199   the predicted score for that category. We know that the classification model will output the category
200
      with the highest score. We define the prediction category of hs for x as y = hs (x), with a little
      abuse of notation, we consider fs (x, y) is the score corresponding to the prediction category of hs
201
      for x. Similarly, we define the scoring function of the target domain model ht as ft :x → RCt , and
202
      we consider ft (x, y) is the score corresponding to the prediction category of ht for x.
203
204   If we directly apply the traditional SFUDA method to obtain the model htr , the corresponding
205   scoring function is ftr :x → RCs . This model may output only the classes presented in the source
206   domain, which leads to the negative transfer issue. Therefore, we focus on reducing the discrepancy
207   between the source and target domains in partial-set scenarios through Machine Unlearning, thereby
208
      alleviating the negative transfer issue associated with domain adaptation and ultimately improving
      the model’s performance in the target domain. Next, in order to demonstrate that reducing the
209
      discrepancy between the source and target domains in partial-set scenarios is helpful for domain
210
      adaptation, we provide a theoretical analysis of model transfer errors.

300
301
      We can easily obtain the category set Cf that only exists in the source domain through Cf = Cs −Ct .

302

303
      Cf and use these anti-samples to update the model, making the model to forget about Cf and thus

305   domly initializing a batch of noise matrices N through a normal distribution N (0, 1), and optimizing
306   the noise matrix N as our anti-samples by solving the following optimization problem:

382   stage. We set K = 7 for each category of Ct and generate 32 noise samples for each category of

386   To simulate complex partial-set scenarios in reality, we have set multiple different target domain

533
      In this paper, we aim to address the negative transfer problem in the partial-set scenario of SFUDA,
534   to achieve satisfactory performance of the model in the target domain. We have demonstrated that
535   reducing the differences between source and target domains in a partial-set scenario is beneficial.
536   Based on this, we propose a Machine Unlearning framework to solve the negative transfer problem,
537   and experiments show that our method significantly reduces the number of negative transfer samples,
538   effectively alleviating the issue of negative transfer in partial-set, thereby improving the accuracy of
539   the model. In addition, as a simple and effective pluggable method, our method is suitable for
      various deep networks.
