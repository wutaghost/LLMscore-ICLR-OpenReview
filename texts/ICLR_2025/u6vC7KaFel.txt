008                                               A BSTRACT
009
010            While Foundation Models provide a general tool for rapid content creation, they
011            regularly require task-specific adaptation. Traditionally, this exercise involves
012            careful curation of datasets and repeated fine-tuning of the underlying model.
013
               Fine-tuning techniques enable practitioners to adapt foundation models for many
               new applications but require expensive and lengthy training while being notably
014
               sensitive to hyper-parameter choices. To overcome these limitations, we introduce
015
               Text-to-LoRA (T2L), a model capable of adapting Large Language Models on
016            the fly solely based on a natural language description of the target task. T2L is a
017            hypernetwork trained to construct LoRAs in a single inexpensive forward pass.
018            After training T2L on a suite of 9 pre-trained LoRA adapters (GSM8K, Arc, etc.),
019            we show that the ad-hoc reconstructed LoRA instances match the performance
020            of task-specific adapters across the corresponding test sets. Furthermore, T2L
021            can compress hundreds of LoRA instances and zero-shot generalize to entirely
022            unseen tasks. This approach provides a significant step towards democratizing
023            the specialization of foundation models and enables language-based adaptation
024
               with minimal compute requirements. Our code and pre-trained checkpoints will be
               available through GitHub and HuggingFace upon publication.

027   1   I NTRODUCTION
028
029   Biological systems are capable of rapid adaptation, given limited sensory cues. For example, the
030   human visual system can tune its light sensitivity and focus through neuromodulation of the fovea and
031   rod cells (Wurtz et al., 2011; Digre & Brennan, 2012). While recent Large language models (LLMs)
032   exhibit a wide variety of capabilities and knowledge, they remain rigid when adding task-specific
033   capabilities. In such cases, practitioners often resort to re-training parts of the model (Gururangan
034   et al., 2020; Wei et al., 2021; Dettmers et al., 2022; Tay et al., 2021) using parameter-efficient fine-
035   tuning techniques, e.g., Low-Rank Adaptation (LoRA, Hu et al., 2022). Typically, a LoRA adapter
036
      has to be optimized for each downstream task and requires task-specific dataset and hyperparameter
      setting. This fine-tuning scheme for adaptation significantly limits the possibility of transferring
037
      knowledge between tasks and induces engineering overhead.
038
039   Recently, it has been observed that by inducing structural constraints, the low-rank matrices learned
040   by LoRA adapters can be further compressed. For example, one can train lossy versions of the
041   original adapter while maintaining downstream performance (Brüel-Gabrielsson et al., 2024; Kim
042
      et al., 2024; Kopiczko et al., 2024). Furthermore, multiple LoRAs can be combined for new tasks
      at inference time (Ostapenko et al., 2024). At the core of these approaches lies the explicit use of
043
      decomposition or dimensionality reduction techniques (e.g., SVD or routing) for better compression
044
      and online composition of existing LoRAs. This raises the following questions:
045
046
047
                1. Can we end-to-end train a neural network to compress many pre-trained LoRAs?
048             2. Can we decode new task-specific LoRA adapters from this network solely based on
049                natural-language instructions for an unseen task at test time?
050
051   We hypothesize that different LoRA adapters share the same underlying adaptation mechanism and
052   can be optimized simultaneously without any explicit structure or recipe for combining them. To
053   explicitly test this hypothesis, we propose T2L (see Section 1), a hypernetwork (Ha et al., 2016) that
      compresses task-specific LoRAs and generates new LoRA adapters zero-shot at inference time. T2L

123   Low-Rank Adaptation (LoRA, Hu et al., 2022): LoRA is a parameter-efficient fine-tuning method
124   that freezes the pre-trained weights of a base model and only learns low-rank weight matrices, which
125
      serve as an adapter to the base model. For each selected linear transformation h = W0 x, the
      fine-tuned transformation is given by h = W0 x + ∆W x = W0 x + B T Ax, where A, B ∈ Rr×d
126
      are weight matrices of rank r < d. We drop the layer index and module type of the LoRA weights
127
      when referring to all LoRA weights simultaneously. Otherwise, we use subscripts to represent the
128   layer index and module type, e.g., ∆Wm,l , where m is the module type (e.g., query projection) and l
129   is the layer index.
130
131
      Hypernetworks: A hypernetwork is a neural network that generates parameters for another ‘base’
      network (Ha et al., 2016). It serves as an indirect encoding (Schmidhuber, 1997; Stanley & Miikku-
132
      lainen, 2003; Zhang et al., 2018; Schug et al., 2024) of the base network, given that the parameter
133
      count of the hypernetwork is much smaller. This compression is achieved by learning to share
134   parameters indirectly. More specifically, given a layer-specific descriptor vector ϕl , a hypernetwork
135   with parameters θ generates the parameters of the base model at layer l ∈ {1, . . . L} as follows:
136   Wl = hθ (ϕl ). Traditionally, the layer descriptors are either one-hot or learned vectors. The weights
137   θ are then trained via end-to-end optimization on a downstream task.

192   3.2   T RAINING H YPER L O RA VIA L O RA R ECONSTRUCTION (D ISTILLATION )
193
      The most straightforward way to train T2L is to reconstruct pre-trained task-specific LoRAs. This
194
      setup allows us to utilize publicly available libraries of LoRAs (Brüel-Gabrielsson et al., 2024;
195
      Zhao et al., 2024). Alternatively, one can also use a two-stage procedure, in which a library of
196   LoRAs is pre-trained in the first stage and then train T2L to reconstruct them. For the sole purpose
197   of compressing LoRAs, we can train T2L using one-hot or learnable vectors as task embeddings.
198   However, these embeddings do not allow zero-shot LoRA generation for unseen tasks. To enable
199   zero-shot LoRA generation, we additionally condition T2L with embeddings of natural language
200   task descriptions, which allows T2L to generate LoRA adapters for various tasks—including unseen
201   ones—given corresponding task descriptions. Given a suitable library of LoRA adapters Ω, the
202   reconstruction loss for T2L can be written as
203
                                     L(Ω, θ) = E∆W i ∼Ω |∆W i − hθ (ϕi )|.                                   (5)

206   3.3   T RAINING H YPER L O RA VIA S UPERVISED F INE -T UNING
207
      Alternatively, T2L can be directly optimized on fine-tuning datasets. Training T2L with SFT sidesteps
208
      the need for intermediate target LoRA adapters and allows for end-to-end training of the hypernetwork.
209
      This training scheme is preferred if existing trained LoRAs are not naturally clustered by their
210   functionalities or downstream tasks. For instance, t1 and t2 could be two related tasks requiring a
211   similar LLM capability, but ∆W 1 and ∆W 2 could be in different minima. Thus, T2L trained via
212   reconstruction training would have to compress numerically different ∆W 1 and ∆W 2 , making it
213   less likely to generalize. In fact, we empirically find that a T2L trained via reconstruction fails to
214   generalize to unseen tasks (Section 5.6). In contrast, an SFT-trained T2L can implicitly learn to cluster
215   tasks, which has been shown to improve zero-shot LoRA routing performance (Ostapenko et al.,
      2024). The SFT loss for T2L is given by Eq. (4).

      4     E XPERIMENTS
217
218
      We investigate the effectiveness of the different T2L architectures and training schemes in terms of the
219   compression of adapters (Section 4.1) and zero-shot LoRA generation for unseen tasks (Section 4.2).
220   As baselines, we consider task-specific LoRAs, element-wise averaged LoRA, and multi-task LoRA—
221   a LoRA adapter trained on all training tasks. We also implement Hyperdecoders (Ivison & Peters,
222   2022)—a hypernetwork that generates LoRAs on a per-sequence basis—based on our proposed
223   architectures. To boost the performance of the base models without fine-tuning, we utilize few-shot
224   in-context learning (ICL, Brown et al., 2020; Dong et al., 2024) and task description prepending, i.e.,
225   providing task description at the beginning of each query. Additionally, we include results of Arrow
226   Routing zero-shot performance from Ostapenko et al. (2024). Note that the performance can only be
227
      compared indirectly as it uses a different set of LoRA adapters and training tasks. Furthermore, there
      are likely differences in the benchmark evaluation prompts.
228
229   In most experiments, we use Mistral-7B-Instruct (Jiang et al., 2023) as the base LLM model
230   except in Tables 3 and 4 where Llama-3.1-8B-Instruct and Gemma-2-2b-Instruct are used as
231   the base models, respectively. We use gte-large-en-v1.5 (Li et al., 2023; Zhang et al., 2024) for
232   extracting the task embedding from a natural language task description. All LoRA adapters are of
233
      rank 8 and only target the query and the value projection modules in every attention block of the base
234   LLM (totaling 3.4M parameters). With this LoRA configuration, L , M , and S have 55M, 34M,
235
      and 5M trainable parameters respectively. We utilize the SNI dataset (Wang et al., 2022) for training
      LoRA adapters. We use a subset of 500 tasks following Brüel-Gabrielsson et al. (2024), 10 of which
236
      are manually chosen while the rest are randomly sampled. We use 11 tasks for hold-out validation,
237
      and remove 10 datasets due to data contamination from the evaluation benchmark tasks, leaving 479
238   datasets for training. All samples are in English.
239
240   For evaluation, we choose 10 widely used benchmarks that collectively cover a variety of LLM
241
      capability assessments, e.g., reasoning, math, science, coding, and world knowledge. Specifically, we
      include the following benchmarks: Arc-challenge (ArcC) and Arc-easy (ArcE) (Clark et al., 2018),
242
      BoolQ (Clark et al., 2019), GSM8K (Cobbe et al., 2021), Hellaswag (HS) (Zellers et al., 2019),
243
      OpenBookQA (OQA) (Mihaylov et al., 2018), PIQA (Bisk et al., 2020), Winogrande (WG) (Keisuke
244   et al., 2019), HumanEval (HE) (Chen et al., 2021), and MBPP (Austin et al., 2021). Task descriptions
245   for the training datasets and the benchmarks are fully generated, as described in Appendix G. When
246   we use a language task embedding as a part of the input, we average T2L performance using three
247   descriptions for each benchmark.

      Hypernetworks for Adaptation: Hypernetworks (Ha et al., 2016) provide a general indirect encoding
543   method for neural network weights. They have been applied to different architectures (e.g., in
544   attention, Schug et al., 2024) and training paradigms (e.g., in continual learning, Von Oswald et al.,
545   2019). Here, we focus on generating low-rank adapters using natural language instructions. Previous
546   work (Mahabadi et al., 2021; He et al., 2022; Ortiz-Barajas et al., 2024) considers hypernetworks
547   for LLM adaptation in a multi-task context but only uses learned task identifiers instead of natural
548   language for adaptation. Thus, these approaches do not enable task-wise zero-shot generalization.
549   Hypernetworks for Zero-Shot LLM Adaptation: Xiao et al. (2023) explore the use of HyperLoRA
550   on a limited set of English dialects; they only consider five dialects, one of which is unseen. Fur-
551   thermore, the hypernetwork relies on an expert-based transformation of the dialects, limiting the
552   possibility of generalization. Mu et al. (2024) propose Gisting, a method that learns to compress
553   an in-context task description to prefix tokens, allowing the language model to follow instructions
554   with less token count. However, Gisting is limited to prefix tokens—only influencing the attention
555   matrices of the base model. Thus, prefix tokens are less flexible compared to LoRA that can modify
556   different parts of LLMs, e.g., MLP blocks. Hyperdecoders (Ivison & Peters, 2022) is a hypernetwork
557
      that generates adapters on the fly based on the input sequence. While per-sequence adaptation is
      desirable for benchmark evaluation—where the LLM should always output the correct answer—we
558
      argue that description-based adaptation gives more control to users since they can steer the LLM in
559
      creative ways based on user-generated descriptions. Furthermore, the generated adapters cannot be
560   efficiently fused into the base model, leading to significant overhead for each query.
561
562   Closely related to our work are HyperTuning (Phang et al., 2023), HNET-LM (Deb et al., 2022), and
563
      HINT (Ivison et al., 2023). Differ from prior work that heavily focuses on pre-trained encoder-decoder
      models, e.g., T5 (Raffel et al., 2020) or BART (Lewis, 2019), we use frontier instruction fine-tuned
564
      models as the base models, i.e., Mistral, Llama, Gemma. Further, prior work typically relies on
565
      initializing a part of their hypernetworks from the base model (e.g., tying task encoder’s weights
566   to the base model) to achieve good performance or stable training as opposed to ours that are task-
567   embedder agnostic and can freely change the task embedding model (Section 5.4). Additionally, our
568   work utilizes generated descriptions instead of the ones provided by the SNI dataset. Overall, our work
569   improves upon prior work in several ways, including achieving task-wise zero-shot generalization on
570   various frontier instruction-tuned language models, simpler and more general hypernetwork input
571   requirements, investigation of training regimes, and more comprehensive experiments, ablations, and
572   analyses.
573   Concurrent to our work, Lv et al. (2024) propose a similar approach that utilizes a hypernetwork to
574   generate LoRA adapter at inference time. However, their hypernetwork assumes that the context
575   vector provided to the hypernetwork contains few-shot examples. In contrast, T2L only assumes a
576   task description, which users can produce themselves within seconds.
577
578
      8   C ONCLUSION
579
580
      Discussion. We rely on generated descriptions from GPT-4o mini to ensure high-quality and
581
      consistent task descriptions. It is plausible that when T2L is deployed in real-world scenarios, users
582   might not input high-quality descriptions, which could cause performance degradation on generated
583   adapters. Our results have primarily focused on LLM adaptation. However, T2L can be directly
584   applied to other LLMs or adapting vision language models. Finally, the potential for T2L trained on a
585   smaller base model to transfer effectively to larger models within the same architecture class remains
586   an open area for exploration.
587
      Limitations. We only consider LoRA as the output space of the hypernetwork. We believe there
588   are more efficient ways to modulate LLMs given a text description, e.g., directly modulate the
589   activations of the base model. Additionally, we believe the compression achieved by T2L can be
590   further optimized using well-designed inductive biases. Though T2L performs better when trained
591   on more tasks, it is yet to take full advantage of scaling up to a larger set of training tasks shown in
592   Section 5.1. Finally, although T2L exhibits robustness and signs of scalability, it still does not reach
593   the benchmark performance of task-specific LoRAs in a zero-shot manner. Achieving such potent
      zero-shot adaption is still one of the biggest challenges for T2L.

      Future Work. We plan to expand T2L across multiple architectures and explore the potential for
595   transfer between different model sizes. In addition, we intend to rigorously study the scaling laws
596   of LoRA compression as the number of T2L parameters, dataset, and compute increases. Finally,
597   our goal is to provide an openly accessible service for various T2L configurations. We envision a

952   All models trained in this work fit in a single H100 GPU (80GB of VRAM). Notably, SFT requires
953   much more memory because of the need to backpropagate the gradient through the base LLM.
954   Reconstruction training, on the other hand, should be possible in a modern consumer-grade GPU.
955
      For reconstruction training, we fix the training epochs to be 100K but scale the batch size to match
956
      the number of target LoRA adapters. This means the model trains much faster for a lower number
957   of target LoRAs while maintaining the same number of optimizer steps. For reference, training to
958   reconstruct 9 benchmark-specific LoRAs takes around 10 minutes to complete, while training to
959   reconstruct 489 SNI LoRA adapters takes around 10 hours.
960
      For SFT training with fixed compute budget, we aim to keep the number of optimizer steps the same
961
      as we do for reconstruction training. However, since we cannot fit all fine-tuning samples, we scale
962
      the number of epochs inverse to the number of training tasks.
963
964   Additionally, for reconstruction training, instead of predicting the weights directly, T2L learns to
965   predict the z-score of a normal distribution of each weight entry in the low-rank A, B matrices. At
966
      test time, the output is multiplied by the standard deviation of each element before adding to the
      mean, converting the prediction to the correct scale.
