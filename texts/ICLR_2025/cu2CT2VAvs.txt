009                                                A BSTRACT
010
011            One essential advantage of recurrent neural networks (RNNs) over transformer-
012            based language models is their linear computational complexity concerning the
013            sequence length, which makes them much faster in handling long sequences during
014            inference. However, most publicly available RNNs (e.g., Mamba and RWKV) are
015            trained on sequences with less than 10K tokens, and their effectiveness in longer
016            contexts remains largely unsatisfying so far. In this paper, we study the cause of the
017            inability to process long context for RNNs and suggest critical mitigations. First,
018
               we investigate state explosion (SE) in Mamba-2 when processing long sequences,
               a phenomenon where some channels of the state exhibit exploding values that
019
               cause severe performance degradation. With controlled experiments, we discover
020
               that the model fails to forget the earlier tokens when there is more information
021            than it can remember. We attribute this to overfitting due to the recurrent state
022            being overparameterized for the training length, thereby establishing a relationship
023            between SE and the capacity of the state. To support this hypothesis, we make an
024            important empirical observation: for any given state size, there exists a training
025            length threshold such that SE is exhibited if and only if the training length is greater
026            than this threshold. Empirically searching for this threshold for different state
027            sizes reveals that it is a linear function of the state size. We also search for the
028            maximum context length at which the model can recall contextual information
029
               and find that this context length scales exponentially to the state size. Based on
               this, we empirically train a Mamba-2 370M with near-perfect passkey retrieval
030
               accuracy on 256K context length. This suggests a promising future for RNN-based
031
               long-context modeling. Code and model checkpoints will be publicly released.

034   1   I NTRODUCTION
035
036   Recent transformer-based language models (Vaswani et al., 2017; Brown et al., 2020; Touvron et al.,
037   2023; Achiam et al., 2023; Dubey et al., 2024) have demonstrated impressive capabilities in reasoning
038   over long sequences with thousands and even millions of tokens (Team, 2024a;b; GLM et al., 2024).
039   However, they rely on the attention mechanism that scales quadratically regarding the sequence
040   length, making them extremely costly for inference over long sequences. In contrast, recurrent neural
041   networks (RNNs) (Bengio et al., 1994) have a contextual memory with constant state size. Thus,
042   during inference, their per-token computational and memory complexity scales linearly with the
043   sequence length, making them much more efficient in processing long sequences.
044   Despite the great efficiency of RNNs in processing long contexts, their long-context performances
045   are far from satisfying. Most recent state-of-the-art (SOTA) RNN-based language models (hereafter
046   referred to as RNNs for simplicity), such as Mamba-1 (Gu and Dao, 2023), Mamba-2 (Dao and Gu,
047   2024), the RWKV series (Peng et al., 2023a; 2024), and GLA (Yang et al., 2023) are trained on
048   sequences with less than 10K tokens. Existing works have shown that Mamba-1 and RWKV-4 suffer
049   from severe performance drops when the context length exceeds their training length (Ben-Kish et al.,
050
      2024; Zhang et al., 2024a; Waleffe et al., 2024).
051   In this paper, we study the problem that what causes the current RNNs’ inability to handle long
052   contexts and what are the possible solutions for supporting long contexts. When applying RNNs
053   to longer contexts, we observe two critical problems. (1) RNNs are unable to generalize along the
      sequence length. They exhibit abnormal behavior when the context length exceeds the training length,


                                                          1

054
      resulting in poor long-context performance. (2) Since their memory size is constant, although they
055   can process infinitely long inputs, there is an upper bound to the amount of information the state
056   can represent. Therefore, there is an upper bound of the contextual memory capacity (the maximum
057   number of tokens that can be remembered), and tokens beyond that limit will be forgotten.
058
      Then we dive deeper into the formation of the above problems. We first attribute the cause of
059
      the length generalization failure of SOTA RNNs to a phenomenon we call state explosion (SE). It
060
      describes the exploding channels in the recurrent state that cause performance degradation as the
061   model consumes too many tokens. We inspect the memory state distribution over time and discover
062   that the explosion is largely contributed by a few dominant outlier channels. These outliers cause
063   vanishing values in other channels when the output hidden representation is normalized. By analyzing
064   various components of the state update rule, we show that SE is caused by the inability to forget the
065   earliest tokens (by decaying the state with a smaller multiplier) when there is more information than it
066   can remember. To support this finding, we empirically show that it is possible to mitigate SE without
067   training by forcing the model to forget contextual information by (1) reducing the memory retention
068   and insertion strength and (2) reformulating the recurrence into an equivalent sliding window state
069
      (which makes it forget everything outside of the window). These methods allow Mamba-2 to consume
      more than 64K tokens without SE.
070
071   Additionally, we present the state overparameterization hypothesis as a possible explanation for why
072   the model fails to learn a robust forgetting mechanism. This hypothesis tells us that one can avoid
073   SE by training on sufficiently long sequences that exceed the model’s state capacity, which forces
074   the model to learn how to forget excessive information to prevent SE. We empirically validate this
075
      hypothesis by discovering that for any state size, there is a threshold for training length beyond which
      the model will not exhibit SE. This insight allows us to establish a relationship between state capacity
076
      and state size. Then, by training Mamba-2 models of different sizes, we establish that the empirical
077
      capacity is a linear function of the state size. Furthermore, we conduct the same experiments on the
078   widely used passkey retrieval task (Mohtashami and Jaggi, 2023), and show that the length where
079   Mamba-2 has near-perfect passkey retrieval accuracy is an exponential function of the state size.
080   The experiment results in a Mamba-2 370M model that can achieve near-perfect passkey retrieval
081   accuracy on 256K context length, significantly outperforming transformer-based models of the same
082   size in both retrieval accuracy and length generalizability. Our results show that the commonly
083   used training lengths for RNN-based models may be suboptimal and that RNN-based long-context
084   modeling has promising potential.
085   The main findings of this paper can be summarized as follows.
086
087
      State explosion: In Mamba-2, there exists a small number of channels with exploding values that
      cause severe performance degradation on sequences longer than the training length. (Section 4)
088
089   State overparameterization hypothesis: By analyzing the hidden representations, we find that state
090   explosion occurs because the model fails to forget when the state is being overflowed with excessive
091   information. We attribute this to state overparameterization, which establishes the connection between
092   SE and state capacity. (Section 4.3)
093   Training length threshold: In alignment with the state overparameterization hypothesis, we empiri-
094   cally discover that for any state size, there exists a training length threshold where SE is exhibited if
095   and only if the training length is below that threshold. (Section 5)
096
      Relationship between state capacity and state size: Utilizing the previous finding, we can empiri-
097   cally search for the relationship between the training length threshold and state size, and discover that
098   it is linear. We also empirically search for state capacity on passkey retrieval, and find that it scaled
099   exponentially with the state size. (Section 7.1)

      Length Generalization Most SOTA language models in the last few years have been based on the
135
      transformer (Vaswani et al., 2017) architecture. These models, when using certain variants of position
136
      encoding, can process arbitrarily long sequences. However, they exhibit severe performance drops on
137   tokens beyond the training length (Zhao et al., 2024). To alleviate this shortcoming, many works have
138   focused on modifying positional encoding (Peng et al., 2023b; Zhu et al., 2023; Ding et al., 2024; Jin
139   et al., 2024), some achieving training-free length generalization to certain extents. Similarly in spirit,
140   this study also explores some post-training modifications for enhancing RNN-based models’ length
141   generalization capabilities.
142
143   Length Generalization of Mamba Some prior works investigated the performance of Mamba as
144   a function of context length (Park et al., 2024; Wen et al., 2024). Jelassi et al. (2024) empirically
145   showed a sharp drop in performance beyond the training length for Mamba and LSTM on a copying
146   task and also showed that Mamba struggles to copy from context unless its state size grows linearly
147   with the context length. Arora et al. (2023) discussed the capacity and empirical ability to perform
148
      associative recall of transformer and some recurrent language models. In contrast, our paper explores
      the empirical relationship between state capacity and length generalization failure.
149
150   Some concurrent works have explored extending Mamba’s context length by controlling the dis-
151   cretization term (∆t in Eq. 3) (Ben-Kish et al., 2024), such as dividing it by a constant to make it
152   smaller1 . This essentially makes the memory decay factor (αt in Eq. 5) closer to 1, which makes
153   the state retain more contextual information. However, it also unnecessarily diminishes the inserted
154
      information on all tokens. Consequently, although it can mitigate SE, it results in poor performance
      in the passkey retrieval task (details in Appendix C).

      4     S TATE E XPLOSION
197
      We first evaluate Mamba-2 and RWKV-6 on language modeling and passkey retrieval and find that
198
      they fail to extrapolate beyond their training lengths. Then, we analyze the state’s distribution on
199
      different context lengths and find that a few outlier channels exhibit exploding values. Based on this
200   finding, we coin the term state explosion, which describes the phenomenon where the recurrent state
201   exhibit exploding values, causing performance degradation.
202
203
      By analyzing the components of the state’s update rule, we discover that SE is caused by the failure
      to forget past information when the context is too long. We then propose a hypothesis that explains
204
      SE from the perspective of overparameterization.

      4.4      H OW TO M ITIGATE S TATE E XPLOSION ?
355
      To further support the state overparameterization hypothesis in the previous section, we demonstrate
356
      that it is possible to mitigate SE by simply forcing the model to forget more and/or remember less
357   information. This can be achieved by training on sufficiently long sequences or by training-free
358   modifications to the update rule. We emphasize that the training-free methods serve as demonstrations
359   of the link between forgetting and SE, and we recommend training on longer sequences in practice.
360
361
      4.4.1         T RAINING ON L ONGER S EQUENCES
362
363   We employ two techniques to improve effectiveness and efficiency when training on longer sequences.
364
365
      Data Engineering To ensure that the data contains as much long-term structure as possible, we
366   filter out sequences with less than 4K tokens. Buckman and Gelada (2024) have shown that this
367   is critical for training effective long-context models. Although we train on sequences longer than
368   4K tokens, we do not use a higher length threshold because the above threshold already removes
369   about 97.6% of the data in the original corpus. To train on longer sequences, we simply concatenate
370   sequences and delimit them with a special EOS (End-of-Sequence) token.
371
372   Truncated Backpropagation Through Time In the vanilla Mamba-2, the states are initialized to
373   zeros for each data sample. Instead, we initialize the states as the final state of the previous sequence.
374   This is equivalent to concatenating multiple sequences, but stopping the backpropagation of gradients
375   at certain intervals. This technique has been shown to help extend the context length of RNNs (Yang
376   et al., 2023) and alleviate the memory cost of caching activations for computing gradients. Based on
377   Yang et al. (2023) and our preliminary tests, we use concatenate 12 sequences with this technique by
      default.

      4.4.2       T RAINING -F REE M ITIGATION M ETHODS
379
380   Here, we propose some tricks for mitigating SE by inducing more forgetting without training. Since
381   the performance of these tricks is not the focus of this paper, we report their evaluation result in
382   Appendix I. The main takeaway from these methods is that SE can be prevented by simply inducing
383   more forgetting by modifying the update rule without additional training.
384
385   Reduced Retention and Insertion Strength (RRI) This method assumes that αt and Bt control
386   the memory retention and insertion strength, respectively. We scale them with a multiplier smaller
387   than 1. The actual value is chosen by validation on the “newlines” prompt. Existing works (?) have
388   experimented with modifying ∆t , but it controls both the insertion and decay strength, making it hard
389   to analyze and control.

410   5        S TATE C APACITY
411
412   Based on the discussion and hypothesis in Section 4.3, we can avoid SE by training on sufficiently
413   long sequences such that the amount of contextual information exceeds the capacity of the state,
414   thereby forcing the model to learn how to forget. This hypothesis implies the following law:
415
416                  Let PS and Ttrain denote the recurrent state size and training length, respectively,
417                  there exists a training length threshold T∗ (PS ) such that SE is exhibited if and only
418                  if Ttrain < T∗ .
419
420   In Section 7.1, we empirically validated this by sweeping different training lengths for different
421   state sizes, and checking whether the model exhibits SE. Hence, T∗ can be viewed as an indirect
422   measurement of state capacity because it tells us when the amount of contextual information exceeds
423   the capacity. Here, it is important to disambiguate two notions of state capacity: the theoretical
424   capacity of the state and the empirically observed capacity on a given task. In this paper, we refer to
425
      the latter, which is an empirical description of the amount of context information a state can store
      without forgetting.
426
427   We conduct the training as described in Section 4.4.1. We train multiple Mamba-2 with different state
428   sizes and training lengths to find the relationship between T∗ and PS . To determine whether a state
429   has exploded, we feed the “newlines” prompt with 1M tokens to the model, and define explosion as
430   the point where perplexity is more than 2x the maximum perplexity within Ttrain tokens.

512   Figure 11 shows the state capacity of Mamba-2 on language modeling and passkey retrieval. The
513   rightmost data point in both plots corresponds to Mamba-2 370M. We have confirmed that the 780M
514   model (with a state size of 19.3M) also exhibits SE at training lengths below 128K, but do not have
515   enough resources to train the model beyond this length. The results establish a linear relationship
516   T∗ = 5.172 · PS − 4.469 between the length Ttrain = T∗ at which SE stops occurring and the state
517   size PS . The R2 value is over 0.999. This indicates that to train a Mamba-2 with robust length
518   generalization, one should use training lengths are grow linearly with the state size.
519   The second plot of Figure 11 shows that the capacity of Mamba-2 on passkey retrieval is exponential
520   concerning the state size, the function is Cpasskey = 4.756 · (1.365PS − 1) − 0.742, with an R2 value
521   over 0.999. This is because the amount of information in the context does not increase with its length.
522   In other words, we are storing a constant amount of information while the number of combinations
523   of the state grows exponentially with the number of elements. Figure 10 shows the best checkpoint
524   of Mamba-2 370M on passkey retrieval. The result is very promising because, to the best of our
525   knowledge, no previous models with less than 1B model parameters have near-perfect accuracy at
526
      this length in this task.

      8     C ONCLUSION
529
530
531
      This paper discovers and presents the first systemic study on state explosion (SE), a phenomenon in
      RNNs that causes length generalization failure. Controlled experiments on Mamba-2 reveals that SE
532
      occurs because the model fails to forget when the state is about to overflow, and we conclude that this
533
      phenomenon is caused by an overparameterized state with excessive state capacity. Then we show
534   that SE can be mitigated by inducing more forgetting by modifying the update rule or by training
535   on context lengths that exceed the state capacity. We discover that for any state size, there exists a
536   training length beyond which SE will not occur. With this insight, we empirically estimate the state
537   capacity of Mamba-2 on language modeling and the passkey retrieval task. With some simple data
538   engineering and state initialization tricks, we achieve much better performance with Mamba-2 on
539   the passkey retrieval task than existing models. Our results indicate that Mamba-2 not only is highly
      efficient in handling long sequences but also has great performance potential.

      L IMITATIONS
541
542
      All models studied in this work can be seen as a specific case of linear attention models, whose
543   recurrent state is decayed by an element-wise or scalar gate (they can be viewed as variants of Gated
544   Linear Attention (Yang et al., 2023)). We have chosen to study these models because of their strong
545   capabilities, yet, some conclusions may not be directly transferred to other variants of RNNs.
546
      Our continued pre-training approach for extending the context length of RNNs can be rather expensive,
547
      some of the models require training with up to 50B tokens, which is 1/6 of their pre-training amount
548
      of data. Also, to ensure simplicity, controllability, and generality, we have not used more advanced
549   techniques for training long-context models, such as upsampling longer data samples, better data
550   order or format, using data with more long-distance dependencies, etc.
551
552
      The passkey retrieval task that we used extensively in this study is very simple. Hence, high accuracy
      on this task may not reflect the capabilities of the model on real-world long-context tasks, because
553
      that requires more advanced capabilities such as high-resolution retrieval, reasoning, state-tracking,
554
      etc. The result is nonetheless promising because it indicates that the model can recall the correct
555   information and further capabilities may be achieved by building on the recalled information.
556
557   SE is somewhat prompt-dependent. While we found that for models with greatly overparameterized
558
      states, SE is highly consistent across different prompts, certain models with less overparameterization
      may exhibit SE on some prompts while successfully extrapolating indefinitely on others. Attributing
559
      SE to specific features of the prompt is a promising future research direction. Moreover, as evident
560
      in Figure 1, length generalization failure is less severe in RWKV-6. This is because RWKV-6 has a
561   much smaller state, as shown in Figure 12. Some architectural choices may also affect the occurrence
562   of SE, but we leave such an exploration for future work.

796   A.1       S TATE S IZE
797
798   The authors of Mamba-2 always set P = 64, N = 128, and H = 2d/P . Thus, the state size of each
799   Mamba-2 layer is HP N = 2dN = 256d. In transformer-based models, when using multi-headed
800
      attention, usually, the product of the head count H and head dimension P equals the hidden dimension
      d. Therefore, the KV cache of a transformer-based model is 2T d, which means that when using the
801
      same hidden dimension, the state of a Mamba-2 layer is equal in size to a KV cache of 128 tokens.
802
803   Compared to many other recurrent models (e.g., the RWKV series (Peng et al., 2023a; 2024), GLA
804   (Yang et al., 2023), and RetNet (Sun et al., 2023)), Mamba-2 does not have a state-less feed-forward
805   network and has considerably more heads in each layer, making the state size much larger than other
806
      recurrent models. Compared to Mamba-1 (Gu and Dao, 2023), Mamba-1 uses N = 16, which means
      that the state size in Mamba-2 is 8 times larger than the state in a Mamba-1 model of roughly the
807
      model parameter count. Figure 12 shows the relationship between state size and model size of the

      RNN models in this study.
