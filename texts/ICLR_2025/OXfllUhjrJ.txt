009                                                A BSTRACT
010
011            Existing methods for critical tasks such as out-of-distribution (OOD) detection,
012            uncertainty quantification, and adversarial robustness often focus on measuring
013            the output of the last or intermediate layers of a neural network such as logits and
014            energy score. However, these methods typically overlook the geometric properties
015            of the learned representations in the latent space, failing to capture important
016            signals that relate to model reliability, fairness, and adversarial vulnerability.
017            Innovations: We introduce an innovative method, termed Tropical Geometry Fea-
018            tures (TGF), for detecting out-of-distribution data and enhancing overall model
019            evaluation. This approach leverages the geometric properties of polytopes derived
020            from a trained neural network’s learned representations. By integrating these
021
               geometric features with the data used during training, TGF establishes a unique
               signature of in-distribution data points. Our framework extends beyond OOD
022
               detection, providing insights into model uncertainty, adversarial robustness, inter-
023
               pretability, and fairness. Through TGF, we enhance interpretability techniques
024            to detect OOD, uncertainty, adversarial robustness in dynamic and unpredictable
025            environments.

028   1   I NTRODUCTION
029
030   In the domain of machine learning (ML), the efficacy of a model is not solely measured by its
031   predictive accuracy but also by its ability to provide reliable, interpretable, and fair outcomes under
032   varying conditions. Beyond classification tasks, machine learning models must demonstrate versatility
033   in tasks such as uncertainty quantification, out-of-distribution detection, adversarial robustness,
034   fairness and bias detection, interpretability, and generalization to new tasks or domains (Mohseni
035   et al., 2022; Amodei et al., 2016; Xu et al., 2018). These aspects ensure that the model not only works
036   well on known data but can also be trusted and used effectively in dynamic, real-world environments.
037   For example, a well-trained classifier must not only provide high accuracy but also produce meaningful
038   outputs that can be used for confidence calibration, enabling better uncertainty estimation (Guo et al.,
039   2017), and for detecting adversarial examples, which are malicious inputs designed to fool the
040   model (Carlini & Wagner, 2017). Additionally, the learned representations should support model
041   interpretability, enabling humans to understand why a model made a particular prediction, and help in
042   transfer learning, where features learned on one task can be reused on related tasks, thereby improving
043   generalization (Yosinski et al., 2014).
044   While ML models have traditionally been developed with the assumption of closed-world scenarios,
045   where test data is similar to the training data (Krizhevsky et al., 2012; He et al., 2015), the true
046   utility of a model expands when it is capable of addressing diverse real-world challenges. For
047   instance, models should be able to detect when they encounter out-of-distribution (OOD) data,
048   handle adversarial threats, and ensure that their decisions remain fair and unbiased across different
049   demographic groups. Furthermore, the model’s internal representations should provide insights into
050
      its decision-making process, enhancing interpretability and explainability in complex systems.
051   In this work, we explore the theoretical and practical aspects of these learned representations, focusing
052   on how neural networks manage uncertainty, detect OOD data, defend against adversarial inputs, and
053   maintain fairness. Additionally, we build on the connection between neural networks and geometric
      structures, such as the relationship between feedforward neural networks with ReLU activation

055   understanding of how models represent data and adapt to different tasks and challenges, providing new

064
      A neural network with a ReLU activation function can be decomposed into polytopes in the vector
065
      space where the data points reside, as discussed by Sudjianto et al. (2020). This property arises
066
      because the composition of linear functions with the ReLU activation function results in a piecewise
067   linear function. Essentially, each piecewise linear segment corresponds to a distinct polytope in the
068   vector space, delineating different regions where the neural network’s behavior is linear.
069
070
      The training process facilitates the learning of these polytopes, suggesting that the geometric attributes
      of the polytopes could be pivotal in delineating in-distribution (ID) data. We hypothesize that
071
      parameters such as the volume, the number of ID points contained within the polytopes, and their
072
      density could serve as indicators to characterize ID data effectively.
073
074   In fig. 1, we present a schematic representation of our proposed algorithm which we use to detect and
075   explain out-of-distribution inputs to a pre-trained machine learning model. In the case that the input is
076
      tabular data and the machine learning model is a ReLU neural network, we analyse the entire network.
      In the case that the model expects some other form of data, e.g. a CNN for image detection, the
077
      model may consist of arbitrary initial layers that extract features (e.g. the flattened features extracted
078
      by a CNN), followed by ReLU feed-forward layers which we analyse. As ReLU feed-forward layers
079   occur as a component of many types of networks, including transformers, there are many potential
080   applications of our approach.
081
082
      Traditional features for uncertainty quantification are based on just the final layer of the network. In
      many cases, the logits—i.e., the raw, unnormalized outputs of the neural network—are utilized as
083
      features for tasks such as out-of-distribution (OOD) detection or assessing adversarial vulnerability.
084
085   However, our approach diverges from this standard practice. We introduce novel features that
086   incorporate a more comprehensive understanding of the ReLU network’s structure and behavior, such
087   as volume and density metrics. These features capture the complex interactions and characteristics of
088
      the entire ReLU network, rather than focusing solely on the final output layer. By considering the
      network holistically, our method aims to enhance the detection of OOD data, offer interpretations
089
      that may assist in understanding cases where inputs are misclassified (for example, due to outliers,
090
      dataset shift, or as a result of an adversarial attack) as well as provide additional characteristics to
091   assist in detection of bias and vulnerabilities to adversarial attacks.
092
093   In this paper, we first define Tropical Geometry Features (TGF) as the set of geometric characteristics
094
      (e.g., volume, density) of the polytopes formed by the linear regions of a ReLU-activated neural
      network in the input space. We make our code publicly available for others that wish to experiment
095
      with our features1 and outline the potential real-world applications of these features. Following this,
096
      we conduct an experiment to evaluate the suitability of TGF for OOD detection to see if our features
097   are able to capture differences between in-distribution data and out-of-distribution data not detected
098   by traditional logit-based features. Finally we conclude, noting that even weak performance on the
099   task of detecting a single OOD point can lead to strong results on tasks such as dataset shift detection.

185   2.1   D EVISION OF SPACE USING HYPERPLANES CORRESPONDING TO R E LU ACTIVATION POINT
186
187   A hyperplane in an m-dimensional vector space Rm is a subspace that has dimension m − 1. It can
188
      be described as the set of vectors that satisfy a linear equation of the form:
189                                          ax1 + bx2 + · · · + kxm = d                                          (4)
190
191   where a, b, . . . , k and d are constants in the field R (the real numbers), and at least one of a, b, . . . , k
192   is non-zero. Here, x1 , x2 , . . . , xm are the coordinates of a vector in Rm .
193   In terms of a linear map, a hyperplane in Rm can be interpreted as the kernel (or null space) of a
194   non-zero linear function. A linear functional f : Rm → R maps vectors in Rm to scalars in R. The
195   kernel of f , denoted as ker(f ), is the set of all vectors v ∈ Rm such that f (v) = 0. When f is
196   non-zero, ker(f ) is a hyperplane in Rm .
197   Thus, the hyperplane can be formally defined as:
198
199                                      Hyperplane = {v ∈ Rm | f (v) = d}
200
      where f : Rm → R is a non-zero linear functional and d is a constant in R.
201
202   In this generalized setting of a linear map f : Rm → Rn , each component function of f , considering
203   f as a vector of n scalar-valued functions f = (f1 , f2 , . . . , fn ), can define a hyperplane in Rm . For
204
      each linear functional fi : Rm → R, a hyperplane can be defined as:
205                                             {v ∈ Rm | fi (v) = di }
206
207   for some constant di .

237
238   Overview In this subsection, we examine the decomposition of polytopes derived from a ReLU
239   neural network, as detailed in our neural network model. The analysis focuses on the transformation
240   of input space through each layer of the network, leading to a piecewise linear mapping that can be
241   conceptualized as a collection of hyperplanes in Rm .
242
243   First Layer Transformation Consider the first transformation in the network:
244                                               (W1 ,b1 )
245                                            Rm −−−−−→ ReLU →
                                                              − Rh1
246   As ReLU is piecewise linear, this transformation results a piecewise linear map from Rm to Rh1 . Each
247   linear region in this map is a polytope bounded by hyperplanes corresponding to the non-linearity in
248   the ReLU activation at a = 0 for each of the hidden neurons.
249
250   Recursive Process through Layers The process extends recursively to subsequent layers. For
251   instance, consider the second layer, which involves the following composite mapping:
252
                                   (W1 ,b1 )                      (W2 ,b2 )
253                           Rm −−−−−→ ReLU →
                                             − Rh1 −−−−−→ ReLU →
                                                               − Rh2
254
      As the first layer transformation results in a piecewise linear map composed of linear regions in Rm ,
255
      we can apply the process recursively to each of these linear regions. The final result is therefore
256   another piecewise linear map from Rm to Rh2
257
258   Hyperplane-Induced Decomposition             Each hyperplane effectively divides Rm into two distinct
259   regions:
260
261          • The region where ax1 + bx2 + · · · + kxm − d > 0 is assigned a binary value of 1.
262          • The region where ax1 + bx2 + · · · + kxm − d < 0 is assigned a binary value of 0.
263
264   Classification Without loss of generality, in this paper we examine neural networks designed to
265   predict a probability (e.g. the probability that an image is of a cat); as such the final output passes
266   though a sigmoid function to transform a logit of range R to a probability within the range [0, 1]. Our
267   approach allows for optionally including one final decomposition of positive logits (i.e. corresponding
268   to probabilities > 0.5) from negative logits (corresponding to probabilities < 0.5). However, our
269   approach does not allow for hidden sigmoid layers as unlike ReLU layers, sigmoid operations cannot
      be composed as a piecewise linear map.

330   3   U SE CASE OF TGF IN M ACHINE LEARNING MODELS
331
332   In this section we outline potential applications of TGF. In our experiment we focus on OOD detection,
333
      noting that if TGF can provide an improvement over a traditional OOD baseline, such features are
      likely to also find applications in many other use cases outside of just OOD. We leave it to the
334
      community to experiment and find more applications of the TGF.
335
336
             • Model Reliability and Robustness Techniques Uncertainty Quantification, Out-of-
337            Distribution (OOD) Detection, adversarial Example Detection, and Network Robustness
338            Analysis are techniques closely related to the evaluation of model reliability and robustness.
339            For instance, in traditional OOD detection, the logits, or the output from the final fully
340            connected layer of a neural network, are typically employed to identify whether an input lies
341            outside the distribution of the training data Djurisic et al. (2022); Wang et al. (2022). In this
342            work, we demonstrate how our Tropical Geometry Features (TGF) approach can be effec-
343            tively utilized for OOD detection. Furthermore, we argue that TGF has potential applications
344            in other related areas, such as uncertainty quantification and adversarial robustness.
345          • Model Interpretability The paper by Geva et al. (Geva et al., 2020) reveals that feed-
346            forward layers in transformer models store and utilize patterns from the training data to
347            make predictions about subsequent words. Simpler patterns are captured in the lower
348            layers, while more complex patterns are learned in the upper layers. This insight enhances
349            our understanding of the role these layers play in the overall language comprehension of
350
               transformer models. A crucial aspect of the feed-forward layers is that their activation
               function is ReLU, which, when considered within the TGF framework, suggests a potential
351
               for exploring deeper connections. Other works such as Pollano et al. (2023), can be seen
352
               through the lens of TGF.
353
354          • Bias Evaluation In regions that are data rich, neural networks are able to learn a detailed
355
               approximation of how inputs map to the predicted output, which may require many polytopes.
               However, in regions that are data poor, we would expect a course approximation. Therefore,
356
               in cases where the distribution of tropical geometry features differs between groups, e.g.
357
               groups based on age, race, or gender, this may be an indicator that the neural network does
358            not perform equally well on all groups, most likely as a result of insufficient training data
359            for certain groups.
360
               From an implementation standpoint, this is similar to out of distribution detection, but in
361            the case of protected attributes should be interpreted as a failure of the network to behave
362            appropriately for that input rather than an issue with the input.
363
             • adversarial attack adversarial attacks involve subtly manipulating the input data of a
364
               classifier, such that models like deep neural networks misclassify the altered input with
365
               high confidence, even though the changes are almost imperceptible. This vulnerability
366            raises serious concerns about the reliability of these models, particularly in safety-critical
367            applications Dalvi et al. (2004); Lowd & Meek (2005). Although several countermeasures
368            have been proposed Gu & Rigazio (2014); Goodfellow et al. (2014), these methods exhibit
369            significant limitations. In their work, Hein et al. Hein & Andriushchenko (2017) investigated
370            the extent to which a classifier’s decision remains stable (i.e., does not change) when the
371            input is perturbed within a certain radius in the input space. We anticipate that our proposed
372            TGF approach could enhance robustness against adversarial attacks, as it incorporates
373            additional information about the data residing within specific polytopes and leverages its
374
               geometric features.
375          • Dataset Shift Even weak evidence that a data point is out-of-distribution or anomalous can
376            become strong evidence when we observe many such values. Thus a practical application of
377            our approach could be early alerts of potential dataset shift when the distribution of volume
               or density features differs to that of the training data.

      4.1   E QUIVALENCE OF BASELINE OOD METHODS WHEN APPLIED TO BINARY CLASSIFICATION
389
390   Existing baseline techniques for OOD detection include maximum softmax probability Hendrycks &
391   Gimpel (2017) and energy score Liu et al. (2020). These are computed from the logits for each class
392   in the last layer of a neural network. While these techniques differ in performance for multi-class
393   prediction, in the case of a neural network that performs binary classification, the output may consist
394   of only a single logit that is passed through a sigmoid function to produce a probability between 0
395   and 1. Any monotonic transformation will have no effect on the separability of in-distribution data
396
      from OOD data.
397   As such, the baseline method used in this paper for OOD detection is the absolute value of the logit
398   score, which can be thought of as a measure of how confident the neural network is in its prediction.
399   The absolute value is taken as negative values of the logit corresponding to probabilities below 0.5
400   can be interpreted as confidence in the negative class. For ease of visualising large values, we further
401   log transform the result when plotting distributions, noting that monotonic transformations, such as
402
      log, exp, and sigmoid, have no effect on the separability of in-distribution data from OOD data.
403

511   5      D ISCUSSION
512
513   A key benefit of our proposed volume feature is that it can be computed from the model weights
514   without requiring any access to the training data. This makes the approach well suited for applications
515   where the training dataset cannot be shared for privacy or commercial reasons, for example, a classifier
516   trained on sensitive information. While the false positive rate was too high to safe-guard against
517   single OOD points, as argued in section 3 even a weak OOD detector for a single data point can be a
518   strong detector for other applications. In particular, we showed that the features it extracts differe to
519   baseline approaches based on logits, performing well in some cases where logits fails.
520   Our density feature performed well, but computation requires access to the count of training data
521   points that fall inside each polytope. Nevertheless, we argue that these counts could be provided in a
522   privacy preserving manner. For example, k-anonymity Sweeney (2002) could be achieved by only
523   sharing counts for polytopes that contain at least k points. When an input lies within a polytope that
524   doesn’t meet this threshold (or if the threshold is met but the density is low), the output could be
525   treated as untrusted. Contrast this to traditional density based methods such as Bishop (1994) that
526   require access to the raw training data.

624
625   Out-of-Distribution (OOD) data generation plays a pivotal role in evaluating the robustness and gener-
626   alization of machine learning models. By intentionally creating data samples that significantly deviate
627   from the training distribution, OOD data tests models against scenarios not covered during training,
628   highlighting potential weaknesses and biases. Statistically, OOD data introduces distributional shifts.
629   Mathematically, this process explores the model’s ability to handle unexpected data, emphasizing the
630   importance of designing systems that are resilient to a wide array of real-world conditions. Thus,
631
      generating and analyzing OOD data is crucial for advancing machine learning methodologies.

653        2. Color Inversion (OOD_rearrange): In this procedure for generating out-of-distribution
654           (OOD) data, we start with the original test data from the Skin dataset, which comprises
655           pixel values represented by three columns corresponding to the Red, Green, and Blue (RGB)
656           color channels.
657           To create an OOD dataset, we perform a color inversion transformation on the RGB values.
658           The inversion process is achieved by multiplying the values of these three columns by -1.
659           This mathematical operation effectively inverts the color representation, resulting in new
660           RGB values that are the opposite of the original colors in terms of intensity. For instance, a
661           pixel originally having a high intensity in the red channel will now have a high intensity in
662           the opposite direction, and similarly for the green and blue channels.
663           This transformation significantly alters the nature of the pixel data while preserving the
664           dimensional structure of the dataset, thus producing OOD data that is fundamentally different
665
              in appearance but retains the same overall format as the original dataset. This approach
              is commonly used to test the robustness of machine learning models in handling data that
666
              deviates from the training distribution.
667
              This method can be generalized and applied to other datasets, particularly when the columns
668
              are permuted to create an out-of-distribution (OOD) dataset.

670        3. Generating Out-of-Distribution Data Using Gaussian Noise Perturbation (OOD_noise):
671
              In this procedure for generating out-of-distribution (OOD) data, we start with an original
              dataset consisting of multiple features organized across different columns. The goal is to
672
              introduce randomness into each feature to create an OOD dataset that diverges significantly
673
              from the original data while maintaining the same dimensional structure.
674
              For each column in the dataset, a noise component is added. This noise is generated from a
675
              Gaussian (normal) distribution with a mean of 0 and a standard deviation of 1. Specifically,
676           a noise vector with values drawn from this distribution is independently generated for each
677           column, matching the number of rows in the dataset. The generated noise is then added
678           element-wise to the values within that column.
679           By repeating this process across all columns, the entire dataset is subjected to random per-
680           turbations, ensuring that every feature experiences unique fluctuations. This transformation
681           alters the original values, producing data that deviates from the initial distribution, thus
682           making it out-of-distribution.
683        4. Generating Out-of-Distribution Data Using Sine Transformation and Gaussian Noise
684           Addition (OOD_nonlin): In this procedure, the goal is to create out-of-distribution (OOD)
685           data by applying a combination of non-linear transformation and noise addition to the
686           original dataset, thereby producing data that is fundamentally different from the original
687           while retaining the same dimensional structure.
688           For each feature (column) in the dataset, we perform the following steps:
689            (a) Non-linear Transformation: Apply the sine function to every value in the column.
690                This transformation changes the original values into their sine equivalents, introducing
691                non-linearity and altering the data’s nature. Mathematically, if x represents the original
692                data values in a column, the transformation becomes:
693

694

696                vector ϵ from a Gaussian (normal) distribution with a mean of 0 and a standard deviation
697                of 1, matching the number of rows in the dataset. This noise vector is then added
698                element-wise to the transformed values within the column. The resulting transformed
699                data for each feature becomes:
700                                                   xood = sin(x) + ϵ
