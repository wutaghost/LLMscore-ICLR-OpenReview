009                                              A BSTRACT
010
011            Processing long-context input imposes a heavy computational burden when de-
012            ploying large language models. Recently proposed inference-time methods accel-
013            erate generation by attending only to local context. Despite its efficiency gains,
014            this approach fails to capture all relevant information in the input, showing sub-
015            stantial performance drop in long-context benchmarks. We propose recycled at-
016
               tention, an efficient and effective method which alternates between full context
               attention and attention over a subset of input tokens. When performing partial
017
               attention, we leverage the attention pattern of a nearby token that has performed
018
               full attention and attend only to the top K most attended tokens. We evaluate
019            our methods on RULER, a suite of tasks designed to comprehensively evaluate
020            long-context abilities, and long-context language modeling tasks. Applying our
021            inference method to off-the-shelf LLMs achieves comparable speedup to base-
022            lines which only consider local context while improving the performance by 2x.
023            We further experiment with continued pre-training the model with recycled atten-
024            tion to improve the performance-efficiency trade-off.

      1   I NTRODUCTION
027
028
      Recent large language models (LLMs) are trained to ingest extremely long inputs and generate long
029
      outputs (Meta, 2024; Gemini, 2024) to support a wide range of applications. However, deploying
030   such long-context LLMs can be very costly. As the context length increases, LLMs see a linear
031   increase in memory to store the Key-Value (KV) cache and a quadratic increase in time for attention
032   computation. These two factors lead to high latency during inference; Adnan et al. (2024) showed
033   that as context length increased 16x for the MPT-7B model (MosaicML, 2023), the inference latency
034   increased by 50x, where 40% of the increase was due to the data movement of the KV cache.
035
      To improve efficiency, prior work put a limitation on the size of KV cache, i.e. the number of past
036   tokens that are available at each generation step. This leads to a meaningful gain along two axes:
037   memory requirement and time for attention computation. To form a smaller KV cache, they make
038   a locality assumption, only keeping most recent input tokens (Beltagy et al., 2020; Child et al.,
039   2019) along with a fixed number of globally available initial tokens (i.e., StreamingLLM (Xiao
040   et al., 2023)). Another line of work (e.g., H2 O (Zhang et al., 2024), Keyformer (Adnan et al.,
041   2024)) maintains a dynamically constructed fixed sized KV cache by identifying key past tokens
042   from observed attention patterns and dynamically evicting the rest during generation.
043   These approaches reported little degradation in perplexity-based evaluation for the next token pre-
044   diction task. However, they show a significant drop in performance (Sun et al., 2024) on long-context
045   benchmarks that require synthesizing information from non-local contexts (Hsieh et al., 2024). For
046   example, on the simple needle-in-a-haystack (NIAH) task, both StreamingLLM (Xiao et al., 2023)
047   and H2 O (Zhang et al., 2024) report less than 8% accuracy compared to 100% for vanilla attention.
048   Keeping a smaller KV cache is problematic when LLMs is tasked with synthesizing information
049   from long context, going beyond next token prediction where local contexts suffice. Once a key in-
050
      put token is eliminated from the KV cache (either through locality assumption or by eviction during
      the generation process), there is no way to recover access to the eliminated token. When LLMs are
051
      tasked to generate long text, it gets harder to predict which input tokens are useful in advance.
052
053   In this work, we propose a novel approach, Recycled Attention, that focuses on reducing inference
      time while comprehensively capturing long-context inputs. We keep the full KV cache throughout

      2.1       P ROBLEM S ETTING AND N OTATION
102
103   Let M be a language model trained to estimate the conditional probability of all output sequences
104   given an input x. At inference time, M generates an output ŷ ∼ M (x) in two steps: (1) Pre-filling
105   stage: M ingests the input x = x1 , · · · xL and stores the KV cache for all L tokens across all
106   layers of the transformer model, and (2) Generation stage: generate one token yi at a time from the
107   conditional distribution PM (yi |x, y1 · · · yi−1 ). At each step, the model attends to the KV cache of
      all previous tokens, and also updates the KV cache to include the current token’s key-value pairs.

108
      Our goal is to reduce the inference latency during this second stage of the generation process.
109   There are two main factors that contribute to this increased latency; first, the attention computation
110   increases quadratically with input length L. Second, a large L necessitates maintaining a large KV
111   cache of past tokens, and 40% of the inference latency can be attributed to the data movement of
112   this large KV cache from the GPU HBM (Adnan et al., 2024).
113
      Prior approaches (Xiao et al., 2023; Zhang et al., 2024; Adnan et al., 2024) achieve inference time
114
      speed-ups by limiting the size of the KV cache to a fixed size K. StreamingLLM (Xiao et al.,
115   2023) constructs this fixed size KV cache by retaining only the initial and recent tokens (illustrated
116   in Figure 1(b)) while H2 O (Zhang et al., 2024) retain a mix of local tokens and key past tokens
117   dynamically identified during generation (see Figure 1(c)). For both approaches, once tokens are
118   evicted from the KV cache, they cannot be recovered in subsequent generation steps. This can be
119   particularly catastrophic in long-context scenarios where key tokens are challenging to identify in
120   advance, e.g. cases where instructions inquiring about the past tokens are located towards the end of
121   the input. Consequently, methods that evict tokens from the KV cache often report poor performance
122   on benchmarks like RULER (Hsieh et al., 2024) that are designed to test reasoning and information
123
      synthesis capabilities over long-contexts.
124   Instead of permanently evicting tokens for all future steps, we ask: can we distinguish between
125   important and unimportant tokens for the attention computation for the next S time steps?
126   Our key hypothesis is that consecutive tokens in a sequence likely place the majority of the atten-
127   tion weights over a similar subset of tokens in the context, and this can be leveraged to increased
128   inference efficiency. We test this hypothesis for the LLaMA-3.1-8B model in the subsection below.

186
      Our approach maintains two separate KV caches Cf and Cr (size ∝ K), corresponding to the full
187
      and recycle attention steps respectively. Given input x1 , ..., xL , we first prefill M using the vanilla
188
      full attention computation and initialize our full KV cache Cf with the first L tokens. We also obtain
189   the attention scores aL for the last token xL at each layer of model M . We initialize our recycle KV
190   cache Cr with the key-value pairs of the topK tokens at each layer based on attention scores.
191
192
      At each recycle steps, i.e. S − 1 contiguous steps after every full attention step, we generate the next
      token yt ∼ M (Cr ) using the smaller recycled KV cache Cr to compute attention. This leads to a
193
      reduction in both the attention computation FLOPs as well as the latency due to movement of KV
194
      cache (we only move the smaller KV Cr instead of the larger full KV Cf , where |Cr | << |Cf |).
195   As Cr is updated with the KV cache of the new input tokens (i.e. the generated token from previous
196   step) in the forward pass, we remove the recycled token which received the lowest attention score
197   from Cr to maintain a fixed size.
198
      At each full attention step that occurs every S steps, we first update the Cf KV cache with the key-
199
      value pairs of the S − 1 tokens from the recycle step. Next, we generate the next token yt ∼ M (Cf )
200
      using the full KV cache Cf . Finally, we follow the same procedure as above to reset the recycle
201   cache Cr with the topK tokens from each layer of the current time step.
202
203
      Compatibility with Flash Attention FlashAttention (Dao, 2024) improves standard attention
204
      computation on GPU by reducing data movement, significantly improving the memory and speed
205   efficiency. It achieves this by directly producing the output for the attention blocks, without storing
206   the O(L2 ) attention matrix. However, we rely on these attention scores to select the topK tokens
207   during the full attention steps and construct our recycled KV cache Cr (lines 9-10 of Algorithm 3).
208   To make our method compatible with FlashAttention, we implement an extra step to re-compute the
209   attention score when we perform full attention. As we only perform full attention at stride of S, this
210   does not introduce significant overhead. Additionally, note that other methods that use attention pat-
211   terns (e.g. H2 O) will also show reduced speed-gain when using in conjunction with FlashAttention.

      Approach At every S th decode step, we first determine whether we need to perform full attention
      instead of always performing full attention by default. We calculate the cosine similarity between
450
      query vectors of the input token t averaged across all query heads in layer l, with the averaged query
451
      vector of the most recent full attention step for that layer. If the similarity is higher than a threshold
452   s, we decode with recycle cache, and otherwise use full attention for layer l. Our approach offers the
453   flexibility of using different schedules for different layers, but uses the same schedule for all heads
454   in the same layer. Since we perform this similarity check every S steps, setting threshold s = 1 is
455   equivalent to decoding with the fixed stride S. We perform the comparison only every S steps to
456   reduce computational overhead; we call this query comparison (QC) stride.
457
458   Setup We run experiments with Llama-3.1-8B on the Arxiv and Books corpus. As before, we
459   report perplexity and decoding time measured on one A100 with batch size of 1 for the last 256
460   tokens of each test sequence. We run dynamic scheduler with two different query comparison strides
461   {5, 10} and a similarity threshold of 0.8. We compare against Recycled Attention with fixed strides
462   10 and 15. For RULER, we report performance on two tasks which require generating longer outputs
463
      (multi-query and multi-value). We run a dynamic scheduler with two different query comparison
      strides 10 and a similarity threshold of {0.8, 0.9}. We compare against Recycled Attention with
464
      fixed strides {10, 15}. For dynamic schedules, we report the effective stride across layers, i.e. the
465
      average stride at which full attention is performed.
466
467
      Dynamic stride strategy improves perplexity compared to fixed strategy when using similar
468
      decoding times. Table 6 reports our results. We observe that using dynamic strides improves
469   the performance-efficiency trade-off across all settings. Compared to fixed stride of 10 (row 2),
470   dynamic stride with query comparison stride of 5 (row 3) achieves lower perplexity with a slightly
471   faster decoding time on both domains. Similarly, employing a dynamic stride with query comparison
472   stride of 10 (last row) achieves better or on-par performance with less decoding time compared to
473   having a fixed stride of 15 (row 4). We observe a similar trend for RULER tasks. This better trade-off
474   can be attributed to the larger effective stride size, i.e. less frequent full attention steps, that result
475   from using dynamic schedules. Overall, our experiment demonstrates that dynamically deciding
476   when to refresh the recycle cache can improve performance when using similar decode times.

511   Dynamic KV cache Recent work (Sun et al., 2024) introduces a hierarchical speculative decoding
512   method, which uses the model with a small KV cache constructed with attention pattern as draft
513
      model for the model with the full cache. While we share the motivation of using the attention pattern
      to construct a smaller KV cache, we directly leverages the dynamic cache to accelerate inference and
514
      study the performance-efficiency trade-off. Another line of recent work (Xiao et al., 2024a) proposes
515
      building a dynamic KV cache by mapping distant tokens into chunks and retrieving chunks that are
516   similar to the current token, with the focus of extending the context size of the language model.
517   Quest (Tang et al., 2024) uses the minimal and maximal key values to estimate import tokens for the
518   query embedding of the current input token and load the KV cache of these tokens to decode.
519
520
      KV cache eviction As performing attention over the full KV cache imposes a high memory and
521   computation burden, KV cache eviction methods have been extensively studied. Strategies include
522   keeping only “sink” and recent tokens in the KV cache (Xiao et al., 2023); or tokens with high
523   accumulative attention scores (Zhang et al., 2024). Building on the idea of attention-based eviction
524   strategy, PyramidInfer(Yang et al., 2024b) retains different number of tokens per layer. Another line
525   of work proposed query-aware eviction strategies, using the attention scores of the last few tokens
526   in the prompt to select tokens to keep (Li et al., 2024; Cai et al., 2024b; Chen et al., 2024). Other
527   works design eviction strategies based on attention patterns of different heads (Ge et al., 2024; Xiao
528   et al., 2024b) or different layers (Yang et al., 2024b).

530   8   C ONCLUSION
531
532
      We propose recycled attention, an inference-time method which maintains a small, dynamic KV
533
      cache based on attention patterns of neighboring tokens. Our work follows a line of work leveraging
534   the locality assumption during the attention computation. Instead of using locality to directly decide
535   which tokens to attend to (only selecting nearby tokens), we recycle the attention pattern of nearby
536   tokens, allowing more flexible and dynamic sparse attention patterns. We apply our method to two
537   off-the-shelf long-context model and show that our method reduces inference wall-clock time while
538   better preserving performance compared to prior methods which keep a KV cache of recent tokens.
539   Finally, we show that continued pre-training the model with recycled attention and employing a
      dynamic stride can further improve the performance-efficiency trade-off.

736   A.1   A BLATING K AND S
737
738   Our method depends on two hyperparameters, the size of the recycle cache K and the stride S which
739
      governs how often we perform full attention and update the recycle KV cache. Here, we analyze
      the impact of varying these two values for Llama-3.1. We experiment with 100 ArXiv sequences
740
      with L = 16, 354 and 14 RULER tasks with L = 32, 768. Results are reported in Table 8 and
741
      Figure 4. We see that Recycled Attention outperforms baselines with similar inference time budget
742   for both tasks. For example, Recycled Attention with K = 2048, S = 16 achieves better perplexity
743   than StreamingLLM with K = 4096. In fact, Recycled Attention with K = 4096 achieves better
744   accuracy than StreamingLLM with a larger K = 8192 for RULER. Overall, we find that increasing
745   K is more effective than decreasing the stride S. While decreasing stride S generally benefits
746   Recycled Attention, it has negligible effect on StreamingLLM++. This shows that the improvement
747   does not merely come from performing full attention, but also from refreshing the recycle cache.
748   We further report the detailed breakdown of each RULER tasks in Table 9. We see that decreasing
749   the stride benefits certain tasks, such as the multi-key version of NIAH and common word extraction.

      speed. SnapKV is slower than vanilla decoding in most of the cases for LLaMA-3.1-8B, as it is
      primarily designed for memory efficiency, instead of decoding speed.

852   We see that Recycled Attention outperforms SnapKV and baseline methods for the two tasks that
853   require long generation (QMSUM and GovReport), especially when we decrease the strides. This

856   Results for LongBench datasets with less than 5K input length is reported in Table 11. Overall, we
857   see a similar trend but with less efficiency gains.

       A.8   L IMITATIONS AND F UTURE W ORK
1081
1082   Proposed method While we focus on accelerating inference speed, our method does not reduce
1083   memory requirement for using long-context LLMs, which can be a bottleneck for certain use cases.
1084   Our method is focused on a setting where we generate long output given a long input. When the
1085   output length is very small, the efficiency gain will be minimal. In this study, we focus on employing
1086   a fixed stride across all layers and explore dynamic scheduling based on query-similarity. Setting a
1087   custom stride per layer, or exploring other methods for deciding when to recycle the cache could be
1088
       future avenue to improve performance.
1089
       Experimental Settings We have conducted experiment with two open-sourced long-context mod-
1090
       els and two evaluation tasks setting. We did not test out more language models and other long-
1091
       context benchmarks (An et al., 2023; Karpinska et al., 2024) given our limited compute resources.
1092   Finally, our method is not limited to the language domain. Future work can explore applying recy-
1093   cled attention to other modalities, for instance, vision transformers.
