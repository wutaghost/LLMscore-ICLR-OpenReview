009                                               A BSTRACT
010
011
               Given two neural network classifiers with the same input and output domains, our
012            goal is to compare the two networks in relation to each other over an entire input
013            region (e.g., within a vicinity of an input sample). Towards this, we introduce
014            and quantify the Relative Output Margin (ROM) with which decisions are made.
015            A larger output margin for a network w.r.t. another indicates that this network
016            consistently makes a correct decision every time the other network does, and it does
017            so in the entire input region. More importantly, as opposed to best-effort testing
018            schemes, our framework is able to establish provably-correct (formally verified)
019            bounds on ROM gains/losses over an entire input region. The proposed framework
020
               is relevant in the context of several application domains, e.g., for comparing a
               trained network and its corresponding compact (e.g., pruned, quantized, distilled)
021
               network. We evaluate our framework using the MNIST, CIFAR10, and two real-
022
               world medical datasets, to show its relevance.

026   1    I NTRODUCTION
027
028   Quantitative comparison of neural networks, e.g., in terms of performance, is a fundamental concept
029   in the Machine Learning (ML) domain. One common example is when a network is pruned,
030   quantized, or distilled to run the compact networks on edge devices or smart sensors. In the
031   medical domain, for instance, neural networks can enable implantable and wearable devices to detect
032
      cardiac arrhythmia (Sopic et al., 2018a) or epileptic seizures (Baghersalimi et al., 2024) in real time.
      However, due to their limited computing resources, such devices often adopt the compact networks
033
      corresponding to the original medical-grade networks. It is vital for the compact network to reliably
034
      detect cardiac abnormalities/seizures, as lack of reliable decisions can jeopardize patients’ lives.
035   Therefore, reasoning about the decisions made by the compact network w.r.t. to an original/reference
036   network is vital for the safe deployment of the compact networks.
037
038
      Traditionally, testing techniques have been adopted to quantitatively compare two neural networks.
      However, testing techniques often cannot cover all possible scenarios, hence do not provide any hard
039
      formal guarantees. This is required in the context of safety-critical applications, e.g., in the medical
040
      domain as discussed above. Therefore, here, we investigate providing hard correctness guarantees
041   based on formal verification techniques.
042
043   In this work, we focus on neural network twins, i.e., two neural networks trained for the same
044
      learning/classification task, with the same input and output domains, but not the same weights and/or
      architectures. We define an input region as the region in a vicinity of a given input sample, e.g.,
045
      captured by the absolute-value/Euclidean/maximum norm centered around the input sample. Given
046
      network twins and an input region, we investigate whether it is possible to prove that, in the entire
047   input region, one network consistently makes a correct decision every time the other network does.
048
049   To this end, we consider the Output Margin (OM) of a network. Let us focus on binary classification
050
      for the simplicity of presentation. Given an input sample, the OM for one classifier corresponds to
      the magnitude of the change/perturbation in the output that leads to misclassification of the input
051
      sample. The larger the OM, the larger the adversarial perturbations needed to toggle the decision.
052
053   To quantitatively compare the neural network twins, we examine the OM of the two classifiers relative
      to each other, which we refer to as Relative Output Margin (ROM). Given the network twins and a


                                                         1

054
      common input sample, ROM allows us to quantify the magnitude of the OM for one network relative
055   to the other. Therefore, ROM captures whether the OM for one network is larger than the other.
056
057   We, then, extend the notion of ROM to Local Relative Output Margin (LROM), where we account
058
      for an entire input region (all perturbed inputs captured by the absolute-value/Euclidean/maximum
      norm centered around the input sample). That is, LROM not only enables us to reason about the
059
      given input samples, but also to reason about the entire region in the vicinity of the samples.
060
061   In this paper, we propose a framework to establish safe (provably-correct) bounds on LROM and
062   formal verification guarantees on the decisions made by neural network twins in the entire input
063   region. LROM enables us to formally prove that a network consistently makes a correct decision
064
      every time the other network does, and it does so in the entire input region. We evaluate our proposed
      framework extensively on several datasets to show its relevance, including two real-world medical
065
      applications for detection of cardiac arrhythmia or epileptic seizures. Our main contributions are
066
      summarized below:
067
068          • We formalize the notion of Local Relative Output Margins (LROMs) to quantitatively
069            compare neural network twins in relation to each other in an entire input region.
070          • We propose a sound framework to derive (provably-correct) verified bounds on Local
071            Relative Output Margins (LROMs) in an entire input region, to quantitatively compare the
072            decisions of neural network twins.
073          • We conduct extensive experiments to compare the decisions made by pre-trained classifiers
074            and their corresponding pruned, quantized, or knowledge-distilled counterparts on the
075            MNIST dataset (LeCun, 1998), CIFAR10 dataset (Krizhevsky, 2009), CHB-MIT Scalp EEG
076            database (Shoeb, 2010), and MIT-BIH Arrhythmia database (Goldberger et al., 2000).

      4.2.2       C OMPACT N ETWORKS
259
260   We explain the architecture and design of compact networks. Our experiments involve pruning,
261   quantization, and knowledge distillation. These techniques are used to derive compact DNNs enabling
262   energy-efficient inference on limited resources, and improving generalization and interoperability.
263
      Pruned Networks are derived through a pruning procedure applied to DNNs, selectively nullifying
264   certain weights and biases. Pruned networks maintain the architecture of their original counterparts.
265   For the MNIST and CIFAR10 datasets, we use pruned networks generated by (Ugare et al., 2022)
266   and (Baninajjar et al., 2024) through post-training pruning. Each pruned network generated by (Ugare
267   et al., 2022) eliminates the smallest weights/biases in each layer, which is called Magnitude-Based
268   Pruning (MBP), resulting in nine pruned networks with pruning rates ranging from 10% to 90%.
269

      (Baninajjar et al., 2024) produces Verification-friendly Neural Networks (VNNs) through the op-
271   timization of weights/biases, aiming to preserve their functionality while reducing the number of
272   non-zero weights and biases. For the CHB-MIT and MIT-BIH datasets, we employ MBP pruning
273   procedure where values below 10% of the maximum weight/bias are set to zero. In this context,

275   MIT-BIH datasets, respectively. Additionally, we utilize networks generated by (Baninajjar et al.,
276   2024) with accuracies of 82.5% ± 9.6% and 92.0% ± 9.1%.
277   Quantized Networks are obtained by quantization, where the precision of the networks’ weights is
278   reduced by converting them from 32-bit floating-point numbers to lower-precision representations.
279   Quantized networks have the same architecture as their corresponding original networks. The DNNs

285   Distilled Networks or student networks are compact networks trained using knowledge distillation
286   to transfer information from larger teacher networks to mimic their behavior (Hinton et al., 2015).

288   parameter affects the complexity of the distillation task, and we evaluate nine temperature values

290   CIFAR10 datasets, we produce them using the methodology outlined in (Hinton et al., 2015). We
291

293
      and 89.7% ± 11.2% for CHB-MIT and MIT-BIH datasets, respectively. Other accuracies can be
294   found in the appendix.

                                         (ci , cj ). Here, we use zero as a threshold as it corresponds to checking
307   increases or decreases of OM from one network to the other. However, our approach can easily
308   accommodate other thresholds. We simply state that “N1 has a verified LROM sample” if LROM
309   of N1 w.r.t. N2 is verified on the sample and N2 is clear from the context. Each time we verify
310   LROM of N1 w.r.t. N2 for a sample, then the corresponding OM of any (c, cj ), for correct class
311   c, are guaranteed to be larger in N1 than the OM in N2 . In addition, each time we show the upper
312
      bound of N1 w.r.t. N2 is negative (i.e., the lower bound of N2 w.r.t. N1 is positive) then the upper
      bound is indeed negative. In other words, if verified LROM of N1 w.r.t. N2 , then the OMs for the
313
      correct class in N2 are indeed smaller than those in N1 .

502   5    R ELATED W ORK
503
504   To investigate the impact of quantization on neural network, (Duncan et al., 2020) empirically show
505   that quantization not only maintains robustness but can also enhance it, and generally, accuracy is
506   preserved after the quantization process. Duncan defines robustness as the proportion of robust data
507   points in the original model that are also robust in the quantized model. In contrast, our method also
508   considers the margins with which the networks make their decisions, in addition to classification
509   results. The results in (Duncan et al., 2020) indicate a reduction in number of misclassified inputs and
510   a preservation of the robustness for given perturbations around them. We observe a decrease in the
511
      obtained margins. This suggests that models may remain robust, but they do so with reduced margins.
512   The studies by (LI et al., 2023) and (Jordao & Pedrini, 2021) show network pruning can empirically
513   improve robustness of a trained network. However, our research indicates this is not always the case.
514   (Wang et al., 2018) uses similar pruning methods as ours but applies two white box attacks including
515   fast gradient sign method (FGSM) (Goodfellow et al., 2014) and projected gradient descent (PGD)
516   (Madry et al., 2018). Their findings, which are consistent with ours, suggest setting small weights to
517
      zero can result in less robust networks. It appears that pruning solely for the purpose of reducing
      the number of parameters, without considering the overall accuracy of the network, can diminish
518
      its robustness. However, more deliberate pruning methods, such as stability-based pruning, might
519
      actually improve robustness.

522   6    C ONCLUSIONS
523
524   In this work, we propose a framework to compare the two networks in relation to each other over
525   an entire input region. Towards this, we introduce and quantify the Relative Output Margin (ROM)
526   with which decisions are made. A larger output margin for a network w.r.t. another indicates that this
527   network consistently makes a correct decision every time the other network does, and it does so in the
528   entire input region. In addition, our framework allows establishing lower and upper bounds on the
529   output margins, given an entire input region, to quantitatively compare neural networks twins. The
530
      proposed framework is relevant in the context of several application domains, e.g., for comparing
      a trained network and its corresponding compact (e.g., pruned, quantized, distilled) network. We
531
      evaluate our framework using the MNIST, CIFAR10, and two real-world medical datasets, to show
532
      its relevance.
533
534   Limitations: Built upon decades of progress in linear programming, our framework, while benefiting
535   from the impressive polynomial time complexity of linear programming, is constrained by the
536
      problem sizes manageable by the current state-of-the-art linear programming toolboxes.
537   Broader Impacts: This paper presents work whose goal is to compare two DNNs in relation to each
538   other. This is particularly important in the context of safety-critical applications, e.g., autonomous
539   driving or medical applications, as we show in this paper using two medical case studies. This work
      enables us to quantify the safety of the decisions made by two DNNs in relation to each other.

731   Solving the original minimization problem is not trivial. Indeed, the activation functions result in
732   nonlinear constraints for Equations (4) and (5). Our analysis targets ReLU layers, it can be generalized
733   to accommodate any nonlinear activation function that can be represented in a piece-wise linear form
734   (Ehlers, 2017). ReLU functions are the most widely used activation functions in DNNs. Recall the
735   last layer is a softmax layer, but we are only interested in the possible values of its inputs. We explain
736   in the following how to over-approximate the values computed at each layer using linear inequalities.
737
      The goal is to make possible the computation of a tight lower bound for the minimization problem
      from Section 3.1.
