009                                               A BSTRACT
010
011            Large language models (LLMs) require alignment—such as instruction-tuning or
012            reinforcement learning from human feedback—to effectively and safely follow
013            user instructions. This process necessitates training aligned versions for every
014            model size in each model family, resulting in significant computational overhead.
015            In this work, we propose nudging, a simple, plug-and-play, and training-free
016            algorithm that aligns any base model at inference time using a small aligned
017            model. Nudging is motivated by recent findings that alignment primarily alters
018
               the model’s behavior on a small subset of stylistic tokens, such as “Sure” or
               “Thank”. We find that base models are significantly more uncertain when gener-
019
               ating these tokens. Leveraging this observation, nudging employs a small aligned
020
               model to generate nudging tokens to steer the large base model’s output toward
021            desired directions when the base model’s uncertainty is high. We evaluate the
022            effectiveness of nudging across 3 model families and 13 tasks, covering reason-
023            ing, general knowledge, instruction following, and safety benchmarks. Without
024            any additional training, nudging a large base model with a 7×-14× smaller aligned
025            model achieves zero-shot performance comparable to, and sometimes surpassing,
026            that of large aligned models. For example, nudging OLMo-7b with OLMo-1b-
027            instruct—affecting less than 9% of tokens—achieves a 10% absolute improvement
028            on GSM8K over OLMo-7b-instruct. Unlike prior inference-time tuning methods,
029
               nudging enables off-the-shelf collaboration between model families. For instance,
               nudging Gemma-2-27b with Llama-2-7b-chat outperforms Llama-2-70b-chat on
030
               various tasks. Overall, this work introduces a simple yet powerful approach to
031
               token-level model collaboration, offering a modular solution to LLM alignment.

034   1   I NTRODUCTION
035
036   Large language models (LLMs) pre-trained on extensive text corpora acquire vast general knowl-
037   edge. Still, they often struggle to produce responses that are helpful, safe, and aligned with user
038   instructions without additional fine-tuning. As a result, alignment methods—such as instruction
039   tuning (Wei et al., 2022a) and reinforcement learning from human feedback (RLHF) (Ouyang et al.,
040   2022; Bai et al., 2022a)—have become essential for developing AI assistants like ChatGPT. How-
041   ever, the conventional training pipelines require separate alignment tuning for every model size
042   within each model family. This practice leads to substantial computational overhead, hindering the
043   rapid iteration and development of new model families for both the industry and the academia.
044   Recent studies (Zhou et al., 2024; Mitchell et al., 2023) argue that alignment primarily enhances
045   LLMs’ ability to generate helpful and well-formatted responses, while the foundational knowledge
046   and capabilities stem from pretraining. More concretely, Lin et al. (2023) analyzed the Llama-2
047   (Touvron et al., 2023b) models and found only a small subset of stylistic tokens, such as Sure, is
048   affected after alignment. These findings raise a natural question: If the alignment mainly affects the
049   base models superficially and at the token level, is it necessary to align base models via fine-tuning?
050   In this work, we show that we can inject a few alignment tokens at inference time using a small
051   aligned model to elicit various abilities of base models while ensuring helpfulness and safety. Specif-
052   ically, we introduce N UDGING—a simple, plug-and-play, training-free alignment algorithm that
053   aligns any base model at inference time. We find that base models are significantly more uncertain
      when generating alignment-related tokens, which we define as tokens where the pair of base and

097   dicating that alignment abilities can be achieved with a small model; (2) Scaling up the base model
098   leads to significant improvements, confirming that core abilities stem from the pretraining stage.
099   Our work opens up a new direction in token-level model collaboration, favoring the disentanglement
100   of abilities and offering a fresh perspective on alignment. By reducing the computational burden
101   associated with traditional alignment methods and enhancing model flexibility, nudging paves the
102   way for more efficient, modular, and adaptable AI systems.

209   3     N UDGING : INFERENCE - TIME ALIGNMENT AT THE TOKEN LEVEL
210
211   The analysis in the previous sections suggests that we can predict where a base model would disagree
212   with its aligned version based on the base model’s uncertainty, and at these positions aligned models
213   of different sizes tend to agree with each other. Based on these findings, we introduce N UDGING:
214   a simple training-free algorithm that aligns a base model’s output at inference time using nudging
215   tokens generated by an off-the-shelf small aligned model, which we call the nudging model. Given
      a base and nudging model and a fixed uncertainty threshold γ, N UDGING generates the output in a


                                                        4
      Under review as a conference paper at ICLR 2025


216
      token-level collaborative fashion. As illustrated in Figure 1, for any query q, we first let the base
217   model propose a short completion c = {c1 , c2 , ..., cL }. Then we find the first token position i where
218   the top-1 probability of the base model is less than γ: top-1 (Pbase (·|q, c<i )) < γ. We discard
219   the tokens after i and insert a token generated by the nudging model. Then we let the base model
220   propose again and find the next uncertain token. If all tokens from the base model’s completion have
221   top-1 probabilities larger than γ, we continue to generate another completion using the base model.
222
      We determine where to nudge based on the token probabilities, but for nudging tokens, we find
223   it beneficial to use spaces as boundaries and use the first “word” from the nudging model as the
224   nudging token. In the example shown in Figure 1, we accept “Sure,” instead of “Sure”, in the
225   first nudging round. This leads to better performance and facilitates better collaboration between
226   model families with different tokenizers. We use the nudging model to determine when to stop
227   generating. Specifically, instead of letting the nudging model generate only one token each time, we
228   let it generate a short completion. If the nudging model’s completion finishes with an [EOS] token,
229   we append the full completion from the nudging model to the current answer and stop generation
230   (see Figure 7 for an example). Otherwise, we accept the first word from the nudging completion.
231
      We depict a detailed implementation of N UDGING in Algorithm 1 in Appendix A.

480   6            R ELATED WORK
481
482   Analysis of alignment Many recent studies focus on understanding the nature of alignment. Zhou
483   et al. (2024); Chen et al. (2023) find that a small amount of carefully curated instruction-tuning
484   data is sufficient to teach base models to generate high-quality responses, posing the superficial
485   alignment hypothesis. Lin et al. (2023) provide a token-level view to understand alignment and find
      that only a small subset of stylistic tokens are affected after alignment. Another line of work shows


                                                                    9
      Under review as a conference paper at ICLR 2025


486
      that alignment might hurt certain capabilities of the base models. Wang et al. (2023) find that some
487   base LLMs become significantly worse on factual and reasoning benchmarks after supervised fine-
488   tuning. Ghosh et al. (2024) also show that supervised instruction tuning makes LLama-2 models
489   more likely to hallucinate. Mitchell et al. (2023) analyze the effect of scaling-up pre-training and
490   instruction tuning and find instruction tuning increases the helpfulness of the model while factual
491   knowledge comes from pre-training. Building on top of these findings, we proposed a modular and
492   token-level solution to alignment that favors the disentanglement of alignment and general abilities.
493
494   Inference-time tuning methods As the LLMs (Brown, 2020; Achiam et al., 2023; Touvron et al.,
495   2023b; Dubey et al., 2024) being increasingly large, fine-tuning them becomes prohibitively expen-
496   sive. Therefore, like N UDGING, many works explore using smaller models to adapt the large mod-
497
      els’ behavior at inference time without updating or accessing the model weights. Liu et al. (2024;
      2021); Mitchell et al. (2023) use the distributions of a pair of tuned and untuned small models to
498
      rescale the distribution of the large models. N UDGING offers a simpler, faster, and more flexible so-
499
      lution with better performance. Other works propose to train a small expert model (Ormazabal et al.,
500   2023) or adapter (Lu et al., 2023) to optimize large base models for specific tasks. For alignment
501   specifically, many works (Lin et al., 2023; Han, 2023; Ye et al., 2024) consider in-context learning
502   as a solution to inference-time alignment. However, using in-context examples shortens the usable
503   context length. More importantly, in-context examples can lead to various biases (Zhao et al., 2021;
504   Fei et al., 2023), and effective examples might be task or evaluation-specific. Finally, Shen et al.
505   (2024) explore a similar token-level model collaboration to our work. Compared with N UDGING,
506   their method needs task-specific training for all model pairs and is not specifically about alignment.
507
508   Token-level alignment of LLMs While most popular RLHF methods (Rafailov et al., 2024;
509   Ouyang et al., 2022; Bai et al., 2022b) optimize at a sample-level, token-level alignment meth-
510
      ods get increasing attention recently. Specifically, Mudgal et al. (2023); Zeng et al. (2024) designed
      token-level reward for RLHF to provide more detailed control of model responses. Deng & Raffel
511
      (2023) uses token-level reward at decoding time to adjust the model’s outputs. N UDGING shares the
512
      same motivation with these works in adapting large language models’ outputs from the token level.
513
514
515   7   D ISCUSSION AND C ONCLUSION
516
517   Limitations and future works (1) Currently where to nudge is determined solely based on the
518
      base model’s uncertainty, which assumes that the base model is well-calibrated. However, for many
      practical use cases, we may want to guide the base model’s behavior based on customized rules.
519
      In this case, designing new nudging rules that take the nudging model’s distribution into account is
520
      an interesting direction to explore. (2) N UDGING combines off-the-shelf models. Although this is
521   effective, the small aligned model is not explicitly trained to generate nudging tokens. Therefore,
522   learning a model that predicts where and what to nudge can potentially shrink the size of the nudg-
523   ing model while further improving the answer qualities, e.g., fluency and conciseness. (3) Nudging
524   builds on the motivation that alignment mainly impacts the base models at a few token positions. It
525   is interesting to study to what extent we can nudge a base model without additional training to fol-
526   low more challenging instructions such as ones that involve multiple sub-instructions or extracting
527   information from long contexts. We discuss more about this in Appendix E. (4) When deploying
528   N UDGING via an API, additional inference costs arise since generating a single response requires
529
      multiple calls to both the base and nudging models. However, if we have direct access to the base
      and nudging models, we can significantly optimize N UDGING by caching prefixes during genera-
530
      tion, similar to Speculative Decoding (Leviathan et al., 2023). In this case, the inference speed of
531
      N UDGING can approach that of the base model alone, as the nudging model is substantially smaller.
532
533   In this work, we introduced N UDGING, a simple yet powerful approach to align large language
534   models at inference time without the need for additional training. The simplicity and modularity of
535
      N UDGING present a promising alternative to traditional alignment methods, drastically reducing the
      computational cost of training while delivering significant performance gains across diverse tasks.
536
      By enabling models to collaborate at the token level, N UDGING harnesses the strengths of different
537
      models and effectively disentangles their capabilities. Our work provides a fresh perspective on
538   aligning large language models and offers a promising direction for designing modular AI systems.

      E THICS S TATEMENT
541
542   In this work, we studied the alignment of large language models, which is essential for ensuring that
543   AI systems behave in safe and beneficial ways. Our proposed method, N UDGING, offers a training-
544   free approach to improve any base model’s helpfulness while maintaining safety, thereby making
545   alignment more accessible and reducing computational overhead. Our analysis also provides deeper
546   insights into the behaviors of aligned models, facilitating a better understanding of alignment mech-
547   anisms and promoting future research in this area. Potential harms of using N UDGING instead of an
548   aligned model, such as undesirable discrimination due to implicit bias against certain populations,
549   need further exploration. Additionally, as a novel way to change models’ behavior at inference
550
      time, we recognize that N UDGING could be used adversarially to align a large base model with a
      small model trained to produce harmful or unsafe content. We encourage the research community
551
      to explore this safety concern further and develop safeguards to prevent the potential malicious use
552
      of LLMs. We acknowledge that our study focuses exclusively on English datasets. We encourage
553   future research to explore how N UDGING can assist in aligning models in other languages. Our
554   work does not involve human subjects or releasing new data sets, but we have adhered to ethical
555   standards throughout our research and believe that our contributions will have a positive impact on
556   the development of ethical AI technologies.

      A     M ORE IMPLEMENTATION DETAILS
757
758
      In this section, we provide more implementation details about N UDGING and the two baselines we
759   compare N UDGING with, the average ensemble and proxy tuning. We implement all methods based
760   on vllm (Kwon et al., 2023) with A6000 GPUs.
761
762   A.1    N UDGING
763
764   We depict a detailed implementation of N UDGING in Algorithm 1. For our implementation, we set
765   the completion length L to be 16 as it balances the computational cost and gives the nudging model
766   better control of when to stop generating. We set the max nudging round R = 100 and the max token
767   number T = 512. When passing the query prompt and the current answer to the nudging models, we
768   adapt them using the instruction templates of the corresponding model families accordingly. Finally,
769
      we use a simple heuristic for repetition control: When the base model’s completion appears in the
      current answer, we end the round and pass to the nudging model. If the nudging words for three
770
      consecutive rounds are the same, we terminate generation.
771
772
      Using words instead of tokens For finding nudging tokens, we use spaces, i.e., “ ”, to split the
773
      nudging completion and use the first word as the nudging tokens. We find that using the first word,
774   rather than the first token, leads to better performance. We hypothesize that this is because words, as
775   the basic semantic units of language, provide more meaningful guidance for steering base models,
776   whereas sub-word level tokens may sometimes lack the semantic coherence needed for effective
777   nudging. For example, for LLama-2 models on GSM8K, the nudging model mostly starts the answer
778   with “Sure”, and the base model would complete the word with “ly”, ending up with “Surely”, which
779   usually leads to worse answers. Also, using full words as nudging tokens makes the collaboration
780   of different model families easier when they have different tokenizations of words.

      A.2    BASELINES
805
806   Average ensemble We choose the average ensemble as a baseline as it is one of the simplest
807   ways to combine two different models. We average the top-5 token distributions of the base and
808   nudging models at each token position before sampling. To put the baseline in a similar condition
809   with N UDGING, we assume that we only have access to the top-5 top log probs from the models,
      which is the maximum number of top log probs for most API service providers like Fireworks AI.


                                                           15
      Under review as a conference paper at ICLR 2025


810
      At inference time, for each token position, we retrieve the top-5 token probabilities from both the
811   base and the nudging model and then average the probability of each token. If a token appears only
812   in the top-5 tokens of one model, its probability is halved. This ensemble operation is applied to
813   each token position, meaning the number of calls made to both the base and the nudging model
814   corresponds to the number of answer tokens.
815
816   Proxy tuning Proxy tuning works by rescaling the large base model’s distribution by contrasting
817   the distribution of a pair of small models. Ideally, it requires the full distribution from all models
818   to work. This requirement cannot be satisfied for API-based implementation, which is the base for
819   most practical applications. Following (Liu et al., 2021), we use the top-100 probabilities from the
820   models due to the limited computational resources, and following their implementation we only
821   focus on tokens that appear in the top 100 tokens of all models. When the top log probs number is
822   small, e.g., 5, the top tokens from all three models might not intersect at all, making proxy tuning
823
      not feasible for most API service providers.

       C.2    A NALYSIS OF THE DISCARDED BASE TOKEN RATIOS .
986
       The ratio of the number of tokens generated by the base models that are discarded is another im-
987    portant aspect of efficiency. In the following analysis, we focus on the discarded token ratio of the
988    base model, as the nudging model is much smaller and has a minor effect on the inference speed. In
989    Figure 11, we reported the nudging token ratios, and here we show that it is strongly connected to
990    the discarded base token ratio, which is defined as the number of discarded base model tokens due
991    to the nudging model divided by the total number of tokens generated by the base model. Assuming
992    in a nudging answer there are N nudging tokens, B base tokens, and T = N + B total tokens. The
993    nudging token ratio is therefore defined as RN = N   T . In each nudging round, the nudging model
994    generates 1 nudging token and then the base model continues by generating L completion tokens
995
       each time. As a result, there can be at most L base tokens discarded in each round. So an upper
       bound of the discarded token ratio RD can be derived as
996

998                                       B+N ×L  RN + L − 1
