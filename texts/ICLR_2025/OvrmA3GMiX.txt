009                                                A BSTRACT
010
011            Transfer is a key promise of hierarchical reinforcement learning, but requires first
012            learning transferable skills. For an agent to effectively transfer a skill it must
013            identify features that generalize and define the skill over this subset. However,
014            this task is under-specified from a single context as the agent has no prior knowl-
015            edge of what future tasks may be introduced. Since successful transfer requires
016            a skill to reliably achieve a sub-goal from different states, we focus our attention
017            on ensuring sub-goals are represented in a transferable way. For each sub-goal,
018
               we train an ensemble of classifiers while explicitly incentivizing them to use min-
               imally overlapping features. Each ensemble member represents a unique hypoth-
019
               esis about the transferable features of a sub-goal that the agent can use to learn a
020
               skill in previously unseen portions of the environment. Environment reward then
021            determines which hypothesis is most transferable for the given task, based on the
022            intuition that useful sub-goals lead to better reward maximization. We apply these
023            reusable sub-goals to M INI G RID and M ONTEZUMA’ S R EVENGE, allowing us to
024            learn previously defined skills in unseen parts of the state-space.

027   1   I NTRODUCTION
028
029   Hierarchical reinforcement learning (HRL) (Barto & Mahadevan, 2003) is a promising approach for
030   scaling RL to challenging, long-horizon problems. HRL methods learn temporally extended skills
031   that abstract away the details of low-level action executions. The most popular HRL framework, the
032   options framework (Sutton et al., 1999), models skills using three components: a set of states from
033
      which execution can begin, a policy which picks low-level actions, and a collection of states where
      execution ceases, which can be represented by a sub-goal.
034
035   To fully realize the benefits of HRL, learned options should be transferable. For example, a robot
036   trained to open a door in a factory should be able to open doors in a user’s home with minimal addi-
037   tional training. However, it is challenging to generalize an option from a single context to another.
038   This is primarily because all three components of the option are conditioned on the entire state—
039
      which includes spurious features unnecessary for successful execution—making generalization from
      one example challenging because it is severely under-specified. However, if these components are
040
      defined over generalizable features of the state, the option can transfer to novel contexts from a
041
      single instance. The core problem here is that, in practice, an option will first be discovered in a
042   single context (like a factory) without foreknowledge of the circumstances under which it can be ap-
043   plied again in the future. When defined in a single context, a sub-goal will have many confounding
044   factors, which cannot be untangled. We further argue this point with a toy example in Section 3.2.
045
      How can the agent learn a state representation that will result in seamless transfer? This problem
046
      can be easily tackled in supervised learning by incorporating data from different contexts or by ap-
047
      plying class-preserving data augmentations. However, in supervised learning, the important features
048   have been selected prior to learning, through the dataset. If we define the class dog, a human has
049   previously chosen what features represent a dog and selected a dataset accordingly. If we wish to
050   transfer from a single context, this information is missing, resulting in a set of data that could be de-
051   fined accurately in multiple ways. If we have a sub-goal that results in us opening a cupboard which
052   contains a red ball, is the sub-goal defined by opening the cupboard or finding the red ball? From
053   this one instance, we do not have enough information to disentangle these two features. Instead,
      we propose that the RL agent maintain several, diverse hypotheses over which features of the state

      Under review as a conference paper at ICLR 2025


054
      might generalize to upcoming tasks, learning a set of classifiers with minimal confounding features.
055   The agent then selects from among these hypotheses, testing them in the environment, and updates
056   its beliefs over which members of the ensemble have learned transferable features for a given task.
057
058
      We focus on transferring sub-goals, since the agent can learn an option given a sub-goal (as we
      argue in Section 3). We begin by assuming access to a sub-goal (potentially discovered using an off-
059
      the-shelf option discovery method), defined for a single context. This sub-goal is used to generate
060
      labeled data which acts as a training set for an ensemble of classifiers. Each ensemble member
061   maintains a unique hypothesis of the generalizing state-features and is, individually, a new sub-goal;
062   then, for each hypothesis, we learn a low-level policy that aims to reach the hypothesized sub-goal.
063   Finally, a high-level policy selects among the low-level policies to maximize extrinsic reward.
064
      We test our approach in two simulated environments: M ONTEZUMAS R EVENGE (Bellemare et al.,
065
      2013; Machado et al., 2018) and M INI G RID (Chevalier-Boisvert et al., 2023). Given a set of options
066
      and their execution data from the first room of M ONTEZUMAS R EVENGE, we show that our algo-
067   rithm can successfully transfer those options to other, visually dissimilar rooms. In M INI G RID, we
068   demonstrate how our agent learns transferable options and uses them to rapidly solve a challenging
069   exploration problem.

072   2   BACKGROUND AND R ELATED W ORK
073
074   We consider Markov Decision Processes (MDP) M = (S, A, r, p, γ) where S denotes the state-
075   space, A is the action-space, p(st+1 |st , at ) denotes the transition dynamics and r(st , at ) is the
076   scalar reward function. The RL objective is to find a policy π(a|s) that maximizes the cumulative
077   discounted reward over the lifetime of the agent.
078
      An option o is a temporally extended action and, for this work, is defined by a tuple (Io , πo , βo ).
079   The initiation set, Io : S → {0, 1}, is the set of states in which option o can initiate. The termination
080   set, βo : S → {0, 1} is the set of states in which option o successfully terminates. The option policy
081   πo : S → A is a controller that transitions the agent from states in Io to states in βo .
082
      Identifying Sub-goals. For this definition, the option termination set is analogous to a sub-goal.
083
      Identifying sub-goals is an important aspect of hierarchical reinforcement learning. One popular
084
      method by Andrychowicz et al. (2017) is a simple but powerful technique of generating goals from
085   previously visited states and has lead to several branch-off works (Pitis et al., 2020; Fang et al.,
086   2019). Nair et al. (2020) learn a latent dynamics model to generate sub-goals and Florensa et al.
087   (2018) focus on selecting solvable sub-goals. Bagaria et al. (2023) generate a protogoal space which
088   consists of many possibly useful sub-goals and an adaptive function to map this space to a small set
089   of useful goals. All these methods focus on the initial discovery of sub-goals while our work focuses
090   on transferring an existing sub-goal and so can be used in tandem with any of these methods.
091   Learning transferable skills. A task is defined by a goal an agent must achieve in the environment.
092   For example, a task can be navigating to a location or obtaining an object. Sub-goals can exist along
093   the trajectory towards the goal, which are intermediate goals that must be first achieved before a final
094   goal can be completed, or a set of unrelated goals the agent is tasked with completing. An option can
095   be used to achieve these sub-goals, by setting the termination set to be a sub-goal. The use of options
096   promises to aid in task learning as we only need to learn to reach each sub-goal once but may need
097   to reach this sub-goal many times in future. However, in their most basic formulation, options do
098   not generalize. This is because an option is defined over a specific subset of the state space and has
099
      no prior knowledge about what features are needed for successful transfer. One approach to transfer
      is via derived input spaces with transferable semantics. Konidaris & Barto (2007) showed that an
100
      agent-centric representation, analogous to an egocentric space, would be sufficient for option gener-
101
      alization but requires a hand-designed agent-centric input. Dayan (1993) and Barreto et al. (2018)
102   use successor features to learn policies that generalize across reward functions but not changes in
103   transition dynamics. Touati et al. (2022) expand on the successor features framework to learn rapidly
104   generalizing policies under changing reward dynamics. Other attempts at skill transfer learn from
105   demonstration (Konidaris et al., 2011; Ranchod et al., 2015), using demonstrations from several
106   tasks to identify and learn common skills. Wang et al. (2014) require first learning a large number of
107   skills and retroactively compressing these skills into a single transferable skill and sub-goal. Barreto
      et al. (2019) combine previously learned options into new options, called the option keyboard, which

                                                         2
      Under review as a conference paper at ICLR 2025


108
      allows for combining previously learned, possibly irrelevant, skills into something more beneficial.
109   A common approach to skill generalization Agarwal et al. (2021); Frans et al. (2017); Gomez et al.
110   (2022) assumes access to a distribution of relevant tasks that can be sampled from during training.
111   These methods leverage this distribution to either learn useful state representations or general poli-
112   cies and sub-goals, however, carry an implicit assumption that an expert has predefined which tasks
113   should be generalized to. Many of these works require either a large library of skills or assume a
114   distribution of relevant tasks. As such, these methods either delay the gain of generalized skills or
115   assume the skill has already been predefined and so are not compatible with existing skill discovery
116   methods. These methods are retroactive, looking backwards on what has been learned or provided.
117
      We wish to generalize forward—transferring a sub-goal from a single option instance.
118   Unsupervised Representation Learning. Representation learning is often used to compress high
119   dimensional data to aid future learning (Bengio et al., 2013). One method to learn a useful represen-
120   tation is leveraging auxiliary tasks such as Denton et al. (2017) who use the downstream tasks such
121   as classification and video prediction to learn a compact representation. Higgins et al. (2017) learn a
122   disentangled representation of the state space to improve an agents domain adaptation. Representa-
123
      tion learning has also been used to represent goals to improve exploration (Laversanne-Finot et al.,
      2018) and sample efficiency (Nair et al., 2018). Our work differs from these by learning multiple
124
      hypotheses, each of which is a unique state representation.
125
126   The Diversity-By-disAgreement Training (D-BAT) algorithm from Pagliardini et al. (2022) is
127   designed to learn a classifier able to transfer to out-of-distribution data. This is done by making use
128   of both labeled and unlabeled data in a semi-supervised setting using the loss function defined in

      3   L EARNING T RANSFERABLE S UB - GOALS
143
144   Hierarchical reinforcement learning simplifies complex problems by splitting a single task into mul-
145   tiple sub-tasks, each of which can be defined by a sub-goal which can be provided to the agent or
146   discovered through a skill discovery algorithm. These sub-goals are used to learn a policy to com-
147
      plete the corresponding sub-task, giving rise to a skill. These sub-goals can be represented in many
      ways, for example a sub-goal to reach the door of a room can be defined over the agent’s (x, y) lo-
148
      cation or the agent’s distance from the door. While both feature sets can be used to effectively learn
149
      an option in the discovering context, the agent’s position sub-goal will not generalize to new states.
150   Many previous works tackle skill transfer retroactively, learning a large collection of skills that are
151   then compressed into a small collection of reusable skills or assuming a predefined distribution of
152   relevant tasks. These skills are either found after countless interactions with the environment—
153   delaying access to a transferable skill for a long time and still need to answer the question of when
154   to group a collection of skills—or require an expert to define a set of tasks. Instead, we propose gen-
155   eralizing forwards, making these skills available to the agent after their initial discovery and allows
156   us to leverage the rich collection of existing skill discovery algorithms.
157   Sub-goals are commonly modeled as classifiers. The classifier returns 1 for sub-goal states and 0
158   otherwise; when executing a skill the agent runs a policy until reaching a sub-goal state, where it
159   receives a reward. This is the most important component of the skill and the output of most skill
160   acquisition algorithms. With only a transferable skill policy, it is not clear when this policy has
161   completed. A transferable initiation set only tells us some policy can be started to reach some sub-
      goal but this information alone has no value. However, if the agent has access to a transferable

      termination set, the agent can learn a policy and initiation set that achieves the same sub-goal in a
163   new context.

      3.1   H YPOTHESIS G ENERATION
166
167   Given state space S, we sample a dataset D which defines a space of functions F, consisting of all
168   functions that accurately fit all points in D. This space of functions is very large as there are many
169   ways to define a function that accurately fits some set of points. A function’s ability to generalize
170   is measured by how accurately the function can predict data points that have not been included in
171   D. When learning a classifier to approximate one of these functions, the burden of generalization is
172   placed on the quality of data collection; which must fully encompass the tasks you wish to generalize
173   to in S. If the collected data does not accurately represent the full distribution of relevant contexts,
174
      the classifier cannot generalize reliably. Unfortunately, this is precisely the scenario RL agents find
      themselves in when generalizing forwards.
175
176   We therefore propose that, instead of sampling a single function from F, we can instead sample
177   multiple functions, which form a collection of hypotheses, H. Each function, h ∈ H, describes
178   different features of S which accurately describe D. By creating hypotheses over D we increase
179   the chance of learning generalizable features. For example consider an agent learning to open a
180
      cupboard door and all training samples have been collected on the same cupboard which contains
      a red ball. One hypothesis attends to the red ball inside the cupboard—which is only visible if the
181
      door is open—while another learns to distinguish the location of the cupboard door when it is open
182
      or closed. Both of these hypotheses accurately describe the training data D but make use of different
183   sets of features. To further improve the quality of H, we can require that our sampling process
184   focuses on diverse functions, ensuring each function h differs meaningfully on the features of our
185   data D.
186
      We leverage the D-BAT algorithm to ensure diversity among h ∈ H, however, any algorithm that
187
      would result in a collection of diverse functions can be used instead. This algorithm makes use of
188
      labeled data D but also requires a set of unlabeled data U ∈ S to train an ensemble of classifiers,
189   taking the environment state as input and outputs 1 if the state is a sub-goal or 0 otherwise. Each
190   ensemble member is encouraged to reduce labeled loss while also decreasing agreement on U. As a
191   result, we learn an ensemble which is our collection of hypotheses H, with each ensemble member
192   attending to different sets of features in S, informed by U. Given some discovered sub-goal, we
193   collect labeled data which—due to the nature of discovered sub-goals—is collected from a small
194   subset of states in the state-space. We can leverage the exploration required during policy learning
195   to collect an unlabeled dataset which contains states in which the sub-goal is not defined.

225   3.3   H YPOTHESIS S ELECTION
226
227
      Once we have learned H, we must select the best hypothesis. This task however is challenging
      and cannot be done without additional information. Returning to the example in Figure 1, can you
228
      determine if the third state satisfies the original sub-goal? The answer is it depends; the red key
229
      is present, so our red key sub-goal is not satisfied. Similarly the yellow key has been collected
230   so our yellow key sub-goal is also not satisfied. There are, however, two keys in the environment
231   so our two key sub-goal has been met. Which hypothesis is correct? If we were an agent in the
232   environment, how do we decide which hypothesis is most beneficial? It is very likely that only one
233   of these hypotheses is transferable given our task, however, without knowledge of what that task is,
234   we cannot confidently select the most general hypothesis. But in an RL problem, the task is encoded
235   in the environment reward. Therefore, we propose that the most transferable hypothesis will lead to
236   higher external reward. We provide a high-level policy with an action space that contains options
237   defined by all h ∈ H, providing one option for each hypothesis. This high-level policy then learns
238
      to maximize the extrinsic reward, which encodes the task. As this high-level policy begins reward
      maximization, it will naturally begin selecting hypotheses that best support transfer.

      ronment places the agent in one of two rooms separated by a locked door. The agent must collect
      the key to unlock the door and travel to the goal location in the other room. Traditionally, this envi-
412
      ronment has a red door and a single red key. We modify this task to include a blue and green key to
413
      increase the difficulty of the task. The agent must collect the correct key—the red key—to unlock
414   the door and reach the goal to receive a reward of 1 or 0 for every other time step, making this a
415   sparse reward problem. We use the full RGB image as our state.
416
417
      We define 5 sub-goals; C OLLECT B LUE K EY, C OLLECT G REEN K EY, C OLLECT R ED K EY,
      O PEN R ED D OOR and G OT O G OAL. Labeled data is collected from seed 0 and unlabeled data is
418
      collected from seeds 1 and 2. Data from test seeds is not shared among agents. The lower-level
419
      ensemble policies are not pre-trained, and must be learned in conjunction with the higher-level pol-
420   icy. We use a PPO (Schulman et al., 2017) agent with access to only the option policies, each of
421   which is a Deep-Q Network (Van Hasselt et al., 2016) with a prioritized replay buffer (Schaul et al.,
422   2015). Each ensemble member is an executable skill, with an independent low-level policy. The
423   PPO agent has an action space of size 15, consisting of 3 hypotheses for all 5 original sub-goals. We
424   aim to show that giving the agent skills that are not aligned with the overall task will not irreparably
425   damage policy learning, and the strong performance of our ensemble agent cannot be explained by
426   our use of DQN and PPO. As such we evaluate against a DQN and a PPO agent, each of which have
427   access to the original environment actions. All agents are run for 1.5 million environment steps with
428
      a maximum episode length of 500 steps.
429   Figure 6 shows the option agent is able to learn to reach the goal consistently, while the DQN and
430   PPO agents fail to complete the task. We can conclude that hypothesizing helps the agent complete
431   the task goal by allowing the agent to leverage previously defined useful sub-goals, simplifying the
      original task.

456   4.4   H YPOTHESIS S ELECTION THROUGH R EWARD O PTIMIZATION
457
458   Our previous experiments have shown that we can learn at least one generalizing hypothesis but has
459   not yet shown how to identify the corresponding ensemble member. We previously claimed that
460   reward maximization is a useful proxy for identifying the hypothesis that best transfers. To evalu-
461   ate this claim, we compare the termination sets of the most and least selected ensemble members.
462
      Overall, we find that the most popular hypothesis closely resembles our hand-defined generalized
      sub-goals. Figure 7 overlays multiple termination sets for two members of the O PEN R ED D OOR
463
      sub-goal. For the least selected ensemble member, the sub-goal does not fully correlate with our
464
      definition of the O PEN R ED D OOR sub-goal; i.e. positioned in front of an open door. On the other
465   hand, Figure 7a shows the most picked ensemble member which requires the agent always stands at
466   the open door, matching our hand-defined O PEN R ED D OOR sub-goal.
467
468
      For the G OT O G OAL sub-goal, shown in Figure 8, we show two ensemble members, neither of which
      seems to be favored. Both of these sub-goals require the agent to be standing on the goal position
469
      which again aligns with our previously defined sub-goal. The final member of this ensemble, which
470
      has a significantly lower selection probability, does not trigger for this D OOR K EY environment seed.
471   This supports our statement that reward maximization leads to the selection of the most generalized
472   sub-goals. We can conclude that, not only does diversity allow for successful sub-goal generalization
473   for a given option but also that reward maximization is a good measure for identifying the most
474   transferable hypothesis.

      5     C ONCLUSION
477
478
      In this work we tackle sub-goal generalization from a single instance; the most crucial step for
479
      forward skill generalization. We do this by learning an ensemble of diverse hypotheses over gen-
480   eralizing feature sets using the D-BAT algorithm. Our experiments show that this ensemble can
481   successfully transfer the C LIMB D OWN L ADDER sub-goal in M ONTEZUMAS R EVENGE with 70%
482   accuracy when exposed to a single set of ladders in the game. These classifiers can be used to
483   learn options that takes the agent to the bottom of ladders throughout M ONTEZUMAS R EVENGE.
484   We also show that these sub-goal classifiers can be used for learning a modified D OOR K EY task
485   in M INI G RID and reward maximization can be used to identify the hypothesis that best supports
      transfer.
