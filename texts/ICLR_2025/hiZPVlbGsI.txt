009                                               A BSTRACT
010
011            Generally, it can be challenging to recognize the table contents from scanned PDF
012            documents containing some red stamps and reconcile the recognized table con-
013            tents. In this paper, we address the reconciliation challenge involving matching
014            the handwritten invoice amounts overlapped with a red stamp against the invoice
015            amounts found in multi-page tables extracted from scanned PDF documents and
016            we propose a context-splitting Transformer-based table recognition method for
017            recognizing the handwritten invoice amounts overlapped with a red stamp against
018
               the invoice amounts. Firstly, we recognize the layout structure of the table which
               is detected from the scanned PDF document containing some red stamps on the
019
               handwritten invoice amounts of the table. Secondly, we represent the table cells
020
               as context-splitting embedding vectors which involve spatial context embedding,
021            position context embedding, lexical context embedding, and colored context em-
022            bedding. Finally, we apply a stack of Transformer-based self-attention encoders
023            to recognize the cross-modality table cells where we multiply the length of query
024            vector and the length of key vector with the scaling factor of the original Trans-
025            former in order to make the training process more stable. We improve the recog-
026            nition accuracy of table cells with a red stamp on handwritten invoice amounts.

029   1   I NTRODUCTION
030
031   Tables are a common form in Portable Document Format (PDF) documents. Especially, some tables
032   in scanned PDF documents contain some handwritten signatures and some red stamps on them. We
033   present an issue in scanned PDF recognition, namely red stamp occlusions and the recognition of
034
      handwritten text under the red stamp. However, it can be challenging to recognize such table con-
      tents as handwritten invoice amounts from scanned PDF documents containing some red stamps and
035
      reconcile the recognized table contents. For example, PP-StructureV2 Li et al. (2022a) which con-
036
      tains a subsystem ”Layout Information Extraction” and subsystem ”Key Information Extraction” can
037   detect the table structures and recognize the table cells from the scanned PDF documents. However,
038   the effectiveness will be greatly reduced when a PDF document is obtained by scanning the printed
039   document and the tables in this PDF document involve some handwritten invoice amounts and some
040   red stamps on these handwritten invoice amounts. Figure 1 shows an example of the results on table
041   structure detection and table cell recognition from the scanned PDF documents which involve some
042   handwritten invoice amounts and some red stamps on them by importing PPStructure and applying
043   it. In Figure 1, PPStructure divides the document image of left side into predefined areas such as
044   text, title, table, and figure based on an ultra lightweight detector PP-PicoDet. For example, the table
045
      symbol on the upper left corner in Figure 1 represents a table area identified by PP-PicoDet detector.
      However, the table cell recognition performance need to be improved in the invoice amount column
046
      containing a red stamp on a handwritten invoice amount, shown as the red arrow in Figure 1. There

      are at least three errors on the recognition results of table elements under the red stamp in Figure

      2.1   TABLE DETECTION FROM S CANNED PDF DOCUMENTS
090
      Table detection from scanned PDF documents is a necessary step for localizing the bounding boxes
091   of tables in PDF documents Zhong et al. (2020). TableSense Dong et al. (2019) is a spreadsheet Ta-
092   ble Detection framework with Convolutional Neural Network (CNN). Multi-Type-TD-TSR Fischer
093   et al. (2021) uses a fully data-driven approach based on a CNN. Graph Neural Network (GNN) is
094   used to describe the local repetitive structural information of tables in invoice documents Riba et al.
095   (2021). PSENet can be used to detect each text line in the table image Ye et al. (2021). CatBoost
096   Prokhorenkova et al. (2018) is the SOTA Tree-based boosting model on tabular tasks. TableFormer
097   Nassar et al. (2022) provides bounding boxes by using Cell BBox Detector which can be improved
098   with post-processing during inference. However, both PSENet and TableFormer are not very ideal
099
      to detect curve texts such as red stamps or stamps. Differentiable Binarization Liao et al. (2020)
      Liao et al. (2023) can perform the binarization process in a segmentation network which can more
100
      accurately describe scene text of various shapes such as curve text. PP-OCR Du et al. (2020) uses
101
      Differentiable Binarization as text detector which is based on a simple segmentation network. The
102   simple post-processing of Differentiable Binarization is efficient. PP-OCRv2 Du et al. (2021) uses
103   Collaborative Mutual Learning (CML) as text detector to solve the problem of text detection distil-
104   lation. In CML method, text detector uses ResNet18 as teacher model and MobileNetV3 as student
105   model. PP-OCRv3 Li et al. (2022b) still uses CML as text detector which adopts a Large Kernel
106   PAN (LK-PAN) module for a teacher model and a RSE-FPN module with residual attention mecha-
107   nism for a student model. PP-StructureV2 Li et al. (2022a) introduces a lightweight layout detector
      PP-PicoDet to divide document images into predefined areas such as text, title, table, and figure.

      2.2   TABLE RECOGNITION OF S CANNED PDF DOCUMENTS
109
110   Table recognition of scanned PDF documents is an important step for parsing only the structural
111   information on row and column layout of tables Zhong et al. (2020) and extracting the logi-
112   cal and physical structure of unstructured table images into a machine-readable format Lin et al.
113   (2022)Huang et al. (2023). Table Structure Recognition (TSR) involves bordered TSR, Unbordered
114   TSR, and Partially Bordered TSR Fischer et al. (2021). Cycle-CenterNet Long et al. (2023) on the
115   top of CenterNet detects and groups tabular cells into structured tables. A bi-directional Recurrent
116   Neural Network (RNN) with Gated Recurrent Units (GRU) Khan et al. (2020) is used to extract
117
      rows and columns from a detected table in document images. Cycle-CenterNet Long et al. (2021)
      detects and groups tabular cells into structured tables. TGRNet Xue et al. (2021) reformulates the
118
      problem of table structure recognition as the table graph reconstruction. Donut Kim et al. (2022) is
119
      an OCR-free document Understanding Transformer which is robust to the handwritten documents.
120   TabTransformer Huang et al. (2020) architecture comprises a column embedding layer to learn effi-
121   cient contextual of table, a stack of N Transformer layers, and a Multi-layer Perceptron (MLP). FT-
122   Transformer Gorishniy et al. (2021) transforms all categorical and numerical features of tabular data
123   to embeddings by Feature Tokenizer and applies a stack of Transformer layers to the embeddings.
124   MATE Eisenschlos et al. (2021) presents a sparse-attention Transformer architecture for modeling
125   documents that contain large tables. TRUST guo et al. (2022) is an end-to-end Transformer-based
126   table structure recognition method with multi-oriented table row/column splitting step and table grid
127   merging step. DETR Carion et al. (2020) employs a Transformer Encoder and Decoder that looks for
128
      a specific number of object queries. TableFormer Nassar et al. (2022) replaces the LSTM decoders
      with Transformer-based decoders which can obtain the content of the table-cells from PDF source
129
      with some non-English tables by using bounding boxes. TSRFormer Lin et al. (2022) contains a
130
      two-stage DETR-based split module to predict all row/column separation lines from each input ta-
131   ble image and a relation network based cell merging module to recover spanning cells. VAST Huang
132   et al. (2023) contains a CNN image encoder, a HTML sequence decoder which is a Transformer with
133   a stack of N identical layers, a coordinate sequence decoder triggered by the representation of the
134   non-empty cell from HTML sequence decoder where the logical structure of the table is represented
135   in HTML format. XTab Zhu et al. (2023) handles the cross-table pre-training of tabular Trans-
136   formers on datasets from various domains by separating the models into data-specific and shared
137   components to learn general knowledge for tabular prediction. LayoutLMv2 Xu et al. (2021) is
138   a multi-modal pre-training method for visually-rich document understanding. PP-StructureV2 Li
139
      et al. (2022a) use a visual-feature independent LayoutXLM architecture to extract key information.

141   3     M ETHODOLOGY
142
143
      In order to correctly recognize the handwritten invoice amounts under a red stamp in the column
144
      ”invoice amount” (in Chinese) of the image-based table in Figure 1, we propose a color-wise context-
145   splitting Transformer-based method for analyzing the table structures and recognizing the table cells
146   with red stamp occlusions and handwritten invoice amount. Figure 2 shows an architecture of our
147   proposed color-wise context-splitting Transformer-based method for recognizing the table cells with
148   red stamp occlusions and handwritten invoice amount. In Figure 2, our proposed color-wise context-
149   splitting Transformer-based method includes at least six parts for analyzing the table structures and
150   recognizing the table cells with red stamp occlusions and handwritten invoice amount. The first part
151   is an input layer which inputs the table image containing a red stamp in the table detected from a
152   multi-pages scanned PDF document. The second part is a table image standardization layer where
153
      the direction of input table image is corrected through a table alignment pre-processing step. The
      third part is a table layout structure analysis layer where the physical structure of a table image refers
154
      to the bounding box coordinates of all non-empty table cells and the logical structure of this table is
155
      its layout structure sequence. The forth part is an embedding layer where we represent the spatial
156   contexts, positional contexts, lexical (tokenized) contexts, and colored (tinctorial) contexts of the
157   table cells in a layout structure sequence as low dimensional embedding vectors. The fifth part is
158   a stack of Transformer-based layers where we modify the self-attention mechanism of the original
159   Transformer Vaswani et al. (2017) by multiplying the length of query vector and the length of key
160   vector with the scaling factor of the original Transformer in order to make the training process more
161   stable. The sixth part is an output layer which outputs the probabilities of predicted table cells in a
      table layout structure sequence.

      4.1   DATASETS
331
332   Unlike other machine learning sub-fields such as computer vision or natural language processing
333   (NLP), there are no standard benchmarks for tabular data Grinsztajn et al. (2022). A large-scale
334   TabRecSet Yang et al. (2023) dataset for end-to-end table recognition in the wild is introduced and
335   compared between the TabRecSet dataset and the existing datasets such as the PubTabNet Zhong
336   et al. (2020) dataset, the TableX Desai et al. (2021) dataset, and the PubTables-1M Smock et al.
337   (2022) dataset. The PubTabNet dataset is a public dataset which contains 568k table images with
338   corresponding structured HTML representation and includes 548,592 training samples and 9,115
339
      validation samples. The FinTabNet dataset is a large dataset containing complex tables from the
      annual reports of the S&P 500 companies Zheng et al. (2020). Financial tables often have very dif-
340
      ferent styles with fewer graphical lines and larger gaps within each table and more colour variations.
341
      We conduct experiments on the PubTabNet and FinTabNet dataset with generated ground-truth for
342   reproducibility to verify the effectiveness of our proposed color-wise context-splitting Transformer-
343   based method for recognizing the table cells containing the red stamps from scanned PDF docu-
344   ments. In the PubTabNet and FinTabNet dataset, the ground truth data is derived from the presence
345   of nearby vertical graphical lines (as detected by a PDF parser) for each cell Zheng et al. (2020).
346   In addition, we collect some real dataset from our project in order to train our proposed color-
347   wise context-splitting Transformer-based table recognition model and compare the results with eight
348   baselines. Our collected real dataset is divided into two parts. The first part is some scanned PDF
349   documents which are obtained by scanning printed documents. In the PDF documents of the fist
350
      part, the tables contain some red stamps on them. The second part is some born-digital PDF docu-
      ments which are obtained by directly saving from Word documents. In the PDF documents of the
351
      second part, the tables don’t contain any red stamp on them.

355
      TableMaster Ye et al. (2021) is customized based on MASTER Lu et al. (2021) which a powerful
356
      image-to-sequence model originally designed for scene text recognition, and includes one branch
357   to predict the HTML item sequence and the other branch to conduct the box regression. LGPMA
358   Qiao et al. (2021) recognizes complicated table structure with local and global pyramid mask align-
359   ment. TableFormer Nassar et al. (2022) takes a table image and creates HTML structure predictions
360   and bounding boxes. PP-StructureV2 Li et al. (2022a) introduces an ultra lightweight layout de-
361   tector PP-PicoDet to divide document images into predefined areas such as text, title, table, and
362   figure. TabTransformer Huang et al. (2020) is a novel deep tabular data modeling architecture for
363   supervised and semi-supervised learning which is built upon self-attention based Transformers. FT-
364   Transformer Gorishniy et al. (2021) is a revisiting simple adaptation of the Transformer architecture
365
      for tabular data. MATE Eisenschlos et al. (2021) is a novel Transformer architecture designed to
      model the structure of web tables which uses sparse attention in a way to allow heads to efficiently
366
      attend to either rows or columns in a table. DONUT Kim et al. (2022) is a novel OCR-free VDU
367
      model which stands for document understanding Transformer. We use TableMaster Ye et al. (2021),
368   PP-StructureV2 Li et al. (2022a), TableFormer Nassar et al. (2022), LGPMA Qiao et al. (2021),
369   TabTransformer Huang et al. (2020), FT-Transformer Gorishniy et al. (2021), MATE Eisenschlos
370   et al. (2021), and Donut Kim et al. (2022) as eight baselines.

373
374   We evaluate our proposed color-wise context-splitting Transformer-based model and eight baselines
375   with the field-level F1 score metric which checks whether the extracted field information is in the
376   ground truth. Even a single character is missed, the score assume the field extraction is failed.
377   Although F1 is simple and easy to understand, there are some limitations Kim et al. (2022). First,
      it does not take into account partial overlaps. Second, it can not measure the predicted structure

384   4.4     T RAINING OBJECTIVE
385
386   In table recognition stage where we recognize the table cells containing the red stamps on some
387   handwritten signatures, we use Intersection Over Union (IOU) to measure the overlap between two
388   bounding table cells by measuring the correctness of the prediction with the ground truth in the
389
      HTML format which is extracted from table image. During the training processes, we need pay
      attention to some key points including the input size of table image and the minimum area size.
390
      For example, when the maximum number of recognizable tokens in a table structure sequence are
391
      set to 500, the table images which involve more than 500 tokens will be ignored when calculating
392   the accuracy and the TEDS. In addition, the training data of the PubTabNet dataset involve some
393   ambiguous annotations about empty table cell. In order to train our proposed color-wise context-
394   splitting Transformer-based model, we use an ADAM optimizer with the initializing learning rate
395   0.001 and then reduce the learning rate to 0.0001.

525   5     C ONCLUSIONS
526
527   Tables are a common form in PDF documents. However, it is challenging to recognize the table
528   contents from scanned PDF documents containing some red stamps on some handwritten invoice
529   amounts and reconcile the recognized table contents. In this paper, we propose a color-wise context-
530   splitting Transformer-based table structure analysis and table cell recognition method to recognize
531
      the table cells containing red stamps on some handwritten invoice amounts from scanned PDF doc-
      uments. The contributions of our proposed color-wise context-splitting Transformer-based table
532
      recognition method are three-folds. Firstly, we construct efficiently the feature representation of
533
      each table cell by fusing spatial, positional, lexical, and colored context features extracted by the
534   context-splitting based decoders. Secondly, we represent the spatial contexts, positional contexts,
535   lexical (tokenized) contexts, and colored contexts of the table cells in a table structure sequence
536   as low dimensional embedding vectors. Finally, we modify the self-attention mechanism of origi-
537   nal Transformer Vaswani et al. (2017) by multiplying the length of query vector and the length of
538   key vector with the scaling factor of the original Transformer in order to make the training process
539   more stable. As a result, we improve the recognition accuracy of table cells with a red stamp on
      handwritten invoice amounts and the efficiency of reconciliation.
