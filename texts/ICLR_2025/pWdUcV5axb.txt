009                                                     A BSTRACT
010
011            Visual hallucination (VH) occurs when a multimodal large language model (MLLM) generates
012            responses with incorrect visual details for prompts. Existing methods for generating VH test
013            cases primarily rely on human annotations, typically in the form of triples: (image, question,
014            answer). In this paper, we introduce VHExpansion, the first automated method for expanding
015            VH test cases for MLLMs. Given an initial VH test case, VHExpansion automatically expands
016
               it by perturbing the question and answer through negation as well as modifying the image using
               both common and adversarial perturbations. Additionally, we propose a new evaluation metric,
017
               symmetric accuracy, which measures the proportion of correctly answered VH test-case pairs.
018
               Each pair consists of a test case and its negated counterpart. Our theoretical analysis shows
019            that symmetric accuracy is an unbiased evaluation metric that remains unaffected by the im-
020            balance of VH testing cases with varying answers when an MLLM is randomly guessing the
021            answers, whereas traditional accuracy is prone to such imbalance. We apply VHExpansion to
022            expand three VH datasets annotated manually and use these expanded datasets to benchmark
023            seven MLLMs. Our evaluation shows that VHExpansion effectively identifies more VH test
024            cases. Moreover, symmetric accuracy, being unbiased, leads to different conclusions about the
025            vulnerability of MLLMs to VH compared to traditional accuracy metric. Finally, we show that
026            fine-tuning MLLMs on the expanded VH dataset generated by VHExpansion mitigates VH more
027
               effectively than fine-tuning on the original, manually annotated dataset. We will publish code
               and data upon paper acceptance.

030   1   I NTRODUCTION
031
032   Given a prompt containing both an image and a question, multimodal large language models (MLLMs) (Li et al.,
033   2024b; Liu et al., 2023; Bai et al., 2023a; Li et al., 2023a; Tong et al., 2024a; Li et al., 2024a) generate a text
034   response. MLLMs extend the capabilities of large language models (LLMs) (AI@Meta, 2024; Bai et al., 2023b;
035   Touvron et al., 2023; Yang et al., 2024; Chiang et al., 2023) by enabling them to understand visual inputs. An
036   MLLM typically comprises three main components: a vision encoder, a vision-language connector, and an LLM.
037   Specifically, the vision encoder extracts visual embedding vectors from the image in the prompt, while the vision-
038
      language connector aligns these visual embedding vectors with the token-based input used by the LLM. The LLM
      then generates the text response based on the outputs of the vision-language connector and the text in the prompt.
039
      This integration allows MLLMs to tackle complex tasks like Visual Question Answering (VQA) (Tong et al.,
040
      2024b; Huang et al., 2024; Li et al., 2023b; Fu et al., 2023).
041
042   Despite significant advancements, MLLMs are prone to a critical
043   flaw known as visual hallucination (VH) (Huang et al., 2024; Liu
044
      et al., 2024), where the model generates responses containing in-
      correct or misleading visual information. For example, Figure 1
045
      illustrates a VH case where the MLLM provides an incorrect re-
046
      sponse regarding the number of spots on a butterfly’s wings. VH
047   can lead to catastrophic outcomes, especially in high-stakes ap-
048   plications such as autonomous driving (Wen et al., 2023; Chen
049   et al., 2024), medical diagnostics (Qiu et al., 2022), and content
050   moderation (Kumar et al., 2024). Therefore, VH poses signif-
051   icant obstacles to the safe deployments of MLLMs. This con-
      cern is highlighted in the U.S. Executive Order on Trustworthy       Figure 1: An example of visual hallucination

057   Existing VH testing relies on either manual (Li et al., 2023b) or semi-automated (Huang et al., 2024; Tong et al.,
      2024b) methods to construct test cases, both of which require extensive human annotations. As MLLMs evolve

058
      rapidly, these methods struggle to scale up VH testing, limiting the number of test cases and thus hindering
059   comprehensive testing of MLLMs’ vulnerability to VH. Furthermore, existing VH testing methods do not consider
060   adversarial testing (Goodfellow et al., 2014; Carlini & Wagner, 2017) in a white-box setting, where an adversary
061   with full knowledge of the target MLLM can craft adversarial examples to trigger VH through adding human-
062   imperceptible perturbations to the images. This is particularly relevant for open-sourced MLLMs whose model
063   parameters are public. Thus, automated and adversarial methods for generating VH test cases are urgently needed.
064
065   Our work: To address these challenges, we introduce VHExpansion, the first automated framework to generate
066
      VH test cases for MLLMs. Given an initial VH test case, VHExpansion generates additional ones using a com-
      bination of negation as well as common and adversarial image perturbations. Each VH test case is a VQA triple
067
      consisting of an image, a question, and a ground-truth answer. Negation flips the question and answer. To automate
068
      negation, we leverage an LLM with a specifically designed prompt. For common image perturbations, we process
069   the image via frequently encountered image processing operations such as JPEG compression, Gaussian noise,
070   etc.. For adversarial image perturbations, we add a human-imperceptible perturbation to the image so that the
071   resulting embedding vector, generated by the vision encoder, differs significantly from the original. We formulate
072   finding the perturbation as a constrained optimization problem, solved using Projected Gradient Descent (Madry
073   et al., 2018) or the iterative Fast Gradient Sign Method (Kurakin et al., 2018). We apply VHExpansion to expand
074   three existing VH datasets annotated manually and use these expanded datasets to benchmark seven MLLMs. Our
075   evaluation demonstrates that VHExpansion effectively identifies more VH test cases.
076   Moreover, we introduce a new evaluation metric called symmetric accuracy, which measures the proportion of
077   correctly answered VH test-case pairs, where each pair includes a VH test case and its negated counterpart.
078   Symmetric accuracy captures the consistency of an MLLM in accurately answering both the original and negated
079   questions. In fact, we theoretically show that symmetric accuracy is an unbiased evaluation metric that remains
080   unaffected by the imbalance of VH testing cases with varying answers when an MLLM is randomly guessing
081   the answers, whereas traditional accuracy is prone to such imbalance. Our empirical benchmark results show
082   that symmetric accuracy and traditional accuracy can lead to different conclusions about MLLMs’ vulnerability
083
      to VH. For instance, on the POPE dataset (Li et al., 2023b), Cambrian-1 (Tong et al., 2024a) achieves a higher
      traditional accuracy than LLaVA-NeXT (Li et al., 2024a) (0.887 vs. 0.879), but performs worse in symmetric
084
      accuracy (0.745 vs. 0.798).
085
086   Finally, we demonstrate that fine-tuning an MLLM on the expanded VH test cases generated by VHExpansion
087   significantly mitigates visual hallucinations. For example, our experiments show that when fine-tuning LLaVA-
088   1.5 on the POPE dataset, using randomly sampled 200 VH test cases results in a symmetric accuracy of 0.180,
089
      whereas fine-tuning on both the sampled VH test cases and the corresponding expanded 1,800 more VH test cases
      achieves a symmetric accuracy of 0.711. This highlights the effectiveness of VHExpansion in mitigating VH in
090
      MLLMs. Additionally, our evaluation shows that fine-tuning does not compromise the model’s performance on
091
      other general-purpose VQA datasets, preserving its broader functionality.
092
093   To summarize, we make the following contributions in this work:
094
095          • We introduce VHExpansion, the first automated framework for generating VH test cases in MLLMs,
096
               combining negation and common and adversarial image perturbations.
097          • We propose a new evaluation metric, symmetric accuracy, to quantify an MLLM’s performance. Sym-
098            metric accuracy is unaffected by the imbalance of test cases when an MLLM makes random guessing.
099
             • We demonstrate that fine-tuning MLLMs on the expanded test cases generated by VHExpansion signifi-
100            cantly mitigates VH while maintaining general performance on other VQA datasets.

136   2.2   M ETHODS TO G ENERATE VH T EST C ASES
137
      To detect and mitigate VH in MLLMs, several methods to generate VH test cases (Li et al., 2023b; Huang et al.,
138
      2024; Tong et al., 2024b; Guan et al., 2023) have been proposed. These methods can be categorized into two
139   types: manual and semi-automatic. Manual methods (Li et al., 2023b; Guan et al., 2023) involve creating each
140   VH test case through human effort. For example, POPE (Li et al., 2023b) requires human annotation for each
141   image to identify the objects within it and then design corresponding questions based on these objects, including
142   some randomly introduced non-existent objects. Note that the images in POPE are also human-created.
143
      To reduce the human labor involved in generating VH test cases, semi-automatic methods (Huang et al., 2024)
144
      have been developed. For instance, VHTest uses GPT-4V (Achiam et al., 2023) and DALL·E-3 (Betker et al.,
145   2023) to facilitate construction of VH test cases. Specifically, it first employs CLIP (Radford et al., 2021) and
146   DINO (Oquab et al., 2023) to detect images from benchmark datasets that may trigger VH in MLLMs. These im-
147   ages are then passed to GPT-4V to generate textual descriptions. The generated text descriptions are subsequently
148   passed to DALL·E-3 to create more images. Based on these AI-generated images, human workers manually iden-
149   tify objects within them and design questions, along with the corresponding ground-truth answers. Similarly,
150   MMVP (Tong et al., 2024b) uses CLIP and DINO to identify image pairs that have a high CLIP score but a low
151   DINO score. Human workers then manually examine the differences between these paired images and formulate
152   questions/answers based on those differences.
153
154   2.3   M ITIGATING VH VIA F INE - TUNING
155
      With VH datasets constructed by these methods, MLLMs can be fine-tuned on them to mitigate VH (Huang et al.,
156
      2024). This approach enables the MLLMs to learn from instances of VH, allowing them to distinguish between
157
      accurate visual representations and hallucinated content. By exposing the models to diverse VH instances during
158   fine-tuning, they can better generalize and reduce the occurrence of hallucinations (Huang et al., 2024).

187   The primary intuition behind negation is that an MLLM may simply guess the answer (i.e., “yes” or “no”) correctly
188   for binary questions without really understanding the image. In particular, some MLLMs such as LLaVA-1.5 tend
189
      to answer “yes” for binary questions (Liu et al., 2023). Therefore, if the VH test cases are imbalanced and a major-
      ity of them have “yes” as ground-truth answers, such MLLMs would have high accuracy without understanding
190
      the images, misleading developers to think that the MLLMs are not vulnerable to visual hallucination. However,
191
      such MLLMs would be likely to answer incorrectly for the negated questions, leading to low accuracy on them.
192   Thus, the VH test cases and their negated versions can better quantify the vulnerability of an MLLM to visual
193   hallucination. In fact, in Section 4, we propose a new evaluation metric, called symmetric accuracy, which mea-
194   sures the percentage of correctly answered VH test-case pairs, each of which includes a test case and its negated
195   version. In Section 4, we theoretically show that symmetric accuracy is unaffected by the imbalance of VH test
196   cases with answers “yes” and “no” when the MLLM makes random guessing, while accuracy on the original VH
197   test cases alone is prone to such imbalance.

199   3.2     M ODIFYING I MAGE XI
200
201   Common image perturbations: In real-world scenarios, images often undergo standard editing operations for
202   various purposes. For example, images are frequently compressed using formats like JPEG to reduce trans-
203   mission costs over the Internet. These image edits are known as common image perturbations (Hendrycks &
204   Dietterich, 2019). Our VHExpansion uses these perturbations to generate additional VH test cases. Given a VH
205   test case {xI , xQ , yA }, we apply a common perturbation method T to the image xI , creating a new VH test case
206   {T (xI ), xQ , yA }. The intuition is that for a slightly perturbed image T (xI ), the ground-truth answer yA should
207   remain unchanged for the same question xQ . However, this subtle alteration may trigger VH in an MLLM. We
208   focus on four common image perturbations: Gaussian Noise, Brightness Adjustments, Defocus Blur, and JPEG
209
      Compression. Further details on these common perturbations are provided in Section C of the Appendix.
210
      Adversarial image perturbations: In the context of adversarial image perturbations, we consider a white-
211
      box setting where an adversary, with full knowledge of the target MLLM’s model parameters, crafts nearly-
212   imperceptible adversarial perturbations to generate VH test cases. Given an original VH test case {xI , xQ , yA },
213   the adversarial image perturbation generates a new test case {xI + δ ∗ , xQ , yA }, where δ ∗ is the adversarial per-
214   turbation. Our intuition is that for a VH test case that does not trigger VH in an MLLM M , VHExpansion creates
215   perturbations that cause the projected visual embedding vector from the vision-language connector to differ from
216   the original. Conversely, if the test case already triggers VH in M , VHExpansion generates perturbations that
217   make the projected visual embedding vector similar to the original. Formally, for an MLLM M with vision en-
218   coder ME and vision-language connector MC , we formulate finding δ ∗ as the solution to the following constrained
219   optimization problem:
220
                         arg minδ (− cos (ME ◦ MC (xI ), ME ◦ MC (xI + δ))) , if xI does not trigger VH,
221              δ∗ =
                         arg minδ (cos (ME ◦ MC (xI ), ME ◦ MC (xI + δ))) ,           if xI triggers VH,
222
223                     s.t.   ||δ||∞ ≤ ϵ,                                                                                    (1)
224
      where ME ◦ MC denotes the concatenation of the vision encoder and the vision-language connector, cos denotes
225
      cosine similarity, and ϵ is the ℓ∞ -norm constraint on the perturbation δ added to the image xI . Note that when
226
      the VH test case already triggers VH, we initialize δ to be a non-zero vector with random value and apply early
227   stopping to avoid the optimization result to be identical with the original image input xI ; and when the VH
228   test case does not trigger VH, we initialize δ to be zero. Our algorithm solves the optimization problem in
229   Equation 1 using either Projected Gradient Descent (PGD) (Madry et al., 2018) or the iterative Fast Gradient
230   Sign Method (I-FGSM) (Kurakin et al., 2018). PGD iteratively updates δ via gradient ascent: δ = δ − γ · ∇δ l,
231   where l = cos (ME ◦ MC (xI ), ME ◦ MC (xI + δ)), followed by projecting δ onto the feasible region using
      δ = clip(δ, −ϵ, ϵ). I-FGSM differs from PGD by using the sign of the gradient instead: δ = δ − γ · sign(∇δ l).

242
243   In this section, we theoretically analyze the standard accuracy metric and our proposed symmetric accuracy metric
244   for evaluating an MLLM model’s performance when the model is making random guessing. Suppose we are given
245   a VH test case t = {xI , xQ , yA }, sampled from the distribution T of VH test cases, i.e., t ∼ T . Our analysis
246   focuses on binary questions, i.e., yA is either “yes” or “no”. Specifically, we denote by q the probability that a
247
      randomly sampled t has a ground-truth answer “yes”. In other words, a randomly sampled t has a ground-truth
      answer “no” with probability 1 − q. q quantifies the imbalance of the VH test cases with answers “yes” and “no”.
248
249   We denote by f an MLLM model and f (xI , xQ ) the MLLM’s answer for the VH test case. f (xI , xQ ) ̸= yA
250   indicates that the MLLM hallucinates. When the MLLM model makes random guessing to answer the test case
251   without understanding the image xI and question xQ , it outputs an answer “yes” or “no” randomly. Suppose the
252   MLLM model guesses “yes” with probability p and “no” with probability 1 − p.
253   An evaluation metric measures the performance of an MLLM model f on the VH test cases whose distribution is
254   T . Specifically, an evaluation metric takes T and f as input and outputs a number (e.g., between 0 and 1), with
255   a smaller number indicating that f is more vulnerable to VH test cases from the distribution T . An evaluation
256   metric is unbiased if it does not depend on the imbalance of the VH test cases when the model f makes random
257   guessing, i.e., it does not depend on q. Otherwise, the evaluation metric is biased. Formally, we have the following
258   definition.
259   Definition 1 (Unbiased Evaluation Metric). An evaluation metric is said to be unbiased if does not depend on q
260   when the MLLM model makes random guessing.
261
262
      Next, we formally define accuracy and prove that accuracy is a biased evaluation metric.
263   Definition 2 (Accuracy). Accuracy is the probability that an MLLM model f correctly answers a VH test case
264   t = {xI , xQ , yA } sampled from T . Formally, we have: accuracy = Prt∼T (f (xI , xQ ) = yA ).
265   Theorem 1. Accuracy is a biased evaluation metric when p ̸= 12 , where p is the probability that the MLLM model
266   guesses answer “yes”.

333   across various VQA benchmarks and have diverse model architectures. Table 2 shows details of these MLLMs.
334
335   Evaluation metrics: We use accuracy and symmetric accuracy as our evaluation metrics, both of which are
336   formally defined in Section 4. Our theoretical analysis demonstrates that symmetric accuracy is an unbiased eval-
337   uation metric, whereas traditional accuracy is biased. In our experiments, we illustrate how symmetric accuracy
338   leads to different conclusions about the vulnerability of MLLMs to VH compared to the traditional accuracy met-
339   ric. Subsequently, we use symmetric accuracy as our default evaluation metric unless otherwise mentioned. We
340   also report the number of successful VH test cases generated by our VHExpansion.
341
      Parameter settings: Unless otherwise mentioned, we use LLaVA-1.5 on MMVP dataset by default. We use
342
      GPT-4o as the LLM to negate all questions in VH test cases due to its state-of-the-art performance. We use the
343   default parameter settings for all MLLMs. For common image perturbations, the parameters are set as follows:
344   Gaussian Noise standard deviation σ = 0.08, Brightness Hue-Saturation -Value space constant c = 0.5, Defocus
345   Blur radius r = 5 , and JPEG Compression quality factor q = 30. More details of these common perturbations are
346   shown in Section C in Appendix. For adversarial image perturbations, the default setting is: ℓ∞ -norm constraint
347   ϵ = 8/255, with 500 epochs for non-hallucinated VH test cases and 100 epochs for hallucinated test cases. In
      hallucinated test cases, each pixel of the initial perturbation is set to 5/255 or −5/255 uniformly at random.

      5.2   E XPERIMENTAL R ESULTS
349
350   Symmetric accuracy v.s. accuracy: Table 3 shows accuracy and symmetric accuracy of the seven MLLMs
351   across the three datasets MMVP, VHTest, and POPE. We have three main observations. First, symmetric accuracy
352   reveals different conclusions about MLLM vulnerability to VH compared to traditional accuracy. For example, on
353   the POPE dataset, Cambrian-1 has higher traditional accuracy than LLaVA-NeXT (0.887 vs. 0.879) but performs
354   worse in symmetric accuracy (0.745 vs. 0.798). Second, when comparing symmetric accuracy across MLLMs,
355   GPT-4o achieves the highest scores on MMVP and VHTest, particularly on MMVP with 0.663, indicating it is
356   less prone to visual hallucinations than other models. LLaVA-OneVision scores the highest symmetric accuracy
357   on POPE (0.843), likely due to its fine-tuning on simpler existence-based questions and possible overlap between
358
      POPE and its training data. Third, across VH datasets, all MLLMs perform worse on VHTest, with InstructBLIP
      scoring only 0.117. This is likely because VHTest contains VH test cases with AI-generated images and more
359
      complex questions that the models have not trained on, making it more challenging. In addition, expanding the
360
      dataset using negation allows us to generate more new VH instances, providing additional training data for fine-
361   tuning, which leads to more robust models.
362
363   Common and adversarial image perturbations generate more VH test cases: Table 4 shows the symmetric
364   accuracy on three datasets of different MLLMs before and after common image perturbations. We observe that
365   symmetric accuracy slightly decreases after common image perturbations in most cases. This shows that most
366   MLLMs are generally robust against common perturbations. For instance, LLaVA-NeXT’s symmetric accuracy
367
      on the VHTest dataset drops from 0.287 to 0.272 under Gaussian Noise. However, there are still some notable
      exceptions. For example, Defocus Blur significantly reduces LLaVA-OneVision’s accuracy on POPE, from 0.843
368
      to 0.646; while three of four common perturbations even increase InstructBLIP’s symmetric accuracy on VHTest.
369
370   Table 5 shows the symmetric accuracy on the three datasets of different MLLMs before and after adversarial
371   image perturbations. Our main observation is that adversarial perturbations cause significant drops in symmetric
372   accuracy for all MLLMs. For example, LLaVA-1.5’s symmetric accuracy on POPE drops sharply from 0.468 to
373
      0.017 when performing I-FGSM to craft adversarial perturbations. When comparing I-FGSM with PGD, I-FGSM
      consistently results in a larger decrease in accuracy, indicating it is more effective.
374
375   To conclude, MLLMs are fairly robust to common image perturbations but remain vulnerable to adversarial ones,
376   highlighting the need for more adversarially robust training strategies. Furthermore, both common and adversarial
377   perturbations lead to the generation of more new VH instances, with adversarial perturbations producing more VH
378   instances than common perturbations, since MLLMs are more vulnerable to them.
379   Manual verification for negation: The correctness of our proposed symmetric accuracy metric relies on the
380   validity of the negated questions, which are generated by LLMs. Since LLMs may exhibit hallucinations, these
381   negated questions might not always be the negated counterparts of the original questions. Thus, it is necessary to
382   verify whether the negated questions generated by the LLM are correct.
383
      To validate the correctness of these negated questions, we randomly sampled 200 VQA triples (100 original-
384
      negation pairs) from each of the MMVP, VHTest, and POPE datasets, which were evaluated by four independent
385   annotators. The task of the annotators was to judge whether each negated question was the correct negation of
386   the corresponding original question generated by the LLM. The annotators unanimously agreed that all negated
387   questions were correctly generated by the LLM. This result demonstrates the reliability of the LLM in generating
388   valid negations.

573
574
      In this paper, we introduce VHExpansion, an automated framework to generate VH test cases for MLLMs. VHEx-
575   pansion significantly advances VH testing by automating the generation of test cases through techniques such as
576   negation and image perturbations, both common and adversarial. We also propose an unbiased evaluation metric,
577   symmetric accuracy, to measure the consistency of MLLMs in answering VH test cases and their negated coun-
578   terparts. Our experiments demonstrate that, given VH test cases, VHExpansion can find more successful VH test
579   cases. Importantly, fine-tuning MLLMs on the expanded VH test cases generated by VHExpansion significantly
      mitigates VH, while maintaining general performance on standard VQA tasks.

      Proof. Standard accuracy is defined as: Accuracy = Prt∼T (f (xI , xQ ) = yA ). Since the model’s predictions are
699   independent of yA when random guessing:
700
701              E[Accuracy] = P (yA = Yes) · P (f (xI , xQ ) = Yes) + P (yA = No) · P (f (xI , xQ ) = No)
702                          = q · p + (1 − q) · (1 − p)
703                          = 1 + (2p − 1) · q − p.
704
705   This expression shows that E[Accuracy] depends on the class distribution (P (yA = Yes)) if p ̸= 21 . If the model’s
706   bias aligns with the majority class (e.g., p is large when P (yA = Yes) is large), E[Accuracy] is artificially inflated,
707   even though the model is merely guessing.
708
      Therefore, standard accuracy is biased due to class imbalance and model bias.

713   Proof. Symmetric accuracy is defined as: Symmetric Accuracy = Prt∼T (f (xI , xQ ) = yA ∧f (xI , ¬xQ ) = ¬yA ).
714   Since model predictions are independent of yA and independent between xQ and ¬xQ under random guessing:
715               E[Symmetric Accuracy] = P (f (xI , xQ ) = yA ) · P (f (xI , ¬xQ ) = ¬yA )
716
                                        = P (yA = Yes) · P (f (xI , xQ ) = Yes) · P (f (xI , ¬xQ ) = No)
717
718
                                          + P (yA = No) · P (f (xI , xQ ) = No) · P (f (xI , ¬xQ ) = Yes)
719                                     = q · p(1 − p) + (1 − q) · p(1 − p)
720                                     = p(1 − p).                                                                       (2)
721

723   Thus, symmetric accuracy is an unbiased evaluation metric with respect to class imbalance.

729          • Gaussian Noise In this method, Gaussian noise is randomly sampled from a distribution with zero mean
730
               and a standard deviation of σ. The image pixel values are first converted to the range [0, 1], and the
               generated noise is then added to these values. This process simulates the noise real-world images might
731
               experience during transmission. In our experiments, the standard deviation σ is set to 0.08.
732
733          • Brightness This method adjusts image brightness by modifying its V (value) channel in the HSV color
734
               space. The input image is first normalized to [0, 1] and converted from RGB to HSV. The brightness is
               then altered by adding a constant c to the V channel, with values clipped to the range [0, 1]. The image
735
               is finally converted back to RGB. In our experiments, the constant c is set to 0.5.
736
737          • Defocus Blur This method applies a defocus blur to the image using a disk-shaped kernel. The input
738
               image is normalized to [0, 1], and a disk kernel of radius c is generated. Each of the three RGB channels
               is filtered independently with this kernel, then recombined and clipped to the range [0, 1]. The radius c
739
               is set to 5 in our experiments.
740
741          • JPEG Compression This method compresses the input image using a specified quality factor q. Lower
742
               q values result in higher compression and more artifacts, while higher values retain more image quality.
               In our experiments, the quality factor q is set to 30.
