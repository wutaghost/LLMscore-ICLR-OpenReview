011                                              A BSTRACT
012
013            A 3D scene graph represents a compact scene model, storing information about the
014            objects and the semantic relationships between them, making its use promising for
015            robotic tasks. When interacting with a user, an embodied intelligent agent should
016
               be capable of responding to various queries about the scene formulated in natural
               language. Large Language Models (LLMs) are beneficial solutions for user-robot
017
               interaction due to their natural language understanding and reasoning abilities.
018
               Recent methods for creating learnable representations of 3D scenes have demon-
019            strated the potential to improve the quality of LLMs responses by adapting to the
020            3D world. However, the existing methods do not explicitly utilize information
021            about the semantic relationships between objects, limiting themselves to informa-
022            tion about their coordinates. In this work, we propose a method 3DGraphLLM for
023            constructing a learnable representation of a 3D scene graph. The learnable repre-
024            sentation is used as input for LLMs to perform 3D vision-language tasks. In our
025            experiments on popular ScanRefer, RIORefer, Multi3DRefer, ScanQA, Sqa3D,
026            and Scan2cap datasets, we demonstrate the advantage of this approach over base-
027            line methods that do not use information about the semantic relationships between
028            objects.

031   1   I NTRODUCTION
032
033   In this paper, we consider scene understanding in the context of solving 3D vision-language prob-
034   lems: 3D referred object grounding task, 3D dense scene captioning and 3D visual question an-
035   swering. The 3D referred object grounding task involves identifying a region in a 3D scene that
036   corresponds to a complex natural language query. This query may describe object properties (e.g.,
037
      color, size) and spatial relationships (e.g., a mug on a table). A common approach to solving this
      problem is to assume that one is given a 3D reconstruction of the scene (e.g., a point cloud, mesh, or
038
      NeRF). The goal is to predict the bounding boxes of the region that matches the query. The goal of
039
      the dense scene captioning task is to describe the selected object. Finally, the goal of the 3D visual
040
      question answering task is to generate text answer to various questions about the properties of the
041
      scene. It seems promising to explicitly use a three-dimensional scene graph to solve these tasks.
042
043   The 3D scene graph not only allows storing multimodal information about individual objects within
044   a scene but also captures the semantic relationships (Wang et al., 2023b; Koch et al., 2024) and
045   hierarchical organization between them (Werby et al., 2024; Honerkamp et al., 2024). Additionally,
046   the graph scene representation enables real-time updates for dynamic environments (Rosinol et al.,
047
      2021; Özsoy et al., 2023), and supports the application of graph algorithms for tasks such as nav-
      igation (Zhou et al., 2023b; He & Zhou, 2024; Honerkamp et al., 2024) or object search based on
048
      textual queries (Feng et al., 2021; Chang et al., 2023; Werby et al., 2024; Gu et al., 2024).
049
050   The solving of 3D vision-language tasks (Chen et al., 2020; 2021; Azuma et al., 2022) is crucial for
051   embodied intelligent agents. To interact with the user, an intelligent agent must be able to describe
052   the environment and answer questions about its properties using natural language. Large language
053   models (LLMs) are particularly well-suited for this task, as their advanced capabilities in natural
      language understanding and common-sense reasoning make them highly effective in interpreting and

082
      designed for use as input to a LLM (see Figure 1). This representation consists of a list of learnable
083
      embeddings for objects within the scene, where each object is represented by a subgraph containing
084   the object itself along with several of its nearest neighbors. These object subgraphs are provided to
085   the LLM as a sequence of triplets (object1, relation, object2). Semantic relations between objects
086   are embedded using features derived from the semantic edges of the graph, which is generated
087   using state-of-the-art methods for 3D semantic graph generation such as VL-SAT (Wang et al.,
088   2023b). Our experiments demonstrate that incorporating semantic relationships between objects
089   significantly improves the accuracy of LLM responses for 3D vision-language tasks, outperforming
090   baseline approaches for creating learnable scene representations.

      2   R ELATED WORKS
109
110
111   Scene Graphs. The concept of a scene graph was initially developed for 2D images, providing a
112
      structured representation of a scene’s semantics by incorporating relationships between the semantic
      elements (Johnson et al., 2015). In the context of images, scene graphs have proven effective for
113
      tasks such as content-based image retrieval (Johnson et al., 2015; Pei et al., 2023), 2D referring ex-
114
      pression comprehension (Yang et al., 2019a; Shi et al., 2023; Han et al., 2024), image caption (Yang
115   et al., 2019b; Phueaksri et al., 2023), image generation (Johnson et al., 2018; Farshad et al., 2023).
116
117   In 3D scenes, a scene graph is commonly used to address robotics challenges such as plan-
118
      ning (Werby et al., 2024; Honerkamp et al., 2024), object grounding for navigation (Werby et al.,
      2024; Gu et al., 2024; Linok et al., 2024; Honerkamp et al., 2024) and manipulation (Honerkamp
119
      et al., 2024), as well as scene generation (Zhai et al., 2024; Gao et al., 2024).
120
121   Our approach is part of a class of methods that utilize an implicit representation of the scene graph,
122   such as OVSG (Chang et al., 2023), which frames the problem of 3D object grounding as subgraph
123   retrieval. 3DGraphQA (Wu et al., 2024) proposes to use the bilinear graph neural network for
124
      feature fusion between scene and question graphs for question answering task. Feng et al. (2021)
      build a graph based on a text query, which is used to refine the visual graph in order to select from its
125
      vertices the one that best fits the description. However, the application scope of this method is limited
126
      to specific tasks as 3D referred object grounding with one referred object or question answering. In
127   contrast, we propose a more versatile method capable of solving various 3D vision-language tasks.
128
129   3D Language Scene Understanding. 3D scene understanding is a complex computer vision task
130
      that involves identifying the semantic, physical, and functional properties of objects, as well as
      their mutual relations. One of the goals of 3D scene understanding is to develop methods capable
131
      of responding to natural language queries about the scene. The queries may correspond to differ-
132
      ent visual-language tasks such as 3D referred object grounding (Chen et al., 2020; Zhang et al.,
133   2023; Miyanishi et al., 2024), question answering (Azuma et al., 2022), and dense scene caption-
134   ing (Chen et al., 2021). Recent approaches address these queries by reconstructing the scene as a
135   3D mesh (Peng et al., 2023) or point cloud (Zhao et al., 2021; Chen et al., 2022; Zhu et al., 2023),
136   often enhanced with instance segmentation (Zhu et al., 2023).
137
      The emergence of transformer models (Vaswani, 2017) has enabled the development of neural net-
138
      work models that create a learnable representation of a scene for answering various language queries.
139   MultiCLIP (Delitzas et al., 2023) proposes to align 3D scene representation with text queries and
140   multi-view 2D CLIP (Radford et al., 2021) embeddings to improve the quality of question answer-
141   ing. 3DVG-Transformer (Zhao et al., 2021) and Vil3DRef (Chen et al., 2022) methods introduce
142   modules for modeling spatial relationships between objects to improve the quality of object ground-
143   ing. 3D-VisTA (Zhu et al., 2023) presents a transformer model for aligning 3D object and text
144   representations, coupled with an unsupervised pre-training scheme to solve various 3D vision-text
145   problems using specialized task-specific heads. However, these fully supervised approaches face
146   challenges in generalizing to new tasks and domains. In contrast, leveraging large language mod-
147
      els (LLMs) for scene understanding enhances generalization capabilities and taps into the extensive
      knowledge LLMs contain about the physical world (Hong et al., 2023b).
148
149   Large Language Models for Scene Understanding. Large language models (LLMs) offer several
150   advantages for scene understanding, notably enhancing the ability to address complex queries that
151   require common knowledge. LLMs can serve as agents that decompose user queries into elementary
152   tasks, which can then be addressed by other methods (Yang et al., 2024; Yuan et al., 2024). Addi-
153
      tionally, LLMs can act as an interface for reasoning by processing textual descriptions of the scene
      as input (Linok et al., 2024; Gu et al., 2024). BBQ (Linok et al., 2024) and ConceptGraphs (Gu et al.,
154
      2024) demonstrate that using a text-based graph representation with an LLM interface significantly
155
      improves the quality of object retrieval compared to using CLIP features of objects. HOV-SG (Werby
156   et al., 2024) construct a hierarchical graph consisting of objects, rooms, and floors, and demonstrate
157   the effectiveness of such a representation for the task of object grounding given a query containing
158   object location hints. The authors of the MOMA (Honerkamp et al., 2024) method propose using
159   a hierarchical scene graph together with a navigational Voronoi graph as input to LLM to predict a
160   high-level policy for object search for navigation and manipulation. However, using text to describe
161   an object in a scene graph inevitably leads to the loss of some of the information contained in its
      RGB point cloud. Additionaly, in the case of using a text graph, several hundred tokens may be


                                                         3
      Under review as a conference paper at ICLR 2025


162
      required to describe one object (its semantic class, pose), which will significantly slow down LLM
163   inference in the case of a large number of objects in the scene.
164
165   Recent advancements have successfully integrated point cloud data into LLMs by employing pre-
166
      trained point cloud encoders and training adapters to align the resulting representations with the
      LLM embedding space. 3D-LLM (Hong et al., 2023a) aggregates 3D point cloud features from a
167
      sequence of 2D images and then solves the grounding problem as a prediction of a sequence of lo-
168
      cation tokens added to the LLM dictionary. Chat3D-v2 (Huang et al., 2023) generates a 3D feature
169   for each object in the scene and then treats the grounding problem as an object selection problem.
170   LLA3D (Chen et al., 2023) proposes to use a set of trainable fixed-length query tokens obtained by
171   interacting potential visual cues, text cues, and object point cloud features in a transformer model.
172   Grounded 3D-LLM (Chen et al., 2024) uses referent tokens to decode object masks in point clouds.
173   Additionally, research has demonstrated that incorporating spatial information, such as object coor-
174   dinates (Huang et al., 2023) or depth maps (Cheng et al., 2024), enhances the accuracy of responses
175   to user queries.
176   Despite recent advances, existing methods do not fully leverage the rich semantic information in
177   object relationships. In this paper, we introduce 3DGraphLLM, a method that demonstrates the
178   effectiveness of utilizing semantic relationships between objects to enhance performance across
179   various scene understanding tasks.

182   3     M ETHOD
183
184   Our approach uses a set of point clouds of scene objects as input. The objects’ point clouds can be
185   obtained either from ground-truth annotations or through state-of-the-art point cloud instance seg-
186   mentation methods. These point clouds are used to extract scene graph features (see Section 3.1).
187
      A scene graph consists of nodes representing the objects and edges corresponding to semantic rela-
      tionships between them. To convert the scene graph into a token sequence, we represent each object
188
      by an identifier, followed by a subgraph comprising the object’s k nearest neighbors. The rela-
189
      tionships between an object and its neighbors are encoded as triplets (objecti , relationij , objectj ).
190   The scheme of the 3DGraphLLM approach is shown in Figure 2. For more details on the scene
191   graph representation, refer to Section 3.2. Our training process is two-stage. First, we pre-train the
192   model on a dataset for various 3D scene understanding tasks using ground-truth instance segmenta-
193   tion. Next, we fine-tune 3DGraphLLM with predicted instance segmentation of scene point clouds,
194   considering a scenario where ground-truth segmentation is unavailable (see Section 3.3).
195
196
      3.1   M ODEL A RCHITECTURE
197
198   The model architecture includes pre-trained encoders for 3D point clouds and their semantic re-
199   lationships, alongside a pre-trained LLM. We train projection layers to map the extracted object
200   features and their relationships into the LLM’s token embedding space. Following the approach of
201   Chat-Scene (Huang et al., 2024), we introduce additional object identifier tokens {< OBJi >}ni=1
202   into the LLM’s vocabulary. Here and throughout, we use n to denote the number of objects in the
203   scene. These learned identifiers, along with the features from object subgraphs composed of nearest
204   neighbors for each object, are used to create a flat representation of the scene graph, which is then
205   fed into the LLM.
206   Object Proposals. We use point clouds of objects in the scene as vertices in the scene graph
207   G. In our experiments, we evaluate 3DGraphLLM in various modes, including ground-truth
208   scene segmentation and instance segmentation using state-of-the-art neural network methods like
209   Mask3D (Schult et al., 2023) and OneFormer3D (Kolodiazhnyi et al., 2024). Thus, the set V of ver-
210   tices of the graph consists of n point clouds {Pi }ni=1 , where Pi ∈ Rmi ×6 . Here, mi is the number
211   of points in the i-th object proposal of instance segmentation of scene point cloud, and 6 dimensions
212
      of each point correspond to its 3D coordinates and RGB color.
213   Object Identifiers. Following the approach in Chat3D-Scene, we add a set of learnable identifier
214   tokens {< OBJi >}ni=1 to the LLM’s vocabulary for object identification. These tokens allow the
215   model to identify objects in the scene by simply predicting the corresponding object identifier token.
      In our experiments, we assume a maximum of 200 objects per scene.

292   3.2      F LAT G RAPH R EPRESENTATION
293
294   The scene graph is a complete graph because we can generate connections between all pairs of ob-
295   jects. However, such a graph contains n · (n − 1) edges between objects, and using the complete
296   graph as a sequence for the LLM would significantly increase the sequence length. However, in-
297   tuitively, the most relevant relationships for answering user questions are those between an object
298   and its nearest neighbors. Therefore, for each object, we consider a subgraph of its k nearest neigh-
299
      bors. The relationships between objects are encoded using features extracted from point clouds
      {Fiv }ni=1 and semantic relations features {Fije , i ∈ {1, ..., n}, j ∈ {1, ..., n}}, represented as a
300
      triplet (Fiv , Fije , Fjv ). When using the complete scene graph, the number of tokens required to de-
301
302
      scribe the scene is 2 · n + 3n · (n − 1). For 100 objects, which matches the number of objects in the
303
      Mask3D (Schult et al., 2023) instance segmentation, this totals 29900 tokens. By using a k-nearest
      neighbor subgraph, we reduce the token count to n + 3n · k. As shown in Section 4.3 (see Figure 4),
304
      setting k = 2 improves accuracy in 3D visual-language tasks while reducing the number of tokens
305
      needed to describe a scene with 100 objects to 800.

347
348
      Datasets. We conduct experiments using the ScanNet (Dai et al., 2017) and 3RScan (Wald et al.,
349   2019) scene datasets. For training 3DGraphLLM on ScanNet scenes (Dai et al., 2017), we uti-
350   lize data from five 3D vision-language benchmarks: visual grounding tasks (ScanRefer (Chen et al.,
351   2020), Multi3DRefer (Zhang et al., 2023)), scene description (Scan2Cap (Chen et al., 2021)), and 3D
352   visual question answering (ScanQA (Azuma et al., 2022), SQA3D (Ma et al., 2022)). Each of these
353   datasets follows a standard split into training and validation sets, corresponding to 1201 training
354   scans and 312 validation scans from ScanNet. Additionally, we include the RioRefer dataset (Miyan-
355   ishi et al., 2024), which provides referring expressions for objects in 3RScan scenes (Wald et al.,
356   2019) splitting into standard training and validation sets (1175 training scans and 157 validation
357
      scans). Since our method primarily targets visual grounding tasks, the majority of validation exper-
      iments are performed on the ScanRefer, Multi3DRefer, and RioRefer datasets.
358
359   Implementation details. The projection layers for 3D object features and their semantic relations
360   are three-layer MLPs. In our experiments, we use LLAMA3-8B-Instruct (AI@Meta, 2024), a state-
361   of-the-art large language model, as well as Vicuna-1.5-7B (Zheng et al., 2023) for ablation. For
362   fine-tuning the language model, we apply LoRA (Hu et al., 2021) with a rank of 16. We use a batch
363
      size of 8 and train 3DGraphLLM for 3 epochs with an initial learning rate of 0.00002, following
      a cosine annealing schedule. Training is performed on a server equipped with an NVIDIA A100
364
      GPU, and the entire training process takes approximately 36 hours. In our experiments, we select
365
      k = 2 nearest neighbors to construct object subgraphs and, in the case of using Mask3D (Schult
366   et al., 2023) instance scene point cloud segmentation, we use a NMS filter and a filter that ensures a
367   minimum distance between nearest neighbors of 1 cm (see Section 4.3).
368
369
      Evaluation metrics. For the visual grounding task on the ScanRefer (Chen et al., 2020) and Ri-

370
      prediction is considered a true positive if the intersection-over-union (IoU) between the predicted
371
      object’s 3D bounding box and the ground truth exceeds the thresholds of 0.25 and 0.5, respectively.
372   The Multi3DRefer (Zhang et al., 2023) dataset contains queries that may refer to multiple objects.
373   Therefore, we use the benchmark-standard F1 score at IoU thresholds of 0.25 and 0.5. During
374   ablation experiments, we also assess the quality of object descriptions using the Scan2Cap (Chen
375   et al., 2021) benchmark metrics CIDEr@0.5 and BLEU-4@0.5. For the visual question answering
376   task, we follow the validation strategy from Chat3Dv2, applying CIDEr (Vedantam et al., 2015) and

467
468   We conduct a series of experiments to explore methods for constructing a scene graph repre-
469   sentation from a point cloud. In these experiments, we use a frozen version of LLAMA3-8B-
470   Instruct (AI@Meta, 2024), training only the projection layers. We do not introduce new object
471   tokens into the LLM’s dictionary and follow a three-stage training process, including 3D Object
472   Alignment, 3D Scene Alignment, and Instruction Tuning, as outlined in Chat3D (Wang et al.,
473   2023a).
474   Quality of instance segmentation. We evaluate how the quality of scene segmentation into objects
475   impacts the performance of 3DGraphLLM. As shown in Table 4, even with noisy neural network
476   segmentation, representing the scene as a graph with semantic relationships is still more effective
477   than using a simple list of objects. We conduct experiments with different object proposal methods,
478   including OneFormer3D (Kolodiazhnyi et al., 2024) and Mask3D (Schult et al., 2023), but found no
479   significant difference between them for our task. Therefore, in subsequent experiments, we use the
480   Mask3D method to maintain consistency with the baseline Chat3Dv2 approach.
481   Neural network segmentation imperfections impact both the quality of object embeddings generated
482   by the 3D Object Encoder and the embeddings of semantic relations between objects. We perform a
483   PCA analysis of Uni3D object embeddings and VL-SAT relation embeddings, comparing results for
484   ScanNet training scenes using GT instance segmentation and Mask3D instance segmentation (see
485   Appendix A). Our analysis shows that, with the standard selection of nearest neighbors, the relation
      embeddings differ significantly between GT and Mask3D three-dimensional masks.

526   5    C ONCLUSION
527
528   In this paper, we propose a new learnable approach to using a 3D semantic scene graph for a large
529   language model solving the 3D vision-language tasks. Detailed experiments demonstrate the ef-
530   fectiveness of this approach, which explicitly takes into account semantic relations between objects
531   represented as 3D point clouds. Our approach, called 3DGraphLLM, demonstrated state-of-the-art
532   quality on popular ScanRefer, Multi3DRefer, and Scan2Cap datasets.
533   A limitation of the method is a significant increase in resource consumption with an increase in the
534   edge number for each graph node. At the same time, we showed that taking into account only two
535   edges for each object demonstrates an acceptable trade-off between performance and model quality.
536
      For further development of the work, it seems appropriate to search for the methods to reduce token
537
      usage for encoding object relationships in our graph representation.. Another important aspect for
538   further work is the creation of methods for generating semantic relations between objects that are
539   robust to imperfections in the instance segmentation of the scene point cloud.
