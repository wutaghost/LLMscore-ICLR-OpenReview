011                                              A BSTRACT
012
013            Conversational recommender systems (CRS) have emerged as a key enhance-
014            ment to traditional recommendation systems, offering interactive and explain-
015            able recommendations through natural dialogue. Recent advancements in pre-
016
               trained language models (PLMs) have significantly improved the conversational
               capabilities of CRS, enabling more fluent and context-aware interactions. How-
017
               ever, PLMs still face challenges, including hallucinations—where the gener-
018
               ated content can be factually inaccurate—and difficulties in providing precise,
019            entity-specific recommendations. To address these challenges, we propose the
020            PCRS-TKA framework, which integrates PLMs with knowledge graphs (KGs)
021            through prompt-based learning. By incorporating tree-structured knowledge from
022            KGs, our framework grounds the PLM in factual information, thereby enhanc-
023            ing the accuracy and reliability of the recommendations. Additionally, we de-
024            sign a user preference extraction module to improve the personalization of rec-
025            ommendations and introduce an alignment module to ensure semantic consis-
026            tency between dialogue text and KG data. Extensive experiments demonstrate
027
               that PCRS-TKA outperforms existing methods in both recommendation accu-
               racy and conversational fluency. The code is anonymously open-sourced at
028
               https://anonymous.4open.science/r/PCRS-TKA-9496.

031   1 I NTRODUCTION
032   Recommendation systems are pivotal in intelligent assistants, helping users discover relevant items
033   more efficiently. However, traditional recommendation systems typically focus on item suggestions
034   without interactive dialogues with users (Chen et al., 2017; He et al., 2020). To address this limita-
035   tion, conversational recommender systems (CRS) have gained significant attention in recent years,
036   which enhance not only the flexibility of the recommendations but also their explainability by al-
037   lowing more natural and intuitive user interactions (Christakopoulou et al., 2016; Tran et al., 2020).
038   The recent advancements in pretrained language models (PLMs) have significantly expanded the
039
      capabilities of CRS with their powerful language understanding and generation abilities (Zhang &
      Wang, 2023; Wu et al., 2021). However, PLMs face their own set of challenges in recommenda-
040
      tion tasks, particularly hallucination, where the models may generate factually incorrect informa-
041
      tion (Hal; Zhang et al., 2023). To counter this, knowledge graphs (KGs) can be used as auxiliary
042   tools, grounding PLMs in factual external knowledge (Wang et al., 2022b; Tong et al., 2024), thus
043   enhancing both the accuracy and reliability of CRS.
044
045
      In the literature, conventional CRS primarily relied on structured conversations centered around
      item attributes such as genre or price (Gao et al., 2021a). Later, KG-based CRS (Wang et al., 2019;
046
      Petroni et al., 2019; Bouraoui et al., 2020) incorporated external knowledge resources and developed
047
      specialized alignment strategies to ensure semantic consistency. For instance, KBRD (Chen et al.,
048   2019) combined KGs with relational graph convolutional networks (RGCNs) (Schlichtkrull et al.,
049   2017) to enhance interaction between recommendation and dialogue modules. KGSF (Zhou et al.,
050   2020a) extended this by incorporating word-level KGs and using mutual information maximization
051   (MIM) to align word and entity representations. RevCore (Lu et al., 2021) enriched dialogues by
052   incorporating unstructured review data for more diverse responses, while C2CRS (Zhou et al., 2023)
053   employed multi-granularity contrastive learning to align multimodal data and improve semantic con-
      sistency. Despite these advancements, natural language capabilities remained limited. More recent


                                                        1

054
      works, such as BARCOR (Wang et al., 2022a) and UniCRS (Wang et al., 2022b), have adopted pre-
055   trained language models (PLMs) and prompt learning (Chen et al., 2022) to generate higher-quality
056   conversational responses. These PLM-based models also integrate KGs to address hallucination
057   issues and improve domain-specific knowledge.
058
      However, several key challenges remain in integrating PLMs and KGs for conversational recom-
059
      mendation. First, existing methods often rely on graph convolutional networks (GCNs), such as
060
      RGCN, to extract relational information from KGs. While effective, these approaches do not fully
061   exploit the reasoning capabilities of PLMs over graph relationships, thereby limiting the potential
062   of CRS for more sophisticated knowledge integration. Second, dialogue text in current methods is
063   often treated merely as textual input data, neglecting valuable latent user collaborative preference
064   information embedded in conversations. This oversight can result in suboptimal personalized rec-
065   ommendations that fail to align with users’ true preferences. Third, CRS typically involve multiple
066   data types, such as textual dialogue data and structured knowledge from graphs, which reside in dis-
067   tinct semantic spaces (Li et al., 2021). Without proper semantic alignment, the integration of these
068   heterogeneous data sources may introduce noise, ultimately degrading recommendation quality.
069   In response to these challenges, we propose a novel framework, PCRS-TKA (Prompt-based Con-
070   versational Recommender System with Tree-structured Knowledge Augmentation), that effectively
071   integrates KG and dialogue data through prompt-based learning using a PLM. Specifically, our ap-
072   proach first extracts a knowledge tree from the KG based on historically interacted items, leveraging
073   the PLMs’s logical reasoning abilities to capture entity relationships. Additionally, we design a user
074   preference extraction module that captures user collaborative preferences through dialogue interac-
075
      tion, guiding more personalized recommendations. Then, our framework introduces an alignment
      module that harmonizes data from different semantic spaces, minimizing noise and improving rec-
076
      ommendation accuracy. Finally, through extensive experiments on public datasets, we demonstrate
077
      that PCRS-TKA outperforms existing methods on both the recommendation and conversation tasks.

079   2     R ELATED W ORK
080   The related work can be categorized into two main areas: Conversational Recommendation Sys-
081   tems and Combining PLMs and KGs for Recommendation.
082
083
      2.1   C ONVERSATIONAL R ECOMMENDATION S YSTEMS
084   Early CRS methods (Chen et al., 2019; 2017) focused on structured conversations that gathered user
085   preferences through item attributes like genre or price, relying on predefined templates and algo-
086   rithms such as multi-armed bandits or reinforcement learning. However, these approaches lacked
087   flexibility and natural language generation capabilities. To improve this, KG-based CRS methods
088
      were developed. KBRD (Chen et al., 2019) introduced KGs and RGCNs to better connect recom-
      mendation and dialogue tasks by modeling complex relations between items and users. KGSF (Zhou
089
      et al., 2020a) extended this by incorporating word-level KGs and using MIM to align word and
090
      entity representations, resulting in more coherent responses. RevCore (Lu et al., 2021) enriched
091   dialogue generation by leveraging unstructured review data, adding diversity to system responses,
092   while C2CRS (Zhou et al., 2023) used multi-granularity contrastive learning to align multimodal
093   data (e.g., text, images, KG). Despite these advancements, KG-based methods often treated recom-
094   mendation and dialogue modules separately, limiting their ability to fully utilize dialogue content.
095   With the rise of PLMs, prompt learning has been introduced to improve conversational capabili-
096   ties. For example, BARCOR (Wang et al., 2022a) employs BART (Lewis et al., 2020) to generate
097   higher-quality responses, while UniCRS (Wang et al., 2022b) uses prompt learning to integrate rec-
098   ommendation and dialogue generation. These PLM-based systems also incorporate KGs to mitigate
099
      hallucination issues and improve domain-specific knowledge, providing better alignment between
      dialogue and recommendations.
100

106   feature representations. This is commonly achieved by modifying the Transformer architecture with
107   cross-attention mechanisms, enabling the model to process both dialogue text and KG information
      simultaneously. For instance, works like KGSF (Zhou et al., 2020a) use mutual information maxi-

137   3      M ETHODOLOGY
138   In this section, we will first introduce our task formulation and the overview of the proposed PCRS-
139   TKA framework. Then we will provide the technical details of our proposed framework.
140
141
      3.1         TASK F ORMULATION
142   The goal of CRS is to recommend relevant items while engaging in a natural, ongoing conversation
143   with the user. During each turn of the conversation, the system analyzes the dialogue history, infers
144   the user’s preferences, and generates a response that includes recommended items within the natural
145   language utterance. If the recommended items do not meet the user’s needs, the system continues
146
      the conversation in subsequent turns, refining the recommendations based on the user’s feedback
      and evolving preferences.
147
148   Formally, let u denotes a user from user set U, i denotes an item from item set I, and w denotes a
149   word from vocabulary V. A multi-turn conversation C consists of a set of utterances, denoted by
150   C = {st }nt=1 , in which st denotes the sentence in the t-th turn and composed of words from the
151   vocabulary V, denoted as st = {wj }m   j=1 . For a given n-turn conversation C, the task of CRS is to
152
      generate a response utterance sn+1 and select a set of recommended items In+1 from the item set I,
      where In+1 can be empty. We denote the externeal KG as G . It stores a semantic fact with a triple
153
      < e1 , r, e2 >, where e1 , e2 from the entity set E and r from the relation set R, here we assume all
154
      the candidate items can be find in E, i.e.I ⊂ E.
155
156   3.2         OVERVIEW OF THE A PPROACH
157   As shown in Figure 1, we propose the PCRS-TKA framework for CRS, which integrates PLMs
158   and KGs through knowledge-enhanced prompt learning. For a given dialogue text C, we previously
159   extract the KG entities that appear in C, and denote as E. Then the framework consists of four main
160   components: (1) Feature Encoder Module: We employ RoBERTa (Liu et al., 2019) and RGCN
161   (Schlichtkrull et al., 2017) to encode dialogue text C and the KG G. The dialogue embeddings C
      and entity embeddings E (extracted from G) are fused to form the first part of the prompt, denoted


                                                                                                                                       3
      Under review as a conference paper at ICLR 2025


162
      as Pfeature . E is used for recommendation, while C is used for dialogue generation. (2) Knowl-
163   edge Tree Enhanced Module: We transform the KG triples related to entities E into a knowledge
164   tree, which is then converted into a long text T . RoBERTa encodes this text, and the resulting em-
165   beddings T are aligned with E to form the second part of the prompt, Ptree . (3) User Preference
166   Extraction Module: This module infers user preferences through an auxiliary recommendation
167   task, generating a preference matrix U. Fused with E, this forms the third part of the prompt, Puser .
168   (4) Soft-Prompt Module: A learnable soft prompt Psoft is appended to the end of the prompt for
169   additional task-specific guidance. Since the PLM remains fixed during prompt learning, our focus is
170   on constructing a knowledge-enhanced prompt template to integrate different information. Finally,
171
      all components of the prompt (Pfeature , Ptree , Puser , Psoft ) are concatenated with the dialogue text C
      and fed into DialoGPT for conversation and recommendation tasks.

173   3.3   F EATURE E NCODER M ODULE
174
      For natural language understanding, we adopt RoBERTa (Liu et al., 2019), a bidirectional pre-trained
175   language model, to encode the dialogue text C and generate word embedding matrix C, where
176   C ∈ RnW ×dW , nW is the number of words in the dialogue text and dW the hidden size of RoBERTa.
177   RoBERTa excels at capturing the contextual meaning of words, making it suitable for understanding
178   user intents from conversation.
179
      Furthermore, since pre-trained language models lack domain-specific knowledge, we incorporate a
180
      KG G to provide external knowledge about entities mentioned in the conversation. KGs contain rich
181   relational information. To encode this information, we use RGCN, which learns the representations
182   of entities in KGs. By aggregating information from neighboring nodes, we obtain the embedding
183   matrix of all entities in G, and denote it as G , where G ∈ RnG ×dE , nG is the number of entities
184   in G, i.e.the size of the entity set E, nE is the hidden size of RGCN. The entity embeddings corre-
185   sponding to E (i.e., entities mentioned in C) are retrieved from matrix G and denoted as E, , where
186   E ∈ RnE ×dE , nE is the number of entities in E.
187   Dialogue text typically captures user preferences and intents, while entity embeddings from the
188   R-GCN provide external knowledge about entities. However, there is a significant semantic gap
189   between the two modalities. To bridge this gap, we apply a cross-interaction mechanism (Wang
190   et al., 2022b), which aligns the semantic spaces of the word and entity embeddings:

      3.4   K NOWLEDGE T REE E NHANCED M ODULE
197
198   To fully leverage the information from the KG G, we propose to integrate KG entities into the orig-
199   inal dialogue context by constructing a tree-structured knowledge tree. This structure, as illustrated
200
      in Figure 2, enhances the dialogue by embedding relevant background knowledge, taking advantage
      of PLM’s ability to process and analyze complex semantic structures.
201
202   Specifically, given the dialogue C and the mentioned entities E, we begin by selecting these entities
203   as seed nodes and extract their n-hop triples from the KG G. These triples are then reorganized
204   into t n-layer trees. To unify these t trees, we introduce a common parent node, constructing a
205   comprehensive knowledge tree Gtree corresponding to the entities in E. To preserve the relational
206
      structure of Gtree , we represent the edges between consecutive layers of nodes as virtual nodes.
      By performing a depth-first traversal of the entire tree, we generate an ordered list of nodes in the
207
      form of “#Moana (2016)”, where “Moana (2016)” is the text of the node, and “#” represents the
208
      node’s depth in the tree (e.g., depth 1). This depth marker ensures that the tree structure can be
209   fully preserved and easily reconstructed from the text list. Finally, we concat the node text in the
210   pairs in order to get the equivalent text of the knowledge tree, and input them to the RoBerta to
211   get the embedding matrix T for the knowledge tree, where T ∈ RnT ×dW , nT is the length of the
212   knowledge tree text.

328   4     E XPERIMENTS
329   In this section, we first introduce the datasets and experimental settings. Next, we present the over-
330   all evaluation of our framework, comparing it against several state-of-the-art baselines on both
331   the recommendation and conversation tasks. Finally, we provide detailed analyses through ab-
332   lation studies and sensitivity experiments for key hyperparameters. The source code and data
333   have been uploaded to an anonymous repository: https://anonymous.4open.science/
334   r/PCRS-TKA-9496.
335   4.1   E XPERIMENTAL S ETUP
336
      Dataset. We conducted multiple experiments on the REDIAL (Li et al., 2019) and INSPIRED (Hay-
337   ati et al., 2020) datasets. The REDIAL dataset is a conversational dataset for movie recommen-
338   dations, containing 10,006 dialogues with 182,150 utterances about recommendations for 6,281
339   movies. INSPIRED is a dataset containing 1001 dialogues with 35,811 utterances about recommen-
340   dations for 1472 movies. Both datasets are constructed by Amazon Mechanical Turk (AMT). In the
341   experiment, we split the datasets into training, validation, and test sets in an 8:1:1 ratio. For each
342   dialogue, we incrementally added one round of utterances starting from the first round to create new
343   data, thereby expanding the dataset.
344   Knowledge Graph. DBpedia (Auer et al., 2007) is a large-scale, multilingual KG extracted from
345   the structured content of Wikipedia. It represents real-world entities and their relationships, such as
346   people, places, and organizations, along with their attributes, containing 5,040,986 high-frequency
347   entities with their corresponding 927 relations and 24,267,796 triplets. For the entire DBpedia graph
348   is too huge, we collected all the entities appearing in the dataset corpus via the Tagme tool. Start-
349   ing from these entities as seeds, we extracted their one-hop triples on the DBpedia graph, and the
350   subgraph obtained is used as the external KG in our experiment.
351   Evaluation Metrics. We conducted two types of evaluations: recommendation evaluation and dia-
352   logue evaluation. For the recommendation task, we used recall@k (k=10, 50), NDCG@k (k=10,
353   50), and MRR@k (k=10, 50) as metrics. For the conversation task, we employed both auto-
354   matic and manual evaluations. For automatic evaluation, we used word-level distinct-n (n=1,
355   2, 3, 4) to measure response diversity. Additionally, we randomly selected 100 conversations
356   and their corresponding model-generated responses, and invited ten annotators to manually score
357
      the responses. The manual evaluation assessed three aspects: Fluency, Informativeness, and
      Question-Answer Consistency, with scores ranging from 0 to 5. The details of the manual eval-
358
      uation, along with the original survey data, have been uploaded to an anonymous repository:
359
      https://anonymous.4open.science/r/PCRS-TKA-9496.
360
361   Benchmark Models. We selected several state-of-the-art recommendation models as baselines,
362   including: ReDial (Li et al., 2019): This model was introduced with the Redial dataset. It inte-
363
      grates an autoencoder-based recommendation module and an HRED-based conversation module.
      KBRD (Chen et al., 2019): This model utilizes DBpedia to enhance the semantic representation of
364
      entities, combining a self-attention-based recommendation module with a Transformer-based con-
365
      versation module. KGSF (Zhou et al., 2020a): It integrates ConceptNet (Speer et al., 2018) and
366   DBpedia to enhance word and entity representations, and uses mutual information maximization
367   to align the semantic spaces of both KGs. TG-ReDial (Zhou et al., 2020b): This model intro-
368   duces a topic prediction task, employing SASRec (Kang & McAuley, 2018) for recommendation, a
369   BERT (Devlin et al., 2019) encoder for topic prediction, and GPT-2 (Gao et al., 2021b) for response
370   generation. UniCRS (Wang et al., 2022b): This model uses prompt learning to guide a pretrained
371   large language model for both recommendation and conversation tasks, enhancing the semantic rep-
372   resentation of entities with DBpedia.
373   Implementation Details. We chose the DialoGPT-small model as the base PLM for prompt learning
374   and used the RoBERTa-base model as the encoder for text, freezing all parameters of these two
375   modules during the entire training process. We have used grid search to choose the hyperparameters.
376   After searching, we used AdamW with epsilon set to 0.01, learning rate set to 5e-4 for first-stage
377   pre-training, and 1e-4 for second-stage training for both recommendation and conversation tasks.
      The batch size was set to 64 for the recommendation task and 8 for the conversation task. The soft

462   4.4     A BLATION S TUDY
463   Recommendation Task. We performed ablation experiments to evaluate the contribution of each
464   component in our approach across both datasets. Specifically, we excluded the knowledge tree-
465   enhanced prompts, user preference prompts, and the information alignment module, separately dur-
466   ing both the pre-training and training phases. As shown in Figure 3, removing any component leads
467   to performance degradation, confirming that all components are crucial for improving the recom-
468   mendation task. Notably, removing the alignment mechanism caused the most significant drop,
469   highlighting its critical role. This is likely because entity embeddings E play a central role in the
470   recommendation task, and the alignment module helps entities capture and integrate information
471
      more effectively, resulting in better representations for recommendations.
472   Conversation Task. We conducted similar ablation experiments for the conversation task, remov-
473   ing the knowledge tree-enhanced prompts, user preference prompts, and the information alignment
474   module one by one in both the pretraining and training stages. Figure 4 shows that removing any
475   component results in a performance drop, indicating that all components contribute positively to the
476   conversation task. The most significant decline occurred when removing the tree structure, demon-
477
      strating its importance in the conversation task. This is because the tree structure provides rich
      entity-related information from the KG, which is effectively utilized by large language models to
478
      enhance conversational quality.

532   5            C ONCLUSION
533
      In this paper, we presented PCRS-TKA, a novel framework that integrates pretrained language mod-
534   els with KGs for conversational recommender systems. Our approach enhances the recommendation
535   process through tree-structured knowledge augmentation and user preference extraction from multi-
536   turn dialogues, improving both recommendation accuracy and conversational fluency. Addition-
537   ally, the alignment module ensures effective integration of heterogeneous data from dialogues and
538   KGs, minimizing semantic inconsistencies and noise. Finally, extensive comparative experiments
539   on benchmark datasets clearly demonstrated the effectiveness of the proposed model compared to
      state-of-the-art baselines in terms of both recommendation quality and dialogue generation.

      A.1             EXPERIMENTAL SETUP
705
706   Except for UniCRS, the baseline models are implemented using the CRSLab tool (Zhou et al., 2021).
707   CRSLab is an open-source toolkit for building CRS, providing ready-made code where users only
708   need to modify configuration files to run the systems in the tool. Furthermore, CRSLab has already
709   set default values for model-specific parameters, such as the token hidden size of transformer mod-
710   ule and the hidden size of RGCN module in KGSF, based on the code from the baseline repository
711
      papers, and has validated the correct selections of parameters on the TG-Redial dataset. So users
      only need to adjust some common parameters according to the dataset, including hidden size, num-
712
      ber of training epochs, batch size, and optimizer configurations. For these parameters, we opted for
713
      a grid search approach, choosing training epochs from [5, 10, 20, 50], batch size from [8, 16, 32, 64,
714   128], and learning rates from [1e-4, 5e-4, 1e-3, 5e-3]. Both the recommendation and task modules
715   use an early stopping strategy with the endurance epoch set to 3. After numerous experiments, we
716   identified the best-performing parameter settings and uploaded the configuration file to the codebase.
717   For the UniCRS task, we used the code and parameter configuration provided by the repository of
718   UniCRS ar https://github.com/rucaibox/unicrs. We examined the evaluation met-
719   rics in all mentioned codebases and confirmed that the implementations of all evaluation metrics are
720   identical. We eventually achieved results similar to those reported in the corresponding paper,
