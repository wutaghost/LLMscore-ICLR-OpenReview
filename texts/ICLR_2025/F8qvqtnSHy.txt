009                                                     A BSTRACT
010
011             In many causal learning problems, variables of interest are often not all measured
012             over the same observations, but are instead distributed across multiple datasets
013             with overlapping variables. Tillman et al. (2008) presented the first algorithm for
014             enumerating the minimal equivalence class of ground-truth DAGs consistent with
015             all input graphs by exploiting local independence relations, called ION. In this
016             paper, this problem is formulated as a more computationally efficient answer set
017             programming (ASP) problem, which we call ION-C, and solved with the ASP
018
                system clingo. The ION-C algorithm was run on random synthetic graphs with
                varying sizes, densities, and degrees of overlap between subgraphs, with overlap
019
                having the largest impact on runtime, number of solution graphs, and agreement
020
                within the output set. To validate ION-C on real-world data, we ran the algorithm
021             on overlapping graphs learned from data from two successive iterations of the Eu-
022             ropean Social Survey (ESS), using a procedure for conducting joint independence
023             tests to prevent inconsistencies in the input.

026   1    I NTRODUCTION
027
028   Many inference problems require the use of data from different sources. Ideally, these data can be
029   merged and collected into a single unified dataset (e.g., in tabular form) that is suitable for most
030   learning methods. However, this type of data merging is not always possible. For example, suppose
031   we have two distinct datasets, one from a financial institution and one from a healthcare provider. We
032   might reasonably suspect that information about health outcomes and financial outcomes are related
033   to one another; that is, we might want a unified model over these datasets. In practice, though, these
034   datasets almost certainly cannot be integrated together for privacy reasons. Even worse, the datasets
035
      might be about different samples (even if from the same population), preventing us from directly
      linking observations from each dataset. At the same time, we might be able to leverage the variables
036
      that are measured in both datasets, such as someone’s age, postal code, and so forth. We thus aim
037
      to learn about relationships between variables that are not co-measured in any dataset (existing or
038   integrated), but where there are some variables that are measured in multiple datasets.
039
040   Formally, we examine a method for enumerating the complete set of ground-truth graphs Hi ∈ H
041
      consistent with a set of input graphs Gi ∈ G, each learned locally from a source dataset.1 The
      first algorithm for solving this problem, Integration of Overlapping Networks (ION) (Tillman et al.,
042
      2008), used a constructive solution that iterated through sets of changes to the complete graph that
043
      were faithful to independence relations in the input graphs. However, this formulation was compu-
044   tationally expensive, and only able to be tested on 4- and 6-node ground-truth graphs. In this work,
045   we present a more efficient answer set programming formulation that, when solved, yields the same
046   output set of graphs as ION; we call this ION-C, or ION via Constraints.
047
      In Section 2, we describe previous approaches to learning from data distributed across datasets.
048
      Section 3 presents and explains the answer set programming formulation of ION-C. In Section 4,
049
      we provide evaluation results for ION-C for a range of synthetic input graphs. In Section 5, we
050   evaluate ION-C on real-world data from two iterations of the European Social Survey. In Section 6,
051   we discuss limitations and potential extensions of the ION-C algorithm.

      2   R ELATED W ORK

      Most structure learning methods (causal or otherwise) have focused on learning from a single
057
      dataset. As a result, there has been significant work on methods to unify datasets involving dis-
058
      tinct variable sets (i.e., some variables are never co-measured) so that existing methods can be used.
059   Most notably, since the 1960s, statistical matching approaches match individual observations from
060   each dataset to observations from other datasets on the basis of distance in the co-measured features
061   (Budd & Radner, 1969; Okner, 1972). These matches provide the basis for either imputations of
062   unobserved variable values, or other statistical information connecting non-comeasured variables
063   (Leulescu & Agafitei, 2013).
064
      Traditional statistical matching approaches are only provably reliable when non-overlapping vari-
065   ables from each input dataset are conditionally independent of one another given the overlapping
066   variables. More precisely, in the two dataset case where D1/2 is over V1/2 ∪ Vc , these methods
067   assume that V1 ⊥ V2 |Vc . This assumption is both rarely true in practice, and also untestable given
068   only the input datasets (Sims, 1972; Rodgers, 1984). While methods to overcome this conditional
069   independence assumption exist, they usually require the provision of additional data (Paass, 1986;
070   Singh et al., 1993), or the existence of informational proxy variables (Zhang, 2015).
071
      Federated learning (FL) methods also aim to combine distinct information sources. In this case, we
072   typically aim to learn a single model (at a central server) from multiple data sources, ideally without
073   exchanging any observations and without assuming i.i.d. data across the different sources (Kairouz
074   et al., 2021). In typical “horizontal” FL problems, each data source contains a partition of obser-
075   vations over a shared feature space; in “vertical” FL, data sources contain different features about
076   shared observations (Wei et al., 2022). Some vertical FL methods also require sample alignment
077   between data sources via cryptographic communication protocols (Lu & Ding, 2020). Federated
078   transfer learning approaches aim to find single central models learned from information sources
079   with both different sets of features and different observations, but typically with some small overlap
080
      in observations (Liu et al., 2020; Sharma et al., 2019).
081   While there are some high-level similarities, FL approaches inhabit a different problem space to the
082   ION problem, since they seek efficient learning of a single best model rather than the full space of
083   possible models given the data. They also are typically designed for distributed learning where a
084   unified dataset could (in theory) be constructed. As a result, they usually face constraints of privacy
085   and information flow that do not arise in our setting.
086   This paper is most directly related to Tillman et al. (2008), which presented an asymptotically correct
087   algorithm that outputs the equivalence class of directed acylic graphs (DAGs) consistent with an
088   input set of partial ancestral graphs (PAGs). Their Integration of Overlapping Networks (ION)
089   algorithm starts with a complete graph, then encodes edge absence and orientation information
090   from each input PAG, including propagation of all entailments Zhang (2007). ION then finds all
091   minimal sets of changes that would block paths between variables that are d-separated in at least one
092
      input PAG. These minimal changes are applied and propagated, and the resulting graph is accepted
      if it does not contradict the input PAGs. Finally, additional edge removals are tested to discover
093
      additional valid graphs. ION was shown to be both complete and sound, but is NP-complete and
094
      requires a superexponential number of operations. As a result, Tillman et al. (2008) were only able
095   to run ION on 4- and 6-node ground-truth graphs.
096
097   ION takes PAGs as input; Tillman & Spirtes (2011) developed the Integration of Overlapping
098
      Datasets (IOD) algorithm that takes datasets as input. Their approach is closely related to the orig-
      inal ION algorithm, except that independence and association information is derived from p-value
099
      pooling over multiple datasets, rather than inferred from the input PAGs. IOD requires less mem-
100
      ory than ION, and also outperformed ION in precision and recall, largely because IOD smoothly
101   resolves (statistical) inconsistencies between input datasets.
102
103   Boolean satisfiability (SAT) solvers have also been applied to versions of this problem. Triantafillou
104
      et al. (2010) used a SAT solver to find a single graph that encodes all possible pairwise causal
      relationships between variables. Hyttinen et al. (2013) used a SAT formulation of d-separation to
105
      discover cyclic causal models from a set of overlapping input graphs.
106
107   Our approach uses answer set programming (ASP), a declarative problem-solving framework
      in which logical rules are provided to describe solution conditions for the problem (Marek &


                                                        2
      Under review as a conference paper at ICLR 2025


108
      Truszczyński, 1999; Gelfond & Lifschitz, 1988). Relative to other problem-solving methods, ASP
109   benefits from a simple problem formulation and high expressiveness (Eiter et al., 2009; Brewka
110   et al., 2011), while leveraging optimization of the boolean SAT problem (Gebser et al., 2007). ASP
111   has been used to encode other causal learning problems Sonntag et al. (2015); Rantanen et al. (2020).
112   For example, Hyttinen et al. (2014) used ASP to represent causal discovery as an optimization prob-
113   lem, providing a set of dependence and independence relations with weights corresponding to their
114   probabilities, and returning the optimal causal graph according to these weights.

      The problem that ION and ION-C aim to solve is to determine the complete set of ground-truth
119   DAGs over all variables (that appear in at least one dataset) that are consistent with a set of overlap-
120   ping input graphs. More formally: our inputs are a set of partial ancestral graphs (PAGs) Gi ∈ G,
121   such that every graph Gi shares at least one node with at least one other graph in the set (and these
122   overlaps for a connected structure; see footnote 1). Importantly, although all output graphs are
123   DAGs, the input graphs do not have to be DAGs. In this problem, there are known latent variables
124   for every input graph (namely, variables that are only in a different graph). Some of those latents
125   could be common causes, which produce bidirected edges in the input PAG.
126   The output is a complete set of solution graphs H, where each graph Hi ∈ H is a DAG containing
127   the union of all nodes in every input graph Gi , such that each Hi does not violate any of the local
128   independence or association information encoded in the input graphs. Specifically, this means that
129   all d-separation and d-connection relations in every input graph Gi are preserved in every Hi .
130
      As a concrete example, suppose that G1 = X → Y → Z and G2 = X → W → Z. Exactly two
131   graphs (over {W, X, Y, Z}) preserve the d-separation and d-connection relations in these graphs:
132   H = {X → Y → W → Z, X → W → Y → Z}. Interestingly, in this example, we can learn that
133   there must be a direct connection between Y and W (but not orientation of the edge), even though
134   Y and W are never jointly measured.
135
      In this paper, we present an answer set programming formulation of the integration of overlapping
136
      networks problem, which is implemented in the ASP system clingo (Gebser et al., 2019), based on
137   the solver clasp (Gebser et al., 2007). We define the ION problem by providing the graph as a set of
138   facts, then define a set of rules that must hold in any valid solution. clingo then outputs the set of all
139   possible graphs that follows all of these facts and rules (see Listing 1).
140
      The input PAGs are specified through sets of statements involving three different predicates:
141
142
            1. edge(X,Y,T)., denoting an edge from node X to node Y in input PAG T
143
144         2. bidirected(X,Y,T)., denoting a bidirected edge between X and Y in T
145         3. nedge(X,Y,T)., denoting absence of an edge in either direction between X and Y in T
146
147   We additionally explicitly indicate all nodes in PAG T with the command varin(T, X).. Finally,
148   we provide the number of subgraphs and nodes as constants, and define all nodes with the command
149
      node(0..n).
150   Listing 1 describes the problem specification in a format suitable for clingo. Line 1 defines any set
151   of edge declarations between nodes as a valid solution. Lines 3 through 5 specify constraints for the
152   solution: (3) self-loops are not allowed; (4) if an edge is absent in some input graph, then it cannot
153   appear in a solution;2 and (5) a valid solution must be acyclic. Lines 7 and 8 recursively define a
154   directed path from Y and X. Lines 10 and 11 provide a recursive definition of a directed edge from
155
      X to Y relative to the input graph T . Such an edge could be explained by a direct edge in the output
      graph, and also by a directed path that involves only nodes that do not appear in T (since such a
156
      path would be an edge in T ). Lines 13 and 14 define a causal connection between nodes X and Y
157
      in input graph T as a directed edge between nodes, or an unobserved common cause of both nodes.
158   Line 15 states that a bidirected edge in the input graph T implies a causal connection between nodes
159   without a directed edge in the solution graph, due to an unobserved common cause.

188   In order to show that the ION-C ASP formulation leads to the correct output equivalence class, we
189
      show that the problem statement is complete and sound.
190   Theorem 3.1. Soundness: If nodes X and Y are d-separated (d-connected) given nodes Z in some
191   Gi ∈ G, then X and Y are d-separated (d-connected) given Z in every output Hi ∈ H.
192
193   Proof. Suppose X and Y are d-separated given Z in some Gi , but d-connected in some output
194   Hi . This implies that there is a path between X and Y in Hi that is active given Z. X and Y are
195   not adjacent in Gi , and so (by line 17) the output graph d-connection cannot be a directed path or
196   common cause. The only remaining possibility is that some variable in R ∈ Z is a descendant of a
197
      collider in Hi on a path between X and Y . This implies, however, that Hi includes paths from X to
      R and Y to R that are active given Z R. However, this implies (per lines 10-11) that each of these
198
      paths corresponds to a sequence of edges in Gi that contradict the known d-separation in Gi .
199
200   Now suppose that X and Y are d-connected given Z. Line 18 specifies that if an edge exists between
201   two nodes X and Y in input graph T , then the property directed(X,Y,T) is true. Per lines 10
202   and 11, directed(X,Y,T) holds true only when there is an edge from X to Y in the output, or
203
      when the solution includes multiple edges from X to Y consisting of intermediate nodes that were
      not observed in graph T . This means that any pair of nodes connected by an edge in an input T will
204
      be connected either by a single edge, or by a directed path of nodes that were not included in T .
205
      This, in turn, entails the necessary d-connection relation.
206
207   Theorem 3.2. Completeness: Let Hi be a partial ancestral graph over variables V such that for
208   every {(X, Y )} ⊆ V, if X and Y are d-separated (d-connected) given Z ⊆ V/{X, Y } in some
209   Gi ∈ G, then X and Y are d-separated (d-connected) given Z in Hi . Then, Hi is in H.
210
211   Proof. In order to show completeness, we must show that no d-separations or d-connections present
212   in the input graph are unnecessarily removed from the output set H. All edge removals in line
213   4 are necessary to translate d-separations from the inputs, as is the acyclicity constraint in line 5.
214   Remaining edge removals only occur in line 14 and 15 by removing bidirected edges X ↔ Y and
215   retaining the relevant d-connections by creating directed paths to X and Y from the unobserved
      common cause, or in line 11 to replace a directed edge X → Y with a previously unobserved path

217   constraints, and because none of the changes specified by these constraints would preclude such an
218   output Hi from the solution set, ION-C is complete for the problem.

      4   S IMULATION R ESULTS
223
224   In Tillman et al. (2008), the ION algorithm was only evaluated on 4- and 6-node directed acyclic
225   graphs (DAGs) due to computational constraints. In order to establish the usability of the ION-C al-
226   gorithm on larger graphs with the faster ASP formulation (and additional computational resources),
227   we tested ION-C on graphs of varying sizes, densities, and overlap between subgraphs.
228
      We randomly generated “ground truth” graphs using four control parameters: (i) the total number
229   of nodes N ; (ii) pdegree that controls ground-truth density; (iii) the number of input subgraphs s;
230   and (iv) poverlap that controls the extent of input graph overlap. More precisely, each ground-truth
231   graph was generated with N nodes, and random edges such that each node makes connections to
232   a other nodes, with a ∼ Bin(N − 1, pdegree ). As pdegree increases, more connections are made,
233   and ground-truth graphs are denser. Finally, we check that the DAG is connected, and add required
234   edges to connect the graph if not. To generate input subgraphs, we first split the nodes evenly
235   into s partitions, and for each partition set, we sample poverlap of the nodes from other partitions.
236   As poverlap increases, each subgraph will contain more nodes, and the level of overlap between
237
      subgraphs will increase.
238   Given the ground-truth graph and a subset of nodes, we analytically generate the input subgraph
239   by marginalizing out the variables not in the subset. The resulting input PAG is provably causally
240   faithful to the ground-truth. For example, if the ground-truth contains X → Z → Y but the
241   subgraph does not include Z, then the input PAG will have X → Y . In addition, we connect nodes
242   X ↔ Y if they share a common cause that is not observed in that subgraph. Given a set of input
243
      PAGs for a single ground-truth graph, we convert the inputs (as described in Section 3) and run the
      ASP solver to find the full set of possible ground-truth graphs consistent with the input graphs.
244
245   We ran 100 simulated ground-truth graphs for each possible combination of parameters, with N ∈
246   {6, 8, 10, 15, 25}, poverlap ∈ {0.25, 0.5, 0.75}, pdegree ∈ {0.1, 0.25, 0.5, 0.75}, and S ∈ {2, 3, 4}.
247   For graphs with 15 and 25 nodes, due to the high complexity of denser graphs, we additionally used
248   pdegree values of 0.025, 0.05, and 0.075. In total, we considered 234 sets of 100-graph simula-
249
      tions. All instances were run with four-hour timeouts for the clingo solver on nodes with 24 GB of
      RAM. We only report results for parameterizations that resulted in at least 95 of 100 ground-truths
250
      completing (and all reported proportions are relative to the completed runs). 153 parameterizations
251
      resulted in completion of at least 95 of 100 output solution sets.
252
253   For each simulation, we initially report two key statistics. First, prop same is the proportion of all
254   possible edges or edge absences that are shared across 75%, 90%, and 100% of the solution set. This
255
      statistic provides a measure of the similarity of graphs in the solution set. Second, prop accurate
      indicates, as a proportion of the edges/absences shared in 75%, 90%, or 100% of the solution set,
256
      what proportion are found in the ground-truth graph itself (ignoring orientation). This statistic pro-
257
      vides a measure of the “accuracy” of the output set: are the most common edges/absences correct?
258   Complete results for all parameterizations are available in Appendix A. Figures 1, 2, and 3 show
259   these statistics for all 8-node graphs, for which all graphs ran at all parameterizations. Tables 1 and
260   2 display prop same and prop accurate for completed parameterizations among 15- and 25-node
261   graphs with two subgraphs.
262
      As expected, the most important factor controlling the number of output graphs, and consequently
263
      the runtime of the algorithm, was the amount of overlap between the input subgraphs. For example,
264   in 8-node ground-truth graphs with pdegree = 0.75 with two subgraphs (the rightmost set of bars in
265   the left graph in Figure 3), the three settings of overlap corresponded to two subgraphs with 5, 6, and
266   7 nodes each. The median number of solution graphs was 25648, 161, and 5, respectively. (In many
267   settings with poverlap = 0.75, there was only one valid solution graph.) The degree of overlap in the
268   graphs is also the largest factor in the coherence of the output set; as Figure 1 indicates, proportion
269   of edge adjacencies or absences that is shared across 90% of the solution set is closely related to the
      overlap in nodes.

362   5   A PPLICATION TO R EAL - WORLD DATA
363
364   In order to examine the real-world performance and utility of ION-C, we use data from rounds 8
365   and 9 of the European Social Survey (ESS), from years 2016 and 2018, respectively (ERIC, 2017;
366   2019). The ESS survey, conducted every two years, asks participants a core set of questions in every
367   survey, in addition to a rotating set of topical modules that vary in each iteration. Rotating modules
368
      not asked in the same survey round are thus not co-measured, but ION-C can potentially be used to
      enumerate possible ground-truth graphs based on graphs learned within each survey round.
369
370   We selected 8 variables from the ”welfare attitudes” module from ESS round 8; 8 from the ”justice
371   and fairness” module from ESS round 9; and an overlap group of 8 variables that were measured in
372   both survey rounds. We suspected that there might be connections between participants’ attitudes
373   about the round-specific topics; for example, someone who is particularly concerned about fairness
374
      might plausibly want a strong, supportive welfare system.
375   We learn causal graphs for each survey round using the PC algorithm (Spirtes et al., 2001), allow-
376   ing for missing data using the method in Tu et al. (2019) implemented in the causal-learn Python
377   package (Zheng et al., 2024). (Missing values correspond to nonresponses, refusals, and other non-

425   6   D ISCUSSION
426
427   While the output of the ION-C algorithm is provably correct—that is, it returns all possible ground-
428   truth graphs consistent with the input—there are limitations to this approach as a methods of causal
429   discovery given overlapping graphs. Just as with the constructive formulation of ION in Tillman
430   et al. (2008), contradictory information in input datasets, whether due to differences in underlying
431   distributions in the data or statistical errors in the causal discovery process, can make the constraint
      formulation unsatisfiable, with no possible ground truths satisfying this conflict. The number of


                                                         8
      Under review as a conference paper at ICLR 2025


432
      conditional independence tests required in the PC algorithm is potentially super-exponential in the
433   number of variables, and therefore the likelihood of mistaken edge commissions, deletions, or ori-
434   entations drastically increases as dimensionality increases.
435
436
      Not only can statistical errors lead to unsatisfiable ION-C problems, but if statistical errors occur in
      multiple input graphs, it is possible for ION-C to return a solution set that, while valid for the input
437
      graphs as stated, is inaccurate to the ground truth. Potential methods for improving such errors in-
438
      clude the p-value pooling approach outlined in Tillman & Spirtes (2011), which ensures consistency
439   in the causal structures over the overlapping nodes. Another option is to find the closest satisfiable
440   set of graphs to the input set, using a metric like the structural Hamming distance (Tsamardinos
441   et al., 2006) to compare to the original input. This latter approach will find valid ground-truths
442   that require the fewest changes to the provided input graphs, even if the learned causal graphs are
443   inconsistent with each other.
444
      An additional limitation is in the interpretation of the output equivalence class of graphs. As seen
445   in the results, these sets can range into the tens or hundreds of millions of graphs, even given rela-
446   tively small input graphs. Of course, these large output sets are still much smaller than the super-
447   exponential number of n-node DAGs, but large output sets might have limited real-world utility.
448
      In this paper, we use the proportion of the solutions in which a given edge or edge absence appears
449
      as a sort of ad-hoc confidence metric; for example, a node that appears in 90% of the solution set
450
      is very likely to be present in the ground truth. This is not entirely baseless – ION provides all
451   possible graphs consistent with the input, and barring input errors, the actual ground-truth is one of
452   these graphs. Therefore, if we start with a flat prior over possible global graphs, then this measure
453   accurately describes the likelihood of output graphs in our beliefs.
454
      Indeed, in our results, we found that edges or edge absences that were in large proportions of the
455
      output set were very likely to be accurate. However, in order to more clearly determine the single
456
      ground truth, additional information or experiments would be needed to disambiguate ION-C so-
457   lution set graphs. In this way, ION-C could serve to indicate edges of interest that are likely, but
458   not certain to exist, or indicate edges that the solution set has high disagreement over, allowing an
459   intervention on these edges to most efficiently cut down the set of possible ground truths as part of
460   an experimental process. In Section 5, for example, we saw that the ION-C output, with a solution
461   size in the thousands, involves disagreement over only a small number of edges, highlighting which
462   variables and relationships we do not currently have the information to understand.
463   Even without leveraging other information, there are potentially other methods or assumptions that
464   could help to deal with the size the ION-C solution set. To provide one example, suppose two
465   potential ground-truth graphs H1 and H2 are returned by ION-C, where the edges in H1 are a
466   proper subset of those in H2 . We might make a simplicity assumption that leads us to focus on H1 ,
467   the graph with fewer causal connections. In this fashion, by leveraging additional assumptions or
468   requirements from the data, we can take the often very large solution set returned by ION-C and
469   reduce it into more useful constructs for analysis.
