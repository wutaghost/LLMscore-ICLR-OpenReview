009                                                A BSTRACT
010
011            Open-vocabulary multi-object tracking (OVMOT) represents a critical new chal-
012            lenge involving the detection and tracking of diverse object categories in videos,
013            encompassing both seen categories (base classes) and unseen categories (novel
014            classes). This issue amalgamates the complexities of open-vocabulary object de-
015            tection (OVD) and multi-object tracking (MOT). Existing approaches to OVMOT
016
               often merge OVD and MOT methodologies as separate modules, predominantly
               focusing on the problem through an image-centric lens. In this paper, we pro-
017
               pose VOVTrack, a novel method that integrates object states relevant to MOT and
018
               video-centric training to address this challenge from a video object tracking stand-
019            point. First, we consider the tracking-related state of the objects during tracking
020            and propose a new prompt-guided attention mechanism for more accurate local-
021            ization and classification (detection) of the time-varying objects. Subsequently,
022            we leverage raw video data without annotations for training by formulating a
023            self-supervised object similarity learning technique to facilitate temporal object
024            association (tracking). Experimental results underscore that VOVTrack outper-
025            forms existing methods, establishing itself as a state-of-the-art solution for open-
026            vocabulary tracking task. We have included the source code in the supplementary
027
               material.

029   1   I NTRODUCTION
030
031   Multi-Object Tracking (MOT) is a fundamental task in computer vision and artificial intelligence,
032   which is widely used for video surveillance, media understanding, etc. In the past years, plenty of
033   datasets, e.g., MOT-20 (Dendorfer et al., 2020), DanceTrack (Sun et al., 2022), KITTI (Geiger et al.,
034   2012), as well as the algorithms, e.g., SORT (Wojke et al., 2017), Tracktor (Sridhar et al., 2019),
035   FairMOT (Zhang et al., 2021), have been proposed for MOT problem. However, most previous
036   works focus on the tracking of simple object categories, i.e., humans and vehicles. Actually, it is
037   important for the perception of various categories of objects in many real-world applications. Some
038   recent works have begun to study the tracking of generic objects. TAO (Dave et al., 2020) is the first
039
      large dataset for the generic MOT, which includes 2,907 videos and 833 object categories. The later
      GMOT-40 (Bai et al., 2021) includes 10 categories and 40 videos with dense objects in each video.
040
041   With the development of Artificial General Intelligence (AGI) and multi-modal foundation models,
042   open-world object perception has become a popular topic. Open-vocabulary object detection (OVD)
043   is a new and promising task because of its generic settings. It aims to identify the various categories
044   of objects from an image, including both the categories that have been seen during training (namely
045
      base classes) and not seen (namely novel classes). Although OVD has been studied in a series of
      works (Dhamija et al., 2020; Joseph et al., 2021; Doan et al., 2024), the literature on open-vocabulary
046
      (multi-) object tracking is rare. The nearly sole work (Li et al., 2023) builds a benchmark for
047
      open-vocabulary multi-object tracking (OVMOT) based on TAO (Dave et al., 2020). The authors
048   also develop a simple framework for this problem consisting of a detection head and a tracking
049   head. The detection head is directly taken from an existing OVD algorithm, i.e., DetPro (Du et al.,
050   2022), which is used to detect the open-vocabulary categories of objects in each frame without
051   considering any tracking-related factors. Then the tracking head is used to learn the similarities
052   among the detected objects in different frames. Since the lack of video data with open-vocabulary
053   tracking annotations, the approach in Li et al. (2023) uses a data hallucination strategy to generate the
      image pairs for training the tracking head. However, the generated image pairs ignore the adjacent

102          2    R ELATED W ORK
103
104          Multiple object tracking. The prevailing paradigm in MOT is the tracking-by-detection frame-
105          work (Andriluka et al., 2008), which involves detecting objects in each frame first, and then associat-
106          ing objects across different frames using various cues such as object appearance features (Bergmann
107          et al., 2019; Fischer et al., 2023; Leal-Taixé et al., 2016; Milan et al., 2017; Pang et al., 2021;
             Sadeghian et al., 2017; Wojke et al., 2017; Cai et al., 2022), 2D motion features (Zhou et al., 2020;


                                                                    2
      Under review as a conference paper at ICLR 2025


108
      Saleh et al., 2021; Xiao et al., 2018; Qin et al., 2023; Du et al., 2023), or 3D motion features (Huang
109   et al., 2023; Luiten et al., 2020; Wang et al., 2023; Krejčı́ et al., 2024; Ošep et al., 2018; Sharma
110   et al., 2018b). Some methods leverage graph neural networks (Bochinski et al., 2017; Ding et al.,
111   2023) or transformers (Meinhardt et al., 2022; Sun et al., 2020; Zeng et al., 2022; Zhou et al., 2022d)
112   to learn the association relationships between objects, thereby enhancing tracking performance. To
113   broaden the object categories of the MOT task, the TAO benchmark (Dave et al., 2020) has been
114   proposed for studying MOT under the long-tail distribution of object categories. On this bench-
115   mark, relevant methods include AOA (Du et al., 2021), GTR (Zhou et al., 2022d), TET (Li et al.,
116   2022), QDTrack (Fischer et al., 2023), etc. While these methods perform well, they are still lim-
117
      ited to pre-defined object categories, which makes them unsuitable for diverse open-world scenarios.
      Differently, this work handles OVMOT problem, which contains categories not seen during training.
118
119   Open-world/vocabulary object detection. Unlike traditional object detection with closed-set cate-
120   gories that appear at the training time. The task of open-world object detection aims to detect salient
121   objects in an image without considering their specific categories (Dhamija et al., 2020; Joseph et al.,
122   2021; Doan et al., 2024). This allows the method to detect objects of categories beyond those present
123
      in the training set. However, such methods do not classify objects into specific categories and only
      regard the object classification task as a clustering task (Joseph et al., 2021), achieving classification
124
      by calculating the similarity between objects and different class clusters. Consistent with the ob-
125
      jective of open-world detection, open-vocabulary object detection requires the identification of cat-
126   egories not seen in the training set. However, unlike open-world detection, open-vocabulary object
127   detection needs to predict the specific object categories (Zareian et al., 2021). To achieve this, some
128   works (Bansal et al., 2018; Rahman et al., 2020) train the detector with text embeddings. Recently,
129   pre-trained vision-language models ,e.g., CLIP (Radford et al., 2021) connect visual concepts with
130   textual descriptions, which has a strong open-vocabulary classification ability to classify an image
131   with arbitrary categories described by language. Based on this, many works (Gu et al., 2022; Zhou
132   et al., 2022c; Wu et al., 2023) utilize pre-trained vision-language models to achieve open-vocabulary
133   object detection and few-shot object detection. In addition, some studies (Du et al., 2022; Zhou et al.,
134
      2022a;b) further enhance the effectiveness of prompt embeddings of class descriptions using prompt
      learning methods, thereby improving the results of open-vocabulary detection. Different from the
135
      above detection methods developed for single image, in this work, we propose a prompt-guided
136
      training method designed for open-vocabulary detection in continues videos, which can effectively
137   enhance the detection performance for the video tracking task.
138
139   Open-world/vocabulary object tracking. There are relatively few works addressing the task of
140
      open-world tracking. Related approaches aim to segment or track all the moved objects in the
      video (Mitzel & Leibe, 2012; Dave et al., 2019) or handle the generic object tracking (Ošep et al.,
141
      2016; 2018; 2020) using a class-agnostic detector. Recently, the TAO-OW benchmark (Liu et al.,
142
      2022) is proposed to study open-world tracking problems, but its limitation lies in evaluating only
143   class-agnostic tracking metrics without assessing class-related metrics. To make the setting more
144   practical, OVTrack (Li et al., 2023) first brings the setting of open vocabulary into the tracking task,
145   which also develops a baseline method and benchmark based on the TAO dataset. However, the
146   method in Li et al. (2023) directly uses an existing OVD algorithm for detection, and its training
147   process only utilizes static image pairs and ignores the information from video sequences. Differ-
148   ently, we consider the tracking-related object states for detection, and also propose a self-supervised
149   video-based training method designed for open-vocabulary tracking, making full use of video-level
150   information to enhance the performance of open-vocabulary tracking.

241   Tracking-state-aware prompt. In the classification head of existing open-vocabulary detection
242
      methods, when selecting region candidates for calculating classification loss, they only consider
      whether the maximal IOU between the candidates and ground-truth box exceeds a threshold. For
243
      this tracking problem, we further consider whether the states of the candidates are appropriate for
244
      training the detection network.
245
246   As is well known, the objects present many specific states during tracking, such as occlusion/out-
247   of-view/motion blur, etc, which are more frequent than the object detection in static images. So, it
248
      is important to identify such object states to achieve more accurate detection and tracking results.
      However, these states have often been overlooked in past methods because the labels of these states
249
      are difficult to obtain, not to mention incorporating them into the network training.
250
251   Prompt, as a burgeoning concept, can be used to bridge the gap between the vision and language
252   data based on the cross-modal pre-trained models. We consider using specially designed prompts
253   to perceive the tracking states of the objects and integrate such states into the model training. We
254
      refer to such prompts as ‘tracking-state-aware prompts’. We specifically employ pairs of adjectives
      with opposite meanings to describe the object states during tracking. For example, ‘unoccluded and
255
      occluded’. As shown in classification head of Figure 2, we add M pairs of tracking-state-aware

      prompt pairs, denoted as {pm        m M

281   3.3     S ELF - SUPERVISED O BJECT A SSOCIATION WITH R AW V IDEO DATA
282
      Considering the lack of annotated videos for OVMOT, we develop a self-supervised approach to
283
      train the association network by leveraging the consistency among the same objects during a video.
284
285
      3.3.1    F ORMULATION
286
287   Objective function. To learn the object appearance feature, we consider two aspects. The first one
288   is intra factor, i.e., the self-consistency of appearance for the same object at different times. The
289   second one is inter factor, i.e., the mutual-consistency between the appearance and motion cues
290   during tracking. This way, we formulate the optimized objective function as
291
                    max    S = Sintra (ti , tj ) + α · Sinter (ti , tj ), s.t.   ∀ti , tj ∈ Tc ,   ∀c ∈ C,   (4)
292
293   where S represents the overall consistency objective to be maximized, Sintra and Sinter denote the
294   intra and inter consistency measures respectively, while α is a weight to balance them. Besides,
295   considering the diversity of object categories in the OVMOT problem, we add the object category
296   consistency constraint for the consistency learning. Specifically, we construct the intervals, e.g., Tc ,
297
      which contain several frames only with the same object category c, in which we select ti , tj . This is
      because we aim to learn the feature in a self-supervised manner without ID annotation, the objects
298
      with various categories may bring about interference.
299
300   Long-short-interval sampling. First, we consider the interval splitting of Tc in Eq. (4). We split
301   the original videos into several segments of length L and randomly sample the shorter sub-segments
302   with various lengths from each segment. These short-term sub-segments are then concatenated to
303
      form the training sequence. Such training sequences include long-short-term intervals. Specifically,
      we select the adjacent frames from the same sub-segment, which allow the association head to learn
304
      the consistency objectives under minor object differences. We also select the long-interval video
305
      frames from different sub-segments, which allow the association head to learn the similarity and
306   variation of objects under large differences.
307
308   Category-consistency constraint. Then we consider the category consistency constraint in Tc .
309
      After obtaining the sampled training sequences as discussed above, we utilize the localization head
      to extract object bounding boxes from each frame in the sequence. Since we only use unlabeled
310
      raw videos for training, we cannot directly obtain the object categories. To address this issue, we
311
      employ a clustering approach to group the bounding boxes based on their classification features.
312   Specifically, we utilize the classification head to obtain the category features for the candidate objects
313   from all frames. Then we cluster the candidate objects’ category features as different clusters. After
314   clustering the candidate objects’ category features into distinct groups, we can treat each cluster as a
315   separate category. As illustrated in Figure 2, we proceed to conduct self-supervised learning on the
316   objects that belong to the same category cluster and are sourced from different frames.
317

321   Intra-consistency loss. After getting the training samples (object bounding boxes in different
322   frames of the sample training sequence within the category clustering), we first use the CNN network
323   to extract the appearance feature from the association head for all objects in frame ti to construct the
      feature matrix Fi . The main idea of the intra-consistency loss is to leverage the self-consistency of

      the same objects at different times (frames). We utilize the following two types of similarity transfer

367   margin m ≥ 0 is a parameter used to control the punishment scope between E(r, c) and E(r, r).
368   This loss helps E approach the identity matrix while addressing cases where there are unmatched
369   targets in a row through a margin m. Finally, we define our self-supervised consistency learning loss

373   box list Bi and Bj of adjacent frames i and j in the video, we compute the Intersection over Union
374   (IoU) matrix Mij between bounding boxes Bi and Bj , which quantifies the overlap between them.

379   the optimization of the spatial consistency objective. Furthermore, we utilize the previously ob-
380   tained normalized similarity matrix Sij to establish the inter-consistency loss, using the binary cross-
381   entropy loss function LBCE , as Linter = LBCE (Sij , Aij ). This formulation allows us to measure the

387   thereby effectively exploring the potentiality of videos to enhance the association for OVMOT.

532
533
      In this work, we have developed a new method namely VOVTrack to handle the OVMOT problem
534   from the perspective of video object tracking. For this purpose, we first consider the object state
535   during tracking and propose tracking-state-aware prompt-guided attention, which improves the ac-
536   curacy of object localization and classification (detection). Second, we develop an object similarity
537   learning strategy for the temporal association (tracking) using only the raw video data without an-
538   notation, which unveils the power of self-supervised learning for open-vocabulary tracking tasks.
539   Experimental results demonstrate the effectiveness of the proposed method and each component for
      open-vocabulary tracking.

      A PPENDIX 1. OVMOT P ROBLEM
760
      OVMOT requires the tracker to be capable of tracking objects from the open-vocabulary categories
761
      of objects. We first present the problem formulation of this task from the training and testing stages.
762
763   At training stage, the training data is Xtrain , Atrain that contains video sequences Xtrain and their
764   respective annotations Atrain of the objects. Given one frame in the video, each annotation α ∈ Atrain
765   consists of a 2D bonding box b = [x, y, w, h], a unified ID d over the whole video, and a category
766   label c, where (x, y) is the center pixel coordinates and (w, h) is the width and height of the box,
767   the category belongs to the base class set, i.e., c ∈ C base .
768   At the testing stage, the inputs consist of video sequences Xtest and the set of all object classes
769   C = C base ∪ C novel , where C novel denotes the novel categories not appearing in the training set, i.e.,
770   C novel ∩C base = ∅. OVMOT aims to obtain the trajectories of all objects in Xtest belonging to classes
771   C. Each trajectory τ consists of a series of tracked objects τt at frame t, and each τt is composed
772   of a 2D bounding box b, and its object category c. Note that, during the testing stage, we need to
773   evaluate not only the results on the base class C base , but also on the novel class C novel . The results on
774
      C novel can validate the tracker’s capability when facing objects from the open-vocabulary categories.
