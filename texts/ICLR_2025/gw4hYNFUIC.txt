009                                                A BSTRACT
010
011            In automated task planning, state estimation is the process of translating an agent’s
012            sensor input into a high-level task state. It is important because real-world environ-
013            ments are unpredictable, and actions often do not lead to expected outcomes. State
014            estimation enables the agent to manage uncertainties, adjust its plans, and make
015            more informed decisions. Traditionally, researchers and practitioners relied on
016            hand-crafted and hard-coded state estimation functions to determine the abstract
017
               state defined in a specific task domain. Recent advancements in Vision-Language
               Models (VLMs) enable autonomous retrieval of semantic information from visual
018
               input. We present Semantic Symbolic State Estimation (S3E), the first general-
019
               purpose symbolic state estimator based on VLMs that can be applied in various
020            settings without specialized coding or additional exploration. S3E takes advantage
021            of the foundation model’s internal world model and semantic understanding to as-
022            sess the likelihood of certain symbolic components of the environment’s state. We
023            analyze S3E as a multi-label classifier, reveal different kinds of uncertainties that
024            arise when using it, and show how they can be mitigated using natural language
025            and targeted environment design. We show that S3E can achieve over 90% state
026            estimation precision in our simulated and real-world robot experiments.

      1   I NTRODUCTION
030
      Automated task planning is a crucial component for intelligent agents to solve complex and ever-
031
      changing tasks (Ghallab et al., 2016; Geffner & Bonet, 2013). In some cases, it is appropriate to
032   assume that an agent has full domain knowledge (the Closed World Assumption (CWA) (Reiter,
033   1981)) and that all facts about the world are known. However, an agent’s observations are often
034   based on its sensing capabilities, especially in real-world applications, from which extracting these
035   facts is non-trivial. State estimation is the process of obtaining a high-level state of the environment,
036   i.e., translating numeric sensor input into semantic facts. (Chen et al., 2024a; Castaman et al., 2021;
037   Lagriffoul et al., 2018). State estimation is crucial for monitoring the execution of a plan. Should an
038   action lead to an unexpected state, there may be grounds for replanning or reporting of task failure.
039
      Example 1 A robotic arm is tasked with rearranging groceries on multiple tables. The goal is to
040
      move a box of cereal and a carton of milk to a specific table where the hungry human would like to
041
      prepare her breakfast. A task planner chooses the following plan: “pick-up(milk, table1)”, “put-
042   down(milk, table3)”, “pick-up(cereal, table2)”, “put-down(cereal, table3)”. While moving to place
043   the cereal on table number 3, the the object is dropped due to a bad grasp, and lands on table
044   number 1. Using a state estimator, we notice that the expected state where the cereal box is on table
045   number 3 has not been achieved. We thus call the task planner once more to obtain the following
046   plan that will lead us to the goal state: “pick-up(cereal, table1)”, “put-down(cereal, table3)”.
047
048   Current state-of-the-art methods for task planning rely on hand-crafted and hard-coded state estima-
049   tion functions (Moreno et al., 2024; Garrett et al., 2020). This is time-consuming work that relies
050
      on advanced sensing equipment, which results in domain-specific outputs that do not adapt to any
      changes in the environment or task. We desire a general state estimation function that requires no
051
      specialized coding, no additional exploration, and generalizes to a large scope of tasks.
052
053   With the rise of powerful instruction-based Vision-Language Models (VLMs), i.e., vision-based
      foundation models, it is now possible to answer complex semantic questions about a scene based

071   In this work, we introduce Semantic Symbolic State Estimation (S3E), the first zero-shot state esti-
072   mator based on these VLMs. Our objective is to provide a general, versatile, and performant solution
073   for state estimation that will accelerate the process of constructing state estimation functions for re-
074   searchers and practitioners of task planning. S3E takes advantage of the foundation model’s internal
075
      world model and semantic understanding (Schneider et al., 2024; Smeaton, 2024) to assess the like-
      lihood of certain symbolic components of the environment’s state that are relevant to the task being
076
      solved. It consists of two stages: (1) translating symbolic predicate definitions into natural language
077
      questions and (2) answering them given visual input. We show that the translation stage signifi-
078   cantly improves performance in Appendix B. Fig. 1 demonstrates the usage of S3E in a real-world
079   robotics task. We show that a zero-shot solution is indeed possible in some cases. To further improve
080   performance alongside the VLM’s strong priors, we use natural language instruction and targeted
081   environment design to remove ambiguities and reduce uncertainties about the environment or task.
082
      We analyze S3E as a multi-label classifier where the labels are the set of grounded predicates that
083
      make up all possible facts about the task state. Our experiments focus on usability and showcase a
084   simulated and real-world robotic domain. S3E also achieves high performance in a photorealistic
085   blocksworld with ever-changing objects in Appendix C, showing that it is truly general-purpose and
086   versatile. We propose task-specific solutions to handle two kinds of uncertainties in our proposed
087   state estimator. The first is the model’s uncertainty regarding the state. The second stems from the
088   subjective nature of the actual state relative to the intent of the task designer, i.e., whether a certain
089   property holds for a given state is in the eye of the beholder. We show examples of these uncertainties
090   and how they can be reduced using natural language instruction and minimal environment design.
091   This improves on previous work that elicit uncertainties in language models Ren et al. (2023); Xiong
092
      et al. (2023) by leveraging this idea for symbolic state estimation in the context of task planning.
      Regardless of these uncertainties, general-purpose state estimation is a needed change from the
093
      specialized solutions offered by today’s state-of-the-art.

105
106   State estimation is vital for automated agents performing complex tasks, especially in Task and
107   Motion Planning (TMP), where recognizing the state amidst uncertain dynamics is crucial for in-
      tegrating high-level actions with physical motions (Curtis et al., 2024; Garrett et al., 2021; La-


                                                                                   2
      Under review as a conference paper at ICLR 2025


108
      griffoul et al., 2018; Kaelbling & Lozano-Pérez, 2011). Traditionally, state estimators have been
109   hand-crafted (Moreno et al., 2024; Garrett et al., 2020), and modern task planning toolkits still rely
110   on such manual coding (Wertheim et al., 2024). Learning-based methods, like Pankert & Hutter
111   (2023), address specific tasks but lack generalizability and require exploration steps. In contrast, our
112   approach eliminates manual coding and exploration, leveraging Vision-Language Models (VLMs)
113   for zero-shot generalization.
114
      Large Language Models (LLMs) have been explored in task planning (Kambhampati et al., 2024;
115   Liu et al., 2023a; Huang et al., 2022) but typically overlook environment dynamics and sensing.
116   Chen et al. (2024b) automate TMP using LLMs but depend on textual domain representations and
117   ignore state uncertainty. Unlike these, S3E incorporates sensor input for state estimation.
118
      Open-world planning has introduced LLMs in tasks where the agent lacks prior knowledge of its
119
      environment. Singh et al. (2023) assume known initial states and action effects, while Ding et al.
120
      (2023) predict planning obstacles without full state estimation. VLM-based state estimation ap-
121   proaches, such as Chen et al. (2024a), rely on external action success indicators. Duan et al. (2024b)
122   uses VLMs in a robot manipulation pipeline that includes state estimation. However, the method
123   requires a scene representation with task-specific elements and demonstrations while being coupled
124   into a manipulation-centric system. Duan et al. (2024a) and Liu et al. (2023c) also use VLMs for
125   scene understanding but focus on reasoning over action failure rather than state estimation.
126
      Another string of related work is that of scene graph generation. These methods are designed to
127   construct a matching graph of semantic entities and their relationships from a given scene represen-
128   tation. The scene representation can be 3D, e.g., mesh (Armeni et al., 2019) or point cloud Wang
129   et al. (2023b) (which may be hard to come by), while others use images Zhao et al. (2023); Wu et al.
130   (2023). Some approaches are even specifically geared toward perception and planning Maggio et al.
131   (2024); Gu et al. (2024). Unlike these approaches, S3E requires no additional training beyond that
132   of the pre-trained VLM. It does not need task-specific adaptation, making it more versatile out-of-
133   the-box. Furthermore, S3E and scene graphs do not cancel each other out, i.e., they can be used
134   together to further improve state estimation accuracy. For example, scene graph data may be used
135
      as additional input for S3E to mitigate uncertainties about the scene.

137   3    BACKGROUND
138
139   Symbolic Task Domains In this work, we support agents with tasks for which the task domain
140   (Geffner & Bonet, 2013) is defined as a tuple Σ = ⟨S, s0 , SG , A, T, c⟩. s0 ∈ S is the initial state,
141   SG ⊆ S is a set of goal states, A(s) ⊆ A is the set of actions applicable at state s, T is a deterministic
142   transition function where T (a, s) ∈ S is the state that follows s after performing action a, and c(a, s)
143   is a positive cost of performing action a at state s. A task plan is a sequence of actions (a1 , ..., an )
144   that are applicable in order from s0 onwards and ultimately reach some state s ∈ SG .
145   We focus on symbolic representations of task domains. Namely, s is defined over a finite feature
146   space F . Thus, each state s ∈ S is an assignment over the set of features, i.e., s = (f1 , f2 , ..., f|F | ).
147   This representation makes it easy to support representations such as Planning Domain Definition
148   Language (PDDL) McDermott et al. (1998) which define actions using three sets of propositions
149   over the features denoting the preconditions, add effects, and delete effects for that action. In Fig. 1,
150   the pick-up action has ’robot-gripper-empty()’ as a precondition, and ’robot-gripper-empty()’ and
151   ’in-table-section(spray-bottle, blue)’ as delete effects. This highlights the important role state esti-
152
      mation has on the ability of automated agents to assess what is achievable in the current state.
153
154   Instruction Tuned Vision-Language Models A Vision-Language Model (VLM) is a machine-
155
      learning model that combines natural language processing and computer vision (Li et al., 2022;
      Radford et al., 2021; Li et al., 2019) to perform various tasks. Visual Question Answering (VQA) is
156
      a task that uses VLMs to answer natural language questions about visual input, making them highly
157
      versatile question-answering functions.
158
159   In the general case, a VQA model is a parameterized function gϕ that accepts an image Xv and text
160   Xq as input and outputs a probability distribution over a vocabulary of predefined tokens V . The
161   input text is a sequence of tokens Xq = (Xq1 , ..., Xqn ) in V , with sequence length n. The output
      sequence Xa = (Xa1 + ... + Xam ) of length m is generated by sampling tokens in an autoregressive


                                                            3
      Under review as a conference paper at ICLR 2025


162
      fashion. First, the token Xa1 ∼ gϕ (Xv , Xq ) is sampled. The next token is then sampled with a
163
      concatenation of the same text and the new token Xa2 ∼ gϕ (Xv , Xq +Xa1 ), and repeating this process
164
      sequentially until an “end-of-response” token is generated Xam ∼ gϕ (Xv , Xa + Xa1 + ... + Xam−1 ).
165
      The final model’s response is the sequence of all sampled tokens Xa = (Xa1 + ... + Xam ). VQA
166   models are trained such that Xq is a question about Xv and Xa is likely the correct answer to the
167   question. For more information on instruction-tuning these models, see Appendix A.

      4    U SING S EMANTICS FOR S TATE E STIMATION
171
      Our objective is to provide a general and versatile solution for state estimation that will accelerate the
172
      construction of state estimation functions for researchers and practitioners of task planning. We want
173
      to provide a function that, given a symbolic world model, estimates the individual state features.
174
175   Let Φ be the set of possible agent image observations in a fully observable setting. Given task
176   domain Σ = ⟨S, s0 , SG , A, T, c⟩ where S is defined over features F , let ξΣ be the ground truth
177
      state estimator, that is, ξΣ (Xv ) = (f1 , .., f|F | ) ∈ S is the true feature assignment corresponding to
      observation Xv ∈ Φ. We want to find meta function ξG that accepts a task domain as input and
178
      outputs an approximate state estimator function, i.e., ∀Xv ∈ Φ, ξG (Σ)(Xv ) ≈ ξΣ (Xv ). In simple
179
      terms, we want to find a global function ξG that outputs a task-specific state estimator function
180   ξG (Σ) that reliably approximates ground truth estimator ξΣ .
181
182   A naı̈ve approach to defining ξG (Σ) is to learn from applying actions from different states and pair-
183
      ing the resulting observations with the expected state according to the domain description. But if
      we could fully trust actions and the controller executing their trajectory, the downward refinement
184
      property (Bacchus & Yang, 1991) would hold, meaning low-level actions would guarantee the de-
185
      sired task state. Thus, the agent can blindly execute planned actions sequentially and reach the goal
186   with certainty, making the state estimator redundant. Since this assumption is unrealistic, a different
187   approach is needed. In this work, we turn to semantics as the source from which the state is derived.
188
189
      In real-world symbolic task planning, it is common for the task designer to maintain states and
      actions that contain some semantic value. In Example 1, the “pick-up(item)” action has a clear
190
      semantic meaning: the item is picked up by the agent. Therefore, we assume that the task domain
191
      components can be clearly described using natural language. We use these descriptions as a semantic
192   guideline for state estimation.
193
194   We present Semantic Symbolic State Estimation (S3E), a semantic approach to state estimation that
195
      uses pre-trained vision-based foundation models to provide a general and versatile solution that
      generates a joint probability distribution over the symbolic state components. This way, we take ad-
196
      vantage of one of the great strengths of foundation models: their internal world model and semantic
197
      understanding (Schneider et al., 2024; Smeaton, 2024). Specifically, we translate a textual domain
198   description into a collection of natural language queries for which answers determine the features
199   of a state using a Large Language Model (LLM). We then answer these questions in reference to
200   vision-based observations, denoted Xv , using a VQA model, thus estimating the current task state.
201
      Using a VQA model as a state estimator requires making some assumptions: (1) the image obser-
202
      vations contain the information required to determine the task state (e.g., no object occlusion); (2)
203
      the domain description is unambiguous (e.g., use object names like “white-table” and “black-table”
204   instead of “table1” and “table2”); (3) all objects are visually distinguishable (no identical objects).
205   Note that while assumption 3 seems particularly demanding, it can easily be overcome by labeling
206   objects using a combination of object detection and object tracking methods, e.g., YOLO (Redmon
207   et al., 2016) and SORT (Bewley et al., 2016). Additionally, S3E assumes that the state space features
208   are defined as semantic predicates that refer to one or more objects.
209
      A predicate P is a function that that represents a property of relation between compatible object
210   parameters ω = (ω1 , ..., ωm ). Denote ΩP the set of object sequences that are valid arguments for
211   predicate P . A grounded predicate is a predicate-parameters pair P (ω) where ω ∈ ΩP . The feature
212   space F corresponding to state space S is defined as the set of all grounded predicates. For each
213   grounded predicate, we would like to answer the question “In this image, does P (ω) hold true?”.
214   However, we would like to ask these questions in natural language, e.g., if P is “on-table” and
215   Ω is “milk-carton, wood-table”, then we would like to answer the question “In this image, is the
      milk carton on the wooden table?”. We perform this translation from grounded predicate to natural

279   5   U NCERTAINTY IN S EMANTIC S TATE E STIMATION
280
281   When employing vision-based semantic state estimation, as outlined in Fig. 2 and under the as-
282   sumptions of Section 4, two distinct kinds of uncertainties emerge: model uncertainty and task
283   uncertainty.
284   Model uncertainty pertains to the system’s lack of knowledge about the world. For instance, if
285   the training data does not include a cereal box (neither in images nor text), the state estimation, as
286   demonstrated in Example 1, would likely result in random guesses for any predicate involving the
287   ”cereal” object. This type of uncertainty reflects the model’s limited exposure during training and
288   its inability to generalize beyond its learned domain.
289   Task uncertainty, on the other hand, arises from the subjective nature of task design and interpre-
290   tation. A pertinent example can be seen in Fig. 2, where it may be ambiguous whether a coffee
291   mug is ”on the table” if a small plate is separating the two. This uncertainty stems from differing
292   perspectives or intentions in defining predicates.
293
      We classify these uncertainties based on their origins: aleatoric or epistemic (Hüllermeier & Waege-
294
      man, 2021). Model uncertainty can be viewed as aleatoric because it involves inherent randomness
295   in the world, yet it can also be epistemic since expanding the training dataset could mitigate it. Task
296   uncertainty, similarly, can be aleatoric due to variability in task design but may also be epistemic, as
297   clearer feedback or instructions from the task designer can reduce it. For clarity, we adopt the follow-
298   ing conventions: (a) Model uncertainty is treated as aleatoric, with the model parameters fixed. Our
299   mitigation strategy focuses on designing experiments to avoid ambiguous or unclear states entirely.
300   (b) Task uncertainty is considered epistemic, and we address it by providing explicit, clarifying
301   instructions to refine predicate definitions and reduce ambiguity.
302   The uncertainties faced by S3E are akin to those encountered by humans during manual state esti-
303   mation. To manage these uncertainties effectively, we implement two strategies inspired by human
304   practices: (1) Few-shot adaptation: In our simulated pick-and-place experiment, we provide exam-
305   ples to guide the state estimator, such as describing specific object appearances; (2) Environmental
306   and action design: We structure the environment and robot behavior to make states unambiguous.
307   For instance, in our experiments, the robot assumes a standardized configuration after each action,
308   ensuring clarity on whether it is holding an object. These approaches, detailed and demonstrated
309
      in Section 6, serve to minimize both types of uncertainties and enhance the robustness of our state
      estimation framework.

346   The task is defined in PDDL (McDermott et al., 1998), a common description language for task
347   domains, which requires explicitly declaring predicates and objects of interest. These are extracted
348   using an off-the-shelf parser (Micheli & Bit-Monnot, 2022) and fed into the S3E pipeline.
349   The advantage of running in simulation is that we can collect data more easily. The simulation can
350   run faster than a real-world equivalent, and constructing a ground-truth state estimator is easier with
351   access to privileged simulator information. We collect pairs of rendered and task states by randomly
352   setting the items’ positions on the tables and having the robotic agent perform random actions that
353   perturb the environment and produce a new rendered state. We collect 2,000 data points1 using the
354   procedure described in Appendix D. The agent performs the actions in full instead of us directly
355   placing the items on the tables or in the robot’s gripper to achieve natural-looking states that may be
356
      reached when solving the task, e.g., items knocked over, some on the floor, various grip positions,
      etc. Actions are implemented imperfectly to achieve these real-world situations.
357
358   Each data point is processed through the pipeline depicted in Fig. 2. The quality of estimation will
359   be determined using evaluation tools for multilabel classification. We expect to witness two kinds
360   of failures (see Section 5). First, how the actions are designed sometimes makes it unclear whether
361   the robot is gripping an object. This is an example of aleatoric uncertainty relative to the scene and
362
      the underlying task. In an attempt to overcome this, we run the same experiment on an alternative
      dataset wherein we assert a certain robot “home pose” after picking up and placing an object. The
363
      home pose is the pose of the robot as shown in Fig. 3, where it can be clearly seen whether or not
364
      an object is within the gripper jaws. These experiments are denoted with “Pose”. Second, because
365   this is a simulated environment and VQA systems are normally trained on real-world data, we fully
366   expect low performance on predicates that consider the less realistic items. This will illustrate the
367   uncertainty introduced by the limitation of the model’s training distribution. To overcome this, we
368   prompt the VLM with additional information about the task and its objects, e.g., “The milk carton is
369   a clean white rectangular box with a triangular top”. We denote these experiments with the “Instruct”
370   label. The exact instruction prompt used in these experiments is provided in Appendix E.3.

375   sections on a table. The items in the scene are a green mug, a water bottle, a soda can, and a window
376   cleaner spray bottle. The table is divided into a white section and a blue section. A single camera

410   Experiment 1: Simulated Environment This section evaluates the S3E state estimators in Ex-
411   periment 1, treating them as multilabel classifiers. Since most predicates (approximately 75%) are
412   false (e.g., when one object is being gripped, others are not), a baseline accuracy of around 75%
413   can be achieved by predicting all predicates as false. Therefore, we focus on the Average Precision
414   (AP) (Average Precision) score with micro and macro averaging as our primary metrics (Sokolova
415   & Lapalme, 2009).
416   Table 1 presents accuracy and AP scores for different S3E VLM models. As anticipated, larger
417   models show better performance. The 0.5B model performs poorly, with an AP score below 50%.
418   The 7B model achieves AP scores ranging from 66% to 77%, while the 72B model scores between
419   74% and 91%. The ”Pose” modification significantly improves performance across all models,
420   particularly for the macro average, which emphasizes the gripping predicates: improvements are
421   approximately 21% for the 0.5B model, 16% for the 7B model, and 12% for the 72B model.
422   Adding natural language instructions negatively impacts the 0.5B and 7B models, likely due to
423   confusion from the additional context. In contrast, the 72B model shows consistent AP performance
424   for micro averages (about a 1% difference) and substantial macro average gains (approximately 4%
425   without ”Pose” and 9% with it). Combining both enhancements (”Instruct + Pose”), the 72B model
426   improves by approximately 9.5% (micro) and 22% (macro). Similar trends are observed in the
427   photorealistic blocksworld domain (see Appendix C).
428   Although accuracy alone offers limited insight, examining it across different thresholds reveals
429   model certainty. The 72B model maintains balanced certainty around the 50% mark, effectively dis-
430   tinguishing between true and false predicates. In contrast, the 0.5B and 7B models show increased
431   accuracy at higher thresholds, suggesting they often assign high probabilities to false values. Setting
      a higher threshold for these models may reduce false positives.

505    6.3                      D ISCUSSION
506
507    Our experiments demonstrate that S3E can approximate symbolic states with over 90% precision in
508    both real and simulated tasks. We also see similar results for our additional experiments that test our
509
       method’s adaptability (see Appendix C). This high accuracy showcases the potential of our novel
       pipeline to estimate symbolic states in task planning environments. By harnessing the VQA model’s
510
       semantic understanding, we bridge the gap between visual inputs and symbolic state representations.
511
512    While S3E struggles with out-of-distribution scenarios, targeted environment manipulation and task-
513    specific natural language instructions help mitigate this issue, particularly in larger models. Notably,
514    S3E combined with the OV model performs better in real-world environments than simulated ones,
515
       likely due to the composition of its training data. These findings highlight both the strengths of our
       approach and areas for future refinement in VQA, particularly for state estimation.

518    7                        C ONCLUSION
519
520
       In this work, we presented S3E, a general-purpose vision-based symbolic state estimator using
521
       VLMs. S3E offers a versatile replacement for hand-crafted state estimator functions that are spe-
522    cialized for the individual task. While our framework was used deterministically, it can easily be
523    adapted for probabilistic estimations and account for belief state updates in partially observable
524    domains Kaelbling et al. (1998). We intend to explore this in future work.
525
       We empirically evaluated S3E coupled with LLaMA 3 and LLaVA OV as a multiclass classifier
526
       of task-specific predicates for robot pick-and-place tasks. We showed that this combination can
527
       achieve over 90% state estimation precision with no task-specific coding involved. In a simulated
528    environment, we demonstrated two kinds of uncertainties brought on by using a VQA model for
529    state estimation. We showed how to reduce these uncertainties with low-effort modifications to the
530    environment and natural language instructions to the model. In a real-world setting, we showed that
531    high performance can be achieved without providing any task-specific information.
532
       While S3E offers a general solution to visual state estimation, it comes with some limitations relative
533
       to hand-crafted state estimators. It requires a visual input setup with full observability. Furthermore,
534    it requires that all objects be visually distinguishable. We also perform an exhaustive search over all
535    grounded predicates, which can become computationally expensive in more complex environments.
536    Finally, the task must be defined in descriptive language to generate high-quality queries for the
537    VQA model. Future work should address the above limitations, uncertainty detection (e.g. using
538    confidence scores, discrepancy analysis, or external knowledge integration) and further mitigation
539    (e.g. Bayesian state estimation, conformal prediction), and improve performance with task-specific
       information (e.g., using predicate correlations).

745   LLM instruction-tuning is a method for improving a model’s ability to follow natural language
746   instructions (Wang et al., 2023a; Ouyang et al., 2022). To achieve this, models are fine-tuned using
747   enhanced datasets of instruction-output pairs. These datasets often include question-answer pairs,
748   task-completion examples, dialogue simulations, solved coding exercises, and more. Training is
749   typically performed using supervised learning. A reward model is learned through supervised output
750   and the language model is trained to maximize reward. To ensure human compatibility, this process
751   involves human-curated datasets, annotations, and response rankings (Ouyang et al., 2022), but this
752
      is also done using another LLM for scalability (Wang et al., 2023a).
753   A cross-breed of VLMs and instruction-tuning yielded LLaVA (Liu et al., 2023b), a VLM trained to
754   follow natural-language instructions based on visual input. This allows giving the model a certain
755   context in which to answer input questions. We take advantage of this ability to put the agent into
      the context of the task planning problem at hand.

991    E.1   PDDL P REDICATES T RANSLATION
992
993    To translate PDDL predicates to natural language questions, we use the following system prompt to
994
       instruct the LLM.
995    The following is a PDDL domain
996    {DOMAIN}
997    Here are the names of all the objects in the current problem, sorted by their type:
998    {OBJECTS BY TYPE}
999    Given a grounded predicate with concrete variables, write a natural language yes-no query whose
1000
       answer determines the truth value of the predicate.
       Respond only with this natural language query and nothing else.
1001
1002   The DOMAIN variable is the string description of the entire domain. In our case, this was the content
1003   of the PDDL domain. The OBJECTS BY TYPE variable is a comma-separated list of strings of the
1004   form:
1005   {OBJECT TYPE NAME} type: [{OBJECT1 NAME},{OBJECT2 NAME},...]
1006
       where the OBJECT TYPE NAME and OBJECTi NAME variables are the names as they appear in
       the PDDL domain file (for the types) and problem file (for the objects).
1007
1008   With this system prompt, the LLM is given a user prompt of the form:
1009   {PREDICATE}({VARIABLE1},{VARIABLE2},...)
1010   where the PREDICATE is a predicate from the PDDL domain file and VARIABLEi are objects from
1011   the PDDL problem file whose types match the predicate’s variables. We do this for all ground
1012
       predicates and create a mapping from predicates to their corresponding natural language query.
1013
1014   E.2   VQA M ODEL P ROMPTS
1015
1016
       The following is the system prompt used to calibrate the VQA model for state estimation:
       A curious human is asking an artificial intelligence assistant yes or no questions.
1017
       The assistant answers with one of three responses: YES or NO.
1018
       The assistant’s response should not include any additional text.
1019
1020
