009                                                                                   A BSTRACT
010
011                            Sparse attention can effectively mitigate the significant memory and throughput
012                            demands of Large Language Models (LLMs) in long contexts. Existing methods
013                            typically employ a uniform sparse attention mask, applying the same sparse pattern
014                            across different attention heads and input lengths. However, this uniform approach
015                            fails to capture the diverse attention patterns inherent in LLMs, ignoring their
016                            distinct accuracy-latency trade-offs. To address this challenge, we propose the
017
                               Mixture of Attention (MoA), which automatically tailors distinct sparse attention
                               configurations to different heads and layers. MoA constructs and navigates a search
018
                               space of various attention patterns and their scaling rules relative to input sequence
019
                               lengths. It profiles the model, evaluates potential configurations, and pinpoints
020                            the optimal sparse attention compression plan. MoA adapts to varying input sizes,
021                            revealing that some attention heads expand their focus to accommodate longer
022                            sequences, while other heads consistently concentrate on fixed-length local contexts.
023                            Experiments show that MoA increases the effective context length by 3.9× with
024                            the same average attention span, boosting retrieval accuracy by 1.5 − 7.1× over
025                            the uniform-attention baseline across Vicuna-{7B,13B}, and Llama3-{8B,70B}
026                            models. Moreover, MoA narrows the capability gaps between sparse and dense
027                            models, reducing the maximum relative performance drop from 9% − 36% to
028
                               within 5% across two long-context understanding benchmarks. MoA achieves a
                               1.2−1.4× GPU memory reduction, boosting decode throughput by 6.6−8.2× and
029
                               1.7 − 1.9× over FlashAttention2 and vLLM, with minimal performance impact.

050   Large Language Models (LLMs) exhibit remarkable versatility across numerous applications (Brown
051   et al., 2020; Tay et al., 2022; Wan et al., 2023). Central to LLM is the attention mechanism (Vaswani
052   et al., 2017), which computes interactions among tokens within a certain span, thereby enabling
053   context understanding. Scaling input length is crucial for enhancing LLM capabilities (Chen et al.,
      2023; Tworkowski et al., 2023), including fact retrieval, summarization, few-shot learning, question


                                                                                                  1
054
      answering and so on (Bai et al., 2023; Yuan et al., 2024). However, the ever-growing attention
055   computation and Key-Value Cache (KV-Cache) pose significant efficiency challenges (Sheng et al.,
056   2023; Xiao et al., 2024c; Han et al., 2023; Kwon et al., 2023).
057
058
      Previous work proposes sparse attention methods to address the efficiency challenges of long contexts
      in generative LLMs. These methods typically employ a uniform, fixed-span sliding window mask
059
      across all heads and input lengths, limiting attention to local contexts only (Xiao et al., 2024c;
060
      Han et al., 2023). This approach allows the LLM to take long inputs with a fixed attention span,
061   keeping bounded attention computation and KV caching overhead. Following previous works (Chen
062   et al., 2023; Tworkowski et al., 2023), we quantify the effective context length as the maximum
063   input length where content retrieval accuracy exceeds a 90% threshold. In principle, fixed-span
064   local attention can gradually aggregate global information through multiple model layers, yielding
065   a longer effective context length than each attention span (Feng et al., 2022; Zaheer et al., 2020).
066   Nonetheless, we reveal that uniform masks, like StreamingLLM (Xiao et al., 2024c), hardly extend
067   effective context length beyond the span, as shown in Figure 6. Figure 1(b) further illustrates such
068   limitation: with a 50% attention span mask, StreamingLLM fails to accurately retrieve content from
069
      the earlier half of the input and performs even worse at longer input lengths. Figure 2 reveals one
      possible explanation for the problem: while some attention heads focus on local contexts, others
070
      encompass the broad span of the entire input sequence. Consequently, the uniform approach fails
071
      to achieve a long effective context length as it limits the attention span of the global-context heads,
072   while excessively allocates compute and memory budget for local-context heads. Additionally, as the
073   input length increases, some attention heads need a faster increase in attention span than others to
074   avoid serious performance degradation, as shown in Table 1. Unfortunately, the uniform approaches
075   do not include heterogeneous rules to scale the attention spans differently for various heads. Besides,
076   existing model compression methods (Men et al., 2024; Lin et al., 2023; Xiao et al., 2024b; Li et al.,
077   2024a; Kim et al., 2023; Li et al., 2024b) use general language modeling corpora to decide the
078   compression plan, which cannot accurately profile the influence of compression on long-context
079   tasks.
080   In this work, we propose Mixture of Attention (MoA), a training-free sparse attention method. As
081   illustrated in Figure 3, MoA constructs the search space of heterogeneous elastic rules of attention
082   spans. For automatic LLM compression, MoA first utilizes gradient-based profiling to inspect the
083   influences of each attention position on the prediction loss. Based on the profiling results, MoA
084   tailors heterogeneous sparse attention configurations for each model layer and attention head. During
085   profiling, MoA employs a calibration dataset with long-range dependencies and uses the original
086
      dense model’s response instead of the human-written response as the reference to calculate the loss.
      This ensures an accurate profiling of the attention influences to facilitate better compression results.
087
      Our contributions are summarized as follows.
088
089
             • Heterogeneous Elastic Rules. We propose heterogeneous elastic rules for masks of each
090            attention head. We formulate MoA compression search space to include a diverse range of
091            elastic rules that tailor the local attention span relative to the input length for each attention
092            head. The heterogeneous elastic rules improve the fact retrieval accuracy of MoA from 25%
093            to 98% compared with masks with uniform span and scaling function for each head.
094
             • Calibration Dataset Construction We emphasize the importance of data engineering in
095            LLM compression. Our findings demonstrate that, instead of relying on general language
096            modeling datasets and human responses, using datasets with long-range dependencies and
097            referencing the original LLM’s responses is essential for accurately profiling the effects of
098            compression.
099
             • Automatic Optimization. We propose an automatic pipeline to find the optimal compression
100
               plan encompassing heterogeneous elastic rules for various attention heads. This pipeline can
101            efficiently find the optimal plan within several hours, for example, two hours for compressing
102            Vicuna-13B.
103
104   Experiments show that MoA achieves 6.6 − 8.2× throughput improvements over FlashAttention2,
105   1.7 − 1.9× over vLLM framework on 7B and 13B dense LLMs at a 50% density (the average of KV-
106   Cache length / input length), with only a 1% average relative degradation in retrieval accuracy. The
107   significant throughput improvements of MoA over FlashAttention2 can be attributed to four factors:
      (1) the static size of the KV-cache, (2) reduced attention computations, (3) increased batch size


                                                          2
108
      enabled by reduced memory usage, and (4) a specialized kernel implementation. Additionally, MoA
109   achieves over 90% retrieval accuracy with just 25% average density, far surpassing sparse attention
110   baselines that need a density of 75% to 100% for similar performance. On long-context understanding
111   benchmarks, MoA performs comparably to dense models, with a maximum relative performance drop
112   of less than 5%, which is about one-sixth of that observed with the uniform sparse attention baseline.
113   Our code is available at https://anonymous.4open.science/r/MoA-Review.

117   2.1   ATTENTION MECHANISM
118
119   The Multi-Head Self Attention (MHA) mechanism (Vaswani et al., 2017) is crucial to the functionality
120   of LLMs. It starts with an input sequence transformed into query (Q), key (K), and value (V) matrices
121   through linear projections. These matrices, combined with the cached K and V (KV-Cache) from
122   previous sequences, compute the attention matrix (A). This calculation is modified by a causal mask
123   (M) to ensure autoregressive properties, resulting in the output (O), as depicted in Equation 1:
124                             S = QKT ,      A = softmax(S + M),         O = AV                           (1)
125   Autoregressive inference in LLMs involves two stages: prefill and decode. During prefill, the
126   model processes the entire input sequence to generate the initial response token. In the subsequent
127   decode stage, it uses the newly generated token and previously cached K and V matrices to produce
128   subsequent tokens until the generation concludes. Although effective, this iterative process increases
129   memory and computation demands due to the expanding KV-Cache.
130
131   2.2   E FFICIENT ATTENTION
132
133   Efficient methods are proposed to mitigate the computation and memory costs associated with
134   attention. One branch of work uses dynamic sparse attention masks to adaptively skip attention
135
      computations during prefill stage (Pagliardini et al., 2023; Qu et al., 2022; Roy et al., 2021; Wang
      et al., 2021; Lu et al., 2021; Kitaev et al., 2020) or drop KV-Cache during decode stage (Anagnostidis
136
      et al., 2023; Zhang et al., 2023; Ge et al., 2023; Sheng et al., 2023; Liu et al., 2023a) based on the
137
      input sequences. However, due to the complex control and computation flow, dynamic prefill often
138   requires specific hardware to achieve substantial wall-time speedup (Qu et al., 2022; Wang et al.,
139   2021; Lu et al., 2021; Ham et al., 2021; 2020). Additionally, dynamic KV-Cache pruning in the
140   decode stage may require extensive retraining (Anagnostidis et al., 2023), additional KV-Cache score
141   computation (Sheng et al., 2023; Zhang et al., 2023; Liu et al., 2023a; Ge et al., 2023; Li et al., 2024c;
142   Cai et al., 2024), or extensive memory swap for KV-Cache retrieval (Tang et al., 2024b; Xiao et al.,
143   2024a).
144   Another branch of work uses static sparse attention, where predefined masks are applied consistently
145   across all processed sentences. Thanks to the fixed computation flow, static sparse attention is
146   generally more efficient and GPU-friendly. For language understanding models such as BERT (Devlin
147   et al., 2018), various masks are used (Zaheer et al., 2020; Beltagy et al., 2020; Child et al., 2019; Zhou
148   et al., 2024; Xiao et al., 2024c; Han et al., 2023). But for generative LLMs, the predominant method
149   is the fixed-span sliding window mask with global attention on a few initial tokens (Xiao et al., 2024c;
150   Han et al., 2023). With the local attention pattern, the KV-Cache beyond the current attention span
151   can be dropped, saving much memory for long sequence scenarios. However, the uniform static
152
      masks across different attention heads and input lengths are model- and data-agnostic, which can
      compromise LLMs’ effective context length and lead to suboptimal performance in long sequence
153
      scenarios. Our method falls within this category, benefiting from the efficiency and training-free
154
      advantages, while addressing the performance limitations encountered by previous methods.
155
156   In addition to sparse attention, alternative mechanisms have been proposed to replace traditional
157   attention for long-sequence modeling (Gu & Dao, 2023; Peng et al., 2023; Sun et al., 2023; Poli
158
      et al., 2023; Li et al., 2022; Kacham et al., 2023; Peng et al., 2021; Choromanski et al., 2020; Wang
      et al., 2020). However, these new mechanisms often require different weights compared to vanilla
159
      transformers, imposing significant re-training overhead for LLMs.
160
161   Previous works also propose LLM acceleration frameworks (Gugger et al., 2022; Aminabadi et al.,
      2022; Sheng et al., 2023; Kwon et al., 2023), as well as kernel-level optimizations (Dao et al., 2022;

      3     M IXTURE OF ATTENTION (M OA)
176
177
      We first illustrate the heterogeneity of the attention patterns in pre-trained LLMs in Section 3.1. Based
178   on this insight, we define the search space for our Mixture-of-Attention (MoA) method in Section 3.2.
179
180
      3.1   M IXTURE OF ATTENTION PATTERNS AND E LASTIC RULES
181
182   Heterogeneous Attention Patterns. Different attention heads in LLMs exhibit heterogeneous
183   attention patterns, as shown in Figure 2. For example, the first head primarily focuses on local
184   contexts with a narrow-span sliding window, while the third head covers nearly the entire input,
185   indicating global attention. The attention spans of different heads mostly remain constant across
186   various tasks and datasets, as shown in Appendix D.1. Table 1 demonstrates that applying the
187
      same sliding-window sparse attention mask across model layers can lead to a 65% variance in
      retrieval accuracies. It conforms to the multi-head self-attention design principle of capturing varied
188
      information (Vaswani et al., 2017), as well as the findings from concurrent research that identifies
189
      specific attention heads for global text retrieval (Wu et al., 2024).
190
191   Heterogeneous Elastic Rules. In addition to heterogeneity at a certain length, different attention
192   heads also exhibit varying elastic behaviors as the input length changes. Figure 2 illustrates this
193
      variability: for shorter inputs (the upper left part of the attention matrix), the second and third heads
      initially show global attention. However, as input length increases, the second head remains the
194
      medium-span local focus, while the third head continues to expand as global attention. Table 1 further
195
      evidences the diverse elastic rules. For example, at 4k input length, a 2k sliding-window sparse
196   attention mask on layers 9 to 11 yields better retrieval accuracy than on layers 17 to 19. However,
197   the opposite is true for an 8k input length. This data supports the visual observations from Figure 2,
198   highlighting that attention patterns respond to input length scaling differently. Leveraging these
199   insights, MoA encompasses heterogeneous elastic rules as the search space.
200
201   3.2   H ETEROGENEOUS E LASTIC RULE S EARCH S PACE
202
203   In designing the search space for the MoA mask, we consider the inherently heterogeneous and
204   elastic nature of LLM attention patterns. As shown in Figure 3(a), we adopt a hardware-friendly
205
      sliding-window mask as our base sparse attention mask (Beltagy et al., 2020). Following previous
      work (Xiao et al., 2024c; Han et al., 2023), the initial few tokens (64 tokens for MoA) are not masked.
206
      The attention span equals the sliding-window-span plus the number of initial unmasked tokens. We
207
      define the attention span S of head h at input length N using a straightforward linear function:
208
                                                Sh = αh + βh · N,                                         (2)
209
210   where αh and βh are hyperparameters that control the base span and its expansion rate with input
211   length of a specific attention head.
212   The α and β hyperparameters for each attention head are chosen from multiple discrete options. By
213   default, MoA uses 6 and 9 options for α and β, respectively. For LLMs with many heads and layers,
214   the search space can become quite large. For example, for a 7B model consisting of 32 attention
215   heads and 32 layers, the potential search space expands to 541024 configurations. Thus, we design
      the automatic pipeline to efficiently pinpoint the optimal αs and βs for any LLM.

      4     AUTOMATIC P IPELINE FOR M OA C OMPRESSION
233
      This section outlines the MoA automatic compression pipeline as shown in Figure 3(b). Starting
234
      with a trained LLM and a calibration dataset, MoA first profiles the influence of each attention
235
      value on the model’s prediction loss for various input sequences from the calibration dataset. The
236   masked sum of the influences represents the accuracy loss associated with each mask at different
237   input lengths, showing the accuracy loss each candidate elastic rule could cause at that length. Then,
238   MoA optimizes the compression plan by selecting the optimal elastic rule for each head, which
239   minimizes the accuracy loss across various lengths while adhering to specified density constraints.
240   The following sections provide detailed discussions of each step in this pipeline.
241

291

297   solutions as the final plan.

321   5     DATASET AND S UPERVISION
322
323   In this section, we highlight the overlooked importance of calibration dataset design and its supervision
      objective in LLM compression. Calibration datasets are essential for sensitivity analysis across various


                                                          6
324
      compression techniques, including weight pruning (Men et al., 2024; Lee et al., 2024; Liu et al.,
325   2023b) and quantization (Lin et al., 2023; Xiao et al., 2024b; Li et al., 2024a; Kim et al., 2023).
326   In this work, MoA profiles the attention influence on the calibration dataset, which is crucial for
327   subsequent automatic optimization.
328
      Current Approach. General language modeling datasets, such as human-written text corpus RedPa-
329
      jama (Computer, 2023), are commonly used as the calibration dataset. These datasets, supervised
330
      by next-token-prediction on the entire corpus, primarily capture attention patterns coherent with
331   immediately preceding tokens. However, they lack long context dependencies, failing to address the
332   global attention crucial for tasks like long-range retrieval.
333
334
      Moreover, a notable misalignment exists between the model response and the human-written supervi-
      sion. Consequently, it leads to inaccuracies when using human responses to compute attention values
335
      and gradients during profiling. For example, given the same question, a human might answer ‘Blue’,
336
      while the model could generate ‘The blue color’. Using the human answer for supervision, attention
337   influence is inaccurately quantified based on probability shift for predicting ‘Blue’; this diverges
338   from the objective of maintaining crucial attention for the original model prediction, ‘The’. These
339   inconsistencies arise from various factors, including mismatched positions, tones, and synonyms.
340
      MoA’s Approach. MoA enhances the calibration dataset by integrating long-range dependencies and
341
      model alignment. Specifically, we utilize the long-contextual MultiNews dataset (Fabbri et al., 2019),
342
      which includes summaries heavily dependent on long-range content. The summaries are generated by
343   the original dense model and serve as supervision. Compared to current approaches that adopt human
344   responses as the reference to calculate the cross-entropy loss L, using the responses generated by the
345   original model as the supervision can facilitate accurate profiling, thus benefiting the compression.
346
      Approach Comparison. We evaluate our design’s effectiveness by varying dataset choices, supervi-
347
      sion types, and summary references, while standardizing data item count and length to 50 and 8k
348
      words, respectively. Additional setups and evaluations are in Appendices A and B.3.1.
349
350   We show the importance of long-range dependencies by comparing the MoA compression plan
351   generated with different datasets and supervisory methods. In Table 2, RedPajama (Computer, 2023)
352
      represents the general language modeling dataset, while MultiNews (Fabbri et al., 2019) highlights
      long-range contexts by aggregating multiple documents on a single incident. Additionally, each
353
      MultiNews item includes a human-written summary, providing even stronger long-range dependencies
354
      and better performance. Calculating loss on the summary of MultiNews leads to significantly better
355   performance, with a 60% increase in retrieval accuracy and a 0.98 decrease in perplexity.
356
357   Furthermore, using summaries generated by the original dense model as supervision promotes higher

522
      We investigate MoA’s elastic rules for each head. As shown in Figure 11, masks in the initial and
523
      middle layers exhibit high density, aligning with the conclusions from previous research on LLM’s
524
      intrinsic dimensions (Valeriani et al., 2023) and layer sensitivities (Yuan et al., 2023). Conversely, in
525   the final layers, most heads require low density, while few need high density. Figure 12 shows that
526   layers with lower average density typically display more diverse densities among heads, confirming
527   the need for heterogeneity within the same layer. Further details and insights are in Appendix C.

      7     C ONCLUSION AND F UTURE W ORK
531
      MoA automates the selection of heterogeneous elastic masks for each attention head and input length,
532
      significantly extending the effective context length of LLMs by 3.9×. It enhances retrieval accuracy
533   by 1.5× to 7.1× over uniform sparse attention method and increases throughput to over 7× at 50%
534   average density, maintaining performance on par with dense models in rigorous benchmarks.
535
536
      Limitations and Future Work. Under an extremely low-density budget, MoA fails to maintain
      good performance. Designing a dynamic MoA method has the potential to address this issue, which
537
      we leave for future work. Using non-linear elastic rules with bounded attention spans is also worth
538
      exploring. Additionally, MoA’s profiling method can be adapted to evaluate the influence of weights
539   and other activations, facilitating other compression methods such as quantization.

865
866   A.1   M AIN S ETUP
867
868   Baselines. In the setup for our experiment, we adhere to specific configurations outlined in the
869   respective papers. In the case of StreamingLLM (Xiao et al., 2024c), the initial four tokens remain
870   unmasked, serving as the attention sink, except for the 70b model in Table 4 and the super long
871   setting in Figure 5, where we use 64 tokens as the attention sink. For InfLLM (Xiao et al., 2024a), we
872   adhere to the original configuration by maintaining the same local window size and selected memory
873
      size, using 128 initial tokens as specified in their setup. For H2O (Zhang et al., 2023), we ensure the
      same number of heavy hitter tokens and recent tokens. Note that H2O uses dense prefill since it relies
874
      on the column sum of the attention matrix to calculate the importance of every token for KV-Cache
875
      eviction. StreamingLLM, InfLLM and MoA use sparse prefill.
876
877   Models and Benchmarks. Since vicuna-7b-v1.5-16k and vicuna-13b-v1.5-16k (Chiang et al., 2023)
878   can only take in 16k context length, we use the 16k split of LV-Eval benchmark (Yuan et al., 2024),
879
      truncating the input to 15500 for model input in Table 4. For the LongBench benchmark (Bai et al.,
      2023), we use the LongBench-E split, which features a balanced number of data items at every length
880
      level. The LongBench dataset is segmented into ranges of 0-4k, 4-8k, and 8k+ tokens. We test each
881
      split using the input length truncation thresholds of 3,500, 7,500, and 15,500 tokens, respectively.
882
883   Perplexity Evaluation. We construct a comprehensive yet concise test set by sampling 50 × 4
884   data items for each length level from the test split of four long-context understanding datasets:
885
      Qasper (Dasigi et al., 2021), MultiNew (Fabbri et al., 2019), TREC (Li & Roth, 2002; Hovy et al.,
      2001) and LCC (Mohler et al., 2016), representing the question answering, summarization, few-
886
      shot learning, and code completion abilities of the LLM. Following LongBench, the data items are
887
      organized as question-answer pairs. The questions and answers are written by humans and come
888   with the dataset. The perplexity is calculated solely on the answer part of the data, demonstrating the
889   model’s coherence in responding to user requests.
890
891
      Validation Dataset. The validation dataset is used to select the optimal compression plan among the
      Pareto front solutions during the optimization step. The validation dataset is similarly constructed as
892
      the perplexity test dataset, but on the respective validation split of the datasets. 50 × 4 data items
893
      are sampled from the same four long-context understanding datasets: Qasper (Dasigi et al., 2021),
894   MultiNew (Fabbri et al., 2019), TREC (Li & Roth, 2002; Hovy et al., 2001) and LCC (Mohler et al.,
895   2016). The additional 50 data items from the LongEval (Li et al., 2023a) dataset are also added to
896   validate the retrieval ability. For the datasets that do not contain the validation split, namely TREC,
897   MultiNews and LCC, we sample from the test split and ensure different data items with the perplexity
898   evaluation dataset.
899
      MoA Settings. MoA uses the block sparse attention pattern with a block size of 64, where each grid
900   depicted in Figure 3(a) represents a block. The first block of tokens is not masked as the attention
901   sink. For the profile stage, we use the MultiNews (Fabbri et al., 2019) calibration dataset with
902   model response as supervision, as described in Section 5. We use 50 × 3 data items at 2k, 4k, 8k
903   lengths. The data items are padded to their corresponding length level in order to ensure a unified
904   shape of attention influence tensors for each length level. We adopt block granularity during the
905   profiling stage, calculating the average attention influence within each block to represent the block’s
906   overall influence. For hyperparameter search space α and β, we use 6 values for α and 9 values
907   for β, creating a search space of 54 pairs for each attention head. α is uniformly sampled from the
908
      range [−2048, 8192], and β is uniformly sampled from [0, 1]. The resulting attention span lengths
      are clipped to the range between 0 and the current input length. The optimization is done with the
909
      multi-objective optimization at the same set of lengths. We limit the number of distinct rules to at
910
      most two per model layer to ensure inference-time efficiency. Among the Pareto front solutions, we
911   select the one with the lowest perplexity on the validation dataset of length 12k.
912
913   A.2   E FFICIENCY E XPERIMENT S ETUP
914
915   We test the efficiency of different frameworks using a single NVIDIA A100-SXM4-80GB GPU.
916   To improve the runtime profiling accuracy, we first run five forward passes as warmups. Then we
917   use torch.CudaEvent to calculate the runtime for each method. Our experiments are structured
      around three scenarios: including prefilling 3k tokens and decoding 1k tokens; prefilling 6k tokens


                                                        17
918
      and decoding 2k tokens; prefilling 12k tokens and decoding 4k tokens. The labels are marked by the
919   total sequence length, which equals prefill length plus decode length.
920
921   For MoA, The implementation is based on Huggingface Transformers. During the prefill stage,
922
      we use the sparse CUDA kernel designed by us with block size 64. During the decode stage, we
      modify the KV-Cache implementation to support our heterogeneous elastic rules. Thanks to our fixed
923
      sliding-window span during the decode stage, we simply replace the old KV-Cache that exceeds the
924
      span with the latest KV-Cache. Our custom decoding CUDA kernel then handles KV-Cache with
925   varying lengths across different attention heads during the decoding process.
926
927   For H2O, we use its official efficient implementation, which is based on Flexgen (Sheng et al., 2023).
928
      Note that H2O uses dense prefill since it relies on the column sum of the attention matrix to calculate
      the importance of every token for KV-Cache eviction, which requires the attention matrix to be
929
      explicitly calculated. It makes H2O’s prefill stage currently incompatible with kernel optimizations
930
      like FlashAttention. Therefore, H2O is easy to get OOM (Out-Of-Memory) with large prefill length
931   and increased batch size.
932
933   In our efficiency tests across all frameworks, we implemented a simple optimization at the language
934
      modeling head (lm head) during the prefill stage. Specifically, after the final layer of the transform-
      ers, we compute the logits—these are the raw outputs that are transformed into probabilities—for
935
      only the last token. This selective computation avoids generating these probabilities for preceding
936
      tokens, substantially reducing both computational overhead and memory usage. We also set the
937   environment variable PYTORCH_CUDA_ALLOC_CONF to be expandable_segments:True
938   for Hugginface and MoA to mitigate memory fragmentation, allowing larger inference batch size.
939
940
      Following the performance experiments, we use Vicuna-7B and Vicuna-13B for efficiency tests
      whenever possible. However, the official efficient implementation of H2O based on Flexgen only
941
      supports OPT (Zhang et al., 2022). Therefore, we use OPT-6.7b and OPT-13b models for H2O in
942
      Table 11 for comparison.

1387   2021), MultiNews (Fabbri et al., 2019), and TREC (Li & Roth, 2002; Hovy et al., 2001). We use
1388   their training set to construct the calibration dataset, and use their respective test set in LongBench to
1389   calculate the score. Following Section 5, all calibration datasets are constructed using the original
1390   model’s response to the context and questions as the supervision.
1391
       As shown in Table 15, we find that as long as the calibration dataset conforms to the long-range
1392   dependency and model alignment highlighted in section 5, the specific choice of the dataset is less
1393   important. Calibration datasets with long dependency and model alignment show somewhat similar
1394   test results on various datasets. Additionally, they all show strong generalization power to test sets
1395   other than their respective calibration dataset.
1396
       In contrast, the RedPajama dataset without long-range dependency and model alignment shows
1397
       large variance on various test sets. It also differs from the performance of the original dense model,
1398   which may incur unexpected behaviors after compression. Note that though all datasets exhibit long
1399   dependency, the questions in the TREC dataset can be answered without long context. The context
1400   in the TREC dataset of LongBench is the many-shot examples, each showing a short sentence and
1401   its classification result, while the question is to classify a new short sentence. Although the context
1402   helps to determine the complete set of 50 classes, the model can also directly clarify the sentence
1403   without any context based on common knowledge. It may contribute to a high score on the TREC
       test set with the RedPajama calibration dataset.

1687   Current work RazorAttention (Tang et al., 2024a) also proposes to use heterogeneous attention for
1688   better performance. MoA distinguishes from it in the following aspects:
1689
1690         1. Strategies for attention heads: RazorAttention categorizes attention heads into two types:
1691            retrieval and non-retrieval. It adopts bipolar strategies, applying either full attention or
1692            fixed-sized local attention. In contrast, MoA recognizes the diverse attention spans of
1693            different heads and employs a broader range of strategies, covering the entire spectrum from
1694
                very limited local attention to full attention.
1695         2. Adaptation to different input lengths: RazorAttention uses a fixed density for non-retrieval
1696            heads across all input lengths. MoA, on the other hand, applies heterogeneous elastic rules
1697            for each head, dynamically adjusting densities based on input length while maintaining
1698
                overall density constraints.
1699         3. Determination of strategies: RazorAttention relies on a heuristic approach to assign
1700            strategies. It uses attention scores between the current token and specific tokens (e.g., echo
1701            and induction tokens) to identify retrieval heads and assign full attention. MoA employs a
1702
                loss-based method. It uses a first-order Taylor expansion to estimate the impact of a strategy
                on end-to-end prediction loss and optimizes strategies to minimize this loss under density
1703
                constraints.
