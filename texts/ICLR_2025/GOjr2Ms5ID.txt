009                                               A BSTRACT
010
011            Recent studies have demonstrated that learned Bloom filters, which combine ma-
012            chine learning with the classical Bloom filter, can achieve superior memory effi-
013            ciency. However, existing learned Bloom filters face two critical unresolved chal-
014            lenges: the balance between the machine learning model size and the Bloom filter
015            size is not optimal, and the reject time cannot be minimized effectively. We pro-
016
               pose the Cascaded Learned Bloom Filter (CLBF) to address these issues. Our
               optimization approach based on dynamic programming automatically selects con-
017
               figurations that achieve an optimal balance between the model and filter sizes
018
               while minimizing reject time. Experiments with real-world datasets show that
019            CLBF reduces memory usage by up to 24% and decreases reject time by up to 14
020            times compared to the state-of-the-art learned Bloom filter.

023   1   I NTRODUCTION
024
025   Bloom filter (Bloom, 1970) is a ubiquitous data structure for approximate membership queries. By
026   compressing a set S into a bit array, the Bloom filter can determine whether a query q belongs to the
027
      set S, with a small probability of false positives. Due to its high memory efficiency and fast query
      performance, Bloom filters have been employed in various memory-constrained and/or latency-
028
      sensitive applications (Broder & Mitzenmacher, 2004; Chang et al., 2008). Recently, a new class
029
      of Bloom filter called Learned Bloom Filter (LBF) has been proposed (Kraska et al., 2018). LBFs
030   leverage a machine learning model that predicts whether the input belongs to the set S and achieves
031   a superior memory efficiency compared to traditional Bloom filters. Despite numerous attempts to
032   further improve memory efficiency (Mitzenmacher, 2018; Dai & Shrivastava, 2020), existing LBFs
033   still face two critical unresolved issues: (1) the balance between the machine learning model size
034   and the Bloom filter size is not optimal, and (2) the reject time cannot be minimized effectively.
035
      (1) The Balance between Machine Learning Model Size and Bloom Filter Size: Existing LBFs
036   lack mechanisms to automatically adjust the balance between the machine learning model and
037   Bloom filter sizes. LBFs consist of a machine learning model and one or more Bloom filters, aiming
038   to minimize the total memory usage, i.e., the sum of memory consumed by the machine learning
039   model and the Bloom filters. While higher model accuracy can reduce the memory for Bloom filters,
040   optimizing the model size is challenging due to the complex relationship between model size and ac-
041   curacy. Even when increased model size improves accuracy, the balance between increased model
042   size and reduced Bloom filter size must be evaluated. Current LBFs focus solely on optimizing
043   Bloom filter configurations for a fixed model, neglecting model size optimization.
044   (2) The Reject Time: Existing approaches do not provide automatic methods for minimizing reject
045   time in LBFs. The time Bloom filter takes to reject, i.e., answer “q ∈   / S,” is often more important
046   than the time it takes to accept, i.e., answer “q ∈ S.” This is because the accept response may be a
047   false positive, so a heavy verification process is often required afterward, whereas the reject response
048   is always correct, so such a verification process is not required. The reject time is more likely to
049   be a bottleneck than the accept time, so it is important to reduce the reject time of LBF. However,
050
      the prior research on LBFs (Fumagalli et al., 2022; Malchiodi et al., 2024) provides only heuristic
      guidelines to reduce reject time, and does not propose any automatic method for minimizing reject
051
      time.
052
053   To address these issues, we propose a novel learned Bloom filter with a cascaded structure called
      Cascaded Learned Bloom Filter (CLBF). CLFB has two key features: (1) CLBF achieves an optimal

      memory size balance by training a larger machine learning model and then appropriately reducing
055   it. (2) CLBF enables fast rejections through branching based on tentative outputs of the machine
056   learning model and the insertion of intermediate Bloom filters. We can set the hyperparameters to
057   control the trade-off between memory efficiency and reject time. Based on dynamic programming,
058   our optimization approach automatically adjusts the CLBF to the optimal configuration for the given
059   hyperparameter. Our experiments demonstrate that (1) CLBF reduces memory usage by up to 24%
060   compared to the Partitioned Learned Bloom Filter (PLBF) (Vaidya et al., 2021), the state-of-the-art
061   LBF for memory efficiency, and (2) CLBF reduces reject time by up to 14 times compared to PLBF.

064   2   R ELATED W ORK
065
066   Bloom filter (Bloom, 1970) is one of the most fundamental and widely used data structures for
067   approximate membership queries. Bloom filters can quickly respond to queries by only perform-
068   ing a few hash function evaluations and checking a few bits. Although Bloom filters may return
069   false positives, they never yield false negatives. This property makes Bloom filters valuable in
070
      memory-constrained and latency-sensitive scenarios such as networks (Broder & Mitzenmacher,
      2004; Tarkoma et al., 2011; Geravand & Ahmadi, 2013) and databases (Chang et al., 2008; Goodrich
071
      & Mitzenmacher, 2011; Lu et al., 2012). The theoretical lower bound on the number of bits required
072
      for data structures supporting approximate membership queries is known to be n log2 (1/ε), where
073   n is the number of elements in the set, and ε is the false positive rate (Pagh et al., 2005). Bloom
074   filters use log2 (e) · n log2 (1/ε) bits, i.e., they require log2 (e) times more memory than the theo-
075   retical lower bound. Several improved versions of the Bloom filter, such as the Cuckoo filter (Fan
076   et al., 2014), Vacuum filter (Wang & Zhou, 2019), Xor filter (Graf & Lemire, 2020), and Ribbon
077   filter (Dillinger & Walzer, 2021), have been proposed to get closer to the theoretical lower bound.
078
      Recently, the concept of Learned Bloom Filter (LBF), which further enhances the memory efficiency
079   of Bloom filters using machine learning models, was introduced (Kraska et al., 2018). They pro-
080   posed an LBF that uses a machine learning model, which predicts whether the input is included in
081   the set S, as a pre-filter before a classical Bloom filter (Figure 1(a)). In this LBF, elements predicted
082   by the model as included in the set S are not inserted into the classical Bloom filter, while those
083   predicted as not included are. When this LBF answers a query, it immediately answers q ∈ S if the
084   model predicts the query is in the set. In contrast, if the model predicts the query is not in the set,
085   this LBF uses the classical Bloom filter. This design reduces the number of elements stored in the
086   Bloom filter, thus reducing the total memory usage. Numerous subsequent studies have sought to
087
      improve this structure further. For example, the sandwiched LBF (Mitzenmacher, 2018) sandwiches
      the machine learning model with two Bloom filters (Figure 1(b)). it is demonstrated that the mem-
088
      ory efficiency is further improved by optimizing the size of the two Bloom filters. Ada-BF (Dai &
089
      Shrivastava, 2020) and the Partitioned Learned Bloom Filter (PLBF) (Vaidya et al., 2021) further
090   enhance memory efficiency by utilizing the score, which is the prediction of the machine learning
091   model regarding the likelihood that an input element is included in the set (Figure 1(c)). These
092   LBFs employ multiple Bloom filters with different false positive rates, selecting the appropriate fil-
093   ter based on the score. This approach allows for a more continuous and fine-grained utilization of
094   the model predictions.
095
      While most research on LBFs has focused on the optimal configuration of Bloom filters for a fixed
096   trained machine learning model, some studies have investigated the choice of the machine learning
097   model itself (Fumagalli et al., 2022; Dai et al., 2022; Malchiodi et al., 2024). These studies evaluate
098   various machine learning models and LBF configuration methods (such as sandwiched LBF and
099   PLBF) across different datasets, measuring memory efficiency and reject times. The results suggest
100   that the optimal machine learning model varies depending on dataset noisiness, ease of learning, and
101   the importance of minimizing reject time. However, these studies provide only general guidelines
102   based on observed trends, and no method has yet been proposed for automatically selecting the
103   optimal machine learning model.
104   Optimizing the hash functions is another approach to improving accuracy using key and non-key
105   data. Hash Adaptive Bloom Filter (HABF) (Xie et al., 2021b) uses a lightweight data structure
106   called HashExpressor to select suitable hash functions for each key, and Projection Hash Bloom
107   Filter (PHBF) (Bhattacharya et al., 2022a) employs projections as hash functions. Unlike LBFs,
      these approaches avoid classifier training and instead pack information into a lighter structure.

143   This section describes the architecture and construction method of our proposed Cascaded Learned
144   Bloom Filter (CLBF). In Section 3.1, we describe the architecture of CLBF. Then, we formulate the
145
      problem of constructing CLBF in Section 3.2, and the optimization method for configuring CLBF
      using dynamic programming is described in Section 3.3.

148
      3.1    A RCHITECTURE OVERVIEW
149
      CLBF employs a cascade structure consisting of multiple machine learning models and multiple
150   Bloom filters arranged alternately (Figure 2). The structure of CLBF, where models are aligned
151   sequentially, is similar to ensemble learning techniques such as boosting (Freund & Schapire, 1997;
152   Friedman, 2001). In fact, the machine learning models used in our experiments (Section 4) are weak
153   learners trained using a boosting algorithm. By automatically determining the appropriate number
154   of machine learning models (weak learners) and adjusting the false positive rates of each Bloom
155   filter, CLBF not only balances model and filter sizes but also shortens the reject time (as detailed in
156   Section 3.2 and Section 3.3).
157   Now, we explain how the CLBF processes a query. A query q first enters the initial Trunk Bloom
158   filter, TBF1 . If TBF1 determines that q ∈    / S, CLBF immediately concludes q ∈     / S. Otherwise,
159   the query is passed to the first machine learning model, ML1 , which outputs a score indicating
160   how likely q is included in S. If this score exceeds a threshold θ1 , a subsequent Branch Bloom filter,
161   BBF1 , is used to make the final determination. Otherwise, q is passed to the next Trunk Bloom filter,
      TBF2 . In this way, CLBF alternates between branching based on the tentative score and filtering


                                                                3
      Under review as a conference paper at ICLR 2025


162
      using a Bloom filter. When the query reaches the D-th (i.e., final) machine learning model, CLBF
163   adopts the same approach as PLBF. In other words, multiple thresholds are set, and based on the
164   range in which the query score falls, one of the K Final Bloom filters (FBF1 , FBF2 , . . . , FBFK ) is
165   selected for use. When inserting a key into CLBF, the key is inserted into all Bloom filters traversed
166   by the process outlined above, ensuring the absence of false negatives. The exact algorithms for key
167   insertion and query responses are detailed in Appendix A.
168
      The CLBF architecture generalizes existing LBFs. CLBF with only one Trunk Bloom filter and
169   one Final Bloom filter is equivalent to a sandwiched LBF. Moreover, CLBF that omits both Trunk
170   and Branch Bloom filters while employing multiple Final Bloom filters corresponds to a PLBF. By
171   selecting the optimal configuration under this generalized architecture, CLBF can achieve better
172   memory efficiency and shorter reject time.

      3.2   P ROBLEM F ORMULATION
197
198   The following components must be provided to construct the CLBF: several pre-trained machine
199   learning models, two hyperparameters, and a validation dataset. The machine learning models are
200   trained to perform binary classification between key and non-key using all or part of the sets S
201   and Q. The number of pre-trained models is denoted by D̄. Among these, only the first D (≤ D̄)
202   models are used in the CLBF, where D is a parameter optimized during CLBF construction. The two
203   hyperparameters are F (∈ (0, 1)), which represents the target false positive rate for the CLBF, and
204   λ (∈ [0, 1]), which controls the trade-off between memory efficiency and reject time. The CLBF is
205   optimized to minimize memory usage and reject time, subject to the constraint that the expected false
206
      positive rate does not exceed F . As λ increases, greater emphasis is placed on memory efficiency;
      specifically, when λ = 1, only memory efficiency is optimized, while when λ = 0, only reject time
207
      is minimized.
208

      5    L IMITATION
515
      Although our method is highly compatible with learning models composed of multiple weak learn-
516
      ers, such as boosting, it cannot be directly applied to models composed of a single large learner (e.g.,
517   a single deep learning model). To extend our approach for such models, it is necessary to introduce
518   an additional mechanism that outputs tentative scores from intermediate layers. A critical challenge
519   is mitigating the memory and time overhead introduced by this mechanism. Furthermore, a novel
520   optimization algorithm tailored to this framework may be required, and promising directions include
521   leveraging advanced generic optimization techniques, such as Bayesian optimization or Adam.
522
      Additionally, our current optimization method does not always select the optimal intermediate layer
523   thresholds, i.e., θ. While our experimental results demonstrate that selecting the best-performing
524   thresholds from a set of candidates is sufficient to outperform existing LBFs in terms of memory
525   efficiency and reject time, further improvements in threshold optimization could lead to even greater
526   performance gains. Furthermore, while we currently discretize the false positive rates of Trunk
527   Bloom filters, i.e., f (t) , at a certain granularity, future work could achieve better results by exploring
528   more precise solutions.
529
      Moreover, the configuration time of our approach is longer compared to existing methods such as
530   sandwiched LBF or PLBF, which may pose a problem for specific applications. Therefore, acceler-
531   ating the optimization process is another important direction for future work.

      6    C ONCLUSION
535
536
      In this research, we proposed CLBF, solving two critical issues existing LBFs face. (1) By training a
      large machine learning model and reducing it optimally, CLBF achieves an optimal balance between
537
      model and filter sizes. (2) By branching based on tentative scores and the insertion of intermediate
538
      Bloom filters, CLBF significantly reduces reject time. As a result, CLBF not only broadens the
539   applicability of LBFs but also establishes a strong foundation for addressing these issues.

680

684   Branch Bloom filters by the final depth d = D, it is inserted into the appropriate Final Bloom filter
685   based on the final score. This process is repeated for all keys q contained in S, the set stored by the
686   CLBF.

689   certain that q ∈
                     / S, and this result is returned immediately. Otherwise, the output score from ML1 is
690   obtained for q. If this score exceeds the threshold θ1 , the algorithm branches to the Branch Bloom
691   filter; the algorithm queries BBF1 for q, and the result from this filter is used as the final result.
692   Otherwise, q is passed to the next depth d. If q does not branch into any Branch Bloom filters by the
693   final depth d = D, it is queried against the appropriate Final Bloom filter based on the final score.
694   This approach leverages the tentative and final scores of the machine learning models to provide fast
695
      query responses while preserving the false-negative-free property.

902
                      M ODELS
903
904
       Our CLBF improves memory efficiency and reduces reject time by optimizing the configuration, but
905
       it is possible to achieve the same effect by tuning the machine learning model (e.g., hyperparameter
906    tuning, distillation, and quantization). These two approaches can be used at the same time. In other
907    words, it is possible to construct a CLBF with superior performance by the optimization hyperpa-
908    rameters of the machine learning model followed by the optimization of the cascade structure. Here,
909    we show the results of our observations on how the hyperparameters of the machine learning model
910    XGBoost, in particular, the value of max depth, affect the performance of LBF. The smaller the
911    max depth, the smaller each weak learner becomes, and while the size is smaller and inference
912    time is shorter, the discriminative power of each weak learner becomes weaker.
913    The trade-off between memory usage and false positive rate for max depth = 1, 2, 4, 6 is shown
914    in Figure 13, and the trade-off between memory usage and reject time is shown in Figure 14, and the
915    construction time is shown in Figure 15. We confirmed that the properties of CLBF, as shown in the
916    main text, consistently appear for any value of max depth: (1) better memory-accuracy trade-off
917    than existing LBFs, (2) better memory-reject time trade-off than existing LBFs, (3) slightly (up to
       1.8 times) longer construction time than existing LBFs.
