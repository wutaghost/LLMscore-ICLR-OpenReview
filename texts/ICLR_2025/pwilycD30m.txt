011                                              A BSTRACT
012
013            Fine-tuning large language models (LLMs) to adapt them for specialized down-
014            stream tasks is a common practice, yet existing methods overlook a critical issue:
015
               label preference discrepancies among different annotators. Such inconsistencies
               in labeling can significantly impair the model’s robustness and generalization. In
016
               this work, we propose Dynamic Cross-Layer Preference Correction (DCPC), a
017
               novel self-supervised learning framework designed to mitigate these inconsisten-
018            cies. DCPC incorporates a preference-sensitive similarity mechanism, cross-layer
019            prefix alignment, and a Preference Correction Module (PCM) to dynamically ad-
020            just embeddings across transformer layers. By leveraging self-supervision, DCPC
021            effectively aligns semantic representations and ensures consistency in label pre-
022            dictions, even in the presence of preference shifts. We evaluate DCPC across
023            multiple tasks using prominent base models and introduce modified datasets that
024            simulate real-world preference shifts. Our results show that DCPC consistently
025            outperforms state-of-the-art Parameter-Efficient Fine-Tuning (PEFT) methods in
026
               handling label preference discrepancies.

      2   R ELATED WORKS
080
081
      Parameter-Efficient Fine-Tuning As LLMs grow, fine-tuning becomes increasingly resource-
082
      intensive(Xin et al., 2024). Parameter-Efficient Fine-Tuning (PEFT) methods, like LoRA (Hu et al.,
083
      2021; Gao et al., 2024) and P-Tuning v2 (Liu et al., 2021), address this by updating only a small
084   subset of parameters, while freezing the rest. Adapter-based methods (Houlsby et al., 2019; Chen
085   et al., 2024) further reduce the parameter footprint by introducing bottleneck layers. While effective
086   across benchmarks, these techniques do not address label preference inconsistencies across datasets.
087
088   Prompt-Tuning Methods Prompt-tuning methods have emerged as a popular approach for adapt-
089   ing LLMs to various tasks without full model fine-tuning. These methods introduce learnable soft
090   prompts that act as task-specific instructions, guiding the model during inference. Prefix-Tuning
091   (Li & Liang, 2021; Liu et al., 2021; Vu et al., 2021; Ouyang et al., 2023) prepend trainable pre-
092   fix embeddings to input sequences and internal layers, significantly improving model performance
093   while reducing computational costs. Other variations modify internal components like attention
094   mechanisms or bias terms (Tan et al., 2024). However, these techniques do not account for labeling
095
      inconsistencies across datasets. Our DCPC framework addresses this gap by dynamically adjust-
      ing prefix embeddings based on preference-sensitive similarity and cross-layer alignment, offering
096
      a more robust solution for handling heterogeneous datasets with varying annotation styles.
097
098
099
      Learning with inconsistent labels To the best of our knowledge, no existing work in the fine-
      tuning of LLMs has addressed the issue of inconsistent labels. In the context of traditional small- and
100
      medium-scale deep learning models, inconsistent labels have already posed a significant challenge
101
      for real-world applications (Rodrigues & Pereira, 2018; Chen et al., 2020). Several methods have
102   been proposed, such as inferring the unknown true label of each instance from multiple noisy labels
103   (Zhang et al., 2014). Majority Voting (MV) (Raykar & Yu, 2012) is a commonly used technique,
104   which assumes that the labeling quality is balanced across the dataset—an assumption that is often
105   unrealistic. Other approaches, such as RSVMI (Yang et al., 2023), LAWMV (Chen et al., 2022),
106   and AALI (Zheng et al., 2021), utilize instance-specific features. However, these methods struggle
107   when applied to LLM fine-tuning, where subjective or domain-specific annotations introduce more
      complex label inconsistencies, reflecting inherent biases or ambiguities rather than simple noise.

110   3.1   P RELIMINARY
111
112   P-Tuning v2 is a parameter-efficient tuning technique designed for large pre-trained language mod-
113   els. Instead of fine-tuning the entire set of model parameters, P-Tuning v2 optimizes a small set
114   of continuous task-specific prefix embeddings that are inserted into each layer of the transformer
115   model. These prefix embeddings act as learnable prompts that guide the model in adapting to new
116   downstream tasks while keeping the majority of the model parameters frozen.
117   For a given input sequence x = {x1 , x2 , . . . , xn }, the model first computes its embedding represen-
118   tation at each layer. Let elx ∈ Rd denote the embedding representation of x at transformer layer l,
119   where d is the embedding dimension. In P-Tuning v2, a prefix embedding Plx ∈ Rm×d is learned,
120   where m represents the length of the prefix. These prefix embeddings are prepended to the input
121   sequence embeddings before being fed into the transformer layers.
122   The modified input for layer l, combining both the prefix and the original token embeddings, is
123   represented as:
124
125
                                                  ẽlx = [Plx ; elx ]                                     (1)
126
127   where [Plx ; elx ] denotes the concatenation of the prefix embedding Plx and the input embedding elx
128   along the sequence dimension.
129   The transformer layer processes this augmented input using the self-attention mechanism and feed-
130   forward network, updating the hidden representations at each layer. The output of the transformer
131   layer l, denoted as hlx , is computed as:
132
133
                                       hlx = TransformerLayerl ([Plx ; elx ])                             (2)
134
135   During training, the prefix embeddings Plx are learned for each layer l, while the pre-trained trans-
136   former model parameters are frozen.

186   3.2   OVERALL
187
188   The proposed framework, referred to as Dynamic Cross-Layer Preference Correction (DCPC) and
189   illustrated in Figure 3, addresses label preference inconsistencies through a structured sequence
190   of steps. For two input sequences xA and xB , the model computes their respective embeddings
191   elA ∈ Rd and elB ∈ Rd at transformer layer l, where d is the embedding dimension.
192
      The DCPC framework first evaluates the similarity between embeddings elA and elB using a
193
      preference-sensitive similarity mechanism. If cos(θ) between elA and elB is high but LA ̸= LB ,
194
      it indicates a preference inconsistency, triggering corrective mechanisms.
195
196   Next, when significant discrepancies between labels LA and LB arise, the cross-layer prefix align-
197   ment mechanism is activated. This ensures alignment between both the prefix embeddings PlA ,
198   PlB , and token embeddings TlA , TlB across all layers l. The goal is to maintain consistency be-
199   tween representations by minimizing differences in PlA and PlB through each transformer layer. In
200   parallel, the Preference Correction Module (PCM) dynamically adjusts prefix embeddings Pnew by
201
      leveraging a meta-matrix M ∈ Rm×d , where m is the prefix length. New prefixes are computed
      based on auxiliary classifier outputs and preference distributions p(eA ), p(eB ). The PCM minimizes
202
      discrepancies via adjustments to Pnew , ensuring alignment in the preference space.
203
204   This integration of cross-layer prefix alignment and preference correction dynamically corrects in-
205   consistencies, making DCPC an efficient framework designed to optimize both inter-layer consis-
206   tency and preference-driven discrepancies.

266   3.5   P REFERENCE C ORRECTION M ODULE (PCM)
267
268   Simultaneously, when Lambiguity exceeds τambiguity , the Preference Correction Module (PCM) is acti-
269   vated to adjust the prefix embeddings and resolve label preference discrepancies. The PCM consists
      of two key components: auxiliary classifiers and a meta-matrix M. The auxiliary classifiers are


                                                         5
      Under review as a conference paper at ICLR 2025


270
      responsible for predicting preference distributions for the input embeddings. The meta-matrix M
271   stores learned patterns related to preference correction and is used to generate new prefix embed-
272   dings for each layer.
273
274   Auxiliary Classifier Predictions For each input embedding eA and eB with dimension Rd , the
275   auxiliary classifier predicts two parameters, µ ∈ Rd (mean) and σ ∈ Rd (variance), which represent
276   the distribution of preferences for each embedding. The shared parameters ensure consistency across
277   transformer layers. These parameters are predicted as:
278
279                    (µA , σA ) = AuxClassifier(eA ),       (µB , σB ) = AuxClassifier(eB )            (9)
280
281   Here, µA ∈ Rd and σA ∈ Rd are the predicted mean and variance for input A, while µB ∈ Rd and
282   σB ∈ Rd are for input B.
283   Once we have the predicted parameters, we sample a random noise vector ϵ ∈ Rd from a standard
284   normal distribution. Using these elements, we construct a preference distribution based on the pre-
285   dicted mean and variance. To ensure the stability and smoothness of this distribution, we apply a
286   softmax operation:
287
288                                   ppref (µ, ϵ) = softmax(µ + σ · ϵ) ∈ Rd                           (10)
289
290   Here, σ ·ϵ ∈ Rd introduces variability into the distribution by perturbing the predicted mean µ ∈ Rd .
291
292   Generation of New Prefix Embeddings The normalized preference distribution ppref (µ, ϵ) ∈ Rd
293   is then combined with the meta-matrix M ∈ Rm×d , where m represents the length of the new prefix
294   embedding, and d is the dimension of the embeddings. The new prefix embeddings Pnew ∈ Rm are
295   generated via matrix multiplication:
296
297                                           Pnew = M · ppref (µ, ϵ)                                  (11)
298
299   Here, M ∈ Rm×d is multiplied by ppref (µ, ϵ) ∈ Rd , resulting in a new prefix embedding Pnew ∈
300   Rm . This operation adjusts the prefix embeddings based on the learned preference patterns encoded
301
      in M and the input embeddings’ distributions.
302
      KL-Divergence Loss for Preference Alignment To ensure that the newly generated prefix
303
      embeddings align with the original embeddings’ preference distributions, we introduce a KL-
304
      divergence loss. This loss penalizes the divergence between the predicted preference distributions
305   for eA and eB :
306                                LKL = DKL (pA (µA , σA )∥pB (µB , σB ))                          (12)
307
308   The KL-divergence ensures that the distributions for the two inputs become closer, leading to more
309   aligned prefix embeddings across layers.
310   Finally, The overall loss function is defined as:
311
312
                                    Ltotal = λ1 Lambiguity + λ2 Lalign + λ3 LKL                        (13)
313
314   where λ1 , λ2 , and λ3 are hyperparameters controlling the relative importance of each loss term. The
315   objective is to minimize label inconsistency while maintaining alignment across embedding layers
316   and correcting for label preference discrepancies.

351   Implementation Details All experiments are conducted using NVIDIA A100. For our main ex-
352   periments, we fine-tune the LlaMA-2 models(Touvron et al., 2023), specifically the LlaMA-2 7B
353   and LlaMA-2 13B models, as the backbone for the DCPC framework. We also conducted ablation
354   experiments on Mistral-7B(Jiang et al., 2023). The predictions are generated using the standard
355   language modeling (LM) head provided by the LlaMA-2 models. During inference, we apply beam
356   search with a beam size of 3 to enhance the diversity and quality of generated outputs. The hyper-
357
      parameters of the DCPC framework are set as follows: (a) the length of the prefix embeddings m is
      fixed at 16, (b) the meta-matrix M in the Preference Correction Module (PCM) is configured with
358
      dimensions m × d, where d = 4096 for LlaMA-2 7B and d = 5120 for LlaMA-2 13B, correspond-
359
      ing to the hidden dimension of each model. (c) The cross-layer alignment similarity threshold τcos
360   is set to 0.85, and the ambiguity loss threshold τambiguity is set to 0.3.
361
362   We fine-tune the LlaMA-2 7B and 13B models using the HuggingFace Transformers library. The
363
      maximum sequence length is set to 2048 tokens for both models, and training runs for up to 10
      epochs. The batch size is 16 for smaller datasets (e.g., SST-2 and RTE) and 64 for larger datasets
364
      (e.g., ReCoRD and BoolQ). We employ the AdamW optimizer with an initial learning rate of 1 ×
365
      10−4 , utilizing a linear learning rate decay and a warm-up phase covering 6% of the training steps.
366   Evaluation is performed on the development set every 200 steps, and early stopping is applied if no
367   improvement is observed after 10 evaluations. The best checkpoint based on the development set is
368   used for final testing.

481   4.4   ROBUSTNESS A NALYSIS
482
483   In this section, we analyze the robustness of the proposed DCPC framework by studying the impact
484   of key hyperparameters on model performance. We focus on three primary hyperparameters: (1) the
485   length of prefix embeddings (m), (2) the ambiguity loss threshold (τambiguity ), and (3) the cross-layer
      prefix alignment similarity threshold (τcos ).


                                                         9
      Under review as a conference paper at ICLR 2025


486
      Prefix Length (m): The prefix length m controls the dimensionality of the prefix embeddings
487   inserted into each transformer layer. To study its effect, we vary m from 8 to 24 and observe the
488   changes in model performance across datasets. Figure 4 shows that as m increases, the performance
489   improves until saturation is reached at m = 16. Beyond this value, performance either stagnates or
490   slightly declines, suggesting that overlong prefix embeddings may introduce noise and reduce the
491   model’s ability to capture meaningful preference shifts efficiently.
492
493   Ambiguity Loss Threshold (τambiguity ): The ambiguity loss threshold τambiguity determines when
494   the Preference Correction Module (PCM) is triggered to correct label discrepancies. We experi-
495   ment with τambiguity values ranging from 0.1 to 0.5. As shown in Figure 4, a moderate value of
496   τambiguity = 0.3 yields the best performance. Lower thresholds (e.g., τambiguity = 0.1) result in fre-
497   quent activations of the PCM, potentially over-correcting minor discrepancies, while higher thresh-
498   olds (e.g., τambiguity = 0.5) reduce the corrective impact of the PCM, leading to larger inconsistencies
499   in the final predictions.
500
501   Cosine Similarity Threshold (τcos ): The cosine similarity threshold τcos is critical for determining
502   when embeddings are considered semantically similar enough to trigger the ambiguity loss. We vary
503   τcos from 0.7 to 0.95 to assess its impact on performance. Figure 4 shows that setting τcos = 0.85
504   achieves optimal results. Lower values result in too many similarity comparisons being treated as
505
      high, leading to unnecessary corrective actions, while higher values decrease the number of correc-
      tive interventions, reducing the overall effectiveness of the framework.

525   5   C ONCLUSION AND FUTURE WORK
526
527   In this work, we proposed Dynamic Cross-Layer Preference Correction (DCPC) to address label
528   preference inconsistencies in fine-tuning large language models. DCPC effectively reduces the im-
529   pact of subjective labeling, outperforming state-of-the-art Parameter-Efficient Fine-Tuning (PEFT)
530
      methods across multiple datasets. It improves the alignment of semantically similar inputs, ensuring
      consistent label predictions, while highlighting the challenge of handling human preference shifts
531
      often overlooked in traditional fine-tuning techniques.
532
533   Our findings suggest future directions, including enhancing robustness against systematic biases
534   in human annotations and exploring more efficient methods for aligning label preferences, such
535   as innovations in prefix-tuning or preference correction mechanisms. Additionally, the ethical and
536
      practical implications of addressing label biases, particularly in sensitive domains, warrant further
      exploration.
537
538   Future work should extend DCPC’s applicability to more diverse real-world datasets and refine cor-
539   rective techniques, while also considering the societal and ethical challenges of mitigating subjective
      label biases in AI systems.

827
828   We compare our proposed Dynamic Cross-Layer Preference Correction (DCPC) framework with
829
      full-parameter fine-tuning (Full-FT) and several state-of-the-art Parameter-Efficient Fine-Tuning
      (PEFT) methods.
830
831
      Representation Modification Methods: We include two common representation modification
832
      methods: (1) BitFit (Zaken et al., 2021), which introduces learnable parameters directly into the
833   hidden representations by adding trainable bias terms; (2) (IA)3 (Liu et al., 2022a), which modifies
834   the hidden representations by scaling them using trainable vectors. Both methods keep the trainable
835   vectors fixed across different samples. To adjust the number of tunable parameters, we initialize the
836   vectors in a reduced dimension r′ < dmodel and project them back to dmodel using a learnable matrix.
837   For BitFit, r′ = 8, and for (IA)3 , r′ = 16.
838
839   Adapter-Based Tuning: We include two adapter-based methods as baselines: (1) Houlsby-
840   Adapter (Houlsby et al., 2019), which is configured with a bottleneck dimension of 18, and (2)
841   Learned-Adapter(Zhang et al., 2023b), which is configured with a bottleneck dimension of 36.
842
843
      Prompt-Based Tuning: For prompt-based fine-tuning, we compare against: (1) P-Tuning v2(Liu
      et al., 2021), where the number of soft prompt tokens per layer is set to 64, (2) LPT (Liu et al.,
844
      2022b), which uses a bottleneck dimension of 1024 and a soft prompt length of 64 tokens, and (3)
845
      PEDRO(Xie et al., 2024) involves integrating a lightweight vector generator into each Transformer
846   layer.

848   LoRA and Its Variants: We also consider LoRA (Hu et al., 2021) and its variant
849   AdaLoRA(Zhang et al., 2023a) as baselines. For LoRA, the rank of the low-rank adaptation matri-
850   ces is set to 4. For AdaLoRA, the initial rank is set to 8 per module, and half of the rank budget is
