010                                              A BSTRACT
011
012            With the widespread deployment of long-context large language models (LLMs),
013            there has been a growing demand for efficient support of high-throughput infer-
014            ence. However, as the key-value (KV) cache expands with the sequence length,
015            the increasing memory footprint and the need to access it for each token generation
016
               both result in low throughput when serving long-context LLMs. While various dy-
               namic sparse attention methods have been proposed to speed up inference while
017
               maintaining generation quality, they either fail to sufficiently reduce GPU mem-
018
               ory consumption or introduce significant decoding latency by offloading the KV
019            cache to the CPU. We present S HADOW KV, a high-throughput long-context LLM
020            inference system that stores the low-rank key cache and offloads the value cache to
021            reduce the memory footprint for larger batch sizes and longer sequences. To min-
022            imize decoding latency, S HADOW KV employs an accurate KV selection strategy
023            that reconstructs minimal sparse KV pairs on-the-fly. By evaluating S HADOW KV
024            on a broad range of benchmarks, including RULER, LongBench, and Needle In
025            A Haystack, and models like Llama-3.1-8B, Llama-3-8B-1M, GLM-4-9B-1M,
026            Yi-9B-200K, Phi-3-Mini-128K, and Qwen2-7B-128K, we demonstrate that it can
027
               support up to 6× larger batch sizes and boost throughput by up to 3.04× on an
               A100 GPU without sacrificing accuracy, even surpassing the performance achiev-
028
               able with infinite batch size under the assumption of infinite GPU memory.

      1   I NTRODUCTION
032
033
      Large language models (LLMs) have increasingly demonstrated their ability to scale and handle
034
      long contexts (Microsoft, 2024; Liu et al., 2024a; Achiam et al., 2023; Team et al., 2023), enabling
035
      them to tackle complex tasks like multi-document question answering and information retrieval from
036   extensive contexts of up to 1M tokens (Achiam et al., 2023; Wang et al., 2024b). However, efficiently
037   serving these long-context LLMs presents challenges related to the key-value (KV) cache (Liu et al.,
038   2024b; Ge et al., 2023), which stores previous key-value activations to avoid re-computation. As the
039   KV cache scales with sequence length, its growing memory footprint and the need to access it for
040   each token generation lead to low throughput during long-context LLM inference. To address these
041   issues, KV cache eviction or sparse attention methods have been widely explored.
042   However, existing methods face three primary limitations: accuracy degradation, inadequate mem-
043   ory reduction, and significant decoding latency overhead. KV cache eviction strategies (Zhang et al.,
044   2024d;c) aim to reduce the memory footprint by discarding KV pairs based on specific policies, but
045   they often result in information loss and accuracy degradation in tasks such as multi-turn conversa-
046   tions (Yang et al., 2024b; Tang et al., 2024a). Dynamic sparse attention methods (Tang et al., 2024b)
047   preserve all KV pairs on the GPU and accelerate inference by computing attention with selected KV
048   pairs. However, this line of work does not mitigate the memory footprint, thereby limiting the batch
049   size and preventing accommodation of extremely long contexts (e.g., 1M tokens). A naive solution
050
      based on sparse attention involves offloading the KV cache to the CPU to reduce memory usage
      (Lee et al., 2024a; He & Zhai, 2024). Nonetheless, this approach incurs significant overhead due to
051
      the latency of fetching the selected sparse KV pairs from the CPU during decoding.
052
053   Consequently, an ideal effective system for long-context LLM inference with sparse attention
      should: (i) reduce GPU memory usage, (ii) minimize inference latency, and (iii) maintain accuracy

082   Low-rank Keys and Offloaded Values for Storage: In long-context LLM inference, the quadratic
083
      scaling of attention computation with sequence length makes the linear cost of low-rank decom-
      position during pre-filling negligible, as illustrated in Figure 1c1 . To reduce memory footprint, we
084

085
      the value cache does not exhibit low-rank properties, minimizing memory footprint without sacri-
086   ficing accuracy. During decoding with sparse attention, we employ CUDA multi-streams to overlap
087   the recovery of the selected key cache with the fetching of the corresponding value cache. This
088   approach conceals key cache reconstruction and reduces data fetching overhead by 2× compared to
089   the naive offloading strategy, thereby decreasing the latency of sparse attention during decoding.
090

091

092   lected tokens (i.e. the K of TopK), which we refer to as sparse budgets (1.56%). Our analysis reveals

096   accuracy. As shown in Figure 2, our method outperforms the naive sparse attention approach (Tang
097   et al., 2024b) and achieves higher sparsity, accelerating decoding.

143
144   Token Eviction. To reduce memory footprint, eviction-based strategies keep a fixed size of KV
145
      cache to store the critical token KV pairs and discard unnecessary tokens. StreamingLLM (Xiao
      et al., 2023b) addresses the limitations of window attention by retaining attention sinks and recent
146
      KV pairs. H2 O (Zhang et al., 2024d) introduces a low-cost eviction policy, updating the KV cache
147
      based on cumulative attention scores. LESS (Dong et al., 2024b) accumulates evicted token in-
148   formation by a constant-sized low-rank cache, which allows partial access to previously evicted
149   information, along with tokens maintained by a sparse policy. SnapKV (Li et al., 2024) uses the lo-
150   cal window of prompts to select important tokens for future generations. However, they suffer from
151   performance degradation and information loss since the evicted tokens will never be recovered.
152
153
      Dynamic Sparse Attention. This line of work retains all KV cache but performs dynamic sparse
      attention within selected KV pairs to reduce inference latency. SparQ (Ribar et al., 2023) uses the
154
      norm of the query to decide an important subset of the key cache’s channels to calculate a metric
155
      to select relevant tokens. Quest (Tang et al., 2024b) segments tokens into pages and selects pages
156   by approximating the highest attention within each page. Loki (Singhania et al., 2024) performs
157   principal component analysis on key caches using a calibration dataset, selecting tokens based on
158   attention scores computed in low-dimensional space. TriForce (Sun et al., 2024) combines sparse
159   attention with speculative decoding (Leviathan et al., 2023) for lossless acceleration. InfiniGen
160   (Lee et al., 2024a) offloads the entire KV cache to the CPU and prefetches essential entries using
161   an predefined projection matrix via SVD for KV selection. In contrast, S HADOW KV employs an
      online, prompt-dependent SVD for key cache compression rather than for KV selection.

       3.1                 L OW-R ANK K EYS AND O FFLOADED VALUES FOR S TORAGE
190
191    To reduce memory footprint, the low-rank nature of the KV cache has been explored by recent
192    studies (DeepSeek-AI, 2024; Xu et al., 2024; Chang et al., 2024). However, these methods focus on
193    data-independent decomposition, either requiring training or achieving limited compression rates.
194
195    Observation. In our study, by conducting SVD on the model weights Wk , Wv , the input X, the
196    pre-/post-RoPE key cache, and the value cache of Llama-3.1-8B, we visualize the relative singular
197    value distributions in Figure 1a together with the accuracy in Figure 3a. As we observed, pre-RoPE
198    keys have the lowest rank and can be compressed by 6× without performance degradation.
199    We also identify striking dynamic and static behaviors in low-rank keys between and within se-
200    quences, inspired by a related investigation in FFN layers (Dong et al., 2024a). Analogous to cosine
201    similarity, we define D(H1 , H2 ) = ⟨H1 , H2 ⟩/r to be the similarity metric between low-rank sub-
202    spaces of two rank-r projection matrices, H1 and H2 , where ⟨·, ·⟩ is the Frobenius inner product2 . In
203    our case with truncated SVDs of pre-RoPE keys, let K1 , K2 ∈ Rn×d have rank-r truncated SVDs,

234   3.2    ACCURATE KV S ELECTION FOR FAST D ECODING
235
236   To further reduce the latency overhead in sparse attention, including fetching the selected value
237   cache from the CPU and reconstructing the corresponding key cache, an accurate KV selection
238   method is needed to minimize the sparse KV cache budget while maintaining the accuracy.
239   Observation. We found most post-RoPE key cache exhibits spatial locality, with high cosine sim-
240   ilarity to adjacent tokens, except for a few outliers. To quantify this, we conducted inference experi-
241   ments on 128K contexts. We divided the post-RoPE keys into chunks of eight tokens and visualized
242   the minimum cosine similarity between the chunk’s mean and its key cache, as shown in Figure 3b.
243   The results indicate that, apart from a few outliers, there is generally high cosine similarity, suggest-
244   ing the mean values can serve as landmarks to approximate attention well within normal chunks.
245
246   Analysis. This finding suggests that for the majority of chunks, we can maintain the mean value as
247   compressed landmarks to select minimal important KV pairs (1.56%) accurately during decoding.
248   Outlier chunks, which may contain dense or critical information and are difficult to approximate,
249   are retained to ensure accuracy. Given their relatively small number (0.2–0.3%), storing them on the
250   GPU is feasible without affecting memory capacity. Furthermore, as shown in Figure 3c, considering
251   the temporal locality of the KV cache—meaning that the KV pairs selected by the queries of two
252   adjacent decoding steps have a high repetition rate, a cache policy (Zhang et al., 2024a) can be
253   leveraged to further reduce the latency overhead by 60% during decoding with optimized kernels.

       6                                  C ONCLUSION
533
534   We present S HADOW KV, a high-throughput inference system for long-context LLM inference.
535   S HADOW KV optimizes GPU memory usage through the low-rank key cache and offloaded value
536   cache, allowing for larger batch sizes. It reduces decoding overhead by accurate sparse attention,
537   boosting throughput while maintaining accuracy. Our empirical experiments demonstrate S HAD -
538   OW KV can support up to 6× larger batch sizes and enhance throughput by up to 3.04× on an A100
539   across various long-context models, including Llama-3.1-8B, Llama-3-8B-1M, GLM-4-9B-1M, and
      Yi-9B-200K. S HADOW KV holds great promise for improving long-context LLM inference.

      A.2   Q UANTITATIVE A NALYSIS OF GPU M EMORY S AVINGS
811
812   The GPU memory savings provided by S HADOW KV can be quantitatively analyzed as follows. Let
813   each K or V vector have a size of M bytes, with a sequence length S, a chunk size C, a selected
814   chunk budget K, O outliers, and a pre-RoPE key cache rank r. The GPU memory savings of
815   S HADOW KV can then be expressed as:
816                                                           2SM
817
                            Memory Savings =
                                             SM/C + 2(K + O)C + Sr + rM
818
      For example, assuming M = 1024, C = 8, S = 128K, K = 256, O = 48, r = 160, the memory
819   savings of S HADOW KV is calculated as 7.08×. This result demonstrates that S HADOW KV can
820   theoretically reduce the KV cache memory footprint on the GPU by 7.08× for longer sequences and
821   larger batch sizes.

       A.9   D ETAILED C OMPARISON WITH I NFINI G EN
1002
1003   We provide further clarification on the key distinctions and conduct additional experiments between
1004   S HADOW KV and InfiniGen. These experiments show that S HADOW KV significantly outperforms
1005   InfiniGen across a wide range of downstream tasks.
1006
1007   Differences in SVD Usage. Infinigen performs an offline SVD to get a projection matrix, which is
1008   applied to post-RoPE key and query states for KV selection, while S HADOW KV applies an online,
1009   prompt-dependent SVD directly to the pre-RoPE key cache for compression, not for KV selection.
1010
1011   Methodological Differences. While InfiniGen uses SVD for KV selection, it requires fetching
1012   selected, exact KV pairs from the CPU. In contrast, S HADOW KV only fetches the value cache from
1013   the CPU, reconstructing the key cache from its low-rank storage on the GPU. By overlapping these
1014
       processes, S HADOW KV reduces data-fetch overhead and achieves improved efficiency in KV cache
       management.
1015
1016
       Accuracy Comparison. To empirically validate S HADOW KV’s advantages, we conducted accu-
1017
       racy evaluations. Results confirm S HADOW KV’s effectiveness in maintaining accuracy while op-
1018
       timizing memory usage. Although InfiniGen performs well on simpler tasks like RULER-N-S1, it
1019   shows significant accuracy drops on more complex tasks, such as RULER-N-MK2, RULER-FWE,
1020   LongBench-LCC, and others, where S HADOW KV maintains consistently high accuracy.

1033   We implement the framework based on PyTorch (Paszke et al., 2019; Wolf, 2019) and dedicated
1034   kernels (Thakkar et al., 2023). FlashAttention (Dao et al., 2022; Dao, 2023; Hong et al., 2023) is
1035   used for attention computation and some efficient fused kernels in Flashinfer (Ye et al., 2024) and
1036   vLLM (Kwon et al., 2023) are used, including layer norm. To reduce memory movement and kernel
1037   launch overhead, we fuse some operations into CUDA kernels, including attention approximation,
1038   key cache low-rank reconstruction, value cache fetching, cache mechanism, etc. We leverage multi-
1039
       streams to overlap the reconstruction of key cache and value cache fetching. We set the rank of
       pre-RoPE key cache to 160, chunk size to 8, and sparse KV cache budget to 1.56% for most cases.
