                                                  A BSTRACT
008
009
               For gradient-based machine learning (ML) methods commonly adopted in practice
010
               such as stochastic gradient descent, the de facto differential privacy (DP) technique
011            is perturbing the gradients with random Gaussian noise. Data valuation attributes
012            the ML performance to the training data and is widely used in privacy-aware
013            applications that require enforcing DP such as data pricing, collaborative ML, and
014            federated learning (FL). Can existing data valuation methods still be used when
015            DP is enforced via gradient perturbations? We show that the answer is no with
016            the default approach of injecting i.i.d. random noise to the gradients because the
017            estimation uncertainty of the data value estimation paradoxically linearly scales
018            with more estimation budget, producing estimates almost like random guesses.
019
               To address this issue, we propose to instead inject carefully correlated noise to
               provably remove the linear scaling of estimation uncertainty w.r.t. the budget with
020
               some assumptions on the gradient distribution. We also empirically demonstrate
021
               that our method gives better data value estimates on various ML tasks and is
022            applicable to use cases including dataset valuation and FL.

025   1   I NTRODUCTION
026
027   With growing data privacy regulations (Bukaty, 2019; Council of European Union, 2014) and
028   machine learning (ML) model attacks (Shokri et al., 2017), privacy has become a primary concern
029   in many scenarios such as collaborative ML (Sim et al., 2020; Xu et al., 2021) and federated
030   learning (FL) (McMahan et al., 2017; Yang et al., 2019). Differential privacy (DP) (Dwork & Roth,
031   2014) is commonly adopted as the de facto framework to provide privacy protection for training data
032   with theoretical guarantees. In deep learning, DP is typically achieved by perturbing the gradients
033
      w.r.t. the model’s loss on the training data (Abadi et al., 2016).
034   Data valuation (Ghorbani & Zou, 2019) has received increased interest from providing attribution for
035   parties in collaborative ML (Sim et al., 2022) and identifying data quality for data curation (Ghorbani
036   & Zou, 2019) and data marketplace (Agarwal et al., 2019). Data values are often estimated with
037   sampling-based methods such as Monte Carlo (Castro et al., 2009) on user statistics, e.g. gradients
038   of user data (Ghorbani & Zou, 2019). However, the sensitive nature of the user statistics requires
039
      privacy during data valuation to protect the data of participants in collaboration (Sim et al., 2023)
      or to facilitate interaction between data buyers and sellers in a marketplace (Chen et al., 2023).
040
      Specifically, enforcing DP on the data is desirable in these scenarios for two reasons: 1) DP enjoys
041
      the post-processing immunity (Dwork & Roth, 2014), allowing further access to data without risking
042   privacy leakage; 2) DP offers a natural trade-off between privacy protection and data quality: Data
043   contributors can determine at their discretion the level of information protection at the expense of
044   degraded data quality. While some previous works considered DP in data valuation (Sim et al., 2023;
045   Wang et al., 2023; Watson et al., 2022), they either focused on a limited class of ML models or
046   required a trusted central server, both of which are difficult to satisfy in real-world scenarios (Sim
047   et al., 2022). A natural question arises: Can we circumvent these two challenges in data valuation
048   while enforcing DP?
049   One might think of perturbing gradients on user data before using them for updating a gradient-
050   trained parametric model to overcome the challenges. Unfortunately, we demonstrate that the naive
051   approach of adding i.i.d. Gaussian noise (Dwork & Roth, 2014) to user gradients leads to practically
052   useless data value estimates: The perturbation on the gradients erodes the information they carry,
053   leading to a lowered data value and increased data value estimation uncertainty because the noise
      introduced by such perturbation accumulates with repeated evaluations (i.e., sampling methods


                                                        1

054
      methods (Castro et al., 2009; Maleki et al., 2014)). While a lowered (mean of) data value estimate
055   is intuitive and expected as a result of information loss (Dwork et al., 2015; Kairouz et al., 2015),
056   increased estimation uncertainty can render existing data valuation methods useless because the
057   magnitude of the noise scales with the evaluation budget for a fixed DP guarantee. As an illustration,
058   the 3rd figure of Fig. 2 shows that as the number of evaluations k increases, removing data with
059   high data value estimates produces a curve closer to that with random removal, implying that more
060   evaluation samples, paradoxically, have worsened the quality of data value estimates. We theoretically
061   account for this counter-intuitive phenomenon by showing that perturbation introduced by DP can
062   cause the estimation uncertainty of data value estimates, measured in terms of the variance brought
063
      by the perturbation, to scale linearly in the number of evaluations.
064   Fortunately, through our developed theoretical analysis, we derive an insight that leads to a method
065   for controlling the estimation uncertainty. We focus the analysis on the family of semivalues widely
066   adopted as data valuation metrics such as data Shapley (Ghorbani & Zou, 2019), Beta Shapley (Kwon
067   & Zou, 2022), and data Banzhaf (Wang & Jia, 2023). We revisit the paradigm of data valuation in
068   the context of gradient-based DP ML and thus design a technique that perturbs the gradients with
069
      correlated noise in repeated evaluations to mitigate the above issue. Contrary to the linear scaling of
      estimation uncertainty of the data value estimates with the evaluation budget in the naive approach,
070
      our proposed method is shown to control the estimation uncertainty to a constant. Additionally, we
071
      empirically demonstrate that, on various ML tasks, our proposed method produces (i) greater model
072   degradation from removing high-value data; and (ii) higher AUC scores in identifying label-corrupted
073   data. In comparison, the naive approach performs similarly to random selection. We also apply our
074   approach to other scenarios including dataset valuation (Wu et al., 2022) and FL (McMahan et al.,
075   2017). Our specific contributions are summarized as follows:
076
      • We formalize a notion of estimation uncertainty (Eq. (3)) to specifically target the uncertainty due
077
        to DP. We then theoretically identify that under the naive approach of injecting i.i.d. noise for DP,
078     the estimation uncertainty grows in Ω(k) with the evaluation budget k (Prop. 5.1), resulting in
079     low-quality data value estimates.
080   • As mitigation, we propose a simple yet effective approach (in Algorithm 1) via injecting correlated
081     noise to control the estimation uncertainty to O(1) (Prop. 5.4) as opposed to Ω(k).
082   • We empirically demonstrate the implications of the escalating estimation uncertainty shown by
083     the near-random performance on the data removal task (Sec. 6.1) and noisy label detection task
084     (Sec. 6.2). Our approach outperforms the baseline approach on these tasks (Sec. 6.2) and on noisy
085     label detection task in dataset valuation and FL (Sec. 6.3).

091   Prior works (Sim et al., 2023; Wang et al., 2023; Watson et al., 2022; Usynin et al., 2024) that consider
092   privacy-aware data valuation have limited applicability due to limited settings. (Sim et al., 2023)
093
      considered perturbing user statistics to ensure DP but is restricted on the class of Bayesian models
      whereas we consider a wider family of models trained with gradient-based methods including neural
094
      networks. (Wang et al., 2023) proposed a private variant of KNN-Shapley but did not generalize
095
      to other semivalues, whereas our method applies to all semivalues. Watson et al. (2022) directly
096   perturbed the semivalue estimates. However, both (Wang et al., 2023; Watson et al., 2022) require a
097   trusted server to centralize the original gradients which may not reflect real-world scenarios where
098   untrusted central servers pose added privacy risks, whereas our approach does not require a trusted
099   central server. (Usynin et al., 2024) assessed using variance of gradients and privacy loss-input
100   susceptibility score to select useful data points for DP training. The authors further propose methods
101   to compute the DP version of these scores. (Bani-Harouni et al., 2023) considered improving the
102   performance of DP-SGD by utilizing cosine similarity between privatized per-sample gradient and
103   original gradient to decide whether to include the gradient in averaged gradient.
104   Li & Yu (2023); Wang & Jia (2023) identified that stochastic utility functions can lead to noisy data
105   values which deteriorated the rank preservation and proposed to use (weighted) Banzhaf values as the
106   semivalue metric. Although the noise introduced by DP also renders the utility functions stochastic,
107   their methods do not consider the DP setting where noise scales with the number of evaluation
      budgets. We specifically consider mitigating the issue of the scaling noise.

138   Definition 3.1 ((ϵ, δ)-Differential Privacy (Dwork & Roth, 2014, Def. 2.4)). A randomized algorithm
139   M with domain D and range R is said to be (ϵ, δ)-differentially private if for any two neighboring1
140
      datasets d, d′ ∈ D , and for all event S ⊆ R , Pr(M(d) ∈ S) ≤ exp(ϵ)Pr(M(d′ ) ∈ S) + δ.
141
      Importantly, the DP guarantee of M is immune against post-processing (Dwork & Roth, 2014,
142
      Proposition 2.1): The composition f ◦ M with an arbitrary randomized mapping f have the same DP
143   guarantee as M. We adopt this definition of DP to show the linearly scaling effect of perturbation
144   (Sec. 5.1). We elaborate in App. B.2 that our analysis can be extended to other DP frameworks.

247   We first reveal that the propagation of noise from each zt to V can be catastrophic to the estimator ψ:
248   a higher evaluation budget k, surprisingly, leads to a higher estimation uncertainty.
249   The estimator ψ in Eq. (2) aggregates various marginal contributions m(π) = V (θ π ) − V (θ pπ ) over
250   different π ∈ Π. Thus, if ψ requires k evaluations of m, party j needs to reveal its gradients k times.
251   The repeated release of gradients increases the privacy risk, requiring greater perturbation to maintain
252   the same DP guarantee. In the Gaussian mechanism, the best known variance of the Gaussian noise
253   grows in Θ(k) (Abadi et al., 2016, Theorem 1) (more discussed in App. B.3), so each Gaussian noise
254   is expressed as zt ∼ N (0, k(Cσ)2 I) where both C and σ 2 are constants satisfying a fixed (ϵ, δ)-DP

      guarantee hereafter.

371

372   non-square matrix (i.e., relaxing P1). For an arbitrary square matrix X, consider its counterpart Y
373   defined with a hyperparameter q ∈ (0, 1), Y (k−kq)×k := X(kq + 1 : k; 1 : k) where the bracket
374   means taking a sub-matrix with the selected rows and columns. In other words, given a X, Y is a
375   sub-matrix of X with the (kq + 1)th row to the last row, and thus a counterpart to X and relaxes
376   P1. The intuition is that in the first few iterations, t is small, causing g̃π∗ t to still incur relatively
377   large variances even with correlated noise via X. As a remedy, Y effectively discards these highly
      fluctuating marginal contributions to yield an asymptotically even lower total variance than X.

522   7       C ONCLUSION , L IMITATIONS , AND F UTURE W ORKS
523
524   In this work, we identify a problem in data valuation where DP is enforced via perturbing gradients
525   with i.i.d. noise: The estimation uncertainty scales linearly (i.e., Ω(k)) with more budget k and
526   renders the data value estimates almost useless (i.e., close to random guesses in some investigated
527   cases). As a solution, we propose to use correlated noise and theoretically show that using a weighted
528
      sum via matrix form using X provably reduces the estimation uncertainty of semivalues from Ω(k)
      to O(1) and empirically demonstrate the implications of our method on various ML tasks and data
529
      valuation metrics. One limitation is the need to store the gradients. However, this limitation is
530
      alleviated when the number of parties is small (e.g. dataset valuation) or when the memory load
531   can be distributed across parties (e.g. FL). Another limitation is that our theoretical result assumes
532   an diagonal multivariate sub-Gaussian distribution of the gradients. Nevertheless, we empirically
533   demonstrate that our method works for neural networks where the assumption is not explicitly
534   satisfied. A future direction is to explore other possible X to reduce estimation uncertainty further.

782   B.1        S EMIVALUE AS A R ANDOM VARIABLE .
783
784
      The exact computation of semivalues is often intractable in practice due to the need to compute an
      exponential number of marginal contributions V (S ∪ {i}) − V (S). Moreover, in data valuation, the
785
      utility function V is not deterministic w.r.t. S: V is commonly defined as the test accuracy or test
786
      loss (Ghorbani & Zou, 2019), which is stochastic due to (stochastic) gradient descent and random
787   model initialization. Such stochasticity becomes more pronounced when gradients are injected with
788   artificial noise to ensure privacy such as in DP-SGD (Abadi et al., 2016) (recalled later). Hence, the
789   semivalue estimator ψi is treated as a random variable whose randomness comes from the marginal
790   contributions.
791
792
793
      B.2        D IFFERENTIAL P RIVACY F RAMEWORK .
794
      We emphasize that our analysis adopts this definition as we rely on the Gaussian mechanism and
795
      leverage its composition property (Dwork & Roth, 2014). Our analysis holds for other DP frame-
796   works capturing privacy guarantee with Gaussian mechanism and possessing composition and
797   post-processing properties, such as Rényi DP (Mironov, 2017) and z-CDP (Bun & Steinke, 2016).
798   Typically, both Rényi DP and z-CDP satisfy post-processing immunity and have composition proper-
799   ties. Moreover, both provide a DP guarantee for the Gaussian mechanism. Notably, while both DP
800   frameworks provide a modestly tighter privacy analysis than compared to (Dwork & Roth, 2014),
801   to the best of our knowledge, they still require the variance of the Gaussian noise to linearly scale
802   with k to satisfy a given level of DP guarantee. In fact, some existing works have identified that
803   the moment accountant method (Abadi et al., 2016) which we adopt is an instantiation of Rényi
804
      DP (Ouadrhiri & Abdelhadi, 2022) translated back to (ϵ, δ)-DP. Lastly, we note that our theoretical
      analysis is independent of the choice of DP framework except for requiring zt ∼ N (0, kσ 2 I).
805
806
807   Gaussian mechanism. We note that other mechanisms exist for achieving privacy with DP guaran-
808   tees. We choose the Gaussian mechanism for the convenience of mathematical analysis compared to
809   other mechanisms such as the Laplace mechanism as well as a manifold of existing works discussing
      the theoretical properties of the Gaussian mechanism.

      B.3    C OMPOSITION OF DP M ECHANISMS .
811
812
      While it is possible to find a theoretical privacy budget lower than the result in (Abadi et al., 2016,
813   Theorem 1), the current asymptotic bound σ 2 = Ω(k) is the best we know. We also note that
814   there have been efforts at improving the bound (Asoodeh et al., 2021). However, the results are
815   not asymptotically better in terms of k, hence do not affect our theoretical results. Moreover, we
816   emphasize that our theoretical analysis and proof strategy do not depend on the exact form of σ 2
817   other than assuming it depends on k. If a lower bound is found in the future, our analysis can be
818   readily adapted.
819
820
      Note on per evaluation DP guarantee and final DP guarantee. We are concerned about the final
821
      DP guarantee (i.e., (ϵ, δ)-DP guarantee after k compositions) unless otherwise stated. Notably, by
822
      setting zt ∼ N (kσ 2 I), each per-evaluation DP-guarantee is stronger, or put differently, the privacy
823   budget required at each evaluation to satisfy a given (ϵ, δ)-DP guarantee is lowered such that, after k
824   compositions, the desired final DP-guarantee can be achieved.
825
826
827   B.4    DP G UARANTEE L EVEL AND DATA VALUE E STIMATES
828
829   Noise introduced by DP causes a lower mean of data value estimates. Mathematically, DP
830   imposes a limit on the leave-one-out property of the privatized mechanism M. In particular, if
831   V (·) ∈ [0, 1] (e.g., when test accuracy is used as V ), then we have, for any party i and subsets
832   S ⊆ [n] \ {i}, the following (Dwork et al., 2015, Lemma 6):
833
834                              |ψi | := |ES [V (S ∪ {i}) − V (S)]| ≤ eϵ − 1 + δ
835
836   where (ϵ, δ) are the parameters satisfying (ϵ, δ)-DP. Assuming a fixed δ, stronger DP implies a lower
837   ϵ, hence decreased right-hand side value, i.e., the upper bound for the marginal contribution. This
838   inequality suggests that stronger DP results in lower absolute value of the data value estimates. We
839
      find this a reasonable behavior as the decreased value reflects the erosion of information carried by
      the data. Instead, having a lower absolute data value does not forbid us to still preserve the relative
840
      order of the estimates by minimizing the estimation uncertainty.
841
842
843   Impact of different (ϵ, δ)-DP guarantee levels on our method. Through our theoretical devel-
844   opment, we have established that the estimation uncertainty can be controlled to be independent of
845   the evaluation budget k, i.e., the noise due to DP is only affected by the final DP guarantee level.
846   That said, an overly large final DP guarantee level shrinks the gap between the performance of
847   i.i.d. noise and without DP noise since perturbation is small, and vice versa. Therefore, to highlight
848   the effectiveness of our method in controlling the estimation uncertainty, we fix a moderate final DP
849
      guarantee level (ϵ, δ) = (1.0, 5 × 10−5 ) throughout the main text s.t. a clear contrast in performance
      on various ML tasks can be observed. We also provide additional experiments on different ϵ values in
850
      App. D.5.

2181

2182

2183   marginal contribution of party j which is averaged over all participation rounds. Note that in a total
2184   of k rounds, each party still needs to release the gradients k times, thus incurring the problem of
2185   linearly scaling variance of the Gaussian noise which can be alleviated with our method. However,
2186   the setting of FL is not identical to the conventional data valuation setting as the global model keeps
2187   updating which causes the gradients to change drastically, especially in the first few rounds. To
2188   address this challenge, we use a weighted sum that puts more emphasis on the more current gradients
2189   in the first few rounds: X t,t = 0.75 − 0.7 × t/k. Note that this choice is consistent with Prop. 5.3
2190   which suggests setting a larger X t,t when σg2 is large. This can be implemented algorithmically by
2191   updating g̃πt with g̃π∗ t = (0.25 + 0.7 × t/k)g̃π∗ t−1 + (0.75 − 0.7 × t/k)g̃πt . Moreover, as the global
2192   model converges with more collaboration rounds, more information about the data values is revealed
2193
       in the first few rounds. Therefore, setting q too large causes ψj to miss important information about
       the data value, leading to inaccurate data value estimates, even though the estimation uncertainty is
2194
       lowered. As such, we set a moderately small burn-in ratio q = 0.2.
