009                                                       A BSTRACT
010
011             Transformers exhibit In-Context Learning (ICL), a phenomenon in which these
012             models solve new tasks by using examples in the prompt without additional train-
013             ing. In our work, we analyze two key components of ICL: (1) context-scaling,
014             where model performance improves as the number of in-context examples in-
015             creases and (2) task-scaling, where model performance improves as the number
016             of pre-training tasks increases. While transformers are capable of both context-
017             scaling and task-scaling, we empirically show that standard Multi-Layer Percep-
018
                trons (MLPs) with vectorized input are only capable of task-scaling. To under-
                stand how transformers are capable of context-scaling, we first propose a signifi-
019
                cantly simplified transformer that performs ICL comparably to the original GPT-2
020
                model in statistical learning tasks (e.g., linear regression, teacher-student settings).
021             By analyzing a single layer of our proposed model, we identify classes of feature
022             maps that enable context scaling. Theoretically, these feature maps can implement
023             the Hilbert estimate, a model that is provably consistent for context-scaling. We
024             then show that using the output of the Hilbert estimate along with vectorized input
025             empirically enables both context-scaling and task-scaling with MLPs. Overall, our
026             findings provide insights into the fundamental mechanisms of how transformers
027             are able to learn in context.

      Thus far, the ability of models to learn in context has been broadly defined as their ability to general-
059
      ize to unseen tasks based on context examples without updating the model parameters. We observe
060
      that there are two settings for studying generalization in ICL. The first, which we call context-
061   scaling, refers to the ability of the model to improve as the context length N increases while the
062   number of pre-training tasks T is fixed. The second, task-scaling refers to the ability of a model to
063   improve as T increases while N is fixed.

080
      vice-versa. For example, as we show in Figure 1A, both transformers and standard Multi-Layer
081   Perceptrons (MLPs) are capable of task-scaling on ICL of linear regression tasks. In contrast, only

084                              What mechanism enables transformers, but not MLPs, to context-scale?
085
086   To identify such a mechanism, we begin by constructing a bare-bones transformer with all key,
087
      query, and value weight matrices fixed to be the identity matrix. We refer to our simplified model
      as Simplified GPT (SGPT). Despite its simplicity, we find that SGPT is competitive with GPT-
088
      2 architecture transformers (Radford et al., 2019) for a variety of ICL problems including linear
089
      regression, student-teacher networks, decision trees, and sparse linear regression (Garg et al., 2022).

      We summarize our findings as follows:
114
             • We show that a simplified transformer, SGPT, with all key, query, and value weight matrices
115            fixed to the identity matrix, is competitive with GPT-2 on a range of ICL tasks.
116
             • We analyze a one-layer version of SGPT, demonstrating that this model is capable of
117
               context-scaling solely through the use of a feature map applied to input data.
118
119
             • We theoretically show that this feature map can be modified to perform kernel smoothing
               to impute the missing element in each task using the other in-context examples. Upon
120
               selecting the Hilbert estimate as the choice of smoother (Devroye et al., 1998), this im-
121
               puted element becomes a statistically optimal (consistent) estimate as the context length
122            approaches infinity.
123
             • We show that by providing MLPs with a concatenation of features from our proposed fea-
124
               ture map and the vectorized input for each task, the model can simultaneously achieve both
125            context-scaling and task-scaling.
126
127   Overall, we identify key mechanisms that transformers can use to provably generalize to new, unseen
128   tasks given a large context.

130   2   P RIOR WORK
131
132   ICL in controlled settings. The work by Garg et al. (2022) initiated the study of ICL in statistical
133   learning tasks, such as linear regression, decision tree learning, and teacher-student neural network
134   learning. They showed that transformers such as GPT-2 (Radford et al., 2019), when pre-trained
135   with a huge number of independent tasks (around 3.2 × 107 independent tasks, each presented
136   once), can exhibit a strong ICL ability, in the sense that during inference time, the performance of
137   pre-trained transformers matches the Bayes optimal or the best-known algorithms for these tasks.
138   These results were later extended to other settings (see Akyürek et al., 2022; Raventós et al., 2023;
139
      Bai et al., 2023; Li et al., 2023; Tong & Pehlevan, 2024, for examples). In particular, Raventós
      et al. (2023) empirically showed transformers can achieve nearly Bayes optimal ICL even when pre-
140
      trained with multiple passes over a much smaller number of independent tasks, and Bai et al. (2023)
141
      showed by construction that transformers can select optimal algorithms in context if the task prior
142   is a mixture of distributions. These works together suggest that transformers can achieve both task-
143   scaling and context-scaling for ICL. On the other hand, Tong & Pehlevan (2024) empirically showed
144   that pretrained MLPs can achieve ICL when the context length is fixed and is the same during pre-
145   training and inference. However, it was unclear whether MLPs can achieve context-scaling, and
146   our work empirically gives a negative answer. Motivated by this, we study the mechanism inside
147   transformers that enable both task and context scaling.
148
149   Linear attention and one-step gradient descent. The theory of ICL is best understood in linear
150   regression with a fixed context length (therefore only concerning the task-scaling). Specifically,
151
      Von Oswald et al. (2023) showed by construction that single linear attention can implement one-step
      gradient descent (GD) in context. Ahn et al. (2023); Zhang et al. (2024a); Mahankali et al. (2024);
152
      Zhang et al. (2024b) proved that optimally pre-trained single linear attention is equivalent to a one-
153
      step GD. Wu et al. (2024) proved that one-step GD nearly matches the Bayes optimal algorithm
154   for inference and can be pre-trained efficiently with finite independent tasks. Later works such as
155   Cheng et al. (2024) connected nonlinear attention to functional one-step GD in feature space. These
156   works together have substantially furthered our understanding of single linear attention for ICL of
157   linear regression with a fixed context length. However, these results assume (or are only sharp the
158   context length is fixed, and in particular, they fail to explain the observed context-scaling ability of
159   pre-trained transformers.
160
161   Softmax attention and kernel smoothers. The connection between softmax attention and kernel
      smoothers was first pointed out by Tsai et al. (2019). Specifically, by setting the query and key


                                                         3
      Under review as a conference paper at ICLR 2025


162
      matrices to be the same, an attention component can be viewed as a kernel-smoother (that uses
163   a learnable semi-positive definite kernel). Empirical evidence suggests that using shared a matrix
164   for query and key matrices does not significantly impact the performance of transformers (Tsai
165   et al., 2019; Yu et al., 2024). Later, theoretical works Chen et al. (2024); Collins et al. (2024)
166   utilized this connection to study the ICL of softmax attention in both linear (Chen et al., 2024) and
167   nonlinear regression tasks (Collins et al., 2024). In these settings, softmax attention achieves ICL by
168   implementing a kernel smoother with an optimal bandwidth parameter (achieved by training query
169   and key matrices). Compared with these works, we demonstrate that transformers can achieve ICL
170   using an attention component that does not contain trainable parameters (that is, setting query, key,
171
      and value matrices to identity matrices). Our results suggest that the transformer architecture is
      capable of performing ICL without needing to explicitly learn any hyperparameters in the attention
172
      head. We explain this by connecting to a hyper-parameter-free yet statistically consistent kernel
173
      smoother given by the Hilbert estimate (Devroye et al., 1998).
174
175
      Approximation ability of transformers. The transformer is known to be a versatile architecture
176   that can implement efficient algorithms (by forwarding passing an input prompt) in many scenarios
177   (see Akyürek et al., 2022; Bai et al., 2023; Guo et al., 2023; Lin et al., 2023; Gatmiry et al., 2024,
178   for a non-exhaustive list of examples). These constructive results, while explaining ICL from an
179   approximation theory perspective, heavily exploit the trainability of query, key, and value weight
180   matrices. In this work, we show that transformers are competitive with GPT-2 architecture trans-
181   formers for ICL even when all of their query, key, and value weight matrices are fixed to be the
182   identity matrix. Our results suggest the prior constructive results may not fully explain the ICL
183   ability of transformers.

465   5.2                    T RAINING MLP S THAT SIMULTANEOUSLY CONTEXT- SCALE AND TASK - SCALE
466
467   As the Hilbert estimate provides a consistent estimate, our results above show that transformers
468   provably generalize to unseen tasks, when the context length approaches infinity. Nevertheless,
469   the issue with using the Hilbert estimate alone is that the Hilbert estimate is only computed using
470   examples provided in a context. As such, it cannot task-scale unlike MLPs trained on vectorized
471
      inputs. We now show that training MLPs on vectorized inputs concatenated with features estimated
      using ψK (A) result in MLPs that can both context-scale and task-scale.
472
473   Namely, we revisit the experiment presented in Figure 1 and extend our analysis by training MLPs
474   on three distinct input configurations: (1) vectorized input data; (2) features from ψK (A)N,: ; and

476   consider the feature maps ψL and ψH discussed in the previous section. The results of training these
477
      MLPs is presented in Figure 7 and we summarize the results below.
478
479                          1. MLPs with vectorized input data: Figure 7A demonstrates that these MLPs exhibit task-
480

483                             scale, as performance does not improve with an increasing number of tasks. This behavior

496   6   CONCLUSION AND LIMITATIONS
497
498   Summary. In this work, we observed that transformers, unlike MLPs, are able to simultaneously
499   context-scale (improve performance as the context length increases) and task-scale (improve perfor-
500   mance as the number of pre-training tasks increases). To better understand this property of trans-
501   formers, we first identified a simplified transformer (SGPT) that could solve ICL tasks competitively
502   with GPT-2 architecture transformers despite having no trainable key, query, and value weights in at-
503   tention layers. By studying a one-layer version of SGPT, we identified that the attention operator of
504   SGPT applied a feature map, ψ, on input data that enabled context-scaling. In particular, we showed
505   that this feature map could implement kernel smoothers such as the Hilbert estimate, which is a
506
      statistically consistent estimator as the context length goes to infinity. As such, our work provably
      demonstrates that transformers can context-scale, generalizing to new, unseen tasks when provided
507
      a large context. We demonstrated the effectiveness of the feature map, ψ, for context-scaling by
508
      showing that MLPs trained on both features from ψ and vectorized inputs could simultaneously
509   context-scale and task-scale.
510
511   Future work and limitations. While we have provably established that one-layer transformers
512   can context-scale, we empirically observe that one-layer transformers are not as sample-efficient as
513   deep transformers for both context-scaling and task-scaling. Thus, an important future direction is
514   understanding how depth improves the sample complexity of transformers in both context-scaling
515   and task-scaling settings. Exploring this aspect remains a promising avenue for future research and
516   could provide a comprehensive understanding of ICL, and more broadly, a better understanding of
517   how transformers are able to generalize to new tasks when provided large contexts.
