009                                              A BSTRACT
010
011            In the maximum coverage problem we are given d subsets from a universe [n],
012            and the goal is to output at most k subsets such that their union covers the largest
013            possible number of distinct items. The input can be formalized as an n × d matrix
014            A where entry Aij ̸= 0 if item i is covered by subset j and Aij = 0 otherwise. In
015            this paper we create the first linear sketch to solve the maximum coverage prob-
016            lem. The sketch has size sublinear in the input and is directly applicable to dis-
017            tributed and streaming settings, often offering significant runtime improvements.
018            We focus on the application to the turnstile streaming model which supports inser-
019
               tions and deletions. In this model, updates take the form (i, j, ±1) which update
               Aij to Aij + 1 or Aij − 1, depending on the sign. Previous work has largely fo-
020
               cused on more restrictive models, such as the set-arrival model where each update
021
               reveals an entire column of A, or the insertion-only model which does not allow
022
               deletions. We design an algorithm with an Õ(d/ε3 ) space bound for all k ≥ 0. We
023            note that when k is constant, this space bound is nearly optimal up to logarithmic
024            factors. We then turn to fingerprinting for risk measurement. The input is an n × d
025            matrix A where there are n users and d features, and the goal is to determine which
026            k features (or columns in A) together pose the greatest re-identification risk. Our
027            maximum coverage sketch directly enables a solution to targeted fingerprinting
028            for risk measurement. Furthermore, we present a result of independent interest:
029            a linear sketch of the complement of Fp , the pth frequency moment, for p ≥ 2.
030            We use this sketch to solve general fingerprinting for risk management. Empirical
031
               evaluation confirms the practicality of our fingerprinting algorithms, demonstrat-
               ing a speedup of up to 210x over prior work. We also demonstrate that our general
032
               fingerprinting algorithm can serve as a dimensionality reduction technique, with
033
               an application to facilitating enhanced feature selection efficiency.

      1   I NTRODUCTION
038
039
      Maximum coverage is a classic NP-hard problem with applications including information retrieval
      (Anagnostopoulos et al., 2015), influence maximization (Kempe et al., 2003), and sensor placement
040
      (Krause & Guestrin, 2007). Given d subsets of a universe with n items and cardinality constraint
041
      k ≥ 0, the goal is to output the k subsets whose union covers the greatest number of distinct items.
042   A simple greedy algorithm solves this problem by running for k rounds, selecting the subset with the
043   largest marginal gain in each round. This algorithm, in polynomial time and space, achieves a 1−1/e
044   relative approximation, an approximation factor which is tight for polynomial time algorithms unless
045   P = N P (Feige, 1998). However, its polynomial time and space complexity make it impractical
046   for handling massive datasets. Our objective, consequently, is to study algorithms for maximum
047   coverage with sublinear time and memory requirements.
048   We formalize the input to the maximum coverage problem as an n × d matrix A where entry Aij is
049   nonzero if item i is in subset j and 0 otherwise. In this paper we create the first linear sketch with
050   size sublinear in the input matrix to solve maximum coverage, to the best of our knowledge. Linear
051   sketches compress large input matrices while preserving essential information used to form the final
052   output. They also support updates, including both insertions and deletions, to the input matrix. In
053   the context of maximum coverage, an update takes the form (i, j, ±1) which modifies entry Aij by
      adding or subtracting one, effectively adding or removing an item from a subset (or doing nothing

054
      if entry Aij was nonzero before and after the update). After all updates, we can query the sketch to
055   form our output (in this case to select the k subsets to output).
056
057   Linear sketches are far more powerful than algorithms tailored to specific models, as they enable
058
      significant runtime improvements while being applicable to a wide range of settings including dis-
      tributed and streaming contexts1 . We focus on the application to the turnstile streaming setting
059
      where updates come one-by-one in a stream and each update modifies Aij by adding or subtract-
060
      ing one. To our knowledge, our linear sketch provides the first streaming algorithm for maximum
061   coverage which allows arbitrary deletions of items from subsets. Deletions are critical for a number
062   of applications. For example, we use them to extend our algorithm to fingerprinting for dataset risk
063   measurement.
064
      Related Work. There is an extensive body of work on the maximum coverage problem, and we
065
      do not attempt to give a comprehensive overview here. Instead, we focus on the related streaming
066
      literature. Specifically, we discuss one-pass streaming algorithms with polynomial time complexity,
067   where a pass refers to a single traversal of the stream of updates. In the following, a (x) relative
068   approximation means that the number of distinct items covered by the k subsets selected by the
069   algorithm is at least x · OPT, where OPT denotes the number of items covered by the optimal
070   solution. In addition, Õ(·) notation is used to suppress poly-logarithmic factors in its argument.
071
072
      McGregor & Vu (2018) provide a one-pass algorithm that outputs a (1−1/e−ε) relative approxima-
073
      tion for ε ∈ (0, 1) in Õ(d/ε2 ) space. They consider the insertion-only set-arrival streaming model
      which given our input n × d matrix A is equivalent to seeing an entire column of A in each update.
074
      In other words, each update reveals a subset and the items it covers, and deletions are not supported.
075
      At a high level, their algorithm first subsamples rows of A such that OPT in this smaller universe is
076
      Õ(k/ε2 ). They then argue that achieving a (1 − 1/e) relative approximation to maximum coverage
077
      on this smaller universe achieves a (1 − 1/e − ε) relative approximation overall.
078
079   Bateni et al. (2017) give a one-pass (1−1/e−ε) relative approximation algorithm that uses Õ(d/ε3 )
080   memory. They consider the insertion-only streaming model which given our input matrix A is
081   equivalent to receiving updates of the form (i, j, 1). Note that negative updates (i.e., deletions) are
082
      not supported. They specifically provide an algorithm that carefully samples a number of nonzero
      entries of input A, and they show that any (α) relative approximation on this smaller subsampled
083
      universe achieves an (α − ε) relative approximation for the original input. We use their sketch as a
084
      starting point (see Section 3 for details).
085
086   There has also been work that achieves different approximation factors (Saha & Getoor, 2009; Mc-
087   Gregor et al., 2021), in random-arrival streams (Warneke et al., 2023; Chakrabarti et al., 2024), and
088
      in more general submodular maximization in the insertion-only set-arrival model (Badanidiyuru
      et al., 2014; Kazemi et al., 2019).
089
090   In contrast to all of the above, our sketch (and therefore turnstile streaming algorithm) allows dele-
091   tions and arbitrarily ordered updates to any individual entry of A.
092   We note that there has also been work on submodular maximization in the somewhat related dynamic
093   model (Monemizadeh, 2020; Chen & Peng, 2022; Lattanzi et al., 2020). We briefly outline the
094   differences between the dynamic model and streaming model. While both models process updates
095   sequentially, the key distinction lies in their primary objectives. The dynamic model prioritizes
096   achieving minimal update time, whereas the streaming model, which we consider here, emphasizes
097   minimizing space usage. It is worth noting that most algorithms designed for the dynamic model
098   do not achieve sublinear space and, in some cases, require exponential space. Despite our emphasis
099   on space efficiency, the linear sketches we present that we will apply to the streaming model also
100
      maintain sublinear update times.

104   et al., 2016). In targeted fingerprinting, the input is an n × d matrix A, where n represents the
105   number of users and d represents the number of features, and a target user u ∈ [n] = {1, 2, . . . , n}.

134   time Õ(kd/ε3 ) such that given d subsets of a universe [n], integer k ≥ 0, and ε ∈ (0, 1), running

138
      k ≥ 0) appears in poly-logarithmic factors. Moreover, since we can assume k ≤ d - otherwise,
139   we can simply output all the input subsets - these poly-logarithmic factors in k can be treated as
140
      poly-logarithmic factors in d, which are hidden by the Õ(·) notation.

143   accomodate insertions and deletions.
144   Corollary 1.1. Given d subsets of a universe [n], integer k ≥ 0, and ε ∈ (0, 1), there exists a

      We note that the space complexity of our algorithm matches that of Bateni et al. (2017) and, for
      constant ε, that of McGregor & Vu (2018). Additionally, several lower bounds exist.
149
150   Assadi (2017) shows that achieving a (1 − ε) relative approximation in a constant number   √ of passes
151   requires Ω(d/ε2 ) space. Assadi & Khanna (2018) shows that even achieving a n1/3 or k relative
152   approximation in one pass with a sketch requires the sketch to have size Ω(d/k 2 ). McGregor & Vu

154   requires Ω(d/k 2 ) space. Therefore (while our algorithm works for all k ≥ 0 with space Õ(d/ε3 )),
155   if k is constant, our result is optimal up to poly-logarithmic factors. Bateni et al. (2017) also show
156   that any (1/2 + ε) relative approximation multi-pass streaming algorithm requires Ω(d) space.

      To do this, we first present a framework for submodular maximization under cardinality constraints
      over monotone, linearly sketchable functions in turnstile streams 2 . Submodular functions exhibit the
185
      property of diminishing returns, and we specifically focus on maximizing monotone, non-negative
186
      submodular functions that are defined over subsets of a given universe. In our context, this means
187   that there are d subsets of a universe [n], and the function takes as input some of these subsets and
188   returns a positive real number. A function is defined to be linearly sketchable if its input can be
189   compressed by a linear sketch and this sketch can be queried to efficiently produce the function’s
190   output value on some given subsets. For formal definitions of submodular functions and linearly
191   sketchable functions, see Appendix A.1.2 and Appendix A.1.2. Here, a (x) relative approximation
192   means that the output of the function on the k subsets selected by the algorithm is at least x · OPT,
193   where OPT denotes the maximum output of the function on k subsets. The proof of the following
194   is deferred to Appendix A.4.

229   Experimental Results. We also illustrate the practicality of our fingerprinting algorithms by run-
230   ning experiments on two different datasets of size 32, 000 × 80 and 2, 500, 000 × 120. In a direct
231   comparison with the implementations of Gulyás et al. (2016), our algorithms show significantly
232   improved efficiency while retaining high comparative accuracy. Specifically, for targeted finger-
233   printing, we achieve a speedup of up to 49x, with accuracy that converges rapidly to that of Gulyás
234   et al. (2016). For general fingerprinting, we gain a speedup of up to 210x while again achieving high
235   comparative accuracy.
236   Finally, we believe that our general fingerprinting algorithm can serve as a dimensionality reduction
237   technique. To illustrate this, we apply it in the context of feature selection for machine learning
238   models where feature spaces are often extremely large. Feature selection is a process that identifies a
239   subset of relevant features from the original dataset to improve model performance or computational
240   efficiency. By using our general fingerprinting algorithm to perform feature selection and therefore
241   reduce the dimensionality of the input dataset, we can mitigate issues such as overfitting, improve
242   interpretability, and greatly speed up machine learning algorithms.
243   In particular, since the time complexity of many popular clustering algorithms such as k-means
244   scales with the dimensionality of the data, we use our general fingerprinting algorithm to select x
245   features that best separate the data. We therefore have reduced the dimension of the feature space to
246   x. We then use k-means on these x features instead of the full feature space and demonstrate that this
247   approach significantly increases efficiency while sacrificing little in terms of accuracy. We believe
248   our techniques to be general and extendable to other clustering and machine learning algorithms
249
      outside of k-means.

      Linear Sketches We begin by defining what a linear sketch is and then provide an overview of
256
      the specific linear sketches used in this paper. Given a n × d matrix A, we can compress it while
257
      retaining essential information to solve the problem by multiplying it with a r × n linear sketching
258   matrix S. A linear sketch is a matrix drawn from a certain family of random matrices independent
259   of A. This independence ensures that S can be generated without prior knowledge of the contents of
260   A. Linear sketches support insertions and deletions to the entries of A, as S(A + cij ) = SA + Scij
261   holds for any update cij , which adds or subtracts one from an entry of A. This property allows us to
262   maintain SA throughout updates without requiring storage of A itself. Furthermore, S is typically
263   stored in an implicit, pseudorandom form (e.g., via hash functions) rather than explicitly, enabling
264   efficient sketching of updates cij . The primary focus is on minimizing the space requirement of a
265   linear sketch, specifically ensuring that the sketching dimension r is sublinear in n and ideally much
266
      smaller. Alongside space efficiency, there are two additional important performance metrics: update
      time and reporting time. Update time refers to the time complexity required for the sketch to process
267
      an update, and reporting time refers to the time complexity needed to return an answer to a query.
268

676   A.1.1    T URNSTILE S TREAMING M ODEL
677
678   In this paper, we represent the input as a n × d matrix A. In the streaming model, it is standard
679   to initialize all the entries to zero before the stream of updates. The algorithm then processes a
680
      stream of updates which come one-by-one, each of the form (i, j, ±1). This modifies entry Aij by
      performing Aij = Aij + 1 or Aij = Aij − 1 depending on the sign. This is referred to as the
681
      turnstile streaming model, where both insertions and deletions (or positive and negative updates)
682
      are allowed. The updates can appear in arbitrary order in the stream, and we make the standard
683   assumption that the length of the stream is at most poly(n). The goal of the streaming algorithm
684   is to process the stream efficiently, using sublinear space in the size of the input matrix A (and
685   therefore cannot store all the updates) and a small constant number of passes over the stream. In
686   this work, restrict our focus to one-pass algorithms. At the end of the stream, the algorithm can do
687   some post-processing and then must output the answer. While streaming algorithms are not required
688   to maintain a stored answer at every point during the stream, there is no restriction on when the
689   stream may terminate. Any time or space used before or after processing the stream is attributed
690   to pre-processing or post-processing, respectively. Generally, our primary focus is on optimizing
691
      the memory usage and update time during the stream. Here the update time is the time complexity
      required by the algorithm to process an update.

962   A.4   P ROOF OF T HEOREM 2 (S UBMODULAR M AXIMIZATION F RAMEWORK )
963
964   Here, we outline a framework to design algorithms to maximize monotone non-negative submodular
965   functions that are linearly sketchable subject to a cardinality constraint. At a high level we will
966   receive a linear sketch of the input matrix A such that querying the sketch will produce the function’s
967   output value on some union of subsets. We then adapt the classical greedy algorithm for maximizing
968
      a monotone submodular function to query the linear sketches instead of accessing the input matrix
      directly.
969
970   We note that setting γ = ε/k for many linear sketches introduces poly(k) factors in the final memory
971   usage. However, setting γ = ε/k is provably necessary when performing submodular maximization
      over queried function values that are preserved up to a (1 ± γ) factor to achieve a 1 − 1/e − ε


                                                         18
       Under review as a conference paper at ICLR 2025


972
       approximation (see Theorem 5 of Horel & Singer (2016)). Note that this applies to all algorithms
973    that perform submodular maximization that have this property.
974
975    We now prove Theorem 2. Theorem 2 allows us to create an algorithm to maximize a specific
976
       monotone non-negative submodular function subject to a cardinality constraint by simply sketching
       the input A via a linear sketch that satisfies the properties of the theorem.
977
978    Let C be a subset of the column vectors of A. In the following, {S · ai }i∈C can be thought of as
979    the sketch of A restricted to C. As described in Appendix A.1.2, we say that our function f has a
980    corresponding sketching matrix S and corresponding gS . For any two subsets of columns X and Y ,
981    let gS ({S·ai }i∈X|Y ) denote the marginal gain of adding X, or gS ({S·ai }i∈X∪Y )−gS ({S·ai }i∈Y ).
982    c ∈ d \ C denotes a column c which is not already in subset C.
983    We now describe our algorithm, sketchy-submodular-maximization (Algorithm 7). We first create k
984    independent linear sketches (recall that the process of creating a linear sketch for the input function
985    is given as input to the algorithm). Then we run the following classical greedy submodular maxi-
986    mization algorithm with the modification that instead of directly evaluating the input function f it
987    queries the given sketch. Note that in each of the k adaptive rounds, we use a different sketch. The
988    classical greedy algorithm in each round simply looks at all subsets that have not been chosen and
989
       adds the one with the largest marginal gain to the output set (Nemhauser et al., 1978).
990
991    Algorithm 7 sketchy-submodular-maximization
992     1: Initialize C ← ∅.
993     2: while |C| ≤ k do
994     3:    C ← C ∪ argmaxc∈d\C gS ({S · ai }i∈c|C ).
995     4: end while
996     5: Return C.
997
998    We first analyze the memory usage. We are given that each sketch takes O(s) space. Since there are
999    k rounds of adaptivity, the total space taken by the sketches is O(sk). Both the update and reporting
1000   time will depend on the specific linear sketch.
1001   Now, let us prove correctness. We assume by our theorem statement that our sketch S and corre-
1002   sponding function gS give us a (1 ± γ)-approximation to the queried values of our input function
1003   f . There are k adaptive rounds. Since we create as many sketches and use a different one in each
1004   round, adaptivity between the rounds does not introduce error. In addition, despite getting (1 ± γ)-
1005   approximations to all our queried values instead of the true queried values of our input function,
