                                                 A BSTRACT
008
009
010            This paper studies continual learning after model deployment. A real-world appli-
011
               cation environment is often an open world filled with novel or out-of-distribution
               (OOD) objects that have not been seen before. We can call continual learning in
012
               such an environment open-world continual learning (OWCL). OWCL incremen-
013
               tally performs two main tasks: (1) detecting OOD objects, and (2) continually
014            learning the OOD or new objects on the fly. Although OOD detection and con-
015            tinual learning have been extensively studied separately, their combination for
016            OWCL has barely been attempted. This is perhaps because in addition to the ex-
017            isting challenges of OOD detection and continual learning such as catastrophic
018            forgetting (CF), OWCL also faces the challenge of data scarcity. As novel ob-
019            jects appear sporadically, when an object from a new/novel class is detected it, is
020            difficult to learn it from one or a few samples to give good accuracy. This paper
021            proposes a novel method called OpenLD to deal with these problems based on
022
               linear discriminant analysis (LDA) and a pre-trained model. This method enables
               OOD detection and incremental learning of the detected samples on the fly with
023
               no CF. Experimental evaluation demonstrates the effectiveness of OpenLD.

027   1   I NTRODUCTION
028
029   Traditional machine learning operates in a closed world, where everything encountered during test-
030   ing has already been seen during training, meaning nothing novel can appear after model deploy-
031   ment.(Bendale & Boult, 2015; Liu et al., 2023a). In contrast, the real world is an open environment,
032   full of unknowns and novelties, also referred to as out-of-distribution (OOD) objects. To function ef-
033
      fectively in this open world, an AI agent must continuously learn on the fly after deployment, rather
      than relying on periodic offline retraining initiated by human engineers. We call this paradigm the
034
      open-world continual learning (OWCL). OWCL must have three key capabilities: (1) detecting
035
      OOD objects, (2) obtaining the class labels of the detected OOD objects, and (3) learning the OOD
036   objects on the fly incrementally. (3) is the class-incremental learning setting of continual learning
037   as the system learns more and more classes. This paper focuses on (1) and (3). As humans, when
038   we see some unknown objects, we usually ask others for their names (i.e., class labels). Thus, for
039   (2), we assume the agent can ask humans or other agents to provide labels for the detected OOD
040   objects, but we want to minimize the number of asks. Furthermore, in OWCL, the data comes in a
041   stream and OOD detection and learning of the detected novel objects have to be done online.
042   OWCL Problem Setting. Since OWCL aims to learn continually after model deployment, the initial
043   model M to be deployed is assumed to be well-trained with a set of initial classes C of labeled data.
044   After M is deployed in its application, it detects and learns more and more new classes. At the
045   steady state, the set of all classes that the system has encountered is C A = C ∪ C L ∪ C E , where
046   C L is the set of new classes that have appeared after model deployment and are well learned after
047   seeing a good number of training samples, and C E is a set of emerging new classes that have been
048   seen but are not well learned yet, i.e., not enough labeled training data have been seen to well-learn
049   the classes. We denote C + = C ∪ C L as the set of well-learned classes so far and C N = C L ∪ C E
050
      as the set of all new classes seen after deploying M . With incremental learning, M becomes M + ,
      covering all classes in C A . Each iteration of OWCL performs two main functions.
051
052   (1). OOD Detection and Classification. M + detects whether an incoming test sample x is OOD.
053   If not, it is classified to one of the classes in C + . OOD classes include those emerging classes
      in C E as they still need some more data to be well-learned, but OOD detection can leverage the

074   OWCL is thus related to two main areas of research, OOD detection (Yang et al., 2021b; Chandola
075   et al., 2009) and the class incremental learning (CIL) setting of continual learning (Van de Ven &
076   Tolias, 2019; De Lange et al., 2021; Wang et al., 2023; Mai et al., 2022). These topics have been
077   studied separately and extensively. However, in OWCL, OOD detection has to be done continually
078   unlike the traditional static OOD detection with a set of fixed in-distribution (ID) classes. The
079
      number of ID classes increases as the agent learns new classes of objects. CIL, which learns a
      sequence of tasks (where each task consists of a set of classes) incrementally, still faces a major
080
      challenge of catastrophic forgetting (CF). CF refers to the phenomenon that in learning a new task,
081
      the learner needs to modify the parameters learned for previous tasks, which may cause performance
082   degradation for previous tasks. Furthermore, existing CIL methods do not do OOD detection.
083
084   A novel approach called OpenLD (Open-world continual learning via Linear discriminant analysis
085
      and OOD Detection) is also proposed to learn in the OWCL setting, i.e., performing the above
      two main functions. The method is based on linear discriminant analysis (LDA) (Pang et al., 2005).
086
      LDA assumes that given the class, the data follows a normal distribution with a mean and covariance
087
      matrix. It further makes the assumption that the class covariances are identical, i.e., all classes
088   share one covariance matrix but have different means. LDA uses the means and covariance for
089   classification. However, LDA is not suitable for OOD detection because LDA is based on likelihood
090   ratio, which is only suitable for closed-world classification, as OOD detection needs a measure of
091   absolute distance from a sample to a distribution. In this work, we use Mahalanobis distance (MD)
092   and related methods for OOD detection with a novel idea (see below).
093
      After obtaining the label of each detected OOD sample, OpenLD learns it immediately. Each new
094   class still uses the covariance matrix learned initially in M , but the mean of the class is updated. The
095   pipeline of the proposed method is shown in Figure 1. This paper makes the following contributions.
096
097
      1. It proposes a realistic open-world continual learning setting, which is important as an AI agent
      working in the real open world needs the capability to continually learn new knowledge on the fly af-
098
      ter deployment to make it more and more knowledgeable and ultimately achieve learning autonomy,
099
      i.e., incremental learning for self-improvement without the involvement of human engineers.
100
      2. It proposes a novel approach based on incremental updating of the model with a shared covariance
101
      and different means for different classes, which has no CF and gives good accuracy without a large
102
      number of training samples from each new class. This is particularly important because it is hard to
103   obtain many labeled samples after model deployment.
104
      3. In continual OOD detection, OpenLD not only uses the ID classes but also already detected OOD
105
      samples to help detect OOD data more accurately. To our knowledge, this has not been done before.
106
107   Experiments have been conducted to demonstrate the effectiveness of the proposed OpenLD. The
      code of OpenLD has been submitted in Supplementary Materials.

      2   R ELATED WORK
109
110
111   OOD Detection. OOD detection has been studied under many names, e.g., novelty detection, outlier
112   or anomaly detection, and open set recognition (Ghassemi & Fazl-Ersi, 2022; Cui & Wang, 2022;
113
      Shen et al., 2021; Cortes et al., 2016; Geifman & El-Yaniv, 2017; Andrews et al., 2016; Liu et al.,
      2008; Tax, 2002; Scheirer et al., 2012; Bendale & Boult, 2015; 2016; Ge et al., 2017; Neal et al.,
114
      2018; Malinin & Gales, 2018; 2019; Malinin et al., 2019; Zhou et al., 2023). In recent years, neural
115
      network-based approaches have produced state-of-the-art algorithms. They all generate some kinds
116   of OOD scores (Yang et al., 2021b). One popular category of methods uses logits to compute such
117   scores (Hendrycks & Gimpel, 2017; Hendrycks et al., 2019; Wei et al., 2022). Some other works
118   also use additional mechanisms (Liang et al., 2017; Sun et al., 2021; Liu et al., 2020a; Wang et al.,
119   2021; Liu et al., 2023b). Many also improve the architecture and features (DeVries & Taylor, 2018;
120   Yu & Aizawa, 2019; Hsu et al., 2020; Huang & Li, 2021; Shama Sastry & Oore, 2019; Sun &
121   Li, 2022). Yet, some others use ensembles (Lakshminarayanan et al., 2017; Gal & Ghahramani,
122   2016; Guo et al., 2017). Some approaches also expose the system with some OOD data during
123   training (Hendrycks et al., 2018; Yang et al., 2021a; Chen et al., 2021; Papadopoulos et al., 2021).
124
      Some work has been done to cluster the detected OOD samples into potential classes (Han et al.,
      2021). However, we do not do that as we learn each OOD sample immediately after it is detected.
125
126   Our OOD detection method is most related to distance-based methods like Mahalanobis distance
127   (MD) (Lee et al., 2018). Ren et al. (2021) addressed some limitations of MD and proposed the
128   Relative Mahalanobis distance (RMD) method. However, our work does both OOD detection and
129   continual learning. Unlike existing OOD detection methods, in our case, the number of ID classes
130
      is not fixed but keeps increasing. We also use newly identified OOD data to detect more OOD data.
131   Continual learning. The existing work mainly focuses on overcoming CF (Castro et al., 2018;
132   Fernando et al., 2017; Kemker & Kanan, 2017). Current methods belong to several categories. Reg-
133   ularization-based methods try to deal with CF in learning a new task by using a regularization term
134   to penalize changes to existing network parameters that are important to previous tasks (Kirkpatrick
135   et al., 2017; Zenke et al., 2017; Li & Hoiem, 2016; Ritter et al., 2018; Schwarz et al., 2018; Xu &
136
      Zhu, 2018; Lee et al., 2019; Ahn et al., 2019). Replay-based methods overcome CF by storing some
      data from previous tasks in a memory buffer. When learning a new task, the saved data and the new
137
      task data are used jointly to train the new task while also adjusting the previous task parameters so
138
      that their performance will not deteriorate significantly (Rusu et al., 2016; Lopez-Paz & Ranzato,
139   2017; Rebuffi et al., 2017; Aljundi et al., 2019a; Chaudhry et al., 2019; Wu et al., 2019; Rolnick
140   et al., 2019; Liu et al., 2020b; Yan et al., 2021; Wang et al., 2022; Guo et al., 2022; Kim et al.,
141   2022a). Pseudo-replay-based methods build a data generator that can generate previous task data
142   to be used in learning a new task (Shin et al., 2017; Wu et al., 2018; Seff et al., 2017; Kemker &
143   Kanan, 2018; Ostapenko et al., 2019; Zhu et al., 2021). The generated data replaces the replay data
144   in the replay-based approach. Parameter-isolation-based methods learn and use masks to protect
145   the learned models for previous tasks so that they will not be updated in learning a new task, which
146   avoids CF (Mallya & Lazebnik, 2017; Abati et al., 2020; Von Oswald et al., 2019; Rajasegaran et al.,
147
      2020; Henning et al., 2021; Serra et al., 2018; Wortsman et al., 2020). Orthogonal projection-based
      methods learn each task in an orthogonal space to the previous task spaces to reduce task interference
148
      or CF (Zeng et al., 2019; Chaudhry et al., 2020; Lin et al., 2022). Recently, the parameter-isolation
149
      approach and OOD detection are combined for class-incremental learning (CIL) (Kim et al., 2022b).
150   However, this approach is not for open-world continual learning, but for traditional CIL.
151
152   Most of the above methods were proposed for batch continual learning, meaning that when a task
153
      arrives, all its training data are available and the learning can be done in any number of epochs.
      However, our work is more related to online continual learning, which learns from a data stream
154
      (Mai et al., 2022). Almost all approaches in online continual learning are based on replay (Chaudhry
155
      et al., 2020; Aljundi et al., 2019a; Shim et al., 2021; Prabhu et al., 2020; Aljundi et al., 2019b;
156   Buzzega et al., 2020; Mai et al., 2021; Cha et al., 2021; Pham et al., 2021; Bang et al., 2021;
157   2022; Koh et al., 2022). Since our method learns after model deployment, we use online continual
158   learning. However, there are some major differences. First, we do not use any replay data. Second,
159   we perform OOD detection and incrementally learn the detected OOD classes on the fly. Our work
160   is most closely related to the work in Hayes & Kanan (2020), which used incremental LDA (Pang
161   et al., 2005) for online continual learning, but it does not detect OOD instances or work in open-
      world continual learning after model deployment. Further, our work does not use incremental LDA.

162
      Some attempts have been made to solve OWCL. Gummadi et al. (2022) proposed the SHELS
163   method for OOD detection and continual learning. However, their method does not truly integrate
164   OOD detection and continual learning like ours. Their two functions can only be evaluated sep-
165   arately and its continual learning is not on streaming data but on traditional batch-based data and
166   training. Bendale & Boult (2015) detected OOD data using SVM and computing the mean of each
167   class for classification. However, for OOD detection, it builds a separate model for each class, which
168   is not in the spirit of continual learning, learning all tasks or classes incrementally in a single model.
169   Rios et al. (2022) proposed the deep learning method (IncDFM) that does OOD detection using a
170   pre-trained model. However, it does not continually learn or integrate with continual learning as we
171
      do. Roy et al. (2022) investigated class-incremental novel class discovery (class-iNCD), focusing on
      strategies for discovering and adapting to new classes over time by training a joint classifier for both
172
      base and novel classes. He & Zhu (2022) addressed OOD detection challenges in the unsupervised
173
      continual learning setting. None of these methods truly integrated OOD detection and continual
174   learning such that the system can learn in the open world after model deployment.

177   3     P ROPOSED APPROACH : O PEN LD
178
179   The proposed new OWCL setting has been discussed in the introduction section. From this subsec-
180   tion, we present the details. We start with the key challenges of OWCL and the main idea of the
181   proposed techniques and their novelties for dealing with the challenges. Recall that OWCL aims to
182   learn after model deployment and the two main steps are: (1) continual OOD detection, and (2) class
183   incremental learning (CIL). As discussed earlier, we assume that the class label for each detected
184   OOD sample can be obtained by asking a human user or another agent while working with them.
185   Both steps are highly challenging. For (1), continual OOD detection is dynamic in nature. Unlike
186   a traditional OOD detection model, which is built based on a set of fixed ID classes. The key
187   novelty of our OOD detection is that we also use the identified OOD samples to detect more OOD
188   samples. For (2), the key challenge is that the AI agent should not ask human users for labels of
189   the detected OOD samples too many times, which means that the proposed method OpenLD must
190   have a strong learning capability without using many labeled samples. For both (1) and (2), there
191   is also the challenge of catastrophic forgetting (CF) in continual learning. OpenLD deals with all
192
      these challenges with the help of linear discriminant analysis (LDA), which we introduce next.
193
194   3.1   L INEAR DISCRIMINANT ANALYSIS (LDA)
195
196   The original LDA classifies two classes {0, 1}. It assumes that the conditional probability density
197   functions p(x|y = 0) and p(x|y = 1) are both normal distributions with parameters of covariance
198   and mean for each class, i.e., (Σ0 , µ0 ) and (Σ1 , µ1 ) (Fisher, 1936). LDA computes (Σ0 , µ0 ) and
199
      (Σ1 , µ1 ) given the training data and learns a classifier based on them.
200   Most LDA methods make the further simplifying homoscedasticity assumption that the class covari-
201   ances are identical, i.e., Σ0 = Σ1 = Σ. Thus, the differences between different classes are in only
202   their means µi ’s. This assumption is particularly useful for continual learning as the system does not
203   have to save one covariance matrix for each class, which can consume a huge amount of memory as
204   more classes are learned. Otherwise, the space required to save covariance matrices for all classes
205
      will be huge. LDA also makes it possible without using any replay data. OpenLD uses multiclass
      LDA, which allows any number of classes to be classified (Rao, 1948).
206
207
208   3.2   O PEN LD METHOD
209
210   The proposed method OpenLD uses a pre-trained model (which is frozen throughout), a continual
211   OOD detection method, and the linear discriminant analysis (LDA) method to solve the OWCL
212
      problem. The pre-trained model will be described in the experiment section. The proposed system
      OpenLD consists of the following steps.
213
214   1. Building the Initial Model M . OpenLD uses the pre-trained model f to provide features for the
215   input samples, which is used by LDA to build a model or classifier M using the initial classes C. As
      mentioned earlier, LDA’s classifier building is based on a mean µi for each class i ∈ C and a single


                                                          4
      Under review as a conference paper at ICLR 2025


216
      shared covariance matrix Σ across all classes. Thus, it produces the shared Σ and a separate mean
217   µi for each class i ∈ C. The resulting model M is deployed in its application (Figure 1).
218
219   2. Post-deployment Continual Learning. After deployment, it continues to learn, which will
220
      update M after new classes are incrementally learned and M becomes M + . In the continual learning
      process, Σ remains unchanged or frozen and it is also used by the newly detected classes.
221
222   Each iteration of post-deployment continual learning has two sub-steps.
223
224
      2.1. Continual OOD Detection and Classification. OpenLD uses M + to detect whether each
      income sample x in the online stream is an OOD sample. If not, it is classified to its class (see
225
      Figure 1). Note that M + is M initially. OpenLD employs the covariance matrix Σ and the µi ’s for
226
      all classes encountered or seen so far to perform the tasks.
227
228   At a steady state, the set of all classes that the system has encountered is C A = C ∪ C L ∪ C E ,
229   where C is the initial set of classes learned in M , C L is a set of new classes that have appeared after
230
      model deployment and are well learned after seeing a good number of training instances, and C E
      is a set of emerging new classes that have been seen but are not well learned yet, i.e., not enough
231
      labeled training data have been seen to well learn the classes. Well learned means that the mean of
232
      the class has stabilized or converged, i.e., it does not change much after more samples are added.
233   A class becomes well-learned if ||µnew − µold || < th, where µold and µnew denote the mean of
234   the class before and after being updated by a sample of the class, and th is a threshold. We denote
235   C + = C ∪ C L as the set of well-learned classes so far and C N = C L ∪ C E as the set of all new
236   classes seen after the deployment of M . With incremental learning, M becomes M + , covering all
237   classes in C A .
238
      What is important here is that OOD detection not only uses the classes in C + but also leverages
239   the covariance Σ and the current un-converged means of the classes in C E to detect OOD samples
240   belonging to C E and other new classes. To our knowledge, no existing method does that. This is
241   advantageous because a new sample may be similar to a class in C E , which makes OOD detection
242   more effective. In Sec. 3.3, we discuss the OOD detection methods used in our OpenLD. If a test
243   sample x is near a class in C E , it is also considered an OOD sample.
244
      2.2 Continual Learning - class-incremental learning (CIL). Here the continual learning setting
245   is CIL, which incrementally learns more and more new classes. Specifically, OpenLD learns each
246   detected novel instance x after obtaining its class label by asking a human user or another knowl-
247   edgeable agent. If x is assigned a class label in C, do nothing (i.e., no learning). If x is assigned
248   a class label ci in C N , the current model M + is updated by updating the mean µi of the class ci
249   (covariance matrix Σ is not changed) as

255   This approach has two desirable properties.
256   (1) OpenLD has no catastrophic forgetting (CF) during OWCL after model deployment because we

      3.3   OOD DETECTION METHODS
264
265   Since OpenLD produces a shared covariance Σ and one mean µi for each class i, we can naturally
266   use Σ and µi related OOD detection methods, i.e., Mahalanobis distance (MD), relative Maha-
267   lanobis distance (RMD), and distance to mean. Each of these methods produces a confidence score
268   using all classes k ∈ C + for the given feature vector z = f (x), where x is the input. If the confi-
269   dence score remains below a threshold level, or it belongs to any class in C E , that input is marked
      as OOD. Note that apart from these methods, there are numerous existing OOD detection methods

      4.1     DATASETS , COMPARED METHODS , PRE - TRAINED MODEL , AND IMPLEMENTATION
312
313   Datasets. We utilize three well-known continual learning benchmark datasets in our experiments.
314   (1) CIFAR-10 (Krizhevsky et al., 2009): This dataset comprises 60,000 color images with dimen-
315   sions of 32x32 pixels. It includes 50,000 training images and 10,000 testing images, distributed
316   evenly across 10 classes. (2) CIFAR-100 (Krizhevsky et al., 2009): This dataset contains 50,000
317   training images (500 per class) and 10,000 testing images (100 per class) of 100 classes, all of which
318   are colored and sized at 32x32 pixels. (3) TinyImageNet (Le & Yang, 2015): This dataset has 200
319   classes, each with 500 training images of 64x64 pixels. The validation set includes 50 images per
320
      class. Since the test data labels are unavailable, we use the validation set for testing.
321   – ID Class Set and OOD Class Set: For each experiment using a dataset, we divide the classes
322   in the dataset into an equal number of ID (in-distribution) classes and OOD classes. The ID class
323   set is used for building the initial model M for deployment, while the OOD class set is used in
      incremental learning after model deployment.


                                                        6
      Under review as a conference paper at ICLR 2025


324
      – ID Train Set and ID+OOD APP Set: We further divide the training set of each class in the ID
325   class set into ID Train set (80%) and ID APP set (20%). We do the same for the OOD class set.
326   ID+OOD APP set includes the data from both the ID and OOD APP sets. The ID+OOD APP set
327   simulates the application (APP) data from a real-life data stream that needs to be classified.

      For CIFAR-100 and TinyImageNet, each class has 500 samples in its original training set. After the

      10 has 5000 samples per class in its original training set, but since we want to simulate the situation
331   where the system does not ask the human user too many questions, we selected 4000 samples per
332   class for the ID Train set and only 100 samples per class for the ID+OOD APP set.
333
334
      – Pre- and Post-Deployment: In pre-deployment, we perform joint training using LDA and a pre-
      trained model (see below) to build the model M using only the ID Train set. ID+OOD APP set is
335
      used only post-deployment.
336
337   Compared Methods. Although there are several related papers (Bendale & Boult, 2015; Gummadi
338   et al., 2022; Rios et al., 2022; Roy et al., 2022), as discussed in Sec 2, no existing system can
339   perform OWCL after model deployment as proposed in this paper. However, we found that the recent
340
      continual learning system TPL (Lin et al., 2024) can be adapted for OWCL as it is based on OOD
      detection. We also create variations of the proposed method OpenLD as ablation experiments.
341
342   – TPL: TPL (Lin et al., 2024) is a state-of-the-art CIL system for traditional continual learning.
343   Since each task in TPL is learned as an OOD detection model and the OOD score for each task is
344   used to predict the task-id of each test sample, it can naturally be adapted for OWCL. Our OpenLD
345   also uses the same pre-trained model as TPL. To suit OWCL, we made it learn in one epoch with a
346
      batch size of 1 to simulate learning from a data stream like OpenLD in post-deployment. ID Train
      set is used to learn ID classes to build the initial model M as the first task, and OOD APP set is
347
      used to learn the OOD classes incrementally. The setting assumes that TPL can do perfect OOD
348
      detection (thus ID APP set is not used). We did not use TPL to do OOD detection as it is hard to
349   set an OOD score threshold because it depends on two factors. However, even in this ideal case for
350   TPL, it performs markedly poorer than OpenLD.
351
352
      – OpenLD[X]: This creates three variations of OpenLD, using three OOD detection methods (X),
      i.e., MD, RMD, or Mean for OOD detection (see Section 3.3).
353
354   – OpenLD[X](-C E ): This also creates three variations of OpenLD without leveraging the un-
355   converged means of the classes in C E to help detect OOD samples. These are for ablation ex-
356   periments and they will show that using C E in OOD detection is very helpful.
357   Two upper-bound methods are also created, which are not continual learning methods.
358
359
      – Joint LDA: This method applies LDA to learn a classifier using the data from all classes together
      or jointly, i.e., ID Train set and OOD APP set. ID APP set is not used as there is no post-deployment
360
      learning or OOD detection in this setting. This method gives the upper bound results of LDA.
361
362   – Joint Fine-tune: This joint method fine-tunes the pre-trained model using ID Train set and OOD
363   APP set. We used the AdamW optimizer with a learning rate of 0.0005, batch size of 256, and
364   trained the model for 20 epochs, which is sufficient for convergence.
365   Pre-trained Models. For feature extraction, our primary pre-trained model is DeiT-S/16 (Touvron
366   et al., 2021) as in (Kim et al., 2022a). To prevent information leaks between the pre-training and
367   continual learning phases, 389 classes in ImageNet, which are similar to the classes in CIFAR-10,
368   CIFAR-100, and TinyImageNet, were excluded and pre-training was performed with the remaining
369   611 classes. In Section 4.3.1, we will also see the results of using several other pre-trained models.
370   Implementation and Resource Usage. We used the LDA implementation in (Hayes & Kanan,
371   2020) and the pre-trained model in (Kim et al., 2022a), which is a DeiT-S/16 Transformer (Touvron

399        4.2               T WO EXPERIMENTAL SETUPS AND EVALUATION MEASURES
400
401       Setup 1) Random ID+OOD APP Data Arrival. This is our main setup as it reflects a normal real-
402       life application scenario. In this setup, after model deployment, the samples in the ID+OOD APP
403       set arrive randomly in a data stream. The current model M + classifies each to its class or detects
404       it as OOD. For each detected OOD sample (which may be correct or wrong), the proposed system
405       OpenLD asks the human user for its class label, and then OpenLD incremental/continual learning is
406
          performed by updating its class mean. Clearly, in our experiment, no human user is involved. The
          system just uses the class label of the sample in the original data.
407
408       Setup 2) Class-Incremental OOD APP Data Arrival. This setup is for scenarios where an AI
409       agent keeps going to completely new environments. This is very unlikely. We use it to show the
410       robustness of our approach. Each environment has a set of new OOD classes as a new task. This is
411       similar to class incremental learning (CIL). In our case, we divide the classes in the OOD class set
412
          evenly into 5 tasks (5 environments). Each task is created to include one-fifth of random samples
          from the ID APP set and all samples in the ODD APP set of the classes in the task. The OOD
413
          samples from one task finish before the data from the next task arrive. Acquiring the OOD sample
414
          class label and updating the class means for incremental/continual learning are the same as above.
415
416       Evaluation Measures. We conduct two types of evaluations.
417       (i). Draw a curve to show how the accuracy changes as the number of asks increases. Setup 1):
418       For CIFAR-10, for every 25 asks (i.e., detected OOD samples, some of which may be true OOD
419       samples and some of which may be not), accuracy is computed using the test data from all classes.
420       For CIFAR-100 and TinyImageNet, for every 100 asks, accuracy is computed as these two datasets
421       have many more OOD classes. Setup 2): It is similar. For every 25 asks for CIFAR-10 and every
422       100 asks for CIFAR-100 and TinyImageNet, accuracy is computed using the test data of the classes
423       that the model has seen so far.
424       (ii). Final test accuracy after the system has seen all the data in ID+OOD APP set. Final F score
425       for OOD detection also after the system has seen all the data in ID+OOD APP set.

475   constantly, the results will improve further and there will be less and less need to learn new features
476   or fine-tune the pre-trained models, which can cause catastrophic forgetting (CF).
477
      Efficiency. Since OpenLD uses the statistical techniques LDA in learning and MD for OOD detec-
478   tion, it is much more efficient than deep learning. In the post-deployment stage, the system performs
479   only OOD detection and updating of the mean of each new class (no training is involved), which
480   takes almost no time (less than 15 milliseconds), because the features are already learned in the
481   pre-trained model. This makes OpenLD especially suitable for OWCL and real-time processing.

           5                 C ONCLUSION
528
529       This paper proposed the setting of open-world continual learning (OWCL) after model deployment.
530       The OWCL setting is getting more and more important as more and more AI agents are deployed
531       in real-life applications. It is highly desirable that these agents can learn continually after deploy-
532       ment to improve themselves and become more and more knowledgeable over time. This paper also
533       proposed a method (called OpenLD) based on LDA and a pre-trained model with several novel
534       techniques to improve OOD detection in the OWCL process and to learn new classes easily by only
535       updating class means, which has no catastrophic forgetting (CF). Experiment results have demon-
536
          strated the effectiveness of the proposed method OpenLD.
537       Limitations. (1) The pre-trained model is frozen, meaning no feature learning from new data,
538       although OpenLD is better than TPL that does feature learning. (2) The type of OOD detection
539       method is limited to distance-based methods because LDA only generates a shared covariance and
          a mean for each class. In our future work, we will try to overcome these limitations.
