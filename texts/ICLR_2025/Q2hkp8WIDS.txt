011                                               A BSTRACT
012
013            Deep reinforcement learning has achieved remarkable success in learning con-
014            trol policies from pixels across a wide range of tasks, yet its application remains
015
               hindered by low sample efficiency, requiring significantly more environment inter-
               actions than humans to reach comparable performance. Model-based reinforcement
016
               learning (MBRL) offers a solution by leveraging learnt world models to generate
017
               simulated experience, thereby improving sample efficiency. However, in visually
018            complex environments, small or dynamic elements can be critical for decision-
019            making. Yet, traditional MBRL methods in pixel-based environments typically rely
020            on auto-encoding with an L2 loss, which is dominated by large areas and often
021            fails to capture decision-relevant details. To address these limitations, we propose
022            an object-centric MBRL pipeline, which integrates recent advances in computer
023            vision to allow agents to focus on key decision-related elements. Our approach
024            consists of four main steps: (1) annotating key objects related to rewards and
025            goals with segmentation masks, (2) extracting object features using a pre-trained,
026
               frozen foundation vision model, (3) incorporating these object features with the
               raw observations to predict environmental dynamics, and (4) training the policy
027
               using imagined trajectories generated by this object-centric world model. Building
028
               on the efficient MBRL algorithm STORM, we call this pipeline OC-STORM. We
029            demonstrate OC-STORM’s practical value in overcoming the limitations of con-
030            ventional MBRL approaches on both Atari games and the visually complex game
031            Hollow Knight. Code and videos are available in the supplementary materials.

034   1    I NTRODUCTION
035
036   Over the past decade, deep reinforcement learning (DRL) algorithms have demonstrated remarkable
037   capabilities across a wide-range of tasks (Silver et al., 2016; Mnih et al., 2015; Hafner et al., 2023).
038   However, applying DRL to real-world scenarios remains challenging due to low sample efficiency,
039
      meaning DRL agents require significantly more environment interactions than humans to achieve
      comparable performance. A promising solution to this problem is model-based reinforcement
040
      learning (MBRL) (Sutton & Barto, 2018; Ha & Schmidhuber, 2018). By utilizing predictions from a
041
      learned world model, MBRL enables agents to generate and learn from simulated trajectories, thereby
042   reducing reliance on direct interactions with the real environment and improving sample efficiency.
043
044   Recent MBRL methods typically train the world model in a self-supervised manner (Hafner et al.,
045
      2023; Zhang et al., 2023; Micheli et al., 2023; Alonso et al., 2024). These methods first map
      environmental observations into low-dimensional latent variables and then train an autoregressive
046
      sequence model over these latent variables. This mapping is usually achieved through a variational
047
      autoencoder (Kingma & Welling, 2014; van den Oord et al., 2017), with the learning objective often
048   being the reconstruction of the input observation, typically using L2 loss or Huber loss (Huber, 1964).
049   While auto-encoding is simple and effective in many cases, it can fail to capture decision-relevant
050   information. For example, when decision-relevant targets are too small, the background is dynamic,
051   or there are too many decision-irrelevant objects in the scene, the reconstruction can easily miss these
052   key targets, leading to poor agent performance. Moreover, even if the autoencoders produce good
053   reconstruction, there’s no guarantee the resulting latent variables are useful for control tasks (Zhang
      et al., 2021).

054
      Meanwhile, recent advances in computer vision, such as open-set detection and segmentation tech-
055   nologies including SAM (Kirillov et al., 2023; Ravi et al., 2024), Cutie (Cheng et al., 2023), and
056   GroundingDINO (Liu et al., 2023), have revolutionized our ability to identify objects in diverse
057   environments. These models excel at detecting or segmenting objects in out-of-domain cases without
058   further finetuning. By integrating these capabilities into reinforcement learning, then the agent can
059   immediately focus on essential decision-relevant elements, bypassing the need to study how to extract
060   key information from raw observations.
061   To mitigate the limitations of reconstruction losses in previous MBRL methods, we propose an
062   object-centric model-based reinforcement learning pipeline that leverages these advances in
063   computer vision. This pipeline involves four steps:
064
            1. Annotating key objects in a small number of frames using segmentation masks.
065
            2. Extracting object features through a parameter-frozen pre-trained foundation vision model
066
               conditioned on these annotations. In this work, we use Cutie (Cheng et al., 2023).
067         3. Utilizing both these object features and the raw observations as inputs for training an object-
068            centric world model that predicts the dynamics of the environment while considering the
069            relationships between different objects and the scene.
070         4. Training the policy with imagined trajectories generated by the world model.
071
      Since the MBRL component of this pipeline is based on STORM (Zhang et al., 2023), we name
072
      our method OC-STORM. To our knowledge, we are the first to successfully adopt object-centric
073   learning on Atari and the visually more complex Hollow Knight without relying on an extensive
074   number of labels or accessing internal game states (Delfosse et al., 2023; Jain, 2024). OC-STORM
075   outperforms the baseline STORM on 18 of 26 tasks in the Atari 100k benchmark and achieves the
076   best-known sample efficiency on several Hollow Knight bosses.
077
      In summary, the main contribution of this work is an object-centric model-based reinforcement
078
      learning pipeline, OC-STORM, that enables the agent to immediately focus on useful object features,
079   increasing sample efficiency and in many cases enabling better performance. Furthermore, this work
080   represents a successful integration of modern vision models with reinforcement learning frameworks,
081   which may provide insights for the research community.

087   2.1   O BJECT EXTRACTION
088
089   Model selection Object detection and segmentation have been active areas of research, leading to
090
      the development of various influential methods. Appendix A provides a brief review of these methods
      which we considered for extracting object representations for reinforcement learning agents. After
091
      considering many possible methods (Cheng & Schwing, 2022; Kirillov et al., 2023; Zhang et al.,
092
      2024; Ravi et al., 2024; Redmon et al., 2016; Jocher et al., 2023; Liu et al., 2023; Locatello et al.,
093   2020; Kipf et al., 2022; Elsayed et al., 2022; Wang et al., 2023; Xu et al., 2023), we selected Cutie
094   (Cheng et al., 2023) as the object feature extractor for the following reasons:
095
096
             • Cutie provides a compact vector representation of objects. Compared to using bounding
               boxes or segmentation masks, this representation directly offers a compressed, high-level
097
               summary of an object’s state and position, allowing downstream models to bypass the need
098
               for re-learning and extracting visual features.
099          • Cutie is a video object segmentation algorithm capable of generating consistent representa-
100            tions across frames. While image-based methods can track objects in videos using matching
101            algorithms, such an approach would require additional complexity.
102          • Cutie is a retrieval-based algorithm and handles few-shot annotations effectively with a
103            memory system. In complex environments, one-shot (single-frame) annotations or natural
104            language descriptions can be insufficient to cover the different states of an object.
105          • Cutie generalizes well to out-of-domain inputs. None of the mentioned algorithms were
106            trained on frames from Hollow Knight or Atari games. Our experiments show that Cutie is
107
               robust for use on these out-of-domain games. This generalization capability demonstrates
               that our approach could be applied to other environments as well.

      2.2   M ODEL - BASED REINFORCEMENT LEARNING
136
137   MBRL involves two steps. In the model learning step, the agent uses collected data to train a
138   predictive model of the environment’s dynamics. In the policy optimization step, the agent uses
139   this model to simulate the environment to improve the policy. Although real experience could also
140   be used for training, modern MBRL methods often rely solely on simulated trajectories for policy
141   optimization.
142   Ha & Schmidhuber (2018) first demonstrated the feasibility of learning by imagination in pixel-based
143   environments. SimPLe (Kaiser et al., 2020) further extended this idea to Atari games (Bellemare
144   et al., 2013), though with limited efficiency. The Dreamer series (Hafner et al., 2019; 2021; 2023)
145   employs categorical variational autoencoders and recurrent neural networks (RNNs), achieving robust
146   performance across diverse domains. Dreamer introduces both a stable discretization method and
147   a set of techniques for robust optimization across domains with diverse observations, dynamics,
148   rewards, and goals. TWM (Robine et al., 2023) and STORM (Zhang et al., 2023) replace the RNN
149
      sequence model in Dreamer with transformers, enhancing parallelism during training. TWM encodes
      the observation, reward, and termination as three input tokens for the transformer, while STORM
150
      encodes them as a single token, demonstrating better efficiency. IRIS (Micheli et al., 2023; 2024)
151
      and its variant REM (Cohen et al., 2024) utilize VQ-VAE (van den Oord et al., 2017) for multi-token
152   latent representations. DIAMOND (Alonso et al., 2024) employs a diffusion process as the world
153   model, further improving the final performance. All of these methods predominantly use an L2
154   reconstruction loss for self-supervised learning.
155

      2.3   O BJECT- CENTRIC REINFORCEMENT LEARNING
157
158   Object-centric learning has gained increasing attention in both the machine learning and cognitive
159   psychology fields (Driess et al., 2023; Delfosse et al., 2023). Human infants inherently possess an
160   understanding of objects (Spelke, 1990), suggesting that object extraction from visual observations
161   may be fundamental for high-level decision-making. From a data processing perspective, utilizing
      object information can significantly reduce computational costs compared to raw visual inputs.


                                                          3
      Under review as a conference paper at ICLR 2025


162
      Many attempts have been made to introduce object-centric learning to reinforcement learning systems.
163   However, to our knowledge, no existing methods could be directly applied to Atari games or Hollow
164   Knight without leveraging internal game states or an extensive number of annotations. These
165   object-centric learning methods broadly follow two main trends: two-stage and end-to-end.
166
      Two-stage methods usually first use computer vision models or techniques to detect objects, then
167
      train the policy based on this object-level information. Current approaches often require labour-
168
      heavy task-specific fine-tuning (Devin et al., 2018; Liu et al., 2021), access to game memories
169   (Delfosse et al., 2023; Jain, 2024), or leverage game-specific observation structures (Stanic et al.,
170   2024). FOCUS (Ferraro et al., 2023), the most similar work to ours, is a model-based method that
171   uses TrackingAnything (Yang et al., 2023) to generate segmentation masks, which are then fed
172   into DreamerV2 (Hafner et al., 2021) for policy training. However, using binary masks for object
173   representation limits efficiency, which will be discussed in Section 5.2. Moreover, FOCUS has
174   only been tested on six robot control tasks and hasn’t been fully explored in more visually complex
175   environments.
176   End-to-end methods jointly learn object perception and policy, often using unsupervised slot-based
177   approaches (Locatello et al., 2020) to discover and represent objects. While these methods allow the
178   visual module to be trained alongside the world model or policy network, their unsupervised learning
179   nature leads to poor object detection quality, especially in noisy, real-world scenes. As a result,
180   they are typically limited to simple object-centric benchmarks (Watters et al., 2024; Ahmed et al.,
181   2021) and struggle to generalize to visually complex tasks. Several model-based (Veerapaneni et al.,
182   2019; Lin et al., 2020; van Bergen & Lanillos, 2022) and model-free (Yoon et al., 2023; Haramati
183
      et al., 2024) algorithms have used these ideas. Nakano et al. (2024) added slot attention to STORM,
      achieving stronger performance on the OCRL benchmark (Yoon et al., 2023). Our work also builds
184
      on STORM, but we use a pre-trained vision model instead of unsupervised slot attention, allowing us
185
      to better handle visually complex environments.

215   to play a game without being told the rules, there is no reason not to inform agents of key objects.

246
      Modelling an autoregressive sequence model on raw inputs often results in compounding errors
247
      (Hafner et al., 2023; Zhang et al., 2023; Alonso et al., 2024). To mitigate this, we employ a categorical
248   VAE (Kingma & Welling, 2014; Hafner et al., 2023), which transforms input states st into a discrete
249   latent stochastic variable zt , as formulated in Equation 2. The VAE encoder (qϕ ) and decoder (pϕ ) are
250   implemented as multi-layer perceptrons (MLPs) for object feature vectors and convolutional neural
251   networks (CNNs) for visual observations:
252
253                            Categorical VAE encoder:                     zt ∼ qϕ (zt |st ) ∈ RK×16×16 or R32×32 ,
254
                                                                                                                                                                                        (2)
                               Categorical VAE decoder: ŝt = pϕ (zt ) ∈ RK×2048 or R3×64×64 .
255
256   As sampling from a distribution lacks gradients for backpropagation, we apply the straight-through
257   gradient trick (Bengio et al., 2013; Hafner et al., 2021) to retain them. The VAE treats each of the
258   K objects independently. Each latent variable comprises 16 categories with 16 classes for an object
259   and 32 categories with 32 classes for the visual input. The configuration of 32 × 32 is inherited from
260   prior work (Hafner et al., 2023; Zhang et al., 2023; Robine et al., 2023), while the 16 × 16 design is
261   motivated by the fact that a single object contains less information than the entire scene.

      3.5   O PTIMIZATION METHODS FOR THE WORLD MODEL AND THE POLICY
294
295   The world model is trained in a self-supervised manner, optimizing it end-to-end. The policy is
296   trained over simulated trajectories generated by the world model and is optimized with a model-free
297   actor-critic algorithm. Our setup closely follows DreamerV3 (Hafner et al., 2023), which is also
298   similar to other MBRL methods (Zhang et al., 2023; Micheli et al., 2023; Robine et al., 2023). Full
299   details are provided in Appendix C.

301   4     E XPERIMENTS
302
303   We first evaluate the performance of our method on the Atari 100k benchmark (Bellemare et al.,
304   2013), which serves as a standard testbed for measuring the sample efficiency of MBRL methods
305   (Kaiser et al., 2020; Micheli et al., 2023; Hafner et al., 2023). We then further test our method on
306   Hollow Knight (TeamCherry, 2017), which is a highly acclaimed game released in 2017. The core
307   gameplay of Hollow Knight revolves around world exploration and combat with enemies, and we
308   focus on combat with bosses in this work. Compared to Atari games, Hollow Knight’s boss fights are
309   visually more complex, with most key information representable as objects, making it well-suited to
310   demonstrating the capabilities of our proposed pipeline.
311   As outlined in Section 3, our method can utilize either object features, visual observations, or both.
312   In this section, all reported results from our method incorporate both types of inputs. A more detailed
313   analysis of input selection will be presented in Section 5.2.

472   6     L IMITATIONS AND FUTURE WORK
473
474   Our method has two main limitations, each of which corresponds to a potential future enhancement:
475   Duplicated instances: Current video object segmentation algorithms are primarily developed and
476   trained to track a single object. When a scene contains two or more identical or similar objects,
477   approaches like Cutie (Cheng et al., 2023) may fail to segment each object correctly. For example,
478   this could occur with the beans in Atari MsPacman, the two lords in Hollow Knight’s Mantis Lords
479   (as shown in Appendix H), or a group of sheep in the wild. Even if the model provides accurate
480   segmentation for these duplicated instances, it may still generate only a single feature vector for these
481   multiple instances, which may confuse the model and lead to performance degradation. Although
482
      addressing this issue is beyond the scope of this work, detecting duplicated instances in a scene is
      feasible (Redmon et al., 2016; Ren et al., 2017; Jocher et al., 2023; Carion et al., 2020), and we
483
      believe that future methods will be able to manage such situations effectively.
484
485   Background representation: Our object representations do not capture elements that cannot be
      easily described as objects or compact vectors, such as walls, map boundaries, or the overall scene


                                                                                                                  9
      Under review as a conference paper at ICLR 2025


486
      layout. However, some of these elements may be important for reward signals and decision-making,
487   and so we must also provide the raw visual observation. For example, the tunnel in Atari Gopher is
488   critical, yet difficult to encode as an object within our model structure (also shown in Appendix H).
489   Detail changes in such non-object information could also suffer from L2 reconstruction loss so that it
490   may not be fully represented. Including information around the object, as in Lin et al. (2020), is a
491   potential workaround. However, this assumes that everything impacting an object is close to it, which
492   might not hold true in cases of contactless interactions, such as with two magnets. This represents a
493   general limitation of any object-centric representation method.
494
495   7   C ONCLUSIONS
496
497   In this work, we introduced OC-STORM, an MBRL pipeline designed to improve sample efficiency
498   in visually complex environments. By integrating recent advances in object segmentation and
499   detection, we mitigate the limitations of traditional reconstruction-based MBRL methods, which
500   may be dominated by large background areas and overlook decision-relevant details. Through
501   experiments on Atari and Hollow Knight, we demonstrated that object-centric learning could be
502   successfully implemented without relying on internal game states or extensive labelling, highlighting
503   the adaptability of our method to complex, visually rich environments. OC-STORM represents a
504
      meaningful step toward combining modern computer vision with reinforcement learning, offering an
      efficient framework for training agents in visually complex settings.
505
506   As the development of foundation vision models continues to advance, this pipeline is well-positioned
507   to evolve alongside these improvements, harnessing even more powerful and efficient techniques
508   for object detection and segmentation. This evolution will further enhance our ability to disentangle
509   perceptual learning from policy learning, simplifying the optimization challenges of reinforcement
510
      learning and paving the way for more intuitive and robust agent behaviours in increasingly complex
      scenarios.

903   B    R ECONSTRUCTION ANALYSIS IN STORM
904
905   As mentioned in Section 1, MBRL methods that rely on L2 reconstruction loss may miss key elements
906   for controlling. Here, we present a qualitative reconstruction example using STORM (Zhang et al.,
907   2023), as shown in Figure 5. Since the 64 × 64 image processed by the model may be hard to interpret,
908
      a high-resolution sample is provided in Figure 6.
909   The two main characters in the game are the Knight on the left (in white and black) and the boss
910   Hornet Protector on the right (in red). The 9 “masks” in the top left represent the Knight’s remaining
911   health, showing 1 health point remaining and 8 lost in the case depicted in Figure 6.
912   The autoencoder captures static or large-area features, such as lighting, shadows, streaks, smoke,
913   and health indicators, which are not crucial for gameplay or rewards. However, the model struggles
914   with character positions and states. While MBRL methods have shown nearly perfect reconstruction
915   and simulation in some simpler environments like Atari games (Micheli et al., 2023; Zhang et al.,
916   2023), they face challenges in visually complex environments such as Hollow Knight or real-life
917   scenes. Similar difficulties could also be observed in Minecraft (Guss et al., 2021), as depicted in the
      DreamerV3 paper (Hafner et al., 2023).

1002   action space is employed.

       Despite its popularity among players, Hollow Knight has seen limited use as a benchmark for research

1053
       that leverage or explore Hollow Knight as a benchmark. Cui (2021) employs DQN (Mnih et al., 2015)
1054

1055   Yang (2023) uses the Rainbow algorithm (Hessel et al., 2018) with additional techniques like DrQ

1057   repository has been widely forked and adopted. Building on his work, Lee (2023) studies the effect

1059   interaction configuration and switching to the PPO algorithm (Schulman et al., 2017). Jain (2024)
1060   leverages internal game states to extract hitboxes as input for the algorithm, representing them as
1061   segmentation masks that are passed to DQN or PPO.

1063   D.2   E NVIRONMENT CONFIGURATION
1064
1065
       Hollow Knight is a modern video game developed with Unity (Technologies, 2005). To our knowl-
       edge, efficient simulators for this game, such as those available for Atari (Brockman et al., 2016;
1066
       Towers et al., 2023), do not exist. Therefore, we developed a custom wrapper that captures screenshots
1067
       of the game at 9 FPS and sends keyboard signals to execute actions. The 9 FPS rate is a choice
1068   based on the author’s experience with the game and considerations for computational efficiency. To
1069   obtain reward signals, we developed a modding plugin (Bham & Wyza, 2017) that logs when the
1070   player-controlled character (the Knight) either hits an enemy or is hit. Our wrapper then parses this
1071   log file to generate reward and termination signals.
1072
       The game execution and agent training are conducted on a Windows machine. To monitor training
1073
       progress and statistics without interrupting the game, we needed a method to send keyboard inputs to
1074   a background or unfocused window. However, Windows lacks an API for this purpose. As a result,
1075   the game window must remain in the foreground, fully occupying the training device and hindering
1076   monitoring. To address this, we utilized a Hyper-V (Cooley, 2022) Windows virtual machine to
1077   run the game in the background, with Ray (Moritz et al., 2018) facilitating communication between
1078   the host and virtual machine. Training and processing occur on the host machine, while the virtual
1079   machine handles interactions with the environment. This setup can be extended to distributed nodes,
       with some handling game rendering and others managing training tasks.

1084   D.3    ACTION SPACE
1085
1086   All previous works utilize a human-specified action space rather than the original keyboard inputs.
1087   For example, in the Yang (2023) implementation, short and long jumps are treated as two distinct
1088   actions, which are originally controlled by the duration of the jump button press. His environment
1089   wrapper handles this difference with a fixed command. While this design reduces the exploration and
1090   computational costs for reinforcement learning agents, it cannot capture the full range of possible
1091
       actions in the game. Advanced operations, as demonstrated in this video (CrankyTemplar, 2018),
       require full control of the keyboard. Therefore, we design the action space as a multi-binary-discrete
1092
       one that directly binds to the press and release of the physical keyboard, which will be explained in
1093
       Section D.3.
1094
1095   We design the action space as a multi-binary-discrete space directly tied to the press and release
1096   states of eight specific keys on the keyboard. These keys include W, A, S, and D for movement
1097
       directions, and J, K, L, and I for attack, jump, dash, and spell actions, respectively. Each key’s state
       is represented as a binary variable, where 0 corresponds to a key release and 1 corresponds to a key
1098
       press. The action can therefore be described as a ∈ {0, 1}8 , with each element representing the
1099
       binary state of one key. The key’s state is maintained between frames, and a toggle signal is sent only
1100   when there is a change in the key state from 0 → 1 (press) or 1 → 0 (release).
1101
1102   The probability of an action is determined by the independent probabilities of each key’s state:

1105                                                       k=1

1114   complex manoeuvres while maintaining a tractable exploration space for reinforcement learning.

1116   D.4    R EWARD SHAPING IN H OLLOW K NIGHT
1117
1118
       Most existing methods (Yang, 2023; Sun, 2024; Jain, 2024; Cui, 2021; Lee, 2023) for Hollow Knight
       use a reward structure of +1 for hitting an enemy and -1 for taking damage. Some approaches
1119
       modify the weighting ratios, while others introduce auxiliary rewards for performing specific actions.
1120
       However, we found that these settings are suboptimal for training reinforcement learning agents.
1121
1122   Our method assigns a +1 reward signal for hitting an enemy and a virtual termination signal upon
1123   being hit. The game continues until the episode naturally ends. The termination signal is stored in
1124
       the replay buffer for training the world model, treating health loss as a life-loss event. Leveraging
       life-loss information is a common technique that aids in value estimation (Ye et al., 2021; Micheli
1125
       et al., 2023; Zhang et al., 2023; Alonso et al., 2024). Additionally, the Knight can damage enemies
1126
       in multiple ways, and these damages are normalized against the base attack damage to compute the
1127   positive reward.
1128

       improvement can be analyzed from two perspectives:

1219   E.1    ATTENTION - BASED POLICY OR MLP- BASED POLICY
1220
1221   When handling multiple objects, naturally one would think of an attention-based policy network like
1222   the self-attention predictor described in Figure 2. A previous work OC-SA (Stanic et al., 2024) has
1223   also explored such structure. However, we still design our actor and critic networks as MLPs which
1224   take the concatenation of object latent variables and hidden states as input.
1225   We found that the attention-based policy tends to overfit pre-learned behaviours and makes it hard
1226   to learn new knowledge. This won’t be a major issue in stationary games like Boxing but will face
1227   trouble in non-stationary games like Pong. For example, the attention-based policy can quickly learn
1228   how to catch the ball but can’t efficiently learn how to score against the opponent. On the one hand,
1229   we can confirm that by visually checking the rendered episodes. On the other hand, numerically
1230   speaking, we can observe the episode length of playing Pong. If the episode length increases while
1231   the episode returns remain at the same level, then we can tell that the agent learns how to catch the
1232
       ball, but is stuck in that local optimum.
1233   As the results plotted in Figure 10, we can tell that attention-based policy suffers from that issue. The
1234   episode length of both policies rises at a similar speed before 50k steps, but it declines slower for the
1235   attention-based policy after that. The experiments are conducted using only the object module, so the
1236   MLP-based policy curves are identical to the ”vector” ones in Figure 4. As the visual latent itself
1237   contains all the information, the agent can choose only to use that part of the information and thus
1238
       may affect our judgement on the effectiveness of the attention-based policy.
1239   Though the attention-based policy has the potential to handle a dynamic number of objects, our
1240   experiments are conducted on a fixed number. As it doesn’t demonstrate superior performance
1241   than the MLP-based policy in our case, we always use MLP-based in other tasks for consistency in
       evaluation.

       F.2   M ODIFICATIONS FOR INTERGRATION
1297
1298   We make no modifications to the official implementation, except for caching and copying internal
1299   variables.
1300
       The only special process involves setting the object feature vector to 0 when Cutie loses track of
1301
       the object. Cutie uses an attention guidance mask within its object transformer, which restricts
1302   which visual features the object feature can attend to. This mask is trained as part of an auxiliary
1303   segmentation task. When the attention guidance mask is set to all 1s (0 allows attention and 1
1304   rejects it), indicating that Cutie cannot find strong evidence of the object’s presence in the scene, the
1305   transformer theoretically should reject all attention from the visual features.
1306
       However, in this situation, Cutie inverts the mask, allowing the object feature to attend to all visual
1307
       features in an attempt to search for the object in the scene. As a result, the attention becomes scattered
1308   across the observation space, leading to unpredictable output for the object feature. This unpredictable
1309   behaviour complicates learning in the world model.
1310
1311
       To address this, we set the object feature vector to 0 when the attention guidance mask is entirely 1.
       This informs the world model that the object feature is missing, rather than reflecting a random state.
