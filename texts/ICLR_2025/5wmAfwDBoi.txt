009                                               A BSTRACT
010
011            Building autonomous UI agents that automate user interactions with interfaces has
012            long been a vision in the field of artificial intelligence. Central to these agents is
013            the capability for UI element grounding, which involves accurately locating UI
014            elements (e.g., buttons and links) based on referring expression, such as user intents
015            and functionality descriptions. Developing these agents with robust grounding
016
               capabilities using vision-language models (VLMs) offers a promising path forward.
               However, a practical framework for creating VLMs with strong element grounding
017
               capabilities remains under-explored. To address this gap, we conduct systematic
018
               experiments within the design space of VLMs to uncover an effective recipe for
019            building VLMs with strong UI element grounding ability. Firstly, we find that
020            fine-tuning with general visual grounding tasks as a warming-up step mitigates the
021            challenges of fine-tuning with downstream UI element grounding data. Next, we
022            explore different fine-tuning sequences of UI grounding training data from various
023            sources and find that a simple-to-complex fine-tuning curriculum can maximize
024            data utility. Moreover, we find that scaling up the size of either the warming-up
025            data or the UI grounding data in downstream fine-tuning significantly enhances UI
026            element grounding accuracy. Lastly, we explore various image feature compression
027
               techniques and find that using a convolution-based compressor to compress UI sub-
               image features significantly enhances the grounding capabilities on high-resolution
028
               UI images. Integrating these insights, we successfully develop UI-Pro, an expert
029
               VLM that achieves state-of-the-art UI grounding accuracy with fewer parameters
030            across multiple benchmarks. We hope this work serves as a valuable roadmap for
031            researchers in the UI-VLM domain and inspires future research.

034   1    I NTRODUCTION
035
036   The concept of autonomous UI agents capable of clicking, typing, and scrolling on behalf of humans
037   as personal assistants is an enticing prospect, as illustrated in Fig. 1. Imagine a UI agent navigating
038   the Internet to perform daily tasks such as using search engines and managing emails, as well as more
039
      complex activities like comparing prices across e-commerce platforms and collecting the latest news
      on stock markets.
040
041   At the core of autonomous UI agents is UI element grounding, which involves recognizing and locating
042   elements associated with referring expressions. These elements serve as the fundamental building
043   blocks that carry UI functionalities. Accurate grounding allows UI agents to interact effectively
044   with UI components such as buttons, text fields, and images, enabling them to perform tasks like
045
      clicking, filling out forms, and extracting information according to user instructions. Developing these
      agents based on vision-language models (VLMs) (Yin et al., 2023) offers a promising pathway toward
046
      realizing this vision. A VLM-based UI agent can directly perceive and interact with UIs as humans do,
047
      provided the agent possesses vision and comprehension capabilities that align with those of humans.
048   Although a few prior studies, such as SeeClick (Cheng et al., 2024) and CogAgent (Hong et al.,
049   2023) have explored developing UI-related VLMs, a practical recipe for building VLMs with robust
050   UI grounding capabilities from scratch remains under-explored. Specifically, it is unclear which
051   combination of data is most effective for instilling UI grounding capabilities in VLMs and whether
052   non-UI-related multimodal understanding tasks can serve as useful training data. Furthermore,
053   the optimal design of model architectures and training procedures to enhance the models’ ability
      to perform element grounding on high-resolution UI screenshots is still uncertain. Addressing

071   these questions necessitates thorough exploration within the vast design space of VLMs. While
072   several studies (Tong et al., 2024; McKinzie et al., 2024; Laurençon et al., 2024) have examined
073   the significance of various model components and data choices, they predominantly focus on visual
074   question answering (VQA) in natural images, overlooking the complex challenges presented by UI
075
      grounding scenarios.
076   This paper aims to bridge the gap in existing research by providing a practical framework for building
077   VLMs capable of UI grounding. Drawing on pioneering research (Tong et al., 2024; McKinzie et al.,
078   2024; Baechler et al., 2024), we pinpoint three key areas where different studies make distinct design
079   choices: (a) model architecture, particularly vision-language connection modules that enhance the
080   accuracy of locating small elements within high-resolution UI screenshots, presenting new challenges
081
      rarely addressed in visual grounding for natural images; (b) training data curation; and (c) fine-tuning
      procedures. We systematically compare various design choices in a controlled setting to derive
082
      empirical insights for each area.
083
084   Our findings reveal that: (a) warming up VLMs with general visual grounding task data is essential
085   before fine-tuning the models on downstream UI grounding tasks; (b) organizing UI grounding
086   training data in a simple-to-complex curriculum significantly maximizes data utility through multi-
087
      stage fine-tuning; (c) increasing the sizes of both the warming-up data and UI grounding data
      results in substantial performance gains, well beyond saturation; and (d) a lightweight convolution-
088
      based connector is effective for compressing visual features of high-resolution UI images, enabling
089
      processing at the original ratio.
090
091   Our work distinguishes itself from previous studies (Yao et al., 2022; Cheng et al., 2024; Hong et al.,
092   2023; You et al., 2024b) on UI-oriented VLMs by exploring previously unexamined areas, including
093
      warming-up data selection, multi-stage training methodologies, data scaling effects, and UI-oriented
      connector design. Our findings provide a hidden recipe for building powerful UI VLMs from scratch,
094
      circumventing reliance on fine-tuning open-source models.
095
096   Based on these insights, we have trained UI-Pro, a VLM with 2.8 billion parameters. UI-Pro demon-
097   strates exceptional performance across multiple UI grounding benchmarks, including grounding by
098   action intents, element appearance, and complex functionality descriptions. Notably, UI-Pro matches
099
      the performance of previous UI-oriented models that are nine times its size. We hope our work will
      benefit the research community and accelerate advancements in UI autonomous agents.

104   2.1   UI E LEMENT G ROUNDING TASK
105
106   UI element grounding is to locate visual elements within UIs given element annotations. These
107   annotations can be brief, including details such as element appearance, location, and displayed text,
      or complex, encompassing contextual functionality and action intents, as shown in Fig. 1.


                                                                                2
      Under review as a conference paper at ICLR 2025


108
      Several element grounding benchmarks have been established for research purposes (Fig 1).
109   ScreenSpot (Cheng et al., 2024) is a benchmark for mobile, desktop, and web scenarios, requiring
110   models to locate elements based on brief descriptions. RefExp (Bai et al., 2021) focuses on locating
111   elements on mobile devices using crowd-sourced referring expressions. VisualWebBench (Liu
112   et al., 2024c) evaluates VLMs in content-rich web environments. In contrast to these benchmarks,
113   AutoGUI Test (FuncPred) features complex tasks that require models to locate elements specified
114   by context-specific functionality descriptions. For all these benchmarks, we report the grounding
                              PN
115   accuracy (%): Acc = i=1 1 (predi inside GT bbox i ) /N × 100
116
      where 1 is an indicator function and N is the number of test samples. This formula denotes the
117   percentage of samples for which the predicted points lie within the bounding boxes of the elements.
118
119   Unlike visual grounding aimed at natural scenes (Yu et al., 2016), UI element grounding introduces
120
      distinct challenges: 1) High resolution: UIs are typically rendered as high-resolution images, ne-
      cessitating models that can process large visual inputs effectively. 2) Fine-grained comprehension:
121
      UIs often display numerous small elements, which occupy significantly less area than objects in
122
      datasets like RefCOCO (Yu et al., 2016) and Visual Genome (VG) (Krishna et al., 2016), requiring
123   enhanced visual understanding to distinguish between highly similar elements. 3) Data insufficiency.
124   Due to the substantial cost of human annotation, the scale of existing open-source datasets for UI
125   understanding is significantly lower than natural image datasets such as COCO (Lin et al., 2014) and
126   LAION-5B (Schuhmann et al., 2022).
127
128   2.2      BASICS OF VLM S
129
130   We adopt the popular architecture used by recent VLMs, such as LLaVA (Liu et al., 2023) and
131   Qwen-VL (Bai et al., 2023). These architectures typically combine a pre-trained visual backbone
132
      fϕ (e.g., ViT (Dosovitskiy et al., 2021)) and a large language model (e.g., Llama Touvron et al.
      (2023)) to build a model capable of processing both textual and visual inputs. Formally. the visual
133
      backbone maps an input image I ∈ RH×W ×3 to an L-length sequence of patch features Vimg ∈ RL×h
134
      that are then projected into the embedding space of the LLM. Subsequently, the projected visual
135
      features Eimg ∈ RL×D are concatenated with the S-length textual embeddings Etxt ∈ RS×D
136   before being fed to the LLM for response generation. The generation process can be formulated
137   as o = LLMθ ([P rojω (fϕ (I)), Embed(t)]; where t, P roj, and Embed denotes the text prompt, the
138   vision-language projector, and the embedding module in the LLM, respectively. Given a training
139   sample (I, t, o), the VLM is optimized by minimizing the loss L(ϕ, ω, θ) = − log p(o|I, t) via
140   gradient descent.
141
      In this paper, we aim to explore the intricate design space of VLMs to develop a comprehensive
142   recipe for building VLMs capable of UI element grounding.

      2.3      D OWNSTREAM UI E LEMENT G ROUNDING T RAINING DATASETS
145
146   To fulfill our aim, two training datasets are utilized as the sources for downstream fine-tuning:
147
      SeeClick (Cheng et al., 2024) provides a dataset that integrates existing tasks in UI element grounding,
148
      captioning, and summarization. It comprises two parts: (a) a web portion containing element text
149
      grounding and OCR tasks derived from 300k web pages in the latest Common Crawl repository1 ;
150   and (b) a mobile device portion that includes element grounding and captioning tasks generated by
151   applying instruction-following templates to the Widget Captioning and RICO annotations. As shown
152   in Fig. 1, this dataset mainly comprises brief element annotations.
153
      AutoGUI (AutoGUITeam, 2024) is introduced to complement SeeClick. AutoGUI contains 625k UI
154
      functionality grounding and captioning tasks that require VLMs to grasp the functional semantics of
155
      various UI elements. This dataset is collected on multi-resolution and multi-device screenshots across
156   diverse data domains, providing detailed element functionality annotations related to UI contexts (see
157   Fig. 1). Given that the functionality annotations in this AutoGUI dataset are more detailed and longer
158   than those in SeeClick and that associating these annotations with unique elements among hundreds
159   of counterparts is challenging, this dataset is expected to enhance VLMs’ UI element grounding
160   capabilities, albeit with increased complexity.

191   3      E XPLORING THE D ESIGN S PACE OF VLM S FOR I MPROVED UI G ROUNDING
192
193   This section investigates design choices related to instruction-tuning data, training procedures, and
194   patch feature compression techniques for UI VLMs. A LLaVA model (Liu et al., 2023) with a
195   pre-trained vision-language projector is utilized as the base model. To process high-resolution UI
196   screenshots, a parameter-free image division strategy (Ye et al., 2023a; Zhang et al., 2024b) is
197   employed to crop the shape-variable screenshots into fixed-size image patches. Please see Fig. 2 for
198   the full pipeline. More implementation details are listed in the Appendix.

200   3.1     W HICH DATA T YPE C AN BE U SED TO WARM UP VLM S ?
201
      Our preliminary studies found that directly fine-tuning the base VLM using UI element grounding
202
      data resulted in poor grounding accuracy. We hypothesize that the base model needs warming up to
203   adapt to UI element grounding tasks, which require models to output precise numerical coordinates
204   based on cluttered UI screenshots.
205

      To explore suitable warming-up data sources, multiple instruction-tuning tasks shown in Fig. 3 are

211   2016) into visual grounding and referring tasks by applying instruction-tuning templates, resulting in
212
      5.7M samples in total.

214   lowing text instructions, without coordinate outputs. We utilize the VQA subset, including LLaVA-
215   Pretrain (Liu et al., 2023), COCO Lin et al. (2014), and SAM (Kirillov et al., 2023), of the
      ShareGPT4V-SFT dataset (Chen et al., 2023a), resulting in 530k samples in total.

406   3.4 W HICH PATCH F EATURE C OMPRESSION A PPROACH I S B ENEFICIAL ?
407
      UIs captured at super-high resolutions result in thousands of visual tokens when processed with the
408   division technique. For example, a 720p screenshot will be converted into 4608 tokens and a 4kHD
409   will exceed 48k. These extensive features may possess redundant details that interfere with VLM
410   inference and cause an unbearably high computational budget.
411
      To build a UI VLM capable of efficiently processing high-resolution screenshots, we explore various
412
      designs of the patch feature compressor, as shown in Fig. 5: (a) Merging compressor (Merger)(Ye
413
      et al., 2023a): This compressor concatenates N = 4 adjacent tokens in square regions along
414   the channel dimension andcompresses the concatenated features using a single-layer MLP. (b)
415   Resampler (Alayrac et al., 2022): This compressor reduces visual features to a fixed number of tokens
416   by utilizing a set of learnable queries to cross-attend to the visual features. (c) Convolution-based
417   compressor: This compressor reshapes the visual features to align them with image dimensionality,
418   processes them through a convolutional network, and flattens them back into visual tokens. Apart
419   from convolution with square-shaped kernels, i.e., C-Abstractor (Cha et al., 2024), we also test
420   the H-reducer (Dong et al., 2024), a type with stripe-shaped kernels (1 × 4), which is tailored for
421   horizontal text layouts in document understanding scenarios. (d) MLP: Directly using an MLP to
422
      process the patch features without compression.
423   This experiment uses 1M warming-up samples, 1.1M SeeClick data, and 125k AutoGUI data to
424   fine-tune the base model for one epoch. To ensure a fair comparison, the numbers of parameters of
425   these compressors are roughly equalized by adjusting their hyper-parameters.
426   The results in Tab. 3 show that the two convolution-based compressors achieve superior grounding
427   accuracy, with C-Abstractor leading on five benchmarks. Although H-Reducer is tailored for docu-
428   ment inputs, it is inferior to C-Abstractor which uses square-shaped kernels, probably because the UI
429   screenshots display a higher proportion of flexibly arranged visual contents (i.e., icons and images),
430   compared to text-rich documents. Resampler performs poorly as its cross-attention mechanism
431   possibly leads to a loss of spatial information, which is crucial for grounding tasks that require
      precise annotation-region associations (Cha et al., 2024). In contrast, Merger simply fuses nearby

471   4     S TATE OF THE A RT P ERFORMANCE OF UI-P RO
472
473
      Finally, we leverage the findings from the previous experiments to build UI-Pro. We train UI-Pro
474   using Gemma-1.1-2B (Team et al., 2024) and LLaMA-3.2-Instruct-3B AI@Meta (2024) as the base
475   LLM and OpenAI CLIP ViT-L/14@336 (Dosovitskiy et al., 2021) as the visual encoder. The training
476   process begins with 5M samples of visual grounding on natural images for warming up, followed
477   by fine-tuning with 5.3M SeeClick and 125k AutoGUI samples, adhering to a simple-to-complex
478   curriculum. We utilize C-Abstractor as the patch feature compressor.
479
      Tab. 4 show that compared with existing VLMs, UI-Pro exhibits impressively high accuracy on the
480   UI element grounding benchmarks. Notably, it achieves this with only one-fifth the model size of
481   CogAgent, a leading UI-oriented VLM, surpassing it across five benchmarks and establishing a new
482   state-of-the-art. In contrast, VLMs primarily designed for universal multimodal comprehension,
483   such as Qwen-VL, MiniCPM-V-2.6, and LLaVA-1.6, struggle with UI element grounding tasks,
484   highlighting a potential disconnect between their design strategies and the complexities of UI
485   grounding scenarios. Overall, with our four key findings, we can build VLMs that possess strong UI
      element grounding capabilities.

      5.1   R ECENT A DVANCEMENT OF VLM S
489
490
      There has been a significant rise in reasearch focused on improving LLMs by integrating both visual
491   and textual data (Alayrac et al., 2022; Chen et al., 2023b; Li et al., 2023; Lin et al., 2023a; Liu et al.,
492   2023; Lin et al., 2023b; Chen et al., 2023c; Lu et al., 2024; Bai et al., 2023; Wang et al., 2024b; Zhu
493   et al., 2024; Wang et al., 2024c; Li et al., 2024b; Zhang et al., 2024a; You et al., 2024a; Laurençon
494   et al., 2024; Peng et al., 2024; Driess et al., 2023), which has led to the development of VLMs.
495   Flamingo (Alayrac et al., 2022), utilizes combined visual and language prompts and has demonstrated
496   exceptional few-shot visual question-answering abilities. With the advancements brought by GPT-
497   4 (Team, 2024), both academic and industrial efforts have been made to make its multimodal
498   reasoning capabilities more accessible. LLaVA (Liu et al., 2023) and LLaMA-Adapter (Zhang et al.,
499
      2024a) have worked to align vision encoders (Dosovitskiy et al., 2021) with LLMs to support visual
      instruction following. Models like VisionLLM (Wang et al., 2024c), Ferret (You et al., 2024a), and
500
      Qwen-VL (Bai et al., 2023) have further developed strong visual grounding abilities. LLaVA-Next Liu
501
      et al. (2024b), Monkey Li et al. (2024b), LLaVA-UHD Guo et al. (2024), Qwen2-VL Wang et al.
502   (2024b) enhanced the perception resolution of VLMs. Moreover, research is expanding to VLM
503   applications in contexts with rich textual imagery (Tang et al., 2022; Ye et al., 2023b;a; Liu et al.,
504   2024d) and embodied interactions (Driess et al., 2023; Mu et al., 2023), unlocking new possibilities
505   in multimodal reasoning. Additionally, some works Laurençon et al. (2024); McKinzie et al. (2024)
506   give a comprehensive study on building VLMs, highlighting the impact of various design components
507   and data choices on model performance. Despite these advancements, there has been no systematic
508   approach proposed for data collection, model design, or training frameworks specifically targeting
509   VLMs in UI environments, highlighting a critical gap in the research.
510
511   5.2   E XISTING UI DATASETS AND B ENCHMARKS
512
513   In contrast to well-established natural image datasets (Russakovsky et al., 2014; Schuhmann et al.,
514   2022), datasets focused on UI understanding have received less attention in the field of computer
515   vision. Some efforts have been made to develop datasets for mobile UI modeling (Wang et al., 2021;
516
      Li et al., 2020a;b; Bai et al., 2021; Burns et al., 2022), with many of these efforts centered on the
      RICO dataset (Deka et al., 2017), which contains 72K Android app screenshots. Notable examples
517
      include Widget Captioning (Li et al., 2020a), which examines the captions and linguistic features of
518
      UI elements, and RICOSCA (Li et al., 2020b), which maps single-step instructions to corresponding
519   UI elements. More recently, MoTIF (Burns et al., 2022) has gained attention alongside the growing
520   interest in web-based scenarios. WebShop (Yao et al., 2022), for instance, was an early effort to
521   introduce a simplified simulator for web navigation tasks. Subsequent projects like Mind2Web (Deng
522   et al., 2024) and WebArena (Zhou et al., 2023) have focused on creating realistic and reproducible
523   web environments to enhance web agent performance. VisualWebBench (Liu et al., 2024c) has
524   also contributed by establishing a robust evaluation framework for VLMs, specifically targeting UI
525   grounding. To address the issue of limited data, recent studies like SeeClick (Cheng et al., 2024) and
526   CogAgent (Hong et al., 2023) have leveraged Common Crawl data to construct large-scale datasets,
527
      though these datasets often include noisy HTML code snippets. AITW (Rawles et al., 2023) has
      been introduced to focus on interpreting high-level instructions in Android environments. Existing
528
      UI-VLMs have primarily focused on fine-tuning open-source models using these datasets, but there
529
      is still a lack of detailed solutions for effectively enhancing their UI grounding capabilities.
530

532   6     C ONCLUSION
533
534   This paper introduces a practical framework for building VLMs with strong UI element grounding
535   capability. We identified key strategies, including warming up with visual grounding tasks, employing
536   a simple-to-complex fine-tuning curriculum, and scaling data sizes, all of which significantly optimize
537   grounding accuracy. Our findings on image feature compression further improve grounding accuracy
538   for high-resolution UI images. The integration of these findings resulted in UI-Pro, a state-of-the-
539   art VLM that achieves impressive grounding accuracy with fewer parameters. Hope this research
      provides a roadmap for future studies in intelligent UI agent development.

937   A.1    DATA C URATION D ETAILS
938
939   We found that certain datasets, such as RefCOCO and SeeClick include samples containing multi-turn
940   dialogues while others do not, leading to potential imbalance issues caused by this multi-turn trait and
941   resulting in unfair comparison. Additionally, excessively long dialogs exceed the context window of
942
      2048 of UI-pro, causing training bugs. To resolve these issues, we reorganize the multi-turn dialogs
      to ensure each dialog contains no more than 650 text tokens to balance all samples.

      A.2    F INE - TUNING D ETAILS

946   The hyper-parameters of training UI-Pro is shown in Tab. A. All experiments are conducted with
947   8 L20 GPUs each with 48GB memory. The three-stage fine-tuning (stage 1: warming-up; stage
948   2: simple UI grounding task fine-tuning; stage 3: complex UI grounding task fine-tuning) spend

955   B     L IMITATIONS
956
957   Despite the impressive UI element grounding capability, UI-Pro still encounters several limitations:
958   1. Model Diversity. This paper is targeted at LLaVA-like architectures that typically comprise a
959   vision encoder, a vision-language connector, and an LLM. In practice, there exist various VLM
960   architectures, such as Flamigo (Alayrac et al., 2022) and Emu (Sun et al., 2024) with multi-modal
961   inputs and outputs. Future work can extend our findings to these architectures to generalize the
962   insights.
963
      2. UI Data Diversity. As UI-related datasets are much more scarce than natural image datasets,
964   this work mainly conducts experiments with SeeClick and AutoGUi training datasets, which are the
965   largest ones to date. This data insufficiency issue may be the cause of slight over-fitting observed in
966   the data size scaling experiments, as shown in Fig 4. We hope future work can collect more diverse
967   data to consolidate our findings.
968
      3. Resource Intensiveness. Fine-tuning VLMs like UI-Pro can be extremely resource-intensive, re-
969
      quiring substantial computational power and time, which may limit accessibility for some researchers
970   or developers. Due to such a resource restriction, this work uses small LLMs, i.e. Gemma-2B. We
971   will extend our work to a larger scale by integrating larger LLMs like Llama-3-8B.
