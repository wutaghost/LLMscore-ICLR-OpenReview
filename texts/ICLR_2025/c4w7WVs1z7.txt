                                                  A BSTRACT
008
009
               Recently, the use of transformers in offline reinforcement learning has become a
010
               rapidly developing area. This is due to their ability to treat the agent’s trajectory
011
               in the environment as a sequence, thereby reducing the policy learning problem
012            to sequence modeling. In environments where the agent’s decisions depend on
013            past events (POMDPs), it is essential to capture both the event itself and the
014            decision point in the context of the model. However, the quadratic complexity of
015            the attention mechanism limits the potential for context expansion. One solution
016            to this problem is to extend transformers with memory mechanisms. This paper
017            proposes a Recurrent Action Transformer with Memory (RATE), a novel model
018            architecture that incorporates a recurrent memory mechanism designed to regulate
019            information retention. To evaluate our model, we conducted extensive experiments
020
               on memory-intensive environments (ViZDoom-Two-Colors, T-Maze, Memory
               Maze, Minigrid-Memory), classic Atari games, and MuJoCo control environments.
021
               The results show that using memory can significantly improve performance in
022
               memory-intensive environments, while maintaining or improving results in classic
023            environments. We believe that our results will stimulate research on memory
024            mechanisms for transformers applicable to offline reinforcement learning. The code
025            is available at https://anonymous.4open.science/r/RATE-B01F/.

057   In RL memory has two senses. One is using past information within an episode to make de-
058
      cisions (Lampinen et al., 2021; Ni et al., 2023). The other is transferring experience from one
      environment to another (Melo, 2022; Kang et al., 2023; Team et al., 2023), improving generalizability,

060

061   episode.
062
063   In this paper, we propose the Recurrent Action Transformer with Memory (RATE, Figure 1),
064
      a model that uses several memory mechanisms: memory embeddings, caching of previous hidden
      states of previous tokens, and Memory Retention Valve (MRV). We empirically show that memory
065
      mechanisms effectively preserve information from previous steps, allowing the model to use past
066
      information when making decisions in the present. MRV is designed to control the process of updating
067   memory embeddings and prevent the loss of important information when processing long sequences,
068   thus enabling the processing of highly sparse tasks. To evaluate the memory mechanisms, we perform
069   extensive experiments in memory-intensive environments: ViZDoom-Two-Colors (Sorokin et al.,
070   2022), Memory Maze (Pasukonis et al., 2022), Minigrid-Memory (Chevalier-Boisvert et al., 2023),
071   and Passive T-Maze (Ni et al., 2023), as well as on standard RL benchmarks: Atari (Bellemare et al.,
072   2013) and MuJoCo (Fu et al., 2021). We also study the impact of memory on the performance of the
073   proposed model.
074   The proposed model interpolates and extrapolates well outside the transformer context, is able to
075   retain important information for a long time when operating in highly sparse environments, and
076   allows to compensate the effect of bias in the training data.
077
      Our contribution can be summarized as follows:
078
079         1. We propose the Recurrent Action Transformer with Memory (RATE), a transformer model
080            for offline RL that makes use of memory mechanisms: memory embeddings, caching of
081            hidden states of previous tokens, and the Memory Retention Valve (MRV). The proposed
082            MRV is based on the cross-attention architecture and is designed to prevent information
083            loss from memory embeddings and significantly improve the performance of RATE in
084            memory-intensive environments with sparse structure (see section 3).
085
            2. We show that RATE significantly outperforms strong baselines with and without memory
086            mechanisms in memory-intensive environments, including ViZDoom Two-Colors, Memory
087            Maze, Minigrid-Memory, and T-Maze (see subsection 4.2).
088
089
            3. We demonstrate that RATE achieves better or comparable results in classic Atari games and
               MuJoCo control tasks, demonstrating that the proposed model is suitable for different types
090
               of tasks and emphasizing its universality (see subsection 4.2).

095   2.1    O FFLINE R EINFORCEMENT L EARNING
096
097
      In RL (Sutton & Barto, 2018), we assume that the task can be described by a Markov Decision
      Process (MDP) as a tuple ⟨S, A, P, R⟩. The process consists of states s ∈ S, actions a ∈ A, a
098
      state transition function P(s′ |s, a), and an immediate reward function r = R(s, a). The states are
099
      assumed to have the Markov Property, that is P(st+1 |st ) = P(st+1 |s1 , . . . , st ). Given a timestep t,
100   we use rt = R(st , at ) to denote the immediate reward that the agent receives at state st performing
101   action at at that timestep. We describe trajectory τ of length T as a sequence of states si , actions
102   ai , and immediate rewards ri : τ = (s0 , a0 , r0 , s1 , a1 , r1 , . . . , sT −1 , aT −1 , rT −1 ). We denote return-
103   to-go (Chen et al., 2021) Rt of trajectory τ at the timestep t as a sum of future rewards from the
                                                  PT −1
104   timestep t to the end of trajectory: Rt = t′ =t rt′ . The goal of a RL agent is to learn policy π that
105   maximizes the expected return. In online RL, the trajectories used to train an agent are obtained
106   iteratively as the agent interacts with the environment. In offline RL, the agent does not interact with
107   the environment during training. A fixed set of trajectories collected by an arbitrary policy is used for
      training. Although such a setting is more difficult because it does not allow additional exploration of

      the environment or generation of new trajectories, it is preferable for tasks where interaction with the
109   environment is costly or risky, such as in robotics.

      2.2    PARTIALLY O BSERVABLE M ARKOV D ECISION P ROCESS
112
113   In the real world, there are frequent situations where the full state of the environment is not available
114   to the agent, so the Markov Property is violated, and the agent is said to receive observations instead
115   of state as input. In this case, observations are no longer sufficient statistics of the past to make
116   a decision in the current step. An example would be a robot navigating an environment based
117   on a camera image or a situation in which a decision must be made based on information from
118
      the past that is not available in the current observation. A Partially Observable Markov Decision
      Process (POMDP) is used in such cases. POMDP is a generalization of the MDP and is written as
119
      ⟨S, A, O, P, R, Z⟩, where o ∈ O – observations, s ∈ S – states, a ∈ A – actions, r = R(s, a) –
120
      immediate reward function, P(s′ |s, a) – state transition function and Z is an observation function
121   Zsa′ o = P (O = ot+1 |St+1 = s′ , At = a). To work successfully in such environments, mechanisms
122   such as memory are needed to allow the use of information from the past (Parisotto et al., 2020;
123   Lampinen et al., 2021; Esslinger et al., 2022). In our work, we propose an approach of adding memory
124   to the agent in the form of memory embeddings, caching of hidden states of previous tokens, and
125   MRV. In this paper we consider the setting of an offline model-free RL, where learning is formulated
126   as a sequence modeling problem (Chen et al., 2021).

176   4     E XPERIMENTAL E VALUATION
177
178   We designed experiments to accomplish two primary objectives: (a) to demonstrate the advantage
179   of our RATE model in memory-intensive environments (T-Maze (Ni et al., 2023), ViZDoom-Two-
180
      Colors (Sorokin et al., 2022), Memory Maze (Pasukonis et al., 2022), Minigrid-Memory (Chevalier-
      Boisvert et al., 2023)), and (b) to investigate the effectiveness of the proposed model in classical
181
      MDPs to demonstrate its versatility (Atari (Bellemare et al., 2013) and MuJoCo (Fu et al., 2021)).
182
183   For comparison with RATE, we chose DT (Chen et al., 2021) as the main baseline, and adapted the
184   memory-augmented architectures Recurrent Memory Transformer (RMT) (Bulatov et al., 2022) and
185   Transformer-XL (TrXL) (Dai et al., 2019) developed for NLP tasks to the RL domain. Information
186
      about the environments used can be found in Table 7.
187
188   4.1   M EMORY- INTENSIVE ENVIRONMENTS
189
190
      To test RATE memory mechanisms, we use memory-intensive environments Figure 2, i.e., envi-
      ronments where the agent requires memory to operate successfully. A brief description of these
191
      environments is presented below, and a full description and data collection methodology can be found
192
      in the Appendix B.
193
194   1. ViZDoom-Two-Colors (Sorokin et al., 2022) – an agent in an acid-filled room observes a quickly
195      disappearing green or red pillar. To stay alive, the agent must recall the pillar’s color and gather
196      items of the same color.
197
      2. T-Maze (Ni et al., 2023) – an agent navigates a T-shaped corridor, receiving a clue at the start
198
         about which direction to turn at the end of the corridor. The task tests memory in a prolonged
199      sparse reward environment (the agent receives a reward only at the end).
200
201
      3. Memory Maze (Pasukonis et al., 2022) – an agent navigates a maze, seeking objects matching
         the color of its view frame. The frame color changes after each successful find. The goal is to
202
         collect the most matching objects within a time limit.
203
204   4. Minigrid-Memory (Chevalier-Boisvert et al., 2023) – a similar task to T-Maze, but with different
205      observation spaces and reward functions (see Appendix B, Table 7). Another important difference
206      is that the agent appears at a random point at the beginning of the episode, not at the beginning
207
         of the corridor. Thus, in the case of Minigrid-Memory, it is necessary to reach that clue first
         (in T-Maze it is a memory problem, in Minigrid-Memory it is a memory and credit assignment
208
         problem (Ni et al., 2023)).
209
210   In the experiments, the same hyperparameters presented in Table 8 were used for RATE, DT, RMT, and
211   TrXL to simplify the comparison. The context length K and the number of segments N were chosen
212   so that the effective context Kef f = K × N for RATE, RMT, and TrXL covers important events in
213   memory-intensive environments during training. In turn, since DT has no memory mechanisms, for it
214   K = Kef f . Thus, the context length K for RATE, RMT and TrXL is less than the context length for
215   DT by a factor of N , but all models process the same parts of trajectories. More information about
      the training procedure for each environment can be found in Appendix D.

      In this section, the main experimental results for each of the environments are presented in the corre-
237   sponding paragraphs. Additional results can be found in the Appendix F. For each of the experiments,
238   the same techniques were used for all models to obtain the results presented in the Appendix E.
239   Unless otherwise indicated, all baselines were trained from scratch.
240
241   ViZDoom-Two-Colors. The dataset for this environment was collected using a pre-trained Advan-
242   tage Actor Critic (A2C) (Beeching et al., 2019), which has a slight bias in favor of selecting green
243   items even if red items are required. However, the dataset is balanced by the pillars colors. Figure 3
244   (a) shows the inference results on all pillars, separately only on red pillars and separately only on
245   green pillars. As illustrated in the Figure 3 (a), for inference with the presence of a disappearing pillar
246   at the beginning of the episode, all baselines have an average total reward for inference on green
247   pillars greater than for inference on red pillars.
248   To prove that this is not due to the peculiarities of the algorithms, but to the presence of bias in
249   the data, we performed an additional inference without a pillar at the beginning of the episode,
250   demonstrating the ability of all baselines to collect exclusively green items. As can be seen from
251   Figure 3 (b), DT learns the distribution of training data and is unable to remember the pillar color.
252   In turn, baselines with memory mechanisms such as RATE, RMT and TrXL are successful in this
253   task. The poor performance of baselines with memory on red pillars without the pillar at the episode
254
      beginning proves that it is the color of the pillar that they remember.
255   This conclusion of DT’s inability to use information out of context as opposed to RATE is supported
256   by the experimental results presented in Figure 3 (c, d), which illustrates the inference results for
257   the first 90 steps, where the pillar are entirely captured in the effective context, and the subsequent
258   90 steps, where the pillar color information begins to disappear from the effective context (the
259   context window moves as a sliding window). As a result, there is a drop in total reward for red-pillar
260
      environments by almost a factor of two, indicating DT’s inability to memorize information to use it
      out of context. In turn, for RATE, RMT, and TrXL, the values of total reward in the first and second
261
      cases are almost unchanged, indicating their ability to utilize information outside of the current
262
      context window.
263
264   Figure 3 (e, f, g) demonstrates the dependence of model performance on the return-to-go. In this paper,
265   we used an empirical estimate of the target reward as the average of the top-10 total rewards in the
266
      training dataset. As can be seen, RATE not only significantly outperforms DT, but also outperforms
      the memory-augmented models RMT and TrXL.

411   RQ 3. Memory Retention Valve architecture ablation. Without MRV in the T-Maze environment
412   at corridor lengths of L ≫ K, the performance of the RATE model decreased with each segment
413   processed at inference, resulting in almost SR = 50% on long trajectories (see Table 6). For example,
414   in the T-Maze task, the important information to be remembered goes into memory embeddings when
415
      processing the first segment of the sequence, and then it must be retrieved when making decisions
      on the last segment. At the same time, due to the recurrent structure of the architecture, memory
416
      embeddings continue to be updated during the processing of intermediate segments when no new
417
      information needs to be memorized, causing important information from memory embeddings to
418   leak out. To retain important information in memory embeddings, MRV mechanism was added to the
419   architecture. We considered the five different schemes detailed in subsection F.3 to implement MRV:
420
421         1. MRV-CA-1: cross-attention-based MRV which uses an attention mechanism to control
422            the updating of memory embeddings. The updated memory embeddings Mn+1 are fed to
423            Query, and the incoming Mn are fed to Key and Value.
424         2. MRV-CA-2: uses the same mechanism as MRV-CA-1 but the incoming memory embeddings
425            Mn are fed to Query, and the updated Mn+1 are fed to Key and Value.
426
            3. MRV-G: gated MRV which uses a gating mechanism similar to the one used in
427            GTrXL (Parisotto et al., 2020).
428
429
            4. MRV-GRU: uses a GRU (Chung et al., 2014) block to process updated Mn with hidden
               states.
430
431         5. MRV-LSTM: uses a LSTM (Hochreiter & Schmidhuber, 1997) block to process updated
               Mn with cached states.

449   6     R ELATED W ORK
450
451   6.1    T RANSFORMERS FOR R EINFORCEMENT L EARNING
452   Transformers have found application in various areas of RL (Agarwal et al., 2023; Li et al., 2023):
453   online RL (Parisotto et al., 2020; Lampinen et al., 2021; Esslinger et al., 2022; Melo, 2022; Zheng
454   et al., 2022; Pramanik et al., 2023; Team et al., 2023), offline RL (Chen et al., 2021; Janner et al.,
455   2021; Lee et al., 2022; Reed et al., 2022; Jiang et al., 2023), and model-based RL (Chen et al., 2022;
456   Micheli et al., 2023; Robine et al., 2023). The use of transformers as a general policy for many
457   environments is also being explored (Lee et al., 2022; Melo, 2022; Reed et al., 2022). In our work,
458
      we consider the formulation of an offline model-free RL, where learning is formulated as a sequence
      modeling problem (Chen et al., 2021). Prominent representatives of such models are (Chen et al.,
459
      2021; Janner et al., 2021; Lee et al., 2022; Jiang et al., 2023), although planning in latent space (Jiang
460
      et al., 2023) is considered, which can be seen as modeling of the environment. Moreover, (Janner
461   et al., 2021; Jiang et al., 2023) are specialized for control tasks with vector observations and do not
462   generalize to environments with observations in the form of images. Therefore, we consider the
463   Decision Transformer (Chen et al., 2021), which has no memory mechanism, as the main baseline for
464   comparison.
465
466   6.2    R ECURRENT N EURAL N ETWORKS FOR R EINFORCEMENT L EARNING
467   For long input sequences, recurrent networks may have computational advantages over transformers.
468   The RNNs recurrent unit maintains a hidden state, which is essentially a form of memory that is
469   important for solving POMDPs. In Decision LSTM (DLSTM) (Siebenborn et al., 2022) in DT the
470   transformer is replaced by an LSTM unit (Hochreiter & Schmidhuber, 1997).
471
      6.3    S TATE S PACE M ODELS FOR R EINFORCEMENT L EARNING
472
473   Recently, State Space Models (SSMs) (Gu et al., 2021) have shown significant success in sequence
474
      modeling, particularly in offline RL (Bar-David et al., 2023; Cao et al., 2024; Gu & Dao, 2023; Ota,
      2024). In Decision S4 (DS4) (Bar-David et al., 2023), sequence modeling is executed using S4 (Gu
475
      et al., 2021) layers within the framework of offline RL, whereas Decision Mamba (DMamba) (Ota,
476
      2024) utilizes the most recent Mamba (Gu & Dao, 2023) sequence model instead of causal self-
477   attention. Mamba Decision Maker (MambaDM) (Cao et al., 2024) integrates the unique features of
478   SSMs to effectively combine local and global features with Global-local fusion Mamba (GLoMa)
479   module.
480
481   6.4    M EMORY IN T RANSFORMERS
482   There are many ways to implement the memory mechanism for transformers (Bulatov et al., 2022;
483   Dai et al., 2019; Ding et al., 2020; Lei et al., 2020; Rae et al., 2019; Wu et al., 2020; 2022). In
484   Transformer-XL (TrXL) (Dai et al., 2019), it is proposed to split a long data sequence into segments
485   and to access past segments at the expense of memory, but to ignore very distant segments. This
      increases the effective length of the context. The Compressive Transformer (Rae et al., 2019) uses


                                                           9
      Under review as a conference paper at ICLR 2025


486
      compressed memory, allowing previous versions of memory to be compressed rather than discarded
487   as in TrXL. ERNIE-Doc (Ding et al., 2020) suggests using the retrospective feed mechanism and the
488   enhanced recurrence mechanism. Memformer (Wu et al., 2020) uses external dynamic memory to
489   encode and retrieve past information. MART (Lei et al., 2020) extends this idea by adding a memory
490   update mechanism similar to a recurrent neural network (Cho et al., 2014; Hochreiter & Schmidhuber,
491   1997). The Memorizing Transformer (Wu et al., 2022) proposes to store the internal representations
492   of past inputs. The Recurrent Memory Transformer (RMT) (Bulatov et al., 2022) includes additional
493   read and write memory tokens at each segment’s beginning and end. This method allows the effective
494   context to be expanded to over 1 million tokens (Zhu et al., 2020).
495   An Adaptive Agent (AdA) (Bauer et al., 2023) uses memory architectures to store and employ infor-
496   mation previously acquired by the agent. The default memory architecture is TrXL with normalization
497   before each layer (Parisotto et al., 2020), and the use of gating on the feedforward layers (Shazeer,
498   2020) to stabilize training. We also use TrXL in our work but refrain from using additional modifica-
499   tions to stabilize training. Another distinctive feature of using a transformer in AdA, as opposed to
500   DT, is that pixel observations, past actions, past rewards, and additional information are not tokenized
501
      separately but are combined into a single vector that feeds the transformer. The transformer itself
      predicts not only actions but also value function values.

      7    C ONCLUSION
505
506
507   In this paper, we propose Recurrent Action Transformer with Memory (RATE), a transformer
508
      model for offline RL that exploits memory mechanisms in the form of memory embeddings and
      caching of hidden states of previous tokens, and the Memory Retention Valve (MRV), which
509
      controls memory updating and prevents the loss of important information in sparse tasks. In extensive
510
      experiments in memory-intensive environments such as ViZDoom-Two-Colors, Memory Maze,
511   Minigrid-Memory, and T-Maze, we have shown that RATE outperforms recurrent and transformer
512   baselines. The proposed model interpolates and extrapolates well outside the transformer context, is
513   able to retain important information for a long time when operating in highly sparse environments,
514   and allows to compensate for the effect of bias in the training data.
515
      We also show that the proposed model achieves better or comparable results to state-of-the-art
516
      Mamba-based models in Atari and MuJoCo environments, indicating that RATE is suitable for all
517   tasks: both memory-intensive and not. We have thoroughly investigated the influence of memory
518   mechanisms on the performance of the model and have clearly shown that the model uses them in
519   decision making. This method shows great potential for tackling complex tasks with long sequences,
520   especially in robotics, where training agents on pre-collected data sets is highly advantageous.
521
522
      Limitations. Limitations of the proposed model include its inability to design K and N in memory-
523
      intensive environments so that all important events fall into the efficient context of Kef f = K × N .
524
      In addition, the approach based on dividing trajectories into segments during inference does not allow
525   for memory updates effectively at each step using a sliding window. Also, there are currently no
526   studies of the memory capacity of the proposed model, so the practical amount of information that
527   can be stored remains unknown.

773   In this section, we provide an extended description of the environments used in this paper, as well as
774   the methodology used to collect the trajectories. Table 7 summarizes the observations type, rewards
775   type, and actions type for each of the environments considered in this paper.
776
777   ViZDoom-Two-Colors. We used a modified ViZDoom-Two-Colors environment from (Sorokin
778   et al., 2022) to assess the model’s memory abilities. The agent initially having 100 hit points (HP) is
779   placed in a room without inner walls filled with acid. At each step in the environment, the agent loses
780   a fixed amount of health (10/32 HP per step). In the center of the environment, there is a pillar of
781
      either green or red color, which disappears after 45 environment steps. Throughout the environment,
      objects of two colors (green and red) are generated. When the agent interacts with an object of the
782
      same color as the pillar, it gains an increase in health of +25 and a reward of +1. When the agent
783
      interacts with an object of the opposite color, it loses a similar amount of health. The agent receives
784   an additional reward of +0.02 for each step it survives. The episode ends when the agent has zero
785   health. Thus, the agent needs to remember the color of the pillar to select items of the correct color,
786   even if the pillar is out of sight or has disappeared. The agent does not receive information about its
787   current health or rewards, as these observations essentially convey the same information as the color
788   of the pillar but persist beyond step 45.
789
      We collected a dataset of 5000 trajectories of 90 steps in length using a trained A2C (Beeching et al.,
790   2019) agent (an agent trained with a non-disappearing pillar). The average reward for these 90 steps
791   is 4.46. When collecting trajectories, to ensure that the agent saw the pillar before it disappeared,
792   the agent always appeared facing the pillar in the same place – midway between the pillar and the
793   nearest wall. In order to successfully complete this task, the agent needs to remember the color
794   of the pillar. This environment tests the long-term memory mechanism, since the agent needs to
795   retain information about the pillar for a time much longer than the pillar has been in the environment.
796   Using only short-term memory and, for example, collecting the next item of the same color as the
797   previous collected item, it will not be possible for the agent to survive for a long time, as this policy
798
      is extremely unstable. This is due to the fact that in the training dataset the agent occasionally makes
      a mistake and picks up an object of the opposite color. Thus, irrelevant information about the desired
799
      color may enter the transformer context and the agent will start collecting items of an opposite color,
800
      which will quickly lead to a failure.
801
802
      T-Maze. To investigate agent’s long-term memory on very long environments (the inference
803
      trajectory length is much longer than the effective context length Kef f ) we used a modified version of
804   the T-Maze environment (Ni et al., 2023). The agent’s objective in this environment is to navigate from
805   the beginning of the T-shaped maze to the junction and choose the correct direction, based on a signal
806   given at the beginning of the trajectory using four possible actions a ∈ {lef t, up, right, down}. This
807   signal, represented as the clue variable and equals to zero everywhere except the first observation,
808   dictates whether the agent should turn up (clue = 1) or down (clue = −1). Additionally, a constraint
809   on the episode duration T = L + 2, where the maximum duration is determined by the length of the
      corridor L to the junction, adds complexity to the problem. To address this, a binary flag, represented


                                                           15
      Under review as a conference paper at ICLR 2025


810
      as the f lag variable, which is equal to 1 one step before the junction and 0 otherwise, indicating
811   the arrival of the agent at the junction, is included in the observation vector. Additionally, a noise
812   channel is added to the observation vector, with random integer values from the set {−1, 0, +1}.
813   The observation vector is thus defined as o = [y, clue, f lag, noise], where y represents the vertical
814   coordinate. The reward r is given only at the end of the episode and depends on the correctness of
815   the agent’s turn at the junction, being 1 for a correct turn and 0 otherwise. This formulation deviates
816   from the traditional Passive T-Maze environment (Ni et al., 2023) (different observations and reward
817   functions) and presents a more intricate set of conditions for the agent to navigate and learn within
818   the given time constraint.
819   The dataset consists of 2000 of trajectories for each segment of length 30 (i.e. 6000 trajectories for
820   the Kef f = 3 × 30 = 90) and consists only of successful episodes. An artificial oracle with a priori
821   information about the environment was used to generate the dataset.

1003   It is worth noting that we presented raw scores for Atari, rather than gamer-normalized scores as

1009   such actions hold little significance. For instance, the emergence of enemies and divers in SeaQuest

1014   episode could remain unchanged, provided the agent’s policy performs consistently well. To rectify
1015   this discrepancy, we propose a novel evaluation strategy for MuJoCo tasks. In this approach, each
1016
       segment commences with the maximum return, simulating the scenario where the agent initiates the
       trajectory anew. This method effectively mitigates the aforementioned issue, enhancing the accuracy
1017
       of our evaluation process. Our MuJoCo experiments in Table 4 show that this benefits performance
1018
       significantly for some environments. Thus, using RATE allowed us to obtain the best metrics for
1019   MuJoCo in 4/9 cases compared to the other baselines. RATE also outperforms DT in 9/9 tasks.

1060
1061   To determine the optimal hyperparameters associated with memory mechanisms, additional ablation
1062   studies were performed in ViZDoom-Two-Colors and T-Maze environments, and the results are
1063   presented in Figure 11 and Figure 10 (right). From the ablation studies results, it was found that for
1064   environments like ViZDoom-Two-Colors with continuous reward signal and image observations, the
1065   best results can be obtained using number of cached memory tokens mem_len = (K × 3 + 2 ×
1066   num_mem_tokens) × N , where K – context length and N – number of segments.
1067   On the other hand, for environments with sparse events like T-Maze, it has been found that using
1068   caching of hidden states of previous tokens (mem_len > 0) prevents remembering important
1069   information. In this case, gating with n_head_ca = 4 and moderate number of memory tokens
1070   num_mem_tokens = 5 gives the best results (see Figure 10 (right)).

1118   F.2   C URRICULUM L EARNING
1119
1120   Since in the T-Maze environment, the number of actions at the junction relates to the number of actions
1121   when moving straight along the corridor as L1 and tends to 0 as L increases, there is a significant
1122
       imbalance in the agent’s action distribution, which can cause problems when performing rare class
       (turning actions) prediction. Theoretically, this situation can be remedied through curriculum learning.
1123
1124   Curriculum learning (CL) is a technique in which a model is trained on examples of increasing
1125   difficulty. In this approach, the model is first trained on the set of trajectories Q1 = q1 of length
1126   K × 1, then the trained model is re-trained on the set of trajectories Q2 = q1 ∪ q2 , where the set
1127   q2 is formed by trajectories of length K × 2, and so on (in order of increasing complexity of the
                                                                                            SN
1128   trajectories). Thus, for the N segments considered during training, the set QN = i=1 qi is used.
1129   In the T-Maze environment, DT, RATE, RMT, and TrXL were trained with and without curriculum
1130   learning because this approach theoretically produces better results. However, it is important to note
1131   that the T-Maze task is successfully solved by the RATE model without using curriculum learning,
1132   and even vice versa – its use slightly degraded performance on long corridors. However, with respect
1133   to TrXL, the use of CL yielded slightly better results. The work showed that using CL does not
       achieve significantly better performance on the T-Maze task. The results of using the CL on the
