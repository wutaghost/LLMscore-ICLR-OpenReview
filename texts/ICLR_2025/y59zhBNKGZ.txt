008                                               A BSTRACT
009
010
               The original softmax based “all-to-all” attention mechanism in the extremely
011            successful transformer architecture computes the attention between N tokens
012            embedded in a D-dimensional head, in O(N 2 D) time and O(N 2 ) memory. Several
013            linearly scaling attention mechanisms have been proposed over the seven years
014            since the transformer algorithm was proposed, to address the quadratic complexity
015            in N , and these “Linear” algorithms have been shown to achieve reasonable
016            training performance in the papers they were introduced in. Despite the fact
017            that most current LLM applications are trending towards processing larger token
018            sequences in context (increasing N into the millions and beyond), and transformers
019
               have helped create foundational models in diverse domains beyond language (e.g.,
               image, video, and audio processing), the proposed linear mechanisms surprisingly
020
               do not find wide usage, and moreover remain restricted to relatively small sized
021
               problems. We address two possible reasons for this. The first is that many of
022            the proposed mechanisms, despite their linear complexity in N , have practically
023            a relatively large memory footprint which is quadratic in D, which quickly fills
024            memory of available GPUs. We show the memory complexity can be reduced
025            by approaching calculations from a novel perspective in both the forward and
026            backward passes in training. A second reason is that techniques that accelerate
027            convergence, such as dropout, cannot be used during training in the current linear
028            algorithms. This is mitigated via an alternative dropout mechanism. Results and
029            comparison studies demonstrate the usefulness of the proposed approaches, while
030
               maintaining the linear scaling in N in both wall-clock time and memory usage.

033   1   I NTRODUCTION
034
035   Transformers are the single deep learning architecture that underpin many recent successful applica-
036   tions in diverse fields such as natural language processing, speech, computer vision, biology, and
037   so on. Transformers incorporate a Softmax-based all-to-all “Attention" mechanism between the
038   input tokens Vaswani et al. (2017). While this has proven to be extraordinarly effective in learning
039
      tasks, it has a time and memory complexity of O(N 2 D) and O(N 2 ) respectively, where N is the
      number of tokens and D the dimension per attention head. Efficient hardware implementation can
040
      offer constant speedup, and significant memory reduction. A notable example is the widely used
041
      FlashAttention-2 Dao (2023), where the memory complexity is reduced to O(N D), but the time
042   complexity remains O(N 2 D). Given how the industrial models are shifting towards ever larger N
043   (∼ 107 in current implementations), searching for a linearly scaling attention mechanism started
044   shortly after the introduction of the original paper.
045
      Previous proposals for a linearly scaling attention mechanism used either Sparsified/localized Atten-
046
      tion, or Kernel Separation. In the former, the all-to-all attention, and as a result long-range context
047
      is not captured. In the former approach, all-to-all attention is not captured, resulting in a loss of
048   long-range context. While this may be effective in certain situations (particularly for approximations
049   during inference), it compromises the model’s ability to learn from larger sequence, which was the
050   goal in the first place. Kernel Separation, in contrast, successfully captures all-to-all attention and
051   has demonstrated promising results on small benchmarks. However, its major drawback is the high
052   memory consumption of O(N D2 ). This drawback has hindered its application in large language
053   models (LLMs), limiting researchers’ ability to study its effectiveness at scale. To put things in
      perspective, with a memory consumption of O(N D2 ), TinyLlama-1.1B Zhang et al. (2024b) would


                                                        1
      Under review as a conference paper at ICLR 2025


054
      require 280GB1 of memory per batch. This makes the Kernel Separation approach impractical on
055   any production GPU unless the memory consumption is addressed.
056
057   We propose a new viewpoint for calculating Linear Attention, and show how the memory consumption
058
      can be reduced to O(N D). Using our method, the memory consumption of TinyLlama-1.1B with
      Linear Attention is reduced to 2.2 GB memory. Our work does not introduce a new Attention
059
      mechanism, and our focus is addressing the high memory consumption issue in the many linear
060
      attention mechanisms proposed. Furthermore, since these methods do not explicitly calculate the
061   attention matrix, dropout cannot be applied. This is problematic given the robustness and accelerated
062   convergence that dropout offers. To address this, we introduce an alternative mechanism. In summary,
063   we make the following contributions:
064
065          • Reduce the memory consumption of Linear attention from O(N D2 ) to O(N D) (Section 3).
066          • Introduce an alternative mechanism to dropout (Section 4).

      2.2   R ELATED W ORK
095
      The most successful approach to speedup the vanilla attention mechanism that has seen widespread
096   use is efficient hardware implementation and the use of analytical gradients. The most notable
097   such mechanism, which offers a constant speedup, and reduces memory consumption to O(N D) is
098   FlashAttention-2 Dao (2023), Dao (2024).
099
      However, this method does not address the issue of quadratic cost of large context attention. Because
100
      of this there is a lot of research on systems approaches to use multi-GPU systems, e.g., Liu et al.
101
      (2023).
102
103   The goal of this paper is to make sub-quadratic attention usable. Two common algorithm-based
104   approaches proposed in the literature for this are Kernel Separation, and Sparsification.

115   Katharopoulos et al. (2020); Kasai et al. (2021); Zhang et al. (2024a); Choromanski et al. (2020); Qin

121   Sparsification In this approach, we only calculate the Attention for a fixed number of keys for
122   each query. Common approaches consider keys with tokens that are spatially close-by to the query’s
123
      token Zaheer et al.; Zhou et al.; Yu et al. (2023a); Beltagy et al. (2020); Han et al. (2023), randomized
      sampling Wiegreffe & Pinter (2019), or taking advantage of locality aware hashing Kitaev et al.,
124
      where the query and key vectors are hashed based based on their position in the D−dimensional
125
      space, and attention is only calculated for the keys close to the query with this metric. However,
126   these approaches contradict the goal of capturing attention for a long sequence, as it only considers
127   limited token pairs. This is especially problematic for training.
128
129
      There are works which that combine both of these approaches, where they calculate the exponential-
      based kernel for a sparse number of query-key pairs, and separable-based kernel for the rest of the
130
      tokens Qin et al. (2022a).
131
132
      Other Algorithmic Approaches Other approaches include assuming that the Attention matrix is
133   numerically low rank and derive a smaller matrix Wang et al., hierarchically packing the tokens Ma
134   et al. (2021), using the Gaussian function as the Attention kernel and approximating it in linear
135   time Chen et al. (2021) or calculating the result of the Attention and the value matrix multiplication
136   using Fast Multipole Methods Nguyen et al. (2021). Despite their novel approaches, these methods
137   have yet to be used in large context LLMs.
138
139   Non-transformer Models New learning architectures are also being proposed as an alternative to
140   Transformers. Recent ones which have generated considerable interest include Mamba Gu & Dao,
141   Retentive networks Sun et al., Hyena Poli et al. and CRATE Yu et al. (2023b). These approaches are
142
      complementary to the factored attention kernels which are the focus of this paper.

144   3     M EMORY R EDUCTION
145
146   We first discuss factorization of the attention kernel, which is the computational trick enabling
147   speedup. This enables the time and memory complexity of factorization based kernels can be reduced
148   both in the forward and backward passes. The derivation of the analytical gradient of the attention
149
      head in this paper is a novel contribution, which allows us to reduce memory consumption. We show
      these results for the linear attention kernel f (x) = a + bx, and demonstrate how to achieve time and
150
      memory complexities of O(N D2 ) and O(N D). Our method can also be applied to the attention
151
      kernel of f (x) = a + bx + cx2 , which we show in Appendix A, and results in O(N D3 ) time and
152   O(N D) memory complexity.

464            6                 C ONCLUSION
465
466           Transformers use a Softmax-based mechanism known as Attention, which has a quadratic scaling
467           with number of tokens taken in context. Many approaches have proposed subquadratic mechanisms
468           for both training and inference. This problem is particularly important since the current applications
469           are moving towards higher number of tokens, and training can only be done in large datacenter
470           scale systems. The search for a linearly scaling attention mechanism has become an important and
471           active area of research. One of the promising approaches is Linear Attention (Kernel Separation).
472
              However, current implementations had very high memory consumption of O(N D2 ), where N is
              the number of tokens and D the model dimension per head. This memory consumption makes
473
              kernel separation impractical since even small transformers would require a high amount of memory
474
              per batch. We present a solution to reduce the memory scaling of Linear Attention to O(N D). In
475           summary, we derive the analytical format of the attention layer gradient and calculate it in linear
476           time. By implementing this approach instead of relying on a differentiable programming library,
477           we eliminate the need to store the variables for the backward pass, and bring down the memory
478           consumption to O(N D). We provide the details of how to implement the forward and backward pass
479           using Factorization, and confirm the time and memory scaling of our implementation in context of an
480           attention layer and training an LLM. Furthermore, since Linear Attention does not directly compute
481           the attention matrix, dropout cannot be applied. Therefore, we have also introduced an alternative
482           mechanism to dropout, to mimic its effect, and confirmed our mechanism’s effectiveness through an
483
              ablation study.

      C     CUDA I MPLEMENTATION
821
822
      To implement custom gradients, we wrote the forward and backward pass of the Attention head in
823   CUDA. As a reminder, the matrices Q, K, V, O are 4-dimensional tensors with dimensions batch,
824   Attention heads, tokens, and dimension per head. We parallelized our code within the batch, heads,
825   and dimension per head, meaning we lunch a total of B × H × D threads. For a CUDA code to
826   be efficient, the number of threads should be either an integer multiple of number of CUDA cores
827   (hardware dependant), or be much bigger. For small benchmarks, we have B ≥ 32, H ≥ 8, D ≥ 32
828   resulting in launching 8, 192 threads. For comparison, the NVIDIA A100 GPU has 6, 912 CUDA
829   cores, meaning this method of parallelization is sufficient for reaching maximum parallelization, even
830   for small benchmarks. Note that it is not efficient for the calculations to be parallelized within the
831
      tokens dimensions, since in the case of Attention with casual mask, the factorized coefficients for
      each token is calculated with respect to the previous token’s, as shown in Eq. 8182.
832
833   Since the computations are MVP, and that MVP has a low memory reuse rate, the calculations are
834   memory bound. To alleviate this issue, we take advantage of shared memory. Shared memory, is
835   cache-like memory, which means it has a much lower read/write time but has a very limited capacity
836   (in order of 100 KB for each thread-block). As the name suggests, shared memory is shared between
837
      the threads within each thread-block. The first trick we implement is to use all of the threads within
      a thread-block to fill the shared memory, and then each thread can use all of the loaded data. To
838
      elaborate, assume we have thread-block of n threads, and c variables are needed by each thread.
839
      Assume accessing the global and shared memory takes tg and ts seconds for each variable, where
840   tg >> ts . Without the trick, the time needed would be c × tg since each thread accesses global
841                                                                                  c               c
      memory c times for the calculations. However, using the trick, the time will be × tg + (c + ) × ts
842                                                                                  n               n
                                                                   c
843   since each thread accesses the global and shared memory times to fill the shared memory, and
844                                                                n
      accesses the shared memory c times for the calculations.
845
846   Even by using shared memory, the calculation are memory bound. Therefore, we shape the calcula-
847   tions in a way to take advantage of the available registers within each thread. Registers have a much
848
      lower read/write time compared to shared memory memory, but has limited availability (less than 1
      KB per thread). Registers are useful when a variable needs to be updated many times within each
849
      thread. Instead of accessing this variable from shared memory, the thread stores and updates it locally,
850
      and writes it in shared memory when done. To elaborate, assume accessing the shared memory and
851   register takes ts and tr seconds for each variable, where ts >> tr , and that a variable needs to be
852   updated d times. The time without and with using this trick will be d × ts and ts + d × tr respectively.
