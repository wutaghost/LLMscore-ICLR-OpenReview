009                                                                 A BSTRACT
010
011            With the development of modern deep neural network (DNN), the scale of parameters is increasing, making it difﬁcult
012            to deploy models for use on resource-constrained edge devices. To address this issue, model compression is necessary,
013            and using low-rank matrix decomposition to compress DNN models is an effective research approach. However,
014            traditional studies on low-rank decomposition compression typically apply a single matrix decomposition method to
015            each parameter matrix in the neural network, without considering the structural characteristics of each layer in AI
016
               models, thus failing to achieve the optimal compression effect. Therefore, this paper proposes, for the ﬁrst time, a
               scheme for model compression using multiple decomposition methods, selecting the most suitable decomposition
017
               method for each layer in the model. However, to truly implement this approach, it is essential to balance model
018
               accuracy and compression cost. To address this, we propose a joint optimization paradigm that simultaneously
019            optimizes model accuracy and compression rate. We also introduce a framework LMFBRL based on reinforcement
020            learning that jointly selects the optimal decomposition method and rank. Tests were conducted on ﬁve models such
021            as LeNet-300, ResNet-20, and Vgg-16. Compared to singly using the MF method for compressing the LeNet300
022            model, our approach has shown an improvement of 3.6% in compression rate and a 1.8% increase in accuracy. The
023            test results validate the effectiveness of the algorithm proposed in this paper.

026   1   I NTRODUCTION
027
028   With the continuous advancement of AI model capabilities, the scale of model parameters has grown exponentially. Common models
029
      like VGG and ResNet now have parameter counts reaching into the millions, necessitating signiﬁcant computational resources and time
      to run these models. For the effective execution of these large-scale AI models, they are typically deployed on resource-rich cloud
030
      servers. However, transmitting collected data from edge devices to the cloud, processing it, and then sending back the results involves
031
      considerable transmission latency, resulting in response delays.
032
033   In the ever-expanding landscape of intelligent applications, there is an urgent need to implement edge intelligence at edge nodes to
034   meet the demand for rapid service response in scenarios such as smart homes and autonomous driving. Deploying AI models on edge
035
      nodes allows for the quick utilization of local data for training, without the need to wait for data transmission to cloud servers. This not
      only reduces network latency and transmission costs but also enhances data privacy and security.
036
037   However, edge nodes have limited resources, making it challenging to deploy large-scale AI models. Existing research, as demonstrated
038   by Cheng et al. (Denil M, 2023) (Lyu H, 2019) (Cheng Y, 2015) (Cheng Y, 2018), has shown the redundancy of AI models, providing
039   foundational principles for model compression. Therefore, we can compress AI models and deploy the compressed versions to resource-
040   constrained edge nodes for rapid execution.
041   Existing AI model compression techniques can be primarily categorized into four types: parameter quantization(Wu J, 2016)(Zhou Y,
042   2018), pruning(Luo J H, 2017)(Li Y, 2022), knowledge distillation(Hinton G, 2015), and low-rank decomposition(Tai C, 2015)(Wu K,
043   2019)(Phan A H, 2020a).
044
      a.Parameter quantization reduces the ﬂoating-point precision of model parameters (e.g., from Float32 to Float8) to compress the
045   model. However, when quantizing parameters to speciﬁc bit-widths, many existing training methods and hardware platforms become
046   incompatible, necessitating the design of specialized system architectures with limited ﬂexibility.
047
048
      b.Pruning reduces the model parameter count by removing redundant weights with small values. However, it relies heavily on pre-
      trained models, and platforms like TensorFlow do not support sparse computation, thus hindering computational acceleration.
049
050   c.Knowledge distillation treats large AI models as teacher models, incorporating soft targets related to the teacher model as part of
051   the total loss function to guide the training of student models, resulting in compressed models. However, its assumptions about student
052   models are overly restrictive, limiting its applicability.
053   d.Low-rank decomposition utilizes techniques to decompose weight matrices/tensors of AI models into smaller scale low-rank matri-
054   ces/tensors, achieving parameter compression. This method does not require specialized hardware support, can be used across various
055   platforms, and offers ﬂexibility in compression levels by adjusting the rank of the decomposition matrices.
056
      Based on the analysis of the four AI model compression methods mentioned above, this paper adopts the approach of low-rank decom-
057
      position due to its advantages of low hardware requirements and ﬂexible compression.

      1.1   C HALLENGE
059
060   Current research on AI model compression includes numerous low-rank decomposition methods, primarily categorized into low-rank
061   matrix decomposition and low-rank tensor decomposition (Wang C, 2024) (Tang J, 2022) (Liu Y, 2022). Two-dimensional matrix
062   decomposition is simpler and faster than high-dimensional tensor decomposition, with Singular value decomposition (SVD) and Full-
063   rank matrix factorization (MF) being the main methods. Most existing research employs a single decomposition method for all neural
064   network layers. For instance, Hsu et al. (Hsu, 2017) used SVD for all layers, while Winata et al. (Winata, 2015) used MF. These
065   methods have been effective for model compression.
066   However, our experiments reveal that different neural network layers have distinct structural characteristics, and various decomposition
067   methods have different capacities for representing these structures. This limitation becomes apparent when using a single decompo-
068   sition method. Our tests show that applying different decomposition methods to different network layers yields varying compression
069   effectiveness, with the optimal method differing for each layer (detailed analysis in Section 3).
070
      To address this issue, we explore using multiple decomposition methods for model compression, selecting the most suitable method for
071   each layer. However, achieving this multi-decomposition approach requires addressing two key challenges:
072
073   a.Balancing model accuracy and compression cost: Compressing a model often reduces accuracy. Striving for higher compression
074
      can result in progressively lower accuracy. Existing methods tend to focus on one aspect (Xu Y, 2020) (Yang H, 2020), leading to
      either high compression cost and low accuracy or low compression cost and high accuracy. Balancing the trade-off between accuracy
075
      and compression cost to achieve optimal outcomes for both is extremely challenging.
076
077   b.Jointly selecting the decomposition method and the rank r: The performance of low-rank decomposition compression is in-
078   ﬂuenced by the chosen rank r. Existing methods determine the rank based on a single predetermined decomposition method. Our
079   approach, involving multiple decomposition methods, requires the joint selection of both the decomposition method and the rank,
080
      which is highly challenging.
081
082   1.2   C ONTRIBUTION
083
084   To address these challenges and achieve optimal model compression, this paper proposes the Low-Rank Matrix Factorization Algorithm
085   Based On Reinforcement Learning (LMFBRL). This method selects the optimal decomposition method for each layer, determines the
086
      appropriate rank r, balances the trade-off between compression ratio and accuracy, and achieves optimal results. The contributions are
      as follows:
087
088   a.Joint Optimization of Model Accuracy and Compression Cost: This paper introduces a paradigm that jointly considers model
089   accuracy and cost functions in the context of multiple decomposition methods. By constructing expressions for both and jointly
090   optimizing them, we achieve an optimal balance between model accuracy and compression, resulting in the best possible solution for
091   model compression.
092   b.Alternating Training for Compressed Models Without Retraining: We propose an alternating optimization algorithm that iter-
093   atively performs parameter learning and model compression. Initially, model parameters are updated using the model’s loss function
094   and a compression parameter matrix. Subsequently, each network layer’s parameter matrix undergoes decomposition and compression.
095   This alternating process is repeated through successive training iterations until the ﬁnal training epoch is reached. Notably, the resulting
096   compressed model does not require retraining after the alternating optimization process is completed.
097   c.Joint Selection of Decomposition Method and Rank Based on Reinforcement Learning: Leveraging reinforcement learning,
098   we employ the Q-learning algorithm to establish a Q-value matrix based on two different decomposition methods. Each Q-value is
099   determined by the model’s loss value and an optimal rank selection function. Within the reinforcement learning framework, a selection
100   function is constructed to choose the decomposition method by comparing Q-values. This approach not only solves for the Q-values to
101   select the decomposition method but also determines the optimal rank simultaneously, achieving the goal of jointly selecting the best
102   decomposition method and rank.
103   d.Extensive Experiment: We conducted experiments using ﬁve models: LeNet-300, LeNet-5, ResNet-20, ResNet-32, and Vgg-
104   16, tested on two datasets, MNIST and CIFAR-10. The experimental results demonstrate that, compared to using a single ﬁxed
105   decomposition method for model compression, our LMFBRL approach achieves better compression rates with minimal accuracy loss.
106   Speciﬁcally, LeNet-300 achieved a compression rate of 89.5% and an accuracy of 96.2%. When applying MF decomposition solely
107   to LeNet-300, the compression rate was 83.6% with an accuracy of 95.2%. In comparison, our LMFBRL approach resulted in a 3.6%
108   improvement in compression rate and a 1.8% increase in accuracy. Test results conﬁrm the effectiveness of our algorithm.

      In low-rank decomposition-based AI compression, performance is inﬂuenced by the chosen rank r, affecting both compression rate and
117   accuracy. Thus, selecting the optimal rank is crucial.
118
119   Some studies use a ﬁxed rank r for each layer’s decomposition. For example, Kim et al. (Kim Y D, 2015) use variational Bayesian
120
      matrix decomposition to select ranks for CNNs on mobile devices, then decompose and ﬁne-tune the model. Kim et al. (Kim H,
      2019) propose an effective method considering the entire network to choose the correct rank conﬁguration, using new accuracy metrics
121
      non-iteratively. These ﬁxed-rank methods can achieve high compression rates but often compromise accuracy, leading to poor overall
122
      performance.
123
124   To address these issues, some researchers propose automated rank selection using performance metrics or heuristics. Phan et al. (Phan
125   A H, 2020b) introduced SLR low-rank decomposition, using positive deﬁnite tensors to reduce parameters and costs. Xiao et al.
126
      (Xiao J, 2023) proposed HALOC, a hardware-aware framework for automatic rank determination. Other studies (Kim Y D, 2015)
      (Kim H, 2019) (Acharya A, 2019) (Tai C, 2015) (Lebedev V, 2014) also use rank selection to improve compression but focus solely on
127
      rank r without optimizing model accuracy. It is not possible to achieve optimal values for both compression rate and accuracy.
128
129   Moreover, the aforementioned studies all employ the same decomposition method for compressing neural networks. However, our
130   analysis shows that the optimal decomposition method varies for different layers of the neural network, and also differs based on the
131   structure of those layers. As a result, a ﬁxed decomposition scheme does not yield the best compression performance.
132   To solve these issues, this paper introduces a joint optimization paradigm for model accuracy and compression cost, ensuring both are
133   optimized. Inspired by reinforcement learning, we construct a framework (LMFBRL) for joint selection of decomposition methods and
134   ranks. This framework selects the best method and optimal rank for each layer, achieving the best possible model compression.

362   This section provides a comprehensive analysis of the LMFBRL algorithm
363   designed in this paper. The algorithm adopts an iterative training approach, termed "update before compression selection," which
364   involves training the model parameters ﬁrst and then compressing them using a reinforcement learning-based decomposition algorithm
365   across all layers. The compressed matrices obtained are then used to train the entire model parameters, and this process is iteratively
366   repeated until the ﬁnal training round. The detailed procedure is outlined in Algorithm 2.
367
      Through the aforementioned iterative method of training before compression, as the number of iterations increases, we adjust our
368   penalty factor β towards inﬁnity, causing the model parameter matrix W to gradually approach the low-rank compressed matrix X,
369   becoming more low-rank. After completing all training, based on the recorded optimal decomposition methods for each layer, the
370   trained model parameter matrix W is decomposed into various decomposition matrices, thus achieving model compression.
371
      The entire model training method employs the SGD (Stochastic Gradient Descent) algorithm, and the pre-trained model is incorporated
372
      into the LMBFRL algorithm for training. Lines 1 to 24 of the Algorithm 2 represent the alternating optimization scheme designed in
373
      this paper. Line 2 involves the training process for the entire model parameters, where W represents the original uncompressed model
374   Weight matrix with rank Ri . During the learning steps of the algorithm, parameter updates are performed, while X represents the
375   compressed low-rank matrix with rank ri ≤ Ri , obtained during the compression selection steps of the algorithm.
376
377
      Lines 3 to 21 of the Algorithm 2 outline the model compression process, wherein all network layers of the model undergo decomposition
      and compression. During the reinforcement learning process, the selection of decomposition methods for each layer is determined
378
      by comparing the utility function values Q obtained from different decomposition methods. The selection function E chooses the
379
      decomposition method with the highest utility and simultaneously determines the selected rank r. The resulting compressed matrix
380   X is then used in the parameter update steps. This process is iteratively repeated until the ﬁnal round, where the trained model is
381   decomposed according to the selected method and rank r to obtain the ﬁnal compressed model.
382
383
      The ﬁnal compressed model obtained in this study does not require retraining to restore accuracy, saving a signiﬁcant amount of time.

551   9   C ONCLUSION
552
553   To deploy AI models efﬁciently on resource-constrained edge devices, model compression techniques are commonly used. This paper
554   focuses on using low-rank decomposition for model compression.
555
      Traditional low-rank decomposition research typically employs a single matrix decomposition method for each parameter matrix in a
556
      neural network, neglecting the structural characteristics of each layer and failing to achieve optimal compression. This paper introduces
557   a new scheme called LMFBRL, which uses multiple decomposition methods for model compression, selecting the most suitable method
558   for each layer. However, implementing this plan requires establishing criteria for selecting decomposition methods and balancing
559   model accuracy and compression costs. To address this, we propose a joint optimization approach that simultaneously optimizes model
560   accuracy and compression ratio. We introduce a rank selection scheme to choose the most appropriate rank for each decomposition
561   method and incorporate a reinforcement learning-based framework to select the optimal decomposition method for each network layer.
562
      Experimental results show that the LMFBRL algorithm outperforms other single decomposition methods by selecting the most suitable
563   method for each layer, resulting in higher accuracy and compression rates. The data demonstrate that optimizing both model accuracy
564   and compression cost achieves superior results. Compared to ﬁxed-rank decomposition methods, our rank selection scheme enables
565   the compressed model to achieve higher accuracy and compression rates.

735
736   Reinforcement learning adjusts each step of action based on feedback from the current environment, ultimately maximizing overall
737   rewards. In this paper, the current environment for the reinforcement learning framework refers to the i-th layer of the model, where
738   applying different decomposition methods to the weight matrix yields different feedback. The goal is to select the decomposition
739   method with the highest feedback for the current layer, maximizing the overall reward. This paper employs the Q-learning algorithm
740   based on a greedy strategy. According to the following Eq. (16):
741                                                                [                               ]
742
                                                 K ∗ (hi , bi ) = Q Qi + ϵ · max K ∗ (hi−1 , bi−1 ) .                                     (16)
743                                                                           b
744
745
      Among them, hi ∈ H is the state subspace, which is the set of decomposition methods chosen for each network layer. bi ∈ B is the
      action subspace, representing our comparison operation for each layer’s decomposition method. K ∗ (∗) represents the optimal action
746
      value function, which represents the overall beneﬁt of selecting a decomposition method for the compressed model. Qi represents the
747
      maximum reward of the current state, indicating the beneﬁt after selecting the decomposition method for the i-th network layer. Q
748   represents the maximum expected overall reward, and ϵ is a hyperparameter.

      A.4   D ETAILS O F E XPERIENCE
768   We test the algorithm proposed in this paper using the following ﬁve models:
769
770         • LeNet-300(Y. LeCun and Haffner, 1998): The LeNet-300 model consists of three fully connected layers, with 300, 100,
771           and 10 neurons in each layer respectively. Sigmoid activation functions are used between each fully connected layer, and a
772           Dropout layer is employed between the ﬁrst and second fully connected layers to reduce overﬁtting.
773         • LeNet-5(et al., 2015): LeNet-5 is a classic convolutional neural network model that consists of two convolutional layers, two
774           pooling layers, and three fully connected layers. The convolutional layers have 5x5 kernels, and the fully connected layers
775           contain 120, 84, and 10 neurons, respectively.
776         • Resnet-20(Akamaster, 2017): ResNet-20 is a basic version of the Residual Network (ResNet), consisting of 20 convolutional
777           layers. Its structure includes several convolutional layers, batch normalization layers, global average pooling layers, and a
778           fully connected layer for classiﬁcation.
779         • Resnt-32(Zhu C, 2016): Compared to ResNet-20, ResNet-32 has a deeper network architecture, consisting of 32 convolu-
780           tional layers. The model structure includes a series of convolutional layers, batch normalization layers, and global average
781           pooling layers, followed by a fully connected layer for classiﬁcation.
782         • Vgg-16(Simonyan and Zisserman, 2015): The VGG16 model consists of 16 layers, including 13 convolutional layers and
783           3 fully connected layers. This model utilizes small 3x3 convolutional kernels and a deep network structure, with multiple
784           convolutional layers stacked repeatedly.
785
786   We conducted experimental tests on the aforementioned model using six low-rank decomposition compression algorithms:
787
788
            • SVD-Selected: This method employs SVD (see Section A.2), where the original matrix is decomposed into the product of
              the left singular vectors, a diagonal matrix of singular values, and the transpose of the right singular vectors. We use the
789
              SVD adaptive rank selection algorithm from Yerlan et al.’s (Idelbayev and Carreira-Perpinán, 2020) research for testing and
790
              comparison.
791
792
            • SVD-Fixed: This method also uses SVD, but the rank of the decomposed matrix is ﬁxed. The rank is determined by averaging
              the ranks obtained from the experiments conducted in this study.
793
794         • MF-Selected: This method uses MF decomposition (see Section A.1), where the original matrix is decomposed into the
795           product of two low-rank matrices. We derived a corresponding MF adaptive rank selection algorithm by replacing the SVD in
796
              Yerlan et al.’s adaptive rank selection algorithm with the MF decomposition method.
797         • MF-Fixed: This method also uses MF decomposition, but the rank of the decomposition matrices is ﬁxed. The rank is
798           determined by averaging the ranks obtained from the experiments conducted in this study.
799         • LMFBRL-Selected: This paper proposes the LMFBRL-Selected algorithm, which is based on a reinforcement learning
800           framework. This algorithm selects the most suitable decomposition method for each network layer to achieve optimal model
801           compression performance. The decomposition methods adaptively select the rank.
802         • LMFBRL-Fixed: This paper proposes the LMFBRL-Fixed algorithm, which follows the same framework as LMFBRL-
803           Selected. However, in LMFBRL-Fixed, the decomposition methods use ﬁxed ranks.
