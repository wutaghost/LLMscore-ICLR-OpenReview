009                                                 A BSTRACT
010
011            Inductive GNNs are able to generalize across graphs with the same set of node
012            attributes. However, zero-shot generalization across attributed graphs with disparate
013            node attribute domains remains a fundamental challenge in graph machine learning.
014            Existing methods are unable to effectively make use of node attributes when
015            transferring to unseen attribute domains, frequently performing no better than
016            models that ignore attributes entirely. This limitation stems from the fact that
017            models trained on one set of attributes (e.g., biographical data in social networks)
018
               fail to capture relational dependencies that extend to new attributes in unseen test
               graphs (e.g., TV and movies preferences). Here, we introduce STAGE, a method
019
               that learns representations of statistical dependencies between attributes rather than
020
               the attribute values themselves, which can then be applied to completely unseen
021            test-time attributes, generalizing by identifying analogous dependencies between
022            features in test. STAGE leverages the theoretical link between maximal invariants
023            and measures of statistical dependencies, enabling it to provably generalize to
024            unseen feature domains for a family of domain shifts. Our empirical results show
025            that when STAGE is pretrained on multiple graph datasets with unrelated feature
026            spaces (distinct feature types and dimensions) and evaluated zero-shot on graphs
027            with yet new feature types and dimensions, it achieves a relative improvement in
028            Hits@1 between 40% to 103% for link prediction, and an 10% improvement in
029
               node classification against state-of-the-art baselines.

      1    I NTRODUCTION
033
      Zero-shot generalization, or a model’s ability to handle unseen test data without additional training
034
      or adaptation, has long been a key objective for AI systems (Larochelle et al., 2008; Xian et al.,
035
      2017; Wang et al., 2022). An essential prerequisite to zero-shot generalization is for models to learn
036   prediction rules that can be used across entirely different sets of features (or attributes) at test time.
037   This challenge has been addressed in other domains, such as natural language, through processes that
038   can split up any data into a pre-determined set of tokens (Samuel & Øvrelid, 2023).
039
      However, in the context of graph data with node attributes, zero-shot generalization presents unique
040
      challenges. First, features can vary significantly between graphs in different domains. Any method
041
      must be able to handle heterogeneous node features, including mixtures of continuous and categorical
042   features, as well as features that are highly dataset specific such as RAM on an electronics store and
043   clothes size in a department store, as shown in Figure 1. Second, the interpretation of certain features
044   (such as size) is often context-dependent, requiring graph learning methods that can jointly model
045   both the topology and the node attributes. These challenges make it hard to define a unified input
046   space allowing graph models to zero-shot generalize to unseen attributed graphs.
047
      For these reasons, pretraining general purpose graph models remains an open challenge. This is
048   reflected in the fact that generalizing to new attribute domains is rarely tackled in zero-shot settings (as
049   we discuss in Section 5). One strategy in such cases is to ignore node attributes altogether, focusing
050   on learning node and edge relations. Alternatively, some works textify node features (or the entire
051   graph) using pre-trained text embedding models (Chen et al., 2024a; Huang et al., 2023; Liu et al.,
052   2024; Zhang et al., 2023). However, this latter approach is limited by the text encoder, which may
053   overlook significant predictive signal in numerical attributes (Collins et al., 2024; Gruver et al., 2024;
      Schwartz et al., 2024), and the inclusion of text encoders often makes end-to-end training challenging.

073   Another approach is to focus on test-time adaptation for the GNN node embeddings, accepting that
074   the models representations may degrade on unseen attribute domains. This can be done, for example,
075   by applying linear models to node embeddings using closed-form maximum-likelihood solutions
076   (Zhao et al., 2024). Whilst test-time adaptation may prove to be useful in conjunction with domain
077   generalization, it does not address the fundamental problem of domain generalization itself, which is
078   to bestow the pretrained GNN model with the ability to produce useful representations of any graph.
079   In this work, we introduce STAGE (Statistical Transfer for Attributed Graph Embeddings), a novel
080   design for encoding node features to address the challenges of generalization to new attribute domains
081   on graphs. The core idea behind STAGE is to transform raw node attributes, which exist in an
082   “absolute” natural space that can vary arbitrarily across graphs, into a relative space that captures
083   their statistical dependencies over graph edges. For instance, Figure 1 illustrates purchases triggered
084   by positive correlations between the features in both domains. More specifically, STAGE builds a
085   graphical representation of such statistical dependencies that is invariant to three transformations
086
      commonly observed when transferring across distinct attribute domains—changes in attribute values,
      permutations of attribute dimensions, and permutations of node identities—while preserving the
087
      maximal amount of information about the raw attribute values. In particular, STAGE transforms node
088
      features into edge embeddings through a two-step process:
089
090   1. STAGE-edge-graph construction: For each edge in the original graph, STAGE constructs an
091      STAGE-edge-graph (such as illustrated in Figure 2(b), a fully connected weighted graph whose

123   In order to shift from embedding node features to embedding statistical dependencies on edges, the
124   STAGE-edge-graph is constructed using principles established by Bell (1964) and Berk & Bickel
125
      (1968), which link the theory of maximal invariants to measuring statistical dependencies between
      random variables. Namely, we prove that any function that can measure statistical dependencies
126
      of node features from totally ordered sets (e.g., Rd , d ≥ 1), can be described as a graph regression
127
      task on a variant of our STAGE-edge-graph, as long as its GNN encoder is sufficiently expressive.
128   This means that our method can learn expressive representations independent of the attribute domain
129   for a class of domain shifts, allowing generalization between graph tasks that rely on the interplay
130   between attribute dependencies and graph structure. Note that we do not prove generalization between
131   arbitrary graphs, since there are clear worst-case examples for which transfer is impossible.
132
      Our experiments show that STAGE substantially improves zero-shot out-of-domain generalization
133
      against state-of-the-art baselines for both link prediction and node classification tasks on multiple
134   datasets. Predicting future customer activity on an e-commerce website (link prediction) involving six
135   distinct stores selling beds, desktops, refrigerators, smartphones, shoes and clothes and apparel from
136   another (H&M), STAGE achieves up to a 103% improvement in Hits@1 over the best state-of-the-art
137   baselines. Additionally, in node classification, by pretraining on Friendster and zero-shooting into
138   Pokec, STAGE outperforms state-of-the-art by approximately 10%. Notably, STAGE’s performance
139   improves as the number of training domains increases, showing it is currently the only method
140   capable of learning generalizable patterns across distinct feature domains during pretraining.
141   These results underscore STAGE’s ability to learn transferable representations that are robust to
142   variations in node attribute domains. By embedding feature dependencies rather than their absolute
143   values, STAGE enables robust zero-shot predictions across entirely new and unseen graph domains.

      3     S TATISTICAL UNDERPINNINGS OF STAGE
217
218
      This section explains how STAGE achieves domain transferability. The central result is to show that
219   STAGE generates representations capable of measuring feature dependencies on graphs. This means
220   that STAGE is able to ignore “absolute” feature values, whilst still generalizing through analogous
221   statistical dependecies of the unseen features in the test graph.
222
      Our first step (Section 3.1) connects measures of statistical dependencies with a novel graph regression
223
      task. Then, Section 3.2 shows that our STAGE-edge-graphs (Definition 2.1) can lead to a compact
224
      model for this regression, with a variant that is invariant to a class of shifts between train and test
225   feature domains. The following theoretical results are meant to provide insights and are restricted
226   to domains with a fixed number of features to simplify the proofs, extending them to variable size
227   spaces is left as future work. Detailed proofs are provided in Appendix A.
228
229

230
231   We begin by introducing the framework for building what we call feature hypergraphs. We will show
232   that feature hypergraphs can sufficiently encapsulate the statistical dependencies between features,
233   whilst only leveraging the relative orders rather than the numerical values of the feature, enabling it to
234   be invariant to the order-preserving transformations (defined later in Definition A.2) to achieve better
235   domain transferability. Throughout this exposition we assume one feature space defined over a totally
236   ordered set (e.g., Rd for d ≥ 1, where the total order ≤ is well defined). Since the invariances of
237
      unordered sets are a special case (as these do not need order-preserving transformations), this section
      focuses on totally ordered sets. Before we describe how feature hypergraphs are built, we start with
238
      the concept of order statistic, which captures the relative ordering of the feature values.

      We now consider statistical tests that measure dependencies of the endpoint features. As an illustrating
271   example, consider that if (xu , xv ) ∈ E are samples (not necessarily independently sampled) from a
272   bivariate distribution (x, x′ ) ∼ F , one may be interested in testing the hypothesis
273
274                                       H0 : F (x, x′ ) = F1 (x)F2 (x′ ),
275
      i.e., that x and x′ are independent. Bell (1964); Berk & Bickel (1968) have shown over totally ordered
276   sets, measures (e.g., p-values) of such hypothesis tests, for pairwise independence (H0 above) and
277   more generally higher-order conditional independence between multiple variables, have invariances
278   that simplify the data representation to such a degree that the original values are discarded, retaining
279   only the order relationships between the variable values. And that any such test is therefore a rank test,
280   i.e., it relies solely on indices of the order statistic rather than the numerical values of the features.

      Our first theoretical contribution is the observation that any statistical test that focuses on measuring
282   the (conditional) dependencies of endpoint features in E can be defined as a graph regression task
283   over the feature hypergraph FE of Definition 3.1.

      5    R ELATED W ORK
493
494
      A more comprehensive discussion of related work can be found in Appendix G.
495
496   Graphs Generalization under Distribution Shifts. Several works address distribution shifts between
497   train and test graphs over the same feature space, such as You et al. (2022); Zhu et al. (2021b),
498
      which employ learned augmentations to mitigate distribution shift in node attributes. Meanwhile,
      extensive research has focused on domain adaptation for GNNs (Dai et al., 2022; Li et al., 2020;
499
      Kong et al., 2022; Pei et al., 2020; Veličković et al., 2019; Wiles et al., 2022; Zhang et al., 2019; Zhu
500
      et al., 2021a), which typically assume access to data in both source and target domains. In contrast,
501   our work tackles the more challenging scenario of zero-shot generalization to unseen attribute spaces.
502
503   Foundation Models for Graph Data. Developing foundation models for graph data is a growing
504
      research interest, aiming to create versatile graph models capable of generalizing across different
      graphs and tasks (Mao et al., 2024). Initial efforts in this direction convert attributed graphs into texts
505
      and apply an LLM (Liu et al., 2024; Chen et al., 2024b;a; Tang et al., 2024; Zhao et al., 2023; He
506
      & Hooi, 2024; Huang et al., 2023). However, while promising, this methodology risks information
507   loss and may limit transferability (Collins et al., 2024; Gruver et al., 2024; Schwartz et al., 2024).
508   In contrast, non-LLM approaches attempt to directly address domain transferability in the attribute
509   space (Xia & Huang, 2024; Lachi et al., 2024; Zhao et al., 2024), or forgo the attributes entirely (Gao
510   et al., 2023; Lee et al., 2023; Galkin et al., 2024). However, no definitive solution has emerged, and
511   the search for a universal graph model for node feature domain shift remains an open challenge.
512
      Maximal Invariants and Statistical Testing. Bell (1964) first explored the relationship between
513   invariant and almost-invariant tests in hypothesis testing. Berk & Bickel (1968) and Berk (1970)
514   extended Bell’s approach to show that almost-invariant tests are equivalent to invariant ones under
515   certain conditions, which are those met in our work. Later, Berk et al. (1996) explored the interplay be-
516   tween sufficiency and invariance in hypothesis testing by providing counterexamples that demonstrate
517   how these concepts can differ significantly in other scenarios. More recently, Koning & Hemerik
518   (2024) improves the efficiency of hypothesis testing under invariances for large transformation groups
519   such as rotation or sign-flipping without resorting to sampling.
520
521   6    C ONCLUSION
522
523   Designing graph models with the same intuitive flexibility as vision and language models is limited by
524   the challenge of learning representations that are informative across diverse feature spaces, including
525   new unseen concepts at test time. We tackle this challenge by proposing STAGE, a method designed
526   for zero-shot generalization across graphs with distinct node feature spaces. STAGE provides a
527   principled framework that learns embeddings by capturing how feature dependencies are structured
528   within the graph topology, rather than feature values. STAGE shows strong empirical performance
529   even under entirely new attribute spaces, unlike its natural baselines. We believe STAGE represents a
530
      meaningful advance in graph machine learning, paving the way for more universal pretained models
      that generalize across diverse datasets while fully leveraging node attributes.
531
532   Limitations and Future Work. While our method demonstrates scalability with respect to the
533   number of nodes (Appendix F), it may encounter limitations when dealing with high-dimensional
534   feature spaces. To address this challenge, various techniques can be explored, such as feature selection
535   via association studies, which we identify as a promising direction for future research.
536   Looking ahead, several avenues for further investigation emerge. These include extending our
537   framework to accommodate multi-dimensional and multi-modal attributes, such as text and images,
538   integrating large language models (LLMs) into the pipeline, and evaluating its performance on diverse
539   domains, including biomedical and geospatial networks. Pursuing these directions will be a positive
      step towards realizing a robust graph foundation model for attributed graphs.

      A.1   G ROUPOIDS
759
760   Definition A.1 (Groupoids). A groupoid G consists of the following elements:
761
762   1. A collection of distinct spaces, denoted as Spaces(G).
763
      2. A set of transformations (also called morphisms) between these spaces, denoted as Trans(G).
764
765   3. Each transformation f ∈ Trans(G) maps one space in Spaces(G) to another space (or potentially
766      to itself), denoted as f : X → Y , where X, Y ∈ Spaces(G).
767
      4. There is a rule for combining transformations: for any two transformations f : X → Y and
768
         g : Y → Z, their composition results in a transformation g ◦ f : X → Z.
769
770   5. Each space S ∈ Spaces(G) has an identity transformation idS : S → S that maps S to itself,
771      such that for any space X ∈ Spaces(G) and any transformation f1 : S → X and f2 : X → S, it
772      guarantees f1 ◦ idS = f1 and idS ◦ f2 = f2 .
773   6. Every transformation f : X → Y has a unique inverse transformation f −1 : Y → X such that
774      f −1 ◦ f = idX and f ◦ f −1 = idY .
775
776   A.2   S TATISTICAL TESTS AS GRAPH REGRESSION ON FEATURE HYPERGRAPHS
777
778   To prove the result of Theorem 3.2, we will first show an intermediate result using the notion of
779   maximal invariants. Let G be a transformation group acting on a space X. A function M : X → Z is
780   said to be maximal invariant if it is invariant to transformations of G and if ∀x1 , x2 ∈ X, M (x1 ) =
781   M (x2 ) implies x2 = g ◦ x1 for some group action g ∈ G, that is, if M is constant on the orbits but
782   for each orbit, it takes on a different value (Lehmann et al., 2005, pp. 214). A maximal invariant is a
783
      representation theory counterpart of sufficient statistics.
784   Our intermediate result will show that the feature hypergraph admits a graph representation that is a
785   maximal invariant. But first, we need to formally define the class of invariances, which we show later
786   is essential for STAGE’s domain transferability. Since we are interested in attribute spaces of distinct
787   domains, rather than using groups (which involve automorphisms mapping a space onto itself),
788   we will use groupoids (Definition A.1). Groupoids generalize the concept of groups by allowing
789
      transformations between multiple spaces. In a group, all transformations map a space onto itself,
      while in a groupoid, transformations can map between different spaces, but must still be invertible.
790
791   Definition A.2 (Component-wise order-preserving groupoids for features (COGF)). Let X1 , X2 be
792
      two feature spaces, both with d feature dimensions. A feature transformation f : X1 → X2 is said to
      be component-wise order-preserving if it can be decomposed into a set of maps f1 , . . . , fd , where
793
      each fi maps the i-th dimension of X1 to the i-th dimension of X2 and is a homomorphism that
794
      preserves the total order in X1 , and all dimensions of both X1 and X2 have a mapping.
795
796   Given an edge endpoint features E = {{(xu , xv ) | (u, v) ∈ E}} and a groupoid action f from the
797   COGF (Definition A.2), we define how f acts on E as follows:
798                                f (E) = {{(f (xu ), f (xv )) | (u, v) ∈ E}} .
799
800   Now, we are ready to establish the intermediate result as follows.

857   FE (Definition 3.1) and a most-expressive hypergraph GNN encoder Mθ∗ (FE ), then any test T (E)

863   endpoint features of E is necessarily a rank test that relies solely on the indices of the order statistics
      rather than the numerical values of the features (Bell, 1964; Berk & Bickel, 1968). As such, T (E)

1479   E.1   E VALUATING STAGE WITH GCN AS THE BACKBONE GNN
1480
1481   In the main experiments, we employed GINE + NBFNet for link prediction and GINE + GINE for
1482   node classification as the backbone GNN configurations. A natural question arises: Can STAGE be
1483
       effective when using other backbone GNN architectures? To address this, we propose GCN-STAGE
       (GINE + GCN (Kipf & Welling, 2016)), where we replace the second GINE with a modified GCN to
1484
       perform message passing on the original graph. We choose GCN as it is a well-known baseline for
1485
       node classification tasks. We modified GCN to process edge attributes by applying an MLP layer to
1486   edge attributes before incorporating them into the edge messages. The first GINE model operating on
1487   STAGE-edge-graphs remained unchanged.
1488
1489
       Table 6 presents the results, which demonstrate that GCN-STAGE outperforms all baseline methods in
       terms of average accuracy. Comparing to the other GCN-backbone models, GCN-STAGE outperforms
1490
       with a 7.33% relative improvement, and achieves an order-of-magnitude smaller standard deviation,
1491
       showcasing the stability and consistency of predictions across random seeds. Furthermore, same
1492   as GINE-STAGE, GCN-STAGE also outperforms GraphAny (Zhao et al., 2024), demonstrating
1493   that STAGE is effective on both GCN and GINE. We note that, however, the gain observed with
1494   GCN-STAGE is slightly lower than that of GINE-STAGE as shown in Table 2. This is not surprising,
1495   as GCN has been shown to have lesser expressivity than GINE (Xu et al., 2018).
1496
       These results demonstrate the effectiveness of STAGE regardless of the backbone GNN architecture
1497
       (GINE or GCN), reinforcing the versatility and general applicability of STAGE across tasks and
1498   architectures, further solidifying its strength as a robust framework.

1537
1538   F.1   STAGE C OMPLEXITY
1539
1540
       Here we analyze the runtime complexity of STAGE. In particular, we analyze NBFNet-STAGE (used
       for link prediction) and GINE-STAGE (used for node classification).
1541
1542   Let p be the number of features, d the dimension of internal node and edge embeddings, |E| the
1543   number of edges, and |V | the number of nodes in the input graph. For all tasks, STAGE consists of
1544   three steps:
1545
       1. Fully Connected STAGE-edge-graph Construction: This step requires O(|E|p2 ) operations
1546
          because each fully connected STAGE-edge-graph has 2p nodes, and each edge in the original
1547
          graph induces a fully connected STAGE-edge-graph.
1548
1549   2. Inference on STAGE-edge-graphs: We use 2 shared layers of GINE for all STAGE-edge-graphs.
1550
          A single layer on one fully connected STAGE-edge-graph has complexity O(pd + p2 d) = O(p2 d),
          since we have 2p d-dimensional nodes and (2p)2 d-dimensional edges in each STAGE-edge-graph.
1551
          Obtaining edge embeddings across all STAGE-edge-graphs takes O(|E|p2 d).
1552
1553   3. Inference on the original graph: For link prediction tasks, we use NBFNet to perform message
1554
          passing on the original graph, which requires O(|E|d + |V |d2 ) for one forward pass (Zhu et al.,
          2021c). For node classification tasks, we use GINE again, which requires O(|E|d) time.
1555
1556
       Hence, in total, running one forward pass has a complexity of O(|E|p2 d + |E|d + |V |d2 ) for
1557   NBFNet-STAGE, and O(|E|p2 d + |E|d) for GINE-STAGE.

1589
1590
       Graphs Generalization under Distribution Shifts. Several works address distribution shifts
1591
       between train and test sets over the same feature space. For instance, (You et al., 2022; Zhu et al.,
1592
       2021b) employ learned augmentations to mitigate differences in node attribute distributions, training
1593   a feature extractor that cannot be used to identify the domain of the node, but these methods are not
1594   designed to generalize across distinct attribute spaces. Meanwhile, extensive research has focused on
1595   domain adaptation and transfer learning for GNNs (Dai et al., 2022; Li et al., 2020; Kong et al., 2022;
1596   Pei et al., 2020; Veličković et al., 2019; Wiles et al., 2022; Zhang et al., 2019; Zhu et al., 2021a),
1597   typically assuming access to both source and target domains. In contrast, our work tackles the more
1598   challenging scenario of generalizing not only to unseen domains, but also to entirely new attribute
1599   spaces.
1600
1601   Foundation Models for Graph Data. Foundation models for graph data aim to create versatile
1602   graph models capable of generalizing across different graphs and tasks. Despite growing interest,
1603   achieving a truly universal graph foundation model remains challenging, especially due to the
1604   complexities in designing a suitable graph vocabulary that ensures transferability across datasets
1605
       and tasks (Mao et al., 2024). Initial efforts in this direction convert attributed graphs into texts
       and apply an LLM, but this methodology, while promising, risks information loss and may limit
1606
       transferability (Collins et al., 2024; Gruver et al., 2024; Schwartz et al., 2024). For instance, OFA (Liu
1607
       et al., 2024) uses frozen LLMs to generate features, and then trains a GNN to perform multiple tasks,
1608   while Chen et al. (2024b) explores the potential of LLMs as predictors or enhancersof graph-based
1609   predictions. Other methods, like LLaGA (Chen et al., 2024a) and GraphGPT (Tang et al., 2024), use
1610   instruction tuning to map graph data into the LLM embedding space. Similarly, Graphtext (Zhao
1611   et al., 2023) and Unigraph (He & Hooi, 2024) adopt NLP techniques, with Graphtext (Zhao et al.,
1612   2023) translating graphs into natural language via a syntax tree encapsulating node attributes and
1613   inter-node relationships, and Unigraph (He & Hooi, 2024) learning a unified graph tokenizer in a
1614   self-supervised fashion to generalize across different attribute domains. Prodigy (Huang et al., 2023)
1615   further encodes textual features with an LLM and leverages prompt-based graph representations for
1616
       task generalization.
1617   In contrast, recent approaches forgo LLMs entirely. For instance, Xia & Huang (2024) employs SVD
1618   decomposition of the feature matrix to handle shifts to new datasets. Lachi et al. (2024) employs a
1619   Perceiver-based encoder to compress domain-specific features into a shared latent space. Zhao et al.
       (2024) proposes GraphAny, specifically designed for node classification, which models inference


                                                          30
       Under review as a conference paper at ICLR 2025


1620
       on new graphs as an analytical solution to LinearGNNs, and addresses generalization by learning
1621   attention scores to fuse predictions from multiple LinearGNNs. Mao et al. (2023) introduces the
1622   concept of feature proximity as a key factor in determining the likelihood of links forming between
1623   nodes. Unfortunately, the definition of proximity still depends on the feature space.
1624
       Another line of work addresses zero-shot domain transferability on heterogeneous graphs such as
1625
       knowledge graphs, where both the nodes (entities) and edge types (relations) may be new and unseen
1626
       on the test-time graph. For instance, ISDEA+ (Gao et al., 2023) proposes a set aggregation layer over
1627   the set of edge-type-specific graph representations to ensure equivariance to edge type permutations.
1628   Gao et al. (2023) also proposes a theoretical framework named double equivariance that underlies the
1629   necessary design principles of models capable of tackling such a task. In contrast, the theoretical
1630   framework of our work addresses transferability to unseen attribute domains and proposes a novel
1631   connection between statistical tests and the graph regression task. ULTRA (Galkin et al., 2024), on
1632   the other hand, builds a relation graph that captures the interactions among different edge types, and
1633   applies a two-stage pipeline based on NBFNet (Zhu et al., 2021c) to ensure equivariance to edge type
1634   permutations. Similarly, InGram (Lee et al., 2023) also builds a relation graph, but its relation graph
1635
       differs from ULTRA’s in that it computes a set of affinity scores between pairs of relations and use
       them as edge weights on the relation graph. In comparison, the STAGE-edge-graphs built by our
1636
       method captures the statistical dependencies among different feature dimensions of node attributes
1637
       in the graph. However, all of these methods soly rely on graph structure and disregard attributes or
1638   features in nodes or edges. In contrast, our work focus on attributed graphs, which is capable of
1639   leveraging important information carried in the node attributes.
1640
1641
       However, a definitive solution for a universal graph foundation models is yet to arrive, and the search
       for such a model remains an open challenge.

       Maximal Invariants and Statistical Testing. Bell (1964) first explored the relationship between
1644
       invariant and almost-invariant tests in hypothesis testing. Berk & Bickel (1968) and Berk (1970)
1645
       extended Bell’s approach to show that almost-invariant tests are equivalent to invariant ones under
1646   certain conditions, which are conditions met in our work. Later, Berk et al. (1996) explored the
1647   interplay between sufficiency and invariance in hypothesis testing by providing counterexamples
1648   that demonstrate how these concepts, typically assumed to be equivalent under certain conditions,
1649   can differ significantly in other scenarios. More recently, Koning & Hemerik (2024) improves the
1650   efficiency of hypothesis testing under invariances for large transformation groups such as rotation or
1651   sign-flipping without resorting to sampling.
