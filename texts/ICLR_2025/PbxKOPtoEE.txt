                                                  A BSTRACT
010
011            Complex Query Answering (CQA) over knowledge graphs is a crucial multi-
012            hop reasoning task aimed at addressing first-order logical queries within large
013            and incomplete knowledge graphs. Direct traversal search methods rely solely
014            on graph topology and often miss answers due to the incompleteness of the
015            graph, thus neural models have been proposed to generalize the neglected an-
016            swers from observed facts. There are primarily two lines of research tackling the
017            challenges of CQA. Query embedding models learn representations for complex
018
               queries, offering fast speed but often providing only generic performance. In
               contrast, neural symbolic search methods deliver better performance, although
019
               they tend to be computationally more expensive. In this paper, we propose an
020
               efficient and scalable search framework that combines the precision of symbolic
021            methods with the speed of embedding techniques. Our model utilizes embed-
022            ding methods to compute Neural Logical Indices (NLI) to reduce the search
023            domain for each variable in advance, followed by an approximate symbolic search
024            for fine ranking. The search is precise for tree-form queries and approximates
025            cyclic queries (which are NP-complete) in quadratic complexity with respect
026            to the search domain, matching the complexity of tree-form queries. Experi-
027            ments on various CQA benchmarks show that our framework reduces computation
028            by 90% with a minimal performance loss, alleviating both efficiency and scal-
029
               ability issues for symbolic search methods. Our code is provided in https:
               //anonymous.4open.science/r/efficient_CQA/README.md.

032   1   I NTRODUCTION
033
034   Knowledge Graphs (KGs) are knowledge bases that represent relational facts in graph form. Although
035   KGs have an interpretable structure supporting many real-world applications (Ji et al., 2021), they
036   often suffer from incompleteness (Safavi & Koutra, 2020; Hu et al., 2020). Recently, complex
037   query answering (CQA) (Ren et al., 2023; Wang et al., 2022) over knowledge graphs has attracted
038   significant interest because this practical task performs logical reasoning with new knowledge inferred
039
      from observed knowledge graphs. Currently, the CQA task mainly focuses on answering existential
      first-order logic queries (Ren & Leskovec, 2020; Yin et al., 2023), involving logical operations such
040
      as conjunction, negation, disjunction, and the existential quantifier. Due to the incompleteness of
041
      KGs, many answers are overlooked in direct traversal searching.
042
043   There are primarily two lines of research to address the challenge of CQA. One is query embed-
044   ding methods which represent the query’s answer set using representations like vector, box, beta
045
      distribution of low dimensional space (Hamilton et al., 2018; Ren et al., 2020; Ren & Leskovec,
      2020). In this approach, logical operations are transformed into set operations within an operator
046
      tree (Wang et al., 2021; Ren et al., 2023), modeled by neural networks in alignment with their seman-
047
      tics in the low-dimensional space. Although the representational capabilities have been thoroughly
048   explored (Zhang et al., 2021; Choudhary et al., 2021), current query embedding methods still face
049   limitations in both performance and expressiveness (Yin et al., 2024). The second line of research,
050   neural-symbolic search methods (Arakelyan et al., 2020; Zhu et al., 2022; Bai et al., 2023; Yin et al.,
051   2024), utilizes knowledge graph completion methods (Bordes et al., 2013; Sun et al., 2018; ?) as a
052   backbone to predict missing facts and model logical operations using fuzzy logic inference. Though
053   symbolic search methods usually have both strong performance and interpretability, they typically
      suffer from high complexity and lack scalability, as shown in Fig 1.

071   The differences between the embedding-based approach and the neural-symbolic search approach
072   are not just empirical but fundamentally related to query syntax and complexity (Yin et al., 2024).
073   From a syntactical aspect, the widely used operator tree in query embedding methods is limited to
074   representing a subset of Existential First Order queries with single free variable (EFO1 queries),
075   denoted as tree-form queries (Yin et al., 2024). The tree-form queries in previous datasets (Ren
076   et al., 2020; Ren & Leskovec, 2020; Wang et al., 2021) are solvable with quadratic complexity
077   concerning |E|, where |E| is denoted as the number of entities in KG (Bai et al., 2023; Yin et al.,
078
      2024). Consequently, the real EFO1 dataset (Yin et al., 2024) that includes multigraphs and cyclic
      graph patterns has been proposed, further underscoring the complexity issues. When answering
079
      the EFO1 query with n variable, the existing precise symbolic search method (Yin et al., 2024)
080
      exhibits a worst-case computational complexity O(|E|n ) which grows polynomially with the size of
081   the knowledge graph but increases exponentially with the number of variables. The query embedding
082   methods can only approximate the general EFO1 query by operator tree but the complexity is
083   polynomial with respect to its dimension.
084
      Despite the fundamental differences between these two types of methods, we design a synergistic
085
      way to integrate them together in a mutually beneficial way, thereby delivering a new frontier of
086
      performance and efficiency, as shown in Fig 1. Inspired from the arc consistency (Chen et al., 2011)
087   in constraint satisfaction problem (Gottlob et al., 2000; Tönshoff et al., 2022)1 , it is unnecessary to
088   use all entities of the KG as the search domain for symbolic search methods. Instead, we can leverage
089   the surrounding constraints of each variable to reduce the corresponding search domain in advance,
090   thereby decreasing the computational cost required for originally slow but accurate symbolic searches.
091   Our first contribution is the Neural Logical Index (NLI), which models the surrounding constraints
092   with fast but not accurate embedding-based methods. In terms of the extent of the constraints utilized,
093   we propose two specific strategies, as illustrated in Fig. 2. The local constraints strategy uses the
094   relations directly connected with the variable, with a relation tail prediction task formulated to assist
095
      in computing the local constraints. The global constraints strategy considers broader constraints
      across the entire query to further reduce the search domain. Further details are presented in Section 3.
096
097   The second technical contribution of this work is that we propose a scalable framework, Neural
098   Logical Index for Search Approximately (NLISA) addressing the cyclic query in quadratic complexity
099   with respect to the search domain. In particular, this approximate search framework can be parallelized
100   and is exact in tree-form queries similar to QTO (Bai et al., 2023) and FIT (Yin et al., 2024).
101
      Combining the approximate search framework with neural logical indices, our method can efficiently
      answer the general EFO1 queries, including the cyclic queries. Experiments on various CQA
102
      benchmarks show that our framework reduces computation by 90% with a minimal performance loss,
103
      alleviating both efficiency and scalability issues. Additionally, we demonstrate that our framework
104   can execute neural-symbolic methods on a KG with an order of magnitude more entities than before,
105   highlighting the scalable nature of our approach.

124   2.1   K NOWLEDGE G RAPH
125
126   Definition 1 (Knowledge Graph). Let E be the finite set of entities and R be the finite set of relations,
127   a knowledge graph is a collection of factual triples G = {(si , ri , oi )}, where si and oi are entity
128   objects, and ri is a relation predicate.
129
      We augment the facts by adding reverse relations and denote r+ as the original relation r and r− as
130   its reverse, where each original triple (h, r, t) will result in two triples: (h, r+ , t) and (t, r− , h). The
131   knowledge graph can be represented as a first-order logic knowledge base, where each triple (s, r, o)
132   denotes an atomic formula r(s, o), with r ∈ R a binary predicate and s, o ∈ E its arguments.
133

156
157   By replacing logical operations with their corresponding set operations, some logical queries can be

      represented in the operator tree (Ren et al., 2020). Notably, the existential quantifier introduces a new

161   et al. (2023) in using query graphs to represent general EFO1 queries. Each conjunctive query ϕ is
      represented as a query graph, then the DNF can be represented as the disjoint query graphs.

210   We define the search domain D as the set of candidate entities when the symbolic search algorithm
211   operates over the variables. We denote Dx and Dy as the search domain for x and y, respectively. We
212   argue treating the entire entity set E, as done by previous search algorithms (Bai et al., 2023; Yin

214   constraints (Chen et al., 2011). Overall, for each variable, our framework extracts the surrounding
215   subgraph as constraints and computes the neural logical indices by these constraints to reduce the
      search domain.

      3.1       N EURAL L OGICAL INDICES
217
218   Given a query with the variable set V = {x1 , · · · , xn , y} over KG, neural logical indices are defined
219   as a mapping from a variable in V to a subset of the entity set E: I(x) : V → 2E . To compute the I(x)
220   for each variable, we conceptually define the constraints as the subgraph pattern GSx from the query
221   graph Gϕ . Then we propose the strategy to apply neural embedding models h to identify the entities
222   that satisfy the constraints represented by h(GSx )). Let h(GSx )) denote the ranking of the entities, we
223   determine I(x) by selecting the top k entities from this ranking: I(x) = Topk(h(GSx ), k). Thus,
224   I(x) serves as our reduced search domain Dx , which can accelerate symbolic search algorithms.
225   By using neural logical indices as the search domain for variables, we can narrow down the search
226   space, thereby simplifying the optimization problem in Equation 3 as follows:

339   4.4   A LGORITHM AND C OMPLEXITY A NALYSIS
340
341   Finally, we present the complete procedure of our method, as shown in Algorithm 1. Our objective is
342   to propose an efficient symbolic search method for general EFO1 queries. The key aspect is that we
343   can flexibly reduces the search domain. Additionally, our appropriate search reduces the complexity
344
      of answering cyclic queries from exponential to quadratic with respect to the search domain.
345   The space complexity of our method is O((|E| + |R| + dh ) ∗ d), where d is the embedding dimension
346   of the KG embedding model and dh is the hidden dimension of hyper-network. This complexity is the
347   same as the knowledge graph embedding and scales linearly with the sizes of entities and relations.
348   The time complexity is given by O((|Dx ||Dy | + |Ex |2 + |Dx | + |Dy |)d), where |Dx | and |Dy | are
349   the size of search domain of the existential and free variable, respectively. It is evident that our
350   algorithm exhibits quadratic complexity for any EFO1 query and can flexibly reduce the complexity
351   by adjusting the size of the search domain.
352
      Compared to other approximate symbolic search methods, such as CQD-beam (Arakelyan et al.,
353
      2020) and CQD-CO (Arakelyan et al., 2020), CQD-beam is constrained by the operator tree and only
354   utilizes partial constraints to retain consistent entities as intermediate variables. On the other hand,
355   CQD-CO optimizes the representation of existential variables in continuous space, which may limit
356   expressiveness and result in slower performance. Notably, both methods do not reduce the search
357   domain; instead, they only decrease the amount of information retained for intermediate variables.

360   5     E XPERIMENTS S ETTING
361
362   In this section, we conduct a comprehensive evaluation of our method across diverse tasks to
363   investigate its effectiveness and efficiency. In terms of query structure, we consider tree-form queries
364   and general EFO1 queries. Regarding the scale of the knowledge graph, we consider graphs with
365   15,000, 60,000, and 400,000 entities.

367   5.1   B ENCHMARKS
368
369   The BetaE benchmark (Ren & Leskovec, 2020) is the standard benchmark for complex question
370   answering (CQA), primarily containing tree-form queries. The benchmark is comprised of three
371   knowledge graphs (KGs): FB15k (Bordes et al., 2013), FB15k-237 (Toutanova et al., 2015), and
372   NELL995 (Xiong et al., 2022). Specifically, the BetaE benchmark contains 14 distinct query types,
373   with 5 types involving negation operations. It is important to note that the "pni" query type in BetaE
374
      is a universal first-order logic query, and is therefore excluded from the evaluation.
375   The Real EFO1 benchmark (Yin et al., 2024) proposes 10 new query types beyond the tree-from
376   queries, with the same KGs as BetaE. In particular, the Real EFO1 benchmark introduces new
377   patterns, including multi-graph, and cyclic graphs. The visualization of these query structures of
      BetaE and Real EFO1 benchmark is presented in Appendix E.

378
      The Smore benchmark (Ren et al., 2022) considers the same tree-form queries as BetaE benchmark,
379   but the queries are sampled from a much larger-scale KG. Since only the FB400K dataset with
380   40,0000 entities has been released, we select this KG as the large-scale benchmark.
381
382
      5.2   E VALUATION P ROTOCOL
383
384   To evaluate the effectiveness over incomplete knowledge graphs (KGs), we adopt the evaluation
385   scheme from (Ren & Leskovec, 2020), distinguishing the answers to each query into easy and hard
386   sets. Hard queries are defined as non-trivial queries that cannot be answered by direct traversal along
387   the edges of the KG and require predicting at least one missing link in the test and validation splits.
388   We assess the CQA models on these non-trivial queries by calculating the rank r for each hard answer
389   against non-answers, and we compute the Mean Reciprocal Rank (MRR) and HIT@k.
390   To evaluate the efficiency, we experimentally measure the speed and running memory required to
391   answer complex queries. We sample 100 queries for different query types in each benchmark and
392   calculate the average queries per second (QPS). We record both the model memory and the maximum
393   running memory during evaluations. For a fair comparison between models that support batching and
394   those that do not, the running memory is calculated by excluding the model memory from the total
395   running memory and then dividing the result by the batch size.

399
      We consider various state-of-the-art CQA methods as baselines. In particular, we compare our
      approach with strong baselines from symbolic search methods, including CQD-CO (Arakelyan
400
      et al., 2020), CQD-Beam (Arakelyan et al., 2020),QTO (Bai et al., 2023), and FIT (Yin et al., 2023).
401
      Additionally, we include ConE and LMPNN as baselines for comparison with query embedding
402   methods. We also examine the GNN-QE method, which combines graph neural networks with
403   symbolic methods. To ensure fairness, we use the same checkpoint for those methods requiring
404   a pre-trained neural link predictor, including CQA, QTO, FIT, and our method. For the FB15K,
405   FB15K-237, and NELL datasets, we utilize the checkpoints provided by CQD-CO (Arakelyan et al.,
406   2020). For FB400K, the details for the pretrained checkpoint are provided in Appendix F. Since FIT
407   is equivalent to QTO in tree-form queries (Yin et al., 2024), we don’t distinguish them in this case.

      utilize the local constraints strategy for existential variables. However, NLISA (Local) applies the

      7   C ONCLUSION
526
527   Complex query answering over knowledge graphs is a crucial multi-hop reasoning task aimed at
528   addressing first-order logical queries within large and incomplete knowledge graphs. Although sym-
529   bolic search methods exhibit strong performance and expressiveness, they face efficiency challenge
530   that hinders further development and application. Query embedding models learn representations for
531
      complex queries, offering fast speed but often providing only generic performance. In this paper, we
      integrate their advantage together in a mutually beneficial way and propose an approximate search
532
      framework combined with flexible neural logical indices to address these efficiency issues. The
533
      neural logical indices can be computed rapidly using embedding methods, significantly reducing the
534   search domain of symbolic methods. In particular, the approximate search framework can handle
535   cyclic queries well with a quadratic complexity. Our approximate search is precise for acyclic
536   queries. Experiments on various benchmarks show that, with a 10% reduced search domain, our
537   method achieves 90% performance, including for cyclic queries, while the QSP assess efficiency
538   is improved to be comparable to that of query embedding methods. Additionally, we demonstrate
539   that our framework can execute neural-symbolic methods on a KG with an order of magnitude more
      entities than before, highlighting the scalable nature of our approach.

      A     R ELATED W ORK

      A.1    Q UERY E MBEDDING METHODS
705
706   Answering complex logical queries over knowledge graphs is naturally extended from link prediction
707   and aims to handle queries with complex conditions beyond simple link queries. This task gradually
708   grows by extending the scope of complex logical queries, ranging from conjunctive queries (Hamilton
709   et al., 2018) to Existential Positive First-Order (EPFO) queries (Ren et al., 2020), Existential First-
710   Order (EFO) queries (Ren & Leskovec, 2020), real Existential First-Order queries (Yin et al., 2024).
711   The primary method is query embedding, which maps queries and entities to a low-dimensional
712   space. The form of embedding has been well investigated, such as vectors (Hamilton et al., 2018;
713   Chen et al., 2022; Bai et al., 2022), geometric regions (Ren et al., 2020; Zhang et al., 2021; Liu et al.,
714
      2021), and probabilistic distributions (Ren & Leskovec, 2020; Choudhary et al., 2021; Yang et al.,
      2022; Wang et al., 2023). These methods not only explore knowledge graphs embedding but also
715
      leverage neural logical operators to generate the embedding of complex logical queries.
716
717
718   A.2    S YMBOLIC METHODS
719
      Neural-symbolic CQA models represent variables as fuzzy sets with vector forms µ ∈ [0, 1]E and
720
      apply fuzzy logic to model the logic operations naturally. Built on the operator tree, GNN-QE (Zhu
721
      et al., 2022) represents the intermediate variable as a fuzzy vector, and simultaneously adapts graph
722   neural network from KG completion to execute relation projection and models the logical operations
723   with fuzzy logic. In particular, the neural symbolic search method combining the knowledge graph
724   embedding with symbolic search is of particular interest. CQD-beam (Arakelyan et al., 2020; 2023)
725   uses beam search during the execution of operator tree, maintaining only a beam width of entities for
726   intermediate variables. CQD-CO uses gradient optimization to estimate the embedding of existential
727   variables (Arakelyan et al., 2020). Unlike previous approximate search methods, QTO is a precise
728   symbolic search method because it finds that the tree-form queries can be solved on O(|E|2 ). FIT (Yin
729   et al., 2024) extends its scope to general EFO1 by using the enumeration to handle cyclic queries.
730
      However, the complexity is |E|n (Yin et al., 2024), where n is the variable number of query. In
      general, the above two precise methods constantly remove nodes and preserve results with fuzzy
731
      vectors corresponding to variables.While symbolic methods demonstrate good performance and
732
      strong interpretability, they struggle with high computational complexity.
733
734   There are many other models and datasets proposed to enable answering queries with good perfor-
735   mance and additional features, see the comprehensive survey Ren et al. (2023).

749   variables. We consider the simplest case where (s, r, e, N EG = FALSE), with s as the constant node,
750   r as the relation, e as the existential variable, and N EG = FALSE indicating that this edge is positive.
751   We first construct the symbolic representation of this edge: s = [Pr (s, ei )] ∈ R|De | , ei ∈ De .

754   fuzzy vector of e. The above method can be generalized to other cases, such as when the variable is a
755   free variable y and the edge is negative. A similar update approach can be used, and specific details
      can be found in Yin et al. (2024).

858   Why Use Hypernet?
859   Relation tail prediction is fundamentally similar to knowledge graph completion (KGC) tasks, as
860   both involve inferring missing tails from a knowledge graph. In KGC, the query is presented as
861   (h, r, ?), to predict the tail t. In contrast, relation tail prediction aims to predict the tail t given
862   the relation r. This similarity makes the embedding methods used in KGC equally applicable to
863   relation tail prediction. Hypernet is employed to generate weights dynamically, allowing the model
      to automatically adjust representations for new tasks. This facilitates the sharing and adaptation

887
888   Definition 9 (t-norm). A t-norm ⊤ is a function: [0,1] x [0,1] → [0,1] that satisfies the following
889   properties:
890
                   (i) Communitavity: a⊤b = b⊤a
891
892               (ii) Monotonicity: (a⊤b) ≤ (c⊤b) if a ≤ c
893
894              (iii) Associativity: (a⊤b)⊤c = a⊤(b⊤c)
895               (iv) Neutrality: a⊤1 = a
896
897   Then the t-conorm ⊥ is directly defined by a⊥b = 1 − (1 − a)⊤(1 − b), which follows the De
898   Morgan’s law.
899
      Finally, we introduce some common t-norms which are of interest:
900
901        (i) Godel: a⊤G b = min(a, b)
902
           (ii) Product: a⊤P b = a ∗ b
903
904       (iii) Łukasiewicz: a⊤LK b = max(a + b − 1, 0)
905
      In the main paper, we mainly focus on the Godel and Product t-norm.
